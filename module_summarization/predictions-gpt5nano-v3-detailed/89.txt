augmentations
=============

Overview
--------
This module provides a standard suite of image augmentations used by NVIDIA DALI AutoAugment policies
(AdamNet ImageNet policy, RandAugment, TrivialAugmentWide). The augmentations are implemented as
DALI-based operations and wired through the @augmentation decorator, which handles automatic
computation of the augmentation parameter from a magnitude index rather than a direct numeric value.

Key ideas
- Each augmentation function is defined with an example parameterization using @augmentation.
- At call time, you pass a single batch as the first argument, and supply magnitude_bin and
  num_magnitude_bins to choose the strength of the augmentation.
- The actual magnitude is computed as mag_to_param(magnitudes[magnitude_bin] * ((-1) ** random_sign)),
  where magnitudes = linspace(mag_range[0], mag_range[1], num_magnitude_bins).
- You can customize an augmentation by reusing the function’s .augmentation method to adjust mag_range,
  mag_to_param, or other decorator options, e.g.:
  my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))

Usage pattern
- Call signature (for a decorated augmentation):
  out_batch = augmentation(batch, magnitude_bin=idx, num_magnitude_bins=N)
- You can create a customized augmentation:
  my_aug = augmentation_function.augmentation(mag_range=(new_min, new_max))
  out_batch = my_aug(batch, magnitude_bin=idx, num_magnitude_bins=N)

Augmentations
-----------

shear_x
-------
Shifts image pixels by a horizontal shear.

- Decorator parameters: mag_range=(0, 0.3), randomly_negate=True, mag_to_param=warp_x_param
- Core operation: computes a shear parameter and applies a warp with a shear matrix
- Signature: shear_x(data, shear, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = shear_x(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customized magnitudes:
    my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))
    out = my_shear_x(batch, magnitude_bin=3, num_magnitude_bins=10)

shear_y
-------
Vertical shear of the image.

- Decorator parameters: mag_range=(0, 0.3), randomly_negate=True, mag_to_param=warp_y_param
- Core operation: applies a vertical shear
- Signature: shear_y(data, shear, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = shear_y(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customized magnitudes:
    my_shear_y = shear_y.augmentation(mag_range=(0, 0.5))
    out = my_shear_y(batch, magnitude_bin=4, num_magnitude_bins=10)

translate_x
---------
Horizontal translation (relative offset).

- Decorator parameters: mag_range=(0, 1), randomly_negate=True, mag_to_param=warp_x_param
- Core operation: computes offset = rel_offset * width, applies translation
- Signature: translate_x(data, rel_offset, shape, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = translate_x(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customized magnitudes:
    my_tx = translate_x.augmentation(mag_range=(0, 0.5))
    out = my_tx(batch, magnitude_bin=5, num_magnitude_bins=10)

translate_x_no_shape
-------------------
Horizontal translation with direct offset (no shape required).

- Decorator parameters: mag_range=(0, 250), randomly_negate=True, mag_to_param=warp_x_param, name="translate_x"
- Core operation: applies translation with a fixed offset
- Signature: translate_x_no_shape(data, offset, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = translate_x_no_shape(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Direct offset usage (no shape needed):
    out = translate_x_no_shape(batch, offset=40)

translate_y
---------
Vertical translation (relative offset).

- Decorator parameters: mag_range=(0, 1), randomly_negate=True, mag_to_param=warp_y_param
- Core operation: computes offset = rel_offset * height, applies translation
- Signature: translate_y(data, rel_offset, shape, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = translate_y(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customized magnitudes:
    my_ty = translate_y.augmentation(mag_range=(0, 0.5))
    out = my_ty(batch, magnitude_bin=6, num_magnitude_bins=10)

translate_y_no_shape
--------------------
Vertical translation with direct offset (no shape required).

- Decorator parameters: mag_range=(0, 250), randomly_negate=True, mag_to_param=warp_y_param, name="translate_y"
- Core operation: translation by a fixed offset
- Signature: translate_y_no_shape(data, offset, fill_value=128, interp_type=None)
- Example usage:
  - Default usage:
    out = translate_y_no_shape(batch, magnitude_bin=2, num_magnitude_bins=10)

rotate
------
In-place rotation of the image.

- Decorator parameters: mag_range=(0, 30), randomly_negate=True
- Core operation: rotates the image by the computed angle
- Signature: rotate(data, angle, fill_value=128, interp_type=None, rotate_keep_size=True)
- Example usage:
  - Default usage:
    out = rotate(batch, magnitude_bin=3, num_magnitude_bins=10)

brightness
----------
Adjusts image brightness.

- Helper: shift_enhance_range maps magnitude to [1, 3) around the neutral value 1
- Decorator parameters: mag_range=(0, 0.9), randomly_negate=True, mag_to_param=shift_enhance_range
- Core operation: uses fn.brightness with computed parameter
- Signature: brightness(data, parameter)
- Example usage:
  - Default usage:
    out = brightness(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customize range:
    my_brightness = brightness.augmentation(mag_range=(0, 0.8))
    out = my_brightness(batch, magnitude_bin=4, num_magnitude_bins=10)

contrast
--------
Adjusts image contrast.

- Helper: uses channel-wise mean and a computed center for contrast
- Decorator parameters: mag_range=(0, 0.9), randomly_negate=True, mag_to_param=shift_enhance_range
- Core operation: increases/decreases contrast based on computed parameter
- Signature: contrast(data, parameter)
- Example usage:
  - Default usage:
    out = contrast(batch, magnitude_bin=2, num_magnitude_bins=10)
  - Customize:
    my_contrast = contrast.augmentation(mag_range=(0, 0.6))
    out = my_contrast(batch, magnitude_bin=5, num_magnitude_bins=10)

Notes on design and behavior
- The @augmentation decorator centralizes the mapping from a magnitude_bin to a concrete transformation parameter. This keeps the core function signatures stable while enabling a tunable strength spectrum.
- Magnitude ranges are customizable per augmentation. For example, to broaden the magnitude for shear_x, use:
  my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))
- For translations, using translate_x_no_shape and translate_y_no_shape avoids the need for spatial shape information; the non-no_shape variants compute offsets from the provided shape tensor.
- The contrast augmentation follows a perceptual model based on a channel-weighted mean, mirroring PIL-like behavior with a GPU-friendly approach.

Helper utilities
----------------
warp_x_param and warp_y_param
- Purpose: map a scalar magnitude to a 2D affine parameter vector [dx, dy] for horizontal/vertical shear
- warp_x_param(magnitude) -> [magnitude, 0]
- warp_y_param(magnitude) -> [0, magnitude]

shift_enhance_range
- Purpose: translate a magnitude in [0, 1] (after random negation) to the [1, 2]–style range used by enhance operations
- Returns: 1 + magnitude

Parameters and return values
----------------------------
Common
- batch input: a batch of images/tensors to augment
- magnitude_bin: index into the magnitude grid defined by mag_range and num_magnitude_bins
- num_magnitude_bins: number of discrete magnitudes to sample from mag_range
- Returns: a batch of augmented images with the same batch size and channel layout as input

Function-specific notes
- translate_x and translate_y expect a shape (H, W, C) layout for non-no_shape variants; translate_x_no_shape and translate_y_no_shape do not require shape and take a direct offset.
- rotate includes a rotate_keep_size flag to control whether the output retains the input dimensions.
- contrast and brightness use perceptual cues aligned with common image processing practices; results depend on the input data distribution and dtype.

Usage examples
-------------
Basic usage with default magnitude settings:
```python
from nvidia.dali.auto_aug.augmentations import shear_x, brightness

# Apply horizontal shear with default magnitudes
out_batch = shear_x(batch, magnitude_bin=2, num_magnitude_bins=10)

# Apply brightness with default magnitudes
bright_batch = brightness(batch, magnitude_bin=3, num_magnitude_bins=10)
```

Customize magnitudes via the augmentation interface:
```python
# Create a customized shear_x with a broader magnitude range
my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))

# Use the customized augmentation
out_batch = my_shear_x(batch, magnitude_bin=4, num_magnitude_bins=10)

# Similarly for translate_y without requiring a shape
my_translate_y = translate_y.augmentation(mag_range=(0, 0.8))
out_batch = my_translate_y(batch, magnitude_bin=2, num_magnitude_bins=10)
```

Notes
-----
- The module depends on numpy for some internal computations; ensure numpy is installed.
- The actual parameter passed to the decorated functions is derived from magnitude_bin, not magnitude directly. Pass magnitude_bin and num_magnitude_bins to select the strength.
- The decorator-based approach allows easy extension or customization of the magnitude schedules without modifying the core augmentation logic.
