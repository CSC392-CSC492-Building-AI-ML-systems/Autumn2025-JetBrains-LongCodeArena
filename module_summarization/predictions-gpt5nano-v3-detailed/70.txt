Kernel approximation
====================

The :mod:`sklearn.kernel_approximation` module implements several approximate kernel feature maps based on Fourier transforms and Count Sketches. These mappings enable scalable kernel methods by projecting inputs into a low-dimensional feature space that approximates a chosen kernel.

PolynomialCountSketch
---------------------

PolynomialCountSketch provides an efficient Tensor Sketch-based approximation to the polynomial kernel feature map.

Algorithm overview
- Approximates the polynomial kernel feature map for K(X, Y) = (gamma * <X, Y> + coef0)^degree
- Uses Tensor Sketch via Count Sketch representations and FFTs to efficiently compute outer-product sketches
- Suitable for high-dimensional feature spaces where exact polynomial feature expansion would be prohibitive

Class
-----

PolynomialCountSketch
^^^^^^^^^^^^^^^^^^^^^

PolynomialCountSketch implements the Tensor Sketch method to approximate the polynomial kernel feature map.

Parameters
- gamma (float, default=1.0)
  Parameter of the polynomial kernel whose feature map will be approximated.

- degree (int, default=2)
  Degree of the polynomial kernel whose feature map will be approximated.

- coef0 (float, default=0)
  Constant term of the polynomial kernel whose feature map will be approximated.

- n_components (int, default=100)
  Dimensionality of the output feature space. Usually, a larger value yields a better approximation.

- random_state (int, RandomState instance, default=None)
  Determines random number generation for indexHash and bitHash initialization. Use an int for reproducibility.

Attributes
- indexHash_ : ndarray of shape (degree, n_features), dtype=int64
  Array of indices in [0, n_components) used to represent the 2-wise independent hash functions for Count Sketch computation.

- bitHash_ : ndarray of shape (degree, n_features), dtype=float32
  Array with random entries in {+1, -1}, representing the 2-wise independent hash functions for Count Sketch computation.

- n_features_in_ : int
  Number of features observed during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit (only if the input features have string names).

- _n_features_out : int
  Internal record of the number of output features (typically equal to n_components).

Notes
- The class adheres to scikit-learn’s estimator API, including fit/transform interfaces.
- The transform method returns a feature matrix X_new with shape (n_samples, n_components).

Examples
- Basic usage

```python
from sklearn.kernel_approximation import PolynomialCountSketch
# sample data
X = [[0, 0], [1, 1], [1, 0], [0, 1]]
y = [0, 0, 1, 1]

# create a polynomial kernel approximation with degree 3
pcsk = PolynomialCountSketch(degree=3, random_state=1)

# fit and transform to approximate feature map
X_features = pcsk.fit_transform(X)

# use approximate features with a classifier
from sklearn.linear_model import SGDClassifier
clf = SGDClassifier(max_iter=10, tol=1e-3)
clf.fit(X_features, y)
print(clf.score(X_features, y))
```

Parameters and return values (summary)
- fit(X, y=None)
  - Validates and stores feature information, initializes hashing structures (indexHash_, bitHash_), and sets internal feature counts.
  - Returns: self

- transform(X)
  - Produces the approximate polynomial kernel feature map for X.
  - Returns: X_new, array-like of shape (n_samples, n_components)

Important considerations and edge cases
- coef0 != 0 increases the effective input dimensionality by 1 before hashing, which affects the shape of indexHash_ and bitHash_.
- Works with dense or sparse input (scikit-learn-compatible input validation is used).
- Random state controls the reproducibility of the Count Sketch hash functions.
- The quality of the approximation improves with larger n_components, at the cost of higher memory and compute.
