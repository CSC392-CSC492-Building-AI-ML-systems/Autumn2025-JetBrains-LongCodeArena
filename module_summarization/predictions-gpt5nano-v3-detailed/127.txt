Convolution module built-in kernels
===================================

Overview
--------
This section documents the built-in 1D and 2D kernels provided for use with the Convolution model
in the convolution subsystem. Kernels are represented by Kernel (base), Kernel1D, and Kernel2D
objects and can be used directly or via specialized subclasses (e.g., Gaussian, Box). They are
designed to work with the Convolution model to perform linear filtering on N-dimensional data,
supporting normalization, boundary handling, and shape validation.

Key concepts
------------
- Kernel: Abstract base for all kernels. Encapsulates the weight array and common validation.
- Kernel1D / Kernel2D: Concrete 1D and 2D kernel containers.
- Built-in kernels: Common families (e.g., Gaussian, Box, and other standard filters) available as
  ready-to-use Kernel1D/Kernel2D subclasses.
- Normalization: Kernels are typically normalized so their sum equals 1, ensuring the filtered
  result preserves overall brightness. Normalization can be controlled via constructor options where
  supported.
- Boundary handling: Convolution supports different boundary strategies controlled via boundary
  options. The allowed options align with BOUNDARY_OPTIONS: None, "fill", "wrap", "extend".
- Validation and edge cases: Kernels enforcing odd sizes on certain axes are checked; attempting to
  use an even-sized kernel may raise KernelSizeError via has_even_axis / raise_even_kernel_exception.
- Performance helpers: Private utilities such as _next_fast_lengths help select FFT lengths for faster
  convolution when using FFT-based implementations.

Available classes and relationships
-------------------------------------
- Kernel (base class)
  - Kernel1D: 1D kernel container
  - Kernel2D: 2D kernel container
- Built-in kernels (examples)
  - Gaussian1DKernel, Gaussian2DKernel
  - Box1DKernel, Box2DKernel
  - Other standard filter kernels provided by the module
- Convolution: Model that applies a given Kernel1D/Kernel2D to data, supporting boundary handling
  and integration with the modeling framework.

Important parameters and return values
--------------------------------------
- Kernel constructor parameters
  - kernel_values: 1D or 2D array-like weights defining the kernel
  - normalize: whether to normalize the kernel so its sum equals 1 (where supported)
  - x_size / y_size (for some built-ins): explicit kernel extents
- Convolution constructor parameters
  - kernel: a Kernel1D or Kernel2D instance
  - boundary: one of the values in BOUNDARY_OPTIONS (None, "fill", "wrap", "extend")
  - fill_value: value used when boundary="fill"
- Returns
  - Applying a kernel via Convolution yields a convolved array with the same dimensionality as the input
    (subject to boundary handling and internal padding rules).

Usage examples
--------------

1) Create and use a simple 1D kernel
```python
from astropy.modeling.convolution import Kernel1D, Convolution
import numpy as np

# Simple 1D kernel [1, 2, 1]
k1d = Kernel1D([1, 2, 1], normalize=True)
conv1d = Convolution(k1d)

data = np.arange(10.0)
result = conv1d(data)
```

2) Use a built-in 2D Gaussian kernel
```python
from astropy.modeling.convolution import Gaussian2DKernel, Convolution
import numpy as np

g2d = Gaussian2DKernel(stddev=1.0)
conv2d = Convolution(g2d)

image = np.random.random((64, 64))
smoothed = conv2d(image)
```

3) Boundary handling
```python
# Use wrap-around boundary conditions
conv_wrap = Convolution(k1d, boundary="wrap")
wrapped_result = conv_wrap(data)
```

4) Internal FFT-length optimization (informational)
```python
# _next_fast_lengths is a private helper used to determine efficient FFT sizes.
from astropy.modeling.convolution import _next_fast_lengths

shape = (64, 64)
fft_shape = _next_fast_lengths(shape)
```

5) Handling edge cases with even-sized kernels
```python
from astropy.modeling.convolution import Kernel1D, KernelSizeError

try:
    evn = Kernel1D([1, 1])  # an even-sized kernel
    conv = Convolution(evn)
except KernelSizeError as exc:
    print("Invalid kernel size for this operation:", exc)
```

Internal helpers and notes
--------------------------
- _next_fast_lengths(shape): Compute optimal FFT lengths for each axis to speed up FFT-based
  convolution. This is an internal helper and is not part of the public API for typical usage.
- _copy_input_if_needed(...): Utility used internally to ensure input data is correctly prepared
  for interpolation and processing, respecting mask and fill-value handling.

Important considerations and edge cases
--------------------------------------
- Kernel sizes: Many built-in kernels are defined with odd extents to align with the center of the data.
  If an operation detects an even-sized axis, it may raise KernelSizeError via has_even_axis /
  raise_even_kernel_exception.
- Boundary handling: Choose boundary strategy that matches the intended edge behavior of your data.
  None corresponds to the default behavior; "fill" uses a fill_value; "wrap" wraps around; "extend"
  extends the input with edge values.
- Normalization: Some kernels normalize automatically; explicit normalization is available where supported.
- NDData compatibility: Kernels support NDData-like inputs and can be used with Convolution in typical
  data pipelines.

See also
--------
- Convolution (module): The Convolution model that applies Kernel1D/Kernel2D to data.
- Kernel, Kernel1D, Kernel2D: Core kernel abstractions used by the built-in kernel implementations.

Notes
-----
- The built-in kernels are designed to be convenient defaults for common image processing and signal
  processing tasks. For custom filtering, you can construct Kernel1D/Kernel2D from explicit weight
  arrays or use specialized subclasses (e.g., Gaussian1DKernel, Gaussian2DKernel) as needed.
