Intro to nextcord Guild Widget (Widget)

====================================

Overview
--------
The widget module provides a lightweight representation of a Discord guild widget. A widget is a small, read-only snapshot of a guild’s current state that exposes:

- accessible voice channels in the guild
- online members (partial view of online members)

This data is useful for embedding guild information in external sites or for quick in-app inspection without performing full member data requests. The widget data is partial and may omit offline members or unavailable fields depending on the guild’s widget settings.

Key concepts
------------
- WidgetChannel: a partial representation of a voice channel included in the widget. It exposes minimal channel information such as id, name, and position.
- WidgetMember: a partial representation of an online guild member. It extends BaseUser and includes status, display name (nick or username), avatar, activity, and connection context (which channel they are in, if any).
- Widget: a collection representing the guild widget itself. It includes the guild id, name, a list of channels, and a list of online members.

Classes and relationships
-------------------------
- WidgetChannel
  - id: int
  - name: str
  - position: int
  - str/repr reflect the channel name and identity
  - mention: property returning a Discord channel mention string
  - created_at: property deriving creation time from the channel’s snowflake id

- WidgetMember (inherits from BaseUser)
  - id, name, discriminator, bot
  - status: Status (e.g., online, offline)
  - nick: Optional[str]
  - avatar: Optional[str]
  - activity: Optional[Union[BaseActivity, Spotify]] (derived from the member’s current activity)
  - deafened, muted, suppress: Optional[bool]
  - connected_channel: Optional[WidgetChannel] (the channel the member is connected to)
  - display_name: returns nick if set, otherwise the username

- Widget
  - id: int (guild ID)
  - name: str (guild name)
  - channels: List[WidgetChannel] (accessible voice channels)
  - members: List[WidgetMember] (online members visible in the widget)
  - Note: The widget data is a partial view; not all members may be present, depending on visibility and settings.

Usage examples
--------------
- Accessing widget data from a guild (assuming you have a Widget instance named widget):

.. code-block:: python

    # Print basic widget info
    print(widget.name)
    for channel in widget.channels:
        print(f"Channel: {channel.name} (id={channel.id})")

    for member in widget.members:
        print(f"{member.display_name} - {member.status}")

- Working with member activity and status:

.. code-block:: python

    for member in widget.members:
        print(f"{member.display_name}: {member.status}")
        if member.activity:
            # activity can be a BaseActivity or Spotify instance
            activity_name = getattr(member.activity, "name", None)
            print(f"  Activity: {activity_name}")

- Accessing channel metadata (creation time):

.. code-block:: python

    some_channel = widget.channels[0]
    print(f"Channel {some_channel.name} created at {some_channel.created_at.isoformat()}")

Important considerations and edge cases
---------------------------------------
- Widget availability: Guilds must enable widget data for it to be available. If a widget is not enabled, widget data may be incomplete or absent.
- Partial data: The widget exposes a subset of the full guild state. Offline members may not appear; some fields may be missing or null.
- Activity resolution: The activity attribute is derived from the payload’s 'game' field if present. If the field is absent, activity will be None.
- Time handling: created_at uses the channel’s snowflake id to compute the creation time in UTC.
- Data freshness: Widget data is a snapshot and may lag behind real-time events; use it for quick status checks or embeds, not for precise stateful logic.

Notes

- The widget data model mirrors the small, read-only view provided by Discord’s widget API. Use it to present lightweight guild state without requiring full member details or heavy API calls.
