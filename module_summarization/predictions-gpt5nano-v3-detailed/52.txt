# Generalized Method of Moments (GMM) and Two-Stage Least Squares for Instrumental Variables (IV2SLS)

This module provides a simple implementation of Two-Stage Least Squares (2SLS) within the framework of instrumental variables estimation. It exposes an IV2SLS class that performs 2SLS using a textbook construction and returns a regression-like results object.

## Overview

- Purpose: Estimate linear models with endogenous regressors using instrumental variables (IV) via 2SLS.
- Core idea: 
  - First stage: project endogenous regressors on the instruments to obtain fitted values.
  - Second stage: regress the dependent variable on the fitted values of the endogenous regressors.
- Returns: A RegressionResults-like object encapsulated in a wrapper, with standard regression-style attributes (e.g., params, etc.) and a covariance structure derived from the 2SLS setup.

## API

### IV2SLS(endog, exog, instrument=None)

Instrumental variables estimation using Two-Stage Least Squares.

- Parameters
  - endog: array-like
    Endogenous dependent variable, shape can be 1D or 2D (nobs by 1 or nobs by k).
  - exog: array-like
    Explanatory variables to be instrumented, shape can be 1D or 2D.
  - instrument: array-like
    Instruments for the explanatory variables. Must include both exogenous variables not to be instrumented and any instruments. Internally the instrument matrix is coerced to 2D via _ensure_2d.

- Notes
  - All variables in exog are instrumented in the calculations. If some exog variables should not be instrumented, they must also appear in the instrument matrix.
  - Degrees of freedom for residual variance are computed as df_resid = nobs - k_vars, where k_vars is the number of exog parameters.
  - The class currently uses a textbook 2SLS calculation (no generalized weighting matrix customization in this implementation).

- Attributes (as established during initialization)
  - df_resid: degrees of freedom for residuals
  - df_model: degrees of freedom for the model

- Methods
  - fit(self)
    - Performs 2SLS estimation and returns a RegressionResults-like object wrapped in a RegressionResultsWrapper.
    - Implementation highlights:
      - First stage: compute xhatparams = (Z'Z)^{-1} Z'X, where Z is the instrument matrix and X is the original exog.
      - Construct F = Z * Xhat, then compute the second-stage parameters:
        - params = (F'F)^{-1} F'Y
      - Compute a covariance estimate (normalized_cov_params) using the standard 2SLS formula (via F'F and F'X inverse).
      - Returns a RegressionResultsWrapper around an IVRegressionResults-like object with the estimated parameters and covariance.

- Returns
  - A RegressionResultsWrapper containing an IVRegressionResults instance, representing the 2SLS estimates in a regression-results interface.

### IVRegressionResults

A results class for the 2SLS estimation, derived from RegressionResults. It inherits standard regression result attributes and behavior, augmented by the IV/2SLS context (e.g., the internal xhat-based second-stage results are stored for reference).

## How it works

- Step 1 (First stage): Solve for the coefficients that project X onto Z:
  - xhatparams = (Z'Z)^{-1} Z'X
  - F = Z * Xhat
- Step 2 (Second stage): Regress Y on Xhat by solving:
  - params = (F'F)^{-1} F'Y
- Covariance estimation uses the implied weighting consistent with the 2SLS construction:
  - normalized_cov_params is computed from the inverse of (F'F) and related cross-terms.

- The final results object behaves like a standard regression results object, enabling access to params and other statistics via the typical interface (wrapped by RegressionResultsWrapper).

## Usage examples

```python
import numpy as np
from your_module import IV2SLS  # replace with the actual module path

# Seed for reproducibility
np.random.seed(0)

n = 200
# Instruments (including any exogenous instruments or constants)
Z = np.random.normal(size=(n, 2))

# Endogenous regressor X and outcome Y
X = Z @ np.array([[1.0], [0.5]]) + np.random.normal(size=(n, 1))
Y = 1.5 * X + Z[:, 0:1] * 0.3 + np.random.normal(size=(n, 1))

# Instantiate IV2SLS: endog=Y, exog=X, instrument=Z
iv2sls = IV2SLS(endog=Y, exog=X, instrument=Z)

# Estimate
res = iv2sls.fit()

# Access results
print(res.params)
print(res.bse)  # standard errors if available in the interface
```

Notes:
- The endog and exog shapes may be 1D or 2D; the implementation handles these via internal reshaping compatible with the underlying regression framework.
- The instrument matrix should reflect all variables that are used for identification; if some exog variables are not to be instrumented, they must appear in the instrument matrix as well.

## Important considerations and edge cases

- Whiten/weighting: The current implementation does not support a separate whitening step. The whiten method is a placeholder.
- Optimal weighting matrix: This 2SLS path uses the textbook approach; there is no explicit support for a generalized weighting matrix in this simple 2SLS path. For GMM-like weighting customization, additional development would be required.
- Degrees of freedom: df_resid and df_model are derived from the shapes of endog and exog; ensure your inputs have compatible dimensions.
- Edge cases:
  - Shape mismatches between endog, exog, and instrument will raise errors during matrix operations (e.g., dimension mismatches in Z'Z or (Z'Z)^{-1}.
  - If the instrument matrix is not sufficiently informative for all endogenous regressors, the solution may be numerically unstable or ill-conditioned.

## Implementation notes

- Core goal: provide a straightforward, regression-like interface for 2SLS within the StatsModels ecosystem.
- The approach aligns with standard IV/2SLS practice: instrument all endogenous variables using Z, then perform second-stage regression on the fitted values.
