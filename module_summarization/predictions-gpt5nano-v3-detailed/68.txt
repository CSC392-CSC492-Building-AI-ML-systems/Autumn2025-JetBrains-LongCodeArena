# gaussian_process.rst

Kernels for Gaussian process regression and classification

Introduction
------------
This module provides a collection of kernels for Gaussian process (GP) regression and classification. The kernels can be engineered by composing simple kernels using addition (+), multiplication (*) and scalar exponentiation (**). Scalars in expressions are automatically converted to a constant kernel. All kernels support analytic gradient-based hyperparameter optimization. Hyperparameters are defined with a configurable search space, which can be specified either as bounds (rectangular search space) or by marking certain parameters as fixed (excluded from optimization).

Key concepts
------------
- Hyperparameter
  - Represented by the Hyperparameter namedtuple with the fields:
    - name: the parameter name (e.g., "length_scale" or "constant_value").
    - value_type: the type of the parameter (currently "numeric").
    - bounds: a pair of floats (lower, upper) or the string "fixed".
    - n_elements: number of elements of the hyperparameter (1 for scalar, >1 for vector-valued, e.g., anisotropic length-scales).
    - fixed: boolean indicating whether the parameter is fixed (not optimized). If None, fixed is inferred from bounds.
  - Access hyperparameters via kernel.hyperparameters. Each hyperparameter corresponds to a parameter in the kernel, and kernels expose their hyperparameters so they can be tuned, inspected, and validated.
  - Example:
    - Hyperparameter(name='constant_value', value_type='numeric',
                   bounds=array([[0., 10.]]), n_elements=1, fixed=False)

- Kernel
  - Base class for all kernels. Provides core functionality used by all concrete kernels:
    - Kernel composition via + (sum), * (product), and ** (exponentiation by a scalar).
    - Automatic handling of scalar values in expressions by converting them to a constant kernel.
    - Gradient-based hyperparameter optimization with a configurable search space.
    - Retrieval of parameters and hyperparameters (e.g., via get_params and hyperparameters).
  - Compatibility: designed to be used with GP regression and classification pipelines.

- _check_length_scale
  - Internal helper to validate length-scale vectors:
    - Ensures length_scale is 1D and, for anisotropic kernels, that its size matches the data dimensionality.
    - Raises ValueError if the dimensionality is inconsistent.

Usage patterns
--------------
- Create simple kernels and combine them
  - Kernels can be combined with +, *, or exponentiated with **. Scalars in expressions become constant kernels.
  - Examples (conceptual; class names reflect common kernel nomenclature in GP libraries):

  ```python
  from gaussian_process import ConstantKernel, RBF

  # Simple constant kernel with bounds
  k1 = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 10.0))

  # Anisotropic RBF kernel (conceptual)
  k2 = RBF(length_scale=1.0, length_scale_bounds=(0.1, 10.0))

  # Sum and product of kernels
  k_sum = k1 + k2
  k_prod = k1 * k2

  # Exponentiate a kernel by a scalar
  k_exp = k_sum ** 2.0
  ```

- Inspect hyperparameters and params
  - Access the hyperparameters exposed by a kernel and adjust them programmatically if needed.
  ```python
  for hp in k_sum.hyperparameters:
      print(hp)

  # Get kernel parameters in a dictionary-like form
  params = k_sum.get_params()
  for key in sorted(params):
      print(f"{key} : {params[key]}")
  ```

- Practical notes
  - The hyperparameter search space is rectangular by default (lower and upper bounds for each parameter).
  - If a hyperparameter is marked as fixed, it is excluded from optimization.
  - Length-scale validation ensures correct dimensionality when using anisotropic kernels; mismatches raise clear errors to guide correction.

Important considerations and edge cases
---------------------------------------
- Length-scale handling
  - _check_length_scale enforces that length_scale is 1D. If an anisotropic kernel is used, its length_scale must have a dimension equal to the data dimensionality.
  - If the dimensionality is inconsistent, a ValueError is raised with a descriptive message.

- Hyperparameters
  - Each hyperparameter has a name that corresponds to the kernel parameter (e.g., "length_scale", "constant_value").
  - Bounds can be provided as numeric pairs or the string "fixed" to indicate non-optimizable parameters.
  - For vector-valued hyperparameters, bounds can be specified with multiple elements (n_elements > 1).

- Operator-based composition
  - Kernels support intuitive composition via addition, multiplication, and exponentiation, enabling flexible kernel design for GP models.
  - Scalars encountered in expressions are automatically converted to a constant kernel, ensuring consistent behavior.

API reference (high-level)
--------------------------
- Hyperparameter
  - A lightweight, memory-efficient representation of a kernel's tunable parameter.
  - Fields: name, value_type, bounds, n_elements, fixed.
  - Behavior:
    - __new__ handles bounds normalization and automatic fixed detection when bounds is "fixed".
    - __eq__ compares hyperparameters for equality, useful in testing and caching.

- Kernel
  - Base class for all kernels.
  - Core capabilities:
    - Hyperparameter exposure via kernel.hyperparameters.
    - Parameter inspection via kernel.get_params(deep=True).
    - Support for kernel arithmetic (sum, product) and scalar exponentiation.
    - Gradient-based hyperparameter optimization support.

- _check_length_scale(X, length_scale)
  - Validates length_scale against data X.
  - Returns a 1D array with the appropriate length_scale.
  - Raises ValueError for anisotropic dimension mismatches or invalid shapes.

Notes
-----
- This module is inspired by kernel paradigms used in GP libraries and emphasizes flexibility through kernel algebra and gradient-based hyperparameter optimization.
- The exact concrete kernel implementations (e.g., RBF, Matern, Periodic, etc.) are designed to interoperate through the same interface and support hyperparameter tuning in a consistent manner.
