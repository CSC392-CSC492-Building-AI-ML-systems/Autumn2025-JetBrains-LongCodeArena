Introduction

The sklearn.naive_bayes module provides a family of Naive Bayes classifiers that apply Bayes’ theorem with strong (naive) feature independence assumptions. These estimators are suitable for a variety of data representations and are designed for fast training and probabilistic predictions.

Module contents

Public classes
- BernoulliNB
- GaussianNB
- MultinomialNB
- ComplementNB
- CategoricalNB

Common concepts
- All estimators derive from a shared base class (_BaseNB) and expose a consistent API: fit, predict, predict_proba, predict_log_proba, and predict_joint_log_proba.
- They implement task-specific computations of the joint log-likelihood P(x, y) via _joint_log_likelihood and input validation via _check_X.
- Prediction methods rely on the joint log-likelihood, followed by normalization to obtain probabilities.

Base class: _BaseNB

Overview
- Abstract base class for Naive Bayes estimators.
- Defines common public methods used for prediction and logging:
  - predict_joint_log_proba(X): returns log P(x, y) for each sample and class.
  - predict(X): returns predicted class labels.
  - predict_log_proba(X): returns log P(y|x) (up to a normalization by P(x)).
  - predict_proba(X): returns P(y|x).
- Subclasses must implement:
  - _joint_log_likelihood(X): unnormalized posterior log-probabilities.
  - _check_X(X): input validation specific to the estimator.

Key methods (high level)
- fit(X, y, sample_weight=None): train the model (inherited from BaseEstimator).
- partial_fit(X, y, classes=None, sample_weight=None): online updates (where supported, e.g., GaussianNB).
- predict(X), predict_proba(X), predict_log_proba(X), predict_joint_log_proba(X): as described above.

Usage snippet
- Basic usage pattern across estimators:
.. code-block:: python

   from sklearn.naive_bayes import GaussianNB, BernoulliNB, MultinomialNB

   # Prepare data
   X_train = [[0.5, 1.2], [1.0, 0.1], [0.3, 0.8], [2.0, 1.5]]
   y_train = [0, 1, 0, 1]

   # Gaussian Naive Bayes
   gnb = GaussianNB()
   gnb.fit(X_train, y_train)
   preds = gnb.predict([[0.6, 1.0]])
   probs = gnb.predict_proba([[0.6, 1.0]])

   # Multinomial Naive Bayes (suitable for non-negative counts or frequencies)
   mnb = MultinomialNB()
   mnb.fit(X_train, y_train)
   mnb_preds = mnb.predict([[0.2, 1.3]])
   mnb_probs = mnb.predict_proba([[0.2, 1.3]])

Overview of key estimators

GaussianNB

Purpose
- Handles real-valued features with an assumption of Gaussian class-conditional distributions.
- Supports online updates through partial_fit.

Parameters
- priors: array-like of shape (n_classes,), default=None
  Prior probabilities of the classes. If provided, priors are not adjusted from the data.
- var_smoothing: float, default=1e-9
  Portion of the largest variance of all features that is added to variances for calculation stability.

Attributes (typical)
- class_count_: number of training samples observed in each class.
- class_prior_: probability of each class.
- classes_: array of class labels.
- epsilon_: absolute additive value to variances (for numerical stability).
- n_features_in_: number of features seen during fit.
- feature_names_in_: names of features seen during fit (if provided as strings).
- var_: per-class feature variances.
- theta_: per-class feature means.

Example
.. code-block:: python

   from sklearn.naive_bayes import GaussianNB
   X = [[1.0, 2.0], [1.5, 1.8], [5.0, 2.0], [6.0, 1.0]]
   y = [0, 0, 1, 1]
   gnb = GaussianNB()
   gnb.fit(X, y)
   print(gnb.predict([[2.0, 2.0]]))
   print(gnb.predict_proba([[2.0, 2.0]]))

BernoulliNB

Purpose
- Suitable for binary/boolean feature vectors or binarized features.

Parameters
- priors: array-like of shape (n_classes,), default=None
  See GaussianNB for priors behavior.
- binarize: float or None, default=None
  Threshold for binarizing input if X is not already binary.

Notes
- Works well with binary/boolean features after optional binarization.
- Like other NB estimators, relies on independence assumptions conditioned on class.

Example
.. code-block:: python

   from sklearn.naive_bayes import BernoulliNB
   X = [[0, 1, 0], [1, 0, 1], [0, 0, 1], [1, 1, 0]]
   y = [0, 1, 0, 1]
   bnb = BernoulliNB()
   bnb.fit(X, y)
   print(bnb.predict([[1, 0, 1]]))
   print(bnb.predict_proba([[1, 0, 1]]))

MultinomialNB

Purpose
- Appropriate for discrete counts, such as term frequencies in text classification.

Parameters
- alpha: float, default=1.0
  Additive (Laplace/Lidstone) smoothing parameter.
- fit_prior: bool, default=True
  If True, learn class prior probabilities from the data; otherwise, use class_prior if provided.

Notes
- Features must be non-negative counts or frequencies.

Example
.. code-block:: python

   from sklearn.naive_bayes import MultinomialNB
   X = [[2, 0, 1], [1, 3, 0], [0, 0, 4]]
   y = [0, 1, 0]
   mnb = MultinomialNB()
   mnb.fit(X, y)
   print(mnb.predict([[1, 1, 0]]))
   print(mnb.predict_proba([[1, 1, 0]]))

ComplementNB

Purpose
- Designed to address imbalanced datasets and certain text classification scenarios by using complement class statistics.

Parameters
- alpha: float, default=1.0
  Smoothing parameter for complement counts.
- fit_prior / class_prior: see MultinomialNB for prior handling.

Notes
- Often provides improved accuracy on imbalanced data relative to standard MultinomialNB.

Example
.. code-block:: python

   from sklearn.naive_bayes import ComplementNB
   X = [[1, 0, 2], [0, 3, 0], [1, 1, 0]]
   y = [0, 1, 0]
   cnb = ComplementNB()
   cnb.fit(X, y)
   print(cnb.predict([[0, 1, 0]]))
   print(cnb.predict_proba([[0, 1, 0]]))

CategoricalNB

Purpose
- Handles categorical features directly, without one-hot encoding.

Notes
- Suitable for datasets with categorical features and a high cardinality per feature.

Example
.. code-block:: python

   from sklearn.naive_bayes import CategoricalNB
   # Example with small categorical features encoded as integers
   X = [[0, 1], [1, 0], [0, 0], [1, 1]]
   y = [0, 1, 0, 1]
   cbnb = CategoricalNB()
   cbnb.fit(X, y)
   print(cbnb.predict([[0, 1]]))
   print(cbnb.predict_proba([[0, 1]]))

Important considerations and edge cases

- Independence assumption: Naive Bayes relies on conditional independence of features given the class; performance depends on how well this assumption holds for the data.
- Input requirements:
  - GaussianNB operates on real-valued features; consider standardization if features have very different scales.
  - MultinomialNB, ComplementNB require non-negative feature counts or frequencies.
  - BernoulliNB is suitable for binary/boolean features or binarized representations.
  - CategoricalNB expects categorical features and handles them directly.
- Priors handling:
  - By default, class priors are learned from data; you can provide priors to override data-driven priors.
- Numerical stability:
  - GaussianNB uses var_smoothing to add a small value to feature variances to prevent division-by-zero and numerical issues.
- Online updates:
  - GaussianNB supports partial_fit for online learning; other estimators may not support partial_fit or may have limited online behavior.

Edge cases
- Zero counts or zero variance in features can affect the learning process; apply smoothing (var_smoothing for GaussianNB, alpha for discrete NB variants) as appropriate.
- Highly imbalanced classes can affect priors; consider providing class_prior or using ComplementNB in some scenarios.

Usage and integration notes

- All estimators conform to the scikit-learn estimator interface: fit returns self, and predictions/probabilities are exposed via predict, predict_proba, and related methods.
- The base class provides predict_joint_log_proba, which can be useful for debugging or custom scoring that leverages P(x, y).

Summary

This module offers a set of practical Naive Bayes classifiers for diverse data representations (real-valued, binary, counts, categorical). Each estimator is optimized for fast training and probabilistic prediction, with model-specific considerations documented in their respective sections. Use GaussianNB for continuous features, MultinomialNB for count-based data, BernoulliNB for binary features, ComplementNB for imbalanced tasks, and CategoricalNB for direct handling of categorical features.
