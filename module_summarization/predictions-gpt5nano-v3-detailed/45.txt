Overview
========
log.rst documents the logging module responsible for configuring the application's root logger. It provides support for console output, file-based rotating logs, and syslog sinks, along with a custom NOTICE log level. The configuration is driven by the application's Config object and integrated with the asynchronous Timer system for log rotation.

Key Concepts
------------
- Custom log level: NOTICE (numeric value 25) to publish informational messages visible in non-verbose modes.
- Root logger configuration: The module configures the root logger only if no handlers are already attached.
- Sinks:
  - Console: Enabled when the output is a terminal or when ASABFORCECONSOLE is set.
  - File: Rotating file handler with configurable size and backup count.
  - Syslog: Supports multiple transport schemes (tcp, udp, unix socket, or unix sendto) with formatters for different syslog standards.
- Formatting:
  - File output uses StructuredDataFormatter with user-configurable format and date format.
  - Syslog output uses one of several formatters (MacOSXSyslogFormatter, SyslogRFC5424Formatter, SyslogRFC5424microFormatter, SyslogRFC3164Formatter) depending on configuration.

API Summary
============
Constants
---------
- LOG_NOTICE (int): Custom log level (25) used for NOTICE messages.
  - Registered in the logging module with the name "NOTICE".

- _NAME_TO_LEVEL (dict): Maps string representations of log levels to numeric values recognized by Python's logging module, including aliases like "LOG NOTICE" and "NOT SET".

Classes
--------
Logging
-------
Purpose
-------
Configure and manage the application's root logger, including console, file, and syslog handlers.

Constructor
-----------
Logging(app)

- Parameters:
  - app: An application object expected to provide:
    - Loop: The asyncio event loop used for asynchronous tasks (e.g., Timer integration).
    - Other components referenced by the module (e.g., for creating AsyncIOHandler instances, and for rotation scheduling via Timer).

- Behavior:
  - Creates and attaches handlers to the root logger if none exist.
  - May configure console logging, file-based logging, and syslog logging depending on configuration.

Attributes
----------
- RootLogger: The Python logging root logger.
- ConsoleHandler: Optional console handler (stdout).
- FileHandler: Optional RotatingFileHandler for file-based logging.
- SyslogHandler: Optional AsyncIOHandler for syslog transport.

- Note: The class references additional internal components such as StructuredDataFormatter, MacOSXSyslogFormatter, SyslogRFC5424Formatter, SyslogRFC5424microFormatter, and SyslogRFC3164Formatter, which are assumed to be defined elsewhere in the codebase.

Usage and Configuration
=======================

How to instantiate
------------------
The Logging class is intended to be initialized with your application object. Example:

.. code-block:: python

    from log import Logging

    # Assume `app` is your application object with a running event loop (app.Loop)
    # and necessary attributes used by the logging module.
    logger = Logging(app)

Configuration keys (Config)
--------------------------------
- logging:file/path
  - Path to the log file. If empty, file logging is disabled.
- logging:file/backup_count
  - Number of rotated log backups to keep.
- logging:file/backup_max_bytes
  - Maximum size in bytes before rotation occurs.
- logging:file/rotate_every
  - Optional rotation interval, e.g. "5d" (5 days), "10M" (10 minutes), etc.
  - Supported formats: <integer><unit> where unit is one of d (days), H (hours), M (minutes), s (seconds).
- logging:syslog.enabled
  - Enable or disable syslog output.
- logging:syslog/address
  - Destination address for syslog. Can be:
    - Absolute file path starting with '/' for Unix domain sockets (unix-like transport).
    - URL-like address parsed by urllib.parse (tcp, udp, unix-connect, unix-sendto).
- logging:syslog/format
  - Formatter selection for syslog:
    - 'm' => MacOSXSyslogFormatter
    - '5' => SyslogRFC5424Formatter
    - '5micro' => SyslogRFC5424microFormatter
    - otherwise => SyslogRFC3164Formatter
- logging:sd_id
  - Structured Data IDs used by the syslog formatters.
- logging:file/format
  - Format string for StructuredDataFormatter used in the file handler.
- logging/file/datefmt
  - Date format for the file formatter.
- logging:verbose (if present in Config)
  - Influence on console visibility (not detailed here, but related to console output activation).

Usage examples
--------------
Minimal example with file logging and syslog disabled:

.. code-block:: python

    from log import Logging
    # app should provide Loop and other required attributes
    app = ...  # your application instance
    logger = Logging(app)

Example with console forcing and rotating file logs:

.. code-block:: python

    # Assuming Config is already populated with:
    # logging:file/path=/var/log/myapp.log
    # logging:file/backup_count=7
    # logging:file/backup_max_bytes=10485760 (10 MB)
    # logging:file/rotate_every="1d"
    from log import Logging
    app = ...  # your application instance
    logger = Logging(app)

Edge cases and considerations
-----------------------------
- Early initialization: The root logger is configured only if it has no handlers yet to avoid duplicate handlers.
- Console vs. non-tty: Console logging is enabled when stdout is a TTY or ASABFORCECONSOLE is set; behavior may differ in containers.
- File path handling: The module ensures the directory for the log file exists, creating it if necessary.
- Rotation scheduling: If rotate_every is provided and valid, a Timer-based task is scheduled to manage rotation. Invalid values are logged as errors.
- Syslog configuration:
  - If the provided address is invalid, a warning is logged and the syslog handler is not attached.
  - Multiple transport schemes are supported (tcp, udp, unix-connect, unix-sendto) as well as Unix domain socket support via absolute paths.
- Dependencies:
  - StructuredDataFormatter for file logs.
  - AsyncIOHandler, MacOSXSyslogFormatter, SyslogRFC5424Formatter, SyslogRFC5424microFormatter, and SyslogRFC3164Formatter for syslog formatting.
  - Timer for rotation timing.

Implementation Notes
--------------------
- Custom log level: LOG_NOTICE (25) is added to the logging module with the name "NOTICE".
- The _NAME_TO_LEVEL mapping supports multiple spellings and aliases to map string levels to numeric levels, ensuring flexibility in configuration.
