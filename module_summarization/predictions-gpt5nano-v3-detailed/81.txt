Module: models
Introduction
------------
This module provides two primary capabilities for spectral modeling:

- SimpleFitter wrapper: a lightweight wrapper that exposes a moments helper for spectral analysis.
- SpectralModel framework: a flexible, drop-in wrapper (SpectralModel) for building, annotating, and fitting spectral models. It supports multi-component models, parameter information management, and optional fitting backends, with an auxiliary vheightmodel to add a variable-height background to any base model.

vheightmodel: variable-height background wrapper
-----------------------------------------------
Overview
vheightmodel is a factory that wraps a base “zero-height” model and adds a variable-height (background) component. The resulting function behaves as a background-added model when invoked.

Signature
- vheightmodel(zeroheightmodel) -> vhm
  - zeroheightmodel: a callable f(x, ...params) returning model values without a background

Wrapper behavior
- vhm(xax, *pars, **kwargs)
  - If vheight is True (default; can be overridden with kwargs['vheight']), returns:
    zeroheightmodel(xax, *pars[1:], **kwargs) + pars[0]
  - If vheight is False, returns:
    zeroheightmodel(xax, *pars[1:], **kwargs)
  - Parameter order (within pars) is: height, amplitude, shift, width

Docs
- vhm.__doc__ is extended with zeroheightmodel.__doc__ to preserve the base model documentation.

Usage
- Wrap a base model to add a background component.

Code example
```python
import numpy as np
from models import vheightmodel

def base_model(x, A, mu, sigma):
    return A * np.exp(-0.5 * ((x - mu) / sigma) ** 2)

bg_model = vheightmodel(base_model)

x = np.linspace(-5, 5, 100)
# height (background), amplitude, center, width
y = bg_model(x, height=1.0, A=2.0, mu=0.0, sigma=1.0)
```

Generic SpectralModel wrapper
-----------------------------
Overview
SpectralModel is a powerful wrapper class for building and fitting spectral models. It supports:
- Wrapping a user-supplied model function
- Managing parameter information (names, limits, fixed parameters)
- Handling multiple components and annotations
- Optional use of different fitting backends (mpfit or lmfit)
- Integration with parinfo structures (ParinfoList, Parinfo)

Key attributes and concepts
- modelfunc: the user-provided model function (signature: x-axis, followed by parameters)
- npars: number of model parameters
- default_parinfo: a base Parinfo structure describing parameters
- parinfo: a possibly edited copy of default_parinfo used during fitting
- parinfo-related helpers: ParinfoList, Parinfo (from pyspeckit)
- fitunit: optional unit conversion for the x-axis before evaluating the model
- shortvarnames: TeX-friendly names for parameter annotations
- use_lmfit: whether to use lmfit instead of mpfit
- centroid_par: parameter indicating the model’s centroid (e.g., for centroiding)
- fwhm_func / fwhm_pars: optional analytic FWHM (or equivalent) and its parameters
- integral_func: optional analytic integral for the model
- guess_types: sequence indicating how to interpret user-provided guesses (e.g., 'amplitude', 'center', 'width')

Initialization signature
SpectralModel(
  modelfunc, npars,
  shortvarnames=("A","\\Delta x","\\sigma"),
  fitunit=None,
  centroid_par=None,
  fwhm_func=None,
  fwhm_pars=None,
  integral_func=None,
  use_lmfit=False,
  guess_types=('amplitude', 'center', 'width'),
  **kwargs
)

Key behaviors
- Inherits from fitter.SimpleFitter, integrating with the SimpleFitter API (notably the moments helper).
- Creates default parameter information via _make_parinfo and stores it in default_parinfo and parinfo.
- Supports copying and deep copying via __copy__ and __deepcopy__.
- __call__(self, *args, **kwargs): Core entry point for performing a fit. Delegates to the chosen fitting backend (mpfit or lmfit) based on use_lmfit, and respects overrides passed through kwargs. Performs logging for debugging.

Returns
- Fitting typically yields a tuple of (best-fit parameters, model function, parameter errors, chi-squared), with exact structure determined by the underlying fitting backend and wrapper configuration.

Usage concepts
- Compose multi-component models by supplying a modelfunc that aggregates multiple components.
- Use default_parinfo and parinfo to control per-parameter attributes (names, limits, fixed status, etc.).
- Optionally provide fwhm_func and integral_func for analytic handling of line widths and integrals.
- Use guess_types to help translate generic guesses into parameter-specific initial guesses when needed.

Usage example: simple Gaussian model
```python
import numpy as np
from models import SpectralModel

def gauss(x, amp, cen, sigma):
    return amp * np.exp(-0.5 * ((x - cen) / sigma) ** 2)

sm = SpectralModel(
    gauss,
    npars=3,
    shortvarnames=("A", "x0", "\\sigma"),
    fitunit=None,
    guess_types=('amplitude', 'center', 'width')
)

# Example call (exact usage depends on the fitting backend and data)
# x_data = ...

```

Notes and considerations
- Backends: The class supports mpfit by default and can switch to lmfit via use_lmfit. Ensure the required dependencies are installed for your workflow.
- Parinfo handling: The module uses ParinfoList and Parinfo for parameter metadata. This requires pyspeckit’s parinfo structures.
- Documentation inheritance: The class docstring is designed to be augmented by the wrapped model’s __doc__ to preserve helpful parameter descriptions.
- Python version compatibility: The code includes compatibility shims for OrderedDict; test in your target Python environment.
