w_stateprobs
==============

Overview
--------
w_stateprobs is a backward-compatibility shim that exposes the state-to-state
kinetics analysis functionality via the legacy command name. It delegates to
the w_direct tool’s DStateProbs implementation under the subcommand "trace",
and is deprecated in favor of using w_direct directly.

The shim provides:
- A WStateProbs class that defines the subcommand and default file names.
- A WDirect master command that wires the subcommand into the top-level CLI.
- An entry point that emits a deprecation warning and routes invocation to
  the w_direct-based implementation.

Key concepts
------------
- DStateProbs (from westpa.cli.tools.w_direct) is the underlying, current implementation
  handling the computation of average state populations and confidence intervals.
- WStateProbs: a compatibility wrapper that sets:
  - subcommand = 'trace'
  - help_text = 'averages and CIs for path-tracing kinetics analysis'
  - default_output_file = 'stateprobs.h5'
  - default_kinetics_file = 'assign.h5'
- WDirect: the master command wiring the WStateProbs subcommand into the
  program name w_stateprobs.
- Output structure (as described by the underlying tool):
  - /avg_state_pops [state]
  - Optional /state_pop_evolution [window][state] depending on evolution mode
  - Datasets contain benchmarking and bootstrap metadata via mcbs_* attributes.

Classes and relationships
-------------------------
- WStateProbs(DStateProbs)
  - Subcommand: 'trace'
  - help_text: Averages and CIs for path-tracing kinetics analysis
  - default_output_file: stateprobs.h5
  - default_kinetics_file: assign.h5
  - Purpose: Provide backward compatibility by configuring DStateProbs for the trace path.
- WDirect(WESTMasterCommand, WESTParallelTool)
  - prog: 'w_stateprobs'
  - subcommands: [WStateProbs]
  - subparsers_title: 'calculate state-to-state kinetics by tracing trajectories'
  - description: (see below)
  - Purpose: Top-level CLI wrapper that exposes the WStateProbs subcommand under the legacy name.
- entry_point()
  - Behavior: Print deprecation warning and route to WDirect().main()
  - Mechanism: Ensures 'trace' is present as the subcommand, then executes the master command.
  - Note: This is the public API used when the user runs w_stateprobs.

Important parameters and behavior
--------------------------------
- Default files:
  - --output (-o): stateprobs.h5 (default)
  - --kinetics-file: assign.h5 (default)
- Subcommand:
  - trace: the path-tracing kinetics analysis mode (the default and only subcommand provided by the shim)
- Evolution mode (optional, via underlying w_direct options):
  - --evolution-mode: 'cumulative' or 'blocked' (controls how the state-pop evolution window(s) are formed)
  - --start-iter: starting iteration for evolution windows
  - --step-iter: window width increment (for cumulative) or fixed window size (for blocked)
- Datasets in outputs:
  - /avg_state_pops: mean population of each state over the specified range
  - /state_pop_evolution: population evolution across windows (if evolution mode is requested)
- Attributes in evolution datasets:
  - mcbs_alpha: confidence interval alpha value
  - mcbs_nsets: number of bootstrap datasets
  - mcbs_acalpha: alpha for correlation-length calculation
- Return value:
  - Exit status indicating success or failure; output is written to the specified HDF5 file.

Usage notes and edge cases
--------------------------
- Deprecation:
  - Running w_stateprobs will emit a deprecation warning recommending w_direct instead.
- Subcommand handling:
  - If invoked without a subcommand, the wrapper injects 'trace' as the subcommand to preserve the old behavior.
- Parallel execution:
  - The wrapper inherits WESTParallelTool; behavior with MPI or multiple ranks follows the underlying WDirect/DStateProbs behavior.
- Compatibility caveats:
  - This wrapper exists to preserve scripts and user workflows; prefer invoking w_direct directly for new tooling.

Usage examples
--------------
Basic usage (legacy name, with deprecation warning):
.. code-block:: console

   $ w_stateprobs

Basic usage with explicit trace and custom output:
.. code-block:: console

   $ w_stateprobs trace --output stateprobs.h5 --evolution-mode cumulative --start-iter 0 --step-iter 10

Specifying input kinetics/assignment file (if needed, via underlying options):
.. code-block:: console

   $ w_stateprobs trace --output stateprobs.h5 --kinetics-file assign.h5

Viewing help to understand available options:
.. code-block:: console

   $ w_stateprobs --help

In all cases, the resulting HDF5 file (default: stateprobs.h5) contains:
- /avg_state_pops
- Optional /state_pop_evolution (depending on evolution mode)

Notes
-----
- For new tooling, migrate to w_direct and its DStateProbs tooling.
- The default files can be overridden with the corresponding CLI options as described above.
- The exact option names come from DStateProbs via w_direct; see the w_direct documentation for complete details.
