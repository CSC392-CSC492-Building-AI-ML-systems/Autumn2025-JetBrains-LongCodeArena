diophantine.rst
===============

Overview
--------
This module provides a solver for Diophantine equations as part of the symbolic math framework. The primary entry point, diophantine(eq, param, syms), converts a given equation into a product of factors that must be equal to zero, solves each factor independently, and then combines the results into a set of consistent solutions for the variables involved. Solutions are returned as a set of tuples, with each tuple containing the values for the variables in a deterministic order (alphabetical by default, or as specified via the syms parameter). A symbolic parameter t (default t) is used to express infinite families of solutions.

Public API
----------
diophantine(eq, param=symbols('t', integer=True), syms=None)

- Purpose
  Solve a Diophantine equation by transforming it into factors that must vanish and solving each factor with diop_solve, then merging results.

- Parameters
  - eq: Relational or Expr
    The equation to solve (e.g., Eq(x**2 - y**2, 0) or x**2 - y**2). If a Relational, the left-hand side minus the right-hand side is used.
  - param: Symbol, optional
    The parameter symbol to use for parametric solutions. Default is t, assumed to be integer-valued.
  - syms: sequence of Symbol, optional
    Specifies the order of variables in the returned solution tuples. If provided, only variables present in the equation are kept, and the final tuples are ordered according to this sequence.

- Returns
  - A set of tuples representing the solutions. Each tuple provides a solution for every free symbol (variable) in the equation, ordered according to the variable order determined by syms (or alphabetical order by default). Elements may involve the parameter t. If the equation is identically zero, a trivial solution with the parameter is returned, e.g., {(t,)}.

- Behavior and Notes
  - If eq is an equality (Eq), it is converted to an expression by subtracting rhs from lhs.
  - If the resulting equation is identically zero, a trivial parametric solution is returned.
  - The method attempts to factor and classify the polynomial; for certain known diophantine types, it merges partial solutions into a full solution set.
  - If the expression cannot be treated as a polynomial with rational coefficients, a TypeError is raised.
  - Special handling exists to avoid expensive factoring for expressions classified as general_sum_of_squares, while still attempting to solve them.
  - The function relies on classify_diop and diop_solve internally; See Also for related solvers.

Examples
--------

Basic two-variable difference of squares:

.. code-block:: python

    diophantine(x**2 - y**2)

Output (example):
.. code-block:: none

    {(t_0, -t_0), (t_0, t_0)}

Factoring with three variables (merging results from factors):

.. code-block:: python

    diophantine(x*(2*x + 3*y - z))

Output (example):
.. code-block:: none

    {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}

Quadratic in x and linear in y with a constant term in x:

.. code-block:: python

    diophantine(x**2 + 3*x*y + 4*x)

Output (example):
.. code-block:: none

    {(0, n1), (3*t_0 - 4, -t_0)}

Ordering of variables via syms
--------------------------------
You can control the order of the variables in the returned tuples by supplying syms. The set of variables in the result is ordered according to syms (filtered to the variables present in the equation). If syms is not the full variable list, the results are reordered to match the requested variable order.

.. code-block:: python

    diophantine(x**2 - y**2, syms=[y, x])

Output (example):
.. code-block:: none

    {(-t_0, t_0), (t_0, t_0)}

Notes
-----
- The heart of the algorithm is: convert eq to a polynomial, factorize (when appropriate), classify sub-expressions with classify_diop, and solve each sub-expression with diop_solve. The final solutions are merged to form the complete solution set.
- If factoring would cause excessive computation for certain expressions (e.g., general_sum_of_squares), the implementation may bypass factoring for those terms to avoid hangs.
- If no non-trivial solutions exist, an empty set is returned unless the equation with the trivial zero substitution yields a valid parametric solution, in which case that is returned.

See Also
--------
- classify_diop: Helper that classifies diophantine terms (internal to the module).
- diop_solve: The solver used for individual diophantine subproblems (internal to the solver stack).

Internal helpers (documentation but not intended for normal use)
---------------------------------------------------------------
These are used by diophantine() internally and are not part of the public API. They support low-level operations such as integer checks, tuple normalization, gcd reduction, and rational normalization.

- _is_int(i)
  Returns True if i can be interpreted as an integer (via as_int).

- _sorted_tuple(*i)
  Returns a sorted tuple of the provided items.

- _remove_gcd(*x)
  Attempts to divide all components by their gcd and return the reduced tuple. If gcd cannot be determined, returns the original input.

- _rational_pq(a, b)
  Returns a reduced rational pair (numerator, denominator) for a/b without common factors.

- _nint_or_floor(p, q)
  Returns the integer closest to p/q; in case of a tie, returns floor(p/q).

- _odd(i), _even(i)
  Utility predicates for parity checks.

Diophantine types known to be handled
-------------------------------------
The module maintains a set of known diophantine problem types that influence solving strategies:

- diop_known includes:
  - binary_quadratic
  - cubic_thue
  - general_pythagorean
  - general_sum_of_even_powers
  - general_sum_of_squares
  - homogeneous_general_quadratic
  - homogeneous_ternary_quadratic
  - homogeneous_ternary_quadratic_normal
  - inhomogeneous_general_quadratic
  - inhomogeneous_ternary_quadratic
  - linear
  - univariate

Usage Notes
-----------
- Ensure that the equation is polynomial with rational coefficients for reliable results. If not, a TypeError will be raised.
- The parameter t is an integer symbol by default and is used to express infinite solution families.
- The output is a set of tuples; the exact form depends on the variable ordering and the presence of parameters.

End of documentation.
