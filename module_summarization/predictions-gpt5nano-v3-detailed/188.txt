Kernel approximation
====================

Overview
--------

This module provides tools to approximate feature mappings corresponding to kernel functions. By leveraging Fourier transforms and Count Sketches, these estimators enable scalable kernel methods by mapping input data into finite-dimensional feature spaces that approximate kernel evaluations.

Key components
--------------

- PolynomialCountSketch: Tensor Sketch-based approximation of the polynomial kernel feature map
  - K(X, Y) = (gamma * <X, Y> + coef0)^degree
  - Uses Count Sketch and FFT to efficiently approximate the outer-product feature representation
  - Suitable for large-scale datasets where exact polynomial features would be prohibitive

Related components (context)
----------------------------

Other estimators in this module (not shown here) provide efficient approximations for different kernels, including:
- RBFSampler: Random Fourier features for the RBF kernel
- AdditiveChi2Sampler: Feature map for the additive chi-squared kernel
- SkewedChi2Sampler: Feature map for the skewed chi-squared kernel
- Nystroem: Kernel map approximation using a subset of training data

API reference
-------------

PolynomialCountSketch
~~~~~~~~~~~~~~~~~~~~~

The PolynomialCountSketch estimator provides a fast, approximate feature map for polynomial kernels using Tensor Sketch (Count Sketch) techniques.

Parameters
~~~~~~~~~~

- gamma : float, default=1.0
  Parameter of the polynomial kernel; scales the inner product.

- degree : int, default=2
  Degree of the polynomial kernel to approximate.

- coef0 : float, default=0
  Constant term added to the polynomial kernel (bias).

- n_components : int, default=100
  Dimensionality of the output feature space.

- random_state : int, RandomState instance, default=None
  Controls random number generation for indexHash and bitHash initialization.

Attributes
~~~~~~~~~

- indexHash_ : ndarray of shape (degree, n_features), dtype=int64
  Hash indices used for the Count Sketch representation.

- bitHash_ : ndarray of shape (degree, n_features), dtype=float32
  Random entries in {+1, -1} used by the Count Sketch.

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,), dtype=object
  Names of features seen during fit (if input features have string names).

- _n_features_out_ or _n_features_out
  Internal attribute representing the output dimensionality (equal to n_components).

Notes
~~~~~

- If coef0 != 0, the feature space effectively includes an extra feature to account for the constant term, increasing the effective input feature count by 1 during hashing.

- The estimator accepts dense arrays and sparse CSC input during fitting.

- After fit, transform (or fit_transform) yields an array of shape (n_samples, n_components).

Data flow and methods
~~~~~~~~~~~~~~~~~~~~~

- fit(X, y=None)
  - Validates X, initializes random hashes (indexHash_ and bitHash_), and stores the effective output dimensionality.

- transform(X)
  - Generates the approximate polynomial-kernel feature map for X.

Usage example
-------------

```python
from sklearn.kernel_approximation import PolynomialCountSketch
from sklearn.linear_model import SGDClassifier

# Sample data
X = [[0, 0], [1, 1], [1, 0], [0, 1]]
y = [0, 0, 1, 1]

# Tensor Sketch-based polynomial kernel approximation
ps = PolynomialCountSketch(degree=3, random_state=1)
X_features = ps.fit_transform(X)

# Train a classifier on the approximate features
clf = SGDClassifier(max_iter=10, tol=1e-3)
clf.fit(X_features, y)

# Evaluate
score = clf.score(X_features, y)
print("Accuracy on transformed features:", score)
```

Edge cases and considerations
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- The choice of n_components balances accuracy and computation time; larger values yield closer approximations to the exact polynomial kernel but require more memory and compute.

- Random_state ensures reproducibility of the sketch; set it when reproducibility is important.

- When coef0 is non-zero, expect a slight increase in the effective feature count due to the bias term.

- Ensure input X is compatible (dense or CSC sparse); the implementation validates input shape and type.
