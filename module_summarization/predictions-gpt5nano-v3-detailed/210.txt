Empirical likelihood
====================

Empirical likelihood inference on descriptive statistics. This module provides hypothesis testing and confidence interval construction for descriptive statistics such as the mean, variance, skewness, kurtosis, and correlation. When matplotlib is available, it can also produce multivariate confidence region plots and mean–variance contour plots.

General references
------------------
Owen, A. (2001). "Empirical Likelihood." Chapman and Hall

Overview
--------

The module centers on empirical likelihood (EL) methods for descriptive statistics. It supports both univariate and multivariate settings:

- Univariate (DescStatUV) and multivariate (DescStatMV) implementations are selected by the public factory function DescStat(endog).
- EL-based optimization is encapsulated in an internal helper class _OptFuncts, which assembles estimating equations (est_vect) and solves for the Lagrange multipliers (eta) that maximize the profile likelihood.
- The optimization relies on a modified Newton approach (_modif_newton) with auxiliary gradient (_grad) and Hessian (_hess) calculations. The final EL-based statistics are used to form likelihood-ratio-based confidence regions and hypothesis tests.

Public API
----------

DescStat(endog)
^^^^^^^^^^^^^^^^

Factory that returns a descriptive-statistics EL inference object.

Parameters
- endog : ndarray
  Data array. If endog is 1-dimensional, it is reshaped to (n, 1). If endog has a single column, a univariate EL object (DescStatUV) is returned; if more than one column, a multivariate EL object (DescStatMV) is returned.

Returns
- DescStat instance
  - If k = 1, returns DescStatUV.
  - If k > 1, returns DescStatMV.

Internal: _OptFuncts
^^^^^^^^^^^^^^^^^^^^^^^

An internal class that constructs and optimizes the estimating equations necessary for empirical likelihood.

Notes
- This class is intended for internal use by the DescStat family and is not part of the public API.
- It provides the core optimization machinery used by the descriptive-statistics EL procedures.

Public methods (internal)
- _log_star(eta, est_vect, weights, nobs)
  Computes the transformed log-likelihood “star” values for the EL problem given a Lagrange multiplier eta.

  Parameters
  - eta : float or array-like
    Lagrange multiplier.
  - est_vect : ndarray (n, k)
    Estimating equations vector for each observation.
  - weights : nx1 array
    Observation weights.
  - nobs : int
    Number of observations.

  Returns
  - data_star : ndarray
    The weighted log-star values (same length as nobs).

- _grad(eta, est_vect, weights, nobs)
  Gradient of the log-star objective with respect to eta.

  Returns
  - gradient : ndarray (m, 1)
    The gradient vector.

- _hess(eta, est_vect, weights, nobs)
  Hessian of the objective with respect to eta.

  Returns
  - hess : ndarray
    The Hessian matrix.

- _modif_newton(eta, est_vect, weights)
  Modified Newton’s method to maximize the log-star objective and solve for eta.

  Returns
  - params : 1 x m array
    Lagrange multiplier that maximizes the log-likelihood.

- _find_eta(eta)
  Placeholder for auxiliary routine to locate optimal eta (implementation not shown in excerpt).

Key usage notes
- The _OptFuncts class is designed to be used by the DescStat classes to compute EL-based statistics. Direct usage is not intended for end users.
- The methods rely on the observed estimating equations est_vect and observation weights to form the EL optimization problem.

Important parameters and return values
- endog: ndarray
  Input data. Shape (n,) or (n, k). Reshaping behavior ensures proper univariate/multivariate handling.

- eta: ndarray
  Lagrange multiplier vector in EL optimization.

- est_vect: ndarray
  Matrix of estimating equations with shape (n, k).

- weights: ndarray
  1D array of observation weights (length n).

- nobs: int
  Number of observations (n).

Usage examples
--------------

Basic univariate usage (conceptual)
- Create a descriptive-statistics EL object for a 1D dataset and perform EL-based inference on the mean, variance, etc. The exact API calls depend on the concrete DescStatUV implementation (not shown here).

Example: constructing the descriptor
- Note: This is a minimal illustration of the intended usage pattern.

```python
import numpy as np
from emplike import DescStat

ds = DescStat(data)

# ds now provides EL-based methods to test hypotheses and build CIs
```

Edge cases and considerations
- Input shape handling: If endog is 1D, it is reshaped to (n, 1). If endog has a single column, a univariate EL object is returned; otherwise, a multivariate EL object is returned.
- Numerical stability: The _log_star implementation contains conditional handling for near-extreme log-likelihood values and uses a Newton-based solver with a tolerance setting (tol=1e-8).
- Dependencies: Optional plotting requires matplotlib. Core functionality relies on numpy, scipy.optimize, scipy.stats, and statsmodels optimizer utilities.
- Documentation scope: The internal _OptFuncts class is not part of the public API; it is referenced by the higher-level DescStat classes to perform optimization.

Implementation notes
--------------------
- The module ties together data handling (DescStat factory), estimand construction (via estimating equations), and numerical optimization (modified Newton method) to support EL-based inference for descriptive statistics.
- The design favors a clean separation between input data processing (DescStat) and optimization machinery (_OptFuncts), enabling extensibility to additional statistics (mean, variance, skewness, kurtosis, correlation) without changing the high-level API.
