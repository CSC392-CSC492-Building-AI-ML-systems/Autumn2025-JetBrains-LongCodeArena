Naive Bayes methods
===================

Overview
--------
The sklearn.naive_bayes module implements Naive Bayes classifiers, a family of
supervised learning algorithms based on Bayes' theorem with strong (naive)
feature independence assumptions. They are fast, easy to implement, and often
serve as strong baseline classifiers, especially for high-dimensional data.

Key ideas:
- For each class y, model P(x|y) under simple distributional assumptions (e.g., Gaussian, multinomial, Bernoulli, etc.).
- Combine with class priors P(y) to form the posterior P(y|x) via Bayes' rule.
- Predict by selecting the class with the highest posterior (or joint log-likelihood).

Public classes
--------------
The module exposes the following Naive Bayes classifiers:

- BernoulliNB
- GaussianNB
- MultinomialNB
- ComplementNB
- CategoricalNB

All classes implement a common API (via _BaseNB) that provides:
- predict(X): class labels for samples in X
- predict_proba(X): class probabilities for samples in X
- predict_log_proba(X): log-probabilities for each class
- predict_joint_log_proba(X): unnormalized joint log probabilities log P(x, y)

Common internal mechanism:
- _joint_log_likelihood(X): abstract method implemented by each subclass to compute
  log P(y) + log P(x|y) for all samples and classes.
- _check_X(X): subclass-specific input validation used by predict* methods.
- Internally, joint log-likelihoods are normalized as needed (e.g., using logsumexp)
  to produce probabilities.

GaussianNB
----------
Gaussian Naive Bayes models the class-conditional feature distributions as Gaussians.
It can perform online updates to model parameters via partial_fit.

Parameters
- priors: array-like of shape (n_classes,), default=None
  Prior probabilities of the classes. If None, priors are inferred from the training data.
- var_smoothing: float, default=1e-9
  Portion of the largest variance of all features that is added to variances for
  calculation stability.

Attributes (after fit)
- class_count_: ndarray of shape (n_classes,)
  Number of training samples observed in each class.
- class_prior_: ndarray of shape (n_classes,)
  Estimated prior probability for each class.
- classes_: ndarray of shape (n_classes,)
  Class labels known to the classifier.
- epsilon_ (or equivalent): float
  Absolute additive value to variances for numerical stability.
- n_features_in_: int
  Number of features seen during fit.
- feature_names_in_: ndarray of shape (n_features_in_,)
  Names of features seen during fit (if X has feature names that are strings).
- var_: ndarray of shape (n_classes, n_features)
  Variance of each feature per class.
- theta_: ndarray of shape (n_classes, n_features)
  Mean of each feature per class.

Notes
- Uses log-space computations (log P(y) and log P(x|y)) to maintain numerical stability.
- For P(x) normalization in predict_proba, log probabilities are stabilized via logsumexp.

Examples
.. code-block:: python

    from sklearn.naive_bayes import GaussianNB
    from sklearn.datasets import make_classification

    # Create a simple synthetic dataset
    X, y = make_classification(n_features=4, n_classes=3, random_state=0)

    # Initialize and fit the model
    gnb = GaussianNB(priors=None, var_smoothing=1e-9)
    gnb.fit(X, y)

    # Predict on new samples
    y_pred = gnb.predict(X)

    # Predict class probabilities
    proba = gnb.predict_proba(X)

    # Predict log probabilities
    log_proba = gnb.predict_log_proba(X)

    # Joint log-probabilities (log P(x, y))
    jll = gnb.predict_joint_log_proba(X)

Notes and edge cases
- Independence assumption: Naive Bayes assumes feature independence given the class. Violations may reduce accuracy.
- Numerical stability: var_smoothing helps prevent division-by-zero in variance estimates.
- Feature requirements vary by variant:
  - GaussianNB: suitable for continuous, real-valued features.
  - MultinomialNB / BernoulliNB: typically used for discrete, non-negative features (e.g., text counts or binary/bernoulli features). See the respective class docs for exact requirements.
- Priors: If priors are provided, they override the class distribution inferred from the data.
- Online updates: GaussianNB supports online updates via partial_fit; other NB variants may not implement online updates.

Usage examples (end-to-end)
- Classification with GaussianNB (as shown above) is representative for continuous features.
- For text classification or bag-of-words inputs, MultinomialNB is a common choice:
.. code-block:: python

    from sklearn.naive_bayes import MultinomialNB
    from sklearn.feature_extraction.text import TfidfVectorizer
    from sklearn.pipeline import make_pipeline

    texts = ["spam spam eggs", "eggs and bacon", "spam spam sausage", "bacon eggs"]
    labels = ["spam", "ham", "spam", "ham"]

    model = make_pipeline(TfidfVectorizer(stop_words="english"), MultinomialNB())
    model.fit(texts, labels)
    print(model.predict(["spam eggs"]))  # example prediction
    print(model.predict_proba(["spam eggs"]))  # class probabilities

Edge considerations by variant
- BernoulliNB: appropriate for binary/boolean features; features are typically binarized.
- ComplementNB: designed to handle imbalanced datasets by adjusting the complement distributions.
- CategoricalNB: suitable for categorical features with a finite number of categories; supports smoothing and prior settings.

This documentation provides a concise reference to the Naive Bayes family in sklearn.naive_bayes, with emphasis on GaussianNB due to its explicit parameterization in the core docstring. For complete API signatures and parameter options, refer to the individual class docs.
