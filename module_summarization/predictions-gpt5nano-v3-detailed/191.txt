Validation curves: plotting scores to evaluate models
======================================================

Overview
--------

This module provides visualization utilities for plotting learning curves and validation curves
to assess model performance as training size or hyperparameters vary. It defines a
public display class, LearningCurveDisplay, built on top of internal plotting helpers that
render the curve(s) and associated uncertainty (where available) using Matplotlib.

Key ideas:
- Learn how model performance on training and/or validation sets evolves with more data (learning curves).
- Compare different score curves (train vs. test) in a single plot.
- Support multiple styles for representing score variability:
  - fill_between: shaded region around the mean score using standard deviation
  - errorbar: explicit error bars per point
  - None: plot only the mean scores
- Handle naming and scaling of the score axis, including a deprecation path for log_scale.

Public API
----------

LearningCurveDisplay
^^^^^^^^^^^^^^^^^^^^

Purpose
- Visualize learning curves produced by sklearn.model_selection.learning_curve (and related validation curves).

Constructor parameters
- train_sizes (ndarray, shape (n_ticks,)): The number of training samples used at each point.
- train_scores (ndarray, shape (n_ticks, n_cv_folds)): Scores on the training sets.
- test_scores (ndarray, shape (n_ticks, n_cv_folds)): Scores on the test/validation sets.
- score_name (str, default=None): Name of the score displayed on the y-axis. If None, the display derives a suitable name from the scoring strategy and negate_score.

Attributes
- ax_ (matplotlib.Axes): Axes containing the plot.
- figure_ (matplotlib.Figure): Figure containing the plot.
- errorbar_ (list[Artist] or None): If std_display_style is "errorbar", contains the errorbar artists.
- lines_ (list[Line2D] or None): If std_display_style is "fill_between" or None, contains the line artists for mean scores.
- fill_between_ (list[PolyCollection] or None): If std_display_style is "fill_between", contains the shaded regions.

Notes on usage
- It is recommended to create instances via LearningCurveDisplay.from_estimator(...) for convenience.
- The display supports comparing training and/or test scores via score_type:
  - "train": plot only training scores
  - "test": plot only test scores
  - "both": plot both training and test scores

Plotting behavior and options
- _plot_curve (internal method) handles the actual rendering, given x_data (e.g., train sizes).
- std_display_style controls how variability is shown:
  - "fill_between": shaded region around the mean score (default behavior)
  - "errorbar": per-point error bars
  - None: only mean lines, no uncertainty visualization
- score_type determines which curves to display: "train", "test", or "both".
- Score naming and scaling:
  - If score_name is not provided, a default is derived from the model’s scoring configuration.
  - The y-axis is labeled with the chosen score name.
  - The x-axis scale is chosen by heuristics that consider the spread of x_data, with a deprecated log_scale parameter kept for backward compatibility.

Internal helpers referenced
- _validate_score_name: Used to determine a user-friendly score label when score_name is not explicitly provided.
- _interval_max_min_ratio: Used to decide between linear, log, or symlog x-scales based on the range and spacing of train_sizes.
- check_matplotlib_support: Ensures Matplotlib is available before plotting.

Usage examples
-------------

Example 1: Basic usage with precomputed arrays
```python
from sklearn.model_selection import learning_curve
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_digits
from sklearn.model_selection import LearningCurveDisplay
import numpy as np

X, y = load_digits(return_X_y=True)
train_sizes, train_scores, test_scores = learning_curve(
    LogisticRegression(max_iter=1000),
    X, y, cv=5, scoring="accuracy"
)

display = LearningCurveDisplay(
    train_sizes=train_sizes,
    train_scores=train_scores,
    test_scores=test_scores,
    score_name="Accuracy",
)

display.plot()
```

Example 2: Using from_estimator (recommended)
```python
from sklearn.model_selection import LearningCurveDisplay
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_digits
import numpy as np

X, y = load_digits(return_X_y=True)

display = LearningCurveDisplay.from_estimator(
    estimator=LogisticRegression(max_iter=1000),
    X=X,
    y=y,
    scoring="accuracy",
    cv=5,
    train_sizes=np.linspace(0.1, 1.0, 5)
)

display.plot()
```

Example 3: Validation curve visualization (via scores from validation_curve)
```python
from sklearn.model_selection import validation_curve
from sklearn.svm import SVC
from sklearn.datasets import load_digits
from sklearn.model_selection import LearningCurveDisplay

X, y = load_digits(return_X_y=True)
param_range = [0.1, 1, 10]

train_scores, test_scores = validation_curve(
    SVC(), X, y, param_name="gamma", param_range=param_range, cv=5, scoring="accuracy"
)

# Prepare the display (train vs. test curves across hyperparameter values)
display = LearningCurveDisplay(
    train_sizes=np.arange(len(param_range)),
    train_scores=train_scores,
    test_scores=test_scores,
    score_name="Accuracy",
)
display.plot()
```

Important considerations and edge cases
- Deprecation path: The log_scale argument is deprecated. Use display.ax_.set_xscale / set_yscale on the produced axes instead.
- Score naming: If score_name is not provided, the display derives a human-friendly label from the scoring strategy; see _validate_score_name for details.
- x-scale selection: The code automatically selects an appropriate x-axis scale (linear, log, or symlog) based on the spread of train_sizes, using _interval_max_min_ratio.
- std_display_style validation: Valid options are "fill_between", "errorbar", or None. Any other value raises a ValueError.
- Edge symmetry: When plotting multiple curves (Train vs. Test), the legend is added to help distinguish them.
- Retrieving artists: After plotting, the internal attributes (ax_, figure_, lines_, fill_between_, errorbar_) can be inspected to customize or modify the plotted elements.
