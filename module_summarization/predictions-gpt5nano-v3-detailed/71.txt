Kernel Ridge Regression
=======================

Overview
--------

The kernel_ridge module implements kernel ridge regression (KRR), which combines
ridge regression with the kernel trick. KRR learns a linear function in the
feature space induced by a kernel, enabling non-linear fitting in the original
input space. Unlike support vector regression (SVR), KRR uses a squared-error
loss with L2 regularization and admits a closed-form solution, which can be more
efficient for medium-sized datasets. The learned model is generally non-sparse.

This estimator supports multi-target regression (i.e., y with shape
[n_samples, n_targets]).

Key relationships
-----------------

- Related to SVR in the sense that both can use kernels, but KRR uses squared-error
  loss, is closed-form, and yields non-sparse models.
- Conceptually similar to ridge regression but in the kernel-induced feature space.

Usage is common in scenarios where a closed-form solution is desirable and the
dataset size is moderate.

API reference
-------------

KernelRidge class
~~~~~~~~~~~~~~~~~

This class implements kernel ridge regression and is designed to be used
through scikit-learn’s estimator API (fit/predict) with support for multi-output
regression.

Parameters
----------

alpha : float or array-like of shape (n_targets,), default=1.0
    Regularization strength. Must be positive. If an array is provided, penalties
    are assumed to be target-specific and must match the number of targets.

kernel : str or callable, default="linear"
    Kernel mapping used internally. Passed directly to pairwise_kernels when a
    string is given. If "precomputed", X is treated as a precomputed kernel matrix.
    If a callable, it is ignored by pairwise_kernels and should not be used with
    pairwise_kernels; the callable is not allowed to operate on matrices.

gamma : float, default=None
    Gamma parameter for certain kernels (e.g., RBF, polynomial, sigmoid). Ignored
    by kernels that do not use this parameter. If None, the kernel’s default
    behavior is used.

degree : int, default=3
    Degree of the polynomial kernel. Ignored by other kernels.

coef0 : float, default=1
    Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

kernel_params : dict, default=None
    Additional parameters (keyword arguments) for the kernel function, when the
    kernel is provided as a callable or via pairwise_kernels.

Attributes
----------

dual_coef_ : ndarray of shape (n_samples,) or (n_samples, n_targets)
    Representation of the model in kernel space (the “dual” coefficients).

X_fit_ : ndarray or sparse matrix of shape (n_samples, n_features)
    Training data. If kernel == "precomputed", this holds the precomputed kernel
    matrix of shape (n_samples, n_samples).

n_features_in_ : int
    Number of features observed during fit.

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features observed during fit, if input features have string names.

(Additional notes on attributes may appear in the full documentation.)

Fitting and prediction
----------------------

- fit(X, y) trains the model using the specified kernel and regularization.
- predict(X) returns predicted values with shape (n_samples,) for single-target
  regression or (n_samples, n_targets) for multi-target regression.

Important considerations and edge cases
--------------------------------------

- Kernel choice: The behavior and expressiveness of the model depend strongly on
  the kernel. The RBF, polynomial, and other kernels enable non-linear fitting in
  the input space.
- Precomputed kernel: If kernel="precomputed", X must be a kernel matrix of shape
  (n_samples, n_samples) during fit and predict.
- Callable kernels: If kernel is a Python function, it must accept two 1D
  samples (rows) and return a scalar kernel value. Callables that operate on
  matrices (e.g., some utilities in pairwise) are not allowed in this path.
- Multi-output targets: The estimator supports y with shape (n_samples, n_targets).
  The alpha parameter may be a scalar or an array with length n_targets.
- Numerical stability: Regularization (alpha) improves conditioning; very small
  alpha may lead to numerical issues for ill-conditioned kernel matrices.

Usage examples
--------------

Basic usage with a linear kernel
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

```python
from sklearn.kernel_ridge import KernelRidge
import numpy as np

# Sample data
rng = np.random.RandomState(0)
n_samples, n_features = 10, 5
X = rng.randn(n_samples, n_features)
y = rng.randn(n_samples)

# Kernel ridge with linear kernel
krr = KernelRidge(alpha=1.0, kernel="linear")
krr.fit(X, y)
y_pred = krr.predict(X)
```

Non-linear fitting with RBF kernel
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

```python
from sklearn.kernel_ridge import KernelRidge
import numpy as np

rng = np.random.RandomState(42)
n_samples, n_features = 100, 2
X = rng.randn(n_samples, n_features)
true_coefs = rng.randn(n_features)
y = X @ true_coefs + rng.randn(n_samples) * 0.1  # mildly noisy

krr = KernelRidge(alpha=0.1, kernel="rbf", gamma=0.5)
krr.fit(X, y)
pred = krr.predict(X)
```

Precomputed kernel matrix
~~~~~~~~~~~~~~~~~~~~~~~~

```python
from sklearn.kernel_ridge import KernelRidge
import numpy as np

# Compute a kernel matrix K for training data
# K_train[i, j] = kernel(X[i], X[j])
K = pairwise_kernels(X, X, metric="rbf", gamma=0.5)

krr = KernelRidge(alpha=1.0, kernel="precomputed")
krr.fit(K, y)
```

Notes
-----

- The implementation follows a closed-form solution based on the Cholesky factorization
  of the kernel matrix augmented with the regularization term.
- For large datasets, consider alternatives that produce sparse models (e.g.,
  SVR with epsilon-insensitive loss) or approximate kernel methods.
