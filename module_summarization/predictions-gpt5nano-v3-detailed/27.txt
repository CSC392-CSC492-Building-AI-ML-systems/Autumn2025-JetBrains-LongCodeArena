# w_kinetics

Overview
- The w_kinetics command provides backward-compatible access to WestPA’s path-tracing kinetics analysis. It ultimately delegates to the w_direct tooling infrastructure (trace subcommand) to compute state-to-state kinetics by tracing trajectories.
- This wrapper is deprecated in favor of using w_direct directly. A deprecation warning is emitted when the command is run.

Key components and relationships
- WKinetics
  - Subclass of DKinetics (from westpa.cli.tools.w_direct)
  - Subcommand: trace
  - Help text: “averages and CIs for path-tracing kinetics analysis”
  - Default output: kintrace.h5
- WDirect
  - Inherits WESTMasterCommand and WESTParallelTool
  - prog: w_kinetics
  - subcommands: [WKinetics]
  - subparsers_title: “calculate state-to-state kinetics by tracing trajectories”
  - description: provides the detailed usage and output information for the kin trace workflow
- Entry point behavior
  - entry_point() emits a deprecation warning and forwards the invocation to the appropriate trace subcommand.
  - Backwards compatibility: If invoked as a non-subcommand form, the wrapper will insert the trace subcommand to preserve existing usage patterns.

Input and output
- Input
  - Requires a bin assignment file (usually assign.h5) that includes trajectory labeling. See w_assign --help for guidance on generating this file.
  - The input path is provided as part of the command invocation (the exact option names are inherited from the w_direct tooling).
- Output
  - Default output file: kintrace.h5
  - Output datasets (in kintrace.h5):
    - /conditional_fluxes [iteration][state][state]
      - Floating-point; macrostate-to-macrostate fluxes (not normalized by initial macrostate population)
    - /conditional_arrivals [iteration][stateA][stateB]
      - Integer; number of trajectories arriving at stateB in a given iteration from stateA
    - /total_fluxes [iteration][state]
      - Floating-point; total flux into a given macrostate
    - /arrivals [iteration][state]
      - Integer; trajectories arriving at a given state in a given iteration
    - /duration_count [iteration]
      - Integer; number of event durations recorded in each iteration
    - /durations [iteration][event duration]
      - Structured dataset with fields:
        - istate (Integer): Initial state of the transition
        - fstate (Integer): Final state of the transition
        - duration (Floating-point): Duration of transition, in units of tau
        - weight (Floating-point): Weight of the trajectory at the end of the transition (not normalized by initial state population)
- Notes
  - The fluxes are not normalized by the initial macrostate population, so they cannot be treated as rates without normalization.
  - Normalization to obtain rates (with uncertainties) is performed by the w_kinavg tool, using the trace output as input.

Usage and options
- The command is invoked as a subcommand under w_kinetics, with the trace subcommand selected by default via the wrapper.
- Basic invocation (example):
  - w_kinetics trace -o kintrace.h5 assign.h5
- Backwards-compatibility behavior
  - If you run w_kinetics with a non-subcommand form, the entry point inserts the trace subcommand to preserve older usage:
    - Example: w_kinetics assign.h5
    - Internally becomes: w_kinetics trace assign.h5
  - A deprecation warning is emitted informing users to prefer w_direct directly.

Example: a complete workflow
- Prepare the assignment file (assign.h5) using w_assign or equivalent tooling.
- Run the kinetics trace analysis:
  - w_kinetics trace -o kintrace.h5 assign.h5
- Inspect the produced kintrace.h5 datasets (e.g., with an HDF5 viewer or WestPA tooling).
- Normalize to rates and compute confidence intervals with w_kinavg:
  - w_kinavg trace kintrace.h5

Edge cases and considerations
- Deprecation: The w_kinetics wrapper is deprecated; migrate to using w_direct (trace) directly.
- Input validation: Missing or malformed assign.h5 will lead to errors; ensure the assignment file is present and correctly formatted.
- Output interpretation: Since conditional fluxes are not population-normalized, additional processing is required to derive true rates. Use w_kinavg for normalization and uncertainty estimation.
- Compatibility: The wrapper exists to maintain older scripts; future migrations should target w_direct to avoid deprecation.

Usage examples
- Basic usage:
  - w_kinetics trace -o kintrace.h5 assign.h5

  - w_kinetics trace -o kintrace.h5 --some-direct-option value assign.h5
