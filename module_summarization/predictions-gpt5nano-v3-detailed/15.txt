Ploterr command
===============

Overview
--------
Ploterr provides a command-line interface for plotting error bars (confidence intervals)
associated with expected values over iterations, derived from HDF5 datasets used in WESTPA.
The command supports a generic plotting mode that reads a simple, user-specified
HDF5 dataset and renders a plot of the expected values with their lower and upper
confidence bounds. It can render plots via a graphical backend (matplotlib) when a
display is available, or fall back to a text-based interface when needed.

Key concepts
------------
- CommonPloterrs: Base class that encapsulates shared plotting options and behavior
  (scales, ranges, labels, titles, and interface selection). Handles common argument
  parsing and the plotting operation.
- GenericIntervalSubcommand: Subcommand for plotting arbitrary HDF5 datasets
  that provide iteration and statistical data. Exposes input/output options and
  dataset specification in the form FILENAME/PATH[SLICE].
- Plot data format (required in the dataset ):
  - iter_start
  - iter_stop
  - expected
  - ci_lbound
  - ci_ubound

Relationships
-------------
- GenericIntervalSubcommand inherits from CommonPloterrs.
- The plotting logic resides in CommonPloterrs.do_plot, which is reused by all subcommands.
- GenericIntervalSubcommand adds input/output handling and data loading/validation.

Important parameters and return values
-------------------------------------
CommonPloterrs
- xscale, yscale: Axis scales. Values: linear, log, symlog. Default: linear.
- xrange, yrange: Optional numeric ranges for x and y axes (tuple (min, max)).
- xlabel, ylabel, title: Axis labels and plot title. Defaults vary; xlabel defaults to "Iteration".
- interface: 'matplotlib' or 'text' depending on DISPLAY and --terminal.

CommonPloterrs.do_plot(data, output_filename, title=None, x_range=None, y_range=None, x_label=None, y_label=None)
- Renders the plot using matplotlib (if available) and saves to output_filename.
- Expects data to contain: iter_stop, expected, ci_lbound, ci_ubound.
- Iteration values are computed as iters = data['iter_stop'] - 1.

GenericIntervalSubcommand
- h5file, h5dset: HDF5 file handle and dataset handle opened for reading.
- dset_slice: Numpy-style slice expression selecting the vector to plot.
- output_filename: Path to the output plot file (default: errbars.pdf).
- dsspec: CLI argument of the form FILENAME/PATH[SLICE].

add_args(...)
- Defines:
  - -o/--output: Output file (default: errbars.pdf).
  - dsspec: Dataset specification (FILENAME/PATH[SLICE]).
- dsspec syntax supports an optional slice, e.g., kinavg.h5/rate_evolution[:,0,1].

process_args(args)
- Parses the dataset specification into pathname, slicestr.
- Converts slicestr into a NumPy index expression (sl) or uses a full slice if absent.
- Opens the HDF5 file and resolves the dataset via h5io.resolve_filepath.
- Stores the selected slice as dset_slice.

load_and_validate_data()
- Loads the sliced dataset and validates it is 1D.
- Requires dataset to include: iter_start, iter_stop, expected, ci_lbound, ci_ubound.

Usage and examples
------------------
Overview
- The generic subcommand plots a user-specified dataset that contains an
  expectation and confidence interval bounds over iterations.

CLI usage
- Basic plot (default output): 
  wploterr generic kinavg.h5/rate_evolution[:,0,1]
- Specify output file:
  wploterr generic kinavg.h5/rate_evolution[:,0,1] -o errbars.png
- Customize axes:
  wploterr generic kinavg.h5/rate_evolution[:,0,1] --xrange 0,100 --yrange -0.5,0.5 -o errbars.pdf
- Labels and title:
  wploterr generic kinavg.h5/rate_evolution[:,0,1] --xlabel "Iteration" --ylabel "Rate" --title "Rate Evolution" -o rate_evolution.png
- Terminal (text) interface (no plotting backend):
  wploterr generic kinavg.h5/rate_evolution[:,0,1] --terminal

Dataset specification example
- The path form is FILENAME/PATH[SLICE]. For example:
  kinavg.h5/rate_evolution[:,0,1]
  This selects rate evolution data for state pair (0,1) from the dataset.

What data is required in the HDF5 dataset
----------------------------------------
The dataset must provide, at minimum, the following 1D fields:
- iter_start
- iter_stop
- expected
- ci_lbound
- ci_ubound

Notes and edge cases
--------------------
- The dataset must be 1D. If not, a TypeError is raised.
- If the output filename is not provided or is falsy, plotting is skipped.
- If DISPLAY is available, matplotlib is used by default unless --terminal is specified.
- When using the --terminal option, the interface switches to text rendering.
- The x/y range parsing expects numeric values in "xmin,xmax" and will raise a ValueError on invalid input.
- Slice syntax must be a valid NumPy-style slice expression within brackets, e.g., [:,0,1].

Implementation highlights
------------------------
- Plot rendering leverages matplotlib when available and the DISPLAY environment variable is set.
- The plotting pipeline is designed to be reused by multiple subcommands via CommonPloterrs.
- Dataset loading is integrated with the project’s HDF5 I/O utilities (h5io.resolve_filepath).

Code blocks (for reference)
----------------------------
Example: dataset specification
```text
kinavg.h5/rate_evolution[:,0,1]
```

Example: plotting with custom ranges and labels
```bash
wploterr generic kinavg.h5/rate_evolution[:,0,1] \
  -o rate_evolution.png \
  --xrange 0,100 \
  --yrange -0.5,0.5 \
  --xlabel "Iteration" \
  --ylabel "Rate" \
  --title "Rate Evolution"
```

Example: using the terminal/text interface
```bash
wploterr generic kinavg.h5/rate_evolution[:,0,1] --terminal -o rate_evolution.txt
```

Notes for developers
--------------------
- Maintain consistency with other plotter subcommands by reusing CommonPloterrs for common plotting behavior.
- Ensure dataset schemas follow the required fields to enable robust plotting and validation.
- Consider extending with additional subcommands for dataset-specific conventions while reusing the GenericIntervalSubcommand framework.
