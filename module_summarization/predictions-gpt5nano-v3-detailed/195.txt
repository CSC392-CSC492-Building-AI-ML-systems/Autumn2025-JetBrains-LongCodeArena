# Learn Gaussian Mixture Models

This module provides an extensible framework for learning Gaussian Mixture Models (GMMs) using Expectation-Maximization (EM). At the core is a BaseMixture class that defines the common training workflow and parameter handling. Concrete GMM implementations derive from this base and implement the specifics of parameter updates.

## Overview

- BaseMixture: Abstract foundation for EM-based mixture models. Manages initialization, multiple restarts, convergence checking, and the EM loop orchestration.
- Derived GMMs: Implement the domain-specific parameter updates (e.g., means, covariances, mixture weights) and parameter validation via the abstract methods _check_parameters and _initialize.
- Training loop: For each initialization (n_init), the model iterates E-step (responsibilities) and M-step (parameter updates) up to max_iter or until the change in the objective (likelihood or lower bound) is below tol. If convergence is not reached, a ConvergenceWarning is issued.

## API and Relationships

- BaseMixture
  - Purpose: Provide the EM framework and parameter validation for mixture models.
  - Core methods:
    - __init__(n_components, tol, reg_covar, max_iter, n_init, init_params, random_state, warm_start, verbose, verbose_interval)
    - _check_parameters(X): Abstract; implemented by derived classes to validate X and initial settings.
    - _initialize(X, resp): Abstract; implemented by derived classes to initialize model parameters from X and the initial responsibilities resp.
    - _initialize_parameters(X, random_state): Initialize responsibilities (resp) according to the chosen init_params strategy and call _initialize(X, resp).
    - fit(X, y=None): Fit the model using EM (with multiple restarts if not warming up).
    - fit_predict(X, y=None): Fit the model and return the most likely component label for each sample.
    - (Abstract) The derived class typically exposes attributes such as weights_, means_, covariances_ after a successful fit.
- Derived GaussianMixture
  - Implements _check_parameters and _initialize to perform Gaussian-specific updates.
  - Uses the base class’ EM loop to handle initialization, convergence checks, and multiple restarts.

## Parameters

- n_components (int, >= 1): Number of mixture components.
- tol (float, >= 0.0): Convergence threshold for the EM loop.
- reg_covar (float, >= 0.0): Non-negative regularization added to the diagonal of covariance matrices to ensure they remain positive-definite.
- max_iter (int, >= 0): Maximum number of EM iterations per initialization.
- n_init (int, >= 1): Number of times the algorithm will be run with different initializations. The best result (highest likelihood or lower bound) is kept.
- init_params (str): Initialization strategy. One of:
  - "kmeans": initialize with hard assignments from KMeans.
  - "random": initialize responsibilities randomly.
  - "random_from_data": select random data points as initial component centers.
  - "k-means++": initialize using k-means++ style seeding.
- random_state: Random state or seed to control random number generation.
- warm_start (bool): If True, reuse the solution of the previous fit and incrementally fit (ignore n_init after the first call).
- verbose (int): Controls the verbosity of the fitting process.
- verbose_interval (int): Iteration interval at which to print status when verbose > 0.

Constraints for parameters are enforced to ensure valid values. See the parameter constraint definitions for exact validation rules.

## Initialization Strategies

- kmeans: Runs KMeans with n_components clusters to generate hard assignments; used to form the initial responsibilities.
- random: Starts with random responsibilities that sum to 1 across components for each sample.
- random_from_data: Picks n_components representative samples and initializes with one-hot responsibilities at those samples.
- k-means++: Uses a k-means++ style seeding to place initial component centers and then assigns one-hot responsibilities accordingly.

Trade-offs:
- kmeans and k-means++ often yield faster convergence and better local optima for well-separated clusters.
- random init can help explore different regions of the parameter space when the data are more complex or have overlapping clusters.
- random_from_data provides deterministic, data-driven initialization with a compact set of initial centers.

## Training Workflow

1. Validate and normalize input X (shape: n_samples x n_features; require at least 2 samples for meaningful estimates).
2. Initialize parameters via _initialize_parameters(X, random_state) using the chosen init_params strategy.
3. For each initialization (up to n_init, or a single pass if warm_start is True):
   - E-step: Compute responsibilities (posterior probabilities of component membership for each sample).
   - M-step: Update mixture weights, means, and covariances (with reg_covar regularization).
   - Check convergence by comparing the change in the objective to tol.
   - If converged, store the current model as a candidate.
4. After all restarts, select the best model according to the objective and finalize the fit.
5. Expose fitted parameters (typical attributes such as weights_, means_, covariances_) for downstream use.

## Return Values

- fit(X): Returns self after fitting.
- fit_predict(X): Returns an array of component labels with shape (n_samples,).
- After fitting, typical derived attributes (provided by the concrete implementation) include:
  - weights_: Mixture weights for each component.
  - means_: Component means.
  - covariances_: Component covariance matrices.
  - responsibilities_: Posterior probabilities (E-step results) if available.

## Important Considerations and Edge Cases

- Input validation: X must be a 2D array with at least 2 samples.
- Numerical stability: reg_covar helps guard against singular covariances; extremely small data variances may require larger regularization.
- Convergence: If the EM loop fails to meet tol within max_iter, a ConvergenceWarning is raised, and the best among restarts is kept.
- Warm start: When enabled, subsequent fits reuse the current model state, ignoring further n_init as appropriate.
- Random state: For reproducibility, consistently seed the random_state parameter.
- Parameter consistency: Ensure n_components is appropriate for the dataset size and dimensionality to avoid degenerate solutions.

## Usage Examples

Basic usage with a Gaussian Mixture implementation derived from BaseMixture:

```python
# Assumes a concrete GaussianMixture class exists in your package
from your_package.mixture import GaussianMixture
import numpy as np

# Example data: 2D points from several Gaussians
rng = np.random.RandomState(0)
X = np.vstack([
    rng.randn(100, 2) + np.array([0, 0]),
    rng.randn(100, 2) + np.array([5, 5]),
    rng.randn(100, 2) + np.array([0, 5]),
])

gm = GaussianMixture(
    n_components=3,
    tol=1e-3,
    reg_covar=1e-6,
    max_iter=100,
    n_init=5,
    init_params="kmeans",
    random_state=42,
    warm_start=False,
    verbose=0,
    verbose_interval=10,
)

gm.fit(X)
labels = gm.predict(X)          # shape (n_samples,)
# Access fitted parameters (typical attributes in concrete GMM)
weights = gm.weights_
means = gm.means_
covariances = gm.covariances_
```

Alternative usage with automatic labeling via fit_predict:

```python
labels = gm.fit_predict(X)
```
