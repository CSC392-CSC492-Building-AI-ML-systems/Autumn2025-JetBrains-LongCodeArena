Configuration Options for PyAEDT Configurations
===============================================

Overview
--------
This module defines the configuration options used to control import and export behavior
for PyAEDT configurations. The primary user-facing component is the ConfigurationsOptions
class, which encapsulates a set of boolean flags and a numeric tolerance that determine
which entities are exported or imported when serializing or deserializing project data
to and from JSON.

The module also provides a small private helper, _find_datasets, which traverses nested
data structures to locate dataset references (notably piecewise linear, "pwl", definitions)
within datasets.

Key concepts
- Export flags: control whether specific PyAEDT entities are written to JSON during export.
- Import flags: control whether specific PyAEDT entities are read from JSON during import.
- object_mapping_tolerance: a numerical tolerance used when mapping objects (e.g., boundaries)
  during import/export to establish correspondences robustly.
- Datasets handling: specialized logic to identify and handle datasets referenced by
  free_form_value strings that contain pwl specifications.

Private helper
--------------
_find_datasets(d, out_list)
    Recursively traverses a nested dictionary (or OrderedDict) to collect dataset
    identifiers used in free_form_value entries. It looks for strings containing
    "pwl" and extracts the substring representing the dataset reference.

    Parameters
    ----------
    d : dict or OrderedDict
        Nested data structure containing dataset definitions.
    out_list : list
        Accumulator for discovered dataset references.

    Behavior
    --------
    - If a value is a dict/OrderedDict, the function recurses.
    - If a value is a list, it inspects each element for a free_form_value field containing
      a pwl reference and extracts the corresponding dataset name.
    - If a value is a string containing "pwl", it extracts the dataset reference from the string.

Usage
-----
The ConfigurationsOptions object is typically accessed via a higher-level project or application object
within PyAEDT (for example, through a specific application's configurations property). The common
pattern is to toggle the desired flags to tailor the export/import behavior for a given workflow.

Example: Basic usage
--------------------
- Disable exporting variables and enable importing datasets.

.. code-block:: python

   from pyaedt import Hfss
   hfss = Hfss()
   opts = hfss.configurations.options
   opts.export_variables = False
   opts.import_datasets = True

Example: Adjust object mapping tolerance
----------------------------------------
.. code-block:: python

   opts = hfss.configurations.options
   opts.object_mapping_tolerance = 1e-12

Edge cases and considerations
-----------------------------
- The default values assume a safe, conservative approach to exporting/importing. Change
  only when you understand the implications for your project data.
- object_mapping_tolerance should be chosen based on the numerical stability of the model and
  the precision of your dataset identifiers; too large a value may cause incorrect mappings, while
  too small a value may hinder legitimate mappings.
- The private _find_datasets helper is intended for internal use during dataset extraction and
  is not part of the public API. Changes to its behavior may affect dataset discovery in export/import
  pipelines.
