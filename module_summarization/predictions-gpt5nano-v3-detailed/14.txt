Querying
========

Overview
--------

The Querying module provides a programmatic interface for retrieving entities from an ftrack server. It enables developers to construct server-side queries with:

- Target entity types (e.g. Task, Shot, Asset)
- Projections of fields to fetch
- Flexible filtering criteria (including nested/project fields)
- Sorting and pagination
- Execution against a connected Session, with results returned as entities or dictionaries

Key concepts and relationships
------------------------------

- Session: The authenticated connection to the ftrack server used to execute queries.
- Query: The main object used to describe what to fetch. It aggregates target entity type, selected fields, filters, ordering, and pagination.
- Criteria/Filters: Conditions that restrict which entities are returned. Can be combined with logical operators (and/or/not) and nested as needed.
- Projection: The set of fields to fetch for each result. Projections can include related fields (e.g. project.name) and nested relationships.
- Execution: Running a Query against the server, returning results. By default, results may be returned as ftrack entities or dictionaries depending on options.

Building and running queries
----------------------------

A typical workflow:
1. Create or obtain a Session.
2. Create a Query for a specific entity type.
3. Specify the fields to fetch (projection).
4. Apply filtering criteria.
5. Optionally apply sorting and pagination.
6. Execute to obtain results.

Example: simple query for tasks
--------------------------------

<code-block python>

from ftrack_api import Session
from ftrack_api.query import Query

# Create a session (adjust parameters to your environment)
session = Session(server_url='https://your-ftrack-server',
                  api_key='YOUR_API_KEY', api_user='YOUR_USER')

# Build a query for Task entities
query = Query(session, 'Task')

# Specify fields to fetch
query.select(['id', 'name', 'project.name', 'assignee.username'])

# Apply a simple filter (e.g., tasks with status "In Progress")
query.where(['status.name is_in ["In Progress"]'])

# Execute and obtain results
results = query.execute()

# results is typically a list of Task entities or dictionaries
print(len(results))
</code-block>

Example: multiple criteria, sorting and limiting
-------------------------------------------------

<code-block python>

from ftrack_api import Session
from ftrack_api.query import Query

session = Session('https://your-ftrack-server', 'KEY', 'user')

query = Query(session, 'Task')
query.select(['id', 'name', 'status.name', 'priority', 'project.name'])
query.where(['status.name is_in ["In Progress", "Open"]',
             'project.name is "My Project"'])

# Sort by priority (ascending) then name
query.order_by(['priority', 'name'])

# Paginate: get first 100 results
query.limit(100)
query.offset(0)

results = query.execute()
</code-block>

Example: nested/projection of related fields
---------------------------------------------

<code-block python>

from ftrack_api import Session
from ftrack_api.query import Query

session = Session('https://your-ftrack-server', 'KEY', 'user')

query = Query(session, 'Task')
# Fetch core fields plus related project and assignee information
query.select(['id', 'name', 'project.name', 'assignee.username', 'created_at'])

# Optional deeper filters
query.where(['project.name is "Advertising"',
             'assignee.username is_not_null'])

results = query.execute()

for task in results:
    print(task.id, task.name, task.project.name, task.assignee.username)
</code-block>

Return values and types
-----------------------

- execute(): Executes the query against the server and returns a collection of results.
- Result shape:
  - By default, results may be returned as ftrack entity objects corresponding to the queried type.
  - If a dictionary projection is requested (depending on the API version or options), results may be dictionaries with the projected fields.
- You can typically request specific fields using the select(...) method to control the projection.

Important parameters and behavior
----------------------------------

- Entity type: The first argument to Query(session, 'EntityType') specifies the target entity type.
- Projection (select): Defines which fields are retrieved for each result. Supports nested/related fields (e.g., 'project.name').
- Filters (where): A structured or textual representation of criteria. Supports combining multiple conditions with logical operators.
- Sorting (order_by): A list of field names to sort by. Can specify ascending or descending order if the API supports it.
- Pagination (limit, offset): Controls how many results are returned and from which offset.
- Execution context: Queries run within a Session; authentication and permissions are enforced by the server.
- Error handling: Query-related errors raise appropriate exceptions (e.g., query syntax errors, permission issues, or server-side errors). See ftrack_api.exception for details.

Edge cases and considerations
------------------------------

- Large result sets: Use limit/offset to paginate results and avoid loading massive datasets at once.
- Permissions: If the user lacks permission for certain fields or entities, those results may be omitted or raise an error.
- Missing fields: If a requested projection contains fields that are not present on some entities, the corresponding result entries may have null/None values for those fields.
- Server capabilities: Some advanced query features may depend on server version. Ensure compatibility with your ftrack server.
- Performance: Complex criteria with deep relations can impact performance; prefer targeted projections and indexed fields when available.
- Consistency: If the underlying data changes during iteration, results may reflect a mix of states depending on server behavior and session settings.

API surface (summary)
---------------------

- Query(session, entity_type): Create a new query for a given entity type.
- select(fields): Specify a projection for fields to fetch.
- where(criteria): Apply filtering criteria (structured objects or textual expressions).
- order_by(fields): Define sorting of results.
- limit(n): Limit the number of results returned.
- offset(n): Skip the first n results for pagination.
- execute(): Run the query and return results as a list of entities or dictionaries.

Usage notes and best practices
------------------------------

- Prefer explicit projections to minimize data transfer.
- Use structured criteria (rather than raw textual where clauses) where possible to reduce ambiguity.
- Combine filtering with indexing-friendly fields to improve performance.
- Always paginate large results and handle potential partial results or timeouts gracefully.
- Validate results against your application’s expectations (e.g., ensure required fields are present).

See also
--------

- ftrack_api.session.Session: the entry point to authenticate and interact with the server.
- ftrack_api.exception: for query-related error handling and debugging helpers.

Usage example: end-to-end script
-------------------------------

<code-block python>

from ftrack_api import Session
from ftrack_api.query import Query

# Initialize session
session = Session(
    server_url='https://ftrack.example.com',
    api_key='YOUR_API_KEY',
    api_user='YOUR_USERNAME'
)

# Build and execute a query
query = Query(session, 'Task')
query.select(['id', 'name', 'status.name', 'project.name', 'assignee.username'])
query.where(['status.name is_in ["In Progress", "Open"]'])
query.order_by(['priority', 'name'])
query.limit(50)

results = query.execute()
for r in results:
    print(r.id, r.name, r.project.name, r.assignee.username, r.status.name)
</code-block>
