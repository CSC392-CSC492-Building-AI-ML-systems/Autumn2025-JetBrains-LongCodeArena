The sklearn.feature_extraction.text submodule provides utilities to build numeric feature vectors from text documents. It includes vectorizers that convert raw text into sparse matrices suitable for machine learning, as well as helpers for text normalization and preprocessing.

Overview
- Purpose: Transform text data into numerical representations (e.g., counts, TF-IDF) for machine learning pipelines.
- Core components:
  - CountVectorizer: Converts a collection of text documents to a matrix of token counts.
  - TfidfVectorizer: Converts a collection of raw documents to a matrix of TF-IDF features.
  - TfidfTransformer: Transforms a count matrix to a TF-IDF representation.
  - HashingVectorizer: Applies the hashing trick to convert text to a sparse matrix.
  - ENGLISH_STOP_WORDS: Built-in set of common English stop words.
  - Helper functions: strip_accents_unicode, strip_accents_ascii, strip_tags, _preprocess, _analyze.
- Internal utilities: _VectorizerMixin provides shared tokenization logic used by vectorizers.

Key Functions and Helpers
- _preprocess(doc, accent_function=None, lower=False)
  - Chains optional preprocessing steps on a document.
  - Parameters:
    - doc: str, the input document.
    - accent_function: callable or None, function to handle accented characters.
    - lower: bool, whether to lowercase the text.
  - Returns: preprocessed string.

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  - Orchestrates the text processing pipeline to produce tokens or n-grams.
  - If analyzer is provided, uses it as the primary pipeline; otherwise uses provided preprocessor, tokenizer, and ngrams.
  - Returns: list of tokens/ngrams.

- strip_accents_unicode(s)
  - Converts accented unicode characters to their unaccented equivalents (fallback is slower than ASCII-based method).
  - Returns: string with accents removed.

- strip_accents_ascii(s)
  - Strips accents using ASCII transliteration (faster, language-dependent).
  - Returns: ASCII-only string.

- strip_tags(s)
  - Lightweight HTML/XML tag stripper using a regular expression.
  - Returns: string with tags replaced by spaces.

- _check_stop_list(stop)
  - Resolves stop word lists (e.g., "english" to ENGLISH_STOP_WORDS) or returns user-provided sets.
  - Returns: iterable stop words or None.

Constants
- ENGLISH_STOP_WORDS
  - Built-in set of English stop words used by various vectorizers and preprocessors.

Public API (Classes)
- _VectorizerMixin (internal)
  - Provides common tokenization utilities and a decode method to turn input into a string.
  - Supports input modes such as "filename" and "file".
  - Handles decoding with specified encoding and error handling.

- CountVectorizer
  - Converts text documents to a matrix of token counts.
  - Key parameters (typical, high-level):
    - input, encoding, decode_error
    - analyzer, tokenizer, preprocessor, ngrams
    - stop_words, strip_accents, lowercase
    - max_df, min_df, max_features
  - Returns a sparse matrix of shape (n_samples, n_features).

- TfidfVectorizer
  - Builds a TF-IDF representation from a collection of raw documents.
  - Inherits behavior from CountVectorizer for tokenization and n-grams.
  - Additional behavior:
    - use_idf, norm, smooth_idf, sublinear_tf
  - Returns a TF-IDF weighted sparse matrix.

- TfidfTransformer
  - Transforms a count matrix to a TF-IDF representation.
  - Parameters include use_idf, norm, smooth_idf, sublinear_tf.
  - Returns a TF-IDF weighted dense or sparse matrix (depending on input).

- HashingVectorizer
  - Applies the hashing trick to map tokens to a fixed-size feature space.
  - Parameters include n_features, alternate_sign (default True), non_negative, and other vectorizer-like options.
  - Returns a sparse matrix with a fixed number of features, suitable for online or streaming scenarios.

Usage Patterns
- Simple count vectorization
  - Example:
    from sklearn.feature_extraction.text import CountVectorizer

    docs = ["This is a sentence.", "This is another sentence."]
    cv = CountVectorizer(stop_words='english')
    X = cv.fit_transform(docs)
    print(X.shape)  # (2, n_features)

- TF-IDF vectorization
  - Example:
    from sklearn.feature_extraction.text import TfidfVectorizer

    docs = ["This is a sentence.", "This is another sentence."]
    tfidf = TfidfVectorizer(stop_words='english')
    X = tfidf.fit_transform(docs)
    print(X.shape)

- Hashing vectorization (useful for large-scale or streaming data)
  - Example:
    from sklearn.feature_extraction.text import HashingVectorizer

    docs = ["This is a sentence.", "This is another sentence."]
    hv = HashingVectorizer(n_features=2**16, alternate_sign=False)
    X = hv.fit_transform(docs)
    print(X.shape)

- Custom analyzer chain
  - You can customize the preprocessing pipeline by supplying your own preprocessor, tokenizer, and/or ngrams functions or by using a built-in analyzer.

Important Considerations and Edge Cases
- Stop words handling:
  - Use ENGLISH_STOP_WORDS or provide a custom list; stop words are removed during tokenization unless disabled.
- Accent stripping:
  - strip_accents_unicode vs strip_accents_ascii trade-offs between accuracy and performance; choose based on language requirements.
- Input types:
  - Vectorizers support different input styles (e.g., raw strings, or filenames) and handle decoding with specified encoding and error policies.
- HashingVectorizer:
  - Produces a fixed-dimensional feature space; collisions are possible but acceptable in many text mining tasks.
  - Consider memory and speed trade-offs for large corpora.
- Edge cases:
  - Empty documents or documents that yield zero features after filtering.
  - Handling of Unicode normalization and inconsistent encodings across documents.
  - 32-bit vs 64-bit considerations when dealing with indices; internal utilities may adapt to platform architecture.

Usage Example (Pipeline)
- Demonstrating a typical text processing workflow with a pipeline:
  - Example:
    from sklearn.pipeline import Pipeline
    from sklearn.feature_extraction.text import CountVectorizer, TfidfTransformer
    from sklearn.linear_model import LogisticRegression

    docs = [
        "The quick brown fox jumps over the lazy dog.",
        "Never jump over the lazy dog quickly."
    ]

    pipeline = Pipeline([
        ("vect", CountVectorizer(stop_words="english", lowercase=True)),
        ("tfidf", TfidfTransformer()),
        ("clf", LogisticRegression())
    ])

    y = [0, 1]  # sample labels
    pipeline.fit(docs, y)
    preds = pipeline.predict(["quick dog jumps"])
    print(preds)

Notes
- The module is designed to integrate with scikit-learn’s estimator API, supporting fit/transform patterns and compatibility with pipelines and model selection utilities.
- For advanced usage, combine with custom analyzers, tokenizers, and preprocessor functions to tailor tokenization to specific domains (e.g., code, social media text, multilingual data).
