Polynomial factorization routines in characteristic zero
=======================================================

Overview
--------
This module provides core factorization routines used by a polynomial algebra system to factor polynomials over characteristic zero domains. It includes:
- Low-level trial-division helpers
- Multivariate algebraic-number factorization (aa-factor) via Trager-style approaches
- Zassenhaus-based factorization for polynomials in Z[X], including enhancements for multivariate scenarios (EEZ-like behavior)
- Support utilities such as Mignotte-type bounds used in univariate Z[X] factorization

The implementation is organized as a mixin class (_Factor) intended to be combined with domain/ring specific polynomial representations. It relies on the current polynomial domain (self.domain) and a collection of domain-aware utilities (e.g., coercion, normalization, and factor lists).

Key concepts and components
----------------------------
- _Factor: Mixin class that provides the factorization routines.
- _trial_division(f, factors): Basic post-processing to recover multiplicities of given irreducible factors by repeated division.
- _aa_factor_trager(f): Factor multivariate polynomials over algebraic number fields using algebraic-number techniques (Trager-style refinement via square-free part, cofactors, and shifts).
- _zz_factor(f): Factor polynomials in Z[X], handling both univariate and multivariate cases with primitive normalization, square-free checks, cyclotomic/ZZ-based strategies, and recursive factorization for multivariate components.
- _zz_mignotte_bound(f): Compute a Mignotte-type bound for factoring univariate polynomials in Z[X].
- _zz_zassenhaus(f): Core univariate Z[X] factorization using Zassenhaus decomposition with modular lifting and Hensel lifting; supports choosing primes, lifting factors, and combining to obtain a complete factorization.
- query(...): Configuration switch used to enable/disable advanced strategies (e.g., irreducible-in-factor checks, cyclotomic factorization).
- factor_list / factorization results: Returns typically (content, [(factor, multiplicity), ...]) for Z[X] inputs.

Usage notes
----------- 
- The routines are designed to be used within a larger factorization framework and expect a polynomial object with a domain and properties such as is_univariate, primitive(), sqf_part(), is_ground, LC (leading coefficient), monic(), degree(), and other domain-specific manipulations.
- The behavior is influenced by global query settings:
  - USE_IRREDUCIBLE_IN_FACTOR: Try to stop decomposition early if the remaining cofactor is irreducible.
  - USE_CYCLOTOMIC_FACTOR: Apply cyclotomic factor detection as a preprocessing step.
- Univariate Z[X] factorization: _zz_zassenhaus implements a classical Zassenhaus lattice-based approach with a Mignotte bound and probabilistic (via modular) lifting. It requires the polynomial to be primitive and square-free as part of the main loop.
- Multivariate Z[X] factorization: _zz_factor handles non-square-free and multivariate polynomials by reducing to primitive, factoring the primitive part, and recursively factoring components in a lifted ring.

Important edge cases
--------------------
- Zero polynomial: _zz_factor returns (domain.zero, []).
- Ground polynomials (constants): univariate handling short-circuits to no factors beyond content.
- Non-square-free inputs: factor multiplicities are recovered after initial square-free factorization via trial-division.
- Domain constraints: Some steps assume Z[X] domain and integer-coefficient polynomials; code paths branch based on domain checks (e.g., domain.is_IntegerRing and self.is_univariate).

Documentation examples
----------------------
Univariate Z[X] factorization example (illustrative):

```python
# Example: factorization in Z[X] for a univariate polynomial
# (Provided in the original implementation as a docstring example)
>>> _, x = ring('x', ZZ)
>>> (2*x**4 - 2).factor_list()
(2, [(x - 1, 1), (x + 1, 1), (x**2 + 1, 1)])
```

Multivariate Z[X] factorization example:

```python
>>> _, x, y = ring('x y', ZZ)
>>> (2*x**2 - 2*y**2).factor_list()
(2, [(x - y, 1), (x + y, 1)])
```

Usage guide
-----------
- To perform factorization, instantiate a class that inherits _Factor with the appropriate domain (self.domain) and generator set (self.gens), then invoke the high-level factorization routine (not shown here) which will call:
  - _zz_factor for Z[X] factorization
  - _aa_factor_trager for algebraic-number-extended factorization
  - _trial_division for multiplicity recovery
- Adjust strategy via query(...) switches for performance or problem-specific behavior (e.g., irreducibility checks, cyclotomic factor detection).

This documentation provides a concise map of the factorization routines exposed by _Factor, their roles in the overall algorithm, and the typical inputs/outputs you can expect when factoring polynomials over characteristic zero domains.
