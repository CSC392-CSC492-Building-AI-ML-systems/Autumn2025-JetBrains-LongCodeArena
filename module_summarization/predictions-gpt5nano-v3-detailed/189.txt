Kernel Ridge Regression
=======================

Overview
--------
Kernel ridge regression (KRR) combines ridge regression (L2-regularized least squares)
with the kernel trick. It learns a linear function in the feature space induced by
the chosen kernel, enabling non-linear relationships in the original input space.
For non-linear kernels, KRR corresponds to learning a non-linear function in the
input space. The modeling form is equivalent to that of support vector regression
(SVR) with squared loss and L2 regularization, but KRR admits a closed-form solution
and is typically faster for medium-sized datasets. Unlike SVR, the resulting model
is dense (not sparse) and predictions are straightforward linear combinations of
kernel evaluations.

The estimator supports multi-output regression (y as a 2D array: [n_samples, n_targets]).

Key relationships
- KernelRidge is closely related to SVR, Ridge, and GaussianProcessRegressor:
  - Both KRR and SVR use kernel functions to operate in feature space.
  - KRR uses squared loss with L2 regularization; SVR uses epsilon-insensitive loss.
  - Ridge is linear regression with L2 regularization; KRR generalizes this with kernels.
- The model’s dual representation is given by dual_coef_, with predictions computed as
  y_hat = K(X, X_fit_) @ dual_coef_ (or equivalent with X_precomputed).

API at a glance
- Class: KernelRidge
  - Inherits: MultiOutputMixin, RegressorMixin, BaseEstimator
  - Primary purpose: fit a kernelized ridge regression model and predict on new data.

Parameters
----------
alpha : float or array-like of shape (n_targets,), default=1.0
    Regularization strength. Must be non-negative. If an array is provided, penalties
    are per-target. Corresponds to 1 / (2C) in other linear models.

kernel : str or callable, default="linear"
    Kernel mapping used internally. If a string, it must be one of the supported
    pairwise kernels or "precomputed". If "precomputed", X is treated as a precomputed
    kernel matrix. If a callable, it is invoked on each pair of samples to compute
    the kernel value. When using a callable, kernel_params may provide additional
    arguments.

gamma : float, default=None
    Gamma parameter for the RBF, laplacian, polynomial, exponential chi2, and sigmoid
    kernels. Ignored by kernels that do not use this parameter.

degree : int, default=3
    Degree of the polynomial kernel. Ignored by other kernels.

coef0 : float, default=1
    Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

kernel_params : dict, default=None
    Additional parameters (keyword arguments) for the kernel function passed as a callable
    object.

Attributes
----------
dual_coef_ : ndarray of shape (n_samples,) or (n_samples, n_targets)
    Representation of the weight vector(s) in kernel space.

X_fit_ : {ndarray, sparse matrix} of shape (n_samples, n_features)
    Training data used for fitting. If kernel == "precomputed", this stores the
    training kernel matrix of shape (n_samples, n_samples).

n_features_in_ : int
    Number of features seen during fit.

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features seen during fit (if available).

Notes and edge cases
--------------------
- If kernel == "precomputed", provide the kernel matrix K_train of shape
  (n_samples, n_samples) to fit. Predictions should be made by passing the
  kernel matrix between the test and train samples (K_test of shape
  (n_test, n_train)) to predict.
- The model is dense and scales quadratically with the number of training samples
  in both memory and computation. For very large datasets, consider using a kernel
  approximation or a sparse alternative.
- For ill-conditioned kernel matrices (e.g., very small alpha or near linearly
  dependent samples), a Cholesky-based solver is used. If numerical issues arise,
  increasing alpha or adjusting the kernel parameters often helps.
- The interplay between gamma, degree, coef0 and the chosen kernel is kernel-specific;
  consult the kernel documentation for valid ranges and defaults.

Usage examples
--------------

Basic usage with a linear kernel
- This is equivalent to applying ridge regression in the original feature space.

.. code-block:: python

    from sklearn.kernel_ridge import KernelRidge
    import numpy as np

    rng = np.random.RandomState(0)
    n_samples, n_features = 100, 5
    X = rng.randn(n_samples, n_features)
    y = rng.randn(n_samples)

    krr = KernelRidge(alpha=1.0, kernel="linear")
    krr.fit(X, y)
    y_pred = krr.predict(X)

Single-target regression with RBF kernel
- Non-linear relationships can be captured with an RBF kernel.

.. code-block:: python

    krr = KernelRidge(alpha=0.5, kernel="rbf", gamma=0.1)
    krr.fit(X, y)
    y_pred = krr.predict(X)

Multi-target regression
- Y with multiple targets is supported; alpha can be a scalar or per-target vector.

.. code-block:: python

    Y = rng.randn(n_samples, 3)

    krr = KernelRidge(alpha=[1.0, 0.5, 0.2], kernel="rbf", gamma=0.2)
    krr.fit(X, Y)
    Y_pred = krr.predict(X)

Precomputed kernel matrix
- When using a precomputed kernel, the input to fit is the kernel matrix K_train.

.. code-block:: python

    # Compute a kernel matrix for training data
    from sklearn.metrics.pairwise import pairwise_kernels
    K_train = pairwise_kernels(X, X, metric="rbf", gamma=0.1)

    krr = KernelRidge(alpha=1.0, kernel="precomputed")
    krr.fit(K_train, y)

    # For predictions, compute K_test = K(X_test, X_train)
    X_test = rng.randn(20, n_features)
    K_test = pairwise_kernels(X_test, X, metric="rbf", gamma=0.1)
    y_pred = krr.predict(K_test)

Kernel parameters
-----------------
- The kernel for string-valued names is selected from PAIRWISE_KERNEL_FUNCTIONS; other
  kernels include "precomputed".
- Extra kernel parameters can be supplied via kernel_params when using a callable kernel.

Notes on model interpretation
- dual_coef_ contains the weights in kernel space; predictions are made via the
  kernel evaluations between new inputs and the training data.
- For multi-output targets, dual_coef_ is 2D, with shape (n_samples, n_targets).
