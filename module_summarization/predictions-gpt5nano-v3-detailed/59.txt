# Regression: Quantile Regression (regression.rst)

Introduction
------------
This module implements Quantile Regression using Iteratively Reweighted Least Squares (IRLS). It enables estimating conditional quantiles of a response variable given one or more predictors and provides robust options for standard error estimation via kernel density estimation.

Overview
--------
- Core class: QuantReg
  - Inherits from the Statsmodels RegressionModel framework.
  - Estimates coefficients for a specified quantile q (0 < q < 1) using IRLS.
  - Supports two variance-covariance estimation modes:
    - robust: heteroskedasticity-robust standard errors (default)
    - iid: iid errors (as in some econometrics workflows)
  - Asymptotic covariance is computed using kernel density estimation with selectable kernels and bandwidths.
- Key methods:
  - whiten(data): a placeholder whitening method (no transformation applied in this implementation)
  - fit(q=0.5, vcov='robust', kernel='epa', bandwidth='hsheather', max_iter=1000, p_tol=1e-6, **kwargs)
- Supporting concepts:
  - Kernel options for density estimation: Epanechnikov, Gaussian, and others
  - Bandwidth selection methods: Hall-Sheather, Bofinger, Chamberlain
  - Quantile specification q: 0 <= q <= 1 (0.5 corresponds to the least absolute deviation/LAD estimator)

Class and Relationships
-----------------------
- QuantReg(RegressionModel)
  - The QuantReg model uses the regression infrastructure provided by Statsmodels.
  - Endogenous variable (endog) and exogenous design matrix (exog) are supplied at construction time.
  - The fit method returns a results object (RegressionResultsWrapper) that exposes estimated parameters and diagnostics.

- RegressionModel / RegressionResults
  - Endows QuantReg with standard regression attributes and interfaces (e.g., endog, exog, nobs, rank, df_model, df_resid).
  - The fit method ultimately produces a RegressionResultsWrapper containing:
    - params: estimated coefficients
    - bse: standard errors
    - tvalues, pvalues: hypothesis test statistics for coefficients
    - summary: human-readable summary of results

Important Parameters
--------------------
- endog: array-like
  - The endogenous/response variable (dependent variable).
- exog: array-like
  - The exogenous/predictor variables (design matrix). Should include a constant if desired.
- q: float (default 0.5)
  - Target quantile to estimate. Must satisfy 0 <= q <= 1.
- vcov: {'robust', 'iid'} (default 'robust')
  - Method to compute the variance-covariance matrix of the parameter estimates.
- kernel: {'biw', 'cos', 'epa', 'gau', 'par'} (default 'epa')
  - Kernel used in the kernel density estimation for the asymptotic covariance matrix.
- bandwidth: {'hsheather', 'bofinger', 'chamberlain'} (default 'hsheather')
  - Bandwidth selection method for the kernel density estimation.
- max_iter: int (default 1000)
  - Maximum number of IRLS iterations.
- p_tol: float (default 1e-6)
  - Convergence tolerance for the IRLS loop.
- kwargs: additional arguments passed to base classes or for future extensibility.

Return Values
-------------
- fit(...) returns a RegressionResultsWrapper object containing:
  - params: estimated coefficients for the specified quantile q
  - bse: standard errors (based on the chosen vcov method)
  - tvalues, pvalues: t-statistics and p-values for coefficient significance
  - Convergence/Warn conditions (if applicable)
- The wrapper also provides diagnostic attributes like nobs, rank, df_model, df_resid, and methods for model summary and predictions.

Usage Notes and Edge Cases
--------------------------
- Quantile range:
  - q must be within [0, 1]. Values outside this range raise an exception.
- Kernel and bandwidth:
  - The kernel must be one of the supported names. An invalid name raises an exception.
  - Bandwidth must be one of the specified options; otherwise, an exception is raised.
- Convergence:
  - The IRLS procedure has a convergence tolerance (p_tol) and a maximum iteration count (max_iter).
  - If convergence is not reached within max_iter, a ConvergenceWarning or IterationLimitWarning may be issued depending on the situation.
- Rank and model specification:
  - The design matrix rank is computed to determine df_model and df_resid. Singular or near-singular designs may affect convergence and results.
- References:
  - The method follows quantile regression theory and asymptotic covariance estimation as outlined in Greene (2008), Koenker (2005), LeSage (1999), and related econometrics literature.
- Practical considerations:
  - For LAD (q = 0.5), the estimator reduces to the Least Absolute Deviations estimator.
  - Kernel-based covariance estimation introduces sensitivity to bandwidth choices; consider experimenting with bandwidth and kernel selections for robust inference.

Usage Examples
--------------

Example 1: Basic quantile regression (median)
```python
import numpy as np
import pandas as pd
from statsmodels.regression.quantreg import QuantReg

# Example data
np.random.seed(0)
n = 100
X = np.column_stack((np.ones(n), np.random.randn(n)))
beta_true = np.array([1.0, 0.5])
y = X @ beta_true + np.random.randn(n)

# Fit model for the median (q = 0.5)
model = QuantReg(y, X)
res = model.fit(q=0.5)

print(res.summary())
```

Example 2: Quantile regression with Gaussian kernel and Hall-Sheather bandwidth
```python
model = QuantReg(y, X)
res = model.fit(q=0.9, vcov='robust', kernel='gau', bandwidth='hsheather')
print(res.summary())
```

Example 3: Using a DataFrame for endog/exog
```python
import pandas as pd
np.random.seed(1)

df = pd.DataFrame({
    'y': np.random.randn(120),
    'const': 1.0,
    'x1': np.random.randn(120),
    'x2': np.random.randn(120)
})

endog = df['y'].values
exog = df[['const', 'x1', 'x2']].values

model = QuantReg(endog, exog)
res = model.fit(q=0.25, bandwidth='bofinger', kernel='epa', vcov='iid')
print(res.summary())
```
