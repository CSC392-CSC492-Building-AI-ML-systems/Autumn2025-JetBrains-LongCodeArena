# Basics: Indexing Utilities for Structured Arrays

Overview
- This module provides utilities for manipulating structured (record) arrays in NumPy, including support for nested field structures. It enables tasks such as inspecting field names, flattening nested dtypes, and copying data between arrays with matching fields.

Key concepts
- Structured arrays and recarrays: arrays with named fields, potentially nested or subarray fields.
- Nested fields: fields that themselves have a dtype with further named fields.
- Flattening vs. preserving structure:
  - get_names and get_names_flat return field names with different levels of nesting preserved.
  - flatten_descr converts a nested dtype into a flat sequence of field-name and dtype pairs.
- Internal helpers (not part of the public API):
  - _get_fieldspec(dtype): produce a list of (name, dtype) pairs describing the dtype’s fields; used to reconstruct dtypes (offsets are not preserved).
  - _zip_dtype(seqarrays, flatten=False): internal utility to derive a combined dtype from a sequence of arrays (with optional flattening).

Public API (core functions)

1) recursive_fill_fields(input, output)
- Purpose: Fill fields from the input array into the output array, supporting nested (structured) fields.
- Parameters:
  - input: ndarray (source data)
  - output: ndarray (destination; should be at least as large as input)
- Returns: output with matching fields filled from input
- Behavior notes:
  - For each field in output.dtype.names, if input contains that field, its value is copied.
  - If a field is itself a structured dtype, the function recurses into that sub-structure.
  - If the input does not have a given field, it is skipped.
- Example:
```python
from numpy.lib import recfunctions as rfn
a = np.array([(1, 10.), (2, 20.)], dtype=[('A', np.int64), ('B', np.float64)])
b = np.zeros((3,), dtype=a.dtype)
rfn.recursive_fill_fields(a, b)
# b -> array([(1, 10.), (2, 20.), (0,  0.)], dtype=[('A', '<i8'), ('B', '<f8')])
```

2) _get_fieldspec(dtype)  (internal)
- Purpose: Produce a list of (name, dtype) pairs describing the fields of a dtype. Handles subarrays.
- Parameters:
  - dtype: a NumPy dtype (potentially with nested fields)
- Returns: list of (field_name, field_dtype) where field_dtype may itself describe a nested structure
- Notes:
  - If dtype has no named fields, returns [('', dtype)]
  - Intended for internal use to reconstruct dtypes and to describe field structures without offsets
- Documentation excerpt:
```python
>>> dt = np.dtype([(('a', 'A'), np.int64), ('b', np.double, 3)])
>>> dt.descr
[(('a', 'A'), '<i8'), ('b', '<f8', (3,))]
>>> _get_fieldspec(dt)
[(('a', 'A'), dtype('int64')), ('b', dtype(('<f8', (3,))))]
```

3) get_names(adtype)
- Purpose: Return the field names of a dtype as a nested tuple, preserving structure for nested fields.
- Parameters:
  - adtype: dtype with fields (may be nested)
- Returns: tuple of field names, with nested structures preserved (e.g., ('a', ('b', ('ba', 'bb'))))
- Examples:
```python
>>> from numpy.lib import recfunctions as rfn
>>> rfn.get_names(np.empty((1,), dtype=[('A', int)]).dtype)
('A',)
>>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
>>> rfn.get_names(adtype)
('a', ('b', ('ba', 'bb')))
```

4) get_names_flat(adtype)
- Purpose: Return the flattened field names of a dtype, flattening any nested fields.
- Parameters:
  - adtype: dtype with fields (may be nested)
- Returns: tuple of flattened field names (e.g., ('a', 'b', 'ba', 'bb'))
- Examples:
```python
>>> from numpy.lib import recfunctions as rfn
>>> rfn.get_names_flat(np.empty((1,), dtype=[('A', int)]).dtype)
('A',)
>>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
>>> rfn.get_names_flat(adtype)
('a', 'b', 'ba', 'bb')
```

5) flatten_descr(ndtype)
- Purpose: Flatten a structured dtype description into a simple, one-level sequence of (name, dtype) pairs.
- Parameters:
  - ndtype: a dtype that may contain nested fields
- Returns: a tuple of (field_name, field_dtype) pairs, with no nested structure
- Example:
```python
>>> from numpy.lib import recfunctions as rfn
>>> ndtype = np.dtype([('a', '<i4'), ('b', [('ba', '<f8'), ('bb', '<i4')])])
>>> rfn.flatten_descr(ndtype)
(('a', dtype('int32')), ('ba', dtype('float64')), ('bb', dtype('int32')))
```

Notes and edge cases
- Nested structures: get_names and get_names_flat preserve or flatten nesting, respectively; flatten_descr fully flattens nested descriptors.
- Subarrays: The helper functions handle subarray dtypes (e.g., fields with shapes) when constructing field specifications.
- Offsets: _get_fieldspec deliberately discards field offsets; it describes the field layout without positional information.
- Data integrity: recursive_fill_fields requires the output to be at least as large as the input; fields present in input but absent in output are ignored.

Usage scenarios
- Inspecting field structure: use get_names or get_names_flat to understand the layout of a complex dtype.
- Reconstructing dtypes: use _get_fieldspec in combination with a dtype constructor to rebuild dtypes from field specifications.
- Copying data between arrays with nested fields: use recursive_fill_fields to propagate data from one structured array to another with matching fields.

See also
- Other higher-level utilities in this module (not shown here) provide operations like appending, dropping, or joining fields across structured arrays. The core functions documented above form the building blocks for those operations.
