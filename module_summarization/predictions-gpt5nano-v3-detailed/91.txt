RandAugment
===========

Overview
--------

RandAugment is an augmentation strategy that applies a randomly selected subset of augmentations to each sample in a batch. For each sample, n augmentations are chosen from a predefined suite and applied with magnitudes controlled by m. The approach is motivated by the RandAugment paper (arXiv:1909.13719) and is implemented within NVIDIA's DALI Auto-Augment framework.

This module provides the high-level entry point rand_augment for applying RandAugment to a batch of samples (images or videos) and includes internal helpers used to manage augmentation selection and application.

Public API
----------

- rand_augment(data, n, m, num_magnitude_bins=31, shape=None, fill_value=128, interp_type=None, max_translate_abs=None, max_translate_rel=None, seed=None, monotonic_mag=True, excluded=None) -> DataNode

  Applies RandAugment to the provided batch. Returns a transformed batch as a DataNode.

- apply_rand_augment(augmentations, data, n, m, num_magnitude_bins, seed=None, **aug_kwargs) -> DataNode

  Internal helper that applies a specified list of augmentations to the batch. This is primarily used by rand_augment and is not typically called directly by end users.

Notes
-----

- This module operates on DataNode batches containing images (layout HWC) or videos (layout FHWC) with dtype uint8.
- The augmentation magnitudes are defined over a configurable number of magnitude bins (default 31).
- You can specify a shape to adapt translation magnitudes to the image/video dimensions; otherwise, max translations are defined in pixels for translations.
- fill_value controls padding for warp-based operations. If None, border-padding is clamped to the border value.
- monotonic_mag controls how magnitude scales with the magnitude bin index:
  - True: magnitudes increase monotonically with bin index.
  - False: some color-related operations may have non-monotonic or alternative strength scaling.
- Excluded augmentations allow you to prune the default augmentation suite by name.

Key parameters (rand_augment)

- data: DataNode
  Batch of samples to process. Supported: images (HWC) and videos (FHWC); dtype uint8.

- n: int
  Number of randomly sampled operations to apply to each sample.

- m: int
  Magnitude (strength) of each operation, in the range [0, num_magnitude_bins - 1].

- num_magnitude_bins: int
  Number of bins into which magnitude ranges are divided (default: 31).

- shape: DataNode or tuple (height, width), optional
  Shape to scale magnitude for translation operations. If provided, magnitudes depend on shape via max_translate_rel; otherwise magnitudes depend on max_translate_abs.

- fill_value: int, optional
  Padding value for warp-based operations (translation, rotation, shear). If None, border padding uses clamped border values.

- interp_type: types.DALIInterpType, optional
  Interpolation method for warp-based transformations. Typical values: INTERP_LINEAR (default), INTERP_NN.

- max_translate_abs: int or (int, int), optional
  Maximal translation in pixels when shape is not provided. A tuple limits height and width respectively.

- max_translate_rel: float or (float, float), optional
  Maximal translation as a fraction of the image shape when shape is provided. A tuple limits height and width respectively.

- seed: int, optional
  Seed for random sampling of augmentations and magnitude negation.

- monotonic_mag: bool, optional
  If True (default), magnitude bins increase strength with the bin index. If False, some color operations use alternative scaling.

- excluded: list of str, optional
  Names of augmentations to exclude from the default RandAugment suite.

Return value

- DataNode
  A batch of transformed samples.

Usage examples
--------------

Basic usage with default settings

```python
from nvidia.dali import fn
from nvidia.dali.data_node import DataNode
from nvidia.dali.auto_aug.rand_augment import rand_augment

# Assume `images` is a DataNode with a batch of HWC uint8 images
# Example with default RandAugment settings
augmented = rand_augment(
    data=images,
    n=2,
    m=9
)
```

Excluding specific augmentations

```python
excluded = ["rotate", "shear"]
augmented = rand_augment(
    data=images,
    n=2,
    m=9,
    excluded=excluded
)
```

Using shape to scale translations

```python
shape = (224, 224)
augmented = rand_augment(
    data=images,
    n=3,
    m=15,
    shape=shape
)
```

Internal helper usage (advanced)

```python
# Build a specific set of augmentations (depends on internal API)
# augmentations, data, n, m are defined per your pipeline
from nvidia.dali.auto_aug.rand_augment import apply_rand_augment

aug_list = [...]  # list of _Augmentation instances
augmented = apply_rand_augment(aug_list, images, n=2, m=8, num_magnitude_bins=31, seed=123)
```

Important considerations and edge cases
---------------------------------------

- The excluded parameter must refer to valid augmentation names in the selected augmentation suite. If an invalid name is provided, an exception is raised listing available augmentations.
- The augmentation names are deduplicated to ensure the suite contains unique augmentations; bundles with conflicting names may raise an assertion error.
- When shape is provided, translation magnitudes scale with the shape; otherwise magnitudes rely on max_translate_abs. Choose according to whether you provide per-sample shapes or a fixed translation scale.
- The data must be uint8 images or videos; other dtypes or layouts are not supported.
- If fill_value is None, padding is performed with border clamping behavior.
