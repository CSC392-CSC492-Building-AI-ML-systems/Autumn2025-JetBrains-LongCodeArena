Camera Geometry for Cherenkov Cameras
=====================================


Introduction
------------
This module provides utilities to represent and work with the geometric layout of Cherenkov camera subarrays. At the core is SubarrayDescription, which aggregates telescope descriptions and their ground positions for a subarray, enabling consistent handling of camera geometry, footprint calculations, and ID-to-index mappings used when processing images or data cubes from arrays of telescopes.

Overview of core concepts
-------------------------
- SubarrayDescription: Collects TelescopeDescription objects for all telescopes in a subarray and their positions on the ground. It exposes convenient accessors for ground-based geometry, such as the per-telescope coordinates (tel_coords), the set of telescope IDs (tel_ids), and mappings between telescope IDs and a compact indexing scheme (tel_indices, tel_index_array, tel_ids_to_indices).
- Telescopes and positions: Each telescope in a subarray has a unique tel_id and a ground position. Positions are stored as a dictionary of 3D coordinates and are converted to a GroundFrame-based SkyCoord when requested.
- Helper utilities: Internal helpers (_group_consecutives, _range_extraction) format lists of telescope IDs into human-readable ranges, used by info() to summarize subarray composition.

Public API
----------

SubarrayDescription
-------------------
Fields and initialization
- __init__(name, tel_positions=None, tel_descriptions=None, reference_location=None)
  - name: str
    Name of the subarray.
  - tel_positions: Dict[int, np.ndarray]
    Mapping from tel_id to a 3-element position array on the ground. Values are converted internally to GroundFrame coordinates.
  - tel_descriptions: Dict[int, TelescopeDescription]
    Mapping from tel_id to the corresponding TelescopeDescription.
  - reference_location: EarthLocation
    The Earth location of the array reference position (0, 0, 0) of the coordinate system used for tel_positions.

- CURRENT_TAB_VERSION
  - str
  - Current tabular data version used for compatibility.

- COMPATIBLE_VERSIONS
  - set of str
  - Versions compatible with this implementation.

- tel (property)
  - Returns: Dict[int, TelescopeDescription]
  - A direct mapping from telescope IDs to their descriptions.

- n_tels (property)
  - Returns: int
  - Number of telescopes in the subarray.

- tel_coords (property, lazy)
  - Returns: astropy SkyCoord
  - Telescope positions as a single SkyCoord object in the GroundFrame (x, y, z in meters).

- tel_ids (property, lazy)
  - Returns: np.ndarray
  - Array of telescope IDs present in the subarray.

- tel_indices (property, lazy)
  - Returns: dict[int, int]
  - Mapping from tel_id to a compact index (0-based) used for fixed-length array representations.

- tel_index_array (property, lazy)
  - Returns: np.ndarray
  - Expanded array mapping tel_id to its index. If tel_ids are non-contiguous, missing IDs are mapped to -1.

- Methods
  - __str__(): str
    String representation (subarray name).
  - __repr__(): str
    Debug representation with class name, name, and number of telescopes.
  - info(printer=print)
    Print a summary of the subarray:
    - Subarray name
    - Number of telescopes
    - Footprint (requires self.footprint)
    - Per-type telescope counts and ID ranges (using internal helpers to format ranges)

  - tel_ids_to_indices(tel_ids)
    - Parameters: tel_ids (int or Iterable[int])
    - Returns: np.ndarray
    - Map given telescope IDs to their fixed-length indices (0-based). Non-existent IDs will raise an error via standard NumPy indexing behavior when used.

  - tel_ids_to_mask(tel_ids)
    - Parameters: tel_ids (int or Iterable[int])
    - Returns: np.ndarray (bool)
    - Convert a list of telescope IDs to a boolean mask of length n_tels, where True indicates the presence of the telescopes in the provided ID list.

  Note: A ValueError is raised by __init__ if tel_positions and tel_descriptions have mismatched keys (i.e., a telescope ID appears in one but not the other).

UnknownTelescopeID
------------------
- Exception: Raised when an unknown telescope ID is encountered (definition in the module for error handling when IDs are not recognized).

Helper functions (internal)
---------------------------
- _group_consecutives(sequence)
  - Purpose: Convert a sorted list of integers into a generator of human-friendly range strings (e.g., 1,2,3,5,6 -> "1-3", "5-6").
  - Example:
    - Input: [1, 2, 3, 5, 7, 8]
    - Output: "1-3", "5", "7-8"

- _range_extraction(sequence)
  - Purpose: Create a comma-separated string of consecutive ranges using _group_consecutives.
  - Example:
    - Input: [1, 2, 3, 5, 7, 8]
    - Output: "1-3,5,7-8"

Usage scenarios
---------------

Creating a SubarrayDescription
- Purpose: Represent a subarray's geometry by pairing telescope positions with their descriptions.

Code example (illustrative; adapt imports to your package layout):
- Start by providing tel_positions and tel_descriptions with matching telescope IDs.
- Then instantiate SubarrayDescription and access its geometry-related properties.

Code snippet:
- Note: This example uses placeholder TelescopeDescription objects; replace with real ones from your codebase.

from ctapipe.instrument import SubarrayDescription
from ctapipe.instrument.telescope import TelescopeDescription

# Example telescope positions (in meters)
tel_positions = {
    1: np.array([0.0, 0.0, 0.0]) * u.m,
    2: np.array([5.0, 0.0, 0.0]) * u.m,
    3: np.array([0.0, 5.0, 0.0]) * u.m,
}

# Example telescope descriptions (replace with real TelescopeDescription instances)
tel_descriptions = {
    1: TelescopeDescription(...),
    2: TelescopeDescription(...),
    3: TelescopeDescription(...),
}

sub = SubarrayDescription(
    name="TestSubarray",
    tel_positions=tel_positions,
    tel_descriptions=tel_descriptions,
)

# Access geometry and mappings
coords = sub.tel_coords        # Ground-frame 3D coordinates
ids = sub.tel_ids               # Array of telescope IDs
indices_map = sub.tel_indices     # tel_id -> index
index_array = sub.tel_index_array # mapping tel_id -> index (or -1 for gaps)

# ID to indices
indices = sub.tel_ids_to_indices([1, 3])

# Build a mask for a subset of telescopes
mask = sub.tel_ids_to_mask([1, 3])

# Print a summary table
sub.info()

Querying version and internal helpers
- SubarrayDescription.CURRENT_TAB_VERSION: string indicating the current tabular data version used in this representation.
- SubarrayDescription.COMPATIBLE_VERSIONS: set of strings indicating compatible versions.

Important considerations and edge cases
---------------------------------------
- Key consistency: tel_positions and tel_descriptions must contain identical sets of telescope IDs. A mismatch raises ValueError during initialization.
- Contiguity and indexing: tel_index_array maps tel_id to a flat index starting at 0. If telescope IDs are not contiguous, missing IDs are filled with -1, which is important to consider when using the mapping for fixed-length arrays.
- Access patterns: tel_coords is lazy-loaded; the underlying ground coordinates are constructed only on first access.
- Oversized or missing IDs: tel_ids_to_indices expects valid telescope IDs; passing unknown IDs will result in index lookup behavior that should be handled by the caller (potential KeyError orOut-of-range indexing depending on the input).

Edge cases
- If a subarray has no telescopes, tel_coords and related properties will reflect an empty set; ensure downstream code handles this scenario gracefully.
- The footprint attribute referenced in info() is assumed to be defined elsewhere in the class or mixins; ensure your subclass provides this property if you rely on info() for display.

Usage considerations for images and data cubes
- The tel_indices and tel_index_array mappings enable efficient assembly of per-telescope data (e.g., camera images or reconstructed data cubes) into a single array while preserving the correspondence to telescopes by ID.
- tel_coords provides a convenient ground-plane representation of telescope positions, which is useful for footprint calculations, coordinate transforms between CameraFrame and GroundFrame, or visualization of array geometry overlaid on sky maps.

Versioning and compatibility
----------------------------
- The module includes versioning constants to track compatibility:
  - SubarrayDescription.CURRENT_TAB_VERSION
  - SubarrayDescription.COMPATIBLE_VERSIONS

Example visualization (conceptual)
- Use tel_coords to plot telescope footprints on a 3D ground plane.
- Use tel_ids and tel_indices to align per-telescope data with a 2D camera array or a 3D reconstruction pipeline.
