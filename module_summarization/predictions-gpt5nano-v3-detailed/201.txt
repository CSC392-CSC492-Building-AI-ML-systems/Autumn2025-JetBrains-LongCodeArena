Partial Dependence and Individual Conditional Expectation Plots
===============================================================

Overview
--------
This module provides tools to compute and visualize Partial Dependence Plots (PDP) and Individual Conditional Expectation (ICE) plots. PDPs reveal the average effect of one or more features on the model prediction, while ICE plots show the effect for individual instances, enabling examination of heterogeneity in feature effects.

Key concepts
------------
- Partial Dependence Plot (PDP): Shows the average predicted value as a function of one or more features, marginalizing over the distribution of the other features.
- Individual Conditional Expectation (ICE): Extends PDP by displaying the effect for individual data instances, highlighting heterogeneous responses across samples.
- Feature identification: Features can be provided as names or indices. Internal helpers ensure consistent interpretation and validation of feature identifiers.

API overview
------------
This section outlines the core helpers and plotting interface commonly used to generate PDP/ICE plots.

Utility helpers
- _check_feature_names(X, feature_names=None)
  - Purpose: Validate and obtain a list of feature names to use in plotting.
  - Behavior:
    - If feature_names is None:
      - If X has pandas-like columns, use X.columns.tolist() as feature names.
      - Otherwise, generate generic names like ["x0", "x1", ...] based on X.shape[1].
    - If feature_names has a tolist method, convert it to a Python list.
    - Ensure feature names are unique; raise ValueError if duplicates are found.
  - Returns: List[str] feature_names

- _get_feature_index(fx, feature_names=None)
  - Purpose: Resolve a feature specification (name or index) to a numeric index.
  - Behaviour:
    - If fx is a string, look up its position in feature_names.
      - If feature_names is None or the name is not found, raise a ValueError.
    - If fx is already an int, return it.
  - Returns: int (feature index)

High-level plotting interfaces
- PartialDependenceDisplay (class)
  - Purpose: Provide a convenient interface to compute and render PDP/ICE plots for one or more features.
  - Key options:
    - estimator: any fitted estimator supporting predict (or equivalent) required by the plotting routine.
    - X: training data on which PDP/ICE is evaluated.
    - features: list of feature names or indices to plot.
    - grid_resolution: number of grid points per feature to evaluate partial dependence (for continuous features).
    - kind: "average" (PDP) or "individual" (ICE). May also support multiple ICE-related modes depending on the version.
  - Methods (typical):
    - from_estimator(estimator, X, features, grid_resolution=..., kind="average", ...): construct a display object from a fitted estimator.
    - plot()/plot_individual(): render the plots. Returns a figure or axes handle depending on the API version.

- plot_partial_dependence (function) or equivalent: stand-alone helper that computes PDP/ICE and renders the plots without creating a dedicated display object (usage mirrors PartialDependenceDisplay in older APIs). Accepts similar parameters (estimator, X, features, grid_resolution, kind, etc.).

Important parameters and return values
--------------------------------------
- estimator: Fitted model that supports prediction. Used to evaluate model outputs on modified feature grids.
- X: Training data (array-like or DataFrame) used to define the distribution of the other features when marginalizing.
- features: List of feature identifiers (names or indices) to plot.
- feature_names (internal): Optional list of feature names used to resolve string identifiers. Populated by _check_feature_names when not explicitly provided.
- grid_resolution: Integer specifying the number of points along each feature dimension used to construct the prediction grid for continuous features.
- kind: Determines plot type.
  - "average" for PDP (mean prediction across samples).
  - "individual" for ICE (predictions for individual samples or rows).
- Returns: A display object (e.g., PartialDependenceDisplay) or a figure/axes pair, depending on the API, containing computed PDP/ICE values and the corresponding axes data for plotting.

Edge cases and considerations
------------------------------
- Feature naming:
  - If the input X is a pandas DataFrame, feature names default to the column names when feature_names is not provided.
  - For NumPy arrays without column names, generic names ["x0", "x1", ...] are used.
  - Duplicate feature names raise a ValueError.
- Feature lookup:
  - Providing a feature name that is not present in feature_names results in a clear ValueError.
- Data types:
  - Continuous features are interpolated on a grid; categorical features may require different handling or encoding.
- Sample weighting and model types:
  - PDP/ICE computations typically rely on the model’s predict function. Ensure the estimator supports the required interface for your plotting API.
- Computational cost:
  - PDP/ICE can be expensive for large datasets or many features; prefer smaller feature subsets or lower grid resolutions for quick iterates.

Usage examples
--------------
Example 1: PDP for a single feature using the high-level display interface
```python
from sklearn.ensemble import RandomForestRegressor
from sklearn.inspection import PartialDependenceDisplay

# Assume X, y are defined and X is a 2D array or DataFrame
estimator = RandomForestRegressor(n_estimators=100, random_state=0).fit(X, y)

# PDP: average effect of a single feature named 'feature_1'
disp = PartialDependenceDisplay.from_estimator(
    estimator, X, features=['feature_1'], grid_resolution=20, kind='average'
)
disp.plot()
```

Example 2: ICE plots for multiple features
```python
from sklearn.ensemble import RandomForestRegressor
from sklearn.inspection import PartialDependenceDisplay

estimator = RandomForestRegressor(n_estimators=100, random_state=0).fit(X, y)

# ICE: individual conditional effects for two features
disp = PartialDependenceDisplay.from_estimator(
    estimator, X, features=['feature_1', 'feature_2'], grid_resolution=25, kind='individual'
)
disp.plot()
```

Example 3: Using utility helpers directly (internal use)
```python
import numpy as np
import pandas as pd

X = pd.DataFrame(np.random.randn(100, 3), columns=['a', 'b', 'c'])
# Validate feature names
from sklearn.inspection.partial_dependence import _check_feature_names, _get_feature_index

names = _check_feature_names(X, feature_names=None)
idx = _get_feature_index('b', feature_names=names)  # returns 1
```

Notes
-----
- The helpers _check_feature_names and _get_feature_index are intended for internal use to support robust feature handling in PDP/ICE plotting utilities.
- For up-to-date API details, refer to the specific version of your installed library, as parameter names and available options may evolve across releases.
