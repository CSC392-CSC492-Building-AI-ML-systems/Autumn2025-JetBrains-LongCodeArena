# w_kinavg.rst

Introduction
- This module provides the w_kinavg command, a backwards-compatible wrapper around kinetic averaging and confidence-interval calculation for weighted ensemble data. It computes average rates and associated uncertainties from path-tracing data.

Overview
- The w_kinavg command is implemented as a shim that delegates to the direct kinetics analysis tooling, preserving backwards compatibility with older command names.
- The primary functionality is to average state-to-state rates (and related flux quantities in trace mode) over a specified window of iterations, optionally with evolving window widths.
- This wrapper deprecates in favor of a more modern interface exposed by w_direct.

Module Structure and Key Classes
- WKinAvg (inherits from DKinAvg)
  - subcommand: 'trace'
  - help_text: Description of the trace-based averaging and confidence-intervals
  - default_kinetics_file: 'kintrace.h5'
  - default_output_file: 'kinavg.h5'
  - Relationship: Extends DKinAvg to fix the subcommand and default file names for trace-based analysis.
- WDirect (inherits from WESTMasterCommand and WESTParallelTool)
  - prog: 'w_kinavg'
  - subcommands: [WKinAvg]
  - subparsers_title: 'direct kinetics analysis schemes'
  - description: Detailed information about file requirements, output datasets, and data structures
  - Relationship: Provides the master-parallel command framework; delegates actual processing to WKinAvg (via DKinAvg) under the trace subcommand.

Important parameters and return values
- Default files
  - default_kinetics_file: kintrace.h5
  - default_output_file: kinavg.h5
- Subcommands
  - trace: Performs averages and confidence intervals for path-tracing kinetics analysis.
- Output datasets (in kinavg.h5)
  - /avg_rates [state, state]
    - Structured dataset of state-to-state rates over the selected iterations.
  - In trace mode, additional datasets:
    - /avg_total_fluxes [state]
    - /avg_conditional_fluxes [state, state]
- Evolution mode (optional)
  - /rate_evolution [window][state][state]
  - If --evolution-mode=cumulative: windows begin at --start-iter and grow by --step-iter
  - If --evolution-mode=blocked: windows are width --step-iter starting at --start-iter
  - In trace mode, additional evolution datasets:
    - /target_flux_evolution [window, state]
    - /conditional_flux_evolution [window, state, state]
- Dataset structure (for each window)
  - iter_start: Integer, start iteration of the window (inclusive)
  - iter_stop: Integer, stop iteration of the window (exclusive)
  - expected: Floating-point, mean rate within the window (units: 1/tau)
  - ci_lbound / ci_ubound: Confidence interval bounds (units: 1/tau)
  - corr_len: Integer, correlation length of the rate within the window (units: tau)
- Attributes (attached to each dataset)
  - mcbs_alpha: Floating-point, alpha for confidence intervals
  - mcbs_nsets: Integer, number of bootstrap samples
  - mcbs_acalpha: Floating-point, alpha for determining correlation lengths

Usage details
- The w_kinavg tool is essentially a compatibility shim around the w_direct tooling, exposing a stable wrapper for users who previously relied on w_kinavg.
- Deprecation: A deprecation warning is emitted on invocation, advising users to switch to w_direct.
- Parallel execution: The command supports WestPA’s parallel tooling via WESTParallelTool.

Usage and examples
- Basic run (uses defaults)
  - This will execute the trace subcommand with default inputs.
  - Command:
    - w_kinavg
- Explicit subcommand and file specification
  - Uses the same trace-based analysis with specified input and output files.
  - Command:
    - w_kinavg trace --kinetics-file kintrace.h5 --output kinavg.h5
- Evolution-mode analysis (cumulative)
  - Configure windows to start at a given iteration and grow by a step size.
  - Command:
    - w_kinavg trace --start-iter 0 --step-iter 100 --evolution-mode cumulative
- Evolution-mode analysis (blocked)
  - Configure windows to have a fixed width (excluding the last potentially shorter window).
  - Command:
    - w_kinavg trace --start-iter 0 --step-iter 100 --evolution-mode blocked
- Help and deprecation notice
  - Since this command is deprecated in favor of w_direct, consider:
    - w_direct trace --kinetics-file kintrace.h5 --output kinavg.h5

Important considerations and edge cases
- Deprecation and migration
  - w_kinavg is deprecated; use w_direct for ongoing development and support.
- Prerequisites
  - Requires that assignments and kinetics data have been generated (see w_assign and w_kinetics equivalents for data preparation).
  - The trace datasets (if used) rely on properly generated flux and state-to-state data.
- Data compatibility
  - Default file names are kintrace.h5 and kinavg.h5, but users may override with --kinetics-file and --output (as supported by DKinAvg).
- Parallel execution
  - The command supports parallel execution paths via WESTParallelTool; ensure proper MPI/parallel environment if using with multiple ranks.
- Edge cases
  - If no iterations are selected (empty window), expected, ci_lbound, ci_ubound may be undefined; ensure window configuration is valid.
  - If --evolution-mode is specified without corresponding --start-iter or --step-iter, defaults or errors may occur depending on the underlying DKinAvg implementation.

Migration notes
- The wrapper is primarily for backward compatibility. For new development and usage, prefer:
  - w_direct trace
- Behavior and outputs are equivalent to the trace-oriented path of w_direct, but the recommended entry point is w_direct.
