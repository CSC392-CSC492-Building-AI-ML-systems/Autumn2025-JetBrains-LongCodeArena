Introduction
------------
w_ipa (WESTPA Interactive Python Analysis) is a stateful command-line tool that provides an interactive, Python-based interface for inspecting and analyzing WESTPA simulation data. It exposes the current analysis context (scheme, iteration, states, bins, weights, and related metadata) and offers utilities to inspect traces, weights, progress coordinates, and more. The data you work with is pulled from the current analysis scheme and iteration, by default using the first enabled scheme.

Overview
--------
- Purpose: Provide an interactive shell to explore WESTPA data in the context of the active analysis scheme and iteration.
- Core concepts:
  - Scheme and iteration: The tool derives its context from the current WESTPA configuration (west.cfg) and the active analysis scheme.
  - Current object: Access to an object named current that exposes walkers, summary, states, seg_id, weights, parents, kinavg, pcoord, bins, populations, and auxdata (if present).
  - Trace function: w.trace(seg_id) runs a history trace for a given segment and returns a detailed dictionary of its history.
- Interface: The tool is designed as a stateful wrapper around WESTPA data sources. It can present output graphically (via matplotlib) or in a text terminal, depending on the selected interface.
- Dependencies: Integrates with WESTPA data access components (WESTDataReader, h5io) and common WESTPA tooling (Plotter, ProgressIndicatorComponent, WIPIDataset, WIPIScheme).

-----------------------
- WIPI (class)
  - Inheritance: WESTParallelTool
  - Purpose: Implement the w_ipa command as a parallel-aware interactive analysis tool.
  - Key collaborators:
    - WESTDataReader: Handles reading of WESTPA data across iterations.

  - Primary responsibilities:

Key methods
-----------
- add_args(parser)
  - Adds command-line options for:
    - --analysis-only / -ao: Run analysis and return to terminal.
    - --reanalyze / -ra: Delete existing files and reanalyze.
    - --ignore-hash / -ih: Ignore argument hashing and avoid regenerating files.
    - --debug / -d: Enable debug output.
    - --terminal / -t: Plot output in terminal (text interface).
    - -f / --f: Hidden/auxiliary argument placeholder.
  - Sets default compression to True.
- process_args(args)
  - Binds runtime options to internal state.
  - Uses data_reader to determine the current iteration and derives the active scheme from west.cfg configuration.
  - Updates internal flags:
    - analysis_mode, reanalyze, ignore_hash, debug_mode
  - If --terminal is specified, switches interface to a text-based mode.
- hash_args(args, extra=None, path=None)
  - Generates a unique hash to determine whether stored results/files are out-of-date with respect to the given arguments.
  - Normalizes path information, removes certain arguments, and returns an MD5-based hash (base64-encoded string of the argument representation).
  - Optional debug output when debug_mode is enabled.
- stamp_hash(h5file_name, new_hash)
  - Opens an h5 file (WESTPAH5File) in read/write mode to stamp an arg_hash attribute.
  - Intended to associate a hash with the stored data to detect changes in configuration/arguments.
  - Note: The provided snippet ends mid-implementation; the documented behavior is stamping the file with the new hash for change detection.

Important parameters and return values
-------------------------------------
- Command-line options:
  - --analysis-only / -ao: If set, run analysis and return to the terminal.
  - --reanalyze / -ra: If set, delete existing analysis files and re-run analysis.
  - --ignore-hash / -ih: Do not regenerate files based on argument hashing.
  - --debug / -d: Enable verbose debug output.
  - --terminal / -t: Render output in a terminal-friendly (text) interface.
  - -f / --f: Placeholder/hidden option for compatibility with existing tooling.
- Data access:
  - data_reader handles iteration data and yields self.niters (iterations minus one, as set in process_args).
  - self.__config and self.__settings provide access to westrc configuration, from which the active analysis scheme is chosen.
  - self.scheme is the selected analysis scheme identifier.
- Return semantics:
  - hash_args returns a string MD5 hash representing the argument state (base64-encoded).
  - stamp_hash returns an opened h5 file handle (read-only semantics implied by context, though the snippet shows read/write access is opened).

Usage notes and considerations
------------------------------
- Stateful nature:
  - w_ipa pulls data from the current analysis scheme and the current iteration; changes to scheme or iteration are reflected in real-time when you access the current property.
- Interface choices:
  - Default interface is matplotlib (graphical output).
  - Use --terminal to switch to a text-based interface for environments without a GUI.
- Reproducibility:
  - The hash_args and stamp_hash mechanisms help determine whether the analysis output is up-to-date with the given arguments and configuration.
  - Use --reanalyze to regenerate analysis when the underlying data or configuration changes.
- Edge cases:
  - If no analysis schemes are enabled, the code path selecting the active scheme may need to handle that gracefully.
  - When ignore_hash is used, the system bypasses argument-change checks, which may lead to stale outputs if configurations change.
  - The stamp_hash function is shown incompletely; in a complete implementation, it should robustly handle file I/O and attribute updates.

Usage examples
--------------
Basic invocation (CLI)
- Run w_ipa with default settings (matplotlib interface):
  w_ipa

- Run in terminal/text mode:
  w_ipa --terminal

- Analyze and return to terminal after completion:
  w_ipa --analysis-only

Force re-analysis (regenerate analysis files):
  w_ipa --reanalyze

Skip argument hashing (do not regenerate on arg changes):
  w_ipa --ignore-hash

Inspect current iteration and scheme (in the interactive shell)
- After launching, in the w_ipa session:
  w.iteration            # Access the current iteration number
  w.scheme               # Access the active analysis scheme
  w.states               # Access defined states in the current scheme
  w.bin_labels           # Access bin labels for current scheme
  w.current              # Access the current data object
  w.current.walkers      # Access walkers for the current iteration
  w.current.weights      # Access weights for the current iteration

Run a trace for a specific segment (in the interactive shell)
- seg_id = 123
- trace_info = w.trace(123)
- trace_info contains history and relevant attributes for the requested segment
