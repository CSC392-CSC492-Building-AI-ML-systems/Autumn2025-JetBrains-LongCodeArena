# Biclustering

Biclustering, also known as co-clustering, seeks to discover submatrices (blocks) within a data matrix where a subset of rows exhibits coherent patterns across a subset of columns. Unlike traditional clustering, biclustering simultaneously clusters rows and columns, enabling the discovery of locally coherent structures in data, such as subgroups of samples that respond similarly to a subset of features.

## Overview

- Core idea: identify rectangular blocks in a matrix where rows and columns form a coherent pattern.
- Typical outputs: row and column cluster memberships that define biclusters (blocks).
- Common algorithms in the module:
  - SpectralBiclustering: uses spectral decomposition to uncover checkerboard-like block structure.
  - SpectralCoclustering: a coclustering variant that finds checkerboard patterns with a focus on clustering rows and columns jointly.
  - Bimax (if available): finds exact constant-valued blocks by enumerating row/column subsets.
- Data expectations: usually dense numerical arrays (2D) with numeric values. Preprocessing or normalization may improve results.

## Key classes and functions

- SpectralBiclustering
  - Purpose: uncover block structure in a data matrix via spectral methods.
  - Primary outputs:
    - rows_: boolean/membership matrix for row clusters (shape: n_rows × n_row_clusters)
    - cols_: boolean/membership matrix for column clusters (shape: n_cols × n_col_clusters)
  - Parameters (typical):
    - n_clusters: tuple (n_row_clusters, n_col_clusters)
    - method: {'log', 'linear'}
    - random_state: int or RandomState for reproducibility
  - Relationship: focuses on row and column clustering simultaneously; suitable for detecting checkerboard patterns.

- SpectralCoclustering
  - Purpose: coclustering where both rows and columns are clustered jointly to reveal coco-clusters.
  - Primary outputs:
    - rows_: membership matrix for row clusters (shape: n_rows × n_row_clusters)
    - columns_: membership matrix for column clusters (shape: n_cols × n_col_clusters)
  - Parameters (typical):
    - n_clusters: int (or similar to SpectralBiclustering depending on version)
    - method: {'log', 'linear'}
    - random_state: int or RandomState
  - Relationship: similar to SpectralBiclustering but geared toward coclustering with a focus on joint block structure.

- Bimax (optional/legacy)
  - Purpose: finds exact constant-valued blocks by exploring row and column subsets.
  - Primary outputs:
    - rows_ and cols_ membership structures indicating blocks
  - Parameters: typically specify the number of row/column blocks or similar controls.

## Important parameters and attributes

- n_clusters
  - SpectralBiclustering: a tuple (n_row_clusters, n_col_clusters)
  - SpectralCoclustering: typically an integer or a tuple depending on implementation
  - Determines the expected number of row and column blocks.

- method
  - Choice between 'log' and 'linear'. Affects how patterns are emphasized during decomposition and extraction of blocks.

- random_state
  - For reproducibility of stochastic steps in initialization or optimization.

- rows_ and cols_ (attributes after fit)
  - rows_: membership (or indicator) matrix for row clusters with shape (n_rows, n_row_clusters)
  - cols_: membership (or indicator) matrix for column clusters with shape (n_cols, n_col_clusters)
  - These matrices allow reconstruction of the individual biclusters:
    - For bicluster (r, c), use rows_[:, r] and cols_[:, c] to select the corresponding rows and columns.

- fit(X)
  - X: 2D numeric array-like
  - Returns: self

- predict or transform (if provided by specific estimator)
  - May return cluster assignments for rows/columns; check the estimator’s API.

## Usage examples

- Example 1: SpectralBiclustering on a synthetic matrix with block structure

```python
import numpy as np
from sklearn.cluster import SpectralBiclustering

# Synthetic data with a checkerboard-like block structure
np.random.seed(0)
n_rows, n_cols = 120, 100
X = np.zeros((n_rows, n_cols))
# Create 3 row blocks and 4 column blocks
row_blocks = [(0, 40), (40, 80), (80, 120)]
col_blocks = [(0, 25), (25, 60), (60, 85), (85, 100)]
for i, (r0, r1) in enumerate(row_blocks):
    for j, (c0, c1) in enumerate(col_blocks):
        X[r0:r1, c0:c1] = (i + 1) * (j + 1) + np.random.randn(r1 - r0, c1 - c0) * 0.5

# Add small noise
X += np.random.randn(n_rows, n_cols) * 0.2

model = SpectralBiclustering(n_clusters=(3, 4), random_state=0)
model.fit(X)

print("Row blocks (rows_ shape):", model.rows_.shape)
print("Column blocks (cols_ shape):", model.cols_.shape)

# Inspect a specific bicluster (row block r, column block c)
r = 0
c = 0
row_indices = np.where(model.rows_[:, r] > 0.5)[0]
col_indices = np.where(model.cols_[:, c] > 0.5)[0]
block = X[np.ix_(row_indices, col_indices)]
print("Block shape for bicluster (row {}, col {}): {}".format(r, c, block.shape))
```

## Important considerations and edge cases

- Data preprocessing
  - Normalize or standardize data if required by the chosen biclustering method.
  - Handle missing values appropriately; many implementations expect complete data.
- Block interpretation
  - Membership matrices (rows_ and cols_) define blocks; ensure you understand their shape and indexing to extract biclusters.
- Sparsity and scale
  - Some methods assume dense data and may perform poorly or exhaust memory on very large sparse matrices.
- Noise sensitivity
  - Outliers and noise can affect block detection; consider robust preprocessing or smoothing.
- Randomness
  - For reproducibility, set random_state; results can vary between runs if randomness is involved.
- Parameter choices
  - The choice of n_clusters dramatically affects the number and size of discovered blocks; start with a reasonable guess based on domain knowledge and refine via validation.

## API relationships

- SpectralBiclustering and SpectralCoclustering share a common goal of discovering block structure but differ in their emphasis on the clustering pattern (biclusters vs. coclusters) and how blocks are represented.
- When exact constant blocks are required, Bimax (if available) offers a different search strategy with its own trade-offs.

End-to-end workflow typically involves:
1) Prepare a numeric data matrix X.
2) Choose a biclustering estimator (e.g., SpectralBiclustering with a tuple for n_clusters).
3) Call fit(X) and inspect rows_ and cols_ to identify the biclusters.
4) Extract and analyze specific blocks for downstream tasks.
