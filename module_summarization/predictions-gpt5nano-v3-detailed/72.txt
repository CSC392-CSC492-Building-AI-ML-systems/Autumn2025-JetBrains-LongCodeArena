# metrics.rst

Introduction
------------
The metrics module provides internal utilities for validating and preparing inputs used in pairwise distance computations. It ensures that inputs (dense arrays or sparse matrices) are properly shaped, typed, and finite, and it supports both regular and precomputed distance scenarios. These helpers are intended for developers building or interfacing with distance-based metrics.

API Overview
------------
This module exposes a few core utilities:

- _return_float_dtype(X, Y)
  - Determines an appropriate floating-point dtype for inputs X and Y and ensures they are in a compatible format.
  - Returns possibly coerced X, Y, and the selected dtype.

- check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse='csr', force_all_finite=True, copy=False)
  - Validates and standardizes X and Y for pairwise distance computations.
  - Converts inputs to numpy arrays when needed, checks dimensionality, and ensures compatible feature dimensions.
  - Supports precomputed distance matrices and finite-value requirements.
  - Returns safe_X, safe_Y (both array-like, with safe_Y possibly being a reference to X when Y is None or X).

- check_paired_arrays(X, Y)
  - Validates inputs for paired distance computations.
  - Ensures appropriate shapes and data types for both X and Y.

Key Concepts
------------
- Dense vs Sparse: Inputs may be dense arrays or sparse matrices. The accept_sparse parameter governs allowed formats.
- Precomputed Distances: When precomputed=True, X represents distances to samples in Y. The function enforces dimension compatibility (X.shape[1] == Y.shape[0]).
- Finite Values: force_all_finite controls whether np.inf, np.nan, and pd.NA are allowed. The special option 'allow-nan' permits NaN values but disallows infinities. pd.NA is treated as NaN.
- Copy Behavior: The copy parameter controls whether a forced copy may be triggered during conversion.

Function Details
---------------
1) _return_float_dtype(X, Y)
- Purpose: Decide the appropriate floating dtype for inputs X and Y.
- Behavior:
  - If both X and Y have dtype float32, returns dtype float32.
  - Otherwise returns a default floating dtype (typically float).
  - Ensures X (and Y, when needed) are numpy arrays for dtype inspection.
- Returns:
  - X, Y: possibly coerced to numpy arrays
  - dtype: selected floating dtype (np.float32 or float)

Usage:
```python
X, Y, dtype = _return_float_dtype(X, Y)
```

2) check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse='csr', force_all_finite=True, copy=False)
- Purpose: Validate and prepare X and Y for pairwise distance computations.
- Parameters:
  - X: array-like or sparse matrix of shape (n_samples_X, n_features)
  - Y: array-like or sparse matrix of shape (n_samples_Y, n_features) or None
  - precomputed: bool, default False
    - If True, X is treated as a precomputed distance matrix to samples in Y.
    - Enforces X.shape[1] == Y.shape[0].
  - dtype: None or dtype
    - If None, dtype is determined by _return_float_dtype.
  - accept_sparse: str, bool, or list/tuple of str, default 'csr'
    - Allowed sparse formats; inputs may be converted to the first allowed format.
    - True allows any sparse format; False rejects sparse inputs.
  - force_all_finite: bool or 'allow-nan', default True
    - Controls finiteness checks:
      - True: all values must be finite
      - False: allows inf and NaN
      - 'allow-nan': allows NaN but not infinite values
    - pd.NA is converted to NaN (and treated accordingly)
  - copy: bool, default False
    - Whether to force a copy during conversion.
- Behavior:
  - Ensures X and Y are arrays (or converted to arrays) with a consistent dtype.
  - If Y is X or Y is None, both are validated against the same array via check_array.
  - If precomputed is False, checks that X.shape[1] == Y.shape[1].
  - If precomputed is True, checks that X.shape[1] == Y.shape[0].
- Returns:
  - safe_X: validated array-like of shape (n_samples_X, n_features)
  - safe_Y: validated array-like of shape (n_samples_Y, n_features)
- Edge cases:
  - Shape or dtype mismatches raise ValueError with informative messages.
  - Incompatible dimensions between X and Y raise ValueError.
  - Non-finite values (when force_all_finite is True) raise DataConversionWarning or ValueError as appropriate.

Usage:
```python
# Basic usage with dense arrays
X = np.array([[1.0, 2.0], [3.0, 4.0]])
Y = None  # will be treated as a pointer to X
safe_X, safe_Y = check_pairwise_arrays(X, Y)

# Precomputed distances example
# X represents distances to samples in Y
X = np.array([[0.0, 1.0], [1.0, 0.0]])
Y = np.array([[0.0, 1.0, 2.0], [1.0, 0.0, 3.0]])
safe_X, safe_Y = check_pairwise_arrays(X, Y, precomputed=True)
```

3) check_paired_arrays(X, Y)
- Purpose: Validate inputs for paired distance metrics.
- Behavior:
  - Ensures X and Y are arrays, with at least 2 dimensions.
  - Converts elements to floating types as needed.
  - Establishes compatibility between X and Y for paired distance computations.
- Returns:
  - Typically processed X, Y (as numpy arrays) ready for downstream distance calculations.

Usage:
```python
X = np.array([[1.0, 2.0], [3.0, 4.0]])
Y = np.array([[5.0, 6.0], [7.0, 8.0]])
safe_X, safe_Y = check_paired_arrays(X, Y)
```

Important Considerations and Edge Cases
---------------------------------------
- Y is None: Y is treated as a reference to X, and both X and Y are validated accordingly.
- Sparse inputs: If accept_sparse is a string (e.g., 'csr'), sparse inputs will be converted to the first allowed format. True allows any format; False rejects sparse inputs.
- Precomputed distances: When precomputed=True, the shape constraint X.shape[1] == Y.shape[0] must hold. This reflects the interpretation that X contains distances from query samples to indexed samples in Y.
- Finite-value handling: Use force_all_finite to control how NaN, Inf, and pd.NA are treated. pd.NA is converted to NaN; NaN is allowed only when force_all_finite is set to 'allow-nan' or False.
- Copy behavior: If copy=False, conversions may not duplicate data; a copy may still occur depending on the input type and conversion steps.

Usage Examples
--------------
- Validating with dense inputs:
```python
import numpy as np
from some_module.metrics import check_pairwise_arrays

X = np.array([[1.0, 2.0], [3.0, 4.0]])
safe_X, safe_Y = check_pairwise_arrays(X, None)
```

- Validating with precomputed distances:
```python
import numpy as np
from some_module.metrics import check_pairwise_arrays

X = np.array([[0.0, 1.0], [1.0, 0.0]])   # distances to samples in Y
Y = np.array([[0.0, 1.0, 2.0], [1.0, 0.0, 3.0]])
safe_X, safe_Y = check_pairwise_arrays(X, Y, precomputed=True)
```
