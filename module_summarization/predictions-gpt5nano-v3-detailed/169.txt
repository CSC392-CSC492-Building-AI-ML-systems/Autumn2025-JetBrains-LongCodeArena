Contributing to Qiskit Metal
=============================

This guide provides practical information for contributing to the Qiskit Metal analyses package, including the CPW parameter calculations and related utilities. It covers repository structure, how to extend existing analyses, testing, and contribution workflows.

Overview of the analyses package
--------------------------------
The analyses package contains mathematical helpers used by Qiskit Metal components to compute physical parameters for planar superconducting circuits. It includes two primary submodules:

- cpw_calculations: Functions for calculating CPW (coplanar waveguide) related parameters such as guided wavelength and lumped equivalent circuit elements.
- lumped_capacitive: Helpers related to lumped-capacitance modeling (in practice, part of the same family of CPW analyses).

Key functions
-------------
The following functions are central to current CPW-related analyses. Contributing authors should ensure these APIs remain stable or are clearly versioned when breaking changes are introduced.

- guided_wavelength(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45)
  - Purpose: Compute the guided wavelength for a planar CPW transmission line, accounting for dielectric loading and film thickness.
  - Parameters:
    - freq (float): Frequency in Hz.
    - line_width (float): CPW center conductor width in meters.
    - line_gap (float): CPW gap width in meters.
    - substrate_thickness (float): Dielectric substrate thickness in meters.
    - film_thickness (float): Thin film thickness in meters.
    - dielectric_constant (float, default 11.45): Relative permittivity of the substrate.
  - Returns:
    - lambdaG (float): Guided wavelength in meters.
    - etfSqrt (float): Effective dielectric constant (square root form) accounting for film thickness.
    - q (float): Filling factor.
  - Usage example:
    .. code-block:: python
       from qiskit_metal.analyses.cpw_calculations import guided_wavelength
       lambdaG, etfSqrt, q = guided_wavelength(freq=5e9,
                                             line_width=10e-6,
                                             line_gap=6e-6,
                                             substrate_thickness=760e-6,
                                             film_thickness=200e-9)

- lumped_cpw(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45, loss_tangent=1e-6, london_penetration_depth=30e-9)
  - Purpose: Provide a lumped-element equivalent model for a CPW line, suitable for fast circuit-level analyses.
  - Parameters:
    - freq (float): Frequency in Hz.
    - line_width (float): CPW center conductor width in meters.
    - line_gap (float): CPW gap width in meters.
    - substrate_thickness (float): Dielectric substrate thickness in meters.
    - film_thickness (float): Thin film thickness in meters.
    - dielectric_constant (float, optional): Relative permittivity of the substrate.
    - loss_tangent (float, optional): Dielectric loss tangent.
    - london_penetration_depth (float, optional): Superconductor London penetration depth.
  - Returns:
    - Lk (float): Series kinetic inductance (Henries).
    - Lext (float): Series geometric external inductance (Henries).
    - C (float): Shunt capacitance (Farads).
    - G (float): Shunt admittance (Siemens).
    - Z0 (float): Characteristic impedance sqrt(L/C).
    - etfSqrt2 (float): Square of the effective dielectric constant.
    - Cstar (float): External inductance (Henries).
  - Usage example:
    .. code-block:: python
       from qiskit_metal.analyses.cpw_calculations import lumped_cpw
       Lk, Lext, C, G, Z0, etfSqrt2, Cstar = lumped_cpw(freq=5e9,
                                                        line_width=10e-6,
                                                        line_gap=6e-6,
                                                        substrate_thickness=760e-6,
                                                        film_thickness=200e-9)

Notes and edge cases
--------------------
- Dependencies: These analytical helpers rely on NumPy and SciPy (for special functions such as elliptic integrals). Ensure these dependencies are installed when contributing.
- Parameter validation: The functions assume physically reasonable input values. Consider adding input validation if extending the API to improve robustness.
- Edge cases:
  - Very thin or very thick films (t approaching 0 or large values) can affect etfSqrt calculations and numerical stability.
  - Substrate/permittivity values outside typical ranges may yield non-physical results; consider safeguarding or emitting warnings in extended code.
  - For lumped_cpw, the kinetic inductance Lk can dominate at very small geometries or very large London depths; users should be aware of the regime where the lumped model remains valid (Pearl-length considerations).

Getting started
---------------
- Prerequisites:
  - Python (compatible with the project’s Python version)
  - numpy, scipy
  - pytest (for running tests)
- Setting up a development environment:
  - Fork the repository, clone your fork, and install dependencies in a virtual environment.
  - Run the test suite locally to verify changes.

Code structure and contribution guidelines
------------------------------------------
- Follow the repository’s coding style and docstring conventions.
- When adding new analyses, document each public function with a clear docstring describing purpose, parameters, returns, and edge cases.
- Add or update unit tests to cover typical usage and boundary conditions.
- Update Sphinx documentation as needed to reflect new functionality.
- Ensure backward compatibility for minor changes; if substantial changes are required, propose a clear deprecation plan or a new API version.

API stability and documentation
-------------------------------
- Public APIs should be well-documented with clear input/output semantics.
- Update contributor or developer docs to reflect any changes to the analyses API or expected usage.
- Consider adding autosummary entries for new modules and functions to keep documentation in sync.

Testing and quality checks
--------------------------
- Run unit tests:
  - pytest -q
- Run linters and style checks (as per project guidelines, e.g., flake8 or isort if configured).
- Validate numerical stability with representative parameter sets (typical device geometries and cryogenic material properties).

Contribution workflow
---------------------
- Create a feature branch based on the latest main branch.
- Implement changes with accompanying tests and updated docs.
- Run the full test suite and ensure all tests pass.
- Open a pull request with a clear description of changes, rationale, and any migration notes.
- Respond to reviews and iterate as needed.
