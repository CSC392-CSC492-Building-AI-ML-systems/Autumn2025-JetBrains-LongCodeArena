Channel Options
===============

Overview
--------
This module provides a lightweight IPython/Jupyter integration for rendering Vega-Lite specifications. It focuses on how to configure and apply channel encodings (the x, y, color, size, etc. channels) within Vega-Lite specs, and how to supply data to those specs from Python variables or from external sources.

Key components and relationships
-------------------------------
- vegalite (cell magic)
  - A cell/magic function that renders Vega-Lite specifications inside IPython environments.
  - Usage: %%vegalite [data_variable] [--json] [--version='v5']
  - If a data variable is provided, the data is injected into the spec using the specified transformer pipeline.
  - Supports spec input in JSON (default), or YAML when YAML is available.
- RENDERERS
  - Maps renderer names and version strings to the corresponding Vega-Lite rendering class.
  - Example: RENDERERS["vega-lite"]["5"] -> VegaLite class for Vega-Lite v5.
- TRANSFORMERS
  - Maps renderer/version to data transformation utilities used to prepare pandas DataFrames for Vega-Lite.
- _prepare_data(data, data_transformers)
  - Converts Python data into a format suitable for Vega-Lite spec["data"].
  - Behaviors:
    - None or dict: returned as-is.
    - pandas.DataFrame: transformed via the data_transformers pipeline.
    - str: treated as a URL payload: {"url": <string>}.
    - others: a warning is issued and the data is returned unchanged.
- _get_variable(name)
  - Retrieves a variable by name from the current IPython namespace.
  - Requires to run inside an IPython environment; otherwise raises an error.
- YAML/JSON handling
  - If --json is used or YAML is not available, input is parsed as JSON.
  - If YAML is available, the spec can be provided in YAML format.
  - If YAML is not available and the input is not valid JSON, a helpful error is raised.
- Deprecated dataset helpers
  - load_dataset and list_datasets provide compatibility shims around the vega_datasets package.
  - They are marked deprecated and will guide users toward using vega_datasets directly.

Parameters and return values
----------------------------
- vegalite(...)
  - line: string containing the magic’s line arguments (positional data variable name, flags).
  - cell: the cell content containing the Vega-Lite specification (JSON or YAML).
  - Returns: an instance of the Vega-Lite renderer (which renders the spec in the notebook).

- _prepare_data(data, data_transformers)
  - Input: data (None, dict, DataFrame, or str), data_transformers (callable or transformer registry).
  - Output: a data payload suitable for spec["data"] (e.g., {"values": ...}, {"url": "..."}).

- _get_variable(name)
  - Input: name (str) of a variable in the notebook namespace.
  - Output: the corresponding Python object from the namespace.
  - Errors: NameError if the variable does not exist; ValueError if not in an IPython environment.

Channel options in Vega-Lite (encoding)
---------------------------------------
- The primary mechanism to customize how data is visualized is through encoding channels:
  - x, y: define the horizontal and vertical data mappings.
  - color, shape, size, opacity: define visual attributes tied to data fields.
  - tooltip, href, detail, text: provide additional data-driven affordances.
- Each channel supports sub-properties, such as:
  - field, type (quantitative, ordinal, nominal, time), bin/aggregate, scale, axis/legend configurations.
  - For example:
    - x: {"field": "Horsepower", "type": "quantitative"}
    - color: {"field": "Origin", "type": "nominal", "scale": {"scheme": "category20"}}
- Channel options are defined by the Vega-Lite spec itself; this module merely passes the spec (and optionally injects data) to the Vega-Lite renderer.

Usage examples
--------------
Basic usage (no data variable)
- Renders a simple Vega-Lite spec embedded in the cell:
.. code-block:: json

   {
     "mark": "point",
     "encoding": {
       "x": {"field": "Horsepower", "type": "quantitative"},
       "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
       "color": {"field": "Origin", "type": "nominal"}
     }
   }

Using a pandas DataFrame from the notebook
- Suppose you have a DataFrame named df in scope and you want to render with its data:
.. code-block:: none

   %%vegalite df
   {
     "mark": "point",
     "encoding": {
       "x": {"field": "Horsepower", "type": "quantitative"},
       "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
       "color": {"field": "Origin", "type": "nominal"}
     }
   }

- The DataFrame df will be transformed via the configured data transformers and injected as spec["data"].

Using JSON vs YAML for the spec
- By default, the cell content is parsed as JSON.
- If YAML support is installed (YAML_AVAILABLE), you can provide YAML in the cell.
- You can also force JSON parsing with the --json flag:
.. code-block:: none

   %%vegalite --json df
   (cell content with JSON spec)

Edge cases and considerations
------------------------------
- IPython environment: _get_variable requires an active IPython runtime; otherwise a ValueError is raised.
- Data injection: providing a data variable name injects data into the spec, replacing spec["data"].
- Data types: DataFrame input is transformed via the configured data_transformers; unsupported types warn and pass through.
- YAML vs JSON: If YAML is not installed, non-JSON specs will raise a parsing error; if YAML is installed, YAML is accepted.
- Versioning: The default renderer version is v5; you can specify a different version via --version, e.g., v5.

Version information
-------------------
- This module exposes __version__ = "4.3.0.dev0" reflecting the development snapshot.

Notes
-----
- The deprecated dataset helpers load_dataset and list_datasets are provided for compatibility but are superseded by the vega_datasets package.
- The channel options described here are expressed via the Vega-Lite spec; this module does not provide a separate API for channel tuning beyond standard Vega-Lite spec syntax.

Code references (high level)
----------------------------
- vegalite: main IPython cell magic for rendering
- RENDERERS: mapping to Vega-Lite 5 renderer
- TRANSFORMERS: data transformation utilities for DataFrame inputs
- _prepare_data, _get_variable: internal helpers for data handling and namespace access
- load_dataset / list_datasets: deprecated helpers for dataset access
