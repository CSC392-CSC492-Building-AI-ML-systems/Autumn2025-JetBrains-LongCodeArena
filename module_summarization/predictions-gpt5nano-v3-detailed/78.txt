A concise API reference and usage guide for the filenode.py module.

Introduction
------------
The filenode module provides a binary file-like interface to PyTables file nodes. It allows treating a PyTables file node as a standard Python file object, enabling random access, read/write (depending on mode), and typical file-like operations. The module is designed to be compliant with the Python io interfaces (as of version 3.0, it was rewritten to align with io).

Key concepts
-----------
- NodeType: A constant indicating the node type, with the value 'file'.
- NodeTypeVersions: Supported NODE_TYPE_VERSION values, currently [1, 2].
- RawPyTablesIO: A binary I/O class deriving from io.RawIOBase that provides a low-level file-like interface for PyTables file nodes. It handles position tracking, mode validation, reading, seeking, and other core IO functionality.

Usage overview
--------------
- Acquire a PyTables file node (via your project’s node creation/opening workflow, e.g. new_node() or open_node()).
- Wrap the node with RawPyTablesIO to obtain a binary, file-like interface.
- Use standard file-like operations such as seek, tell, readinto, and others as provided by the IO base class, subject to the mode used when opening.

Important notes
--------------
- Read access is always available.
- Write access is enabled for new files and files opened with mode 'a+' (append). Writing only appends data to the end of the file node; writing to arbitrary positions is not implied in the shown API.
- The module has been rewritten in version 3.0 to be fully compliant with the Python io interfaces.
- The implementation relies on PyTables node attributes (e.g., NODE_TYPE_VERSION) and internal node methods such as read(), nrows, and _v_file for fileno().

API reference
-------------

Constants
- NodeType
  - Description: Value for the NODE_TYPE node system attribute.
  - Example:
    - NodeType = 'file'
- NodeTypeVersions
  - Description: Supported values for NODE_TYPE_VERSION node system attribute.
  - Example:
    - NodeTypeVersions = [1, 2]

Class: RawPyTablesIO
- Inheritance: io.RawIOBase

Purpose
- Provide a binary file-like interface to a PyTables file node, allowing reading, writing (when permitted by mode), seeking, and flush/close operations in a manner compatible with Python’s io framework.

Constructor
- RawPyTablesIO(node, mode=None)
  - Parameters:
    - node: The PyTables file node to operate on.
    - mode: Optional string indicating the access mode (e.g., 'r', 'w', 'a', 'r+', etc.). If None, the mode is taken from node._v_file.mode.
  - Behavior:
    - Validates the node and required attributes.
    - If mode is provided, validates and cross-checks against the underlying PyTables file mode.
    - Initializes internal state, including position, version, shape mapping, and data type.

Public attributes and methods

- mode (property)
  - Returns the current access mode as a string.

- tell()
  - Returns the current stream position (byte offset).

- seek(pos, whence=0)
  - Repositions the stream to a new byte offset.
  - whence values:
    - 0: start of stream (pos must be >= 0)
    - 1: current position
    - 2: end of stream (typically negative offset from end)
  - Returns the new absolute position.
  - Raises ValueError for invalid whence or negative positions.

- seekable()
  - Always returns True (random access is supported).

- fileno()
  - Returns the underlying file descriptor, if available.
  - Raises an error if the IO object does not use a file descriptor.

- close()
  - Flushes and closes the IO object if not already closed.
  - Releases the node object to allow the file to close.

- flush()
  - Flushes write buffers, if applicable.
  - Calls node.flush().

- truncate(pos=None)
  - Truncates the file to a given size in bytes.
  - If pos is None, uses the current position.
  - Only supports growing the file in the provided implementation (cannot shrink the data within existing content).
  - Returns the new file position after truncation.

- readable()
  - Returns True if the IO was opened for reading ('r' or '+').

- writable()
  - Returns True if the IO was opened for writing ('w', 'a', or '+').

- readinto(b)
  - Reads up to len(b) bytes into the provided writable buffer b.
  - Returns the number of bytes read (0 indicates EOF).
  - Updates the internal position accordingly.
  - Uses the node’s read() method to fetch data as needed.
  - Handles end-of-file conditions based on node.nrows.

- readline(limit=-1)  (description)
  - Reads and returns a line from the stream, up to a newline or the specified limit.
  - The exact behavior depends on the underlying node’s line-oriented semantics and the implementation.

Usage examples
--------------

Example 1: Opening an existing node for read-only access
- This example assumes you have a PyTables file node obtained from your project’s node management (e.g., open_node).
- It demonstrates reading raw bytes into a buffer.

```python
from pytables import filenode as fn  # hypothetical module path

# Acquire an existing PyTables file node (implementation-dependent)
node = get_existing_pytable_node()

# Open a binary file-like interface for read-only access
with fn.RawPyTablesIO(node, mode='r') as f:
    buf = bytearray(4096)
    n = f.readinto(buf)
    data = buf[:n]
    # process data as needed
```

Example 2: Opening a node with write/append access
- Demonstrates append-oriented writing behavior (mode and underlying implementation permitting writes).

```python
from pytables import filenode as fn

node = get_existing_pytable_node()

# Open with append mode (write access, appending data)
with fn.RawPyTablesIO(node, mode='a+') as f:
    # Example: append data
    payload = b'some binary payload'
    n = f.write(payload)  # write implementation provided by full IO class
    f.flush()
```

Notes
- The above examples assume integration with your PyTables-based node management utilities (new_node, open_node, or equivalent) that provide the node object compatible with RawPyTablesIO.
- For actual usage, consult your project’s node creation/opening workflow to obtain a valid node instance compatible with the filenode module.

Edge cases and considerations
-----------------------------
- Negative seek positions raise ValueError.
- Invalid whence values raise ValueError.
- Truncation only supports growing the file in the current implementation; attempting to shrink or truncate to a lower size raises an error.
- Readable and writable properties depend on the mode string supplied at construction.
- The module emphasizes compliance with io API semantics (as of 3.0), including proper closed-state handling and cross-checking of modes against the underlying PyTables file.
