Using lists
===========

Overview
--------
This section documents how to work with lists in Python and with the collection types provided by the ftrack API. Lists are the primary data structure for storing ordered sequences of items, including entities returned by the API or values derived from them.

Basic list operations
---------------------
Common Python list operations include creation, indexing, slicing, and mutating the list.

- Creation
  - Example: create a simple list of integers
    ```python
    values = [1, 2, 3]
    ```

- Access and slicing
  - Access the first element
    ```python
    first = values[0]  # 1
    ```
  - Slicing
    ```python
    sub = values[1:3]  # [2, 3]
    ```

- Length and membership
  - Length
    ```python
    n = len(values)  # 3
    ```
  - Check membership
    ```python
    2 in values  # True
    4 in values  # False
    ```

- Mutating a list
  - Append, extend, insert
    ```python
    values.append(4)          # [1, 2, 3, 4]
    values.extend([5, 6])     # [1, 2, 3, 4, 5, 6]
    values.insert(2, 'a')     # [1, 2, 'a', 3, 4, 5, 6]
    ```
  - Remove, pop, clear
    ```python
    values.remove('a')        # [1, 2, 3, 4, 5, 6]
    last = values.pop()         # 6, values -> [1, 2, 3, 4, 5]
    values.clear()              # []
    ```

Working with ftrack collections
-------------------------------
The ftrack API may return collection objects (e.g., ftrack_api.collection.Collection) that behave like iterables or lists. You can treat them as lists for most purposes, or explicitly materialize them to a Python list.

- Materialize to a Python list
  ```python
  # Suppose 'results' is a ftrack collection of entity objects
  results_list = list(results)
  ```
- Iterate directly
  ```python
  for entity in results:
      do_something(entity)
  ```

- Access properties from entities
  ```python
  names = [e.name for e in results]
  ids = [e.id for e in results]
  ```

Conversion and iteration
------------------------
- Converting to a list is useful when you need random access, repetition, or Python list utilities.
  ```python
  items = list(results)          # materialize
  first_five = items[:5]           # slicing on a concrete list
  ```
- For large result sets, consider streaming or paging strategies to avoid high memory usage.

Sorting and filtering
---------------------
- Sorting a list of entities by a property
  ```python
  entities = list(results)
  entities.sort(key=lambda e: e.name)
  ```
- Using sorted (without in-place modification)
  ```python
  sorted_entities = sorted(results, key=lambda e: e.created_at)
  ```
- Filtering with list comprehensions
  ```python
  active_entities = [e for e in results if getattr(e, 'is_active', False)]
  ```

Copying and mutability
----------------------
- Shallow copy (new list, same element references)
  ```python
  copy_of_results = list(results)
  ```
- Deep copy (new list and recursively copied elements)
  ```python
  import copy
  deep_copy = copy.deepcopy(results)
  ```
- Be mindful: mutating the list structure does not automatically mutate the elements themselves.

Edge cases and performance
--------------------------
- Empty lists and None values
  - An empty list is common when a query returns no results.
  - Items within a list may be None; guard access as needed.
- Large lists
  - Python lists hold references to objects; large lists can consume significant memory.
  - If possible, process results in chunks or use generators where supported by the API.

Code examples: common patterns
------------------------------
- Extract IDs from a collection of entities
  ```python
  entity_ids = [e.id for e in results]
  ```

- Build a mapping from name to entity
  ```python
  mapping = {e.name: e for e in results}
  ```

- Filter and transform in one pass
  ```python
  active_names = [e.name for e in results if getattr(e, 'is_active', False)]
  ```

Notes
-----
- The ftrack API collection types are designed to be convenient to use as Python iterables. When you need full Python list capabilities (random access, slicing, or list-specific utilities), materialize the collection with list(your_collection).
- If a collection represents a lazy-loading query, iterating over it or converting it to a list will trigger data retrieval from the server.

Usage examples
--------------
- Example: simple list usage
  ```python
  # Basic Python list
  fruits = ['apple', 'banana', 'cherry']
  print(len(fruits))        # 3
  print(fruits[1])           # 'banana'
  fruits.append('date')
  print(fruits)              # ['apple', 'banana', 'cherry', 'date']
  ```

- Example: working with a ftrack collection of entities
  ```python
  # Assume 'results' is a ftrack collection of Project entities
  project_names = [p.name for p in results]
  # Or materialize for further processing
  results_list = list(results)
  for p in results_list:
      process(p)
  ```
