# Grid search: Tuning the hyper-parameters of an estimator

This module provides a systematic way to exhaustively search over specified parameter values to optimize an estimator. By evaluating all combinations of parameters via cross-validation (or a user-provided split strategy), GridSearchCV helps identify the best performing configuration and (optionally) refit the best estimator on the full training data.

## Overview

- GridSearchCV performs an exhaustive search over a parameter grid.
- It integrates with any estimator that follows the standard fit/predict API.
- Supports cross-validation strategies, custom scoring, and parallel execution.
- Works naturally with Pipelines, using parameter names that include step prefixes (e.g., pipeline steps named with double underscores).

## Key concepts and relationships

- GridSearchCV
  - estimator: The base estimator to tune.
  - param_grid: Dictionary or list of dictionaries describing parameter values to search. Keys are parameter names; values are lists of parameter settings to try. When using Pipelines, prefix parameters with the step name (e.g., 'svc__C').
  - cv: Cross-validation strategy (integer for number of folds or a cross-validation splitter).
  - scoring: Scoring metric or callable used to evaluate parameter combinations. If None, uses the estimator’s default score method.
  - refit: Whether to refit an estimator using the best parameters on the entire training set. Can be True, False, or a string naming a scoring target.
  - return_train_score: Whether to include train scores in the results.
  - n_jobs: Number of jobs to run in parallel.
  - pre_dispatch: Controls the number of candidate parameter settings that get dispatched for fitting.
  - verbose: Controls the verbosity of the output during fitting.
- ParameterGrid (and the underlying mechanism)
  - Builds the complete cartesian product of parameter values from param_grid.
  - Used to determine all combinations that GridSearchCV will evaluate.
- Pipeline compatibility
  - When tuning parameters of a Pipeline, use the step name followed by two underscores and the parameter name (e.g., 'scaler__with_mean').

## Important parameters and return values

- GridSearchCV parameters
  - estimator: The estimator object to tune.
  - param_grid: Dict[str, list] or List[Dict[str, List[Any]]], specifying parameter values to search.
  - cv: int or CV splitter; default 5.
  - scoring: str or callable; default=None (uses estimator.score).
  - refit: bool or str; default True. If True, refit the best estimator on the entire training set. If a string, refit using the specified scoring target.
  - n_jobs: int; default=None (uses single processor).
  - verbose: int; controls the verbosity.
  - return_train_score: bool; default=False.
  - pre_dispatch: str or int; default '2*n_jobs'.
  - error_score: float or 'raise'; default=np.nan. Value to assign if an exception occurs in a candidate fit.
- Return values and attributes
  - best_params_: Parameters of the best estimator.
  - best_score_: Best cross-validation score.
  - best_estimator_: Estimator refitted on the full training data with best parameters (if refit=True).
  - cv_results_: Dict containing detailed cross-validation results for all parameter combinations.
  - best_index_: Index of the best parameter set in the grid.
  - n_splits_: Number of cross-validation splits used.
  - grid_scores_ (deprecated in favor of cv_results_): Historical alias for results.
- Other considerations
  - When using Pipelines, parameter names must reflect the pipeline structure (e.g., 'svc__C', 'scaler__with_mean').
  - If param_grid is large, GridSearchCV can be time and memory intensive.

## Usage examples

### Example 1: Simple grid search for a classifier

```python
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import GridSearchCV

# Load data
X, y = load_iris(return_X_y=True)

# Define parameter grid
param_grid = {
    'C': [0.1, 1.0, 10.0],
    'solver': ['liblinear', 'lbfgs'],
    'max_iter': [200, 500],
}

# Create GridSearchCV object
grid = GridSearchCV(
    estimator=LogisticRegression(),
    param_grid=param_grid,
    cv=5,
    scoring='accuracy',
    return_train_score=True,
)

# Run search
grid.fit(X, y)

# Best results
print("Best params:", grid.best_params_)
print("Best cross-validated score:", grid.best_score_)
print("Best estimator:", grid.best_estimator_)
```

### Example 2: Grid search with a Pipeline

```python
from sklearn.datasets import load_digits
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC
from sklearn.model_selection import GridSearchCV

# Data
X, y = load_digits(return_X_y=True)

# Pipeline: Standardize then classify
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('svc', SVC())
])

# Parameter grid targeting pipeline steps
param_grid = {
    'svc__C': [0.5, 1.0, 2.0],
    'svc__gamma': ['scale', 'auto'],
    'svc__kernel': ['rbf', 'linear'],
}

grid = GridSearchCV(
    estimator=pipe,
    param_grid=param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1
)

grid.fit(X, y)

print("Best params:", grid.best_params_)
print("Best cross-validated score:", grid.best_score_)
print("Best estimator:", grid.best_estimator_)
```

## Important considerations and edge cases

- Grid size and performance
  - The total number of parameter combinations is the product of the lengths of the lists in param_grid. Large grids can be very time-consuming.
  - Use pre_dispatch to limit the number of fits dispatched concurrently and control memory usage.

- Pipeline interactions
  - When tuning a Pipeline, ensure parameter names include the correct step name (e.g., 'clf__C' for a step named 'clf').

- Scoring and refitting
  - If scoring is not provided, the estimator’s default score method is used.
  - If refit is True, GridSearchCV refits the best parameters on the full training data. If a string is provided, it uses the corresponding scoring metric for refitting the best estimator.

- Handling errors
  - error_score controls what value is assigned if a candidate parameter combination raises an exception. By default, it is set to NaN, which allows the search to continue.

- Cross-validation strategy
  - The cv parameter accepts an integer (the number of folds) or a cross-validation splitter. Custom CV strategies can be used to tailor evaluation to your data.

## Notes

- GridSearchCV exposes results via:
  - best_params_, best_score_, best_estimator_, cv_results_, best_index_, n_splits_.
- For more flexibility in parameter search (e.g., probabilistic sampling), consider RandomizedSearchCV or Bayesian optimization methods.
