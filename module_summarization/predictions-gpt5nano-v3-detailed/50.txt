GAM Spline Smoothers: B-spline Basis Utilities (gam.rst)

Introduction
------------
This module provides the internal utilities used to construct and evaluate B-spline bases for Generalized Additive Models (GAMs). It includes knot placement helpers, basis evaluation (and derivatives), and convenience functions to assemble knot vectors and spline bases for use in smoothing terms (e.g., BSplines) within GLMGam and related GAM components.

Overview of key concepts
------------------------
- Knot placement: choose how knots are distributed along the predictor domain (quantile-based or equally spaced).
- Basis evaluation: compute B-spline basis functions (and first/second derivatives) at given x values for a specified degree.
- Knot assembly: generate the full knot vector including boundary/extreme knots and inner knots.
- Basis construction: create full basis matrices (basis, derivative bases) for modeling.

Public API and key functions
----------------------------
- _equally_spaced_knots(x, df)
  - Purpose: generate equally spaced interior knots given data x and desired degrees of freedom (df).
  - Returns: an array of knot locations.

- _R_compat_quantile(x, probs)
  - Purpose: compute quantile-based knot locations in a way compatible with R/patsy conventions.
  - Returns: quantiles shaped like the input probs.

- _eval_bspline_basis(x, knots, degree, deriv='all', include_intercept=True)
  - Purpose: evaluate B-spline basis functions (and optional derivatives) at points x for a given knot vector and degree.
  - Parameters:
    - x: 1D array-like of predictor values.
    - knots: 1D array-like of knots (must be sorted).
    - degree: degree of the spline (e.g., 3 for cubic).
    - deriv: 'all' to return (basis, der1, der2) or 0/1/2 to return the corresponding derivative basis.
    - include_intercept: whether to include the intercept term in the basis.
  - Returns:
    - If deriv='all': (basis, der1_basis, der2_basis)
    - Otherwise: the requested derivative basis array(s)

- compute_all_knots(x, df, degree)
  - Purpose: compute full knot vector and related bounds for B-splines.
  - Returns: (all_knots, lower_bound, upper_bound, inner_knots)

- make_bsplines_basis(x, df, degree)
  - Purpose: convenience wrapper to construct B-spline basis and first/second derivatives for x.
  - Returns: (basis, der_basis, der2_basis)

- get_knots_bsplines(x=None, df=None, knots=None, degree=3, spacing='quantile', lower_bound=None, upper_bound=None, all_knots=None)
  - Purpose: public API to obtain knot vectors for B-splines using different strategies.
  - Parameters:
    - x: 1D array-like input (used when calculating knots via spacing).
    - df: degrees of freedom (affects the number of inner knots).
    - knots: user-specified knots (if provided, returned as-is).
    - degree: spline degree (default 3).
    - spacing: 'quantile' (default) or 'equal' to determine knot placement strategy.
    - lower_bound, upper_bound: optional bounds for knots (used with equal spacing or extant bounds).
    - all_knots: if provided, returned directly (bypasses knot computation).
  - Returns:
    - all_knots: computed or provided knot vector (1D array)

Notes and edge cases
--------------------
- Out-of-bounds data points: _eval_bspline_basis raises NotImplementedError if any x value falls outside [min(knots), max(knots)]. Ensure your data lies within the knot span or extend knots accordingly.
- Intercept handling: include_intercept in _eval_bspline_basis affects the number of basis functions. The effective number of bases is len(knots) - (degree + 1) - k_const, where k_const = 1 - int(include_intercept).
- Knot strategy: get_knots_bsplines allows flexible knot selection:
  - Provide all_knots directly via all_knots to reuse a precomputed knot vector.
  - Use df and degree with spacing='quantile' or spacing='equal' to derive knots automatically.
- Compatibility: the implementation mirrors Patsy/R's approach for B-splines; internal helpers ensure consistent knot placement and basis construction for GAM smoothing terms.

Usage example
-------------
The following example demonstrates generating a B-spline basis for a synthetic dataset, using different approaches to knots and basis construction.

Code example 1: Direct knot computation and basis evaluation
```python
import numpy as np
# synthetic predictor
x = np.linspace(0, 1, 100)
df = 8          # degrees of freedom for the spline (affects number of inner knots)
degree = 3      # cubic splines

# compute full knot sequence and bounds
all_knots, lower_bound, upper_bound, inner_knots = compute_all_knots(x, df, degree)

# evaluate basis and derivatives at x
basis, der1_basis, der2_basis = _eval_bspline_basis(x, all_knots, degree)

# shapes
print(basis.shape)      # (100, n_bases)
print(der1_basis.shape) # (100, n_bases)
print(der2_basis.shape) # (100, n_bases)
```

Code example 2: High-level convenience for B-spline basis
```python
# using the helper that returns the basis and derivatives
B, dB, d2B = make_bsplines_basis(x, df, degree)
```

Code example 3: Knot selection via get_knots_bsplines
```python
# automatic knot construction (quantile-based)
all_knots_auto = get_knots_bsplines(x=x, df=df, degree=degree)

# build basis with the auto-generated knots
B2, dB2, d2B2 = _eval_bspline_basis(x, all_knots_auto, degree)

print(B2.shape, dB2.shape, d2B2.shape)
```

Code example 4: Specifying knots directly
```python
# provide a custom set of knots
custom_knots = np.linspace(0, 1, 12)
B3, dB3, d2B3 = _eval_bspline_basis(x, custom_knots, degree)

print(B3.shape)
```

Practical considerations
------------------------
- Choosing df and degree: Higher df yields more flexible splines but can lead to overfitting; balance with model complexity and cross-validation.
- Boundary treatment: The knot vector typically includes boundary multipliers to control behavior at the edges; ensure x data falls within the outermost knots when evaluating.
- Integration with GLMGam: These utilities feed into smoothing components (e.g., BSplines) that integrate with the design matrix construction in GAM modeling pipelines.

Usage notes for developers
--------------------------
- When extending smoothing terms, these helpers provide consistent basis construction and derivative evaluations needed for penalties, constraints, or smoothness components.
- If you need a fully external-facing API, consider wrapping these internals in a small high-level class or function that directly returns a Patsy-compatible design matrix for GLMGam.
