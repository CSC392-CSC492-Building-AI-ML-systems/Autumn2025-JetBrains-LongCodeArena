Simulation Utilities
====================

Overview
--------
This module provides lightweight helpers for visualizing and validating Bluesky plans that drive motor-position devices. It includes:

- plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2): Plot the raster path described by a plan.
- summarize_plan(plan): Print a concise summary of a plan’s moves and event triggers.
- print_summary (alias): Backwards-compatible alias for summarize_plan.
- check_limits(plan): Validate that a plan does not move devices beyond their limits.

Functions and usage
-------------------

plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)
--------------------------------------------------------------------------------
Plot the raster trajectory described by a plan.

Parameters
- plan: Iterable of Msg objects. Must yield Msgs and not be a coroutine.
- x_motor, y_motor: Names of the x and y motors (strings).
- ax: Optional matplotlib.axes.Axes to plot onto. If None, a new figure/axes are created.
- probe_size: Optional float. If provided, draw probe markers as circles with this radius (in the same units as motor positions). If None, points are drawn with a scatter plot.
- lw: Optional float. Line width for the path and, when probe_size is None, for the scatter points.

Returns
- A dict with:
  - 'path': The Line2D object for the plotted path.
  - 'events': The scatter collection (or circle-based patch collection) representing readback points.

Notes and considerations
- The function assumes plan yields messages with:
  - cmd == 'set': updates cur_x or cur_y based on msg.obj.name matching x_motor or y_motor, taking the value from msg.args[0].
  - cmd == 'save': appends the current (cur_x, cur_y) to traj.
- If ax is None, a new figure/axes is created and the aspect is set to equal.
- If there are no saved points, or the first relevant 'set' messages occur after a 'save', traj may be incomplete. If traj is empty, attempting to unzip x, y will raise an exception.
- Edge cases include plans that emit 'save' events before any corresponding 'set' for one or both motors, or plans that do not contain both motors before a save.

Example
```python
import matplotlib.pyplot as plt
from bluesky.plan_stubs import Msg  # examples only; Msg shape depends on the Bluesky version

# Assume `plan` yields Msg objects with fields: command, obj.name, args
# For illustration purposes only; replace with your actual plan/messages.

# Plot a raster path between 'x' and 'y' motors
fig, ax = plt.subplots()
plot = plot_raster_path(plan, x_motor='x', y_motor='y', ax=ax, probe_size=None, lw=2)
plt.show()
```

summarize_plan(plan)
--------------------
Print a minimal, human-readable summary of the plan, focusing on moves and where events are created.

Parameters
- plan: Iterable of Msg objects. Must yield Msg objects.

Returns
- None. The function prints to stdout via print_summary_wrapper(plan).

Notes
- This function uses Bluesky's print_summary_wrapper to render the summary.
- The exact formatting depends on the wrapper implementation.

print_summary
-------------
Backwards-compatible alias for summarize_plan.

check_limits(plan)
------------------
Check that a plan will not move devices outside of their limits.

Parameters
- plan: Iterable of Msg objects. Must yield Msg objects.

Returns
- None. May warn to stdout if a device lacks a check_value() method.

Behavior
- Iterates over the plan; for each 'set' message whose object is not in the ignore list:
  - If the object has a check_value method, call obj.check_value(msg.args[0]).
  - Otherwise, emit a warning that the object has no check_value method and add the object to an ignore list to skip future checks.

Edge cases and considerations
- If a motor object lacks check_value(), a warning is emitted and the object is ignored for subsequent messages.
- Devices with complex validation logic not exposed via check_value() will be skipped with a warning.
- Like plot_raster_path, this function expects a Bluesky Msg-style interface (command, obj, args).

Usage examples
--------------

Basic plotting and validation workflow
```python
from simulation import plot_raster_path, summarize_plan, check_limits

# Assume `plan` is an iterable of Bluesky Msg objects
# 1) Plot the raster path
fig, ax = plt.subplots()
plot_raster_path(plan, x_motor='x', y_motor='y', ax=ax, probe_size=None, lw=2)

# 2) Print a plan summary
summarize_plan(plan)

# 3) Check limits
check_limits(plan)
```

Alternative usage with the alias
```python
from simulation import print_summary

# Print the same plan summary using the alias
print_summary(plan)
```

Dependencies
------------
- matplotlib (for plotting utilities)
- bluesky (for Msg structure and print_summary_wrapper)

Edge considerations
- Ensure the plan yields at least one 'set' followed by a 'save' to form a valid trajectory; otherwise plotting may fail due to empty trajectory data.
- The plotting function assumes motor names provided exist within the plan messages; mismatches will lead to incomplete paths.
