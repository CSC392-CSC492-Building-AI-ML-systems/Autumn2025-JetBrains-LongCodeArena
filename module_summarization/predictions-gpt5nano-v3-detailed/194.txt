Pairwise metrics, Affinities and Kernels
========================================

Overview
--------
This module provides utilities for computing pairwise relationships between samples. It covers:
- Distances (pairwise metrics) between samples
- Affinities (similarities) derived from pairwise computations
- Kernels used in kernel-based learning methods

It is designed to handle dense and sparse inputs, support precomputed distance matrices, and provide robust input validation. Internal fast implementations are used for certain metrics (e.g., chi-squared, Manhattan) to improve performance.

Key utilities and their relationships
--------------------------------------
The following core functions are used to validate and prepare inputs for pairwise computations:

- _return_float_dtype(X, Y)
  - Determines an appropriate floating-point dtype for X and Y.
  - If both inputs are float32, returns float32; otherwise uses a floating dtype (float64).
  - Also coerces non-ndarray inputs to numpy arrays as needed.

- check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse='csr', force_all_finite=True, copy=False)
  - Validates and prepares X and Y for pairwise operations.
  - If Y is None or Y is X, X and Y are both converted to arrays (or checked if already arrays).
  - If Y is provided, both X and Y are checked for the same number of features.
  - Handles sparse inputs according to accept_sparse, enforces finiteness according to force_all_finite, and may copy data if requested.
  - If precomputed is True, X is treated as a distance matrix to Y; requires X.shape[1] == Y.shape[0].
  - Returns safe_X and safe_Y as numpy arrays (or pointers to the same object when appropriate).

- check_paired_arrays(X, Y)
  - Similar to check_pairwise_arrays but tailored for paired-distance metrics.
  - Ensures inputs are suitable for pairwise operations and validates shapes and dtypes as needed.

- Internal fast kernels
  - _chi2_kernel_fast, _sparse_manhattan
  - Provide optimized implementations for specific metrics when inputs are sparse or when performance is critical.

Important parameters and return values
--------------------------------------
- X, Y
  - Input data for pairwise computations. Can be dense arrays or sparse matrices (CSR/CSC formats, depending on accept_sparse).
  - If Y is None, Y points to X for self-pairwise computations.

- precomputed
  - If True, X is treated as a precomputed distance matrix to Y. This affects shape validation.

- dtype
  - Desired data type for X and Y. If None, a suitable float dtype is inferred via _return_float_dtype.

- accept_sparse
  - Allowed sparse formats. If the input is sparse and not in an allowed format, it will be converted to the first listed allowed format. True allows any format; False disallows sparse inputs.

- force_all_finite
  - Finite value constraint:
    - True: all values must be finite
    - False: allows inf/nan
    - 'allow-nan': allows only NaN/NA values; infinite values are not allowed
  - pd.NA is converted to np.nan where applicable.

- copy
  - If True, a forced copy will occur during conversion; otherwise a copy may still occur if necessary.

- Returns
  - safe_X: array-like of shape (n_samples_X, n_features)
  - safe_Y: array-like of shape (n_samples_Y, n_features) or a pointer to X if Y is None

Usage examples
--------------
Compute pairwise Euclidean distances between two sets of samples:

```python
from sklearn.metrics import pairwise_distances
import numpy as np

X = np.array([[0, 0], [1, 1], [2, 2]])
Y = np.array([[0, 1], [1, 0]])

D = pairwise_distances(X, Y, metric='euclidean')
print(D)
```

Validate inputs explicitly (useful for advanced validation workflows):

```python
from sklearn.metrics.pairwise import check_pairwise_arrays

X = [[1.0, 2.0], [3.0, 4.0]]
Y = [[5.0, 6.0]]

safe_X, safe_Y = check_pairwise_arrays(X, Y, dtype=float, accept_sparse='csr',
                                       force_all_finite=True, copy=False)
```

Notes and edge cases
--------------------
- Sparse inputs: Internal fast implementations optimize certain metrics for sparse data (e.g., _chi2_kernel_fast, _sparse_manhattan). If you rely on sparse inputs, ensure the formats align with accept_sparse.
- Precomputed distances: When precomputed is True, the function expects a distance matrix in X with shape (n_queries, n_indexed); Y is treated as the indexed data. The shape validation enforces X.shape[1] == Y.shape[0].
- Finite values: By default, values must be finite. If your data may contain NaNs or Infs, set force_all_finite appropriately (e.g., 'allow-nan') or preprocess data to handle missing values.
- pd.NA: pd.NA values are converted to np.nan as part of force_all_finite handling in newer versions.
- dtype handling: If you pass heterogeneous inputs (some float32, some float64), the resulting computations will use a common floating dtype determined by _return_float_dtype.

Related concepts
---------------
- Pairwise distances: Distances between samples, used in clustering, nearest-neighbor methods, and similarity-based algorithms.
- Affinities: Similarity measures derived from pairwise comparisons, often used in spectral methods and clustering.
- Kernels: Kernel functions mapping inputs into a feature space to enable linear methods to operate in a non-linear space.
