Developing scikit-learn estimators
===================================


Overview
--------

This module provides the infrastructure for building and using meta-estimators that extend single-output estimators to multi-output tasks. It enables developers to wrap a base estimator in order to:

- Perform multi-output regression or classification with a single wrapper
- Chain estimators so that predictions for one target can condition the next
- Reuse common fitting and validation logic across different multi-output strategies

Key classes and relationships
-----------------------------

- _MultiOutputEstimator
  - Abstract base class for multi-output meta-estimators.
  - Mixin of MetaEstimatorMixin and BaseEstimator.
  - Core parameters:
    - estimator: a base estimator with fit/predict methods
    - n_jobs: optional integer controlling parallelism
  - Provides common construction semantics and guarantees a fresh clone of the base estimator for fitting.

- MultiOutputRegressor
  - Meta-estimator that fits one regressor per target.
  - Handles multi-output regression by wrapping a single base regressor.

- MultiOutputClassifier
  - Meta-estimator that fits one classifier per target.
  - Handles multi-output classification by wrapping a single base classifier.

- ClassifierChain
  - Chain-based meta-estimator for multi-label classification.
  - Builds a sequence of binary classifiers where each classifier uses the previous predictions as additional features.

- RegressorChain
  - Chain-based meta-estimator for multi-output regression.
  - Similar to ClassifierChain but for regression targets.

- Helper functions (internal API)
  - _fit_estimator(estimator, X, y, sample_weight=None, **fit_params)
    - Clones the given estimator and fits it on X, y (including optional sample_weight and fit_params).
    - Returns the fitted estimator instance.
  - _partial_fit_estimator(estimator, X, y, classes=None, partial_fit_params=None, first_time=True)
    - Supports partial_fit for online/incremental learning.
    - First-time call clones the estimator; subsequent calls reuse the fitted instance.
    - If classes is provided, passes it to partial_fit.
  - _available_if_estimator_has(attr)
    - Returns a conditional availability check (decorator) that evaluates to True only if the sub-estimator exposes the given attribute.
    - Useful for exposing or hiding parameters based on the base estimator’s capabilities.

API details
-----------

Public constructor for the base multi-output meta-estimator:

- _MultiOutputEstimator.__init__(self, estimator, *, n_jobs=None)
  - estimator: Base estimator to wrap. Must implement fit and predict.
  - n_jobs: Optional int or None. Controls parallelism where supported by the base estimator.

Common internal helpers (private API)
- _fit_estimator(estimator, X, y, sample_weight=None, **fit_params)
  - Returns: a fitted clone of the estimator.
- _partial_fit_estimator(estimator, X, y, classes=None, partial_fit_params=None, first_time=True)
  - Returns: the (potentially updated) fitted estimator.
- _available_if_estimator_has(attr)
  - Returns: a decorator enabling conditional parameter availability based on sub-estimator attributes.

Important considerations and edge cases
---------------------------------------

- Independence and cloning
  - Each target or chain element operates on a fresh clone of the base estimator to avoid shared state across targets.

- Base estimator requirements
  - The wrapped estimator must implement fit and predict. Partial fit is supported only if the base estimator implements partial_fit.

- Parallelism
  - The n_jobs parameter enables parallelism where the base estimator supports it. Not all base estimators will parallelize; behavior depends on the underlying estimator.

- Chaining behavior
  - ClassifierChain and RegressorChain introduce a dependency chain among targets. Errors can propagate through the chain; consider ordering and potential error amplification when configuring chains.

- edge-case handling for multi-output
  - Multi-output wrappers typically manage separate instances or sequential processing per target, ensuring correct handling of sample_weight, fit_params, and per-target outputs.

Usage examples
--------------

Example 1: MultiOutputRegressor with a base regressor

```python
from sklearn.multioutput import MultiOutputRegressor
from sklearn.ensemble import RandomForestRegressor
import numpy as np

# Synthetic data: 100 samples, 4 features, 3 targets
X = np.random.randn(100, 4)
y = np.random.randn(100, 3)

base_reg = RandomForestRegressor(n_estimators=50, random_state=0)
mor = MultiOutputRegressor(base_reg, n_jobs=2)

mor.fit(X, y)
pred = mor.predict(X)
```

Example 2: ClassifierChain for multi-label classification

```python
from sklearn.multioutput import ClassifierChain
from sklearn.linear_model import LogisticRegression
import numpy as np

# Simple binary-target example: 4 samples, 2 targets
X = np.array([[0, 0], [1, 0], [0, 1], [1, 1]])
y = np.array([[0, 1], [1, 0], [0, 0], [1, 1]])

clf = ClassifierChain(LogisticRegression(max_iter=1000))
clf.fit(X, y)
pred = clf.predict(X)
```

Example 3: RegressorChain for multi-output regression

```python
from sklearn.multioutput import RegressorChain
from sklearn.ensemble import GradientBoostingRegressor
import numpy as np

# Synthetic data: 50 samples, 3 targets
X = np.random.randn(50, 5)
y = np.random.randn(50, 3)

base_reg = GradientBoostingRegressor(random_state=0)
rchain = RegressorChain(base_reg)

rchain.fit(X, y)
pred = rchain.predict(X)
```

Notes
-----

- The internal helpers are intended for use by the meta-estimators in this module and are not part of the public API for end-users.
- For advanced use, consider how partial_fit support in the base estimator interacts with multi-output wrappers, especially in streaming data scenarios.
