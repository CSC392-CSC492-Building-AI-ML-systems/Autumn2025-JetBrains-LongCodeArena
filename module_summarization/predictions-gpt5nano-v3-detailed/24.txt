w_init: Initialize a new WEST simulation

Overview
--------
The w_init command initializes a new WEST simulation. It creates the WEST HDF5 file and prepares the first iteration's segments. Initial states are constructed from basis states (specified via --bstates-from or --bstate arguments). Target states define the recycling behavior for non-equilibrium simulations, while start states specify where trajectories begin. If no basis or target states are provided, sensible defaults are used (a default basis state; and an equilibrium, sink-free simulation unless targets are supplied).

Key concepts
------------
- BasisState: Represents a starting state for new segments. Each basis state has a label, a probability, and an optional auxiliary reference. Multiple basis states can be supplied from a file or on the command line.
- TargetState: Represents a target state for recycling in non-equilibrium simulations. States can be loaded from a file or specified inline.
- Start state (sstate): Optional starting state(s) for the simulation, used to seed initial segments.
- Work Manager: Initialization runs under a WEST work manager. Master processes perform the setup; worker processes participate as configured by the WM.

Main functions and their relationships
-------------------------------------
- entry_point(): Entry point for the w_init CLI. Builds and parses the argument parser, applies WESTPA configuration, and delegates to initialize().
- initialize(tstates, tstate_file, bstates, bstate_file, sstates, sstate_file, segs_per_state, shotgun): Core function that constructs the target states, basis states, and start states, then initializes the simulation via the configured work manager.
  - Reads:
    - Target states from tstate_file and/or inline tstates.
    - Basis states from bstate_file and/or inline bstates.
    - Start states from sstate_file and/or inline sstates.
  - Creates the initial set of segments according to segs_per_state.
  - Uses the WESTPHYS/WestPA runtime configuration (westpa.rc) to obtain system, sim manager, and data manager, and to determine master/worker behavior.

Important parameters
--------------------
From entry_point / initialize:

- force (bool): Overwrite any existing simulation data.
- bstate-file / bstates-from (str): Path to file containing basis state definitions.
- bstate (str, repeatable): Basis state as "label,probability[,auxref]".
- tstate-file / tstates-from (str): Path to file containing target state definitions.
- tstate (str, repeatable): Target state as "label,pcoord0[,pcoord1[,...]]".
- sstate-file / sstates-from (str): Path to file containing start state definitions.
- sstate (str, repeatable): Start state as "label,probability[,auxref]".
- segs-per-state (int): Number of segments to initialize per basis state (default: 1).
- shotgun / no-we (bool): If set, do not run the weighted-ensemble bin/split/merge algorithm on newly-created segments.

- WM args: Additional work-manager specific arguments are added via work_managers.environment.add_wm_args(parser).

Return values
-------------
- entry_point: No return value; side effects are the creation of the WEST HDF5 file and initial segments.
- initialize: No return value; side effects are the initialization of the simulation state within the WESTPA environment.

Usage notes and edge cases
--------------------------
- Default basis state: If neither --bstates-from nor any --bstate is provided, a default basis state with probability one and label "basis" (state ID zero) will be created, and a warning will be emitted.
- Default target states: If neither tstate-file nor tstates are provided, an equilibrium simulation (no sinks) is performed.
- Inline state specification: Inline states supplied via --tstate, --sstate, and --bstate are parsed by splitting on commas to extract label, probability, and optional auxref.
- File formats: Basis, target, and start states can be loaded from their respective files via --bstates-from, --tstates-from, and --sstates-from. The exact file formats are defined by BasisState.states_from_file, TargetState.states_from_file, etc.
- Master process: Initialization logic executes on the WM master only; worker processes coordinate with the master via the work manager.

Code references (high level)
----------------------------
- entry_point():
  - Builds argument parser with options described above.
  - Integrates WESTPA runtime configuration (westpa.rc) and WM args.
  - Calls initialize(...) with parsed arguments.
- initialize(...):
  - Retrieves work manager, system driver, sim manager, and data manager from westpa.rc.
  - On master:
    - Reads target states from file and inline definitions.
    - Reads basis states from file and inline definitions.
    - Reads start states from file and inline definitions.
    - Creates the initial set of segments according to segs_per_state.
  - The actual creation of the simulation file and segments is performed as a side effect.

Usage examples
--------------
- Basic: initialize with basis and target states from files
  .. code-block:: shell

     w_init --bstates-from basis_states.txt --tstates-from target_states.txt --segs-per-state 2

- Inline definitions: basis, target, and number of segments per state
  .. code-block:: shell

     w_init --bstate "basis1,1.0" --tstate "target1,0.5" --segs-per-state 3

- Disable weighted-ensemble logic during init
  .. code-block:: shell

     w_init --bstate "basis1,1.0" --tstate "target1,0.5" --no-we

- Default behavior (no states provided): rely on defaults (default basis state; equilibrium if no targets)
  .. code-block:: shell

     w_init
