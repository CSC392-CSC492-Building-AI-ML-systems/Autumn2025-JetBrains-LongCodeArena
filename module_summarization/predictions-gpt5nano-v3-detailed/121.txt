Cradmin Menu System (crmenu)
============================

Introduction
------------
This module provides a flexible set of renderable objects for building Cradmin menus. It defines a collection of link items, expandable menu items, separators, and a toggle control, all designed to integrate with Cradmin’s rendering pipeline and the BEM (Block-Element-Modifier) naming conventions used in the templates.

Overview
--------
- The system is built around renderable objects that describe menu items and their appearance.
- Each renderable class maps to a template (via template_name) and encapsulates behavior for its portion of the menu.
- The AbstractMenuRenderable class serves as the base for building custom menus; concrete subclasses supply specific renderable types for links, separators, and buttons.
- A helper factory, get_default_expandable_menu_renderable, lets you instantiate a default expandable menu class defined in Django settings.

Key Concepts and Relationships
------------------------------
- BaseMenuLinkRenderable: A reusable renderable for link-like menu items. Handles label, URL, active state, and BEM block/element naming.
  - Subclasses:
    - NavLinkItemRenderable: Simple menu link in the main menu.
    - NavLinkButtonItemRenderable: Menu link rendered as a button (uses an extra BEM variant).
- ExpandableMenuItem: A link item that can expand to show a nested list of items. Uses a dedicated BEM block for its markup.
- ExpandableMenuSeparator: Visual separator used within an expandable menu.
- MenuToggleItemItemRenderable: A toggle control that can open/close the expandable menu; provides label and icon styling hooks.
- AbstractMenuRenderable: Core base class for building a menu. It:
  - Exposes renderable_class attributes for links, separators, and buttons.
  - Provides an interface to enumerate the contained renderables.
  - Requires a concrete link renderable class to be defined (otherwise raises ValueError).

Key Functions and Classes
-------------------------

1) get_default_expandable_menu_renderable(**kwargs)
- Purpose: Factory helper to create the default expandable menu renderable as configured in Django settings.
- Behavior:
  - If DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS is not set, returns None.
  - Otherwise imports the class path from settings, instantiates it with the provided kwargs, and returns the instance.
- Parameters:
  - kwargs: Passed to the menu class constructor (e.g., request, cradmin_instance, etc.).
- Returns:
  - An instance of the configured expandable menu renderable or None.

Usage notes:
- Requires DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS to be set to a valid class path.
- Import errors or incorrect class paths will raise exceptions at runtime.

Code example:
.. code-block:: python

    from django.conf import settings
    from django_cradmin.crmenu import get_default_expandable_menu_renderable

    # Set in settings.py:
    # DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.mymenu.MyExpandableMenuRender'

    def build_menu(request, cradmin_instance=None):
        menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin_instance)
        # Add items to menu.renderable_list or configure as needed
        return menu

2) BaseMenuLinkRenderable
- Purpose: Base renderable for menu items that behave like links.
- Key attributes:
  - label: Text label for the menu item.
  - url: Destination URL when clicked.
  - is_active: If True, the item is styled as active.
  - bem_variant_list / parent_bem_block: Controls CSS/BEM naming.
- Template: django_cradmin/crmenu/menuitem/link.django.html
- Important methods:
  - __init__(label, url, is_active=False, bem_variant_list=None, parent_bem_block=None, **kwargs)
  - get_fallback_parent_bem_block(): Returns 'adminui-page-header' by default.
  - get_parent_bem_block(): Uses explicit parent_bem_block or fallback.
  - get_bem_element(): Returns the appropriate BEM element for this link.
  - get_active_bem_variant_list(): Returns ['active'] when active.
  - get_bem_variant_list(): Includes active variant when is_active is True.

Usage notes:
- Subclasses can override template and BEM behavior.
- If no link_renderable_class is defined on an AbstractMenuRenderable, a ValueError will be raised when accessed.

3) NavLinkItemRenderable
- Subclass of BaseMenuLinkRenderable.
- Purpose: Convenience class for a standard main menu link.

Usage example:
.. code-block:: python

    link = NavLinkItemRenderable(label='Home', url='/', is_active=True)

4) NavLinkButtonItemRenderable
- Subclass of NavLinkItemRenderable.
- Purpose: A main menu link rendered as a button.
- Behavior: Overrides get_bem_variant_list to add 'button'.

Usage example:
.. code-block:: python

    button_link = NavLinkButtonItemRenderable(label='Actions', url='/actions/', is_active=False)

5) ExpandableMenuItem
- Subclass of BaseMenuLinkRenderable.
- Purpose: A menu item that can expand to reveal a nested list of items.
- Template: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
- Special methods:
  - get_fallback_parent_bem_block(): Returns 'expandable-menu'
  - get_bem_element(): Returns '{parent}__link'
  - get_list_element_bem_element(): Returns '{parent}__listitem'

Usage notes:
- Intended to contain nested items in its renderable_list.

6) ExpandableMenuSeparator
- Purpose: Visual separator to group items within an expandable menu.
- Template: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
- Parameters:
  - parent_bem_block=None: Optional BEM block; defaults to 'expandable-menu'.
- Important methods:
  - __init__(parent_bem_block=None, **kwargs)
  - get_fallback_parent_bem_block(): Returns 'expandable-menu'
  - get_parent_bem_block(): Uses explicit parent_bem_block or fallback
  - get_bem_element(): Returns '{parent}__separator'

7) MenuToggleItemItemRenderable
- Purpose: Toggle control to open/close the expandable menu in the header.
- Template: django_cradmin/crmenu/menuitem/menutoggle.django.html
- Parameters:
  - parent_bem_block=None: Optional BEM block for naming.
- Important methods:
  - __init__(parent_bem_block=None)
  - get_label(): Returns a translatable label via pgettext_lazy('cradmin default header menu toggle label', 'Menu')
  - get_fallback_parent_bem_block(): Returns 'adminui-page-header'
  - get_parent_bem_block(): Uses explicit parent or fallback
  - get_bem_element(): Returns '{parent}__navmenutoggle'
  - get_label_css_class(): Returns '{parent}__navmenutoggle-label'
  - get_icon_css_class(): Returns '{parent}__navmenutoggle-icon'

8) AbstractMenuRenderable
- Core base class for rendering a menu.
- Key attributes:
  - link_renderable_class: The class used for rendering links (must be defined by concrete subclasses).
  - separator_renderable_class: The class used for separators.
  - button_renderable_class: The class used for button items.
- Initialization:
  - __init__(request, cradmin_instance=None): Stores request/cradmin_instance and initializes an empty renderable_list.
- Core methods:
  - iter_renderables(): Returns an iterator over renderable_list.
  - get_link_renderable_class(): Returns the configured link renderable class; raises ValueError if not defined.
  - get_separator_renderable_class(): Returns the configured separator renderable class.
- Notes:
  - Concrete menus should set the appropriate renderable classes to enable full rendering.
  - If a required renderable class is missing, attempting to render the menu will raise an error.

Usage considerations and edge cases
-----------------------------------
- Configuration dependency: get_default_expandable_menu_renderable relies on a Django setting (DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS). If not set, the function returns None.
- Classpath validity: Importing the configured class path must point to a valid class; invalid paths will raise ImportError/AttributeError at runtime.
- Required renderable classes: AbstractMenuRenderable enforces that a concrete link_renderable_class is defined. If not, get_link_renderable_class raises ValueError.
- BEM naming: The renderables use a Block-Element-Modifier naming convention. If you override parent blocks, ensure consistency with your CSS (the default fallbacks are designed to work with Cradmin’s admin UI CSS).
- Translation: Menu toggle label uses pgettext_lazy for translation; ensure your translation bundles include the provided context string if you override labels.

Usage Examples
--------------

1) Simple link item
.. code-block:: python

    from django_cradmin.crmenu import NavLinkItemRenderable
    link = NavLinkItemRenderable(label='Dashboard', url='/admin/dashboard/', is_active=True)

2) Button-style link item
.. code-block:: python

    from django_cradmin.crmenu import NavLinkButtonItemRenderable
    btn_link = NavLinkButtonItemRenderable(label='Create', url='/admin/create/', is_active=False)

3) Expandable menu item (with nested items)
.. code-block:: python

    from django_cradmin.crmenu import ExpandableMenuItem
    parent_item = ExpandableMenuItem(label='Settings', url='/admin/settings/', is_active=False)
    child_item = NavLinkItemRenderable(label='User management', url='/admin/settings/users/', is_active=False)
    # Assuming your menu renderer supports nesting, you would attach child_item to parent_item.renderable_list
    parent_item.renderable_list.append(child_item)

4) Configure default expandable menu class
.. code-block:: python

    # settings.py
    DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.menus.DefaultExpandableMenuRender'

5) Build a menu using the factory
.. code-block:: python

    from django_cradmin.crmenu import get_default_expandable_menu_renderable

    def build_menu(request, cradmin_instance=None):
        menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin_instance)
        # Build your menu by appending items to menu.renderable_list
        home = NavLinkItemRenderable(label='Home', url='/', is_active=True)
        menu.renderable_list.append(home)
        return menu
