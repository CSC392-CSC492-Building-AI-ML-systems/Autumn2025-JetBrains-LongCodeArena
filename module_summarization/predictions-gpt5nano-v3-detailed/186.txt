Imputation of missing values
============================

Overview
--------
This module provides transformers for imputing missing values in arrays or matrices. The public entry point is SimpleImputer, which can fill in missing values per feature using a univariate strategy. It supports numerical strategies (mean, median, most_frequent) as well as a constant fill value. An optional missing-value indicator can be appended to the transformed data to indicate which entries were imputed.

Key concepts
-------------
- Missing values placeholder: missing_values (default np.nan). For pandas data with nullable dtypes, pd.NA can be used as missing_values.
- Strategies:
  - mean: replace missing values with the mean of each column (numeric data only).
  - median: replace missing values with the median of each column (numeric data only).
  - most_frequent: replace missing values with the most frequent value per column.
  - constant: replace missing values with a user-specified fill_value.
- add_indicator: when True, appends a binary indicator feature per original feature indicating where values were imputed.
- keep_empty_features: when True, keeps features that are entirely missing rather than dropping them during transformation.

Public API
----------
- SimpleImputer
  - Purpose: Univariate imputer for completing missing values with simple strategies.
  - Inherits from: _BaseImputer (internal base class) and TransformerMixin, BaseEstimator.
  - Primary responsibilities: validate inputs, compute per-feature statistics during fit, impute missing values during transform, optionally attach a missing-value indicator.

Parameters
----------
- missing_values: int, float, str, np.nan, None, or pandas.NA
  - Placeholder for missing values. All occurrences are imputed.
  - For pandas nullable dtypes, pd.NA is supported.
- strategy: {'mean', 'median', 'most_frequent', 'constant'}, default='mean'
  - Imputation strategy.
- fill_value: str or numeric value, default=None
  - Used when strategy='constant'. If None, defaults to 0 for numeric data and "missing_value" for strings/objects.
- copy: bool, default=True
  - If True, a copy of X is created; otherwise imputation may be done in-place where possible.
- add_indicator: bool, default=False
  - If True, appends a MissingIndicator to the transformed data to indicate imputed positions.
- keep_empty_features: bool, default=False
  - If True, keeps features that are entirely missing instead of removing them.

Return values
-------------
- imputed_X: array-like of shape (n_samples, n_features) with missing values replaced according to the selected strategy.
- If add_indicator is True, the result is augmented with additional indicator features corresponding to imputed positions.

Usage examples
--------------
Basic mean imputation on numeric data
```python
import numpy as np
from sklearn.impute import SimpleImputer

X = np.array([[1.0, 2.0], [np.nan, 3.0], [7.0, np.nan]])
imputer = SimpleImputer(strategy="mean")
X_imputed = imputer.fit_transform(X)
print(X_imputed)
```

Constant value imputation (e.g., for categorical or mixed data)
```python
import numpy as np
from sklearn.impute import SimpleImputer

X = np.array([["a", "b"], [np.nan, "c"], ["d", np.nan]], dtype=object)
imputer = SimpleImputer(strategy="constant", fill_value="missing")
X_imputed = imputer.fit_transform(X)
print(X_imputed)
```

Imputation with missing-value indicator
```python
import numpy as np
from sklearn.impute import SimpleImputer

X = np.array([[1.0, 2.0], [np.nan, 3.0], [7.0, np.nan]])
imputer = SimpleImputer(strategy="mean", add_indicator=True)
X_imputed = imputer.fit_transform(X)
print(X_imputed.shape)  # (n_samples, n_features + n_indicators)
```

Important considerations and edge cases
---------------------------------------
- Data types: mean/median imputation requires numeric data. non-numeric data must use most_frequent or constant.
- Missing values placeholder: for numeric data, both np.nan and pd.NA (where supported) are handled; for string/object data, strategy='constant' or 'most_frequent' is typically used.
- All-missing features: behavior depends on strategy and parameters; the helper _most_frequent includes logic for special handling when necessary.
- Indicator handling: when add_indicator is True, the transformed data includes additional features representing the missing-value indicator; internal MissingIndicator is fit during _fit_indicator and used during _transform_indicator.
- In-place vs copy: by default imputation creates a copy; set copy=False to attempt in-place operation where possible.
- Interaction with sparse data: internal utilities support sparse data; concatenation of imputed data with indicator handles dense and sparse cases.

How it works (implementation overview)
--------------------------------------
- Input validation:
  - _check_inputs_dtype ensures that X and missing_values have compatible numeric types when required.
- Fitting:
  - For mean/median: compute per-feature statistics from non-missing values.
  - For most_frequent: determine per-feature most frequent value (with deterministic tie-breaking).
  - For constant: store fill_value to use during transform.
  - If add_indicator is True, instantiate and fit a MissingIndicator to generate an indicator mask.
- Transform:
  - Replace missing values per feature using the fitted statistics or fill_value.
  - If add_indicator is True, concatenate the indicator mask to the imputed data.
  - If keep_empty_features is True, preserve features that might be empty after transformation.

Related components
------------------
- _BaseImputer: Internal base class providing common logic for imputers, including support for add_indicator and feature-name handling.
- MissingIndicator: Used when add_indicator is True to create a mask indicating imputed positions.
- _most_frequent: Helper function implementing the logic to select the most frequent value, with special handling for ties and empty arrays.

Notes
-----
- This documentation reflects the behavior implemented in the module, including support for pandas NA, numeric dtype handling, and indicator concatenation.
- Public API centers on SimpleImputer; _BaseImputer is an internal helper class.
