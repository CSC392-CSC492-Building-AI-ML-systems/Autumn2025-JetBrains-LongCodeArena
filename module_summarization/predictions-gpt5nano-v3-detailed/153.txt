# geometry.subpix.rst

Geometric image transformations performed under a Pinhole camera model. This module provides a batched PinholeCamera abstraction that encapsulates camera intrinsics and extrinsics and exposes convenient accessors for common subpixel and geometric quantities. It is designed to work efficiently with PyTorch tensors to leverage batch processing.

## Overview

- Represents a pinhole camera with full 4x4 intrinsics and 4x4 extrinsics (pose) in batch form.
- Provides convenient properties to access focal lengths, principal point, translation, and derived matrices.
- Supports batched shapes for users who operate on multiple cameras or multiple parameterizations per batch.
- Intended as a lightweight container to drive higher-level geometric transforms (e.g., projection, unprojection, and coordinate frame conversions) in conjunction with Kornia utilities.

## Class: PinholeCamera

The PinholeCamera class models the camera using:
- intrinsics: (B, 4, 4) or (B, N, 4, 4)
- extrinsics: (B, 4, 4) or (B, N, 4, 4)
- height: (B)
- width: (B)

It stores and validates these inputs and exposes convenient properties for common camera parameters.

Key concepts:
- Batch-aware design: operations are performed on batched tensors to maximize PyTorch parallelism.
- Separation of concerns: intrinsics describe the camera calibration; extrinsics describe the pose; height/width describe image dimensions.
- Convenience accessors: fx, fy, cx, cy, translation components (tx, ty, tz), and derived matrices (rt_matrix, camera_matrix, rotation_matrix, translation_vector).

## Constructor

Parameters:
- intrinsics: tensor of shape (B, 4, 4) or (B, N, 4, 4)
- extrinsics: tensor of shape (B, 4, 4) or (B, N, 4, 4)
- height: tensor of shape (B)
- width: tensor of shape (B)

Notes:
- The class validates shapes and ensures batch compatibility.
- Height and width must be 1D tensors with length B.

## Properties

- intrinsics: (B, 4, 4)
- extrinsics: (B, 4, 4)
- batch_size: int (B)
- fx: (B)
  - Focal length in x direction from intrinsics[*, 0, 0]
- fy: (B)
  - Focal length in y direction from intrinsics[*, 1, 1]
- cx: (B)
  - Principal point x-coordinate intrinsics[*, 0, 2]
- cy: (B)
  - Principal point y-coordinate intrinsics[*, 1, 2]
- tx: (B)
  - Translation x component from extrinsics[*, 0, -1]
- ty: (B)
  - Translation y component from extrinsics[*, 1, -1]
- tz: (B)
  - Translation z component from extrinsics[*, 2, -1]
- rt_matrix: (B, 3, 4)
  - Rotation-translation matrix derived from extrinsics
- camera_matrix: (B, 3, 3)
  - 3x3 camera intrinsics (upper-left submatrix)
- rotation_matrix: (B, 3, 3)
  - 3x3 rotation extracted from extrinsics
- translation_vector: (B, 3, 1)
  - Translation vector extracted from extrinsics

Setters:
- tx, ty, tz allow in-place updates of the translation components in the extrinsics.

## Usage

```python
import torch
from geometry.subpix import PinholeCamera  # hypothetical import path

# Example batch size
B = 2

# Example intrinsics and extrinsics (identity-like for simplicity)
intrinsics = torch.eye(4).unsqueeze(0).repeat(B, 1, 1)  # (B, 4, 4)
extrinsics = torch.eye(4).unsqueeze(0).repeat(B, 1, 1)  # (B, 4, 4)

# Image dimensions per batch
height = torch.tensor([480, 720], dtype=intrinsics.dtype)
width = torch.tensor([640, 1280], dtype=intrinsics.dtype)

cam = PinholeCamera(intrinsics, extrinsics, height, width)

# Accessing properties
print("Batch size:", cam.batch_size)
print("fx:", cam.fx)
print("fy:", cam.fy)
print("cx:", cam.cx)
print("cy:", cam.cy)

# Accessing derived matrices
print("Camera matrix shape:", cam.camera_matrix.shape)        # (B, 3, 3)
print("Rotation matrix shape:", cam.rotation_matrix.shape)   # (B, 3, 3)
print("Translation vector shape:", cam.translation_vector.shape)  # (B, 3, 1)

# Update translations (example)
new_tx = torch.tensor([0.0, 0.5])
cam.tx = new_tx  # updates extrinsics[..., 0, -1]
```

## Parameters and Return Values (in detail)

- intrinsics (Tensor): shape (B, 4, 4) or (B, N, 4, 4)
  - Full camera calibration matrix.

- extrinsics (Tensor): shape (B, 4, 4) or (B, N, 4, 4)
  - Full rotation-translation matrix (pose).

- height (Tensor): shape (B,)
  - Image height per batch.

- width (Tensor): shape (B,)
  - Image width per batch.

- intrinsics property: Tensor shape (B, 4, 4)
- extrinsics property: Tensor shape (B, 4, 4)
- batch_size: int
- fx, fy: Tensor shape (B,)
- cx, cy: Tensor shape (B,)
- tx, ty, tz: Tensor shape (B,)
- rt_matrix: Tensor shape (B, 3, 4)
- camera_matrix: Tensor shape (B, 3, 3)
- rotation_matrix: Tensor shape (B, 3, 3)
- translation_vector: Tensor shape (B, 3, 1)

## Validation and Edge Cases

- Shapes:
  - intrinsics/extrinsics must be (B, 4, 4) or (B, N, 4, 4)
  - height/width must be 1D with length B
- Mismatched batch sizes across inputs will raise errors during validation.
- The internal validation checks may reflect the intended design (e.g., 3D vs 4D intrinsics) and can be sensitive to shape conventions. Ensure inputs conform to the documented shapes to avoid runtime errors.
- The translation components (tx, ty, tz) are mutable in-place via setters; changes affect the corresponding extrinsics.

## Related Utilities

- inverse_transformation
- transform_points

These utilities support common geometric operations (e.g., inverting pose, transforming points between coordinate frames) and can be used in conjunction with PinholeCamera for end-to-end geometric pipelines.

## Edge Considerations

- This module emphasizes batched operation and compatibility with PyTorch's autograd for differentiable pipelines.
- When using 4D shapes (B, N, 4, 4), each batch element can carry multiple intrinsic/extrinsic configurations, enabling per-sample parameterization within a batch.
- As this is a data container with properties and simple setters, ensure that any downstream computations re-use the same batched shapes to preserve performance and correctness.
