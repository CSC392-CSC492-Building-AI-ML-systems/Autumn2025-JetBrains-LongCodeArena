Discretemod
===========

Overview
--------
This module implements zero-inflated count models designed to handle data with excess zeros. It provides three public classes:

- ZeroInflatedPoisson
- ZeroInflatedGeneralizedPoisson
- ZeroInflatedNegativeBinomialP

Each model combines a zero-inflation component with a main count model (Poisson, Generalized Poisson, or Negative Binomial P) to capture both the probability of excess zeros and the distribution of positive counts.

Public API
----------
__all__ = ["ZeroInflatedPoisson", "ZeroInflatedGeneralizedPoisson",
           "ZeroInflatedNegativeBinomialP"]

Classes
-------
GenericZeroInflated (base class)
- Purpose: Internal base class used by the concrete zero-inflated count models. It implements the shared logic for mixing a zero-inflation process with a main count model.
- Inheritance: CountModel
- Key responsibilities:
  - Manage endog (response), exog (main model design), and exog_infl (zero-inflation design).
  - Configure the inflation model (Logit or Probit) and its parameters.
  - Provide log-likelihood calculations for the mixed model (loglike, loglikeobs).
  - Support parameter estimation via fit.

ZeroInflatedPoisson
- Purpose: Zero-inflated Poisson model, mixing a Poisson main model with a zero-inflation component.
- Notes: Public alias intended for users needing a Poisson-based zero-inflated count model.

ZeroInflatedGeneralizedPoisson
- Purpose: Zero-inflated Generalized Poisson model, mixing a Generalized Poisson main model with a zero-inflation component.
- Notes: Public alias for users requiring generalized Poisson counts with excess zeros.

ZeroInflatedNegativeBinomialP
- Purpose: Zero-inflated Negative Binomial P model, mixing a NB-P main model with a zero-inflation component.
- Notes: Public alias for users needing NB-P counts with zero inflation.

Key Parameters
--------------
Zero-inflated models share the following parameters (via GenericZeroInflated and the main model):

- endog: array-like
  Endogenous response variable (counts).

- exog: array-like
  Exogenous design matrix for the main count model.

- exog_infl: array-like or None
  Exogenous design for the zero-inflation (mixture) model. If None, a constant term is used.

- offset: array-like
  Offset added to the linear predictor with coefficient 1.

- exposure: array-like
  Log(exposure) is added to the linear predictor with coefficient 1.

- inflation: string, 'logit' or 'probit'
  The model for the zero inflation (mixing probability). Defaults to 'logit'.
  - 'logit': Logistic regression for the inflation probability.
  - 'probit': Probit regression for the inflation probability.

- Missing handling: 'none' by default
  Controls how missing observations are treated.

- Additional internal attributes (used internally):
  - k_inflate: number of inflate parameters (from exog_infl)
  - k_extra: same as k_inflate (alias)
  - model_infl: underlying binary model (Logit or Probit) used for the inflation process
  - _init_keys, _null_drop_keys: internal bookkeeping for parameter handling

Important Methods
-----------------
loglike(params)
- Returns the total log-likelihood of the mixed zero-inflated model given parameter vector params.

loglikeobs(params)
- Returns the per-observation log-likelihoods for the mixed model.

fit(start_params=None, method='bfgs', maxiter=35, full_output=1, disp=1, callback=None, cov_type='nonrobust', cov_kwds=None, use_t=None, **kwargs)
- Estimate model parameters.
- If start_params is None, a sensible default starting point is computed.
- Returns a results object (consistent with other Statsmodels models).

Notes and Formulae
------------------
- The log-likelihood combines the zero-inflation probability w_i with the main model likelihood L_main:
  log L_i = log(w_i + (1 - w_i) * P_main(y_i)) for y_i = 0
  log L_i = log(1 - w_i) + L_main(y_i) for y_i > 0
  where P_main is the pdf of the main count model and L_main is its log-likelihood.
- Inflation probability w_i is derived from the inflation model (Logit or Probit) applied to exog_infl.
- If exog_infl is None, a constant inflation is assumed (k_inflate = 1).
- If the lengths of exog and exog_infl do not match, a ValueError is raised (missing handling is not supported).
- Inflation must be either 'logit' or 'probit'; otherwise a ValueError is raised.
- The implementation uses the underlying main model’s loglikeobs method to obtain L_main for each observation.

Usage Examples
--------------

Basic usage with Zero-Inflated Poisson (logit inflation)
```python
from discretemod import ZeroInflatedPoisson
import numpy as np

# Example data (y: counts, X: main model design, X_infl: inflation design)
y = np.array([0, 1, 0, 2, 0, 3, 0])
X = np.ones((len(y), 2))  # example main model design (with intercept)
X[:, 1] = np.random.randn(len(y))
X_infl = np.ones((len(y), 1))  # inflation design (intercept only)

model = ZeroInflatedPoisson(endog=y, exog=X, exog_infl=X_infl, inflation='logit')
res = model.fit()
print(res.summary())
```

Usage with Probit inflation
```python
from discretemod import ZeroInflatedGeneralizedPoisson

# y, X, X_infl as above
model = ZeroInflatedGeneralizedPoisson(endog=y, exog=X, exog_infl=X_infl, inflation='probit')
res = model.fit()
print(res.summary())
```

Notes
- The zero-inflated design (exog_infl) can be used to model factors that influence the probability of an observation being a structural zero.
- For small datasets or borderline cases, consider adjusting maxiter or providing a good starting parameter vector start_params to aid convergence.
- This module relies on Statsmodels’ wrapper and result classes; the returned results object exposes standard attributes and methods (e.g., params, bse, pvalues, summary()) consistent with other discrete models.

Edge Cases and Considerations
-----------------------------
- exog and exog_infl must have the same number of observations; otherwise, a ValueError is raised.
- Missing data handling is not supported for exog_infl alignment; ensure data are prepared accordingly.
- Inflation type must be 'logit' or 'probit'. Any other value triggers a ValueError.
- If exog_infl is None, a single constant inflation parameter is used (k_inflate = 1).
