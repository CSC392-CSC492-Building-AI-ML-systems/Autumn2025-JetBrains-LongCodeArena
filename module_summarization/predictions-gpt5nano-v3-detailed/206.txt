Support Vector Machines (svm)
=============================

Introduction
------------
This module provides a scalable implementation of linear Support Vector Machines (SVM) for classification, backed by the liblinear library through Cython bindings. It focuses on linear kernels and is suited for large-scale datasets where traditional kernel SVMs would be impractical. The primary estimator exposed is LinearSVC, which offers flexible penalties, loss functions, and multi-class strategies.

API overview
------------
Key components
- LinearSVC: Main estimator for linear SVM classification. Wraps liblinear functionality and exposes a Pythonic API compatible with the project’s estimator conventions.
- Internal helpers: Low-level Cython/C library bindings (e.g., problem, model, parameter, and liblinear helpers) are used under the hood to configure problems, train models, and extract results.

Relationships
- LinearSVC relies on liblinear’s primal/dual optimization formulations.
- Multi-class handling is performed via one-vs-rest (ovr) or crammer-singer strategies, controlled by the multi_class parameter.
- Internals expose options to select penalty types, loss functions, and duality via the _validate_dual_parameter helper.

LinearSVC
---------
LinearSVC implements linear (kernel-free) SVM classification using liblinear. It mirrors the behavior of a linear SVC model but is optimized for large-scale problems and provides fine-grained control over the optimization formulation.

Key concepts
- Kernel: Linear only. This implementation does not provide non-linear kernels.
- Formulation: Supports both primal and dual optimization problems. The dual choice is automatically determined by the solver unless explicitly set.
- Multi-class: Supports ovR and crammer_singer strategies.

Parameters
----------
- penalty : {'l1', 'l2'}, default='l2'
  Norm used in the regularization term. 'l2' is standard; 'l1' yields sparse coef_ (note: not all combinations of penalty and loss are supported).
- loss : {'hinge', 'squared_hinge'}, default='squared_hinge'
  Loss function to optimize. 'hinge' is the standard SVM loss; 'squared_hinge' is its squared variant.
- dual : "auto" or bool, default=True
  Selects between solving the dual or primal problem. 'auto' chooses based on data shape and configuration; setting explicitly can impact performance.
- tol : float, default=1e-4
  Tolerance for stopping criteria.
- C : float, default=1.0
  Regularization strength; inversely proportional to C.
- multi_class : {'ovr', 'crammer_singer'}, default='ovr'
  Strategy for multi-class classification.
- fit_intercept : bool, default=True
  Whether to fit an intercept term.
- intercept_scaling : float, default=1.0
  If fit_intercept is True, the intercept is scaled by this factor (intercept term is treated as a synthetic feature with this value).

Notes
------
- The combination penalty='l1' with loss='hinge' is not supported.
- The duality and solver choice depend on data shape (n_samples, n_features), loss, and multi_class settings.
- Internally, the estimator uses C bindings to configure a liblinear problem, train the model, and extract coefficients and intercept.

Return values and attributes
----------------------------
- fit(X, y) -> self
  Trains the model on X and y.
- predict(X) -> ndarray of shape (n_samples,)
  Predicted class labels for X.
- decision_function(X) -> ndarray of shape (n_samples,) or (n_samples, n_classes)
  Confidence scores or distance to the separating hyperplane.
- coef_ : ndarray of shape (n_features, n_classes) or (n_features,)
  Weight vector(s) for the features.
- intercept_ or bias_ : float
  Intercept term for the decision function.
- n_iter_ : int
  Number of iterations performed during training (if available from the underlying solver).

Usage examples
--------------
Basic binary classification
```python
import numpy as np
from your_svm_module import LinearSVC

# Simple linearly separable data
X = np.array([[0.0, 0.0],
              [1.0, 1.0],
              [2.0, 2.0],
              [3.0, 3.0]])
y = np.array([0, 0, 1, 1])

model = LinearSVC(penalty='l2', loss='squared_hinge', C=1.0)
model.fit(X, y)

pred = model.predict(np.array([[0.5, 0.5],
                               [2.5, 2.5]]))
print(pred)
```

Multi-class classification (ovr)
```python
from sklearn.datasets import load_iris
import numpy as np
from your_svm_module import LinearSVC

X, y = load_iris(return_X_y=True)

model = LinearSVC(multi_class='ovr', penalty='l2', loss='squared_hinge', C=1.0)
model.fit(X, y)

pred = model.predict(X[:10])
print(pred)
```

Internal implementation overview (for developers)
-----------------------------------------------
- The Python API delegates to Cython-wrapped liblinear routines.
- Parameter and problem objects are constructed via set_parameter and csr_set_problem (or set_problem) to configure the optimization task.
- The train function executes the optimization (nogil region) and returns a model object from which coefficients and intercepts are extracted.
- Auxiliary helpers provide parameter validation (e.g., _validate_dual_parameter) to ensure compatible configurations.

Important considerations and edge cases
---------------------------------------
- Large feature spaces: LinearSVC is designed for scalability, but the choice of penalty, loss, and dual formulation can impact memory usage and training time.
- Multi-class strategy: OvR is faster but may be less optimal in some settings; crammer_singer can be more accurate but is more computationally intensive.
- Regularization and loss compatibility: Be aware of the constraints (e.g., l1 with hinge is not supported) when configuring parameters.
- Input data: Accepts dense and sparse inputs; ensure features are properly preprocessed (e.g., standardized if needed for certain loss/penalty combinations).

Edge cases
- Binary vs multi-class behavior: Multi-class handling depends on the multi_class parameter.
- Non-finite values: Ensure input data contains finite values; NaNs or infs should be handled/cleaned prior to training.
- Convergence: If the solver fails to converge, consider adjusting tol, C, or using a different loss/penalty combination.

Reference and further reading
-----------------------------
- liblinear documentation for problem, model, and parameter interfaces.
- The _validate_dual_parameter helper for dual/primal feasibility checks.
- Corresponding Cython wrappers and their documentation for available options and expected input types.

End-to-end example pipeline
----------------------------
- Prepare data (X, y) with appropriate preprocessing.
- Instantiate LinearSVC with desired configuration.
- Call fit(X, y) and then use predict or decision_function as needed.
- Access model attributes such as coef_ and intercept_ for inspection or downstream processing.
