Probability calibration
======================

Overview
--------
This module provides facilities to calibrate the predicted probabilities of a classifier. Calibration adjusts the raw scores produced by a base estimator to yield better probability estimates, enabling more reliable probabilistic interpretation and decision making. Two calibration strategies are supported:

- Sigmoid calibration (Platt scaling) via a logistic regression model.
- Isotonic calibration, a non-parametric approach.

CalibratedClassifierCV implements cross-validated calibration. It can:
- Train a fresh copy of the base estimator on training folds and calibrate on corresponding test folds, then average predictions across folds (ensemble mode).
- Use cross-validated predictions on training data to calibrate, while training the base estimator on all data for final prediction (non-ensemble mode).
- Support the cv="prefit" mode where the user supplies a pre-fitted estimator and all data are used for calibration.

The calibration uses the estimator’s decision_function when available; otherwise it uses predict_proba.

Key class
---------
CalibratedClassifierCV
- Inherits from: ClassifierMixin, MetaEstimatorMixin, BaseEstimator
- Purpose: Provide probability calibration for a base classifier using isotonic regression or sigmoid calibration with cross-validation.

Parameters
----------
estimator : estimator instance, default=None
    The classifier whose probability outputs are to be calibrated. If None, a
    default estimator (LinearSVC) is used.

method : {'sigmoid', 'isotonic'}, default='sigmoid'
    Calibration method:
    - 'sigmoid': Platt scaling (logistic regression on top of the classifier outputs)
    - 'isotonic': Non-parametric isotonic regression

cv : int, cross-validation generator, iterable or "prefit", default=None
    Cross-validation strategy to use for obtaining calibration data.
    - None: use 5-fold cross-validation by default
    - int: number of folds (StratifiedKFold for binary/multiclass)
    - CV splitter: any compatible splitter
    - iterable: yields train/test splits as arrays of indices
    - "prefit": estimator is already fitted; no cross-validation is performed for calibration

n_jobs : int, default=None
    Number of jobs to run in parallel. Parallelism occurs only when cv != "prefit".

ensemble : bool, default=True
    If True, train an estimator/calibrator pair for each cv split and average
    their predicted probabilities at prediction time.
    If False, calibrate using cross_val_predict predictions and train the final
    estimator on all data.

base_estimator : estimator instance
    Deprecated alias for estimator. Use estimator instead.
    Deprecated since 1.2; will be removed in a future release.

Attributes
----------
classes_ : ndarray of shape (n_classes,)
    The class labels.

n_features_in_ : int
    Number of features seen during fit (if the base estimator exposes this).

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features seen during fit (if the base estimator exposes this).

calibrated_classifiers_ : list
    List of calibrated classifiers (one per cross-validation fold when ensemble
    is True, or a single classifier when ensemble is False or cv="prefit").

Usage and behavior
------------------
- Calibration target: The calibration is performed on the probability estimates
  produced by the base estimator. If the estimator exposes a decision_function,
  those values are calibrated. Otherwise, predict_proba outputs are used.

- Cross-validation with ensemble:
  - For each fold, fit a copy of the base estimator on the training portion and
    calibrate using the test portion.
  - Predictions are averaged across folds for final probabilities.

- Cross-validation without ensemble (cv-based calibration with cross_val_predict):
  - Use cross_val_predict to obtain unbiased predictions for calibration.
  - Train the final base estimator on all data for actual predictions.

- Prefit mode (cv="prefit"):
  - The provided estimator is assumed pre-fitted.
  - All data are used for calibration; no internal cross-validation is performed.

Edge cases and considerations
-----------------------------
- Isotonic calibration can overfit with small calibration samples (especially << 1000 samples).
- For multiclass problems, calibration typically applies in a one-vs-rest fashion; the interface supports multiclass targets.
- When using prefits, ensure that the data used for calibration do not overlap with any data used for fitting the estimator if you intend unbiased evaluation.

Examples
--------

Basic usage with sigmoid calibration (default)
```python
from sklearn.datasets import load_iris
from sklearn.svm import LinearSVC
from sklearn.calibration import CalibratedClassifierCV

# Load data
X, y = load_iris(return_X_y=True)

# Calibrate probabilities with sigmoid (Platt scaling) using 5-fold CV
ccvc = CalibratedClassifierCV(
    estimator=LinearSVC(),
    method='sigmoid',
    cv=5
)
ccvc.fit(X, y)

# Predict calibrated probabilities
probas = ccvc.predict_proba(X[:5])
print(probas)
```

Isotonic calibration
```python
ccvc_iso = CalibratedClassifierCV(
    estimator=LinearSVC(),
    method='isotonic',
    cv=5
)
ccvc_iso.fit(X, y)
probas_iso = ccvc_iso.predict_proba(X[:5])
```

Prefit calibration with a pre-fitted estimator
```python
# Pre-fit the base estimator on the full dataset (user-supplied data)
svc = LinearSVC()
svc.fit(X, y)

# Use the pre-fitted estimator; no internal cross-validation for calibration
cc_prefit = CalibratedClassifierCV(
    estimator=svc,
    method='sigmoid',
    cv='prefit'
)
cc_prefit.fit(X, y)
probas_prefit = cc_prefit.predict_proba(X[:5])
```
