GILDAS CLASS file reader
=========================

Overview
--------
This module provides foundational utilities and constants used by the GILDAS CLASS file reader. It is designed to interpret CLASS binary headers and records, and to facilitate conversion of a CLASS file into a pyspeckit ObsBlock. The implementation emphasizes compatibility across endianness/byte-order variants and Python 2–3, with progress-bar support that gracefully degrades if the optional dependency is unavailable.

Public API
----------
- print_timing(func)
  - A decorator that measures and logs the execution time of the wrapped function.
  - Usage:
    @print_timing
    def my_func(...):
        ...
- ensure_bytes(string)
  - Converts a given string to its byte representation, handling Python 2/3 differences.
  - Returns bytes on Python 3; returns the string representation on Python 2.
  - Usage:
    key = ensure_bytes('1A  ')

Constants and helper data structures
------------------------------------
The module defines several dictionaries and constants to support parsing CLASS files. These are intended to be used by the actual parser (not shown in the excerpt) to interpret header fields, determine data types, and map codes to human-readable labels.

- filetype_dict
  - Maps CLASS file type codes to human-readable type names (e.g., '1A  ' -> 'Multiple_IEEE').
  - Keys are provided as strings and are expanded to include byte-string keys for Python 3 compatibility via ensure_bytes.
  - Example:
    filetype_dict[b'9A  '] -> 'Single_IEEE'
- fileversion_dict
  - Maps CLASS file version codes to version identifiers (e.g., '1A  ' -> 'v1', '2A  ' -> 'v2').
  - Also extended with byte-key variants.
  - Example:
    fileversion_dict[b'2A  '] -> 'v2'
- record_lengths
  - Record lengths for different CLASS file formats (e.g., '1A' -> 512 bytes, '2A' -> 4096 bytes).
  - Keys may be used as strings or bytes after ensure_bytes expansion.
  - Example:
    record_lengths.get(b'1A') -> 512
- header_id_numbers
  - Maps internal header section codes to human-readable section names (e.g., 0 -> 'USER CODE', -2 -> 'GENERAL', -4 -> 'SPECTRO').
- header_id_lengths
  - Specifies fixed lengths for certain header sections. Some sections are variable length (None) or have special handling.
  - Example:
    header_id_lengths[-2] -> 9
- filedescv2_nw1
  - A constant used in the CLASS v2 description (value is 14 in this module).
- keys_lengths
  - Detailed field definitions for header blocks, organized by logical section names (e.g., 'unknown', 'COMMENT', 'GENERAL').
  - Each entry is a tuple:
    (NAME, LENGTH, TYPE)
  - Examples:
    - 'NUM', 1, 'int32'
    - 'TELES', 3, '|S12'
    - 'UT', 2, 'float64'
  - The mapping helps drive a generic parser for reading binary records into structured data.
- Unknown/auxiliary
  - The module includes large commented sections mirroring the Fortran structure from GILDAS/classic_mod.f90, which informs the field layout used by the parser.

Important considerations and edge cases
---------------------------------------
- Byte vs string keys: The dictionaries are initialized with string keys but are augmented to include byte-key variants via ensure_bytes. This allows flexible access using either bytes or strings, accommodating different I/O paths.
- Endianness and data type mapping: The module encodes multiple possible endianness and data representations (e.g., 'Multiple_IEEE', 'Single_Vax', etc.). The actual parser should select the appropriate interpretation based on the CLASS file header.
- Variable length fields: Some header sections have variable lengths (None or other constraints). Parsing logic must handle these gracefully, using header_id_lengths as guidance.
- Optional progress bar: A ProgressBar is imported if available; otherwise a no-op fallback is used. Parsing progress will not fail if the optional dependency is absent.
- Python 2/3 compatibility: The code includes compatibility shims (via six) to support both Python 2 and Python 3 environments.
