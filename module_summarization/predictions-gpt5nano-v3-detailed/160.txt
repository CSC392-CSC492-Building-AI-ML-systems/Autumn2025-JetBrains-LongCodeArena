Compatibility Policy
====================

Introduction
------------
This document defines the Compatibility Policy for the npyrandom module, outlining how we maintain binary and API compatibility across platforms, Python and NumPy versions, and build environments. It covers ABI/API stability, platform-specific considerations, build-time constraints, and testing expectations to ensure reliable integration for downstream projects.

Scope
-----
- Applies to the C core sources bundled in src/distributions and the Python-facing extensions under numpy/random.
- Covers static library construction (npyrandom) and the corresponding Cython/Numpy extension modules (numpy.random/*).
- Addresses build-time flags, platform quirks, and runtime expectations that affect compatibility.

Key Concepts and Relationships
------------------------------
- npyrandom_lib: Static library built from core C sources (e.g., logfactorial.c, distributions.c, random_mvhg_count.c, random_mvhg_marginals.c, random_hypergeometric.c). Linked by the Python extensions via link_with.
- random_pyx_sources: List of Python extension modules implemented with Cython, each built against npyrandom_lib and possibly other dependencies (e.g., npymath_lib). Modules are installed under numpy/random.
- _bounded_integers.*: A small pxd/pyx pair generated and installed to support a stable interface across build environments. A custom_target ensures the .pxd is installed, preserving compatibility with downstream builds.
- NP_DEPRECATED_API handling: The build deliberately allows use of older NumPy C-API by setting NP_NO_DEPRECATED_API=0 to maximize compatibility with older NumPy versions.
- Large File Support: cflags_large_file_support is included to ensure compatibility with large data files across platforms.
- Platform-specific adjustments: On Cygwin, an export-all-symbols flag is added to ensure proper symbol visibility for the extension module on that platform.
- Legacy paths: A legacy mode (e.g., -DNP_RANDOM_LEGACY=1) is used for certain modules to preserve behavior expected by existing tests.

Important parameters and behaviors
-----------------------------------
- NP_NO_DEPRECATED_API=0: Enables usage of deprecated NumPy API to maintain compatibility with older NumPy versions.
- cflags_large_file_support: Enables large file support for handling files > 2 GiB.
- -Wl,--export-all-symbols (Cygwin): Ensures exported symbols for shared libraries on Cygwin.
- _bounded_integers.pxd/.pyx: Generated and installed to ensure consistent inclusion across the build, enabling compatibility with the rest of the Cython extensions.
- Linking: Each Python extension module links against both npyrandom_lib and any per-module libraries (e.g., npymath_lib or other generated objects).
- Install layout: Libraries installed to an appropriate random/lib location, and Python extension modules installed under numpy/random, preserving a predictable layout for downstream consumers.
- Legacy tests and distributions: Legacy build flags (e.g., NP_RANDOM_LEGACY) are used to retain compatibility with existing tests and behaviors.

Build-time and Platform Considerations
---------------------------------------
- Large file support is enabled to ensure compatibility with large datasets across platforms.
- Cygwin requires an explicit symbol export flag to ensure ABI compatibility for dynamic linking.
- The build uses a templating step to generate and install _bounded_integers.pxd, ensuring that consumers relying on this pxd file remain compatible with the built extension.
- Dependencies: The Python extensions rely on core NumPy and the project’s core dependencies (e.g., np_core_dep). The build enforces these relationships to maintain compatibility at link time.
- API/ABI surface: Exposed interfaces are designed to be stable across minor releases; where breaking changes are anticipated, deprecation and migration paths are documented in release notes.

Usage and Practical Guidance
----------------------------
- Ensure your Python environment matches the minimum supported NumPy version and Python version documented in the release notes.
- When building on Windows/Cygwin, verify that the toolchain supports the necessary export/link options to avoid missing symbol errors.
- If you rely on legacy NumPy C-API features, the current policy preserves compatibility by default (NP_NO_DEPRECATED_API=0). Plan migrations to newer API as part of future releases.

Edge Cases and Troubleshooting
------------------------------
- Mismatched NumPy version: If linking fails due to API changes, verify that NP_NO_DEPRECATED_API is configured as intended and consult release notes for a compatible NumPy version.
- Missing generated pxd/pxi: The custom_target ensures installation of _bounded_integers.pxd. If the pxd isn’t present at runtime, verify the templating step and installation path.
- Large file support: If you encounter issues with large datasets, confirm that cflags_large_file_support is active in the build and that the target platform supports large file operations.
- Cross-platform builds: Differences in toolchains across Linux, macOS, Windows, and Cygwin may require platform-specific adjustments; ensure the Meson build environment aligns with the target platform.

Code Examples
-------------
Examples illustrate typical usage and validation steps for compatibility.

1) Check Python and NumPy versions at runtime
```python
import sys
import numpy as np

print("Python version:", sys.version)
print("NumPy version :", np.__version__)
```

2) Ensure large file support is active in a build (conceptual)
- Verify that the build configuration includes cflags_large_file_support.
- On systems where you customize the Meson build, confirm that the flag is present in the compiler arguments for the relevant targets.

3) Validate the presence of the compatibility layer (_bounded_integers)
```python
import importlib
mod = importlib.import_module('numpy.random._bounded_integers')
print("Bounded integers module loaded:", mod is not None)
```

4) Basic import and smoke test for the random extension
```python
import numpy as np
from numpy import random as npr

# Simple smoke test to ensure extension modules are importable
try:
    x = npr.mtrand.rand()
    print("Sample random:", x)
except Exception as e:
    print("Extension import or execution failed:", type(e).__name__, e)
```
