Cross decomposition
===================

Overview
--------

The cross_decomposition module implements Partial Least Squares (PLS) methods, providing a family of estimators that relate two data sets X (predictors) and Y (responses) through a shared latent subspace. This module exposes three main variants:

- PLSRegression: regress Y from X using PLS latent components.
- PLSCanonical: symmetric decomposition that emphasizes shared structure between X and Y (canonical-like). Often used for exploratory data analysis.
- PLSSVD: PLS via singular value decomposition of XᵀY; a compact, algebraic interpretation of the PLS objective.

All classes follow the scikit-learn estimator API (fit/transform/predict), and rely on a common internal center-and-scale preprocessing when scale is enabled.

Key concepts
------------

- Latent components: Each variant extracts a sequence of latent axes (components) that capture the maximal covariance or shared structure between X and Y.
- Weight/loadings and scores:
  - x_weights_, y_weights_: weight vectors in the original feature spaces for X and Y.
  - x_loadings_, y_loadings_: loadings that map latent components back to the original feature spaces.
  - x_scores_, y_scores_: projections of X and Y onto the latent components.
  - x_rotations_, y_rotations_: alternative representations often used to relate components back to X and Y.
- Centering and scaling: By default, X and Y are centered; if scale is True, they are scaled to unit variance per feature prior to component extraction. This can be controlled per estimator.
- Algorithm choices:
  - nipals (power-method) approach for iterative extraction of the first singular vectors.
  - svd approach that uses a full singular value decomposition of XᵀY for the first latent vectors.

Classes and relationships
-------------------------

- PLSRegression
  - Purpose: regression from X to Y using PLS latent variables.
  - Fit/predict workflow follows scikit-learn conventions.
  - Outputs components that can be used to predict Y from X.
- PLSCanonical
  - Purpose: symmetric extraction of latent structure in X and Y; emphasizes shared information.
  - Useful for exploring relationships between X and Y rather than pure prediction.
- PLSSVD
  - Purpose: direct SVD-based decomposition of XᵀY to obtain latent structure.
  - Often used as a compact, algebraic variant of PLS.

Common parameters
-----------------

- n_components
  - Type: int
  - Description: Number of latent components to extract.
  - Note: The actual maximum usable components are limited by the data shapes (n_samples, n_features, n_targets).
- scale
  - Type: bool
  - Description: If True, X and Y are standardized to unit variance per feature before extraction; otherwise, features with varying scales may bias the components.
  - Default: True
- max_iter
  - Type: int
  - Description: Maximum iterations for the iterative solver (e.g., Nipals/power method) when extracting a component.
  - Default: 500
- tol
  - Type: float
  - Description: Convergence tolerance for the iterative solver.
  - Default: 1e-06
- algorithm
  - Type: {'nipals', 'svd'}
  - Description: Strategy to obtain the first latent vectors:
    - 'nipals': use the power-method iteration (default, more scalable for large data).
    - 'svd': compute the first singular vectors via a full SVD of XᵀY.
  - Default: 'nipals'
- copy
  - Type: bool
  - Description: Whether to copy X and Y during fitting.
  - Default: True

Returned values and attributes
-----------------------------

After fitting, each estimator exposes several attributes (ending with an underscore) that summarize the learned decomposition:

- x_weights_, y_weights_: component weight vectors for X and Y.
- x_loadings_, y_loadings_: loading matrices mapping components back to the original feature spaces.
- x_scores_, y_scores_: scores (latent representations) of X and Y.
- x_rotations_, y_rotations_: alternative representations linking components to original variables.
- n_features_in_: number of features fed to the estimator during fit.
- (For multi-output cases) y_loadings_ and y_scores_ have corresponding shapes aligned with Y.

Notes and edge cases
--------------------

- Constant Y: If Y is constant along a column, some methods (especially iterative ones) may raise a StopIteration or converge slowly. The power-method variant handles this via early stopping conditions.
- Scaling interactions: When scale is True, features with near-zero variance are safeguarded to avoid division by zero (typically by substituting 1.0 for zero-variance terms).
- Component limits: The practical maximum number of components is limited by the data size and target dimensionality. Exceeding the feasible maximum will raise an error.
- Centering: By default, inputs are centered. If you want to preserve means, you may post-process results using the stored means.

Usage examples
--------------

Basic regression with PLSRegression
----------------------------------

```python
from sklearn.datasets import make_regression
from sklearn.cross_decomposition import PLSRegression

# Synthetic data
X, Y = make_regression(n_samples=200, n_features=10, n_targets=3, random_state=0)

pls = PLSRegression(n_components=2, scale=True)
pls.fit(X, Y)

# Predict on new data
Y_pred = pls.predict(X)
```

Canonical cross-decomposition with PLSCanonical
------------------------------------------------

```python
from sklearn.cross_decomposition import PLSCanonical
import numpy as np

# Example data
X = np.random.randn(150, 6)
Y = np.random.randn(150, 4)

pls_can = PLSCanonical(n_components=3, scale=True)
pls_can.fit(X, Y)

# Access loadings/scores for interpretation
X_loadings = pls_can.x_loadings_
Y_loadings = pls_can.y_loadings_
```

SVD-based cross decomposition with PLSSVD
----------------------------------------

```python
from sklearn.cross_decomposition import PLSSVD
import numpy as np

X = np.random.randn(100, 8)
Y = np.random.randn(100, 2)

pls_svd = PLSSVD(n_components=2, scale=False)
pls_svd.fit(X, Y)

# Inspect latent representations
X_scores = pls_svd.x_scores_
Y_scores = pls_svd.y_scores_
```

Practical tips
--------------

- When interpretability of components is important, examine both x_loadings_ and y_loadings_ to understand how original features relate to latent variables.
- If you expect strong, linear relationships with differing scales, consider enabling scale to normalize features.
- For exploratory analysis of relationships between X and Y, PLSSVD or PLSCanonical can provide meaningful insights beyond pure predictive performance.

This documentation outlines the purpose, usage, and practical considerations for cross decomposition via PLS variants in the module.
