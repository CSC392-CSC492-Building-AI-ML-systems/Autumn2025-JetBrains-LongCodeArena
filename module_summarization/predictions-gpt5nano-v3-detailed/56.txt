MixedGLM
========

Overview
--------
This module implements Bayesian inference for generalized linear mixed models (GLMMs). It currently supports families without additional scale or shape parameters (notably binomial and Poisson). Two estimation approaches are provided: Laplace approximation (maximum a posteriori, MAP) and variational Bayes (mean-field approximation to the posterior). All realizations of random effects are modeled as independent in this implementation.

Key concepts
-------------
- Random effects: The random effects are captured by exog_vc. Each column of exog_vc corresponds to an independent random effect realization. These realizations are assumed to have mean zero and unknown standard deviations.
- Variance structure (vcp): Standard deviation parameters for the random effects are constrained to be equal within subsets of the columns. The vcp parameters have log-normal priors (implemented via log-standard deviations).
- ident: An array that groups columns of exog_vc into variance-structure blocks. Columns with the same ident value share the same standard deviation. When using formulas, ident is constructed from the formula-derived groupings.
- exog_vc sparsity: exog_vc is often sparse. A sparse matrix may be provided, or converted internally to a sparse representation if a dense matrix is supplied. When formulas are used, there may be an intermediate dense representation during construction.
- Formula support: When formulas are used, columns derived from the same formula are constrained to share the same variance.

Model and parameterization
---------------------------
The joint density factors as:

p(y | vc, fep) p(vc | vcp) p(vcp) p(fe)

- p(y | vc, fep): The GLM likelihood for the given family (depends on the specific GLM being fit).
- p(vc | vcp): Independent Gaussian prior for random effects, with variances determined by vcp (shared according to ident).
- p(vcp): Prior for the log-standard deviations of the random effects (log-normal distribution).
- p(fe): Prior for the fixed effects parameters (Gaussian with mean 0 and standard deviation fe_p).

In the documentation context, the random effects realizations (vc) are the individual values used to form the linear predictors, while the vcp parameters control their variability, and fe parameters control the fixed-effects part of the mean model.

Important parameters
--------------------
- endog: Array-like, the response vector.
- exog: Array-like, design matrix for fixed effects.
- exog_vc: Array-like or scipy.sparse matrix, design matrix for random effects.
- ident: Array-like of integers, labels grouping exog_vc columns into variance-structure blocks.
- vcp_p: Float, prior standard deviation for variance component parameters (log-standard deviations have this as their prior std).
- fe_p: Float, prior standard deviation for fixed effects parameters.
- family: statsmodels.genmod.families instance, the GLM family (e.g., Binomial, Poisson).
- fep_names: List[str], names of fixed effects parameters (default names constructed if None).
- vcp_names: List[str], names of variance component parameters (default names constructed if None).
- vc_names: List[str], names of the random effect realizations.

Estimation methods
------------------
- fit_map(): Fit using the Laplace approximation to the posterior (maximum a posteriori).
- fit_vb(): Fit using variational Bayes (mean-field approximation to the posterior).

Return values
-------------
- Both fit_map() and fit_vb() return a MixedGLMResults object (containing estimated parameters and diagnostics).

Notes and edge cases
--------------------
- Sparse handling: exog_vc can be sparse; dense inputs are converted to sparse internally. When formulas are used, a temporary dense version of exog_vc may be created.
- ident alignment: If not using formulas, ident must have the same length as the number of columns in exog_vc. Columns with the same ident value share the same variance. When formulas are used, ident is derived from the formula groupings.
- Numerical considerations: For some families (e.g., Binomial), the posterior mode can be numerically challenging to locate if vcp_p is too large. Values around 0.5 for vcp_p are often reasonable starting points.
- Prior choices: The fixed effects have Gaussian priors with mean 0 and standard deviation fe_p. The variance components have log-normal priors with standard deviation vcp_p for the log-sd.

Code structure (high level)
---------------------------
- The module defines a MIXED GLM modeling class that encapsulates data, priors, and the likelihood.
- It supports both MAP and VB fitting procedures.
- Internally, it handles the joint density, prior structure, and the parameterization of random effects via exog_vc and ident.

Usage example
-------------
- Basic setup and MAP fitting (pseudocode; adapt imports to your environment):

.. code-block:: python

    # Prepare data
    endog = ...      # response vector
    exog = ...       # fixed effects design matrix
    exog_vc = ...     # random effects design matrix (sparse allowed)
    ident = ...      # integer labels for variance groups

    # Create GLMM model
    model = MixedGLM(
        endog=endog,
        exog=exog,
        exog_vc=exog_vc,
        ident=ident,
        vcp_p=0.5,
        fe_p=0.5,
        family=family  # e.g., families.Binomial() or families.Poisson()
    )

    # MAP estimation
    res_map = model.fit_map()
    print(res_map.summary())

    # Variational Bayes
    res_vb = model.fit_vb()
    print(res_vb.summary())

Notes
-----
- The exact import paths for MixedGLM and MixedGLMResults may vary by version of the library. Consult your version's API reference for the precise module and class names.
- If you rely on formulas to build exog_vc, the ident vector is constructed from the formula groups.
