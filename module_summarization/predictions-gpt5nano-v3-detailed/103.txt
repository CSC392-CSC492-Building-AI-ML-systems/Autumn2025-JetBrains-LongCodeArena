Code generation utilities for Diofant expressions
===============================================

Introduction
------------

This module provides facilities to translate Diofant expressions into
executable code in C, C++, Fortran (77/90) and Octave/Matlab. It is a
work in progress; the API is designed around a generic Routine description
and language-specific CodeGen backends. The goal is to produce compilable
code (headers, includes, and language-specific nuances) while allowing
advanced workflows such as sharing intermediate results via common subexpression
elimination and evaluating constants at code-generation time.

Key ideas
---------

- A generic Routine describes what should be computed, including inputs,
  outputs, local temporaries, and global (constant) data.
- CodeGen subclasses translate Routine instances into source code for a
  specific target language (C, Fortran, Octave).
- Friendly helpers (codegen, make_routine) offer a higher-level API on top of
  the Routine/CodeGen abstraction for common use cases.

Module API overview
-------------------

- Routine
  - A language-agnostic description of an evaluation routine.
  - Constructor: Routine(name, arguments, results, local_vars, global_vars)
  - Validates that all symbols used by expressions are provided by
    input arguments, local variables, or global data.
  - Exposes:
    - .name: routine name
    - .arguments: list of argument descriptors (InputArgument, OutputArgument, InOutArgument)
    - .results: list of Result descriptors
    - .local_vars: list of local variables or identifiers
    - .global_vars: list of global Symbols
  - Properties:
    - .variables: the set of all variables possibly used in the routine
    - .result_variables: the set/list of variables that hold the results (see notes below)

- Data types
  - DataType: abstract representation of a language datatype (e.g., integer, real, double, etc.)
  - default_datatypes, get_default_datatype: utilities for selecting default datatypes

- Argument and Result objects
  - InputArgument: represents an input variable (name is used for binding)
  - OutputArgument: represents an output computed from an expression
  - InOutArgument: input/output argument (pass-by-reference style in some languages)
  - Result: represents a computed result with an associated target variable
  - The precise constructors capture the binding between names, expressions,
    and the variables used in the generated code.

- Code generation backends
  - CodeGen: base class for code generation backends
  - CCodeGen: generates C code (headers, includes, etc.)
  - FCodeGen: generates Fortran code (77/90)
  - OctaveCodeGen: generates Octave/Matlab code
  - Each backend translates a Routine into a complete, compilable code unit for its target language.

- Friendly API
  - codegen: high-level helper to generate code from a Routine (via an appropriate CodeGen backend)
  - make_routine: convenience helper to build a Routine from expressions and arguments

Usage notes and considerations
------------------------------

- The Routine validation ensures that every symbol used in results and
  outputs is provided by the inputs or local/global data. If a symbol is
  required but not supplied, a ValueError is raised.
- Early milestones focus on scalar inputs and C code generation; more advanced
  features (arrays, non-contiguous data, complex numbers, and multi-language
  support) are planned for future milestones.
- Some languages support multiple return values differently; the framework
  documents and enforces language-specific limitations (e.g., C may restrict
  to a single return value in certain contexts).
- The module provides an interface to extract or generate constants at
  code-generation time to optimize runtime performance.

Illustrative usage
------------------

Note: The exact constructors for InputArgument, OutputArgument, InOutArgument, and Result
may vary with versions of the library. The following illustrates the intended usage
at a conceptual level.

- Define a routine that computes z = sin(x) + x*y

Code generation workflow (conceptual):

- Create a Routine describing inputs, outputs, locals, and global data.
- Instantiate a language-specific CodeGen (e.g., CCodeGen) with the Routine.
- Retrieve the generated code as a string or write it to a file.

Pseudo-code example (illustrative):

"""
# Pseudo-code illustrating intended usage (API may vary)
from diofant import symbols, sin
from diofant.utilities.codegen import Routine, InputArgument, OutputArgument, Result, CodeGen, CCodeGen

# Define symbols
x, y = symbols('x y')

# Define expressions
expr = sin(x) + x*y

# Build routine components (exact constructors depend on the API)
inputs = [InputArgument('x'), InputArgument('y')]
outputs = [Result('z', expr, result_var='z')]  # z will hold the computed value

# Create a Routine (local_vars and global_vars can be provided as needed)
routine = Routine(name='compute', arguments=inputs, results=outputs, local_vars=[], global_vars=[])

# Generate C code
cg = CCodeGen(routine)
code = cg.get_code()  # or cg.write(filename)

print(code)
"""

- Friendly API example (conceptual):
"""
from diofant.utilities.codegen import make_routine, codegen

# make_routine builds a Routine from a simple specification
routine = make_routine('compute', inputs=['x','y'], outputs=[('z', 'sin(x) + x*y')])

# Generate C code using the friendly codegen API
code = codegen(routine, language='C')
print(code)

Edge cases and error handling
-----------------------------

- If a symbol appearing in any output expression is not provided by
  inputs, local_vars, or global_vars, Routine initialization raises a ValueError.
- In C-based codegen milestones, multiple return values may require special
  handling; the generator may not support arbitrary multiple returns in C.
- Complex numbers, Python-style long integers, or non-portable data types may
  require additional handling or are not yet supported in the current milestones.
