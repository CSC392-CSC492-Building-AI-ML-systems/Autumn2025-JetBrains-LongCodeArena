Overview
--------
This module implements the preview+confirm+delete workflow for deleting objects in Django-cradmin applications. It provides a mixin (DeleteViewMixin) that adds standardized behavior and context for a delete confirmation, and a concrete view (WithinRoleDeleteView) that wires the mixin into a role-aware context and a sane default template.

Key components
--------------
- DeleteViewMixin
  - Purpose: Provide the core delete-confirm UI and messaging for a Django DeleteView integration.
  - Responsibilities:
    - Supply page title, action and button labels.
    - Provide a preview of the object to be deleted.
    - Render a confirmation message that references the object preview.
    - Define the success URL and success message behavior after deletion.
    - Populate common template context data for delete views.
    - Ensure success messages are added after deletion.
  - Important methods:
    - get_pagetitle(): Returns the page title. Default: "Confirm delete".
    - get_action_label(): Returns the action label used for the title and defaults for buttons. Default: "Delete".
    - get_delete_button_label(): Label for the delete button. Defaults to get_action_label().
    - get_cancel_button_label(): Label for the cancel button. Default: "Cancel".
    - get_object_preview(): Returns a string representation of the object to delete. Default: str(object).
    - get_confirm_message(): Message shown in the focus area asking for confirmation. Default: 'Are you sure you want to delete "<object_preview>"?'.
    - get_success_url(): URL to redirect to after deletion. Default: app index URL for the current app.
    - get_success_message(object_preview): Optional override to customize the success message. Default: 'Deleted "<object_preview>"'.
    - add_success_messages(object_preview): Adds a success message if get_success_message returns a value.
    - add_delete_view_mixin_context_data(context): Populates context with template data like model_verbose_name, success_url, object_preview, pagetitle, confirm_message, delete_button_label, and cancel_button_label.
    - delete(request, *args, **kwargs): Performs deletion, then adds success messages, and returns the response.
- WithinRoleDeleteView
  - Purpose: Concrete, role-aware delete view with a sensible default template for Cradmin-based views.
  - Inheritance: QuerysetForRoleMixin, DeleteViewMixin, DjangoDeleteView, CommonCradminViewMixin, javascriptregistry.viewmixin.WithinRoleViewMixin.
  - Template: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html
  - Behavior: Extends GetContextData to incorporate JavaScript registry IDs and common view data, then adds DeleteViewMixin data.

Relationships and workflow
--------------------------
- WithinRoleDeleteView is a fully-fledged DeleteView that:
  - Resolves the object via the role-aware queryset mixin (QuerysetForRoleMixin).
  - Renders a confirmation page with a preview and a confirm button.
  - On POST (confirm), performs the delete via DjangoDeleteView, then:
    - Adds a success message (if configured).
    - Redirects to the app index URL.
  - Context data is augmented with template-specific data (object preview, pagetitle, messages, etc.) and with JavaScript component IDs when using the JavaScript registry.

Parameters and return values (highlights)
-----------------------------------------
- get_pagetitle(self) -> The page title for the delete view. Default: gettext_lazy('Confirm delete').
- get_action_label(self) -> The action label used as the prefix for the page title and as a default for delete button label. Default: gettext_lazy('Delete').
- get_delete_button_label(self) -> Label for the delete button. Default: value of get_action_label().
- get_cancel_button_label(self) -> Label for the cancel button. Default: gettext_lazy('Cancel').
- get_object_preview(self) -> str: String representation of the object being deleted.
- get_confirm_message(self) -> str: Confirmation message shown in the focus area. Default: 'Are you sure you want to delete "%(object_preview)s"?'.
- get_success_url(self) -> URL to redirect to after deletion. Default: app index URL for the current app.
- get_success_message(self, object_preview) -> Optional str: Message shown after successful deletion. Default: 'Deleted "%(what)s"'.
- add_success_messages(self, object_preview) -> None: Adds the success message to Django messages if provided.
- add_delete_view_mixin_context_data(self, context) -> None: Populates context with keys like model_verbose_name, success_url, object_preview, pagetitle, confirm_message, delete_button_label, cancel_button_label.
- delete(self, request, *args, **kwargs) -> HttpResponse: Performs the deletion, then adds success messages and returns the response.

Usage examples
--------------
1) Using WithinRoleDeleteView in a Cradmin app

.. code-block:: python

    from django_cradmin.viewhelpers.formview import WithinRoleDeleteView
    from django_cradmin.viewhelpers.mixins import QuerysetForRoleMixin, CommonCradminViewMixin
    from django.views.generic import DeleteView
    from myapp.models import MyResource

    class MyResourceWithinRoleDeleteView(QuerysetForRoleMixin,
                                        WithinRoleDeleteView,
                                        DeleteView,
                                        CommonCradminViewMixin):
        model = MyResource
        template_name = 'myapp/delete_confirm.django.html'
        # The queryset for the role is provided by QuerysetForRoleMixin
        # Optionally override get_success_url/get_success_message if needed

2) Wire into URLs

.. code-block:: python

    from django.urls import path
    from .views import MyResourceWithinRoleDeleteView

    urlpatterns = [
        path('resource/<int:pk>/delete/', MyResourceWithinRoleDeleteView.as_view(), name='resource-delete'),
    ]

Notes and considerations
------------------------
- Template: WithinRoleDeleteView uses a sane default template: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html. You can override template_name to customize presentation.
- Context data: The view provides keys such as model_verbose_name, object_preview, pagetitle, confirm_message, delete_button_label, cancel_button_label to be used in the template.
- Messaging: Deletion success messages integrate with Django’s messages framework via add_success_messages; override get_success_message to customize or suppress messages.
- Edge cases:
  - If get_object_preview returns a long string, consider truncation in the template or a custom override.
  - Ensure the object is available in the appropriate queryset for the current role when using QuerysetForRoleMixin.
  - If you override get_success_url to a different location, ensure permissions and redirects remain consistent.
- Extensibility: You can subclass DeleteViewMixin to tailor labels, messages, or context data for non-role-based views. WithinRoleDeleteView provides a ready-to-use, role-aware implementation.

Template and integration details
--------------------------------
- Template path for WithinRoleDeleteView: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html
- The view integrates with the django-cradmin JavaScript registry to support dynamic UI components (WithinRoleViewMixin).

End-user behavior
---------------
- User navigates to the delete view.
- The page displays a preview of the object, a confirmation message, and Delete/Cancel buttons.
- Clicking Delete deletes the object, shows a success message, and redirects to the app index.
- Clicking Cancel returns the user to the previous page or designated cancel path as defined by the template.
