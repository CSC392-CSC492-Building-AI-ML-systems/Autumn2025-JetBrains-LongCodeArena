Introduction
-------------
This module provides general-purpose plotting routines based on the Bokeh backend for Masci-tools. It exposes helpers to configure the Bokeh plotter defaults and two core plotting functions for interactive scatter plots. The design centers on a single BokehPlotter instance and utility wrappers that ensure consistent usage across plots.

Key components
--------------
- BokehPlotter (plotter engine)
  - Central class that handles figure creation, default settings, plot keyword handling, tooltips, legends, and saving.
- Default management helpers
  - set_bokeh_plot_defaults(**kwargs): Update BokehPlotter defaults.
  - reset_bokeh_plot_defaults(): Restore hardcoded defaults.
  - show_bokeh_plot_defaults(): Print current defaults.
  - get_bokeh_help(key): Describe a specific plotter option.
  - load_bokeh_defaults(filename='plot_bokeh_defaults.json'): Load defaults from a JSON file.
  - save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False): Save current defaults to a JSON file.
- Data processing helper
  - process_data_arguments(data, x, y, copy_data, single_plot, same_length, use_column_source) (via import): Normalizes inputs, creates a ColumnDataSource, and enforces lengths/single vs. multi-plot semantics.
- Plotting routines
  - bokeh_scatter(...): Create a single interactive scatter plot.
  - bokeh_multi_scatter(...): Create an interactive scatter plot that supports multiple data series.

Important concepts and relationships
-------------------------------------
- Consistency: The @ensure_plotter_consistency(plot_params) decorator ensures the function uses a consistent BokehPlotter instance.
- Data model: Plots are built from a data source (data) plus x/y accessors. process_data_arguments converts inputs into a uniform ColumnDataSource and returns an entry describing the dataset (with fields x and y) and the corresponding source for the figure.
- Figure lifecycle: A figure is prepared with plot_params.prepare_figure, data is plotted with the appropriate p.scatter call, tooltips are added, optional lines and limits are drawn, and the plot is saved via plot_params.save_plot.
- Defaults and customization: The BokehPlotter contains defaults (size, toolbar, styling, etc.). Users can modify and persist these defaults and retrieve help describing available options.

General plotting routines

1) bokeh_scatter
- Purpose: Create an interactive single dataset scatter plot with Bokeh.
- Signature:
  - x: array-like or key for x-axis data, or a data source (deprecated usage emits a warning)
  - y: array-like or key for y-axis data
  - data: optional data source (e.g., pandas DataFrame)
  - xlabel, ylabel, title: axis labels and plot title
  - figure: optional existing Bokeh figure to add to
  - saveas: string name used to save the plot (via plot_params)
  - copy_data: whether to copy the input data before plotting
  - plus additional kwargs passed to the plotting backend
- Behavior:
  - Normalizes inputs using process_data_arguments.
  - Derives a dataset entry and a ColumnDataSource.
  - Applies default plot type and name hints to the plotter.
  - Creates the figure, computes plotting args for a scatter, and issues the scatter call.
  - Adds tooltips and optional styling (level, straight lines, limits).
  - Saves the plot to a file/endpoint as specified by saveas.
- Return: The created Bokeh Figure object (p).

Example:
```python
from masci_tools.vis.plotting import bokeh_scatter, set_bokeh_plot_defaults

set_bokeh_plot_defaults(width=800, height=420)

x = [0, 1, 2, 3, 4]
y = [0.0, 0.8, 0.9, 0.1, -0.2]

p = bokeh_scatter(x, y, xlabel='Time (ps)', ylabel='Signal', title='Sample Scatter')
```

2) bokeh_multi_scatter
- Purpose: Create an interactive scatter plot that can display multiple data series in one figure.
- Signature:
  - x, y, data, figure, xlabel, ylabel, title, saveas, copy_data
  - set_default_legend: whether to automatically generate a legend from data names
  - Additional kwargs passed to the plotting backend
- Behavior:
  - Similar data handling as bokeh_scatter, but supports multiple datasets within a single plot.
  - Optionally generates default legend entries from the provided data if set_default_legend is True.
  - Utilizes the same figure preparation and saving workflow as bokeh_scatter.
- Return: The created Bokeh Figure object (p).

Example:
```python
import pandas as pd
from masci_tools.vis.plotting import bokeh_multi_scatter

df = pd.DataFrame({
    'x': [0, 1, 2, 3, 4],
    'y1': [0.1, 0.4, 0.8, 0.4, 0.2],
    'y2': [0.0, 0.2, 0.5, 0.7, 0.9]
})

# First dataset is x, y references are taken from df
p = bokeh_multi_scatter(x='x', y=['y1', 'y2'], data=df, title='Multi-series Scatter')
```

Parameters and return values
----------------------------
- set_bokeh_plot_defaults(**kwargs)
  - Updates the global BokehPlotter defaults.
  - Returns: None
- reset_bokeh_plot_defaults()
  - Restores hardcoded defaults.
  - Returns: None
- show_bokeh_plot_defaults()
  - Prints current defaults to stdout.
  - Returns: None
- get_bokeh_help(key)
  - Displays description for a given key in the BokehPlotter.
  - Returns: None
- load_bokeh_defaults(filename='plot_bokeh_defaults.json')
  - Loads defaults from a JSON file.
  - Returns: None
- save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)
  - Persists current defaults to a JSON file.
  - Returns: None
- bokeh_scatter(...)
  - Returns: A Bokeh Figure object representing the plotted scatter.
- bokeh_multi_scatter(...)
  - Returns: A Bokeh Figure object representing the plotted multi-series scatter.

Important considerations and edge cases
--------------------------------------
- Deprecated input pattern: Passing the data source as the first positional argument (x) is deprecated. A DeprecationWarning is issued; prefer supplying data via the data keyword and x/y through their respective arguments.
- Data length: process_data_arguments enforces consistent data lengths (same_length=True). Mismatches will raise errors.
- Single vs. multi plot: bokeh_scatter uses single_plot semantics; bokeh_multi_scatter supports multiple datasets and can auto-generate legends if requested.
- Plotter consistency: The ensure_plotter_consistency decorator helps ensure that the same BokehPlotter configuration is used across calls, avoiding inconsistent visuals.
- Saving behavior: saveas controls how the plot is saved or exported. The exact behavior depends on the BokehPlotter’s save_plot implementation, but typically includes saving to a file or embedding into a notebook.
- Tooltips and styling: Tooltips and default styling are managed by the BokehPlotter; customizations can be provided through kwargs as long as they are recognized by the backend.

Usage quickstart
----------------
- Inspect and adjust defaults
  - set_bokeh_plot_defaults(width=800, height=420, toolbar_location='right')
  - show_bokeh_plot_defaults()
  - get_bokeh_help('width')
- Basic single scatter
  - bokeh_scatter([0,1,2], [0,1,4], xlabel='X', ylabel='Y', title='Example')
- Multi-series scatter
  - bokeh_multi_scatter(x=[0,1,2], y=[[1,2,3], [2,3,4]], data=my_df, set_default_legend=True, title='Multi-series')
