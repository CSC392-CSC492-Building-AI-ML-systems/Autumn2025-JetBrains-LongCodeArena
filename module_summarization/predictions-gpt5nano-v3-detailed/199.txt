# Neural Network Models (Supervised)

This module provides a collection of utilities to support supervised neural networks. It includes in-place activation functions and their derivatives, as well as common loss functions used for regression and classification tasks. The functions are designed to be efficient by mutating inputs in place where appropriate.

## Overview

- In-place activation functions for performance-sensitive forward passes:
  - Identity, Logistic (sigmoid), Tanh, ReLU, Softmax
- In-place derivatives for backpropagation:
  - Derivatives corresponding to the activation functions
- Loss functions for supervised learning:
  - Squared loss (regression)
  - Logistic loss (multiclass cross-entropy)
  - Binary logistic loss (binary/multilabel case)

The module exposes two key mappings:
- `ACTIVATIONS`: maps activation names to their in-place implementations.
- `DERIVATIVES`: maps activation names to their in-place derivative implementations.

Note: All in-place functions mutate their input arrays. The forward activations expect arrays with shape `(n_samples, n_features)` where applicable, and derivatives expect the same shapes for `Z` (activation output from the forward pass) and `delta` (backpropagated error).

## Activation functions (in-place)

The following activations are available and mutating the input array in place.

- Identity
- Logistic (sigmoid)
- Tanh
- ReLU
- Softmax

Key details:
- Softmax uses a numerically stable formulation by subtracting the row-wise maximum before exponentiation.
- The `ACTIVATIONS` dictionary provides a convenient lookup by name.

Example:
```python
import numpy as np

from neural_networks_supervised import ACTIVATIONS

X = np.random.randn(5, 3)
ACTIVATIONS['softmax'](X)  # X is mutated in place to contain row-wise probabilities
```

## Activation derivatives (in-place)

Backpropagation requires derivatives of the activations with respect to their inputs. The module provides in-place derivative functions that modify `delta` using the activation output `Z` (the result of the forward pass).

- Identity derivative: no change
- Logistic derivative: uses the property that d(sigmoid(z))/dz = s(z) * (1 - s(z)) and is expressed in terms of the forward output `Z`
- Tanh derivative: `1 - Z^2` where `Z = tanh(z)`
- ReLU derivative: zero where the forward output `Z` equals zero

Available derivatives are accessible via the `DERIVATIVES` mapping.

Example:
```python
import numpy as np

from neural_networks_supervised import DERIVATIVES

Z = np.random.randn(4, 3)  # output from forward pass (activation applied)
delta = np.ones_like(Z)

DERIVATIVES['logistic'](Z, delta)  # delta is updated in place
```

## Loss functions

Loss functions are provided for common supervised learning scenarios.

- Squared loss (regression)
  - Function: `squared_loss(y_true, y_pred)`
  - Formula: mean((y_true - y_pred)^2) / 2
  - Returns: scalar loss

- Logistic loss (multiclass cross-entropy)
  - Function: `log_loss(y_true, y_prob)`
  - `y_prob` is the predicted probability distribution with shape `(n_samples, n_classes)`
  - Handles numerical stability via clipping
  - If inputs are in the binary form (shape `(n_samples, 1)`), they are converted to a two-class one-hot representation internally
  - Returns: scalar loss

- Binary logistic loss
  - Function: `binary_log_loss(y_true, y_prob)`
  - Used for binary classification or multilabel setups
  - Treats `y_prob` as shape `(n_samples, 1)` and applies a binary cross-entropy formulation
  - Returns: scalar loss

Important notes:
- Loss functions clip probabilities to avoid log(0) and ensure numerical stability.
- For `log_loss`, if inputs are in 1-column form, they are expanded to two-class one-hot representations internally.
- Shapes must be compatible (e.g., one-hot encoded `y_true` and `y_prob` with matching number of classes).

Examples:
```python
import numpy as np
from neural_networks_supervised import squared_loss, log_loss, binary_log_loss

y_true = np.array([[1.0, 0.0], [0.0, 1.0]])
y_pred = np.array([[0.6, 0.4], [0.3, 0.7]])

loss_reg = squared_loss(y_true, y_pred)
loss_log = log_loss(y_true, y_pred)
loss_bin = binary_log_loss(np.array([[1], [0]]), np.array([[0.9], [0.2]]))
```

## Edge cases and considerations

- In-place behavior: Mutates input arrays. Preserve originals if needed.
- Numerical stability: Softmax uses row-wise max subtraction; loss functions clip probabilities to `[eps, 1 - eps]`.
- Shape handling:
  - `log_loss` and `binary_log_loss` gracefully handle 1D/2D inputs by expanding to two-class representations when necessary.
  - Ensure `y_true` and `y_prob` are compatible in shape for the chosen loss function.
- Derivative usage: The derivative functions expect `Z` to be the forward-pass activations and will mutate `delta` accordingly.

## API reference (summary)

- Activation functions (in-place)
  - `ACTIVATIONS`: mapping of activation name to in-place function
    - Keys: `'identity'`, `'tanh'`, `'logistic'`, `'relu'`, `'softmax'`
- Activation derivatives (in-place)
  - `DERIVATIVES`: mapping of activation name to in-place derivative function
    - Keys: `'identity'`, `'tanh'`, `'logistic'`, `'relu'`
- Loss functions
  - `squared_loss(y_true, y_pred)`
  - `log_loss(y_true, y_prob)`
  - `binary_log_loss(y_true, y_prob)`

Usage examples above illustrate common patterns for applying activations, computing derivatives during backpropagation, and evaluating model performance with standard loss measures.
