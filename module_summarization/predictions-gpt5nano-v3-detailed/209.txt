Working with Text Data
======================

Analyzing a collection of text documents
---------------------------------------

This module provides utilities to convert raw text documents into numeric feature vectors suitable for machine learning. It focuses on tokenization, normalization, stop-word handling, and the generation of token-based features (including n-grams). The public API centers on vectorizers that map text collections to sparse numeric matrices and, optionally, to TF-idf representations.

Overview
--------

- Convert text documents into feature vectors for machine learning workflows.
- Support for word- and character-level analysis, by-words or by-characters n-grams.
- Built-in stop-word handling (including an English stop-word list).
- Preprocessing options such as lowercasing and accent stripping.
- Multiple vectorizers with varying trade-offs:
  - CountVectorizer: simple term frequency counts.
  - TfidfVectorizer: TF-IDF weighted features (often improves performance).
  - HashingVectorizer: scalable, stateless hashing-based vectorization.

Public API
----------

Note: The module exposes several key classes and helper functions. The following are the primary public entry points and utilities you would typically use when analyzing a collection of text documents.

- CountVectorizer
  - Purpose: Convert a collection of text documents to a matrix of token counts.
  - Typical workflow:
    - Instantiate CountVectorizer with desired analyzer (word/char), n-gram range, stop words, etc.
    - Call fit_transform on your corpus to obtain a sparse count matrix.
  - Common parameters:
    - input: {'filename','content','object'} (default 'content')
    - encoding, decode_error
    - strip_accents: {'ascii','unicode', None}
    - lowercase: bool
    - stop_words: built-in list (e.g., 'english') or a collection
    - analyzer: {'word','char'} or callable
    - tokenizer, preprocessor, ngrams
    - ngram_range: tuple (min_n, max_n)
    - max_df, min_df, max_features
  - Output: sparse matrix of shape (n_samples, n_features) with integer counts.

- TfidfVectorizer
  - Purpose: Convert text documents to TF-IDF weighted feature vectors.
  - Typical workflow:
    - Instantiate TfidfVectorizer with desired settings (analyzer, n-grams, stop words, etc.).
    - Call fit_transform to get a TF-IDF sparse matrix.
  - Common parameters (in addition to CountVectorizer options):
    - use_idf: bool (default True)
    - norm: {'l2','l1','none'} (default 'l2')
    - smooth_idf: bool (default True)
    - sublinear_tf: bool (default False)
  - Output: sparse matrix with TF-IDF weights.

- HashingVectorizer
  - Purpose: Stateless, scalable vectorization using a hashing trick.
  - Typical workflow:
    - Instantiate HashingVectorizer with n_features, analyzer, n-grams, etc.
    - Call transform on raw text data (or fit_transform if supported by your workflow).
  - Common parameters:
    - n_features: int, number of features (buckets)
    - input: {'filename','content','object'} (default 'content')
    - alternate_sign: bool (default True)
    - strip_accents, stop_words, analyzer, ngram_range
  - Output: sparse matrix of hashed features.

- TfidfTransformer
  - Purpose: Transform a term-document matrix (counts) into TF-IDF representation.
  - Typical workflow:
    - Use after CountVectorizer to convert counts to TF-IDF features.
  - Common parameters:
    - use_idf: bool (default True)
    - norm: {'l2','l1','none'} (default 'l2')
    - smooth_idf: bool (default True)
  - Output: sparse TF-IDF matrix.

- ENGLISH_STOP_WORDS
  - A built-in stop-word set intended for English text. Used with stop_words='english' in vectorizers or fed directly as a collection.

- Helper functions (internal; for advanced customization)
  - _preprocess(doc, accent_function=None, lower=False)
    - Preprocess a single document by optionally lowercasing and applying an accent transformation function.
  - _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
    - Compose the processing pipeline for a single document (preprocessor, tokenizer, ngrams, stop words handling).
  - _check_stop_list(stop)
    - Resolve stop words: 'english' -> ENGLISH_STOP_WORDS, None -> None, or a custom collection.
  - _VectorizerMixin
    - Shared logic for tokenization and handling of preprocessing steps used by vectorizers.

Usage Examples
--------------

CountVectorizer: simple term counting

```python
from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "This is the first document.",
    "This is the second document.",
    "And this is the third one."
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

print("Feature names:", vectorizer.get_feature_names_out())
print("Shape:", X.shape)
print("Dense representation:\n", X.toarray())
```

TfidfVectorizer: TF-IDF features with default settings

```python
from sklearn.feature_extraction.text import TfidfVectorizer

corpus = [
    "This is the first document.",
    "This is the second document.",
    "And this is the third one."
]

tfidf = TfidfVectorizer()
X = tfidf.fit_transform(corpus)

print("Feature names:", tfidf.get_feature_names_out())
print("TF-IDF matrix shape:", X.shape)
print("TF-IDF values (dense):\n", X.toarray())
```

Using English stop words and n-grams

```python
from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "The quick brown fox jumps over the lazy dog.",
    "The quick brown dog jumps over the lazy fox."
]

vec = CountVectorizer(stop_words='english', ngram_range=(1, 2), analyzer='word')
X = vec.fit_transform(corpus)

print("Features:", vec.get_feature_names_out())
print("Matrix shape:", X.shape)
```

HashingVectorizer for scalable text representation

```python
from sklearn.feature_extraction.text import HashingVectorizer

corpus = [
    "This is the first document.",
    "This is the second document."
]

hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
X = hasher.transform(corpus)

print("Matrix shape:", X.shape)
```

Important Considerations and Edge Cases
--------------------------------------

- Stop words
  - Use ENGLISH_STOP_WORDS or a custom collection via stop_words. Passing 'english' uses a built-in English stop-word list.
  - Stop words filtering occurs after optional lowercasing and accent stripping, depending on configuration.

- Accent handling
  - strip_accents can be 'ascii' or 'unicode' (or None). Accent stripping is applied during tokenization/preprocessing depending on vectorizer configuration.

- Preprocessing pipeline
  - The typical pipeline is preprocessor -> tokenizer -> ngrams, with optional decoders for decoding input documents.
  - You can customize the pipeline by providing your own tokenizer, preprocessor, or analyzer.

- Input sources
  - Vectors can be built from raw content or from file names, depending on the input parameter.

- Output representations
  - CountVectorizer yields raw term counts.
  - TfidfVectorizer yields TF-IDF weighted features (good for many text classification tasks).
  - HashingVectorizer provides a memory-efficient, assignment-free representation suitable for very large corpora.

- Edge cases
  - Empty documents or documents with no recognized tokens result in rows of zeros.
  - Unicode normalization and encoding issues are handled via the vectorizer’s encoding and decode_error settings.
  - Large vocabularies can lead to high memory usage; prefer max_features or hashing with a fixed feature size for large corpora.

Notes
-----

- This module emphasizes practical tokenization strategies and easy-to-use vectorizers for common text analysis tasks.
- For advanced customization, you can supply your own analyzer, tokenizer, preprocessor, or stop words, or use internal helpers to implement specialized preprocessing pipelines.

Usage Context
-------------

- Text classification, clustering, topic modeling, and information retrieval workflows often rely on turning text documents into numerical feature vectors via the vectorizers described here.
- The choice between CountVectorizer, TfidfVectorizer, and HashingVectorizer depends on data size, desired feature properties, and memory considerations.
