Contingency Tables
===================

Overview
--------
This module provides tools for analyzing two-way contingency tables (frequency tables cross-classifying two categorical variables). The primary public API is the Table class, which supports both nominal and ordinal (trend) association tests on a contingency table built from raw data. The design emphasizes independence assumptions, standard chi-squared testing for nominal factors, and Cochran-Armitage style trend testing for ordinal factors.

Key components
--------------
- Table
  - Represents a two-way contingency table.
  - Handles zero-cell adjustments, construction from raw data, and statistical tests.
  - Public methods:
    - from_data(data, shift_zeros=True): Construct a Table from raw data using the first two columns to form a cross-tabulation.
    - test_nominal_association(): Assess independence of two nominal factors using a chi-squared test.
    - test_ordinal_association(row_scores=None, col_scores=None): Assess association of two ordinal factors using a linear-by-linear (Cochran-Armitage) style test.
  - Internal helpers:
    - _make_df_square(table): Ensure a DataFrame is square by reindexing rows and columns to share the same index set and order (private utility).
  - Result container:
    - _Bunch: Simple container used to return test results with a readable representation.

Usage notes
-----------
- Data handling
  - from_data accepts either a pandas DataFrame or a 2D array-like input. It constructs the contingency table using the first two columns via pandas crosstab.
  - If shift_zeros is True and the constructed table contains zeros, all zero entries are increased by 0.5 to enable certain test computations.

- Nominal association test
  - test_nominal_association computes the chi-squared statistic as the sum of the cell-wise contributions.
  - Degrees of freedom: (number of rows - 1) × (number of columns - 1).
  - p-value is computed from the chi-squared distribution.
  - Returned values are accessible via a _Bunch with attributes: statistic, df, pvalue.

- Ordinal association test
  - test_ordinal_association evaluates a linear-by-linear (trend) statistic using supplied row and column scores.
  - If scores are not provided, default scores are 0, 1, 2, ... for rows and columns.
  - The method returns a _Bunch with statistics such as statistic, null_mean, null_sd, zscore, and pvalue (exact fields may depend on the internal implementation).

- Robustness and edge cases
  - Zero handling via shift_zeros helps stabilize tests when zero counts appear.
  - For test_ordinal_association, lengths of row_scores and col_scores must match the table dimensions.
  - The internal representation caches the original table as table_orig for reference.

Code examples
-------------
Below are representative usage patterns for developers.

Create a Table from raw data and perform a nominal association test:
```python
import numpy as np
import pandas as pd
from contingency_tables import Table  # adjust import path as needed

# Example raw data: two categorical variables
data = pd.DataFrame({
    'A': np.random.choice(['x', 'y', 'z'], size=100),
    'B': np.random.choice(['u', 'v'], size=100)
})

# Build a contingency table
tbl = Table.from_data(data)

# Nominal association test (independence)
res_nominal = tbl.test_nominal_association()
print("Chi-squared statistic:", res_nominal.statistic)
print("df:", res_nominal.df)
print("p-value:", res_nominal.pvalue)
```

Perform an ordinal association test with custom scores:
```python
# Optionally provide scores for rows and columns
row_scores = [1, 2, 3]       # length must match number of rows
col_scores = [0, 1]          # length must match number of columns

res_ordinal = tbl.test_ordinal_association(row_scores=row_scores, col_scores=col_scores)
print("Statistic:", res_ordinal.statistic)
print("p-value:", res_ordinal.pvalue)
```

Notes and considerations
------------------------
- The module focuses on two-way contingency tables and standard tests for nominal and ordinal data.
- The _make_df_square helper ensures compatibility when operations expect square matrices, by expanding the index/columns to a common set and ordering consistently.
- The behavior of zeros is configurable via shift_zeros to avoid issues with certain statistical procedures.
- The internal _Bunch class provides a convenient, readable summary of test results.

Related concepts
----------------
- The nominal test aligns with classical chi-squared tests for independence in contingency tables.
- The ordinal test implements a trend-oriented statistic (linear-by-linear association) that gains power when order information is present in both dimensions.
