Transforms
==========

Overview
--------
This module provides utilities for converting images between PIL/NumPy representations and PyTorch tensors, as well as helpers for querying image dimensions and channel information. It includes an InterpolationMode enumeration compatible with common image processing backends and internal mappings to PIL’s resizing/interpolation options.

Key components
--------------
- InterpolationMode: Enum of supported interpolation methods (nearest, bilinear, bicubic, etc.).
- _interpolation_modes_from_int(i): Internal helper to map integer codes to InterpolationMode.
- pil_modes_mapping: Mapping from InterpolationMode values to PIL’s mode indices.
- _is_pil_image: Delegate to determine if an object is a PIL image.
- get_dimensions(img): Return image dimensions as [C, H, W] for a PIL image or a Tensor.
- get_image_size(img): Return image size as [width, height].
- get_image_num_channels(img): Return the number of channels in the image.
- to_tensor(pic): Convert a PIL Image or NumPy array to a PyTorch tensor. Produces a float tensor in [0, 1] for PIL inputs by default; handles numpy arrays and accimage images when available.
- pil_to_tensor(pic): Convert a PIL Image to a tensor preserving the underlying dtype (no scaling).

Public API
----------
- InterpolationMode
- _interpolation_modes_from_int(i: int) -> InterpolationMode
- pil_modes_mapping: Dict[InterpolationMode, int]
- _is_pil_image
- get_dimensions(img: Tensor) -> List[int]
- get_image_size(img: Tensor) -> List[int]
- get_image_num_channels(img: Tensor) -> int
- to_tensor(pic) -> Tensor
- pil_to_tensor(pic) -> Tensor

Usage notes
-----------
- to_tensor handles:
  - PIL Images: converts to CHW float32 tensor in [0, 1].
  - NumPy arrays:
    - If 2D, treated as a single-channel image.
    - If 3D, expected to be HWC and will be rearranged to CHW.
  - accimage (if installed) inputs are supported.
  - PIL mode "1" (1-bit) is converted to 0/255 then scaled to [0, 1].
- pil_to_tensor preserves the original PIL image dtype in the resulting tensor (e.g., uint8 for typical RGB images), without scaling.
- get_dimensions, get_image_size, and get_image_num_channels accept either a PIL Image or a Tensor and return consistent metadata:
  - get_dimensions returns [C, H, W].
  - get_image_size returns [W, H].
  - get_image_num_channels returns the number of channels.
- The module relies on internal helpers F_pil and F_t for PIL and tensor implementations respectively.

Important considerations and edge cases
- torch.jit is_scripting and is_tracing checks are performed to log API usage appropriately; behavior is consistent whether scripting/tracing is active.
- If input is a NumPy array with dimensions != 2 or 3, to_tensor raises an error.
- When using accimage, an explicit import is attempted; if not available, accimage is None and normal PIL/NumPy paths are used.
- The InterpolationMode enum provides compatibility with PIL and internal mappings for different interpolation methods.

Examples
--------

1) Convert a PIL image to a tensor
```python
from torchvision.transforms.functional import to_tensor
from PIL import Image

img = Image.open("path/to/image.jpg").convert("RGB")
t = to_tensor(img)
print(t.shape)  # torch.Size([3, H, W])
```

2) Convert a NumPy array to a tensor
```python
import numpy as np
from torchvision.transforms.functional import to_tensor

arr = np.zeros((100, 100, 3), dtype=np.uint8)
t = to_tensor(arr)
print(t.shape, t.dtype)  # torch.Size([3, 100, 100]), torch.float32
```

3) Convert a PIL image to a tensor without scaling (pil_to_tensor)
```python
from torchvision.transforms.functional import pil_to_tensor
from PIL import Image

img = Image.open("path/to/image.jpg").convert("RGB")
t = pil_to_tensor(img)
print(t.shape, t.dtype)  # torch.Size([3, H, W]), e.g., torch.uint8
```

4) Query image metadata
```python
import torch
from torchvision.transforms.functional import get_dimensions, get_image_size, get_image_num_channels

t = torch.randn(3, 64, 64)
print(get_dimensions(t))       # [3, 64, 64]
print(get_image_size(t))        # [64, 64]
print(get_image_num_channels(t))  # 3
```

Notes on internal mappings
--------------------------
- _interpolation_modes_from_int maps integer codes to InterpolationMode values, supporting a range used by various backends.
- pil_modes_mapping translates InterpolationMode values into PIL-specific mode indices for compatibility.

Terminology reference
---------------------
- PIL: Python Imaging Library, used for image loading and processing.
- accimage: Optional accelerated image loader; if available, its Image type is supported by to_tensor.
