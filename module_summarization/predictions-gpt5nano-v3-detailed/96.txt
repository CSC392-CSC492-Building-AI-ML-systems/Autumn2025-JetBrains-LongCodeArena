Getting Started with the mitogen.core Shared Utilities
=====================================================

Introduction
------------
This module defines functionality common to master and parent processes in Mitogen. It is intended to be sent to child contexts that are about to become a parent via recursive connections. The utilities address core tasks such as execution context handling, platform-specific considerations, and on-demand access to the Mitogen core source.

Overview
--------
- Provides a central set of helpers used by both master and parent contexts.
- Includes platform and OS checks (e.g., SELinux, open file limits).
- Exposes a safe, fallback mechanism for determining the Python executable.
- Supports on-demand preparation of the Mitogen core source for use in child contexts.
- Defines commonly used control messages (for error conditions and diagnostics).

Key Concepts
------------
- Pre-exec hook (_preexec_hook): A hook that can be registered prior to exec() in a new child process. Used to perform setup (for example, affinity changes) before the new process runs.
- Core source access (_get_core_source, get_core_source_partial): Mechanisms to obtain the Mitogen core code. get_core_source_partial prepares a compressed representation of the core source for transmission to children, with thread-safety to avoid contention.
- Platform flags and messages: Constants such as SELINUX_ENABLED, BROKER_SHUTDOWN_MSG, OPENPTY_MSG, and SYS_EXECUTABLE_MSG provide standardized behavior and messaging across platforms.

Public API
----------
Note: Some names are considered internal (prefixed with an underscore). The public-facing helpers are described here.

- get_log_level()
  - Returns the effective log level currently configured for the module.
  - Returns: int — the numeric log level (e.g., logging.INFO).

- get_sys_executable()
  - Returns the path to the Python executable to be used for child processes.
  - Behavior: If sys.executable is set, returns it; otherwise logs a warning and returns '/usr/bin/python'.
  - Returns: str — path to the Python interpreter.

- get_core_source_partial()
  - Ensures that a compressed, partial representation of mitogen.core is prepared for child contexts.
  - Behavior: Lazily initializes a global PartialZlib containing the core source (encoded as UTF-8). Uses a thread lock to prevent race conditions when accessed by multiple threads.
  - Returns: None. The partial is stored in a module-global cache for later use.

- _get_core_source()
  - Internal helper that returns the textual source of mitogen.core.
  - Returns: str — the source code of mitogen.core.

Constants and Messages
----------------------
- BROKER_SHUTDOWN_MSG
  - Description: Human-readable message indicating that a connection was cancelled because the associated Broker began to shut down.

- OPENPTY_MSG
  - Description: Error message used when PTY creation fails, typically due to system limits. Provides guidance on OS-specific tuning and configuration.

- SYS_EXECUTABLE_MSG
  - Description: Warning message emitted when sys.executable is not set, explaining that the child will default to '/usr/bin/python'.

- SELINUX_ENABLED
  - Description: Boolean flag indicating whether SELinux is enforcing. Used to influence certain IPC/PTY strategies to avoid problematic patterns on SELinux-enabled systems.

Platform and Compatibility Notes
--------------------------------
- The module includes compatibility shims for older Python versions (e.g., special handling for Python 2.4/2.5 when dealing with ioctl parameter sign conventions).
- On SELinux-enabled systems, certain IPC mechanisms (e.g., socketpairs) may be avoided as a precaution.

Usage Examples
--------------
Basic usage: determine the interpreter to use for a child process.

.. code-block:: python

    from mitogen.core import get_sys_executable

    python_path = get_sys_executable()
    print("Child Python executable:", python_path)

Preparing the Mitogen core source for a child context (thread-safe, lazy initialization):

.. code-block:: python

    from mitogen.core import get_core_source_partial

    # This call initializes the partial representation if needed.
    get_core_source_partial()
    # The prepared PartialZlib is stored in a module-global cache for later use.

Handling a potential pre-execution setup (illustrative, internal hook):

.. code-block:: python

    # This demonstrates the intended usage pattern; actual hooks are managed internally.
    # Define a pre-exec hook function (for example, to set resource limits or affinity).
    def my_preexec_hook():
        pass  # Implement setup before child process exec

    # Register the hook (note: in practice, the module provides controlled access to this hook)
    # _preexec_hook = my_preexec_hook

Edge Cases and Best Practices
-----------------------------
- Race conditions: get_core_source_partial() uses a global lock to avoid multiple threads attempting to initialize the partial simultaneously.
- SELinux considerations: If SELinux is enforcing, some IPC mechanisms may be avoided to prevent failures in constrained environments.
- Fallback safety: get_sys_executable() provides a safe default (/usr/bin/python) if the interpreter path cannot be determined.

Additional Notes
----------------
- The module is intended for internal use by Mitogen’s master/parent contexts; public APIs are designed for safe, minimal interactions.
- When upgrading, verify that any internal helpers or flags (e.g., SELINUX_ENABLED) remain compatible with your target OS version and Python interpreter.
