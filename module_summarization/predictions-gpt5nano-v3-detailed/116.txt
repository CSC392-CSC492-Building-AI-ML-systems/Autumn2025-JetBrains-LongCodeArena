Introduction
------------
This document provides a comprehensive reference for the AsdfSearchResult utility, which supports searching and manipulating values within ASDF trees. It covers the class API, helpers, and usage patterns for filtering, formatting, and updating leaf nodes discovered by a search.

API Overview
------------
- Core class: AsdfSearchResult
- Key capabilities:
  - Build up search queries across an ASDF tree
  - Apply formatting options to rendered trees (rows, columns, value display)
  - Replace leaf values that match the accumulated filters
  - Retrieve the single leaf node when a single result is found

Class: AsdfSearchResult
-----------------------
Synopsis
- Represents the result of an ASDF tree search.
- Encapsulates the matched identifiers, the node tree, and a collection of filter predicates that define the current search criteria.
- Supports chaining additional search criteria via the search method and rendering customization via format.

Constructor
- Parameters
  - identifiers: sequence describing the path to the current node(s) within the tree
  - node: the root of the tree (or subtree) being examined
  - filters: list of filter callables (default is an empty list)
  - parent_node: reference to the parent container for potential replacements
  - max_rows: maximum number of display lines (default from DEFAULT_MAX_ROWS)
  - max_cols: maximum line length (default from DEFAULT_MAX_COLS)
  - show_values: whether primitive values are displayed (default from DEFAULT_SHOW_VALUES)

Public API
1) format(max_rows=NotSet, max_cols=NotSet, show_values=NotSet)
- Purpose: Return a new AsdfSearchResult with updated display/formatting options.
- Behavior:
  - If any argument is NotSet, retain the current value.
  - Returns a new AsdfSearchResult with the updated formatting settings.
- Returns: AsdfSearchResult

2) search(key=NotSet, type=NotSet, value=NotSet, filter=None)
- Purpose: Narrow the search by additional criteria.
- Parameters
  - key: NotSet, str, or any object
    - NotSet: unconstrained
    - str: treat as a regular expression pattern to match the node key or index
    - other: node key/index must equal this value
  - type: NotSet, str, typing.Pattern, or builtins.type
    - NotSet: unconstrained
    - str: treat as a regular expression over the fully qualified node type
    - typing.Pattern: compiled pattern to test against fully qualified type
    - builtins.type: node must be an instance of this type
  - value: NotSet, str, or any object
    - NotSet: unconstrained
    - str: treat as a regular expression against the node value
    - other: node value must equal the queried value
  - filter: callable
    - Optional predicate that can inspect the node and its key/index
    - Signature: filter(node, identifier) or filter(node) or filter(node, identifier) depending on arity
    - Returns True to retain the node, False to exclude
- Behavior:
  - Patterns are compiled for string inputs
  - Values that are patterns will not be evaluated for container representations
  - A new filter predicate is appended to the existing filters
- Returns: AsdfSearchResult (new chained result)

3) replace(value)
- Purpose: Assign a new value to all leaf nodes that match the current search filters.
- Behavior:
  - Traverses the tree breadth-first to collect leaf nodes that pass all filters
  - Replaces leaf values in their respective parent containers with the provided value
- Returns: None

4) node (property)
- Purpose: Retrieve the leaf node for a single-result search
- Behavior:
  - If the search yields exactly one leaf node, returns that node
  - The exact behavior for zero or multiple results depends on implementation details not shown here
- Returns: leaf node or raises/returns None if not a single result

Internal Helpers
----------------
1) _maybe_compile_pattern(query)
- If query is a string, compiles it into a regular expression.
- If query is already a compiled pattern or another type, returns it unchanged.
- Returns: compiled pattern or original value

2) _safe_equals(a, b)
- Compares a and b with error tolerance.
- Catches exceptions and returns False if comparison fails.
- Returns: bool

3) _get_fully_qualified_type(value)
- Determines a value’s fully qualified type name.
- If the type’s module is builtins, returns just the type name.
- Otherwise returns "module.TypeName".
- Returns: str

Usage Patterns
--------------
Basic search
- Start from a known ASDF tree and narrow down by key, type, or value.

Example:
```python
# Pseudo-example: actual usage depends on the ASDF API in your environment
# Assume af is an opened ASDF file object and .search returns AsdfSearchResult
result = af.search(key="config")  # search by key name
```

Chaining search criteria
```python
# Narrow by key and type (as a fully qualified name) and by value
result = af.search(key="version").search(type="my_module.MyType", value=42)
```

Formatting the display
```python
# Create a formatted result that hides primitive values and limits lines
formatted = result.format(max_rows=100, max_cols=80, show_values=False)
```

Replacing matched leaves
```python
# Replace all matched leaf nodes with a new value
formatted.replace({"new": "value"})
```

Edge cases and considerations
-----------------------------
- NotSet semantics: If an argument to format or search is NotSet, the existing value is retained.
- Type matching:
  - Strings are treated as regex patterns against the fully qualified type name.
  - Builtins.type can be used to enforce isinstance-type checks.
- Value matching:
  - Strings are treated as regex patterns against the node’s string representation.
  - Containers (e.g., lists, dicts) behave specially when value is a string pattern; the implementation avoids matching on the string representation of containers.
- Filters:
  - Additional filters are accumulated; all must pass for a node to be included.
  - The filter function can inspect the node and its key/index (identifier) for complex criteria.
- Replacement:
  - The replace method operates on leaves only and uses a breadth-first traversal to identify eligible nodes.
  - Ensure you have the appropriate permissions to modify the underlying tree structure.
