Transforming the prediction target

This module focuses on techniques for transforming the target variable y in regression tasks. Transforming the target can stabilize variance, handle skewed distributions, or linearize relationships, making it easier for models to learn. The primary tool provided for this purpose is TransformedTargetRegressor, which wraps a regressor and applies a transformation to y during fitting and prediction.

Key concepts

- Target transformation vs. feature transformation: Target transformations modify the response variable y (not the input features X). This is complementary to feature preprocessing and can lead to improved performance and interpretability.
- Invertible transformations: The transformation should be invertible so that predictions can be mapped back to the original target space for interpretation and evaluation.
- Monotonic transformations: Monotonic and differentiable transformations are commonly used (e.g., log, log1p, power transforms). Non-invertible or non-monotonic transformations are not suitable for this wrapper.

API overview

- TransformedTargetRegressor
  - Purpose: Wraps a regressor and applies a transformation to the target y before fitting, with an inverse transformation applied to predictions to return them to the original scale.
  - Key behavior:
    - During fit: y is transformed by func and the wrapped regressor is trained on (X, transformed_y).
    - During predict: the regressor predicts in the transformed space, and inverse_func is applied to map predictions back to the original space.
  - Important note: TransformedTargetRegressor supports single-target regression. For multi-output targets, apply per-target or use an appropriate wrapper.

Parameters

- regressor: The underlying regressor to be wrapped (e.g., LinearRegression, RandomForestRegressor).
- func: Callable that transforms y for fitting (e.g., np.log, np.log1p, np.sqrt). Must be invertible via inverse_func.
- inverse_func: Callable that inverts func (e.g., np.exp, np.expm1, lambda x: x**2). Used to transform predictions back to the original space.
- check_inverse: If True, validate that inverse_func approximately inverts func on a sample of y. Default is True.
- transform_y, inverse_transform_y: Deprecated aliases in some versions; use func and inverse_func instead.
- Additional fit/predict behavior is inherited from the wrapped regressor.

Return values

- fit returns self
- predict returns predictions mapped back to the original target space via inverse_func
- score uses the transformed target during fitting as configured, with predictions mapped back for evaluation if applicable

Usage examples

- Example 1: Regression with log transformation of the target

```python
from sklearn.compose import TransformedTargetRegressor
from sklearn.linear_model import LinearRegression
import numpy as np

# Example data
X = np.random.randn(100, 3)
y = np.exp(np.random.randn(100))  # positive target with skew

model = TransformedTargetRegressor(
    regressor=LinearRegression(),
    func=np.log1p,        # transform y for training
    inverse_func=np.expm1   # inverse transform for predictions
)

model.fit(X, y)
y_pred = model.predict(X)
```

- Example 2: Regression with a square-root transform

```python
from sklearn.compose import TransformedTargetRegressor
from sklearn.ensemble import RandomForestRegressor

model = TransformedTargetRegressor(
    regressor=RandomForestRegressor(n_estimators=200, random_state=0),
    func=np.sqrt,
    inverse_func=lambda x: x**2
)

model.fit(X, y)
y_pred = model.predict(X)
```

Edge cases and considerations

- Invertibility: Ensure func and inverse_func are true inverses (or close) over the range of y values you expect.
- Non-positive targets: Functions like log require positive y. Use transforms such as log1p for y that can include zero, or apply a shift if appropriate.
- Multi-output targets: TransformedTargetRegressor is designed for single-target regression. For multi-target problems, apply the transformation to each target independently (e.g., wrap multiple regressors or use a per-target transformer).
- Interpretability: Coefficients and feature importances relate to the transformed space. Inference in the original space requires applying the inverse transformation to predictions.
- Validation: If check_inverse is True, the wrapper will verify that the inverse_func approximately inverts func on a sample of y; if not, it will raise a warning or error depending on implementation.

Practical notes

- Use target transformation when residuals or errors are heteroscedastic, or when the relationship between X and y is multiplicative or non-linear in y.
- Combine with a robust regressor if the transformed target reduces sensitivity to outliers.
