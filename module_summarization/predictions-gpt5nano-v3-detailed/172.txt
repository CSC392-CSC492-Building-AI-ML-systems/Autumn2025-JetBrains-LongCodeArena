SANS to SESANS conversion
=========================

Overview
--------
This module provides utilities related to converting between SANS representations and SESANS mappings. It includes a small parameter-inspection utility used by sasmodels to enumerate model parameters, as well as a mathematical core that exposes an elliptic-function based implementation intended for coordinate transformations (e.g., a Guyou-like projection) that can be employed in SESANS coordinate handling.

Public components
-----------------
- Parameter inspection utilities (sasmodels.list_pars):
  - find_pars(kind=None)
    - Collects all kernel parameters used across models.
    - Returns a dictionary of the form {parameter_name: [model, model, ...]}.
    - If kind is provided, only parameters of that kind are included.
  - list_pars(names_only=True, kind=None)
    - Prints the parameters found.
    - If names_only is True, prints only parameter names.
    - If names_only is False, prints each parameter followed by the models that use it.
  - main()
    - CLI entry point for listing parameters.
    - Options:
      - -v / --verbose: list models that use each parameter in addition to the parameter name.
      - kind: restricts to a specific parameter kind (volume, orientation, sld, none, any).
- Elliptic-function based core for coordinate conversion (ellipticJi and helpers):
  - ellipticJi(u, v, m)
    - Returns [sn, cn, dn](u + i v | m).
    - Accepts scalars or broadcastable arrays; returns a scalar if inputs are scalar.
  - _ellipticJi_real(u, m)
    - Helper for real-axis evaluation of ellipticJi.
  - _ellipticJi_imag(v, m)
    - Helper for imaginary-axis evaluation of ellipticJi.
  - _ellipticJi(u, v, m)
    - Core computation for mixed real/imaginary inputs.
  - Note: These functions rely on SciPy’s elliptic J and related functions.
  - The implementation supports broadcasting and handles basic edge cases for real, imaginary, and mixed inputs.

Usage
-----
Parameter listing utility (CLI)
- List all parameter names used across models:
  python -m sasmodels.list_pars
- List parameter names with the models that use each parameter:
  python -m sasmodels.list_pars -v volume

Elliptic-function based core (programmatic usage)
- Import and evaluate a simple case:
  from sans_to_sesans import ellipticJi
  import numpy as np
  u = 0.0
  v = 0.0
  m = 0.5
  sn, cn, dn = ellipticJi(u, v, m)
- For vector inputs:
  u = np.array([0.0, 0.5])
  v = np.array([0.0, 0.2])
  m = np.array([0.5, 0.7])
  sn, cn, dn = ellipticJi(u, v, m)

Important considerations and edge cases
- Dependency: The ellipticJi implementation relies on SciPy's ellipj; ensure SciPy is available in your environment.
- Broadcasting: The functions are designed to accept scalar or array inputs and broadcast accordingly.
- Parameter kinds: When using the CLI to list parameters, a kind can be provided to restrict the results (volume, orientation, sld, none, or any). The special value any corresponds to listing all parameters; none lists only those not assigned to a specific kind.
- Private helpers: The _ellipticJi_* helpers are intended for internal use. They illustrate the decomposition of the mixed real/imaginary evaluation and are not part of the public API.

Notes
-----
- The module includes legacy copyright and provenance text from a d3-geo-projection lineage, adapted here for the elliptic-function core.
- The current implementation in this file is a lightweight foundation for coordinate transformations that may be employed in SESANS mappings; it is not a complete, end-to-end SANS-to-SESANS transformer by itself. Integrations should build on top of the publicly exposed functions described above.
