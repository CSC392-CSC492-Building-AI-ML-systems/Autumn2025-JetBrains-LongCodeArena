Schema generation
=================

Overview
--------
This module provides utilities to generate and apply the SQL schema for a Tortoise ORM
database backend. It relies on a schema generator exposed by the database client to build
the appropriate CREATE statements and to apply them to the underlying database.

Key concepts
------------
- BaseDBAsyncClient: Abstract database client type. The actual backend client provides a
  schema_generator capable of producing SQL for the schema and executing it.
- schema_generator: Created via client.schema_generator(client). It exposes:
  - get_create_schema_sql(safe): Returns the SQL statement to create the schema. If
    safe is true, the SQL should use safe-create semantics (e.g., CREATE TABLE IF NOT EXISTS).
  - generate_from_string(schema): Executes the provided SQL against the database.

Functions
---------

get_schema_sql(client: "BaseDBAsyncClient", safe: bool) -> str
- Purpose: Generate the SQL schema string for the given client.
- Parameters:
  - client: The DB client to generate Schema SQL for (type: BaseDBAsyncClient).
  - safe: If true, the generated SQL should create tables only if they do not already exist.
- Returns: A string containing the SQL schema. If no schema is produced, returns an empty string.
- Behavior notes:
  - Internally, it creates a generator via client.schema_generator(client) and calls
    get_create_schema_sql(safe) on it.

generate_schema_for_client(client: "BaseDBAsyncClient", safe: bool) -> None
- Purpose: Generate and apply the SQL schema directly to the given client.
- Parameters:
  - client: The DB client to generate and apply schema for (BaseDBAsyncClient).
  - safe: If true, uses safe-create semantics.
- Returns: None (applies the schema asynchronously).
- Behavior notes:
  - Uses the same generator workflow as get_schema_sql to obtain the schema.
  - Logs the schema at debug level.
  - If the generated schema is non-empty, it executes generator.generate_from_string(schema).

Usage examples
--------------

Example 1: Generate the schema SQL string for a client
```python
from tortoise.backends.base.client import BaseDBAsyncClient

# Assume `client` is an instance of a concrete BaseDBAsyncClient
schema_sql = get_schema_sql(client, safe=True)
print(schema_sql)
```

Example 2: Generate and apply the schema to the database
```python
import asyncio
from tortoise.backends.base.client import BaseDBAsyncClient

# Assume `client` is an instance of a concrete BaseDBAsyncClient

async def apply_schema(client: BaseDBAsyncClient, safe: bool = True) -> None:
    await generate_schema_for_client(client, safe)

# Run in an async event loop
# asyncio.run(apply_schema(client, safe=True))
```

Edge cases and considerations
------------------------------
- Empty schema: If the generator returns an empty string, no schema is applied.
- Safe flag: The safe parameter influences the SQL generation (e.g., using IF NOT EXISTS). Ensure your backend supports the expected semantics.
- Client availability: The module expects a functioning database client. Ensure the client is connected and the backend schema generator is available.
- Logging: The operation logs the SQL schema at debug level for troubleshooting.

See also
--------

- Schema generator interface: get_create_schema_sql and generate_from_string.
