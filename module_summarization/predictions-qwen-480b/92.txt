====================
distutils
====================

.. module:: numpy.distutils

The `numpy.distutils` module is a collection of tools for building and distributing Python packages, particularly those that include compiled extensions. It extends the standard `distutils` module with support for Fortran code, better handling of dependencies, and integration with BLAS/LAPACK libraries.

This module provides utilities for configuration management, source file handling, and build process customization. It also includes functions for colorizing terminal output and managing temporary directories during builds.

Functions
=========

get_num_build_jobs()
--------------------

Get number of parallel build jobs set by the --parallel command line argument of setup.py. If the command did not receive a setting, the environment variable NPY_NUM_BUILD_JOBS is checked. If that is unset, return the number of processors on the system, with a maximum of 8 (to prevent overloading the system if there are a lot of CPUs).

Returns:
    int: Number of parallel jobs that can be run.

---------------------------

get_path_from_frame(frame, parent_path=None)
--------------------------------------------

Return path of the module given a frame object from the call stack. Returned path is relative to parent_path when given, otherwise it is absolute path.

Parameters:
    frame (frame): Frame object from the call stack.
    parent_path (str, optional): Base path for computing the relative path.

Returns:
    str: Absolute or relative path of the module.

njoin(*path)
------------

Join two or more pathname components:
- Convert a /-separated pathname to one using the OS's path separator.
- Resolve `..` and `.` from path.

Either passing n arguments as in njoin('a','b'), or a sequence of n names as in njoin(['a','b']) is handled, or a mixture of such arguments.

Parameters:
    *path (str or list): Path components to join.

Returns:
    str: Joined and normalized path.
