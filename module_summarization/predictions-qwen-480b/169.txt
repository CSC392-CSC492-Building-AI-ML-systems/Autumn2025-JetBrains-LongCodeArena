Contributing to Qiskit Metal
============================

Qiskit Metal is an open-source project aimed at enabling researchers and developers to design, simulate, and optimize superconducting quantum circuits. Contributions from the community are essential to the continued growth and improvement of the project. This guide outlines how you can contribute to Qiskit Metal.

Ways to Contribute
------------------

There are several ways to contribute to Qiskit Metal:

1. **Code Contributions**: Help improve the core functionality, add new features, or fix bugs.
2. **Documentation**: Improve existing documentation or create new guides and tutorials.
3. **Testing**: Write tests to ensure code quality and help identify issues.
4. **Issue Reporting**: Report bugs or suggest enhancements by creating issues on GitHub.
5. **Community Support**: Answer questions on forums, help new users, and participate in discussions.

Getting Started
---------------

To get started with contributing to Qiskit Metal, follow these steps:

1. **Fork the Repository**: Fork the `Qiskit Metal <https://github.com/Qiskit/qiskit-metal>`_ repository on GitHub.
2. **Clone Your Fork**: Clone your forked repository to your local machine.
3. **Set Up Your Environment**: Install the necessary dependencies and set up your development environment.
4. **Create a Branch**: Create a new branch for your feature or bugfix.
5. **Make Changes**: Implement your changes, ensuring that your code follows the project's style guidelines.
6. **Write Tests**: Add tests for any new functionality or bug fixes.
7. **Commit and Push**: Commit your changes and push them to your fork.
8. **Submit a Pull Request**: Open a pull request to the main Qiskit Metal repository.

Code Style and Guidelines
-------------------------

Qiskit Metal follows the `Qiskit Style Guide <https://github.com/Qiskit/qiskit/blob/master/QISKIT_STYLE_GUIDE.md>`_ and uses `Black <https://black.readthedocs.io/en/stable/>`_ for code formatting. Please ensure your code adheres to these standards before submitting a pull request.

Documentation
-------------

Improving documentation is one of the easiest ways to contribute. You can help by:

- Fixing typos or grammatical errors.
- Clarifying existing documentation.
- Adding examples or tutorials.
- Writing new guides for features or tools.

Documentation is written in reStructuredText (`.rst`) format and built using Sphinx.

Testing
-------

Qiskit Metal uses `pytest <https://docs.pytest.org/en/latest/>`_ for testing. When contributing code, please include tests that cover your changes. Tests should be placed in the `test` directory and follow the existing structure.

Reporting Issues
----------------

If you encounter a bug or have a feature request, please open an issue on the `Qiskit Metal GitHub repository <https://github.com/Qiskit/qiskit-metal/issues>`_. When reporting a bug, include as much detail as possible, including steps to reproduce the issue and any relevant error messages.

Community
---------

Join the Qiskit community to connect with other contributors and users:

- `Qiskit Slack <https://qiskit.slack.com/>`_
- `Qiskit GitHub Discussions <https://github.com/Qiskit/qiskit-metal/discussions>`_

By contributing to Qiskit Metal, you agree to abide by the `Qiskit Code of Conduct <https://github.com/Qiskit/qiskit/blob/master/CODE_OF_CONDUCT.md>`_.
