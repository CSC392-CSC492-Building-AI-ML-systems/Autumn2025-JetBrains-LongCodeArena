Generate Documentation for This Code
====================================

This module provides a set of functions for generating experimental plans used in data acquisition systems, particularly within the context of the Bluesky project. These plans define sequences of actions that control hardware such as detectors and motors to perform scans and measurements.

Functions
---------

### `count(detectors, num=1, delay=None, *, per_shot=None, md=None)`

Takes one or more readings from a list of detectors.

**Parameters:**
- `detectors` (list): A list of 'readable' objects.
- `num` (int, optional): Number of readings to take; default is 1. If None, capture data until canceled.
- `delay` (iterable or scalar, optional): Time delay in seconds between successive readings; default is 0.
- `per_shot` (callable, optional): Hook for customizing action of inner loop (messages per step). Expected signature: `def f(detectors: Iterable[OphydObj]) -> Generator[Msg]:`.
- `md` (dict, optional): Metadata.

**Notes:**
If `delay` is an iterable, it must have at least `num - 1` entries or the plan will raise a `ValueError` during iteration.

---

### `list_scan(detectors, *args, per_step=None, md=None)`

Scans over one or more variables in steps simultaneously (inner product).

**Parameters:**
- `detectors` (list): A list of 'readable' objects.
- `*args`: For one dimension: `motor, [point1, point2, ...]`. In general:
  ```python
  motor1, [point1, point2, ...],
  motor2, [point1, point2, ...],
  ...
  motorN, [point1, point2, ...]
  ```
  Motors can be any 'settable' object (motor, temperature controller, etc.).
- `per_step` (callable, optional): Hook for customizing action of inner loop (messages per step). Expected signature: `f(detectors, motor, step) -> plan (a generator)`.
- `md` (dict, optional): Metadata.

**See Also:**
- :func:`bluesky.plans.rel_list_scan`
- :func:`bluesky.plans.list_grid_scan`
- :func:`bluesky.plans.rel_list_grid_scan`

---

### `rel_list_scan(detectors, *args, per_step=None, md=None)`

Scans over one or more variables in steps relative to their current positions.

**Parameters:**
- `detectors` (list): A list of 'readable' objects.
- `*args`: For one dimension: `motor, [point1, point2, ...]`. In general:
  ```python
  motor1, [point1, point2, ...],
  motor2, [point1, point2, ...],
  ...
  motorN, [point1, point2, ...]
  ```
  Motors can be any 'settable' object (motor, temperature controller, etc.). Points are relative to the current location.
- `per_step` (callable, optional): Hook for customizing action of inner loop (messages per step). Expected signature: `f(detectors, motor, step)`.
- `md` (dict, optional): Metadata.

**See Also:**
- :func:`bluesky.plans.list_scan`
- :func:`bluesky.plans.list_grid_scan`
- :func:`bluesky.plans.rel_list_grid_scan`
