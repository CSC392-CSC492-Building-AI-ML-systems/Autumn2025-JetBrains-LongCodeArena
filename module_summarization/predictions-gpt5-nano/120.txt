Channel Options
===============

Overview
--------
Channel options define how data fields and values are mapped to visual properties of marks through encoding channels in Vega-Lite. Each channel can receive a field definition (FieldDef), a value definition (ValueDef), or a conditional encoding. Channel options control data type, transformations, scales, axes, legends, and other rendering details.

Common encoding properties
--------------------------
The following properties can be specified for most channels (as part of a FieldDef/ValueDef or a conditional encoding):

- field / datum: the data field to map (or a constant value for ValueDef)
- type: data type for the channel (quantitative, temporal, ordinal, nominal)
- aggregate: aggregation to apply (mean, sum, min, max, count, etc.)
- bin: binning configuration (boolean or BinParams)
- timeUnit: time unit for temporal fields (year, month, day, etc.)
- scale: scale configuration (domain, range, nice, clamp, zero, etc.)
- axis: axis configuration for x/y channels (title, labels, grid, etc.)
- legend: legend configuration for color/size/shape channels (title, format, orientation, etc.)
- sort: sorting specification for discrete channels
- stack: stacking configuration for stackable channels (e.g., y with a quantitative field)
- format: formatting rules for axis or tooltip
- condition: conditional encoding (e.g., show one value when a condition is true, another when false)
- scale properties can be nested (e.g., scale.domain, scale.range)

Per-channel details
-------------------
X channel
- Maps data to horizontal position.
- Common uses: quantitative x fields (Q), temporal fields (T), or ordinal/nominal (N) with appropriate type.
- Typical options: type, aggregate, bin, timeUnit, scale, axis, legend, format, stack (where applicable).

Y channel
- Maps data to vertical position.
- Similar options to X: type, aggregate, bin, timeUnit, scale, axis, legend, format, stack.

Color channel
- Maps data to color hue/intensity.
- Supports qualitative/nominal/ordinal/quantitative mappings.
- Options: type, scale (including color schemes), legend, condition, bin/aggregation for derived color values.

Size channel
- Maps data to the size of marks.
- Options: type, scale, legend, axis (where applicable), aggregate, timeUnit, bin.

Shape channel
- Maps data to mark shapes (where supported by the mark type).
- Options: type, scale, legend, sort, aggregate, timeUnit.

Opacity channel
- Maps data to transparency.
- Options: type, scale, legend, aggregate, timeUnit, bin.

Tooltip channel
- Maps data to interactive tooltips.
- Options: multiple fields can be included; can use format and aggregation as needed.

Detail channel
- Encodes additional data points that do not affect position or appearance directly but can influence aggregation, grouping, or interaction.
- Options: multiple fields; typically used to create derived groups.

Text channel
- Maps data to text labels for text marks.
- Options: type, scale, axis, legend, format, aggregation as needed.

Notes
-----
- Channel options are defined by the underlying Vega-Lite FieldDef/ValueDef schemas. Altair (as used by Vega-Lite) exposes these through the encode() interface, e.g., x=..., y=..., color=..., size=..., shape=..., tooltip=..., etc.
- For JSON/YAML specifications, many channel options mirror Vega-Lite's encoding structure. See Vega-Lite v5 documentation for exact field names and allowed values.
- Some options are global (e.g., top-level config) rather than channel-specific; this section focuses on per-channel encoding options.

Examples
--------
Basic numeric mapping with explicit types and scales:

.. code-block:: python

  import altair as alt
  chart = (
      alt.Chart(data)
      .mark_point()
      .encode(
          x=alt.X('Horsepower:Q', bin=True, axis=alt.Axis(title='Horsepower')),
          y=alt.Y('Miles_per_Gallon:Q', aggregate='mean', axis=alt.Axis(title='MPG')),
          color=alt.Color('Origin:N', scale=alt.Scale(scheme='category10')),
          tooltip=['Name', 'Origin', 'Miles_per_Gallon']
      )
  )

Temporal and aggregated encoding:

.. code-block:: python

  chart = (
      alt.Chart(data)
      .mark_line()
      .encode(
          x=alt.X('Year(Timestamp):T', timeUnit='year', axis=alt.Axis(title='Year')),
          y=alt.Y('Sales:Q', aggregate='sum', axis=alt.Axis(title='Total Sales')),
          color=alt.Color('Region:N', legend=alt.Legend(title='Region'))
      )
  )

This file describes the options you can configure for encoding channels that determine how data is mapped to the visual properties of Vega-Lite charts.
