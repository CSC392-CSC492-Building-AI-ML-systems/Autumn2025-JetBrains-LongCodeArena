# Bluesky Plans Documentation

## count

```{eval-rst}
.. autofunction:: count
```

**Description:**
Take one or more readings from detectors.

**Parameters:**
- `detectors` (list): List of 'readable' objects
- `num` (integer, optional): Number of readings to take; default is 1. If None, capture data until canceled
- `delay` (iterable or scalar, optional): Time delay in seconds between successive readings; default is 0
- `per_shot` (callable, optional): Hook for customizing action of inner loop (messages per step)
- `md` (dict, optional): Metadata

**Notes:**
If `delay` is an iterable, it must have at least `num - 1` entries or the plan will raise a `ValueError` during iteration.

## list_scan

```{eval-rst}
.. autofunction:: list_scan
```

**Description:**
Scan over one or more variables in steps simultaneously (inner product).

**Parameters:**
- `detectors` (list): List of 'readable' objects
- `*args`: Motor and position list pairs in the format: `motor, [point1, point2,...]`
- `per_step` (callable, optional): Hook for customizing action of inner loop (messages per step)
- `md` (dict, optional): Metadata

**See Also:**
- `rel_list_scan`
- `list_grid_scan`
- `rel_list_grid_scan`

**Notes:**
All position lists must be of the same length. Motors can be any 'settable' object (motor, temp controller, etc.)

## rel_list_scan

```{eval-rst}
.. autofunction:: rel_list_scan
```

**Description:**
Scan over one variable in steps relative to current position.

**Parameters:**
- `detectors` (list): List of 'readable' objects
- `*args`: Motor and relative position list pairs in the format: `motor, [point1, point2,...]`
- `per_step` (callable, optional): Hook for customizing action of inner loop (messages per step)
- `md` (dict, optional): Metadata

**See Also:**
- `list_scan`
- `list_grid_scan`
- `rel_list_grid_scan`

**Notes:**
This function appears to be incomplete (marked with TODO in the code). Points are relative to the current location of each motor.
