SANS to SESANS Conversion
=========================

This module provides functionality for converting between Small Angle Neutron Scattering (SANS) data and Spin-Echo Small Angle Neutron Scattering (SESANS) data.

Coordinate Transformation
-------------------------

The conversion utilizes the Guyou map projection for coordinate transformations between latitude-longitude and projected coordinates. The implementation includes special elliptic functions for handling the mathematical transformations.

Elliptic Functions
~~~~~~~~~~~~~~~~~

The module provides several elliptic function implementations:

- ``ellipticJi(u, v, m)``: Returns [sn, cn, dn](u + iv|m) for complex arguments
- ``_ellipticJi_real(u, m)``: Handles real component transformations
- ``_ellipticJi_imag(v, m)``: Handles imaginary component transformations
- ``_ellipticJi(u, v, m)``: Main implementation for mixed real/imaginary components

These functions are essential for the mathematical transformations required in the SANS to SESANS conversion process.

Parameter Management
-------------------

The module includes utilities for managing parameters across different scattering models:

- ``find_pars(kind=None)``: Finds all parameters in all models and returns a reference table mapping parameters to the models that use them
- ``list_pars(names_only=True, kind=None)``: Prints all parameters across models, with options for verbose output

The parameter system supports filtering by parameter type including:
- Volume parameters
- Orientation parameters
- SLD (Scattering Length Density) parameters

Usage
-----

The module can be used both programmatically and via command line:

Programmatic usage:
```python
from sans_to_sesans import find_pars, list_pars

# Get all parameters
parameters = find_pars()

# List parameters with model information
list_pars(names_only=False)
```

Command line usage:
```bash
python -m sans_to_sesans.list_pars [-v] [volume|orientation|sld|none|any]
```

The ``-v`` flag provides verbose output showing which models use each parameter.

Mathematical Foundation
-----------------------

The conversion relies on advanced mathematical operations including elliptic integrals and Jacobi elliptic functions. The implementation uses both numpy for basic mathematical operations and scipy for special functions, with support for mpmath as an alternative implementation.

Note: The coordinate conversion functions work with degrees rather than radians for angular measurements.
