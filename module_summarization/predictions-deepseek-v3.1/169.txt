===================================
Contributing to Qiskit Metal
===================================

We welcome contributions to Qiskit Metal! This document provides guidelines and instructions for contributing to the project.

Getting Started
---------------

- Fork the repository on GitHub.
- Clone your fork locally.
- Install the development dependencies (see the main README for instructions).
- Create a branch for your changes.

Code Style and Guidelines
-------------------------

- Follow PEP 8 for Python code style.
- Use descriptive variable and function names.
- Include docstrings for all public functions and classes, following the format used in existing code.
- Add unit tests for new functionality.
- Ensure all tests pass before submitting a pull request.

Documentation
-------------

- Update relevant documentation when adding or modifying features.
- Use reStructuredText for documentation files.
- Include examples where appropriate.

Submitting Changes
------------------

- Push your changes to your fork.
- Submit a pull request to the main repository.
- Provide a clear description of the changes and any related issues.
