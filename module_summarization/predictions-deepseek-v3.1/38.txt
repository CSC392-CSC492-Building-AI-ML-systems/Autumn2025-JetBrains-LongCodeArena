.. _fields:

Fields
======

The ``Field`` class is the base for all field types in Tortoise ORM. It defines the common attributes and methods that all fields share.

Examples and Usage
-----------------

Defining a basic field:

.. code-block:: python

    from tortoise import fields, models

    class MyModel(models.Model):
        # A simple integer field
        my_field = fields.IntField()

        # A string field with a default value
        name = fields.CharField(max_length=255, default="Unknown")

        # A nullable boolean field
        is_active = fields.BooleanField(null=True)

        # A field with a custom source field name in the database
        custom_name = fields.CharField(max_length=100, source_field='db_column_name')

Using primary keys:

.. code-block:: python

    class User(models.Model):
        # Custom primary key
        user_id = fields.IntField(pk=True)
        
        # Or use the default auto-incrementing primary key named 'id'

Setting unique and indexed fields:

.. code-block:: python

    class Product(models.Model):
        # Unique product code
        code = fields.CharField(max_length=50, unique=True)
        
        # Indexed price field
        price = fields.DecimalField(max_digits=10, decimal_places=2, index=True)

Using field defaults:

.. code-block:: python

    import uuid
    from datetime import datetime

    class Document(models.Model):
        # Default with callable
        uuid = fields.UUIDField(default=uuid.uuid4)
        
        # Default with static value
        created_at = fields.DatetimeField(default=datetime.now)
        
        # Default with lambda
        status = fields.CharField(default=lambda: "pending")

Field descriptions:

.. code-block:: python

    class Customer(models.Model):
        email = fields.CharField(
            max_length=255,
            description="Customer's primary email address for communication"
        )

Note: The ``Field`` class itself is typically not used directly. Instead, use the specific field types like ``IntField``, ``CharField``, ``BooleanField``, etc., which inherit from ``Field`` and provide appropriate type-specific implementations.
