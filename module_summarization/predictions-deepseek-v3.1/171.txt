Calculator Interface
===================

The calculator interface provides utilities for listing parameters used across models and converting between coordinate systems.

Parameter Listing
-----------------

The module includes functionality to list all parameters used in models, with options to filter by parameter type and to show which models use each parameter.

### Functions

- `find_pars(kind=None)`: Returns a dictionary mapping each parameter to a list of models that use it. The `kind` parameter can filter parameters by type (e.g., 'volume', 'orientation', 'sld').

- `list_pars(names_only=True, kind=None)`: Prints all parameters. If `names_only` is True, only parameter names are printed; otherwise, it also lists the models containing each parameter.

### Command-Line Usage

The module can be run as a script to list parameters:

    python -m sasmodels.list_pars [-v] [kind]

Options:
- `-v` or `--verbose`: List models containing each parameter.
- `kind`: Optional filter for parameter type (choices: 'volume', 'orientation', 'sld', 'none', 'any').

Coordinate Conversion
---------------------

The module also includes functions for converting between latitude-longitude and Guyou map coordinates using elliptic functions.

### Functions

- `ellipticJi(u, v, m)`: Returns the Jacobi elliptic functions [sn, cn, dn] for complex input u + iv with parameter m. Handles real, imaginary, and mixed cases.

- `_ellipticJi_real(u, m)`: Helper function for real inputs.
- `_ellipticJi_imag(v, m)`: Helper function for purely imaginary inputs.
- `_ellipticJi(u, v, m)`: Helper function for general complex inputs.

### Dependencies

- numpy: Used for array operations and mathematical functions.
- scipy.special: Provides `ellipj` (Jacobi elliptic functions) and `ellipkinc` (incomplete elliptic integral of the first kind).

### Notes

The coordinate conversion functions are based on a conversion from JavaScript (D3 geo projection library) to Python, with adjustments for degrees instead of radians.
