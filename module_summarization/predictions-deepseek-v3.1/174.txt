Array API Support
=================

scikit-learn provides experimental support for the `Array API standard`_ for some of its estimators and functions. This allows using scikit-learn with different array libraries that conform to the standard, such as CuPy, PyTorch, and others, in addition to the traditional NumPy arrays.

.. _Array API standard: https://data-apis.org/array-api/latest/

Supported Estimators
-------------------

The following multi-output estimators currently support the Array API:

- ``MultiOutputRegressor``
- ``MultiOutputClassifier``
- ``ClassifierChain``
- ``RegressorChain``

These meta-estimators can work with any base estimator that supports the Array API specification and implements the required ``fit`` and ``predict`` methods.

Requirements
------------

To use Array API compatible arrays with scikit-learn, you need:

- Python 3.9 or newer
- NumPy 1.22 or newer (for NumPy arrays)
- A compatible array library that implements the Array API standard
- scikit-learn 1.3 or newer

Usage
-----

To use Array API compatible arrays, simply pass them to the supported estimators:

.. code-block:: python

    from sklearn.multioutput import MultiOutputRegressor
    from sklearn.linear_model import LinearRegression
    import cupy as cp  # or other array library

    # Create Array API compatible arrays
    X = cp.random.random((100, 5))
    y = cp.random.random((100, 3))

    # Use with multi-output estimator
    estimator = MultiOutputRegressor(LinearRegression())
    estimator.fit(X, y)
    predictions = estimator.predict(X)

Current Limitations
-------------------

- Array API support is currently experimental and may change in future releases
- Not all scikit-learn estimators support the Array API standard yet
- Some advanced functionality may not be available with all array types
- Performance characteristics may differ from traditional NumPy arrays

The Array API support is continuously being improved, and more estimators will be added in future releases.
