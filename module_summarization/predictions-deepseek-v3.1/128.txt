Miscellaneous Features
=====================

This section covers miscellaneous classes and utilities used internally by
`astropy.io.fits` for verification, error handling, and notification systems.

Verification Framework
----------------------

The verification framework provides a consistent way to validate and correct
data structures with various levels of strictness.

.. autoclass:: astropy.io.fits.VerifyError
   :members:

.. autoclass:: astropy.io.fits.VerifyWarning
   :members:

The `VERIFY_OPTIONS` list contains all supported verification options:

- ``"ignore"``: Ignore all verification issues
- ``"warn"``: Issue warnings for all verification issues
- ``"exception"``: Raise exceptions for all verification issues  
- ``"fix"``: Fix issues and report actions taken
- ``"silentfix"``: Fix issues without reporting
- Combined options: Any combination of fix options (``"fix"``/``"silentfix"``)
  with reporting options (``"ignore"``/``"warn"``/``"exception"``)

Notification System
-------------------

The notification system allows objects to register listeners that are notified
of changes to the object.

.. autoclass:: astropy.io.fits.NotifierMixin
   :members: _add_listener, _notify

Example usage:

.. code-block:: python

   class Widget(NotifierMixin):
       state = 1
       def __init__(self, name):
           self.name = name
       def update_state(self):
           self.state += 1
           self._notify('state_change', self.state)

   class Listener:
       def _update_state_change(self, new_state):
           print(f"Widget state changed to {new_state}")

   widget = Widget('test')
   listener = Listener()
   widget._add_listener(listener)
   widget.update_state()  # Prints: Widget state changed to 2

Utility Functions and Constants
-------------------------------

- ``path_like``: Tuple of types representing path-like objects (str, bytes, os.PathLike)
- ``cmp``: Three-way comparison function equivalent to (a > b) - (a < b)
- ``all_integer_types``: Tuple of all integer types (int, numpy.integer)

Note: These components are primarily for internal use but may be useful for
extending `astropy.io.fits` functionality.
