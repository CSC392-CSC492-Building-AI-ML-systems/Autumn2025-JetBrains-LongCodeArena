Pipelines and Composite Estimators
==================================

The :mod:`sklearn.pipeline` module implements utilities to build composite estimators
as chains of transforms and estimators. These tools enable streamlined preprocessing
and modeling workflows while ensuring proper data handling during cross-validation
and parameter tuning.

Pipeline
--------

The :class:`~sklearn.pipeline.Pipeline` class allows sequential application of
transforms with a final estimator. This enables assembling several steps that can be
cross-validated together while setting different parameters for each component.

Key Features
~~~~~~~~~~~~

- **Sequential Processing**: Applies a list of transforms in order, followed by a final estimator
- **Parameter Tuning**: Enables setting parameters of various steps using double underscore syntax (e.g., `stepname__paramname`)
- **Caching Support**: Optional caching of fitted transformers using the `memory` parameter
- **Time Tracking**: Optional verbose mode to display fitting time for each step

Basic Usage
~~~~~~~~~~~

.. code-block:: python

    from sklearn.pipeline import Pipeline
    from sklearn.preprocessing import StandardScaler
    from sklearn.svm import SVC
    
    pipe = Pipeline([
        ('scaler', StandardScaler()), 
        ('svc', SVC())
    ])
    
    # Fit and score like any other estimator
    pipe.fit(X_train, y_train)
    score = pipe.score(X_test, y_test)
    
    # Set parameters using __ syntax
    pipe.set_params(svc__C=10)

Parameters
~~~~~~~~~~

- **steps**: List of (name, transform) tuples that are chained in sequential order
- **memory**: Used to cache fitted transformers (string path or joblib.Memory object)
- **verbose**: If True, prints time elapsed while fitting each step

Attributes
~~~~~~~~~~

- **named_steps**: Dictionary-like object to access step parameters by name
- **classes_**: Class labels (only if final step is a classifier)
- **n_features_in_**: Number of features seen during fit
- **feature_names_in_**: Names of features seen during fit

Convenience Functions
---------------------

:func:`~sklearn.pipeline.make_pipeline`
    Simplified pipeline construction without explicit naming of steps

:func:`~sklearn.pipeline.make_union`
    Convenience function for creating feature unions

Use Cases
---------

- **Data Preprocessing Pipelines**: Chain scaling, imputation, and feature selection
- **Model Selection**: Grid search over pipeline parameters
- **Reproducible Workflows**: Ensure consistent preprocessing during training and prediction
- **Avoiding Data Leakage**: Prevent test set information from influencing preprocessing steps
