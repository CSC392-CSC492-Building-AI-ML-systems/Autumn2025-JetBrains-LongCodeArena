Controlling chunking
===================

Dask Array provides several mechanisms to control how arrays are chunked, which is essential for optimizing performance and memory usage in parallel computations.

Chunking Dispatchers
--------------------

Dask uses a dispatch system to handle array operations across different backends. The following dispatchers are available for controlling array operations:

- ``concatenate_lookup``: Handles array concatenation operations
- ``tensordot_lookup``: Manages tensor dot product operations
- ``einsum_lookup``: Controls Einstein summation operations
- ``empty_lookup``: Handles creation of empty arrays
- ``divide_lookup``: Manages division operations
- ``percentile_lookup``: Controls percentile calculations
- ``numel_lookup``: Handles number of elements operations
- ``nannumel_lookup``: Manages NaN-aware number of elements operations
- ``to_numpy_dispatch``: Controls conversion to NumPy arrays
- ``to_cupy_dispatch``: Manages conversion to CuPy arrays

Masked Array Operations
-----------------------

Dask Array provides support for masked array operations with controlled chunking through various functions:

Basic Masked Operations
~~~~~~~~~~~~~~~~~~~~~~~

- ``filled(a, fill_value=None)``: Return array with masked values filled
- ``masked_greater(a, value)``: Mask array where greater than value
- ``masked_greater_equal(a, value)``: Mask array where greater than or equal to value
- ``masked_less(a, value)``: Mask array where less than value
- ``masked_less_equal(a, value)``: Mask array where less than or equal to value
- ``masked_not_equal(a, value)``: Mask array where not equal to value

Advanced Masked Operations
~~~~~~~~~~~~~~~~~~~~~~~~~~

- ``masked_equal(a, value)``: Mask array where equal to value (scalar values only)
- ``masked_invalid(a)``: Mask array containing invalid values (NaN/inf)
- ``masked_inside(x, v1, v2)``: Mask array inside the interval [v1, v2]
- ``masked_outside(x, v1, v2)``: Mask array outside the interval [v1, v2]
- ``masked_where(condition, a)``: Mask array where condition is True
- ``masked_values(x, value, rtol=1e-05, atol=1e-08, shrink=True)``: Mask array values close to given value
- ``fix_invalid(a, fill_value=None)``: Replace invalid values with fill_value
- ``getdata(a)``: Return the data part of masked array
- ``getmaskarray(a)``: Return the mask of masked array

Chunking Control Mechanisms
---------------------------

The operations use various chunking strategies:

1. ``map_blocks``: Applies functions block-wise, preserving chunk structure
2. ``blockwise``: General block-wise operations with controlled output chunking
3. Automatic dtype preservation: Operations maintain the input array's data type
4. Shape consistency: Operations validate and maintain shape compatibility

These mechanisms ensure that chunking is maintained consistently across operations while allowing for efficient parallel computation and memory management.
