`BestEffortCallback`
==================

The `BestEffortCallback` class provides a comprehensive callback implementation for visualizing and monitoring data collection in real-time. It supports live table displays, baseline readings, and various plot types with intelligent field hinting.

Overview
--------
The callback automatically generates appropriate visualizations based on plan hints and document structure. It handles multiple data streams and provides both tabular and graphical output with several configuration options.

Key Features
------------
- Live table display for primary stream data
- Baseline stream monitoring
- Automatic plot generation (LivePlot, LiveGrid, LiveScatter)
- Peak statistics calculation
- Configurable output through enable/disable methods
- Support for multiple data streams
- Intelligent field selection through hinting system

Configuration Methods
---------------------
- ``enable_heading()``/``disable_heading()`` - Control timestamp and ID display
- ``enable_table()``/``disable_table()`` - Toggle primary stream table
- ``enable_baseline()/disable_baseline()`` - Toggle baseline stream output
- ``enable_plots()/disable_plots()`` - Control all plot generation

Options
-------
- ``overplot`` (bool): Whether to overplot subsequent scans (default: True)
- ``noplot_streams`` (list): Stream names to exclude from plotting (default: ['baseline'])
- ``omit_single_point_plot`` (bool): Skip plotting for single-point scans (default: True)

Public Data
-----------
- ``peaks`` (PeakResults): Container for peak statistics results

Usage Notes
-----------
The callback automatically handles document processing through the ``__call__`` method. It processes start documents, descriptors, and event documents to generate appropriate visualizations based on the configured settings and available hints.

The implementation uses hinting information from plans to determine which fields to display in tables and plots. When hints are unavailable or incomplete, it falls back to reasonable defaults.
