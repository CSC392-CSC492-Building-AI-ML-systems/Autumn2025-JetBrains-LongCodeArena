Ploterr command
=================

Overview
--------

Ploterr provides plotting support for error bars and CI data contained in HDF5 datasets. It exposes subcommands (at least a generic one) that read a 1D dataset and its associated statistics, then generate a plot of the expected values with confidence intervals.

Key features
- Plots the following against iteration:
  - expected (black)
  - ci_lbound (gray)
  - ci_ubound (gray)
- Configurable axis scaling (x and y): linear, log, or symlog
- Optional axis ranges: xrange and yrange
- Optional axis labels and title
- Output format inferred from file extension
- Interface choice based on environment:
  - If a display is available and not using terminal mode, uses matplotlib
  - If a terminal mode is requested or no DISPLAY, uses a text interface

Subcommands
-----------

generic
-------

Description
- Plots generic expectation/CI data from a provided HDF5 file and dataset.
- The targeted dataset must be specified as FILENAME/PATH[SLICE], where SLICE is a NumPy-style slice.
- The dataset must be 1-dimensional.

Dataset requirements inside the HDF5 file
- Fields expected by the plotting routine:
  - iter_start
  - iter_stop
  - expected
  - ci_lbound
  - ci_ubound

How it works (high level)
- The dataset is located via a path string of the form FILENAME/PATH[SLICE].
- If SLICE is omitted, all elements are selected. If present, a NumPy slice is applied.
- The code computes the x-axis values as iters = iter_stop - 1.
- It generates a plot with:
  - x-axis: iters
  - y-axis: data['expected'] (black)
  - CI bounds: data['ci_lbound'] and data['ci_ubound'] (gray)
- Styling and axis configuration are controlled via the common plotting options.

Command-line interface and options
- Output file
  - -o, --output OUTPUT
  - Default: errbars.pdf
  - The format is inferred from the extension (e.g., PDF, PNG, SVG)
- Data specification
  - dsspec: FILENAME/PATH[SLICE]
  - Example: kinavg.h5/rate_evolution[:,0,1]
- Plot options (group)
  - --xscale: linear, log, or symlog (default: linear)
  - --yscale: linear, log, or symlog (default: linear)
  - --xrange: xmin,xmax (optional)
  - --yrange: ymin,ymax (optional)
  - --xlabel: X-axis label (default varies; usually 'Iteration')
  - --ylabel: Y-axis label (default varies)
  - --title: Plot title (default varies)
  - --terminal, -t: Plot output in terminal (text interface)

Interface behavior
- If --terminal is specified or no DISPLAY is available, the command uses the text interface.
- Otherwise, it uses a matplotlib-based interface to render plots.

Usage examples
- Basic plot (matplotlib interface):
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -o rate_evolution_errbars.pdf
- With custom labels and ranges:
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -o rate_evolution_errbars.pdf \
    --xlabel 'Iteration' --ylabel 'Rate' --title 'Rate evolution' \
    --xrange 0,100 --yrange 0,0.5
- Terminal/text interface (no GUI, suitable for headless environments):
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -t -o rate_evolution_errbars.txt

Internal implementation notes (derived from code)
- CommonPloterrs (base class)
  - Handles generic plot options and interface decisions
  - do_plot(...) renders and saves the plot given a data dictionary and optional plot metadata
  - Expects data to contain:
    - iter_stop
    - expected
    - ci_lbound
    - ci_ubound
  - Implements x/y scaling, range constraints, labels, and title
- GenericIntervalSubcommand
  - Inherits from CommonPloterrs
  - add_args(...) defines input/output options, notably -o/--output and the dsspec positional argument
  - process_args(...) parses dsspec into (pathname, slicestr) using a regex; converts slicestr into a NumPy slice expression; resolves the HDF5 file and dataset via h5io.resolve_filepath
  - load_and_validate_data(...) loads the dataset with the specified slice and validates that it is 1-dimensional; prepares for plotting
  - Expected to be followed by further validation against the required fields (iter_start, iter_stop, expected, ci_lbound, ci_ubound)

Notes and caveats
- The code expects the dataset to be 1D. If not, a TypeError is raised.
- The dataset must contain the specified fields; if missing, behavior is not shown in the snippet but should be verified by the broader codebase.
- The output format is inferred from the output filename extension; ensure you provide an appropriate extension for the desired format (e.g., .pdf, .png, .svg).
- The default x-axis label is typically derived from Ploterr’s internal logic (not a hard-coded constant in the snippet). Override with --xlabel if needed.
