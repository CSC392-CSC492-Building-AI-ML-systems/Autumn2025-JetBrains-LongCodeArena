Random Projection transformers
===============================

A simple and computationally efficient approach to reducing dimensionality

Overview
--------
Random projection offers a straightforward way to reduce the dimensionality of high-dimensional data while preserving pairwise distances with high probability. It trades a controlled amount of accuracy (represented by the distortion parameter eps) for faster processing times and smaller model sizes. The projection is achieved by multiplying the data by a random matrix, producing a lower-dimensional representation that approximately preserves distances between samples.

Key ideas
----------
- Johnson-Lindenstrauss lemma: A set of n samples in high-dimensional space can be projected into k dimensions (k much smaller than the original dimension) so that pairwise distances are preserved within a factor of (1 ± eps) with high probability.
- The random projection matrix can be:
  - Dense Gaussian: entries drawn from a normal distribution with appropriate scaling.
  - Sparse Achlioptas: a sparse matrix with a simple ±1/0 pattern to speed up computations.
- The approach is linear in the data and projection dimensions, enabling efficient dimensionality reduction for large datasets, including sparse data such as text representations.

How it works (high level)
-------------------------
- Generate a random projection matrix W of shape (n_components, n_features).
- Transform data X (shape: n_samples, n_features) to X' = X dot W.T (or an equivalent sparse/dense multiply).
- Distances between transformed points approximate the original distances within a user-specified distortion bound eps with high probability.

API overview
------------
The module provides two Transformer implementations and a helper for estimating a safe target dimensionality.

- johnson_lindenstrauss_min_dim(n_samples, eps=0.1)
  - Compute the minimal number of components needed to achieve an eps-embedding with high probability, per the JL bound:
    n_components >= 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)
  - Accepts scalar or array-like inputs for n_samples and eps; supports array-wise computation.

- GaussianRandomProjection
  - A Transformer that projects data using a dense Gaussian random matrix.
  - Matrix entries are drawn from N(0, 1/n_components).
  - Typically used when dense projection is acceptable and you want a straightforward implementation.

- SparseRandomProjection
  - A Transformer that projects data using a sparse random matrix (sparse Achlioptas style).
  - Reduces computation and memory footprint for high-dimensional sparse data.
  - Density controls the sparsity of the projection matrix (default or user-specified).

Practical usage
---------------
- Basic usage with Gaussian projection:
  from sklearn.random_projection import GaussianRandomProjection
  rp = GaussianRandomProjection(n_components=100, random_state=42)
  X_reduced = rp.fit_transform(X)

- Using JL bound to choose dimensions:
  from sklearn.random_projection import johnson_lindenstrauss_min_dim
  n_samples = X.shape[0]
  eps = 0.1
  k = johnson_lindenstrauss_min_dim(n_samples, eps=eps)
  rp = GaussianRandomProjection(n_components=int(k), random_state=42)
  X_reduced = rp.fit_transform(X)

- Sparse projection for large sparse data:
  from sklearn.random_projection import SparseRandomProjection
  rp = SparseRandomProjection(n_components=1000, density='auto', random_state=0)
  X_reduced = rp.fit_transform(X)

Notes on parameters and behavior
--------------------------------
- density (SparseRandomProjection):
  - If density is "auto", it is computed as 1 / sqrt(n_features) per Li et al. guidance.
  - If a numeric value is provided, it must be in (0, 1].
  - Higher density increases accuracy but reduces sparsity and performance gains; lower density yields sparser projections and faster computation.
- random_state:
  - Ensures reproducible projections across runs.
- The transformation is linear; the reduced representation is suitable for use in downstream models and can be integrated into pipelines.

Examples
--------
- Example: reduce a dataset to 200 components using a Gaussian projection
  >>> from sklearn.random_projection import GaussianRandomProjection
  >>> rp = GaussianRandomProjection(n_components=200, random_state=0)
  >>> X_rp = rp.fit_transform(X)

- Example: estimate safe dimensionality with JL bound and apply
  >>> from sklearn.random_projection import johnson_lindenstrauss_min_dim, GaussianRandomProjection
  >>> k = johnson_lindenstrauss_min_dim(X.shape[0], eps=0.1)
  >>> rp = GaussianRandomProjection(n_components=int(k), random_state=0)
  >>> X_rp = rp.fit_transform(X)

- Example: sparse projection for high-dimensional sparse data
  >>> from sklearn.random_projection import SparseRandomProjection
  >>> rp = SparseRandomProjection(n_components=1000, density='auto', random_state=0)
  >>> X_rp = rp.fit_transform(X)

Implementation notes
--------------------
- The implementation uses check_random_state to ensure reproducible random matrices.
- Input validation helpers enforce positive component counts and feature counts, and validate densities for sparse projections.
- The provided helpers and classes are designed to integrate seamlessly with scikit-learn pipelines.
