============================================================
w_stateprobs: command for state-to-state kinetics by tracing
============================================================

Overview
--------
w_stateprobs is a deprecated wrapper around the w_direct tool. It exposes the same functionality as w_direct’s DStateProbs under the subcommand trace, providing a backward-compatible interface for computing average state populations and associated errors from weighted ensemble data.

Key points
- Deprecated: This command is being deprecated. Please use w_direct instead.
- Backward-compatible shim: It forwards to the underlying DStateProbs implementation.
- Default outputs:
  - Output file: stateprobs.h5 (default_output_file)
  - Kinetics file: assign.h5 (default_kinetics_file)
- Underlying behavior: Calculates average populations and confidence intervals for state populations, requiring bin assignments (including macrostate definitions). See w_assign for bin definitions.

Command name and usage
----------------------
- Program: w_stateprobs
- Subcommand: trace (the actual implementation in this shim)
- Minimal invocation (the wrapper will insert the subcommand if omitted):
  - w_stateprobs
  - w_stateprobs trace

Notes
- This command is a compatibility shim. For current usage, prefer:
  - w_direct instead of w_stateprobs
- The help text and behavior are aligned with the underlying DStateProbs tool.

Output format
-------------
The output file (stateprobs.h5 by default) contains the following datasets:

1) /avg_state_pops [state]
   - Structure: Population of each state across the full range.

2) If evolution mode is enabled (--evolution-mode specified):
   /state_pop_evolution [window][state]
   - Structure: State populations computed over windows of iterations (varying width depending on mode).

Window definitions and modes
- iter_start (Integer): Start iteration (inclusive) of the averaging window.
- iter_stop (Integer): End iteration (exclusive) of the averaging window.
- expected (Floating-point): Mean rate within this window (in units of inverse tau).
- ci_lbound (Floating-point): Lower bound of the confidence interval for the rate in this window (inverse tau).
- ci_ubound (Floating-point): Upper bound of the confidence interval for the rate in this window (inverse tau).
- corr_len (Integer): Correlation length of the rate within this window (in units of tau).

Evolution modes (when --evolution-mode is specified)
- cumulative:
  - Windows start at --start-iter and grow in length by --step-iter for each successive element.
- blocked:
  - All windows have width --step-iter (the last may be shorter), starting at --start-iter.

Dataset attributes (shared by the datasets above)
- mcbs_alpha (Floating-point): Confidence interval alpha (e.g., alpha=0.05 for a 95% CI).
- mcbs_nsets (Integer): Number of bootstrap datasets used to generate confidence intervals.
- mcbs_acalpha (Floating-point): Alpha used to determine correlation lengths.

Inputs and dependencies
-----------------------
- Bin assignments and macrostate definitions are required. See w_assign for details on bin/macrostate definitions.
- This tool relies on the same inputs as the underlying DStateProbs tool (WStateProbs in this shim delegates to DStateProbs).

Examples
--------
- Basic invocation (wrapper inserts trace if omitted):
  w_stateprobs

- Explicit trace subcommand with default outputs:
  w_stateprobs trace -o stateprobs.h5

- With evolution mode (cumulative):
  w_stateprobs trace --evolution-mode cumulative --start-iter 100 --step-iter 50 -o stateprobs_evol.h5

- Note: For bin/macrostate definitions, refer to w_assign and its help output.

Relation to other tools
-----------------------
- Underlying implementation: westpa.cli.tools.w_direct.DStateProbs
- Primary recommended replacement: w_direct (preferred interface for direct state-probability analyses)
- The shim exists to preserve existing workflows that used w_stateprobs.

Implementation notes
--------------------
- The entry point issues a deprecation warning and then routes to the same command flow as w_direct by ensuring the subcommand trace is present.
- The default output file and kinetics file names are provided to preserve prior behavior in the wrapper:
  - default_output_file = 'stateprobs.h5'
  - default_kinetics_file = 'assign.h5'

Related commands
----------------
- w_direct: The non-deprecated, recommended interface that provides the same functionality.
- w_assign: For bin/macrostate definitions used by state population analysis.
