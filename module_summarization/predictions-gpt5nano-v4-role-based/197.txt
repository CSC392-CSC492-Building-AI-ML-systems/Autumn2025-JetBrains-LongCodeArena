Naive Bayes methods
====================

Overview
--------
The naive Bayes family in scikit-learn provides probabilistic classifiers that apply Bayes’ theorem with strong (naive) feature independence assumptions. Each estimator learns a model of P(y) and P(x|y) from labeled data and then uses these to predict class labels and probabilities for new samples.

What to know about the implementation
- All Naive Bayes estimators derive from the abstract base class _BaseNB, which implements the common estimator API for prediction.
- The core of the model is the joint log-likelihood: log P(y) + log P(x|y) for each class y and input x.
- Prediction is performed by selecting the class with the highest joint log-likelihood (predict), or by converting posterior log-probabilities (predict_log_proba) to probabilities (predict_proba) using normalization with log P(x).
- predict_joint_log_proba returns the unnormalized joint log probabilities log P(x, y) for all classes.
- Each concrete Naive Bayes estimator implements its own _joint_log_likelihood and _check_X methods appropriate for its feature type.
- GaussianNB supports online updates via partial_fit and uses a Gaussian distribution assumption for continuous features. It includes a smoothing parameter var_smoothing to stabilize variance estimates.

Base class: _BaseNB
-------------------
Abstract foundation for all Naive Bayes estimators. Key responsibilities:

- _joint_log_likelihood(X)
  - Abstract method that computes the unnormalized posterior log probability log P(c) + log P(X|c) for all samples X and all classes c.
  - Implementations are provided by concrete NB estimators (e.g., GaussianNB, BernoulliNB, MultinomialNB, etc.).

- _check_X(X)
  - Abstract input validation method. Subclasses implement feature-type specific checks (e.g., dense vs sparse, dimensionality).

- Public prediction methods (inherited by all NB estimators)
  - predict_joint_log_proba(X): returns joint log probabilities log P(x, y) for each sample and class.
  - predict(X): returns predicted class labels.
  - predict_log_proba(X): returns log-posterior log P(y|x) for each sample and class.
  - predict_proba(X): returns posterior probabilities P(y|x).
  - All prediction methods call _check_is_fitted and _check_X before delegating to _joint_log_likelihood.

Notes
- The term “joint log likelihood” is synonymous with the unnormalized log posterior log P(y) + log P(x|y).
- The posterior normalization used in predict_log_proba is performed by subtracting log P(x) (computed via logsumexp over classes).

GaussianNB
----------
Gaussian Naive Bayes (GaussianNB) assumes continuous features are distributed according to class-specific Gaussian distributions and that features are conditionally independent given the class.

Parameters
- priors : array-like of shape (n_classes,), default=None
  Prior probabilities of the classes. If provided, priors are used as-is and not estimated from the data.

- var_smoothing : float, default=1e-9
  Portion of the largest variance of all features that is added to variances for numerical stability.

Notes
- GaussianNB can be updated online via partial_fit, allowing incremental learning.
- See the User Guide entry for Gaussian Naive Bayes for more details.

Attributes (learned after fit)
- class_count_ : ndarray of shape (n_classes,)
  Number of training samples observed in each class.

- class_prior_ : ndarray of shape (n_classes,)
  Estimated prior probability of each class (unless priors were provided).

- classes_ : ndarray of shape (n_classes,)
  Class labels known to the classifier.

- epsilon_ : float
  Absolute additive value added to variances (related to var_smoothing for numerical stability).

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit (if input has feature names that are all strings).

- var_ : ndarray of shape (n_classes, n_features)
  Variance of each feature per class.

- theta_ : ndarray of shape (n_classes, n_features)
  Mean of each feature per class.

Algorithm sketch
- For each class c, estimate:
  - Prior P(c) from the training data (or use user-provided priors).
  - For each feature f, the mean mu_{f,c} and variance sigma^2_{f,c} of the feature values among samples in class c.
- For a new sample x, compute:
  - log P(y=c) + sum_f log P(x_f | y=c), where P(x_f | y=c) is evaluated using a Gaussian density with mu_{f,c} and sigma^2_{f,c} (with var_smoothing applied to sigma^2_{f,c} for stability).
- Use the resulting joint log-likelihoods to predict labels and probabilities via the base class methods described above.

Practical usage example
- Training and predicting with GaussianNB:
  - from sklearn.naive_bayes import GaussianNB
  - gnb = GaussianNB()
  - gnb.fit(X, y)  # where X is a 2D array-like and y is the target
  - y_pred = gnb.predict(X_new)
  - proba = gnb.predict_proba(X_new)

Compatibility with other Naive Bayes estimators
- Other NB estimators in this module (BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB) implement similar prediction APIs but differ in how they model P(x|y) to suit different data types:
  - BernoulliNB: binary/boolean features.
  - MultinomialNB: integer feature counts (e.g., word/token counts in text classification).
  - ComplementNB: variant designed for imbalanced classes.
  - CategoricalNB: handles categorical features directly.

Design and architecture notes
- Core logic is centralized in _BaseNB, ensuring a consistent API across all Naive Bayes variants.
- Numerical stability is addressed via log-domain computations (log-sum-exp) and variance smoothing (var_smoothing).
- The public API emphasizes interpretability: the joint log-likelihoods can be inspected via predict_joint_log_proba, while posterior probabilities are obtained through predict_proba/predict_log_proba.
