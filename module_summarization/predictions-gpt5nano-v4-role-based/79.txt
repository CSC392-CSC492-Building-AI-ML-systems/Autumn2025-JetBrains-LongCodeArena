GILDAS CLASS file reader (classfiles)
======================================

Summary
-------
This module provides the low-level machinery to interpret GILDAS CLASS binary files and prepare their contents for use with pyspeckit. It defines the mappings and layout specifications used to decode CLASS headers and records, and includes small utilities to aid Python 2/3 compatibility and debugging.

Key concepts
------------
- CLASS filetypes and versions: CLASS files can be encoded in different endianness and layouts. The module defines mappings to human-readable names and versions.
- Header and record layout: The CLASS file header is composed of labeled blocks (e.g., GENERAL, COMMENT). The exact structure (field names, counts, and data types) is captured in the module so that a reader can parse binary data into Python values consistently.
- Compatibility utilities: Helpers for Python 2/3 compatibility and optional progress reporting.

Public API and data structures
------------------------------

Note: The following API items are defined in the module and are intended for use by developers integrating or extending the reader.

- print_timing(func)
  - Description: Decorator that prints the execution time of the decorated function.
  - Purpose: Useful for profiling CLASS file reads, which can be time-consuming.
  - Example:
    - @print_timing
      def read_class(...): ...
- ensure_bytes(string)
  - Description: Convert a string to a bytes object in a Python-2/3 compatible way.
  - Behavior:
    - If running under Python 3, returns bytes(string, 'utf-8').
    - Otherwise, returns a byte-string representation for Python 2.
  - Example:
    - ensure_bytes('1A  ') -> b'1A  '
- filetype_dict
  - Description: Map of CLASS filetype codes to human-readable type names.
  - Original keys map to values like 'Multiple_IEEE', 'v2', etc.
  - The dictionary is extended to use byte-string keys for Python 3 compatibility.
  - Example:
    - Ensure-byte lookup: filetype_dict[ensure_bytes('2A  ')] -> 'v2'
- fileversion_dict
  - Description: Map of file version codes to version strings (e.g., 'v1', 'v2').
  - Also extended to byte-string keys for Python 3.
  - Example:
    - fileversion_dict[ensure_bytes('2A  ')] -> 'v2'
- record_lengths
  - Description: Record length (in bytes) for different CLASS file formats.
  - Examples:
    - '1A' -> 512
    - '2A' -> 4096 (1024*4)
- header_id_numbers
  - Description: Mapping of header block IDs (integers) to human-friendly names.
  - Examples:
    -  -2 -> 'GENERAL'
    -  -3 -> 'POSITION'
    -  -4 -> 'SPECTRO'
    -  -6 -> 'HISTORY'
- header_id_lengths
  - Description: Lengths for specific header blocks (in items or bytes depending on implementation).
  - Some blocks have fixed lengths (e.g., -2: 9, -3: 17, -4: 17) while others are variable (e.g., -5, -6).
- filedescv2_nw1
  - Description: A constant used by the v2 description/definition (value in the module is 14).
  - Purpose: Part of the specification imported from the Fortran-era layout.
- keys_lengths
  - Description: The core header field specification used to decode header blocks.
  - Structure: A dictionary mapping a header category name to a list of field descriptors.
  - Each field descriptor is a tuple: (field_name, count, dtype)
  - Examples (from the snippet):
    - 'unknown': [
        ('NUM'    ,1,'int32'),  # Observation number
        ('VER'    ,1,'int32'),  # Version number
        ('TELES'  ,3,'|S12'),   # Telescope name
        ('DOBS'   ,1,'int32'),  # Date of observation
        ('DRED'   ,1,'int32'),  # Date of reduction
        ('TYPEC'  ,1,'int32'),  # Type of coordinates
        ('KIND'   ,1,'int32'),  # Type of data
        ('QUAL'   ,1,'int32'),  # Quality of data
        ('SCAN'   ,1,'int32'),  # Scan number
        ('SUBSCAN',1,'int32'),  # Subscan number
      ],
    - 'COMMENT': [
        ('LTEXT',1,'int32'),       # Length of comment
        ('CTEXT',1024//4,'|S1024'), # Comment string (ctext*1024)
      ],
    - 'GENERAL': [
        ('UT'     ,2,'float64'),  # UT of observation (rad)
        ('ST'     ,2,'float64'),  # LST of observation (rad)
        ('AZ'     ,1,'float32'),  # Azimuth (rad)
        ('EL'     ,1,'float32'),  # Elevation (rad)
        ('TAU'    ,1,'float32'),  # Opacity (neper)
        ('TSYS'   ,1,'float32'),  # System temperature (K)
        ('TIME'   ,1,'float32'),  # Integration time (s)
        # Note: XUNIT is not present in this GENERAL block
      ],
  - The actual set of blocks in keys_lengths may be larger in the full module; the example shows the intended structure.

Implementation notes
--------------------
- Python compatibility: Uses six to support both Python 2 and 3, including iteritems and xrange compatibility shims.
- Optional progress reporting: Attempts to import astropy.utils.console.ProgressBar. If unavailable, ProgressBar becomes a no-op to avoid hard dependency.
- Logging: Uses astropy.log for timing and informational messages (e.g., timing results from print_timing).

Usage guidance
--------------
- The module serves as the decoding backbone for CLASS files. A higher-level reader (likely in the same package) would use the dictionaries above to parse binary data into Python types, ultimately populating a pyspeckit.spectrum.ObsBlock.
- To explore the supported formats, inspect:
  - filetype_dict and fileversion_dict to see how codes map to friendly names.
  - record_lengths to understand per-format byte widths.
  - keys_lengths to see how header blocks are laid out and how to decode fields.
- Practical tips:
  - Use ensure_bytes to safely form dictionary lookups in a Python 3 environment.
  - If reading performance is a concern, apply the @print_timing decorator to the actual read function to profile bottlenecks.
