Spline and other smoother classes for Generalized Additive Models
===============================================================

Overview
--------
This module provides the smoothing utilities used by the Generalized Additive Models (GAM) implementation in StatsModels. It contains:
- GLMGam: GAM model that combines linear terms with smooth terms for nonparametric fitting.
- BSplines: B-spline basis wrapper to form smooth terms from continuous covariates.
- CyclicCubicSplines: Cyclic (periodic) spline basis suitable for seasonal or angular data.
- MultivariateGAMCVPath: cross-validation path utilities for GAMs (useful for multivariate smoothing setups).

In addition, several internal helpers are provided to construct knot sequences and evaluate spline bases:
- _equally_spaced_knots(x, df)
- _R_compat_quantile(x, probs)
- _eval_bspline_basis(x, knots, degree, deriv='all', include_intercept=True)
- compute_all_knots(x, df, degree)
- make_bsplines_basis(x, df, degree)
- get_knots_bsplines(...)

Notes
-----
- The underscore-prefixed functions (_equally_spaced_knots, _R_compat_quantile, _eval_bspline_basis, etc.) are internal utilities intended for internal use by the smoothing classes. They are documented here for completeness but are not part of the public API.
- The public API focuses on constructing smooth terms (BSplines, CyclicCubicSplines) and combining them in GLMGam or related GAM fitters.

Example
-------
The following examples illustrate how to construct and fit a simple GAM with a single smooth term using B-splines, and how to leverage cyclic splines for periodic data.

Example 1: One smooth term with B-splines
-----------------------------------------
This example fits y ~ s(x) where s(x) is represented by a B-spline basis.

.. code-block:: python

    import numpy as np
    import statsmodels.api as sm
    from statsmodels.gam.api import GLMGam, BSplines

    np.random.seed(0)
    n = 200
    x = np.linspace(-3, 3, n)
    y = np.sin(x) + 0.3 * np.cos(2 * x) + np.random.normal(scale=0.2, size=n)

    # Build a B-spline basis for x with 6 degrees of freedom and cubic degree
    # include_intercept=False ensures the smoother does not absorb the intercept term
    x_s = x.reshape(-1, 1)
    splines = BSplines(x_s, df=[6], degree=[3], include_intercept=False)

    # Exogenous linear part (intercept only)
    exog = np.ones((n, 1))

    # Fit the GAM: y ~ s(x)
    gam = GLMGam(y, exog=exog, smoother=[splines])
    res = gam.fit()

    print(res.summary())

Example 2: Periodic data with Cyclic Cubic Splines
---------------------------------------------------
This example shows how to model a periodic covariate with cyclic splines.

.. code-block:: python

    import numpy as np
    from statsmodels.gam.api import GLMGam, CyclicCubicSplines

    np.random.seed(0)
    n = 300
    theta = 2 * np.pi * np.random.rand(n)
    y = 1.0 + 0.5 * np.sin(theta) + 0.25 * np.cos(3 * theta) + 0.1 * np.random.normal(size=n)

    # Cyclic cubic splines for the periodic covariate theta
    # Here x should be shaped as (n, 1)
    theta_s = theta.reshape(-1, 1)
    cyclic = CyclicCubicSplines(theta_s, df=[8], degree=[3])

    exog = np.ones((n, 1))  # intercept
    gam = GLMGam(y, exog=exog, smoother=[cyclic])
    res = gam.fit()

    print(res.summary())

Example 3: Computing knots and building a basis (advanced)
----------------------------------------------------------
The module provides internal helpers to compute knots and evaluate BSpline bases. These are primarily used by the public smoothing classes, but can be used directly for advanced customization.

.. code-block:: python

    import numpy as np
    from statsmodels.gam.smooth_basis import compute_all_knots, _eval_bspline_basis

    x = np.linspace(0, 1, 100)
    df = 6
    degree = 3

    all_knots, lower, upper, inner_knots = compute_all_knots(x, df=df, degree=degree)
    basis, der1, der2 = _eval_bspline_basis(x, all_knots, degree)

    print("Basis shape:", basis.shape)
    print("First few basis coefficients:\n", basis[:5, :3])

How to decide between knot placement options
---------------------------------------------
The module supports different knot-placement strategies:
- Quantile spacing with boundary multiplicity (as used by Patsy): good default for data-driven smoothing.
- Equal spacing extended to boundary knots (P-Splines): useful when you want evenly spaced knots across the domain.

The get_knots_bsplines helper can be used to obtain knot sequences according to the chosen spacing strategy and to drive the creation of spline bases.

Public API summary (high level)
-------------------------------
- GLMGam(endog, exog=None, smoother=None, family=None, ...): Fit a generalized additive model with one or more smooth terms.
- BSplines(x, df, degree, knot_type='quantile', include_intercept=False, knots=None, ...): Build a B-spline basis for a covariate (or matrix of covariates) to be used as a smooth term in GLMGam.
- CyclicCubicSplines(x, df, degree, ...): Build a cyclic (periodic) cubic spline basis for periodic covariates.
- MultivariateGAMCVPath: Utilities for cross-validation paths in multivariate GAM settings (for model selection and CV).

For more details, refer to the specific class docstrings and the code in the module.
