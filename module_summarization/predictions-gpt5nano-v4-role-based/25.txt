WESTPA Interactive Python Analysis (w_ipa)
------------------------------------------

Overview
--------

The w_ipa command provides a stateful, interactive Python-based interface to the
current WESTPA analysis. It exposes a persistent environment from which you can
inspect traces, weights, progress coordinates, states, bins, and other data
pertaining to the active analysis scheme and iteration. By default, w_ipa uses the
first analysis scheme defined in west.cfg and starts at iteration 1. All
interactive properties are accessed via the objects exposed as w or west (e.g.,
w.iteration, west.iteration).

Key capabilities
- Inspect the current analysis state:
  - Access iteration, scheme, states, bin_labels, and the comprehensive
    current data object (current), which includes walkers, summary, seg_id, weights,
    parents, kinavg, pcoord, bins, populations, and auxdata (if present).
- Run traces:
  - The trace(seg_id) function returns a dictionary with the history and
    pertinent information for the specified seg_id in the current iteration.
- Navigate analysis schemes and iterations:
  - Query and set the active iteration and analysis scheme (w.iteration, w.scheme, w.list_schemes).
- Interactive data exploration:
  - The interface supports both a matplotlib-based plotting interface and a text-based terminal output, selectable via runtime options.

Commands and runtime options
-----------------------------

WIPI inherits behavior from WESTParallelTool and wires in common WESTPA interactive tooling. The following runtime options are exposed to control behavior and I/O:

- --analysis-only, -ao
  - Run the analysis and return to the terminal.
- --reanalyze, -ra
  - Delete existing analysis files and re-run analysis from scratch.
- --ignore-hash, -ih
  - Ignore the argument/file hash mechanism and do not trigger regeneration.
- --debug, -d
  - Enable debug output intended for development and troubleshooting.
- --terminal, -t
  - Plot output in a terminal/text interface.
- -f, --extra
  - Hidden/less-documented extra argument (used internally in some flows).

Implementation notes
--------------------

- Data access:
  - The tool uses a WESTDataReader (WESTDataReader) to load data for the current
    iteration and scheme. The reader is initialized in __init__ and its lifecycle is
    managed during processing of arguments (process_args).
- Configuration:
  - The current analysis configuration is loaded from westrc via self.__config and
    self.__settings. The active scheme is chosen by scanning the configured schemes for
    enabled = True or enabled is None.
- Statefulness:
  - WIPI is designed to be "stateful": it pulls data from the current analysis scheme and
    the active iteration, so changes in scheme/iteration are reflected in the data you see.
- Hashing:
  - hash_args(args, extra=None, path=None) builds a hash to determine whether a given set of
    arguments (and optional extra context) has changed sufficiently to warrant re-generation.
  - stamp_hash(h5file_name, new_hash) is intended to stamp the HDF5 file with the new hash, aiding
    incremental work. The implementation in the provided code is incomplete.
- Interface mode:
  - The default interface is matplotlib-style, but you can switch to a text interface when
    --terminal is used.

Data model and nomenclature
---------------------------

- current (object)
  - The primary container for per-iteration data. It may include:
    - walkers, summary, states, seg_id, weights, parents, kinavg, pcoord, bins, populations, auxdata (if present).
- iteration and scheme
  - Accessed via w.iteration and w.scheme. The active scheme is derived from the west.cfg
    analysis_schemes configuration.
- states and bin_labels
  - Convenience accessors to inspect the analysis’ defined states and their bin labels.
- trace(seg_id)
  - A method to retrieve a detailed history snapshot for a given segmentation ID. It can be expensive,
    so it is often advantageous to store or cache the result.

Usage examples
--------------

 Quick start
- Start the w_ipa interactive tool (in a WestPA-enabled shell):
  - w_ipa

- Inspect the current iteration
  - w.iteration
  - west.iteration

- List and switch schemes
  - w.list_schemes
  - w.scheme = "NAME_OF_A_SCHEME"  # as returned by w.list_schemes

- Access current data
  - current = w.current
  - current.walkers
  - current.weights

- View states and bins
  - w.states
  - w.bin_labels

- Run a trace for a segment
  - trace_info = w.trace(seg_id)

- Change iteration
  - w.iteration = 100

Notes
-----

- The tool is designed to provide convenient access to data for the current analysis
  scheme and iteration. If you need to persist results or avoid recomputation, consider
  the hashing utilities (hash_args/stamp_hash) and the --reanalyze option.

References to implementation details
------------------------------------

- Core class: WIPI (inherits from WESTParallelTool)
- Data access: WESTDataReader
- Progress: ProgressIndicatorComponent
- Interface mode: interface = 'matplotlib' by default; switched to 'text' if --terminal is used
- Argument processing lifecycle:
  - add_args(parser): registers command-line options
  - process_args(args): loads current_iteration, determines active scheme, stores options
- Hashing helpers:
  - hash_args(args, extra=None, path=None): computes a unique hash for the current invocation
  - stamp_hash(h5file_name, new_hash): stamps an HDF5 file with arg_hash (implementation incomplete)

If you need to customize or extend behavior
-------------------------------------------

- Extend or replace the data reader to support additional datasets or projections.
- Override or extend the process_args to adjust how the active scheme is chosen or to pre-load additional caches.
- Implement or refine stamp_hash to ensure hash stamping integrates with your storage workflow.
