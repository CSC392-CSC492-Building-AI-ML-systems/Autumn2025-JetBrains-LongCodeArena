Validation curves: plotting scores to evaluate models
======================================================

Overview
--------

Validation curves plot a model’s performance as a function of a hyperparameter value. They help diagnose how changing a parameter (e.g., regularization strength, number of estimators) impacts bias and variance, enabling informed hyperparameter selection.

In the learning_curve module, the plotting infrastructure used for validation curves shares the same visualization backbone as learning curves. The core plotting logic is encapsulated in the _BaseCurveDisplay._plot_curve method, which:

- Accepts x_data (the hyperparameter values or training sizes).
- Supports plotting training scores, test scores, or both.
- Visualizes score variability across cross-validation folds with:
  - fill_between (mean ± std)
  - errorbar (mean with error bars)
  - None (no variability shown)
- Manages score naming, axis labeling, and legend.
- Dynamically selects an appropriate x-axis scale (log, linear, or symlog) based on the x_data distribution, with a deprecated log_scale argument kept for backward compatibility.

Key concepts
------------

- Parameters and data shapes
  - param_range (x_data): 1D array of hyperparameter values (e.g., C, gamma) or other parameter values.
  - train_scores / test_scores: arrays with shape (n_params, n_folds). Each row corresponds to a parameter value, and each column to a CV fold.
  - mean(axis=1): the mean score across folds for each parameter value.
  - std(axis=1): the standard deviation of scores across folds for each parameter value.

- Display options (std_display_style)
  - "fill_between": shaded region around the mean score (mean ± std).
  - "errorbar": error bars showing mean with std for each point.
  - None: no variability visualization.

- Score selection (score_type)
  - "train": plot training scores.
  - "test": plot validation/test scores.
  - "both": plot both training and test scores.

- Negating scores (negate_score)
  - If the raw scores should be negated (common when lower is better), set negate_score to True to plot the inverted scores.

- Score naming (score_name)
  - If not explicitly provided, the displayed score name is inferred from the scoring parameter, with appropriate handling of negation.

- Axis scales (log_scale)
  - Deprecated: the parameter log_scale is deprecated and will be removed; the code now selects an appropriate x-axis scale automatically.
  - If needed, you can set the axis scale directly on the produced axes (e.g., display.ax_.set_xscale("log")).

Public patterns (how this maps to validation curves)
----------------------------------------------------

- ValidationCurveDisplay (the public API) typically uses a method like from_estimator(...) to construct a display from an estimator, data, and a param_name/param_range. The underlying plotting follows the same _BaseCurveDisplay logic described above, enabling consistent visualization across learning and validation curves.

Usage example
-------------

Basic validation curve with a scalar hyperparameter:

- Compute the curve (typical usage):
  - from sklearn.model_selection import validation_curve
  - param_range = [0.1, 1, 10, 100]
  - train_scores, test_scores = validation_curve(
      estimator, X, y, param_name="C", param_range=param_range, scoring="accuracy", cv=5
    )

- Visualize with the display class:
  - from sklearn.model_selection import ValidationCurveDisplay
  - ValidationCurveDisplay.from_estimator(
      estimator, X, y, param_name="C", param_range=param_range, scoring="accuracy", cv=5
    ).plot()
  - This yields a plot where:
    - The x-axis shows C (or the chosen parameter) values.
    - The y-axis shows accuracy (or the chosen scoring metric).
    - Mean scores are shown (train/test or both), with variability displayed according to std_display_style.

Alternatively, plot with custom style:
  - ValidationCurveDisplay.from_estimator(...).plot(std_display_style="fill_between")
  - Or std_display_style="errorbar" to show error bars.

Interpreting the plot
---------------------

- If both curves (train and test) converge and the test curve peaks at a particular parameter value, that point is a strong candidate for the optimal hyperparameter.
- A large gap between train and test curves indicates potential overfitting (high variance). A small gap with low test performance suggests underfitting (high bias).
- The spread (std) across folds informs about the stability of the model with respect to data splits.

Internal notes (implementation highlights)
------------------------------------------

- The plotting routine handles:
  - mean and std across CV folds for each parameter value.
  - Optional display of multiple curves (train, test, or both).
  - Multiple plotting styles for standard deviation visualization.
  - Dynamic axis scaling based on x_data distribution, with a deprecation notice for the log_scale parameter.

- Public attributes (as used in displays)
  - ax_: Matplotlib Axes object containing the plot.
  - figure_: The associated Figure.
  - train_scores / test_scores: Original score arrays used to compute means and stds.
  - lines_, errorbar_, fill_between_: Plot artists created depending on std_display_style.

Notes
-----

- The log_scale parameter is deprecated; prefer using axis methods (e.g., ax_.set_xscale) to adjust the scale.
- The automatic x-scale selection uses the interval-to-min/max ratio to decide between linear, log, or symlog scales.
