General Plotting routines
=========================

Overview
--------
This module provides core, interactive plotting capabilities built on top of Bokeh. It centralizes configuration for the Bokeh backend, and exposes high-level plotting helpers for common tasks such as scatter plots (single and multiple data series). The plotting helpers leverage a shared BokehPlotter instance to ensure consistent styling, tooltips, limits, and figure handling across plots.

Key components
- plot_params: a shared BokehPlotter instance used to configure defaults, prepare figures, and apply consistent styling.
- Default management helpers: set_bokeh_plot_defaults, reset_bokeh_plot_defaults, show_bokeh_plot_defaults, get_bokeh_help, load_bokeh_defaults, save_bokeh_defaults.
- General plotting routines: bokeh_scatter for single-series scatter plots and bokeh_multi_scatter for multi-series scatter plots.
- Data handling: process_data_arguments orchestrates how the input data (x, y, optional data source) is prepared for plotting, returning a data source and accessors for x/y.

Public API
1) Default management

- set_bokeh_plot_defaults(**kwargs)
  - Description: Update defaults for the Bokeh backend.
  - How it works: Delegates to BokehPlotter.set_defaults with provided keyword arguments.
  - Typical use: Configure default aesthetics (colors, sizes, tools) globally for subsequent plots.

- reset_bokeh_plot_defaults()
  - Description: Reset Bokeh defaults to hardcoded, library-provided values.
  - Typical use: Restore a known baseline if custom defaults caused issues.

- show_bokeh_plot_defaults()
  - Description: Print the currently active Bokeh defaults.
  - Output: A dictionary-like view of active defaults via pprint.

- get_bokeh_help(key)
  - Description: Print the description/documentation for a given default key in the Bokeh backend.
  - Typical use: Discover valid configuration options and their meanings.

- load_bokeh_defaults(filename='plot_bokeh_defaults.json')
  - Description: Load Bokeh defaults from a JSON file.
  - Parameter: filename - path to the defaults file.

- save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)
  - Description: Save current Bokeh defaults to a JSON file.
  - Parameters:
    - filename: destination file.
    - save_complete: if True, store both user-modified and unmodified hardcoded defaults.

2) General plotting routines

- bokeh_scatter(...)
  - Purpose: Create an interactive scatter plot using Bokeh.
  - Signature (highlights):
    - x: array-like or key/name for the x-axis data, or a data source-like object (deprecated usage if passed as first positional arg).
    - y: array-like or key/name for the y-axis data.
    - data: optional data source (e.g., pandas DataFrame) containing x/y data.
    - xlabel, ylabel: axis labels.
    - title: plot title.
    - figure: optional existing Bokeh figure to add the plot to.
    - copy_data: if True, copy the provided data argument to avoid side effects.
    - saveas: output target name/identifier used when saving the plot (default 'scatter').
    - Other kwargs: passed through to the Bokeh scatter function or consumed by the plotting helper.
  - Behavior:
    - If x is a dict, DataFrame, or ColumnDataSource (or None), a deprecation warning is emitted. The code expects data to be supplied via the data keyword and x/y data to be provided as xdata/ydata in kwargs or via standard x/y arguments.
    - Data preparation is done with process_data_arguments(data, x, y, copy_data, single_plot=True, same_length=True, use_column_source=True). This returns a mapping from which the lone entry (entry) and its associated data source (source) are extracted.
    - The plotting helper sets defaults (default_type='function', name=entry.y), processes kwargs, and prepares a figure with plot_params.prepare_figure(title, xlabel, ylabel, figure=figure).
    - The scatter is drawn with p.scatter(x=entry.x, y=entry.y, source=source, **plot_kwargs, **kwargs).
    - Tooltips are added via plot_params.add_tooltips(p, res, entry).
    - Optional z-leveling: if plot_params['level'] is set, apply to the glyph.
    - Additional refinements: draw straight lines, apply axis limits, and save the plot via plot_params.save_plot(p, saveas).
  - Returns: The created Bokeh figure.

- bokeh_multi_scatter(...)
  - Purpose: Create an interactive scatter plot that supports multiple data sets (multiple series) on a single axes.
  - Signature (highlights):
    - Similar to bokeh_scatter with additional controls:
      - set_default_legend: bool to derive default legends from data names.
      - data, figure, xlabel, ylabel, title, copy_data, saveas, and other kwargs as in bokeh_scatter.
  - Behavior:
    - Uses the same data processing and plotting pipeline, extended to accommodate multiple data series and automatic legend generation when requested.
  - Returns: The created Bokeh figure.

Data handling and plotting workflow
- Data sources and extraction:
  - The code relies on process_data_arguments to harmonize inputs (data, x, y). It supports common data structures like pandas DataFrame and Bokeh ColumnDataSource.
  - For single plots, the function retrieves a single entry with x/y accessors and a corresponding data source.
- Consistency and typing:
  - A decorator ensure_plotter_consistency(plot_params) enforces that the plotting call uses the same underlying configuration, ensuring consistent behavior across different plotting helpers.
- Plot configuration and rendering:
  - The BokehPlotter instance (plot_params) is used to:
    - Set defaults and override them via kwargs
    - Prepare the figure (title, axis labels, figure object)
    - Build plot-specific kwargs (plot_kwargs) for the scatter glyph
    - Attach tooltips and interactive features
    - Apply optional level, straight lines, and axis limits
    - Persist the resulting plot to a file or display context through save_plot

Usage examples
- Simple single-series scatter
  - from masci_tools.vis.plotting import bokeh_scatter
  - bokeh_scatter(x=[1, 2, 3], y=[4, 5, 6], xlabel='Index', ylabel='Value', title='Sample Scatter')
- Scatter with data frame
  - df = pandas.DataFrame({'x': [...], 'y': [...]})
  - bokeh_scatter(x='x', y='y', data=df, xlabel='X', ylabel='Y', title='DataFrame Scatter')
- Default configuration
  - set_bokeh_plot_defaults(tools='pan,wheel_zoom,box_zoom,reset')
  - show_bokeh_plot_defaults()
  - reset_bokeh_plot_defaults()

Notes and considerations
- Backends and compatibility:
  - The plotting routines are designed around Bokeh; defaults and behaviors are centralized in BokehPlotter.
- Deprecation warning:
  - Passing the data source as the first positional argument to bokeh_scatter (or bokeh_multi_scatter) is deprecated. Use the data keyword and x/y data via xdata/ydata in kwargs or provide x/y as direct arguments.
- Data integrity:
  - The copy_data flag allows protecting original inputs from in-place modification during processing.
- Extensibility:
  - The design expects new plot types to follow the same pattern: data processing via process_data_arguments, figure preparation via plot_params, and final rendering with the plotter.
