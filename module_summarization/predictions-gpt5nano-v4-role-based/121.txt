Create Cradmin menu system
==========================

Overview
--------

The crmenu module provides a flexible, template-driven system for building the Cradmin header menu. It defines a set of renderable items (links, buttons, expandable items, separators, and a menu toggle) and a base menu renderable to compose these items into a complete menu. The system is intentionally extensible and relies on Django’s template rendering and BEM-style CSS classes for styling.

Key concepts
------------

- Renderables: Small, reusable objects that render to HTML using Django templates. Each item type has its own template and a small API to customize its appearance and behavior.
- BEM naming: The renderables expose methods to determine their parent BEM block and element names, enabling consistent CSS hooks like block__element and block__modifier.
- Expandable menus: Support for items that expand to show nested lists, with dedicated templates for items and separators.
- Extensibility via settings: A default expandable menu class can be configured, allowing you to inject your own menu implementation at runtime.

Public API
----------

Note: The following API pieces are part of crmenu.rst to document how to create and customize menus.

1) get_default_expandable_menu_renderable
   - Purpose: Create the default expandable menu renderable as configured by Django settings.
   - Signature: get_default_expandable_menu_renderable(**kwargs)
   - Behavior:
     - Reads DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS from settings.
     - If not set, returns None.
     - If set, imports the class path and instantiates it with the provided kwargs.
   - Practical usage:
     - Allow a project to provide a custom expandable menu class that integrates with the rest of the Cradmin UI.
   - Example:
     - settings.DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.menus.MyExpandableMenu'
     - menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin)

2) BaseMenuLinkRenderable
   - Purpose: Shared base for menu link items (normal links and button-styled links).
   - Template: 'django_cradmin/crmenu/menuitem/link.django.html'
   - Constructor:
     - label: Text shown for the link
     - url: Destination URL
     - is_active: If True, render as the active link
     - bem_variant_list: Optional list of BEM variants
     - parent_bem_block: Optional explicit BEM block name
     - **kwargs: Passed to the parent renderable
   - Key methods:
     - get_fallback_parent_bem_block(): Returns 'adminui-page-header' by default
     - get_parent_bem_block(): Returns the explicit parent_bem_block or the fallback
     - get_bem_element(): Returns the BEM element for the link (parent block __navlink)
     - get_active_bem_variant_list(): Returns ['active'] when active
     - get_bem_variant_list(): Returns base variants; appends 'active' when is_active is True
   - Use case: Create standard navigation links in the Cradmin header.

3) NavLinkItemRenderable
   - Subclass of BaseMenuLinkRenderable
   - Purpose: Simple navigation link to add to the main menu.

4) NavLinkButtonItemRenderable
   - Subclass of NavLinkItemRenderable
   - Purpose: Navigation link rendered as a button
   - Behavior: get_bem_variant_list() returns ['button']

5) ExpandableMenuItem
   - Subclass of BaseMenuLinkRenderable
   - Template: 'django_cradmin/crmenu/menuitem/expandable-menu-item.django.html'
   - Behavior:
     - get_fallback_parent_bem_block(): Returns 'expandable-menu'
     - get_bem_element(): Returns '{parent}__link'
     - get_list_element_bem_element(): Returns '{parent}__listitem'
   - Use case: Menu items that can expand to reveal sub-items.

6) ExpandableMenuSeparator
   - Subclass of renderable.AbstractBemRenderable
   - Template: 'django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html'
   - Constructor: accepts optional parent_bem_block
   - Behavior:
     - get_fallback_parent_bem_block(): Returns 'expandable-menu'
     - get_parent_bem_block(): Uses explicit parent or fallback
     - get_bem_element(): Returns '{parent}__separator'
   - Use case: Visual separation between groups of items inside an expandable menu.

7) MenuToggleItemItemRenderable
   - Subclass of renderable.AbstractBemRenderable
   - Template: 'django_cradmin/crmenu/menuitem/menutoggle.django.html'
   - Constructor: accepts optional parent_bem_block
   - Behavior:
     - get_label(): Returns a translatable label, defaulting to 'Menu'
     - get_fallback_parent_bem_block(): Returns 'adminui-page-header'
     - get_parent_bem_block(): Uses explicit parent or fallback
     - get_bem_element(): Returns '{parent}__navmenutoggle'
     - get_label_css_class(): Returns '{parent}__navmenutoggle-label'
     - get_icon_css_class(): Returns '{parent}__navmenutoggle-icon'
   - Use case: The interactive control to toggle the visibility of an expandable menu.

8) AbstractMenuRenderable
   - Base class for rendering a menu
   - Inherits from renderable.AbstractRenderableWithCss
   - Class attributes (to be configured by subclasses or instances):
     - link_renderable_class: The renderable class used for links
     - separator_renderable_class: The renderable class used for separators
     - button_renderable_class: The renderable class used for buttons
   - Constructor:
     - __init__(self, request, cradmin_instance=None)
       - Stores request and cradmin_instance
       - Initializes an empty renderable_list
   - API methods:
     - iter_renderables(): Returns an iterator over the items in renderable_list
     - get_link_renderable_class():
       - Returns the configured link_renderable_class
       - If not defined, raises ValueError indicating no link_renderable_class is defined for the concrete class
     - get_separator_renderable_class(): Returns the configured separator_renderable_class
       - (Documentation note: The code snippet indicates this method exists; its exact behavior is described in the project’s implementation.)

Usage patterns
--------------

- Building a menu
  - Create concrete renderable instances (NavLinkItemRenderable, NavLinkButtonItemRenderable, ExpandableMenuItem, etc.) and append them to an AbstractMenuRenderable-derived instance's renderable_list.
  - The concrete menu class should specify which renderable classes to use via link_renderable_class, separator_renderable_class, and button_renderable_class.
  - The menu object is then passed to templates via your Cradmin view context so it can render the header menu.

- Customizing the default expandable menu
  - Provide a custom class (subclassing AbstractMenuRenderable or an existing menu class) and wire it via DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS.
  - The helper get_default_expandable_menu_renderable(**kwargs) will instantiate your class with the supplied kwargs (e.g., request, cradmin_instance).

- Translation and accessibility
  - The Menu toggle label is retrieved via pgettext_lazy, enabling translation of the default label ('Menu').

Templates
---------

The menu items render using the following templates:

- Link items: django_cradmin/crmenu/menuitem/link.django.html
- Expandable menu items: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
- Expandable menu separators: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
- Menu toggle: django_cradmin/crmenu/menuitem/menutoggle.django.html

Implementation notes
--------------------

- The system relies on Django’s import_string to resolve a class path from settings, enabling easy swapping of menu implementations.
- Each renderable uses BEM-style class naming to keep CSS predictable and extensible.
- The default behavior is designed to be Safe for most projects, while still allowing deep customization by subclassing or configuring renderable classes.

Examples
--------

- Example: Create and render a simple set of links using the provided NavLinkItemRenderable and a hypothetical menu class

  - Create a menu class (subclass of AbstractMenuRenderable) that builds its renderable_list with instances of NavLinkItemRenderable, NavLinkButtonItemRenderable, and possibly ExpandableMenuItem with nested items.
  - In a view, instantiate the menu with the current request and Cradmin instance, and provide it to the template for rendering.

- Example: Use a custom default expandable menu

  - In settings.py:
    DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.menus.MyExpandableMenu'
  - In code:
    menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin)

Notes for maintainers
---------------------

- If you introduce new renderable classes, ensure they provide sensible defaults for get_parent_bem_block and related BEM helpers to keep styling consistent.
- When overriding templates, keep the same context variables expected by the base renderables to avoid breaking changes.

This documentation provides the essential pieces to understand and build the Cradmin menu system as exposed by crmenu.rst, including how to customize defaults, what each renderable represents, and how to compose a menu for the Cradmin header.
