Kernel approximation (kernel_approximation.rst)
===============================================

Overview
--------
The kernel_approximation module implements several approximate kernel feature maps based on Fourier transforms and Count Sketches. These feature maps provide scalable, low-dimensional representations that mimic the behavior of kernel methods for various kernels.

This documentation focuses on the functions/classes that approximate the feature mappings corresponding to specific kernels, enabling linear models to operate in an implicit kernel space.

Polynomial kernel approximation
-------------------------------
Class: PolynomialCountSketch

Purpose
-------
PolynomialCountSketch provides an approximate feature map for the polynomial kernel

    K(X, Y) = (gamma * <X, Y> + coef0) ** degree

by using a Tensor Sketch (Count Sketch) combined with FFTs to efficiently produce a lower-dimensional representation of the polynomial feature space.

Key ideas
- Outer product hashing: for each degree, features are hashed with two 2-wise independent hash functions to a common output dimension.
- Count Sketch: random bit signs (+1 / -1) and index hashes are used to compress high-dimensional interactions into a compact representation.
- Tensor Sketch via FFT: the combination across feature interactions is performed efficiently in the Fourier domain, enabling a scalable approximation of the degree-th polynomial feature map.

Parameters
----------
gamma : float, default=1.0
    Coefficient multiplying the inner product <X, Y> in the polynomial kernel.

degree : int, default=2
    Degree of the polynomial kernel to approximate.

coef0 : float, default=0
    Constant term added to the inner product before exponentiation.

n_components : int, default=100
    Dimensionality of the output feature space (the size of the approximate feature map).

random_state : int, RandomState instance, default=None
    Seed or state used to initialize the random hash functions (indexHash_ and bitHash_).

Attributes
----------
indexHash_ : ndarray of shape (degree, n_features)
    Integer indices in the range [0, n_components) used to map degree-wise interactions
    to the output feature space via the Count Sketch.

bitHash_ : ndarray of shape (degree, n_features)
    Random entries in {+1, -1} used to sign features in the Count Sketch.

n_features_in_ : int
    Number of features seen during fit (i.e., the number of features in the input).

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features seen during fit (only if input feature names are provided as strings).

Notes on behavior
-----------------
- The implementation validates input shapes and supports both dense arrays and sparse matrices (CSC format during fitting).
- The approximate feature map is designed to preserve kernel similarities while reducing dimensionality, trading some approximation error for computational efficiency.
- For practical performance, a typical target dimensionality is around 10 times the original number of features, though the optimal value depends on the data and task.

Usage
-----
- Create the transformer with the desired kernel parameters.
- Call fit(X) to initialize internal random hashes based on the training data.
- Call transform(X) (or fit_transform) to obtain the approximate feature representation.

Example
-------
>>> from sklearn.kernel_approximation import PolynomialCountSketch
>>> X = [[0, 0], [1, 1], [1, 0], [0, 1]]
>>> ps = PolynomialCountSketch(degree=3, random_state=1)
>>> X_features = ps.fit_transform(X)
>>> X_features.shape
(4, 100)

Notes
-----
- The class is part of the larger kernel_approximation module, which also includes other approximate kernel feature maps such as AdditiveChi2Sampler, Nystroem, RBFSampler, and SkewedChi2Sampler. See See Also below for references.

Parameters (validation)
-----------------------
The class enforces parameter constraints consistent with its role as a kernel-approximation transformer:
- gamma: Real, gamma >= 0
- degree: integer >= 1
- coef0: Real (unconstrained, can be any real number)
- n_components: integer >= 1
- random_state: random_state generator

Implementation and internal workflow
-----------------------------------
- fit(X): determines n_features from X, adjusts for coef0 if present, and initializes:
  - indexHash_ with random integers in [0, n_components)
  - bitHash_ with random signs in {-1, +1}
  - _n_features_out is set to n_components
- transform(X): computes the approximate polynomial feature map using Tensor Sketch and FFT-based convolution across degrees, returning an array of shape (n_samples, n_components).

See Also
--------
- AdditiveChi2Sampler: Approximate feature map for additive Chi-squared kernel.
- SkewedChi2Sampler: Approximate feature map for a skewed Chi-squared-like kernel.
- RBFSampler: Random Fourier features for approximating the RBF kernel.
- Nystroem: Kernel map approximation using a subset of training data.
- sklearn.metrics.pairwise.kernel_metrics: List of built-in kernels.

Examples (from the class docstring)
-----------------------------------
>>> from sklearn.kernel_approximation import PolynomialCountSketch
>>> from sklearn.linear_model import SGDClassifier
>>> X = [[0, 0], [1, 1], [1, 0], [0, 1]]
>>> y = [0, 0, 1, 1]
>>> ps = PolynomialCountSketch(degree=3, random_state=1)
>>> X_features = ps.fit_transform(X)
>>> clf = SGDClassifier(max_iter=10, tol=1e-3)
>>> clf.fit(X_features, y)
SGDClassifier(max_iter=10)
>>> clf.score(X_features, y)
1.0

See Also Reference
------------------
- PolynomialCountSketch is primarily designed for polynomial kernel approximation. For other kernels, consider the alternative kernel-approximation classes (RBFSampler, AdditiveChi2Sampler, SkewedChi2Sampler, Nystroem).

Notes for developers
--------------------
- The class relies on SciPy’s FFT interface (scipy.fft if available, otherwise scipy.fftpack for older SciPy versions) to perform efficient tensor sketches.
- The random hashing (indexHash_, bitHash_) is degree-wise, enabling the combination of feature interactions up to the specified degree in a memory-efficient way.
