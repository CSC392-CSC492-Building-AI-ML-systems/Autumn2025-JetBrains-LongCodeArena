Condition syntax in PyTables
=============================

This section documents the syntax and semantics of conditions used by PyTables to filter data. Conditions are boolean expressions evaluated against table columns to select rows (e.g., via read_where, where, or similar APIs).

Overview
--------

- A condition is a boolean expression written as a string that references table columns.
- PyTables evaluates conditions chunk-by-chunk, using optimised search paths for numeric data when possible.
- For performance, PyTables builds and maintains a condition cache. If the underlying data changes in a way that affects the condition evaluation, the cache is invalidated (nail/unnail) to ensure correct results.

Key concepts in this file
-------------------------

- Column references: Conditions reference table columns by their names (or their full path when necessary). Refer to a column directly by name within the condition string when evaluated in the context of a Table.
- Operators: Standard comparison operators are supported (at least: >, <, >=, <=, ==, !=). Complex conditions are created by combining simple expressions.
- Boolean combination: Use bitwise operators to combine conditions within strings (typical PyTables usage uses & for logical AND and | for logical OR; parentheses are recommended to enforce precedence).
- NaN handling: PyTables provides NaN-aware comparison helpers (nan_aware_gt, nan_aware_ge, nan_aware_lt, nan_aware_le) to ensure robust behavior when columns contain NaN values. These are used internally by the condition evaluation engine to produce predictable results.

Supported data types and optimisations
--------------------------------------

- The engine supports a range of numeric types with optimised search implementations (e.g., int8, int16, int32, int64, uint8, uint16, uint32, uint64, float32, float64).
- For these types, PyTables can employ specialised (faster) search paths during condition evaluation.
- The default sorting method used for internal condition processing is mergesort (defsort = "mergesort").

Condition cache and indexing
---------------------------

- PyTables maintains a condition cache to accelerate repeated condition evaluations when the underlying data has not changed.
- If a table’s index becomes dirty (i.e., the data it depends on has been modified), the condition cache is notified:
  - When a column/index becomes dirty, the cache is nails (conditioncache.nail()).
  - When the data is revalidated and the index becomes clean again, the cache is unnail()’ed.
- The Index class exposes a dirty property to reflect this state and to trigger cache updates accordingly.

Practical usage
---------------

- Basic single-condition query:
  - read_where("temperature > 25.0")
  - Reads rows where the temperature column is greater than 25.0.

- Combined conditions (use parentheses for clarity):
  - read_where("(temperature > 25.0) & (humidity < 0.8)")
  - Reads rows where temperature > 25.0 AND humidity < 0.8.

- String comparisons:
  - read_where("status == 'OK'")
  - Reads rows where the status column equals the string "OK".

- Be mindful of NaN values:
  - NaN-aware comparisons can be used to obtain robust results in the presence of missing data. PyTables uses nan_aware_* helpers internally to ensure consistent behavior when evaluating conditions involving NaN.

Syntax notes and best practices
--------------------------------

- Use parentheses to group expressions and avoid precedence pitfalls.
- Use & for AND and | for OR when combining conditions inside a condition string.
- Quote string literals using standard Python string syntax (e.g., 'OK', "FAILED").
- If a column name is unusual (e.g., contains spaces or special characters), you may need to reference its full path in the condition context (as a path string). Refer to the table documentation for exact path syntax in such cases.

Examples
--------

- Simple numeric filter:
  - read_where("age >= 18")

- Composite condition:
  - read_where("(age >= 18) & (income > 50000)")

- String comparison:
  - read_where("category == 'A'")

- NaN-aware usage (internal guidance; rely on PyTables defaults for NaN handling):
  - PyTables may apply nan_aware_* semantics for comparisons involving NaN values in numeric columns.

Notes
-----

- Condition syntax is tightly coupled with the table’s indexing and the condition cache. If you mutate data in a way that affects indexed columns, PyTables may require reindexing and will invalidate the condition cache to preserve correctness.
- The engine automatically selects fast search paths for supported numeric types (see opt_search_types) to speed up condition evaluation on large datasets.
