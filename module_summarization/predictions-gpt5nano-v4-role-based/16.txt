Plothist command
=================

Overview
--------
The Plothist command plots histogram data produced by WestPA’s fast histograms. Implemented as a WestPA subcommand, it reads an HDF5 file containing histogram data, selects a dimension (or 2D dimensions for a heatmap), and renders a plot using matplotlib. It supports multiple output formats, axis scaling options (linear, log10, or an energy-based scale), optional contour overlays for 2D plots, and a post-processing hook that can modify the plot before saving.

Key behavior
- Input: HDF5 file containing histogram data.
- Dimension selection: Choose a dimension to plot, with optional bounds and axis label.
- Plot types: 1D heatmaps (histogram vs. bin midpoints) or 2D heatmaps (two dimensions forming a 2D histogram).
- Scales: linear, log10, or an inverted natural-log energy scale (default).
- Range control: Y-range for 1D plots; color-range for 2D plots.
- Contours: Optional contour overlay for 2D plots.
- Postprocessing: Optional function to modify the plot before saving.
- Outputs:
  - Plot file (default hist.pdf). Can be suppressed by giving an empty string for the plot output.
  - Optional HDF5 output containing plot data.
  - Internal plot rendering relies on matplotlib’s NonUniformImage for heatmaps.

Usage synopsis
--------------
plothist input_h5_file [DIMENSION_SPEC] [options]

Where:
- input_h5_file: Path to the HDF5 file containing histogram data.
- DIMENSION_SPEC: Optional specification describing the dimension to plot, in the form described below (DIMENSION: LB,UB:LABEL). If omitted, dimension 0 with full range is used.

Arguments
---------
- input_h5_file
  - Path to the HDF5 file containing histogram data.

- DIMENSION_SPEC (optional)
  - Description: Plot for a given dimension identified by an index and optionally constrained by lower/upper bounds and labeled for the axis.
  - Format: INT[:[LB,UB]:LABEL]
    - INT: zero-based dimension index within the histogram data.
    - LB, UB: optional lower and upper bounds for plotting (per-dimension units).
    - LABEL: axis label for the dimension.
  - Default: dimension 0, full range, with label “dimension 0”.

Options
-------
Output options
- -o, --output, --plot-output
  - Destination for the generated plot. If set to an empty string, plotting is suppressed.
  - Default: hist.pdf
- --hdf5-output
  - Path to an HDF5 file where plot-related data will be stored.
- --plot-contour
  - If set, overlay a contour on 2D objects (heatmaps).

Plot options
- --title
  - Include a TITLE at the top of the graph.
- --linear
  - Plot the histogram on a linear scale.
- --energy
  - Plot the histogram on an inverted natural-log scale (default). Interpreted as a (free) energy-like scale.
- --zero-energy
  - Sets the zero of the energy scale. Can be a numeric value or one of 'min' or 'max'.
- --log10
  - Plot the histogram on a base-10 logarithmic scale.
- --range
  - Plot range for the ordinate (Y axis for 1D plots; color axis for 2D plots). Format: LB,UB
  - Default: full range
- --postprocess-function
  - Name of a function (module.function) to call just before saving the plot.
  - Signature: postprocess(hist, midpoints, binbounds)
    - hist: the histogram array to be plotted
    - midpoints: the bin midpoints for each dimension
    - binbounds: the bin boundaries for each dimension (2D plots only) or None otherwise
  - The function modifies the plot in place via the matplotlib state.

Defaults and interpretation
- Default plot scale: energy
- Default energy zero point: min
  - Accepts numeric values, or strings 'min' or 'max'
- If no dimension is supplied, dimension 0 is used with the default full range.

Iteration controls (average/evolution vs. single point)
- avail_iter_start / avail_iter_stop / avail_iter_step
  - Represent the available iteration range for average/evolution plots, as read from the HDF5 input.
- iter_start / iter_stop / iter_step
  - Represent the specific iteration range to plot (for a single-point plot, for example).
- These controls are derived from the input HDF5 data and are used to constrain the plotted data.

Internal mechanics (brief)
- The command opens the input HDF5 file with h5py and uses WestPA’s h5io utilities to infer iteration ranges.
- The plotting logic uses matplotlib, and for heatmaps it utilizes NonUniformImage.
- A small helper, sum_except_along, reduces arrays by summing over all axes except a specified subset; this supports dimension selection and aggregation when constructing plots.

Output format details
- Plot files: The output format is inferred from the file extension of plot_output (e.g., .pdf → PDF). If the path ends with an empty string, plotting is suppressed.
- Data output: If --hdf5-output is provided, the script stores relevant plot data into the specified HDF5 file for later reuse or inspection.

Implementation notes
- Core classes:
  - PlotHistBase (inherits from WESTSubcommand): encapsulates common argument parsing and processing shared by plothist.
  - The class handles argument parsing, input HDF5 opening, and preparation of plotting state (dimension, range, scale, etc.).
- Helpers:
  - sum_except_along(array, axes): utility to reduce arrays by summing along all axes except those listed, used for preparing data along specific dimensions.
- Dependencies:
  - h5py for HDF5 interaction
  - NumPy for numerical operations
  - Matplotlib for plotting
  - WestPA core modules (h5io, textio) and utilities (normhistnd, get_object)

Examples
-------
1) Basic 1D plot (dimension 0, full range)
   plothist input.h5 0

2) 1D plot with a specific range and custom label
   plothist input.h5 0:-1.0,1.0:Time --range -0.5,0.5 --plot-output=hist_time.pdf

3) 2D plot with contour overlay and explicit outputs
   plothist input.h5 1:-5,5:Density --plot-contour --range -2,2 --plot-output=density2d.pdf --hdf5-output=density2d.h5

4) Suppress plotting and only emit data
   plothist input.h5 0 --plot-output=''

See also
--------
- WestPA plotting utilities and histograms (fasthist) for related functionality.
- h5io, textio: WestPA core input/output utilities.
- NonUniformImage, matplotlib: rendering of irregular heatmaps.

Notes
-----
- The exact DIMENSION syntax is documented in the help text of the command, which should be consulted when providing a dimension specification. The parser accepts a form like INT[:[LB,UB]:LABEL].
- The Plothist command is designed to be extended or wrapped by higher-level workflows through the postprocess-function hook, allowing custom styling or annotation after the histogram is prepared but before it is saved.
