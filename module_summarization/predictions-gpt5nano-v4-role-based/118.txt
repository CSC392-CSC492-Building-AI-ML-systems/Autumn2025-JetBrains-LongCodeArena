Configuration
=============

Overview
--------
This document describes the configuration options used to control the import/export behavior
for various components within the PyAEDT configuration system. It centers on the
ConfigurationsOptions class, which provides a set of boolean toggles and a tolerance
setting to govern which items are exported to or imported from JSON representations, as
well as a small helper function used to extract dataset references from nested data
structures.

Helper: _find_datasets
----------------------
- Purpose: Traverse a nested dictionary/list structure to locate dataset references encoded
  as Power-Law (pwl) values within datasets.
- Behavior:
  - Recursively navigates dictionaries (including OrderedDict) to locate dataset entries.
  - For entries that appear as lists, iterates over elements to find “free_form_value” fields
    containing a pwl reference, and extracts the substring between the first “$” and the first comma.
  - For string values, collects pwl references when encountered.
- Output: Appends discovered dataset identifiers (as strings) to the provided out_list.
- Notes: The function is intended for internal use during the serialization/deserialization
  process to track referenced datasets.

Class: ConfigurationsOptions
----------------------------
A dedicated options object that lets users enable or disable specific export/import features.
This class is the central place to configure how a project is serialized to or from JSON.

Usage
-----
Typical usage pattern:

- Disable exporting of variables:
  - hfss.configurations.options.export_variables = False

- Disable exporting of setups and optimizations:
  - hfss.configurations.options.export_setups = False
  - hfss.configurations.options.export_optimizations = False

- Enable or adjust the object mapping tolerance if needed:
  - hfss.configurations.options.object_mapping_tolerance = 1e-8

Notes
-----
- The ConfigurationsOptions class is designed to be a lightweight, explicit container for
  user preferences regarding import/export behavior. It does not perform the import/export
  itself; it merely communicates preferences to the underlying serialization logic.
