Naive Bayes classifiers
=======================

The naive_bayes module implements Naive Bayes algorithms for supervised classification.
Naive Bayes classifiers use Bayes’ theorem with strong (naive) feature independence
assumptions. They are typically fast, simple, and work well with high-dimensional data.

Public API
----------

Classes
^^^^^^^

- BernoulliNB
- MultinomialNB
- GaussianNB
- ComplementNB
- CategoricalNB

Abstract base class
^^^^^^^^^^^^^^^^^^^

- _BaseNB
  - Abstract base class for Naive Bayes estimators.
  - Public methods implemented in this base class:
    - predict_joint_log_proba(X): returns joint log probabilities log P(x, y) for each sample and class.
    - predict(X): returns predicted class labels for X.
    - predict_log_proba(X): returns log-probability estimates log P(y|x) (up to a normalization by P(x)).
    - predict_proba(X): returns probability estimates P(y|x).

Key concepts
************

- Joint log likelihood: The base interface exposes _joint_log_likelihood(X) which computes log P(y) + log P(x|y) for all samples and classes. Subclasses implement the specific P(x|y) model.
- Input checks: predict*, predict_proba*, and related methods perform input validation and _check_X before computing the joint log likelihood.

Gaussian Naive Bayes
********************

GaussianNB
----------

Gaussian Naive Bayes models the features as conditionally independent Gaussians given the class.
It supports online updates via partial_fit and provides a standard implementation of Gaussian NB
for continuous features. See the user guide for details.

Parameters
----------

- priors : array-like of shape (n_classes,), default=None
  Prior probabilities of the classes. If specified, priors are not adjusted from the data.

- var_smoothing : float, default=1e-9
  Portion of the largest variance of all features that is added to variances for numerical stability.

Notes
-----

- GaussianNB is designed for continuous features and estimates per-class means (theta_) and variances (var_).
- It computes class_count_ (number of training samples per class) and class_prior_ (class prior probabilities).
- Attributes added during fit (and versions):
  - classes_ : array of shape (n_classes,)
  - theta_     : array of shape (n_classes, n_features)
  - var_       : array of shape (n_classes, n_features)
  - class_count_ : array of shape (n_classes,)
  - class_prior_ : array of shape (n_classes,)
  - epsilon_    : float (absolute additive value to variances for stability)
  - n_features_in_ : int (number of features seen during fit)
  - feature_names_in_ : ndarray of shape (n_features_in_,) (feature names, if provided as strings)
- partial_fit: supports online updates to model parameters.

Example
-------

from sklearn.naive_bayes import GaussianNB

# Training data
X = [[1.0, 2.0], [1.1, 1.9], [3.0, 4.0], [3.2, 4.1]]
y = [0, 0, 1, 1]

gnb = GaussianNB(priors=None, var_smoothing=1e-9)
gnb.fit(X, y)

# Predict on new samples
print(gnb.predict([[1.2, 2.1]]))
print(gnb.predict_proba([[1.2, 2.1]]))

API usage notes
****************

- predict_joint_log_proba(X) returns log P(x, y) for each sample and class.
- predict_log_proba(X) returns log P(y|x) up to a normalization by P(x); the normalization is performed internally using log-sum-exp.
- predict_proba(X) is the exponentiated form of predict_log_proba(X).

Other Naive Bayes classifiers (brief)
-------------------------------------

- BernoulliNB: Naive Bayes for binary/boolean features.
- MultinomialNB: Naive Bayes for discrete counts (e.g., word/frequency features).
- ComplementNB: An improvement of MultinomialNB for imbalanced datasets.
- CategoricalNB: Naive Bayes for unconditional categorical features.

Internal design
****************

- The module defines a common interface through _BaseNB, including the joint log-likelihood contract and the standard prediction methods.
- Subclasses implement _joint_log_likelihood(X) and _check_X(X) to handle their specific feature representations and checks.
- Input validation utilities and preprocessing helpers (e.g., LabelBinarizer, binarize, label_binarize) are leveraged by concrete estimators as needed.

Implementation notes
********************

- The module uses log-domain computations for numerical stability.
- The joint log-likelihood is defined as log P(y) + log P(x|y).
- The public methods expose a consistent API across different Naive Bayes variants, while each class handles its own feature modeling assumptions.
