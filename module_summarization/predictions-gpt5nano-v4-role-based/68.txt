Gaussian Process Kernels
========================

Kernels for Gaussian process regression and classification.

Overview
--------

This module provides a collection of kernels that can be used with Gaussian
process models. Key features include:

- Kernel algebra: kernels can be combined with addition (+), multiplication
  (*), or exponentiation (**). Scalar values appearing in expressions are
  automatically converted to a constant kernel.
- Gradient-based hyperparameter optimization: all kernels expose
  hyperparameters that can be optimized analytically.
- Hyperparameter search space: each hyperparameter can be described by lower
  and upper bounds, forming a rectangular search space. Hyperparameters can be
  declared as "fixed" to exclude them from optimization.
- Strongly typed hyperparameters: hyperparameters are represented using a
  Hyperparameter namedtuple, containing metadata about the parameter.

The module is strongly inspired by the kernel module of the george package.

Key concepts
------------

- Kernel expressions
  - Kernels may be combined using:
    - Addition: kernel1 + kernel2 represents a sum-kernel.
    - Multiplication: kernel1 * kernel2 represents a product-kernel.
    - Exponentiation: kernel1 ** scalar exponent (e.g., 0.5) for exponentiated
      kernels.
  - Scalars in expressions are automatically converted to a constant kernel.

- Hyperparameters and optimization
  - Hyperparameters define tunable aspects of kernels (e.g., length-scales,
    variances).
  - Hyperparameters can be constrained by a rectangular search space using
    bounds, or marked as fixed to disable optimization.
  - The Hyperparameter data structure captures:
    - name: parameter name (used to access properties on the kernel)
    - value_type: type of the parameter (e.g., "numeric")
    - bounds: 2D bounds array or the string "fixed"
    - n_elements: number of elements (1 for scalar, >1 for vector-valued)
    - fixed: whether the parameter is fixed (not optimized)

- Length-scale handling
  - The _check_length_scale helper validates length scales for isotropic vs
    anisotropic kernels and ensures dimensional consistency with the input
    data.

Key classes and functions
-------------------------

Hyperparameter
^^^^^^^^^^^^^^

A lightweight, memory-efficient representation of a kernel hyperparameter.

Attributes
- name: str
  The hyperparameter name. A kernel using a hyperparameter named "x" should expose
  an attribute self.x and a corresponding self.x_bounds.
- value_type: str
  The type of the hyperparameter (currently "numeric" is supported).
- bounds: array-like of shape (n_elements, 2) or the string "fixed"
  The lower and upper bounds for the parameter. If n_elements > 1, a 2D array
  with n_elements rows is used. If "fixed" is provided, the parameter is fixed.
- n_elements: int
  Number of elements of the hyperparameter value. 1 for scalar, >1 for vector-valued
  hyperparameters (e.g., anisotropic length-scales).
- fixed: bool
  Whether the hyperparameter is fixed (not optimized). If None, it is inferred
  from bounds.

Notes
- Hyperparameter.__new__ enforces bounds shape compatibility and defaulting of
  the fixed flag when bounds are provided as a non-"fixed" value.
- Hyperparameter implements equality (__eq__) to facilitate testing and
  comparison of hyperparameter configurations.

Example
.. code-block:: python

   # Conceptual example; actual kernels may provide concrete hyperparameters
   # depending on their implementation.
   h = Hyperparameter(name="length_scale", value_type="numeric",
                      bounds=[[0.1, 3.0]], n_elements=1)

   # Accessing hyperparameters of a kernel (iterable):
   for hp in kernel.hyperparameters:
       print(hp)

_check_length_scale
^^^^^^^^^^^^^^^^^^^

Validates the length-scale specification used by kernels with potential
anisotropy.

Behavior
- Squeezes and converts the length_scale to a 1D float array.
- If the length-scale has more than one dimension, an error is raised
  (anisotropic kernels must be handled via a vector-valued length-scale).
- If the length-scale is anisotropic (1D array) and
  X.shape[1] != length_scale.shape[0], a ValueError is raised to ensure
  that the kernel dimensionality matches the data.

Kernel
^^^^^^

Abstract base class for all kernels.

Public API
- get_params(self, deep=True)
  Retrieve kernel parameters as a dictionary mapping parameter names to
  their values. The implementation introspects the constructor signature to
  discover parameter names. If deep is True, parameters of contained estimators
  are also retrieved (mirroring scikit-learn conventions).

Notes
- The Kernel class serves as the foundation for all concrete kernel
  implementations in this module. Subclasses declare their tunable parameters
  via the constructor, and may expose a hyperparameter collection (e.g.,
  kernel.hyperparameters) used by optimization routines.

API compatibility and imports
-----------------------------

This module relies on:

- clone from the package’s base utilities
- ConvergenceWarning for numerical optimization notices
- pairwise_kernels for pairwise kernel computations
- _num_samples from the validation utilities
- SciPy’s distance metrics and special functions for advanced kernel forms

Usage patterns
--------------

- Combine kernels with + and * to form composite kernels.
- Use exponentiation to raise a kernel to a scalar power.
- Include scalar constants in expressions by automatic conversion to a constant kernel.

Example (conceptual)
.. code-block:: python

   # Conceptual composite kernel (actual classes depend on implementation)
   k = (RBF(length_scale=1.0) + ConstantKernel(constant_value=2.0)) ** 0.5

   # Access parameters for inspection or optimization
   params = k.get_params()

   # Iterate over hyperparameters for tuning
   for hyper in k.hyperparameters:
       print(hyper)
