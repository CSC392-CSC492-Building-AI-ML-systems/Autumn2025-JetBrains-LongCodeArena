Working with Text Data
======================

Analyzing a collection of text documents
---------------------------------------

This section describes how the text feature extraction components in sklearn analyze a collection of text documents. It covers the preprocessing, tokenization, n-gram generation, stop-word handling, and how input is decoded and transformed into numeric feature representations.

Key concepts
------------

- Preprocessing pipeline
  - Decoding: text input can be a filename, a file-like object, or a string. Decoding handles binary to text conversion using a specified encoding and error policy.
  - Lowercasing: option to convert all text to lower-case prior to tokenization.
  - Accent handling: choice of strategies to normalize or remove accented characters (strip_accents_unicode, strip_accents_ascii).
  - Optional preprocessing and tokenization steps can be composed or replaced by a single analyzer.

- Analyzers and how they compose steps
  - The _analyze function chains together multiple processing steps to convert a document into tokens or n-grams.
  - If an analyzer is supplied, it replaces the preprocessor, tokenizer, and n-grams steps. Otherwise, the document passes through optional preprocessor, tokenizer, and n-grams functions.
  - Stop words can be supplied to filter tokens during n-gram construction.

- Stop words
  - A stop-list can be a built-in list (e.g., "english"), a user-provided collection, or None.
  - The helper _check_stop_list converts the input into a usable form (e.g., a frozenset) for fast membership testing.

- Tokenization and n-grams
  - Tokens can be filtered by stop words and combined into n-grams (e.g., unigrams, bigrams, trigrams) as part of the tokenization step.
  - The _word_ngrams method demonstrates how stop words are removed and how token sequences can be expanded into n-grams.

- Input handling (decoding)
  - The Vectorizer mixin provides a decode method to read and decode input documents:
    - input="filename": open the file in binary mode and read bytes.
    - input="file": read from a file-like object.
    - If the input is bytes, decode using the configured encoding and error policy.
    - np.nan is treated as an invalid document.

- Text normalization helpers
  - strip_accents_unicode: converts accented characters to their closest ASCII-like representation using Unicode normalization, with a slower but more general approach.
  - strip_accents_ascii: a faster approach that transliterates to ASCII where possible.
  - strip_tags: removes basic HTML/XML tags using a regex-based stripper.

Documentation references (from the code)
----------------------------------------

- _preprocess(doc, accent_function=None, lower=False)
  - Chains optional lowercasing and an accent-function to a document.

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  - Builds the processing pipeline, either via a custom analyzer or via preprocessor/tokenizer/ngrams with optional stop words.

- strip_accents_unicode(s)
  - Remove accents using Unicode normalization; fast path for ASCII-only strings; otherwise strip combining marks.

- strip_accents_ascii(s)
  - Normalize to NFKD and encode to ASCII, ignoring non-ASCII; returns ASCII-only text.

- strip_tags(s)
  - Remove HTML/XML-like tags with a simple regular expression.

- _check_stop_list(stop)
  - Resolve stop-list specification:
    - "english" -> ENGLISH_STOP_WORDS
    - None -> None
    - collection -> frozenset(collection)
    - string (other than built-in) raises ValueError

- _VectorizerMixin.decode(self, doc)
  - Decode input documents to strings or raise errors:
    - input == "filename": read file in binary mode and decode
    - input == "file": read from file-like object
    - decode with self.encoding and self.decode_error
    - raise ValueError if doc is np.nan

- _word_ngrams(self, tokens, stop_words=None)
  - Convert tokens into a sequence of n-grams, with optional stop-word filtering.

Practical usage example
-----------------------

Goal: Analyze a collection of documents and obtain a numeric representation (e.g., term-frequency or TF-IDF).

Code example (conceptual):

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer

   documents = [
       "The quick brown fox jumps over the lazy dog.",
       "A quick movement of the enemy will ruin the well-being of the fox.",
       # more documents...
   ]

   # Basic analysis with English stop words and lowercasing
   vectorizer = CountVectorizer(
       stop_words="english",
       lowercase=True,
       strip_accents="ascii"  # uses strip_accents_ascii under the hood
   )
   X = vectorizer.fit_transform(documents)
   feature_names = vectorizer.get_feature_names_out()

   # Or using TF-IDF representation
   tfidf_vectorizer = TfidfVectorizer(
       stop_words="english",
       lowercase=True,
       strip_accents="ascii"
   )
   X_tfidf = tfidf_vectorizer.fit_transform(documents)

What this achieves
------------------

- Converts a collection of raw text documents into a numeric matrix suitable for machine learning.
- Applies preprocessing steps (lowercasing, accent stripping, tag removal if desired) before tokenization.
- Filters out common stop words to focus on informative terms.
- Generates unigrams or higher-order n-grams to capture context and phrasing.
- Offers a flexible pipeline where an analyzer can replace multiple steps for custom tokenization.

Tips and design notes
---------------------

- Analyzer replacement:
  - If you supply an analyzer, it replaces preprocess/tokenize/ngrams. Use this for custom tokenization or language-specific processing.
- Accent handling:
  - strip_accents_ascii is fast and often sufficient; use strip_accents_unicode if you need broader character normalization.
- Stop words:
  - Use built-in "english" for common English words; supply a custom set for domain-specific stop words.
- Input sources:
  - For large corpora stored on disk, you can feed filenames directly; the vectorizer will decode as needed.
- Memory considerations:
  - HashingVectorizer or similar approaches can help with very large vocabularies or streaming scenarios.

See also
--------

- ENGLISH_STOP_WORDS: built-in stop word list used by _check_stop_list
- FeatureHasher, HashingVectorizer: alternative scalable text feature representations

This section summarizes how to analyze a collection of text documents using the vectorizers and preprocessing utilities provided. It focuses on the steps from raw text to a numeric representation suitable for downstream modeling, emphasizing the configurable and modular nature of the processing pipeline.
