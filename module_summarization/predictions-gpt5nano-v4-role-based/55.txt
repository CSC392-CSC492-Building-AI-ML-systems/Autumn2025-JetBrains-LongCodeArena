miscmodels
===========

Description
-----------

This module provides robust statistical models that replace the usual Gaussian error assumption with a Student-t distribution. The heavier tails of the t distribution yield more robust parameter estimates in the presence of outliers or observations with larger-than-typical deviations. The implementations here are designed as practical examples for likelihood-based estimation within Statsmodels:

- TLinearModel: Maximum Likelihood Estimation of a linear model with t-distributed errors. It extends GenericLikelihoodModel and relies on numerical differentiation for gradients and Hessians. The parameter vector includes regression coefficients, the degrees of freedom (df), and a scale parameter. The degrees of freedom can be fixed or estimated.

- TArma: Univariate ARMA model with t-distributed errors. It inherits from the standard Arma class and overrides the likelihood calculations to use a t-distributed error structure. As with the linear model, the scale parameter is not the same as the standard deviation, and the implied error variance depends on df and scale.

Key ideas
---------

- Robustness: By modeling errors with a Student-t distribution, the models downweight outliers naturally in the likelihood framework.

- Parameterization: For TLinearModel, parameters consist of:
  - beta: regression coefficients
  - df: degrees of freedom (can be fixed or estimated)
  - scale: scale parameter (absolute value to ensure positivity)

  If df is not fixed, the total number of parameters is k_vars + 2; if df is fixed, it is k_vars + 1.

- Inference: Inference is based on the likelihood of the t-distributed errors. Gradients and Hessians are obtained via numerical differentiation (as is typical for GenericLikelihoodModel-based implementations).

- Variance interpretation (note for TArma): The variance implied by the t-errors is scale-dependent and, for df > 2, Var(error) = df/(df-2) * scale^2. The scale parameter is not the standard deviation of the error term.

Implemented classes
-------------------

TLinearModel
------------

Overview
^^^^^^^^
A maximum likelihood estimator for a linear regression model with t-distributed errors. It follows the GenericLikelihoodModel interface, so most methods are generic and rely on numerical differentiation to obtain gradients, Hessians, and statistics.

Key features
------------
- Parameters:
  - beta: regression coefficients (exog.shape[1] parameters)
  - df: degrees of freedom (optional; can be estimated or fixed)
  - scale: scale of the t-distribution (positive)

- df handling:
  - If fix_df is False (default): df is estimated along with beta and scale.
  - If fix_df is set (e.g., a numeric value): df is fixed to that value and only scale and beta are estimated.

- Starting values:
  - If not provided, OLS estimates are used for beta; df defaults to a reasonable starting value (e.g., 5 or 6 based on kurtosis if requested); scale is initialized from OLS residuals.

- Log-likelihood:
  - Uses the standard Student-t log-density. The implementation explicitly computes the log-pdf to avoid numerical issues and applies a correction for the scale parameter.

- Prediction:
  - Predicts y_hat = exog @ beta, consistent with a linear model given the estimated coefficients.

Usage example
-------------
Example usage (path names may vary by installation):

from statsmodels.miscmodels.tlinear_model import TLinearModel

# endog: dependent variable array, exog: design matrix with a column of ones for intercept
model = TLinearModel(endog, exog)

# fit with df estimated (default)
res = model.fit()

# fit with fixed df=5
model.fix_df = 5
res_fixed_df = model.fit()

res.params contains [beta..., df, scale] (or [beta..., scale] if df fixed)

Notes
-----
- The log-likelihood is defined so that res.loglike() corresponds to the usual maximized log-likelihood.
- The scale parameter is not identical to the standard deviation of the errors.
- The implementation includes a TODO list in the code for future improvements and bug fixes.

TArma
-----

Overview
^^^^^^^^
Univariate ARMA model with t-distributed errors. It inherits from the standard Arma class and replaces the Gaussian error assumption with a Student-t distribution for the innovation terms.

Key features
------------
- Parameters:
  - df: degrees of freedom for the t-distribution (can be fixed or estimated)
  - scale: scale of the t-distribution (positive)

- Error structure:
  - The model uses the same ARMA structure as the base Arma class, but the innovation terms follow a t-distribution.

- Log-likelihood:
  - per-observation log-likelihood is computed from the t-distribution. The overall log-likelihood is the sum over observations (via the negative of nloglikeobs, as required by the GenericLikelihoodModel interface).

- Variance interpretation:
  - For df > 2, the implied variance is df/(df-2) * scale^2, not simply scale^2.

Usage example
-------------
Example usage (path names may vary by installation):

from statsmodels.tsa.arma_mle import Arma
from statsmodels.miscmodels.tarma import TArma  # or appropriate import path

# endog: univariate time series, order: (p, q)
model = TArma(endog, order=(p, q))
res = model.fit()

Notes
-----
- The class overrides loglike and nloglikeobs to implement t-distributed errors.
- The scale parameter is constrained to be non-negative in practice (the code uses abs(scale)).

Notes for developers
---------------------
- The module is intended as an illustrative example of using Statsmodels’ generic MLE machinery to model heavy-tailed error distributions.
- The implementation relies on numerical differentiation for gradients and Hessians; performance can be improved with analytic derivatives if needed.
- The code contains a number of TODOs and potential bugs noted in the source (e.g., startup values, parameter fixing behavior). Use with awareness of these limitations.
