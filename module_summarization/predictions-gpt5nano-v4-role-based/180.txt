Cross-validation: evaluating estimator performance
====================================================================

Overview
--------
Cross-validation is a robust approach to estimate how an estimator will perform on unseen data. In the cross-validation workflow documented here, performance is assessed by computing scores across multiple train/test splits and, for learning curves, across a range of training set sizes. The resulting curves enable diagnosing bias and variance issues and comparing different estimators or hyperparameters.

Key concepts
------------
- Score computation across folds: For each cross-validation split, the estimator is trained on the training portion and evaluated on the test portion. Scores from all folds are aggregated to produce mean and standard deviation estimates.
- Learning curve visualization: Shows how the estimator’s performance scales with the size of the training data. Useful for diagnosing underfitting or overfitting and for estimating data requirements.
- Validation curve visualization: (Referenced by this module) Shows how a metric changes as a function of a hyperparameter, enabling hyperparameter tuning from a visual perspective.
- Plotting styles for score uncertainty: 
  - fill_between: shaded region around the mean to show ±1 standard deviation.
  - errorbar: error bars representing standard deviation at each point.
  - None: no uncertainty visualization.

Code architecture highlights (relevant to documentation)
-------------------------------------------------------
- _BaseCurveDisplay: Internal plotting backbone for curve visualizations. Handles:
  - Data interpretation for train/test scores.
  - Support for multiple display styles (fill_between, errorbar, or none).
  - Score naming and axis labeling.
  - Automatic axis scaling with a deprecation path for log_scale.
- LearningCurveDisplay: Public visualization class for learning curves. Uses _BaseCurveDisplay to plot training/test scores across training sizes.
- Scoring and naming:
  - Scores may be negated for plotting convenience (negate_score).
  - score_name controls the label used on the y-axis; if not provided, it is inferred from the scoring configuration.
- Plot options:
  - std_display_style: "fill_between", "errorbar", or None.
  - score_type: "train", "test", or "both" to select which curves to display.
  - log_scale: deprecated; a deprecation warning is issued if used. The x-axis scale is chosen automatically otherwise.

Public API (highlights)
-----------------------
- learning_curve and validation_curve (computed values for curves; used by visualization tools to render plots).
- LearningCurveDisplay (visualization class):
  - Recommended usage: LearningCurveDisplay.from_estimator(...) to create an instance, followed by display.plot().
  - Attributes:
    - ax_: Matplotlib Axes with the curve.
    - figure_: Matplotlib Figure containing the curve.
    - train_scores, test_scores: raw score arrays with shape (n_points, n_folds).
    - lines_/errorbar_/fill_between_: handles to plotted elements depending on std_display_style.

What you can customize
-----------------------
- std_display_style:
  - "fill_between": show mean scores with shaded ±1 std.
  - "errorbar": show mean scores with error bars representing std.
  - None: only the mean curves are plotted.
- score_type:
  - "train": plot training scores.
  - "test": plot test scores.
  - "both": plot both training and test scores.
- score_name:
  - Explicitly set the name for the y-axis; otherwise inferred from scoring configuration.
- negate_score:
  - If True, negate train/test scores before plotting (useful when higher is worse).

Usage examples
--------------
Example 1: Basic learning curve with default settings
- Objective: visualize how performance improves with more training data.

.. code-block:: python

   from sklearn.model_selection import learning_curve
   from sklearn.pipeline import Pipeline
   from sklearn.linear_model import LogisticRegression
   from sklearn.datasets import load_iris

   X, y = load_iris(return_X_y=True)
   estimator = Pipeline([("clf", LogisticRegression(max_iter=1000))])

   train_sizes, train_scores, test_scores = learning_curve(
       estimator, X, y, cv=5, scoring="accuracy", train_sizes=[0.1, 0.3, 0.5, 0.7, 1.0]
   )

   from sklearn.model_selection import LearningCurveDisplay

   disp = LearningCurveDisplay.from_estimator(
       estimator, X, y, train_sizes=train_sizes, cv=5, scoring="accuracy"
   )
   disp.plot()
   disp.set_title("Learning Curve (default)")

Example 2: Validation curve (hyperparameter influence; not shown directly in the class, but part of the cross-validation suite)
- Objective: inspect how a hyperparameter affects score.

.. code-block:: python

   from sklearn.model_selection import validation_curve
   from sklearn.svm import SVC
   import numpy as np

   param_range = np.logspace(-3, 2, 6)
   train_scores, test_scores = validation_curve(
       SVC(), X, y, param_name="gamma", param_range=param_range, cv=5, scoring="accuracy"
   )

   # Visualization can be achieved via a dedicated visualization utility (not shown here).

Interpreting the results
------------------------
- Learning curve:
  - High bias (underfitting): both training and test scores are low and converge. Additional data is unlikely to help much.
  - High variance (overfitting): training score is high, test score is significantly lower; gaps may shrink with more data.
  - Suitable data size: plateaus in the learning curve indicate sufficient data for the model.
- Validation curve:
  - Identify a sweet spot for the hyperparameter where the validation score is maximized (or variance is minimized).
  - Watch for overfitting/underfitting patterns as a function of the hyperparameter.

Notes and caveats
-----------------
- Matplotlib is required for plotting; the module checks for plotting support.
- The log_scale parameter is deprecated (will be removed); prefer using the axis scaling methods on the displayed plot (ax_.set_xscale / set_yscale) as indicated in the code.
- The x-axis scale choice is automatic when log_scale is not used, based on the spread of x_data to select linear, log, or symlog scales.

Related functions/docs
----------------------
- sklearn.model_selection.learning_curve: compute the learning curve data.
- sklearn.model_selection.validation_curve: compute the validation curve data.
- LearningCurveDisplay.from_estimator: recommended entry-point to create a learning curve visualization.

Implementation notes (for maintainers)
-------------------------------------
- _BaseCurveDisplay centralizes plotting logic for learning/validation visualizations, handling:
  - Validation of std_display_style and score_type.
  - Handling of negated scores for "higher is better" semantics.
  - Derivation of score_name when not provided.
  - Automatic axis scaling with a deprecation pathway for log_scale.
- LearningCurveDisplay exposes artifacts (ax_, figure_, lines_, errorbar_, fill_between_) consistent with Matplotlib plotting primitives.

This section provides a concise, developer-oriented overview of how cross-validation performance is computed and visualized, with guidance on interpretation and customization. For practical usage, leverage LearningCurveDisplay.from_estimator and the associated plotting options to diagnose estimator performance across training sizes.
