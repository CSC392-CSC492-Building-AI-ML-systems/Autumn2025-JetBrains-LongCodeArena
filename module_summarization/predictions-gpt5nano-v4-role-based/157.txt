Introduction to Nextcord Widgets (intro.rst)

Overview
--------
This section documents the widget-related data models used by Nextcord to represent a guild's widget payloads. A widget is a lightweight snapshot of a guild's voice channels and online members, intended for quick, read-only display (often used on guild-rich dashboards or dashboards that show live activity). Nextcord provides three core classes to work with these payloads:

- WidgetChannel: a partial representation of a voice channel exposed by the widget.
- WidgetMember: a partial representation of an online member in the widget.
- Widget: the guild widget as a whole, containing accessible channels and online members.

Key design notes
- Lightweight/partial data: WidgetChannel and WidgetMember are designed to be small, partial views of the full channel/member objects. They include only the subset of fields provided by the widget payload.
- Memory efficiency: these classes use __slots__ to reduce per-instance memory overhead.
- Creation time: WidgetChannel exposes created_at, computed from the channel's snowflake ID via snowflake_time.
- Activity handling: WidgetMember may include an activity (e.g., playing a game) and gracefully handles the absence of activity data.

Core classes

1) WidgetChannel
- Purpose: Represents a partial channel included in a guild widget.
- Primary attributes:
  - id (int): The channel's ID.
  - name (str): The channel's name.
  - position (int): The channel's position within the guild.
- Key behavior:
  - str(widget_channel) returns the channel's name.
  - mention property returns a string that can be used to mention the channel: <#channel_id>.
  - created_at property returns the channel's creation time (UTC) derived from the snowflake ID.
- Example usage:
  - channel_name = widget_channel.name
  - creation_time = widget_channel.created_at
  - mention = widget_channel.mention

2) WidgetMember
- Purpose: Represents a partial member in the widget's guild.
- Inheritance: Extends BaseUser (thus includes standard user fields such as id, name, discriminator, bot, avatar, etc.).
- Primary attributes (as exposed by the widget payload):
  - id (int): Member’s ID.
  - name (str): Member’s username.
  - discriminator (str): Member’s discriminator.
  - bot (bool): Whether the member is a bot.
  - status (Status): Member’s online status.
  - nick (Optional[str]): Member’s nickname.
  - avatar (Optional[str]): Member’s avatar hash.
  - activity (Optional[Union[BaseActivity, Spotify]]): Member’s current activity, if any.
  - deafened (Optional[bool]): Whether the member is deafened.
  - muted (Optional[bool]): Whether the member is muted.
  - suppress (Optional[bool]): Whether the member is suppressed.
  - connected_channel (Optional[WidgetChannel]): The channel the member is connected to.
- Initialization:
  - Requires a state (ConnectionState) and a data payload (WidgetMemberPayload).
  - connected_channel can be provided to link to the corresponding WidgetChannel.
  - Activity is created from the payload’s game data when present; if not present, activity is None.
- Key behavior:
  - __repr__ provides a compact representation including name and discriminator.
  - display_name property returns the nickname if present, otherwise the username.
- Example usage:
  - display = member.display_name
  - current_activity = member.activity
  - connected_to = member.connected_channel

3) Widget
- Purpose: Represents a guild widget as a whole.
- Primary attributes:
  - id (int): The guild’s ID.
  - name (str): The guild’s name.
  - channels (List[WidgetChannel]): The accessible voice channels in the guild.
  - members (List[WidgetMember]): The online members in the server (offline members do not appear).
- Behavior notes:
  - The widget object is typically constructed from a widget payload received from Discord (via the state/ConnectionState).
  - String representation (str(widget)) is documented to return the widget’s JSON URL.
  - Equality/inequality and other operations behave as standard Python objects (as described in the class docstring).
- Example usage:
  - first_channel = widget.channels[0]
  - online_members = widget.members
  - json_url = str(widget)

Practical usage tips
- Accessing data:
  - channels and members are lists; iterate safely and handle the possibility of empty lists.
- Looking up creation time:
  - Use WidgetChannel.created_at to order or group channels by their creation time.
- Display names:
  - Use WidgetMember.display_name for user-facing names, preferring nick if available.

Code-level notes
- Type checking and forward declarations:
  - The module uses TYPE_CHECKING to reference ConnectionState and payload types without importing them at runtime.
- Data flow:
  - WidgetChannel and WidgetMember are constructed from widget payloads (WidgetPayload) and may be associated with a specific connected_channel (for WidgetMember).
- Error handling:
  - When the widget payload lacks activity data (e.g., missing 'game' key), the code gracefully sets activity to None.

Example integration snippet (conceptual)
- Given a widget_payload dict obtained from Discord:
  - widget = Widget(state=state, data=widget_payload)
  - print(widget.name)
  - for ch in widget.channels:
      print(ch.name, ch.created_at)
  - for m in widget.members:
      print(m.display_name, m.status, m.activity)

Note
- The documentation strings in the code emphasize equality, string representations, and other container-like operations for WidgetChannel, WidgetMember, and Widget. These behaviors are consistent with the broader Nextcord design where similar classes implement helpful dunder methods for debugging and readability.

End of introductory guide to Nextcord widgets.
