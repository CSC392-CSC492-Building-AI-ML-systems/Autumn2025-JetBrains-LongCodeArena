Transforms: Image to Tensor Utilities and Interpolation Modes
============================================================

Overview
--------
This module provides core utilities to work with images in TorchVision’s transforms framework. It includes:
- An InterpolationMode enum used by resampling transforms.
- Helpers to convert between PIL Images, NumPy arrays, and PyTorch tensors.
- Functions to query image properties such as dimensions, size, and number of channels.

Public API
----------

InterpolationMode
^^^^^^^^^^^^^^^^^
Enumeration of supported interpolation methods. These modes mirror common image resizing options and are designed to be used with transforms that perform image resampling.

- NEAREST
- NEAREST_EXACT
- BILINEAR
- BICUBIC
- BOX
- HAMMING
- LANCZOS

Notes:
- The enum is designed to be PIL-compatible in its mode mappings.
- This enum is used to select an interpolation algorithm in transforms that resize images.

Internal helpers (documented for completeness)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
- _interpolation_modes_from_int(i: int) -> InterpolationMode
  Maps an integer code to an InterpolationMode. Used for interoperability with legacy int codes.
- pil_modes_mapping: Dict[InterpolationMode, int]
  Maps InterpolationMode values to PIL’s internal resampling mode numbers.
- _is_pil_image: bool check whether an object is a PIL.Image.
- _is_numpy(img: Any) -> bool
  Returns True if the input is a NumPy array.
- _is_numpy_image(img: Any) -> bool
  Returns True if the NumPy array represents a 2D or 3D image.

Functions
---------

get_dimensions(img: Tensor) -> List[int]
Returns the image dimensions as [C, H, W].

- Args:
  - img: A PIL Image or a Tensor.
- Returns:
  - List[int]: [channels, height, width]
- Notes:
  - If a Tensor is provided, delegates to F_t.get_dimensions.
  - If a PIL Image is provided, delegates to F_pil.get_dimensions.
  - Logs API usage for non-traced/scripted code paths.

get_image_size(img: Tensor) -> List[int]
Returns the image size as [width, height].

- Args:
  - img: A PIL Image or a Tensor.
- Returns:
  - List[int]: [width, height]
- Behavior mirrors get_dimensions, but returns width and height.

get_image_num_channels(img: Tensor) -> int
Returns the number of channels in the image.

- Args:
  - img: A PIL Image or a Tensor.
- Returns:
  - int: number of channels

to_tensor(pic) -> Tensor
Converts a PIL Image or NumPy array to a PyTorch Tensor.

- Supports:
  - PIL Images
  - NumPy arrays (2D or 3D)
  - accimage Images (if available)
- Behavior:
  - For NumPy arrays:
    - If 2D, adds a channel dimension.
    - Converts to CHW format.
    - If the input is 8-bit (ByteTensor), scales to [0, 1] using the default floating dtype.
  - For PIL Images:
    - Handles common PIL modes and converts to a PyTorch tensor in CHW layout.
    - Converts to the default floating dtype and scales to [0, 1] if the underlying tensor is ByteTensor.
  - Returns:
    - Tensor with shape [C, H, W]

pil_to_tensor(pic: Any) -> Tensor
Converts a PIL Image to a tensor of the same underlying dtype (and performs a deep copy).

- Note:
  - The conversion is a deep copy of the underlying NumPy array.

Example usage
-------------
Basic conversions and property queries between PIL Images, NumPy arrays, and tensors.

Code examples (Python):

.. code-block:: python

    from torchvision.transforms import functional as F
    from PIL import Image
    import numpy as np
    import torch

    # Create a simple PIL image (RGB)
    pil_img = Image.new("RGB", (64, 32), color=(255, 0, 0))

    # Convert PIL image to tensor (C x H x W), values in [0, 1]
    t = F.to_tensor(pil_img)
    print(t.shape)          # torch.Size([3, 32, 64])
    print(t.dtype)            # torch.dtype('float32')

    # Convert a NumPy array (H x W x C) to tensor
    np_img = np.zeros((32, 64, 3), dtype=np.uint8)
    t_np = F.to_tensor(np_img)
    print(t_np.shape)       # torch.Size([3, 32, 64])

    # Query image properties
    dims = F.get_dimensions(t)        # [3, 32, 64]
    size = F.get_image_size(t)          # [64, 32]
    channels = F.get_image_num_channels(t)  # 3
    print("dims:", dims, "size:", size, "channels:", channels)

    # PIL to tensor with same dtype (explicit)
    t_pil = F.pil_to_tensor(pil_img)
    print(t_pil.shape)        # [3, 32, 64]
    print(t_pil.dtype)          # matches input PIL->likely uint8 or float

Notes and considerations
------------------------
- Dependency requirements:
  - NumPy, PIL (Pillow), and PyTorch are required.
  - accimage is optional; if available, it is used for acceleration of certain image conversions.
- Logging:
  - API usage is logged via _log_api_usage_once for non-scripted/traced paths to help track usage of these utilities.
- Image modes:
  - Common PIL modes (e.g., "L", "RGB", "RGBA") are supported. Special handling exists for mode "1" (1-bit images), which is converted to 255-scale.
- Internal helpers:
  - Functions prefixed with _ (e.g., _is_pil_image) are intended for internal use and may be omitted from public API users’ day-to-day code.

See also
--------
- _functional_pil as F_pil: PIL-based image utilities (helpers for PIL images).
- _functional_tensor as F_t: Tensor-based image utilities (helpers for PyTorch tensors).
