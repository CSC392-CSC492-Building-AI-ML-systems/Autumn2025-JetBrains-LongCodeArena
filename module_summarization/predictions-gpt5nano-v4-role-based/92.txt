distutils.rst
=============

Module overview
---------------
This module provides a collection of utilities used by the distutils/numpy.distutils build system. It includes facilities for:
- per-thread temporary directory management and cleanup
- path manipulation helpers that normalize and join path components
- tooling to determine the number of parallel build jobs
- frame inspection helpers to locate file paths from stack frames
- a lightweight container for installable libraries

Public API
----------

Classes
- InstallableLib
  - Purpose: Container to hold information on an installable library.
  - Parameters
    - name (str): Name of the installed library.
    - build_info (dict): Dictionary holding build information.
    - target_dir (str): Absolute path specifying where to install the library.
  - Notes:
    - Intended to be used in coordination with Configuration.add_installed_library.
  - Attributes (stored with the same names): name, build_info, target_dir

Functions
- get_num_build_jobs()
  - Purpose: Determine how many parallel build jobs can be run.
  - Behavior:
    - Checks the --parallel setting via numpy.distutils.core.get_distribution and related command objects.
    - Falls back to the environment variable NPY_NUM_BUILD_JOBS or a system CPU count (capped at 8 to avoid overload).
    - Considers explicit parallel settings on build commands (build, build_ext, build_clib) if provided; otherwise uses env/cpu-based value.
  - Returns: int
  - Notes: The function imports get_distribution from numpy.distutils.core and gracefully handles cases where distribution information is unavailable.

- quote_args(args)
  - Purpose: Quote a list of arguments.
  - Deprecation: Deprecated (as of version 1.22).
  - Behavior:
    - Emits a DeprecationWarning.
    - Returns a new list where certain arguments are quoted (implementation detail reflects a specific condition in the code).
  - Returns: list
  - Notes: Do not rely on this function for new code; consider alternatives for argument quoting.

- allpath(name)
  - Purpose: Convert a /-separated pathname to the OS-specific path separator.
  - Behavior:
    - Splits the input on '/' and joins using OS path separators.
  - Returns: str

- rel_path(path, parent_path)
  - Purpose: Return path relative to a given parent_path.
  - Behavior:
    - Resolves real paths to avoid symlink-related issues.
    - If the absolute path is shorter than the parent path, returns the original path.
    - If path equals parent_path, returns ''.
    - If parent_path is a prefix of path, returns the portion after the prefix.
  - Returns: str

- get_path_from_frame(frame, parent_path=None)
  - Purpose: Return the directory of the module associated with a given frame.
  - Behavior:
    - Attempts to extract __file__ from frame.f_globals/f_locals; if unavailable, uses __name__ to import and locate the module.
    - If module has no __file__, falls back to the current directory.
    - If parent_path is provided, returns the path relative to parent_path using rel_path.
  - Returns: str
  - Notes: Useful for determining module paths during dynamic execution, such as setup/build scripts.

- njoin(*path)
  - Purpose: Join path components with enhanced handling.
  - Behavior:
    - Accepts multiple arguments, which may be individual strings or sequences.
    - If an argument is a sequence, it is expanded (e.g., njoin(['a', 'b'], 'c') -> 'a/b/c').
    - Converts /-separated components to OS-specific separators and resolves .. and . components.
  - Returns: str
  - Notes: Flexible path-joining utility that supports both spread and sequence inputs.

Internal state and helpers
- _tdata (thread-local storage)
  - Purpose: Store temporary directory information per thread to ensure only one temporary directory per thread is created.

- _tmpdirs (list)
  - Purpose: Keep track of all created temporary directories so they can be cleaned up on exit.

- clean_up_temporary_directory()
  - Purpose: Cleanup hook registered with atexit to remove all temporary directories tracked in _tmpdirs.
  - Behavior:
    - Iterates over _tmpdirs and attempts to remove each directory with shutil.rmtree.
    - Swallows OSError exceptions to avoid cleanup failures affecting process shutdown.

- atexit integration
  - The module registers clean_up_temporary_directory with atexit to ensure cleanup at process exit.

Notes on API surface
- __all__ includes a broad set of names, indicating the module aims to expose a wide public API. The portions documented here reflect the concrete implementations present in the provided code snippet (InstallableLib, get_num_build_jobs, quote_args, allpath, rel_path, get_path_from_frame, njoin, and the internal cleanup machinery).

Usage examples
- Determine parallel build jobs
  .. code-block:: python

     from distutils import get_num_build_jobs
     print("Parallel build jobs:", get_num_build_jobs())

- Normalize a path with OS separators
  .. code-block:: python

     from distutils import allpath
     print(allpath("src/include/../lib"))

- Compute a path relative to a parent directory
  .. code-block:: python

     from distutils import rel_path
     print(rel_path("/usr/local/lib/python3.9/site-packages", "/usr/local"))

- Get a module path from a stack frame
  .. code-block:: python

     import inspect
     import distutils
     frame = inspect.currentframe()
     print(distutils.get_path_from_frame(frame))

- Join path components with sequence support
  .. code-block:: python

     from distutils import njoin
     print(njoin("a", ["b", "c"], "d"))

Deprecation notice
- quote_args is deprecated. Use explicit quoting logic or the standard library equivalents instead.

See also
- The InstallableLib class is designed to be used in conjunction with a Configuration object (referenced in the class docstring) to manage libraries that are installed by a build process.
- The per-thread temporary directory management utilities help ensure isolation in multi-threaded builds and safe cleanup at program exit.
