Transforming the prediction target
===================================

Overview
--------
The code in this preprocessing module focuses on transforming input features (X) using encoders such as OneHotEncoder and OrdinalEncoder. It does not implement target (y) transformation. To transform the prediction target, wrap your estimator with a target-transforming regressor (e.g., TransformedTargetRegressor) that applies a forward transformation to y before fitting and an inverse transformation to predictions after inference.

How target transformation works
------------------------------
- Before fitting: the target y is transformed with a user-supplied function func.
- After predicting: the model outputs are transformed back to the original scale using inverse_func.
- This separation allows feature encoding to remain independent of target scaling, which is handled at the estimator level.

API (conceptual)
------------------
- regressor: the underlying estimator to wrap (e.g., LinearRegression, RandomForestRegressor).
- func: callable applied to y before fitting (e.g., np.log1p for positive targets).
- inverse_func: callable applied to predictions to return them to the original scale (e.g., np.expm1).
- check_inverse: whether to verify that inverse_func correctly inverts func (if supported by the implementation).

Practical example
-------------------
Transform a positive target with a log1p transformation, fit, and obtain predictions on the original scale:

.. code-block:: python

    from sklearn.compose import TransformedTargetRegressor
    from sklearn.linear_model import LinearRegression
    import numpy as np

    # regressor that will operate on the transformed target
    reg = TransformedTargetRegressor(
        regressor=LinearRegression(),
        func=np.log1p,        # transform y -> log(1 + y)
        inverse_func=np.expm1,  # invert back: exp(y') - 1
        check_inverse=True       # optional: verify invertibility
    )

    reg.fit(X, y)           # y is transformed before fitting
    y_pred = reg.predict(X)  # predictions are inverse-transformed to original scale

Notes and considerations
------------------------
- Applicable to regression tasks. Classification targets typically do not use target transformations.
- Choose func carefully to match the scale and distribution of y (e.g., log for positive targets, or a custom monotonic transform).
- If y contains non-positive values, avoid transforms that require strictly positive input (e.g., log). Consider alternatives like a monotonic yet invertible transform or wrapping with a custom function.
- For multi-output regression, TransformedTargetRegressor can handle multiple target dimensions.

Relation to feature transformers
-------------------------------
- The encoder classes documented in this module (OneHotEncoder, OrdinalEncoder) transform X and are agnostic to y.
- Target transformation is a separate concern, typically implemented by wrapping the final estimator with a target-transforming regressor in a pipeline, ensuring clean separation between feature preprocessing and target scaling.
