Overview
- This module provides the building blocks for pairwise computations used across distances, similarities, affinities, and kernels.
- Core responsibilities include input validation, dtype handling, and support for both dense and sparse inputs as well as precomputed distance/similarity matrices.
- It also references fast kernel implementations (e.g., chi-square kernel) and reductions used in pairwise computations.

Pairwise metrics
Definition
- Pairwise metrics quantify relationships between samples in two sets:
  - Distances (e.g., Euclidean, Manhattan) measure dissimilarity.
  - Similarities or affinities measure likeness (kernels such as linear, RBF, polynomial, chi-squared, etc.).

Key utilities
- _return_float_dtype(X, Y)
  - Purpose: Decide a safe numeric dtype for pairwise computations.
  - Behavior:
    - If both inputs are dense and float32, returns float32.
    - Otherwise returns a real (floating) dtype (float).
  - Handling:
    - Works with sparse inputs by converting to numpy arrays only as needed for dtype decisions.

- check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)
  - Purpose: Validate and normalize inputs for pairwise computations.
  - Behavior:
    - If Y is None, Y is set as a view on X (no copy) to enable efficient self-pairings.
    - Ensures X and Y (or the single array) are array-like and of an appropriate float dtype.
    - If precomputed is True, X is treated as a distance/similarity matrix with shape conventions that depend on Y.
      - Precomputed: X.shape[1] must equal Y.shape[0].
    - If precomputed is False, X.shape[1] must equal Y.shape[1].
    - Supports sparse inputs via accept_sparse (e.g., 'csr'). If a sparse input is not in an allowed format, it will be converted to the first allowed format.
    - Force finite values handling via force_all_finite and copy semantics via copy.
  - Returns:
    - safe_X, safe_Y: validated and (if needed) coerced numpy arrays ready for computation.
  - Practical implications:
    - Central entry point for enforcing consistent shapes and dtypes before heavy computations.
    - Helps catch mismatched dimensions or inappropriate input types early with informative errors.

- check_paired_arrays(X, Y)
  - Purpose: Validate and prepare inputs for paired-distance metrics.
  - Similar to check_pairwise_arrays but tailored for paired-distance workflows (e.g., metrics that take two input sets and compute pairwise relations).
  - Ensures both X and Y are at least 2D and of appropriate numeric dtype before computation.

Affinities
Concepts
- Affinities refer to similarity measures derived from data, often via kernel functions.
- In practice, affinities are represented as matrices where entry (i, j) encodes the similarity between samples i and j (or i and j in Y, depending on alignment).
- Kernels are a family of functions that produce these affinities, frequently used in spectral methods, clustering, and kernelized learning.

Architecture notes
- The codebase expects pairwise computations to drive affinity-related operations by returning a consistent matrix shape and dtype.
- Affinities can be computed from pairwise distances (e.g., via a kernel transformation) or directly via kernel functions. The utilities here ensure that inputs are valid for such transformations.

Kernels
Concepts
- Kernels are functions that map two samples to a similarity score, often enabling non-linear relationships to be captured.
- Common kernels include linear, polynomial, radial basis function (RBF), and chi-squared kernels.
- There are fast-path implementations referenced in the module, enabling efficient computation for specific kernels on sparse data.

Fast path and reductions
- _chi2_kernel_fast
  - Fast kernel implementation for the chi-squared kernel, optimized for performance.
- _sparse_manhattan
  - Fast path for computing pairwise Manhattan distances on sparse data.

Input validation and stability
- The utilities emphasize robust handling of:
  - Dense vs. sparse inputs
  - Precomputed vs. on-the-fly distance/similarity matrices
  - Finite value enforcement (with nuanced handling for NaN and infinite values)
  - Consistent dtypes to avoid unexpected casting or precision loss

Practical examples
- Self-pairwise distance (X vs. X)
  - Use check_pairwise_arrays(X, None) to ensure X is prepared for self-comparison (Y points to X).
- Distances with a separate set (X vs. Y)
  - Use check_pairwise_arrays(X, Y) to validate both matrices.
- Precomputed kernel matrix
  - If you already have a kernel/affinity matrix K between X and Y, call check_pairwise_arrays(K, None, precomputed=True) or align with the expected shapes (K should be shaped appropriately as a distance/affinity matrix).

Notes for contributors
- The core validation logic (dtype selection, shape checks, and sparse handling) is centralized in _return_float_dtype and check_pairwise_arrays to ensure consistency across all pairwise metrics.
- When extending with new kernels or affinity measures, reuse the same validation utilities to maintain API stability and predictable error messages.
- Pay attention to precomputed matrix expectations:
  - Distances/affinities represented as 2D matrices with compatible inner dimensions.
  - Clear separation between "precomputed" usage and normal pairwise computation to avoid shape mismatches.
