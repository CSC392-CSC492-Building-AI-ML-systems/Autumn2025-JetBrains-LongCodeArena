Fitting SESANS Data
===================

This document describes the SESANS fitting parameter discovery utility used to analyze and document the parameters leveraged by SESANS models in the SASmodels framework.

Overview
--------
The sesans_fitting module provides a lightweight command-line tool to enumerate all parameters used across all available SESANS models. It interacts with the core model metadata (load_model_info, list_models) to collect parameters defined as kernel parameters for each model and surfaces them to the user in two formats:
- Names only (one parameter per line)
- Parameter → models mapping (showing which models use each parameter)

Key concepts:
- kernel_parameters: The list of parameters that a model exposes for fitting. Each parameter has attributes such as name and type.
- kind filter: Optional filtering by parameter kind (volume, orientation, sld). If no filtering is applied, all parameters are returned.

What is documented here is the behavior of the SESANS parameter listing tool, which is used when preparing, auditing, or documenting SESANS fitting workflows.

API and Behavior
----------------
Core functionality is provided by three components within the module:

- find_pars(kind=None)
  - Purpose: Scan all available models and collect a mapping from parameter name to the list of models that use that parameter.
  - Signature: find_pars(kind=None)
  - Behavior:
    - Iterates over all models from list_models()
    - Loads model_info for each modelname
    - Iterates over model_info.parameters.kernel_parameters
    - If kind is None or parameter.type == kind, includes the parameter
    - Returns a dictionary: { parameter_name: [model_name, model_name, ...] }

- list_pars(names_only=True, kind=None)
  - Purpose: Print parameters either as a names-only list or as a parameter-to-models mapping.
  - Signature: list_pars(names_only=True, kind=None)
  - Behavior:
    - Calls find_pars(kind)
    - If names_only is True: prints the sorted, unique parameter names in columns (via columnize)
    - If names_only is False: prints lines of the form "param_name: model1, model2, ..."

- main()
  - Purpose: Command-line interface for listing SESANS parameters.
  - Behavior:
    - Accepts a flag:
      - -v / --verbose: if set, list models which use each parameter (i.e., names_only becomes False)
    - Accepts an optional positional argument kind with choices:
      - volume, orientation, sld, None, any
      - The underlying conversion maps 'any' and 'none' to None, so these values effectively disable filtering in current implementation.
      - When a concrete kind is provided (e.g., 'volume'), only parameters of that kind are listed
    - Invokes list_pars with appropriate arguments

CLI Usage
---------
- Basic (names only):
  python -m sasmodels.list_pars

- Names with model mappings:
  python -m sasmodels.list_pars -v

- Filter by parameter kind (if supported by models):
  python -m sasmodels.list_pars volume

Notes on kind handling:
- The code uses a filter: kind is None or p.type == kind to include parameters.
- The CLI accepts: volume, orientation, sld, none, any
- In the current implementation, 'any' and 'none' map to None, effectively disabling the filter. If you need strict filtering, provide a specific kind (e.g., volume) and rely on model metadata supporting p.type values.

Practical Examples
-------------------
- List all parameter names used across all SESANS models:
  python -m sasmodels.list_pars

- List all parameter names, with no model context:
  python -m sasmodels.list_pars

- List parameters and the models that use them:
  python -m sasmodels.list_pars -v

- List parameters of a specific kind (e.g., volume):
  python -m sasmodels.list_pars volume

Output formats
- Names only (default):
  param1
  param2
  param3
  ...

- Names-to-models mapping (verbose):
  param1: modelA, modelB
  param2: modelC
  param3: modelA, modelD

Data structures and how they relate to SESANS models
---------------------------------------------------
- Model metadata source:
  - list_models(): returns a collection of model names available in the environment.
  - load_model_info(name): loads a metadata object for a given model name.
- Kernel parameters:
  - model_info.parameters.kernel_parameters is the list of Parameter objects used for fitting in that model.
  - Each Parameter object exposes:
    - name: parameter’s name used in fitting
    - type: parameter’s kind (e.g., 'volume', 'orientation', 'sld', etc.)
- Aggregation:
  - find_pars() aggregates across all models, producing a comprehensive view of which models share which parameters.

Architectural notes
-------------------
- This utility is designed to aid in documentation and audit trails for SESANS fitting workflows by revealing the parameter landscape across models.
- It relies on the centralized model registry (list_models, load_model_info) and a simple formatting helper (columnize) to present results consistently.
- It is safe to run in environments with many SESANS models; the operation is read-only and does not modify model state.
