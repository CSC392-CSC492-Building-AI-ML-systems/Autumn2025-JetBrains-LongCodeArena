Module: Imputation (MICE)

Description
------------
This module implements the Multiple Imputation through Chained Equations (MICE)
approach for handling missing data in statistical analyses. It provides the
core machinery to construct and manage multiply-imputed data sets, fit
imputation models for each variable with missing values, and (optionally)
fit an analysis model to the imputed data and combine results across
imputations.

Key ideas
- Imputation workflow
  - Initialize missing values (e.g., with mean imputation).
  - For each variable with missing data (the focus variable), fit an imputation
    model regressing the focus variable on observed and current-imputed values
    of other variables.
  - Impute missing values for the focus variable using predictive mean matching
    (PMM) when imputing.
  - After all variables are imputed, fit the analysis model to the completed data.
  - Repeat the process and combine results using a combining rule to obtain
    point estimates and standard errors for the analysis model parameters.
- Modularity
  - Imputation models are defined per variable via a model class and a formula.
  - Default model is ordinary least squares (OLS) with a formula specifying main
    effects for all other variables.
  - Supports two usage modes: (i) producing multiply imputed data sets, or
    (ii) using the imputations to fit an analysis model and obtain combined results.
- Perturbation methods (for imputations)
  - Gaussian perturbation: fit the imputation model and draw parameters from a
    Gaussian approximation to the posterior (default).
  - Bootstrap perturbation: fit on bootstrapped data to obtain parameter vectors.

Class structure
--------------
The module defines three public components:

- MICEData
  - Wraps a Pandas DataFrame and stores imputation model specifications for
    variables with missing values.
  - Provides facilities for producing multiply imputed datasets and plotting
    imputation diagnostics.
  - History hook: history_callback (or history_func in some usage) allows
    arbitrary information from intermediate imputations to be saved.
  - Key usage patterns (from examples):
    - Set an imputer for a focus variable with a specified formula:
      - imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
    - Iterate imputations and optionally save each completed dataset:
      - imp.update_all()
      - imp.data.to_csv('data%02d.csv' % j)
    - Iterate over imputations: for data in imp: ...

- MICE
  - Orchestrates the full multiple-imputation workflow given a MICEData object
    and an analysis model specification.
  - Runs the multiple imputation, fits the analysis models, and combines the
    results to produce a MICEResults object.
  - The MICEResults object provides a summary() method to view target estimands
    and inferential quantities.

- PatsyFormula
  - Lightweight wrapper for a string interpreted as a Patsy formula.
  - Ensures a zero-intercept baseline by prepending "0 + " to the provided
    formula.

- Note on a helper: PatsyFormula is used to construct regression formulas for
  imputation models (the default approach uses OLS with main effects for other
  variables).

Usage patterns (illustrative)
-----------------------------
Code blocks taken from the module's examples illustrate common workflows.

1) Produce imputations and write to files using per-imputation models
.. code-block:: python

    imp = mice.MICEData(data)
    imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
    for j in range(20):
        imp.update_all()
        imp.data.to_csv('data%02d.csv' % j)

2) Iterate over imputations directly
.. code-block:: python

    imp = mice.MICEData(data)
    j = 0
    for data in imp:
        imp.data.to_csv('data%02d.csv' % j)
        j += 1

Notes
-----
- Default memory behavior: MICEData saves very little information from one
  iteration to the next to conserve memory. The input data is copied on entry
  and subsequently overwritten as new imputations are produced.
- If using MICE (i.e., with an analysis model), fitted analysis models and
  results are saved for later combination and reporting.
- History callbacks (history_callback) enable saving arbitrary information from
  intermediate imputations for downstream analysis.

Terminology
-----------
- Analysis model: The model for which inferences are desired. If not specified,
  the workflow produces imputed data for later analysis rather than immediate
  parameter inference.
- Imputation model: A regression model for each variable that has missing values,
  conditioned on observed and current-imputed values of other variables. A separate
  model (one per variable with missing data) defines the imputation for that
  variable.
- Perturbation method: How parameter uncertainty in an imputation model is
  introduced. Gaussian perturbation samples from a Gaussian approximation to the
  fit distribution; bootstrap perturbation uses a bootstrap-resampled fit.

Notes on defaults and configuration
- Default perturbation method: 'gaussian' (unless overridden).
- k_pmm: Number of nearest neighbors used in predictive mean matching during
  imputation (default 20, configurable per MICEData instance or per imputation).
