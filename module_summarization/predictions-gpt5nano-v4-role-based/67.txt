Feature extraction from text (module sklearn.feature_extraction.text)
=====================================================================

This submodule provides utilities to convert text documents into numeric feature vectors that can be consumed by machine learning models. It contains vectorizers (CountVectorizer, TfidfVectorizer, HashingVectorizer), a transformer (TfidfTransformer), and a collection of helper utilities for preprocessing, tokenization, stop words handling, and n-gram generation.

Public API
----------
- HashingVectorizer
- CountVectorizer
- ENGLISH_STOP_WORDS
- TfidfTransformer
- TfidfVectorizer
- strip_accents_ascii
- strip_accents_unicode
- strip_tags

Overview
--------
Text data must be converted into a numeric representation to be used by most ML models. This module provides a flexible set of tools to:
- tokenize and preprocess text
- optionally remove stop words
- generate n-grams
- transform counts into TF or TF-IDF representations
- optionally hash features to a fixed-dimensional space (HashingVectorizer)

Internal helpers and their roles
--------------------------------
- _preprocess(doc, accent_function=None, lower=False)
  - Chains optional preprocessing steps on a single document.
  - Parameters:
    - doc: input string
    - accent_function: callable to handle accented characters
    - lower: whether to convert text to lowercase
  - Returns: preprocessed string
  - Typical usage: apply lowercase and/or accent normalization before tokenization.

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  - Chains together processing steps to convert a single document into tokens/ngrams.
  - If decoder is provided, it is applied first. If analyzer is provided, it replaces preprocessor/tokenizer/ngrams. Otherwise, the sequence preprocessor -> tokenizer -> ngrams is applied.
  - Returns: a sequence of tokens (possibly n-grams)

- strip_accents_unicode(s)
  - Remove accented characters by Unicode normalization (NFKD) and stripping combining marks.
  - Fast path for ASCII-compatible strings; otherwise normalizes to non-accented form.
  - Returns: string with accents removed (unicode aware)

- strip_accents_ascii(s)
  - Remove accents by transliterating to ASCII via NFKD and ASCII encoding.
  - Suitable for languages with direct ASCII transliterations.
  - Returns: ASCII-only string

- strip_tags(s)
  - Basic HTML/XML tag stripper using a regular expression.
  - Returns: string with tags replaced by spaces

- _check_stop_list(stop)
  - Resolve stop word specification to a concrete collection.
  - If stop == "english", returns ENGLISH_STOP_WORDS.
  - If stop is a string other than a built-in, raises ValueError.
  - If stop is None, returns None.
  - If stop is a collection, returns a frozenset(stop).

- _VectorizerMixin
  - Shared functionality for text vectorizers.
  - Attributes and methods:
    - _white_spaces: compiled regex to detect runs of whitespace
    - decode(doc): normalize input to a string. Supports:
      - input="filename": reads from a file path
      - input="file": reads from a file-like object
      - handles bytes and string inputs, decoding with configured encoding and decode_error
      - raises NotFittedError for np.nan inputs
    - This mixin encapsulates the input handling logic so vectorizers can operate on a uniform string representation.

- _word_ngrams(tokens, stop_words=None)
  - Convert a sequence of tokens into word n-grams after optional stop-word filtering.
  - Supports removal of stop words prior to n-gram construction.
  - Intended to be used by vectorizers that support n-gram features.

Public concepts and usage
-------------------------
- Tokenization and preprocessing
  - Vectorizers can be configured with custom preprocessors, tokenizers, and n-gram builders.
  - The special analyzer argument, if provided, bypasses preprocessor/tokenizer/ngrams and handles the chain itself.

- Stop words handling
  - Stop words can be specified as a built-in set (e.g., "english"), a custom collection, or None to disable removal.

- Accent stripping and HTML stripping
  - The module provides multiple strategies to normalize text (strip accents, remove HTML/XML tags) to improve downstream feature extraction.

- Input handling flexibility
  - Text content can come from raw strings, file-like objects, or filenames. The internal _VectorizerMixin.decode method handles these cases.

Examples
--------
Basic CountVectorizer with English stop words and lowercase normalization:
.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = [
       "This is a sample document.",
       "Another example document!"
   ]

   vectorizer = CountVectorizer(stop_words="english", lowercase=True)
   X = vectorizer.fit_transform(docs)

TF-IDF with Unicode accent stripping:
.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   docs = ["Café au lait", "naïve approach to text"]
   vectorizer = TfidfVectorizer(strip_accents="unicode")
   X = vectorizer.fit_transform(docs)

HTML stripping before tokenization:
.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = ["<p>This is <b>bold</b> text.</p>"]
   vectorizer = CountVectorizer(strip_tags=True)
   X = vectorizer.fit_transform(docs)

Notes
-----
- The public API primarily exposes the vectorizers and the simple text preprocessing utilities.
- Internal helpers (_preprocess, _analyze, _VectorizerMixin) underpin the flexible and extensible text processing pipeline used by vectorizers.
- ENGLISH_STOP_WORDS provides a ready-made set of common English stop words for quick use.
