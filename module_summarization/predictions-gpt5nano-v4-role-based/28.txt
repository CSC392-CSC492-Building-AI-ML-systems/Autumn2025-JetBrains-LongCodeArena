w_multi_west
============

Overview
--------
w_multi_west is a WESTPA tool designed to merge results from multiple WESTPA simulations while accounting for reweighting between runs. The command is implemented as a WESTTool-derived class (WMultiWest) and is intended to operate within the WESTPA tooling framework. It prepares a combined output HDF5 file that aggregates data from individual simulations, with optional reweighting and auxiliary datasets.

Command-line interface
----------------------
The w_multi_west command provides the following options:

- -o, --output-file
  - Default: multi.h5
  - Description: Name of the output HDF5 file to store results in.

- -W, --west, --WEST_H5FILE
  - Default: west.h5
  - Description: The name of the main HDF5 file inside each simulation directory.

- -a, --aux
  - Action: append
  - Description: Names of additional auxiliary datasets to be combined.

- -aa, --auxall
  - Action: store_true
  - Description: If set, combine all auxiliary datasets. Default: False.

- -nr, --no-reweight
  - Action: store_true
  - Description: Do not apply reweighting. Default: False.

- -ib, --ibstates
  - Action: store_true
  - Description: Attempt to combine ibstates dataset. Default: False.

Notes:
- The tool relies on the WESTTool/WESTMultiTool infrastructure. It initializes with a progress indicator component and maintains internal state related to simulations, iterations, and walkers.
- The options allow you to control where results are written, which datasets are included, and whether reweighting is applied.

- Segment (inner class)
  - Lightweight container for a simulation segment with attributes:
    - weight, iteration, simid, recycled_in

- Helper functions (module scope)
  - get_bin_mapper(we_h5file, hashval)
    - Purpose: Retrieve a bin Mapper object corresponding to a given hash from the HDF5 bin_topologies group.
    - Behavior: Attempts to locate the index/pickles in /bin_topologies, unpickle the mapper, and return it. Raises KeyError if not found.

    - Returns: A new array with the istate_dtype structure.

- Data types (imported/defined)

  - istate_dtype, n_iter_dtype: Dtypes provided by WESTPA for istates and iteration indices, used to maintain compatibility with WESTPA's expected data layout.
  - h5io, WESTPAH5File: Utilities and file wrappers used to read/write WESTPA HDF5 data.

Data flow and architecture notes
-------------------------------
- Input handling:
  - The tool accepts a list of input simulations (via self.west) and opens their HDF5 files to prepare for merging.
  - open_files creates a destination output file and stamps creator data for provenance.

- Data compatibility:
  - create_idtype_array ensures that merged data conforms to istate_dtype, preserving existing fields while normalizing the destination array.
  - get_bin_mapper provides a mechanism to retrieve binning information if present in the input HDF5 (used for reweighting or binning-based operations).

- Reweighting and auxiliaries:
  - The options allow enabling/disabling reweighting and selecting which auxiliary datasets to include. The actual reweighting logic is intended to reside in the go() method, though the provided snippet ends before the full implementation.

- Progress and user feedback:
  - The ProgressIndicatorComponent is integrated to provide user feedback during long-running operations.

Usage example
-------------
- Basic usage (defaults):
  - Combine simulations with default settings (no-reweight disabled by default flag, aux datasets optional).
  - Command (illustrative; format depends on the WEST tool launcher):
    west w_multi_west -o multi.h5 -W west.h5

- Include specific auxiliary datasets and enable all auxiliaries:
  - Command:
    west w_multi_west -o multi.h5 -W west.h5 -a aux1 -a aux2 -aa

- Disable reweighting:
  - Command:
    west w_multi_west -o multi.h5 -W west.h5 -nr
