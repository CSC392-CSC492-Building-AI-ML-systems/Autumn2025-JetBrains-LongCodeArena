Kernel approximation
====================

The :mod:`sklearn.kernel_approximation` module implements several approximate kernel
feature maps based on Fourier transforms and Count Sketches. These maps enable
computationally efficient approximations of kernel feature spaces, trading exactness
for faster training on large datasets.

PolynomialCountSketch
---------------------

Class overview
^^^^^^^^^^^^^^

PolynomialCountSketch implements Tensor Sketch to approximate the feature map of
the polynomial kernel

    K(X, Y) = (gamma * <X, Y> + coef0)^degree

by efficiently computing a Count Sketch of the outer product of a vector with
itself using Fast Fourier Transforms (FFT). This provides a compact, random,
and scalable approximation of polynomial kernel features.

This class was introduced in version 0.24.

Public API
**********

PolynomialCountSketch
    A transformer that maps input data into a randomized, finite-dimensional
    feature space intended to approximate a polynomial kernel feature map.

Parameters
**********

gamma : float, default=1.0
    Parameter of the polynomial kernel whose feature map will be approximated.

degree : int, default=2
    Degree of the polynomial kernel whose feature map will be approximated.

coef0 : float, default=0
    Constant term of the polynomial kernel whose feature map will be approximated.

n_components : int, default=100
    Dimensionality of the output feature space. Typically, n_components should
    be larger than the number of input features to achieve good performance.
    A common heuristic is around 10 * n_features, though this depends on the data.

random_state : int, RandomState instance, default=None
    Determines random number generation for indexHash and bitHash initialization.
    Pass an int for reproducible results across calls.

Attributes
**********

indexHash_ : ndarray of shape (degree, n_features) or (degree, n_features + 1)
    Array of indices in [0, n_components) used to represent the 2-wise independent
    hash functions for Count Sketch computation. If coef0 != 0, an extra feature is
    included and the second dimension increases accordingly.

bitHash_ : ndarray of shape (degree, n_features) with values in {+1, -1}
    Random entries representing 2-wise independent hash functions used in the
    Count Sketch computation.

n_features_in_ : int
    Number of features seen during fit.

feature_names_in_ : ndarray of shape (n_features_in_,), optional
    Names of features seen during fit. Defined only when X has feature names
    that are all strings. Available only when the version supports feature names.
    Added in version 1.0.

    .. versionadded:: 0.24
       indexHash_ and bitHash_ attributes.

    .. versionadded:: 1.0
       feature_names_in_ attribute.

Notes
*****

- The implementation relies on SciPy’s FFT, with a fallback to fftpack for older
  SciPy versions (< 1.4).
- This transformer follows scikit-learn’s transformer API (fit/transform) and can be
  used in pipelines and with estimators that consume numeric feature matrices.

Examples
********

>>> from sklearn.kernel_approximation import PolynomialCountSketch
>>> from sklearn.linear_model import SGDClassifier
>>> X = [[0, 0], [1, 1], [1, 0], [0, 1]]
>>> y = [0, 0, 1, 1]
>>> ps = PolynomialCountSketch(degree=3, random_state=1)
>>> X_features = ps.fit_transform(X)
>>> clf = SGDClassifier(max_iter=10, tol=1e-3)
>>> clf.fit(X_features, y)
SGDClassifier(max_iter=10)
>>> clf.score(X_features, y)
1.0

Usage notes
***********

- Choose n_components to balance accuracy and performance. A typical guideline is
  to set n_components on the order of 10 * n_features, but optimal values depend
  on the dataset and task.
- For non-zero coef0, an extra feature is included in the sketch to account for the
  constant term in the polynomial kernel.

Implementation notes
*******************

- The class stores indexHash_ and bitHash_ as internal random hash mappings used
  by the Count Sketch procedure. These are created in fit using a provided random
 _state to ensure reproducibility.
- The class inherits from scikit-learn’s estimator interfaces to integrate with
  the broader ecosystem (pipelines, cross-validation, etc.).
