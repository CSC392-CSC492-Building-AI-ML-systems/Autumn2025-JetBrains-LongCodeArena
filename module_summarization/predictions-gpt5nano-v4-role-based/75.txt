Working with Text Data
======================

This documentation provides a practical tutorial on turning raw text documents into numeric feature vectors using scikit-learn's text feature extraction utilities. It focuses on common workflows, from simple counts to TF–IDF weighting, with guidance on preprocessing, tokenization, stop words, and n-grams.

Overview
--------
The sklearn.feature_extraction.text submodule contains tools to convert text documents into numerical feature vectors suitable for machine learning. The primary building blocks are:

- CountVectorizer: converts a collection of text documents to a matrix of token counts.
- TfidfTransformer / TfidfVectorizer: reweights counts by term frequency–inverse document frequency (TF–IDF).
- HashingVectorizer: provides a memory-efficient, hash-based feature representation without an explicit vocabulary.
- Utility helpers for preprocessing: accents stripping, HTML tag stripping, and simple tokenization/analyzing pipelines.
- Stop words support: built-in English stop words set (ENGLISH_STOP_WORDS) and a convenient stop list interface.

Key Concepts
------------
- Tokenization: breaking text into tokens (words or terms).
- Normalization: lowercasing, accent removal, and other text normalization steps.
- Stop words: common words to exclude from feature extraction (e.g., “the”, “and”).
- N-grams: sequences of tokens (e.g., unigrams, bigrams) to capture phrases.
- TF–IDF: weighting scheme that downscales frequently occurring terms while highlighting informative ones.
- Preprocessing pipelines: you can customize the flow via preprocessor, tokenizer, analyzer, and n-gram functions, or supply a complete analyzer to replace the pipeline.

Preprocessing and Tokenization Helpers
--------------------------------------
- _preprocess(doc, accent_function=None, lower=False)
  - Optional steps: lowercasing and an accent-stripping function.
  - Use to normalize text before tokenization.

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  - Chains preprocessing steps into a single pipeline.
  - If an explicit analyzer is provided, it replaces preprocessor, tokenizer, and ngrams steps.

- strip_accents_unicode(s) vs strip_accents_ascii(s)
  - strip_accents_unicode: converts accented characters to their unaccented counterparts where possible, preserving characters when no ASCII equivalent exists.
  - strip_accents_ascii: transliterates to ASCII (best for languages with direct ASCII equivalents).
  - These can be used as accent_function in preprocessing.

- strip_tags(s)
  - Lightweight HTML/XML tag stripper (useful for cleaning web-sourced text).

- ENGLISH_STOP_WORDS
  - Built-in set of English stop words.

Tutorial: Getting Started with Text Vectorization
-------------------------------------------------

1) Quick start: Count-based representation (unigrams)
- Goal: Convert raw text documents into a matrix of token counts.

Code example:
- from sklearn.feature_extraction.text import CountVectorizer
- docs = ["This is a sample.", "This is another example sample."]
- vec = CountVectorizer()
- X = vec.fit_transform(docs)
- print(X.shape)  # (n_samples, n_features)
- print(vec.get_feature_names_out())  # vocabulary

What you get:
- A sparse matrix where each row represents a document and each column corresponds to a token (word) from the vocabulary.
- Entry (i, j) is the count of token j in document i.

2) Weighting with TF–IDF
- Goal: Downscale common terms and highlight informative terms.

Code example:
- from sklearn.feature_extraction.text import TfidfVectorizer
- docs = ["This is a sample.", "This is another example sample."]
- tfidf = TfidfVectorizer(stop_words="english", ngram_range=(1, 2))
- X = tfidf.fit_transform(docs)
- print(X.shape)
- print(tfidf.get_feature_names_out())

What you get:
- A sparse matrix with TF–IDF weights instead of raw counts.
- stop_words="english" (or ENGLISH_STOP_WORDS) excludes common English words.
- ngram_range=(1, 2) includes unigrams and bigrams.

3) Customizing preprocessing: accents, HTML, and tokenization
- Accent stripping
  - Use as part of preprocessing to normalize characters before tokenization.
  - Example:
    - accent_fn = strip_accents_unicode
    - vec = CountVectorizer(preprocessor=lambda s: _preprocess(s, accent_function=accent_fn, lower=True))

- HTML tag stripping
  - Use strip_tags to clean HTML content:
    - vec = CountVectorizer(preprocessor=strip_tags)

- Full preprocessing pipeline
  - You can combine lowercasing, accent stripping, and stripping tags:
    - vec = CountVectorizer(preprocessor=lambda s: _preprocess(strip_tags(strip_accents_unicode(s)), lower=True))

4) Stop words management
- Built-in English stop words
  - ENGLISH_STOP_WORDS provides a ready-made set.
- Using the built-in English stop list
  - vec = CountVectorizer(stop_words="english")
  - The helper _check_stop_list("english") returns ENGLISH_STOP_WORDS.
- Supplying a custom stop list
  - vec = CountVectorizer(stop_words={"a", "an", "the"})

5) Tokenization and n-grams customization
- Custom tokenizer
  - You can supply a function tokenizer(doc) -> list[str]
- Custom analyzer
  - analyzer accepts a callable that receives a document and returns an iterable of tokens or a complete sequence of n-grams.
  - If you provide an analyzer, you typically bypass preprocessor/tokenizer/ngrams in favor of a single, unified function.

6) HashingVectorizer (memory-friendly, no vocabulary)
- Goal: Produce a fixed-size feature space without building a vocabulary.
- Example:
  - from sklearn.feature_extraction.text import HashingVectorizer
  - hv = HashingVectorizer(n_features=2**20)  # 1M features
  - X = hv.transform(docs)
- Notes:
  - No explicit vocabulary is built.
  - Colocations and collisions are possible due to hashing.
  - Useful for very large datasets or streaming data.

7) End-to-end pipelines with scikit-learn
- Typical workflow combines vectorization with a downstream estimator via Pipeline.
- Example:
  - from sklearn.pipeline import Pipeline
  - from sklearn.linear_model import LogisticRegression
  - docs = ["text data here", "more documents"]
  - pipeline = Pipeline([
      ("vect", TfidfVectorizer(stop_words="english", ngram_range=(1, 2))),
      ("clf", LogisticRegression(max_iter=1000))
    ])
  - pipeline.fit(docs, y)  # y contains labels
  - preds = pipeline.predict(["new text to classify"])

- You can insert any combination of:
  - CountVectorizer, TfidfVectorizer, or HashingVectorizer
  - Preprocessing steps (strip_accents_unicode/ascii, strip_tags)
  - Custom analyzer/tokenizer
  - Stop words configuration
  - N-gram ranges

Advanced Topics
---------------
- The analyzer parameter
  - If you supply an analyzer, it replaces preprocessor, tokenizer, and ngrams steps. This provides maximum flexibility for specialized text processing pipelines.

- Handling input sources
  - The internal _VectorizerMixin.decode supports input types such as filename or file, enabling flexible data ingestion when used in custom pipelines.

- Choosing between vocabulary-based and hashing approaches
  - Vocabulary-based (CountVectorizer / TfidfVectorizer) yields interpretable features (tokens) and is suitable for smaller to medium datasets.
  - HashingVectorizer is memory-efficient and suitable for very large datasets or streaming; interpretability of features may be limited due to hashing.

API at a Glance
---------------
- CountVectorizer
  - fit_transform(raw_documents)
  - get_feature_names_out() or get_feature_names()
  - vocabulary_ attribute
  - stop_words handling via stop_words parameter

- TfidfTransformer
  - fit, transform
  - Converts raw term counts to TF–IDF features

- TfidfVectorizer
  - Combines CountVectorizer and TfidfTransformer in one step

- HashingVectorizer
  - transform(raw_documents)
  - n_features parameter controls the feature space size

- Utilities
  - strip_accents_unicode(s)
  - strip_accents_ascii(s)
  - strip_tags(s)
  - ENGLISH_STOP_WORDS
  - _preprocess(doc, accent_function=None, lower=False)
  - _analyze(...)

Practical Tips
--------------
- Start simple: use CountVectorizer with default settings to establish a baseline.
- Add TF–IDF for better weighting of informative terms.
- Experiment with stop words to balance noise reduction against potential loss of important terms.
- Use ngram_range to capture phrases (e.g., (1, 2) for unigrams and bigrams) when word order and context matter.
- For large datasets or streaming data, consider HashingVectorizer to avoid building a large vocabulary.
- Combine with a machine learning model using scikit-learn’s Pipeline for a clean, repeatable workflow.

Appendix: Example Code Snippet (End-to-End)
--------------------------------------------
- from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer
- from sklearn.pipeline import Pipeline
- docs = ["This is the first document.", "This document is the second document."]
- pipeline = Pipeline([
    ("vect", TfidfVectorizer(stop_words="english", ngram_range=(1, 2))),
    ("clf", None)  # replace with a classifier as needed
  ])
- X = pipeline.named_steps['vect'].fit_transform(docs)
- print(X.shape)
