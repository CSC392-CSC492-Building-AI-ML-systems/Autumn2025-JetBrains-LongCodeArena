basics.indexing
================

Overview
--------
Collection of utilities to manipulate structured arrays. Most of these functions were
initially implemented by John Hunter for matplotlib. They have been rewritten and
extended for convenience.

Public API
----------
The module exposes a set of helpers to work with structured (record) arrays, including
operations to inspect, flatten, and fill fields. The following names are intended for
public use:

- append_fields
- apply_along_fields
- assign_fields_by_name
- drop_fields
- find_duplicates
- flatten_descr
- get_fieldstructure
- get_names
- get_names_flat
- join_by
- merge_arrays
- rec_append_fields
- rec_drop_fields
- rec_join
- recursive_fill_fields
- rename_fields
- repack_fields
- require_fields
- stack_arrays
- structured_to_unstructured
- unstructured_to_structured

Note: The present documentation focuses on the implemented portions visible in the code
sample (and marks internal utilities accordingly).

Public functions
---------------

recursive_fill_fields(input, output)
-----------------------------------
Fill fields from output with fields from input, with support for nested structures.

Parameters
- input: ndarray
  Input array.
- output: ndarray
  Output array (must be at least as large as input).

Notes
- The output array should be at least the same size as the input.

Examples
- Basic usage (from the codebase example):

  >>> from numpy.lib import recfunctions as rfn
  >>> a = np.array([(1, 10.), (2, 20.)], dtype=[('A', np.int64), ('B', np.float64)])
  >>> b = np.zeros((3,), dtype=a.dtype)
  >>> rfn.recursive_fill_fields(a, b)
  array([(1, 10.), (2, 20.), (0,  0.)], dtype=[('A', '<i8'), ('B', '<f8')])

Notes
- Recursively fills nested fields when present. Non-matching fields are skipped.

_get_fieldspec(dtype)
---------------------
Produce a list of name/dtype pairs corresponding to the dtype fields.

Parameters
- dtype: numpy.dtype
  The structured dtype to analyze.

Returns
- List of pairs describing fields. For nested dtypes, the second element of the pair is a dtype
  object representing the substructure.

Examples
- Example from the code:

  >>> dt = np.dtype([(('a', 'A'), np.int64), ('b', np.double, 3)])
  >>> dt.descr
  [(('a', 'A'), '<i8'), ('b', '<f8', (3,))]
  >>> _get_fieldspec(dt)
  [(('a', 'A'), dtype('int64')), ('b', dtype(('<f8', (3,))))]

Internal note
- Intended for internal use within the module; not part of the stable public API.

get_names(adtype)
---------------
Returns the field names of the input datatype as a tuple. The input datatype must have fields,
otherwise a KeyError/ValueError would typically be raised by NumPy if accessed improperly.

Parameters
- adtype: numpy.dtype
  Input datatype.

Returns
- tuple of field names. Nested structures are represented as nested tuples.

Examples
- Basic usage:

  >>> from numpy.lib import recfunctions as rfn
  >>> rfn.get_names(np.empty((1,), dtype=[('A', int)]).dtype)
  ('A',)
  >>> rfn.get_names(np.empty((1,), dtype=[('A',int), ('B', float)]).dtype)
  ('A', 'B')
  >>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
  >>> rfn.get_names(adtype)
  ('a', ('b', ('ba', 'bb')))

get_names_flat(adtype)
----------------------
Returns the field names of the input datatype as a flattened tuple. Nested structures are flattened.

Parameters
- adtype: numpy.dtype
  Input datatype.

Returns
- tuple of flattened field names.

Examples
- Basic usage:

  >>> from numpy.lib import recfunctions as rfn
  >>> rfn.get_names_flat(np.empty((1,), dtype=[('A', int)]).dtype) is None
  False
  >>> rfn.get_names_flat(np.empty((1,), dtype=[('A',int), ('B', str)]).dtype)
  ('A', 'B')
  >>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
  >>> rfn.get_names_flat(adtype)
  ('a', 'b', 'ba', 'bb')

flatten_descr(ndtype)
---------------------
Flatten a structured data-type description.

Parameters
- ndtype: numpy.dtype
  The dtype to flatten.

Returns
- A tuple of (field_name, dtype) pairs describing the flattened fields.

Examples
- Basic usage:

  >>> from numpy.lib import recfunctions as rfn
  >>> ndtype = np.dtype([('a', '<i4'), ('b', [('ba', '<f8'), ('bb', '<i4')])])
  >>> rfn.flatten_descr(ndtype)
  (('a', dtype('int32')), ('ba', dtype('float64')), ('bb', dtype('int32')))

Internal utilities
------------------
_zip_dtype(seqarrays, flatten=False)
-----------------------------------
Internal helper to build a new dtype from a sequence of arrays, with optional flattening
of nested structures. This is an internal utility and not part of the public API.

Notes
- Used by internal routines to assemble dtypes when combining or transforming
  structured arrays.

Internal helper
---------------
_get_fieldspec and _zip_dtype are intended for internal use to support the public API.
They are not guaranteed to be stable across NumPy releases.

Examples
- Not typically called directly by users; see public functions above for common use cases.

Usage tips
-----------
- These utilities work with both structured arrays (dtype with named fields) and
  nested structured arrays (fields that themselves have dtypes with their own fields).
- When working with masked arrays, these utilities preserve the masking semantics provided
  by NumPy's MaskedArray and MaskedRecords as appropriate.

See Also
--------
- numpy.lib.recfunctions (the broader collection these utilities belong to)
- numpy.ma (masked array utilities used by this module)

Notes
-----
- The module’s public API is primarily exposed via __all__ and is designed to support
  common structured array manipulations used in data processing pipelines.
- The examples in the docstrings illustrate typical usage patterns and expected outputs.
