diophantine.rst

Diophantine equation solver
===========================

Overview
--------
This module implements a high-level interface for solving Diophantine equations
-- equations that seek integer solutions. The primary entry-point is the
function diophantine(eq, param, syms), which rewrites the input equation as a
product of terms equal to zero and solves each factor independently. The
solutions are combined and returned as a set of tuples, where each tuple contains
the solution for every variable in a canonical order.

Key ideas:
- An equation is converted to a product of factors whose zeroes yield the
  overall solution set.
- Each factor is classified by its Diophantine type and solved using diop_solve.
- Solutions are merged consistently to produce a set of tuples, ordered by the
  variables' alphabetic order (or by an explicit syms sequence if provided).
- Special handling for trivial and degenerate cases (e.g., 0 == 0, or equations
  with denominators).

API
---
diophantine(eq, param=<symbol t, integer=True>, syms=None)

- Purpose
  Solve a polynomial Diophantine equation by factoring into zero-product terms
  and solving each term. Returns a set of solutions, where each solution is a
  tuple of expressions for the unknowns.

- Parameters
  - eq: Relational (e.g., Eq) or expression to solve. The equation is interpreted as lhs - rhs = 0.
  - param: A symbolic parameter used in the solution expressions. By default this is a symbol t with integer domain.
  - syms: Optional sequence of Symbols to determine the order of variables in the returned
          tuples. If provided, only variables present in the equation are kept and their
          order is set by syms; otherwise variables are ordered alphabetically.

- Returns
  A set of tuples. Each tuple contains the solution for each variable, ordered
  according to the determined variable order.

- Behavior notes
  - If eq is an equality (Eq), it is transformed to lhs - rhs.
  - If the equation reduces to 0, a trivial solution with the parameter is returned.
  - Equations with a denominator are handled by solving the numerator and denominating
    with respect to the denominator (solutions must satisfy both).
  - If no nontrivial solutions exist, a trivial solution is returned if it satisfies the equation.
  - Output terms are sympified for consistency with the rest of the symbolic framework.

Internal helpers
----------------
The module defines several helper utilities that support diophantine solving:

- diophantine_known types
  - diop_known: a set of known Diophantine problem families handled by the solver
    (e.g., 'binary_quadratic', 'cubic_thue', 'general_pythagorean', etc.).

- _is_int(i)
  - Determines if i is an integer (via as_int). Used in internal checks for
    numeric operands.

- _sorted_tuple(*i)
  - Returns a sorted tuple from the given items (for canonical ordering).

- _remove_gcd(*x)
  - Normalizes a tuple of integers by removing their greatest common divisor.

- _rational_pq(a, b)
  - Returns a reduced rational pair (numerator, denominator) for a/b, with
    gcd removed and sign placed in the numerator.

- _nint_or_floor(p, q)
  - Returns the nearest integer to p/q; in case of a tie, returns floor(p/q).

- _odd(i) / _even(i)
  - Helper predicates to test odd/even parity.

Implementation notes
--------------------
- diophantine(eq, param, syms) flow
  1. If eq is an Eq, convert to eq.lhs - eq.rhs.
  2. If eq == 0, return a trivial solution {(param,)}.
  3. Expand and collect free symbols; optionally re-order according to syms.
  4. If there is a denominator, solve the denominator and numerator separately:
     - solve d and n with diophantine(d) and diophantine(n), then filter to those
       that satisfy the original equation (via substitution).
  5. Factor the equation and obtain independent components (var, eq) suitable for polynomial handling.
  6. Convert to a polynomial in its variables; ensure no numeric coefficients remain in the generators.
  7. Attempt to classify the main Diophantine type via classify_diop(eq) to
     guide factoring (with a fall-back to factor_list(eq)).
  8. For each term, extract its base and variable target representation, classify
     the term type (eq_type) and solve with diop_solve(base, param).
  9. Merge solutions according to eq_type:
     - linear, homogeneous_ternary_quadratic, homogeneous_ternary_quadratic_normal, general_pythagorean
       use merge_solution with the variable mapping.
     - binary_quadratic, general_sum_of_squares, general_sum_of_even_powers, univariate
       merge each individual solution.
  10. Clean up: remove empty tuple (), ensure a trivial solution if none found
      but the equation allows the zero solution, and sympify final results.

- Output handling
  - The final result is a set of sympy expressions (tuples) suitable for further use
    in symbolic computation contexts.

Examples
--------
The following examples illustrate typical usage and expected outputs:

- Solve a simple difference of squares
  >>> diophantine(x**2 - y**2)
  {(t_0, -t_0), (t_0, t_0)}

- Solve a product with multiple variables and a linear relation
  >>> diophantine(x*(2*x + 3*y - z))
  {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}

- Solve a polynomial with a mixture of terms
  >>> diophantine(x**2 + 3*x*y + 4*x)
  {(0, n1), (3*t_0 - 4, -t_0)}

Notes
-----
- diophantine() returns a set of tuples, with each tuple representing a complete
  assignment for the equation’s variables.
- The parameter t (default) is an integer-valued symbol used to express infinite
  families of solutions.
- The ordering of variables in the returned tuples follows either the natural
  alphabetical order of the free symbols or the explicit syms argument.

Code-level hints (for maintainers)
----------------------------------
- The function heavily relies on internal helpers for classification and merging
  solutions (classify_diop, diop_solve, merge_solution, _mexpand).
- Careful handling of denominators and polynomial conversion is essential to avoid
  spurious solutions; the code enforces a polynomial representation with rational
  coefficients before solving.
- The syms parameter allows precise control over the tuple ordering, which is useful
  when multiple variables are involved and a particular ordering is required for
  downstream processing.
