# Biclustering

Biclustering (also known as co-clustering) aims to simultaneously cluster rows and columns of a data matrix to discover submatrices (biclusters) where a subset of rows exhibits coherent patterns across a subset of columns. This is useful for uncovering localized structures in data such as gene expression, user-item interactions, or other tabular data where relationships are not uniform across the entire feature set.

Key ideas
- Unlike traditional clustering, biclustering searches for blocks of the matrix where a subset of rows behaves similarly over a subset of columns.
- Results are expressed as a collection of biclusters, each described by which rows and which columns participate in that bicluster.
- Most scikit-learn biclustering implementations provide binary membership masks for rows and columns, rather than a single cluster label per sample/feature.

Estimators (overview)
- SpectralBiclustering
  - Purpose: Find coherent row-column blocks by performing a spectral (eigenvalue) decomposition on a bipartite representation of the data.
  - Input: Dense real-valued 2D array (n_samples, n_features) with numerical values (the data matrix).
  - Output: For each bicluster, binary masks over rows and columns describing membership.
  - Typical usage: Identify multiple biclusters that capture localized patterns in the data.
- SpectralCoclustering
  - Purpose: Similar to SpectralBiclustering but often framed as coclustering a bipartite graph; also discovers row and column blocks with coherent structure.
  - Input/Output: Analogous to SpectralBiclustering (masks for rows and columns per bicluster).

API highlights
- fit(X, y=None)
  - Learn bicluster structure from the data matrix X.
  - X should be a 2D numeric array (dense) of shape (n_samples, n_features).
- Attributes after fit
  - rows_: ndarray
    - Shape depends on the number of biclusters produced (one mask per bicluster).
    - Each entry along the first axis is a binary mask over rows (length n_samples) indicating membership of that bicluster’s rows.
  - columns_: ndarray
    - Similar to rows_, but for columns (length n_features).
  - The number of biclusters is inferred from the length of rows_ (and columns_) along the first axis.

Interpreting results
- Each bicluster i is defined by:
  - Row indices where rows_[i, :] is True (or nonzero).
  - Column indices where columns_[i, :] is True (or nonzero).
- To inspect bicluster i:
  - rows_i = np.where(model.rows_[i])[0]
  - cols_i = np.where(model.columns_[i])[0]
- You can examine the blocks to understand localized patterns, or use the masks to extract submatrices X[np.ix_(rows_i, cols_i)] for further analysis.

Practical example
- Basic usage (SpectralBiclustering):
  - This example demonstrates fitting and retrieving row/column memberships for each bicluster.
```
import numpy as np
from sklearn.cluster import SpectralBiclustering

# Example data: replace with your real matrix
X = np.random.RandomState(0).randn(100, 80)

# n_clusters can be an int or a tuple (n_row_clusters, n_col_clusters)
model = SpectralBiclustering(n_clusters=(3, 4), random_state=0)
model.fit(X)

# Inspect bicluster memberships
for i in range(len(model.rows_)):
    rows = np.where(model.rows_[i])[0]
    cols = np.where(model.columns_[i])[0]
    print(f"Bicluster {i}: {len(rows)} rows x {len(cols)} cols")
```

Notes and best practices
- Input data should be a dense matrix of numerical values. Sparse inputs are not typically supported directly by these estimators.
- The number of biclusters (and their sizes) is controlled by the n_clusters parameter (or equivalent in your chosen estimator). Choose values that reflect the expected structure in your data.
- Biclustering is most beneficial when you expect localized, block-like patterns rather than global, uniform patterns across all rows and columns.
- Preprocessing (centering/normalization) can affect results. Typical steps include standardizing rows or columns depending on the data and the chosen method.
- Memory considerations: since biclustering operates on the full data matrix and produces masks for each bicluster, very large matrices can be memory-intensive.

Relation to the provided clustering code pattern
- The code snippet in this repository demonstrates a standard estimator pattern used by clustering algorithms:
  - A functional wrapper (dbscan) delegates to a class-based estimator (DBSCAN) and returns core indices and labels.
  - The DBSCAN class inherits from common scikit-learn base classes (BaseEstimator, ClusterMixin) and exposes a clear, consistent API.
- Biclustering estimators follow the same design philosophy but adapt the output to row/column block structures rather than a single label per sample.
