Markov Chain Monte Carlo (MCMC) Framework
=========================================

Overview
--------

This module implements a flexible Markov chain Monte Carlo (MCMC) framework for equilibrium sampling from a biomolecular thermodynamic state. The framework is designed to compose different move types (Langevin dynamics, Hybrid Monte Carlo, Generalized Hybrid Monte Carlo, Monte Carlo barostat moves) into a single workflow. Moves can be combined in sequence or selected stochastically, and may be configured to reuse OpenMM Context objects via a ContextCache to minimize overhead.

Key concepts
------------

- Thermodynamic state
  - Represented by a ThermodynamicState, which describes the underlying molecular system, temperature, pressure, and related thermodynamic quantities.

- Sampler state
  - Represented by a SamplerState, which encapsulates the microscopic state of the system (e.g., positions, velocities).

- MCMCMove
  - Abstract base class for all MCMC moves. Concrete moves must implement the apply(thermodynamic_state, sampler_state) method.
  - Moves may modify the thermodynamic state, the sampler state, or both.
  - Subclasses typically accumulate statistics (e.g., number of attempts, acceptance rates).

- Move composition primitives
  - SequenceMove
    - Executes a list of moves in a fixed order on every iteration.
  - WeightedMove
    - Selects among candidate moves according to specified (unnormalized) weights (probabilistic choice) each iteration.
  - Individual move types (examples shown in documentation):
    - LangevinDynamicsMove
      - Langevin dynamics step(s); typically used for equilibrium sampling with continuous dynamics.
    - HMCMove (Hybrid Monte Carlo)
      - Proposes a molecular dynamics trajectory and accepts/rejects the proposal using a Metropolis criterion.
    - GHMCMove (Generalized Hybrid Monte Carlo)
      - A generalized variant of HMC with extended stochastic components.
    - Monte Carlo barostat moves
      - Volume/pressure adjustment moves used for constant-pressure sampling.

- Context caching and platform configuration
  - The framework defaults to using a global ContextCache, which constructs OpenMM Context objects on the fastest available platform.
  - Users can configure the default OpenMM platform, or supply per-move caches (context_cache) or a dummy cache to disable caching altogether.
  - Supported via openmmtools.cache: global_context_cache, per-move ContextCache instances, and DummyContextCache.
  - Examples include switching to the Reference platform and adjusting time-to-live for OpenMM Context objects.

Public API
----------

MCMCMove
- Abstract base class for all MCMC moves.
- Interface
  - apply(thermodynamic_state, sampler_state)
    - Applies the move to the given thermodynamic and sampler state.
    - May modify either or both states.
- Subclasses should record move statistics (e.g., accepted/rejected counts) as part of their implementation.

MCMCSampler
- The primary driver for running MCMC simulations.
- Purpose
  - Orchestrates a sequence or stochastic selection of MCMC moves to generate equilibrium samples.
- Constructor
  - MCMCSampler(thermodynamic_state, sampler_state, move_set)
    - thermodynamic_state: initial ThermodynamicState
    - sampler_state: initial SamplerState
    - move_set: container of MCMCMove objects
      - If a list or tuple, all moves are attempted each iteration in the specified order.
      - If a dict, unnormalized weights are provided for stochastic selection of moves (e.g., {move1: 0.3, move2: 0.5, move3: 0.9}).
- Attributes
  - thermodynamic_state
    - Current thermodynamic state.
  - sampler_state
    - Current sampler state.
  - move_set
    - Container of MCMCMove objects (ordered sequence or weighted selection).
- Methods (typical usage)
  - minimize()
    - Prepare the system by performing a preparatory minimization step if supported by the configured moves.
  - run(n_iterations)
    - Run the MCMC workflow for the specified number of iterations, applying moves as configured by move_set.
  - Additional methods (not shown in the excerpt) may include state accessors, resetters, and utilities for diagnostics.

Usage patterns
--------------

- Simple sequential moves
  - Create a sequence of moves and run for a number of iterations.
  - Example moves: GHMCMove, LangevinDynamicsMove.

- Mixed moves with fixed sequencing
  - Use SequenceMove to apply a fixed order of moves every iteration.

- Stochastic move selection
  - Use WeightedMove to randomly select a move each iteration according to weights.

- Local and global caching
  - Global OpenMM platform cache (default) via cache.global_context_cache.
  - Per-move context caches to customize resources and platforms.
  - DummyContextCache for no caching behavior.

Examples
--------

The following example snippets illustrate typical usage and mirror the capabilities described in the module documentation.

Create and run a simple GHMC/Langevin workflow
.. code-block:: python

   from simtk import unit
   from openmmtools import testsystems, cache
   from openmmtools.states import ThermodynamicState, SamplerState

   # Create initial state for alanine dipeptide in vacuum
   test = testsystems.AlanineDipeptideVacuum()
   thermodynamic_state = ThermodynamicState(system=test.system,
                                            temperature=298*unit.kelvin)
   sampler_state = SamplerState(positions=test.positions)

   # Define moves
   ghmc_move = GHMCMove(timestep=1.0*unit.femtosecond, n_steps=50)
   langevin_move = LangevinDynamicsMove(n_steps=10)

   # Simple sequence
   sampler = MCMCSampler(thermodynamic_state, sampler_state, move=SequenceMove([ghmc_move, langevin_move]))
   sampler.minimize()
   sampler.run(n_iterations=2)

Configure caching and per-move caches
.. code-block:: python

   # Optional: configure global context cache
   reference_platform = openmm.Platform.getPlatformByName('Reference')
   cache.global_context_cache.platform = reference_platform
   cache.global_context_cache.time_to_live = 10  # number of read/write operations

   # Local caches for specific moves
   local_cache1 = cache.ContextCache(capacity=5, time_to_live=50)
   local_cache2 = cache.ContextCache(platform=reference_platform, capacity=1)

   sequence_move = SequenceMove([HMCMove(), LangevinDynamicsMove()],
                                context_cache=local_cache1)
   ghmc_move = GHMCMove(context_cache=local_cache2)

   sampler = MCMCSampler(thermodynamic_state, sampler_state, move=sequence_move)

   # Alternate with a specific move using a dummy cache (no caching)
   dummy_cache = cache.DummyContextCache(platform=reference_platform)
   ghmc_move = GHMCMove(context_cache=dummy_cache)

Notes
-----

- The framework emphasizes using the fastest available OpenMM platform by default but provides straightforward mechanisms to override platform choices and context caching on a per-move basis or globally.
- Langevin dynamics are provided as a convenient, continuous-dynamics-based move, but users should be aware of potential integration error depending on settings.
- The MCMC framework is designed to enable composition of different sampling strategies for robust equilibrium sampling.
