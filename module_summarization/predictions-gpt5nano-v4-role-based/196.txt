Multiclass and Multioutput Algorithms
=====================================

Overview
--------

This module provides meta-estimators that extend a single base estimator to handle multiclass/multioutput problems. The public API focuses on multioutput regression/classification, as well as classifier/regressor chaining strategies that model dependencies between multiple targets.

Key ideas
- Meta-estimators wrap a base estimator and train one instance of that estimator per target (for regression) or per label (for classification).
- ClassifierChain and RegressorChain build a dependency chain among targets by feeding prior targets (predictions) as additional features to subsequent estimators.
- These estimators support standard features such as cloning of base estimators, optional parallelization via n_jobs, and incremental learning via partial_fit when supported by the base estimator.

Public API
----------

The following classes constitute the primary public API of this module:

- MultiOutputRegressor
  - Wraps a regression estimator to support multioutput regression.
  - Trains one regressor per target with the same input features X.

- MultiOutputClassifier
  - Wraps a classification estimator to support multioutput (multi-label) classification.
  - Trains one classifier per target with the same input features X.
  - Validates that the underlying estimator is suitable for classification (via classification-target checks).

- ClassifierChain
  - Builds a chain of classifiers where predictions for earlier targets are appended as features for later classifiers.
  - Captures dependencies among targets to improve predictive performance on structured multi-label problems.

- RegressorChain
  - Analogous to ClassifierChain but for regression targets.
  - Each subsequent regressor can use the previous targets’ predictions as additional features.

Common parameters
- estimator: The base estimator to be wrapped. Must implement fit and predict.
- n_jobs: Optional integer to control parallelism across the per-target learners.

Internal helpers and design
---------------------------

The module uses internal helpers to support multioutput training and dynamic API behavior:

- _fit_estimator(estimator, X, y, sample_weight=None, **fit_params)
  - Clones the base estimator and fits it on X and a single target y.
  - If sample_weight is provided, it is passed to fit.

- _partial_fit_estimator(estimator, X, y, classes=None, partial_fit_params=None, first_time=True)
  - Supports incremental learning via partial_fit when available on the base estimator.
  - First time usage clones the estimator; subsequent calls reuse the cloned instance.
  - If classes are provided, they are passed to partial_fit for the initial call.

- _available_if_estimator_has(attr)
  - Helper to conditionally expose parameters in documentation based on whether the sub-estimator(s) expose a given attribute.
  - Returns a decorator that enables a parameter only if the sub-estimator(s) have the specified attribute.
  - Useful for Chain implementations to reflect capabilities of the wrapped estimators.

- _MultiOutputEstimator (abstract base)
  - Internal base class for multioutput meta-estimators.
  - Enforces parameter constraints:
    - estimator must have fit and predict methods (HasMethods(["fit", "predict"]))
    - n_jobs must be an integer or None
  - __init__(self, estimator, *, n_jobs=None): stores the base estimator and the optional parallelism setting.
  - Subclasses provide concrete behavior for training and prediction.

Implementation notes
--------------------

- These are meta-estimators; they do not replace the underlying estimator but extend it to handle multiple targets.
- Training typically involves cloning the base estimator for each target and fitting them independently (or sequentially in a chain).
- When partial_fit is available, the _partial_fit_estimator helper enables incremental training across targets.
- The design emphasizes:
  - Independence of per-target models (via clone) to avoid unintended parameter sharing.
  - Optional parallelism to improve training speed on multi-target problems.
  - Validation of target structure for classification tasks to ensure appropriate use with MultiOutputClassifier.

Usage examples
--------------

Example 1: MultiOutputRegressor with a regressor backbone
- Purpose: Predict multiple continuous targets from the same input features.

  from sklearn.datasets import make_regression
  from sklearn.ensemble import RandomForestRegressor
  from sklearn.multioutput import MultiOutputRegressor

  # Create a dataset with multiple targets
  X, Y = make_regression(n_samples=200, n_features=4, n_targets=3, random_state=0)

  # Base estimator
  base = RandomForestRegressor(n_estimators=100, random_state=0, n_jobs=-1)

  # Wrap for multioutput regression
  mor = MultiOutputRegressor(base)
  mor.fit(X, Y)
  Y_pred = mor.predict(X)

Example 2: MultiOutputClassifier with multilabel data
- Purpose: Train one classifier per label; suitable for multilabel classification.

  from sklearn.datasets import make_multilabel_classification
  from sklearn.linear_model import LogisticRegression
  from sklearn.multioutput import MultiOutputClassifier

  X, Y = make_multilabel_classification(n_samples=200, n_features=20, n_classes=3, random_state=0)

  clf = LogisticRegression(max_iter=1000)
  morc = MultiOutputClassifier(clf)
  morc.fit(X, Y)
  Y_pred = morc.predict(X)

Example 3: ClassifierChain
- Purpose: Model dependencies among labels by forming a chain of classifiers.

  from sklearn.datasets import make_multilabel_classification
  from sklearn.linear_model import LogisticRegression
  from sklearn.multioutput import ClassifierChain

  X, Y = make_multilabel_classification(n_samples=200, n_features=20, n_classes=3, random_state=0)

  base_clf = LogisticRegression(max_iter=1000)
  chain = ClassifierChain(base_clf)
  chain.fit(X, Y)
  Y_pred = chain.predict(X)

Example 4: RegressorChain
- Purpose: Model dependencies among regression targets via a chain of regressors.

  from sklearn.datasets import make_regression
  from sklearn.ensemble import RandomForestRegressor
  from sklearn.multioutput import RegressorChain

  X, Y = make_regression(n_samples=200, n_features=4, n_targets=3, random_state=0)

  base_reg = RandomForestRegressor(n_estimators=100, random_state=0, n_jobs=-1)
  rchain = RegressorChain(base_reg)
  rchain.fit(X, Y)
  Y_pred = rchain.predict(X)

Notes and recommendations
-------------------------

- Shapes:
  - For multioutput regressors, y should have shape (n_samples, n_targets).
  - For multioutput classifiers, y is typically a binary/multi-class indicator matrix with shape (n_samples, n_classes) suitable for multilabel classification.

- Parallelism:
  - Use n_jobs to control parallelism when training multiple per-target estimators.
  - Be mindful of memory usage when using large ensembles.

- Validation:
  - MultiOutputClassifier uses classification-target validation to ensure appropriate usage.
