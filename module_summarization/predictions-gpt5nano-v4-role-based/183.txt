Feature Selection and Dimensionality Reduction on Sample Sets
==========================================================

Overview
--------
Feature selection and dimensionality reduction reduce the number of features (columns) in a dataset to
improve model performance, decrease training time, and mitigate overfitting. Feature selection
chooses a subset of existing features, while dimensionality reduction creates new features that
express the data more compactly.

This section covers how to apply feature selection and dimensionality reduction to sample sets (X, y)
in typical machine learning workflows, how they interact with pipelines and estimators, and practical
usage patterns.

Concepts
--------
- Feature Selection vs Dimensionality Reduction
  - Feature Selection: selects a subset of the original features based on some criterion.
  - Dimensionality Reduction: transforms data into a lower-dimensional space (could be linear or nonlinear; creates new features).
- Typical use cases
  - Improve generalization by removing noisy or redundant features.
  - Decrease training time and memory usage.
  - Mitigate the curse of dimensionality in high-dimensional datasets.
- When to apply
  - Before classifiers or regressors to simplify the input space.
  - Within a pipeline to ensure cross-validated selection and prevent data leakage.

API Snapshot
------------
Common classes and approaches (illustrative; import paths follow scikit-learn conventions):
- Feature selection
  - VarianceThreshold: remove features with low variance.
  - SelectKBest: select the top k features by a scoring function (e.g., f_classif, chi2).
  - SelectFromModel: select features based on importance weights from a fitted model (embedded methods).
  - Recursive Feature Elimination (RFE) / RFECV: recursively remove least-important features.
- Dimensionality reduction
  - PCA (Principal Component Analysis): linear dimensionality reduction via orthogonal components.
  - TruncatedSVD: suitable for sparse data (e.g., text data with TF-IDF features).
  - NMF (Non-negative Matrix Factorization): parts-based, non-negative features generation.
  - FactorAnalysis: probabilistic dimensionality reduction.

Data flow with sample sets
--------------------------
- Input X: shape (n_samples, n_features)
- Target y: shape (n_samples,) or (n_samples, n_outputs) for multi-target problems
- Output after transformation: reduced feature matrix X′ with shape (n_samples, n_features′)

Usage patterns
--------------
- Integrate feature selection or dimensionality reduction in a Pipeline to ensure proper cross-validation.
- Use with a classifier/regressor or a multi-output estimator as a preprocessing step.

Examples
--------

1) PCA for dimensionality reduction before classification
.. code-block:: python

   from sklearn.decomposition import PCA
   from sklearn.pipeline import Pipeline
   from sklearn.linear_model import LogisticRegression

   pipe = Pipeline([
       ('pca', PCA(n_components=0.95, random_state=42)),
       ('clf', LogisticRegression(max_iter=1000, random_state=42))
   ])
   pipe.fit(X_train, y_train)
   preds = pipe.predict(X_test)

2) SelectKBest with chi-squared feature scoring
.. code-block:: python

   from sklearn.feature_selection import SelectKBest, chi2
   from sklearn.pipeline import Pipeline
   from sklearn.naive_bayes import GaussianNB

   pipe = Pipeline([
       ('sel', SelectKBest(chi2, k=20)),
       ('clf', GaussianNB())
   ])
   pipe.fit(X_train, y_train)
   preds = pipe.predict(X_test)

3) VarianceThreshold to drop near-constant features
.. code-block:: python

   from sklearn.feature_selection import VarianceThreshold
   from sklearn.linear_model import SGDClassifier
   from sklearn.pipeline import Pipeline

   pipe = Pipeline([
       ('var', VarianceThreshold()),
       ('clf', SGDClassifier())
   ])
   pipe.fit(X_train, y_train)
   preds = pipe.predict(X_test)

4) Feature selection with a multi-output estimator (example pattern)
.. code-block:: python

   from sklearn.ensemble import RandomForestClassifier
   from sklearn.multioutput import MultiOutputClassifier
   from sklearn.pipeline import Pipeline
   from sklearn.feature_selection import SelectFromModel

   base = RandomForestClassifier(n_estimators=100, random_state=42)
   multi_target = MultiOutputClassifier(base)

   pipe = Pipeline([
       ('sel', SelectFromModel(base)),
       ('clf', multi_target)
   ])
   pipe.fit(X_train, y_train_multi)
   preds = pipe.predict(X_test)

Implementation notes inspired by the codebase
---------------------------------------------
- Dependency metadata (build/install requirements) is defined to ensure consistent environments across
  CI, tests, and docs. Key minimum versions referenced include numpy, scipy, joblib, threadpoolctl,
  pytest, and cython, among others. This helps guarantee compatibility for feature selection and
  dimensionality reduction components in the broader project.
- The code structure includes meta-estimator scaffolding (e.g., _MultiOutputEstimator) to support
  applying estimators in multi-output settings. Feature selection and dimensionality reduction steps can
  be integrated as part of a pipeline or wrapped in a meta-estimator where appropriate.
- fit/transform flow is standard: fit(X, y) learns a transformation or feature subset; transform(X) (or
  the pipeline’s fit_transform) yields the reduced feature representation ready for the downstream estimator.

Implementation notes
--------------------
- For multi-output workflows, a pipeline can include a feature selector or dimensionality reducer prior to
  a MultiOutputClassifier/Regressor to ensure consistent feature spaces across targets.
- When using certain feature selection methods (e.g., chi-squared), ensure input features meet method-specific
  requirements (non-negativity for chi-squared; appropriate scaling for some scoring functions).

Best practices
--------------
- Normalize or scale data when required by the selected method (e.g., PCA itself is sensitive to scaling; some
  feature scores assume standardized data).
- Use cross-validated selection (e.g., RFECV or nested CV) to avoid overfitting the feature selection process.
- For text data or very sparse data, consider TruncatedSVD instead of PCA.
