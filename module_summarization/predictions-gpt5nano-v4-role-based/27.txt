w_kinetics.rst
==============

Overview
--------

w_kinetics is a backward-compatible wrapper around the kinetic analysis tooling used for path-tracing. It exposes a single subcommand, trace, which computes state-to-state kinetics by tracing trajectories and generating a structured output file for downstream averaging and uncertainty analysis.

Notes
- This command is deprecated in favor of using w_direct directly. A deprecation warning is emitted when w_kinetics is invoked.
- The actual functionality is provided by the DKinetics trace subcommand; w_kinetics simply forwards to that implementation for backwards compatibility.

What it does
-------------

- Traces trajectories to compute macrostate-to-macrostate fluxes, arrivals, and durations.
- Produces an output file (default: kintrace.h5) containing:
  - conditional_fluxes: [iteration][state][state] floating-point data
    Macrostate-to-macrostate fluxes (not normalized by the initial macrostate population)
  - conditional_arrivals: [iteration][stateA][stateB] integer
    Number of trajectories arriving at stateB given departure from stateA
  - total_fluxes: [iteration][state] floating-point
    Total flux into a given macrostate
  - arrivals: [iteration][state] integer
    Number of trajectories arriving at a given state in an iteration
  - duration_count: [iteration] integer
    Number of event durations recorded in the iteration
  - durations: [iteration][event duration] structured
    istate: integer, initial state of the event
    fstate: integer, final state of the event
    duration: float, duration in units of tau
    weight: float, trajectory weight at the end of the transition (not normalized by initial state population)

Output and normalization
------------------------

- The flux datasets are not normalized by the initial macrostate population. Therefore, the data by itself cannot be interpreted as rates without further processing.
- Use w_kinavg trace to normalize fluxes and durations while accounting for statistical fluctuations and correlations. Target fluxes (total flux into a state) do not require normalization.

Input requirements
------------------

- A bin assignment file (typically assign.h5) that includes trajectory labeling is required. See w_assign --help for details on generating this file.

Command-line and usage
----------------------

- Program: w_kinetics
- Subcommand: trace
- Description: averages and confidence intervals for path-tracing kinetics analysis
- Default output file: kintrace.h5
- Output flag: -o/--output can be used to specify a different file name

Migration note
--------------

- This wrapper prints a deprecation warning and forwards to w_direct trace. It is recommended to migrate to:
  - w_direct trace [options]
  - For standard usage, see the w_direct help and the DKinetics trace subcommand documentation.

Practical examples
------------------

- Basic invocation (uses defaults; emits deprecation warning)
  - w_kinetics trace

- Specify an output file
  - w_kinetics trace -o kintrace.h5

- Forward compatibility (recommended approach)
  - w_direct trace -o kintrace.h5
  Note: Options are shared with the DKinetics implementation; see w_direct --help for details.

Reference
---------

- Command: w_kinetics
- Subcommand: trace (implemented by WKinetics, a thin wrapper around DKinetics)
- Default output: kintrace.h5
- Input requirements: assign.h5 (bin assignment file with trajectory labeling)
- Output datasets:
  - /conditional_fluxes [iteration][state][state] (float)
  - /conditional_arrivals [iteration][stateA][stateB] (int)
  - /total_fluxes [iteration][state] (float)
  - /arrivals [iteration][state] (int)
  - /duration_count [iteration] (int)
  - /durations [iteration][event duration] (structured: istate, fstate, duration, weight)

See also
--------

- w_kinavg trace: Normalizes fluxes to compute rates and confidence intervals.
- w_assign --help: Guidance on creating the bin assignment file required by this analysis.
