Partial Dependence and Individual Conditional Expectation plots
===============================================================

Overview
--------
Partial Dependence (PD) plots reveal the marginal effect of one or two features on a model’s predicted outcome by averaging (or inspecting individual instances) the model’s predictions across the domain of the feature(s). Individual Conditional Expectation (ICE) plots extend PD by showing how predictions change for individual samples as the feature(s) vary, illustrating heterogeneity in the model’s behavior across the data.

Key ideas:
- PD: Averages the model response over the distribution of all other features.
- ICE: Plots one line per sample, showing how the prediction changes as a feature varies, highlighting variability across samples.
- 1-feature PD/ICE gives a 1D line (or a set of lines for ICE) per feature; 2-feature PD extends to a 2D surface (or grid) for two features.

Input handling and internal utilities (relevant to PD/ICE workflows)
-------------------------------------------------------------------

The internal helpers shown in this codebase assist with reliable feature naming and indexing, which are essential for selecting features to plot.

- _check_feature_names(X, feature_names=None)
  - Purpose: Validate and derive feature names for plotting.
  - Behavior:
    - If feature_names is None:
      - If X has columns (e.g., a pandas DataFrame), use X.columns.tolist().
      - Otherwise, generate default names ["x0", "x1", ..., "x(n_features-1)"] for NumPy arrays.
    - If feature_names is array-like, convert to a Python list.
    - Checks for duplicates and raises a ValueError if duplicates are found.
  - Returns: A list of feature names (str), one per feature in X.

- _get_feature_index(fx, feature_names=None)
  - Purpose: Resolve a feature’s index from either a string name or an integer index.
  - Behavior:
    - If fx is a string, requires feature_names to map the name to its index.
    - If the name is not found, raises a ValueError.
    - If fx is already an int, returns it unchanged.
  - Returns: The integer feature index.

Practical implications:
- You can pass feature identifiers as integers (0-based indices) or as feature names (strings). The internal helpers will map names to indices when needed, provided feature_names is available.
- When plotting, feature_names may be inferred automatically from X (DataFrame columns) or supplied by the user via feature_names.

Usage workflow (typical)
------------------------

1) Fit your estimator on your data.

2) Prepare data for PD/ICE:
- If you’re using a DataFrame, you can rely on column names for feature selection.
- If you’re using a NumPy array, you should supply feature_names or accept the default names generated by _check_feature_names.

3) Select features to analyze:
- You may specify features as integers or as strings (feature names).

4) Compute PD/ICE data and plot:
- PD (average effect): plot the mean model response over the feature grid.
- ICE (individual conditional): plot lines for individual samples across the grid, illustrating heterogeneity.

Examples
--------

Example 1 — Using feature names inferred from a DataFrame
- Data: X is a pandas.DataFrame with columns ["age", "income", "education"].
- Objective: Plot PD and ICE for the "age" feature.

- Pseudocode:
  - feature_names = _check_feature_names(X)  # returns ["age", "income", "education"]
  - idx_age = _get_feature_index("age", feature_names)  # -> 0
  - Use idx_age (0) to configure the PD/ICE calculation for the first feature.

Example 2 — Using a NumPy array with automatic naming
- Data: X is a NumPy array of shape (n_samples, n_features).
- Objective: Plot PD/ICE for the second feature (index 1).

- Pseudocode:
  - feature_names = _check_feature_names(X)  # returns ["x0", "x1", "x2", ...]
  - idx_feature = _get_feature_index(1, feature_names)  # -> 1
  - Use idx_feature (1) to compute PD/ICE for the second feature.

Example 3 — Mixed naming and error handling
- If you pass feature_names=["f1", "f2"] and request PD for "f3", _get_feature_index will raise a clear ValueError: Feature 'f3' not in feature_names.

Plotting notes
--------------

- For a single feature:
  - PD plot shows the model’s average prediction as the feature varies.
  - ICE plot shows one line per sample, illustrating how individual predictions evolve with the feature.

- For two features:
  - PD can be extended to a 2D grid (surface) showing the joint effect of two features.
  - ICE curves can be drawn as a set of colored surfaces or lines if supported by the plotting helper.

- Naming alignment:
  - When using pandas, feature names naturally align with column names.
  - When using NumPy arrays, ensure feature_names (or the inferred default names) correctly reflect the columns, so the correct features map to the intended axes.

Implementation notes (internal)
-------------------------------

- The helper functions enforce robust naming and indexing so that PD/ICE computations consistently map feature selections to the correct model inputs.
- Duplicates in feature_names are rejected to avoid ambiguity when selecting features by name.
- The approach works seamlessly whether you pass feature selections as integers or strings, as long as the mapping (feature_names) is available.

API references (high-level)
---------------------------

- _check_feature_names(X, feature_names=None)
  - Returns: List[str] of feature names suitable for downstream PD/ICE calculations.
  - Raises: ValueError if duplicates are found.

- _get_feature_index(fx, feature_names=None)
  - Returns: int index for the requested feature.
  - Raises: ValueError if a named feature is not found in feature_names.

Best practices
--------------

- Prefer DataFrame inputs when planning to plot PD/ICE, as feature names are automatically detected.
- When using NumPy arrays, supply meaningful feature_names to improve interpretability of plots.
- Use ICE plots to diagnose feature-wise heterogeneity across samples; if ICE lines are highly divergent, a simple PD may obscure important variation.
- For multi-feature PD, ensure the visualization is comprehensible (2 features is typically the practical maximum for clear PD surfaces).

Changelog and compatibility
---------------------------

- Internal helper utilities (_check_feature_names, _get_feature_index) help maintain consistent behavior across PD/ICE plotting workflows.
- These utilities are stable under typical usage where feature naming and indexing are required for feature selection in plots.

Appendix: API sketch (conceptual)
----------------------------------

- partial_dependence_plot(estimator, X, features, feature_names=None, grid_resolution=20, kind="average")
  - Produces PD plots for specified features.

- partial_dependence_plot_ice(estimator, X, features, feature_names=None, grid_resolution=20)
  - Produces ICE plots for specified features (single-feature plots by default; multi-feature plotting follows a lower-dimensional visualization strategy).
