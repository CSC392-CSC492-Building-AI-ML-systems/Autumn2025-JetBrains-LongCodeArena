# Simulation API Reference

This module provides utilities for visualizing and validating Bluesky plans in simulation contexts. It includes plotting helpers, plan summarization, and a simple plan limit-checker.

## Functions

### plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)

Plot the raster path for a given Bluesky plan.

- Signature
  - `plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)`

- Parameters
  - `plan` (iterable)
    - An iterable that yields Bluesky `Msg` objects. The iterable must not be a coroutine.
  - `x_motor` (str)
    - Name of the motor to be treated as the x-coordinate.
  - `y_motor` (str)
    - Name of the motor to be treated as the y-coordinate.
  - `ax` (matplotlib.axes.Axes, optional)
    - Axes to plot on. If `None`, a new figure/axes is created.
  - `probe_size` (float, optional)
    - If provided, draw a probe as circles of this radius (in the same units as motor positions). If `None`, probe points are drawn as markers.
  - `lw` (float, optional)
    - Line width for the path and markers.

- Behavior
  - Iterates through the plan and tracks current x and y positions by handling:
    - `cmd == 'set'`: updates `cur_x` or `cur_y` when the corresponding motor is encountered.
    - `cmd == 'save'`: appends the current `(cur_x, cur_y)` to the trajectory.
  - Plots the path as a line connecting saved points.
  - Labels axes with `x_motor` and `y_motor`.
  - If `probe_size` is `None`, plots read points using a scatter plot.
  - If `probe_size` is provided, renders a collection of black semi-transparent circles at each trajectory point.
  - Returns a dictionary with:
    - `'path'`: the Line2D path object.
    - `'events'`: the scatter plot collection or patch collection of read points.

- Notes
  - The function assumes valid `set`/`save` messaging in the plan. If the plan does not provide meaningful trajectory data, behavior may raise errors (e.g., due to uninitialized coordinates).

- Example
  - Basic usage (pseudo-plan shown for illustration):
    ```
    # plan is an iterable of Bluesky Msg objects
    path = plot_raster_path(plan, x_motor='x', y_motor='y')
    ```
  - With probe visualization:
    ```
    path = plot_raster_path(plan, x_motor='x', y_motor='y', probe_size=0.5)
    ```

- Dependencies
  - Matplotlib (for plotting)

### summarize_plan(plan)

Print a minimal, human-friendly summary of a Bluesky plan.

- Signature
  - `summarize_plan(plan)`

- Parameters
  - `plan` (iterable)
    - An iterable that yields Bluesky `Msg` objects.

- Returns
  - `None`
  - The function delegates to `print_summary_wrapper(plan)` from Bluesky to produce the summary output. The loop over the wrapper ensures the summary is printed.

- Backwards compatibility
  - `print_summary` is an alias for `summarize_plan` (for compatibility with older code).

- Example
  ```
  summarize_plan(plan)
  ```

- Notes
  - Requires Bluesky’s `print_summary_wrapper`.
  - The function itself does not return data; it prints to stdout.

### check_limits(plan)

Validate that a plan does not move devices beyond their defined limits.

- Signature
  - `check_limits(plan)`

- Parameters
  - `plan` (iterable)
    - An iterable that yields Bluesky `Msg` objects.

- Returns
  - `None`
  - The function consumes the plan, checking each `Msg` with `cmd == 'set'` for limit compliance by:
    - If the target object has a `check_value` method, it calls `obj.check_value(value)` with the proposed value.
    - If the target object does not have `check_value`, it emits a warning and adds the object to an ignore-list so it won’t be re-checked for subsequent messages.

- Behavior details
  - Maintains an `ignore` list of objects for which limit checks are not possible or have previously failed.
  - For each `Msg` where `cmd == 'set'` and the object is not ignored:
    - If `hasattr(msg.obj, "check_value")` is true, call `msg.obj.check_value(msg.args[0])`.
    - Otherwise, emit a warning:
      - f"{msg.obj.name} has no check_value() method to check if {msg.args[0]} is within its limits."
      - The object is added to `ignore` to skip further checks.

- Notes
  - The function is a plan consumer; it does not yield or transform the plan.
  - This simple checker relies on objects exposing a `check_value` method to validate planned values.

- Example
  ```
  check_limits(plan)
  ```
  - If a motor lacks `check_value`, a warning is emitted and that motor is ignored for future checks.

- Dependencies
  - The standard library `warnings.warn` is used to issue messages.

## How these utilities fit into the workflow

- plot_raster_path is useful for visual inspection of a plan’s motion path, helping verify trajectories against two motors (x/y) and optionally visualize probe placements.
- summarize_plan (via print_summary alias) provides a concise textual sketch of a plan, aiding quick inspection without executing the plan.
- check_limits offers a lightweight early check to catch potential limit violations before or during plan execution, assuming devices expose a `check_value` method.

---

## Requirements and caveats

- These utilities assume Bluesky Msg objects and, for plotting, a compatible Matplotlib environment.
- plot_raster_path expects a plan that yields `Msg` objects with meaningful `set` and `save` commands.
- check_limits depends on devices exposing a `check_value` method; otherwise, a warning is issued and the object is ignored for subsequent checks.
- The code relies on Bluesky’s print summary preprocessing (`print_summary_wrapper`) for the plan summarization.
