Module: optimization.rst

Title
- Optimization Utilities (general-purpose fitting utilities)

Overview
- This module provides general, model-agnostic optimization helpers that can be reused with any objective function, detached from a specific LikelihoodModel. It encapsulates a flexible interface to SciPy’s optimization routines, enabling fitting via various solvers (e.g., Newton-Raphson, Nelder–Mead, BFGS, Powell, conjugate gradient, Newton–conjugate gradient, basin-hopping, or a generic minimize wrapper). The goal is to enable re-use across models by supplying an objective function, its gradient, and optional Hessian information.

Public API
- _check_method(method, methods)
  - Purpose: Validate that a requested optimization method is among the supported methods.
  - Parameters:
    - method: str, the requested solver name.
    - methods: iterable of str, allowed solver names.
  - Behavior: Raises ValueError if method is not in methods.

- Optimizer
  - A lightweight, model-agnostic optimizer wrapper.
  - Core method: _fit(objective, gradient, start_params, fargs, kwargs, hessian=None, method='newton', maxiter=100, full_output=True, disp=True, callback=None, retall=False)
  - Purpose: Fit an objective function (and its gradient, with optional Hessian) to data by optimizing parameters.
  - Returns:
    - xopt: array
      The optimized parameter vector.
    - retvals: dict or None
      If full_output is True, a dictionary containing solver-specific information returned by the chosen solver; otherwise None.
    - optim_settings: dict
      A dictionary containing the solver arguments that were passed to the optimizer.

API Details

1) _check_method(method, methods)
- Description:
  - Small helper to ensure the requested optimization method is supported.
  - If the provided method is not in the allowed set, a ValueError is raised with a descriptive message.
- Typical usage:
  - _check_method('newton', ['newton','nm','bfgs','powell','cg','ncg','basinhopping','minimize'])

2) Optimizer._fit(
- Signature:
  - _fit(self, objective, gradient, start_params, fargs, kwargs,
         hessian=None, method='newton', maxiter=100, full_output=True,
         disp=True, callback=None, retall=False)
- Purpose:
  - Fit a model by maximizing/minimizing a provided objective using a selected solver.
  - The framework is designed to be generic and usable with any model where an objective function (e.g., log-likelihood, loss) and its gradient (and optionally Hessian) are supplied.
- Parameters:
  - objective: callable
    The objective function to optimize. Typically a scalar-valued function of the parameter vector.
  - gradient: callable
    The gradient (first derivative) of the objective with respect to the parameters.
  - start_params: array_like
    Initial guess for the solution. By default, this is an array of zeros.
  - fargs: tuple
    Extra positional arguments passed to the objective (and gradient) after the parameter vector, i.e., objective(x, *fargs).
  - kwargs: dict
    Additional keyword arguments passed to the objective (and gradient) as needed.
  - hessian: callable or None
    Optional Hessian (second derivative) of the objective. If None, the Hessian is not provided to solvers that support it.
  - method: str
    Optimization method to use. Supported values include:
      - 'newton' (Newton-Raphson),
      - 'nm' (Nelder-Mead),
      - 'bfgs' (Broyden–Fletcher–Goldfarb–Shanno),
      - 'powell' (Powell’s method),
      - 'cg' (conjugate gradient),
      - 'ncg' (Newton–conjugate gradient),
      - 'basinhopping' (global basin-hopping solver),
      - 'minimize' (wrapper around scipy.optimize.minimize)
  - maxiter: int
    Maximum number of iterations to perform. Note: basinhopping ignores this argument.
  - full_output: bool
    If True, return a detailed retvals dictionary from the underlying solver.
  - disp: bool
    If True, print convergence messages.
  - callback: callable or None
    Callback function invoked after each iteration with the current parameter vector.
  - retall: bool
    If True, return a list of iterates (solution at each iteration) within retvals.
- Returns:
  - xopt: array
    The optimized parameter vector that minimizes/maximizes the objective.
  - retvals: dict or None
    Solver-specific information returned when full_output is True; otherwise None.
  - optim_settings: dict
    A dictionary capturing the solver parameters that were used (the explicit arguments to the solver).
- Notes and solver-specific behavior:
  - The basinhopping solver ignores maxiter, retall, and full_output explicit arguments.
  - Each solver has its own set of optional arguments (as described in the docstring notes for available solver options). These are typically surfaced via optim_settings or the retvals dictionary, and any solver-specific semantics (like gradient tolerance, step sizes, etc.) follow SciPy’s minimize/optimizers conventions.
  - The wrapper is designed to be compatible with various objective/gradient/hessian configurations. If a Hessian is not supplied, some solvers may fall back to numerical approximations.

Practical usage patterns

- Simple quadratic objective (illustrative example):
  - Goal: minimize f(x) = 0.5 x^T Q x - b^T x
  - Setup:
    - objective(x) returns 0.5 * x.T @ Q @ x - b.T @ x
    - gradient(x) returns Q @ x - b
  - Example:
    - optimizer = Optimizer()
    - x0 = np.zeros(n)
    - fargs = tuple()  # if needed
    - ret_x, retvals, settings = optimizer._fit(objective, gradient, x0, fargs, {}, hessian=None, method='bfgs', maxiter=500, full_output=True, disp=True, callback=None, retall=False)

- Generic usage with a callback and Hessian:
  - Provide a Hessian function hinv or hessian to enable second-order convergence (if available in the chosen method).
  - Supply a callback to monitor progress after each iteration.

Notes and caveats
- The module purposefully decouples optimization from a specific model type, enabling reuse across different fitting problems.
- Behavior and available options largely align with SciPy’s optimization ecosystem; the documentation within the docstring reflects the mapping of solver options to the underlying SciPy routines.
- For basinhopping, be aware of its different semantics and the fact that some common options (maxiter, full_output, retall) are ignored.

Reference to related components
- SciPy optimization (scipy.optimize): The _fit method delegates to SciPy solvers, following the method selection and argument conventions outlined by the underlying SciPy API.
- LikelihoodModel integration (not required): While the original design referenced LikelihoodModel, this module provides a generic interface that can be used without a LikelihoodModel, enabling more flexible reuse.

Usage rationale
- This module provides a compact, reusable foundation for fitting models against an objective function, accommodating a range of optimization strategies and exposing a consistent return structure for integration with downstream tooling (e.g., results objects, logging, diagnostic plots).

End-user guidance
- To adopt this in your project, define your objective and gradient (and optionally Hessian), decide on an appropriate solver, and invoke _fit with sensible defaults for your problem scale and convergence criteria.
- Refer to SciPy’s optimization documentation for deeper details on solver-specific parameters and convergence behavior.
