MixedGLM
==========
Bayesian inference for generalized linear mixed models (GLMMs).

Overview
--------
This module implements Bayesian estimation for GLMMs with families that do not introduce additional scale or shape parameters (currently binomial and Poisson). It provides two estimation approaches:
- Laplace approximation to the posterior (maximum a posteriori, fit_map)
- Variational Bayes (mean-field approximation to the posterior, fit_vb)

All realizations of random effects are modeled as mutually independent. The exog_vc matrix supplies covariates for the random (random-effect) part of the model. Each column of exog_vc corresponds to an independent random effect realization. The standard deviation parameters of these random effects are constrained to be equal within subsets of the columns, as defined by ident. When formulas are used, columns derived from a common formula share the same standard deviation.

Data formats and sparsity
-------------------------
- exog: design matrix for fixed effects (n_obs by n_fix)
- exog_vc: design matrix for random effects (n_obs by n_rand), can be scipy.sparse
  - If a dense matrix is provided, it is converted to a sparse matrix internally.
  - If formulas are used, a temporary dense version of exog_vc may be created, which is then converted to sparse.
- ident: integer labels (length = number of columns in exog_vc)
  - Equal ident values indicate that those columns share the same random-effect standard deviation.
  - When using formulas, ident is constructed internally from the specified vc_formulas.
- Prior and parameterization are designed so that random effects realizations are independent given the variance-structure parameters.

Model and parameterization
--------------------------
The joint density factors as:

p(y | vc, fep) p(vc | vcp) p(vcp) p(fe)

- p(y | vc, fep): likelihood of the data given the random effects vc and fixed-effect parameters fep (depends on the chosen GLM family).
- p(vc | vcp): prior for random effects, Gaussian and independent across realizations and ident groups.
- p(vcp): prior for the standard-deviation parameters of the random effects (vcp). The vcp parameters represent the log standard deviations; priors on vcp are Gaussian with mean 0 and a user-specified standard deviation vcp_p.
- p(fe): prior for fixed effects, Gaussian with mean 0 and standard deviation fe_p.

Parameter names
---------------
- fep: fixed effects parameters (columns of exog)
- vc: random effects realizations (columns of exog_vc)
- vcp: standard deviation parameters for variance components (distinct labels in ident)

Fitting methods
--------------
- fit_map(): Laplace approximation to the posterior (MAP estimate)
- fit_vb(): Variational Bayes approximation (mean-field)

Configuration options
- family: a statsmodels.genmod.families instance (e.g., Binomial, Poisson)
- vcp_p: prior std dev for the log standard deviations of random effects
- fe_p: prior std dev for fixed effects parameters
- fep_names: names for fixed effects parameters (default constructed if None)
- vcp_names: names for variance component parameters (default constructed if None)
- vc_names: names for random-effect realizations (optional)

Notes
-----
- The implementation only supports families without extra scale or shape parameters (binomial and Poisson).
- All random effects are modeled as independent Gaussian values, with variance structures determined by ident.
- When formulas are used to define random-effects structure, ident is derived from the formulas; otherwise ident must be supplied to group columns of exog_vc into variance-structure blocks.
- For large or sparse exog_vc, the sparse representation can be crucial for performance.

Example
-------
The following illustrates constructing a model and running a MAP fit. Adjust data shapes to your problem.

- endog: vector of responses (n_obs,)
- exog: fixed-effects design matrix (n_obs, n_fix)
- exog_vc: random-effects design matrix (n_obs, n_rand)
- ident: integer labels for variance-structure (length n_rand)
- family: e.g., families.Binomial() or families.Poisson()

from statsmodels.genmod import families
import numpy as np
import scipy.sparse as sp

# Example data (replace with real data)
endog = np.array([0, 1, 0, 1, ...])
exog = np.ones((len(endog), 2))  # example fixed effects
exog[:, 1] = np.linspace(-1, 1, len(endog))

# Random effects design
exog_vc = sp.csr_matrix(np.random.randn(len(endog), 3))  # or a dense matrix
ident = np.array([0, 1, 0])  # two variance blocks share variance for ident=0, ident=1 unique

# Build model
model = MixedGLM(endog, exog, exog_vc, ident, vcp_p=0.5, fe_p=1.0, family=families.Binomial())

# Fit using MAP
results = model.fit_map()
print(results.summary())

Notes on naming defaults
------------------------
If fep_names, vcp_names, or vc_names are None, the implementation will generate default names based on the input shapes and labels.
