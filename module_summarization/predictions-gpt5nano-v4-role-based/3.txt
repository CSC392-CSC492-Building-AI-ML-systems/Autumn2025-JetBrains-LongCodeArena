Publishing for web review

Overview
- Web review publishing relies on the session’s ability to connect to the ftrack event hub and publish events to the server so that web-based review interfaces can subscribe to and present reviewable data.
- The embedded event hub can run in a background thread and, when auto_connect_event_hub is enabled, will connect automatically to the event server to enable non-local publishing and subscribing.

Key concepts
- Event Hub: A mechanism for publishing and subscribing to events across process boundaries and servers. Publishing non-local events enables web review clients to react to changes in real time.
- Local vs non-local events: Non-local events are propagated to the event server for consumption by other processes or web clients; local events stay within the current process or host.
- Automatic hub connection: If auto_connect_event_hub is True, the hub connects in the background at startup. If False, you must connect manually via EventHub.connect when required.
- Authentication: All requests to the server must be authenticated. SessionAuthentication attaches the necessary headers (ftrack-api-key and ftrack-user) to each HTTP request.

Prerequisites
- Server connection details:
  - server_url: URL of the ftrack server (including port if needed). Can be supplied directly or via FTRACK_SERVER environment variable.
  - api_key: API key for authentication. Can be supplied directly or via FTRACK_API_KEY (or legacy FTRACK_APIKEY) environment variable.
  - api_user: User to perform operations as. Can be supplied directly or via FTRACK_API_USER (or Python’s getpass default).
- Event publishing capability:
  - Enable non-local publishing by enabling the embedded event hub (auto_connect_event_hub=True) or by manually connecting the hub later.
  - Plugins that implement web review publishing are discovered via plugin_paths or environment defaults (e.g., FTRACK_EVENT_PLUGIN_PATH).
- Optional cache and schema: A session can be configured with a cache and a schema cache path, which is optional but recommended for performance.

Configuring a session for web review publishing
- Create a session with required authentication and enable the event hub:
  - server_url: ftrack server URL
  - api_key: API key
  - api_user: API user
  - auto_connect_event_hub: True (enable background hub connection for non-local events)
- Plugin discovery and web review integrations:
  - Provide plugin_paths to include web review plugins, or rely on default plugin search paths provided by the environment.
- Caching and performance:
  - Optionally configure cache and cache_key_maker; the session will layer the provided cache on top of a MemoryCache, improving performance for repeated lookups.
- Example (conceptual):
  - A session is created with auto_connect_event_hub=True so that non-local web review events can be published as soon as the session starts.

Operation flow for web review publishing
- Session initialization:
  - Validates server_url, api_key, and api_user (from arguments or environment).
  - Sets up authentication (SessionAuthentication) and prepares an event hub.
- Event hub behavior:
  - If auto_connect_event_hub is True, the hub connects to the event server in a background thread, enabling publishing and subscribing to non-local events.
  - Subscribing to events does not require a connected hub (subscription can be established independently of hub status).
- Publishing lifecycle:
  - Web review plugins receive relevant events (e.g., publish actions, review status changes) and forward or transform them for the web review interface.
  - Web clients subscribe to the appropriate event channels to reflect updates in real time.
- Operational notes:
  - If a plugin requires a connected hub, its initialization should verify hub status and handle disconnections gracefully.
  - Access is secured via the API key and user, injected into all HTTP requests by SessionAuthentication.

Practical examples
- Enabling web review publishing via automatic hub connection:
  - Create a Session with auto_connect_event_hub set to True and ensure web review plugins are discoverable through plugin_paths or environment defaults.
- Publishing a review event (conceptual flow):
  - Use the session’s event hub to publish a review-related event.
  - Web review frontend subscribed to the corresponding event channel receives the update and refreshes its view.

Notes and considerations
- Security: Keep API keys secure; the session uses ftrack-api-key and ftrack-user headers for authentication on every request.
- Startup and performance: The hub runs in a background thread to improve startup time; for plugins that require an active hub, status checks should be performed and hub connections should be ensured.
- Local vs non-local events: Publishing for web review relies on non-local events being propagated to the event server so web clients can subscribe remotely.
- Configuration flexibility: You can customize plugin discovery, caching, and schema caching to balance startup time and runtime performance.

Reference to code (conceptual)
- SessionAuthentication: Attaches authentication headers to requests:
  - Headers include ftrack-api-key and ftrack-user.
- Session (relevant parameters):
  - server_url, api_key, api_user: required for authentication and server access.
  - auto_connect_event_hub: enables automatic background hub connection for non-local event publishing.
  - plugin_paths: locations to search for plugins (including web review publishers).
  - cache and cache_key_maker: control session caching strategy.
  - schema_cache_path: optional path to cache schema metadata for faster access.

This documentation provides guidance on configuring and using the session for publishing to a web review workflow, highlighting the role of the embedded event hub, authentication, and plugin-based web review integrations.
