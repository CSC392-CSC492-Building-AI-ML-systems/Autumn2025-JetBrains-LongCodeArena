Channel Options
===============

Overview
--------
Vega-Lite charts are defined by mapping data fields to visual encoding channels (x, y, color, size, shape, etc.). Each channel can be configured with a variety of options to control how data is represented and how the visualization is presented (scales, axes, legends, tooltips, formatting, etc.). This document explains how to express and apply Channel Options when using the Vega-Lite integration in this project, including how to supply specifications via YAML or JSON and how to bind data from notebook variables.

Where Channel Options live in the code
-------------------------------------
- Channel options are provided as part of the Vega-Lite specification (the encoding block) that you pass to the cell magic.
- The core rendering path accepts a spec in YAML or JSON (depending on availability of PyYAML and the --json flag) and hands it to Vega-Lite v5 for rendering.
- If you supply a DataFrame via the cell magic’s data argument, the data is transformed into a Vega-Lite-friendly format using the provided data transformers.

Key Concepts
------------
- Encoding channels: The main channels you configure in the encoding object (for example, x, y, color, size, shape, opacity, tooltip, detail, etc.).
- Channel options: Properties nested under each channel that control mapping and presentation, including:
  - field: the data field name to map
  - type: data type (quantitative, nominal, ordinal, temporal)
  - aggregate, bin, timeUnit: data transformation hints
  - scale: visual scale configuration (domain, range, scheme, etc.)
  - axis: axis presentation (title, labelAngle, etc.) for x and y channels
  - legend: legend configuration for channels like color, shape, size
  - format, labelAngle, etc.: formatting and display options for axes or legends
  - additional channel-level properties as defined by Vega-Lite (e.g., bin, timeUnit, sort, etc.)

Supported Encoding Channels (typical examples)
------------------------------------------------
- x: Defines the horizontal position
- y: Defines the vertical position
- color: Color/{} mapping for category or value
- size: Encodes a quantitative measure as point size
- shape: Encodes a categorical variable by shape
- opacity: Encodes a measure by transparency
- tooltip: Additional information shown on hover
- detail: Breaks marks into sub-groups
- order: Determines rendering order of marks
- text, href, key, etc.: Other specialized channels supported by Vega-Lite v5

Examples
--------

1) Basic point chart with x and y channels
.. code-block:: yaml

  mark: point
  encoding:
    x:
      field: horsepower
      type: quantitative
    y:
      field: mpg
      type: quantitative

2) Color with legend and axis title customization
.. code-block:: yaml

  mark: bar
  encoding:
    x:
      field: origin
      type: nominal
      axis:
        title: "Origin"
    y:
      field: count
      type: quantitative
    color:
      field: origin
      type: nominal
      legend:
        title: "Origin"
      scale:
        scheme: category10

3) Adding tooltips and aggregations
.. code-block:: yaml

  mark: bar
  encoding:
    x:
      field: origin
      type: nominal
    y:
      aggregate: count
      type: quantitative
    tooltip:
      field: origin
      type: nominal

4) Using a DataFrame in the notebook (via the cell magic)
Notes:
- Provide the DataFrame name as the optional data argument to the magic.
- The data block in the spec is then populated from the DataFrame using the configured data_transformers.

.. code-block:: yaml

  # Example in a notebook with a DataFrame named `df`
  %%vegalite df
  mark: point
  encoding:
    x:
      field: horsepower
      type: quantitative
    y:
      field: mpg
      type: quantitative
    color:
      field: origin
      type: nominal

Spec and data handling cautions
--------------------------------
- YAML availability:
  - If PyYAML is installed, you may provide the spec in YAML in the cell, and the magic will parse it as YAML.
  - If PyYAML is not installed, the spec must be valid JSON unless you pass --json to force JSON parsing.
- JSON mode:
  - When using --json, the cell content is parsed strictly as JSON.
  - If parsing JSON fails and YAML is unavailable, you will receive a ValueError indicating invalid JSON and the need to install pyyaml for YAML parsing.
- Data binding:
  - If you supply a data variable (via the optional data argument to the magic), the data is transformed to Vega-Lite-friendly values using the configured data transformers (e.g., for pandas DataFrames).
  - If the data argument is a string, it is interpreted as a URL (data: {"url": "<string>"}).
  - If the data argument is a dict, it is passed through as-is (useful for inline data).
- Versioning:
  - The magic supports a --version option (default v5) and maps to the corresponding Vega-Lite runtime (e.g., v5).

Using the Channel Options in practice
-------------------------------------
- Build your Vega-Lite spec with a clear encoding block that defines the channels you want to visualize.
- Customize axis/legend/scale per channel using axis, legend, and scale properties.
- Use data binding to apply the encoding to a DataFrame or other data sources.
- Choose JSON or YAML input based on your environment and the presence of PyYAML.

API and integration notes
-------------------------
- The rendering pipeline expects a spec object with a top-level mark and encoding definitions.
- Channel options live inside each encoding channel and must conform to Vega-Lite v5 schema expectations.
- The cell magic vegalite orchestrates:
  - Argument parsing (data binding, version, json mode)
  - Spec parsing (YAML or JSON)
  - Optional data preparation via _prepare_data
  - Rendering via VegaLite(spec)
