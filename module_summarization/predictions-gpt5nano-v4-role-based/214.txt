# optimization.rst

Overview
- The optimization primitives are designed to be reusable for any model fitting task, decoupled from a specific LikelihoodModel.
- The core entry point is the _fit function exposed via Optimizer, which delegates to a variety of SciPy optimization solvers.
- Three broad families of estimation algorithms are supported for parameter estimation, spanning second-order, first-order, and derivative-free/global search strategies.

Three algorithm families for parameter estimation
1) Newton-type (second-order) methods
   - Use curvature information (Hessian or its approximation) to guide updates.
   - Typical convergence can be fast when the objective is well-behaved and Hessian information is informative.
   - Supported methods:
     - newton: Newton-Raphson with Hessian information (explicit or via approximation)
     - ncg: Newton-Conjugate-Gradient (Hessian-vector products used to guide steps)
     - bfgs: Broyden-Fletcher-Goldfarb-Shanno; quasi-Newton with an inverse Hessian approximation
   - Notes:
     - These methods generally rely on gradient and Hessian information (or their efficient approximation).
     - The optimizer exposes an hessian argument; if unavailable, these solvers may rely on internal approximations.

2) Derivative-free / first-order methods
   - Do not require explicit Hessian information; leverage gradients (when available) or derivative-free heuristics.
   - Supported methods:
     - cg: Conjugate Gradient; a gradient-based method suitable for large-scale problems
     - (The wrapper 'minimize' can internally select among derivative-based methods via SciPy’s minimize)
   - Notes:
     - Useful when Hessian information is unavailable or expensive to compute.
     - Performance depends on problem conditioning and gradient availability.

3) Derivative-free local & global search
   - Do not require gradient or Hessian information; robust to non-smooth or poorly scaled problems; include global search components where applicable.
   - Supported methods:
     - nm: Nelder-Mead
     - powell: Powell’s method
     - basinhopping: global basin-hopping strategy that combines local minimizers with a stochastic exploration
   - Notes:
     - Basinhopping performs a global search and may ignore some explicit arguments (e.g., maxiter, retall, full_output).
     - Suitable when the objective has multiple local minima or complex landscape.

How to use the optimizer
- Core signature (illustrative)
  - _fit(objective, gradient, start_params, fargs, kwargs, hessian=None, method='newton', maxiter=100, full_output=True, disp=True, callback=None, retall=False)
- Parameters (summary)
  - objective: function to maximize/minimize (e.g., log-likelihood)
  - gradient: function computing the gradient (score); may be None if not available for certain solvers
  - start_params: initial guess for the parameter vector
  - fargs, kwargs: extra arguments passed to objective/gradient
  - hessian: optional Hessian or Hessian-vector product provider
  - method: solver selection among 'newton', 'ncg', 'bfgs', 'cg', 'nm', 'powell', 'basinhopping', or 'minimize' (SciPy wrapper)
  - maxiter: maximum iterations (basinhopping ignores this)
  - full_output: return additional solver output when available
  - disp: print convergence messages
  - callback: user-provided callback after each iteration
  - retall: return all intermediate solutions if supported

Supported methods at a glance
- Newton-type / second-order
  - newton: Newton-Raphson with Hessian information
  - ncg: Newton-Conjugate-Gradient
  - bfgs: BFGS quasi-Newton
- Gradient-based / first-order
  - cg: Conjugate Gradient
  - minimize: SciPy minimize wrapper (flexible, depends on internal method choice)
- Derivative-free / global search
  - nm: Nelder-Mead
  - powell: Powell’s method
  - basinhopping: global basin-hopping strategy (with local minimizers)

Practical examples (conceptual)
- Newton-type optimization
  - optimizer = Optimizer()
  - xopt, retvals, settings = optimizer._fit(objective, gradient, start_params=np.zeros(n), fargs=(), kwargs={}, method='newton', maxiter=200)
- Derivative-free local optimization
  - optimizer = Optimizer()
  - xopt, retvals, settings = optimizer._fit(objective, gradient, start_params=np.zeros(n), fargs=(), kwargs={}, method='nm', maxiter=500)
- Global optimization via basin-hopping
  - optimizer = Optimizer()
  - xopt, retvals, settings = optimizer._fit(objective, gradient, start_params=np.zeros(n), fargs=(), kwargs={}, method='basinhopping', maxiter=1000)

Notes and caveats
- The basinhopping solver ignores maxiter, retall, and full_output explicit arguments.
- The internal _fit function is designed to be reusable for any model fitting task, not just a specific LikelihoodModel.
- When using methods that rely on Hessian information, provide a Hessian or Hessian-vector product if available; otherwise, the solver may approximate or rely on numerical schemes.
- The wrapper 'minimize' delegates to SciPy’s minimize, enabling additional solvers beyond the explicit list above. The exact solver used depends on the arguments passed to the wrapper.
