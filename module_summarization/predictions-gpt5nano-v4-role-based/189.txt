Kernel Ridge Regression
=======================

Kernel ridge regression (KRR) combines ridge regression (linear least squares with L2 regularization) with the kernel trick. It learns a linear function in the feature space induced by the chosen kernel, which corresponds to a non-linear function in the original input space for non-linear kernels. The model is trained in closed form and supports multi-target regression.

Key properties
- Closed-form solution: The dual coefficients solve (K + alpha I) alpha = y, where K is the kernel matrix and alpha is the regularization strength.
- Multi-target support: y can be a 1D array of shape (n_samples,) or a 2D array of shape (n_samples, n_targets).
- Non-sparse model: Unlike SVR, the learned model is typically dense (not sparse) in kernel space.
- Kernel flexibility: Supports a wide range of kernels via the pairwise_kernels interface. Can operate with a precomputed kernel matrix.

How it works
- Given training data X (n_samples, n_features) and target y (n_samples, [n_targets]):
  - Select a kernel K(X, X) or supply a precomputed kernel matrix.
  - Solve (K + alpha I) α = y for the dual coefficients α (shape: (n_samples,) or (n_samples, n_targets)).
  - Predictions for new samples X' are f(X') = K(X', X) α, where K(X', X) is the kernel matrix between X' and the training data.
- If kernel is callable, the kernel is computed by calling the function on pairs of samples. If kernel is a string, it must be one of the supported pairwise kernels (or "precomputed").

Parameters
- alpha: float or array-like of shape (n_targets,), default=1.0
  Regularization strength. Must be non-negative. If an array is provided, it applies a separate regularization for each target.
  Alpha corresponds to 1/(2C) in some other linear models.

- kernel: str or callable, default="linear"
  Kernel mapping used internally. If a string, it must be one of the supported kernels from pairwise_kernels or "precomputed".
  If "precomputed", X is interpreted as a precomputed kernel matrix.
  If a callable, it is invoked on pairs of samples; it should accept two 1D arrays (rows) and return a scalar kernel value.

- gamma: float, default=None
  Gamma parameter for certain kernels (e.g., RBF, Laplacian, polynomial, exponential chi-square, sigmoid). Ignored by kernels where it is not applicable.

- degree: int, default=3
  Degree of the polynomial kernel. Ignored by kernels other than polynomial-based ones.

- coef0: float, default=1
  Independent term in polynomial and sigmoid kernels. Ignored by other kernels.

- kernel_params: dict, default=None
  Additional keyword arguments for the kernel function when kernel is a callable or when additional parameters are needed.

Attributes
- dual_coef_: ndarray of shape (n_samples,) or (n_samples, n_targets)
  The learned coefficients in kernel space. These are used to make predictions via K(X', X) @ dual_coef_.

- X_fit_: ndarray or sparse matrix of shape (n_samples, n_features)
  Training data used for fitting. If kernel == "precomputed", this is the precomputed kernel matrix of shape (n_samples, n_samples).

- n_features_in_: int
  Number of features seen during fitting.

- feature_names_in_: ndarray of shape (n_features_in_,)
  Feature names seen during fitting (available when X has feature names).

Notes
- The kernel must be compatible with pairwise_kernels for non-callable kernels. When using a non-callable kernel, gamma, degree, and coef0 control kernel behavior as in standard kernel definitions.
- If kernel is "precomputed", you must provide a training kernel matrix in X and pass X_test as a kernel matrix between test samples and training samples.
- KernelRidge is particularly convenient for medium-sized datasets where a closed-form solution is preferable to iterative solvers. It yields dense models and is typically faster than SVR for such data, but does not produce sparse models.

Examples
Basic usage with a linear kernel
>>> from sklearn.kernel_ridge import KernelRidge
>>> import numpy as np
>>> n_samples, n_features = 10, 5
>>> rng = np.random.RandomState(0)
>>> y = rng.randn(n_samples)
>>> X = rng.randn(n_samples, n_features)
>>> krr = KernelRidge(alpha=1.0)
>>> krr.fit(X, y)
KernelRidge(alpha=1.0)
>>> y_pred = krr.predict(X)

Using an RBF kernel
>>> krr_rbf = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.5)
>>> krr_rbf.fit(X, y)
KernelRidge(alpha=1.0, kernel='rbf', gamma=0.5)

Precomputed kernel
>>> K = np.dot(X, X.T)  # example precomputed kernel (not recommended in practice)
>>> krr_precomputed = KernelRidge(alpha=1.0, kernel="precomputed")
>>> krr_precomputed.fit(K, y)
KernelRidge(alpha=1.0, kernel='precomputed')
>>> K_test = np.dot(np.random.randn(4, n_features), X.T)
>>> # For precomputed, provide K_test = K(X_test, X_train)
>>> y_pred = krr_precomputed.predict(K_test)

Implementation details (internal)
- The solution uses a Cholesky-based solver for the kernel-regularized linear system, via _solve_cholesky_kernel.
- The implementation supports multi-output targets by solving the regressor for each target column in y separately.

Examples from the module
- The class docstring includes a concise example showing initialization, fitting, and a simple invocation:
  >>> from sklearn.kernel_ridge import KernelRidge
  >>> import numpy as np
  >>> n_samples, n_features = 10, 5
  >>> rng = np.random.RandomState(0)
  >>> y = rng.randn(n_samples)
  >>> X = rng.randn(n_samples, n_features)
  >>> krr = KernelRidge(alpha=1.0)

  KernelRidge(alpha=1.0)
