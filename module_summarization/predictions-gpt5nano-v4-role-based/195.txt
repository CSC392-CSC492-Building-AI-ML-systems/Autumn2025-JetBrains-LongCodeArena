Learn Gaussian Mixture Models
=============================

Overview
--------
Gaussian Mixture Models (GMMs) describe data as arising from a mixture of Gaussian components. They provide soft assignments (responsibilities) of samples to components and are trained with the Expectation-Maximization (EM) algorithm. The base class for mixture models exposes a common API and shared EM-based training behavior, while specific implementations (e.g., with different covariance structures) implement problem-specific initialization and parameter updates.

Key concepts:
- n_components: number of Gaussian components in the mixture.
- EM algorithm: alternates between estimating responsibilities (E-step) and updating parameters (M-step) to maximize the likelihood or its lower bound.
- tol and max_iter: convergence control (stop when change in log-likelihood/lower bound falls below tol or after max_iter iterations).
- init_params: how to initialize the responsibilities before EM begins.
- reg_covar: added regularization to covariance matrices to ensure numerical stability.
- n_init and warm_start: how many random restarts to perform and whether to reuse previous results.

API reference (base class)
--------------------------
The core functionality is provided by the BaseMixture class. Derived classes implement problem-specific parameter updates, while the base class handles initialization, fitting, and validation.

Parameters
- n_components (int, >= 1): number of mixture components.
- tol (float, > 0): convergence threshold for the EM algorithm.
- reg_covar (float, >= 0): regularization term added to diagonal of covariances to ensure positive-definiteness.
- max_iter (int, >= 0): maximum number of EM iterations per run.
- n_init (int, >= 1): number of initializations to perform; the best solution is kept.
- init_params (str): method to initialize the responsibilities. Options:
  - "kmeans": initialize using a single run of KMeans (one initialization step).
  - "random": random responsibilities, normalized per sample.
  - "random_from_data": select random samples as initial centers.
  - "k-means++": initialize centers using k-means++ strategy.
- random_state (int, RandomState, or None): seed/PRNG for initialization.
- warm_start (bool): if True, reuse the solution from the previous call to fit and continue training.
- verbose (int): amount of verbosity during fitting.
- verbose_interval (int): how often to print progress when verbose > 0.

Initialization options
- kmeans: uses KMeans (n_components, n_init=1) to generate a hard assignment, which is converted to initial responsibilities.
- random: creates a random soft assignment and normalizes it per sample.
- random_from_data: selects n_components samples from the data and uses them as one-hot initial responsibilities.
- k-means++: initializes centers with k-means++ and converts to initial responsibilities.

Training flow
- The fit method delegates to fit_predict for compatibility with the estimator API.
- fit_predict validates input X and runs the EM procedure across n_init initializations, selecting the solution with the best (log-likelihood or lower bound).
- Each EM run alternates:
  - E-step: compute responsibilities (soft assignments) given current parameters.
  - M-step: update mixture weights, means, and covariances based on responsibilities.
- Convergence is declared when the change in the objective is less than tol within max_iter iterations.
- If convergence is not achieved within max_iter, a ConvergenceWarning is raised.
- If warm_start is True, subsequent calls resume training from the current state.

Implementation notes
- _check_shape(param, param_shape, name) validates the shape of input parameters to ensure correctness.
- _initialize_parameters handles initialization based on init_params and eventually calls _initialize(X, resp), which must be implemented by derived classes to set up the model-specific parameters.
- _check_parameters(X) is an abstract method that derived classes must implement to validate any problem-specific constraints before fitting.
- The code uses _validate_data to ensure X is numeric and has sufficient samples for fitting.

Practical usage
- Typical workflow:
  - Instantiate a Gaussian Mixture model (a concrete subclass of BaseMixture, e.g., GaussianMixture with a chosen covariance type).
  - Call .fit(X) to train the model (potentially with multiple inits if n_init > 1).
  - Use .predict(X) to obtain the most probable component labels for each sample.
  - Optionally use .predict_proba(X) to obtain the full posterior probabilities (responsibilities) for each sample.
  - Inspect learned parameters (weights, means, covariances) via model attributes.

Example
- Quick start with a common Gaussian Mixture implementation:
  from sklearn.mixture import GaussianMixture
  import numpy as np

  # X is a 2D array of shape (n_samples, n_features)
  gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=0, init_params='kmeans')
  gmm.fit(X)
  labels = gmm.predict(X)
  probs = gmm.predict_proba(X)  # optional

Notes
- The base class focuses on the general EM-based learning of mixtures. Derived classes implement the specifics of updating parameters for different covariance structures (e.g., full, tied, diag, spherical) and any additional constraints.
- The helper _check_shape ensures parameters supplied to the model (where applicable) have the expected shapes, reducing runtime errors during fitting.

References
- For more detailed usage and advanced options, consult the documentation for the concrete Gaussian Mixture implementation in your environment (e.g., GaussianMixture in scikit-learn or equivalent), including notes on convergence behavior and initialization choices.
