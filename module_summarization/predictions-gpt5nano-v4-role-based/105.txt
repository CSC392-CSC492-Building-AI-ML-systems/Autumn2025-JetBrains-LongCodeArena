g-functions
===========

Overview
--------
The g-functions module is a dynamic documentation generator used by the project to present the Meijer G-function-based formulas discovered at runtime. When the module is imported, it builds a large, Sphinx-friendly reStructuredText string describing the elementary functions and the more complex functions that involve additional operands. This generated content is assigned to the module’s __doc__ attribute, so Sphinx can render it as part of the documentation.

Purpose
-------
- Provide auto-generated, readable documentation for the set of functions discovered by the internal lookup mechanism.
- Distinguish between elementary functions and compound functions (those involving other expressions).
- Display each function as a LaTeX math block, with an optional conditional clause rendered in LaTeX.

How it works (high level)
--------------------------
- The module imports a lookup-building function _create_lookup_table and initializes a mapping t of the form:
  t: dict[tuple[type, ...], list[typing.Any]]
  Each key is a tuple of types (the “about” key), and each value is a list of entries (the category).
- _create_lookup_table populates t with entries describing formulas, their generators (gs), any conditions (cond), and a hint.
- The code iterates over t in a sorted order (using default_sort_key) and builds a reStructuredText string:
  - If about == (), a top-level section titled "Elementary functions:" is created.
  - Otherwise, a section titled "Functions involving <names>:" is created, where the names are derived by inspecting the first formula in the category and extracting function names via atoms(func) and LaTeX rendering.
- For each entry (formula, gs, cond, hint) in a category:
  - If gs is not a list, the generated expression g is marked as generated using Symbol('\\text{generated}').
  - If gs is a list, g is constructed as an additive combination Add(*[fac*f for (fac, f) in gs]).
  - The entry is represented as an equation Eq(formula, g) and rendered in LaTeX.
  - If cond is True, no condition is shown; otherwise, a conditional clause is appended as LaTeX: ,\text{ if } <latex(cond)>.
  - Each entry becomes a Sphinx math block:
    .. math::
      <latex(Eq(...))><cond>
- The final string is assigned to __doc__ so that importing the module exposes the generated documentation.

Data model (internal)
---------------------
- t: A defaultdict(list) mapping an “about” key to a list of category entries.
- about: A tuple of types describing the context of the formulas. about == () indicates elementary functions.
- category: A list of entries for a given about key. Each entry is a 4-tuple:
  - formula: The target formula (a symbolic expression) to be presented.
  - gs: Either a list of (factor, function) pairs used to build g, or a non-list value indicating a generated form.
  - cond: A condition under which the formula holds. May be True (no condition) or a LaTeX-renderable expression.
  - hint: Optional additional information (not explicitly rendered in the excerpted code, but available for future use).

Public API surface (as exposed by this module)
----------------------------------------------
- __doc__: The dynamically generated documentation string (RST) produced during import.
- t: Internal data structure populated by _create_lookup_table (not part of the public API, but useful for understanding the content).

Notes and considerations
------------------------
- The documentation content is data-driven and dependent on the data produced by _create_lookup_table at import time.
- The LaTeX rendering relies on the project’s latex printer to convert Python/SymPy objects into LaTeX.
- If gs is not a list, the generated expression is labeled as generated: Symbol('\\text{generated}').
- The output is designed for Sphinx to render as a collection of math blocks, with sections grouping related functions.

Usage examples (illustrative)
-----------------------------
- Importing the module causes the documentation string to be generated:
  - from g_functions import __doc__  or simply viewing the module in Sphinx will render the content.
- The generated sections may resemble:
  - Elementary functions:
    .. math::
      <latex(Eq(formula, g))>
  - Functions involving `<functionName>`:
    .. math::
      <latex(Eq(formula, g))><cond>

Illustrative template of generated content (actual content varies at runtime)
----------------------------------------------------------------------------
Elementary functions:

.. math::
  <latex(Eq(formula_for_an_elementary_function, g_for_elementary))>

Functions involving <SomeFunction>:

.. math::
  <latex(Eq(formula_for_function, g)>)(, \text{ if } <latex(condition)>)

Notes
-----
- This is a documentation-generation mechanism; the actual content depends on the runtime data produced by the lookup table builder.
- For comprehensive details, refer to the underlying components:
  - meijerint._create_lookup_table
  - g-functions' latex printer
  - default_sort_key for stable ordering
