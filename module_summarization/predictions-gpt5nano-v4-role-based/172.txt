SANS to SESANS conversion
=========================

Overview
--------
This module provides the tools and workflow to convert small-angle neutron
scattering (SANS) data, I(q), into SESANS (spin-echo SANS) representations,
specifically the SESANS polarization P(y) as a function of the spin-echo
displacement y. The conversion path leverages standard Fourier-transform
relationships between the SANS scattering length density correlations and the
SESANS polarization signal.

Key ideas
---------
- SANS data I(q) encodes information about the pair-distance distribution and
  the two-point correlation function of the scattering length density.
- The SESANS signal P(y) is related to the projected two-point correlation
  function along the beam direction and can be obtained by transforming the
  real-space correlation function.
- A typical conversion pipeline uses:
  1) an indirect Fourier transform (IFT) or equivalent regularized inversion to
     obtain the pair-distance distribution p(r) from I(q),
  2) a cosine transform (Fourier-like) of p(r) to produce the SESANS polarization
     P(y).

Data inputs
-----------
- SANS data I(q): intensity as a function of momentum transfer q.
- Optional metadata: wavelength, sample thickness, instrumental constants,
  background and normalization information.
- Optional prior constraints for the IFT (e.g., maximum particle size Rmax,
  smoothness, non-negativity).

Outputs
-------
- SESANS polarization P(y) as a function of y (the SESANS displacement).
- Optional diagnostic plots or intermediate representations (e.g., p(r) from I(q)).

How the conversion works (conceptual workflow)
----------------------------------------------
- Step 1: Preprocess I(q)
  - Background subtraction, normalization, unit alignment.
- Step 2: Obtain p(r) from I(q)
  - Use an indirect Fourier transform or equivalent regularized inversion to
    estimate the pair-distance distribution p(r) from the SANS data I(q).
- Step 3: Compute P(y) from p(r)
  - Apply a cosine-type transform to p(r) to yield the SESANS polarization:
    P(y) ∝ ∫ p(r) cos(y r) dr over the accessible r-range.
- Step 4: Postprocess
  - Optional smoothing, uncertainty propagation, and unit handling.
  - Save P(y) with associated y-values and metadata.

Practical considerations
------------------------
- Noise and regularization: Inverting I(q) to p(r) can amplify noise; regularization and
  physically motivated constraints are important for stable results.
- Rmax and data range: The choice of maximum radius Rmax (or equivalent constraint)
  affects resolution and fidelity of P(y). Use physically motivated limits based on
  the sample.
- Units and conventions: Keep q in inverse length units (e.g., 1/Å), r in length units
  (e.g., Å), and y in the SESANS displacement units used by your instrument.
- Model assumptions: The accuracy of the SESANS P(y) signal depends on the
  assumptions embedded in the I(q) interpretation (e.g., isotropy, monodispersity,
  and particle-homogeneity). If these are not met, consider model-based SESANS
  fitting instead of a purely transform-based approach.

API and usage notes
-------------------
- The module exposes a workflow-oriented interface intended for use with SANS
  datasets and SESANS calculations. Typical usage involves:
  - loading SANS data (q, I(q))
  - performing an indirect Fourier transform to obtain p(r)
  - transforming p(r) to P(y)
  - exporting the SESANS results (y, P(y)) for comparison with SESANS experiments or for
    further modeling
- The implementation relies on numerical routines from numpy and scipy
  for transforms, optimization, and regularization.

Example workflow (conceptual)
-----------------------------
- Load I(q) data and metadata
- Preprocess: subtract background, normalize
- Compute p(r) using an indirect Fourier transform with a chosen Rmax
- Compute P(y) from p(r) via a cosine transform
- Save P(y) and accompanying y-values to an output file
