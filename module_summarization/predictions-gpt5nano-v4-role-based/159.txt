Memory Alignment
================

Overview
--------

Memory alignment refers to how data is laid out in memory with respect to natural boundaries (e.g., 2-, 4-, 8-, or 16-byte boundaries). Proper alignment can improve performance, enable certain SIMD/vectorized instructions, and ensure compatibility with C extensions and interop layers.

In NumPy, alignment considerations arise from:
- The dtype’s alignment requirements (dtype.alignment)
- Structured dtypes that request C-struct-like padding (align=True)
- Build- and run-time CPU features that enable vectorized paths and alignment assumptions
- Interoperability with C through buffers, ctypes, and external libraries

What determines alignment in NumPy builds
----------------------------------------

NumPy captures system information and CPU feature data at build time, which informs how memory is laid out and which optimization paths are enabled. The alignment-related configuration is exposed via the internal CONFIG structure, including:
- Machine Information: host and build CPU characteristics and endianness
- SIMD Extensions: a baseline set of CPU features and the features discovered (baseline vs. found)

The project provides a runtime show() utility to inspect this information:
- show(mode="stdout") prints a YAML (or JSON-backed) summary of the configuration
- show(mode="dicts") returns a Python dictionary with the same information

Example:
- Imported as alignment (or the module providing show()):
  - alignment.show(mode="dicts") -> dict with keys such as "Machine Information" and "SIMD Extensions"
  - alignment.show(mode="stdout") -> human-readable output (or a YAML representation if PyYAML is installed)

Practical guidance for developers
---------------------------------

1) Check dtype alignment
- Use dtype.alignment to determine the alignment requirement for a dtype.
  Example:
  - dt = np.dtype(np.float64)
  - print(dt.alignment)  # typically 8 on most platforms

- For structured dtypes, alignment is the maximum alignment of the fields. You can request C-struct-style alignment using align=True:
  Example:
  - dt = np.dtype([("a", np.int8), ("b", np.int64)], align=True)
  - print(dt.alignment)  # will reflect the required struct alignment (often 8)

2) Interoperability with C and memory buffers
- When sharing memory with C code, ensure the memory layout matches expectations of the receiving side. Using aligned structured dtypes (align=True) helps align padding to C-struct rules.
- ctypes interoperability can leverage aligned buffers from NumPy arrays:
  - Use numpy.ctypeslib.as_ctypes(array) to obtain a C-compatible pointer when the dtype alignment matches the C struct.

3) SIMD considerations
- NumPy’s build may enable SIMD-accelerated code paths based on CPU features. These paths often assume properly aligned data to maximize throughput and avoid faults on certain architectures.
- Inspecting the run-time SIMD capabilities via the CONFIG “SIMD Extensions” section helps you understand which vectorized paths are available and what alignment assumptions they make.

4) Inspecting alignment-related information at runtime
- To understand what your current NumPy build expects/affords, view the runtime configuration:
  - import alignment  # module containing show()
  - alignment.show(mode="dicts")
  - Look under:
    - "Machine Information" for host/build CPU characteristics
    - "SIMD Extensions" for baseline vs. found features that influence alignment and vectorization decisions

Code-derived examples
---------------------

- Checking a simple dtype’s alignment:
  - import numpy as np
  - dt = np.dtype(np.float64)
  - print(dt.alignment)  # e.g., 8

- Forcing C-struct alignment in a structured dtype:
  - dt = np.dtype([("x", np.int32), ("y", np.float64)], align=True)
  - print(dt.alignment)  # typically reflects the maximum alignment of fields (often 8)

- Inspecting build/run alignment configuration:
  - import alignment
  - config = alignment.show(mode="dicts")  # returns a dict with "Machine Information" and "SIMD Extensions"
  - baseline = config["SIMD Extensions"]["baseline"]
  - found = config["SIMD Extensions"]["found"]

Notes and limitations
---------------------

- Exact alignment values depend on the platform and the NumPy build. Use dtype.alignment for portable checks on a given platform.
- The alignment information exposed at runtime is influenced by CPU features available to the compiled NumPy binary (as indicated by the CONFIG data).
- When creating or interfacing with external memory buffers, ensure your buffers meet the alignment requirements of your dtype and any consuming C/ASM code to avoid performance penalties or correctness issues.
