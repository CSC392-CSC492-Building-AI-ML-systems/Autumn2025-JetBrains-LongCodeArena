w_kinavg
========

Deprecation
-----------

This command is deprecated. It delegates to w_direct with the trace subcommand. Please use w_direct instead (e.g., w_direct trace ...).

Overview
--------

w_kinavg is a wrapper that runs the direct kinetics averaging tool in the trace mode. It computes average state-to-state kinetics and associated confidence intervals from weighted-ensemble data. It relies on pre-generated inputs (assignments and kinetics data) and produces an output file containing averaged rates (and, in trace mode, flux-related quantities) along with bootstrap-based confidence information.

Key defaults introduced by w_kinavg
------------------------------------

- Default kinetics input file: kintrace.h5
- Default output file: kinavg.h5

Command structure
-----------------

w_kinavg acts as a master command that exposes a single subcommand:

- trace (provided by WKinAvg)
  - Description: averages and confidence intervals for path-tracing kinetics analysis
  - Implemented via the WKinAvg class, which inherits from the underlying DKinAvg driver

The subcommand is selected by:
- w_kinavg trace

The trace subcommand is designed to be compatible with the broader westpa tooling (w_direct) and shares the same data and options framework, while using the defaults above.

Output format
-------------

The output file (default kinavg.h5, overridable with -o/--output) contains datasets describing kinetics averaged over the chosen window of iterations. The exact datasets depend on whether you are in plain mode or in trace mode, and whether evolution-mode is requested.

Common datasets (present in both plain and trace modes)
- /avg_rates [state, state]
  -Structured dataset: state-to-state average rates over the selected iterations

Trace-mode additional datasets
- /avg_total_fluxes [state]
  -Total flux into each state over the selected window

- /avg_conditional_fluxes [state, state]
  -State-to-state fluxes conditioned on the current window

Evolution-mode datasets (if --evolution-mode is specified)
- /rate_evolution [window, state, state]
  -State-to-state rates as a function of a variable-width window
  -Window semantics:
    - cumulative: windows begin at --start-iter and grow by --step-iter
    - blocked: all windows have width --step-iter (except possibly the last), starting at --start-iter

Trace-mode evolution datasets
- /target_flux_evolution [window, state]
  -Total flux into a macro state with varying window widths (as in /rate_evolution)

- /conditional_flux_evolution [window, state, state]
  -State-to-state conditional fluxes with varying window widths (as in /rate_evolution)

Dataset structure
- iter_start
  - Integer: start iteration of the averaging window (inclusive)

- iter_stop
  - Integer: end iteration of the averaging window (exclusive)

- expected
  - Floating-point: mean rate within the window, in inverse-time units

- ci_lbound
  - Floating-point: lower bound of the confidence interval for the rate within the window

- ci_ubound
  - Floating-point: upper bound of the confidence interval for the rate within the window

- corr_len
  - Integer: estimated correlation length of the rate within the window, in units of tau

Attributes (associated with each dataset)
- mcbs_alpha
  - Floating-point: alpha for the bootstrap confidence interval (e.g., alpha=0.05 for 95% CI)

- mcbs_nsets
  - Integer: number of bootstrap resamples used

- mcbs_acalpha
  - Floating-point: alpha for determining correlation lengths

Command-line options
--------------------

w_kinavg defers to the same option surface used by the underlying w_direct/ WKinAvg implementation. The documented options in practice include:

- -o, --output PATH
  - Output file (default kinavg.h5)

- --evolution-mode [none|cumulative|blocked]
  - Enable computation of rate evolution datasets over windows of varying width
  - If not specified, only the base /avg_rates (and trace-mode base datasets) are produced

- --start-iter N
  - Iteration at which the averaging window begins (inclusive)

- --step-iter N
  - Window increment size (for cumulative mode) or fixed window width (for blocked mode)

Notes
-----

- The command is a backward-compatible shim intended to ease migration to w_direct. The underlying logic and data formats are defined by the DKinAvg/w_direct tooling.
- For a full list of options and their meanings, refer to the w_direct/kinetics tooling documentation and the DKinAvg component documentation.

Examples
--------

Basic usage (default inputs, plain averaging)
- w_kinavg trace -o kinavg.h5

Evolution-mode with cumulative windows
- w_kinavg trace --evolution-mode cumulative --start-iter 0 --step-iter 50 -o kinavg.h5

Trace-mode with blocked windows
- w_kinavg trace --evolution-mode blocked --start-iter 100 --step-iter 25 -o kinavg.h5

Notes for maintainers
---------------------

- The WKinAvg class provides the trace subcommand and sets:
  - subcommand = 'trace'
  - default_kinetics_file = 'kintrace.h5'
  - default_output_file = 'kinavg.h5'
- The entry_point() function issues a deprecation warning and forwards to WDirect, ensuring backward compatibility while guiding users toward the newer interface.
