Contributing to Qiskit Metal
============================

This guide explains how to contribute to Qiskit Metal and describes the architecture related to CPW analyses in particular. It is intended for developers who want to add new analyses, improve existing ones, or contribute documentation and tests.

Overview
--------
Qiskit Metal is an open-source project licensed under the Apache License, Version 2.0. Contributions are welcome from the community. This guide covers:

- Project structure related to analyses (CPW calculations and related computations)
- How to add new analyses or extend existing ones
- Documentation, testing, and coding standards
- How to submit changes via pull requests

Key modules for CPW-related analyses
-------------------------------------
Within the analyses package, there are submodules responsible for planar CPW (coplanar waveguide) parameter calculations and related electrical characterizations:

- CPW calculations (analyses.em.cpw_calculations)
  - Provides functions to compute guided wavelength and lumped-parameter equivalents for CPW structures.
  - Example functions:
    - guided_wavelength(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45)
      - Returns lambdaG (guided wavelength), etfSqrt (effective dielectric constant, accounting for film thickness), and q (filling factor).
    - lumped_cpw(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45, loss_tangent=1e-5, london_penetration_depth=30e-9)
      - Returns Lk, Lext, C, G, Z0, etfSqrt**2, and Cstar, representing the lumped-element equivalents and characteristic impedance.
- Lumped-capacitive analysis (analyses.quantization.lumped_capacitive)
  - Provides lumped-element representations for CPW structures in a quantum-device context.

Usage notes
-----------
- The analyses package exposes submodules via:
  - qiskit_metal.analyses.em.cpw_calculations
  - qiskit_metal.analyses.quantization.lumped_capacitive
- Typical usage (illustrative):
  - CPW guided wavelength:
    - from qiskit_metal.analyses.em import cpw_calculations
    - lambdaG, etfSqrt, q = cpw_calculations.guided_wavelength(
        freq=5e9,
        line_width=10e-6,
        line_gap=6e-6,
        substrate_thickness=760e-6,
        film_thickness=200e-9,
        dielectric_constant=11.45
      )
  - Lumped CPW parameters:
    - from qiskit_metal.analyses.quantization import lumped_capacitive
    - Lk, Lext, C, G, Z0, etfSqrt2, Cstar = lumped_capacitive.lumped_cpw(
        freq=5e9,
        line_width=10e-6,
        line_gap=6e-6,
        substrate_thickness=760e-6,
        film_thickness=200e-9,
        dielectric_constant=11.45,
        loss_tangent=1e-6,
        london_penetration_depth=30e-9
      )

Contribution guidelines
-----------------------
1. Find or propose an issue
   - Check the issue tracker for open enhancements, bug fixes, or feature requests.
   - Propose new analyses or improvements with a clear scope and expected impact.

2. Create a feature branch
   - Use a descriptive branch name, e.g., feature/cpw-extended-analyses or fix/docs-contributor-guide.

3. Implement changes
   - Add new modules under the appropriate package (e.g., qiskit_metal/analyses/em or qiskit_metal/analyses/quantization).
   - Document new functions with clear docstrings (type hints preferred).
   - Update autosummary and reference sections in Sphinx docs as needed.

4. Testing
   - Add unit tests for new functionality.
   - Run the project tests locally (pytest) and ensure existing tests pass.

5. Documentation
   - Provide or update reST documentation:
     - Document new modules and functions with param and return value descriptions.
     - Use autosummary for new modules to appear in the API docs.
   - Update contributor guides if new conventions are introduced.

6. Submit a PR
   - Include a concise description of changes, motivation, and any trade-offs.
   - Reference related issues or design notes.
   - Ensure tests pass on CI and docs build succeeds.

Code quality and standards
--------------------------
- Follow the project’s style and conventions:
  - Python style (PEP 8) and descriptive docstrings.
  - ReST-compatible docstrings with clear parameter sections and return values.
  - Use numpy/scipy where appropriate; include necessary imports and handling of edge cases.
- Ensure backward compatibility where feasible.
- Keep public APIs stable; deprecate with proper release notes if necessary.

Documentation expectations
--------------------------
- Each public function in CPW analyses should have a docstring describing:
  - Purpose
  - Parameters (types and units)
  - Returns (types and meaning)
  - Any side effects or exceptions
  - Examples (optional, but encouraged)
- The contributor guide itself should be clear about:
  - How to locate relevant modules (analyses subpackages)
  - How to extend the analyses (coding and testing workflow)
  - How to document new functionality in Sphinx

Development environment setup
-----------------------------
- Clone the repository and install in editable mode:
  - pip install -e .
- Install test and documentation dependencies as described in the repository’s setup and docs requirements.
- Run tests:
  - pytest -k analyses
- Build docs (if you are contributing to API docs):
  - sphinx-build -b html docs build/html

Notes on the code architecture shown in this guide
-------------------------------------------------
- CPW-related computations reside in the analyses package:
  - analyses.em.cpw_calculations: guided_wavelength, lumped_cpw
  - analyses.quantization.lumped_capacitive: lumped_cpw (lumped-element model)
- The code uses standard numerical libraries (numpy, scipy) and relies on explicit physical constants (c0, e0, u0).
- Documentation style follows reST conventions with autosummary directives for automatic API documentation generation.

Example contribution scenario
-----------------------------
Goal: Add a new CPW parameter calculator for a different substrate or geometry.

Steps:
- Create new module under qiskit_metal/analyses/em/ (or the appropriate existing directory) e.g., new_cpw_extensions.py
- Implement functions with clear docstrings, using the same parameter conventions as guided_wavelength and lumped_cpw.
- Update __init__.py to expose the new module via from .em import new_cpw_extensions
- Add unit tests covering typical use cases and edge conditions.
- Update docs: add autosummary entry for the new module and provide short usage examples in the API docs.
- Open a PR with a brief description, tests, and documentation changes.

This guide provides the essential steps and architectural context for contributing to Qiskit Metal, with a focus on CPW-related analyses. For specific, up-to-date contributing practices, always refer to the project’s CONTRIBUTING.md and the PR template.
