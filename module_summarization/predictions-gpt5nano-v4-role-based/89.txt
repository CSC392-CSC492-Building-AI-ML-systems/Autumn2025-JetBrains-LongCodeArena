Augmentations for AutoAugment
=============================

Overview
--------
This module provides a standard suite of image augmentations used by AutoAugment policies
for ImageNet, RandAugment, and TrivialAugmentWide. The augmentations are implemented as
DALI operators and are exposed as decorated functions via the @augmentation decorator.

Key idea
--------
- The @augmentation decorator abstracts how the augmentation parameter is chosen.
- Each augmentation function expects a single positional argument (a batch of samples) and
  instead receives magnitude-related information through mag_range, randomly_negate, and
  mag_to_param. The final transform parameter is computed using mag_to_param applied to
  a magnitude drawn from a linspace over mag_range.
- You can create customized variants by re-exposing the augmentation with a different mag_range:
  my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))

Important notes
--------------
- The module depends on numpy for certain computations (notably inside contrast). If numpy is
  unavailable, importing this module raises a RuntimeError.
- The augmentations are designed to work with DALI’s environment (operating on batches of
  images) and leverage DALI operators such as transforms and warp_affine.
- Parameters passed to the decorated functions are automatically computed; callers provide
  magnitude_bin and num_magnitude_bins rather than the direct magnitude value.

Public API
----------
The following augmentations are defined in this module. Each is a decorated function; after
decoration, you call them with a batch and magnitude-bin related arguments.

- shear_x(data, shear, fill_value=128, interp_type=None)
  - Description: Shear transformation along the x-axis.
  - Internal parameter mapping: mag_to_param = warp_x_param -> [magnitude, 0]
  - mag_range: (0, 0.3)
  - randomly_negate: True
  - How to call: shear_x(batch, magnitude_bin=..., num_magnitude_bins=..., shear=..., fill_value=..., interp_type=...)

- shear_y(data, shear, fill_value=128, interp_type=None)
  - Description: Shear transformation along the y-axis.
  - Internal parameter mapping: mag_to_param = warp_y_param -> [0, magnitude]
  - mag_range: (0, 0.3)
  - randomly_negate: True
  - How to call: shear_y(batch, magnitude_bin=..., num_magnitude_bins=..., shear=..., fill_value=..., interp_type=...)

- translate_x(data, rel_offset, shape, fill_value=128, interp_type=None)
  - Description: Horizontal translation.
  - Internal parameter mapping: mag_to_param = warp_x_param -> [magnitude, 0]
  - mag_range: (0, 1)
  - randomly_negate: True
  - Shape-aware: offset is computed as rel_offset * shape[1]
  - How to call: translate_x(batch, magnitude_bin=..., num_magnitude_bins=..., rel_offset=..., shape=..., fill_value=..., interp_type=...)

- translate_x_no_shape(data, offset, fill_value=128, interp_type=None)
  - Alias for translate_x without shape dependency.
  - mag_range: (0, 250)
  - randomly_negate: True
  - Name override: name="translate_x"
  - How to call: translate_x_no_shape(batch, magnitude_bin=..., num_magnitude_bins=..., offset=..., fill_value=..., interp_type=...)

- translate_y(data, rel_offset, shape, fill_value=128, interp_type=None)
  - Description: Vertical translation.
  - Internal parameter mapping: mag_to_param = warp_y_param -> [0, magnitude]
  - mag_range: (0, 1)
  - randomly_negate: True
  - Shape-aware: offset is computed as rel_offset * shape[0]
  - How to call: translate_y(batch, magnitude_bin=..., num_magnitude_bins=..., rel_offset=..., shape=..., fill_value=..., interp_type=...)

- translate_y_no_shape(data, offset, fill_value=128, interp_type=None)
  - Alias for translate_y without shape dependency.
  - mag_range: (0, 250)
  - randomly_negate: True
  - Name override: name="translate_y"
  - How to call: translate_y_no_shape(batch, magnitude_bin=..., num_magnitude_bins=..., offset=..., fill_value=..., interp_type=...)

- rotate(data, angle, fill_value=128, interp_type=None, rotate_keep_size=True)
  - Description: Rotation.
  - mag_range: (0, 30)
  - randomly_negate: True
  - How to call: rotate(batch, magnitude_bin=..., num_magnitude_bins=..., angle=..., fill_value=..., interp_type=..., rotate_keep_size=...)

- brightness(data, parameter)
  - Description: Brightness adjustment.
  - Magnitude mapping: shift_enhance_range maps magnitude to [1, 2)
  - mag_range: (0, 0.9)
  - randomly_negate: True
  - How to call: brightness(batch, magnitude_bin=..., num_magnitude_bins=..., parameter=...)

- contrast(data, parameter)
  - Description: Contrast adjustment.
  - Magnitude mapping: shift_enhance_range maps magnitude to [1, 2)
  - mag_range: (0, 0.9)
  - randomly_negate: True
  - Additional details: center is computed from image mean with established RGB weights to align with PIL-style contrast behavior; then data is adjusted by center and parameter.
  - How to call: contrast(batch, magnitude_bin=..., num_magnitude_bins=..., parameter=...)

Utility
-------
- mag_range: Tuple (min, max) defining the magnitude search space.
- randomly_negate: If True, the computed parameter may be negated to apply inverse effect.
- mag_to_param: Callable that converts a scalar magnitude into the parameter(s) used by the underlying operator.

Examples
--------
- Create a broader magnitude range for a shear_x augmentation:
  my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))

- Apply a single augmentation to a batch with a specific magnitude bin:
  result = my_shear_x(batch, magnitude_bin=2, num_magnitude_bins=5, shear=0.0)

- Use translate_x_no_shape with a fixed offset:
  result = translate_x_no_shape(batch, magnitude_bin=1, num_magnitude_bins=4, offset=10)

Notes
-----
- The module uses numpy for numerical operations in some augmentations (notably contrast). Ensure numpy is installed when using these augmentations.
- All augmentations operate on batches and rely on DALI operators for performance on accelerators.

Implementation highlights
------------------------
- The core behavior is driven by the @augmentation decorator, which computes the actual transform parameter from a magnitude bin and mag_range.
- The module exposes a compact and interchangeable set of augmentations suitable for AutoAugment-style policies, while allowing easy customization through the augmentation API.
