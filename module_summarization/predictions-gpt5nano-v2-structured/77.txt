Transforms
==========

This module provides utilities for image handling and conversion used by image processing
transforms. It includes an interpolation mode enumeration, helpers to query image
dimensions and sizes, and conversion utilities to turn PIL Images or NumPy arrays into
PyTorch tensors.

InterpolationMode
-----------------

Enum of supported interpolation modes. These values map to corresponding methods used by
image resampling operations.

- NEAREST: "nearest"
- NEAREST_EXACT: "nearest-exact"
- BILINEAR: "bilinear"
- BICUBIC: "bicubic"
- BOX: "box" (PIL compatibility)
- HAMMING: "hamming"
- LANCZOS: "lanczos"

Usage
```
from transforms import InterpolationMode
mode = InterpolationMode.BILINEAR
```

Helper mappings
- _interpolation_modes_from_int(i: int) -> InterpolationMode
  Converts an integer code to an InterpolationMode.
- pil_modes_mapping
  Mapping from InterpolationMode to the corresponding PIL resampling constant.

Image property query helpers
----------------------------

- get_dimensions(img: Tensor or PIL Image) -> List[int]
  Returns image dimensions as [channels, height, width]. Accepts a PyTorch Tensor
  (C x H x W) or a PIL Image.

- get_image_size(img: Tensor or PIL Image) -> List[int]
  Returns image size as [width, height].

- get_image_num_channels(img: Tensor or PIL Image) -> int
  Returns the number of channels in the image.

Notes:
- These helpers log API usage once per function call (when not scripted/traced).
- For PIL inputs, the results reflect the PIL-based dimensions.

Conversion utilities
--------------------

- to_tensor(pic) -> Tensor
  Converts a PIL Image or NumPy ndarray to a PyTorch tensor.
  - Does not support TorchScript.
  - Accepts PIL Images or NumPy arrays (2D or 3D).
  - For NumPy arrays:
    - 2D inputs are interpreted as a single-channel image (added as a channel).
    - The output is in CHW format (C, H, W).
  - For uint8 inputs (common for images), values are scaled to [0, 1].
  - If accimage is available and the input is an accimage.Image, a corresponding tensor is produced.
  - For PIL Images:
    - Non-floating inputs are converted to the appropriate NumPy type, then to a tensor.
    - The resulting tensor is in CHW format and scaled to [0, 1] for uint8 inputs.

- pil_to_tensor(pic: Any) -> Tensor
  Converts a PIL Image to a tensor of the same type and performs a deep copy of the underlying array.
  - This function does not support TorchScript.

Examples
--------

Convert a NumPy array to a PyTorch tensor
```python
import numpy as np
import torch
from transforms import to_tensor

# H x W x C numpy array (uint8)
arr = np.zeros((64, 64, 3), dtype=np.uint8)
arr[16:48, 16:48] = [255, 0, 0]

t = to_tensor(arr)
print(t.shape)  # torch.Size([3, 64, 64])
print(t.dtype)  # torch.float32
print(t.min().item(), t.max().item())  # 0.0 1.0
```

Convert a PIL Image to a tensor
```python
from PIL import Image
from transforms import to_tensor

img = Image.new('RGB', (32, 32), color=(0, 128, 255))
t = to_tensor(img)
print(t.shape)  # torch.Size([3, 32, 32])
print(t.dtype)  # torch.float32
```

Inspect image properties from a tensor
```python
import torch
from transforms import get_dimensions, get_image_size, get_image_num_channels

x = torch.randn(3, 128, 64)

print(get_dimensions(x))          # e.g., [3, 128, 64]
print(get_image_size(x))            # e.g., [64, 128]
print(get_image_num_channels(x))    # 3
```
