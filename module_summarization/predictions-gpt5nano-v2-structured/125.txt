# FITS Headers

FITS headers are the metadata accompanying data in a FITS (Flexible Image Transport System) file. They are made up of a sequence of 80-character ASCII "cards" that describe the data contained in the corresponding HDU (Header/Data Unit). Each card typically contains a keyword, a value, and an optional comment. The header’s primary role is to document the data, its origin, units, coordinate systems, processing steps, and other provenance information.

Key points
- A FITS header is mandatory for each primary HDU and any subsequent HDU with data.
- Cards are fixed-width (80 characters) and the entire header must occupy a multiple of 2880 bytes.
- Keywords are conventionally written in uppercase; many keywords have standard meanings (see common keywords).
- Values can be numeric, boolean (T/F), or strings (strings are enclosed in quotes).
- Comments and history can be added using the COMMENT and HISTORY cards to aid reproducibility and provenance.

## Structure of a FITS header

- Each header card is 80 characters long.
- A typical card has the form:
  - KEYWORD = value / comment
- Examples:
  - SIMPLE  =                    T / file conforms to FITS standard
  - NAXIS   =                    2 / number of data axes
  - NAXIS1  =                 1024 / length of data axis 1
  - NAXIS2  =                 768 / length of data axis 2
  - BITPIX  =                   16 / number of bits per data value

- Non-numeric cards include COMMENT and HISTORY, which do not have a value field but can have text appended.

- Cards may be continued by the software when long comments or values are present; many tools (including Astropy) provide convenient ways to handle long text across multiple cards.

## Data types and value formats

- Numeric values: integers and floating-point numbers (e.g., NAXIS, BITPIX).
- Strings: enclosed in single quotes, e.g., OBJECT = 'NGC 1234'.
- Booleans: T (true) or F (false), e.g., SIMPLE = T.
- Comments: introduced by a slash (/) after the value, e.g., NAXIS = 2 / number of data axes.
- Nulls: some fields may be left blank or use FITS-standard null representations depending on the context and convention.

Commonly used header keywords
- SIMPLE: Whether this is a standard FITS file.
- BITPIX: Number of bits per data value.
- NAXIS: Number of data axes.
- NAXISn: Length of axis n (e.g., NAXIS1, NAXIS2).
- EXTEND: Whether there are extensions beyond the primary HDU.
- DATE: File creation date.
- ORIGIN, CREATOR, AUTHOR: Provenance information about the data.
- TELESCOP, INSTRUME: Instrument details.
- OBJECT: Target object name.
- OBSERVER, OBSTIME: Observer information and observation time.
- COMMENT, HISTORY: Free-form notes about the data processing or provenance.
- CTYPE1, CRVAL1, CRPIX1, CDELT1 (and corresponding for other axes): World Coordinate System (WCS) keywords describing the mapping from pixel to world coordinates.
- CRVALn, CTYPEn, CDELTn, CRPIXn, PC or CD (transformation keywords) for more advanced WCS representations.

Astropy-friendly usage
- Reading a header from a FITS file:
  - from astropy.io import fits
  - with fits.open('example.fits') as hdul:
  -     header = hdul[0].header
  -     print(header)
- Accessing a keyword value:
  - date = header['DATE']
  - naxes = header['NAXIS']
- Accessing a keyword comment:
  - comment = header.comments.get('DATE')
- Checking for a keyword:
  - if 'OBJECT' in header:
  -     obj = header['OBJECT']
- Adding or modifying keywords:
  - header['OBJECT'] = ('NGC 1234', 'Object name')
  - header['OBSERVER'] = 'Jane Doe'
- Adding comments or history entries:
  - header['COMMENT'] = "This file contains calibrated data."
  - header.add_history('Calibrated with pipeline v1.2')
- Deleting a keyword:
  - del header['OBSERVER']
- Writing data with a header:
  - hdu = fits.PrimaryHDU(data, header=header)
  - hdu.writeto('out.fits', overwrite=True)
  - or fits.writeto('out.fits', data, header=header, overwrite=True)

Code examples
- Read and print a header:
  - from astropy.io import fits
  - with fits.open('image.fits') as hdul:
  -     header = hdul[0].header
  -     for key in header.keys():
  -         print(f"{key} = {header[key]}  ({header.comments.get(key, '')})")

- Create a minimal header for a new FITS file:
  - from astropy.io import fits
  - import numpy as np
  - data = np.zeros((100, 100), dtype=np.float32)
  - header = fits.Header()
  - header['SIMPLE'] = True
  - header['BITPIX'] = -32
  - header['NAXIS'] = 2
  - header['NAXIS1'] = 100
  - header['NAXIS2'] = 100
  - header['DATE'] = ('2025-11-21', 'File creation date')
  - header['OBJECT'] = ('NGC 1234', 'Target object')
  - hdu = fits.PrimaryHDU(data, header=header)
  - hdu.writeto('new_image.fits', overwrite=True)

Best practices and notes
- Aim for descriptive, concise comments. Use COMMENT for notes that aid provenance; use HISTORY to record processing steps.
- Keep standard keywords in lowercase or uppercase as appropriate, but rely on the conventional uppercase form for compatibility.
- Be mindful of header size and alignment: the combined header must be a multiple of 2880 bytes.
- Use a header object (Astropy’s fits.Header) to manipulate metadata safely before writing to disk.
- When sharing data, ensure coordinate systems (WCS) are well-documented via CTYPE, CRVAL, CRPIX, CDELT, and related keywords to preserve scientific context.
- If you need to validate or repair headers (e.g., to fix non-standard entries or ensure compatibility), use the FITS verification facilities provided by your tooling of choice (Astropy offers header/file verification utilities). See your library’s documentation for exact usage and options.

Related concepts
- World Coordinate System (WCS) keywords (CTYPE*, CRVAL*, CRPIX*, CDELT*, etc.) for mapping pixel coordinates to world coordinates.
- HISTORY and COMMENT cards for provenance and notes.
- Extension headers when working with multi-extension FITS files (beyond the primary HDU).
- Data integrity and versioning considerations when editing headers.
