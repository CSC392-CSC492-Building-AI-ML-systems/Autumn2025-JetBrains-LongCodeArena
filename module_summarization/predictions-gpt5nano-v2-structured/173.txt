Fitting SESANS Data
===================

This document describes how to fit Spin-Echo Small-Angle Neutron Scattering (SESANS) data
to models provided by sasmodels. SESANS data typically come as a function of the spin-echo length
z (or a closely related SESANS kernel parameter) and are compared to model predictions to obtain
estimates of structural and contrast parameters.

Overview
--------

- SESANS data are related to the sample's scattering properties through a kernel that maps
  model form factors to the measured SESANS observable P(z).
- Fitting aims to estimate model parameters (e.g., size, shape, scattering length density contrasts,
  background) by minimizing the discrepancy between observed SESANS data and model predictions.
- Sasmodels provides a catalog of models. Each model exposes a set of kernel_parameters that participate
  in SESANS calculations. Parameters have names and “types” (e.g., volume, orientation, SLD) that can be
  inspected or filtered programmatically.

Data Format
-----------

Typical SESANS data file format (one row per data point):

- z: spin-echo length (or equivalent SESANS kernel parameter)
- Pz: measured SESANS observable
- dPz: uncertainty on Pz (optional)

Example (space- or tab-delimited):

0.5 0.92 0.03
1.0 0.85 0.04
1.5 0.70 0.05
...

Notes:

- If uncertainties are not available, you can still fit using equal-weights or an assumed error model.
- Normalization or background parameters may be included in the model depending on the data and instrumental setup.

Models and Parameters
---------------------

- Models are loaded from sasmodels. Each model defines a set of kernel_parameters used in SESANS
  calculations. Parameters can be filtered by type (volume, orientation, SLD, none, any) when listing.
- To discover which parameters are available and which models use them, you can list all parameters
  via the sasmodels helper:

  - Command: python -m sasmodels.list_pars [-v]

  - The -v (verbose) option lists, in addition to parameter names, the models that use each parameter.

- Example (no verbose): lists all parameter names across all models.
- Example (verbose): lists parameter names followed by the models that include each parameter.

Fitting Workflow
----------------

1. Prepare your data
   - Ensure data is in a simple text or CSV/TSV format with columns for z, Pz, and dPz (if available).
   - Optionally normalize or baseline-correct your data prior to fitting.

2. Choose a model
   - Inspect available models using sasmodels (e.g., by listing models or querying model info).
   - Select a model that plausibly represents your system (sphere, cylinder, core-shell, etc.).

3. Inspect parameters
   - Use the list_pars tool to understand which parameters the chosen model exposes and their types.

4. Configure initial parameters and constraints
   - Provide reasonable initial guesses for parameters.
   - Optionally fix parameters or set bounds (lower/upper) to guide the fit.
   - Decide which parameters should be varied during fitting.

5. Run the fit
   - Use the SESANS fitting routine (as exposed by sasmodels) to minimize the objective function
     (typically chi-squared) between the data and model predictions.
   - For more robust results, consider multiple starting points or global optimization techniques.

6. Evaluate results
   - Examine best-fit parameter values and uncertainties.
   - Assess goodness-of-fit via chi-squared, reduced chi-squared, residuals, and visual inspection
     of data vs. model overlay.
   - Check parameter correlations to identify poorly constrained parameters.

7. Advanced options
   - Global fitting: fit multiple SESANS data sets simultaneously with shared parameters.
   - Priors and constraints: apply physical bounds or priors to parameters.
   - Uncertainty estimation: bootstrap, MCMC, or other approaches to quantify parameter uncertainties.

Usage Notes
-----------

- The parameterization and the interpretation of fit results depend on the selected model. Always
  verify that the model is physically appropriate for your system.
- Some models expose many parameters; use sensible bounds and/or fix uninformative parameters to
  stabilize the fit.
- If your data include systematic uncertainties (e.g., background, normalization), include those as
  fit parameters or treat them separately in preprocessing.

Code Examples
-------------

The following illustrative Python-style example demonstrates the typical workflow conceptually. The exact
function names and APIs may differ in your sasmodels installation; consult the API reference for the exact calls.

.. code-block:: python

    import numpy as np
    # Load SESANS data
    data = np.loadtxt('sesans_data.dat')
    z, Pz, dPz = data[:,0], data[:,1], data[:,2]

    # Choose a model (e.g., Sphere)
    model = sasmodels.Model('sphere', initial_params={
        'radius': 50.0,      # nm
        'sld_core': 6e-6,    # 1e-6 A units or equivalent
        'sld_solvent': 2e-6,
        'background': 0.0
    })

    # Configure fitting (bounds, fixed parameters, etc.)
    model.set_bounds({'radius': (10.0, 200.0),
                      'background': (-0.01, 0.01)})

    # Run fit
    fit_result = sasmodels.fit(data=(z, Pz, dPz), model=model)

    print("Best-fit parameters:", fit_result.params)
    print("Chi-squared:", fit_result.chi2)

    # Optional: inspect residuals and goodness-of-fit
    residuals = Pz - fit_result.model_values
    # Plot data vs. model to visually assess fit
    # plot(z, Pz, label='Data')
    # plot(z, fit_result.model_values, label='Best-fit')

Appendix A: Parameter Listing
-----------------------------

- Use the parameter listing utility to discover available parameters across models and their usage.
- Command: python -m sasmodels.list_pars [-v]
- This returns a mapping of parameter names to the models that include them (verbose mode includes model lists).

Appendix B: Troubleshooting
---------------------------

- Non-convergence: try different initial parameter guesses, widen or tighten bounds, or fix poorly determined parameters.
- Inconsistent units: ensure units for radii, lengths, and SLDs match the expectations of the chosen model.
- Data quality: high noise or missing uncertainty information can degrade fit reliability; consider weighting or excluding problematic regions.
