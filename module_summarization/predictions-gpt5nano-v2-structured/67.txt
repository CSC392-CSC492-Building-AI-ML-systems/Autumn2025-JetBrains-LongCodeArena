Feature extraction for text
===========================

Overview
--------
The text feature extraction submodule provides utilities to build numeric feature
vectors from text documents. It includes vectorizers that convert raw text into
sparse matrices of token counts or term-weighted scores, as well as preprocessing
helpers for common text normalization tasks. The public API focuses on the core
tools used to prepare text data for machine learning:

- CountVectorizer
- TfidfVectorizer
- TfidfTransformer
- HashingVectorizer
- ENGLISH_STOP_WORDS
- Auxiliary utilities for text preprocessing:
  - strip_accents_unicode
  - strip_accents_ascii
  - strip_tags

Public API
----------
- HashingVectorizer: A fast, memory-efficient vectorizer that uses the hashing
  trick to map tokens to features without building a vocabulary.
- CountVectorizer: Converts a collection of text documents to a matrix of token
  counts.
- TfidfTransformer: Transforms a count matrix to a TF-IDF representation.
- TfidfVectorizer: Combines CountVectorizer and TF-IDF weighting in a single
  transformer.
- ENGLISH_STOP_WORDS: Built-in stop word list for English.
- strip_accents_unicode: Remove accents from unicode strings using a unicode
  normalization-based approach.
- strip_accents_ascii: Remove accents by transliterating to ASCII where possible.
- strip_tags: Basic HTML/XML tag stripper utility.

Key concepts
------------
- Preprocessing and tokenization
  - _preprocess: Chain optional steps like lowercasing and accent handling.
  - _analyze: Chain of preprocessor, tokenizer, and n-gram generation, with
    support for a supplied analyzer. If an analyzer is provided, the other
    steps may be bypassed.
- Stop words handling
  - _check_stop_list: Resolve stop words from built-ins (e.g., "english"),
    a provided collection, or None.
- Accent and tag normalization
  - strip_accents_unicode and strip_accents_ascii: Normalize accented characters.
  - strip_tags: Strip HTML/XML tags from text.
- Vectorization workflow
  - _VectorizerMixin: Shared logic for vectorizers, including input decoding
    and n-gram handling.
  - decode: Convert input documents to unicode strings, supporting input kinds
    such as filename, file-like objects, or raw strings.
  - _word_ngrams: Generate word-level n-grams with optional stop word filtering.

Usage examples
--------------

Basic CountVectorizer
.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = [
       "The quick brown fox jumps over the lazy dog.",
       "Never jump over the lazy dog quickly."
   ]

   vectorizer = CountVectorizer()
   X = vectorizer.fit_transform(docs)
   feature_names = vectorizer.get_feature_names_out()
   print(X.shape)           # (2, number_of_features)
   print(feature_names[:10])  # first 10 feature names

Tf-idf vectorization
.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   docs = [
       "The quick brown fox jumps over the lazy dog.",
       "Never jump over the lazy dog quickly."
   ]

   tfidf = TfidfVectorizer()
   X = tfidf.fit_transform(docs)
   print(X.shape)

Using English stop words
.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = ["This is a sample document. It contains several words."]
   vectorizer = CountVectorizer(stop_words="english")
   X = vectorizer.fit_transform(docs)
   print(X.shape)

HashingVectorizer usage
.. code-block:: python

   from sklearn.feature_extraction.text import HashingVectorizer

   docs = [
       "The quick brown fox",
       "jumps over the lazy dog"
   ]

   hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
   X = hasher.transform(docs)  # fit is a no-op for HashingVectorizer
   print(X.shape)

Accent and tag utilities
.. code-block:: python

   from sklearn.feature_extraction.text import strip_accents_unicode, strip_accents_ascii, strip_tags

   s = "résumé & naïve façade"
   print(strip_accents_unicode(s))  # "resume & naive facade"
   print(strip_accents_ascii(s))    # "resume & naive facade"
   print(strip_tags("<p>Hello</p>"))  # " Hello "

Input handling and preprocessing
--------------------------------
Vectorizers accept text in various forms and may apply:
- decoding (filename, file-like objects, or raw strings)
- lowercasing
- accent normalization
- custom tokenization and n-gram generation
These capabilities are exposed through parameters such as:

- input: {'content', 'filename', 'file'}
- encoding and decode_error: used during decode

Tip: Use built-in stop words by passing stop_words='english' or an explicit
collection of words to filter common terms from your corpus.

Internal utilities (for advanced customization)
-----------------------------------------------
- _preprocess(doc, accent_function=None, lower=False)
  Chains optional lowercasing and accent handling.
- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None,
  decoder=None, stop_words=None)
  Orchestrates the sequence of text processing steps. If an analyzer is supplied,
  only the decoder is used; otherwise preprocessor, tokenizer, and ngrams are applied.
- _check_stop_list(stop)
  Returns a frozenset of stop words given a built-in alias ('english') or a
  user-provided collection.
- strip_accents_unicode, strip_accents_ascii, strip_tags
  Standalone utilities for preprocessing outside of vectorizers.

Notes
-----
- The module aims to provide memory-efficient and scalable text feature extraction
  suitable for large corpora.
- See the specific vectorizer classes (e.g., :class:`~sklearn.feature_extraction.text.CountVectorizer`,
  :class:`~sklearn.feature_extraction.text.TfidfVectorizer`) for detailed parameter
  options and behavior.
