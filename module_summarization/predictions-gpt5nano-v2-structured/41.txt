QuerySet: Building Queries
===========================

This document describes how to use QuerySet to build and execute queries against your models. It covers filtering, joining, ordering, annotation, field selection, and related data handling using the QuerySet abstraction built on top of PyPika’s QueryBuilder.

Overview
--------

- QuerySet provides a fluent API to construct SQL queries for your models.
- Queries are built using a combination of:
  - Filters (including complex Q objects)
  - Joins for related fields
  - Annotations (computed fields)
  - Ordering (including nested/related fields)
  - Field selections (values, values_list, and only)
  - Prefetching related data
- The underlying mechanism uses PyPika’s QueryBuilder and a set of helpers to translate Python expressions into SQL.

Key Concepts
------------

- AwaitableQuery: QuerySets are awaitable. Awaiting returns the results (a single object, a list, or a dictionary/tuple depending on the API used).
- Q objects: Compose complex filter criteria (AND, OR, etc.) in a structured way.
- Joins: Related fields are joined as needed to filter, order, or annotate by fields on related models.
- Annotations: Compute additional fields (e.g., counts) as part of the query.
- Ordering: Supports simple and nested fields, including related models.

Basic Usage
-----------


- Retrieve all objects that match simple criteria
  - Example: fetch active users
    .. code-block:: python

       active_users = await User.filter(is_active=True)

- Retrieve with selected fields
  - Values (dictionary) or Values_list (tuples)
    .. code-block:: python

       # As dictionaries
       users = await User.values("id", "name", "email")

       # As a list of tuples
       users = await User.values_list("id", "name")

- Retrieve a single field as a flat list
  - Example: list of IDs
    .. code-block:: python

       user_ids = await User.all().values_list("id", flat=True)

- Fetch with ordering
  - Basic ordering by a field
    .. code-block:: python

       ordered_users = await User.order_by("name")

  - Descending ordering
    .. code-block:: python

       ordered_users = await User.order_by("-created_at")

  - Ordering by related fields (via joins)
    .. code-block:: python

       ordered_by_profile_city = await User.order_by("profile__city")

- Subset of fields (only)
  - Example: fetch only id and name
    .. code-block:: python

       minimal_users = await User.only("id", "name")

Filtering with Q Objects
------------------------

- Use Q objects to build complex criteria (AND, OR, NOT, etc.)
  - Example: users who are active and at least 18 years old
    .. code-block:: python

       from tortoise.query_utils import Q
       adult_active_users = await User.filter(Q(age__gte=18) & Q(is_active=True))

- Complex nested conditions (OR)
  - Example: users in either city A or city B
    .. code-block:: python

       cities = Q(profile__city="CityA") | Q(profile__city="CityB")
       users_in_cities = await User.filter(cities)

- Combining filters with related fields
  - Example: filter by a field on a related model
    .. code-block:: python

       users_in_ny = await User.filter(profile__city="New York")

Prefetching and Related Data
----------------------------

- Prefetch related data to avoid extra queries when accessing related objects
  - Example: prefetch profile for each user
    .. code-block:: python

       users_with_profiles = await User.filter(is_active=True).prefetch_related("profile")

- Prefetching multiple relations
  - Example: prefetch profile and groups
    .. code-block:: python

       users = await User.filter(is_active=True).prefetch_related("profile", "groups")

Annotating Queries
------------------

- Add computed fields using annotations (e.g., counts)
  - Example: annotate each user with the number of posts
    .. code-block:: python

       from tortoise.functions import Count
       users_with_post_count = await User.annotate(post_count=Count("posts"))

- Use annotations in filters or ordering
  - Example: order by an annotated field
    .. code-block:: python

       users = await User.annotate(post_count=Count("posts")).order_by("post_count")

Selecting and Ordering by Nested/Related Fields
-----------------------------------------------

- Ordering can traverse related fields using the double-underscore notation
  - Example: order by city of a related profile
    .. code-block:: python

       users = await User.order_by("profile__city")

- Default ordering
  - If you have a model with a default ordering, you can rely on it unless overridden
    .. code-block:: python

       users = await User.all()

Selecting Specific Fields and Values
-------------------------------------

- values: return dictionaries keyed by field names
  - Example: fetch id and name
    .. code-block:: python

       users = await User.values("id", "name")

- values_list: return tuples (or flatten a single field)
  - Example: fetch id and name as tuples
    .. code-block:: python

       users = await User.values_list("id", "name")

  - Flatten one field
    .. code-block:: python

       ids = await User.values_list("id", flat=True)

Error Handling and caveats
--------------------------

- FieldError: Unknown field or invalid field usage (e.g., filtering by a relation directly without going through a nested field)
- DoesNotExist, MultipleObjectsReturned, ParamsError: typical query-related errors that may be raised during execution
- When filtering by aggregates or having clauses, a group by may be applied automatically to support the aggregation

Putting It All Together: End-to-End Example
-------------------------------------------

- Retrieve active users, with profile data preloaded, annotate the number of posts, order by the post count, and return id, name, and post_count
  .. code-block:: python

     from tortoise.functions import Count

     users = (
         await (
             User
             .filter(is_active=True)
             .prefetch_related("profile")
             .annotate(post_count=Count("posts"))
             .order_by("post_count")
             .values("id", "name", "post_count")
         )
     )

Notes
-----

- The QuerySet implementation uses PyPika’s QueryBuilder under the hood.
- Joins are performed as needed when filtering or ordering by related fields.
- You can combine filters, annotations, and ordering to build sophisticated queries in a readable, Pythonic way.
