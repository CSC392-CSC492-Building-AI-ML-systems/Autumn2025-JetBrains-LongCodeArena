Array API support
=================

Overview
--------

Array API is a standard interface for array-like data that enables interoperability across backends (NumPy, JAX, PyTorch, CuPy, etc.). This documentation describes how scikit-learn handles inputs and outputs that conform to the Array API, with a focus on multioutput estimators (e.g., MultiOutputRegressor, MultiOutputClassifier) and related components.

Key concepts
-----------

- Input compatibility: Estimators accept array-like inputs. When inputs come from an Array API backend, they should expose a numpy-compatible interface (e.g., support common operations like indexing, shape, and basic arithmetic). Where necessary, inputs can be converted to NumPy arrays to ensure full compatibility with internal computations.
- Output compatibility: Predicted arrays maintain a consistent shape and numeric type. If an Array API backend is used, outputs are provided in a numpy-friendly form or returned as the original array type when feasible.
- Duck-typing and minimal conversions: Scikit-learn relies on duck-typing for array-like behavior. Minimal or explicit conversions to NumPy arrays are performed to preserve broad compatibility and predictable behavior.

Usage guidance
--------------

- When using Array API-backed inputs, you may rely on NumPy interoperability in practice. If your Array API array implements the NumPy array protocol (or provides a smooth conversion path), scikit-learn estimators can operate on it with little or no modification.
- If you encounter any compatibility issues, convert inputs to NumPy arrays explicitly before fitting or predicting.

Code examples
-------------

Example 1: Using Array API arrays with a multioutput estimator (conceptual)
Note: This example assumes the Array API backend provides NumPy-compatible array objects or a straightforward conversion path.

```python
import array_api as xp  # Array API backend
import numpy as np
from sklearn.multioutput import MultiOutputRegressor
from sklearn.linear_model import LinearRegression

# Create Array API arrays (conceptual)
X = xp.asarray([[1.0, 2.0], [3.0, 4.0]])
y = xp.asarray([[1.0], [2.0]])

# Base estimator
base = LinearRegression()

# Multioutput estimator
model = MultiOutputRegressor(base)

# Fitting (scikit-learn will use NumPy-underlying operations if needed)
# If the Array API arrays are not fully NumPy-compatible, convert first:
# model.fit(np.asarray(X), np.asarray(y))
model.fit(X, y)

# Prediction
pred = model.predict(X)
```

Example 2: Explicit conversion when necessary
If your Array API backend does not present a NumPy-compatible interface, convert inputs explicitly.

```python
import array_api as xp
import numpy as np
from sklearn.multioutput import MultiOutputRegressor
from sklearn.linear_model import LinearRegression

X = xp.asarray([[1.0, 2.0], [3.0, 4.0]])
y = xp.asarray([[1.0], [2.0]])

model = MultiOutputRegressor(LinearRegression())

# Explicit conversion to NumPy arrays
model.fit(np.asarray(X), np.asarray(y))
pred = model.predict(np.asarray(X))
```

Limitations and notes
---------------------

- Not all Array API backends may be fully supported out-of-the-box. If you rely on a backend without NumPy compatibility for array operations, perform explicit conversions to NumPy arrays before estimator calls.
- Internal computations follow NumPy/SciPy semantics. Deviations in dtypes, broadcasting, or sparse support may occur if the Array API backend differs significantly from NumPy.
- Tests and validations: Compatibility with Array API inputs is exercised through general array-like input tests. For best results, ensure inputs are numeric and shape-consistent with estimator expectations.
