Executing notebooks
===================

Overview
--------

This module provides preprocessors for executing notebook code cells and for tagging cells that use language magics to aid syntax highlighting. The primary component is ExecutePreprocessor, which runs all code cells in a notebook and updates their outputs. It also includes HighlightMagicsPreprocessor, which detects magic language extensions (e.g., %%R, %%bash) and marks the cell metadata so the appropriate highlighter can be used.

Public API
-----------

ExecutePreprocessor
^^^^^^^^^^^^^^^^^^^

Purpose
~~~~~~~~
Execute all code cells in a notebook and update their outputs. The preprocessor initializes a kernel, executes cells in order, and records execution state and results back into the notebook.

Key features
~~~~~~~~~~~~~
- Executes code cells in-place, updating outputs.
- Captures kernel information and stores language metadata in the notebook.
- Supports per-notebook resources that can influence execution context (e.g., execution path).
- Reuses traitlets-based initialization, compatible with nbconvert 0.x behavior.

Constructor and signatures
~~~~~~~~~~~~~~~~~~~~~~~~~~
class ExecutePreprocessor(Preprocessor, NotebookClient)

- __init__(self, **kw)
  - nb: NotebookNode (optional). If omitted, a new NotebookNode is created.
  - Other keyword arguments are passed to both Preprocessor and NotebookClient as appropriate.

Public methods
~~~~~~~~~~~~~~
preprocess(nb, resources=None, km=None) -> (NotebookNode, dict)
- nb: NotebookNode, the notebook to execute (modified in place).
- resources: dict, optional resources used during conversion (e.g., {'metadata': {'path': run_path}}).
- km: KernelManager, optional kernel manager to reuse or customize the kernel.

Returns
-------
(nb, resources): The executed notebook and any updated resources.

- Implements the execution flow by reinitializing the client, preparing a kernel, and executing each cell via preprocess_cell.
- Updates nb.metadata['language_info'] from the kernel info.
- Each code cell is executed in order; non-code cells are left as-is.

preprocess_cell(cell, resources, index) -> (NotebookNode, dict)
- cell: NotebookNode, the cell to be processed.
- resources: dict, execution resources.
- index: int, index of the cell being processed.

Returns
-------
(cell, resources): The potentially modified cell and resources.

Notes
~~~~~
- If a cell raises an error during execution, a CellExecutionError is raised. The module re-exports CellExecutionError for backwards compatibility.

Example
-------

Execute a notebook and write the results back to disk:

.. code-block:: python

    import nbformat
    from nbconvert.preprocessors.execute import ExecutePreprocessor

    # Load the notebook
    with open("notebook.ipynb", "r", encoding="utf-8") as f:
        nb = nbformat.read(f, as_version=4)

    # Create the preprocessor and execute
    ep = ExecutePreprocessor(timeout=None, kernel_name="python3")
    nb, resources = ep.preprocess(nb, resources={})

    # Save the executed notebook
    with open("notebook_executed.ipynb", "w", encoding="utf-8") as f:
        nbformat.write(nb, f)

Notes
~~~~~
- You can pass a KernelManager instance via the km argument to reuse or customize the kernel.
- The timeout parameter controls how long to wait for cell execution; set to None for no timeout.

executenb (deprecated)
^^^^^^^^^^^^^^^^^^^^^

- This function is a compatibility wrapper that forwards to nbclient.execute.executenb.
- It issues a FutureWarning and is kept for backward compatibility.
- Recommended to import and call nbclient.execute directly.

Usage example (deprecated pathway)
.. code-block:: python

    from nbconvert.preprocessors.execute import executenb
    executenb(notebook, timeout=None)

HighlightMagicsPreprocessor
+++++++++++++++++++++++++

Purpose
~~~~~~~~
Detects code cells that use language magics (e.g., %%R, %%bash) and tags them with the corresponding language for syntax highlighting. This tagging enables proper highlighting in outputs that render code cells with language-specific lexers.

Default language mapping
~~~~~~~~~~~~~~~~~~~~~~~
- "%%R" -> "r"
- "%%bash" -> "bash"
- "%%cython" -> "cython"
- "%%javascript" -> "javascript"
- "%%julia" -> "julia"
- "%%latex" -> "latex"
- "%%octave" -> "octave"
- "%%perl" -> "perl"
- "%%ruby" -> "ruby"
- "%%sh" -> "sh"
- "%%sql" -> "sql"

Construction and usage
~~~~~~~~~~~~~~~~~~~~~~
class HighlightMagicsPreprocessor(Preprocessor)

- languages: traitlet Dict, user-configurable mapping that extends default_languages.
- default_languages: Dict of built-in language mappings.
- preprocess_cell(cell, resources, cell_index) -> (NotebookNode, dict)
  - If the cell is a code cell and contains a magic language, it adds metadata.magics_language with the corresponding language.

Example
-------

Apply the preprocessor in a pipeline (as part of a nbconvert config or a custom export pipeline):

.. code-block:: python

    from nbconvert.preprocessors.execute import HighlightMagicsPreprocessor

    hmp = HighlightMagicsPreprocessor()
    # Add hmp to the preprocessor chain in your exporter configuration

Notes
~~~~~
- This preprocessor does not execute code; it only tags cells for highlighting.
- It operates on code cells and only if a magic language is detected at the start of the cell source.

Getting started and integration
-------------------------------

- Typical usage is via nbconvert exporters or pipelines where preprocessors are configured in sequence.
- The ExecutePreprocessor is the primary tool for running notebooks programmatically and updating cell outputs.
- The HighlightMagicsPreprocessor is often used in conjunction with rendering steps that apply syntax highlighting.

References
----------

- Executing notebooks is implemented by combining Preprocessor and NotebookClient capabilities for robust in-notebook execution.
- For compatibility, CellExecutionError is exposed for handling execution errors in cells.
