w_eddist: Time-resolved transition-event duration distribution
------------------------------------------------------------

Overview
--------

w_eddist computes time-resolved histograms of transition-event durations from
kinetics results produced by WE simulations. It reads source data generated by
w_kinetics trace and builds normalized multi-dimensional histograms that capture
how long transition events last as a function of one or more descriptor
dimensions.

Key concepts
------------

- Source data: Derived from the results of w_kinetics trace (see the w_kinetics
  trace command help for how this data is produced). By default, the tool looks
  for kinetics results in kintrace.h5 unless overridden by options.

- Duration dataset: A specialized view (DurationDataset) that applies an
  iteration-specific mask to select durations within a given iteration.

- Histogram construction: For each iteration, durations are binned into a
  multi-dimensional histogram. The implementation uses WestPA’s histnd and
  normhistnd helpers to accumulate and normalize histograms.

- Binning: By default, 100 uniform bins are used per dimension. Binning can be
  customized with the -b/--bins option, described below.

Source data
-----------

- The input data are generated by w_kinetics trace and stored in a WE results
  file. The w_eddist tool reads the relevant datasets from this results file
  to compute the duration distributions.

- The durations are retrieved via a mask that selects the relevant segments for
  each iteration. If an iteration contains no data, that iteration contributes no
  counts to the histogram.

Histogram binning
-----------------

- Default: 100 bins per dimension.

- -b/--bins accepts several forms:

  - A single integer N: use N uniformly spaced bins for every dimension.
  - A comma-separated sequence N1,N2,...: use Ni bins for the i-th dimension.
  - A list of lists [[B11, B12, B13,...], [B21, B22, B23,...], ...]: explicit
    bin boundaries per dimension. Boundaries may be non-uniform and are evaluated
    with Python’s eval, with NumPy available (np). This form allows arbitrary
    bin spacing and explicit control over bin edges.

- Note: The first two forms (single integer or a sequence of integers) require
  scanning data to determine min/max per dimension, which may be expensive for
  large datasets. The list-of-lists form avoids this cost by using explicit
  boundaries.

Output format
-------------

The file produced (default: pdist.h5) contains:

- histograms
  Normalized histograms. The first axis corresponds to iteration, and the
  remaining axes correspond to the dimensions of the input dataset.

- /binbounds_0, /binbounds_1, ...
  Datasets containing the bin boundaries for each dimension.

- /midpoints_0, /midpoints_1, ...
  Datasets containing the bin midpoints for each dimension.

- n_iter
  Vector of iteration numbers corresponding to the stored histograms
  (i.e., the first axis of histograms).

Subsequent processing
---------------------

The output file can be plotted or further processed with plothist (see plothist
--help). The histogram datasets are arranged so that iteration is the leading
axis, followed by the dimensions corresponding to the input descriptor(s).

Parallelization
---------------

- This tool supports parallelized binning and input data reading by default.
- For simpler cases or smaller datasets, running serially (--serial) may be more
  efficient.

Command-line options
--------------------

- -o, --output
  Path to the output HDF5 file. Default: pdist.h5

- -b, --bins
  Bin specification as described in the Histogram binning section.

- --serial
  Run the computation serially instead of using the parallel work manager.

- Other options
  See the help output (w_eddist --help) for a complete list of options, including
  input data selection, iteration range, and any additional knobs exposed by
  the tool.

Examples
--------

- Basic usage with default settings:
  - w_eddist --output pdist.h5

- Specify a uniform binning of 100 bins per dimension:
  - w_eddist --output pdist.h5 --bins 100

- Use explicit per-dimension bin counts (N1 for dim 0, N2 for dim 1, ...):
  - w_eddist --output pdist.h5 --bins 50,60

- Use non-uniform, explicit bin boundaries for two dimensions:
  - w_eddist --output pdist.h5 --bins [[0, 1, 2, 5, 10], [0, 0.5, 1.5, 3]]

- Run serially (instead of the default parallel processing):
  - w_eddist --serial --output pdist.h5

Notes
-----

- The tool reads kinetics results from kintrace.h5 by default. This can be overridden
  by appropriate command-line options (see w_eddist --help in your installation).

- The produced histograms are intended for plotting with plothist or compatible tools.

See Also
--------

- w_kinetics trace: for generating the source kinetics data used by w_eddist
- plothist: for plotting and analyzing histograms produced by w_eddist

Implementation highlights
-------------------------

- DurationDataset: applies an iteration-specific mask to extract relevant
  duration data for that iteration.
- _remote_min_max: computes per-dimension data ranges (min/max) to initialize
  histogram bounds when explicit boundaries are not provided.
- _remote_bin_iter: performs the per-iteration binning using histnd, then
  normalizes the histogram with normhistnd.
