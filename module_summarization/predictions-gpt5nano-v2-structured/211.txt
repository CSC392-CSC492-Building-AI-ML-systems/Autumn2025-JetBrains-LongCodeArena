Generalized Linear Mixed Effects Models (GLMM)
===============================================

Overview
--------
Generalized Linear Mixed Effects Models enable Bayesian inference for GLMs that include both fixed effects and random effects. This implementation focuses on families without additional scale or shape parameters (currently binomial and Poisson) and supports two estimation approaches:

- Laplace approximation (maximum a posteriori, MAP)
- Variational Bayes (mean-field approximation to the posterior)

All random effects are modeled as independent realizations in this framework.

Key concepts
------------
- exog_vc: Design matrix for the random effects. Each column corresponds to an independent random effect realization.
- ident: Integer labels indicating which random effects share the same variance. Columns with the same ident value have equal standard deviation.
  - When using formulas, columns derived from a common formula share the same variance automatically.
- Random effects (vc): Realizations drawn from Gaussian distributions with structure determined by vcp.
- vcp: Log-standard deviation parameters of the random effects. These have Gaussian priors on the standard deviations (with log-normal distributions for the standard deviations themselves).
- fep: Fixed effects parameters. These have Gaussian priors with mean 0 and standard deviation fe_p.
- Priors:
  - p(fe) is Gaussian with mean 0 and standard deviation fe_p.
  - p(vcp) is Gaussian in log-standard deviations (log-normal for the standard deviations) with prior standard deviation vcp_p.
  - p(vc | vcp) is an independent Gaussian distribution for the random effects given vcp.
- Families: Currently supported are binomial and Poisson (no extra scale/shape parameters).

Model and parameterization
--------------------------
The joint density of data and parameters factors as:

.. math::

    p(y | vc, fep) \; p(vc | vcp) \; p(vcp) \; p(fe)

- p(y | vc, fep): GLM likelihood given random effects vc and fixed effects fep (depends on the chosen family).
- p(vc | vcp): Independent Gaussian distribution for random effects, with variance structure determined by vcp and ident.
- p(vcp): Prior on the log-standard deviations of random effects.
- p(fe): Prior on fixed effects parameters.

Implementation notes
--------------------
- All random effects are assumed independent within and between groups defined by ident.
- exog_vc may be provided as a scipy.sparse matrix and will be handled accordingly. If a dense matrix is passed, it is converted to sparse internally. When using formulas, a temporary dense version of exog_vc may be created.
- Formula support: When using formulas for random effects, the ident structure is created from the formula terms, and all columns generated from a common formula share variance.
- Sparsity: The implementation supports sparse exog_vc. Dense inputs are converted to sparse internally.
- Independent realizations: Each column of exog_vc corresponds to a separate random effect realization.

Inputs
------
- endog: array_like
  Vector of response values.
- exog: array_like
  Covariates for the fixed effects (mean structure).
- exog_vc: array_like
  Covariates for the random effects. May be scipy.sparse; otherwise converted to sparse.
- ident: array_like
  Integer labels indicating which random terms share a variance.
- vcp_p: float
  Prior standard deviation for variance component parameters (log standard deviations of random effects).
- fe_p: float
  Prior standard deviation for fixed effects parameters.
- family: statsmodels.genmod.families instance
  GLM family (e.g., Poisson, Binomial).
- fep_names: list of str, optional
  Names for fixed effects. If None, default names are constructed.
- vcp_names: list of str, optional
  Names for variance component parameters. If None, default names are constructed.
- vc_names: list of str, optional
  Names for the random effect realizations.

Outputs
-------
- MixedGLMResults
  The result object produced by fit_map() or fit_vb(), containing parameter estimates, uncertainty measures, and model diagnostics.

Fitting methods
---------------
- fit_map(): Fit using the Laplace approximation to the posterior (MAP).
- fit_vb(): Fit using variational Bayes (mean-field approximation to the posterior).

Usage example
-------------
Assuming you have prepared data: endog, exog, exog_vc, and ident, and you choose a Poisson family:

.. code-block:: python

    from statsmodels.genmod import families
    from your_module import MixedGLM  # replace with the actual import path

    # Example inputs (numpy arrays or sparse matrices)
    # endog = ...
    # exog = ...
    # exog_vc = ...
    # ident = ...

    model = MixedGLM(endog, exog, exog_vc, ident,
                     vcp_p=0.5, fe_p=0.5,
                     family=families.Poisson())

    # MAP estimation (Laplace approximation)
    res_map = model.fit_map()
    print(res_map)

    # Variational Bayes
    res_vb = model.fit_vb()
    print(res_vb)

Notes
-----
- For some families (e.g., Binomial), setting vcp_p too large may hinder convergence to the posterior mode.
  A practical value such as vcp_p = 0.5 often works well.
- The priors on fixed effects (fe_p) and on the log-standard deviations (vcp_p) influence shrinkage and uncertainty.
- When using formulas for random effects, the ident structure is derived automatically from the formulas.

Internal details
----------------
- The implementation relies on Bayesian inference for GLMMs and supports both Laplace MAP and VB approaches.
- Random effects are modeled as independent realizations with variance structure determined by ident and vcp.
- The design matrix exog_vc supports sparse representations to improve efficiency in large problems.

If you need a quick reference for the API, see the class constructor and the fit methods in the source module mixed_glm.rst.
