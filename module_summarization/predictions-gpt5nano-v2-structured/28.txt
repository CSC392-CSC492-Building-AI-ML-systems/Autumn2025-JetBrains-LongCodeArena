w_multi_west
============

Overview
--------
w_multi_west is a WESTPA tool designed to combine multiple WESTPA simulations into a single HDF5 file, with optional reweighting across runs. It is implemented as a WESTMultiTool-based command and handles stitching together core datasets and selected auxiliaries from multiple WESTPA runs into one consolidated output.

Purpose
-------
- Merge results from multiple independent WESTPA simulations into one coherent dataset.
- Optionally apply reweighting across simulations to produce a combined ensemble that reflects relative weights between runs.
- Provide control over which auxiliary datasets to include in the merge.
- Optionally include ibstates data if present in the inputs.

Key Functionality
-----------------
- Creates an output file (default name: multi.h5) and stamps creator metadata.
- Collects input WESTPA HDF5 files from one or more simulation directories.
- Merges core data (and optionally auxiliary datasets) across simulations.
- Supports reweighting across simulations unless disabled with --no-reweight.
- Supports combining ibstates datasets when requested.

Usage
-----
w_multi_west [options]

Options
-------
- -o, --output-file
  - Default: multi.h5
  - Description: Name of the output HDF5 file to store merged results.

- -W, --west, --WEST_H5FILE
  - Default: west.h5
  - Description: The name of the main WESTPA H5 file inside each simulation directory.

- -a, --aux
  - Action: append
  - Description: Names of additional auxiliary datasets to be combined.

- -aa, --auxall
  - Action: store_true
  - Description: Combine all auxiliary datasets. Default: False

- -nr, --no-reweight
  - Action: store_true
  - Description: Do not reweight. Default: False

- -ib, --ibstates
  - Action: store_true
  - Description: Attempt to combine ibstates dataset. Default: False

Notes
-----
- The tool is built on WESTPA's WESTMultiTool framework and leverages WESTPA I/O utilities to open and write HDF5 datasets.
- By default, reweighting is applied across simulations to create a consistent ensemble; use --no-reweight to disable this behavior.
- Auxiliary data handling is flexible:
  - Use -a to specify particular auxiliary datasets to merge.
  - Use -aa to merge all available auxiliary datasets.

Example
-------
Merge two simulations with reweighting and include a specific auxiliary dataset:

$ w_multi_west -o multi.h5 -W west.h5 -a dist -aa

Merge with no reweighting and include ibstates (if present):

$ w_multi_west -o multi.h5 -W west.h5 -nr -ibstates

Notes on Implementation (for reference)
---------------------------------------
- The tool is implemented as a subclass of WESTMultiTool and uses a progress indicator for user feedback.
- Key steps include:
  - Creating an output file via h5io.WESTPAH5File and stamping creator data.
  - Generating a list of input WESTPA HDF5 files from the provided simulation roots.
  - Preparing to merge datasets (core and optionally auxiliaries) across simulations.
- The internal Segment class represents per-segment metadata for merged data (weight, iteration, simid, recycled_in).

Code references (selected)
--------------------------
- CLI options and wiring are defined in WMultiWest.add_args.
- Output preparation is performed in WMultiWest.open_files (creates output file and stamps metadata).
- Input collection is performed via generate_file_list([self.west]).
- Core data handling scaffolding (e.g., auxiliary dataset handling, ibstates) is controlled by the provided flags.

Example Code Snippet
--------------------

$ w_multi_west -o multi.h5 -W west.h5

$ w_multi_west -o multi.h5 -W west.h5 -a dist -aa

$ w_multi_west -o multi.h5 -W west.h5 -nr

Limitations
-----------
- The go() method in the provided snippet appears incomplete; production usage may require the full implementation to perform the actual merge and reweighting across runs.
- Ensure input simulation directories conform to WESTPA's expected layout and HDF5 schema for successful merging.
