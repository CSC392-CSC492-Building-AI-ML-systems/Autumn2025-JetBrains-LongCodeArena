Generalized Additive Models: Spline and Smoother Utilities
--------------------------------------------------------

This module provides utilities to generate and evaluate B-spline bases for smooth terms in Generalized Additive Models (GAMs). It includes helpers to choose knots, compute spline bases and their derivatives, and obtain ready-to-use design matrices for fitting with standard regression tools or GAM frameworks (e.g., GLMGam).

Public helpers
- get_knots_bsplines(x=None, df=None, knots=None, degree=3, spacing='quantile', lower_bound=None, upper_bound=None, all_knots=None)
  - Compute or return knot locations for B-splines. Supports quantile-based spacing or equal/exterior spacing suitable for P-Splines.
- make_bsplines_basis(x, df, degree)
  - Produce a spline basis (and its first and second derivatives) evaluated at x for a given degrees-of-freedom (df) and spline degree.
- _equally_spaced_knots(x, df)
  - Internal helper to create equally spaced interior knots (not typically used directly in user code).
- _R_compat_quantile(x, probs)
  - Internal helper to compute quantile-based knot locations in a manner compatible with R/Patsy conventions.
- _eval_bspline_basis(x, knots, degree, deriv='all', include_intercept=True)
  - Internal routine that evaluates B-spline basis (and optionally derivatives) at x for given knots and degree. Requires SciPy.

Note: _eval_bspline_basis is intended for internal use; use make_bsplines_basis for a user-friendly API.

Example: A basic B-spline smooth in a linear model
This example demonstrates how to generate a B-spline basis for a predictor x and fit a simple linear model y ~ smooth(x). The approach shown can serve as a building block for GAMs (e.g., when integrating with GLMGam or other GAM frameworks).

Code
.. code-block:: python

    import numpy as np
    import statsmodels.api as sm
    from gam import make_bsplines_basis

    # Simulate data: nonlinear relationship with noise
    np.random.seed(0)
    n = 200
    x = np.linspace(0, 1, n)
    y = np.sin(2 * np.pi * x) + 0.2 * np.random.randn(n)

    # Generate a B-spline basis for x
    df = 8        # number of basis functions (controls smoothness)
    degree = 3    # cubic splines

    basis, der_basis, der2_basis = make_bsplines_basis(x, df=df, degree=degree)

    # Fit a simple linear model with an intercept
    X = sm.add_constant(basis)  # add intercept term
    model = sm.OLS(y, X).fit()

    print(model.summary())

    # Predict on the training data (optional)
    y_pred = model.predict(X)

    # Quick evaluation (optional): compute fit quality
    ss_res = np.sum((y - y_pred) ** 2)
    ss_tot = np.sum((y - y.mean()) ** 2)
    r2 = 1 - ss_res / ss_tot
    print("R^2:", r2)

Discussion
- The make_bsplines_basis function returns three arrays:
  - basis: the evaluated B-spline basis functions (shape: [n, n_bases])
  - der_basis: the first derivative basis (same shape as basis)
  - der2_basis: the second derivative basis (same shape)
- The resulting basis is suitable as a design matrix for smooth terms in regression or as part of a GLMGam smoother specification.
- Knot placement:
  - The default usage is df-based quantile-spaced interior knots with cubic splines (degree=3). For P-spline-like behavior, you can control knot placement via get_knots_bsplines and pass a custom all_knots to the internal routine (via make_bsplines_basis indirectly through the API).

Example: Knot placement and custom knots
Code
.. code-block:: python

    import numpy as np
    from gam import get_knots_bsplines, make_bsplines_basis

    x = np.sort(np.random.rand(150))
    df = 10
    degree = 3

    # Get quantile-based knots (default behavior)
    all_knots, lb, ub, inner_knots = get_knots_bsplines(x=x, df=df, degree=degree, spacing='quantile')
    # Alternatively, you can compute and reuse a custom all_knots array
    # all_knots = custom_all_knots_array

    # Evaluate basis
    basis, der_basis, der2_basis = make_bsplines_basis(x, df=df, degree=degree)

Further usage notes
- Integration with GLMGam and other GAM frameworks:
  - The bases produced by make_bsplines_basis (and the underlying knot logic) are designed to be used as smooth terms in GLMGam or similar models. You would typically pass the basis matrices as part of the exogenous design matrix or through the framework’s smoother specifications.
- Dependency
  - The spline evaluation relies on SciPy (scipy.interpolate.splev). If SciPy is unavailable, the code will raise an ImportError with a helpful message.
- Internal helpers
  - The module contains several internal helpers (_equally_spaced_knots, _R_compat_quantile, _eval_bspline_basis) that implement the low-level knot management and evaluation logic. For typical usage, prefer make_bsplines_basis and get_knots_bsplines.
