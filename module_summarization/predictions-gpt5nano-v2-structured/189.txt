# Kernel Ridge Regression

Kernel ridge regression (KRR) combines ridge regression (linear least squares with L2 regularization) with the kernel trick. It learns a linear function in the feature space induced by a chosen kernel, which corresponds to a non-linear function in the original input space for non-linear kernels. The model learned by KRR is mathematically identical to that of support vector regression (SVR) with squared-loss, but uses a closed-form solution and L2 regularization, making it typically faster to fit on medium-sized datasets. Unlike SVR, the resulting model is not sparse.

KRR supports multi-target regression, i.e., predicting multiple outputs simultaneously when `y` is a 2D array of shape `(n_samples, n_targets)`.

> See also: Gaussian Process Regressor, Ridge, and SVR for related approaches to kernelized regression.

---

## When to use Kernel Ridge Regression

- You want a non-linear regression model without the need to tune many hyperparameters.
- You prefer a closed-form solution with a simple, predictable training procedure.
- You are working with medium-sized datasets where a dense (non-sparse) kernel solution is feasible.
- Your problem involves predicting multiple targets (multi-output regression).

---

## How it works

- The model represents the function as a linear combination of kernel evaluations on training data:
  f(x) = sum_i alpha_i k(x_i, x)
- The dual coefficients alpha are obtained by solving:
  (K + Alpha I) alpha = y
  where K is the kernel matrix with K_ij = k(x_i, x_j) and Alpha is the ridge regularization parameter (alpha in code).
- Predictions for new inputs X are made by computing the kernel between X and the training data and applying the learned coefficients.

Key implications:
- Training is in closed form; prediction is efficient after training.
- The approach is non-sparse (all training samples contribute to predictions).
- You can use a variety of kernels (linear, polynomial, RBF, sigmoid, etc.) or supply a precomputed kernel matrix.

---

## Parameters

- **alpha**: float or array-like of shape `(n_targets,)`, default `1.0`
  Regularization strength. Positive values improve conditioning and reduce variance. If an array is provided, penalties apply per target.

- **kernel**: str or callable, default `"linear"`
  Kernel mapping used internally. If a string, it must be one of the available kernel names or `"precomputed"`. If a callable, it is applied to pairs of samples to compute the kernel value. When using a callable, you may pass additional parameters via `kernel_params`.

- **gamma**: float, default `None`
  Gamma parameter for certain kernels (e.g., RBF, polynomial, sigmoid). Ignored by kernels that do not use it.

- **degree**: int, default `3`
  Degree of the polynomial kernel. Ignored by other kernels.

- **coef0**: float, default `1`
  Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

- **kernel_params**: dict, default `None`
  Additional parameters for the kernel function, passed as keyword arguments when the kernel is a callable.

Notes:
- The kernel can also be `"precomputed"`, in which case the input `X` is treated as a precomputed kernel matrix of shape `(n_samples, n_samples)`.

---

## Attributes

- **dual_coef_**: ndarray of shape `(n_samples,)` or `(n_samples, n_targets)`
  Representation of the weight vector(s) in kernel space (the learned dual coefficients).

- **X_fit_**: ndarray or sparse matrix of shape `(n_samples, n_features)` or the training kernel matrix when `kernel='precomputed'`
  Training data used for fitting. If `kernel == "precomputed"`, this stores the training kernel matrix.

- **n_features_in_**: int
  Number of features seen during fit.

- **feature_names_in_**: ndarray of shape `(n_features_in_,)`
  Names of features seen during fit (if available).

---

## Usage

### Basic example

```python
from sklearn.kernel_ridge import KernelRidge
import numpy as np

# Generate sample data
rng = np.random.RandomState(0)
n_samples, n_features = 100, 5
X = rng.randn(n_samples, n_features)
y = np.sin(X[:, 0]) + 0.1 * rng.randn(n_samples)

# Create and fit a kernel ridge model
krr = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.1)
krr.fit(X, y)

# Predict on new data
X_new = rng.randn(10, n_features)
y_pred = krr.predict(X_new)
```

### Precomputed kernel example

```python
from sklearn.metrics.pairwise import pairwise_kernels
from sklearn.kernel_ridge import KernelRidge
import numpy as np

rng = np.random.RandomState(0)
X = rng.randn(50, 3)
y = rng.randn(50)

# Compute a kernel matrix for training data
K = pairwise_kernels(X, X, metric="rbf", gamma=0.2)

# Use precomputed kernel
krr = KernelRidge(alpha=0.5, kernel="precomputed")
krr.fit(K, y)

# Predict requires the kernel between new samples and training samples
K_pred = pairwise_kernels(np.vstack([X[:5], X[5:]] + [X[:5]]), X, metric="rbf", gamma=0.2)
y_pred = krr.predict(K_pred)
```

### Multi-target regression

```python
# y is of shape (n_samples, n_targets)
y_multi = rng.randn(n_samples, 3)
krr = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.1)
krr.fit(X, y_multi)
y_hat = krr.predict(X)
# y_hat has shape (n_samples, 3)
```

---

## Practical notes

- Computational complexity scales cubically with the number of training samples due to the matrix inversion/Cholesky factorization, making it best suited for medium-sized datasets.
- The model is dense (non-sparse) in its representation; predictions are straightforward to compute but may be slower at large scales compared to sparse models like some SVR variants.
- Kernel choice and hyperparameters (gamma, degree, coef0) strongly influence performance. Use cross-validation or kernel-specific heuristics to select them.
- For large datasets or automatic hyperparameter tuning, consider alternative approaches (e.g., SVR with an RBF kernel, or Gaussian Processes) or approximate kernel methods.
