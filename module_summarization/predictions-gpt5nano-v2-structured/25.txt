WESTPA Interactive Python Analysis (w_ipa)
------------------------------------------

WIPI (WESTPA Interactive Python Analysis) is a stateful interactive tool that provides convenient access to the current WESTPA analysis state. It allows you to inspect and interact with data from the currently active analysis scheme and iteration, including traces, weights, progress coordinates, states, bins, and more.

Overview
--------

- Purpose: Explore and analyze the data of the current WESTPA analysis scheme and iteration from an interactive session.
- Statefulness: All data accessed by w_ipa reflects the current analysis scheme and the current iteration. Data updates automatically as you navigate iterations and schemes.
- Access: Information about the current state is exposed via attributes such as w, west, and current (e.g., walkers, weights, pcoord, bins, populations, and auxdata).
- Tracing: The w.trace(seg_id) function can compute a trace over a specific segment and return a dictionary with history. Traces can be expensive, so store results when needed.

Key Concepts and Data Access
----------------------------

- Global namespace access:
  - w: provides properties and methods for the current iteration and analysis state.
  - west: a parallel/west layout interface for global access; both w and west expose similar data.

- Current iteration and scheme:
  - To view the current iteration: w.iteration or west.iteration
  - To set a new iteration: w.iteration = 100
  - To view/list analysis schemes: w.list_schemes
  - To set the current scheme: w.scheme = <scheme_name>

- Current data snapshot:
  - w.current contains data for the current iteration, including:
    - walkers, summary, states, seg_id, weights, parents, kinavg, pcoord,
      bins, populations, and auxdata (if present)

- Additional data views:
  - w.states: the states defined in the current analysis scheme
  - w.bin_labels: the labels for bins in the current scheme
  - w.trace(seg_id): run a trace for a given seg_id and return a dictionary with trace history

- Related tools:
  - w_assign, w_direct, w_reweight are available via the same WESTPA CLI environment for common workflow tasks.

Usage
-----

- Start an interactive w_ipa session (default interface uses matplotlib for plotting).

  $ w_ipa

- Access current iteration and scheme:

  - View current iteration:
    >>> w.iteration
    1
  - View current scheme:
    >>> w.scheme
    'Your_Default_Scheme_Name'

- Change iteration or scheme:

  - Set a new iteration:
    >>> w.iteration = 2
  - Change to a different scheme:
    >>> w.scheme = 'AnalysisSchemeB'

- Inspect data:

  - Current data snapshot:
    >>> current = w.current
  - Available states and bin labels:
    >>> w.states
    >>> w.bin_labels

- Trace a segment:

  - Run a trace for a segment id (seg_id) and inspect the history:
    >>> trace = w.trace(seg_id)

- Access current iteration data via the w object:

  - Access walkers, weights, and pcoord:
    >>> w.current['weights']
    >>> w.current['pcoord']

- Interactive plotting:
  - By default, w_ipa uses matplotlib. To plot output in a terminal, run with the terminal option.

CLI Options
-----------

w_ipa supports several runtime options to control analysis execution and output:

- --analysis-only, -ao
  - Run the analysis and return to the terminal.

- --reanalyze, -ra
  - Delete existing files and reanalyze from scratch.

- --ignore-hash, -ih
  - Ignore the stored hash and do not regenerate files.

- --debug, -d
  - Enable debug output for development.

- --terminal, -t
  - Plot output in the terminal.

- -f, --extra
  - Hidden/auxiliary option placeholder (supplied for compatibility).

Note: The tool also reads configuration from the WestPA configuration (west.cfg) to determine the default analysis schemes and active iteration. The primary interface remains the w object (and west) for interactive data access.

Key Implementation Details (for advanced users)
----------------------------------------------

- Data access:
  - WIPI initializes a WESTDataReader to fetch data, and uses the current WESTPA analysis configuration to determine the active scheme.

- Hashing and reanalysis:
  - w_ipa includes hash_args for generating a hash that fingerprints a particular set of arguments for a given iteration. This supports logic for updating or regenerating files only when inputs change.
  - stamp_hash can stamp an HDF5 file with an arg_hash attribute to indicate the configuration used for generation.

- Performance considerations:
  - Traces can be expensive to compute; store results if you need repeated access.

Code References (conceptual)
----------------------------

- Core class and purpose:

  - WIPI is defined as a WESTParallelTool subclass that provides an interactive, stateful interface to the current analysis data.

- Typical interactive flow:

  - Access iteration and scheme:
    - w.iteration
    - w.scheme
  - Inspect data:
    - w.current
    - w.states
    - w.bin_labels
  - Run a trace:
    - w.trace(seg_id)

Examples
--------

- Quick session: view and adjust iteration and scheme, then inspect data

  .. code-block:: python

     # Start w_ipa session
     # In your shell:
     # $ w_ipa

     # Within the session:
     w.iteration            # e.g., 1
     w.scheme               # e.g., 'DefaultScheme'
     w.states               # list of defined states
     w.bin_labels           # labels for bins

     # Change iteration and/or scheme
     w.iteration = 2
     w.scheme = 'AnalysisSchemeB'

     # Inspect current data snapshot
     current = w.current
     walkers = current['walkers']
     weights = current['weights']
     pcoord = current['pcoord']

     # Run a trace for a specific segment
     trace = w.trace(seg_id)

- Reanalysis workflow (with CLI options)

  .. code-block:: bash

     # Reanalyze from scratch, re-generating files as needed
     $ w_ipa --reanalyze

- Use with terminal plotting

  .. code-block:: bash

     # Plot output in terminal instead of graphical backend
     $ w_ipa --terminal

Notes
-----

- The w_ipa tool is designed to be used interactively within a WESTPA-enabled environment. It relies on the currently active analysis scheme and iteration, and is therefore best used after selecting the desired scheme and iteration via w or west.
- The documentation herein reflects the capabilities exposed by the WIPI class as implemented in the provided source.
