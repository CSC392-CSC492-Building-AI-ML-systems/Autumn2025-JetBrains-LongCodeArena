Working with Text Data
======================

Analyzing a collection of text documents
---------------------------------------

This document describes how to analyze a collection of text documents using scikit-learn’s text feature extraction tools. The core idea is to convert raw text into numeric feature vectors that can be used for machine learning. The primary building blocks are CountVectorizer and TfidfVectorizer, which transform documents into sparse matrices of token counts or TF-IDF scores, respectively. Text processing is achieved through a configurable pipeline that may include decoding, pre-processing, tokenization, stop-word removal, and n-gram generation.

Key concepts
------------

- Decoding input: Documents can be provided as plain text, or as filenames or file-like objects. The vectorizers handle decoding automatically according to the input type and the configured encoding.
- Preprocessing: Optional steps such as lowercasing and accent-stripping to normalize text before tokenization.
- Tokenization and n-grams: Breaking text into tokens and optionally combining tokens into n-grams (e.g., unigrams, bigrams).
- Stop words: Removal of common words that may not be informative for modeling.
- Analyzers: A high-level interface that defines the entire processing chain. You can supply custom analyzer, preprocessor, tokenizer, or ngrams, or rely on the built-in defaults.
- Output: A sparse matrix where rows correspond to documents and columns correspond to features (tokens or n-grams). The vocabulary maps features to column indices.

Preprocessing and tokenization options
--------------------------------------

The processing chain can be customized with:

- analyzer: A callable that takes a document and returns a sequence of tokens (or a pre-processed document). If provided, this replaces the preprocessor, tokenizer, and ngrams steps.
- preprocessor: A callable that transforms the raw text before tokenization (e.g., removing HTML tags).
- tokenizer: A callable that splits a preprocessed document into tokens.
- ngrams: A callable that converts tokens into n-grams (e.g., 1- to 2-grams).
- stop_words: A list or set of stop words to remove, or a built-in list (e.g., "english").

Accent stripping and HTML stripping utilities
---------------------------------------------

- strip_accents_unicode(s): Replace accented characters with their unaccented equivalents, using Unicode normalization.
- strip_accents_ascii(s): Remove accents by transliterating to ASCII.
- strip_tags(s): Remove HTML/XML tags using a lightweight regular-expression-based approach.

Example: Analyzing a simple collection
--------------------------------------

This example demonstrates a basic CountVectorizer run on a small collection of documents.

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = [
       "The quick brown fox jumps over the lazy dog.",
       "Never jump over the lazy dog quickly.",
       "A quick movement of the enemy will jeopardize five gunboats."
   ]

   vec = CountVectorizer()
   X = vec.fit_transform(docs)

   print("Document-term matrix shape:", X.shape)
   print("Vocabulary:", vec.get_feature_names_out())

Example: Using TF-IDF with stop words
-------------------------------------

TF-IDF often yields better performance by down-weighting common words.

.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   docs = [
       "The quick brown fox jumps over the lazy dog.",
       "Never jump over the lazy dog quickly.",
       "A quick movement of the enemy will jeopardize five gunboats."
   ]

   tfidf = TfidfVectorizer(stop_words="english")
   X = tfidf.fit_transform(docs)

   print("Document-term matrix shape:", X.shape)
   print("Vocabulary:", tfidf.get_feature_names_out())

Example: N-grams
-----------------

Generating n-grams can capture more context than unigrams alone.

.. code-block:: python

   vec = CountVectorizer(ngram_range=(1, 2))  # unigrams and bigrams
   X = vec.fit_transform(docs)
   print("Vocabulary size:", len(vec.get_feature_names_out()))
   print("First 10 features:", vec.get_feature_names_out()[:10])

Example: Using a custom stop word list
--------------------------------------

You can supply your own set of stop words or reuse built-in lists (e.g., ENGLISH_STOP_WORDS).

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, ENGLISH_STOP_WORDS

   vec = CountVectorizer(stop_words=ENGLISH_STOP_WORDS)
   X = vec.fit_transform(docs)
   print("Vocabulary size with English stop words removed:", len(vec.get_feature_names_out()))

Example: Stripping HTML tags with a preprocessor
------------------------------------------------

Strip HTML/XML tags before tokenization.

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, strip_tags

   docs_html = ["<p>The <b>quick</b> brown fox.</p>"]
   vec = CountVectorizer(preprocessor=strip_tags)
   X = vec.fit_transform(docs_html)
   print("Vocabulary:", vec.get_feature_names_out())

Example: Custom analyzer
------------------------

Create a custom analysis function to fully control tokenization.

.. code-block:: python

   import re

   def my_analyzer(doc):
       # Simple alphanumeric tokens, lowercase
       return re.findall(r"\b\w\w+\b", doc.lower())

   from sklearn.feature_extraction.text import CountVectorizer

   docs = [
       "Hello world! This is a test.",
       "World of tests and examples."
   ]

   vec = CountVectorizer(analyzer=my_analyzer)
   X = vec.fit_transform(docs)
   print("Vocabulary:", vec.get_feature_names_out())

Input types and usage notes
---------------------------

- The input can be raw text, or if you set input="filename" or input="file", the vectorizer will read the content accordingly and decode using the configured encoding.
- The default analyzer is word-based tokenization with simple whitespace and punctuation handling, but you can customize the entire pipeline via analyzer, preprocessor, tokenizer, ngrams, and stop_words.
- ENGLISH_STOP_WORDS is available as a built-in stop list to simplify removing common English words.

See also
--------

- hash-based vectorization (HashingVectorizer) for fixed-size, memory-efficient feature spaces.
- The combination of CountVectorizer and TfidfTransformer (or the built-in TfidfVectorizer) for scalable text representations.
