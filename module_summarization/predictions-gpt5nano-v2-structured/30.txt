w_pdist.rst
============

w_pdist - Time-resolved, multi-dimensional probability distributions

Overview
--------
Calculate time-resolved, multi-dimensional probability distributions of WE datasets.
The tool supports parallel processing by default and can also operate serially.

Source data
-----------
Source data is provided either by a user-specified function (--construct-dataset) or
a list of "data set specifications" (--dsspecs). If neither is provided, the
progress coordinate dataset pcoord from the main WEST HDF5 file (west.h5) is used.

- Custom data extraction function
  Specify the function in STANDARD Python MODULE.FUNCTION syntax as the argument to
  --construct-dataset. The function is called as function(n_iter, iter_group) and
  must return data indexable as [segment][timepoint][dimension].

- Data set specifications
  Use --dsspecs followed by one or more data set specifications:
  NAME[,file=FILENAME,slice=SLICE]

  - NAME: dataset name in the HDF5 file
  - file: optionally specify the HDF5 file (default: west.h5)
  - slice: Python slice expression to select data (e.g. [0:2])

  The slice option is particularly useful for selecting one or more columns from a
  multi-dimensional dataset (e.g. a multi-dimensional progress coordinate).

Histogram binning
-----------------
Histograms are constructed with 100 bins per dimension by default. The -b/--bins option
alters binning and accepts several forms:

- A single integer N
  N uniformly spaced bins are used in every dimension.

- A sequence of integers N1,N2,... (comma-separated)
  N1 bins for the first dimension, N2 for the second, etc.

- A list of lists [[B11, B12, B13,...], [B21, B22, B23,...], ...]
  Per-dimension bin boundaries. These boundaries need not be uniformly spaced and
  are evaluated using Python's eval (np is available).

Notes
- The first two forms (integer, list of integers) trigger a scan of all data to
  determine min/max values for each dimension, which can be expensive for large
  datasets.
- The list-of-lists form explicitly specifies bin boundaries and avoids a global scan.
- The binning here is for histogram construction and is not the same as WE sampling
  binning.

Output format
-------------
The output file (default: pdist.h5, configurable via -o/--output) contains:

- histograms
  Normalized histograms. The first axis corresponds to iteration; remaining axes
  correspond to the dimensions of the input dataset.

- /binbounds_0, /binbounds_1, ...
  Datasets containing the bin boundaries for each dimension.

- /midpoints_0, /midpoints_1, ...
  Datasets containing the bin midpoints for each dimension.

- n_iter
  Vector of iteration numbers corresponding to the stored histograms (first axis of
  the histograms dataset).

Subsequent processing
---------------------
The produced pdist.h5 file can be plotted with plothist. See plothist --help for
details on generating plots and additional processing.

Parallelization
---------------
This tool supports parallelized binning and data reading. Parallel processing is the
default. For simple cases (e.g., pre-computed input data or a modest number of
segments), serial processing (--serial) may be more efficient.

Command-line options
--------------------
Key options include (the defaults are noted where applicable):

- -o, --output
  Output file name (default: pdist.h5)

- -b, --bins
  Bin specification. See "Histogram binning" for accepted forms:
  - A single integer N
  - A comma-separated sequence N1,N2,...
  - A Python-evaluated list of lists [[...], [...]]

- --construct-dataset MODULE.FUNCTION
  Specify a Python function to construct data to be histogrammed.

- --dsspecs "NAME[,file=FILENAME,slice=SLICE]"

- --serial
  Run in serial rather than parallel mode.

- --help
  Show help message and exit.

Examples
--------
- Default usage (uses pcoord from west.h5):
  .. code-block:: bash

     w_pdist -o pdist.h5

- Use a custom data extractor function:
  .. code-block:: bash

     w_pdist --construct-dataset mymodule.get_pcoord -o pdist.h5

- Use explicit data set specifications:
  .. code-block:: bash

     w_pdist --dsspecs "pcoord" -o pdist.h5
     w_pdist --dsspecs "pcoord,file=west.h5,slice=[0:2]" -o pdist.h5

- Change binning to 50 bins per dimension:
  .. code-block:: bash

     w_pdist -o pdist.h5 -b 50

- Per-dimension bins:
  .. code-block:: bash

     w_pdist -o pdist.h5 -b 50,100

- Nonuniform bin boundaries (explicit boundaries per dimension):
  .. code-block:: bash

     w_pdist -o pdist.h5 -b "[[0,0.5,1.0],[0,1,2,3]]"

- Run serially:
  .. code-block:: bash

     w_pdist --serial -o pdist.h5
