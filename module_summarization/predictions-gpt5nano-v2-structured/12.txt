A quick dive into using the API
===============================

Overview
--------
The API provides a Session object as the primary entry point for interacting with an ftrack server. It handles connection details, authentication, optional automatic data population, plugin discovery, caching, and event hub integration. For low‑level HTTP interactions, a SessionAuthentication helper is available to attach required authentication headers to requests.

Key concepts
- Session: Encapsulates connection details (server URL, API key, user), caching, plugin loading, and optional features like auto-population and event hub support.
- SessionAuthentication: Attaches ftrack-specific authentication headers to HTTP requests.
- auto_populate: When enabled, accessing attributes of retrieved entities will lazy-load data from the server as needed.
- Cache and cache_key_maker: Control how server data is cached locally.
- Event hub: Optional background connection to publish/subscribe to non-local events.
- schema_cache_path: Path to enable schema caching for faster startup and reduced server round-trips.
- plugin_paths and plugin_arguments: Configure plugin discovery and how plugins are invoked.

Getting started
---------------
Basic usage involves creating a Session with the required server URL, API key, and API user. The following minimal example demonstrates the essentials.

.. code-block:: python
   :caption: Minimal session creation

   import ftrack_api

   server_url = "https://your-ftrack-server"
   api_key = "your-api-key"
   api_user = "your-username"

   session = ftrack_api.Session(
       server_url=server_url,
       api_key=api_key,
       api_user=api_user
   )

Environment variable fallbacks
------------------------------
If you prefer not to pass credentials directly, the following environment variables can be used:

- FTRACK_SERVER for the server URL
- FTRACK_API_KEY for the API key
- FTRACK_API_USER for the API user

The Session constructor will read these automatically when explicit values are not provided.

Authentication helpers
----------------------
For scenarios outside the Session, you can construct a requests-compatible authentication object to attach ftrack credentials to HTTP requests:

.. code-block:: python
   :caption: Using SessionAuthentication with requests

   from ftrack_api import SessionAuthentication
   import requests

   auth = SessionAuthentication(api_key="your-api-key", api_user="your-user")
   response = requests.get("https://your-ftrack-server/api/endpoint", auth=auth)

Session configuration and features
----------------------------------

auto_populate
-------------
- Description: When True (default), accessing attributes of entities will lazy-fetch data from the server if not already loaded.
- Usage: You can disable this behavior by setting auto_populate=False when constructing the Session.

Example:
.. code-block:: python
   :caption: Disable auto-population

   session = ftrack_api.Session(
       server_url=server_url,
       api_key=api_key,
       api_user=api_user,
       auto_populate=False
   )

plugin loading
--------------
- plugin_paths: List of filesystem paths to search for plugins during discovery.
- plugin_arguments: Optional mapping of keyword arguments passed to plugin registration functions. If a plugin cannot accept all provided arguments, the mechanism will trim to compatible ones with a warning.

Caching
-------
- cache: An instance implementing the ftrack_api.cache.Cache interface, or a callable returning a suitable cache. If not provided, a default layered cache is used with a top-level MemoryCache.
- cache_key_maker: Responsible for generating cache keys. The default is StringKeyMaker.

Examples:
.. code-block:: python
   :caption: Custom cache and key maker

   from ftrack_api.cache import MemoryCache, StringKeyMaker

   memory_cache = MemoryCache()
   key_maker = StringKeyMaker(prefix="my_session")

   session = ftrack_api.Session(
       server_url=server_url,
       api_key=api_key,
       api_user=api_user,
       cache=memory_cache,
       cache_key_maker=key_maker
   )

auto_connect_event_hub
----------------------
- If True, an embedded event hub will automatically connect to the event server in the background, enabling publish/subscribe to non-local events. If False, local events are supported until you connect manually via EventHub.connect().

schema caching
--------------
- schema_cache_path: Path to a folder to cache schema data for faster startup. If not set, the environment variable FTRACK_API_SCHEMA_CACHE_PATH is used; if that is not set, a temporary directory may be used. Set to False to disable schema caching entirely.

Examples:
.. code-block:: python
   :caption: Enable schema caching

   session = ftrack_api.Session(
       server_url=server_url,
       api_key=api_key,
       api_user=api_user,
       schema_cache_path="/path/to/schema_cache"
   )

Working with the API
--------------------
- Entity access: With auto_populate enabled, entity attributes may be fetched lazily from the server when accessed. This reduces initial payloads and improves startup time for large datasets.
- Event hub: If enabled, the session will manage a background connection for event publishing and subscription. Subscribing to local events does not require a connected hub, but interacting with non-local events benefits from an active hub.

Common usage patterns
---------------------
- Minimal session in a script using environment variables.
- Advanced session with custom caching and plugin behavior.
- Connecting to the event hub for real-time event processing.

Example: complete minimal script
--------------------------------
.. code-block:: python
   :caption: End-to-end minimal usage

   import ftrack_api

   # Prefer environment-based configuration when possible
   session = ftrack_api.Session(
       server_url=os.environ.get('FTRACK_SERVER', 'https://your-ftrack-server'),
       api_key=os.environ.get('FTRACK_API_KEY', 'your-api-key'),
       api_user=os.environ.get('FTRACK_API_USER', 'your-username')
   )

   # Access entities with lazy loading (if auto_populate is True by default)
   # e.g., entity = session.find('Project', id='12345')
   # name = entity.name

   # Optional: explicitly connect to the event hub
   session = ftrack_api.Session(
       server_url=server_url,
       api_key=api_key,
       api_user=api_user,
       auto_connect_event_hub=True
   )
