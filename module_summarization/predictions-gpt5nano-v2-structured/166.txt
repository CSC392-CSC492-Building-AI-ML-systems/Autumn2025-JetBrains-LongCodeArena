filenode - simulating a filesystem with PyTables
-----------------------------------------------

Overview
--------

The filenode module provides a file-like interface to PyTables nodes. It enables you to store binary data inside a PyTables (.h5) file and interact with that data through a standard Python file-like API. A brand-new file node can be created with new_node() and later opened with open_node() for read-only or read/write access. Read access is always available; write access is allowed only when the node is opened with a mode that permits writing (e.g., append or read/write modes). This module is designed for binary I/O only.

Key concepts
------------

- Node: A PyTables dataset used as a binary blob that can be read and written through a file-like interface.
- New node: Creates a fresh file node inside a PyTables file, ready for I/O.
- Open node: Opens an existing file node (created by new_node()) for reading and/or writing.
- IO compliance: The filenode implementation is aligned with Python’s io module interfaces (as of version 3.0, the module was rewritten to be fully compliant with the io interfaces).
- Versioning: The node carries a NODE_TYPE attribute and a NODE_TYPE_VERSION attribute. Supported versions include 1 and 2. The version determines data shape handling and behavior.

Usage overview
--------------

- Create a new file node that can be read and written like a Python file
- Open an existing file node for read-only or read/write access
- Perform binary I/O through a file-like interface
- Use standard file operations such as read, write, seek, tell, truncate, and close

Code examples
-------------

Note: The exact signatures for new_node() and open_node() depend on how you manage the PyTables file context. The examples illustrate the intended usage pattern rather than exact function signatures.

Create a new filenode inside an existing PyTables file and obtain a file-like interface:

.. code-block:: python

   # Pseudo-code illustrating the intended usage
   import tables as tb
   from filenode import new_node, open_node

   # Open or create a PyTables file
   with tb.open_file('storage.h5', mode='a') as h5file:
       # Create a brand-new filenode inside the PyTables file
       node = new_node(h5file, '/filenodes/data')  # details depend on actual API

       # Open the filenode for read/write using its file-like interface
       with open_node(h5file, '/filenodes/data', mode='r+') as fnode:
           # Write data
           fnode.write(b'Hello PyTables filenode!')
           # Seek and read
           fnode.seek(0)
           data = fnode.read(1024)
           # data is a bytes-like object

Open an existing filenode for read-only access:

.. code-block:: python

   import tables as tb
   from filenode import new_node, open_node

   with tb.open_file('storage.h5', mode='r') as h5file:
       # Open an existing filenode (previously created with new_node)
       with open_node(h5file, '/filenodes/data', mode='r') as fnode:
           fnode.seek(0)
           first_chunk = fnode.read(4096)
           # first_chunk is a bytes-like object

API reference
-------------

Constants
^^^^^^^^^

- NodeType: Value for NODE_TYPE node system attribute. Indicates that the node represents a file.
- NodeTypeVersions: Supported values for NODE_TYPE_VERSION node system attribute (e.g., [1, 2]).

Classes
^^^^^^^

RawPyTablesIO
~~~~~~~~~~~~~

Base class that provides a raw binary I/O interface on top of a PyTables node.

- Inheritance: io.RawIOBase
- Purpose: Expose a file-like binary interface to a PyTables node so that data can be read and written as with a normal Python file.

Properties
++++++++++

mode
    Read/write mode in effect for this file-like object.

Methods (highlights)
++++++++++++++++++++

tell()
    Return the current stream position.

seek(pos, whence=0)
    Change stream position. whence values:
    - 0: start of stream
    - 1: current position
    - 2: end of stream

seekable()
    Return True, as random access is supported.

fileno()
    Return the underlying file descriptor if available.

close()
    Flush and close the IO object; releases the node reference.

flush()
    Flush write buffers to the underlying PyTables file (if writable).

truncate(pos=None)
    Truncate the file to a given size. Only growth (expansion) is supported.
    If pos is None, truncate to the current position.

readable()
    Return True if the file is opened for reading.

writable()
    Return True if the file is opened for writing.

readinto(b)
    Read up to len(b) bytes into the provided buffer b. Returns the number of bytes read.

readline(limit=-1)
    Read and return a line from the stream, up to limit bytes if specified.

Notes and behavior
------------------

- Binary I/O only: This interface is optimized for binary data storage within a PyTables dataset.
- Access semantics: Read access is always available; write access is controlled by the mode used to open the node.
- Append semantics: Writing to a newly created or opened filenode in append mode appends data to the end of the file node.
- Truncation: Truncation is meaningful primarily for growth; truncating to a smaller size is not supported for in-place data modification.
- Versioning: The behavior may depend on NODE_TYPE_VERSION (1 or 2). The size-to-shape mapping guides how data is interpreted when reading and writing.

Limitations
-----------

- This interface focuses on binary I/O for file-like interaction with PyTables nodes. Advanced text I/O or high-level filesystem semantics are not the primary goal.
- The available methods and exact signatures depend on the underlying PyTables node API and the specific version of PyTables in use.
