augmentations.rst
===================


Module overview
---------------
This module defines a standard suite of data augmentations used by NVIDIA DALI AutoAugment policies
(ImageNet, RandAugment, TrivialAugmentWide). The augmentations are implemented as wrappers around
DALI operators and are exposed via the @augmentation decorator.

Key ideas
- The @augmentation decorator automatically computes the actual transformation parameter from a magnitude
  index (magnitude_bin) and the total number of magnitude bins (num_magnitude_bins).
- Magnitudes are derived from mag_range and mapped through mag_to_param, with an optional random sign flip
  (randomly_negate).
- Examples in this module demonstrate typical augmentation setups; users can customize ranges or mapping
  functions to tailor magnitudes for their domain.

How to use
- Each augmentation function is designed to be called with a batch of samples as the first argument.
- After applying the decorator, the function accepts:
  - a single positional argument: batch of samples
  - magnitude_bin and num_magnitude_bins to compute the actual parameter
- The decorator handles converting magnitude_bin to the concrete parameter value (e.g., shear, offset, angle).

Example usage
- Basic usage (default mapping and magnitudes):
  - out = shear_x(batch, magnitude_bin=3, num_magnitude_bins=10)

- Customizing magnitude range or mapping:
  - my_shear_x = shear_x.augmentation(mag_range=(0, 0.5), randomly_negate=True)
  - out = my_shear_x(batch, magnitude_bin=4, num_magnitude_bins=8)

- Custom translation without needing shape-based offsets:
  - out = translate_x_no_shape(batch, offset=15)

Notes
- The module raises if NumPy is not installed, since some augmentations rely on NumPy (e.g., color weights).
- Data layout considerations: some augmentations (e.g., contrast) assume FHWC or HWC layouts.

Available augmentations
-----------------------

Shear
^^^^^^
- shear_x
  - Parameters (via decorator): mag_range=(0, 0.3), randomly_negate=True, mag_to_param=warp_x_param
  - Signature: shear_x(data, shear, fill_value=128, interp_type=None)
  - Behavior: Applies a horizontal shear. Uses the computed shear value as the transformation parameter.

- shear_y
  - Parameters: mag_range=(0, 0.3), randomly_negate=True, mag_to_param=warp_y_param
  - Signature: shear_y(data, shear, fill_value=128, interp_type=None)
  - Behavior: Applies a vertical shear.

Translation
^^^^^^^^^^^
- translate_x
  - Parameters: mag_range=(0, 1.0), randomly_negate=True, mag_to_param=warp_x_param
  - Signature: translate_x(data, rel_offset, shape, fill_value=128, interp_type=None)
  - Behavior: Horizontal translation proportional to image width (uses shape to compute absolute offset).

- translate_x_no_shape
  - Parameters: mag_range=(0, 250), randomly_negate=True, mag_to_param=warp_x_param, name="translate_x"
  - Signature: translate_x_no_shape(data, offset, fill_value=128, interp_type=None)
  - Behavior: Horizontal translation with explicit offset (no shape information required).

- translate_y
  - Parameters: mag_range=(0, 1.0), randomly_negate=True, mag_to_param=warp_y_param
  - Signature: translate_y(data, rel_offset, shape, fill_value=128, interp_type=None)
  - Behavior: Vertical translation proportional to image height.

- translate_y_no_shape
  - Parameters: mag_range=(0, 250), randomly_negate=True, mag_to_param=warp_y_param, name="translate_y"
  - Signature: translate_y_no_shape(data, offset, fill_value=128, interp_type=None)
  - Behavior: Vertical translation with explicit offset (no shape information required).

Rotation
^^^^^^^^
- rotate
  - Parameters: mag_range=(0., 30), randomly_negate=True
  - Signature: rotate(data, angle, fill_value=128, interp_type=None, rotate_keep_size=True)
  - Behavior: Rotation by a given angle (degrees). Keeps output size by default.

Brightness, contrast, color, and sharpness
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
- shift_enhance_range(magnitude)
  - Helper: Converts 0–0.9 magnitude to enhancement magnitude around 1 (the neutral value).
  - Behavior: Enhancement operations (brightness, contrast, color, sharpness) use magnitudes in [0, 2] with a neutral value of 1.

- brightness
  - Parameters: mag_range=(0, 0.9), randomly_negate=True, mag_to_param=shift_enhance_range
  - Signature: brightness(data, parameter)
  - Behavior: Adjusts image brightness.

- contrast
  - Parameters: mag_range=(0, 0.9), randomly_negate=True, mag_to_param=shift_enhance_range
  - Signature: contrast(data, parameter)
  - Behavior: Adjusts image contrast. Uses a center computed from the image mean to align with the PIL approach.

  Notes:
  - The implementation computes a per-channel mean and a center, then applies the transformation:
    center + (data - center) * parameter
  - The transformation is cast to the input data type to preserve channels/layout.

- color and sharpness
  - Placeholder in this module: see the source for potential future augmentation functions following the same decorator pattern.

Internal helpers
----------------
- warp_x_param(magnitude)
  - Returns parameter for horizontal operations: [magnitude, 0]

- warp_y_param(magnitude)
  - Returns parameter for vertical operations: [0, magnitude]

- shift_enhance_range(magnitude)
  - Explanation: Maps magnitude in [0, 0.9] to an enhancement factor around 1, with a neutral value at 1.

Implementation notes
--------------------
- The augmentations rely on NVIDIA DALI operators (fn and related transforms).
- Each augmentation is defined with a preconfigured setup using the @augmentation decorator.
- Example configurations can be easily overridden by creating a modified augmentation via .augmentation(...).

API reference (signatures)
--------------------------
- shear_x(data, shear, fill_value=128, interp_type=None)
- shear_y(data, shear, fill_value=128, interp_type=None)
- translate_x(data, rel_offset, shape, fill_value=128, interp_type=None)
- translate_x_no_shape(data, offset, fill_value=128, interp_type=None)  # name: translate_x
- translate_y(data, rel_offset, shape, fill_value=128, interp_type=None)
- translate_y_no_shape(data, offset, fill_value=128, interp_type=None)  # name: translate_y
- rotate(data, angle, fill_value=128, interp_type=None, rotate_keep_size=True)
- brightness(data, parameter)
- contrast(data, parameter)

Examples
--------
Code snippets illustrating usage and customization:

.. code-block:: python

   from augmentations import shear_x, translate_y, brightness

   # Basic usage (default magnitudes)
   out1 = shear_x(batch, magnitude_bin=2, num_magnitude_bins=10)

   # Customize a specific augmentation
   my_shear = shear_x.augmentation(mag_range=(0, 0.5), randomly_negate=True)
   out2 = my_shear(batch, magnitude_bin=4, num_magnitude_bins=8)

   # Brightness example
   out3 = brightness(batch, magnitude_bin=3, num_magnitude_bins=10)

Notes on layout
---------------
- Contrast augmentation assumes FHWC or HWC layouts when computing the mean center.
- Other augmentations operate on tensors shaped as NHWC or similar, and rely on DALI's numeric types.
