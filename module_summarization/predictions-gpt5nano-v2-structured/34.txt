w_stateprobs
==============

Overview
--------
w_stateprobs is a deprecated wrapper around the w_direct tool's trace subcommand. It provides backwards-compatible access to the functionality that computes average state populations and associated errors from weighted-ensemble (WE) data, including optional evolution-mode views of the populations over sliding windows.

This command is a shim to ensure compatibility and can be invoked as a regular tool (not as a subcommand). It forwards to the underlying w_direct trace implementation. A deprecation warning is emitted when invoked, and users are encouraged to migrate to using w_direct directly.

Key points
- Purpose: Calculate average state populations and confidence intervals (CIs) for path-tracing kinetics analysis using WE data.
- Backwards compatibility: Implemented as a shim for the regular tool interface; equivalent to running w_direct trace with the same options.
- Output: Default output file is stateprobs.h5. Default kinetics file is assign.h5.
- Evolution mode: Optional dataset showing state-population evolution over iteration windows when --evolution-mode is specified.

Usage
-----
w_stateprobs trace [options]

Notes:
- If you run w_stateprobs without a subcommand, the tool automatically uses the trace subcommand.
- This command accepts the same options as the underlying w_direct trace implementation (see “Command-line options” below).

Output format
-------------
The main output file (specified with -o/--output, by default stateprobs.h5) contains:

- /avg_state_pops [state]
  - Population of each state across the full analysis range.

If evolution mode is enabled (via --evolution-mode), an additional dataset is created:

- /state_pop_evolution [window][state]
  - Population data for sliding windows of varying width (or fixed width, depending on mode).

Dataset fields (for each window and state)
- iter_start: Integer. Iteration where the averaging window begins (inclusive).
- iter_stop: Integer. Iteration where the averaging window ends (exclusive).
- expected: Floating-point. Mean rate within this window (in inverse tau).
- ci_lbound: Floating-point. Lower bound of the CI on the rate (inverse tau).
- ci_ubound: Floating-point. Upper bound of the CI on the rate (inverse tau).
- corr_len: Integer. Correlation length of the rate within this window (in tau).

Attributes for both datasets
- mcbs_alpha: Floating-point. Alpha value for CI calculation (e.g., alpha=0.05 for 95% CI).
- mcbs_nsets: Integer. Number of bootstrap realizations used to generate CIs.
- mcbs_acalpha: Floating-point. Alpha value for determining correlation lengths.

Examples
--------

Basic usage (default output and no evolution):
.. code-block:: console

   w_stateprobs trace -o stateprobs.h5

Evolutionary windows (cumulative):
.. code-block:: console

   w_stateprobs trace -o stateprobs.h5 --evolution-mode cumulative --start-iter 0 --step-iter 10

Evolutionary windows (blocked):
.. code-block:: console

   w_stateprobs trace -o stateprobs.h5 --evolution-mode blocked --start-iter 100 --step-iter 20

Migration note
-------------
This command is deprecated. Please use w_direct instead:

.. code-block:: console

   w_direct trace -o stateprobs.h5 <other options>

Rationale
---------
- w_stateprobs exists to preserve compatibility with existing workflows that reference the w_stateprobs command.
- It delegates to the more general and actively supported w_direct tool, which provides the same functionality in a unified interface.

See Also
--------
- w_direct: primary tool for direct execution of WE-based analyses.
- w_assign: related utility for macrostate assignments used by state-probability analyses.
