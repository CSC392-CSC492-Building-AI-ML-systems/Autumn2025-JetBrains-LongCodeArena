Generate example for this module
---------------------------------

This module defines the base class Family for one-parameter exponential family
distributions used by generalized linear models (GLMs). It provides utilities
for link management, variance functions, IRLS weights, and measures such as
deviance, deviance residuals, and fitted values.

Key concepts
- Family: Base class representing a one-parameter exponential family.
- Link: Transformation between the linear predictor and the mean of the distribution.
- Variance: Variance function Var(mu) as a function of the mean.
- IRLS: Iteratively Reweighted Least Squares quantities (weights, starting values).

Usage overview
- Create a concrete family by supplying a link instance and a variance function.
- Use starting_mu to obtain a starting value for the mean mu in IRLS.
- Use weights(mu) to obtain IRLS weights at a given mu.
- Use deviance(endog, mu, ...) and resid_dev(endog, mu, ...) to assess fit.
- Use fitted(lin_pred) to obtain mu from a linear predictor (via the link).

Example
The following example demonstrates a minimal, self-contained usage pattern of the
Family base class with small, self-defined link and variance components. In
real code, you would typically use the concrete link/variance implementations
provided by the library (e.g., Poisson with a log link, Binomial with a logit
link, etc.).

.. code-block:: python

    import numpy as np
    from gee import Family
    # In real usage, import the actual link/variance implementations, e.g.:
    # from gee import links as L
    # from gee import varfuncs as V

    # Minimal demonstration classes (duck-typing; override with real implementations)
    class IdentityLink:
        """Inverse link: mu = eta (identity transformation)."""
        def __call__(self, mu):  # maps mu to mu (identity)
            return mu
        def deriv(self, mu):
            """Derivative g'(mu) = 1 for all mu."""
            return np.ones_like(mu)

    class ConstantVariance:
        """Constant variance function Var(mu) = 1."""
        def __call__(self, mu):
            return np.ones_like(mu)

    # Instantiate a simple (dummy) family
    family = Family(link=IdentityLink(), variance=ConstantVariance())

    # Example data
    y = np.array([0., 1., 2., 3.])

    # Starting value for mu in IRLS
    mu_0 = family.starting_mu(y)

    # Weights for IRLS
    w = family.weights(mu_0)

    # Deviance measures
    D = family.deviance(y, mu_0)

    # Deviance residuals
    resid = family.resid_dev(y, mu_0)

    # Fitted values from a linear predictor (example lin_pred)
    lin_pred = np.array([0.0, 0.5, 1.0, 1.5])
    mu_fitted = family.fitted(lin_pred)

    print("starting_mu:", mu_0)
    print("weights:", w)
    print("deviance:", D)
    print("deviance residuals:", resid)
    print("fitted mu from lin_pred:", mu_fitted)

Notes
- The IdentityLink and ConstantVariance classes above are placeholders to illustrate the API of Family. In practice, use the concrete link and variance implementations provided by the library (for example, a log link with a Poisson variance, or a logit link with a binomial variance).
- The example focuses on demonstrating how to access the key methods of the Family class:
  starting_mu, weights, deviance, resid_dev, and fitted.
