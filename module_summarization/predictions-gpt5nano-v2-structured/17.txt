w_assign
========

Assign walkers to bins and emit an assign.h5 file for downstream analysis.

Overview
--------
The w_assign command assigns each walker (trajectory) to a bin (based on a progress coordinate or other user-supplied data) and generates an HDF5 file (default: assign.h5) containing per-iteration, per-segment, and per-timepoint bin assignments, along with labeled populations by macrostate. This consistent, full assignment is required for analyses that trace trajectories back to their originating bin.

Key concepts:
- Each trajectory timepoint is assigned to a bin.
- Optional macrostates can be defined so that each timepoint is labeled with the most recently visited macrostate.
- The output file contains datasets for assignments, npts, nsegs, and labeled populations, plus bin labels.

Source data
-----------
Source data for the progress coordinate (or other data) can be provided in one of two ways:

- Custom dataset function (--construct-dataset)
  - Provide a Python function using MODULE.FUNCTION syntax.
  - The function is called as function(n_iter, iter_group) and must return data indexable as [segment][timepoint][dimension].

- Data set specifications (--dsspecs)
  - Pass one or more data set specifications of the form NAME[,file=FILENAME,slice=SLICE].
  - NAME is a dataset in the HDF5 file (default: west.h5); slice selects elements along axes (e.g., to pick a column).
  - Example: --dsspecs pcoord[,file=west.h5,slice=[0:2]].
  - If omitted, the default progress coordinate dataset pcoord is used.

- Default: if neither construct-dataset nor dsspecs is provided, the pcoord dataset is used.

Specifying macrostates
---------------------
Kinetic macrostates can be defined to label trajectories by the most recently visited macrostate at each timepoint. Three ways:

1) States for single bins on the command line
   - Use --states with multiple arguments, each a coordinate tuple, optionally prefixed with a label.
   - Examples:
     - bound:1.0
     - unbound:(2.5,2.5)
   - unlabeled states receive automatic names stateN, where N is the position in the list.

2) States from a YAML file
   - Use --states-from-file <FILE.yaml>.
   - The YAML file defines a list of states with coordinates. Each state contains:
     - label: string (optional)
     - coords: list of coordinate tuples
   - Bins containing those coordinates define the corresponding state.

   Example YAML:
   ---
   states:
     - label: unbound
       coords:
         - [9.0, 1.0]
         - [9.0, 2.0]
     - label: bound
       coords:
         - [0.1, 0.0]

3) States from a function
   - Use --states-from-function MODULE.FUNCTION.
   - The function is called with the bin mapper and must return a list of dictionaries, each describing a state.
   - Each dictionary must contain:
     - coords: a vector of coordinate tuples
     - label: optional state name

Output format
-------------
The output file (default: assign.h5) contains the following attributes and datasets:

Attributes
- nbins (Integer): Number of valid bins. Assignments range from 0 to nbins-1.
- nstates (Integer): Number of valid macrostates (may be zero). Trajectory ensemble assignments range from 0 to nstates-1 when states are defined.

Datasets
- /assignments [iteration][segment][timepoint]
  Integer per-segment and per-timepoint bin assignments.

- /npts [iteration]
  Integer number of timepoints in each iteration.

- /nsegs [iteration]
  Integer number of segments in each iteration.

- /labeled_populations [iterations][state][bin]
  Floating-point per-iteration and per-timepoint bin populations labeled by the most recently visited macrostate.
  The last state entry (nstates-1) corresponds to trajectories initiated outside of a defined macrostate.

- /bin_labels [bin]
  Labels for each bin (human-readable or symbolic tags).

Notes
-----
- For consistency in downstream analyses, all timepoints of the trajectory data must be assigned, even if only a subset is used later.
- The last macrostate entry in /labeled_populations corresponds to trajectories not currently in any defined macrostate.
- The w_assign tool benefits from progress indicators and parallel execution support via WEST’s tooling, enabling scalable assignment.

Examples
--------

Minimal usage (default output file assign.h5):
.. code-block:: console

   w_assign

Specify the source data via the pcoord dataset (default behavior):
.. code-block:: console

   w_assign --dsspecs pcoord

Provide a custom macrostate specification via a YAML file:
.. code-block:: console

   w_assign --dsspecs pcoord --states-from-file macrostates.yaml

Define macrostates using a Python function:
.. code-block:: console

   w_assign --dsspecs pcoord --states-from-function mymodule.define_macrostates

Explicitly name the output file:
.. code-block:: console

   w_assign --dsspecs pcoord --output assign.h5

Construct a dataset with a custom function:
.. code-block:: console

   w_assign --output assign.h5 --construct-dataset mymodule.extract_pcoords

Additional notes
----------------
- The output file is intended for use with subsequent WESTPA analyses (e.g., kinetic tracing, kinmat, kintrace).
- See related tools (e.g., w_kintrace, w_kinmat) for kinetic analyses that consume the assign.h5 file.
