Publishing for web review
=========================

Overview
--------

Publishing for web review enables external stakeholders to view, comment on, and provide feedback on content via a secure, web-based interface. This feature focuses on creating a shareable review session (or object) that contains the assets to be reviewed and exposes a web URL for reviewers.

Key concepts
------------

- Web Review: a dedicated construct that groups content to be reviewed and exposes a public or protected web URL.
- Review Items: the content included in the web review (e.g., versions, assets, or other entities).
- Access controls: options to limit who can view the review (password, expiration, user-based permissions).
- Workflow: creation -> addition of items -> configuration -> publishing -> reviewer feedback -> update or retire.

Usage outline
-------------

- Prepare the content you want reviewed.
- Create a web review session/object.
- Attach the items to be reviewed.
- Configure access and expiration if desired.
- Publish to generate a shareable web URL.
- Distribute the URL to reviewers and collect feedback.
- Update, resubmit, or retire the review as needed.

API overview (typical workflow)
-------------------------------

Note: The exact API surface may vary in your environment. The following illustrates the common workflow for publishing web reviews via the API.

- Create a web review object
- Add items to the review
- Publish to generate the URL
- Retrieve the web URL for sharing

Example
-------

Here is a representative Python usage example (pseudo-code; adapt to your API surface):

.. code-block:: python

    from ftrack_api import Session

    # Establish a session (replace with your server, key, and user)
    session = Session(
        server_url='https://your-ftrack-server',
        api_key='YOUR_API_KEY',
        api_user='YOUR_USERNAME'
    )

    # Create a web review session/object
    web_review = session.create('WebReview', {
        'name': 'Q1 2025 External Review',
        'description': 'Review for external stakeholders',
        'expires_at': '2025-04-30T23:59:59Z'  # optional
    })

    # Attach items to be reviewed (versions, assets, etc.)
    web_review.add_item({'type': 'Version', 'id': 'v123'})
    web_review.add_item({'type': 'Version', 'id': 'v124'})

    # Configure access (optional)
    web_review.set_access_control(password='secret', allowed_groups=['external_reviewers'])

    # Publish to generate and obtain the URL
    web_review.publish()

    # Obtain the shareable URL
    print('Web review URL:', web_review.url)

Configuration and prerequisites
-------------------------------

- Server readiness: Ensure the ftrack server supports web reviews and that the feature is enabled for your project.
- Permissions: You need appropriate permissions to create, configure, and publish web reviews.
- Content readiness: Prepare the items you want reviewed (e.g., versions) before publishing.
- Access options: Consider whether to password-protect the review, set an expiration date, or restrict who can view.

Security and best practices
---------------------------

- Use expiration dates to limit how long a review URL remains active.
- Prefer password protection for external reviews and share credentials securely.
- Limit access to the minimum necessary groups or individuals.
- Regularly audit active web reviews and retract those that are no longer needed.

Troubleshooting
---------------

- Publish failures: Check server connectivity, permissions, and whether the web review feature is enabled on the server.
- URL not available: Ensure the review has been published and that it has not been retracted or expired.
- Access denied for reviewers: Verify password and group permissions, and confirm reviewer accounts (if applicable).
