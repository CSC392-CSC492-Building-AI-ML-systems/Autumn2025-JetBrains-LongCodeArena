Cross decomposition
===================

Cross decomposition models in scikit-learn provide a family of methods to analyze and predict the relationship between two data matrices X and Y by extracting latent variables that capture their shared structure. These methods are built around Partial Least Squares (PLS) and include three public classes:

- PLSCanonical
- PLSRegression
- PLSSVD

Key ideas
---------

- Data: X ∈ R^(n_samples × n_features) and Y ∈ R^(n_samples × n_targets)
- Goal: Find latent components that maximize (or align with) the relationship between X and Y
- Components: For each component, the model learns:
  - X scores (t) and X loadings (p)
  - Y scores (u) and Y loadings (q)
  - Weights for projecting X to t (w) and Y to u (c)
- Deflation: After extracting a component, X and Y are deflated to remove the captured covariance, enabling subsequent components to capture new structure.
- Centering and scaling: Both X and Y are centered; optional scaling (scale=True) standardizes features to unit variance before computing components.
- Algorithms: Internally, cross decomposition can compute the first latent vectors either via an iterative power-method approach or via a direct SVD on the cross-covariance, with the choice impacting numerical stability and performance.

Public classes
--------------

- PLSCanonical: Performs a symmetric cross-decomposition akin to Canonical Correlation Analysis (CCA) but within the PLS framework, extracting pairs of canonical variates that maximize the correlation between X and Y.
- PLSRegression: Finds latent components that maximize covariance between X and Y with the aim of predicting Y from X (regression setting). Suitable when Y is continuous or multivariate.
- PLSSVD: A variant that leverages a singular value decomposition (SVD) approach on the cross-covariance to obtain the latent structure.

Common API and usage
---------------------

All classes share a similar API. Typical usage involves:

- Importing the desired model
- Fitting with training data X and Y
- Transforming data to latent space (scores)
- Reconstructing or predicting Y for new X (PLSRegression)

Example: PLSRegression
----------------------

```python
from sklearn.model_selection import train_test_split
from sklearn.cross_decomposition import PLSRegression
import numpy as np

# Example data (n_samples, n_features) and (n_samples, n_targets)
X = np.random.randn(100, 20)
Y = np.random.randn(100, 3)

# Create and fit the model
pls = PLSRegression(n_components=2, scale=True)
pls.fit(X, Y)

# Predict on new data
Y_pred = pls.predict(X)

# Access latent components and loadings
X_scores = pls.x_scores_
Y_scores = pls.y_scores_
X_loadings = pls.x_loadings_
Y_loadings = pls.y_loadings_
```

Example: PLSCanonical
---------------------

```python
from sklearn.cross_decomposition import PLSCanonical
import numpy as np

X = np.random.randn(150, 25)
Y = np.random.randn(150, 15)

pls_can = PLSCanonical(n_components=3, scale=True)
pls_can.fit(X, Y)

# Canonical variates (scores)
X_scores_can = pls_can.x_scores_
Y_scores_can = pls_can.y_scores_
```

Example: PLSSVD
----------------

```python
from sklearn.cross_decomposition import PLSSVD
import numpy as np

X = np.random.randn(80, 40)
Y = np.random.randn(80, 6)

plssvd = PLSSVD(n_components=2, scale=True)
plssvd.fit(X, Y)

# Access resulting components
X_scores = plssvd.x_scores_
Y_scores = plssvd.y_scores_
```

Internal notes
--------------

- Core computations rely on:
  - _get_first_singular_vectors_power_method: an iterative approach (power method) to compute the leading singular vectors when appropriate.
  - _get_first_singular_vectors_svd: a direct SVD-based computation of the leading singular vectors.
- Centering and optional scaling are applied before component extraction via _center_scale_xy.
- Helper utilities handle numerical robustness (e.g., svd flipping via svd_flip, 1D svd normalization).

Input and output shapes
-----------------------

- After fitting, attributes such as x_scores_, y_scores_, x_loadings_, and y_loadings_ are available.
- Shapes:
  - X_scores_: (n_samples, n_components)
  - Y_scores_: (n_samples, n_components)
  - X_loadings_: (n_features, n_components)
  - Y_loadings_: (n_targets, n_components)

Implementation notes
--------------------

- The module is designed as part of sklearn.base and follows scikit-learn conventions for estimators (fit/transform/predict where applicable).
- It supports multi-output Y and works with both univariate and multivariate responses.
- By default, data are centered; scaling is optional via the scale parameter.

References
----------

- Underlying theory follows Partial Least Squares and its cross-decomposition variants, including symmetric (canonical-like) and regression-oriented formulations. The implementation integrates both iterative and direct SVD approaches to extract latent structure from X and Y.
