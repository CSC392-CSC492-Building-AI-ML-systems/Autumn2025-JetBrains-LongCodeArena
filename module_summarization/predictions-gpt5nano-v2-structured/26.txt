w_kinavg
==========

Overview
--------

w_kinavg is a deprecated shim that provides a command-line interface for
averaging and confidence-interval estimation in path-tracing kinetics analysis
using weighted-ensemble data. It delegates to the core w_direct machinery
through the WDirect command class, and is intended for backward compatibility.

Deprecation notice
------------------

.. warning::
   The w_kinavg command is deprecated. Please use w_direct instead.

Key points
----------

- Default files:
  - Kinetics input: kintrace.h5
  - Output file: kinavg.h5
- Purpose: compute average state-to-state rates and associated confidence
  intervals over selected windows of iterations. In trace mode, also compute
  total and conditional fluxes between states.
- Evolution mode: optional analysis over windows of varying width or fixed width
  as a function of iteration, controlled by --evolution-mode, --start-iter,
  and --step-iter.
- Input data: Requires pre-generated input data (e.g., assignments.h5 and
  kintrace.h5 or kinmat.h5). See the related w_assign and w_kinetics tools for
  generation guidance.

What w_kinavg does
------------------

- Computes average rates between states over a user-specified window of iterations.
- Estimates confidence intervals (MCBS bootstrap-based) for those rates.
- In trace mode, provides:
  - /avg_total_fluxes: total flux into each state over the window
  - /avg_conditional_fluxes: state-to-state fluxes over the window
- Optional evolution-mode analysis (if requested):
  - /rate_evolution: rate evolution across windows of varying width
  - When in trace mode, /target_flux_evolution and /conditional_flux_evolution are also generated to reflect flux evolution.

Output data file format
-----------------------

The output is an HDF5 file (default kinavg.h5) containing the following datasets.

- Primary dataset
  - /avg_rates [state, state]
    - Structured dataset of state-to-state rates over the selected window.

- Trace-mode extras
  - /avg_total_fluxes [state]
  - /avg_conditional_fluxes [state, state]

- Evolution-mode datasets (if requested)
  - /rate_evolution [window][state][state]
  - If in trace mode:
    - /target_flux_evolution [window, state]
    - /conditional_flux_evolution [window, state, state]

Dataset structure
-----------------

Each window yields a row with the following fields:

- iter_start
  - Integer. Iteration at which the averaging window begins (inclusive).

- iter_stop
  - Integer. Iteration at which the averaging window ends (exclusive).

- expected
  - Floating-point. Expected (mean) value of the rate within the window
    (units: inverse tau).

- ci_lbound
  - Floating-point. Lower bound of the confidence interval on the rate
    within the window (units: inverse tau).

- ci_ubound
  - Floating-point. Upper bound of the confidence interval on the rate
    within the window (units: inverse tau).

- corr_len
  - Integer. Correlation length of the rate within the window (in units of tau).

Attributes (per dataset)
------------------------

Each dataset is annotated with bootstrap and correlation metadata:

- mcbs_alpha
  - Floating-point. Alpha value for the confidence intervals (e.g., 0.05 for a 95% CI).

- mcbs_nsets
  - Integer. Number of bootstrap data sets used to generate CIs.

- mcbs_acalpha
  - Floating-point. Alpha value for determining correlation lengths.

Input and usage notes
---------------------

- Pre-generated inputs: You must provide assignments and kinetics data
  generated by other WestPA tools (e.g., w_assign, w_kinetics).
  See those tools’ help text for details on producing the required input files.

- Default file names:
  - Kinetics input: kintrace.h5
  - Output: kinavg.h5
  You can override these through the usual tool options (inherited from
  w_direct/DPkinAvg). See the command-line options section below or the
  respective tool help for exact flag names.

- Compatibility:
  - This command is a wrapper around the core w_direct functionality and
    is being kept only for backward compatibility. If you are starting a new
    analysis, prefer using w_direct directly.

Command-line options
--------------------

w_kinavg exposes the same interface as the underlying w_direct-based
kinetics analysis, with defaults set as described above. Since this is a
wrapper, please refer to the w_direct (and the DKinAvg) help text for a full
list of options. The following features are commonly used:

- -o / --output FILE
  - Specify the output HDF5 file (default: kinavg.h5).

- --evolution-mode [cumulative|blocked]
  - Enable evolution-window analysis.

- --start-iter N
  - Starting iteration for evolution or averaging windows.

- --step-iter N
  - Step size (or window width) for evolution analysis.

- Other inputs
  - Input data files (assignments.h5 and kintrace.h5 or kinmat.h5) are required
    and are typically specified via the options defined by the underlying tool.

For exact usage, run:
- w_kinavg --help
- w_direct --help
- w_kinavg trace --help
These will show the precise option names and defaults for your installation.

Examples
--------

Minimal invocation (uses defaults and pre-generated input files in the CWD):

.. code-block:: console

   w_kinavg

Specify an output file explicitly:

.. code-block:: console

   w_kinavg -o kinavg.h5

Enable evolution-mode analysis with a cumulative window and starting at iteration 0:

.. code-block:: console

   w_kinavg --evolution-mode cumulative --start-iter 0 --step-iter 10 -o kinavg.h5

Notes
-----

- The command shown here is deprecated and will be removed in a future WestPA release.
  For new analyses, use w_direct directly (see w_direct --help).
- Datasets and structure described here are generated by the underlying kinetics
  analysis pipelines and may vary with version; consult the help text in your
  installation for the exact schema.
