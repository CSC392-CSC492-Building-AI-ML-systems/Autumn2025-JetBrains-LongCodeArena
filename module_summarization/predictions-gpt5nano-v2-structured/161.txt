Internals: Internal organization of NumPy arrays
===============================================

Overview
--------

NumPy’s core data container is the ndarray, an N-dimensional array that stores elements of a uniform dtype in a single memory buffer. The array object encapsulates metadata describing how to interpret the raw memory, including the shape (dimensions), strides (step sizes in bytes to move along each axis), and the dtype (element type). NumPy supports views into existing arrays, enabling non-copying reshapes and slicing through a shared memory buffer.

Key concepts
------------

- Data buffer: a contiguous block (by default) of memory containing the actual elements.
- Shape: a tuple describing the size along each axis.
- Strides: the number of bytes to move in memory to advance one step along each axis.
- Dtype: describes the element size, kind, and layout; supports structured dtypes with fields.
- Base and views: an array can be a view into another array's data; the original array becomes the base owner of the memory.
- Flags: metadata about contiguity, ownership of memory, writability, alignment, etc.
- Array interfaces: lightweight introspection of memory layout via __array_interface__ and related mechanisms.

 ndarray structure at a glance
 ----------------------------

- Data buffer pointer: points to the start of the memory block containing elements.
- Shape: (n0, n1, ..., n{N-1})
- Strides: (s0, s1, ..., s{N-1}) where moving one step along axis i increases the memory address by si bytes.
- Dtype: describes element size (itemsize), byte order, and field structure if structured.

Memory layout and contiguity
----------------------------

- C-contiguous (row-major): last axis varies fastest; strides are arranged so that the last axis has itemsize as its stride.
- Fortran-contiguous (column-major): first axis varies fastest; strides are arranged accordingly.
- Non-contiguous: obtained via slicing, transposing, or advanced indexing; strides describe the non-unit steps to access elements.

Examples
--------

- Basic array and introspection

  - Compute expected strides for a C-contiguous (2, 3) int64 array:
    - shape = (2, 3)
    - itemsize = 8
    - strides = (3 * 8, 8)  # (row skip, column skip)

- Inspect internal layout via the array interface

  - A = np.arange(6).reshape(2, 3)
  - iface = A.__array_interface__
  - data_ptr = iface["data"][0]      # memory address
  - strides = iface.get("strides", None)

- Example: non-contiguous view

  - B = A[:, ::2]  # takes a view with non-unit stride
  - B.flags  # shows C_CONTIGUOUS may be False, as strides reflect non-contiguous layout
  - B.base is A  # memory buffer is shared with A

Dtypes and structured dtypes
----------------------------

- Dtype describes how to interpret each element, including its byte size and representation.
- itemsize is available as dtype.itemsize.
- Structured dtypes define named fields with their own offsets and dtypes; each field contributes to the overall memory layout.
- Subarray dtypes enable nested shapes within a single element.

Ownership, bases, and views
---------------------------

- Base array: the underlying array that actually owns the memory buffer. A view’s base points to the original memory owner.
- Copy vs view: slicing and basic indexing often produce a view (no data copy). explicit copy() yields a new, independent memory block.
- OWNDATA flag indicates whether the array owns its memory; views may not own memory.

Array flags
-----------

- C_CONTIGUOUS: memory is laid out in C order.
- F_CONTIGUOUS: memory is laid out in Fortran order.
- OWNDATA: memory is owned by the array (free on destruction).
- WRITEABLE: elements can be modified.
- ALIGNED: memory alignment constraints are satisfied.
- UPDATEIFCOPY: used in certain copy-on-write scenarios.

Internal API introspection
--------------------------

NumPy exposes lightweight interfaces to inspect memory layout:

- __array_interface__: a dict describing data pointer, shape, strides, and descr
  - data: (pointer, read-only flag)
  - shape: tuple of dimensions
  - strides: tuple of strides in bytes
  - descr: dtype descriptor

- numpy.ctypeslib can be used to interface with C code, using the array’s memory buffer.

Code excerpts (illustrative)
----------------------------

- Inspect core layout (Python-level exploration)

  import numpy as np

  A = np.arange(6, dtype=np.int64).reshape(2, 3)
  print("shape:", A.shape)
  print("strides (bytes):", A.strides)
  print("dtype:", A.dtype)
  print("flags:", A.flags)

  iface = A.__array_interface__
  print("interface data pointer:", iface["data"][0])
  print("interface strides:", iface.get("strides", None))

- Ensure contiguity for performance

  B = np.ascontiguousarray(A)
  print("B is C-contiguous:", B.flags.c_contiguous)

Best practices
--------------

- Favor contiguous arrays for predictable performance with contiguous memory access.
- Be mindful of views: operations on a view affect the underlying memory.
- Use ascontiguousarray or np.copy when a contiguous copy is required for performance-critical paths.

Notes
-----

- The internal layout is designed to be flexible to support views, broadcasting, and advanced indexing without excessive copying.
- For structured dtypes, accessing fields requires careful handling of offsets and alignment.
