BestEffortCallback
===================

Overview
--------
BestEffortCallback is a live data visualization and logging helper intended for use with a Run Engine (RE) in interactive data acquisition workflows. It aggregates and displays information from streaming documents (start, descriptor, etc.) in a best-effort fashion, including:

- Live plots (via LivePlot, LiveGrid, LiveScatter)
- A Run table (via LiveTable)
- Peaks statistics (via PeakStats)
- Optional headings with timestamps and identifiers
- Handling of multiple data streams (e.g., primary, baseline)

Key behavior:
- Prints a header with transient scan information at the start of a run (configurable).
- Automatically builds dimensional information from start/document hints or motor specifications.
- Supports enabling/disabling of heading, table, baseline data, and plotting.
- Maintains internal state to map descriptors to visualizations and to ordinate data against dimensional fields.
- Includes compatibility handling for older documents and streams, and a fallback heuristic when dimension hints are insufficient.

-------------
The callback depends on the following libraries and project modules:
- matplotlib (for plotting)
- numpy
- cycler
- datetime, io, itertools, re, sys, threading, time

Usage
-----
Instantiate and wire into a Run Engine (RE). Attach to a plan to visualize data as it is produced.

Example
-------
This example demonstrates basic usage: creating the callback, attaching it to a RunEngine, and running a plan (the actual plan will depend on your environment).

.. code-block:: python

    from bluesky.run_engine import RunEngine
    from your_module.callbacks import BestEffortCallback

    bec = BestEffortCallback(fig_factory=None, table_enabled=True)
    RE = RunEngine({})
    RE.subscribe(bec)

    # Example plan invocation (your actual plan will vary)
    # RE(your_plan())

Configuration and API
---------------------
Constructor
-----------
BestEffortCallback.__init__(*, fig_factory=None, table_enabled=True, **kwargs)

- fig_factory: Optional factory for creating figures/axes (defaults to internal handling if None).
- table_enabled: Enable printing and updating the LiveTable with readings from the primary stream.
- **kwargs: Passed to the underlying QtAwareCallback superclass (if applicable).

Public methods and toggles
--------------------------
- enable_heading() / disable_heading()
  - Enable or disable printing the header (timestamp and IDs) at the start of a run.

- enable_table() / disable_table()
  - Enable or disable the LiveTable output for the primary stream.

- enable_baseline() / disable_baseline()
  - Enable or disable printing the fields from the baseline stream.

- enable_plots() / disable_plots()
  - Enable or disable plotting for all streams except those explicitly listed in noplot_streams.

- __call__(name, doc, *args, **kwargs)
  - Dispatch method invoked by the Run Engine for each document.
  - If all outputs are disabled (no table, baseline, or plots), the method becomes a no-op.

- start(doc)
  - Called at the start of a run.
  - Clears internal state, stores start document, and reads hints for planning.
  - Attempts to infer dimensional structure (dimensions) and the primary data stream.
  - Prints a header if enabled and derives a dimension guess from motors or time.

- descriptor(doc)
  - Called for each descriptor document.
  - Stores the descriptor, detects new streams, and optionally prints a “New stream” message.
  - Determines columns to plot or table based on hints and current configuration.
  - Handles legacy documents and attempts to resolve dimension fields, including a fallback heuristic when hints are insufficient.

Internal state (high-level)
----------------------------
- _start_doc: The start document for the current run.
- _descriptors: Mapping of descriptor UID to descriptor document.
- _table: LiveTable instance (if enabled).
- _heading_enabled: Whether to print the header at start.
- _table_enabled: Whether the table output is enabled.
- _baseline_enabled: Whether baseline fields are included.
- _plots_enabled: Whether plotting is enabled.
- _fig_factory: Figure factory for external axes (if provided).
- _live_plots / _live_grids / _live_scatters: Mappings from descriptor UID to plotting objects per data key.
- _peak_stats: Mapping for peak statistics per live plot descriptor.
- _cleanup_motor_heuristic: Flag indicating motor-dimension heuristic needs cleanup.
- _stream_names_seen: Set of stream names encountered.

Public data
-----------
- peaks: Peak results container (PeakResults) to hold peak statistics across plots.
- overplot: Flag to overlay multiple plots when applicable.
- noplot_streams: List of streams that should not be plotted (default: ['baseline']).
- omit_single_point_plot: If True and a run has a single point, skip plotting.

Notes on behavior and design
----------------------------
- The callback is designed to be a best-effort consumer of run documents. If plotting, table, or baseline outputs are disabled, it will be inert to subsequent documents to minimize overhead.
- Dimension handling strives to extract independent variables from start hints or motor names. When hints are insufficient or inconsistent across streams, a fallback guess is used with a warning.
- For legacy documents, the descriptor handler attempts to map object names to actual fields, ensuring compatibility with older schemas.
- The implementation uses a combination of LivePlot, LiveGrid, and LiveScatter instances for dynamic visualization, together with a LiveTable for tabular readings.

Customization
-------------
- The callback is designed to be extended or customized via the fig_factory and by adjusting the public toggles. This allows tailoring the balance between performance and visibility in live data sessions.

End-to-end example (high level)
-------------------------------
- Create BestEffortCallback with desired features enabled.
- Attach it to a RunEngine.
- Run a plan that emits start, descriptor, event, and close documents.
- Observe headings, streams, and plots updating in real time according to the configuration.
