Support Vector Machines
=======================

Overview
--------

Support Vector Machines (SVMs) are supervised learning models that find a decision boundary with maximum margin between classes. This implementation focuses on linear SVMs using LIBLINEAR for speed and scalability, suitable for large datasets. It supports both dense and sparse input and provides options for regularization, loss functions, and multi-class handling.

Key characteristics
-------------------

- Linear decision boundaries (no kernel tricks like RBF; for non-linear problems, consider a kernel-based SVM).
- Regularization controlled by C (inverse of regularization strength).
- Choice of penalty: L1 or L2
- Loss functions: hinge or squared hinge
- Dual vs. primal optimization controlled by the dual parameter
- Multiclass strategies: one-vs-rest (ovr) or crammer_singer
- Handles dense and sparse input formats efficiently
- Intercept handling with fit_intercept and intercept_scaling

API at a glance
---------------

Linear SVM classifier exposed by this module:

- Class: LinearSVC
- Primary purpose: Linear classification via LIBLINEAR with flexible regularization and loss options

Parameters
----------

- penalty : {'l1', 'l2'}, default='l2'
  Specifies the norm used for regularization. L2 is standard for SVC-like behavior; L1 promotes sparsity in the model coefficients.

- loss : {'hinge', 'squared_hinge'}, default='squared_hinge'
  Determines the loss function. 'hinge' is the classic SVM loss; 'squared_hinge' is its squared variant.

- dual : "auto" or bool, default=True
  Selects the optimization problem (dual or primal). 'auto' chooses based on data shape and problem configuration. See notes for details.

- tol : float, default=1e-4
  Tolerance for the stopping criterion.

- C : float, default=1.0
  Regularization strength. Larger C means weaker regularization; smaller C increases regularization.

- multi_class : {'ovr', 'crammer_singer'}, default='ovr'
  Strategy for multiclass classification when y contains more than two classes.
  - ovr: one-vs-rest approach
  - crammer_singer: joint optimization over all classes (more expensive, but sometimes more accurate)

- fit_intercept : bool, default=True
  Whether to fit an intercept term. If True, an intercept is learned.

- intercept_scaling : float, default=1.0
  When fit_intercept is True, an additional synthetic feature is appended to the input vector with this scaling. This helps regularize the intercept separately from other features.

Notes
-----

- This implementation is optimized for speed and scalability on large datasets.
- It is designed for linear decision boundaries. For non-linear separation, consider kernel-based SVMs (e.g., SVC with an appropriate kernel).
- Input data can be NumPy arrays or sparse matrices (e.g., CSR/CSC) to improve memory efficiency with high-dimensional data.

Usage example
-------------

Binary classification (dense data):

.. code-block:: python

    from sklearn.svm import LinearSVC
    from sklearn.datasets import make_classification
    from sklearn.metrics import accuracy_score

    X, y = make_classification(n_samples=200, n_features=20, random_state=42)
    clf = LinearSVC(penalty='l2', loss='squared_hinge', C=1.0, dual='auto',
                    tol=1e-4, multi_class='ovr')
    clf.fit(X, y)
    preds = clf.predict(X)
    print("Accuracy:", accuracy_score(y, preds))

Multiclass with crammer_singer (default is ovr):

.. code-block:: python

    clf = LinearSVC(penalty='l2', loss='hinge', C=0.5, dual='auto',
                    tol=1e-4, multi_class='crammer_singer')
    clf.fit(X, y)
    preds = clf.predict(X)

Sparse data example:

.. code-block:: python

    from scipy.sparse import csr_matrix
    X_sparse = csr_matrix((values, (row_indices, col_indices)),
                          shape=(n_samples, n_features))
    clf = LinearSVC()
    clf.fit(X_sparse, y)

Performance and tips
---------------------

- Scale features before training for improved convergence (standard practice for linear models).
- Increase C cautiously to reduce regularization if underfitting occurs.
- When using high-dimensional sparse data, the L1 penalty can yield sparser models.
- For non-linear decision boundaries, explore kernel-based SVMs (e.g., SVC with an RBF kernel) instead of LinearSVC.
