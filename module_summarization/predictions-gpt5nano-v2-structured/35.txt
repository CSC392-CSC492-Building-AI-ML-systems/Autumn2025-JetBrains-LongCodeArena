w_states
========

Display or manipulate basis (initial) or target (recycling) states for a WEST simulation.

Overview
--------

w_states provides a command-line interface to inspect and modify the basis and target states used by a WEST simulation for the current or next iteration. By default, states are displayed to stdout (or written to files when requested). The command supports:

- Showing current basis and target states
- Replacing the current iteration's states with a new set
- Appending new target states to the existing list (basis states append is not supported)

The tool integrates with WESTPA’s data management and work-management backends.

Usage
-----

w_states [OPTIONS]

Modes of operation (mutually exclusive)

- --show: Display current basis/target states (or dump to files)
- --append: Append new basis/target states to those currently in use
- --replace: Replace current basis/target states with the provided ones

Common options

- --bstate-file BSTATE_FILE
  Read (for append/replace) or write (for show) basis state data from/to BSTATE_FILE.

- --bstate "label,probability[,auxref]"
  Add a basis state to the list (format: label,probability[,auxref]).
  May be specified multiple times; appended after any states in --bstate-file.

- --tstate-file TSTATE_FILE
  Read (for append/replace) or write (for show) target state data from/to TSTATE_FILE.

- --tstate "label,pcoord0[,pcoord1[,...]]"
  Add a target state to the list (format: label,pcoord0[,pcoord1...]).
  May be specified multiple times; appended after any states in --tstate-file.

Notes

- Basis state appending is not permitted in practice. To add basis states, use a show operation to export current states to a file, edit the file to include new basis states, then use --replace to apply the updated set.
- The default mode (if no mode is specified) is --show.

File formats
------------

Basis states file (written by --show)

- A human-readable header is written, followed by one line per basis state.
- Each line contains: label,probability[,auxref]

Example (basis states file)

# Basis states for iteration 42
stateA,0.25
stateB,0.50
stateC,0.25,aux_ref_x

Target states file (written by --show)

- A header is written, followed by one line per target state.
- Each line contains: label,pcoord0[,pcoord1[,...]]

Example (target states file)

# Target states for iteration 42
tstate1,0.0,1.2
tstate2,2.5,0.8

Examples
--------

Show current states and dump to files

- Show current basis and target states to stdout
  w_states --show

- Show and write to files
  w_states --show --bstate-file bstates.txt --tstate-file tstates.txt

Replace states for the next iteration (with files and/or inline definitions)

- Replace using files
  w_states --replace --bstate-file bstates.txt --tstate-file tstates.txt

- Replace using inline definitions (basis and/or target)
  w_states --replace --bstate "stateX,0.3" --bstate "stateY,0.7" --tstate "tstateX,0.0,1.0"

Append target states for the next iteration (not generally used for basis states)

- Append to existing target states
  w_states --append --tstate "newt,0.2,1.1"

Behavior and implementation details
-----------------------------------

- Only the master work manager performs state display or modification actions; data is retrieved from or written to the WESTPA data manager for the current iteration.
- When replacing states, the total basis-state probability must sum to unity. If not, probabilities are rescaled to enforce normalization.
- Progress coordinates for basis states are assigned via the simulation manager prior to creating the new basis-state group and reporting them.
- Writing state dumps with --show includes a header indicating the iteration number and the corresponding state lists.
