Ploterr command
===============

Overview
--------
Ploterr provides a plotting utility to visualize the evolution of a quantity and its confidence intervals across iterations. It reads data from an HDF5 file and a specified dataset containing arrays for iteration indices and statistics, then generates a plot of the expected value alongside its confidence interval bounds.

Key behaviors
- Output is generated by the Generic interval subcommand, which accepts a path of the form FILENAME/PATH[SLICE].
- The plot uses:
  - x-axis: iterations (derived from iter_stop)
  - y-axis: expected value, lower bound, and upper bound
- Interactive plotting uses matplotlib when a display is available; otherwise, or when explicitly requested, a text interface is used.
- Output file format is inferred from the filename extension (e.g., .pdf, .png).

Subcommands
------------
Generic interval plotting
- Command: ploterr generic FILENAME/PATH[SLICE]
- Description: Plots generic expectation/CI data stored in an HDF5 dataset.
- Example:
  - ploterr generic kinavg.h5/rate_evolution[:,0,1] -o errbars.pdf
  - The example accesses the rate evolution data for states 0 and 1 and writes to errbars.pdf.

Input/Output
------------
- dsspec (positional):
  - Format: FILENAME/PATH[SLICE]
  - FILENAME is the HDF5 file to read.
  - PATH is the HDF5 dataset path.
  - SLICE (optional) is a NumPy-style slice selecting a 1-D vector, e.g. [:,0,1].
  - If no slice is provided, a full ellipsis slice [...] is used.
- -o, --output (default: errbars.pdf):
  - Output file path. The format is inferred from the extension.
- Output format:
  - Determined by the filename extension (e.g., .pdf, .png, etc.).

Data requirements
-----------------
The HDF5 dataset must contain, at minimum, the following 1-D arrays:
- iter_start
- iter_stop
- expected
- ci_lbound
- ci_ubound

Validation:
- The dataset must be 1-dimensional. If not, an error is raised.
- The x-axis values are derived from iter_stop minus one (iterations).

Plot options
------------
CommonPloterrs supports the following options to tailor the plot:

- --xscale <linear|log|symlog> (default: linear)
  - Scaling for the x-axis.
- --yscale <linear|log|symlog> (default: linear)
  - Scaling for the y-axis.
- --xrange xmin,xmax
  - Restrict the x-axis to the specified range.
- --yrange ymin,ymax
  - Restrict the y-axis to the specified range.
- --xlabel XLABEL
  - Label for the x-axis. Default varies; typically "Iteration".
- --ylabel YLABEL
  - Label for the y-axis. Default varies.
- --title TITLE
  - Plot title. Default varies.
- --terminal, -t
  - Plot output in terminal (text interface). If DISPLAY is not available or -t is used, a text interface is used.

Interface behavior
------------------
- If -t/--terminal is provided or no graphical display is available, the command uses a text interface.
- Otherwise, when a graphical display is available, and not forced to text, matplotlib is used to render the plot.

Usage notes
------------
- The path spec (FILENAME/PATH[SLICE]) allows flexible extraction of 1-D data from a possibly multi-dimensional dataset.
- The x-axis is implicitly defined as iter_stop - 1, aligning the plot with iteration indices.
- The core plotting shows:
  - expected values as a black line
  - ci_lbound and ci_ubound as gray lines
- Label customization (xlabel, ylabel) and title can be provided to clarify the plot.

Examples
--------
- Basic plot with default settings:
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -o errbars.pdf
- Plot with custom axis labels and title:
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -o errbars.png --xlabel "Iteration" --ylabel "Rate" --title "Rate evolution (0 -> 1)"
- Plot in text mode (no GUI), e.g. when DISPLAY is not available:
  ploterr generic kinavg.h5/rate_evolution[:,0,1] -t

Implementation notes (high level)
--------------------------------
- Data loading and validation:
  - The command loads data from the specified HDF5 dataset slice.
  - Validates that the dataset is 1-D and contains the required fields.
- Plotting:
  - Uses matplotlib when available; otherwise falls back to a text interface.
  - Plots three series: expected, ci_lbound, ci_ubound.
  - Applies requested scales, ranges, and labels.
  - Saves the figure to the specified output file.
