# distutils.rst

Distutils Utilities
===================

Overview
--------
This module provides a collection of small, generally useful utilities that support distutils-based build workflows (notably in numpy.distutils). It includes:

- Thread-safe management of temporary directories
- A lightweight container for information about installable libraries
- Path manipulation helpers (joining, normalizing, and relative paths)
- Utilities to determine the number of parallel build jobs
- Lightweight helpers for working with frames and frames’ module paths
- Miscellaneous helpers and constants exposed as part of the public API

Public API
----------
The module exposes a number of public API items. The following sections summarize the main public classes and functions, and what they are used for.

InstallableLib
--------------
Container to hold information on an installable library.

Parameters
- name: str
  Name of the installed library.
- build_info: dict
  Dictionary holding build information for the library.
- target_dir: str
  Absolute path specifying where to install the library.

Attributes
- name
- build_info
- target_dir

Example
```python
lib = InstallableLib('mylib', {'sources': ['src/mylib.c']}, '/usr/local/lib')
print(lib.target_dir)  # /usr/local/lib
```

get_num_build_jobs
------------------
Determine the number of parallel build jobs.

Behaviour
- Honors the --parallel option from setup.py if available
- If not configured, checks the environment variable NPY_NUM_BUILD_JOBS
- Falls back to the system CPU count, capped at 8 to avoid overloading the machine
- Uses numpy.distutils.core.get_distribution to consult build commands (build, build_ext, build_clib) if present

Returns
- int: number of parallel build jobs that can be run

Example
```python
from distutils import get_num_build_jobs
n = get_num_build_jobs()
print("Parallel build jobs:", n)
```

quote_args
----------
Quote a list of arguments.

Deprecated
- This function is marked as deprecated (since 1.22). It will be removed in a future release.

Usage note
- A warning is emitted when called.

Example
```python
from distutils import quote_args
args = ['-DNAME="Value with spaces"']
print(quote_args(args))
```

allpath
-------
Convert a '/'-separated pathname to the OS-specific path separator.

Parameters
- name: str

Returns
- str: path with OS-specific separators

Example
```python
print(allpath('src/utils/helpers.py'))  # on Windows: 'src\\utils\\helpers.py'
```

rel_path
--------
Return path relative to a given parent path.

Parameters
- path: str
- parent_path: str

Returns
- str: the path relative to parent_path or the original path if not related

Example
```python
print(rel_path('/home/user/project/src/module', '/home/user/project'))  # 'src/module'
```

get_path_from_frame
-------------------
Return the path of the module given a frame object from the call stack.

Parameters
- frame: frame object
- parent_path: optional str to be used as a base for relative paths

Returns
- str: relative path to parent_path if provided, otherwise absolute path

Example
```python
import inspect
def myfunc():
    frame = inspect.currentframe()
    print(get_path_from_frame(frame, '/home/user/project'))
myfunc()
```

njoin
-----
Join two or more pathname components with path normalization.

Features
- Accepts multiple arguments or a sequence of components
- Converts '/' separators to OS-specific separators
- Resolves '.' and '..' components

Parameters
- path: one or more pathname components or a single sequence of components

Returns
- str: the joined and normalized path

Example
```python
print(njoin('src', 'module', ['utils', '..', 'helpers']))
# Example output: 'src/module/helpers'
```

Notes on API and Usage
----------------------
- The module also defines several other helpers and constants (not shown in this excerpt) such as colorized text helpers (red_text, green_text, yellow_text, blue_text, cyan_text) and platform/implementation checks (cyg2win32, mingw32), among others. These are exposed via the module’s public API list (__all__).
- There is internal state to support thread-safe operations:
  - _tdata: thread-local storage to keep per-thread data
  - _tmpdirs: list of temporary directories created during execution
  - clean_up_temporary_directory: function registered with atexit to clean up temporary directories
  - The cleanup function is automatically invoked when the process exits

Usage Context
-------------
These utilities are intended for internal use by distutils-based build workflows and by extensions that need robust path handling, parallel build configuration, and lightweight build metadata containers. They are designed to be small, dependency-light helpers that can be used in a variety of build scripts and configuration situations.

Examples of common workflows
----------------------------
- Determine the number of parallel builds for a setup script:
  - Call get_num_build_jobs() to automatically pick a sensible default based on CPU count and environment configuration.

- Normalize and join paths across platforms:
  - Use allpath, rel_path, and njoin to build file paths in a cross-platform way.

- Represent and track installable libraries:
  - Use InstallableLib to carry library name, build metadata, and target install directory.

Deprecations and Compatibility
------------------------------
- quote_args is deprecated (since 1.22). Prefer alternative argument handling strategies in new code and plan to migrate away from this helper in future releases.

References
----------
- numpy.distutils and distutils build machinery for context on how these utilities integrate with build commands and configuration.
