w_kinetics
==========

Overview
--------

w_kinetics is a thin, backward-compatible shim around the w_direct kinetics tracing tool. It provides the trace subcommand to compute state-to-state kinetics by tracing trajectories and writes results to an output file (default: kintrace.h5). The produced data are intended to be post-processed by w_kinavg to obtain normalized rates and confidence intervals.

Deprecation notice
------------------

This command is deprecated. Please use w_direct instead. The shim is provided for backward compatibility but may be removed in a future release.

Usage
-----

- Program: w_kinetics
- Subcommands:
  - trace: averages and CIs for path-tracing kinetics analysis
- Default output file: kintrace.h5

The w_kinetics command delegates to the underlying w_direct tool’s trace functionality. To see the full set of options, run:

  w_kinetics trace --help
  w_direct trace --help

Output file
-----------

The output file (by default kintrace.h5) contains the following datasets:

- /conditional_fluxes [iteration][state][state]
  - Floating-point: Macrostate-to-macrostate fluxes. These are not normalized by the population of the initial macrostate.

- /conditional_arrivals [iteration][stateA][stateB]
  - Integer: Number of trajectories arriving at stateB in a given iteration, given that they departed from stateA.

- /total_fluxes [iteration][state]
  - Floating-point: Total flux into a given macrostate.

- /arrivals [iteration][state]
  - Integer: Number of trajectories arriving at a given state in a given iteration, regardless of origin.

- /duration_count [iteration]
  - Integer: The number of event durations recorded in each iteration.

- /durations [iteration][event duration]
  - Structured (see below): Event durations for transition events ending during a given iteration. Stored as:
    - istate: Integer, initial state of transition event.
    - fstate: Integer, final state of transition event.
    - duration: Floating-point, duration of transition in units of tau.
    - weight: Floating-point, weight of trajectory at end of transition (not normalized by initial state population).

Notes
-----

- State-to-state fluxes stored in kintrace.h5 are not normalized by the initial macrostate population. They cannot be used as rates without additional processing.
- Normalization and statistical treatment (including correlation) are handled by w_kinavg. See w_kinavg trace --help for details. Target fluxes (total flux into a given state) do not require normalization.

Related tools and workflow
--------------------------

- The kintrace.h5 output is designed to be consumed by w_kinavg to produce average rates with confidence intervals.
- For normalization and rate computation:
  - See: w_kinavg trace --help
- For trajectory-to-trajectory labeling and data preparation:
  - See: w_assign and related documentation.

Examples
--------

- Basic invocation (via the shim, using the underlying w_direct trace functionality):

  w_kinetics trace -o kintrace.h5 <additional options>

- Request help to see all available options (via the wrapper and underlying tool):

  w_kinetics trace --help
  w_direct trace --help

- Post-processing with w_kinavg (example workflow):

  w_kinavg trace -i <iterations> -o kinavg.h5 kintrace.h5

Notes
-----

- The w_kinetics command is a deprecated shim intended for compatibility with scripts and workflows that use the old command name. Prefer using w_direct directly for new workflows.
