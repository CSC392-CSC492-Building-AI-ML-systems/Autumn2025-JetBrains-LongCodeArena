Gaussian Process Kernels
========================

Overview
--------

This module provides kernels for Gaussian process regression and classification. The kernels support kernel-engineering, meaning you can build complex kernels by combining simpler ones:

- Addition: x + y
- Multiplication: x * y
- Exponentiation: x ** scalar

Scalar values in expressions are automatically converted to a ConstantKernel. All kernels support analytic gradient-based hyperparameter optimization. Hyperparameters are controlled by a defined search space, which can be specified either as bounds (a rectangular space) or by marking parameters as fixed (excluded from optimization).

Key Concepts
------------

- Kernel algebra
  - Combine kernels with +, *, and ** to form rich kernels for GP modeling.
  - Scalar terms in expressions become constant kernels automatically.

- Hyperparameters
  - Each kernel exposes hyperparameters that can be optimized analytically.
  - The search space is defined by lower and upper bounds for each hyperparameter.
  - Hyperparameters can be declared as "fixed" to exclude them from optimization.

- Hyperparameter representation
  - Hyperparameter is a namedtuple with fields:
    - name: parameter name (e.g., constant_value)
    - value_type: type of the parameter (e.g., "numeric")
    - bounds: numeric bounds or the string "fixed"
    - n_elements: number of elements (1 for scalar, >1 for vector-valued, e.g., anisotropic length-scales)
    - fixed: whether the parameter is fixed (True/False)

- Length-scale validation
  - Length scales can be anisotropic. The length_scale argument is validated to ensure correct dimensionality with respect to the input data X.

- Kernel base API
  - Kernel is the base class for all kernels in this module.
  - Kernels expose parameters via get_params for introspection (useful for cloning, hyperparameter tuning, etc.).

Usage
-----

- Kernel composition

  Kernels can be combined to form more expressive kernels. For example:

  - Sum of kernels
  - Product of kernels
  - Exponentiation to scale a kernel by a constant

  Scalars in expressions become constant kernels automatically.

- Hyperparameter optimization

  - Define lower/upper bounds for each hyperparameter to specify the search space.
  - Mark hyperparameters as fixed to exclude them from optimization.

- Hyperparameters inspection

  Each kernel exposes its hyperparameters, which can be inspected and manipulated as shown below.

Examples
--------

The following example demonstrates creating a simple constant kernel with hyperparameters and inspecting them. This mirrors the approach commonly used in similar GP kernel ecosystems.

.. code-block:: python

    from sklearn.gaussian_process.kernels import ConstantKernel
    from sklearn.datasets import make_friedman2
    from sklearn.gaussian_process import GaussianProcessRegressor
    from sklearn.gaussian_process.kernels import Hyperparameter

    X, y = make_friedman2(n_samples=50, noise=0, random_state=0)

    # Define a constant kernel with a bound on the constant value
    kernel = ConstantKernel(constant_value=1.0,
                          constant_value_bounds=(0.0, 10.0))

    # Access hyperparameters
    for hyperparameter in kernel.hyperparameters:
        print(hyperparameter)

    # Examine parameter values (as shown by get_params)
    params = kernel.get_params()
    for key in sorted(params):
        print(f"{key} : {params[key]}")

- More complex kernel expressions

  You can build richer kernels by combining base kernels:

  .. code-block:: python

      from sklearn.gaussian_process.kernels import RBF, ConstantKernel

      k1 = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 5.0))
      k2 = RBF(length_scale=1.0, length_scale_bounds=(0.1, 10.0))

      # Kernel algebra: sum, product, or scalar-exponentiation
      k = (k1 + k2) * (k2 ** 0.5)

Notes
-----

- This module is strongly inspired by the kernel module of the george package.
- The implementation relies on numpy, SciPy, and related utilities for distance computations, hyperparameter handling, and validation.

API Summary
-----------

- Hyperparameter: namedtuple describing a kernel hyperparameter (name, value_type, bounds, n_elements, fixed).

- _check_length_scale(X, length_scale): Validates and normalizes the length_scale for anisotropic kernels, ensuring compatibility with the input dimensionality.

- Kernel (base class): Abstract base class for all kernels. Includes utility methods such as get_params for parameter introspection and cloning support.

- get_params(self, deep=True): Returns the kernel's parameters (and subobjects when deep=True) for inspection and cloning.

Implementation notes
--------------------

- The module defines minimal scaffolding for hyperparameter management, including automatic handling of scalar constants in composite kernels.
- The Hyperparameter class ensures bounds are consistently shaped and that vector-valued hyperparameters have matching n_elements.
