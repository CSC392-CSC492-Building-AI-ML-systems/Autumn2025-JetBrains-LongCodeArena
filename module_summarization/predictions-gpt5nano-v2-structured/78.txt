Filе Node: File Interface for PyTables Nodes
=============================================

Overview
--------

The FileNode module provides a file-like interface to PyTables database nodes. It enables using a PyTables file node as if it were a ordinary Python file, with binary I/O operations. Use:

- new_node() to create a brand-new file node
- open_node() to open an existing node (created with new_node())

Read access is always available. Write access is allowed only when the file is opened with write-capable modes (e.g., mode that enables appending). Currently, only binary I/O is supported.

This module is designed to be fully compatible with the Python io module (as of version 3.0, see the version history below).

See also: filenode_usersguide for concrete usage instructions.

Key Constants
-------------

- NodeType
  - Description: The node system attribute value used for File nodes.
  - Value: 'file'
  - Purpose: Identifies the node type in the PyTables metadata.

- NodeTypeVersions
  - Description: Supported NODE_TYPE_VERSION values for File nodes.
  - Value: [1, 2]
  - Purpose: Determines how data is shaped when read from the node.

Core Class
----------

RawPyTablesIO
~~~~~~~~~~~~~

Base class implementing a raw binary I/O interface on PyTables file nodes. It inherits from io.RawIOBase and provides the low-level, byte-oriented operations required to read and write data through the PyTables FileNode.

Constructor
-----------

RawPyTablesIO(node, mode=None)

- node: The PyTables node to operate on.
- mode: Optional file mode (e.g., 'r', 'w', 'a', 'r+'). If omitted, the mode from node._v_file.mode is used.
- Behavior:
  - Validates the node and its attributes.
  - Determines the I/O version from node.attrs.NODE_TYPE_VERSION.
  - Establishes the current position (_pos) and derives the value shape (_vshape) based on version.
  - Determines the element type (_vtype) from the node's atom dtype.

Properties
----------

mode
    Read-only property exposing the current file mode.

Public API
----------

tell()
    Return the current stream position (absolute byte offset).

seek(pos, whence=0)
    Move the stream position to a new byte offset.
    - whence:
      - 0: start of stream (default)
      - 1: current position
      - 2: end of stream
    Returns the new absolute position.

seekable()
    Always returns True, indicating random access is supported.

fileno()
    Return the underlying file descriptor of the PyTables host file, if available.

close()
    Close the IO object. Releasing the node allows the underlying file to be closed.

flush()
    Flush write buffers, if applicable (not implemented for read-only streams).

truncate(pos=None)
    Truncate the file to the given size in bytes.
    - If pos is None, use the current position.
    - Truncation is meaningful for growing files in this implementation.
    Returns the new size (position after the operation).

readable()
    Return True if the file was opened for reading ('r' or '+').

writable()
    Return True if the file was opened for writing ('w', 'a', or '+').

readinto(b)
    Read up to len(b) bytes into the provided writable buffer b.
    Returns the number of bytes read (0 for EOF).

readline(limit=-1)
    Read a line from the stream (implementation present in the module).

Notes on Usage
--------------

- The FileNode interface exposes a file-like API that mirrors Python’s built-in file semantics, but operates over a PyTables node inside a database file.
- Read operations pull data from the underlying PyTables node via node.read(start, stop) and convert results to bytes.
- Write operations append data to the node and may require opening in a mode that permits writing.
- The implementation is designed to be compatible with Python’s io module, enabling usage with standard patterns such as context managers and binary I/O workflows.

Usage Examples
--------------

Create and write to a new file node (example; actual API may vary slightly depending on PyTables version):

.. code-block:: python

    import tables as tb
    # Create a brand-new file node (mode allows read/write)
    with tb.new_node('example.filenode', mode='w+') as f:
        f.write(b'Hello PyTables FileNode!')
        f.seek(0)
        data = f.read()
        print(data)

Open an existing node for read/write:

.. code-block:: python

    import tables as tb
    with tb.open_node('example.filenode', mode='r+') as f:
        f.write(b' Additional data.')
        f.seek(0)
        content = f.read()
        print(content)

Version history and design notes
--------------------------------

- Version 3.0: The module was rewritten to be fully compliant with the interfaces defined in the io module.
- The implementation relies on node attributes:
  - NODE_TYPE: 'file'
  - NODE_TYPE_VERSION: 1 or 2
- Support for multiple data layouts is handled via _size_to_shape, mapping versions to shapes used during I/O.
