Working with assignments and allocations
======================================

This document describes how to work with assignments and allocations using the
ftrack API. It explains the concepts, typical workflows, and provides working
code examples to get started with creating, querying, updating and removing
assignments and allocations via a Session.

Overview
--------

Assignments represent the relationship between a resource (typically a user)
and a work item (such as a task) within a project. Allocations extend this by
defining the time-bound distribution of a resource’s work across tasks or
work items. The ftrack API provides a unified Query API and entity models to
manage these concepts through a Session.

Key concepts
------------

- Assignment: A linkage between a user (owner) and a target work item (e.g. a Task).
- Allocation: A time-bound association indicating how much effort or time a user
  is assigned to a work item (start date, end date, and duration/hours).
- Session: The primary entry point for all operations. Use the Session to obtain
  entities, perform queries, and submit changes to the server.

Prerequisites
-------------

- A configured ftrack server URL.
- API credentials (api_key and api_user).
- A Session object to interact with the server.

Working with the session
-----------------------

All operations for assignments and allocations go through a ftrack_api.Session
instance. The session handles authentication, server communication, and provides
the Query API to list, create, update or delete assignments and allocations.

Example: establishing a session (typical pattern)

.. code-block:: python

    from ftrack_api import Session

    session = Session(
        server_url='https://ftrack.example.com',
        api_key='YOUR_API_KEY',
        api_user='your_username'
    )

Basic workflows
---------------

1) Listing assignments
- Purpose: Retrieve all assignments for a given work item, user, or project.

.. code-block:: python

    # List all assignments for a specific task
    assignments = session.query(
        "Assignment where task.id is 'TASK_ID'"
    ).all()

    # List assignments for a specific user
    user_assignments = session.query(
        "Assignment where owner.id is 'USER_ID'"
    ).all()

2) Listing allocations
- Purpose: Retrieve time-bound allocations for a user or task.

.. code-block:: python

    # List allocations for a user within a date range
    allocations = session.query(
        "Allocation where owner.id is 'USER_ID' and start_date >= '2025-01-01' and end_date <= '2025-01-07'"
    ).all()

    # List allocations for a task
    task_allocations = session.query(
        "Allocation where task.id is 'TASK_ID'"
    ).all()

3) Creating assignments
- Purpose: Create a new assignment linking a user to a work item.

.. code-block:: python

    new_assignment = session.create('Assignment', {
        'owner': {'id': 'USER_ID'},
        'task': {'id': 'TASK_ID'},
        'role': 'Assignee'  # example field; actual fields depend on server schema
    })
    session.commit()  # Persist changes to the server

4) Creating allocations
- Purpose: Define a new time-bound allocation of a user to a work item.

.. code-block:: python

    new_allocation = session.create('Allocation', {
        'owner': {'id': 'USER_ID'},
        'task': {'id': 'TASK_ID'},
        'start_date': '2025-02-01',
        'end_date': '2025-02-05',
        'hours': 8
    })
    session.commit()

5) Updating assignments or allocations
- Purpose: Modify fields such as dates, duration, or relations.

.. code-block:: python

    a = session.query("Assignment where id is 'ASSIGN_ID'").one()
    a.role = 'Reviewer'          # example field
    session.commit()

    alloc = session.query("Allocation where id is 'ALLOC_ID'").one()
    alloc.end_date = '2025-02-06'
    alloc.hours = 6
    session.commit()

6) Deleting assignments or allocations
- Purpose: Remove an assignment or allocation from the server.

.. code-block:: python

    session.delete(a)
    session.commit()

    session.delete(alloc)
    session.commit()

Query patterns and tips
-----------------------

- Use the Query API to filter by owner, task, date ranges, or project.
- Use fields selection to reduce payload, e.g., fields=['id', 'owner', 'task', 'start_date', 'end_date', 'hours'].
- Use session.commit() (or equivalent) to persist changes to the server after create/update/delete operations.
- Handle server-side validation and permissions errors gracefully; the API will raise appropriate exceptions.

Date handling and time zones
----------------------------

Allocations are defined with dates (start_date and end_date). Use ISO date strings
or the date handling utilities provided by your environment. Be mindful of time zone
implications if the server stores or presents dates in a specific zone.

Error handling
--------------

- Use try/except around API calls to catch ftrack_api.exception related errors.
- Validate IDs and relationships before performing create/update operations.
- Check for optimistic locking or concurrency issues if multiple clients edit the same
  assignments or allocations concurrently.

Notes
-----

- The Session object centralizes authentication (via SessionAuthentication) and
  server interaction. Ensure environment variables or explicit arguments provide
  valid credentials.
- The exact field names for Assignment and Allocation entities depend on the server
  schema. Use the server’s schema or query introspection to determine available
  fields.
