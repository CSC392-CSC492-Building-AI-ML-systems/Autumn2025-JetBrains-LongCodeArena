Getting Started Tutorial
========================

This tutorial walks you through the essential concepts and small, practical steps to start using the common functionality defined in the module used by mitogen masters and child processes. You’ll learn how to determine the interpreter for child contexts, inspect basic logging behavior, and understand the core source handling used to bootstrap remote contexts.

Prerequisites
-------------
- Python 3.x (Python 2 compatibility is supported by Mitogen in earlier versions; this guide assumes modern Python).
- A basic Python development environment.
- A running mitogen-enabled workflow (master/child context) is helpful but not required for the quick checks below.

What you will learn
-------------------
- How to determine the Python executable used for child contexts.
- How the library reports and handles logging levels.
- The purpose of pre-exec hooks and how they relate to child process scheduling.
- An overview of core source handling (sending mitogen.core to child contexts) and related constants.

Quick Start
-----------

1) Inspect the current logging level and determine the Python executable used for child contexts.

Example: determine log level and child interpreter
```python
import mitogen.core as core

# Get the effective log level (lower value means more verbose)
print("Current log level:", core.get_log_level())

# Determine the Python executable to use for child contexts
print("Child interpreter:", core.get_sys_executable())
```

2) Understand the pre-exec hook concept (brief overview)

- _preexec_hook is a global hook that, if set, runs just before exec() in a new child process. It enables the child to be scheduled more freely before the new program starts.
- This is an advanced mechanism used when you need fine-grained control over process startup sequencing.

Note: Setting or using _preexec_hook is an internal detail. For typical usage, you won’t need to manipulate it directly.

3) A note on core source transmission to children

- Mitogen optimizes bootstrap by sending a lightweight, partially compressed representation of mitogen.core to child contexts.
- This reduces the payload required when establishing new contexts.
- The related infrastructure uses a thread-safe lock to initialize a cached partial core source.

What this touches in code (high level)
---------------------------------------
- get_log_level()
  - Purpose: Return the effective logging level used by mitogen.
  - Typical use: Adjust verbosity or gate debugging output in your application.

- get_sys_executable()
  - Purpose: Return sys.executable if it is set; otherwise fall back to "/usr/bin/python" and log a warning.
  - Behavior: If Python was started in a way where sys.executable is not set, the function logs a warning once and returns a sensible default.

- _preexec_hook
  - Purpose: A global hook intended to run before executing a new child process. Allows the child to be scheduled more freely.
  - Note: This is an internal mechanism; use with care in advanced scenarios.

- SELINUX_ENABLED
  - Purpose: Flag indicating whether SELinux enforcement is active on the host.
  - Behavior: If SELinux is enabled, Mitogen avoids using socketpairs to work around potential restrictions.

- LINUX_TIOCGPTN and LINUX_TIOCSPTLCK
  - Purpose: Constants used for PTY control on Linux.
  - Behavior: Cast-aware definitions to ensure compatibility across Python versions.

- get_core_source_partial()
  - Purpose: Retrieve a partial, compressed representation of mitogen.core to send to a child context.
  - Behavior: This operation is thread-safe and avoids costly recomputation by caching the partial result.

- _core_source_lock and _core_source_partial
  - Purpose: Internal synchronization and cache for core source transmission.

Code references (illustrative)
------------------------------
- Determining the child interpreter and log level:
  ```python
  import mitogen.core as core
  print("Log level:", core.get_log_level())
  print("Child interpreter:", core.get_sys_executable())
  ```

- Internal pre-exec hook (advanced usage; typically not modified by end users):
  ```python
  # Example (do not rely on this in general unless you understand the implications)
  core._preexec_hook = some_hook_function  # a function to run before exec()
  ```

- Core source partial (conceptual illustration):
  ```python
  from mitogen.core import get_core_source_partial
  core_partial = get_core_source_partial()
  print("Core source partial type:", type(core_partial))
  ```
