w_ntop
======

Overview
--------

The w_ntop command selects walkers from bins using an assignment file that maps walkers to bins at each timepoint. The file is typically produced by w_assign. By default, w_ntop selects the highest-weight walkers within each bin, but you can choose to select the lowest-weight walkers or to select walkers randomly instead.

Output format
-------------

The output file (default: ntop.h5) contains the following datasets:

- /n_iter [iteration]
  Integer iteration numbers corresponding to entries in the other datasets.

- /n_segs [iteration][bin]
  Integer number of segments in each bin/state for the given iteration. Usually equal to the requested count, but may be smaller if fewer walkers exist.

- /seg_ids [iteration][bin][segment]
  Integer segment IDs for matching walkers in each bin. For an iteration n_iter, only the first n_segs[n_iter] entries are valid. Example: seg_ids[0][0][:n_segs[0]] gives the full list of matching seg_ids in bin 0 for the first stored iteration.

- /weights [iteration][bin][segment]
  Floating-point weights corresponding to each matching segment in /seg_ids.

Notes
- The number of bins used is derived from the assignments file attribute nbins as nbins + 1.
- Iteration range is controlled by the IterRangeSelection used by WEST tools.

Command-line arguments
----------------------

Input and selection

- -a, --assignments
  Path to the assignments file. Default: assign.h5. This file maps walkers to bins at each timepoint.

- -n, --count
  Number of walkers to select from each bin per iteration (default: 1).

- -t, --timepoint
  Timepoint within each iteration to base the selection on. Default: -1 (the last timepoint).

Selection mode (mutually exclusive)

- --highweight
  Select COUNT highest-weight walkers from each bin. (default)

- --lowweight
  Select COUNT lowest-weight walkers from each bin.

- --random
  Select COUNT walkers randomly from each bin.

Output

- -o, --output
  Output filename (default: ntop.h5).

Progress display

- The tool can display a progress indicator during processing (via the WEST tool framework).

Usage examples
--------------

Basic usage (default high-weight selection)

- w_ntop -a assign.h5 -n 2 -o ntop.h5

Random selection with two walkers per bin

- w_ntop -a assign.h5 -n 2 --random -o ntop_random.h5

Select from the first timepoint (timepoint index 0)

- w_ntop -a assign.h5 -n 3 -t 0 -o ntop_tp0.h5

See also
--------

- w_assign(1) for generating the assignments file used by w_ntop

- westpa.tools.WESTTool documentation for general tool usage and options

Notes for developers
--------------------

- The output datasets use predefined dtypes:
  - /n_iter uses n_iter_dtype
  - /n_segs uses unsigned integers
  - /seg_ids uses seg_id_dtype
  - /weights uses weight_dtype
- The output is stored with chunking, shuffle, and compression settings for efficient I/O.
