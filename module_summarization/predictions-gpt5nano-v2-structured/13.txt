Understanding sessions
======================

This document describes the concept and usage of a Session in the ftrack_api
library. A Session provides an isolated context for interacting with an ftrack
server, handling authentication, caching, plugin discovery, and optional
integration with the event hub.

Overview
--------

- A Session encapsulates all state required for communicating with an ftrack
  server, including server URL, authentication credentials, and optional
  components such as a cache, plugin loading, and an embedded event hub.
- Accessing entities through a Session can automatically fetch missing data from
  the server when auto-population is enabled.
- The Session supports:
  - Environment-variable driven defaults for server URL and credentials
  - Optional layered caching with a top-level memory cache
  - Plugin discovery and argument handling
  - Optional schema caching to speed up repeated operations
  - Optional embedded event hub support for local and non-local events

Key concepts
------------

- Server URL: The base URL of the ftrack server you connect to. If not supplied,
  the environment variable FTRACK_SERVER is consulted.
- API key and user: Credentials used to authenticate with the server. The
  library looks up FTRACK_API_KEY (or FTRACK_APIKEY) for the key and
  FTRACK_API_USER for the user. If the user is not provided, the current system
  user may be used as a fallback.
- Auto populate: When enabled (default), accessing entity attributes will fetch
  required data from the server if not already present locally.
- Cache: A pluggable cache mechanism. By default, a layered cache with a MemoryCache
  at the top is used.
- Plugin support: Plugins can be discovered and loaded from specified paths.
  If plugin_paths are not provided, the library attempts to use
  FTRACK_EVENT_PLUGIN_PATH.
- Event hub: Optional embedded event hub support for publishing and subscribing
  to events. The hub can be auto-connected in the background to enable non-local
  event handling.
- Schema caching: Optional caching of API schema to speed up repeated operations.
  The path can be configured via schema_cache_path or the environment variable
  FTRACK_API_SCHEMA_CACHE_PATH. Disable with False.

Creating a Session
------------------

Required arguments
- server_url: The ftrack server URL (including port if needed). If not provided,
  the library will check the FTRACK_SERVER environment variable.
- api_key: The API key used for authentication. If not provided, the library will
  check environment variables FTRACK_API_KEY or FTRACK_APIKEY.
- api_user: The user to operate as. If not provided, the environment variable
  FTRACK_API_USER is checked, or the current system user may be used as a fallback.

If any of the required values are missing, a TypeError is raised with a message
guiding how to supply the value.

Example: Basic session (using explicit values)
----------------------------------------------

.. code-block:: python

   import ftrack_api

   session = ftrack_api.Session(
       server_url='https://ftrack.example.com',
       api_key='YOUR_API_KEY',
       api_user='your.username'
   )

Example: Sessions with environment variables
-------------------------------------------

.. code-block:: python

   # Environment variables:
   # export FTRACK_SERVER=https://ftrack.example.com
   # export FTRACK_API_KEY=YOUR_API_KEY
   # export FTRACK_API_USER=your.username

   import ftrack_api

   session = ftrack_api.Session()

Optional parameters
-------------------

- auto_populate (default: True): If True, entity attributes will be fetched on demand.
- plugin_paths (default: None): List of paths to search for plugins. If None,
  the environment variable FTRACK_EVENT_PLUGIN_PATH is consulted.
- cache (default: None): A cache instance implementing the ftrack_api.cache.Cache
  interface. If provided, it will be integrated into the session's layered cache.
  If the provided object is a callable, it will be invoked with the session as
  the sole argument; it should return None if no suitable cache could be configured.
- cache_key_maker (default: StringKeyMaker): Used to generate keys for objects stored
  in the cache.
- auto_connect_event_hub (default: None): If True, an embedded event hub will be
  automatically connected to the event server to allow non-local events. If False,
  only local events can be published or subscribed until you manually connect the hub.
  If not specified, the behavior is determined by the plugins and environment.
- schema_cache_path (default: None): Path to a folder for schema caching. If not
  provided, FTRACK_API_SCHEMA_CACHE_PATH is used. If both are not provided, a
  temporary directory may be used. Set to False to disable schema caching entirely.
- plugin_arguments (default: None): Optional mapping of keyword arguments passed to
  plugin register functions upon discovery. If a plugin signature does not accept a
  given argument, the discovery mechanism will drop that argument (and log a warning).

Usage notes
-----------

- The session keeps track of pending operations via recorded_operations and
  supports control over operation recording through record_operations.
- The session creates and wires a central memory cache on top of the user-provided
  or default cache to ensure consistent caching behavior.
- Auto-population can significantly affect performance; disable it in hot paths
  where you only want to fetch data explicitly.

Working with the Event Hub
--------------------------

- If auto_connect_event_hub is enabled or if you manually connect the hub, the
  session can publish and subscribe to non-local events via the embedded event hub.
- Subscribing to events does not require a connected hub; connection is only
  necessary for receiving non-local events.

Code example: Enabling schema caching and an embedded event hub
---------------------------------------------------------------

.. code-block:: python

   import ftrack_api

   session = ftrack_api.Session(
       server_url='https://ftrack.example.com',
       api_key='YOUR_API_KEY',
       api_user='your.username',
       schema_cache_path='/path/to/schema_cache',
       auto_connect_event_hub=True
   )

   # The hub can be accessed as needed, e.g.:
   # hub = session.event_hub
   # hub.connect()

Understanding behavior through a simple workflow
------------------------------------------------

- Create a session with required credentials.
- Optionally enable auto-populate for convenient access patterns.
- Use the session to query and manipulate ftrack entities.
- If desired, enable and configure the event hub for asynchronous event handling.
- Optionally configure caching and plugin loading to suit your environment.

Notes and troubleshooting
-------------------------

- Missing required configuration (server_url, api_key, or api_user) raises a
  TypeError with guidance on how to supply them (via arguments or environment
  variables).
- If you rely on environment-based configuration, ensure the relevant variables are
  exported in the environment where your application runs.
- When using plugins, ensure the plugin path environment variable or explicit paths are
  correct to allow proper discovery and argument handling.

Appendix: Environment variables of interest
-------------------------------------------

- FTRACK_SERVER: Server URL
- FTRACK_API_KEY or FTRACK_APIKEY: API key
- FTRACK_API_USER: API user
- FTRACK_EVENT_PLUGIN_PATH: Plugin search paths
- FTRACK_API_SCHEMA_CACHE_PATH: Path for schema cache

This documentation summarizes the purpose, usage, and key features of sessions in the ftrack_api
library, providing practical guidance for creating and working with sessions in typical workflows.
