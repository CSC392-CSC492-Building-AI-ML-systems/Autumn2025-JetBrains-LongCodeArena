polys.rst

Polynomial factorization routines in characteristic zero
======================================================


Overview
--------

This module provides a collection of polynomial factorization routines in characteristic zero. It implements univariate and multivariate factorization over the integers, with support for factoring over algebraic number fields. The routines are designed as a mixin (_Factor) meant to be used by higher-level polynomial ring classes that expose a domain abstraction (e.g., rings over ZZ, QQ, or algebraic number fields).

Key capabilities
- Univariate and multivariate factorization over Z and over algebraic number fields.
- Zassenhaus-based factorization for primitive square-free polynomials in Z[x].
- Enhanced Zassenhaus (EEZ) improvements and cyclotomic factor handling.
- Algebraic number field factorization via Trager-like strategies and cofactors for multivariate cases.
- Trial-division pass to recover multiplicities of factors after a square-free factorization.
- Bound computation (Mignotte bound) for efficient lifting and factoring.

Module structure
----------------

- _Factor: Mixin class that provides core factorization routines used by higher-level polynomial ring implementations.

Core methods
------------

- _trial_division(f, factors)
  Perform trial-division on f using the provided factor list and return the factors with multiplicities, sorted.

- _aa_factor_trager(f)
  Factor multivariate polynomials over algebraic number fields. Uses square-free content and cofactor lifting in the algebraic setting.

- _zz_factor(f)
  Factor polynomials in Z[X], handling both univariate and multivariate cases:
  - Univariate: perform primitive factorization, optional irreducibility checks, and Zassenhaus/EEZ-based factoring.
  - Multivariate: reduce to a univariate primitive part, factor recursively via zassenhaus-based methods on subrings, and combine results.

- _zz_mignotte_bound(f)
  Compute the Mignotte bound for multivariate polynomials in Z[X], used to guide lifting and factor search.

- _zz_zassenhaus(f)
  The Zassenhaus algorithm for primitive square-free polynomials in Z[x]. Finds a nontrivial factorization by modular reduction, lifting, and combinatorial splitting.

Usage patterns
--------------

Typically, factoring is invoked through the public API of a polynomial object in the surrounding library. For example, a polynomial ring over ZZ can be created and factoring performed via factor_list(), which returns:
- The content (gcd of coefficients) of the polynomial, and
- A list of irreducible factors with their multiplicities.

Examples
--------

Univariate factoring over ZZ:

.. code-block:: python

   >>> _, x = ring('x', ZZ)
   >>> (2*x**4 - 2).factor_list()
   (2, [(x - 1, 1), (x + 1, 1), (x**2 + 1, 1)])

Multivariate factoring over ZZ:

.. code-block:: python

   >>> _, x, y = ring('x y', ZZ)
   >>> (2*x**2 - 2*y**2).factor_list()
   (2, [(x - y, 1), (x + y, 1)])

Algebraic number field factoring
-------------------------------

The _aa_factor_trager routine factors multivariate polynomials over algebraic number fields by leveraging square-free factorization, cofactors, and composition with the algebraic embeddings of the generators.

Notes on usage
--------------

- Factorization relies on the domain abstraction associated with the polynomial, including methods like primitive(), sqf_part(), factor_list(), and properties such as is_univariate and is_ground.
- The public API for users typically involves calling factor_list() on a polynomial, rather than calling these internal methods directly.
- When working with multivariate polynomials, the factorization process may involve lifting and recombining factors across subrings or substitutions of generators.

Algorithms and design choices
-----------------------------

- Zassenhaus algorithm: Used for univariate polynomials over Z. This classical approach reduces modulo primes, factors in finite fields, and lifts candidates to Z, followed by a combinatorial search to assemble the full factorization.
- Enhanced Zassenhaus (EEZ): An improvement over the basic Zassenhaus approach to handle more cases efficiently.
- Cyclotomic factorization: Optional preprocessing to factor cyclotomic components early when enabled.
- Wang’s method: Used for certain multivariate cases to factor a square-free part before lifting.
- Hensel lifting: Critical step in lifting factored components from a small modulus back to integers.
- Cofactor handling: In algebraic number fields, cofactors are managed to account for nonlinear embeddings and interactions between generators.

References
----------

- Gathen, Joachim von zur, and Jürgen Gerhard. Modern Computer Algebra. (Foundational reference for polynomial factorization techniques, including Zassenhaus and EEZ methodologies.)

Practical notes
--------------

- The factor_list() output is the canonical form used by users to inspect the irreducible factors and their multiplicities.
- The implementation relies on a robust domain abstraction; ensure the polynomial ring and its domain are properly configured (e.g., rings over ZZ, QQ, or algebraic number fields) to obtain correct results.
