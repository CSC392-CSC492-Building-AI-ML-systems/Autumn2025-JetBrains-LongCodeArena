w_init
======

Initialize a new WESTPA simulation

Overview
--------

The w_init command initializes a new WESTPA project by creating the WEST HDF5 data file and preparing the first iteration's segments. Initial states are derived from one or more basis states, which can be provided via a file (via --bstates-from / --bstate-file) or via one or more --bstate arguments. If no basis states are supplied, a default basis state with probability one, identified by state ID zero and the label "basis", is created (a warning is issued to remind you of this behavior).

Target states define the progress coordinates for (non-equilibrium) steady-state simulations. They can be provided through a file (via --tstates-from / --tstate-file) or via one or more --tstate arguments. If neither target states file nor inline targets are provided, w_init will perform an equilibrium simulation (no sinks).

Start states (optional) define the initial distribution for the system’s start configurations. They can come from a file (--sstates-from / --sstate-file) or from inline --sstate arguments.

Key options include the number of segments per state, and whether to run the weighted-ensemble (WE) algorithm on newly created segments.

Usage
-----

Typically you will invoke w_init with various combinations of state specifications. The command wires through the WESTPA runtime to build the initial blocks of the simulation data structure.

Example usage:
- Minimal initialization (uses defaults for basis, target, and start states when not otherwise provided):
  w_init

- Initialize with basis states from a file and target states from a file:
  w_init --bstates-from basis.csv --tstates-from targets.csv

- Initialize with a basis state and a target state provided inline:
  w_init --bstate "basis,1.0" --tstate "stateA,0.25"

- Initialize with a custom number of segments per state and disable WE for the initial segments:
  w_init --bstates-from basis.csv --tstates-from targets.csv --segs-per-state 2 --no-we

- Overwrite any existing simulation data:
  w_init --force --bstates-from basis.csv --tstates-from targets.csv

Options
-------

- --force
  Overwrite any existing simulation data.

- --bstate-file / --bstates-from
  BSTATE_FILE to read basis state names, probabilities, and (optionally) data references.

- --bstate
  Add a basis state inline as a string: label,probability[,auxref].
  May be specified multiple times. States are appended after any basis states read from a file.

- --tstate-file / --tstates-from
  TSTATE_FILE to read target state names and representative progress coordinates. WESTPA uses the representative progress coordinate of a target state and converts the entire bin containing that progress coordinate into a recycling sink.

- --tstate
  Add a target state inline as a string: label,pcoord0[,pcoord1[,...]].
  May be specified multiple times. States are appended after any targets read from a file.

- --sstate-file / --sstates-from
  SSTATE_FILE to read start state names, probabilities, and (optionally) data references.

- --sstate
  Add a start state inline as a string: label,probability[,auxref].
  May be specified multiple times. States are appended after any start states read from a file.

- --segs-per-state
  Number of segments to initialize from each basis state (default: 1).

- --no-we / --shotgun
  Do not run the weighted ensemble bin/split/merge algorithm on newly-created segments.

- Note on state formats
  - Inline states use the formats:
    - Basis/Start: label,probability[,auxref]
    - Target: label,pcoord0[,pcoord1[,...]]
  - Files read by the --bstates-from, --tstates-from, and --sstates-from options should contain the corresponding state definitions in a compatible format understood by BasisState.states_from_file and TargetState.states_from_file.

How it works
------------

- The command creates a work manager and initializes the WESTPA system, simulation manager, and data manager.
- It then processes states on the master process:
  - Target states are collected from a file (if provided) and from inline specifications.
  - Basis states are collected from a file (if provided) and from inline specifications.
  - Start states are collected from a file (if provided) and from inline specifications.
- The initial configuration is written to the WEST HDF5 file, and, unless disabled with --no-we, the initial segments are subjected to the weighted-ensemble setup.

Code references (for advanced users)
------------------------------------

- Entry point and argument parsing are defined in w_init.rst’s corresponding module:
  - entry_point(): builds the CLI, processes arguments, and calls initialize().
- Initialization workflow:
  - initialize(...): sets up theWESTPA runtime, processes arguments, and builds the initial states and segments.
  - Processing of files and inline states uses:
    - BasisState.states_from_file(...)
    - TargetState.states_from_file(...)
  - Inline state strings are parsed as label/probability[,auxref] (basis/start) or label/pcoord[,pcoords] (target).

Documentation notes
-------------------

- If neither --bstates-from nor --bstate is provided, a default basis state with probability 1, label "basis", and id 0 will be created (a warning is printed).
- If neither --tstates-from nor at least one --tstate is provided, an equilibrium simulation (no sinks) will be performed.

Code example (inline states)
----------------------------

- Example of specifying inline basis, target, and start states:
  w_init --bstate "basis,1.0" --tstate "stateA,0.25" --sstate "start,1.0"

- Example with multiple segments per state and force overwrite:
  w_init --bstates-from basis.csv --tstates-from targets.csv --segs-per-state 3 --force
