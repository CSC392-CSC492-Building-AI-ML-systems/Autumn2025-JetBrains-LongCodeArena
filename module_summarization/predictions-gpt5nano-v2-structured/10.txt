Using metadata
==============

This section explains how to use metadata within the ftrack Python API. Metadata provides a flexible mechanism to attach structured, application-specific data to entities managed by the ftrack server.

Overview
--------

- Metadata is a JSON-serializable data structure associated with an entity (for example, tasks, assets, events, etc.).
- It allows you to store arbitrary key-value information alongside the core entity data without modifying the core schema.
- Metadata is intended for auxiliary information such as provenance, review notes, integration data, or any domain-specific attributes that are not part of the standard entity fields.

Key concepts
------------

- Structure: Metadata is represented as a mapping (dictionary) of keys to values. Values should be JSON-serializable.
- Scope: Metadata is attached to individual entities. Depending on the API version, metadata may be accessed via a dedicated property or helper methods on entity objects.
- Persistence: Metadata changes are persisted together with the entity. The exact persistence mechanism (e.g., on entity save or session flush) depends on the backend and API version.

Supported data types
--------------------

- Strings, numbers, booleans
- Lists and dictionaries composed of the above
- Nested structures are supported as long as they are JSON-serializable

Working with metadata
---------------------

Attaching metadata to an entity
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

If the API exposes a metadata property on entities, you can assign a dictionary to it.

Example:

.. code-block:: python

    # Assuming `entity` is a loaded ftrack entity (e.g. a Task or an Asset)
    metadata = {
        'origin': 'import',
        'confirmed': True,
        'reviewers': ['alice', 'bob'],
        'parameters': {
            'threshold': 0.75,
            'mode': 'auto'
        }
    }

    # Attach metadata to the entity
    entity.metadata = metadata

Reading metadata
~~~~~~~~~~~~~~~~

Access the metadata attribute or use a getter if provided by the API.

Example:

.. code-block:: python

    # Retrieve metadata from an entity
    md = entity.metadata

    origin = md.get('origin')
    reviewers = md.get('reviewers', [])
    threshold = md.get('parameters', {}).get('threshold')

Updating metadata
~~~~~~~~~~~~~~~~~

Update the existing metadata in place and let the API persist the changes.

Example:

.. code-block:: python

    md = entity.metadata or {}
    md['reviewed'] = True
    md['last_checked'] = '2025-11-01'
    md['parameters']['mode'] = 'manual'  # if nested dict exists

    entity.metadata = md
    # Persist as per API (e.g., save the entity or flush the session)

Deleting metadata
~~~~~~~~~~~~~~~~~

Remove specific keys or clear metadata entirely.

Example:

.. code-block:: python

    md = entity.metadata or {}
    md.pop('last_checked', None)
    if not md:
        # If empty metadata should be removed, handle via API semantics
        pass
    else:
        entity.metadata = md

Persistence and lifecycle
~~~~~~~~~~~~~~~~~~~~~~~~~

- Metadata changes typically persist when you save the entity or flush the session, depending on the backend and API version.
- If your workflow relies on immediate persistence, ensure you call the appropriate save/commit methods as documented for your API version.
- Some workflows may lazy-load metadata; accessing `entity.metadata` may trigger a server request if the data is not yet loaded.

Best practices
--------------

- Use a dedicated namespace for your application’s metadata (e.g., keys prefixed with your company or project name) to avoid collisions with system or other plugins.
- Keep metadata JSON-serializable and avoid storing large binary blobs.
- Validate metadata structure before persisting to catch schema or type mismatches early.
- Document metadata conventions in your project to ensure consistent usage across scripts and integrations.

Common patterns
---------------

- Initialize metadata on first use:

.. code-block:: python

    if getattr(entity, 'metadata', None) is None:
        entity.metadata = {}

- Update nested structures carefully to avoid overwriting unrelated keys:

.. code-block:: python

    md = entity.metadata or {}
    parameters = md.get('parameters', {})
    parameters['threshold'] = 0.8
    md['parameters'] = parameters
    entity.metadata = md

Compatibility and extension
---------------------------

- If you are targeting multiple API versions, check the specific release notes for metadata support and the exact API surface (property names, getter/setter semantics).
- If your version does not yet expose a native metadata API, consider storing metadata in a dedicated custom attribute or using a structured field designed for JSON payloads, and map your application’s semantics accordingly.
