RandAugment
===========

Documentation for the RandAugment integration in NVIDIA DALI.

Module
------
nvidia.dali.auto_aug.rand_augment

This module provides the RandAugment augmentation scheme implementation for DALI. It exposes a high-level function to apply a configurable set of random augmentations to a batch of images or videos, with control over magnitude, translation behavior, and other transform parameters.

Public API
----------

rand_augment(data, n, m, num_magnitude_bins=31, shape=None, fill_value=128,
             interp_type=None, max_translate_abs=None, max_translate_rel=None,
             seed=None, monotonic_mag=True, excluded=None) -> _DataNode

Applies RandAugment to a batch of samples.

Purpose
- Implement the RandAugment scheme (as described in arXiv:1909.13719) to randomly select and apply n augmentation operations per sample, each with a magnitude determined by m and the configured magnitude bins.
- Support both monotonic and non-monotonic magnitude behavior across augmentations.
- Allow fine-grained control over transform behavior (shape-aware magnitudes, padding fill, interpolation, translation limits, and exclusions).

Key Parameters
- data: DataNode
  - Batch of samples to process. Supported: images with HWC layout and videos with FHWC layout. Data type: uint8.
- n: int
  - Number of random augmentation operations to apply per sample.
- m: int
  - Magnitude (strength) for each operation. Valid range is [0, num_magnitude_bins - 1].
- num_magnitude_bins: int
  - Number of bins to divide the magnitude ranges into (default 31).
- shape: DataNode or Tuple[int, int], optional
  - Spatial shape (height, width). If provided, translation magnitudes are scaled relative to shape (max_translate_rel) and can span up to max_translate_rel * shape.
- fill_value: int, optional
  - Padding value used by warp_affine-based transforms (translation, shear, rotate). If None, border is replicated (clamped).
- interp_type: types.DALIInterpType, optional
  - Interpolation method for warp transforms. Typical values: types.INTERP_LINEAR (default), types.INTERP_NN.
- max_translate_abs: int or (int, int), optional
  - Absolute translation limit in pixels when shape is not provided. If a tuple is given, first element is height limit, second is width limit. Default: 100.
- max_translate_rel: float or (float, float), optional
  - Translation limit as a fraction of shape when shape is provided. If a tuple is given, first element is height ratio, second is width ratio. Default ~0.45 (100 / 224).
- seed: int, optional
  - Random seed for reproducibility of operation sampling and magnitude negation.
- monotonic_mag: bool, optional
  - If True (default), strength increases with magnitude bins for all operations. If False, some color operations have non-monotonic behavior (e.g., inverse trends for certain operations). See internal guidance for the non-monotonic suite.
- excluded: List[str], optional
  - Names of augmentations to exclude from the default RandAugment suite.

Returns
- DataNode
  - A batch of transformed samples.

Notes
- The function validates the provided exclusions against the available augmentation names in the selected RandAugment suite.
- Internally, rand_augment builds a selected augmentation suite (monotonic or non-monotonic) and applies the chosen subset to the data via apply_rand_augment.
- The augmentation set can be customized further via the excluded parameter, or advanced usage can bypass the higher-level function and call apply_rand_augment directly with a predefined list of augmentations.

Usage Examples
--------------

Basic usage within a DALI pipeline:

.. code-block:: python

   from nvidia.dali.auto_aug.rand_augment import rand_augment

   # Assuming `data` is a DataNode representing a batch of images/videos
   # Apply RandAugment with 2 augmentations per sample and magnitude index 9
   augmented = rand_augment(
       data=data,
       n=2,
       m=9
   )

Shape-aware translation example:

.. code-block:: python

   augmented = rand_augment(
       data=data,
       n=2,
       m=15,
       shape=(224, 224),
       max_translate_rel=0.5,  # up to 0.5 * 224 ~ 112 pixels
       fill_value=128,
       interp_type=None  # default linear interpolation
   )

Excluding augmentations:

.. code-block:: python

   augmented = rand_augment(
       data=data,
       n=2,
       m=9,
       excluded=["rotate", "shear"]
   )

Internal helper (for advanced usage)
------------------------------------

apply_rand_augment(augmentations, data, n, m, ...) is the internal API used by rand_augment. It applies a prepared list of _Augmentation objects to the given DataNode and returns the transformed batch. This is primarily intended for advanced users who want to supply custom augmentation lists.
