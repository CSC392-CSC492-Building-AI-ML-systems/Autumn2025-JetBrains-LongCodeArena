Quantile Regression (QuantReg)
===============================

Module overview
---------------
This module implements a quantile regression estimator using iterated reweighted least squares (IRLS). It provides a QuantReg class that fits quantile regression models for a specified quantile q (0 < q < 1). The method supports robust/homoskedastic covariance estimation via kernel density estimation, following Greene (2008). The implementation also supports several kernel types and bandwidth selection strategies for the asymptotic covariance matrix.

Key features
- Quantile regression via iterative reweighted least squares (IRLS)
- Supports quantiles q in (0, 1); q = 0.5 corresponds to least absolute deviations (LAD)
- Asymptotic covariance estimation using kernel density estimation
- Multiple kernel options:
  - biw (Biweight), cos (Cosine), epa (Epanechnikov), gau (Gaussian), par (Parzen)
- Bandwidth selection methods:
  - hsheather (Hall–Sheather), bofinger (Bofinger), chamberlain
- Flexible variance-covariance options:
  - vcov='robust' (default; heteroskedasticity robust)
  - vcov='iid' (independently and identically distributed errors)
- Return type compatible with Statsmodels RegressionResultsWrapper

API reference
------------
Class: QuantReg
- Inheritance: RegressionModel

Initialization
- __init__(endog, exog, **kwargs)
  - endog: array-like
    Endogenous (response) variable
  - exog: array-like
    Exogenous (design) matrix
  - kwargs: forwarded to base class

Public methods
- whiten(data)
  - Returns data unchanged (no whitening is performed by this model)
- fit(q=0.5, vcov='robust', kernel='epa', bandwidth='hsheather',
       max_iter=1000, p_tol=1e-6, **kwargs)
  - Fits a quantile regression model for the specified quantile q
  - Parameters:
    - q: float in [0, 1]
      Target quantile to estimate
    - vcov: {'robust', 'iid'}
      Method for variance-covariance estimation
    - kernel: kernel to use for density estimation in covariance
      Options: 'biw', 'cos', 'epa', 'gau', 'par'
      Maps to Biweight, Cosine, Epanechnikov, Gaussian, Parzen
    - bandwidth: {'hsheather', 'bofinger', 'chamberlain'}
      Bandwidth selection method for the covariance estimation
    - max_iter: int
      Maximum number of IRLS iterations
    - p_tol: float
      Convergence tolerance for the parameter update
    - **kwargs: additional options forwarded to the estimator
  - Returns: RegressionResultsWrapper
    - Contains estimated coefficients, standard errors, t-stats, etc.

Notes
-----
- The estimator uses iterated reweighted least squares. Convergence is checked by the maximum absolute change in coefficients (diff) against p_tol.
- The asymptotic covariance matrix is computed using kernel density estimation, with the chosen kernel and bandwidth method.
- The LAD estimator (quantile q = 0.5) corresponds to the standard least absolute deviations regression.
- Kernel and bandwidth choices follow the literature cited in the docstring (Greene 2008; Hall & Sheather; etc.).

Usage examples
--------------

Basic usage (fit a median regression)
```python
import numpy as np
from statsmodels.regression.quantile_regression import QuantReg

# Example data
np.random.seed(123)
n = 100
X = np.column_stack((np.ones(n), np.random.randn(n)))  # add constant term
beta = np.array([1.0, 2.0])
y = X @ beta + np.random.randn(n)

# Fit quantile regression for q = 0.5 (median)
model = QuantReg(y, X)
res = model.fit(q=0.5)
print(res.summary())
```

Quantile regression with custom kernel and bandwidth
```python
import numpy as np
from statsmodels.regression.quantile_regression import QuantReg

np.random.seed(123)
n = 200
X = np.column_stack((np.ones(n), np.random.randn(n)))
beta = np.array([0.5, 1.5])
y = X @ beta + 0.8 * np.random.randn(n)

model = QuantReg(y, X)
# Use Gaussian kernel with Hall-Sheather bandwidth and iid errors
res = model.fit(q=0.75, vcov='iid', kernel='gau', bandwidth='hsheather', max_iter=800)
print(res.summary())
```

Dependencies
------------
- NumPy
- SciPy
- Statsmodels (base RegressionModel/Results infrastructure)

Notes for developers
--------------------
- The module is designed to be compatible with the Statsmodels regression framework, exposing a standard fit interface and returning a RegressionResultsWrapper.
- The whitening step is a no-op for this model; the class currently uses direct IRLS updates.
