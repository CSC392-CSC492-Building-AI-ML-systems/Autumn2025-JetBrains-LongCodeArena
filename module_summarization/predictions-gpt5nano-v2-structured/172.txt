SANS to SESANS conversion
=========================

This module provides utilities to convert SANS (Small-Angle Neutron Scattering)
data or model outputs into a SESANS (Spin-Echo Small-Angle Neutron Scattering)
representation. The conversion translates the SANS intensity I(q) into a SESANS
contrast function as a function of spin-echo length z, enabling analysis of
structures using SESANS-like diagnostics from SANS data.

Overview
--------

- Purpose: Bridge SANS and SESANS representations by transforming the
  isotropic SANS intensity I(q) into a SESANS-compatible signal, typically a
  polarization P(z) or contrast function G(z), as a function of the SESANS
  length z.
- Applicable when scattering is approximately isotropic and the sample can be
  described by a 1D (radially symmetric) scattering pattern I(q).
- Core idea: apply a Hankel (0th-order) transform to I(q) to obtain the SESANS
  correlation function, then map that to a SESANS observable (e.g., polarization
  P(z)) via the appropriate transform kernel.

Theory (conceptual)
-------------------

- Input: SANS intensity I(q) on a q-grid (or a model that can be evaluated to yield I(q)).
- Transform: compute the SESANS correlation function C(z) (often denoted G(z) in literature)
  from I(q) using a Hankel transform of order 0:
  C(z) ∝ ∫_0^∞ I(q) J0(q z) q dq
  (J0 is the Bessel function of the first kind of order 0). This relationship is
  derived under the standard SESANS formalism for isotropic scattering.
- Output: C(z) mapped to SESANS observables such as polarization P(z) with the
  appropriate normalization and unit conventions.
- Notes:
  - Real data require background subtraction and normalization as needed.
  - The transform assumes a single, non-multi-scattering sample and may require
    corrections for instrumental resolution and background.

Usage
-----

The typical workflow is:

- Provide I(q) as a measured SANS dataset or as an I(q) curve generated from a model.
- Use the sans_to_sesans tool to obtain SESANS-like output, C(z) or P(z), on a
  chosen z-grid.

CLI usage
---------

- Basic usage (input SANS data or model, produce SESANS output):

  .. code-block:: text

      python -m sasmodels.sans_to_sesans input_sans.dat output_sesans.dat

- Verbose mode (print additional information during processing):

  .. code-block:: text

      python -m sasmodels.sans_to_sesans -v input_sans.dat output_sesans.dat

- Help:

  .. code-block:: text

      python -m sasmodels.sans_to_sesans -h

Input format
------------

- The input should provide q-values and corresponding I(q) values.
- Typical formats include:
  - Two-column text files: q  I(q)
  - Or a JSON/CSV file interface if supported by your installation
- If a model is supplied instead of measured I(q), the model must be evaluable to
  produce I(q) on the required q-grid.

Output format
-------------

- The primary output is SESANS data on a z-grid:
  - z (spin-echo length) and the transformed SESANS observable (e.g., P(z) or C(z))
- Optional: intermediate diagnostic data or plots if running in verbose mode.

API and core functionality
--------------------------

- sans_to_sesans.convert(Iq, q, z_grid=None, normalize=True) -> (z, Pz)
  - Accepts I(q) as arrays and returns SESANS observable on z_grid.
- sans_to_sesans.from_model(model, q, z_grid=None) -> (z, Pz)
  - Evaluates the provided SANS model to obtain I(q) and performs the transform.
- Options:
  - z_grid: desired z-values for the output SESANS observable.
  - normalize: apply intensity normalization prior to the transform.

Notes
-----

- Units:
  - q is typically in inverse-angstroms (1/Å).
  - z is the spin-echo length, with common units depending on the instrument (often nm or μm). Ensure consistent unit handling across input and output.
- Limitations:
  - The accuracy of the SESANS representation depends on the validity of the isotropic, single-scattering assumption.
  - Background, multiple scattering, and instrumental resolution can affect the transform; apply corrections as needed.

Examples
--------

1) CLI example (basic):
   python -m sasmodels.sans_to_sesans input_sans.dat output_sesans.dat

2) Verbose CLI example:
   python -m sasmodels.sans_to_sesans -v input_sans.dat output_sesans.dat

3) Conceptual Python usage (illustrative):
   .. code-block:: python

       import numpy as np
       from sasmodels.sans_to_sesans import transform, from_model

       q = np.linspace(0.001, 0.3, 400)          # SANS q-grid
       Iq = my_sans_model.evaluate(q)            # or load from data
       z, Pz = transform(Iq, q, z_grid=np.linspace(0, 1000, 200))

       # Alternatively, from a model directly
       z2, Pz2 = from_model(my_sans_model, q, z_grid=z)
