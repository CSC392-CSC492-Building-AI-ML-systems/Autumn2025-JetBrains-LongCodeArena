How to generate schema
======================

This documentation explains how to generate and (optionally) apply the SQL schema for a database using the provided helpers.

Overview
--------
- get_schema_sql(client, safe) -> str
  - Generates the SQL schema for the given database client.
  - The safe parameter controls whether tables are created only if they do not already exist.
- generate_schema_for_client(client, safe) -> None
  - Generates the SQL schema and applies it directly to the database via the client.
  - Performs the operation asynchronously and logs the produced SQL.

Key concepts
-------------
- client: An instance of a database async client implementing the schema generation interface. It must provide a schema_generator(client) that returns a generator capable of producing and applying SQL.
- safe: If True, creates tables only if they do not already exist (e.g., using CREATE IF NOT EXISTS semantics as implemented by the underlying generator).

API reference
--------------
- get_schema_sql(client: "BaseDBAsyncClient", safe: bool) -> str
  - Description: Returns the SQL needed to create the schema for the given client.
  - How it works: It obtains a schema generator via client.schema_generator(client) and calls get_create_schema_sql(safe) on that generator.
  - Returns: A string containing the SQL statements (empty if there is no schema to create).

- generate_schema_for_client(client: "BaseDBAsyncClient", safe: bool) -> None
  - Description: Generates the schema and applies it to the database for the given client.
  - How it works: It calls get_schema_sql to obtain the SQL, logs the SQL, and if non-empty, uses the generator to execute the statements asynchronously.
  - Returns: None

Usage examples
--------------
- Get the SQL schema without applying it
```python
from tortoise.backends.base.client import BaseDBAsyncClient

# Assume `client` is an initialized and connected instance of BaseDBAsyncClient
# safe = True to create tables only if they do not already exist
schema_sql = get_schema_sql(client, safe=True)
print(schema_sql)
```

- Generate and apply the schema to the database
```python
import logging
from tortoise.backends.base.client import BaseDBAsyncClient

logger = logging.getLogger("tortoise")

async def apply_schema(client: BaseDBAsyncClient) -> None:
    # Safe = True ensures idempotent creation where possible
    await generate_schema_for_client(client, safe=True)
```

Notes and behavior
------------------
- The generated SQL is obtained via the client’s schema generator:
  - generator = client.schema_generator(client)
  - schema = generator.get_create_schema_sql(safe)
- When applying the schema, the code logs the SQL for debugging purposes:
  - logger.debug("Creating schema: %s", schema)
- If the generated schema string is empty, no actions are taken.
- The safe flag influences whether creation is conditional (e.g., CREATE IF NOT EXISTS) at the SQL level, depending on the underlying database dialect and generator implementation.
- The functionality is asynchronous when applying the schema (generate_schema_for_client).

What you need
-------------
- A functional asynchronous database client compatible with the schema generation interface (i.e., provides schema_generator(client)).
- An asynchronous execution context to run generate_schema_for_client.

This module enables straightforward generation and application of the database schema, with a clear separation between generating the SQL and applying it, and with a safe mode to avoid unnecessary recreation.
