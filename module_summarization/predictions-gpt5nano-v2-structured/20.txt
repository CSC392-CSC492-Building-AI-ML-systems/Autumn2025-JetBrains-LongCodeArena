w_direct — Direct calculation of state-to-state kinetics from WESTPA simulations
================================================================================

Overview
--------
The w_direct command computes state-to-state kinetics by tracing trajectories from a WESTPA simulation.
It operates on a bin assignment file (usually assign.h5) and produces a direct kinetics file
(default: direct.h5) containing fluxes, durations, and related statistics. The results from w_direct
are then used by subsequent subcommands (e.g., w_direct kinetics) to normalize fluxes and produce
rates, averages, and confidence intervals.

Key concepts
- Tracing trajectories: State-to-state fluxes and event durations are extracted by following labeled
  trajectories across macrostate boundaries.
- Confidence intervals: Fluxes and rates are bootstrapped (Monte Carlo Bootstrap, MCBS) with
  configurable parameters to estimate uncertainties and correlations.
- Normalization: The raw fluxes in the direct file are not normalized by the initial macrostate population.
  The w_direct kinetics subcommand performs normalization while accounting for statistical fluctuation and
  correlation.

Subcommands
----------

init
^^^^
Calculate state-to-state kinetics by tracing trajectories and generate a direct kinetics file.

Purpose
- Create a direct file (default: direct.h5) containing raw, unnormalized fluxes, arrivals, and event durations
  derived from trajectory labeling.

Inputs
- An assignments/bins file (commonly assign.h5) containing trajectory labeling information.
- Output file (default: direct.h5) to store the results.

Outputs (datasets created in the output file)
- /conditional_fluxes [iteration][state][state]
  Floating-point macrostate-to-macrostate fluxes (not normalized by the initial macrostate population).

- /conditional_arrivals [iteration][stateA][stateB]
  Integer counts of trajectories arriving at stateB in a given iteration, given departure from stateA.

- /total_fluxes [iteration][state]
  Floating-point total flux into a given macrostate.

- /arrivals [iteration][state]
  Integer counts of trajectories arriving at a given state in a given iteration.

- /duration_count [iteration]
  Integer number of event durations recorded in each iteration.

- /durations [iteration][event duration]
  Structured dataset containing:
  - istate (Integer): Initial state of the transition
  - fstate (Integer): Final state of the transition
  - duration (Floating-point): Duration of the transition, in units of tau
  - weight (Floating-point): Weight of trajectory at the end of the transition (not normalized by initial state population)

Notes
- The values in /durations carry the unnormalized trajectory weights; normalization is performed later
  by w_direct kinetics when estimating rates.

Key features
- Custom evaluation block: The rate estimation uses a dedicated evaluation path (_rate_eval_block)
  to compute rate-like quantities with MCBS-based confidence intervals.
- Confidence intervals: MCBS parameters (alpha, number of sets, autocorrelation alpha) control the
  estimation of uncertainties and correlations.
- Correlations: Optional calculation of correlations between fluxes and arrivals via MCBS.

Options (typical)
- --assignments, -a FILE
  Path to the assignments/labeling file (e.g., assign.h5).

- --output, -o FILE
  Path to the output direct kinetics file (default: direct.h5).

- --start-iteration, --stop-iteration
  Range of iterations to process (inclusive).

- --mcbs-alpha
  Significance level for MCBS confidence intervals (e.g., 0.05 for 95% CIs).

- --mcbs-nsets
  Number of bootstrap resampling sets to generate.

- --mcbs-acalpha
  Autocorrelation bootstrap alpha used in MCBS.

- --do-correl
  Enable calculation of correlations in MCBS (boolean).

- --mcbs-enable
  Enable/disable MCBS (boolean).

kinetics
^^^^^^^^
Compute rates, fluxes, and averages from a direct kinetics file.

Purpose
- Normalize fluxes to obtain state-to-state rates and compute average quantities with optional
  confidence intervals. This subcommand uses the data produced by w_direct init.

Inputs
- Direct kinetics file (default: direct.h5) produced by w_direct init.
- (Optional) Window/range for averaging.

Outputs
- /avg_rates [state, state]
  Structured dataset of rate estimates between states.

- /avg_total_fluxes [state]
  Structured dataset of total flux into each state.

- /avg_conditional_fluxes [state, state]
  Structured dataset of conditional fluxes between states.

- /rate_evolution [window][state][state] (if evolution mode is enabled)
  Structured dataset describing the evolution of rates over a specified window.

Notes
- Rates are normalized by the initial macrostate populations to yield true rate constants.
- The calculation accounts for statistical fluctuations and correlations across bootstrap samples.

Usage examples
--------------

1) Initialize direct kinetics file from trajectory labeling
- Typical invocation (options may vary by your WESTPA installation):
  w_direct init --assignments assign.h5 --output direct.h5 --start-iteration 0 --stop-iteration 10000 --mcbs-alpha 0.05 --mcbs-nsets 200 --mcbs-acalpha 0.1 --do-correl

2) Compute averaged rates and fluxes from the direct file
- Typical invocation:
  w_direct kinetics --input direct.h5 --output averages_direct.h5 --evolution-mode

Notes and tips
- Ensure you have a valid assignments file (assign.h5) prior to running w_direct init.
- The direct file contains unnormalized flux data; use w_direct kinetics to obtain rates with proper normalization.
- Adjust MCBS parameters (alpha, nsets, acalpha) to balance confidence interval width and computational cost.
