# Fleur input/output parsers

This document describes the Fleur parsers for extracting key information from Fleur XML files. The parsers are designed to be robust against version differences and to provide convenient, reusable access to commonly used pieces of information.

## Overview

The Fleur parser utilities in this module focus on extracting distinct, version-aware components from Fleur XML files:

- Calculation modes (e.g., spin setup, SOC, relaxation, LDAC, etc.)
- The number of k-points used in a calculation

The functions rely on a Fleur XML context wrapper (FleurXMLContext) and a version-aware schema dictionary to traverse and interpret the XML regardless of minor version changes.

## Public API

### get_fleur_modes(xmltree, schema_dict, logger=None) -> dict[str, Any]

Determine the calculation modes of Fleur for the given Fleur XML file. These modes describe settings that influence the produced outputs (e.g., out.xml) and can depend on the XML version.

- Parameters
  - xmltree: XML-like object representing the Fleur XML.
  - schema_dict: A Fleur schema dictionary (either InputSchemaDict or OutputSchemaDict) corresponding to the file version.
  - logger: Optional logger for emitting warnings or errors.

- Returns
  - A dictionary with keys describing the detected modes and their values. Supported keys include:
    - jspin: number of spins considered
    - noco: non-collinear calculation flag
    - soc: spin-orbit coupling flag
    - relax: structural relaxation flag
    - spex: GW/Spex-related mode indicator
    - force_theorem: force theorem calculation flag
    - film: film system flag
    - ldau: LDA+U flag
    - cf_coeff: presence of cF coefficients (cFCoeffs) or density terms
    - plot: plotting flag
    - greensf: Greens function calculation flag
    - ldahia: LDA+HIA flag for disordered or correlated systems
    - dos: density of states flag
    - band: band structure flag
    - bz_integration: Brillouin-zone integration mode

- Notes
  - The presence and meaning of some fields depend on the XML version (schema_dict.inp_version).
  - The function uses FleurXMLContext to query attributes and tag existence safely across versions.

- Example
  - Basic usage:
    ```
    from lxml import etree
    from masci_tools.io.parsers import fleur_parser
    from masci_tools.io.parsers.fleur_schema import InputSchemaDict  # or OutputSchemaDict

    xmltree = etree.parse("path/to/fleur_out.xml")
    schema_dict = ...  # obtain appropriate schema dict for this file/version
    modes = fleur_parser.get_fleur_modes(xmltree, schema_dict)
    print(modes)
    ```
  - With logging:
    ```
    import logging
    logger = logging.getLogger(__name__)
    modes = fleur_parser.get_fleur_modes(xmltree, schema_dict, logger=logger)
    ```

### get_nkpts(xmltree, schema_dict, logger=None) -> int

Get the number of k-points used in the calculation described by the given Fleur XML file.

- Parameters
  - xmltree: XML-like object representing the Fleur XML.
  - schema_dict: A Fleur schema dictionary (either InputSchemaDict or OutputSchemaDict) corresponding to the file version.
  - logger: Optional logger for warnings or errors.

- Returns
  - An integer representing the number of k-points.

- Behavior and notes
  - The function resolves the currently selected k-point list by reading the listName attribute and validating it against available kPointList entries.
  - It reads the count from the matching kPointList entry.
  - If the selected listName is not found, a ValueError is raised listing available list names.
  - If the resolved count is not an integer, a ValueError is raised.
  - The implementation is version-aware via the schema dispatch decorator, allowing correct handling across Fleur versions.

- Example
  - Basic usage:
    ```
    from lxml import etree
    from masci_tools.io.parsers import fleur_parser
    from masci_tools.io.parsers.fleur_schema import InputSchemaDict  # or OutputSchemaDict

    xmltree = etree.parse("path/to/fleur_out.xml")
    schema_dict = ...  # obtain appropriate schema dict for this file/version
    nkpts = fleur_parser.get_nkpts(xmltree, schema_dict)
    print(f"Number of k-points: {nkpts}")
    ```
  - With logging:
    ```
    import logging
    logger = logging.getLogger(__name__)
    nkpts = fleur_parser.get_nkpts(xmltree, schema_dict, logger=logger)
    ```

## How these parsers work

- The parsers use FleurXMLContext to provide a consistent, version-aware interface to the XML:
  - attribute(name, ...) retrieves XML attributes with optional defaults and typing.
  - tag_exists(tag, ...) checks for the presence of specific XML elements.
  - find(...) allows scoped queries within a subtree, honoring versioned structures.
- get_fleur_modes inspects a set of well-defined flags that commonly influence input/output behavior:
  - Spin configuration (jspin/noco)
  - Spin-orbit coupling (soc)
  - Relaxation vs. static calculation (relax)
  - Special modes (spex, greensf, ldau, etc.)
  - Output-related flags (plot, dos, band, bz_integration)
- get_nkpts resolves the active k-point set by name and reads its count, ensuring it matches the expected structure for the given Fleur version.

## Versioning and compatibility

- get_nkpts is decorated with @schema_dict_version_dispatch, enabling different implementations depending on the schema version. This ensures compatibility across Fleur releases.
- get_fleur_modes relies on the features available in the provided schema_dict (e.g., presence of greensfCalculation, torqueCalculation, etc.). The function adapts its checks according to schema_dict.inp_version.

## Practical tips

- Ensure you pass the correct schema dictionary corresponding to the XML file version. Mismatched schema dictionaries can lead to incorrect parsing or missing fields.
- When debugging, enable a logger to trace decisions and missing tags (e.g., when a particular mode cannot be determined due to version differences).
- These parsers are designed to be lightweight and reusable; you can call them multiple times on different Fleur XML files once you have the appropriate schema context.

## References

- fleur_schema: defines the schema dictionary types and version handling logic.
