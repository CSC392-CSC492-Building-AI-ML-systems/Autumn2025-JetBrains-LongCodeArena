Using review sessions
=====================


Overview
--------

A review session provides a collaborative workspace to examine and comment on
assets, versions, or other reviewable items within the ftrack environment.
Review sessions leverage the same session infrastructure used to connect to an
ftrack server and can optionally integrate with the event hub to notify
participants about updates and changes.

Key concepts
------------

- ReviewSession: a container for items under review, including metadata such as
  title, description, status, owners, and reviewers.
- ReviewItem: an individual object (e.g. asset, version, or other reviewable
  entity) included in a ReviewSession.
- ReviewComment: feedback added by participants on a ReviewItem within a
  ReviewSession.
- Status lifecycle: typically includes open, in_progress, changes_requested,
  approved, and closed, though exact states may vary by deployment.

Prerequisites
-------------

- A functioning ftrack server reachable from your environment.
- Access credentials (server URL, api_key, api_user) to authenticate with
  the ftrack API.
- Optional: an embedded or external event hub for real-time notifications.

to match your deployment’s API surface and plugin configuration.

Working with review sessions
---------------------------

- Lifecycle management: create, modify, and close ReviewSessions as the review
  progresses. Use status transitions to reflect current state (open, in_progress,
  changes_requested, approved, closed).

- Permissions and ownership: assign owners and reviewers to control who can view
  and contribute to the session.

- Visibility: depending on your server configuration, review sessions and items
  may be visible to a subset of users or teams.

Example workflow: end-to-end review
-----------------------------------

- A creator opens a new ReviewSession for a set of assets/versions.
- Reviewers are assigned and notified (via the event hub if enabled).
- Reviewers add comments on specific items within the session.
- The creator addresses comments, updates items, and updates the session status.
- Once all concerns are resolved, the session is approved and archived.

Code considerations
-------------------

- Use a single Session instance for all review-related API interactions to take
  advantage of caching, background event hub integration, and consistent
  authentication.
- If you enable the embedded event hub, review-related events may be published
  asynchronously. Ensure your application handles asynchronous updates or
  subscribes to relevant events as needed.

API surface references (conceptual)
-----------------------------------

- Session: central entry point for server interaction, including access to
  review-related operations if provided by the deployment.
- ReviewSession: container object representing a review, with attributes such
  as id, name, status, owners, and reviewers.
- ReviewItem: individual items included in a ReviewSession.
- ReviewComment: comments attached to a ReviewItem within a ReviewSession.
- Event hub integration (optional): for publishing and subscribing to review
  related non-local events.

Notes
-----

- The exact class and method names for creating and managing review sessions
  depend on your ftrack API version and any custom plugins installed in your
  environment. Refer to your deployment’s API reference and plugin docs for the
  precise surface area.
