MapPlotter
==========

Overview
--------

The MapPlotter class provides an interactive plotting utility for spectral data cubes. It renders a 2D map (a plane extracted from a cube) and enables interactive operations such as:

- Clicking on the map to plot and inspect the corresponding spectrum
- Overplotting additional spectra in an external window
- Disconnecting the interactive viewer
- Averaging over circular regions by click-drag or by setting circle center and radius via keyboard shortcuts

The component is designed primarily for interactive analysis rather than publication-quality plotting. When available, it leverages APLPy to produce publication-quality figures; otherwise, it falls back to standard matplotlib plotting.

Key features

- Uses APLpy when available to create FITS-based color-scale plots with WCS information.
- Falls back to matplotlib if APLpy is not installed.
- Automatic handling of FITS headers to establish the correct WCS projection.
- Interactive controls via mouse and keyboard (as documented in the class docstring).

Dependencies
------------

- Optional: APLpy (for publication-quality plots)
- Astropy (WCS support via astropy.wcs)
- pyfits or astropy.io.fits (for FITS handling)
- matplotlib (for plotting)
- six, numpy, copy, itertools (utility libraries)

If APLpy is unavailable, MapPlotter uses matplotlib’s imshow for visualization.

Class: MapPlotter
-----------------

Description
-----------

A helper class to plot a 2D map from a 3D data cube and provide interactive spectrum plotting and overplotting.

Initialization
--------------

MapPlotter(Cube=None, figure=None, doplot=False, **kwargs)

- Cube: The cube-like object from which a 2D plane will be extracted. If provided, its header is flattened to create a WCS-aware header for plotting.
- figure: Either a matplotlib Figure instance or an integer figure number. If None, a new figure is created when plotting.
- doplot: If True, immediately generate the map plot by calling mapplot().

- Additional kwargs are forwarded to the internal plotting routines.

Important attributes (set during initialization)
-----------------------------------------------

- figure: Target matplotlib figure
- axis: The plotting axis used for the map
- FITSFigure: If using APLPy, the APLPy FITSFigure instance
- header: Flattened FITS header derived from Cube.header (via cubes.flatten_header)
- wcs: WCS object created from the header (if available)
- plane: 2D data plane derived from the cube (via makeplane)
- _origin: Coordinate origin convention (0 for FITS, 1 for PIXEL-based units)
- _click_marks, _circles: Internal state for interactive annotations
- _clickX, _clickY: Last clicked pixel coordinates
- overplot_colorcycle, overplot_linestyle: Styling for overplots and traces
- Cube: The input cube object used to generate the plane
- canvas: Figure canvas used for event connections

Public methods
--------------

__call__(self, **kwargs)
    See mapplot. Enables a convenient call-style to generate the map plot with optional keyword overrides.

mapplot(self, convention='calabretta', colorbar=True, useaplpy=True,
        vmin=None, vmax=None, cmap=None, plotkwargs={}, **kwargs)
    Generate and display the interactive map plot from the input cube.

    Parameters
    - convention: Projection convention ('calabretta' by default; 'griesen' supported)
    - colorbar: Show a colorbar alongside the map
    - useaplpy: Use APLpy if a FITS header is available
    - vmin, vmax: Data value range for color scaling (None to auto-scale)
    - cmap: Colormap name or object
    - plotkwargs: Additional keyword arguments passed to color scale plotting
    - kwargs: Additional arguments passed to the underlying plotting routine
    - estimator (via kwargs): Optional estimator to select a map plane (handled by makeplane)

    Notes
    - This method handles both APLpy-based plotting and pure matplotlib plotting depending on environment and availability of APLpy.
    - The interactive features described in the MapPlotter docstring (click to plot spectra, overplot, disconnect, circle averaging) are wired up in this method via event connections.

_connect(self)
    Connect mouse and keyboard events to internal handlers:
    - button_press_event -> click
    - button_release_event -> plot_spectrum
    - key_press_event -> plot_spectrum

_disconnect(self)
    Disconnect all previously registered event handlers to gracefully stop interactivity.

Usage
-----

- Basic usage (with a pre-built cube-like object):

  ```python
  from cubes.mapplot import MapPlotter  # path may vary depending on your package layout
  # Assuming `my_cube` is a cube-like object with a `.header` attribute
  mp = MapPlotter(Cube=my_cube)
  mp.mapplot()  # or mp() to use __call__
  ```

- Suppress the colorbar and force a specific color map:

  ```python
  mp = MapPlotter(Cube=my_cube)
  mp.mapplot(colorbar=False, cmap='viridis')
  ```

- Interactive usage:

  ```python
  mp = MapPlotter(Cube=my_cube)
  mp()  # Opens the interactive map window
  # Use mouse buttons and keys as documented in the class docstring to inspect spectra and manage overplots
  ```

Notes
-----

- The MapPlotter relies on a cube-like input that provides necessary metadata (header) and data to generate a 2D plane via makeplane. The header is flattened using cubes.flatten_header for WCS-based projection.
- If APLpy is installed, a FITSFigure backed by APLpy will be used to render the map; otherwise, a standard matplotlib imshow is used.
- The interactive capabilities (spectrum plotting, overplotting, circle averaging) are described in the class’ docstring and are wired through event callbacks.

Example workflow
----------------

1. Load or construct a cube object with a header:
   - Ensure the object has a header attribute accessible as .header.
2. Create a MapPlotter with the cube.
3. Plot the map with mapplot() or via the __call__ method.
4. Interact with the plot to extract and compare spectra, or to overplot multiple spectra.

This module is intended as an interactive analysis aid rather than a final publication-quality plotting tool. For publication-quality visuals, enable APLpy where available.
