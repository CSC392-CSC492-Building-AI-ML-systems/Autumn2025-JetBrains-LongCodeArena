Grid Search: Tuning the hyper-parameters of an estimator
========================================================

Overview
--------
GridSearchCV exhaustively evaluates all parameter combinations in a specified grid to determine the best performing estimator according to a scoring metric. It uses cross-validation to provide a robust estimate of performance and can be applied to any estimator that exposes a parameter grid compatible with scikit-learn’s naming conventions.

Key concepts
- Parameter grid: a dict or list of dicts mapping estimator parameter names to lists of values to try.
- Nested parameters: for pipelines or composite estimators, parameter names use double underscores to reach nested parameters (e.g., 'clf__C' for a parameter C in a nested estimator named 'clf').
- Cross-validation: the cv argument controls the splitting strategy used to evaluate parameter combinations.
- Scoring: the scoring argument specifies the metric used to compare parameter combinations (e.g., 'accuracy', 'neg_mean_squared_error').
- Refitting: after finding the best parameters, the best estimator can be refitted on the entire dataset to produce a final model ready for deployment.

Usage
-----
Basic usage with a standalone estimator:

.. code-block:: python

    from sklearn.model_selection import GridSearchCV
    from sklearn.svm import SVC
    from sklearn.datasets import load_iris

    X, y = load_iris(return_X_y=True)

    param_grid = {
        'C': [0.1, 1.0, 10.0],
        'gamma': [0.001, 0.01, 0.1],
        'kernel': ['rbf']
    }

    grid_search = GridSearchCV(
        estimator=SVC(),
        param_grid=param_grid,
        scoring='accuracy',
        cv=5,
        n_jobs=-1
    )
    grid_search.fit(X, y)

    print("Best parameters:", grid_search.best_params_)
    print("Best score (accuracy):", grid_search.best_score_)
    print("Best estimator:", grid_search.best_estimator_)

Tuning hyper-parameters with pipelines
-------------------------------------
When using a pipeline, prefix parameter names with the step name followed by two underscores.

.. code-block:: python

    from sklearn.pipeline import Pipeline
    from sklearn.preprocessing import StandardScaler
    from sklearn.linear_model import LogisticRegression

    X, y = load_iris(return_X_y=True)

    pipe = Pipeline([
        ('scaler', StandardScaler()),
        ('clf', LogisticRegression(max_iter=1000))
    ])

    param_grid = {
        'clf__C': [0.1, 1.0, 10.0],
        'clf__penalty': ['l1', 'l2'],
        'clf__solver': ['liblinear']  # required for 'l1' with LogisticRegression
    }

    grid_search = GridSearchCV(pipe, param_grid, scoring='accuracy', cv=5, n_jobs=-1)
    grid_search.fit(X, y)

    print("Best parameters:", grid_search.best_params_)
    print("Best estimator:", grid_search.best_estimator_)

Parameter grid
--------------
- A parameter grid can be:
  - A dict of parameter names to lists of values.
  - A list of dicts to explore multiple, potentially disjoint parameter spaces.

- For nested estimators (e.g., inside a Pipeline), use the double-underscore syntax to reach the inner parameter:
  - Example: 'clf__C' targets the parameter C of the estimator named 'clf' in the pipeline.

Cross-validation and scoring
----------------------------
- cv controls the cross-validation splitting strategy. It can be an integer (k-fold cross-validation) or a cross-validation splitter object.
- scoring specifies the metric to optimize. It can be a string (e.g., 'accuracy', 'neg_mean_squared_error') or a callable.
- You can obtain multiple metrics by passing a dictionary to scoring. The results are available in the grid search results under corresponding keys (e.g., 'mean_test_accuracy', 'std_test_accuracy').

Refitting and final estimator
-----------------------------
- refit determines whether the best parameters are refitted on the full training data after the grid search. Default is True.
- If refit is False, the best_params_ and best_score_ are computed, but best_estimator_ is not fitted on the whole dataset.
- You can set refit to a string to specify which scorer to use for refitting when multiple scorers are provided.

Parallelization
---------------
- n_jobs controls the number of jobs to run in parallel. -1 means “use all processors,” None means 1 processor.

Accessing results
-----------------
- best_params_: parameter setting with the best score.
- best_score_: best mean cross-validated score.
- best_estimator_: estimator corresponding to the best parameters (after refitting if enabled).
- cv_results_: dict with detailed cross-validation results, including mean_test_score, std_test_score, and the parameter settings for each candidate.

See Also
--------
- RandomizedSearchCV: for randomized hyper-parameter optimization.
- HalvingGridSearchCV: progressive search strategies (if available in your version).

Examples
--------

1) Basic grid search for a classifier

.. code-block:: python

    from sklearn.model_selection import GridSearchCV
    from sklearn.ensemble import RandomForestClassifier
    from sklearn.datasets import load_breast_cancer

    X, y = load_breast_cancer(return_X_y=True)

    param_grid = {
        'n_estimators': [100, 200],
        'max_depth': [None, 10, 20],
        'min_samples_split': [2, 5]
    }

    grid_search = GridSearchCV(RandomForestClassifier(), param_grid, cv=5, scoring='accuracy', n_jobs=-1)
    grid_search.fit(X, y)

    print("Best parameters:", grid_search.best_params_)
    print("Best score:", grid_search.best_score_)
    print("Best estimator:", grid_search.best_estimator_)

2) Grid search with a pipeline

.. code-block:: python

    from sklearn.pipeline import Pipeline
    from sklearn.preprocessing import StandardScaler
    from sklearn.svm import SVC
    from sklearn.datasets import load_iris

    X, y = load_iris(return_X_y=True)

    pipe = Pipeline([
        ('scaler', StandardScaler()),
        ('svc', SVC())
    ])

    param_grid = {
        'svc__C': [0.1, 1, 10],
        'svc__gamma': [0.001, 0.01, 0.1],
        'svc__kernel': ['rbf']
    }

    grid_search = GridSearchCV(pipe, param_grid, cv=5, scoring='accuracy')
    grid_search.fit(X, y)

    print("Best parameters:", grid_search.best_params_)
    print("Best estimator:", grid_search.best_estimator_)

Results exploration
-------------------
- After fitting, inspect cv_results_ to understand performance per parameter combination:

.. code-block:: python

    import pandas as pd

    results = grid_search.cv_results_
    df = pd.DataFrame(results)
    # Display mean test score for each parameter combination
    mean_scores = df[['params', 'mean_test_score']].sort_values(by='mean_test_score', ascending=False)
    print(mean_scores.head())

Notes
-----
- For large hyper-parameter spaces, consider RandomizedSearchCV or HalvingGridSearchCV for efficiency.
- When using pipelines, ensure parameter names correspond to the correct step (e.g., 'estimator__param').

This documentation covers how to define parameter grids, apply cross-validation, interpret results, and implement practical grid search workflows for tuning estimators.
