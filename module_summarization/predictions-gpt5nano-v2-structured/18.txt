# w_bins

Display information about binning in a WEST simulation or rebuild the binning for the current iteration.

Overview
- w_bins is a WESTTool with two subcommands:
  - info: show information about the current (or a specified) iteration's binning.
  - rebin: rebuild the current iteration using a (potentially) new binning configuration.

- The tool relies on:
  - WESTDataReader to access simulation data.
  - BinMappingComponent to describe and apply bin mappings.
  - write_bin_info to present binning statistics in a readable form.

Subcommands

1) w_bins info

Purpose
- Print summary and, optionally, detailed per-bin information about the binning used in a given iteration.

Usage
- Default: show a summary for the current iteration.
- Optional flags:
  - -n, --n-iter N_ITER: operate on the initial points of segment N_ITER (default: current iteration).
  - --detail: include detailed per-bin information in addition to the summary.
  - Additional options provided by the binning mapper (via BinMappingComponent).

Example
- Show binning info for the current iteration (summary only):
  - w_bins info

- Show binning info for iteration 5 with details:
  - w_bins info -n 5 --detail

What you’ll see
- A header line indicating the iteration being reported.
- Bin counts, weights, and assignments as produced by the current bin mapper.
- If --detail is used, per-bin statistics in addition to the overall summary.

2) w_bins rebin

Purpose
- Rebuild the current iteration using a (potentially) new binning configuration.
- Optionally commit the new binning to the HDF5 backing store.

Usage
- Required: binning configuration provided via the BinMappingComponent (shared options with w_bins info).
- Optional flags:
  - -n, --n-iter N_ITER: operate on the initial points of segment N_ITER (default: current iteration).
  - --detail: display detailed per-bin information in addition to the summary.
  - --confirm: write the revised iteration to HDF5 and update internal state; without this, the operation is computed and shown but not saved.
  - Additional options from the bin mapper (bin target counts, etc.). When using rebin, --bins-from-file may be suppressed to avoid conflicts.

Notes
- Rebin is not supported for the very first iteration. If you attempt rebin on iteration 1, the tool will terminate with an error suggesting use of w_init instead.
- Rebinning operates on the current iteration’s initial points and may merge or split walkers according to the new binning.
- If you pass --confirm, the tool will:
  - Prepare and save the new iteration to disk.
  - Update endpoint statuses for the previous iteration.
  - Save the revised binning and update initial states and backing data structures.

Example
- Rebuild the current iteration with new binning and print details, but do not commit:
  - w_bins rebin --detail

- Rebuild iteration 3 with new binning and commit:
  - w_bins rebin -n 3 --confirm --detail

Key concepts

- Bin mapper (mapper)
  - The binning configuration is provided by a BinMappingComponent.
  - The mapper assigns initial points to bins based on the current or specified iteration.
  - The same mapper is used for both info display and rebin operations.

- Data access
  - WESTDataReader provides access to target states, iteration groups, and per-segment data (weights, pcoords, etc.).
  - For rebin, the West driver (we_driver) computes a new iteration using the current bin mapper and target counts.

- Output
  - write_bin_info prints binning statistics (and per-bin details when requested) using information from the mapper, assignments, and weights.

Code references (high level)
- Subcommands wired via add_args:
  - info: -n/--n-iter, --detail, and binning options.
  - rebin: --confirm, --detail, -n/--n-iter, and binning options (with suppression of --bins-from-file as needed).
- Core functions:
  - cmd_info: reads initial pcoords, maps to bins, prints info via write_bin_info.
  - cmd_rebin: builds a new iteration with we_driver, computes new assignments/weights, prints info, and optionally saves changes.

Examples

- Display info for the current iteration (summary only):
  w_bins info

- Display detailed info for iteration 7:
  w_bins info -n 7 --detail

- Rebuild current iteration with new binning, showing details but not saving:
  w_bins rebin --detail

- Rebuild iteration 4 and commit changes to disk:
  w_bins rebin -n 4 --confirm

See Also
- w_init: initialize a WEST simulation, often used before first rebin.
- w_bins info --detail: available to inspect per-bin statistics in depth.

Notes
- Requires a WEST simulation context (WESTDataReader, BinMappingComponent).
- The exact binning options depend on the bin mapper configuration provided by your WEST setup.
