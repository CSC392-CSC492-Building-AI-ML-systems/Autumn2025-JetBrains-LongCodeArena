Getting Started
===============

Overview
--------
This module provides functionality common to master and parent processes in Mitogen. It is distributed to child contexts that become parents during recursive connections. It contains utilities, environment checks, and helpers that support reliable process creation, PTY handling, and core source management for Mitogen's distributed execution model.

Prerequisites
-------------
- Python compatibility: supports both Python 2 and Python 3 environments (with fallbacks for older Python versions).
- A POSIX-like environment for PTY and process control features (typical for Unix-like systems).
- Optional: SELinux considerations may affect PTY usage and inter-process communication.

Quick Start
-----------
This section highlights a few commonly used helpers. In most workflows, you interact with the higher-level Mitogen interfaces; this module provides support utilities you may leverage in advanced scenarios or when debugging.

- Determine the Python executable used for child processes
  This helper returns the appropriate interpreter to spawn for child contexts, falling back to a sensible default and issuing a warning if the system executable cannot be determined.

  .. code-block:: python

     # Example: Determine the Python executable for child processes
     from mitogen.parent import get_sys_executable

     exe = get_sys_executable()
     print("Child interpreter:", exe)

- Check current logging level
  Retrieve the effective logging level used by Mitogen.

  .. code-block:: python

     from mitogen.parent import get_log_level
     print("Mitogen log level:", get_log_level())

- Access Mitogen core source (for advanced use)
  This module can provide access to the Mitogen core source (minified when used for transmission to children). Use this in advanced scenarios where you need to inspect or transmit the core source.

  .. code-block:: python

     from mitogen.parent import get_core_source_partial
     core_blob = get_core_source_partial()
     # core_blob is a PartialZlib object containing the encoded core source

Key Concepts
------------
- Pre-exec hook
  A pre-exec hook (_preexec_hook) is available to influence scheduling before exec() of a new child process. This is used internally to allow the child to be scheduled without constraints. It is intended for advanced customization.

  .. code-block:: python

     # Advanced: customize behavior before spawning a child
     import mitogen.parent as parent
     def my_preexec():
         # example: adjust process attributes before exec
         pass

     parent._preexec_hook = my_preexec

- Core source minification for child contexts
  In master contexts, the Mitogen core source may be prepared and minified for transmission to child contexts. The helper get_core_source_partial() caches and returns a compressed representation (PartialZlib) of the core source to optimize transfers.

  .. code-block:: python

     from mitogen.parent import get_core_source_partial
     partial = get_core_source_partial()
     # partial contains the compressed core source ready for transfer

- Environment and system capability checks
  The module performs runtime checks to determine features and limitations:
  - SELINUX_ENABLED: detects whether SELinux enforcement is active by probing /sys/fs/selinux/enforce.
  - OPENPTY_MSG: provides guidance when PTY allocation fails (e.g., due to hitting the PTY limit).
  - SYS_EXECUTABLE_MSG: warns when Python cannot determine its own executable name.

  These checks help Mitogen adapt to diverse environments and provide helpful diagnostics.

API Reference (high level)
--------------------------
- get_log_level()
  Return the effective logging level for Mitogen.

- get_sys_executable()
  Return sys.executable if set; otherwise return "/usr/bin/python" and log a one-time warning.

- get_core_source_partial()
  Return a PartialZlib-wrapped, encoded representation of the Mitogen core source. Used to minimize the cost of transmitting core code to child contexts.

- _preexec_hook
  Internal hook that can be set to customize behavior just before exec() in a child process. Intended for advanced usage.

- SELINUX_ENABLED
  Boolean indicating whether SELinux is enabled (read at import time).

- OPENPTY_MSG
  Guidance message for PTY allocation failures.

- SYS_EXECUTABLE_MSG
  Warning message when the Python executable cannot be determined.

- LINUX_TIOCGPTN, LINUX_TIOCSPTLCK
  Platform-specific ioctl constants adjusted for Python 2/3 compatibility to manage PTYs.

Notes
-----
- This module is intended for internal and advanced use. Most users should interact with higher-level Mitogen APIs, while these helpers support process creation, environment adaptation, and efficient core-source handling.
- The get_sys_executable() function logs a one-time warning if the executable cannot be determined, then falls back to a safe default.
- The core source minimization path is primarily relevant when Mitogen is used to spawn and provision child contexts; end users typically do not need to call get_core_source_partial() directly.

Example: Minimal usage in an advanced workflow
-----------------------------------------------
- Inspect environment and capabilities
  .. code-block:: python

     import logging
     from mitogen.parent import SELINUX_ENABLED, OPENPTY_MSG, SYS_EXECUTABLE_MSG

     logging.info("SELinux enabled: %s", SELINUX_ENABLED)
     if not SELINUX_ENABLED:
         logging.info("PTYs are available; be mindful of limits.")

- Prepare a child context with the appropriate Python interpreter
  .. code-block:: python

     from mitogen.parent import get_sys_executable
     child_py = get_sys_executable()
     # Use child_py to spawn a new Mitogen child context
