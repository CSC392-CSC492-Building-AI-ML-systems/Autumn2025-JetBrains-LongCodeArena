BaseUser
==========

Overview
--------
BaseUser is a simple authentication-ready user model built on Piccolo ORM. It defines a
table named piccolo_user and provides out-of-the-box password hashing, validation,
and login capabilities. Passwords are stored securely using PBKDF2 with SHA-256.

Key features
- Automatic password hashing when setting or creating users
- Configurable password policy (minimum/maximum length)
- PBKDF2-SHA256 with a per-password salt and a high iteration count
- Password storage format: pbkdf2_sha256$<iterations>$<salt>$<hash>
- Password validation for creation and updates
- Synchronous and asynchronous helpers for updating passwords and logging in
- Readable representation for debugging and admin interfaces

Module and class
----------------
- Module: baseuser
- Class: BaseUser (subclass of Piccolo’s Table)
- Table name: piccolo_user

Schema and fields
-----------------
- id: Serial
- username: Varchar(length=100, unique=True)
- password: Secret(length=255)  # Stores the hashed password
- first_name: Varchar(null=True)
- last_name: Varchar(null=True)
- email: Varchar(length=255, unique=True)
- active: Boolean(default=False)
- admin: Boolean(default=False, help_text="An admin can log into the Piccolo admin GUI.")
- superuser: Boolean(default=False, help_text="If True, this user can manage other users' passwords in the Piccolo admin GUI.")
- last_login: Timestamp(null=True, default=None, required=False, help_text="When this user last logged in.")

Internal helpers and constants
------------------------------
- _min_password_length: 6
- _max_password_length: 128
- _pbkdf2_iteration_count: 600_000  # OWASP-recommended iterations

Password hashing and validation
-------------------------------
- __init__(**kwargs)
  - If a plain password is provided at construction time, it is hashed automatically unless it already starts with pbkdf2_sha256.

- get_salt()
  - Generates a random hex salt for password hashing.

- hash_password(password, salt="", iterations=None)
  - Hashes a raw password using PBKDF2-HMAC-SHA256 with the given salt and iteration count.
  - If salt is not provided, a new salt is generated via get_salt().
  - Returns a string in the form: pbkdf2_sha256$<iterations>$<salt>$<hash>

- __setattr__(name, value)
  - If setting the password attribute to a non-hashed value, it is automatically hashed.

- split_stored_password(password)
  - Splits a stored password into its components: algorithm, iterations, salt, hash.
  - Validates the format and raises ValueError if it cannot be split.

- _validate_password(password)
  - Validates a raw password against policy:
    - Must be provided
    - Length within [min, max]
    - Must not already be a hashed password (does not start with pbkdf2_sha256)

Password updating
-----------------
- update_password(user, password)
  - Async method to update a user’s password.
  - user can be a username (str) or an id (int).
  - Validates the new password, hashes it, and updates the stored value.

- update_password_sync(user, password)
  - Sync wrapper around update_password (uses run_sync under the hood).

Usage example (password update)
--------------------------------
- Update a user’s password by username:
  await BaseUser.update_password("jdoe", "NewP@ssw0rd")

- Sync variant:
  BaseUser.update_password_sync("jdoe", "NewP@ssw0rd")

Readable representation
-----------------------
- get_readable()
  - Returns a Readable representation for debugging/admin GUIs.
  - Example: username is used as the display column.

Authentication and login
------------------------
- login(username, password)
  - Async method to authenticate a user.
  - Validates input lengths to prevent abuse.
  - Looks up the user by username, compares the provided password against the stored hash.
  - If valid, updates last_login and returns the user’s id; otherwise returns None.

- login_sync(username, password)
  - Sync wrapper around login.

Usage example (login)
---------------------
- Async login:
  user_id = await BaseUser.login("alice", "correcthorsebatterystaple")
  if user_id is not None:
      print(f"Logged in as user id: {user_id}")

- Sync login:
  user_id = BaseUser.login_sync("alice", "correcthorsebatterystaple")
  if user_id is not None:
      print(f"Logged in as user id: {user_id}")

Readable password format
------------------------
- Stored password format example:
  pbkdf2_sha256$600000$saltvalue$hashvalue
- split_stored_password can be used to inspect components during debugging or migration.

Practical notes
---------------
- Security
  - Passwords are never stored in plaintext; hashing uses a strong PBKDF2-HMAC-SHA256 with a large iteration count and a random salt.
  - Do not pass hashed passwords to create/update as raw passwords; the class guards against this and raises an error if a hashed string is provided where a raw password is expected.

- Migration and compatibility
  - If upgrading from older Piccolo versions, existing password hashes may be upgraded to use current iteration counts upon successful login.

- Admin capabilities
  - The admin and superuser flags control access to the admin GUI and password management in the admin UI.
