Zero-Inflated Discrete Models
==============================

This module implements zero-inflated count models for discrete data. It provides a generic base class for zero-inflated models and specialized classes for common count distributions, including ZeroInflatedPoisson, ZeroInflatedGeneralizedPoisson, and ZeroInflatedNegativeBinomialP.

Public API
----------
- __all__ = ["ZeroInflatedPoisson", "ZeroInflatedGeneralizedPoisson", "ZeroInflatedNegativeBinomialP"]
- GenericZeroInflated (internal base class for zero-inflated models)
- ZeroInflatedPoisson
- ZeroInflatedGeneralizedPoisson
- ZeroInflatedNegativeBinomialP

Overview
--------
Zero-inflated models account for excess zeros in count data by combining a binary inflation process with a count model for the main data-generating process. This module relies on a two-part specification:

- A binary inflation model to determine the probability of an observation being an "inflated" zero.
- A main count model (e.g., Poisson, Generalized Poisson, Negative BinomialP) for the non-inflated observations.

Key ideas implemented in GenericZeroInflated
--------------------------------------------
- Endogenous data (endog) and exogenous design (exog) for the main count model.
- Optional exogenous design for the inflation model (exog_infl). If not provided, inflation is modeled with a constant term.
- Inflation link function options: 'logit' (default) or 'probit'.
- Compatibility with the base count-model framework (DiscreteModel / CountModel) and integration with the usual fit machinery.
- Separate parameter vectors for the inflation model and the main count model.

Initialization and attributes
-----------------------------
- endog: array_like
  Endogenous response variable.
- exog: array_like
  Exogenous design matrix for the main count model.
- exog_infl: array_like or None
  Design matrix for the zero-inflation (inflation) model. If None, a constant term is assumed.
- k_inflate: int
  Number of inflation-related parameters (columns of exog_infl, or 1 if None).
- k_exog: int
  Number of main count-model parameters (columns of exog).
- inflation: {'logit', 'probit'}
  The inflation model type.
- model_infl
  Binary model (Logit or Probit) used for the inflation part.
- _hessian_inflate
  Function for computing the Hessian of the inflation model (logit or probit).
- exog_names
  Names for the main exogenous variables and the inflation terms.
- _init_keys, _null_drop_keys
  Internal bookkeeping for state and parameter handling.

Log-likelihood and fitting
--------------------------
- loglike(params): float
  Log-likelihood for the entire zero-inflated model at the given parameters.
- loglikeobs(params): ndarray
  Per-observation log-likelihoods for the current parameters.
- fit(start_params=None, method='bfgs', maxiter=35, ...):
  Fit the model by maximizing the log-likelihood. If start_params is not provided, sensible defaults are computed from the data.

Notes on the underlying model
-----------------------------
The zero-inflated log-likelihood combines the inflation probability with the main-count model, yielding:
- For y_i = 0: log(w_i + (1 - w_i) * P_main(y_i=0))
- For y_i > 0: log(1 - w_i) + L_main(y_i)
where w_i is the inflation probability and L_main is the log-likelihood contribution from the main count model.

Usage
-----
The module exposes specialized classes that implement zero-inflated versions of common count models. Typical usage constructs the data, defines both the main exogenous design and the inflation design (if desired), and fits the model.

Example: Zero-Inflated Poisson
--------------------------------
Note: Import paths may vary slightly by version; adjust imports as appropriate for your environment.

```python
import numpy as np
from statsmodels.discrete.discrete_model import ZeroInflatedPoisson
import statsmodels.api as sm

# Generate synthetic data (example)
n = 200
X = np.column_stack((np.ones(n), np.random.randn(n)))  # main exog, with intercept
Z = np.column_stack((np.ones(n), np.random.randn(n)))  # inflation exog

endog = np.random.poisson(lam=np.exp(X @ np.array([0.5, 0.2])))

# Fit Zero-Inflated Poisson
model = ZeroInflatedPoisson(endog=endog, exog=X, exog_infl=Z, inflation='logit')
result = model.fit()

print(result.summary())
```

Practical notes
---------------
- If exog_infl is not provided, the inflation model uses a constant term.
- The inflation model can be specified with a logit or probit link.
- The module integrates with the usual Statsmodels fitting interface, so you can obtain standard results objects and summaries.

Internal details
----------------
- GenericZeroInflated implements the shared logic for all zero-inflated count models, including the parameter partitioning between inflation and main-count components, and the corresponding log-likelihood calculations.
- The concrete zero-inflated classes (Poisson, Generalized Poisson, Negative BinomialP) instantiate GenericZeroInflated with the appropriate count distribution.
