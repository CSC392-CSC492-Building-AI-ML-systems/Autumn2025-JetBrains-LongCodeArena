Memory alignment
================

Overview
--------

Memory alignment refers to how data is arranged and accessed in memory. Proper alignment ensures that data elements start at addresses that match the requirements of the underlying hardware, which can improve access speed and enable certain SIMD (Single Instruction, Multiple Data) optimizations. NumPy provides facilities to inspect and control alignment through dtypes and array creation.

Key concepts
------------

- dtype.alignment
  - The alignment (in bytes) required for values of a given dtype. For simple, primitive dtypes (e.g., int32, float64), this is the natural alignment of the type on the current platform. For structured dtypes, alignment reflects the alignment of the fields when using an aligned layout.
- Data buffer alignment
  - The data buffer backing an ndarray is allocated with an alignment suitable for the dtype. This ensures that element addresses satisfy the dtype’s alignment requirements.
- Aligned vs. unaligned layouts
  - Simple dtypes typically have natural alignment. Structured dtypes can be laid out with or without strict C-like alignment. The aligned layout (enabled by align=True) mirrors C struct alignment for portability and interoperability.

Inspecting alignment
---------------------

- dtype.alignment gives the required alignment in bytes for a dtype.
- The address of an ndarray’s data buffer can be checked against the dtype’s alignment.

Examples
--------

- Check alignment for common dtypes:

  .. code-block:: python

     import numpy as np

     dt_float64 = np.dtype(np.float64)
     dt_int32 = np.dtype(np.int32)

     print("float64 alignment:", dt_float64.alignment)
     print("int32 alignment:", dt_int32.alignment)

- Alignment for a structured dtype with C-like alignment:

  .. code-block:: python

     dt_struct = np.dtype([('a', np.int32), ('b', np.float64)], align=True)

     print("Structured dtype alignment (aligned):", dt_struct.alignment)
     print("Structured dtype itemsize:", dt_struct.itemsize)

- Verifying an array’s data buffer alignment

  .. code-block:: python

     import numpy as np

     a = np.arange(1024, dtype=np.float64)
     addr = a.__array_interface__['data'][0]
     print("Buffer address:", hex(addr))
     print("Address aligned to dtype.alignment?",
           addr % a.dtype.alignment == 0)

     # For aligned layout, ensure fields are aligned in memory
     b = a.view(np.dtype([('x', np.float64), ('y', np.float64)]))
     print("Structured view itemsize:", b.dtype.itemsize)
     print("Structured view alignment:", b.dtype.alignment)

Usage and behavior
------------------

- Purpose
  - Ensures that elements of a dtype are stored at memory addresses that satisfy their alignment constraints. Proper alignment is crucial for correctness on some architectures and for performance due to vectorization and cache usage.
- When to care
  - Interfacing with C/C++ libraries, ctypes, or FFI layers often requires specific alignment guarantees.
  - Performance-critical code, especially that uses SIMD, can benefit from aligned data buffers.
- How NumPy handles it
  - For primitive dtypes, the data buffer typically adheres to the dtype’s alignment.
  - For structured dtypes, you can opt into a C-like, aligned layout by using align=True. This ensures each field starts at an address aligned to its own alignment requirement, matching the behavior of C structs.

Interoperability with external libraries
----------------------------------------

- When passing NumPy arrays to C libraries (via ctypes, CFFI, or extension modules), rely on arr.ctypes.data as the pointer to the buffer and verify that the alignment matches the library’s expectations.
- For structured dtypes that must match C structs, prefer aligned layouts (align=True) to avoid misalignment issues.

Best practices
--------------

- For interoperability with C code that expects specific field alignments, use aligned dtypes:

  .. code-block:: python

     dt = np.dtype([('a', np.int32), ('b', np.float64)], align=True)

- When using SIMD-accelerated operations, ensure the input data is suitably aligned for the target architecture. If in doubt, test with representative data and consult the library’s alignment requirements.
- If you slice or offset buffers, be mindful that the resulting view may not be aligned even if the base buffer is. Always check the pointer address if strict alignment is required.

Notes
-----

- The actual alignment of an ndarray’s data buffer depends on the platform and allocator. dtype.alignment provides a portable reference for the required alignment of the dtype.
- Not all operations/users require manual alignment checks; for typical NumPy usage, the default allocations and layouts already provide sensible alignment for common dtypes.
