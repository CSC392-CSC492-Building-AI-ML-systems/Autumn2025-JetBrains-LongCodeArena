Random Projection transformers
==============================

Overview
--------
This module provides lightweight transformers to reduce the dimensionality of data
by projecting it into a lower-dimensional space, while trading a controlled amount
of accuracy for faster computations and smaller models. The projection preserves
pairwise distances with high probability, enabling effective distance-based analysis
in lower dimensions.

Key idea
--------
The Johnson–Lindenstrauss lemma guarantees that a small set of points in high-dimensional
space can be embedded into a lower-dimensional space such that pairwise distances are
approximately preserved. A random projection matrix is used for the embedding, typically
drawn from a Gaussian distribution or a sparse distribution, to ensure the distortion is
bounded with high probability.

Main components
---------------
The module exposes two transformer implementations and a utility for estimating a safe
target dimensionality.

- GaussianRandomProjection
  - Dense random projection matrix with entries drawn from a Gaussian distribution.
  - The projection reduces dimensionality by projecting data with a matrix whose entries are
    sampled to ensure approximately preserved pairwise distances.
  - Typical usage: fast dimensionality reduction with a dense embedding.

- SparseRandomProjection
  - Sparse random projection matrix with a configurable density.
  - Density controls the sparsity of the projection matrix (dense_output can be used to obtain a dense
    output if desired).
  - Efficient for very high-dimensional data where a sparse projection yields faster transforms.

- johnson_lindenstrauss_min_dim
  - Utility function that computes a safe (minimum) number of components required to achieve an
    eps-embedding for a given number of samples.

Usage examples
--------------

Basic usage with GaussianRandomProjection
-----------------------------------------
```python
import numpy as np
from sklearn.random_projection import GaussianRandomProjection

# Example data: 1000 samples, 500 features
X = np.random.randn(1000, 500)

# Project to 100 components
rp = GaussianRandomProjection(n_components=100, random_state=42)
X_proj = rp.fit_transform(X)
```

Basic usage with SparseRandomProjection
---------------------------------------
```python
import numpy as np
from sklearn.random_projection import SparseRandomProjection

# Example data: 2000 samples, 1000 features
X = np.random.randn(2000, 1000)

# Project to 200 components with a sparse projection
rp = SparseRandomProjection(n_components=200, density="auto", random_state=42)
X_proj = rp.fit_transform(X)
```

Estimating a safe number of components
--------------------------------------
```python
import numpy as np
from sklearn.random_projection import johnson_lindenstrauss_min_dim

n_samples = 1_000_000
eps = 0.5

# Compute a safe minimal dimensionality for the embedding
n_components = johnson_lindenstrauss_min_dim(n_samples, eps=eps)
print(n_components)  # e.g., 663 for this configuration
```

Parameters and behavior
-----------------------
- GaussianRandomProjection
  - Projects data into a lower-dimensional space using a dense Gaussian random matrix.
  - Distortion of pairwise distances is bounded with high probability according to the Johnson–Lindenstrauss lemma.

- SparseRandomProjection
  - Uses a sparse projection matrix with a controllable density.
  - Density can be set by the user; density="auto" selects a value around 1/sqrt(n_features).
  - More memory- and compute-efficient for very large feature spaces.

- johnson_lindenstrauss_min_dim
  - Computes the minimal number of components n_components to guarantee, with high probability,
    an eps-embedding for n_samples.
  - Formula: n_components >= 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)
  - Valid for eps in (0, 1) and n_samples > 0.

Notes
-----
- All transforms are designed to preserve pairwise distances up to a factor of (1 ± eps) with high probability.
- The choice between Gaussian and Sparse projections depends on data sparsity, memory constraints, and speed requirements.

Notes on compatibility
----------------------
This module follows the scikit-learn estimator API. For best results, ensure the input data
is a NumPy array or a SciPy sparse matrix compatible with the expected formats for fit,
transform, and fit_transform. Data dimensionality checks and warnings propagate through the
transformers as in the base API.
