# Validation curves: plotting scores to evaluate models

Validation curves help you understand how model performance changes as you vary a hyperparameter. By plotting training and validation scores across a range of values, you can diagnose underfitting, overfitting, and select an appropriate hyperparameter that yields good generalization.

## What is a validation curve?

- The validation curve shows how a chosen score (e.g., accuracy, F1, AUC) changes as a specific hyperparameter is varied.
- It typically includes two curves:
  - Train score: performance on the training folds.
  - Validation (test) score: performance on the held-out validation folds.
- Cross-validation is used to compute scores, giving mean and variability (e.g., standard deviation) across folds.
- Interpretation:
  - A high train score with a low validation score indicates overfitting.
  - Both curves converging at a high validation score suggest good generalization.
  - The peak of the validation curve often guides the choice of the best hyperparameter value.

## API overview

- Function: sklearn.model_selection.validation_curve
  - Computes train_scores and test_scores for a given estimator, across a range of hyperparameter values.
  - Returns scores with shape (n_params, n_folds); you typically plot their means with a measure of variability.

- Display: ValidationCurveDisplay
  - A plotting helper (often via ValidationCurveDisplay.from_estimator) to visualize validation curves.
  - Supports different styles for the representation of score variability:
    - std_display_style = "fill_between" (default)
    - std_display_style = "errorbar"
    - std_display_style = None (no variability display)
  - The plot labels include the parameter name and score name, e.g., "Validation Curve for gamma" and the y-axis shows the chosen score.

- Parameters you’ll commonly configure:
  - param_name: hyperparameter to vary (string)
  - param_range: values of the hyperparameter to evaluate (array-like)
  - scoring: scoring metric (string or callable)
  - cv: cross-validation splitting strategy
  - estimator: the estimator to assess
  - std_display_style: one of "fill_between", "errorbar", or None
  - score_name: label for the y-axis (overrides the default derived from scoring)

## Quick start

- Basic usage with a scikit-learn estimator:

```python
import numpy as np
from sklearn.model_selection import validation_curve, ValidationCurveDisplay
from sklearn.svm import SVC
from sklearn.datasets import load_digits
import matplotlib.pyplot as plt

X, y = load_digits(return_X_y=True)

param_name = "gamma"
param_range = np.logspace(-4, 0, 6)

# Compute validation curve (optional: use from_estimator for plotting)
train_scores, test_scores = validation_curve(
    SVC(), X, y, param_name=param_name, param_range=param_range, cv=5, scoring="accuracy"
)

# Plot using the display helper
display = ValidationCurveDisplay.from_estimator(
    SVC(), X, y, param_name=param_name, param_range=param_range,
    scoring="accuracy", cv=5
)
plt.show()
```

- You can customize the display style for showing variability:

```python
display = ValidationCurveDisplay.from_estimator(
    SVC(), X, y,
    param_name=param_name, param_range=param_range,
    scoring="accuracy", cv=5,
    std_display_style="fill_between",  # or "errorbar" or None
)
```

## Example: SVM gamma validation curve

- Objective: assess how the SVM gamma parameter affects classification accuracy on digits data.

```python
import numpy as np
from sklearn.model_selection import ValidationCurveDisplay
from sklearn.svm import SVC
from sklearn.datasets import load_digits
import matplotlib.pyplot as plt

X, y = load_digits(return_X_y=True)
param_name = "gamma"
param_range = np.logspace(-4, 0, 6)

display = ValidationCurveDisplay.from_estimator(
    SVC(), X, y,
    param_name=param_name, param_range=param_range,
    scoring="accuracy", cv=5
)

display.ax_.set_title("Validation curve for SVC gamma")
display.plot()
plt.show()
```

## Customization and internal behavior

- std_display_style controls how score variability is visualized:
  - "fill_between": shows a shaded region around the mean score (mean ± std)
  - "errorbar": plots error bars for each fold’s score
  - None: only the mean curves are drawn
- The y-axis label reflects the score name, which can be customized via score_name.
- The underlying plotting component handles multiple curves (e.g., Train, Test, or both) and automatically adds a legend.
- The x-axis scale is chosen to balance readability and interpretability (linear, logarithmic, or symlog as appropriate).

## Interpretation tips

- Look for a high, stable validation score across a range of hyperparameter values.
- Avoid regions where the validation score drops while the training score remains high (overfitting).
- If the validation score peaks at a particular value, this is often a good choice for the hyperparameter.
- If both curves plateau at a low validation score, the model or feature representation may be unsuitable for the task.

## Notes

- The log_scale parameter is deprecated; prefer using the axis scale methods on the displayed axes (e.g., display.ax_.set_xscale or display.ax_.set_yscale) to control scales.
- Validation curves complement learning curves but answer different questions: validation curves focus on hyperparameter selection, while learning curves diagnose sample-size effects and generalization performance.
