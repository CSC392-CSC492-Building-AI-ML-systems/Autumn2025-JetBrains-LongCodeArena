Utilities for Developers
=========================

Overview
--------

This module provides a collection of compatibility utilities used by the project to support older Python, NumPy, and SciPy versions, as well as helpers for threadpool control and resource loading. The goal is to unify behavior across environments and to ease maintenance as minimum supported versions evolve.

Public API (high level)
-----------------------

- threadpool_limits(limits=None, user_api=None)
  - Wrapper that limits the number of threads used by libraries that respect threadpoolctl, using a shared global controller when available.

- threadpool_info()
  - Retrieves information about the current threadpool configuration.

- delayed(function)
  - Deprecated alias for the function moved to sklearn.utils.parallel.delayed. See deprecation notice below.

- _get_threadpool_controller()
  - Internal helper to obtain a global ThreadpoolController instance (used by threadpool_limits and threadpool_info when available).

- _object_dtype_isnan(X)
  - Internal helper to check for NaNs in object-dtype arrays.

- _percentile(a, q, *, method="linear", **kwargs)
  - Compatibility wrapper for NumPy percentile when the interpolation argument usage differs across NumPy versions.

- percentile
  - If NumPy < 1.22, exposed as a compatibility alias to _percentile; otherwise uses numpy.percentile directly.

- _mode(a, axis=0)
  - Compatibility wrapper around scipy.stats.mode to accommodate differences across SciPy versions (returned shape and behavior).

- _sparse_linalg_cg(A, b, **kwargs)
  - Compatibility wrapper for SciPy's conjugate gradient solver; translates rtol to tol and provides a default for atol on older SciPy versions.

- _open_text(data_module, data_file_name)
- _open_binary(data_module, data_file_name)
- _read_text(descr_module, descr_file_name)
- _path(data_module, data_file_name)
  - Backport helpers for Python 3.9+ importlib.resources usage, with fallbacks for earlier Python versions.

- CSR_CONTAINERS, CSC_CONTAINERS, COO_CONTAINERS, LIL_CONTAINERS, DOK_CONTAINERS, BSR_CONTAINERS
  - Lists of container classes for core SciPy sparse matrix formats; extended with sparse-array variants if available (SciPy >= 1.8).

Notes on compatibility
----------------------

- Version checks are performed using the project’s version parsing utilities to adapt behavior for older environments and to plan for removal once minimum supported versions advance.
- The threadpool controller wrapper is designed to avoid repeatedly scanning loaded libraries when threadpoolctl >= 3.0.0 is used.
- Resource loading helpers provide a uniform API for reading package data across Python 3.8 and 3.9+.

Usage examples
--------------

Thread pool control

```python
from sklearn.utils.fixes import threadpool_limits, threadpool_info

# Limit to a single thread for the enclosed block
with threadpool_limits(limits=1):
    # Code that benefits from a single-thread execution
    ...

# Retrieve current threadpool configuration
info = threadpool_info()
print(info)
```

Percentile compatibility

```python
from sklearn.utils.fixes import percentile

import numpy as np
a = np.array([0.0, 1.0, 2.0, 3.0])
q = [50]
p = percentile(a, q)
print(p)
```

Mode compatibility (SciPy version dependent)

```python
from sklearn.utils.fixes import _mode
import numpy as np

arr = np.array([1, 1, 2, 3])
mode = _mode(arr, axis=0)
print(mode)
```

Conjugate gradient compatibility (SciPy version dependent)

```python
from scipy import sparse
from sklearn.utils.fixes import _sparse_linalg_cg
import numpy as np

A = sparse.csr_matrix([[4.0, 1.0], [1.0, 3.0]])
b = np.array([1.0, 2.0])
x, info = _sparse_linalg_cg(A, b)
print(x, info)
```

Reading packaged data (Python 3.9+ resource API)

```python
# Assuming you have a data module and a file "example.txt" to read
from mypackage import data as data_module
from sklearn.utils.fixes import _open_text

with _open_text(data_module, "example.txt") as f:
    content = f.read()
print(content)
```

Internal helpers and data loading
---------------------------------

- _open_text and _open_binary
  - Open text or binary data files from a package in a way that works across Python versions.
- _read_text
  - Read the contents of a packaged text file in a version-compatible way.
- _path
  - Obtain a path-like handle to a packaged resource, with Python 3.9+ support.

Deprecations
------------

- delayed(function)
  - This function has been moved to sklearn.utils.parallel.delayed. The import path used here will be removed in a future release. See deprecation decorator in the source for details.

Implementation notes for developers
-----------------------------------

- The module relies on parse_version to perform robust version checks for NumPy, SciPy, and related packages.
- Backwards-compatibility shims are isolated behind private helpers (_-prefixed names) where possible, so external users should prefer the public API when available.
- When adding new compatibility logic, consider updating the reported minimum supported versions and include a short note in the source header describing when the fix can be dropped.

Changelog and maintenance
-------------------------

- This documentation reflects the stability and scope of compatibility utilities intended to bridge older environments. Updates may be required as minimum supported versions evolve.

Appendix: Data and containers
-----------------------------

- The CSR/CSC/COO/LIL/DOK/BSR container lists provide ready-made references to common SciPy sparse matrix types. When newer sparse array types are available (SciPy >= 1.8), corresponding container entries (csr_array, csc_array, etc.) are appended to the respective lists.
