MICE (Multiple Imputation by Chained Equations) Imputation
=========================================================

Overview
--------

This module implements the Multiple Imputation through Chained Equations (MICE)
approach for handling missing data in statistical analyses. The process
consists of iteratively imputing missing values for each variable (the
focus variable) using regression models conditioned on the other variables,
then fitting an analysis model to the completed data and combining results
across multiple imputations.

Key ideas
- Initial imputation: replace missing values with simple statistics (e.g., mean)
  to initialize the process.
- Imputation model: for each focus variable, fit a regression model on the
  observed and current imputed values of the other variables. Imputations are
  performed using predictive mean matching (PMM) by default.
- Analysis model: after imputing all variables, fit an analysis model on the
  completed data.
- Repetition and combining: repeat imputations and analysis multiple times and
  combine results to obtain point estimates and standard errors.

Two primary usage modes
- Imputing data only: use MICEData to wrap a DataFrame and generate multiple
  imputed datasets. Includes plotting utilities to assess imputation performance.
- Imputing for analysis: create a MICE instance with an analysis model, run
  the imputation, and then combine results to obtain inference on the analysis
  model parameters.

Terminology
- Analysis model: the model of interest for inference (e.g., regression
  model). If not specified, imputations can still be generated for downstream use.
- Imputation model: a model (typically regression) used to impute each missing
  variable, potentially conditioned on all or a subset of the other variables.
- Perturbation method: how to set the parameter estimates for an imputation
  model. Gaussian perturbation draws from a Gaussian approximation to the
  posterior; bootstrap perturbation uses estimates obtained from a bootstrap
  resample of the data.
  - Gaussian: fit the model, then sample the parameter vector from the Gaussian
    approximation.
  - Bootstrap: fit the model on a bootstrapped sample and use the resulting
    parameter estimates.

Class structure
--------------

There are two main classes in the module:

- MICEData
  - Wraps a Pandas DataFrame and manages the imputation information for each
    variable with missing values.
  - Provides facilities to generate multiply imputed datasets and to visualize
    missing data patterns.
  - history_callback: a hook invoked after each complete imputation cycle to
    collect or save intermediate information.
  - Example usage (see Examples section).

- MICE
  - Orchestrates the full workflow: takes a MICEData object and an analysis model,
    runs the multiple imputation cycles, fits the analysis models, and combines
    results into a MICEResults object.
  - The summary method exposes key estimands and inferential quantities.

- PatsyFormula
  - Lightweight wrapper to treat a string as a Patsy formula, aiding the
    specification of regression relationships for imputation models.

Notes
-----

- Default settings favor memory efficiency: MICEData conserves information across
  iterations, overwriting data between imputations while MICE/saveable results
  are kept when using the MICE pathway for analysis.
- Supported perturbation methods: 'gaussian' and 'boot'.
- The module includes helper plots and a history mechanism to inspect the
  imputation process.

Usage
-----

Imputing data only
~~~~~~~~~~~~~~~~~~

- Wrap a DataFrame with MICEData and iteratively perform imputations.
- Access the imputed datasets and, if desired, save intermediate results.

.. code-block:: python

    import mice
    import pandas as pd

    data = pd.read_csv('my_data.csv')
    imp = mice.MICEData(data)
    # Optional: customize imputation for a variable
    imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
    for j in range(20):
        imp.update_all()
        imp.data.to_csv('data%02d.csv' % j)

Imputing for analysis
~~~~~~~~~~~~~~~~~~~~~

- Create a MICE instance with a MICEData object and an analysis model.
- Run the imputation and combine results to obtain inference.

.. code-block:: python

    import mice
    import numpy as np
    import pandas as pd
    from statsmodels.formula.api import ols

    data = pd.read_csv('my_data.csv')
    mice_data = mice.MICEData(data)
    analysis_formula = 'y ~ x1 + x2 + x3'
    model = ols(analysis_formula, data=None)  # specify via MICE API

    mice_instance = mice.MICE(mice_data, analysis_model=model)
    results = mice_instance.fit()
    summary = results.summary()

Code references
--------------

- PatsyFormula: simple wrapper for a formula string to be used with imputation models.
- MICEData: wraps a DataFrame; manages per-variable imputation models, imputation
  workflow, and optional history callbacks.
- MICE: coordinates the full imputation/analysis workflow and returns a MICEResults
  object with a summary representation.
