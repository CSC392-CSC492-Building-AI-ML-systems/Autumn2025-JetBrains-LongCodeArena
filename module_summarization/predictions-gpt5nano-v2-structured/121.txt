Create Cradmin menu system
==========================

This document describes the Cradmin menu system implemented in the crmenu module. The system provides a flexible, BEM-based rendering model for building complex navigation menus, including expandable sections, link items, and button-styled items.

Overview
--------

The Cradmin menu system centers around renderable objects that describe how menu elements should be rendered in HTML. The rendering is achieved via templates and BEM-style CSS classes. The key concepts are:

- Link renderables: items that render as links in the menu.
- Button renderables: links styled as buttons.
- Expandable menu items: top-level items that can expand to show a list of child items.
- Separators: visual separators within an expandable menu.
- A central abstract menu renderable: a container that aggregates the renderables into a complete menu.

Key components
--------------

The following classes and function are provided to build and render menus:

- get_default_expandable_menu_renderable(**kwargs)
  - Returns the default expandable menu renderable instance, based on the Django setting DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS.
  - If the setting is not defined, returns None.
  - Example:
    - settings.DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.crmenu.MyExpandableMenuRenderable'
    - menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin_instance)

- BaseMenuLinkRenderable
  - Base class for link-like menu items.
  - Template: 'django_cradmin/crmenu/menuitem/link.django.html'
  - Constructor:
    - __init__(self, label, url, is_active=False, bem_variant_list=None, parent_bem_block=None, **kwargs)
  - Key behavior:
    - get_fallback_parent_bem_block(): defaults to 'adminui-page-header'
    - get_parent_bem_block(): uses parent_bem_block if provided, otherwise fallback
    - get_bem_element(): returns '{parent_block}__navlink'
    - get_active_bem_variant_list(): returns ['active'] when active
    - get_bem_variant_list(): includes active variant if is_active is True
  - Usage:
    - NavLinkItemRenderable and related classes derive from this.

- NavLinkItemRenderable
  - Subclass of BaseMenuLinkRenderable.
  - For adding standard links in the main menu.

- NavLinkButtonItemRenderable
  - Subclass of NavLinkItemRenderable.
  - get_bem_variant_list() returns ['button'] to render the item as a button.

- ExpandableMenuItem
  - Subclass of BaseMenuLinkRenderable.
  - Template: 'django_cradmin/crmenu/menuitem/expandable-menu-item.django.html'
  - get_fallback_parent_bem_block(): returns 'expandable-menu'
  - get_bem_element(): returns '{parent}__link'
  - get_list_element_bem_element(): returns '{parent}__listitem'

- ExpandableMenuSeparator
  - Subclass of renderable.AbstractBemRenderable.
  - Template: 'django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html'
  - Constructor: __init__(self, parent_bem_block=None, **kwargs)
  - get_fallback_parent_bem_block(): returns 'expandable-menu'
  - get_parent_bem_block(): uses provided parent_bem_block or falls back
  - get_bem_element(): returns '{parent}__separator'

- MenuToggleItemItemRenderable
  - Subclass of renderable.AbstractBemRenderable.
  - Template: 'django_cradmin/crmenu/menuitem/menutoggle.django.html'
  - Constructor: __init__(self, parent_bem_block=None, **kwargs)
  - get_label(): returns a translatable label, defaults to "Menu" (lazy translation)
  - get_fallback_parent_bem_block(): returns 'adminui-page-header'
  - get_parent_bem_block(): uses provided parent or falls back
  - get_bem_element(): returns '{parent}__navmenutoggle'
  - get_label_css_class(): returns '{parent}__navmenutoggle-label'
  - get_icon_css_class(): returns '{parent}__navmenutoggle-icon'

- AbstractMenuRenderable
  - Base class for rendering a menu as a collection of renderables.
  - Inherits from renderable.AbstractRenderableWithCss
  - Attributes:
    - link_renderable_class: class used to render links (default None; must be defined in a subclass)
    - separator_renderable_class: class used to render separators (default None)
    - button_renderable_class: class used to render button items (default None)
  - Constructor: __init__(self, request, cradmin_instance=None)
    - Initializes with current request and optional Cradmin instance.
    - Maintains a renderable_list that holds the individual renderables.
  - iter_renderables(self)
    - Returns an iterator over the renderable_list.
  - get_link_renderable_class(self)
    - Returns the configured link_renderable_class, or raises ValueError if not defined.
  - get_separator_renderable_class(self)
    - Returns the configured separator_renderable_class (implementation details not shown here).
  - Note: Subclasses typically assemble renderables into renderable_list to produce the final menu.

Usage patterns
--------------

1) Define a default expandable menu class in settings (optional)

- Purpose: Provide a convenient way to instantiate a pre-configured menu renderable.
- Example:
  - In settings.py:
    DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.crmenu.MyExpandableMenuRenderable'
  - In your view or context preparation:
    from django_cradmin.crmenu.crmenu import get_default_expandable_menu_renderable
    menu = get_default_expandable_menu_renderable(request=self.request, cradmin_instance=self)

2) Create individual menu items

- Link item (regular link)
  - Example:
    home_link = NavLinkItemRenderable(label='Home', url='/', is_active=True)

- Button item (link rendered as a button)
  - Example:
    settings_link = NavLinkButtonItemRenderable(label='Settings', url='/settings', is_active=False)

- Expandable menu item (top-level item that can contain a nested list)
  - Example:
    crm_menu_item = ExpandableMenuItem(label='Projects', url='/projects', is_active=False)

- Separator (inside an expandable menu)
  - Example:
    sep = ExpandableMenuSeparator()

- Menu toggle (control for expandable menu in the header)
  - Example:
    toggle = MenuToggleItemItemRenderable()

3) Assembling a menu

- Subclass AbstractMenuRenderable to assemble the menu and supply concrete renderable classes.

Example:

from django_cradmin.crmenu.crmenu import AbstractMenuRenderable, NavLinkItemRenderable, ExpandableMenuItem, ExpandableMenuSeparator

class MyHeaderMenu(AbstractMenuRenderable):
    template_name = 'myapp/menu.html'  # your own template
    link_renderable_class = NavLinkItemRenderable
    separator_renderable_class = ExpandableMenuSeparator
    button_renderable_class = None  # or a button renderable class if you have one

    def __init__(self, request, cradmin_instance=None):
        super(MyHeaderMenu, self).__init__(request, cradmin_instance)
        # Build the menu items
        self.renderable_list = [
            NavLinkItemRenderable(label='Home', url='/', is_active=True),
            ExpandableMenuItem(label='Projects', url='/projects', is_active=False),
            # Add nested items as needed, and additional renderables
        ]

4) Rendering in templates

- Each renderable uses a dedicated template by default (via template_name attributes).
- For example:
  - BaseMenuLinkRenderable uses link.django.html
  - ExpandableMenuItem uses expandable-menu-item.django.html
  - ExpandableMenuSeparator uses expandable-menu-separator.django.html
  - MenuToggleItemItemRenderable uses menutoggle.django.html
- In your template, you typically render the menu by passing the menu object to the template context and looping through renderables in renderable_list.

Template and CSS conventions
----------------------------

- Link items render using the parent BEM block plus __navlink.
- Active items add a ‘active’ modifier to the BEM class list.
- Expandable menus use the expandable-menu BEM block with additional elements:
  - __link, __listitem
- Toggles, separators, and other elements have corresponding BEM element classes derived from their parent block.
- You can customize the parent BEM block by overriding get_parent_bem_block() or by passing a parent_bem_block when instantiating the renderable.

Extending and customization
---------------------------

- To customize rendering, subclass AbstractMenuRenderable and provide concrete renderable classes:
  - Override link_renderable_class, separator_renderable_class, and button_renderable_class as needed.
  - Build the renderable_list in __init__ or via a dedicated method.
- You can override get_fallback_parent_bem_block() or get_parent_bem_block() on individual renderables to control the CSS BEM context.
- The Label for MenuToggleItemItemRenderable is lazily translated via pgettext_lazy to support i18n.

Notes
-----

- The system relies on Django settings for default behavior (e.g., DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS).
- The actual HTML output is determined by the templates referenced in each renderable’s template_name.
- The pgettext_lazy usage in MenuToggleItemItemRenderable supports translation of the default toggle label.

Code references (highlights)
---------------------------

- Default expandable menu renderable selection:

def get_default_expandable_menu_renderable(**kwargs):
    """
    Get the default expandable menu renderable.

    The one set in the :setting:`DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS`.
    """
    if not getattr(settings, 'DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS', None):
        return None
    menu_class = import_string(settings.DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS)
    return menu_class(**kwargs)

- Base menu link renderable (core link item behavior):

class BaseMenuLinkRenderable(renderable.AbstractBemRenderable):
    template_name = 'django_cradmin/crmenu/menuitem/link.django.html'

    def __init__(self, label, url, is_active=False,
                 bem_variant_list=None, parent_bem_block=None, **kwargs):
        self.label = label
        self.url = url
        self.is_active = is_active
        self.parent_bem_block = parent_bem_block
        super(BaseMenuLinkRenderable, self).__init__(bem_variant_list=bem_variant_list, **kwargs)

    def get_fallback_parent_bem_block(self):
        return 'adminui-page-header'

    def get_parent_bem_block(self):
        return self.parent_bem_block or self.get_fallback_parent_bem_block()

    def get_bem_element(self):
        return '{}__navlink'.format(self.get_parent_bem_block())

    def get_active_bem_variant_list(self):
        return ['active']

    def get_bem_variant_list(self):
        bem_variant_list = super(BaseMenuLinkRenderable, self).get_bem_variant_list()
        if self.is_active:
            bem_variant_list.extend(self.get_active_bem_variant_list())
        return bem_variant_list

- Expanded usage: creating a simple menu item and a toggle item

home = NavLinkItemRenderable(label='Home', url='/', is_active=True)
toggle = MenuToggleItemItemRenderable()
