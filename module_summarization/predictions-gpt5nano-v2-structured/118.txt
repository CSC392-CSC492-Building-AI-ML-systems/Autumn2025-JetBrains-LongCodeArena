Configuration.rst
===================


Overview
--------

This module provides the ConfigurationsOptions class, a container for toggling and tuning the import/export behavior of configuration data in PyAEDT. It allows enabling or disabling the export/import of various configuration elements to and from JSON, as well as controlling object mapping precision during imports.

Key functionality
-----------------

- Centralized control over what aspects of a configuration are exported to JSON:
  - Variables, setups, optimizations, parametrics, boundaries, mesh operations, coordinate systems, materials, object properties, and datasets.
- Centralized control over what aspects are imported from JSON:
  - Datasets, variables, setups, optimizations, parametrics, boundaries, mesh operations, coordinate systems, materials, and object properties.
- Fine-grained object matching tolerance used during mapping/import processes.
- Option to skip import if an object already exists.

Usage examples
--------------

Basic usage (default behavior)

- Create an options object with default settings.
  - from pyaedt import Hfss
  - options = Hfss().configurations.options
  - (Defaults reflect the values above)

Disable exporting of setups and datasets

- from pyaedt import Hfss
- hfss = Hfss()
- opts = hfss.configurations.options
- opts.export_setups = False
- opts.export_datasets = False

Adjust object mapping tolerance

- from pyaedt import Hfss
- hfss = Hfss()
- hfss.configurations.options.object_mapping_tolerance = 1e-12

Notes
-----

- These flags provide a straightforward way to tailor the JSON export/import to your project requirements.
- The object_mapping_tolerance is used during object matching/mapping to ensure robust association between imported and existing objects (e.g., when importing boundaries).

Internal helper: _find_datasets
-------------------------------

The module includes a helper function for locating dataset references within a nested dictionary structure.

Signature
- _find_datasets(d, out_list)

Purpose
- Recursively traverses a dictionary (or OrderedDict) to collect dataset identifiers referenced in free_form_value fields that contain "pwl".
- Appends found dataset references to out_list as strings in the form "$<dataset_name>," extracted from the free_form_value field.

Typical usage
- datasets = []
- _find_datasets(configuration_dict, datasets)
- Now datasets contains discovered dataset references for further processing (e.g., exporting/importing).

Example
- Given a nested dictionary d with potential lists/entries containing free_form_value, the function will extract and accumulate pwl-based dataset identifiers into out_list.

Code reference (excerpt)
- _find_datasets(d, out_list):
  - Recursively walks values
  - If a value is a list, iterates elements and attempts to extract el["free_form_value"]
  - If a value is a string containing "pwl", extracts the substring from "$" to the first ","

Summary
-------

Configuring export/import behavior is centralized in ConfigurationsOptions, enabling precise control over what configuration aspects are serialized to or restored from JSON. The provided _find_datasets helper assists in extracting dataset references from complex configuration structures for downstream processing.
