Multiclass and Multioutput algorithms
=====================================

This module provides meta-estimators that extend single-output estimators to handle multiple outputs. They enable multiclass/multilabel classification and multioutput regression by wrapping a base estimator and applying it to each target or by chaining estimators to exploit target dependencies.

Estimators
---------

- MultiOutputClassifier
  - Description: Wraps a classifier to fit one classifier per target output. Suitable for multioutput classification where y has shape (n_samples, n_outputs).
  - Key idea: Trains a separate classifier for each column of y.
  - Parallelism: Supports parallel fitting across outputs via the n_jobs parameter.

- MultiOutputRegressor
  - Description: Wraps a regressor to fit one regressor per target output. Suitable for multioutput regression where y has shape (n_samples, n_outputs).
  - Key idea: Trains a separate regressor for each column of y.
  - Parallelism: Supports parallel fitting across outputs via the n_jobs parameter.

- ClassifierChain
  - Description: Builds a chain of classifiers, where each classifier in the chain sees the original input features augmented with the predictions of earlier classifiers in the chain.
  - Purpose: Captures dependencies among target outputs for multiclass/multilabel classification.
  - Order: The order of outputs can be specified or randomized.

- RegressorChain
  - Description: Builds a chain of regressors, where each regressor sees the input features augmented with the predictions of earlier regressors in the chain.
  - Purpose: Captures dependencies among target outputs for multioutput regression.
  - Order: The order of outputs can be specified or randomized.

How it works
------------

- Target shape: All meta-estimators expect y with shape (n_samples, n_outputs).
- Base estimator: Each output dimension is modeled by a separate estimator (for MultiOutputClassifier/Regressor) or by a chained sequence of estimators (ClassifierChain/RegressorChain).
- Parallelism: When possible, training across outputs is parallelized using n_jobs.
- Incremental learning: If the base estimator supports partial_fit, partial fitting is possible via the appropriate parameters.

Usage examples
--------------

A) MultiOutputClassifier with a Random Forest

.. code-block:: python
   :language: python

   from sklearn.datasets import make_multilabel_classification
   from sklearn.ensemble import RandomForestClassifier
   from sklearn.multioutput import MultiOutputClassifier

   X, Y = make_multilabel_classification(
       n_samples=500, n_features=20, n_classes=3, random_state=0
   )
   base_clf = RandomForestClassifier(n_estimators=100, random_state=0, n_jobs=-1)
   mor = MultiOutputClassifier(base_clf, n_jobs=-1)
   mor.fit(X, Y)
   Y_pred = mor.predict(X)

B) MultiOutputRegressor with a Ridge regressor

.. code-block:: python
   :language: python

   from sklearn.datasets import make_regression
   from sklearn.linear_model import Ridge
   from sklearn.multioutput import MultiOutputRegressor

   X, Y = make_regression(n_samples=500, n_features=20, n_targets=3, random_state=0)
   reg = MultiOutputRegressor(Ridge(alpha=1.0))
   reg.fit(X, Y)
   Y_pred = reg.predict(X)

C) ClassifierChain with Logistic Regression

.. code-block:: python
   :language: python

   from sklearn.datasets import make_multilabel_classification
   from sklearn.linear_model import LogisticRegression
   from sklearn.multioutput import ClassifierChain

   X, Y = make_multilabel_classification(
       n_samples=500, n_features=20, n_classes=3, random_state=0
   )
   chain = ClassifierChain(LogisticRegression(max_iter=1000), order="random", random_state=0)
   chain.fit(X, Y)
   Y_pred = chain.predict(X)

D) RegressorChain with Ridge regression

.. code-block:: python
   :language: python

   from sklearn.datasets import make_regression
   from sklearn.linear_model import Ridge
   from sklearn.multioutput import RegressorChain

   X, Y = make_regression(n_samples=500, n_features=20, n_targets=3, random_state=0)
   chain = RegressorChain(Ridge(alpha=1.0), order=[0, 1, 2])
   chain.fit(X, Y)
   Y_pred = chain.predict(X)

Notes
-----

- Classification targets
  - MultiOutputClassifier and ClassifierChain require classification targets. y should be shaped (n_samples, n_outputs) with discrete labels for each output.
  - For multilabel classification, consider using make_multilabel_classification or similar utilities to generate appropriate y.

- Regression targets
  - MultiOutputRegressor and RegressorChain require continuous targets, with y shaped (n_samples, n_outputs).

- Evaluation
  - You can use cross_val_predict or other multioutput-aware metrics (e.g., per-output accuracy, Hamming loss, or RMSE) to assess performance.

— See also
---------

- Cross-validation with multioutput targets: cross_val_predict, multioutput scoring schemes.
- Metrics for multioutput tasks: accuracy per output, Hamming loss, RMSE, etc.

Implementation highlights (high level)
------------------------------------

- These estimators are implemented as meta-estimators that wrap a base estimator and extend it to handle multiple outputs.
- Internally, they rely on helper mechanisms to fit and predict across outputs, and to aggregate results when appropriate.

These tools enable flexible handling of multiclass and multioutput problems by leveraging familiar single-output estimators in a scalable, organized manner.
