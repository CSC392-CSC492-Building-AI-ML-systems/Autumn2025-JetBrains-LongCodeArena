Nextcord Introduction
===================== 

Nextcord is a modern, easy-to-use library for building Discord bots in Python. It is a maintained fork of the popular discord.py library, designed to be intuitive for beginners while providing the capabilities developers need for complex bots. Nextcord exposes Discord's API through Pythonic objects, events, and helpers, enabling rapid development with clean, well-documented code.

What you can build with Nextcord
- Command-based bots and slash commands
- Event-driven bots that respond to Discord events
- Rich interactions with Discord features (embeds, reactions, components)
- Guild widgets and lightweight data representations for guilds, channels, and members

Getting started
- Installation

  - Install the library with pip:
    .. code-block:: python

      pip install nextcord

- Your first bot (basic example)

  This minimal example creates a bot that prints a message when it connects and responds to a simple command.

  .. code-block:: python

    import nextcord
    from nextcord.ext import commands

    intents = nextcord.Intents.default()
    intents.message_content = True  # Enable if your bot requires message content

    bot = commands.Bot(command_prefix="!", intents=intents)

    @bot.event
    async def on_ready():
        print(f"Logged in as {bot.user} (ID: {bot.user.id})")

    @bot.command(name="ping")
    async def ping(ctx):
        await ctx.send("Pong!")

    bot.run("YOUR_BOT_TOKEN")

Key concepts and objects
- WidgetChannel
  - Represents a lightweight channel reference within a guild widget.
  - Attributes: id, name, position
  - Useful for displaying or referencing accessible channels in a widget context.

- WidgetMember
  - Represents a partial member of a guild for widget data.
  - Attributes: id, name, discriminator, bot, status, nick, avatar, activity, deafened, muted, suppress, connected_channel
  - Provides a display_name property for convenient user-facing name resolution.

- Widget
  - Represents a Guild widget and its data.
  - Attributes: id, name, channels, members
  - Allows access to the widget’s channels and online members as reported by Discord.

Using widgets (overview)
Guild widgets provide a lightweight snapshot of a guild’s activity, including accessible channels and online members. You can access widget data from a guild object if your bot has permission to view the widget.

Example: accessing widget data from a guild (when available)

.. code-block:: python

  guild = bot.get_guild(123456789012345678)  # Replace with your guild ID
  widget = getattr(guild, "widget", None)
  if widget is not None:
      print(f"Guild widget for {widget.name}:")
      for channel in widget.channels:
          print(f"- Channel: {channel.name} (ID: {channel.id})")
      for member in widget.members:
          print(f"- Member: {member.display_name} (Status: {member.status})")

Notes
- Widget data availability depends on Discord’s server widget settings and permissions.
- The WidgetMember.display_name property provides a user-friendly display name, falling back to the username if no nickname is set.
