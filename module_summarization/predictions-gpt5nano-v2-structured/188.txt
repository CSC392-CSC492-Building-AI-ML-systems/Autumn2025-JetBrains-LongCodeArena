Kernel approximation
====================

Overview
--------
The kernel_approximation module implements several approximate kernel feature maps
based on Fourier transforms and Count Sketches. These transformers map input
data into a finite-dimensional feature space such that linear models on the
transformed data approximate nonlinear kernel methods.

Key classes include:
- PolynomialCountSketch: Tensor Sketch approximation for polynomial kernels
  of the form K(X, Y) = (gamma * <X, Y> + coef0)^degree
- AdditiveChi2Sampler, SkewedChi2Sampler: Approximate feature maps for specific
  chi-squared related kernels
- RBFSampler: Random Fourier features for the Gaussian (RBF) kernel
- Nystroem: Kernel map approximation using a subset of the training data

This page focuses on PolynomialCountSketch, the Tensor Sketch-based map for polynomial kernels.

PolynomialCountSketch (Tensor Sketch)
------------------------------------

Purpose
-------
PolynomialCountSketch provides an efficient, low-dimensional approximation of the
polynomial kernel feature map using Tensor Sketch. It approximates the feature map
for K(X, Y) = (gamma * <X, Y> + coef0)^degree by computing a Count Sketch of
the outer product of a vector with itself and applying FFTs.

Formula
-------
K(X, Y) ≈ (gamma * ⟨X, Y⟩ + coef0)^degree

Usage
-----
- Create a PolynomialCountSketch transformer.
- Call fit on training data to initialize internal counts and random hashes.
- Call transform or fit_transform on new data to obtain the approximate feature map.

Key parameters
-------------
- gamma: float, default 1.0
  Parameter of the polynomial kernel.

- degree: int, default 2
  Degree of the polynomial kernel to approximate.

- coef0: float, default 0
  Constant term of the polynomial kernel.

- n_components: int, default 100
  Dimensionality of the output feature space. Typically n_components should be
  larger than the number of input features to capture the variability in the data.

- random_state: int or RandomState, default=None
  Controls the random initialization of the Count Sketch hash functions.

Attributes
----------
- indexHash_ : ndarray of shape (degree, n_features) -> int64
  Hash indices used to map features into the Count Sketch representation.

- bitHash_ : ndarray of shape (degree, n_features) -> float32
  Random signs (+1 or -1) used in the Count Sketch.

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit (only if input features are strings).

Notes
-----
- The transformer accepts sparse input data (CSC format is preferred for internal operations).
- Internally, the implementation uses FFT-based Tensor Sketch to combine
  feature interactions efficiently.

API details
-------------
Constructor
- PolynomialCountSketch(gamma=1.0, degree=2, coef0=0, n_components=100, random_state=None)

Public methods
- fit(X, y=None)
  Initialize the sketch based on the input data X. Determines the number of
  input features and sets up the hash mappings.

- transform(X)
  Compute the approximate polynomial feature map for X. Returns an array of
  shape (n_samples, n_components).

- fit_transform(X, y=None)
  Convenience method that fits and transforms X.

Example
-------
from sklearn.kernel_approximation import PolynomialCountSketch
from sklearn.linear_model import SGDClassifier

X = [[0, 0], [1, 1], [1, 0], [0, 1]]
y = [0, 0, 1, 1]

ps = PolynomialCountSketch(degree=3, gamma=1.0, coef0=0, n_components=100, random_state=1)
X_features = ps.fit_transform(X)

clf = SGDClassifier(max_iter=10, tol=1e-3)
clf.fit(X_features, y)
score = clf.score(X_features, y)
print(score)  # Example: 1.0

Implementation notes
--------------------
- The implementation relies on Count Sketch representations and FFT-based
  convolutions to efficiently approximate higher-degree polynomial feature maps.
- The internal random state controls the hashing and sign assignments to ensure
  reproducibility across calls.
