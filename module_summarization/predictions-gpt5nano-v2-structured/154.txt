Connecting Steps Together
========================

This document describes how to connect individual steps to form a pipeline. A pipeline is built from discrete Step instances, each exposing inputs and outputs (handles). Connections are expressed with Edge objects that link a source output to a destination input.

Overview
--------

- A Step represents a single unit of work with named inputs and outputs.
- Handles are references to a Step’s inputs or outputs, created via helper methods.
- Edge represents a connection from a source handle to a destination handle.
- Connecting steps enables data to flow through a workflow.

Key concepts
------------

- Step
  - A unit of work defined by a configure.yml file.
  - Access inputs and outputs via handles.
- Handles
  - Input handle: (step_name, 'inputs', input_name)
  - Output handle: (step_name, 'outputs', output_name)
  - Helper methods:
    - get_input_handle(name) -> input handle
    - get_output_handle(name) -> output handle
    - i(name) -> shorthand for input handle
    - o(name) -> shorthand for output handle
  - Convenience:
    - all_input_handles()
    - all_output_handles()
- Edge
  - A simple connection between two handles: Edge(src_handle, dst_handle)
  - Accessors:
    - get_src()
    - get_dst()

How to connect steps
--------------------

- Create Step instances for each unit of work.
- Obtain the appropriate handles for the outputs of the producer and the inputs of the consumer.
- Create an Edge that links the producer’s output to the consumer’s input.

Code example
------------

Minimal example showing two steps connected by one edge.

.. code-block:: python

  from edge import Edge
  from step import Step

  # Steps defined in your project (path to each step's directory)
  step_a = Step('step_a')  # has an output named 'data'
  step_b = Step('step_b')  # has an input named 'data'

  # Connect Step A's 'data' output to Step B's 'data' input
  connection = Edge(
    step_a.o('data'),  # source handle: (step_a, 'outputs', 'data')
    step_b.i('data')   # destination handle: (step_b, 'inputs', 'data')
  )

- You can create multiple connections to form a larger pipeline:
  .. code-block:: python

    from edge import Edge
    from step import Step

    a = Step('step_a')
    b = Step('step_b')
    c = Step('step_c')

    # A -> B
    e1 = Edge(a.o('out'), b.i('in'))
    # B -> C
    e2 = Edge(b.o('result'), c.i('input'))

- While the code above shows Edge-based connections, the same handles are used when wiring together graphs or pipelines in your build system.

Inspecting and working with handles
----------------------------------

- List a step’s inputs or outputs:
  - step.all_input_handles()
  - step.all_output_handles()
- Get individual handles:
  - step.i('name') or step.o('name')
  - step.get_input_handle('name')
  - step.get_output_handle('name')

Notes and tips
--------------

- YAML configuration validation is performed when a Step is loaded:
  - Each step must have a name.
  - Empty inputs/outputs are removed.
  - Outputs with tags are validated for a single key/value.
  - If commands are empty, they default to ['true'].
- Output tag shorthands in configure.yml are resolved to real files, e.g.:
  - ~: results/1/2/3/data.txt becomes { 'data.txt': 'results/1/2/3/data.txt' }

Advanced usage
--------------

- Extending steps and commands:
  - You can extend inputs/outputs and commands to adapt steps for more complex pipelines.
  - Example APIs (not shown here) include extend_inputs, extend_outputs, pre_extend_commands, and extend_commands.
- Cloning steps:
  - A Step can be cloned with a deep copy of its configuration (Step.clone()) to reuse a configured step layout in multiple places.
