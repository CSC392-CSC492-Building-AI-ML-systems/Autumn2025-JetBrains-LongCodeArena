Diophantine Equations Solver (diophantine.rst)
===============================================

Overview
--------
This module implements a solver for Diophantine equations, i.e., polynomial
equations with integer solutions. The primary entry point is the diophantine
function, which reformulates the equation as a product of terms that must equal
zero, solves each term independently, and then merges the solutions to produce a
set of all possible integer solutions for the variables involved.

Key ideas:
- Convert equations to a product-of-terms form and solve each term with diop_solve.
- Combine solutions from independent terms in a principled way.
- Support ordering of variables via an optional syms parameter and return
  solutions as tuples ordered by alphabetic variable ordering.
- Works for a variety of diophantine types (e.g., linear, quadratic, Pythagorean-type, etc.) and uses internal classification to dispatch solving.

Public API
----------

diophantine(eq, param=Symbol('t', integer=True), syms=None)

Parameters
- eq: Relational or Expr
  The equation to solve. If it is an equality (Eq), it is converted to lhs - rhs.
- param: Symbol (default: t, with integer=True)
  The parameter symbol used to express parametric solutions.
- syms: sequence of Symbol (optional)
  If provided, it controls the ordering of the variables in the output tuples.
  Only variables present in the equation are considered, and the order is
  restricted to the provided sequence.

Returns
- A set of tuples. Each tuple contains the solutions for the variables in the
  alphabetic order of the variables present in the equation. Each element in a
  tuple is a symbolic expression representing a parametric solution (e.g., t0,
  n1, etc.).

Notes
- If the equation is already 0, the trivial solution {(param,)} is returned.
- The function attempts to factor and classify the polynomial before solving.
  If a factor cannot be handled, a TypeError may be raised with an informative
  message indicating that the equation should be a polynomial with rational
  coefficients.
- If no nontrivial solutions exist, but the trivial solution (all zeros) satisfies
  the equation, that trivial solution is returned.
- The solver relies on internal components such as classify_diop and diop_solve
  (from the solver stack) and may dispatch to different solving strategies based on
  the identified diophantine type.

Diophantine types (diop_known)
--------------------------------
The solver recognizes several diophantine types, guiding how each term is solved and merged:
- binary_quadratic
- cubic_thue
- general_pythagorean
- general_sum_of_even_powers
- general_sum_of_squares
- homogeneous_general_quadratic
- homogeneous_ternary_quadratic
- homogeneous_ternary_quadratic_normal
- inhomogeneous_general_quadratic
- inhomogeneous_ternary_quadratic
- linear
- univariate

Examples
--------

Basic usage with algebraic variables x, y, z:

- Solve a simple difference of squares
  >>> diophantine(x**2 - y**2)
  {(t_0, -t_0), (t_0, t_0)}

- Solve a linear combination with a third variable
  >>> diophantine(x*(2*x + 3*y - z))
  {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}

- Solve a linear-quadratic mix (parametric solutions)
  >>> diophantine(x**2 + 3*x*y + 4*x)
  {(0, n1), (3*t_0 - 4, -t_0)}

Notes on usage
- The output tuples are ordered according to the alphabetic ordering of
  the variables involved unless a custom syms sequence is provided.
- The parameter t (default) can be used to express families of solutions in a
  compact, parametric form.

Internal helpers (overview)
---------------------------

Some auxiliary functions in the module aid readability and robustness:
- _is_int(i): Check if i can be interpreted as an integer.
- _sorted_tuple(*i): Return a sorted tuple of inputs.
- _remove_gcd(*x): Normalize by removing the greatest common divisor.
- _rational_pq(a, b): Return reduced rational a/b as (numer, denom).
- _nint_or_floor(p, q): Nearest integer to p/q, breaking ties by floor.
- _odd(i), _even(i): Parity checks.

Implementation details
----------------------

The diophantine function follows this high-level algorithm:
1. If eq is an Eq, convert to eq.lhs - eq.rhs.
2. If eq == 0, return the trivial parametric solution.
3. Collect free symbols, optionally reorder with syms, and handle rational factors.
4. Factor and obtain a polynomial expression in the variables.
5. Classify the polynomial into a diophantine type via classify_diop.
6. For each term, solve using diop_solve and merge solutions according to type.
7. Remove spurious zero-merge results, add trivial solution if appropriate.
8. Return a set of solutions as SymPy expressions.

  t, a, b = symbols('t a b', integer=True)
  diophantine(x*(2*x + 3*y - z), param=t)
