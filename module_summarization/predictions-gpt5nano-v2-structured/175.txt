Biclustering
===============
Biclustering, or co-clustering, aims to find submatrices (co-clusters) within a data matrix that exhibit coherent patterns across subsets of rows and columns simultaneously. Unlike traditional clustering, which groups either rows or columns alone, biclustering discovers blocks where a subset of rows behaves similarly across a subset of columns. This is particularly useful for applications such as gene expression analysis, feature selection over time-series, and market data where localized patterns exist.

Key ideas
---------
- Simultaneous clustering of rows and columns to reveal checkerboard-like structures.
- Each co-cluster corresponds to a subset of rows and a subset of columns with coherent values.
- Helps discover local patterns that may be obscured when clustering dimensions separately.

Estimators
----------
The biclustering module provides the following estimators:

- SpectralBiclustering
- SpectralCoclustering

Notes:
- Both estimators extend the scikit-learn estimator interface and expose row and column cluster assignments after fitting.
- Typical input is a 2D numeric array X with shape (n_samples, n_features).

API overview
------------
Common concepts across estimators:
- fit(X): Learn the row and column clusters from the data X.
- row_labels_: Array of shape (n_rows,), containing the cluster label for each row.
- column_labels_: Array of shape (n_cols,), containing the cluster label for each column.
- n_row_clusters_: Number of row clusters discovered.
- n_col_clusters_: Number of column clusters discovered.
- n_clusters_: For some estimators, this may be a single int or a tuple (n_row_clusters, n_col_clusters) specifying the number of clusters along each dimension.

Example usage
-------------
Below are representative examples illustrating how to apply the biclustering estimators and inspect the results.

.. code-block:: python

   import numpy as np
   from sklearn.cluster import SpectralBiclustering

   # Example data: a numeric 2D array
   rng = np.random.default_rng(seed=0)
   X = rng.normal(size=(40, 30))

   # Fit model: 3 row clusters and 4 column clusters
   model = SpectralBiclustering(n_clusters=(3, 4), random_state=0)
   model.fit(X)

   # Access results
   row_labels = model.row_labels_        # shape (n_rows,)
   col_labels = model.column_labels_     # shape (n_cols,)
   n_row = model.n_row_clusters_
   n_col = model.n_col_clusters_

   # Inspect a particular co-cluster block
   r, c = 1, 2
   rows_in_r = np.where(row_labels == r)[0]
   cols_in_c = np.where(col_labels == c)[0]
   block = X[np.ix_(rows_in_r, cols_in_c)]
   print("Block shape:", block.shape)

.. code-block:: python

   from sklearn.cluster import SpectralCoclustering

   # Another estimator example
   cocluster_model = SpectralCoclustering(n_clusters=(3, 3), random_state=0)
   cocluster_model.fit(X)

   row_labels_coco = cocluster_model.row_labels_
   column_labels_coco = cocluster_model.column_labels_
   print("Row clusters (coclustering):", cocluster_model.n_row_clusters_)
   print("Column clusters (coclustering):", cocluster_model.n_col_clusters_)

Getting started and when to use
--------------------------------
- Use SpectralBiclustering when you expect a checkerboard-like structure with coherent row and column groups.
- Use SpectralCoclustering when you want a partition of the matrix into disjoint blocks with both row and column clusters covering the data.

Data pre-processing tips
------------------------
- Ensure X is a numeric 2D array (dense inputs are typical for these estimators).
- Centering or scaling can be beneficial depending on the domain and data characteristics.
- For noisy data, you may experiment with different numbers of row/column clusters to capture meaningful blocks.
