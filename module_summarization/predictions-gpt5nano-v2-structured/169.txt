Contributing to Qiskit Metal
============================

This guide explains how to contribute to Qiskit Metal, including code changes, documentation updates, tests, and the review process. It covers the typical workflow from opening an issue to submitting a pull request, plus recommended tooling and coding standards.

Getting started
--------------
Qiskit Metal is an open-source project. Contributions from the community are essential to improve features, fix bugs, and enhance documentation. All contributions should follow the project’s guidelines and pass the project's tests and quality checks.

Setting up a development environment
------------------------------------
1. Clone the repository:
   .. code-block:: bash

      git clone https://github.com/Qiskit/qiskit-metal.git
      cd qiskit-metal

2. Create and activate a Python virtual environment (recommended):
   .. code-block:: bash

      python3 -m venv env
      source env/bin/activate

3. Install development dependencies:
   .. code-block:: bash

      pip install -e .[dev]

4. (Optional) Run pre-commit hooks:
   .. code-block:: bash

      pip install pre-commit
      pre-commit install
      pre-commit run --all-files

How to contribute
------------------
1. Find an issue to work on or propose a new feature:
   - Browse the issue tracker for open bugs or enhancement requests.
   - Open an issue describing the problem, motivation, and proposed solution.

2. Create a feature branch:
   .. code-block:: bash

      git checkout -b feature/your-feature-name

3. Make your changes:
   - Implement code changes, add tests, and update or create documentation as needed.
   - Follow the project’s coding standards and docstring guidelines.

4. Run tests locally:
   .. code-block:: bash

      pytest -q

5. Update documentation if applicable:
   - Update user-facing docs, API docs, and inline code comments as needed.
   - Ensure docstrings follow the project’s style (NumPy/Sphinx).

6. Commit your changes with a clear message:
   .. code-block:: bash

      git add -A
      git commit -m "Add feature: ...; fixes #ISSUE_NUMBER"

7. Push and create a pull request:
   .. code-block:: bash

      git push origin feature/your-feature-name

8. Submit a PR on GitHub:
   - Include a concise title and a detailed description.
   - Reference related issues and explain the motivation, design, and limitations.
   - Add or update tests and documentation as needed.
   - Request reviews from appropriate maintainers.

Testing and validation
----------------------
- Run unit tests to validate functionality:
  .. code-block:: bash

     pytest -q
- Run the full test suite if you have the time:
  .. code-block:: bash

     pytest -q tests
- Run style and quality checks:
  .. code-block:: bash

     black --check .
     isort --check-only .
     flake8
- Local documentation validation:
  .. code-block:: bash

     sphinx-build -b html docs build/docs
  (Ensure no broken links and all pages render correctly.)

Documentation contribution
--------------------------
- Update user guides, API reference, and developer docs to reflect your changes.
- Follow the repository’s reStructuredText (reST) conventions and Sphinx configuration.
- Include examples that demonstrate new behavior or usage.
- Ensure docstrings for new functions/classes follow the NumPy/Sphinx format used in Qiskit Metal.

Coding standards and best practices
-----------------------------------
- Python style: follow PEP 8.
- Naming: use descriptive, consistent names; avoid abbreviations.
- Docstrings: provide clear, complete, and consistent docstrings using NumPy style.
- Type hints: use Python type hints where applicable.
- Tests: accompany code changes with unit tests that cover new functionality and edge cases.
- Logging: use the standard logging module where appropriate.

Submitting your contribution
----------------------------
- Ensure all tests pass and that your changes do not regress existing functionality.
- Include tests and documentation updates with your PR.
- Provide a changelog note if your project uses a release process.
- Respond to reviews promptly, addressing any requested changes.

License and attribution
-----------------------
- Qiskit Metal is distributed under the Apache License 2.0.
- If you modify files, retain the copyright notice and indicate alterations.

Getting help
------------
- If you’re unsure about a contribution, open an issue to discuss the approach with maintainers and the community.
- For quick questions, consider reaching out on the project’s discussion forums or chat channels linked in the repository.

Code examples
-------------

  .. code-block:: bash

     # Make changes, commit, and push…
     git push origin feature/my-improvement
     # Then open a PR on GitHub with a descriptive description
