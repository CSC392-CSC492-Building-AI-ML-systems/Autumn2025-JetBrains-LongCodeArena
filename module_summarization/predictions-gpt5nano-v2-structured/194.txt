# Pairwise metrics, Affinities and Kernels

This module provides utilities for computing pairwise quantities between samples, including distances (pairwise metrics), affinities (similarities), and kernel values. It supports dense and sparse inputs, robust input validation, and options for precomputed matrices.

## Public API (overview)

- pairwise_distances(X, Y=None, metric="euclidean", **kwargs)
  - Compute pairwise distances between rows of X and rows of Y (or X with itself if Y=None).
  - Supported metrics include Euclidean, Manhattan, Cosine, Mahalanobis, and more.
  - Accepts dense or sparse inputs; can operate with precomputed distance matrices when metric="precomputed".

- pairwise_kernels(X, Y=None, metric="linear", **kwargs)
  - Compute pairwise kernel values (similarities) between samples.
  - Supported kernels include linear, polynomial, radial basis function (RBF), sigmoid, and others.

- Internal helpers (for developers and advanced users)
  - _return_float_dtype(X, Y): Determine appropriate floating dtype for computation.
  - check_pairwise_arrays(X, Y, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)
    - Validate and standardize inputs for pairwise computations.
  - check_paired_arrays(X, Y)
    - Validate inputs for paired-distance computations.
  - These helpers ensure inputs are two-dimensional, compatible in feature dimension, and finite when required.

## Pairwise metrics

- Purpose
  - Quantify the distance between samples (rows) in one or two matrices.
  - Useful for clustering, nearest-neighbor methods, and as a preprocessing step for metric-based algorithms.

- Key features
  - Works with dense arrays (NumPy) and sparse matrices (CSR/CSC, subject to support).
  - Supports a variety of metrics via a uniform API.
  - Option to use precomputed distance matrices to avoid recomputation.

- Usage notes
  - X and Y are interpreted as matrices of shape (n_samples_X, n_features) and (n_samples_Y, n_features), respectively.
  - If Y is None, distances are computed within X (i.e., O(n_samples_X^2) pairwise distances).
  - For precomputed distances, metric should be set to "precomputed" and the input X (and Y, if provided) should be interpreted as a distance matrix with appropriate shape.

- Example
  - Basic Euclidean distances between rows of X:
    ```python
    from sklearn.metrics import pairwise_distances
    import numpy as np

    X = np.array([[0, 0], [1, 1], [2, 2]])
    D = pairwise_distances(X, metric="euclidean")
    ```
  - Manhattan distances between X and Y:
    ```python
    Y = np.array([[0, 1], [1, 0]])
    D = pairwise_distances(X, Y, metric="manhattan")
    ```

- Precomputed distances
  - If you already have a distance matrix, you can use the precomputed pathway by setting metric="precomputed" and supplying the appropriate matrix shapes. The internal validation ensures the shapes align (n_queries x n_indexed) when using precomputed data.

## Affinities

- Purpose
  - Affinities describe similarity between samples rather than distance.
  - Often derived from distances (e.g., via 1 - distance, or via kernel-based similarity measures).

- Typical approaches
  - Use pairwise kernels to obtain kernel (similarity) matrices, which serve as affinities in many algorithms (e.g., spectral clustering, kernelized methods).
  - Common affinity notions are implemented via kernels such as linear, polynomial, RBF, and sigmoid, or via cosine-based similarities in certain configurations.

- Usage notes
  - Pairwise similarities are symmetric (K(X, Y) = K(Y, X)) for standard kernels.
  - Values typically lie within a bounded range depending on the kernel (e.g., [-1, 1] for some kernels, [0, 1] for certain normalized similarities).

- Example (kernel-based affinity)
  ```python
  from sklearn.metrics.pairwise import pairwise_kernels
  import numpy as np

  X = np.random.rand(5, 3)
  K = pairwise_kernels(X, metric="rbf", gamma=0.5)
  ```
  The kernel matrix K serves as an affinity matrix between samples.

## Kernels

- Purpose
  - Kernels map samples into a (potentially infinite-dimensional) feature space to measure similarity in a principled way.
  - Pairwise kernels are widely used in kernelized learning algorithms (SVMs, kernel PCA, spectral methods).

- Supported kernels (examples)
  - Linear: K(X, Y) = X @ Y^T
  - Polynomial: K(X, Y) = (gamma X Y^T + coef0)^degree
  - RBF (Gaussian): K(X, Y) = exp(-gamma ||X - Y||^2)
  - Sigmoid: K(X, Y) = tanh(gamma X Y^T + coef0)
  - Others as implemented in the underlying pairwise module

- Usage notes
  - X and Y shapes: (n_samples_X, n_features) and (n_samples_Y, n_features)
  - Optional hyperparameters (gamma, degree, coef0) control the kernel shape.

- Example
  ```python
  from sklearn.metrics.pairwise import pairwise_kernels
  import numpy as np

  X = np.array([[0, 0], [1, 1], [2, 2]], dtype=float)
  K_rbf = pairwise_kernels(X, metric="rbf", gamma=0.5)
  K_poly = pairwise_kernels(X, metric="polynomial", degree=3, coef0=1)
  K_linear = pairwise_kernels(X, metric="linear")
  ```

## Input validation and robustness (implementation notes)

- _return_float_dtype(X, Y)
  - Decides on an appropriate floating dtype for computations.
  - If both inputs are float32, returns float32; otherwise uses a default float dtype.

- check_pairwise_arrays(X, Y, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)
  - Ensures X and Y are array-like (converting sparse inputs if needed and allowed).
  - Ensures dimensional compatibility (feature dimension matches or appropriate constraints for precomputed matrices).
  - Returns safe_X, safe_Y as dense arrays and the chosen dtype.

- check_paired_arrays(X, Y)
  - Similar validation for paired distance computations, ensuring inputs are appropriate for pairwise distance metrics.

- These utilities enable consistent, safe usage across dense and sparse inputs, and they help surface clear errors when inputs are incompatible.
