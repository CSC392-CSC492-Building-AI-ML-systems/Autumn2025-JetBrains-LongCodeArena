Calculator Interface
====================

Overview
--------

The Calculator Interface provides a simple, unified way to evaluate expressions that reference model parameters across SASModels. It supports programmatic usage (Python API) and a command-line interface (CLI) for quick ad-hoc computations. Expressions can reference parameter names discovered from the available models, and you may optionally constrain evaluation to a single model.

Key ideas
- Parameter namespace: reference model parameters directly by name in expressions.
- Model scoping: optionally restrict calculations to one model.
- Safe, controlled evaluation: expressions are evaluated using a defined, restricted environment with common math functions.

Python API
----------

The Calculator exposes a lightweight API to declare parameter values and evaluate expressions that use those parameters.

- Calculator
  - set_model(model_name): Restrict subsequent evaluations to a single model (optional).
  - set_variable(name, value): Assign a numeric value to a parameter name used in expressions.
  - evaluate(expression, context=None): Evaluate a numeric expression using the current variables (and optional context). Returns a float.
  - get_parameters(): Return a mapping of parameter names to the models that reference them (useful for discovery).
  - list_functions(): Return the set of allowed mathematical functions available in expressions.

Example

.. code-block:: python

   from sasmodels.calculator import Calculator

   calc = Calculator()

   # Optional: limit scope to a single model
   calc.set_model('sphere')

   # Provide values for parameters used in the expression
   calc.set_variable('volume', 1.0)
   calc.set_variable('pi', 3.141592653589793)

   # Evaluate an expression
   result = calc.evaluate('volume * pi')
   print(result)  # 3.141592653589793

   # Use standard math functions
   calc.set_variable('r', 0.5)
   result = calc.evaluate('pi * r**2')
   print(result)  # 0.7853981633974483

CLI (Command-Line Interface)
----------------------------

A command-line interface is provided for quick calculations without writing Python code.

Usage
  - python -m sasmodels.calculator [options] EXPRESSION

Options
  -v, --verbose
      List models that reference the parameter names used in EXPRESSION (when applicable).
  -m MODEL, --model MODEL
      Restrict evaluation to a specific model (optional).
  -h, --help
      Show help message and exit.

Examples
  - Basic evaluation:
      python -m sasmodels.calculator "volume * 2"

  - Verbose parameter discovery (lists models containing the parameters in the expression):
      python -m sasmodels.calculator -v "radius"

  - Model-scoped calculation:
      python -m sasmodels.calculator -m sphere "volume / 3"

Notes
  - The CLI prints the numeric result of the evaluation or an informative error if the expression cannot be computed.
  - Expressions should use parameter names that exist in the discovered catalog. You can use sasmodels.list_pars to inspect available parameters and their associated models.

Relation to Model Metadata
--------------------------

The Calculator relies on the same parameter catalog used by SASModels. See:
- sasmodels.list_pars: List all parameters (and optionally the models that use them).
- SASModels core utilities (e.g., load_model_info): Retrieve model metadata and parameter definitions for reference.

Error Handling
--------------

Common error conditions include:
- Unknown parameter name: The expression references a parameter not present in the current catalog or scope.
- Undefined variable: A name used in the expression has not been provided via set_variable.
- Type/shape mismatch: A parameter value is not numeric or is incompatible with the operation.
- Model scoping conflicts: Attempting to reference a parameter from a model outside the current model scope.

Security and Safety
-------------------

Expressions are evaluated in a restricted environment with a defined set of allowed functions (e.g., basic mathematical operations). User-supplied expressions are not executed as arbitrary code.

Related Documentation
---------------------

- sasmodels.list_pars: Discover parameters used by models
- sasmodels.core.load_model_info: Access model metadata and parameter definitions
