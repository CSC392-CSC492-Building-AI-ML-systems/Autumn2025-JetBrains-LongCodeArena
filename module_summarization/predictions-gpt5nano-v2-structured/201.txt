Partial Dependence and Individual Conditional Expectation plots
===============================================================

This page describes Partial Dependence Plots (PDP) and Individual Conditional Expectation (ICE) plots, including their purpose, interpretation, and how to generate them in scikit-learn.

Overview
--------

- Partial Dependence Plot (PDP): Shows the average effect of a feature (or pair of features) on the predicted outcome, marginalizing over the distribution of all other features. It helps reveal the global, average relationship between a feature and the target.

- Individual Conditional Expectation (ICE) plot: Shows the effect of changing a feature for individual instances while keeping all other features fixed. ICE plots reveal heterogeneity in feature effects that may be hidden in the PDP’s average curve.

- Relationship: ICE lines depict how each instance’s prediction changes with a feature; the mean of ICE lines gives the PDP. ICE is useful for detecting interactions and non-uniform effects across instances.

Key concepts
------------

- Features can be inspected one at a time (1D PDP/ICE) or in pairs (2D PDP) to study joint effects.

- PDP is computed by averaging model predictions over the distribution of all other features.

- ICE lines are generated by fixing all features except the target feature to the values observed in the training data, then varying the target feature.

- Limitations: PDP assumes independence between the target feature and the rest of the features when averaging, which can be misleading for highly correlated features. ICE can reveal heterogeneity due to interactions or correlations.

Usage in scikit-learn
----------------------

scikit-learn provides both legacy and modern interfaces for generating PDP and ICE plots.

- Legacy API (deprecated in newer versions)
  - partial_dependence: compute the marginal effect
  - plot_partial_dependence: visualize PDP (and optionally ICE via kind parameter in some versions)

- Modern API (recommended)
  - PartialDependenceDisplay: a unified plotting interface with from_estimator, supporting both average (PDP) and individual (ICE) plots via the kind parameter.

Supported features
- One feature or two features (1D or 2D PDP)
- Different grid resolutions via grid_resolution
- Mapping of feature names via feature_names (e.g., pandas DataFrame columns)

Code examples
-------------

Example 1: PDP (average effect) for a single feature using the modern API
- Assumes a trained regression model and a feature matrix X (e.g., from California housing dataset).

.. code-block:: python

    from sklearn.ensemble import RandomForestRegressor
    from sklearn.datasets import fetch_california_housing
    from sklearn.inspection import PartialDependenceDisplay
    import matplotlib.pyplot as plt

    # Load data and train model
    X, y = fetch_california_housing(return_X_y=True)
    model = RandomForestRegressor(n_estimators=100, random_state=0, n_jobs=-1)
    model.fit(X, y)

    # PDP for the first feature (index 0)
    display = PartialDependenceDisplay.from_estimator(
        estimator=model,
        X=X,
        features=[0],
        grid_resolution=20,
        kind="average",  # PDP
    )
    display.plot()
    plt.show()

Example 2: ICE (individual conditional expectation) for a single feature using the modern API
- Plot individual instance lines in addition to the average curve.

.. code-block:: python

    from sklearn.ensemble import RandomForestRegressor
    from sklearn.datasets import fetch_california_housing
    from sklearn.inspection import PartialDependenceDisplay
    import matplotlib.pyplot as plt

    # Load data and train model
    X, y = fetch_california_housing(return_X_y=True)
    model = RandomForestRegressor(n_estimators=100, random_state=0, n_jobs=-1)
    model.fit(X, y)

    # ICE for the first feature (index 0)
    display = PartialDependenceDisplay.from_estimator(
        estimator=model,
        X=X,
        features=[0],
        grid_resolution=20,
        kind="individual",  # ICE
    )
    display.plot()
    plt.show()

Example 3: PDP for two features (2D PDP)
.. code-block:: python

    from sklearn.ensemble import RandomForestRegressor
    from sklearn.datasets import fetch_california_housing
    from sklearn.inspection import PartialDependenceDisplay
    import matplotlib.pyplot as plt

    X, y = fetch_california_housing(return_X_y=True)
    model = RandomForestRegressor(n_estimators=100, random_state=0, n_jobs=-1)
    model.fit(X, y)

    # PDP for features 0 and 1 (two features)
    display = PartialDependenceDisplay.from_estimator(
        estimator=model,
        X=X,
        features=[0, 1],
        grid_resolution=20,
        kind="average",
    )
    display.plot()
    plt.show()

Notes on usage
--------------

- feature_names: You can provide names via feature_names (or rely on X if it is a pandas DataFrame with columns). This helps label axes meaningfully.

- 1D vs 2D: Use a single feature for simple interpretable plots; use two features for joint effects but be aware that 2D PDPs can be harder to interpret.

- Data distribution: PDP assumes that the distribution of other features is represented by the data you used to fit the model. If your training data is not representative, PDP/ICE may be misleading.

- Interpretation tips:
  - If ICE lines cross or diverge, there are potential interactions or heterogeneous effects.
  - If PDP shows a flat or near-flat curve, the average effect of the feature may be minimal under the data distribution.

API reference (high level)
--------------------------

- Legacy:
  - sklearn.inspection.partial_dependence
  - sklearn.inspection.plot_partial_dependence

- Modern:
  - sklearn.inspection.PartialDependenceDisplay
    - from_estimator(estimator, X, features, grid_resolution=20, kind='average', ...): compute and plot PDP or ICE.
    - The kind parameter accepts 'average' (PDP) or 'individual' (ICE); you may also see combined usage in some versions.

Tips for comparing PDP and ICE
------------------------------

- Start with ICE to inspect heterogeneity, then summarize with PDP to understand the average effect.
- Use ICE when features are correlated with other features or when the model’s behavior varies across instances.
- Consider interactions: 2D PDPs can reveal joint effects between two features; ICE can help detect interactions that PDP might obscure.
