Cross-validation: evaluating estimator performance
===============================================

Overview
--------
Cross-validation is a core technique for evaluating how well an estimator will generalize to unseen data. In scikit-learn, cross-validation-based evaluation is exposed through two primary tools:

- Learning curves: assess how performance improves as the training set size grows.
- Validation curves: assess how performance varies with a hyperparameter.

This module provides convenient functions and plotting utilities to compute and visualize these curves, including support for estimating variance across cross-validation folds.

Key concepts
------------
- Learning curve
  - Purpose: understand how training size affects both training and generalization performance.
  - Computed by repeatedly fitting the estimator on increasing subsets of the training data and evaluating on held-out folds.
  - Returns:
    - train_sizes: numbers of training examples used
    - train_scores: scores on the training portions (shape: n_ticks x n_folds)
    - test_scores: scores on the validation portions (shape: n_ticks x n_folds)

- Validation curve
  - Purpose: study how a model’s performance varies as a specific hyperparameter changes.
  - Computed by training the estimator with different values of a hyperparameter and evaluating on held-out folds.
  - Returns:
    - param_name and param_range describe the parameter variation
    - train_scores and test_scores similar to learning_curve

- Visualization
  - Display options:
    - std_display_style: how to show variability across folds
      - "fill_between": shading between mean ± std
      - "errorbar": plot mean with error bars
      - None: plot only the mean values
  - Score display:
    - score_name controls the label shown on the y-axis
    - Supports train, test, or both curves
  - Automatic axis scaling
    - The plotting code selects an appropriate x-scale (linear, log, or symlog) based on the x-values, with a deprecation path for a deprecated log_scale parameter.

Usage
-----
1) Learning curve: basic usage

- Purpose: evaluate how estimator performance scales with the size of the training set.
- Example:

```python
from sklearn.datasets import load_iris
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import learning_curve
import numpy as np

X, y = load_iris(return_X_y=True)
estimator = LogisticRegression(max_iter=1000)

train_sizes, train_scores, test_scores = learning_curve(
    estimator, X, y, cv=5,
    train_sizes=np.linspace(0.1, 1.0, 5),
    scoring="accuracy",
)
```

2) Learning curve visualization

```python
from sklearn.model_selection import LearningCurveDisplay
import matplotlib.pyplot as plt

display = LearningCurveDisplay.from_estimator(
    estimator, X, y, cv=5,
    train_sizes=np.linspace(0.1, 1.0, 5),
    scoring="accuracy",
)
display.plot()
plt.show()
```

3) Validation curve: basic usage

- Purpose: evaluate how a hyperparameter affects performance.
- Example:

```python
from sklearn.datasets import load_iris
from sklearn.svm import SVC
from sklearn.model_selection import validation_curve
import numpy as np

X, y = load_iris(return_X_y=True)
estimator = SVC()

param_range = np.logspace(-3, 2, 6)  # e.g., C parameter for SVC
train_scores, test_scores = validation_curve(
    estimator, X, y, param_name="C", param_range=param_range, cv=5,
    scoring="accuracy"
)
```

4) Validation curve visualization

```python
from sklearn.model_selection import ValidationCurveDisplay
import matplotlib.pyplot as plt

display = ValidationCurveDisplay.from_estimator(
    estimator, X, y, param_name="C", param_range=param_range, cv=5,
    scoring="accuracy"
)
display.plot()
plt.show()
```

Key parameters and what they mean
----------------------------------
- train_sizes: array-like, shape (n_ticks,)
  - The numbers of training examples to use for generating the learning curve.

- train_scores, test_scores: ndarray, shape (n_ticks, n_cv_folds)
  - Scores for each training size and cross-validation fold.
  - The mean across folds is typically plotted; the std is used for shading or error bars.

- score_name: str
  - Label used for the y-axis in plots. If not provided, a sensible default is used.

- std_display_style: {"fill_between", "errorbar", None}
  - Determines how the standard deviation across folds is visualized.

- score_type: {"test", "train", "both"}
  - Selects which curves to display: training performance, cross-validated performance, or both.

- log_scale (deprecated)
  - Deprecated parameter controlling x-axis scale; see the deprecation note in the plotting code. Prefer using ax.set_xscale on the produced display.

Under the hood
--------------
- The visualization utilities (e.g., LearningCurveDisplay) rely on a common plotting base that:
  - Accepts x_data (train_sizes or param values)
  - Computes mean and std across CV folds
  - Supports plotting either two lines (train/test) or a single line
  - Visualizes uncertainty via fill_between or error bars
  - Attempts to automatically pick an appropriate x-axis scale (linear, log, or symlog)

Notes
-----
- The plotting back-end depends on Matplotlib; ensure it is installed.
- The default visualization shows mean performance with variability across cross-validation folds.
- You can customize the display style to emphasize either the mean curve or the uncertainty region.
