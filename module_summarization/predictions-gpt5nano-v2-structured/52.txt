# gmm.rst — Documentation for Generalized Method of Moments (GMM) and IV2SLS

Overview
--------

This module implements foundational elements for Generalized Method of Moments (GMM) estimation and, more specifically, an instrumental variables estimator using Two-Stage Least Squares (2SLS).

Key components:

- IV2SLS: An instrumental variables estimator using the classic 2SLS approach. It inherits from LikelihoodModel and provides a straightforward 2SLS implementation with a primary focus on clarity and integration with StatsModels results infrastructure.
- IVRegressionResults: A results class designed to hold the output of the IV2SLS estimation, providing parameters and covariance information compatible with the rest of the StatsModels regression framework.
- maxabs: A small helper utility to obtain the maximum absolute value in an array.

Purpose
-------

- Provide a clear, minimal implementation of 2SLS within the StatsModels modeling framework.
- Integrate with the existing regression results workflow (RegressionResults, RegressionResultsWrapper, and OLS for the second-stage results).
- Support basic use cases for instrumental variables when endogenous regressors are present and valid instruments are available.

Usage at a Glance
-----------------

- Create an IV2SLS model by supplying the endogenous variable(s), exogenous variables to be instrumented, and a set of instruments.
- Call fit() to obtain a results object wrapped as a RegressionResults-like object for consistency with other regression models.
- Access standard result attributes such as params and cov_params via the returned results.

Key Functionality
-----------------

- First-stage projection:
  - Compute xhatparams = (Z'Z)^{-1} Z' X, where Z contains the instruments and X the exogenous variables that are being instrumented.
  - Form the fitted X (F = Z XhatParams) for use in the second stage.
- Second-stage IV estimation:
  - Solve (F'F) beta = F'y to obtain the IV parameter estimates.
  - Construct the normalized covariance parameters for the results object.
- Results integration:
  - The fit() method returns a RegressionResultsWrapper around an IVRegressionResults instance, ensuring compatibility with StatsModels’ reporting and diagnostics.
- Flexibility:
  - The module notes that the whitening and other advanced GMM features are not implemented here; the focus is on a clear, reliable IV2SLS estimator that plugs into the StatsModels ecosystem.
- Diagnostics:
  - Degrees-of-freedom are computed in the constructor (df_resid and df_model) in line with typical regression reporting conventions.

API Reference
-------------

Classes

- IV2SLS(LikelihoodModel)
  - Purpose: Instrumental variables estimation via Two-Stage Least Squares.
  - Core methods:
    - __init__(endog, exog, instrument=None): Initialize endog (dependent variable), exog (explanatory variables to be instrumented), and instrument (instruments that explain endog/exog).
    - initialize(): Prepare working copies (wendog, wexog) for estimation.
    - whiten(X): Placeholder for a whitening routine (not implemented).
    - fit(): Perform the 2SLS estimation and return a RegressionResultsWrapper that contains an IVRegressionResults instance and a secondary OLS fit of y on the fitted exog.
    - predict(params, exog=None): Predicts values given parameter estimates.
  - Notes:
    - Degrees of freedom are computed as df_resid = nobs - k_vars and df_model = k_exog - k_constant.
    - All variables in exog are treated as potentially endogenous; any variables not meant to be instrumented should be included in the instrument matrix.
    - The result structure includes both IV parameters and a secondary OLS fit on the fitted values for diagnostic purposes.

- IVRegressionResults(RegressionResults)
  - Purpose: Results container for the IV2SLS estimator.
  - Usage: Returned by IV2SLS.fit() via a RegressionResultsWrapper, consistent with other regression result objects in StatsModels.
  - Notes:
    - Exposes standard regression results attributes, with additional fields added by IV2SLS (e.g., exog_hat_params, exog_hat) for the first-stage fitted values.
    - The code stores a secondary OLS fit on the second-stage fitted exog to provide additional diagnostic information.

Functions

- maxabs(x)
  - Purpose: Convenience wrapper to return the maximum absolute value of an array.
  - Signature: maxabs(x)
  - Returns: float (maximum absolute value)

Usage Example
-------------

A minimal example of performing 2SLS using IV2SLS:

```python
import numpy as np
from statsmodels.sandbox.regression.gmm import IV2SLS  # path may vary in your installation

np.random.seed(0)
n = 100

# Instruments (Z)
Z = np.random.normal(size=(n, 2))

# Endogenous regressor (X) expressed in terms of instruments plus noise
X = Z @ np.array([0.6, -0.2]) + np.random.normal(size=n)

# Exogenous outcome variable (Y) with true relationship Y = beta0 + beta1 * X + error
beta0, beta1 = 1.0, 2.0
error = np.random.normal(scale=1.0, size=n)
Y = beta0 + beta1 * X + error

# IV: instrument matrix Z for endogenous regressor X
# If there are multiple exogenous variables to instrument, put them in X as columns as well.
iv_model = IV2SLS(Y, X, Z)

results = iv_model.fit()
print(results.summary())
```

Notes
-----

- The current implementation focuses on clarity and integration with StatsModels’ result handling. It does not yet implement more advanced GMM weighting matrices or robust_cov computations that depend on a fully specified moment matrix and its omega.

- Whitening and other preconditioning steps are not implemented; the provided method is a straightforward 2SLS estimator suitable for standard IV regression scenarios.

- The design emphasizes compatibility with the rest of the regression framework in StatsModels, including the use of RegressionResults and RegressionResultsWrapper for reporting and downstream diagnostics.

-------------------------------

- The IV2SLS class is designed to be a building block within a broader GMM framework. The current implementation provides a clean, dependable 2SLS estimator that can be extended with additional moment conditions, weighting matrices, and diagnostics as needed.
- The module hints at future enhancements for full GMM support, including handling multiple moments per observation and analytical weighting matrices.
