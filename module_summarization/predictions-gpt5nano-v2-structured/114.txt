Simulation
==========

Overview
--------
This module provides utilities to assist with Visualizing and validating Bluesky plans used to move motors and acquire data. It includes:
- plot_raster_path: Visualize the raster path generated by a plan.
- summarize_plan (alias: print_summary): Print a compact summary of a plan.
- check_limits: Validate that a plan does not move devices beyond their limits.

Dependencies
------------
- matplotlib (for plotting in plot_raster_path)
- bluesky (for print_summary_wrapper used by summarize_plan)
- A plan must yield Msg objects (as produced by Bluesky plans)

Functions
---------

plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Plot the raster path for a Bluesky plan.

Purpose
- Reconstruct and plot the path that a plan will take in the x-y motor space.
- Visualize the readback points either as markers or as circular probes.

Parameters
- plan: iterable
  An iterable that yields Msg objects and is not a coroutine. The function expects messages with:
  - cmd 'set' to update motor positions
  - cmd 'save' to mark a read point in the trajectory
- x_motor: str
  Name of the x motor.
- y_motor: str
  Name of the y motor.
- ax: matplotlib.axes.Axes, optional
  Axes to plot on. If None, a new figure and axes are created.
- probe_size: float, optional
  If provided, draw each read point as a circle of this radius (in the same units as motor positions).
  If None, read points are plotted as markers.
- lw: float, optional
  Line width for the trajectory.

Returns
- dict with:
  - 'path': the Line2D object for the plotted path
  - 'events': a matplotlib collection of the read points (PatchCollection when probe_size is used, or PathCollection when not)

Example
.. code-block:: python

   from simulation import plot_raster_path
   import matplotlib.pyplot as plt

   # plan: an iterable of Bluesky Msg objects
   fig, ax = plt.subplots()
   result = plot_raster_path(plan, x_motor='x', y_motor='y', ax=ax, probe_size=None, lw=2)

   plt.show()

summarize_plan(plan)
--------------------
Print a summary of the plan.

Purpose
- Print a minimal version of the plan, showing only moves and where events are created.
- Uses Bluesky's print_summary_wrapper to generate the summary.

Parameters
- plan: iterable
  An iterable that yields Msg objects.

Returns
- None. The function prints to the console via the summary wrapper.

Notes
- This function is assigned to the name print_summary for backward compatibility.

Example
.. code-block:: python

   from simulation import summarize_plan, print_summary
   # plan: iterable of Msgs
   summarize_plan(plan)
   # or
   print_summary(plan)

check_limits(plan)
-------------------
Check that a plan will not move devices outside of their limits.

Purpose
- Validate a plan by invoking each device's check_value method (if available) when a set command is encountered.
- If a device lacks check_value, emit a warning and skip further checks for that device.

Parameters
- plan: iterable
  An iterable that yields Msg objects.

Returns
- None. May emit warnings if devices do not have check_value or if values are outside limits.

Notes
- Objects without a check_value method are added to an ignore list after a warning, so subsequent set messages for that object are skipped.

Example
.. code-block:: python

   from simulation import check_limits
   # plan: iterable of Msgs
   check_limits(plan)

Design and usage notes
----------------------
- The plots produced by plot_raster_path rely on the plan emitting:
  - set messages to update cur_x and/or cur_y based on msg.obj.name matching x_motor or y_motor
  - save messages to append the current position to the trajectory
- The function assumes motor positions are available at the time of a save command.
- The probe visualization can help quickly assess where reads occur along the path.

Example plan shape (conceptual)
--------------------------------
- set: object with name 'x' and value (args[0])
- set: object with name 'y' and value (args[0])
- save: record the current (x, y) position

Changelog and compatibility
----------------------------
- summarize_plan is exposed as print_summary for backward compatibility.
- The implementation uses matplotlib for plotting and Bluesky’s print_summary_wrapper for plan summarization.

Code references
---------------
- plot_raster_path: Visualizes raster trajectory by processing 'set' and 'save' messages.
- summarize_plan / print_summary: Provides a concise plan summary for quick inspection.
- check_limits: Validates plan safety against device limits.
