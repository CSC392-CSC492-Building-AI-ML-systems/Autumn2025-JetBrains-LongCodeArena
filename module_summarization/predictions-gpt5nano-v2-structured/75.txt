Working with Text Data
======================

A practical tutorial on turning raw text into numeric features suitable for machine learning, using scikit-learn’s text feature extraction tools.

Overview
--------

The sklearn.feature_extraction.text module provides:

- CountVectorizer: converts a collection of text documents to a matrix of token counts.
- TfidfVectorizer: converts documents to a matrix of TF-IDF features.
- HashingVectorizer: performs feature hashing to produce a sparse feature representation without building a vocabulary.
- TfidfTransformer: converts a matrix of term frequencies to TF-IDF features (often used with CountVectorizer).
- Helpers for preprocessing and tokenization, including:
  - strip_accents_unicode and strip_accents_ascii
  - strip_tags
  - ENGLISH_STOP_WORDS

Basic workflow
--------------

1. Prepare your corpus: a list of text documents.
2. Choose a vectorizer (and optional preprocessing).
3. Fit the vectorizer on your corpus and transform the documents into a sparse feature matrix.
4. Use the feature matrix as input to ML models.

Getting started with CountVectorizer
------------------------------------

The CountVectorizer tokenizes text and builds a vocabulary of known words or n-grams. It also supports several preprocessing options such as lowercasing, stop words removal, and custom analyzers.

Example: basic word counts with English stop words removal

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer

    docs = [
        "The quick brown fox jumps over the lazy dog.",
        "Never jump over the lazy dog quickly."
    ]

    vectorizer = CountVectorizer(stop_words='english', lowercase=True)
    X = vectorizer.fit_transform(docs)

    print(X.shape)  # (2, n_features)
    print(vectorizer.get_feature_names_out())

Output (example):
    (2, 7)
    ['brown' 'dog' 'fox' 'jump' 'quick' 'lazy' 'over']

Meaning:
- Each document is represented as a sparse vector of term counts.
- Stop words (like "the", "over") are removed automatically.

TF-IDF features with TfidfVectorizer
-------------------------------------

For many text classification tasks, TF-IDF features work better than raw counts.

Example: word-level TF-IDF with 1- to 2-grams

.. code-block:: python

    from sklearn.feature_extraction.text import TfidfVectorizer

    docs = [
        "The quick brown fox jumps over the lazy dog.",
        "Never jump over the lazy dog quickly."
    ]

    vectorizer = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
    X = vectorizer.fit_transform(docs)

    print(X.shape)  # (2, n_features)
    print(vectorizer.get_feature_names_out())

Notes:
- ngram_range controls the range of n in n-grams (e.g., (1, 2) for unigrams and bigrams).
- By default, terms are normalized and TF is weighted by Inverse Document Frequency (IDF).

HashingVectorizer for scalable feature extraction
-------------------------------------------------

When dealing with very large vocabularies, HashingVectorizer avoids building a vocabulary entirely and hashes features into a fixed-dimensional space.

Example:

.. code-block:: python

    from sklearn.feature_extraction.text import HashingVectorizer

    docs = [
        "The quick brown fox jumps over the lazy dog.",
        "Never jump over the lazy dog quickly."
    ]

    hasher = HashingVectorizer(n_features=2**20, alternate_sign=False)
    X = hasher.fit_transform(docs)

    print(X.shape)  # (2, 1048576)

Notes:
- No training is required; the hashing function deterministically maps terms to feature indices.
- alternate_sign controls the sign of the hashed features; False yields non-negative features.

Preprocessing and custom analyzers
----------------------------------

You can customize how text is prepared before tokenization through preprocessors and analyzers.

Common helpers:

- strip_accents_unicode: convert accented characters to their base form (preserves non-ASCII where appropriate).
- strip_accents_ascii: transliterate to ASCII, removing characters without ASCII equivalents.
- strip_tags: remove simple HTML/XML tags.

Examples:

.. code-block:: python

    from sklearn.feature_extraction.text import strip_accents_unicode, strip_tags

    text = "<p>Résumé of naïve café</p>"

    # Remove HTML tags
    clean = strip_tags(text)

    # Normalize accents
    clean = strip_accents_unicode(clean)

    print(clean)  # "Résumé of naive cafe" with accents removed

Advanced: custom analyzers and tokenization
-------------------------------------------

You can supply a full analyzer function to CountVectorizer or TfidfVectorizer. An analyzer is a callable that takes a document and returns an iterable of tokens (or n-grams).

Example: custom whitespace-based tokenizer with lowercasing

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer

    def custom_analyzer(doc):
        return doc.lower().split()

    docs = [
        "The quick brown fox",
        "Brown fox quick"
    ]

    vectorizer = CountVectorizer(analyzer=custom_analyzer)
    X = vectorizer.fit_transform(docs)
    print(X.shape)
    print(vectorizer.get_feature_names_out())

Input types and decoding
------------------------

Vectorizers can read raw file contents directly when configured.

Example: reading from filenames

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer

    # One document per file path
    file_paths = ["path/to/doc1.txt", "path/to/doc2.txt"]

    vectorizer = CountVectorizer(input='filename', encoding='utf-8')
    X = vectorizer.fit_transform(file_paths)

Notes:
- input='filename' makes the vectorizer read the file contents as strings.
- Other valid input modes include input='content' (default) and input='object' for raw strings.

Stop words and language support
-------------------------------

- ENGLISH_STOP_WORDS provides a built-in list suitable for English text.
- You can also pass stop_words as a list or set of tokens, or use a language-specific built-in list when available.

Example: using built-in English stop words

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer, ENGLISH_STOP_WORDS

    docs = [
        "This is a sample document with several words.",
        "Another sample document for demonstration."
    ]

    vectorizer = CountVectorizer(stop_words=ENGLISH_STOP_WORDS)
    X = vectorizer.fit_transform(docs)

Choosing the right vectorizer
-----------------------------

- For quick prototypes or very large datasets, HashingVectorizer offers a scalable, stateless option.
- For most text classification tasks, TfidfVectorizer (with a reasonable n-gram range) provides strong performance.
- For simple bag-of-words features, CountVectorizer is straightforward and fast.

Practical tips
--------------

- Normalize text (lowercase, remove HTML tags, strip accents) before tokenization when appropriate.
- Use ngram_range to capture phrases; beware the feature space grows quickly.
- Experiment with stop words; removing them can improve performance but may also remove informative terms for some tasks.
- Validate feature quality with simple baselines before investing in complex pipelines.

Reference notes
---------------

- Core preprocessing helpers (strip_accents_unicode, strip_accents_ascii, strip_tags) are accessible from the same module.
- The underlying behavior of analyzers, preprocessor, tokenizer, and ngrams can be customized via parameters on the vectorizers.

This tutorial covers the essentials for generating and using text features with scikit-learn, enabling you to build effective text classification, clustering, or similarity models.
