# Manifold learning: non-linear dimensionality reduction via SMACOF MDS

 Manifold learning encompasses techniques that aim to uncover and embed the intrinsic, often non-linear, structure of high-dimensional data into a lower-dimensional space. The SMACOF-based Multidimensional Scaling (MDS) approach provides a principled way to embed data by preserving pairwise dissimilarities (distances or perceived differences) as faithfully as possible in a reduced-dimensional space. It supports both metric and non-metric variants, enabling flexible handling of different data characteristics.

## Key ideas

- Distance-preserving embedding: MDS seeks coordinates in a low-dimensional space that best preserve the given pairwise dissimilarities.
- Metric vs non-metric MDS:
  - Metric MDS (default): the embedding distances aim to match the provided dissimilarities directly.
  - Non-metric MDS: only the rank order of dissimilarities is preserved; the algorithm uses isotonic regression to derive a monotone mapping from dissimilarities to disparities.
- Stress as a goodness-of-fit: the objective is the stress (sum of squared differences between original dissimilarities and embedding distances). A lower stress indicates a better fit. A normalized stress (Stress-1) can be returned for non-metric MDS.
- Iterative refinement: the SMACOF algorithm iteratively updates the embedding via the Guttman transform until convergence or a maximum number of iterations is reached.
- Robustness and initialization: initialization can be random or user-provided; a random seed ensures reproducibility. A small value is used to avoid division by zero in the update step.

## The SMACOF MDS algorithm

The core routine computes a low-dimensional embedding X from a symmetric dissimilarities matrix D.

- Input
  - dissimilarities: symmetric pairwise dissimilarities (n_samples × n_samples)
  - metric: boolean flag; True for metric MDS, False for non-metric MDS
  - n_components: target embedding dimensionality
  - init: optional initial embedding (n_samples × n_components)
  - max_iter: maximum number of iterations per run
  - eps: convergence tolerance
  - random_state: seed for initialization
  - normalized_stress: whether to return Stress-1 (only valid for non-metric MDS)

- Internal steps
  - Validate symmetry; ensure a valid square distance matrix.
  - Initialize X (randomly or from init).
  - Repeat for up to max_iter:
    - Compute pairwise Euclidean distances from X.
    - If metric is True, set disparities = dissimilarities.
    - If metric is False, compute disparities via isotonic regression of a monotone fit between the non-zero dissimilarities and current distances (handling missing values where dissimilarities are zero).
    - Compute stress = sum((disparities - distances)²) / 2; optionally normalize.
    - Update X via the Guttman transform (B matrix) to minimize stress.
    - Check convergence using change in stress; break if converged.
  - Return the embedding X, the final stress, and the number of iterations performed.

## Input and output details

- dissimilarities
  - Shape: (n_samples, n_samples)
  - Symmetric, representing pairwise dissimilarities. Zeros may indicate missing values in non-metric mode when using dissimilarities with 0 as missing indicators.

- metric
  - True: metric MDS (preserves raw dissimilarities)
  - False: non-metric MDS (preserves monotone rank order)

- n_components
  - Target dimensionality of the embedding (typically 2 or 3 for visualization)

- init
  - Optional initial embedding of shape (n_samples, n_components). If None, a random embedding is used.

- max_iter
  - Maximum iterations for a single run.

- eps
  - Relative tolerance used to determine convergence.

- random_state
  - Seed or RandomState instance to ensure reproducible initialization.

- normalized_stress
  - If True and metric=False, returns Stress-1 (normalized stress).

- Returns
  - X: array of shape (n_samples, n_components)
    Coordinates of the embedding.
  - stress: float
    Final stress value (or Stress-1 if normalized_stress is True in non-metric mode).
  - n_iter: int
    Number of iterations performed.

## Practical usage

While the internal SMACOF routine is exposed via the public MDS class in typical usage, understanding the underlying behavior helps when working with precomputed dissimilarities.

- Public usage pattern (example with precomputed distances):
  - Metric MDS
  - Non-metric MDS

```python
import numpy as np
from sklearn.metrics import pairwise_distances
from sklearn.manifold import MDS

# Suppose X is your high-dimensional data
# D = pairwise_distances(X)  # or use a precomputed distance/dissimilarity matrix

# Metric MDS: preserve actual dissimilarities
mds_metric = MDS(n_components=2, metric=True, dissimilarity='precomputed', random_state=0)
Y_metric = mds_metric.fit_transform(D)

# Non-metric MDS: preserve rank order of dissimilarities
mds_non_metric = MDS(n_components=2, metric=False, dissimilarity='precomputed', random_state=0)
Y_non_metric = mds_non_metric.fit_transform(D)
```

- If you already have a complete distance matrix, you can pass it directly as shown above. The non-metric variant will automatically apply isotonic regression to derive monotone disparities.

## When to use SMACOF MDS for manifold learning

- Non-linear structure: If your data lie on a non-linear manifold where global distances are informative but not preserved by simple linear projections, SMACOF MDS can reveal meaningful low-dimensional structure by focusing on pairwise dissimilarities.
- Visual exploration: 2D or 3D embeddings to visualize relationships and clusters.
- When you have a meaningful distance or dissimilarity measure between samples, not just feature vectors.

## Notes and design considerations

- Non-metric MDS uses isotonic regression to align the embedding with the rank order of dissimilarities, which can be robust to non-linear distortions in the dissimilarity scale.
- The Guttman transform updates the embedding to progressively reduce stress, with convergence controlled by eps.
- A small constant is added to distances to avoid division by zero during updates.
- The normalized_stress option (Stress-1) provides a normalized goodness-of-fit metric for non-metric MDS.
