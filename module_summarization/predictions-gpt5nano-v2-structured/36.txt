w_succ
======

List segments which successfully recycled to a target state.

Overview
--------
The w_succ command analyzes a WestPA run and lists segments that reach a target state by examining the final progress coordinates of recycled segments at each iteration. For every recycled segment, it prints:
- iteration
- segment ID (seg_id)
- segment weight
- final progress coordinate values (one per coordinate)

Output is written to standard output by default, or to a file if the -o/--output option is provided.

Key behavior
------------
- Iterates over iterations from 1 up to current_iteration - 1.
- For each iteration, identifies recycled segments (endpoint_type == Segment.SEG_ENDPOINT_RECYCLED).
- Retrieves the final progress coordinates for those recycled segments.
- Outputs a line per recycled segment containing:
  - iteration, seg_id, weight
  - final progress coordinate values formatted by their data types
- If no recycled segments exist for an iteration, that iteration is skipped.
- Header lines describe the columns unless headers are suppressed.

Usage
-----
Basic:
- w_succ

Output to a file:
- w_succ -o succ.txt

WestPA runtime options:
- The command integrates standard WestPA runtime options via westpa.rc. You can configure the run with rc files and other common options as you would for other WestPA tooling.

Options
-------
- -o, --output OUTPUT_FILE
  Store output in OUTPUT_FILE (default: write to standard output).

Notes
-----
- The header describing the columns is printed by default. It can be suppressed by configuring the tool (output_suppress_headers) if needed.
- The output format is designed to be parseable by post-processing tools; fields beyond column 2 are the final progress coordinate values for each recycled segment.

Output format (example)
-----------------------
# successful (recycled) segments
# column 0:    iteration
# column 1:    seg_id
# column 2:    weight
# column>2:    final progress coordinate value
     1         3            0.01234567890000    0.1234567    0.23456789
     2        15            0.00500000000000    0.1111111    0.2222222

Implementation notes
--------------------
- The tool uses the following core components:
  - WestPA data reader mixins to access segment indices and pcoord datasets.
  - Identification of recycled endpoints via Segment.SEG_ENDPOINT_RECYCLED.
  - Formatting of final progress coordinate values based on their dtype using a mapping from dtype to format string.
- If there are no recycled segments in a given iteration, that iteration is skipped to avoid errors when querying pcoord data.
