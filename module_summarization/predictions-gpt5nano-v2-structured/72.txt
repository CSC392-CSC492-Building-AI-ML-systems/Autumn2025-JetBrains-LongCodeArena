Metrics Utilities for Pairwise Distances
==========================================

Overview
--------
This module provides internal utilities used by pairwise distance computations in the metrics subsystem. It focuses on validating inputs, handling data types for floating-point computations, and ensuring consistency between input matrices X and Y (including the case of precomputed distance matrices). End users generally interact with higher-level pairwise distance functions; these helpers are primarily intended for internal use to guarantee safe and predictable behavior across a variety of input formats (dense arrays, sparse matrices, precomputed distances, etc.).

Public API
----------
The following helpers are provided for internal use:

- _return_float_dtype(X, Y)
- check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse='csr', force_all_finite=True, copy=False)
- check_paired_arrays(X, Y)

Function: _return_float_dtype
----------------------------
Determines the appropriate floating-point dtype to use for pairwise computations based on the input X and optional Y.

Parameters
----------
X : {array-like, sparse matrix}
    Input data with shape (n_samples_X, n_features).

Y : {array-like, sparse matrix} or None
    If provided, the second input for distance computations. If None, Y is treated as X.

Notes
-----
- If both X and Y are float32, the function returns float32; otherwise it returns the default floating dtype (float).

Returns
-------
X : array-like
    X converted to a NumPy array if necessary.

Y : array-like
    Y converted to a NumPy array if necessary (or the same as X when Y is None or Y is X).

dtype : type
    The chosen floating dtype (np.float32 or float).

Example
-------
>>> import numpy as np
>>> from sklearn.metrics.pairwise import _return_float_dtype
>>> X = np.array([[1, 2], [3, 4]], dtype=np.float32)
>>> Y = None
>>> X2, Y2, dt = _return_float_dtype(X, Y)
>>> dt
<class 'numpy.float32'>

Function: check_pairwise_arrays
------------------------------
Validate and prepare input arrays for pairwise distance computations.

Parameters
----------
X : {array-like, sparse matrix} of shape (n_samples_X, n_features)
    First input data.

Y : {array-like, sparse matrix} of shape (n_samples_Y, n_features) or None
    Second input data. If None or identical to X, a notional link is used to pairwise compute with X.

precomputed : bool, default=False
    True if X is treated as precomputed distances to the samples in Y.

dtype : {str, type}, default=None
    Data type required for X and Y. If None, a suitable float dtype is chosen by _return_float_dtype.

accept_sparse : {str, bool, list/tuple of str}, default='csr'
    Sparse matrix format(s) to accept. If the input is sparse but not in an allowed
    format, it will be converted to the first listed format. True allows any format.
    False means that a sparse input will raise an error.

force_all_finite : bool or 'allow-nan', default=True
    Whether to raise an error on np.inf, np.nan, or pd.NA in the array.

    - True: All values must be finite.
    - False: np.inf, np.nan, and pd.NA are allowed.
    - 'allow-nan': Only np.nan and pd.NA are allowed (infinite values are not allowed).

    Note: pd.NA is converted to np.nan where applicable.

copy : bool, default=False
    Whether a forced copy will be triggered. If copy is False, a copy might be
    triggered by a conversion.

Returns
-------
safe_X : {array-like, sparse matrix} of shape (n_samples_X, n_features)
    An array equal to X, guaranteed to be a NumPy array.

safe_Y : {array-like, sparse matrix} of shape (n_samples_Y, n_features)
    An array equal to Y if Y was provided, guaranteed to be a NumPy array.
    If Y is None or X is Y, safe_Y is a pointer to X.

Raises
------
ValueError
    If the second dimension of X and Y does not match, or if precomputed mode is inconsistent
    (e.g., shapes do not align for precomputed distances).

Notes
-----
This helper is typically used by higher-level distance metrics to ensure inputs
are numeric, have compatible shapes, and are handled in a consistent dtype.

Example
-------
>>> import numpy as np
>>> from sklearn.metrics.pairwise import check_pairwise_arrays
>>> X = np.array([[1, 2], [3, 4]])
>>> Y = None
>>> X2, Y2, dt = check_pairwise_arrays(X, Y)
>>> X2.shape, Y2.shape
((2, 2), (2, 2))

Function: check_paired_arrays
-----------------------------
Validate and prepare inputs for paired distance calculations.

Parameters
----------
X : {array-like, sparse matrix} of shape (n_samples_X, n_features)
    First input array for paired distances.

Y : {array-like, sparse matrix} of shape (n_samples_Y, n_features)
    Second input array for paired distances.

Returns
-------
X, Y : tuple
    Prepared X and Y arrays suitable for paired distance computations.

Notes
-----
This helper focuses on validated, consistent shapes for paired distance
metrics, ensuring both inputs are properly converted and dimensionally aligned.

Examples
--------
>>> import numpy as np
>>> from sklearn.metrics.pairwise import check_paired_arrays
>>> X = np.asarray([[1, 2], [3, 4]])
>>> Y = np.asarray([[5, 6], [7, 8]])
>>> Xp, Yp = check_paired_arrays(X, Y)
>>> Xp.shape, Yp.shape
((2, 2), (2, 2))

Usage and Compatibility
-----------------------
- These utilities are intended for internal use by the pairwise distance implementations.
- End users should rely on the higher-level pairwise distance functions (e.g., cosine_distances, euclidean_distances) for standard workflows.
- They support dense and sparse inputs and provide robust handling of edge cases such as precomputed distance matrices.

Dependencies
------------
- NumPy
- SciPy (for sparse matrix handling)
- Internal utilities from the package (e.g., check_array, _mask, validation helpers)

See Also
--------
ArgKmin, _chi2_kernel_fast, _sparse_manhattan
These internal components are leveraged by higher-level distance computations.
