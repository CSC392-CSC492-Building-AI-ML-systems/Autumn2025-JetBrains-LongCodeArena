Controlling chunking
====================

In Dask Array, data is partitioned into chunks along each axis. The size and layout of these chunks determine parallelism, memory usage, and the performance of array operations. This section explains how to control chunking and common workflows.

Overview
--------
- A chunk is a contiguous block of data along one or more axes.
- Chunk sizes are specified when creating arrays and can be changed later with rechunk.
- Proper chunking can improve cache locality, load balancing, and parallelism; poor chunking can increase overhead or memory usage.
- Dask automatically aligns and handles chunking for many operations, but you can explicitly control it to optimize workloads.

Creating arrays with explicit chunks
------------------------------------
Specify chunk sizes when constructing a Dask array. You can pass a chunks argument to creation helpers such as from_array, empty, ones, zeros, etc.

.. code-block:: python

    import numpy as np
    import dask.array as da

    # Create a 1000x1000 array backed by 100 blocks of 10x10 each
    x = np.arange(1000*1000).reshape(1000, 1000)
    a = da.from_array(x, chunks=(10, 10))

    # Create a 10000x10000 array with 1000x1000 blocks (10x10 blocks of 100x100)
    b = da.zeros((10000, 10000), chunks=(1000, 1000))

Tips:
- Choose chunk sizes that balance the number of tasks with the overhead of scheduling.
- Align chunk boundaries with memory access patterns and downstream operations.

Rechunking (changing chunk structure)
-------------------------------------
To change the chunking of an existing array, use rechunk. This can be useful to align blocks for operations or to adjust parallelism.

.. code-block:: python

    # Rechunk to a different block/grid
    c = a.rechunk((20, 20))             # new chunks of 20x20
    d = da.rechunk(a, chunks=(5, 5))    # rechunk using the global function

Notes:
- Rechunking may involve moving data between tasks and can incur additional computation.
- Choose a new chunk layout that optimizes the planned computations.

Aligning chunks for operations
------------------------------
Many Dask Array operations (e.g., elementwise ops, reductions, reductions with masks) require compatible chunking across inputs. Dask will automatically align blocks where possible, often by rechunking to a common grid. You can also proactively rechunk inputs to avoid unnecessary data movement or to improve performance.

.. code-block:: python

    x = da.random.random((1000, 1000), chunks=(250, 250))
    y = da.random.random((1000, 1000), chunks=(500, 250))

    # Resulting chunks will be aligned for the operation
    z = x + y  # Dask aligns to a common chunk grid

    # Proactively align inputs if you know the best layout
    x2 = x.rechunk((500, 250))
    z2 = x2 + y

Choosing chunk sizes
--------------------
Guidelines to pick chunk sizes:

- Balance task granularity: smaller chunks increase parallelism but add scheduling overhead.
- Consider memory constraints: too many small chunks may exhaust scheduler resources; too few large chunks may underutilize parallelism.
- Match downstream operations: operations like convolutions, reductions, or blockwise reductions may benefit from regular, aligned chunk grids.
- Backends: if using CuPy or other backends, ensure chunk sizes map well to device memory and transfer patterns.

Code examples for common workflows
----------------------------------
- Create a large array with a regular grid of chunks
- Rechunk to optimize for a specific operation
- Ensure alignment before a blockwise operation

.. code-block:: python

    # Large array with a regular chunk grid
    large = da.random.random((20000, 20000), chunks=(1000, 1000))

    # Rechunk for a particular operation
    op_input = large.rechunk((2000, 2000))

    # Blockwise operation that benefits from aligned chunks
    from dask import array as da
    a = da.arange(1000000, chunks=(1000,))
    b = a * 2
    c = da.stack([a, b], axis=0)
