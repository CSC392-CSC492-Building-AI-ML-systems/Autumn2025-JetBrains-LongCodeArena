Novelty and Outlier Detection
===============================

Overview
--------

This section documents approaches for identifying unusual observations in data, covering two related tasks:

- Novelty detection (also known as one-class novelty detection): detect anomalies in new data after being trained on normal (in-distribution) data. The model is exposed only to normal examples during training.
- Outlier detection (unsupervised anomaly detection): identify unusual observations in a dataset, where the training data may contain anomalies or the goal is to find any deviations from the overall distribution.

Key distinctions
- Novelty detection
  - Training data consists of normal observations.
  - Predicts whether new samples are normal or novel/outlier.
  - Typical estimators: OneClassSVM, IsolationForest(novelty=True), LocalOutlierFactor(novelty=True), EllipticEnvelope(novelty=True).
- Outlier detection
  - Training data may contain anomalies or no labeled anomalies.
  - Seeks to identify outliers within the observed data.
  - Typical estimators: OneClassSVM, IsolationForest (default, novelty=False), LocalOutlierFactor (novelty=False), EllipticEnvelope (default).

Choosing an approach
- If you have a clean dataset representing normal behavior and you want to flag new observations as novelty, use novelty-enabled estimators (novelty=True) or OneClassSVM.
- If you want to identify anomalies within a dataset without a clear separation between normal and anomalous in the training data, use standard outlier detection (novelty=False).
- Always scale/normalize features where appropriate to improve model performance.

Estimators and usage
--------------------

Below are common estimators supporting novelty and/or outlier detection, with typical usage patterns.

OneClassSVM (novelty and outlier detection)
-------------------------------------------

- Novelty detection (train on normal data, then score new data)
- Outlier detection (unsupervised; typically used without novelty)

Code example (novelty detection):
```python
from sklearn.svm import OneClassSVM
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline

# X_normal: training data containing normal observations only
# X_new: new observations to evaluate

model = make_pipeline(StandardScaler(), OneClassSVM(kernel="rbf", gamma="scale", nu=0.05))
model.fit(X_normal)
pred = model.predict(X_new)  # 1 for normal, -1 for outliers
```

IsolationForest
--------------

- Novelty detection: set novelty=True
- Outlier detection: default (novelty=False)

Code example (novelty detection):
```python
from sklearn.ensemble import IsolationForest

# X_normal: training data containing normal observations
# X_new: new observations to evaluate

clf = IsolationForest(n_estimators=200, contamination="auto", novelty=True, random_state=42)
clf.fit(X_normal)
scores = clf.decision_function(X_new)  # higher = more normal (interpret with care)
pred = clf.predict(X_new)  # 1 for normal, -1 for outliers
```

Local Outlier Factor (LOF)
--------------------------

- Novelty detection: set novelty=True
- Outlier detection: default (novelty=False)

Code example (novelty detection):
```python
from sklearn.neighbors import LocalOutlierFactor

# X_normal: training data containing normal observations
# X_new: new observations to evaluate

lof = LocalOutlierFactor(novelty=True, n_neighbors=20, contamination="auto")
lof.fit(X_normal)
scores = lof.score_samples(X_new)  # lower scores indicate more abnormal samples
pred = lof.predict(X_new)  # 1 for inliers, -1 for outliers
```

Elliptic Envelope
-----------------

- Model of a Gaussian distribution; supports novelty=True for novelty detection
- Useful when data are approximately Gaussian

Code example (novelty detection):
```python
from sklearn.covariance import EllipticEnvelope

# X_normal: training data containing normal observations
# X_new: new observations to evaluate

env = EllipticEnvelope(contamination=0.1, novelty=True)
env.fit(X_normal)
pred = env.predict(X_new)  # 1 for inliers, -1 for outliers
```

Data preparation and best practices
-----------------------------------

- Scaling: many detectors (e.g., OneClassSVM, LOF) benefit from feature scaling.
- Contamination: set the expected proportion of outliers via contamination (or nu for OneClassSVM) to help estimators calibrate thresholds.
- Cross-validation: use hold-out or time-series splits for novelty detection; standard cross-validation is less straightforward when the goal is out-of-distribution detection.
- Thresholds: default decision thresholds may not align with domain needs; tune thresholds based on validation data or domain requirements.
- Evaluation: use appropriate metrics (e.g., ROC AUC, Average Precision, precision/recall, F1) when ground-truth labels for anomalies are available.

Metrics and evaluation
----------------------

- When ground-truth anomaly labels exist, evaluate using ROC AUC or Average Precision to measure ranking quality.
- For binary predictions, compute precision, recall, F1, and confusion matrix.
- For novelty detection, ensure the evaluation split reflects unseen data from a normal distribution or the intended deployment scenario.

Tips and pitfalls
------------------

- Assumptions: each estimator encodes assumptions (e.g., Gaussianity for EllipticEnvelope, local-density for LOF). Validate these against your data.
- Hyperparameters: nu (OneClassSVM), contamination (IsolationForest/LOF), and n_neighbors (LOF) significantly affect results; use domain knowledge or a validation set to tune.
- Practical deployment: in novelty detection, continuously update or retrain your model as normal behavior evolves.
