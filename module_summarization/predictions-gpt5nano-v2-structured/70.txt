Kernel approximation
====================

Module overview
---------------
The :mod:`sklearn.kernel_approximation` module implements several approximate kernel feature maps based on Fourier transforms and Count Sketches. These techniques enable efficient, scalable approximations of kernel methods by transforming input data into a (typically much) lower-dimensional feature space that preserves pairwise similarities.

Key ideas
- Use random Fourier features and related sketching methods to approximate kernels fast.
- Tensor Sketch and Count Sketch enable compact representations of polynomial kernel maps.
- Compatible with scikit-learn’s transformer API for seamless integration in pipelines.

API reference
--------------

PolynomialCountSketch
~~~~~~~~~~~~~~~~~~~

Summary
^^^^^^^
Implements a Tensor Sketch-based approximation of the polynomial kernel

K(X, Y) = (gamma * <X, Y> + coef0)^degree

This class efficiently computes a Count Sketch of the outer product of a vector with itself
using Fast Fourier Transforms (FFT). See the User Guide section on polynomial kernel approximation
for details.

Inheritance
^^^^^^^^^^^
- ClassNamePrefixFeaturesOutMixin
- TransformerMixin
- BaseEstimator

Parameters
^^^^^^^^^^
- gamma : float, default=1.0
  Parameter of the polynomial kernel whose feature map will be approximated.

- degree : int, default=2
  Degree of the polynomial kernel whose feature map will be approximated.

- coef0 : float, default=0
  Constant term of the polynomial kernel whose feature map will be approximated.

- n_components : int, default=100
  Dimensionality of the output feature space. Typically, n_components should exceed the number
  of input features to achieve better approximation quality. Practical trade-offs around
  n_components ≈ 10 * n_features are common, but this depends on the dataset.

- random_state : int, RandomState instance, default=None
  Determines random number generation for indexHash and bitHash initialization. Use an int for reproducibility.

Attributes
^^^^^^^^^^
- indexHash_ : ndarray of shape (degree, n_features), dtype=int64
  Array of indices in [0, n_components) used to represent the 2-wise independent hash functions
  for Count Sketch computation.

- bitHash_ : ndarray of shape (degree, n_features), dtype=float32
  Random entries in {+1, -1} used to represent the 2-wise independent hash functions for
  Count Sketch computation.

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit. Defined only when X has string feature names.

Version added
^^^^^^^^^^^^^^
0.24

Notes
^^^^^
- Backed by FFT-based Tensor Sketch for efficient polynomial kernel approximation.
- Requires SciPy for FFT operations (scipy.fft or scipy.fftpack depending on SciPy version).

See Also
^^^^^^^^

- Nystroem: Approximate a kernel map using a subset of the training data.
- RBFSampler: Approximate a RBF kernel feature map using random Fourier features.
- SkewedChi2Sampler: Approximate feature map for the skewed chi-squared kernel.

Examples
--------

Basic usage with a small dataset:

.. code-block:: python

   from sklearn.kernel_approximation import PolynomialCountSketch
   from sklearn.linear_model import SGDClassifier

   # Sample data
   X = [[0, 0], [1, 1], [1, 0], [0, 1]]
   y = [0, 0, 1, 1]

   # Create the transformer
   ps = PolynomialCountSketch(degree=3, random_state=1)

   # Learn the sketch and transform
   X_features = ps.fit_transform(X)

   # Optional: train a classifier on the approximate features
   clf = SGDClassifier(max_iter=10, tol=1e-3)
   clf.fit(X_features, y)

   # Evaluate
   score = clf.score(X_features, y)
   print("Training score on approximate features:", score)

Notes on usage
^^^^^^^^^^^^^^
- The transform method maps input samples to a fixed-dimensional feature space of size n_components.
- The degree and coef0 parameters control the polynomial kernel being approximated.
- Random state controls the hash functions used in sketching, enabling reproducibility.
