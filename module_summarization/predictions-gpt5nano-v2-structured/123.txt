viewhelpers_formview_deleteview.rst

Overview
--------
Implements the preview + confirm + delete workflow for deleting an object in Django CRAdmin form views. The workflow shows a preview of the object to be deleted, a confirmation step, and then performs the deletion with a success message and a redirect.

Key classes
-----------

DeleteViewMixin
~~~~~~~~~~~~~~~~
Mixin that provides the core behavior for a delete confirmation view.

What it does
- Presents a preview of the object to be deleted (via get_object_preview).
- Displays a confirm message asking the user to confirm the deletion (via get_confirm_message).
- Handles the delete action and then shows a success message (via add_success_messages).
- Redirects to a sensible URL after deletion (via get_success_url).
- Populates template context with useful values (via add_delete_view_mixin_context_data).

Notable methods
- get_pagetitle: Returns the page title. Defaults to "Confirm delete".
- get_action_label: Returns the action label, default "Delete".
- get_delete_button_label: Label for the delete button (defaults to get_action_label).
- get_cancel_button_label: Label for the cancel button (defaults to "Cancel").
- get_object_preview: String representation of the object to be deleted.
- get_confirm_message: Message asking the user to confirm deletion.
- get_success_url: URL to redirect to after deletion (defaults to the app index).
- get_success_message: Message shown after successful deletion.
- add_success_messages: Adds the configured success message to Django messages.

Context data
- add_delete_view_mixin_context_data populates:
  - model_verbose_name
  - success_url
  - object_preview
  - pagetitle
  - confirm_message
  - delete_button_label
  - cancel_button_label

Usage notes
- This mixin is intended to be used with Django DeleteView in CRAdmin contexts.
- Translations are supported via gettext_lazy.

WithinRoleDeleteView
~~~~~~~~~~~~~~~~~~~~
Concrete view class for role-based CRAdmin contexts.

What it does
- Combines the DeleteViewMixin with CRAdmin’s role-based context data and base templates.
- Uses a dedicated template: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html.

Context data flow
- get_context_data integrates:
  - javascriptregistry component IDs
  - common CRAdmin view mixin data
  - delete view mixin context data (as described above)

Import path
- You should import these via:
  - from django_cradmin import viewhelpers
  - viewhelpers.formview.DeleteViewMixin
  - viewhelpers.formview.WithinRoleDeleteView

Template
- WithinRoleDeleteView uses:
  django_cradmin/viewhelpers/formview/within_role_delete_view.django.html

Customization
-------------
You can customize the workflow by overriding methods in your subclass, for example:

- Change labels
- Customize the preview or confirm message
- Add additional success messages
- Redirect to a different URL after deletion

Example
-------
Below is a minimal example showing how to create a delete view for a model in a role-based context and override the success URL and message.

```python
from django_cradmin import viewhelpers
from myapp.models import MyModel

class MyModelDeleteView(viewhelpers.formview.WithinRoleDeleteView):
    # Ensure the view operates on the desired object
    queryset = MyModel.objects.all()

    def get_success_url(self):
        # Redirect to a custom URL after deletion
        return '/myapp/mymodel/'

    def get_success_message(self, object_preview):
        # Customize the success message
        return gettext_lazy('Successfully deleted "%(what)s".') % {'what': object_preview}
```

Example usage in URLs (CRAdmin style may vary)
```python
from django.urls import path
from .views import MyModelDeleteView

urlpatterns = [
    path('mymodel/<int:pk>/delete/', MyModelDeleteView.as_view(), name='mymodel-delete'),
]
```

Context keys for templates
--------------------------
The template receives the following context keys for rendering the preview/confirm UI:

- object: The object to be deleted.
- object_preview: String representation used in the preview.
- pagetitle: Page title for the view.
- confirm_message: Message asking for delete confirmation.
- delete_button_label: Label for the delete button.
- cancel_button_label: Label for the cancel button.
- model_verbose_name: Verbose name of the model.
- success_url: URL to redirect to after deletion.

Notes
-----
- The default behavior is internationalized via gettext_lazy.
- The WithinRoleDeleteView template path is tailored for CRAdmin’s “within role” workflow and context. If you’re using a different template, you can override the template_name accordingly in your subclass.
