g-functions
=============

Auto-generated documentation for the g-functions lookup and equations.

Overview
--------

The g-functions module generates a single documentation string at import time. Its purpose is to be displayed in Sphinx as part of the Meijer G-function documentation. It builds a lookup table t using _create_lookup_table, then renders a reStructuredText (RST) structure describing how various formulas relate to their generated forms.

Key behaviors

- t is a dictionary: dict[tuple[type, ...], list[Any]] filled by _create_lookup_table(t).
- Entries are processed in a stable, human-friendly order using sorted(t.items(), key=default_sort_key).
- Elementary functions (where the group key is ()) are listed under "Elementary functions:".
- Other groups are described as "Functions involving <names>:", where <names> are extracted from the formulas’ atoms and rendered via latex.
- Each entry yields a math block of the form:
  .. math::
     <formula> = <generated or sum of scaled generators>, [optional condition]
  The equation is constructed as Eq(formula, g). If a condition is present, it is appended as ",\text{ if } <condition latex>".

- If gs is not a list, the generated form is represented as \text{generated}. Otherwise g is the sum of scaled generators Add(*[fac*f for (fac, f) in gs]).

- The final content is assigned to __doc__, so importing the module makes the documentation available for Sphinx.

Usage
-----

- This module is intended to be imported as part of the project’s documentation build. The generated content is stored in the module’s __doc__ attribute and consumed by Sphinx.

- After regenerating data (e.g., updating _create_lookup_table), re-build the docs to reflect changes.

Code example (illustrative)

.. code-block:: python

   # Importing the module triggers doc generation
   import your_package.g_functions as gf
   print(gf.__doc__[:200])  # Displays the generated reST content for documentation

Generated output structure (illustrative)
-----------------------------------------

- Elementary functions:

  .. math::
     <f₁(x)> = \text{generated}

- Functions involving <names>:

  .. math::
     <f₂(x)> = <generated expression>,\ \text{ if } <condition latex>

Notes
-----

- The content is auto-generated; the exact equations depend on the lookup table produced by _create_lookup_table.
- The documentation relies on LaTeX rendering via the Sphinx LaTeX/math support (latex(...) output).
