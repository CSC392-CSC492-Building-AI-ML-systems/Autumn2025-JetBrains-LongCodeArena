=========================
w_trace — Trace a trajectory back to its origin
=========================

Overview
--------
w_trace reconstructs the lineage of a trajectory segment back to its origin. Given the last segment identified by its iteration number and segment id, the command builds a Trace object that contains the sequence of segments along the path, including per-segment weights, timing information, and the final pcoord for each segment. It also fetches the initial state information needed to link the trajectory back to its origin.

Key concepts
- Trace: Represents the lineage of a trajectory segment back to its origin.
- Summary: A numpy structured array listing, for each segment in the lineage, the iteration, segment id, weight, wall time, CPU time, and the final pcoord of that segment.
- Initial state: The starting state from which the traced trajectory originates.
- Data manager: WESTPA's data access layer (HDF5-backed) used to retrieve segment data and initial states.

Core functionality (high level)
- Build the trace from the data manager starting from a given (n_iter, seg_id).
- Walk backwards through parent segments until reaching the initial state.
- Produce a summary table with one row per traced segment, including the final pcoord for that segment.
- Retrieve the corresponding initial state (basis and istate) for the traced trajectory.
- Provide utilities to fetch timepoint datasets along the traced trajectory.

Usage
-----
The CLI interface is designed to identify a trace by the last segment in a trajectory, specified by:
- n_iter: iteration number of the last segment
- seg_id: segment identifier within that iteration

Optionally, you can direct output and format.

Synopsis
--------
w_trace <n_iter> <seg_id> [options]

Arguments
- n_iter: Iteration number of the last segment to trace
- seg_id: Segment ID within iteration n_iter

Options
- -o, --outfile OUTFILE
  Write the trace summary to OUTFILE. If omitted, the summary is written to stdout.

- -f, --format FORMAT
  Output format. Choices include:
  - json
  - csv
  - numpy (default)

- -v, --verbose
  Increase verbosity of logging/output.

Notes
- w_trace relies on the WESTPA data manager (westpa.rc.get_data_manager()) to access HDF5 data.
- The trace summary contains the following fields:
  - n_iter: Iteration number of the segment
  - seg_id: Segment identifier
  - weight: Segment weight
  - walltime: Wall-clock time
  - cputime: CPU time
  - final_pcoord: Final pcoord of the segment (shape depends on pcoord dimensionality)

Examples
--------

Basic trace (prints to stdout)
.. code-block:: console

   $ w_trace 10 5

Trace with output to JSON file
.. code-block:: console

   $ w_trace 10 5 -o trace.json -f json

Print the number of segments in the trace and inspect the summary
.. code-block:: python

   from westpa.tools import Trace
   trace = Trace.from_data_manager(n_iter=10, seg_id=5)
   print("Number of segments in trace:", len(trace))
   print("Summary dtype:", trace.summary.dtype)
   print("First few rows:", trace.summary[:5])

Access timepoint data along the traced trajectory
.. code-block:: python

   trace = Trace.from_data_manager(n_iter=10, seg_id=5)
   data_trace, w = trace.trace_timepoint_dataset('pcoord')
   print("Data trace shape:", data_trace.shape)
   print("Weights shape:", w.shape)

API reference
-------------
Trace class
- from_data_manager(n_iter, seg_id, data_manager=None)
  Construct and return a trajectory trace whose last segment is identified by seg_id in iteration n_iter. This method traverses backward through parent segments, collects a summary, and fetches the initial state.

- __len__(self)
  Return the number of segments in the trace summary.

- __getitem__(self, sl)
  Access an element of the trace summary.

- __iter__(self)
  Iterate over the trace summary rows.

- get_segment_data_slice(self, datafile, dsname, n_iter, seg_id, slice_=None, index_data=None, iter_prec=None)
  Retrieve a slice of data for a given dataset from the appropriate iteration group or via an index mapping if index_data is supplied.

- trace_timepoint_dataset(self, dsname, slice_=None, auxfile=None, index_ds=None)
  Return a time-ordered trace along this trajectory for the given dataset. Accounts for overlapping values at segment boundaries and returns a tuple (data_trace, weight).

Implementation notes
- Trace data is reconstructed using the WESTPA data manager and, when necessary, supports legacy HDF5 data versions.
- The initial state can be a Basis State plus an InitialState representation; code paths accommodate both current and old data formats.
