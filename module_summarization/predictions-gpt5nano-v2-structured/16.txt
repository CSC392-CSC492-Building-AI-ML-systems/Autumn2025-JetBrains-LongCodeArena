Plothist command
================

Overview
--------
Plothist plots histograms stored in an HDF5 file produced by WESTPA’s Fast Histograms
(fasthist). It supports both 1-D and 2-D histograms and offers flexible options for
scaling, axis ranges, labeling, and post-processing before saving the plot. Plots can be
printed to an image file (by default a PDF) or suppressed entirely. Additionally, the
histogram data used to generate the plot can be saved back into an HDF5 file.

Key capabilities
- Plot 1-D or 2-D histograms from a WESTPA histogram file
- Plot scaling options: energy (default, inverted natural log), linear, or log10
- Set zero of energy for the energy scale (min, max, or a numeric value)
- Specify plotting range for axis or colorbar
- Overlay a contour on 2-D histograms
- Add a plot title
- Save the plot to an image file or suppress output by using an empty filename
- Optionally store plot data in an HDF5 file
- Invoke a user-supplied post-processing function just before saving the plot

Prerequisites
-------------
- WESTPA environment
- An HDF5 file containing histogram data produced by WESTPA’s fasthist
- Python dependencies: h5py, numpy, matplotlib

Invocation
----------
Plothist is a subcommand in WESTPA’s plotting suite. The basic invocation takes an input
HDF5 file and, optionally, a dimension specification to plot.

Basic usage:
.. code-block:: bash

   plothist input.h5

Parameters can be refined with additional options as described in the sections below.

Arguments
---------
Input options
- input: Path to the HDF5 file containing histogram data
- firstdim: Optional dimension specification to plot a specific dimension.
  Format: DIMENSION_SPEC, where DIMENSION_SPEC is an integer identifying the histogram
  dimension, optionally with bounds and a label. Example formats:
  - 0
  - 1:[LB,UB]:LABEL  (specifies lower/upper bounds and axis label for dimension 1)

Output options
- -o, --output, --plot-output, dest=plot_output:
  Path to save the plot. If set to an empty string, plotting is suppressed.
  The output format is inferred from the file extension (default: PDF).
- --hdf5-output HDF5_OUTPUT:
  Store the plotted data in the given HDF5 file.
- --plot-contour:
  For 2-D plots, overlay a contour on top of the heatmap.

Plot options
- --title TITLE:
  Include TITLE as the top-of-graph title.
- --linear, --energy, --zero-energy, --log10:
  Plot scale options. Mutually exclusive among --linear, --energy, and --log10.
  --energy uses an inverted natural-log scale (default). If using --zero-energy, specify
  an explicit zero point for the energy axis (numeric value, or "min"/"max").
- --range RANGE:
  Plot ranges as "LB,UB". For 1-D plots, this applies to the Y axis; for 2-D plots, to the color axis.
- --postprocess-function FUNCTION:
  Name a function (module.function) to be called just before saving the plot.
  The function will be invoked as postprocess(hist, midpoints, binbounds) and must modify the
  plot in place via the pyplot state.

Defaults
--------
- plotscale: energy (default)
- enerzero: min (unless overridden)
- If firstdim is not provided, a default dimension 0 with label "dimension 0" is plotted.

Output
------
- The primary output is a plotted image file (default hist.pdf) unless suppressed.
- Optionally, histogram plotting data can be written to an HDF5 file (--hdf5-output).

Examples
--------

1) Basic 1-D histogram plot with default energy scale
.. code-block:: bash

   plothist input.h5

2) Plot dimension 0 with energy scale and a custom title
.. code-block:: bash

   plothist input.h5 0 --title "Histogram of Dimension 0" --energy

3) Plot a 2-D histogram with a contour overlay and a specified range
.. code-block:: bash

   plothist input.h5 1 --plot-contour --range "-5,5" --energy -o heatmap.pdf

4) Save the plot data to an HDF5 file and suppress the plot output
.. code-block:: bash

   plothist input.h5 0 --hdf5-output plotted.h5 --plot-output ''

5) Use a post-processing function before saving the plot
.. code-block:: bash

   plothist input.h5 0 --postprocess-function mymodule.postprocess_hist

Notes
-----
- The --plot-output option determines the file format via extension (e.g., .pdf, .png).
- If the input dimension specification is omitted, the command defaults to plotting dimension 0 over its full range.
- The post-processing function must be accessible on Python's import path and implement the expected signature:
  postprocess(hist, midpoints, binbounds)

Implementation notes
--------------------
Plothist is implemented as a WESTSubcommand (PlotHistBase) and handles:
- Argument parsing and validation
- Opening and reading histogram data from the input HDF5 file
- Handling dimension selection, ranges, and labeling
- Plot generation via matplotlib, with optional contour overlay for 2-D plots
- Optional post-processing hook and saving output
- Suppression of divide-by-zero warnings in log space for numerical stability

This documentation reflects the public-facing usage and options; internal details are handled by the WESTPA plotting framework.
