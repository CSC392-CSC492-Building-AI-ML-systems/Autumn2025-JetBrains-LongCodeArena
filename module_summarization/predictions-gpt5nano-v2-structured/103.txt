# Code generation utilities (codegen.rst)

Overview
--------
This module provides an extensible framework for generating compilable code (C, C++, Fortran77/90, and Octave/Matlab) that evaluates diofant expressions. It is a work in progress. The documentation below reflects the current design and milestones as indicated in the module docstring.

Key ideas
- A clear separation between symbolic expressions and generated code.
- A generic Routine description that can be translated into multiple target languages by language-specific CodeGen backends.
- A compact, friendly API on top of the rigorous Routine/CodeGen workflow for common use cases.
- Design considerations that distinguish code generation from simple expression printing (ccode).

Why a dedicated code generator?
- Printing routines alone cannot produce complete compilable source with headers, libraries, and build structure.
- Code generation can handle input/output/argument semantics, multiple routines, and library dependencies.
- It enables optimization opportunities (e.g., common subexpression elimination and constant folding) and cross-language translation.

Public API
----------
The module exposes the following public API:

- Description of routines
  - Routine
  - DataType
  - default_datatypes
  - get_default_datatype
  - Argument
  - InputArgument
  - OutputArgument
  - Result
- Routines -> code
  - CodeGen
  - CCodeGen
  - FCodeGen
  - OctaveCodeGen
- Friendly functions
  - codegen
  - make_routine

Concepts
---------
- Routine: A generic description of a computational routine to be translated into code for one or more languages. It encodes:
  - name, arguments, results, local variables, and global variables
  - validation to ensure all symbols used by outputs are provided by inputs/local/global scope
  - automatic collection of all variables involved
- CodeGen: A base class for language-specific code generators. Descendant classes translate a Routine into compilable code for a target language.
- Language backends:
  - CCodeGen
  - FCodeGen
  - OctaveCodeGen
  Each backend knows about the target language constraints (e.g., return values, argument passing conventions, available types).
- Friendly API:
  - codegen: High-level convenience function to generate code from expressions or simple routines.
  - make_routine: Convenience helper to construct Routine instances without needing to wire every detail manually.

How to use (high level)
------------------------
1. Define a Routine that describes the computation you want to generate code for.
   - Specify inputs, outputs, and any local variables.
   - Ensure all output symbols are provided by inputs or local/global symbols.
2. Choose a target language by selecting an appropriate CodeGen subclass (e.g., CCodeGen, FCodeGen, OctaveCodeGen) or use the friendly API.
3. Generate the code and optionally headers, including any needed includes or library references.

Code examples (illustrative)
----------------------------
Note: The exact constructors for InputArgument, OutputArgument, and Result may vary slightly across versions. Adapt as needed to your installed API.

Example 1: Simple scalar function (C)
- Goal: generate C code for a scalar function y = x^2 + 1

```python
from diofant import symbols
from diofant.utilities.codegen import Routine, InputArgument, OutputArgument, Result
# Define the expression and variables
x = symbols('x')
# Define a simple routine: y = x**2 + 1
# This is illustrative; adjust exact constructor usage to your API
routine = Routine(
    name='f_square',
    arguments=[InputArgument('x')],
    results=[Result('y', x**2 + 1)],
    local_vars=[],
    global_vars=[]
)

# Generate C code (via a CodeGen backend)
from diofant.utilities.codegen import CCodeGen
codegen = CCodeGen(routine)
c_code = codegen.to_string()  # or an appropriate API to obtain the generated source
print(c_code)
```

Example 2: High-level friendly API (codegen)
- Goal: generate code for a single expression using the top-level function

```python
from diofant import symbols
from diofant.utilities.codegen import codegen
x = symbols('x')
expr = x**2 + 1
# codegen returns a list of (filename, code) pairs for the target language
[(c_header, header_code), (c_source, source_code)] = codegen((expr, 'f'), language='C', project='example')
print(source_code)
```

Example 3: Octave/Matlab target
- Goal: generate Octave-compatible code for a simple expression

```python
from diofant import symbols
from diofant.utilities.codegen import codegen
x = symbols('x')
expr = x**2 + 3
[(m_file, octave_code_str)] = codegen((expr, 'f'), language='Octave')
print(octave_code_str)
```
