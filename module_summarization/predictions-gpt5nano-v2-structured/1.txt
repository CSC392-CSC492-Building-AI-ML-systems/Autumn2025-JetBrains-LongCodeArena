Using Custom Attributes
=======================

Overview
--------

Custom attributes are user-defined fields that extend the standard data model of ftrack entities (for example, Tasks, Shots, or Assets). They allow teams to capture project-specific metadata (such as priorities, client codes, or review statuses) without modifying the core schema. Custom attributes are defined at the server side and surfaced through the Python API via the ftrack_api.attribute module.

Key concepts
- Definition: A custom attribute definition describes the name, data type, and display label of an attribute that can be attached to one or more entity types.
- Value: The per-entity value assigned to a custom attribute definition.
- Types: Custom attributes support a variety of data types (e.g. strings, integers, booleans, dates, etc.) and may be validated by the server or API layer.
- Scope: Attributes are defined for specific entity types and may be inherited or restricted according to project configuration.

How custom attributes work in the API
-------------------------------------

- Discover which custom attributes exist for a given entity type.
- Read the value of a custom attribute for a particular entity.
- Write or update the value of a custom attribute on an entity.
- Validate data types and handle missing attributes gracefully.
- Benefit from schema caching in the Session to improve performance when working with many attributes.

Usage guidelines
--------------

- Prefer using the ftrack_api.attribute helpers when reading or writing values to ensure proper validation and type handling.
- When possible, fetch attribute definitions first, then read or assign values using those definitions to avoid ambiguity.
- Be mindful of permissions: reading and writing custom attributes may require appropriate access rights on the server.
- If a custom attribute is missing or not defined for an entity type, handle the situation gracefully (e.g., fall back to defaults or log a warning).

Code examples
-------------

Note: The exact function names in your version of the API may differ. The following examples illustrate the typical workflow and concepts. Replace with the actual API calls available in your environment.

Reading a custom attribute value for an entity
Code (illustrative):
```python
# Pseudo-code illustrating the concept; replace with actual API calls
from ftrack_api import Session, attribute

session = Session(server_url, api_key, api_user)
entity = session.find_one('Task', id=123)  # Retrieve a specific entity

# Read value for a defined custom attribute named 'priority'
value = attribute.get_value(entity, 'priority')
print('Priority:', value)
```

Setting a custom attribute value on an entity
Code (illustrative):
```python
from ftrack_api import Session, attribute

session = Session(server_url, api_key, api_user)
entity = session.find_one('Task', id=123)

# Set value for a defined custom attribute named 'priority'
attribute.set_value(entity, 'priority', 5)

# Persist changes if your workflow requires an explicit save
entity.save()
```

Discovering attribute definitions for an entity type
Code (illustrative):
```python
from ftrack_api import Session, attribute

session = Session(server_url, api_key, api_user)

# Get definitions for the 'Task' entity type
definitions = attribute.get_definitions(session, entity_type='Task')
for d in definitions:
    print(d.name, d.type, d.label)
```

Performance considerations
--------------------------

- Use the Session's schema caching to reduce repeated loads of attribute definitions.
- When iterating over many entities with custom attributes, batch retrieval of definitions and, if available, batch updates to minimize round-trips to the server.

Error handling
--------------

- Missing attribute: If a requested custom attribute is not defined for the entity type, handle the absence gracefully (log a warning or skip).
- Type mismatch: Assigning a value of an incompatible type should raise an appropriate error; prefer validation via the attribute API.
- Permissions: Ensure your API key/user has permission to read or write the custom attributes in question.
