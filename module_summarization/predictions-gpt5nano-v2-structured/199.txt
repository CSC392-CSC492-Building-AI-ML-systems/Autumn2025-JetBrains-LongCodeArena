Neural network models (supervised)
=================================== 

Overview
--------

This module provides a collection of utilities to support supervised neural network modeling. It focuses on in-place operations to improve memory efficiency and performance during training, including activation functions, their derivatives for backpropagation, and loss functions used for regression and classification tasks.

Key features
------------

- In-place activation functions to minimize memory allocations
- Derivatives of common activations for efficient backpropagation
- Loss functions suitable for regression and classification
- Lightweight dictionaries for dynamic selection of activations, derivatives, and losses by name

Activation functions (in-place)
------------------------------

The following activation functions mutate their input array in place. Each function accepts a single argument X with shape (n_samples, n_features).

- Identity: leaves X unchanged
- Tanh: hyperbolic tangent
- Logistic: sigmoid
- ReLU: rectified linear unit
- Softmax: K-way softmax applied row-wise

API
- ACTIVATIONS: mapping from string name to in-place activation function

Examples
.. code-block:: python

    import numpy as np
    from neural_networks_supervised import ACTIVATIONS

    X = np.array([[0.5, -1.2], [2.0, 0.3]], dtype=float)

    # Apply ReLU in-place
    ACTIVATIONS["relu"](X)
    # X is now transformed in-place

    # Softmax in-place
    ACTIVATIONS["softmax"](X)
    # Each row of X sums to 1

Derivatives of activations (in-place)
-------------------------------------

Backpropagation requires the derivatives of activations with respect to the pre-activation input. The following derivatives are provided as in-place operations.

- Identity: derivative is no-op
- Tanh: derivative 1 - Z^2, where Z is the activation output
- Logistic: derivative Z * (1 - Z), where Z is the logistic output
- ReLU: derivative is 0 where Z == 0, otherwise unchanged

Usage
- DERIVATIVES: mapping from string name to in-place derivative function
- Each function accepts:
  - Z: activation output from the forward pass
  - delta: backpropagated error signal to be modified in place

Examples
.. code-block:: python

    import numpy as np
    from neural_networks_supervised import DERIVATIVES

    Z = np.array([[0.2, 0.8], [0.5, 0.1]])
    delta = np.random.randn(*Z.shape)

    # Backprop through ReLU
    DERIVATIVES["relu"](Z, delta)
    # delta is updated in-place

Loss functions
--------------

The module provides loss functions commonly used in supervised learning:

- Squared loss: mean squared error divided by 2 (for regression)
- Log loss: cross-entropy loss for multiclass classification (uses log probabilities)
- Binary log loss: cross-entropy for binary classification (suitable for multilabel contexts)

API
- Loss functions:
  - squared_loss(y_true, y_pred)
  - log_loss(y_true, y_prob)
  - binary_log_loss(y_true, y_prob)

- LOSS_FUNCTIONS: mapping from string name to loss function

Notes
- The log_loss function handles both binary and multiclass scenarios by promoting inputs to two-class representations when needed.
- All losses return a scalar representing the average loss over the samples.

Examples
.. code-block:: python

    import numpy as np
    from neural_networks_supervised import squared_loss, log_loss, binary_log_loss, LOSS_FUNCTIONS

    y_true = np.array([[1.0, 0.0], [0.0, 1.0]])
    y_pred = np.array([[0.9, 0.1], [0.2, 0.8]])

    # Regression-like squared loss
    loss_sq = squared_loss(y_true, y_pred)

    # Multiclass log loss
    loss_log = log_loss(y_true, y_pred)

    # Binary log loss (useful in multilabel contexts)
    y_prob = np.array([[0.7], [0.3]])
    y_b true = np.array([[1.0], [0.0]])
    loss_bin = binary_log_loss(y_true, y_prob)

    # Using the registry to compute a loss
    loss_from_registry = LOSS_FUNCTIONS["squared"](y_true, y_pred)

Usage notes
-----------

- In-place operations reduce memory usage during training, but users should ensure that in-place mutations do not affect concurrent computations.
- The activation and derivative mappings allow dynamic selection by name, enabling flexible neural network implementations where layer behavior is configured at runtime.
- Loss functions are designed to be generic across numpy array shapes and support common conventions for sample-wise averaging.

Typical workflow (illustrative)
- Forward pass: apply an activation in place to the pre-activation outputs
- Backward pass: apply the corresponding derivative in place to the backpropagated error
- Loss computation: compute the appropriate loss using the predicted outputs and ground truth

.. code-block:: python

    import numpy as np
    from neural_networks_supervised import ACTIVATIONS, DERIVATIVES, squared_loss

    # Example tensors
    Z = np.array([[0.2, -0.5], [1.0, 0.0]])
    delta = np.ones_like(Z)

    # Forward: apply activation in place
    ACTIVATIONS["tanh"](Z)

    # Backward: apply derivative in place
    DERIVATIVES["tanh"](Z, delta)

    # Compute loss
    y_true = np.array([[0.0, 1.0], [1.0, 0.0]])
    y_pred = np.array([[0.1, 0.9], [0.6, 0.4]])
    loss = squared_loss(y_true, y_pred)

Reference
---------

- The module provides the following public dictionaries and functions:
  - ACTIVATIONS: activation functions in-place
  - DERIVATIVES: activation derivatives in-place
  - LOSS_FUNCTIONS: mapping of loss function names to implementations
  - squared_loss, log_loss, binary_log_loss: concrete loss implementations
