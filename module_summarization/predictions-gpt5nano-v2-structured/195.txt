Learn Gaussian Mixture Models
=============================

Overview
--------
Gaussian Mixture Models (GMMs) provide a probabilistic approach to clustering. They assume that data are generated from a mixture of several Gaussian distributions, each with its own mean and covariance. Unlike hard clustering methods such as k-means, GMMs assign soft responsibilities to each component, enabling richer modeling of cluster shapes and overlaps.

Key ideas:
- Each data point is generated from one of K Gaussian components.
- The model estimates the mixture weights, component means, and covariances.
- Expectation-Maximization (EM) is used to iteratively refine these parameters until convergence.

Why use GMMs
------------
- Soft clustering: probability of belonging to each component rather than a single label.
- Flexible cluster shapes: full covariance matrices capture ellipsoidal clusters.
- Model selection with likelihood-based criteria and built-in regularization for numerical stability.

Core concepts
-------------
- Components (n_components): number of Gaussian distributions in the mixture.
- Covariances: covariance structure can be full, diagonal, or spherical (depending on covariance_type in some implementations).
- Responsibilities: posterior probabilities that a given sample comes from a particular component.
- EM algorithm: alternates between estimating responsibilities (E-step) and updating parameters (M-step) to maximize the likelihood.

Algorithm at a glance
---------------------
1. Initialization: start with initial responsibilities or parameters.
2. E-step: compute responsibilities given current parameters.
3. M-step: update mixture weights, means, and covariances using the responsibilities.
4. Convergence check: stop when log-likelihood or lower bound changes by less than tol, or after max_iter iterations.
5. Optionally repeat the initialization n_init times and keep the best run.

Parameters
----------
- n_components (int): number of mixture components (clusters).
- tol (float): convergence threshold for the change in log-likelihood/lower bound.
- reg_covar (float): value added to the diagonal of covariance matrices for numerical stability.
- max_iter (int): maximum number of EM iterations per initialization.
- n_init (int): number of initializations to perform; the best result is kept.
- init_params (str): method to initialize parameters. Options typically include:
  - "kmeans": initialize using KMeans clustering.
  - "random": random responsibilities.
  - "random_from_data": select random component centers from data.
  - "k-means++": use k-means++ style initialization.
- random_state (int or RandomState): seed or RNG for reproducibility.
- warm_start (bool): reuse the solution from previous calls to fit.
- verbose (int): level of verbosity during fitting.
- verbose_interval (int): iterations between verbose output.

Initialization strategies
-------------------------
- kmeans: assignments based on a single KMeans fit; creates hard labels and uses them as initial responsibilities.
- random: random soft assignments that sum to 1 for each sample.
- random_from_data: choose a subset of data points as centers and assign one-hot responsibilities accordingly.
- k-means++: select initial centers using a k-means++ style heuristic.

Training and inference
----------------------
- fit(X): estimate model parameters using EM. Executes multiple initializations (n_init) unless warm_start is True.
- fit_predict(X): fit the model and return the most probable component for each sample.
- predict(X): assign each sample to the component with the highest posterior probability.
- predict_proba(X): return posterior probabilities (responsibilities) for each sample.
- score_samples(X): compute log likelihood of each sample under the model.
- get_params / set_params: tune hyperparameters for the estimator.

Data requirements
-----------------
- X: array-like of shape (n_samples, n_features) with numeric features.
- n_samples must be at least 2 for meaningful estimation.

Practical tips
--------------
- Regularization: set reg_covar > 0 to prevent singular covariance matrices.
- Initialization: use kmeans or k-means++ to improve convergence stability and speed.
- Initialization randomness: control with random_state for reproducibility.
- Scaling: standardize features when they have differing scales to improve numerical stability.

Example
-------
A minimal example using scikit-learn's GaussianMixture:

.. code-block:: python

    from sklearn.mixture import GaussianMixture
    import numpy as np

    # Generate synthetic data from two Gaussian clusters
    rng = np.random.default_rng(seed=42)
    X = np.vstack([
        rng.normal(loc=[0, 0], scale=0.5, size=(150, 2)),
        rng.normal(loc=[3, 4], scale=0.8, size=(150, 2))
    ])

    # Create and fit the model
    gmm = GaussianMixture(
        n_components=2,
        covariance_type='full',
        init_params='kmeans',
        random_state=42,
        max_iter=100,
        tol=1e-3
    )
    gmm.fit(X)

    # Inference
    labels = gmm.predict(X)
    probs = gmm.predict_proba(X)
    log_likelihoods = gmm.score_samples(X)

    # Access learned parameters
    weights = gmm.weights_
    means = gmm.means_
    covariances = gmm.covariances_

Notes
-----
- The base class handles parameter validation, initialization, and the EM loop. In practice, you choose:
  - n_components to model the number of clusters.
  - init_params to control initialization.
  - max_iter and tol to control convergence behavior.
  - reg_covar for numerical stability.
- If convergence warnings arise, consider increasing max_iter or adjusting tol or init_params.

API reference (abridged)
------------------------
- fit(X, y=None): Fit the model to data X.
- fit_predict(X, y=None): Fit the model and return component labels for X.
- predict(X): Predict the most likely component for each sample.
- predict_proba(X): Predict posterior probabilities of each component for X.
- score_samples(X): Compute the log probability of X under the model.
- get_params(deep=True) / set_params(**params): Access or set hyperparameters.
