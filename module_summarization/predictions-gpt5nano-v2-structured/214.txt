# optimization.rst

Optimization algorithms for estimating model parameters

This module provides a flexible interface to fit model parameters by maximizing an objective function. It exposes three broad families of algorithms you can choose from, depending on the problem at hand and the properties of your objective (e.g., presence of gradients, desire for global search, etc.).

## Three algorithm families for parameter estimation

### 1) Newton-type optimization (local, derivative-based)

- Methods: `newton`, `ncg`
- Key idea: Use second-order information (Hessian) to perform Newton-Raphson-style updates. These methods can converge rapidly when near the optimum and the Hessian is informative.
- When to use:
  - You have a reasonably well-behaved objective with a meaningful Hessian.
  - You want fast convergence near the optimum.
- Typical usage:
  - Requires the objective function and its gradient, and optionally a Hessian.
- Supported methods:
  - `newton` — Newton-Raphson
  - `ncg` — Newton-Conjugate-Gradient
- Example usage:

```python
import numpy as np
from optimization import Optimizer

# Simple concave quadratic objective (maximize)
def objective(x, data):
    return -(x - data) ** 2

# Gradient of the objective
def gradient(x, data):
    return -2 * (x - data)

data = np.array([3.0])
start = np.array([0.0])

opt = Optimizer()
xopt, retvals, settings = opt._fit(
    objective, gradient, start,
    fargs=(data,), kwargs={},
    hessian=None, method='newton', maxiter=50,
    full_output=True, disp=True, retall=False
)
print("Optimal x:", xopt)
```

### 2) Derivative-free local optimization

- Methods: `nm` (Nelder–Mead), `powell`
- Key idea: Do not require gradient information. These methods explore the parameter space using only objective values, which can be advantageous when gradients are unavailable or the objective is noisy or jagged.
- When to use:
  - You cannot or do not want to compute gradients.
  - The objective is smooth enough but gradients are unreliable or expensive.
- Typical usage:
  - Provide a suitable objective function; gradient may be ignored by the solver.
- Supported methods:
  - `nm` — Nelder–Mead
  - `powell` — Powell’s method
- Example usage (same objective as above; gradient is unused by these solvers):

```python
opt = Optimizer()

# For derivative-free methods, gradient can be a dummy function or unused
def gradient_dummy(x, data):
    return np.zeros_like(x)

xopt, retvals, settings = opt._fit(
    objective, gradient_dummy, start,
    fargs=(data,), kwargs={},
    hessian=None, method='nm', maxiter=100,
    full_output=True
)
print("Optimal x (nm):", xopt)
```

### 3) Global optimization and flexible solvers

- Methods: `basinhopping`, `minimize`
- Key idea:
  - `basinhopping`: performs a global search by combining local minimization with random perturbations, helping escape local minima.
  - `minimize`: a generic wrapper around SciPy’s `optimize.minimize`, enabling a wide range of internal solvers (e.g., BFGS, L-BFGS-B, CG, etc.) with customizable options.
- When to use:
  - You suspect multiple local minima; you want a robust global search (basinhopping).
  - You want to leverage a specific SciPy optimizer via a single interface (minimize wrapper).
- Notes:
  - The basinhopping solver ignores certain explicit arguments (maxiter, retall, full_output) as documented by the underlying implementation.
- Example usage:

```python
# Global search with basinhopping
opt = Optimizer()
xopt, retvals, settings = opt._fit(
    objective, gradient, start,
    fargs=(data,), kwargs={},
    hessian=None, method='basinhopping', maxiter=200,
    full_output=True
)

print("Optimal x (basinhopping):", xopt)

# Flexible usage via the minimize wrapper (SciPy minimize)
# You may pass solver-specific kwargs through `kwargs` and/or `fargs`
xopt_min, retvals_min, settings_min = opt._fit(
    objective, gradient, start,
    fargs=(data,), kwargs={},
    hessian=None, method='minimize', maxiter=300,
    full_output=True
)
print("Optimal x (minimize):", xopt_min)
```

## API overview

- _fit signature:
  - objective: callable
    - The objective function to maximize (e.g., log-likelihood). It must return a scalar.
  - gradient: callable
    - The gradient of the objective with respect to parameters. Used by derivative-based solvers.
  - start_params: array_like
    - Initial guess for the parameters.
  - fargs: tuple
    - Extra arguments passed to the objective and gradient: objective(x, *fargs), gradient(x, *fargs).
  - kwargs: dict
    - Additional keyword arguments passed to the objective/gradient as needed.
  - hessian: callable or None
    - Optional Hessian or Hessian-vector product for Newton-type methods.
  - method: str
    - One of: `newton`, `ncg`, `nm`, `powell`, `bfgs`, `cg`, `basinhopping`, `minimize`.
  - maxiter: int
    - Maximum number of iterations (subject to solver).
  - full_output: bool
    - If True, return rich solver output in retrievable structure.
  - disp: bool
    - If True, print convergence messages.
  - callback: callable or None
    - Called after each iteration.
  - retall: bool
    - If True, return the full history of iterates.
- Returns:
  - xopt: array
    - The optimized parameters.
  - retvals: dict or None
    - Solver-specific information (only if full_output is True).
  - optim_settings: dict
    - The parameters passed to the solver.

## Notes and tips

- The three families cover the common scenarios you’ll encounter in model fitting:
  - Fast local convergence with Newton-type methods when the Hessian is informative.
  - Robust local search without gradients using Nelder–Mead or Powell.
  - Global search and flexible solver choices to mitigate local minima and leverage SciPy optimizers.
- For basinhopping, plan to pair a local minimizer with random perturbations; tune the number of iterations, temperature, and stepsize to your problem scale.
- The minimize wrapper allows you to exploit a broader set of SciPy optimization algorithms with the same interface.

## References

- The optimizer exposes a general interface that decouples the fitting routine from a specific model class, enabling reuse across different models and objective functions.
