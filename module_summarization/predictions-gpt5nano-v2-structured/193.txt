The Metadata Routing API
========================

Overview
--------

The Metadata Routing API provides utilities used by meta-estimators (such as multi-output, chained, or ensemble-style estimators) to distribute and manage metadata (parameters, method calls, and associated data) across multiple underlying estimators. It enables consistent routing of inputs and configuration so that each sub-estimator receives the appropriate subset of parameters and calls.

Public API
----------

The API consists of a small set of classes and helpers intended to coordinate how metadata and method calls are forwarded to constituent estimators.

- MetadataRouter
  - Purpose: Central object that defines and holds routing rules for how metadata should be distributed to sub-estimators.
  - Typical usage: create a router, configure routing rules, and apply them to a payload of parameters or method calls.
  - Key idea: separate routing logic from estimator logic to keep meta-estimators clean and extensible.

- MethodMapping
  - Purpose: Lightweight data structure describing how a method (or set of methods) should be routed to target estimators.
  - Typical usage: construct mappings that express which sub-estimators should handle which methods or parameters.
  - Role: used by MetadataRouter to interpret and apply the routing rules.

- process_routing
  - Purpose: Apply routing rules to a given payload (e.g., a dictionary of parameters or a set of method calls) and produce a routing-aware representation suitable for downstream sub-estimators.
  - Typical usage: take an input payload and a routing configuration, and obtain the routed structure that can be consumed by the constituent estimators.

- _raise_for_params
  - Purpose: Internal helper to validate that provided parameters conform to the routing configuration.
  - Scope: Intended for internal use to ensure consistency and to catch misconfigurations early.

- _routing_enabled
  - Purpose: Internal helper to check whether routing is enabled for a given object.
  - Scope: Internal use by meta-estimators and routing utilities to toggle or guard routing behavior.

Notes
-----

- The Metadata Routing API is primarily leveraged by scikit-learn's multi-output and chained estimators to ensure that parameters and method calls are correctly distributed among sub-estimators.
- The internal helpers (_raise_for_params and _routing_enabled) are intended for internal use; public users interact mainly with MetadataRouter, MethodMapping, and process_routing.
- The API surface may evolve; refer to the source for the most up-to-date usage patterns.

Examples
--------

Basic workflow: define routing rules and apply them to a parameter payload.

.. code-block:: python

   from sklearn.utils.metadata_routing import MetadataRouter, MethodMapping, process_routing

   # Define routing rules
   router = MetadataRouter()
   # Example: route the "fit" method to all underlying estimators
   router.add_mapping(MethodMapping("fit", targets=["all_estimators"]))

   # Example parameter payload to be routed
   params = {
       "estimator_0": {"X": X, "y": y},
       "estimator_1": {"X": X2, "y": y2},
   }

   # Apply routing rules
   routed_params = process_routing(params, router)

   # The routed_params structure is now prepared for consumption by sub-estimators

Advanced usage: integrating routing into a custom meta-estimator

.. code-block:: python

   from sklearn.utils.metadata_routing import MetadataRouter, MethodMapping, process_routing
   from sklearn.base import BaseEstimator

   class MyMultiEstimator(BaseEstimator):
       def __init__(self, estimator, routing=None):
           self.estimator = estimator
           self._routing = routing or MetadataRouter()

       def fit(self, X, y):
           payload = {"estimator": self.estimator, "X": X, "y": y}
           routed = process_routing(payload, self._routing)
