# classfiles.rst

GILDAS CLASS file reader: module reference
==========================================

Overview
--------

This module provides the low-level building blocks used by the GILDAS CLASS file reader to interpret CLASS headers and map them into human-friendly structures suitable for conversion into a pyspeckit ObsBlock. It includes type mappings, header layout specifications, and utility helpers that support parsing CLASS files of various formats and versions.

Public API
----------

The module exposes the following public names:

- print_timing
  - A decorator that prints the execution time of the decorated function.
  - Usage: wrap a long-running parsing function to log its duration.

- ensure_bytes(string)
  - Ensures the input string is returned as bytes (UTF-8 in Python 3; string-like in Python 2).
  - Example:
    - classfiles.ensure_bytes('1A  ') -> b'1A  '

- filetype_dict
  - Mapping from CLASS file type codes to human-friendly labels.
  - Keys include both byte strings (to be compatible with Python 3) and regular strings.
  - Example:
    - filetype_dict[b'9A  '] -> 'Single_IEEE'
    - filetype_dict['9A  '] -> 'Single_IEEE'

- fileversion_dict
  - Mapping from CLASS file version codes to version strings.
  - Example:
    - fileversion_dict[b'2A  '] -> 'v2'

- record_lengths
  - Fixed header lengths by CLASS version.
  - Example:
    - record_lengths['1A'] -> 512
    - record_lengths['2A'] -> 4096

- header_id_numbers
  - Mapping from numeric header IDs to human-readable section names.
  - Example:
    - header_id_numbers[0] -> 'USER CODE'
    - header_id_numbers[-2] -> 'GENERAL'

- header_id_lengths
  - Expected lengths for known header blocks (by header ID).
  - Example:
    - header_id_lengths[-3] -> 17
    - header_id_lengths[-5] -> None (variable length)

- filedescv2_nw1
  - A constant used by the v2 description (value: 14).
  - Context: relates to the CLASS v2 descriptor layout.

- keys_lengths
  - A specification of the layout for header sections, expressed as a dictionary.
  - Each top-level key represents a header section name, with a list of field specifications.
  - Each field spec is a tuple: (field_name, length, dtype)
    - field_name: string key used in parsed output
    - length: number of words or items (dependent on the header section)
    - dtype: numpy/byte-string dtype used to interpret the field
  - Example sections:
    - 'unknown'
      - Includes: NUM, VER, TELES, DOBS, DRED, TYPEC, KIND, QUAL, SCAN, SUBSCAN
    - 'COMMENT'
      - Includes: LTEXT (length of comment), CTEXT (comment text)
    - 'GENERAL'
      - Includes: UT, ST, AZ, EL, TAU, TSYS, TIME, etc.
  - This specification is intended to drive parsing of header blocks and to map raw binary data into structured, named fields.

Notes on usage
--------------

- The module is built to be Py2/Py3 compatible via the six library.
- It gracefully handles environments without an interactive progress bar by falling back to a no-op ProgressBar.
- The dictionaries are designed to be robust against both bytes and string keys, enabling seamless use across Python versions.

Example usage
-------------

- Look up a friendly file type from a raw 2-character code:

```python
import classfiles

raw_code = b'9A  '
friendly = classfiles.filetype_dict.get(classfiles.ensure_bytes(raw_code), 'Unknown')
print(friendly)  # Outputs: Single_IEEE
```

- Look up a file version:

```python
import classfiles

ver_code = b'2A  '
version = classfiles.fileversion_dict.get(classfiles.ensure_bytes(ver_code), 'Unknown')
print(version)  # Outputs: v2
```

- Inspect a header field layout (conceptual example):

```python
import classfiles

layout = classfiles.keys_lengths.get('GENERAL', [])
for field, length, dtype in layout:
    print(field, length, dtype)
```

Design and internal behavior
----------------------------

- Purpose: Provide the constants and header specifications that underpin the CLASS file reader’s parsing logic.
- How it works: The actual parsing logic (not shown here) relies on the dictionaries and the keys_lengths specification to interpret binary header blocks, translate numeric codes to meaningful labels, and assemble data into a pyspeckit ObsBlock.
- Extensibility: New CLASS formats or versions can be supported by updating filetype_dict, fileversion_dict, and record_lengths as needed.

See Also
--------

- pyspeckit.spectrum.ObsBlock: The target structure for data read from CLASS files.
- GILDAS CLASS file documentation: http://iram.fr/IRAMFR/GILDAS/doc/html/class-html/node58.html

Notes
-----

- The module includes a large in-file SPECIFICATION block describing the header layout for different sections (unknown, COMMENT, GENERAL). This is used to guide the parsing process and should be kept in sync with the actual CLASS file formats.
