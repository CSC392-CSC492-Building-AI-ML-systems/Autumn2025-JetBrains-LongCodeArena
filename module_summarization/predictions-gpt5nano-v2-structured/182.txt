Developing scikit-learn estimators
===================================

This guide explains how to develop and integrate new estimators in scikit-learn, with a focus on meta-estimators such as multi-output estimators. It covers design principles, common patterns used in the codebase, and a minimal example to illustrate implementation details.

Overview
--------

Estimators in scikit-learn implement a consistent interface (fit, predict, transform, etc.) and are designed to be composable. Meta-estimators extend single-output estimators to additional capabilities, often by wrapping a base estimator provided at construction time. Examples include multi-output regressors/classifiers and chain models.

Key ideas
- Consistent API: __init__ parameters should be simple, with no heavy logic in the constructor. All configuration should be captured in parameters so get_params/set_params work correctly.
- Cloning: When fitting sub-estimators, clone the base estimator to avoid side effects and enable proper parameter tracking and cross-validation.
- Param validation: Use simple, well-defined validation to ensure sub-estimators provide required methods (e.g., fit, predict).
- Reproducibility: Propagate random_state or other stochastic controls when appropriate, ensuring deterministic behavior when requested.
- Testing: Write unit tests that cover parameter handling, fit/predict on multi-output targets, and compatibility with scikit-learn utilities (e.g., cross_val_predict, clone).

Estimator interfaces and utilities
----------------------------------

Meta-estimators commonly rely on a base estimator and may expose their own hyperparameters. The following patterns appear in multi-output meta-estimators and related utilities:

- Base interfaces:
  - BaseEstimator: provides get_params/set_params
  - MetaEstimatorMixin: helps with parameter management for meta-estimators
  - RegressorMixin / ClassifierMixin: signal the estimator type
- Sub-estimator handling:
  - clone: create a fresh copy of the base estimator for each sub-task
  - fit/predict: delegate to sub-estimators and aggregate results
- Helper utilities (example from scikit-learn’s multi-output module):
  - _fit_estimator(estimator, X, y, sample_weight=None, **fit_params): clone and fit a sub-estimator
  - _partial_fit_estimator(...): support incremental learning when possible
  - _available_if_estimator_has(attr): enable/disable parameters based on sub-estimator capabilities
- Validation helpers:
  - HasMethods, StrOptions: basic parameter validation
  - check_random_state: manage randomness

Example: Multi-output estimator pattern
---------------------------------------

A minimal multi-output estimator wraps a base estimator and fits a separate sub-estimator for each output dimension. The pattern closely follows the approach used by scikit-learn’s MultiOutputRegressor/Classifier, and demonstrates key concepts such as cloning sub-estimators and aggregating predictions.

Code example: Minimal multi-output regressor
--------------------------------------------

.. code-block:: python

    from sklearn.base import BaseEstimator, RegressorMixin, clone
    import numpy as np

    class MinimalMultiOutputRegressor(BaseEstimator, RegressorMixin):
        def __init__(self, estimator):
            # estimator must support fit(X, y) and predict(X)
            self.estimator = estimator

        def fit(self, X, y):
            y = np.asarray(y)
            if y.ndim == 1:
                y = y[:, None]
            self.estimators_ = []
            for i in range(y.shape[1]):
                est = clone(self.estimator)
                est.fit(X, y[:, i])
                self.estimators_.append(est)
            return self

        def predict(self, X):
            preds = [est.predict(X) for est in self.estimators_]
            return np.column_stack(preds)

This skeleton demonstrates:
- Accepting a single base estimator and wrapping it to handle multiple outputs.
- Cloning the base estimator for each target dimension to avoid shared state.
- Aggregating predictions across sub-estimators into a 2-D array.

Notes
- In real scikit-learn meta-estimators, helpers like _fit_estimator and _partial_fit_estimator are used to standardize sub-estimator fitting and support additional features (e.g., partial_fit, sample weighting).
- Handling 1-D vs 2-D targets is important to support both single and multi-output tasks gracefully.
- Ensure get_params/set_params expose sub-estimator parameters (often via the parent estimator’s parameter handling) so users can tune hyperparameters with grid/search tooling.

Implementation guidelines
-------------------------

- __init__ should only assign parameters and perform no heavy computation.
- Validate the presence of required methods on the sub-estimator (e.g., fit, predict) and raise informative errors if missing.
- Use clone(self.estimator) when creating sub-estimators to ensure proper parameter management and avoid shared state across targets.
- When supporting incremental learning, implement partial_fit where feasible and leverage _partial_fit_estimator for consistency.
- For features relying on routing or metadata (e.g., routing decisions in meta-estimators), integrate with the project’s routing utilities (e.g., MetadataRouter) as appropriate.
- Provide clear __all__ exposure for public estimators and ensure docstrings describe the estimator’s purpose, parameters, and behavior.

Usage patterns
--------------

- Multi-output regression
  - Instantiate with a base estimator, then fit(X, Y) where Y has shape (n_samples, n_outputs).
  - Predict returns an array of shape (n_samples, n_outputs).

- Classifier chains and other chain-based meta-estimators
  - Useful when a sequence of models depends on the predictions of previous models.
  - Start with a base estimator and configure the chain length or ordering as needed.

Best practices
--------------

- Write unit tests that cover:
  - Parameter handling and cloning behavior
  - Correct handling of 1-D and 2-D targets
  - Interaction with cross-validation and scoring utilities
- Keep dependency relationships clear and minimal in the constructor to preserve pickling integrity and parameter introspection.
- Document the expected input shapes and output formats in the estimator’s docstring.

Example usage
-------------

.. code-block:: python

    from sklearn.datasets import make_regression
    from sklearn.linear_model import LinearRegression
    from sklearn.metrics import mean_squared_error
    import numpy as np

    # Sample data
    X, y = make_regression(n_samples=100, n_features=20, n_targets=3, random_state=42)

    # Base estimator
    base = LinearRegression()

    # Multi-output estimator (manual pattern)
    from sklearn.base import clone

    class SimpleMultiOutputRegressor(BaseEstimator, RegressorMixin):
        def __init__(self, estimator):
            self.estimator = estimator

        def fit(self, X, y):
            y = np.asarray(y)
            if y.ndim == 1:
                y = y[:, None]
            self.estimators_ = []
            for i in range(y.shape[1]):
                est = clone(self.estimator)
                est.fit(X, y[:, i])
                self.estimators_.append(est)
            return self

        def predict(self, X):
            preds = [est.predict(X) for est in self.estimators_]
            return np.column_stack(preds)

    mreg = SimpleMultiOutputRegressor(base)
    mreg.fit(X, y)
    y_pred = mreg.predict(X)
    print("MSE:", mean_squared_error(y, y_pred))

Conclusion
----------

Developing estimators in scikit-learn emphasizes clean interfaces, predictable behavior, and composability. Meta-estimators like multi-output wrappers follow a consistent pattern: accept a base estimator, clone sub-estimators for each target, delegate fitting and prediction, and expose parameters through the standard get_params/set_params mechanisms. Use the patterns and utilities described here to implement robust, testable estimators that integrate smoothly with the scikit-learn ecosystem.
