Utilities for Searching ASDF Trees
===================================

This document describes the AsdfSearchResult API, the primary entry point for performing fluent searches over ASDF trees. The class enables chaining filters to narrow down leaf nodes, rendering options for displaying results, and in-place replacement of matched values.

Public API
----------

AsdfSearchResult
~~~~~~~~~~~~~~~~

Overview
^^^^^^^^
Represents the result of a search operation on an ASDF tree. Instances can be refined by subsequent calls to search(), and rendered with format().

Constructor
***********
__init__(identifiers, node, filters=[], parent_node=None, max_rows=DEFAULT_MAX_ROWS, max_cols=DEFAULT_MAX_COLS, show_values=DEFAULT_SHOW_VALUES)

Parameters
- identifiers: Sequence of identifiers (keys or indices) that locate the current node within the tree.
- node: The root node for the current search context.
- filters: A list of callables applied to each candidate node to determine if it should be kept.
- parent_node: The parent container of the current node (used for replacement).
- max_rows: Maximum number of rendering rows.
- max_cols: Maximum number of columns per rendered line.
- show_values: Whether primitive values should be shown in the rendering.

Returns
- A new AsdfSearchResult instance (the constructor itself does not render).

format
^^^^^^
Change formatting parameters of the rendered tree.

Parameters
- max_rows: int, tuple, None, or NotSet. Controls the number of lines to display. None means show all lines.
- max_cols: int or None or NotSet. Controls the maximum line length. None means no length limit.
- show_values: bool or NotSet. If False, primitive values are not displayed.

Returns
- AsdfSearchResult: a reconfigured search result (new instance with updated formatting).

Behavior notes
- If a parameter is NotSet, the existing value is retained.
- This method does not modify the underlying tree; it only affects how results are rendered.

_search_
^^^^^^^^
Refine the current search with additional constraints.

Parameters
- key: NotSet, str, or any other object. 
  - NotSet: no constraint on the key/index.
  - str: treat as a regular expression to match keys or indices.
  - other object: require equality with the node’s key or index.
- type: NotSet, str, typing.Pattern, or builtins.type.
  - NotSet: no constraint on type.
  - str: treat as a regular expression over the fully-qualified type name.
  - typing.Pattern: use as a compiled regex against the fully-qualified type name.
  - builtins.type: require the node to be an instance of this type.
- value: NotSet, str, or any other object.
  - NotSet: no constraint on the value.
  - str: treat as a regular expression against the node’s string representation.
  - other object: require equality with the node’s value.
- filter: callable (optional). A predicate taking (node) or (node, identifier) that returns True to keep the node.

Returns
- AsdfSearchResult: a new search result updated with the additional filtering constraints.

Notes
- The type argument supports multiple forms (NotSet, str, compiled regex, or a Python type).
- For value as a Pattern, containers are typically excluded to avoid surprising matches inside complex structures.
- The result of a search() call is chainable, preserving previous filters and formatting settings.

replace
^^^^^^^
Assign a new value in place to all leaf nodes that match the accumulated filters of the current search result.

Parameters
- value: The value to assign to each matched leaf node.

Behavior
- Gathers all (identifiers, parent) pairs for which all active filters pass.
- Replaces each corresponding leaf node in its parent with the provided value.

node
^^^^
Retrieve the leaf node of a tree that has a single search result.

Returns
- The leaf node for the current search result (documented behavior corresponds to a single-result scenario).

Notes
- The specific behavior when multiple results exist is not shown in the excerpt; designed primarily for single-leaf scenarios.

Usage Overview
--------------

- Start from a root ASDF file/search operation that yields an AsdfSearchResult.
- Narrow results with chained search() calls, using key, type, value, or a custom filter.
- Inspect or render the results with format(), optionally adjusting max_rows, max_cols, and show_values.
- If appropriate, replace matched leaf values in place with replace(value).

Key Concepts and Semantics
--------------------------

- NotSet is used to indicate “no constraint” for format and search parameters.
- key, type, and value parameters support flexible matching:
  - Strings are treated as regular expressions against the appropriate textual representation.
  - Patterns (compiled regex) are applied directly.
  - For type, builtins.type allows strict isinstance checks.
  - For value, a Pattern is not applied to containers’ string representations; container objects are skipped for pattern-based value matching.
- _get_fully_qualified_type(value) helps implement type matching by producing a string like "builtins.int" or "package.module.Type".
- _safe_equals(a, b) guards against exceptions during equality checks, returning False if comparison fails.
- _maybe_compile_pattern(query) converts strings to regex patterns, while leaving patterns and non-string objects as-is.
- This API is designed to be fluent: each call to search() returns a new AsdfSearchResult with the cumulative filters.

Example
-------

The following illustrative examples show typical usage. Adapt to your actual AsdfFile/ASDF usage context as needed.

.. code-block:: python

    # Start with a search on an AsdfFile instance (asdf_file)
    # 1) Find all leaves whose type is builtins.int
    result = asdf_file.search(type="builtins.int")

    # 2) Further narrow by key and by value  (if appropriate for your tree)
    result = result.search(key="count", value=42)

    # 3) Disable primitive value rendering and limit to 20 rows
    rendered = result.format(max_rows=20, show_values=False)

    # 4) Apply an additional filter (optional)
    def is_target_node(node, identifier):
        return getattr(node, "name", None) == "target"

    filtered = rendered.search(filter=is_target_node)

    # 5) Replace matched leaf values with a new value
    filtered.replace(0)
