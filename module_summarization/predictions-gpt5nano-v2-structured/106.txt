Hyperexpand: Expand Hypergeometric Functions into Named Special Functions
=======================================================================

Overview
--------
This module is designed to automatically generate the documentation content for
Hypergeometric and Meijer G function expansions. When imported, it builds a
LaTeX-formatted representation of known formulae and injects that content into
the module's docstring. The generated documentation is then rendered by Sphinx.

Key idea: a knowledge base of formulae expresses hypergeometric forms in terms of
named special functions. The module assembles these relations and formats them as
math blocks for inclusion in the documentation.

How the dynamic documentation is generated
-----------------------------------------
At import time, hyperexpand.rst constructs a documentation string by:

- Creating a FormulaCollection instance, which holds a collection of formulae
- Iterating over the collection’s formulae
- For each formula, building an equation that relates the hypergeometric form to its
  closed-form (rewritten in a canonical form)
- Rendering the equation to LaTeX and wrapping it in a Sphinx math directive

Example from the code (conceptual):
.. code-block:: python

   from hyperexpand import FormulaCollection
   from ..core import Eq
   from ..functions import hyper
   from ..printing import latex

   c = FormulaCollection()
   doc = ''
   for f in c.formulae:
       obj = Eq(hyper(f.func.ap, f.func.bq, f.z),
                f.closed_form.rewrite('nonrepsmall'))
       doc += f'.. math::\n  {latex(obj)}\n'

   __doc__ = doc

Usage
-----
- Purpose: Provide rich, automatically generated documentation content for
  Hypergeometric and Meijer G expansions without manually authoring every
  formula block.
- How to use in Sphinx: Include hyperexpand.rst in the documentation build. The
  content is produced at import time and becomes part of the generated docs.
- Output: A sequence of Sphinx math blocks (.. math:: ...) containing the LaTeX
  representations of the hypergeometric expansions.

API and key components
----------------------
- FormulaCollection
  - A container for a set of formulae that describe how hypergeometric
    expressions expand into named special functions.
  - Accessed as c.formulae in the module’s runtime to generate documentation blocks.

- add_formulae(formulae)
  - Populates a given list with Formula objects representing basic hypergeometric
    expansions (e.g., 0F0, 1F0, 2F1, and more complex cases).
  - Demonstrates how the knowledge base is constructed from simple to more
    involved formulas.

- _mod1(x)
  - Internal helper used to compute modular buckets (Mod(x, 1) handling) for
    sorting and organizing terms during expansion logic.

- Hyper_Function, Formula, HyperRep_*, and related constructs
  - Part of the internal machinery that represents hypergeometric structures and
    their target (closed-form) representations.
  - The code references several HyperRep_* helpers (e.g., HyperRep_asin1, HyperRep_power1)
    and a general HyperRep/Hyper function ecosystem that underpins the
    expansion rules.

- 0F0, 1F0, 2F1, and related expansions
  - The module includes a curated set of known expansions that serve as the
    foundation of the lookup-based approach to rewriting hypergeometric forms in
    terms of standard special functions.

Notes on the expansion algorithm
-------------------------------
- The expansion leverages a lookup-table approach, combining:
  - Hypergeometric identities
  - Meijer G-function properties
  - Parameter shifts (via A_J, B_J, C_J, D_J style operators) to move between
    equivalent representations
- Order reduction rules help simplify higher-order G-function expressions when
  certain parameter differences are non-negative integers.
- The implementation emphasizes explicit matching and instantiation of formulas
  rather than relying solely on automatic algebraic manipulation, due to the
  complexity of reachable vs. non-reachable index quadruples.
