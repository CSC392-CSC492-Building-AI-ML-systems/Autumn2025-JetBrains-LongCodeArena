# Data component in an image HDU

This doc describes the Data component of an ImageHDU in FITS files. The ImageHDU stores the actual pixel values of an image (or data cube) in its `data` attribute. The data is exposed as a NumPy array, enabling convenient indexing, slicing, and in-place modification.

## What is stored in the Data component

- The pixel values of the image are stored as a NumPy array accessible via `hdu.data`.
- The array can be 2D (standard image), or higher-dimensional (e.g. a data cube with shape `(nz, ny, nx)`).
- The dtype of the data (e.g., `np.float32`, `np.int16`) is defined by the FITS data type and may be described in the header (often via `BUNIT` for physical units).

## Accessing the image data

- Access the data as a NumPy array:
  - `hdu = fits.open('image.fits')[0]`
  - `arr = hdu.data`
  - `arr.shape` gives the data shape (e.g., `(ny, nx)` for a 2D image).
  - `arr.dtype` gives the data type.

- Coordinate conventions:
  - For 2D images, Python indexing is `(y, x)`, i.e., `arr[y, x]` corresponds to the FITS coordinates `(x+1, y+1)`.
  - For 3D data cubes, indexing is `(z, y, x)`.

- Common operations:
  - Slicing: `subimage = hdu.data[100:200, 50:150]`
  - Access a single pixel: `val = hdu.data[y, x]`

## Inspecting metadata related to the data

- Pixel data type and shape:
  - `hdu.data.dtype`
  - `hdu.data.shape`

- Units (if provided in the header):
  - `unit = hdu.header.get('BUNIT', None)`
  - Many FITS images store physical units in the `BUNIT` keyword (e.g., `'Jy/beam'`, `'K'`, etc.).

- Header alignment:
  - The data array is associated with the header containing the WCS, axes descriptions, and optional calibration information.
  - Keep the header in sync with any changes to the data if you modify values that affect metadata.

## Modifying data and saving changes

- In-place modification (within an open file):
  - Open in update mode and assign to elements:
    - ```
      from astropy.io import fits
      with fits.open('image.fits', mode='update') as hdul:
          hdul[0].data[10, 10] = 42.0
          hdul.flush()
      ```
  - Modifications affect the in-memory array and can be written back to disk with `flush()`.

- Creating a new image with modified data:
  - ```
    from astropy.io import fits
    new_data = hdu.data.copy()
    new_data[10, 10] = 42.0
    new_hdu = fits.ImageHDU(data=new_data, header=hdu.header)
    new_hdu.writeto('image_modified.fits', overwrite=True)
    ```

- Important notes:
  - Ensure the target file is writable and that the data type and shape are preserved unless you explicitly change them.
  - When using memory mapping (see below), write-access modes and `flush()` control when changes are written to disk.

## Working with large images (memory mapping)

- For very large images, enable memory mapping to avoid loading the entire array into memory:
  - ```
    from astropy.io import fits
    with fits.open('large_image.fits', memmap=True) as hdul:
        data = hdul[0].data  # memory-mapped array
        mean = data.mean()
    ```
- To modify a memory-mapped array on disk, open in a write-enabled mode (e.g., `mode='update'`) and use `flush()` as needed.

## Practical tips

- Always verify axis-ordering when performing complex indexing, especially for higher-dimensional data cubes.
- Use `hdu.header` to access/modify metadata such as `BUNIT`, `COMMENT`, or axes information in tandem with data changes.
- When performing I/O-heavy operations, prefer memory mapping for efficient access to large FITS images.

## Example: Open, inspect, modify, and save

- Open and inspect:
  - ```
    from astropy.io import fits
    with fits.open('image.fits') as hdul:
        hdu = hdul[0]
        print("shape:", hdu.data.shape)
        print("dtype:", hdu.data.dtype)
        print("BUNIT:", hdu.header.get('BUNIT', 'Unknown'))
        print("pixel (y=100, x=50):", hdu.data[100, 50])
    ```

- Modify and save:
  - ```
    from astropy.io import fits
    with fits.open('image.fits', mode='update') as hdul:
        hdul[0].data[100, 50] = 0.0
        hdul.flush()
    ```
