Gaussian Processes
==================

Overview
--------

Gaussian processes (GPs) provide a principled, probabilistic framework for
regression and classification. A GP defines a prior over functions, such that
any finite set of function values has a multivariate Gaussian distribution.
Prediction is achieved by conditioning this prior on observed data, yielding
posterior mean and uncertainty estimates.

Key ideas:

- Covariance as a function of inputs: the choice of kernel (covariance
  function) encodes assumptions about smoothness, periodicity, and other
  properties of the underlying function.
- Kernel-based learning: instead of learning a parametric function, we learn
  kernel hyperparameters describing the function space most compatible with the
  data.
- Analytic gradients: hyperparameters can be optimized using gradient-based
  methods with bounds and fixed parameters.

Kernel-based modeling
---------------------

All Gaussian process kernels in this module share common capabilities:

- Composition: kernels can be combined using addition (+) and multiplication
  (*) to form new kernels representing sums and products of covariances.
- Exponentiation: kernels can be scaled by a scalar via exponentiation (**).
- Scalar kernels: scalar values appearing in expressions are automatically
  converted to constant kernels.
- Hyperparameter optimization: the space of hyperparameters is defined by lower
  and upper bounds for each parameter, forming a rectangular search space. Hyperparameters can also be declared as fixed, excluding them from optimization.

Hyperparameters
--------------

Each kernel exposes a set of hyperparameters that control its behavior
(e.g., constant value, length scales, noise levels). Hyperparameters are
represented by the Hyperparameter namedtuple with fields:

- name: parameter name (e.g., "constant_value", "length_scale")
- value_type: type of the parameter (e.g., "numeric")
- bounds: 2D array or "fixed"; bounds for the parameter values
- n_elements: number of elements (scalar or vector-valued)
- fixed: whether the parameter is fixed (not optimized)

Example:

- A ConstantKernel with a tunable constant value:

  - constant_value: bounds (0.0, 10.0)

- A RBF (SE) kernel with anisotropic length-scales:

  - length_scale: bounds (0.1, 10.0) for each input dimension

Both examples illustrate how hyperparameters can be constrained or fixed during optimization. You can inspect hyperparameters via:

.. code-block:: python

    >>> from sklearn.gaussian_process.kernels import ConstantKernel, RBF
    >>> k = ConstantKernel(1.0, (0.0, 10.0)) * RBF(1.0, (0.1, 10.0))
    >>> k.hyperparameters
    Hyperparameter(name='constant_value', value_type='numeric',
                   bounds=array([[ 0., 10.]]), n_elements=1, fixed=False)
    Hyperparameter(name='length_scale', value_type='numeric',
                   bounds=array([[0.1, 10.]]), n_elements=1, fixed=False)

Using Gaussian Processes
------------------------

Typical workflow:

1. Choose a kernel that encodes your prior beliefs about the function.
2. Optionally compose kernels to capture multiple phenomena (e.g., smooth trends plus a noise component).
3. Fit a GaussianProcessRegressor (or GaussianProcessClassifier) by optimizing kernel hyperparameters (with gradient-based methods) subject to the specified bounds.
4. Make predictions with uncertainty estimates.

Code example: simple regression with a composed kernel
-----------------------------------------------------

.. code-block:: python

    import numpy as np
    from sklearn.gaussian_process import GaussianProcessRegressor
    from sklearn.gaussian_process.kernels import ConstantKernel, RBF, WhiteKernel

    # Kernel composition:
    # A constant baseline value times an RBF kernel plus a white noise component
    kernel = (ConstantKernel(1.0, (0.0, 10.0)) * RBF(length_scale=1.0, length_scale_bounds=(0.1, 10.0))
              + WhiteKernel(noise_level=1.0, noise_level_bounds=(1e-5, 1e+1)))

    # Generate some data
    rng = np.random.default_rng(0)
    X = rng.uniform(-5, 5, (40, 1))
    y = np.sin(X).ravel() + rng.normal(0, 0.1, X.shape[0])

    # Create and fit GP model
    gp = GaussianProcessRegressor(kernel=kernel, alpha=0.0, normalize_y=True)
    gp.fit(X, y)

    # Predict at new points with uncertainty
    X_test = np.linspace(-6, 6, 100).reshape(-1, 1)
    y_mean, y_std = gp.predict(X_test, return_std=True)

    # Optional: examine optimized kernel parameters
    optimized_kernel = gp.kernel_
    print("Optimized kernel:", optimized_kernel)

Notes on optimization
---------------------

- The hyperparameter search space is rectangular, defined by lower and upper bounds for each parameter.
- Hyperparameters can be fixed, in which case they are excluded from optimization.
- Gradients with respect to hyperparameters are available, enabling analytic gradient-based optimization.
- You can inspect and retrieve the optimized kernel using the model’s kernel_ attribute after fitting.

API references
--------------

- Hyperparameter: Specification of kernel hyperparameters (name, type, bounds, etc.).
- Kernel: Base class for kernels; supports composition and hyperparameter exposure.
- GaussianProcessRegressor / GaussianProcessClassifier: Models that use kernels to perform regression or classification with Gaussian processes.

Notes
-----

- The kernel module is designed with inspiration from the george package, enabling flexible kernel engineering.
- All hyperparameters and bounds can be configured to guide the optimization process, including fixed parameters for prior knowledge or to reduce search space.
