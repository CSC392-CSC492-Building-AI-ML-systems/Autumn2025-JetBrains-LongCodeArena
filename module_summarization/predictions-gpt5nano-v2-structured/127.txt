Convolution module built-in kernels
====================================

This document describes the built-in kernels provided by the Convolution facilities in the Astropy modeling convolution framework. Built-in kernels are 1D or 2D arrays that define the impulse response used by Convolution to smooth, blur, or filter data objects such as images and spectra.

Overview
--------

- Kernels implement the response shape used in a convolution operation. They come in 1D and 2D varieties and are exposed as Kernel1D/Kernel2D subclasses (e.g., Gaussian1DKernel, Gaussian2DKernel, Box1DKernel, Box2DKernel).
- Kernels are typically normalized to preserve flux (the sum of kernel elements equals 1) by default, unless you explicitly request a non-normalized kernel.
- Convolution operations support different boundary handling modes. Available options include None, "fill", "wrap", and "extend".
- The internal implementation optimizes convolution using FFT when appropriate. The FFT-length padding is chosen to be fast for the underlying FFT engine (scipy.fft if available; otherwise a precomputed set of good sizes).

Available built-in kernel families
----------------------------------

The following built-in kernels are commonly used with the Convolution framework. Each is available as a 1D or 2D kernel class:

- Gaussian kernels
  - Gaussian1DKernel: 1D Gaussian kernel defined by a standard deviation (in pixels).
  - Gaussian2DKernel: 2D Gaussian kernel with a single standard deviation (isotropic) in both axes.
  - Usage (examples):
    - 1D: k = Gaussian1DKernel(2.0)
    - 2D: k = Gaussian2DKernel(2.0)

- Box (uniform) kernels
  - Box1DKernel: 1D rectangular kernel of a given width.
  - Box2DKernel: 2D rectangular kernel (square) of a given width.
  - Usage (examples):
    - 1D: k = Box1DKernel(5)
    - 2D: k = Box2DKernel(5)

- Other built-in kernels (see API reference)
  - The module also provides additional kernels that may be used for specialized filtering. See the public API for the complete list and exact constructor signatures.

Using built-in kernels with Convolution
---------------------------------------

To apply a built-in kernel to data, create the kernel object and then construct a Convolution model with that kernel. You can optionally specify boundary handling to control how edges are treated.

Example: 2D Gaussian smoothing of an image
-----------------------------------------

from astropy.modeling.convolution import Gaussian2DKernel, Convolution
import numpy as np

# Create a 2D Gaussian kernel with sigma = 2.0 pixels
kernel = Gaussian2DKernel(2.0)

# Build a convolution model; default boundary handling is used unless overridden
conv = Convolution(kernel)

# Apply to an input image (2D NumPy array)
image = np.random.random((100, 100))
smoothed = conv(image)

Example: 1D Box smoothing
-------------------------

from astropy.modeling.convolution import Box1DKernel, Convolution
import numpy as np

# Create a 1D box (uniform) kernel of width 7
kernel1d = Box1DKernel(7)

# Convolve a 1D signal
signal = np.random.random(200)
conv1d = Convolution(kernel1d)
smoothed_1d = conv1d(signal)

Boundary handling and FFT optimization
--------------------------------------

- Boundary options
  - You can control how the convolution handles boundaries via boundary options. Valid values include None, "fill", "wrap", and "extend".
  - Example:
    conv = Convolution(kernel, boundary="extend")

- FFT performance
  - When using FFT-based convolution, the internal padding length is chosen to be FFT-friendly. If SciPy is available, scipy.fft.next_fast_len is used to compute optimal lengths. If SciPy is not available, a precomputed list of good sizes is used and scaled as needed.
  - This optimization helps keep large-kernel convolutions fast.

Input handling
--------------

- Inputs are prepared internally to ensure correct dtype and contiguity. Masked inputs and NaN handling are supported through internal copying and filling as needed.

Notes for developers
--------------------

- Kernel objects inherit from Kernel1D or Kernel2D and integrate with the Convolution modeling framework.
- The internal helpers (such as FFT-length selection and input preparation) are designed to minimize user concerns about performance and edge cases while maintaining a clean API.
