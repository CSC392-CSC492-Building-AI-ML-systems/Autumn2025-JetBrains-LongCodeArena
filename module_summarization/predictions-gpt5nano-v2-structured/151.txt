# General Plotting routines

This module provides high-level plotting helpers based on the Bokeh backend. It exposes a small, consistent API to create interactive plots, manage defaults, and handle data sources in a flexible way.

## Overview

- Central plotter instance: `plot_params = BokehPlotter()`
- Convenience functions for common plots:
  - `bokeh_scatter(...)` — single dataset, interactive scatter
  - `bokeh_multi_scatter(...)` — multiple datasets, interactive scatter
- Default management:
  - `set_bokeh_plot_defaults(...)`
  - `reset_bokeh_plot_defaults()`
  - `show_bokeh_plot_defaults()`
  - `load_bokeh_defaults(filename='plot_bokeh_defaults.json')`
  - `save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)`
  - `get_bokeh_help(key)`
- Data handling helpers are used internally to process inputs and align data.

All plotting functions return a Bokeh Figure object and rely on the BokehPlotter to configure appearance and behavior.

---

## Plotter and defaults

- The global plotter instance is created as:
  - `plot_params = BokehPlotter()`
- Defaults control styling and behavior (colors, sizes, tools, etc.). See the BokehPlotter documentation for the complete set of defaults.
- You can:
  - Set defaults once for a session or script
  - Reset to hardcoded defaults
  - Inspect current defaults
  - Persist defaults to a JSON file and load them later

Example:
```python
from masci_tools.vis.plotting import (
    set_bokeh_plot_defaults,
    show_bokeh_plot_defaults,
    reset_bokeh_plot_defaults,
)

set_bokeh_plot_defaults(height=420, width=600, toolbar_location='above')
show_bokeh_plot_defaults()
# ... later
reset_bokeh_plot_defaults()
```

---

## Usage notes

- Both `bokeh_scatter` and `bokeh_multi_scatter` are decorated with `@ensure_plotter_consistency(plot_params)`, which enforces consistency between the provided arguments and the plotter’s capabilities.
- Data handling is flexible:
  - You can pass data directly as a DataFrame (via `data=`) or use raw arrays for the x/y data.
  - If you pass the data source as the first positional argument (e.g., a DataFrame in place of `x`), a deprecation warning is emitted and you should switch to providing the data via the `data=` keyword and use `xdata`/`ydata` to specify column names.
- The `figure` parameter is optional. If provided, the plot will be added to the given Bokeh figure; otherwise a new figure is created.
- The `saveas` parameter controls how the plot is saved (file naming/format) via the internal `save_plot` mechanism.

## Functions

### bokeh_scatter

Create an interactive scatter plot for a single data set.

- Signature (selected): 
  - `bokeh_scatter(x, y=None, *, xlabel='x', ylabel='y', title='', figure=None, data=None, saveas='scatter', copy_data=False, **kwargs)`
- Key behavior:
  - Accepts raw arrays or a data source via `data` and column names via `x`/`y` (or via `xdata`/`ydata` in the deprecation path).
  - Uses `process_data_arguments` to prepare data for plotting.
  - Sets up the figure and renders a scatter glyph with appropriate tooltips.
  - Applies defaults from the plotter and draws auxiliary guidance (e.g., axis limits, reference lines if configured).
- Returns: a Bokeh Figure object.

Example 1 — simple arrays:
```python
import numpy as np
from masci_tools.vis.plotting import bokeh_scatter

x = np.linspace(0, 2 * np.pi, 100)
y = np.sin(x)

fig = bokeh_scatter(x, y, xlabel='Angle [rad]', ylabel='sin(x)', title='Sin Wave')
fig.show()  # or save via saveas/defaults
```

Example 2 — using a DataFrame:
```python
import pandas as pd
import numpy as np
from masci_tools.vis.plotting import bokeh_scatter

df = pd.DataFrame({'angle': np.linspace(0, 2*np.pi, 100), 'val': np.sin(np.linspace(0, 2*np.pi, 100))})
fig = bokeh_scatter(data=df, x='angle', y='val', xlabel='Angle [rad]', ylabel='sin(angle)', title='Sin Wave (DF)')
```

> Note: Passing the data source as the first positional argument (e.g., `bokeh_scatter(df, ...)`) is deprecated. Pass the source via `data=` and specify columns with `x`/`y` (or `xdata`/`ydata`).

### bokeh_multi_scatter

Create an interactive scatter plot that can display multiple datasets. This is useful for comparing several series on the same axes.

- Signature (selected):
  - `bokeh_multi_scatter(x, y=None, *, data=None, figure=None, xlabel='x', ylabel='y', title='', saveas='scatter', copy_data=False, set_default_legend=True, **kwargs)`
- Key behavior:
  - Similar data handling as `bokeh_scatter`, but supports multiple y-datasets (e.g., via a list of columns in `y` or multiple data sources).
  - When `set_default_legend` is True, the data names are used to generate default legend labels.
  - Integrates with the `BokehPlotter` defaults for styling and interaction.
- Returns: a Bokeh Figure object.

Example — multiple datasets from a DataFrame:
```python
import numpy as np
import pandas as pd
from masci_tools.vis.plotting import bokeh_multi_scatter

x = np.linspace(0, 2*np.pi, 100)
df = pd.DataFrame({
    'x': x,
    'y1': np.sin(x),
    'y2': np.cos(x),
})

fig = bokeh_multi_scatter(data=df, x='x', y=['y1', 'y2'],
                          xlabel='Angle [rad]', ylabel='value',
                          title='Sine and Cosine', set_default_legend=True)
```

## Data handling and customization

- Data argument:
  - `data`: a DataFrame or similar data source.
  - When using data sources, specify the columns via `x` and `y` (or `xdata`/`ydata` for the deprecation path).
- Copying:
  - `copy_data` controls whether the input data is copied before plotting (default is often False for performance).
- Figure management:
  - `figure` allows embedding the plot into an existing Bokeh figure.
- Defaults and helpers:
  - `set_bokeh_plot_defaults(...)` to configure defaults.
  - `reset_bokeh_plot_defaults()` to revert to hardcoded defaults.
  - `show_bokeh_plot_defaults()` to inspect current defaults.
  - `get_bokeh_help(key)` to print a description of a specific default key.
  - `load_bokeh_defaults(...)` and `save_bokeh_defaults(...)` to persist defaults to a JSON file.

Example — managing defaults:
```python
from masci_tools.vis.plotting import (
    set_bokeh_plot_defaults, show_bokeh_plot_defaults, load_bokeh_defaults, save_bokeh_defaults
)

set_bokeh_plot_defaults(height=480, width=800, tools='pan,box_zoom,reset')
show_bokeh_plot_defaults()

# Persist and later load
save_bokeh_defaults('plot_bokeh_defaults.json')
load_bokeh_defaults('plot_bokeh_defaults.json')
```

---

## Accessing help and troubleshooting

- Describe a default:
  - `get_bokeh_help(key)` prints the description for a given default key from the BokehPlotter.
- If you encounter warnings about deprecated usage:
  - Transition from passing a data source as the first positional argument to using `data=...` with explicit `x`/`y` (or `xdata`/`ydata`).

## API reference (summary)

- Defaults:
  - `set_bokeh_plot_defaults(**kwargs)`, `reset_bokeh_plot_defaults()`, `show_bokeh_plot_defaults()`
  - `load_bokeh_defaults(filename='plot_bokeh_defaults.json')`
  - `save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)`
  - `get_bokeh_help(key)`
- Plots:
  - `bokeh_scatter(x, y=None, *, xlabel='x', ylabel='y', title='', figure=None, data=None, saveas='scatter', copy_data=False, **kwargs)`
  - `bokeh_multi_scatter(x, y=None, *, data=None, figure=None, xlabel='x', ylabel='y', title='', saveas='scatter', copy_data=False, set_default_legend=True, **kwargs)`

For detailed behavior and all supported keyword arguments, refer to the function definitions and the `BokehPlotter` API.
