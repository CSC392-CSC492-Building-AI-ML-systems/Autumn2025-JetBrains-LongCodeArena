Channel Options
===============

Overview
--------
Channel options define how data is mapped to visual properties in Vega-Lite
encodings. Each encoding channel (e.g., x, y, color, size, shape, tooltip)
can be configured with a set of options that control appearance, scaling,
axes, legends, and formatting. The channel options are exposed as a family of
classes to help construct and customize Vega-Lite specs programmatically.

Common channel categories include:
- Encoding channels: X, Y, Color, Size, Shape, Opacity, Tooltip, Detail, Text, Order, etc.
- Axis options: Axis, AxisConfig, AxisOrient, AxisResolveMap
- Legend options: Legend, LegendConfig
- Scale options: Scale (domain, range, nice, clamp, etc.)
- Color options: Color, ColorDatum, ColorScheme, ColorName, ColorValue
- Miscellaneous: Text, Axis-related conditional properties, and other channel helpers

Usage
-----
Channel options are typically used when building a Vega-Lite specification. A
chart is defined with a data source and a set of encodings. Each encoding
can reference a channel option to customize its behavior.

- JSON spec (illustrative):
  {
    "data": {"url": "data.csv"},
    "mark": "point",
    "encoding": {
      "x": {"field": "Horsepower", "type": "quantitative"},
      "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
      "color": {
        "field": "Origin",
        "type": "nominal",
        "scale": {"scheme": "category10"},
        "legend": {"title": "Origin"}
      }
    }
  }

- Python (Altair-like) style:
  from altair import Chart, Color, Legend
  chart = Chart(source).mark_point().encode(
      x='Horsepower:Q',
      y='Miles_per_Gallon:Q',
      color=Color('Origin:N', legend=Legend(title='Origin'))
  )

Representative Channel Options
------------------------------
The following classes (and related configuration objects) are commonly used to
control channel behavior. This section lists representative options; the full
set is exposed by the module and may evolve over time.

- Axis: Configuration for axis visuals (tick labels, titles, grid, label angles)
- AxisConfig: Detailed configuration options for axes
- AxisOrient: Orientation for axes (e.g., left, right, top, bottom)
- AxisResolveMap: Rules for resolving axis properties in layered/pooled specs
- Color: Encoding for color channel, including legend and scale settings
- ColorDatum, ColorScheme, ColorName, ColorValue: Helpers for color encoding
- Size: Encoding for size channel (marker size or symbol area)
- Text: Encoding for textual output (labels, annotations)
- Tooltip: Rich tooltip configuration for interactive inspection
- Detail: Encoding for additional grouping fields
- Order: Encoding to define sorting order for stacked/ Faceted charts
- Legend: Global legend configuration (position, title, format)
- Scale: Global scale configuration (domain, range, type, nice, clamp)

Examples
--------

1) Customize color channel with a tailored legend and scale
- JSON:
  {
    "data": {"name": "data"},
    "mark": "point",
    "encoding": {
      "color": {
        "field": "Origin",
        "type": "nominal",
        "scale": {"scheme": "category10"},
        "legend": {"title": "Origin", "orient": "right"}
      }
    }
  }

- Python (conceptual illustration):
  from altair import Chart, Color, Legend
  chart = Chart(source).mark_point().encode(
      color=Color('Origin:N',
                  scale={'scheme': 'category10'},
                  legend=Legend(title='Origin', orient='right'))
  )

2) Axis and legend customization for a quantitative x and y
- JSON:
  {
    "data": {"name": "data"},
    "mark": "line",
    "encoding": {
      "x": {"field": "Year", "type": "temporal", "axis": {"title": "Year"} },
      "y": {"field": "Value", "type": "quantitative",
            "axis": {"title": "Value", "labelColor": "gray"} },
      "color": {"field": "Category", "type": "nominal",
                "legend": {"title": "Category"}}
    }
  }

- Python (conceptual illustration):
  chart = Chart(source).mark_line().encode(
      x='Year:T',
      y='Value:Q',
      color=Color('Category:N', legend=Legend(title='Category'))
  )

3) Rich tooltip for interactive data exploration
- JSON:
  {
    "data": {"name": "data"},
    "mark": "point",
    "encoding": {
      "x": {"field": "Horsepower", "type": "quantitative"},
      "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
      "tooltip": [
        {"field": "Model", "type": "nominal"},
        {"field": "Origin", "type": "nominal"},
        {"field": "Cylinders", "type": "quantitative"}
      ]
    }
  }

- Python:
  chart = Chart(source).mark_point().encode(
      x='Horsepower:Q',
      y='Miles_per_Gallon:Q',
      tooltip=['Model:N', 'Origin:N', 'Cylinders:Q']
  )

Deprecations and Compatibility
------------------------------
Channel option classes evolve with Vega-Lite capabilities. Use the latest
spec to ensure compatibility with Vega-Lite v5 features. The documentation
references the Channel Options surface as implemented in the module
aliased as channel_options, which re-exports a broad set of option
classes (e.g., Axis, Color, Legend, Scale) used to compose encoding
channels.

Migration notes
---------------
If upgrading from older channel option implementations, check for renamed
properties or moved configuration items in Axis, Legend, and Scale
categories. The new surface aims to align with Vega-Lite v5 capabilities and
provide a consistent, programmatic approach to channel customization.
