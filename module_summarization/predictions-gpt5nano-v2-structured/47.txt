========================================
Contingency Tables: Two-Way Analysis
========================================

Overview
--------

This module provides tools for analyzing two-way contingency tables (i.e., frequency tables cross-classified by two categorical variables). The primary classes implement common analyses and convenience helpers for working with data in tabular form. In addition, the module exposes functions for conducting McNemar's test and Cochran’s q test.

Main classes
------------

- **Table**: Represents a general two-way contingency table and supports nominal and ordinal association testing.
- **SquareTable**: Implements methods that apply specifically to square two-way contingency tables (same number of rows and columns).
- **Table2x2**: Implements methods that apply to 2×2 contingency tables.
- **StratifiedTable**: Implements methods for operating on a collection of 2×2 contingency tables (stratified analyses).

Internal helpers and utilities
------------------------------

- **_make_df_square(table)**: Internal helper that reindexes a pandas DataFrame to be square by aligning its row and column indices. If the input is not a DataFrame, it is returned unchanged.
- **_Bunch**: Lightweight container used to hold and pretty-print results from statistical tests.

Usage notes
-----------

- The inference procedures assume that the observed units are independent and identically distributed unless otherwise noted.
- The Table class accepts either a raw array or a DataFrame. When zeros are present and shift_zeros is True, zeros are replaced with 0.5 to enable certain calculations.

API Reference
-------------

Table
^^^^^

Description
~~~~~~~~~~
A two-way contingency table with utilities for hypothesis testing and summary statistics.

Constructor
~~~~~~~~~~
Table(table, shift_zeros=True)

Parameters
~~~~~~~~~~
- table: array_like
  The contingency table. Can be a NumPy array or a pandas DataFrame.
- shift_zeros: bool
  If True and any cell count is zero, add 0.5 to all values in the table.

Attributes
~~~~~~~~~~
- table_orig: array_like
  The original input table (cached for reference).

- table: ndarray (float)
  The numeric contingency table used for computations. Zeros may be shifted if requested.

From data
~~~~~~~~~

classmethod
from_data(data, shift_zeros=True)

Construct a Table from raw data by building a cross-tabulation from the first two columns.

Parameters
~~~~~~~~~~
- data: array_like or pandas.DataFrame
  The raw data. If a DataFrame is provided, the first two columns are used.
- shift_zeros: bool
  If True and any cell count is zero, add 0.5 to all values in the table.

Returns
~~~~~~~
A Table instance.

Key methods
~~~~~~~~~~
- test_nominal_association()
  Assess independence for nominal factors using a chi-squared test.

  Returns
  ~~~~~~~
  A _Bunch with attributes:
  - statistic: float
  - df: int
  - pvalue: float

- test_ordinal_association(row_scores=None, col_scores=None)
  Assess independence for ordinal factors using a linear-by-linear (trend) test with optional scores for rows and columns.

  Parameters
  ~~~~~~~~~~
  - row_scores: array_like or None
    Numeric scores for the row categories. If None, defaults to 0,1,2,...
  - col_scores: array_like or None
    Numeric scores for the column categories. If None, defaults to 0,1,2,...

  Returns
  ~~~~~~~
  A _Bunch with attributes:
  - statistic: float
  - null_mean: float
  - null_sd: float
  - zscore: float
  - pvalue: float

Notes
~~~~~
- The scores define the trend to which the test is most sensitive. With default scores for both dimensions, the test corresponds to Cochran-Armitage trend test.

Example
-------

Basic usage with a simple 2×2 table:

.. code-block:: python

   from statsmodels.stats.contingency_tables import Table
   import numpy as np

   tbl = Table([[10, 20], [30, 40]])
   print(tbl)

   res_nominal = tbl.test_nominal_association()
   print("Chi-square statistic:", res_nominal.statistic)
   print("p-value:", res_nominal.pvalue)

Ordinal association example:

.. code-block:: python

   row_scores = [0, 1]
   col_scores = [0, 1]
   res_ordinal = tbl.test_ordinal_association(row_scores=row_scores, col_scores=col_scores)
   print("Statistic:", res_ordinal.statistic)
   print("p-value:", res_ordinal.pvalue)

DataFrame support
~~~~~~~~~~~~~~~~~

If you have a DataFrame with two categorical variables, you can construct a Table directly:

.. code-block:: python

   import pandas as pd
   from statsmodels.stats.contingency_tables import Table

   df = pd.DataFrame({
       'A': ['x', 'y', 'x', 'y', 'x'],
       'B': ['u', 'u', 'v', 'u', 'v']
   })

   t = Table.from_data(df)
   print(t)

Notes on scope
--------------

- The module also includes utilities for McNemar’s test and Cochran’s q test (not shown in this excerpt). See the full module for details.
- Additional classes (SquareTable, Table2x2, StratifiedTable) provide specialized functionality for square tables, 2×2 tables, and collections of 2×2 tables, respectively.

Internal and related functionality
----------------------------------

- _make_df_square(table): Ensures DataFrame inputs are square by extending the index and columns to include all observed categories, aligning order, and filling missing cells with zeros. This is primarily used to normalize input before applying certain tests.
- _Bunch: Simple container for returning multiple named results from tests in a tidy, printable form.
