# plot_data.md

## Supporting data to plotting functions in multiple ways

PlotData is a lightweight wrapper that makes it easy to feed plotting routines with data from diverse sources (dicts, DataFrames, or Bokeh's ColumnDataSource) and to combine multiple data sets into a convenient iteration pattern. It abstracts away the details of how data is stored and allows you to specify which keys to plot, while automatically handling the creation of per-plot-set data.

### Key ideas

- Flexibly consume data from:
  - Python dictionaries
  - Pandas DataFrames
  - Bokeh ColumnDataSource (if Bokeh is installed)
- Support multiple plot sets by providing some keys as lists
  - Example: y can be a list of keys; PlotData will create one plot set per key
- Iterate to obtain a simple, per-set data object
  - Each iteration yields an object with attributes corresponding to the chosen data keys (e.g., `entry.x`, `entry.y`)
  - If you specify multiple y-keys, you get one set per key (e.g., first y-key’s data, then the second, etc.)
- Optional data integrity and handling features
  - `same_length`: ensure all entries have compatible lengths
  - `strict_data_keys`: disallow adding new data keys after initialization
  - `copy_data`: create copies of input sources (and handle Bokeh sources safely)
  - `use_column_source`: enable interaction with ColumnDataSource when available

---

## How it works

- Data holders (when Bokeh is installed): dict, pandas.DataFrame, and ColumnDataSource
- When any value in the keyword arguments is a list, PlotData expands the data into multiple plot sets
- The internal representation uses a namedtuple to describe the columns for each set
- Iterating over PlotData yields per-set data, exposing the requested keys as attributes (e.g., `entry.x`, `entry.y`)

References to internal helpers:
- `_is_bokeh_cds` and `_copy_bokeh_cds` enable safe handling of Bokeh’s `ColumnDataSource`
- `_normalize_dict_entries` is used to align dictionary-based data when requested
- `ALLOWED_DATA_HOLDERS` restricts the kinds of inputs accepted (and includes `ColumnDataSource` only if Bokeh is installed)

---

## Usage

Constructor signature (brief)

- `PlotData(data, use_column_source=False, same_length=False, strict_data_keys=False, copy_data=False, **kwargs)`

- `data`: object or list of objects that can be bracket-indexed with the given keys
  - Examples: dicts or DataFrames
- `**kwargs`: mapping from data key name to the key to plot or to a list of keys
  - If any value is a list, multiple plot sets are created

Key attributes and methods

- `data_keys` (property): returns the registered data keys (names) for the current instance
- `add_data_key(data_key, keys=None)`: add a new data key after initialization (unless `strict_data_keys` is True)
- `columns` (internal): list of namedtuples describing the keys for each plot set

## Examples

### Example 1: Basic dict-based data with multiple y-keys

- Data contains x and three y-series
- Create three plot sets (one for each y)

```python
import numpy as np
from masci_tools.vis.data import PlotData

x = np.linspace(-10, 10, 100)
data = {
    'x': x,
    'y1': np.sin(x),
    'y2': np.cos(x),
    'y3': x**2
}

p = PlotData(data, x='x', y=['y1', 'y2', 'y3'])

for entry in p:
    print(entry.x)  # array of x values
    print(entry.y)  # y1, then y2, then y3
```

### Example 2: Data from multiple sources (DataFrames)

```python
import numpy as np
import pandas as pd
from masci_tools.vis.data import PlotData

x = np.linspace(-5, 5, 50)
df1 = pd.DataFrame({'x': x, 'y1': np.sin(x)})
df2 = pd.DataFrame({'x': x, 'y1': np.cos(x)})

p = PlotData([df1, df2], x='x', y=['y1'])

for entry in p:
    print(type(entry.x), type(entry.y))
```

### Example 3: Copying data to ensure immutability (with Bokeh support)

```python
import numpy as np
data = {'x': np.linspace(-1, 1, 20), 'y': np.random.rand(20)}
p = PlotData(data, x='x', y='y', copy_data=True)
```

### Example 4: Enforcing consistent lengths (when using dict-based data)

```python
p = PlotData({'x': x, 'y1': np.sin(x)}, x='x', y=['y1'], same_length=True)
```

### Example 5: Adding a new data key after initialization

```python
p = PlotData({'x': x, 'y1': np.sin(x)}, x='x', y='y1')
p.add_data_key('y2', keys=[np.cos(x)])  # if you want to extend with a new y-key
```

## API reference (high level)

- PlotData
  - Purpose: Provide a uniform, flexible interface to data used for plotting, supporting multiple sources and multiple plot-sets.
  - Parameters:
    - `data`: initial data source (dict, DataFrame, or ColumnDataSource if available)
    - `use_column_source`: enable ColumnDataSource usage when available
    - `same_length`: ensure data lengths are consistent across keys
    - `strict_data_keys`: disallow adding new data keys after init
    - `copy_data`: copy sources to avoid mutating original data
    - `**kwargs`: mapping of data keys to plotting keys (or lists of keys)
  - Returns:
    - An iterable object; each iteration yields a simple object (with attributes named after the keys) representing one plot set
  - Properties/Methods:
    - `data_keys`: list of registered data keys
    - `add_data_key(data_key, keys=None)`: add a new data key post-initialization
    - `columns`: internal representation of per-set keys (list of namedtuples)

---

## Notes

- Availability of ColumnDataSource dependent on Bokeh installation
  - If Bokeh is not installed, dicts and DataFrames are the only supported data holders
- The design emphasizes minimal data copying for performance, with safe copying when requested
- Iteration semantics are designed to align with plotting workflows: per-plot-set data is exposed with intuitive attribute names

For further details, refer to the source implementation and its integration with plotting backends in masci-tools.
