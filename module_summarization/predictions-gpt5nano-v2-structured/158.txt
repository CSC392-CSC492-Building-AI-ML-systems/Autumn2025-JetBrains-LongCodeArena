Migrating to nextcord
======================

Overview
--------
This guide helps you migrate existing codebases to nextcord with a focus on the Guild Widget API. nextcord provides representations for a guild's widget data via three core models:

- Widget: the overall guild widget data
- WidgetChannel: a partial representation of a channel exposed by the widget
- WidgetMember: a partial representation of a member exposed by the widget

These classes expose a stable API for accessing widget channels, online members, their statuses, nicknames, activities, and basic metadata. The widget data is fetched from the Discord API (when available) and can be updated by calling fetch methods on relevant objects.

Why migrate to nextcord
-----------------------
- Consistent, typed models for widget data (Widget, WidgetChannel, WidgetMember)
- Rich, documented attributes and helpful helpers (e.g., WidgetChannel.mention, WidgetMember.display_name)
- Clear separation of concerns between channel metadata and member state
- Compatibility with typical Discord widget usage patterns (fetching widget data, iterating channels and members)

API overview (core classes)
---------------------------

WidgetChannel
-------------
Represents a partial widget channel.

- id (int): The channel's ID.
- name (str): The channel's name.
- position (int): The channel's position within the widget.

Key behaviors:
- str(channel) returns the channel’s name
- channel.mention returns a string suitable for mentioning the channel in messages
- channel.created_at returns the channel’s creation time (UTC)

WidgetMember
------------
Represents a partial member within the widget’s guild.

- id (int): The member's ID
- name (str): The member's username
- discriminator (str): The member's discriminator
- bot (bool): Whether the member is a bot
- status (Status): The member's online status
- nick (Optional[str]): The member's nickname (if any)
- avatar (Optional[str]): The member's avatar hash
- activity (Optional[Union[BaseActivity, Spotify]]): The member's current activity
- deafened (Optional[bool]): Whether the member is deafened
- muted (Optional[bool]): Whether the member is muted
- suppress (Optional[bool]): Whether the member is suppressed
- connected_channel (Optional[WidgetChannel]): The channel the member is connected to

- display_name: returns nick if available, otherwise the user name

Widget
------
Represents a Guild widget.

- id (int): The guild's ID
- name (str): The guild's name
- channels (List[WidgetChannel]): Accessible voice channels in the guild
- members (List[WidgetMember]): Online members; offline members may not appear depending on data availability

Usage patterns
--------------

Fetching widget data
--------------------
Widget data is retrieved from the Discord API. In practice, you typically fetch the latest widget data for a guild using a fetch operation on the guild object.

Example (async):
- Fetch the latest widget data for a guild
  - from nextcord import Widget
  - widget = await guild.fetch_widget()

Accessing channels and members
------------------------------
Once you have a Widget object, you can inspect channels and online members exposed by the widget.

Example:
- Iterate widget channels
  - for channel in widget.channels:
        print(channel.name)

- Print channel mentions
  - for channel in widget.channels:
        print(channel.mention)

- Iterate widget members and display their names
  - for member in widget.members:
        print(member.display_name)

- Access a member's activity (if available)
  - if member.activity is not None:
        print(member.activity.name)

Migration steps
---------------
1. Upgrade to nextcord (version that includes Widget, WidgetChannel, WidgetMember).
2. Replace direct widget access patterns from the previous library with nextcord equivalents.
   - Old: widget = guild.widget (or another API path)
   - New: widget = await guild.fetch_widget()
3. Update code to work with the Widget API:
   - Use widget.channels to access channels
   - Use widget.members to access online members
   - Use channel.mention for channel mentions
   - Use member.display_name for a member’s display name
4. Ensure you handle cases where widget data is unavailable (privacy settings, guild widget disabled, or API response limitations).
5. Update type hints and imports:
   - from nextcord import Widget, WidgetChannel, WidgetMember (or from nextcord.widget import ... depending on your project structure)

Code examples

Fetch and print basic widget info
--------------------------------
from nextcord import Widget

widget = await guild.fetch_widget()
print(f"Guild: {widget.name} (ID: {widget.id})")

Listing channels
----------------
for channel in widget.channels:
    print(f"Channel: {channel.name} (ID: {channel.id})")

Listing online members
----------------------
for member in widget.members:
    print(f"Member: {member.display_name} — Status: {member.status}")

Accessing a member's activity
-----------------------------
for member in widget.members:
    if member.activity:
        print(f"{member.display_name} is {member.activity.name}")

Notes and compatibility
-----------------------
- Widget data availability depends on Discord’s widget feature and guild settings. If a widget is disabled or hidden, fetch_widget may return partial data or fail depending on permissions.
- The WidgetMember class derives from a base user model and includes partial member data (name, nick, status, activity, etc.). For complete user details, rely on the standard member/user APIs in nextcord.

Summary
-------
Migrating to nextcord for widget handling gives you a stable, well-documented API surface for working with guild widgets. Use fetch_widget to obtain current widget data, then interact with WidgetChannel and WidgetMember to read channels, online members, and their activities in a typed, predictable manner.
