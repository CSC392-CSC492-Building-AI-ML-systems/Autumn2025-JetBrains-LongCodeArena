# Transforming the prediction target

Transforming the target variable (y) can help linearize relationships, stabilize variance, or impose a monotonic mapping that better matches the modelling assumptions of the estimator. The TransformedTargetRegressor wrapper applies a user-specified forward transformation to y during fitting and an inverse transformation to predictions, enabling you to train on a transformed target while returning predictions in the original scale.

- Primary class: sklearn.compose.TransformedTargetRegressor

- Key idea: Fit a regressor on a transformed target y′ = func(y) and, after prediction, map predictions back to the original y using inverse_func.

- Typical use cases:
  - Stabilize variance with a log or Box-Cox-like transform (y > 0).
  - Enforce a monotonic relationship that is easier to learn for the regressor.
  - Predict in a transformed space but report results in the original scale.

- Important notes:
  - The inverse_transform must be the (preferably close) inverse of the forward transform to ensure meaningful predictions.
  - You can pass additional keyword arguments to the transform functions via kw_args.
  - If check_inverse is enabled, the wrapper verifies that the provided func and inverse_func behave as inverses on a sample of inputs.
  - The transformation is applied to the target only (not to X).

API reference
- class: TransformedTargetRegressor (sklearn.compose)
  - regressor: Estimator implementing fit and predict (e.g., LinearRegression, Ridge, RandomForestRegressor).
  - func: Callable that transforms the target y during fitting (e.g., np.log1p, np.sqrt, custom function). If None, no transformation is applied.
  - inverse_func: Callable that inverse-transforms the predictions back to the original scale (e.g., np.expm1, inverse_sqrt, custom function). If None, no inverse transformation is applied.
  - kw_args: dict, optional. Extra keyword arguments passed to both func and inverse_func.
  - check_inverse: bool, default True. If True, check that inverse_func(func(y)) ≈ y.
  - transform_y_in_fit: alias for func; kept for backward compatibility in older versions.
  - validate: bool, default False. If True, enable some basic validation checks (depends on version).
  - Notes:
    - The transformation is applied to y in fit and to the regressor’s predictions in predict.
    - The wrapper preserves the interface of the underlying regressor as much as possible.

Usage overview
- Typical pattern:
  - Construct a regressor.
  - Wrap it with TransformedTargetRegressor, supplying a forward and inverse function.
  - Fit on (X, y) as usual.
  - Predict and obtain outputs in the original target scale.

Quick start example
- Stabilize variance with a log1p transform (for y > 0) and revert with expm1:
  - This is common when the target is positive and skewed.

  from sklearn.datasets import make_regression
  from sklearn.linear_model import Ridge
  from sklearn.compose import TransformedTargetRegressor
  from sklearn.metrics import mean_squared_error
  import numpy as np

  X, y = make_regression(n_samples=200, noise=0.1, random_state=0)
  reg = Ridge(alpha=1.0)

  tr = TransformedTargetRegressor(
      regressor=reg,
      func=np.log1p,
      inverse_func=np.expm1
  )

  tr.fit(X, y)
  y_pred = tr.predict(X)
  rmse = mean_squared_error(y, y_pred, squared=False)
  print(f"RMSE (original scale): {rmse:.4f}")

Worked example: with a monotonic, non-linear transformation
- If y is strictly positive and you want a smoother learning target, you might use a square root transform:

  tr = TransformedTargetRegressor(
      regressor=SomeRegressor(),
      func=np.sqrt,
      inverse_func=lambda z: z**2
  )

  tr.fit(X, y)
  y_pred = tr.predict(X)

Notes on design and behavior
- Inverse consistency:
  - When check_inverse is True, the wrapper checks that inverse_func(func(y)) ≈ y on a sample of input values. This helps catch mistakes where the pair of functions are not true inverses.
- Kw_args:
  - If your transform function requires fixed parameters (e.g., a Box-Cox lambda), pass them via kw_args. For example:
    tr = TransformedTargetRegressor(
        regressor=SomeRegressor(),
        func=lambda y, lam: boxcox(y, lmbda=lam),
        inverse_func=lambda z, lam: inv_boxcox(z, lmbda=lam),
        kw_args={'lam': 0.5}
    )
- Handling NaNs:
  - Transformations that require strictly positive values (e.g., log, Box-Cox) require data preprocessing to handle zeros or negatives. Ensure you clean or suitably preprocess y before applying the transformation.
- Multi-output targets:
  - TransformedTargetRegressor operates on a 1D target y. For multi-output regression, you may need to wrap a separate TransformedTargetRegressor for each target or use aMultiOutputRegressor in combination, depending on your modeling setup.

Advanced usage
- Pipeline integration:
  - TransformedTargetRegressor can be used standalone or inside a Pipeline. The target transformation is applied inside the estimator, so it does not affect transformations applied to X features.
- Choosing transforms:
  - Use transforms that are both differentiable (if your regressor relies on gradient-based optimization) and invertible, to preserve interpretability of predictions.

Common pitfalls
- Inconsistent transforms:
  - If the forward and inverse transforms are not proper inverses, predictions may be biased or inconsistent. Use check_inverse or test the pair on representative inputs.
- Non-positive targets:
  - If y contains non-positive values and you attempt a log or Box-Cox-like transform, you will encounter errors. Ensure y satisfies the transform’s domain.
