# Cross-validation: evaluating estimator performance

Cross-validation provides a principled way to estimate how an estimator will generalize to unseen data. By repeatedly splitting the data into training and validation sets, cross-validation helps assess model performance, compare different estimators, and diagnose bias vs. variance issues. This section describes how to perform cross-validation, interpret the results, and visualize performance with the built-in plotting utilities.

## Overview

- Purpose: Estimate generalization performance on unseen data and compare estimators or hyperparameters.
- Core idea: Split the data into training and validation folds, fit the model on training data, evaluate on validation data, and aggregate results across folds.
- Common patterns:
  - cross_val_score: obtains a single score per fold.
  - cross_validate: obtains multiple metrics (e.g., train/test scores, fit/score times) per fold.
  - learning_curve: shows how training set size affects performance, useful for bias/variance diagnosis.
  - validation_curve: shows performance as a function of a hyperparameter.
- Visualizations: Use LearningCurveDisplay and ValidationCurveDisplay (or their from_estimator helpers) to plot results with uncertainty bands or error bars.

## Key concepts

- CV strategies
  - KFold: random splits into k folds (default for many scenarios).
  - StratifiedKFold: preserves class distribution in each fold (recommended for classification).
  - TimeSeriesSplit: maintains temporal order (useful for time-series data).
  - LeaveOneOut, GroupKFold, and others for specialized needs.
- Scoring
  - Use the scoring parameter to specify the performance metric (e.g., "accuracy", "precision", "recall", "roc_auc", "neg_mean_squared_error").
  - Some metrics are negative by convention (e.g., neg_mean_squared_error). The plotting utilities can negate such scores for display purposes to reflect the original metric.
- Reproducibility
  - Set random_state where applicable (e.g., in the CV splitter) to obtain repeatable results.
- Visualization
  - Training vs. validation scores across folds help diagnose bias (underfitting) vs. variance (overfitting).
  - Learning curves show how performance scales with more training data.
  - Validation curves show how performance changes with hyperparameter values.

## How to measure performance

- Basic score per fold
  - Use cross_val_score(estimator, X, y, cv=..., scoring=...).
  - Returns an array of scores, one per fold.
- Multiple metrics and timings
  - Use cross_validate(estimator, X, y, cv=..., scoring=..., return_train_score=..., return_times=...).
  - Returns a dict with per-fold metrics (e.g., 'test_score', 'train_score', 'fit_time', 'score_time').
- Visualizing learning curves
  - Use learning_curve to compute training sizes, training scores, and test scores.
  - Visualize with LearningCurveDisplay (recommended via LearningCurveDisplay.from_estimator).
- Visualizing validation curves
  - Use validation_curve to study how a hyperparameter affects performance.
  - Visualize with ValidationCurveDisplay (or the equivalent helper).

## Visualization utilities

- LearningCurveDisplay
  - Visualizes learning curves for training and test scores as a function of training set size.
  - Supports different std display styles: mean with fill_between, or error bars.
  - Methods:
    - LearningCurveDisplay.from_estimator(estimator, X, y, train_sizes=..., scoring=..., cv=..., n_jobs=...)
      - Convenience helper to compute and plot a learning curve for a given estimator.
    - plot() to render the figure.
- ValidationCurveDisplay (recommended)
  - Visualizes performance as a function of a hyperparameter value.
  - Similar API to LearningCurveDisplay for consistency.

Note: In the underlying implementation, the plotting supports negating scores for display when needed (e.g., for neg_mean_squared_error). The display also chooses an appropriate x-scale (linear, log, or symlog) and sets the y-axis label to the score name.

## Examples

- Example 1: Basic k-fold cross-validation for a classifier

  - Objective: Evaluate accuracy with 5-fold cross-validation on a classification dataset.

  - Code:
    ```python
    from sklearn.model_selection import cross_val_score
    from sklearn.datasets import load_iris
    from sklearn.linear_model import LogisticRegression

    X, y = load_iris(return_X_y=True)
    clf = LogisticRegression(max_iter=1000, random_state=42)

    scores = cross_val_score(clf, X, y, cv=5, scoring="accuracy")
    print("CV accuracy: %0.3f (+/- %0.3f)" % (scores.mean(), scores.std() * 2))
    ```

- Example 2: Cross-validation with multiple metrics

  - Objective: Obtain both test and train scores for each fold, plus timing information.

  - Code:
    ```python
    from sklearn.model_selection import cross_validate
    from sklearn.datasets import load_breast_cancer
    from sklearn.ensemble import RandomForestClassifier

    X, y = load_breast_cancer(return_X_y=True)
    clf = RandomForestClassifier(n_estimators=100, random_state=42)

    cv_results = cross_validate(
        clf, X, y, cv=5, scoring=("accuracy", "f1"), return_train_score=True
    )

    print("Test accuracy: %0.3f (+/- %0.3f)" % (cv_results["test_accuracy"].mean(), cv_results["test_accuracy"].std() * 2))
    print("Train F1: %0.3f (+/- %0.3f)" % (cv_results["train_f1"].mean(), cv_results["train_f1"].std() * 2))
    ```

- Example 3: Learning curve visualization

  - Objective: Visualize how training size affects training and test accuracy.

  - Code:
    ```python
    import numpy as np
    from sklearn.model_selection import LearningCurveDisplay
    from sklearn.model_selection import train_test_split
    from sklearn.datasets import load_iris
    from sklearn.linear_model import LogisticRegression

    X, y = load_iris(return_X_y=True)
    est = LogisticRegression(max_iter=1000, random_state=42)

    disp = LearningCurveDisplay.from_estimator(
        est,
        X, y,
        train_sizes=np.linspace(0.1, 1.0, 5),
        scoring="accuracy",
        cv=5,
        n_jobs=-1,
    )
    disp.plot()
    ```

- Example 4: Validation curve for a hyperparameter

  - Objective: Study how a hyperparameter (e.g., C for a linear SVM) affects accuracy.

  - Code:
    ```python
    from sklearn.model_selection import ValidationCurveDisplay
    from sklearn.svm import SVC
    from sklearn.datasets import load_digits

    X, y = load_digits(return_X_y=True)
    svc = SVC(kernel="linear", random_state=42)

    disp = ValidationCurveDisplay.from_estimator(
        svc,
        X, y,
        param_name="C",
        param_range=[0.5, 1.0, 1.5, 2.0],
        scoring="accuracy",
        cv=5
    )
    disp.plot()
    ```

## Best practices

- Choose a CV strategy appropriate to the data:
  - Classification with imbalanced classes: use StratifiedKFold.
  - Time-series data: use TimeSeriesSplit to respect temporal order.
  - Small datasets: LeaveOneOut or ShuffleSplit can be explored, but beware high variance.
- Align scoring with the deployment objective (e.g., use f1 for imbalanced classes, roc_auc for ranking tasks).
- Use cross_validate when you need additional information (train scores, fit times) beyond a single test score.
- Use LearningCurveDisplay.from_estimator to quickly inspect how performance scales with data.
- Use ValidationCurveDisplay to understand how hyperparameters impact performance and to guide hyperparameter tuning.

## Developer notes

- The display classes encapsulate plotting logic, handling:
  - Mean and standard deviation across folds
  - Different std display styles: fill_between, errorbar, or None
  - Automatic axis scaling (linear/log) with sensible fallbacks
  - Optional negation of scores for display when metrics are defined as negative values (e.g., neg_mean_squared_error)

- The visuals are designed to be compatible with a variety of scoring schemas, providing consistent naming and labeling through score_name and negation handling.
