Feature Extraction: Text — Documentation
=========================================

Overview
--------

The sklearn.feature_extraction.text module provides utilities to convert raw text data into numeric feature vectors suitable for machine learning. It includes vectorizers based on the bag-of-words (BoW) model and the hashing trick, as well as a TF-IDF transformer to reweight word counts by their informativeness.

Key components
--------------

- CountVectorizer: Converts a collection of text documents to a matrix of token counts.
- TfidfVectorizer: A combination of CountVectorizer and TfidfTransformer; produces TF-IDF features directly.
- HashingVectorizer: Uses the hashing trick to map tokens to a fixed-dimensional feature space.
- TfidfTransformer: Transforms a count matrix to a TF-IDF representation (can be used with CountVectorizer).
- ENGLISH_STOP_WORDS: Built-in stop word list for English.
- strip_accents_unicode / strip_accents_ascii: Helpers to remove diacritics from text.
- strip_tags: Very lightweight HTML/XML tag stripper.
- Internal helpers: _preprocess, _analyze provide extensibility for preprocessing pipelines, tokenization, and analyzers.

Architecture and design
-----------------------

- All vectorizers in this module implement scikit-learn’s Transformer API, following BaseEstimator, TransformerMixin, etc.
- Tokenization and preprocessing are highly customizable via:
  - preprocessor
  - tokenizer
  - analyzer
  - ngrams
  - stop_words
- Stop words support can use the built-in ENGLISH_STOP_WORDS or a custom list.
- Accent handling and stripping are provided via dedicated helpers to normalize text before tokenization.
- The module supports both ASCII-focused and Unicode-aware accent-stripping approaches, with two helper functions:
  - strip_accents_ascii: transliterate accents to ASCII when possible
  - strip_accents_unicode: remove combining diacritics for Unicode strings
- Decoding/input handling is centralized in _VectorizerMixin.decode, supporting input types such as filenames, file-like objects, or raw strings, with configurable encoding and error handling.

Usage basics
------------

1) CountVectorizer (BoW)

- Purpose: convert text documents to a document-term matrix of integer counts
- Typical usage:

  from sklearn.feature_extraction.text import CountVectorizer

  docs = [
      "This is the first document.",
      "This is the second document.",
      "And this is the third one."
  ]

  vect = CountVectorizer()
  X = vect.fit_transform(docs)
  vocab = vect.get_feature_names_out()
  shape = X.shape  # (n_documents, n_features)

- Options you’ll commonly use:
  - stop_words='english' or stop_words=ENGLISH_STOP_WORDS
  - lowercase (default True in many configurations)
  - ngram_range=(1, 2) to include unigrams and bigrams

2) TfidfVectorizer

- Purpose: produce TF-IDF features directly
- Usage:

  from sklearn.feature_extraction.text import TfidfVectorizer

  docs = [...]
  tfidf = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
  X = tfidf.fit_transform(docs)

- Often paired with a machine learning model downstream; you can also use TfidfTransformer separately on a CountVectorizer output if you need more control.

3) HashingVectorizer

- Purpose: apply the hashing trick to map tokens to a fixed-dimensional feature space
- Usage:

  from sklearn.feature_extraction.text import HashingVectorizer

  docs = [...]
  hasher = HashingVectorizer(n_features=2**20, alternate_sign=False)
  X = hasher.fit_transform(docs)

- Pros/Cons:
  - Pros: low memory footprint, no vocabulary to store, scalable
  - Cons: hash collisions possible, no inverse transformation to obtain the vocabulary

4) Stop words and normalization

- ENGLISH_STOP_WORDS provides a built-in stop word list. You can pass stop_words='english' or stop_words=ENGLISH_STOP_WORDS
- Accent stripping helpers:
  - strip_accents_ascii and strip_accents_unicode can be used to normalize text before tokenization (via a custom preprocessor or analyzer)
- strip_tags offers a lightweight HTML/XML tag stripper for preprocessing

Programmatic API overview
-------------------------

- CountVectorizer
  - fit(X): learn the vocabulary from raw text data
  - transform(X): transform raw text data into a document-term matrix
  - fit_transform(X): convenience method
  - parameters commonly adjusted:
    - input, encoding, decode_error
    - lowercase, strip_accents, analyzer, stop_words
    - preprocessor, tokenizer, ngrams
    - max_features, max_df, min_df, vocabulary
- TfidfVectorizer
  - Similar API to CountVectorizer, with integrated TF-IDF weighting
  - Typically uses a built-in or custom analyzer and supports the same preprocessing options
- HashingVectorizer
  - fit_transform operates directly on raw text; no vocabulary is learned
  - n_features controls the dimensionality
  - alternate_sign toggles sign of hashed features (default True in some configurations)
- TfidfTransformer
  - Used to convert a counts matrix to TF-IDF. Useful when you want to apply TF-IDF after a custom vectorization step.

Advanced usage tips
-------------------

- Custom analyzers
  - You can supply an analyzer callable to replace preprocessor, tokenizer, and ngrams steps entirely.
  - If you provide an analyzer, only the decoder remains relevant for decoding input data in vectorizers.
- Custom preprocessing components
  - preprocessor: function(doc) -> str
  - tokenizer: function(doc) -> list[str]
  - ngrams: function(tokens) -> list[str] (or list of tuples for higher-order grams)
- Unicode handling
  - Use strip_accents_ascii for languages with direct ASCII transliterations, or strip_accents_unicode for general Unicode normalization
- Stop words
  - ENGLISH_STOP_WORDS can be merged with other stop word lists or used as-is
  - A custom list can be supplied as a collection, converted to a frozenset internally
- Input handling
  - The vectorizers can accept raw strings or file-like objects depending on the input parameter
  - decode handles filenames, file-like objects, and byte strings with configurable encoding and error handling

Implementation notes
--------------------

- _preprocess(doc, accent_function=None, lower=False): optional chainable preprocessing function that can apply lowercasing and accent-stripping
- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None): orchestrates the mini-pipeline for a single document or delegates to a user-provided analyzer
- _VectorizerMixin.decode(self, doc): centralizes decoding of input data (filename, file-like, or string) into a string
- _check_stop_list(stop): resolves the stop words input to a concrete set (built-in English list, or user-supplied)
- strip_accents_unicode / strip_accents_ascii / strip_tags: small, fast helpers built into the module to simplify common text preprocessing tasks

Examples
--------

- Example 1: Basic CountVectorizer with English stop words

  from sklearn.feature_extraction.text import CountVectorizer

  documents = [
      "The quick brown fox jumps over the lazy dog.",
      "Never jump over the lazy dog quickly."
  ]
  vectorizer = CountVectorizer(stop_words="english")
  X = vectorizer.fit_transform(documents)
  print(X.shape)  # (2, n_features)
  print(vectorizer.get_feature_names_out()[:10])

- Example 2: TF-IDF with unigrams and bigrams

  from sklearn.feature_extraction.text import TfidfVectorizer

  docs = [
      "The quick brown fox",
      "The quick blue hare"
  ]
  tfidf = TfidfVectorizer(stop_words=ENGLISH_STOP_WORDS, ngram_range=(1, 2))
  X = tfidf.fit_transform(docs)

- Example 3: HashingVectorizer with fixed feature space

  from sklearn.feature_extraction.text import HashingVectorizer

  docs = ["Sample document for hashing."]
  hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
  X = hasher.fit_transform(docs)

- Example 4: Custom analyzer with HTML stripping

  from sklearn.feature_extraction.text import CountVectorizer

  import re
  TAG_RE = re.compile(r"<[^>]+>")

  def custom_analyzer(doc):
      # remove HTML tags
      text = TAG_RE.sub(" ", doc)
      # simple whitespace tokenizer
      return text.lower().split()

  vect = CountVectorizer(analyzer=custom_analyzer)
  X = vect.fit_transform(["<p>Hello world!</p>", "<p>Hello again!</p>"])

API reference (high level)
--------------------------

- CountVectorizer
  - fit(X, y=None)
  - transform(X)
  - fit_transform(X, y=None)
  - get_feature_names_out()
  - vocabulary_
- TfidfVectorizer
  - fit, transform, fit_transform
  - get_feature_names_out()
  - vocabulary_
- HashingVectorizer
  - transform, fit_transform
  - get_feature_names_out() is not applicable (no vocabulary)
- TfidfTransformer
  - fit, transform, fit_transform
  - idf_, tfidf_ properties
- ENGLISH_STOP_WORDS
  - Set-like collection of stop words
- strip_accents_ascii / strip_accents_unicode / strip_tags
  - Pure function utilities for preprocessing

Notes for contributors
----------------------

- This module is designed to be extensible via the analyzer/preprocessor/tokenizer framework.
- Keep custom analyzers lean and well-documented; tests should cover edge cases with punctuation, unicode, and mixed case inputs.
- Ensure compatibility with both ASCII and Unicode inputs; document any limitations related to encoding/decoding.

This documentation provides a concise, practical guide for developers using sklearn.feature_extraction.text, including core usage patterns, customization options, and example workflows.
