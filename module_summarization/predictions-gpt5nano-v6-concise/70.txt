kernel_approximation.rst
========================

Overview
--------
The kernel_approximation module provides approximate kernel feature maps designed to
enable scalable kernel methods by explicitly mapping inputs to a finite-dimensional
feature space. The maps are based on Fourier transforms and Count Sketches, enabling
fast, low-memory approximations of common kernels (e.g., polynomial, RBF, chi-squared,
and skewed chi-squared). These features can then be used with linear models to approximate
kernel methods efficiently.

This module contains several estimators that implement different approximation schemes, such as
Tensor Sketch-based polynomial kernel approximation, random Fourier feature maps for RBF kernels,
and other kernel-specific approximations. See the See Also section for related estimators.

Quick start
-----------
- Install scikit-learn (with dependencies) and import the estimator.
- Create an approximate kernel feature mapper.
- Transform data and (optionally) fit a downstream linear model.

Example: Polynomial kernel approximation with Tensor Sketch
----------------------------------------------------------
- This example demonstrates fitting and transforming data using PolynomialCountSketch and then
  training a simple classifier.

.. code-block:: python

   from sklearn.kernel_approximation import PolynomialCountSketch
   from sklearn.linear_model import SGDClassifier
   from sklearn.datasets import make_classification

   # Create synthetic data
   X, y = make_classification(n_samples=200, n_features=4, random_state=0)

   # Polynomial kernel approximation (degree=3)
   ps = PolynomialCountSketch(degree=3, random_state=1)

   # Fit and transform
   X_features = ps.fit_transform(X)

   # Train a linear classifier on the approximate features
   clf = SGDClassifier(max_iter=1000, tol=1e-3, random_state=0)
   clf.fit(X_features, y)

   # Evaluate
   score = clf.score(X_features, y)
   print("Training accuracy with PolynomialCountSketch features:", score)

Notes
-----
- The feature map dimensionality is controlled by `n_components`.
- Random hash functions (indexHash_ and bitHash_) are generated in `fit` and reused in `transform`.
- For reproducible results, provide an integer `random_state`.

API reference
-------------

Module: sklearn.kernel_approximation
------------------------------------

- PolynomialCountSketch
  - Description
    - Approximate feature map for the polynomial kernel using Tensor Sketch and FFT.
    - Maps X to a finite-dimensional feature space such that the inner product approximates
      K(X, Y) = (gamma * <X, Y> + coef0)^degree.
    - Based on Count Sketch with two-wise independent hash functions, implemented via FFTs.
    - See the user guide section on polynomial kernel approximation for more details.

  - Parameters
    - gamma : float, default=1.0
      Parameter of the polynomial kernel (multiplier of the inner product).
    - degree : int, default=2
      Degree of the polynomial kernel to approximate.
    - coef0 : int, default=0
      Independent term in the polynomial kernel; contributes to the bias term in the feature map.
    - n_components : int, default=100
      Dimensionality of the output feature space. Typically larger than the input feature count.
    - random_state : int or RandomState, default=None
      Seed or RandomState for initializing the Count Sketch hash functions.

  - fit(X, y=None)
    - Fit the model to X. Initializes internal hash tables used for the Count Sketch-based map.
    - Returns self.

  - transform(X)
    - Generate the approximate kernel feature map for X.
    - Returns an array of shape (n_samples, n_components).

  - Attributes
    - indexHash_ : ndarray of shape (degree, n_features), dtype=int64
      Hash indices in the range [0, n_components) used for the Count Sketch.
    - bitHash_ : ndarray of shape (degree, n_features), dtype=float32
      Random {+1, -1} signs used in the Count Sketch.
    - n_features_in_ : int
      Number of features seen during fit.
    - feature_names_in_ : ndarray of shape (n_features_in_,)
      Feature names seen during fit, if X had named features (strings).

  - Version added
    - 0.24

  - See Also
    - AdditiveChi2Sampler: Approximate feature map for additive chi-squared kernel.
    - Nystroem: Approximate a kernel map using a subset of the training data.
    - RBFSampler: Approximate a RBF kernel feature map using random Fourier features.
    - SkewedChi2Sampler: Approximate feature map for "skewed chi-squared" kernel.
    - sklearn.metrics.pairwise.kernel_metrics: List of built-in kernels.

Compatibility and notes
-----------------------
- This module relies on numpy and scipy. Some functionality uses FFT-based computations;
  ensure SciPy >= 1.4 for fft/ifft (fallback to fftpack for older versions).
- The estimators are designed to be used with scikit-learn pipelines and cross-validation.
- Feature name handling (feature_names_in_) is available when input X provides string feature names.

Implementation details (polynomial count sketch)
----------------------------------------------
The PolynomialCountSketch estimator implements a Tensor Sketch approach to approximate the feature map of the polynomial kernel
K(X, Y) = (gamma * <X, Y> + coef0)^degree. It uses two-wise independent Count Sketch hash functions, which are
represented by indexHash_ (the hashed feature indices) and bitHash_ (the ±1 signs). During fit, random hash
tables are generated; during transform, the sketch is applied to each sample to produce a fixed-dimensional
feature representation. The resulting features can be fed into linear models to approximate polynomial-kernel classifiers
efficiently on large datasets.
