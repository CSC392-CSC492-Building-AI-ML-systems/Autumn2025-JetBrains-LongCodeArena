# Analyzing a collection of text documents

This section describes how to analyze a collection of text documents using scikit-learn’s text feature extraction utilities. It covers tokenization, preprocessing, n-grams, stop words, and how to transform raw text into numerical feature vectors suitable for machine learning models.

## Overview

- Convert a collection of documents into numeric feature vectors.
- Primary tools: CountVectorizer, TfidfVectorizer, HashingVectorizer, and TfidfTransformer.
- Build a processing pipeline by chaining preprocessing, tokenization, and n-gram generation.
- Support for decoding different input sources (in-memory strings, filenames, or file-like objects).
- Flexible handling of stop words, accents, and HTML/XML tags.

## Key concepts

- Analyzer vs. preprocessor/tokenizer/ngrams:
  - If you supply an analyzer, the vectorizer uses it directly and bypasses the separate preprocessor, tokenizer, and n-grams steps.
  - If no analyzer is provided, you can supply a preprocessor, a tokenizer, and/or an n-grams callable to customize the tokenization pipeline.
- Input sources:
  - input can be content, filename, or file. The internal decode method handles reading and decoding as needed.
- Stop words:
  - Built-in stop word lists (e.g., English) can be selected via stop_words or None to disable filtering.
  - Custom stop word collections are supported.
- Accent and HTML handling:
  - Utilities exist to strip accents (unicode or ASCII), and to remove HTML/XML tags prior to tokenization.

## API overview (for developers)

- CountVectorizer: Converts a collection of text documents to a matrix of token counts.
- TfidfVectorizer: Similar to CountVectorizer but builds a TF-IDF representation.
- HashingVectorizer: Uses the hashing trick to map features to a fixed-dimensional sparse vector (memory-efficient for large corpora).
- TfidfTransformer: Converts a count matrix to a TF-IDF representation (used when you want more control or to transform an existing count matrix).

Internal helpers (for reference; typically not used directly):
- _preprocess(doc, accent_function=None, lower=False): Optional preprocessing steps such as lowercasing and accent handling.
- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None): Chains the preprocessing and tokenization steps to produce tokens or n-grams.
- strip_accents_unicode, strip_accents_ascii: Utilities to normalize/strip accents.
- strip_tags: Basic HTML/XML tag stripper.
- ENGLISH_STOP_WORDS: Built-in English stop word set used by _check_stop_list.

## Input data and configuration

- input parameter:
  - 'content' (default): treats each input as raw text content.
  - 'filename': reads each item as a filename and loads its bytes.
  - 'file': reads from a file-like object.
- encoding and error handling are configurable through the vectorizer's parameters.

- stop words:
  - stop_words='english' uses a built-in English stop word list.
  - stop_words=None disables stop word filtering.
  - stop_words can be a custom collection of words.

- accent and tag handling:
  - You can supply custom accent handling via accent_function when preprocessing.
  - Use strip_accents_unicode or strip_accents_ascii to normalize characters before tokenization.
  - strip_tags can be applied to remove HTML/XML tags before tokenization.

## Quick-start examples

1) Basic word counts with CountVectorizer

```python
from sklearn.feature_extraction.text import CountVectorizer

docs = [
    "The quick brown fox jumped over the lazy dog.",
    "The quick blue fox was very quick."
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(docs)

print(X.shape)  # (2, n_features)
print(vectorizer.get_feature_names_out()[:10])
```

2) TF-IDF representation with English stop words

```python
from sklearn.feature_extraction.text import TfidfVectorizer

docs = [
    "The quick brown fox jumped over the lazy dog.",
    "The quick blue fox was very quick."
]

tfidf = TfidfVectorizer(stop_words="english")
X = tfidf.fit_transform(docs)

print(X.shape)  # (2, n_features)
print(tfidf.get_feature_names_out()[:10])
```

3) Custom n-grams (bigrams) with a simple tokenizer

```python
from sklearn.feature_extraction.text import CountVectorizer
import re

def simple_tokenizer(text):
    return re.findall(r"\b\w+\b", text.lower())

vectorizer = CountVectorizer(tokenizer=simple_tokenizer, ngram_range=(1, 2))
X = vectorizer.fit_transform([
    "This is a sample. This sample is simple.",
    "A different example text for testing."
])

print(X.shape)
print(vectorizer.get_feature_names_out()[:20])
```

4) Stripping HTML tags and removing accents

```python
from sklearn.feature_extraction.text import CountVectorizer

docs = [
    "<p>The café had a crème brûlée.</p>",
    "<div>Résumé of the project: success.</div>"
]

vectorizer = CountVectorizer(
    strip_accents="ascii",
    lowercase=True
)

# You can apply strip_tags and strip_accents_ascii outside before tokenization,
# or implement a custom preprocessor/tokenizer to apply them as part of _preprocess/_analyze.
X = vectorizer.fit_transform(docs)
print(X.shape)
```

5) HashingVectorizer for large corpora

```python
from sklearn.feature_extraction.text import HashingVectorizer

docs = [
    "The quick brown fox.",
    "Lazy dogs sleep."
]

hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
X = hasher.fit_transform(docs)

print(X.shape)  # (2, 262144)
```

## Advanced usage and customization

- Provide a custom analyzer:
  - If you supply an analyzer callable, the vectorizer will bypass preprocessor, tokenizer, and ngrams steps and apply the analyzer directly to each document.
- Compose a pipeline:
  - Combine preprocessing, tokenization, and feature extraction in a scikit-learn Pipeline for clean and reusable workflows.
- Stop words and languages:
  - Use ENGLISH_STOP_WORDS for a built-in list or supply your own stop word set to tailor vocabulary filtering to your domain.
- Accent and tag handling:
  - Use strip_accents_unicode or strip_accents_ascii to normalize text before tokenization.
  - Use strip_tags to remove HTML/XML markup when processing web-scraped data.

## Best practices

- For large document collections, prefer HashingVectorizer or sparse matrices to manage memory efficiently.
- Experiment with ngram ranges (e.g., (1, 1) vs. (1, 2) or (2, 3)) to capture more contextual information.
- Combine stop words removal with suitable preprocessing to reduce noise without discarding informative terms.
- Validate with a simple baseline (e.g., unigrams with English stop words) before adding complexity.

## Reference (developer notes)

- _preprocess: centralizes optional lowercasing and accent handling.
- _analyze: builds the token/gram sequence from a document by applying decoder, analyzer, or chained preprocessor/tokenizer/ngrams.
- strip_accents_unicode / strip_accents_ascii: utilities to normalize or remove diacritics.
- strip_tags: a lightweight HTML/XML stripper to sanitize input prior to tokenization.
- ENGLISH_STOP_WORDS: default English stop word list used by the _check_stop_list helper.

If you need deeper customization, you can implement your own tokenizer, preprocessor, or n-gram generator and pass them to CountVectorizer or TfidfVectorizer via the tokenizer, preprocessor, and ngrams parameters, or supply a fully custom analyzer.
