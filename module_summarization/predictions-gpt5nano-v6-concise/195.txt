Learn Gaussian Mixture Models (GMMs)
=====================================


Overview
--------

Gaussian Mixture Models (GMMs) are a probabilistic approach to clustering and density estimation. A GMM assumes that the data are generated from a mixture of K Gaussian components, each with its own mean and covariance. The overall data distribution is expressed as:

- p(x) = sum_{k=1}^K pi_k * N(x | mu_k, Sigma_k)

where:
- pi_k are the mixture weights (non-negative and sum to 1),
- N(x | mu_k, Sigma_k) is the multivariate Gaussian density with mean mu_k and covariance Sigma_k.

Learning a GMM involves estimating the parameters {pi_k, mu_k, Sigma_k} from data X. The standard approach is the Expectation-Maximization (EM) algorithm, which alternates between:
- E-step: compute responsibilities r_ik = p(z_i = k | x_i, theta) for each sample i and component k.
- M-step: update the parameters to maximize the expected complete-data log-likelihood given the responsibilities.

This implementation follows the classical EM procedure, with several practical options to control initialization, convergence, and numerical stability.

Mathematical formulation
------------------------

- Latent variables: z_i ∈ {1, ..., K} indicate the generating component for each data point x_i.
- Complete-data likelihood: L_complete = ∏_i ∏_k [ pi_k * N(x_i | mu_k, Sigma_k) ]^{1(z_i = k)}
- E-step computes responsibilities r_ik = p(z_i = k | x_i, theta) using current parameters.
- M-step updates:
  - pi_k = (1/N) ∑_i r_ik
  - mu_k = (∑_i r_ik x_i) / (∑_i r_ik)
  - Sigma_k = (∑_i r_ik (x_i - mu_k)(x_i - mu_k)^T) / (∑_i r_ik) + reg_covar * I
- reg_covar is a regularization term to ensure numerical stability and positive-definiteness of Sigma_k.

Initialization and convergence
------------------------------

Initialization strategies to obtain reasonable starting points for the EM iterations:

- kmeans: initialize responsibilities from a K-means clustering assignment.
- random: initialize responsibilities randomly and normalize per row.
- random_from_data: select K data points as initial component representatives.
- k-means++: initialize using a k-means++ seeding strategy for better spread.

Training options:
- max_iter: maximum EM iterations per run.
- tol: convergence threshold on the change in log-likelihood or lower bound.
- n_init: number of independent EM runs with different initializations; the best run (highest likelihood) is kept.
- warm_start: reuse the solution from a previous fit to continue training.
- random_state: seed control for reproducibility.
- verbose and verbose_interval: logging of progress.

Implementation (BaseMixture)
---------------------------

Key responsibilities of the base class used for Gaussian mixtures:

- BaseMixture is an abstract class providing the EM framework. It defines:
  - Parameter constraints (n_components, tol, reg_covar, max_iter, n_init, init_params, random_state, warm_start, verbose, verbose_interval).
  - Initialization flow via _initialize_parameters(X, random_state), which delegates to concrete strategies and then calls _initialize(X, resp).
  - Abstract hooks _check_parameters(X) and _initialize(X, resp) to be implemented by the concrete mixture (e.g., GaussianMixture).
  - fit(X, y=None) and fit_predict(X, y=None) for training and label prediction.

- Initialization parameters
  - init_params supports {"kmeans", "random", "random_from_data", "k-means++"}.

- Validation and stability
  - The internal helper _check_shape(param, param_shape, name) validates parameter shapes when a derived class sets initial values.
  - reg_covar provides diagonal regularization to prevent singular covariance estimates.

- Model updates
  - The EM loop is performed within the base class, while the concrete subclass provides specifics for parameter initialization and parameter updates (via _initialize and _check_parameters).

Extending BaseMixture: implementing a GaussianMixture subclass
--------------------------------------------------------------

To implement a Gaussian Mixture model with this base class, you typically define:

- _check_parameters(X)
  - Validate input matrix X (shape, dtype).
  - Ensure features dimension is compatible with the model (n_features > 0).

- _initialize(X, resp)
  - Given the responsibilities resp (shape: n_samples x n_components), compute initial Gaussian parameters:
    - weights_ or similar array of size (n_components,)
    - means_ or similar array of shape (n_components, n_features)
    - covariances_ or similar array of shape (n_components, n_features, n_features)
  - Add reg_covar to the diagonals for numerical stability.

- Optional: define additional EM helper steps (e.g., _estimate_covariances) if you want more control over the M-step. The base class expects the concrete subclass to manage parameter updates within the EM cycle.

Example: skeleton for a GaussianMixture subclass (conceptual)
--------------------------------------------------------------

Note: The exact attribute names may vary in the complete library; adapt to the concrete implementation you are using.

.. code-block:: python

    import numpy as np
    from your_library import BaseMixture

    class GaussianMixture(BaseMixture):
        def __init__(self, n_components, tol=1e-3, reg_covar=1e-6,
                     max_iter=100, n_init=1, init_params="kmeans",
                     random_state=None, warm_start=False, verbose=0,
                     verbose_interval=10):
            super().__init__(n_components, tol, reg_covar, max_iter, n_init,
                             init_params, random_state, warm_start,
                             verbose, verbose_interval)

        def _check_parameters(self, X):
            X = np.asarray(X)
            if X.ndim != 2:
                raise ValueError("X must be a 2D array of shape (n_samples, n_features)")
            if X.shape[1] <= 0:
                raise ValueError("X must have at least one feature")
            # Additional domain-specific checks could go here

        def _initialize(self, X, resp):
            # Example: compute simple moment-based initialization from resp
            n_samples, n_features = X.shape

            # weights
            weights = resp.sum(axis=0) / n_samples

            # means
            means = (resp.T @ X) / resp.sum(axis=0)[:, np.newaxis]

            # covariances
            covariances = np.zeros((self.n_components, n_features, n_features))
            for k in range(self.n_components):
                resp_k = resp[:, k][:, np.newaxis]
                diff = X - means[k]
                cov_k = (diff * resp_k).T @ diff
                covariances[k] = cov_k / (resp[:, k].sum() + 1e-12)
                covariances[k].flat[::covariances[k].shape[0] + 1] += self.reg_covar

            # set learned parameters
            self.weights_ = weights
            self.means_ = means
            self.covariances_ = covariances

Usage: training a Gaussian Mixture model
----------------------------------------

The base class provides a standard EM-based training loop with flexible initialization. After implementing a concrete GaussianMixture subclass as above, you can train on data as follows:

.. code-block:: python

    import numpy as np
    from your_library import GaussianMixture

    # Generate or load data: X is shape (n_samples, n_features)
    X = np.random.randn(300, 2)

    # Create a GMM with 3 components
    gmm = GaussianMixture(
        n_components=3,
        tol=1e-4,
        reg_covar=1e-6,
        max_iter=100,
        n_init=5,
        init_params='kmeans',
        random_state=0,
        verbose=1,
        verbose_interval=10
    )

    # Fit the model
    gmm.fit(X)

    # Access learned parameters (names depend on the concrete implementation)
    print("Weights:", getattr(gmm, "weights_", None))
    print("Means:", getattr(gmm, "means_", None))
    print("Covariances shape:", getattr(gmm, "covariances_", None))

    # Predict labels for X (if provided by the concrete implementation)
    # labels = gmm.predict(X)  # if available
    # Alternatively, use fit_predict(X) for a new run
    # labels = gmm.fit_predict(X)

Notes for developers
--------------------

- Parameter validation
  - The BaseMixture defines _parameter_constraints for validating initialization and hyperparameters. Implementers should ensure compatibility with these constraints and validate any additional parameters you introduce.

- Numerical stability
  - Use reg_covar to regularize covariances and avoid singular matrices.
  - Consider computing log-densities in log-space to maintain numerical stability (e.g., log-sum-exp in the E-step).

- Convergence warnings
  - If the EM updates fail to converge within max_iter iterations, a ConvergenceWarning will be raised. Consider increasing max_iter or adjusting tol in such cases.

- Warm-start
  - When warm_start is True, successive fit calls reuse the previous solution, enabling incremental training.

- Random state reproducibility
  - Use the provided random_state to seed initialization methods (e.g., kmeans, random, k-means++).

- Extensibility
  - The design separates EM orchestration (in BaseMixture) from parameter initialization and update details (implemented in the subclass). This enables alternative mixture models (e.g., mixtures of Gaussians with diagonal covariances, mixtures of other distributions) to reuse the same EM framework.
