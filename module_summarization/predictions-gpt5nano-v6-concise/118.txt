# Configuration.rst — Generate documentation for this code

Overview
--------
This module provides configuration options used to control the export and import of various elements in PyAEDT workflows. It is designed to be attached to an AEDT-based application (e.g., Icepak, HFSS) via the .configurations object. The central entry point is the ConfigurationsOptions class, which exposes a set of boolean toggles and a few structural options to customize what gets exported to or imported from JSON files.

Key components
- _find_datasets(d, out_list): Internal helper that traverses nested structures to extract dataset references (e.g., power waveform definitions) used in parametric studies.
- ConfigurationsOptions: Public API to enable/disable exporting/importing specific components (variables, setups, optimizations, parametrics, boundaries, datasets, materials, etc.) and other related behavior.

Usage pattern
-------------
- Instantiate an AEDT-based application (e.g., Icepak).
- Access the project’s configurations options via app.configurations.options.
- Toggle desired export/import capabilities.
- Optionally adjust structural behaviors such as tolerance or skip behavior when importing.

Example
-------
 Disable exporting datasets:
```
from pyaedt import Icepak
ip = Icepak()
ip.configurations.options.export_datasets = False
```

 Enable exporting only certain items (variables, setups, and materials):
```
ip.configurations.options.export_variables = True
ip.configurations.options.export_setups = True
ip.configurations.options.export_materials = True
ip.configurations.options.export_boundaries = False
```

 Adjust the object mapping tolerance (used for operations like boundary/object mapping):
```
ip.configurations.options.object_mapping_tolerance = 1e-12
```

 Import control (e.g., prevent overwriting existing items during import):
```
ip.configurations.options.import_datasets = True
ip.configurations.options.skip_import_if_exists = True
```

- The ConfigurationsOptions class primarily stores boolean flags and a tolerance value, exposing property getters/setters with docstrings that describe intended usage and defaults.

Developer notes
---------------
- This API is designed to be minimally invasive: toggling a flag affects what gets serialized to JSON during export/import operations performed by the host application.
- The default configuration aims to export/import all major components to provide a complete snapshot, but may be tuned for performance or size considerations by disabling certain exports.
- The tolerance parameter affects how the system handles object mapping, which can be critical for precise boundary alignment and feature matching across imports/exports.
