# w_kinetics

The w_kinetics command is a deprecated shim that exposes the same functionality as the w_direct-based kinetics tooling, offering a dedicated subcommand for trace-based state-to-state kinetics analysis. It computes fluxes, arrivals, and event durations by tracing trajectories, producing an output file suitable for subsequent averaging and rate calculation with confidence intervals via w_kinavg.

Note: This command is deprecated in favor of the w_direct workflow. The shim prints a deprecation warning on invocation and forwards execution to the appropriate w_direct path. See Migration and Compatibility notes for details.

---

## Overview

- Purpose
  - Compute state-to-state kinetics by tracing trajectory data.
  - Produce fluxes, arrivals, and detailed event durations that can be fed to w_kinavg to obtain normalized rates with uncertainties.
- Input requirements
  - A bin/trajectory assignment file (typically assign.h5) that labels trajectories with macrostate assignments.
  - Trajectory data accessed via the underlying DKinetics implementation (from westpa.cli.tools.w_direct).
- Output
  - By default, kintrace.h5 (configurable via -o/--output).
  - Datasets include conditional fluxes, conditional arrivals, total fluxes, arrivals, and durations.
- Relationship to other tools
  - Output is designed for use with w_kinavg to obtain normalized rates.
  - See w_assign for generating the required assignment file.
- Deprecation status
  - This wrapper emits a deprecation warning directing users to use the w_direct-based workflow instead.

## Output format (kintrace.h5)

The output file contains the following datasets:

- /conditional_fluxes [iteration][state][state]
  - Floating-point
  - Macrostate-to-macrostate fluxes (not normalized by the initial state's population)

- /conditional_arrivals [iteration][stateA][stateB]
  - Integer
  - Number of trajectories arriving at stateB in a given iteration, given departure from stateA

- /total_fluxes [iteration][state]
  - Floating-point
  - Total flux into a given macrostate

- /arrivals [iteration][state]
  - Integer
  - Number of trajectories arriving at a given state in a given iteration (regardless of origin)

- /duration_count [iteration]
  - Integer
  - Number of event durations recorded in each iteration

- /durations [iteration][event duration]
  - Structured (compound) dataset with fields:
    - istate (Integer): initial state of the transition
    - fstate (Integer): final state of the transition
    - duration (Floating-point): duration of the transition, in units of tau
    - weight (Floating-point): weight of the trajectory at the end of the transition (not normalized by the initial state population)

Notes
- State-to-state fluxes are not normalized by initial macrostate population; normalization (to rates) is performed by w_kinavg.
- Durations are stored per iteration and are associated with transition events ending in that iteration.
- Durations are expressed in units of tau (the unit used by the simulation time scale).

---

## How it works (conceptual)

- Traces trajectories using the provided assign.h5 to label macrostate transitions.
- Accumulates fluxes between macrostates, arrival counts, and durations of transition events.
- Writes the collected data to kintrace.h5 (or a user-specified output file).
- The output serves as input for w_kinavg to convert fluxes into normalized rates with confidence intervals, accounting for statistical fluctuations and correlations.

Durations are stored with explicit istate and fstate fields, allowing downstream tools to reconstruct the transition-event distribution.

## Command-line interface

- Program name: w_kinetics
- Subcommands: trace
- Default output file: kintrace.h5
- Important input:
  - A trajectory assignment file (assign.h5) containing labeling information (or equivalent labeling input consumed by the underlying DKinetics logic).
- Options:
  - Options are inherited from the DKinetics tool in westpa.cli.tools.w_direct.
  - Typical options include:
    - -o, --output: specify the output file (default kintrace.h5)
    - --assign: path to the assignment file (e.g., assign.h5)
    - Other kernel options defined by DKinetics (see w_direct --help or w_kinetics trace --help for exact listing)

Note: The top-level w_kinetics command is a compatibility shim. It prints a deprecation warning and forwards to the underlying w_direct-based implementation.

## Examples

- Basic usage (default output, using default conventions)
  - w_kinetics trace

- Specify an output file
  - w_kinetics trace -o kintrace.h5

- Provide an assignment file explicitly
  - w_kinetics trace --assign assign.h5 -o kintrace.h5

- Inspect help for trace subcommand
  - w_kinetics trace --help

- Migration path (recommended): use the w_direct-based workflow directly
  - Prefer the modern workflow provided by w_direct for kinetics analysis and related tooling.

## Developer notes

- Purpose of this shim
  - The WKinetics class is a minimal subclass of DKinetics with:
    - subcommand = 'trace'
    - help_text describing the trace-based kinetics analysis
    - default_output_file = 'kintrace.h5'
  - The WDirect class wires the command into WESTMasterCommand and WESTParallelTool, exposing the w_kinetics CLI in a backwards-compatible form.
- Deprecation handling
  - entry_point() emits a deprecation warning and redirects execution to the trace subcommand under the new w_direct workflow.
  - The shim ensures that existing scripts and users can continue to operate while the recommended path is migrated.
- Data model details
  - The output schema is designed to pair with w_kinavg, which performs normalization and error analysis to produce rates and confidence intervals.
  - The durations dataset encodes each event with initial/final state, duration, and trajectory weight, enabling post-processing with proper weighting.

## Migration and compatibility

- Prefer using the w_direct-based kinetics tooling going forward.
- The deprecation warning indicates the recommended replacement path and helps identify code that still relies on the w_kinetics shim.
- If you maintain automation or pipelines, update commands to invoke the w_direct workflow directly, then rely on the corresponding trace/kinetics and kinavg steps in that path.
