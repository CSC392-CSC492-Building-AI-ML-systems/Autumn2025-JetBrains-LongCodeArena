Channel Options
===============

Overview
--------
Channel options refer to the configuration you apply to Vega-Lite encoding channels to control how data fields map to visual properties. Each channel (x, y, color, size, shape, opacity, tooltip, etc.) accepts a set of options that determine data type, value aggregation, binning, scaling, axis/legend presentation, and formatting. This document describes common channel options, their usage, and practical examples.

What is an encoding channel?
----------------------------
An encoding channel defines how a data field (or a constant value) is projected onto a visual property of a mark. Example channels include:
- x, y: positional channels
- color, fill, stroke: color-related channels
- size: point/line thickness or area
- shape: glyph type
- opacity: transparency
- tooltip: interactive data-informed hints
- detail, text, href, url, order, key: additional encodings for more complex visuals

Channel option model
--------------------
For each channel you specify, you typically supply:
- field or datum: the data source (e.g., a field name) or a constant value
- type: the data type (quantitative, nominal, ordinal, temporal)
- aggregate/bin/timeUnit: data reduction or transformation for the channel
- scale: visual transformation (domain, range, nice, etc.)
- axis/legend: axis or legend configuration when the channel is x/y (or color, etc.)
- format: numeric or temporal formatting
- condition: conditional encoding logic
- stack: stacking behavior for relevant charts
- title: channel title overrides (often via axis/legend)

Common channel options (per-channel keys)
------------------------------------------
- field: string naming the data field (e.g., "Horsepower")
- type: one of "quantitative", "nominal", "ordinal", "temporal"
- aggregate: string (e.g., "mean", "sum", "min", "max")
- bin: boolean or object (enables binning for quantitative data)
- timeUnit: string (e.g., "year", "month", "date")
- scale: object with Vega-Lite scale properties (domain, range, nice, padding, scheme, etc.)
- axis: object configuring axis (title, format, labelAngle, grid, etc.)
- legend: object configuring legend (title, format, orient, etc.)
- sort: string or array for ordering (useful for ordinal/nominal)
- format: string/format for axis/tooltip formatting
- title: string title for the channel (often via axis/legend)
- condition: encoding condition with a test expression and corresponding values
- value: constant value used when encoding does not depend on a field
- stack: "zero", "normalize", or null to enable stacking for compatible marks/channels
- tooltip: array or boolean to enable interactive tooltips

Notes:
- Some options only apply to specific channels (e.g., stack applies to y in bar/area/line charts when stacking).
- The combination of field/type/aggregate/bin/timeUnit determines how the data is interpreted on that channel.
- Axis/legend configurations live under the axis/legend keys of the respective channel.

Examples
--------

Example 1: Basic scatterplot with color by category
- Objective: Map horsepower to x, miles per gallon to y, and origin to color.
- Vega-Lite spec (JSON):

.. code-block:: json

{
  "mark": "point",
  "encoding": {
    "x": {"field": "Horsepower", "type": "quantitative"},
    "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
    "color": {"field": "Origin", "type": "nominal"}
  }
}

- Altair-style (Python) snippet:

.. code-block:: python

import altair as alt
import vega_datasets  # optional, for datasets

cars = alt.Data(values=[
    {"Horsepower": 110, "Miles_per_Gallon": 30, "Origin": "USA"},
    {"Horsepower": 150, "Miles_per_Gallon": 24, "Origin": "Japan"},
])

chart = alt.Chart(cars).mark_point().encode(
    x="Horsepower:Q",
    y="Miles_per_Gallon:Q",
    color="Origin:N"
)

Example 2: Bar chart with mean aggregation on the y-channel
- Objective: Category on x, average horsepower on y, color by category.
- Vega-Lite JSON:

.. code-block:: json

{
  "mark": "bar",
  "encoding": {
    "x": {"field": "Origin", "type": "nominal"},
    "y": {"aggregate": "mean", "field": "Horsepower", "type": "quantitative"},
    "color": {"field": "Origin", "type": "nominal"}
  }
}

- Altair-style (Python):

.. code-block:: python

chart = alt.Chart(data).mark_bar().encode(
    x="Origin:N",
    y="mean(Horsepower):Q",
    color="Origin:N"
)

Example 3: Time-based X axis with formatting
- Objective: Time on x-axis (year granularity), numeric y-axis, with a formatted axis.
- Vega-Lite JSON:

.. code-block:: json

{
  "mark": "line",
  "encoding": {
    "x": {"field": "Date", "type": "temporal", "timeUnit": "year"},
    "y": {"field": "Value", "type": "quantitative"},
    "tooltip": [{"field": "Date", "type": "temporal"}, {"field": "Value", "type": "quantitative"}]
  }
}

- Altair-style (Python):

.. code-block:: python

chart = alt.Chart(data).mark_line().encode(
    x="Date:T",
    y="Value:Q",
    tooltip=[alt.Tooltip("Date:T"), alt.Tooltip("Value:Q")]
)

Example 4: Conditional color encoding
- Objective: Color data points red when a condition holds, otherwise blue.
- Vega-Lite JSON:

.. code-block:: json

{
  "mark": "point",
  "encoding": {
    "color": {
      "condition": {"test": "datum.value > 50", "value": "red"},
      "value": "blue"
    }
  }
}

- Altair-style (Python):

.. code-block:: python

chart = alt.Chart(data).mark_point().encode(
    color=alt.condition("datum.value > 50", alt.ColorValue("red"), alt.ColorValue("blue"))
)

Practical guidelines
--------------------
- Choose channels that preserve data semantics:
  - Use x/y for coordinates, color/size/shape for category/measure distinctions.
  - Reserve tooltip to provide detailed values without clutter.
- Favor quantitative channels for quantitative fields and nominal/ordinal for categorical fields.
- Use axis/legend configuration to improve readability (titles, formats, label angles, legend orientation).
- Leverage scale and timeUnit for appropriate data mapping (e.g., time-based x-axis with timeUnit: "year" or "month").
- When dealing with large datasets, consider aggregation to reduce rendering complexity (aggregate on the y-channel, or pre-aggregate in data processing).

Version notes
-------------
Channel options and their exact capabilities depend on Vega-Lite version (the examples above assume Vega-Lite 5). If you are using a wrapper around Vega-Lite (such as Altair integration), the corresponding encoding API should map these options to the underlying Vega-Lite spec accordingly.
