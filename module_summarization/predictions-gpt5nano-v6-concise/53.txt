Overview
- This module implements Multiple Imputation by Chained Equations (MICE) to address missing data in statistical analyses.
- The core workflow:
  - Initialize imputations for missing values.
  - For each variable with missing data (the focus variable), fit an imputation model (a regression of the focus variable on other variables) and draw imputations, using predictive mean matching (PMM) by default.
  - After all focus variables are imputed, fit the analysis model to the completed data.
  - Repeat the cycle and combine results to produce parameter estimates and standard errors.
- Imputation models are defined per variable via a model class and a regression formula. The default is Ordinary Least Squares (OLS) with a formula specifying main effects for the other variables.
- Two usage modes:
  - MICEData: wrap a DataFrame to produce multiply imputed datasets (with optional plots and a history hook).
  - MICE: run imputations and fit an analysis model, then combine results into MICEResults.

Terminology
- Analysis model: the model to be estimated on the imputed datasets (the target of inference).
- Imputation model: one regression model per variable with missing values, conditioning on the other variables.
- Perturbation method: how to set parameter estimates for an imputation model. Options include:
  - gaussian: draw parameters from a Gaussian approximation to the posterior distribution after fitting.
  - boot: use parameters from a bootstrapped version of the dataset.
- Predictive Mean Matching (PMM): the current default imputation method for filling in missing values.
- MICEData: a wrapper around a Pandas DataFrame that manages per-variable imputation models and generates imputed datasets.
- MICE: an orchestrator that runs imputations, fits the analysis model, and combines results.
- MICEResults: the object produced by MICE that contains summary statistics, estimands, and plots related to the analysis results.

API Reference

PatsyFormula
- Purpose: Lightweight wrapper to treat a string as a Patsy formula.
- Initialization: PatsyFormula(formula)
- Behavior: stores the formula with a zero intercept term prepended (via "0 + ").

MICEData
- Purpose: Wrap a dataset to perform MICE-based imputations and provide utilities for inspection and visualization.
- Constructor
  - __init__(data, perturbation_method='gaussian', k_pmm=20, history_callback=None)
  - data: Pandas DataFrame to be imputed (copied internally).
  - perturbation_method: default perturbation for imputation models ('gaussian' or 'boot').
  - k_pmm: number of nearest neighbors used in Predictive Mean Matching (PMM).
  - history_callback: function invoked after each complete imputation cycle. Its return value is appended to a history list; the MICEData object is passed as the sole argument.
- Important attributes and methods (as demonstrated in examples):
  - data: current working DataFrame (with the most recent imputations).
  - set_imputer(variable, formula): define the imputation model for a given focus variable. The formula specifies the predictors (main effects, transforms, interactions, etc.). Example usage is shown below.
  - update_all(): perform one complete cycle of imputations across all focus variables.
  - __iter__(): enable iteration over imputed datasets; each iteration yields a dataset (or a view of the current imputed data).
- Example usage
  - Define an imputer for a variable using a custom formula:
    ```
    from mice import MICEData
    import numpy as np

    imp = MICEData(data)
    imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
    ```
  - Run multiple imputation cycles and save intermediate datasets:
    ```
    for j in range(20):
        imp.update_all()
        imp.data.to_csv('data%02d.csv' % j)
    ```
  - Iterate over imputations:
    ```
    imp = MICEData(data)
    j = 0
    for imputed_data in imp:
        imputed_data.to_csv('data%02d.csv' % j)
        j += 1
    ```
- Notes
  - By default, MICEData is designed to conserve memory. The input dataset is copied on entry and then overwritten with new imputations in each cycle.
  - If using MICE (as opposed to MICEData only), fitted analysis models and results are saved, and summary plots are available.
  - history_callback can be used to capture additional features of interest from intermediate imputations.

MICE
- Purpose: Orchestrate multiple imputation, fit the analysis model, and combine results to produce a MICEResults object.
- Usage pattern:
  - Create a MICE instance with a MICEData object and an analysis model specification.
  - Run the imputation and analysis steps, then call combine to obtain the final results.
  - Access results and summary plots via the MICEResults object.

Notes on Design and Implementation
- Model specification
  - Imputation models are defined per variable with missing values. Each model may include a subset of the remaining variables, with support for transformations and interactions.
  - The default is OLS with main effects; users can customize via set_imputer and Patsy-formatted formulas.
- Perturbation methods
  - gaussian perturbation: sample parameters from a Gaussian approximation to the fit distribution after a model is estimated.
  - boot perturbation: re-fit on bootstrap-resampled data and use the bootstrap estimate as the parameter vector.
- Plotting and diagnostics
  - The module provides plotting utilities to visualize missing data patterns and imputation diagnostics.
  - The history hook allows users to save intermediate imputations or diagnostics for later analysis.

Examples

Example 1: Basic MICEData usage and saving imputations
- Intent: create imputations for a dataset, define an imputation model for x1, and save each completed dataset.
- Code:
  ```
  from mice import MICEData
  import numpy as np

  # Assuming 'data' is a Pandas DataFrame with missing values
  imp = MICEData(data)
  imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
  for j in range(20):
      imp.update_all()
      imp.data.to_csv('data%02d.csv' % j)
  ```

Example 2: Iterating over imputations
- Intent: loop over generated imputations and process each completed dataset.
- Code:
  ```
  from mice import MICEData

  imp = MICEData(data)
  j = 0
  for imputed_data in imp:
      imputed_data.to_csv('data%02d.csv' % j)
      j += 1
  ```
