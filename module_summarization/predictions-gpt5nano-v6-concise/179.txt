# Cross Decomposition

Cross decomposition encompasses a family of methods designed to uncover and quantify the relationships between two data blocks X and Y. In scikit-learn, this is implemented through several algorithms that project X and Y onto a shared latent space in order to capture the underlying association structure. The primary classes exposed are PLSRegression, PLSCanonical, and PLSSVD.

Key ideas
- Goal: Find latent components (scores) that capture the maximum association between X and Y.
- Data representation: X ∈ Rn×p, Y ∈ Rn×q, with n samples.
- Common steps:
  - Centering (and optional scaling) of X and Y.
  - Computing latent vectors (loadings) and latent scores.
  - Deflation of X and Y to iteratively extract multiple components.
- Output: Projections of X and Y onto latent spaces, along with loadings that describe how features map to these components.

Notational recap
- X: data matrix with n samples and p features
- Y: data matrix with n samples and q targets/features
- T: X score matrix (n × n_components)
- U: Y score matrix (n × n_components)
- P, Q: X and Y loadings for the scores (p × n_components, q × n_components)
- W, C (sometimes): weight matrices used to compute scores
- OP: deflation performed on X
- OQ: deflation performed on Y

Algorithms implemented
- PLSSVD
  - Core idea: Use the singular value decomposition (SVD) of XᵀY to obtain the first pair of latent vectors, then deflate X and Y and repeat to extract multiple components.
  - Key properties:
    - Directly optimizes the covariance between X and Y via the SVD of XᵀY.
    - Simple and robust; often used when a symmetric treatment of X and Y is desired.
  - Typical workflow:
    - Compute C = XᵀY
    - SVD: C = UΣVᵀ
    - First component vectors: w = U[:, 0], c = V[0, :]
    - Scores: t = Xw, s = Yc
    - Deflate X and Y: X ← X − t pᵀ, Y ← Y − s qᵀ (where p and q are corresponding loadings)
    - Repeat for subsequent components
- PLSRegression
  - Goal: Model the relationship Y ≈ Ŷ = X B, by extracting latent variables that maximize the covariance between X scores and Y scores.
  - Core steps:
    - Iteratively compute weights (W) and loadings (P for X, Q for Y), and scores (T for X, U for Y) that maximize covariance between T and U (subject to orthogonality/deflation constraints).
    - Solve X = T Pᵀ + E and Y = U Qᵀ + F
    - Regress Y on X via the latent variables (often via a regression of Y on T).
  - Deflation: Update X and Y by removing the contributions of the extracted components to obtain residuals for the next component.
- PLSCanonical
  - Goal: Find paired latent components that maximize the covariance between X scores and Y scores, with symmetric treatment of X and Y.
  - Core steps:
    - Compute sequences of score vectors for X and Y (T and S) such that Cov(T, S) is maximized.
    - Deflation applied to both X and Y to extract subsequent pairs of components.
  - This is particularly useful when the primary interest is the relationship structure between X and Y rather than predicting one from the other.

Data preprocessing
- Centering: Both X and Y are centered by subtracting their mean.
- Scaling (optional): If scale=True, both X and Y are scaled by their feature standard deviations (ddof=1). Features with zero standard deviation are handled gracefully (set to 1.0 for scaling to avoid division by zero).
- Practical note: Centering (and scaling) is essential for meaningful cross-decomposition results, especially when feature scales vary widely.

Internal utilities (brief overview)
- _center_scale_xy: Centers and optionally scales X and Y; returns centered/scaled X, Y and their means and stds.
- _get_first_singular_vectors_svd: Computes the first left/right singular vectors using a full SVD of XᵀY (used by PLSSVD).
- _get_first_singular_vectors_power_method: Alternative to SVD that uses a power iteration to estimate the top singular vectors.
- _pinv2_old, X_pinv, Y_pinv: Pseudo-inverse utilities used in certain modes (e.g., when certain numerical conditions favor a power-method approach).
- _svd_flip_1d: Small helper to ensure consistent sign conventions for 1D vectors.

API overview (high level)
- PLSRegression
  - Purpose: Predict Y from X using latent variable modeling.
  - Key parameters:
    - n_components: number of latent components to extract
    - scale: whether to scale X and Y
  - Typical usage:
    - model = PLSRegression(n_components=2)
    - model.fit(X, Y)
    - Y_pred = model.predict(X)
- PLSSVD
  - Purpose: Cross-decomposition via SVD-based extraction of latent vectors.
  - Key parameters:
    - n_components
    - scale
  - Typical usage:
    - model = PLSSVD(n_components=2)
    - model.fit(X, Y)
    - T, U (scores) and loadings are accessible as model.x_scores_, model.y_scores_, etc.
- PLSCanonical
  - Purpose: Canonical-like cross-decomposition maximizing covariance between X and Y scores.
  - Key parameters:
    - n_components
    - scale
  - Typical usage:
    - model = PLSCanonical(n_components=2)
    - model.fit(X, Y)
    - access scores and loadings via attributes similar to PLSRegression

Choosing the number of components
- Use cross-validation to select n_components that minimizes prediction error (for PLSRegression) or maximizes explained cross-covariance (for PLSCanonical/PLSSVD).
- Examine the cumulative explained variance or the stability of loadings across folds.
- Beware of overfitting when n_components approaches the smaller of (n_samples, min(p, q)).

Implementation notes for developers
- These implementations rely on center/scale preprocessing utilities to ensure consistent behavior across components.
- Deflation strategies are central: after extracting each component, the corresponding part of X and Y is removed before extracting the next component.
- Numerical stability considerations:
  - Option to switch between SVD-based extraction and power-method-based extraction for large-scale problems.
  - Sign flipping via svd_flip to ensure deterministic sign conventions where applicable.
- In-code examples and tests typically verify shapes of scores/loadings and basic reconstruction/prediction capabilities.

Relation to the rest of the library
- Cross decomposition functions are designed to work with the standard estimator interface (fit, transform/predict, etc.).
- They complement other regression and dimensionality reduction tools by focusing on the relationship between two data sets rather than just predicting one from the other.
- The utility functions for centering/scaling and deflation patterns are shared with related modules to ensure consistency across scikit-learn.
