Feature selection and dimensionality reduction on sample sets
==============================================================

Overview
--------
High-dimensional feature spaces can degrade model performance, increase training time, and lead to overfitting. This section covers techniques to reduce the dimensionality of your feature space for sample sets, including both feature selection (keeping a subset of existing features) and dimensionality reduction (projecting data into a lower-dimensional space).

Key concepts
------------
- Feature selection: Selects a subset of the original features. Keeps the existing feature space but with fewer features. Can be unsupervised or supervised depending on the criterion.
- Dimensionality reduction: Transforms data into a new feature space with fewer dimensions. May produce features that do not correspond directly to the original ones but capture the most informative structure.

When to use
- High dimensionality with many noisy or redundant features.
- You want faster training times and potentially better generalization.
- You need improved visualization or downstream model stability.
- Your data is sparse (text, one-hot encoding) and you want compact representations.

Common categories of methods
----------------------------
- Filter (univariate) feature selection
  - Evaluates each feature independently (e.g., ANOVA F-value, chi-squared, mutual information).
  - Fast and model-agnostic.
- Wrapper methods
  - Use a predictive model to assess feature subsets (e.g., Recursive Feature Elimination).
  - Computationally heavier but can capture feature interactions.
- Embedded methods
  - Feature selection embedded in model training (e.g., model with regularization or feature importances).
  - Often a good balance of performance and cost.
- Dimensionality reduction
  - PCA (linear, unsupervised) and variants (KernelPCA, TruncatedSVD) to project into fewer components.
  - NMF and other factorization methods for specific data types (e.g., non-negative data).
  - Often helps with visualization and reduces redundancy while preserving structure.

Practical considerations
------------------------
- Scaling: Many feature-selection and dimensionality-reduction methods assume features are on a similar scale. Use StandardScaler (or appropriate transformers) when needed.
- Sparsity: For sparse data, some methods (e.g., TruncatedSVD, chi-squared) handle sparsity well; others may require dense input.
- Data leakage: Fit transformers on training data only and apply the same transformation to validation/test data.
- Reproducibility: For stochastic methods (e.g., randomized SVD), set random_state.
- Interpretability: Feature selection preserves original features (easier to interpret). Dimensionality reduction creates new components that may be harder to interpret.

API anchors (typical usage)
----------------------------
- Feature selection
  - Filter: SelectKBest, SelectPercentile, chi2, f_classif, mutual_info_classif
  - Wrapper: RecursiveFeatureElimination (RFE), RFECV
  - Embedded: SelectFromModel (works with models exposing coef_ or feature_importances_)
- Dimensionality reduction
  - PCA (Principal Component Analysis)
  - TruncatedSVD (suitable for sparse data)
  - NMF (Non-negative Matrix Factorization)
  - KernelPCA (nonlinear dimensionality reduction)

Examples
--------

Example 1: Univariate feature selection on a numeric dataset
- Goal: keep the top 2 features by ANOVA F-value.

.. code-block:: python

   from sklearn.datasets import load_iris
   from sklearn.feature_selection import SelectKBest, f_classif

   X, y = load_iris(return_X_y=True)

   selector = SelectKBest(score_func=f_classif, k=2)
   X_reduced = selector.fit_transform(X, y)

   print("Original shape:", X.shape)
   print("Reduced shape:", X_reduced.shape)
   print("Selected feature indices:", selector.get_support(indices=True))

Example 2: Model-based feature selection (embedded) with SelectFromModel
- Goal: select features with non-zero coefficients from a linear model.

.. code-block:: python

   from sklearn.datasets import load_iris
   from sklearn.linear_model import LogisticRegression
   from sklearn.feature_selection import SelectFromModel

   X, y = load_iris(return_X_y=True)

   clf = LogisticRegression(penalty="l1", solver="liblinear")
   sfm = SelectFromModel(clf, max_features=5)
   X_selected = sfm.fit_transform(X, y)

   print("Original shape:", X.shape)
   print("Selected shape:", X_selected.shape)
   print("Selected feature indices (approx):", sfm.get_support(indices=True))

Example 3: Dimensionality reduction with PCA
- Goal: reduce dimensionality while preserving variance.

.. code-block:: python

   from sklearn.datasets import load_iris
   from sklearn.preprocessing import StandardScaler
   from sklearn.decomposition import PCA
   from sklearn.pipeline import Pipeline

   X, y = load_iris(return_X_y=True)

   pipeline = Pipeline([
       ("scaler", StandardScaler()),
       ("pca", PCA(n_components=0.95, random_state=42))
   ])

   X_pca = pipeline.fit_transform(X, y)

   print("Original shape:", X.shape)
   print("PCA-reduced shape:", X_pca.shape)

Example 4: Dimensionality reduction for sparse data with TruncatedSVD
- Goal: apply SVD to sparse text-like data (e.g., TF-IDF).

.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer
   from sklearn.decomposition import TruncatedSVD
   from sklearn.pipeline import make_pipeline

   docs = [
       "the cat sat on the mat",
       "the dog chased the cat",
       "the cat ate the mouse"
   ]

   vectorizer = TfidfVectorizer(stop_words="english")
   X = vectorizer.fit_transform(docs)

   svd = TruncatedSVD(n_components=2, random_state=42)
   X_reduced = svd.fit_transform(X)

   print("Original shape (sparse):", X.shape)
   print("SVD-reduced shape:", X_reduced.shape)

Notes for developers
--------------------
- Pipeline integration: Feature selection and dimensionality reduction steps are typically placed early in a pipeline, often after scaling (for methods that require it) and before the final estimator.
- Reproducibility: When using stochastic reductions (e.g., randomized SVD), fix random_state to ensure reproducibility.
- Sparse data: Prefer TruncatedSVD for sparse matrices (e.g., TF-IDF features) to avoid dense conversions.
- Interpretability: If interpretability of selected features is important, prefer feature-selection methods that retain original feature names (SelectFromModel, SelectKBest, etc.).
