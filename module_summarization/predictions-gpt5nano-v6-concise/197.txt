# Naive Bayes classifiers

Naive Bayes classifiers are a family of simple probabilistic classifiers based on applying Bayes' theorem with strong (naive) independence assumptions between features. They are efficient to train and can work well in high-dimensional settings, especially with text data or other discrete features.

Key idea
- For a feature vector x and class c, the model estimates P(c) and P(x|c) under the assumption that features are conditionally independent given the class:
  - P(c|x) ∝ P(c) ∏_i P(x_i | c)
- In practice, we compute log probabilities to avoid underflow and for numerical stability. Predictions select the class with the highest posterior probability.

Common API
- Estimators implement BaseEstimator and ClassifierMixin.
- Core methods:
  - fit(X, y): train the model
  - predict(X): predict class labels
  - predict_proba(X): predict class probabilities
  - predict_log_proba(X): log-probabilities
  - predict_joint_log_proba(X): joint log probability log P(x, y)
  - partial_fit(X, y, classes=None): incremental learning (online updates)
- Internal methods to override in concrete estimators:
  - _joint_log_likelihood(X): compute unnormalized log P(y) + log P(X|y)
  - _check_X(X): perform feature checks specific to the estimator
  - predict*, predict_proba*, etc. use _check_X and _joint_log_likelihood

Supported Naive Bayes variants
- GaussianNB
- BernoulliNB
- MultinomialNB
- ComplementNB
- CategoricalNB

Common attributes (after fit)
- classes_: array of class labels seen during fit
- class_count_: array of counts of samples in each class
- class_prior_: prior probability of each class (P(y))
- n_features_in_, feature_names_in_: number of features and their names (if provided)
- For variant-specific attributes (examples):
  - GaussianNB: theta_ (class feature means), var_ (class feature variances), epsilon_ (variance smoothing)
  - MultinomialNB: class_log_prior_, feature_count_, feature_log_prob_
  - BernoulliNB: class_log_prior_, feature_count_, feature_log_prob_
  - ComplementNB: class_log_prior_, feature_count_, feature_log_prob_ (design aims to improve on imbalanced data)
  - CategoricalNB: categories_ (categories per feature)

Choosing a variant
- GaussianNB: Continuous features with roughly Gaussian distribution per class. Uses feature-wise means and variances.
- MultinomialNB: Features represent counts or term frequencies (non-negative). Suitable for text classification with word-count vectors.
- BernoulliNB: Binary/boolean features or boolean occurrence indicators.
- ComplementNB: Improved performance on imbalanced datasets; complements of each class used for parameter estimation.
- CategoricalNB: Categorical features with explicit category handling.

Training and prediction workflow (typical)
- Prepare X (n_samples, n_features) and y (n_samples,)
- Choose an estimator (e.g., GaussianNB, MultinomialNB)
- Fit: estimator.fit(X, y)
- Predict: y_pred = estimator.predict(X_test)
- Predict probabilities: proba = estimator.predict_proba(X_test)
- For incremental learning (where supported): estimator.partial_fit(X_batch, y_batch, classes=classes)

Important parameters (high-level)
- priors (optional): explicit priors P(y). If provided, priors are not adjusted from the data.
- var_smoothing (GaussianNB): portion of the largest variance of all features added to variances for numerical stability.
- alpha (MultinomialNB, BernoulliNB): Laplace smoothing parameter; larger alpha reduces zero probability issues.
- categories_ (CategoricalNB): list of categories per feature; used to model categorical features exactly.

Notes for developers
- Internals rely on log-space computations (log probabilities) for numerical stability.
- _joint_log_likelihood returns log P(y) + log P(X|y) for all classes and samples.
- predict_proba returns probabilities by exponentiating log-probabilities; predict_log_proba returns the log-probabilities.
- partial_fit enables online learning for estimators that support it (GaussianNB, others may or may not).

Examples

1) Gaussian Naive Bayes (continuous features)
- Purpose: quick baseline for continuous data
- Code:
  from sklearn.naive_bayes import GaussianNB
  import numpy as np

  X = np.array([[1.0, 2.0],
                [1.5, 1.8],
                [5.0, 4.5],
                [6.0, 5.0]])
  y = np.array([0, 0, 1, 1])

  gnb = GaussianNB()
  gnb.fit(X, y)

  X_test = np.array([[1.2, 1.9],
                     [5.5, 4.7]])
  y_pred = gnb.predict(X_test)
  proba = gnb.predict_proba(X_test)

2) Multinomial Naive Bayes (count data)
- Purpose: text classification, bag-of-words
- Code:
  from sklearn.naive_bayes import MultinomialNB
  import numpy as np

  X = np.array([[2, 0, 0],
                [0, 3, 1],
                [1, 0, 0]])
  y = np.array([0, 1, 0])

  mnb = MultinomialNB(alpha=1.0)
  mnb.fit(X, y)

  X_test = np.array([[1, 2, 0]])
  y_pred = mnb.predict(X_test)
  proba = mnb.predict_proba(X_test)

3) Partial fit (online learning) with GaussianNB
- Code:
  from sklearn.naive_bayes import GaussianNB
  import numpy as np

  X_batch1 = np.array([[1.0, 2.0], [1.5, 1.8]])
  y_batch1 = np.array([0, 0])

  X_batch2 = np.array([[5.0, 4.5], [6.0, 5.0]])
  y_batch2 = np.array([1, 1])

  nb = GaussianNB()
  nb.partial_fit(X_batch1, y_batch1, classes=np.unique(np.r_[y_batch1, y_batch2]))
  nb.partial_fit(X_batch2, y_batch2)

Notes
- Categorical data with explicit categories is best handled by CategoricalNB.
- For imbalanced datasets, consider ComplementNB for improved performance.
- For numerical stability with GaussianNB, var_smoothing can help when variances are very small.
