# GMM and IV2SLS (gmm.rst)

Module purpose
- Provide Instrumental Variables estimation via Two-Stage Least Squares (2SLS) within a framework that aligns with statsmodels’ modeling interfaces.
- Offer a pathway toward Generalized Method of Moments (GMM) style workflows, with a concrete IV2SLS implementation that returns regression results compatible with the standard results wrappers.

Key classes
- IV2SLS: Instrumental variables estimation using Two-Stage Least Squares.
- IVRegressionResults: A results class intended to mirror RegressionResults behavior for IV-type models.

Public API
- IV2SLS(endog, exog, instrument=None)
  - endog: 1D or 2D array, endogenous dependent variable.
  - exog: 1D or 2D array, explanatory variables to be instrumented.
  - instrument: 1D or 2D array of instruments, must include both non-instrumented exogenous variables and additional instruments.

- IV2SLS.initialize()
  - Prepare internal copies of endog and exog for estimation.

- IV2SLS.whiten(X)
  - Placeholder (Not implemented). Intended for whitening operations on a matrix X.

- IV2SLS.fit()
  - Core estimation routine using the textbook 2SLS approach:
    1) Compute z’z and z’T x where z is the instrument matrix and x is exog.
    2) Solve (z’z) xhat = z’t x to obtain xhatparams.
    3) Form F = z xhatparams (the projected design matrix).
    4) Regress y on F to obtain parameter estimates.
    5) Compute a covariance-like quantity via F’F and F’Tx, F’Ty, producing normalized_cov_params.
  - Returns a RegressionResultsWrapper around an IVRegressionResults-like object (lfit) that mirrors standard regression result access (params, etc.).

- IV2SLS.predict(params, exog=None)
  - Predict linear values given a parameter vector. Falls back to the model’s exog if none provided.

- Internal results
  - The estimator stores intermediate objects, including:
    - xhatparams: solution to (z’z) xhat = z’t x
    - xhat: z xhatparams
    - normalized_cov_params: covariance-like matrix derived from the fitting stage
    - _results_ols2nd: a secondary OLS fit on the fitted values (for potential Hausman-type checks)

- IVRegressionResults
  - Subclass of RegressionResults (for IV-type models). Currently documented as the results class used by IV2SLS, allowing compatibility with standard result access patterns.

Usage example
- Basic IV2SLS workflow

  - Synthetic data (endogenous y, endogenous regressors x, instruments z):
    - y ~ x + error
    - Instruments z include valid instruments for x (and possibly exogenous variables)

  - Example code:

    from statsmodels.gmm import IV2SLS
    import numpy as np

    # Example data (replace with real data)
    n = 100
    np.random.seed(0)
    Z = np.random.randn(n, 2)        # instruments
    X = np.column_stack((np.ones(n), np.random.randn(n), Z[:, 0]))
    beta_true = np.array([1.0, 2.0, 0.5])
    u = np.random.randn(n)
    endog = X @ beta_true + u
    instrumented_exog = X  # all columns in this toy example are instrumented

    # IV estimation
    model = IV2SLS(endog, instrumented_exog, instrument=Z)
    results = model.fit()

    print("Params:", results.params)
    print("Covariance:", results.normalized_cov_params)

Notes
- The current implementation focuses on a straightforward 2SLS flow and returns a result wrapper compatible with standard RegressionResults interfaces.
- The whiten method is a placeholder and not used in the current fit path.
- The degrees of freedom handling (df_resid, df_model) follows a simple convention (nobs - k_vars) but may need refinement for specialized IV/GMM settings.

Design and architecture details
- Integration with statsmodels:
  - IV2SLS inherits from LikelihoodModel, enabling compatibility with the broader modeling ecosystem.
  - The fit method assembles the two-stage projection and a final regression in a single cohesive routine.
- Result handling:
  - The final parameter estimates are wrapped in a regression-style results object (RegressionResultsWrapper) for a familiar API surface.
  - The implementation stores intermediate objects (e.g., xhatparams, xhat, F) for potential diagnostic calculations or Hausman-type tests.

Limitations and known issues
- whiten is not implemented; future work could provide a whitening operation for generalized GMM weighting schemes.
- The current DOF logic is minimal and may need refinement for complex IV/GMM scenarios.
- The implementation assumes a straightforward 2SLS workflow; extending to full GMM weighting matrices and overidentification tests would require additional scaffolding (e.g., moment conditions, optimal weighting, and j-test/jval support).
