# plothist.rst – Plothist command

Plothist is a WestPA tool for visualizing histogram data stored in HDF5 files produced by FAS/FAST hist workflows. This developer-oriented documentation covers the Plothist command as implemented in plothist.rst, including its CLI interface, data model, internal workflow, and extension points.

## Overview

- Purpose: Render one- or two-dimensional histograms from an HDF5 histogram file, with options to control axes, ranges, scales, titles, and contour overlays.
- Input: HDF5 file containing histogram data generated by WestPA.
- Output: A plot file (defaults to PDF) and optionally an HDF5 file with serialized plot data.
- Extensibility: The plotting flow is encapsulated in a base command class (PlotHistBase) intended to be subclassed/extended by specific plot implementations.

## Command interface

Plothist exposes a WestPA subcommand with a rich set of options. The primary entry points are provided by the PlotHistBase class:

- Command: plothist
- Input argument(s):
  - input: Path to an HDF5 file containing histogram data
  - firstdim: Optional dimension spec for the histogram to plot, of the form:
    - INT[:[LB,UB]:LABEL]
    - Where:
      - INT is a zero-based dimension index in the histogram
      - LB, UB are optional lower/upper bounds for plotting
      - LABEL is the axis label for the plot
- Output options:
  - -o / --output / --plot-output: Path to store the plot output. If empty, plotting is suppressed. The format is inferred from the extension (default: hist.pdf).
  - --hdf5-output: Path to store the plot data in an HDF5 file
  - --plot-contour: Overlay a contour on 2D heatmaps
- Plot/options:
  - --title: Optional title to place at the top of the graph
  - Mutually exclusive scale options (one must be chosen):
    - --linear: Plot histogram on a linear scale (plotscale = 'linear')
    - --energy: Plot histogram on an inverted natural log scale (default, plotscale = 'energy')
    - --zero-energy E | min | max: Set the zero-energy reference point
      - E: numeric zero point
      - "min"/"max": use the minimum/maximum energy as zero
    - --log10: Plot histogram on a base-10 log scale (plotscale = 'log10')
  - --range: Plot range as "LB,UB" (LB/UB depend on dimension: Y-axis for 1D plots, colorbar axis for 2D plots)
  - --postprocess-function: Name a function (module.function) to call just before saving the plot
- Defaults:
  - plotscale defaults to 'energy'

Notes:
- The actual implementation of parse_dimspec, parse_range, and related helpers are provided by the base WestPA command utilities. PlotHistBase wires these helpers into the argument parsing and processing flow.

## Data model and internal state

PlotHistBase maintains state used for plotting:

- input_h5: Opened HDF5 input file (via h5py.File)
- plot_output_filename: Path for the generated plot
- hdf5_output_filename: Path for serialized plot data (if requested)
- plot_contour: Boolean flag indicating whether to overlay a contour on 2D heatmaps
- plottitle: Optional plot title
- plotrange: Optional range for the plotting axis (parsed from --range)
- enerzero: Zero-energy reference (min, max, or numeric)
- avail_iter_start / avail_iter_stop / avail_iter_step: Iteration range for averages/evolutions
- iter_start / iter_stop / iter_step: Iteration range used for a single-point plot (if applicable)
- n_iter: Number of iterations for a single-point plot
- dimensions: List of dictionaries describing which histogram dimension(s) to plot, including dimension idim and label
- plot_output_filename / text_output_filename: Filenames for various output formats (text, HDF5, etc.)

Important: The code relies on helper routines to derive available iteration ranges from the input file (e.g., h5io.get_iter_range) and to interpret dimension specifications (parse_dimspec) and range specifications (parse_range).

## How it works (high level)

1. Argument parsing (PlotHistBase.add_args):
   - Build the CLI surface for input, output, and plot options.
   - Support specifying a single dimension (firstdim) or default to dimension 0 if not provided.
   - Configure scales, energy zero point, range, and optional post-processing hook.
2. Argument processing (PlotHistBase.process_args):
   - Open the input HDF5 file and store filenames for outputs.
   - Apply plot title and range, if provided.
   - If a dimension is supplied, append its specification to the internal dimensions list; otherwise, default to dimension 0.
   - Interpret the energy zero point (numeric or min/max).
   - Determine iteration ranges for average/evolution and for single-point plotting.
3. Plotting:
   - The actual rendering code is implemented in the derived plotting command(s) (not shown in the snippet). PlotHistBase provides:
     - Access to histogram data (via the opened HDF5 file)
     - Mechanisms to apply energy scaling, axis labeling, and color scaling
     - Optional contour overlays for 2D plots
     - Optional post-processing hook invocation prior to saving the plot
4. Output:
   - The plot is saved to the path specified by --output (or suppressed if empty)
   - If requested, histogram data used for the plot is stored in the HDF5 output file

## Examples

Note: The exact dimension specification syntax is INT[:[LB,UB]:LABEL], where LB/UB are optional bounds and LABEL is the axis label. Examples below illustrate typical usage patterns.

1) Basic 1D histogram plot (default energy scale, full range)

- Command:
  plothist data.h5 0
- Result:
  - A plot of histogram dimension 0, full range, on the energy scale (default)
  - Output file: hist.pdf (or as configured with -o)

2) 1D plot with a named axis and custom range

- Command:
  plothist data.h5 0:[0.0, 2.0]:Distance
  -o distance_hist.pdf
- Result:
  - Y-axis range limited to [0.0, 2.0] (for 1D: Y-axis)
  - Axis label: Distance

3) 2D heatmap with contour overlay

- Command:
  plothist data.h5 0:[-1.0,1.0]:X 1:[-0.5,0.5]:Y --plot-contour
  -o heatmap.pdf
- Result:
  - 2D heatmap of dimensions 0 and 1
  - Contour lines overlaid on the heatmap
  - Axes labeled 0: X and 1: Y (as per the dimension labels)

4) Save plot data to HDF5 and suppress the visual output

- Command:
  plothist data.h5 0 -o '' --hdf5-output plot_data.h5
- Result:
  - Plot is suppressed (no image file written)
  - Histogram data used for the plot is stored in plot_data.h5

5) Use a custom post-processing function

- Command:
  plothist data.h5 0 --plot-output=custom.pdf --postprocess-function=myplotmod:finalize_plot
- Result:
  - After the plot is created, the function myplotmod.finalize_plot is called as:
    postprocess(hist, midpoints, binbounds)
  - The function is expected to modify the plot in place via matplotlib.pyplot

6) Change the zero-energy reference to a numeric value

- Command:
  plothist data.h5 0 --zero-energy 3.14
- Result:
  - Energy axis zero-point set to 3.14 (numeric)

## Developer notes and extension points

- PlotHistBase is designed as a generic base for plothist-like commands. Concrete plot commands should subclass PlotHistBase and implement the actual rendering, leveraging:
  - The parsed dimensions (PlotHistBase.dimensions)
  - The parsed plot range and energy scale
  - The available iteration range information (PlotHistBase.avail_iter_start/stop/step)
- Key helper methods (not shown in the snippet) include:
  - parse_dimspec(string): Convert the dimension spec string into a dictionary with at least 'idim' and 'label', plus optional bounds
  - parse_range(string): Convert "LB,UB" into numeric bounds suitable for plotting
  - get_iter_range(group): Retrieve available iteration ranges from the HDF5 input
- The code uses:
  - h5py for HDF5 I/O
  - NumPy for array manipulation
  - Matplotlib for rendering plots
- The function sum_except_along(array, axes) is a utility used by plotting routines when reducing high-dimensional histograms to the target plotting axes. It preserves the order of kept axes and sums over all others.
- The CLI design follows WestPA conventions:
  - Grouped arguments for input, output, and plot options
  - Mutually exclusive set for plot scales (linear, energy, log10)
  - Optional chaining to suppress output by setting the plot filename to an empty string

## Implementation details (for maintainers)

- The plothist command relies on the WestPA core libraries:
  - WESTMasterCommand, WESTSubcommand
  - westpa.core.h5io, westpa.core.textio
  - westpa.fasthist.normhistnd
  - westpa.core.extloader.get_object
- The actual rendering path (specific plots, color schemes, etc.) is delegated to derived commands. PlotHistBase provides the shared plumbing for:
  - Argument parsing and validation
  - Histogram data access and range handling
  - Coordinate/axis labeling scaffolding
  - Option hooks (plot-contour, postprocess-function, etc.)

## Troubleshooting

- If no output file is generated and --plot-output is not an empty string:
  - Check that the input HDF5 file contains the requested dimension (idim) and that LB/UB bounds are within valid ranges.
  - Verify the HDF5 file’s structure and that the expected histograms exist.
  - Ensure the plotting backend is available (Matplotlib configured with a valid backend).
- If parsing of the dimension specification fails:
  - Revisit the firstdim string format: it must conform to INT[:[LB,UB]:LABEL].
  - Ensure LB and UB are numeric (if provided) and that LABEL is a valid string.
