duration.rst
============

Overview
--------

This module provides core functionality for survival analysis with right-censoring, including
calculation of the survival function and cumulative incidence function (CIF) for competing risks.
The implementations are designed for use by developers who are integrating survival analysis into
larger projects or building higher-level APIs around these estimators.

Key capabilities
- Compute the one-group (right-censored) survival function with standard errors.
- Compute the cumulative incidence function for competing risks and its standard error.
- Validate input arguments consistently.
- Provide a class-based interface (CumIncidenceRight) for estimating CIFs in a reusable object.

Public API
-----------

Note: Several helpers in this module are internal (prefixed with an underscore) and not guaranteed to be part of the stable public API. They are documented here for completeness and for developers extending or debugging the module.

Functions

- _calc_survfunc_right(time, status, weights=None, entry=None, compress=True, retall=True)

  Purpose
  - Compute the survival function S(t) for a single group, with optional standard error.

  Parameters
  - time: array_like
    Event or censoring times.
  - status: array_like
    Indicator of event type at each time:
    - status >= 1 indicates an event occurred (1..K for K event types if used in CIF context).
    - status == 0 indicates censoring.
  - weights: array_like or None
    Optional frequency weights (same length as time/status).
  - entry: array_like or None
    Optional entry times (left-truncation). If provided, risk sets are adjusted for delayed entry.
  - compress: bool
    If True, retain only times where an event occurred (d > 0) to speed up computation.
  - retall: bool
    If True (default), return the full set of outputs (survival, SE, utime, rtime, n, d).
    If False, return a reduced tuple (sp, utime, rtime, n, d).

  Returns (retall=True)
  - sp: array_like
    Estimated survival probabilities S(t) at event times.
  - se: array_like
    Standard errors of the survival probabilities.
  - utime: array_like
    Unique event/censoring times.
  - rtime: array_like
    Ranks mapping each time to its position in the unique time grid.
  - n: array_like
    At-risk counts just prior to each event time.
  - d: array_like
    Number of events (deaths) at each unique time.

  Returns (retall=False)
  - sp, utime, rtime, n, d
    Reduced set of outputs (no se).

  Notes
  - Greenwood's formula is used for SE when weights is None.
  - Tsiatis’ formula is used for SE when weights are provided.
  - The function enforces numerical stability by capping denominators and clamping survival values.

  Typical usage
  - sp, se, utime, rtime, n, d = _calc_survfunc_right(time, status)

- _calc_incidence_right(time, status, weights=None)

  Purpose
  - Calculate the cumulative incidence function (CIF) and its standard error for competing risks.

  Parameters
  - time: array_like
    Times of censoring or events.
  - status: array_like
    Event indicator:
    - status >= 1 indicates the event type (1..ngrp for ng rp groups).
    - status == 0 indicates censoring.
  - weights: array_like or None
    Optional weights.

  Returns
  - ip: list of array_like
    CIF estimates for each cause (length ng rp).
  - se: list of array_like or None
    Standard errors for CIFs (None if weights are provided).
  - utime: array_like
    Unique times used for CIF computation.

  Notes
  - All-cause survival is used internally to derive CIFs.
  - If weights are provided, the function returns CIF estimates with no standard error (se is None).

- _checkargs(time, status, entry, freq_weights, exog)

  Purpose
  - Validate inputs to ensure consistency before performing survival or CIF computations.

  Validations
  - time and status must have the same length.
  - If entry is provided, it must have the same length as time.
  - If entry is provided, all entry times must occur before any event times.
  - If freq_weights is provided, it must have the same length as time.
  - If exog is provided, its first dimension must match the length of time.

Class

- CumIncidenceRight

  Purpose
  - Encapsulate estimation and inference for the cumulative incidence function.
  - Supports right-censoring and, optionally, frequency weights or covariate adjustments.

  Notes
  - The docstring indicates it is designed to work with CIFs where J = 1, 2, ..., J indicates the event type, and I(t, j) = P(T <= t and J = j).
  - Right censoring is supported; frequency weights enable point estimation with no standard error.
  - Additional features such as exogenous adjustment (exog), bandwidth factor (bw_factor), and dimension reduction (dimred) are exposed, but the complete interface is partially shown in the code snippet. Refer to the module source for full API details and intended usage.

  Parameters (from the docstring)
  - time: array_like
  - status: array_like
  - title: string
  - freq_weights: array_like
  - exog: array_like
  - bw_factor: float
  - dimred: boolean

  Example usage (conceptual)
  - CIF estimator = CumIncidenceRight(time, status, title="My CIF", freq_weights=None, exog=None, bw_factor=1.0, dimred=False)
  - CIF estimator would provide methods to fit and extract CIFs and SEs (exact methods depend on the full class implementation).

Implementation notes
--------------------

- Dependencies
  - numpy
  - pandas
  - scipy (for chi-squared and normal distributions; used in statistical testing or plot utilities)
  - statsmodels (for graphics utilities)

- Design choices
  - Internal helpers (_prefixed names) are used to modularize the computations and keep the public API clean.
  - Distinction between all-cause survival and cause-specific CIF is implemented to support competing risks analyses.
  - Weights and entry times (left truncation support) are optional, enabling flexible data scenarios.

Typical workflows
-----------------

A. Compute survival function for a single group

- Given time and status arrays, obtain survival probabilities and diagnostic data.

Code example
- Imports
  - import numpy as np
  - from duration_module import _calc_survfunc_right  # adjust import path to your package

- Data
  - time = np.array([5, 6, 8, 12, 15])
  - status = np.array([1, 0, 1, 0, 1])

- Calculation
  - sp, se, utime, rtime, n, d = _calc_survfunc_right(time, status)

- Output
  - sp: survival probabilities at observed event times
  - se: standard errors
  - utime, rtime: timing indices
  - n: at-risk counts prior to each event
  - d: number of events at each time

B. Compute cumulative incidence function for competing risks

- Given time and status with multiple event types

Code example
- time = np.array([2, 3, 5, 6, 8])
- status = np.array([1, 0, 2, 1, 0])  # 1 and 2 indicate two competing risks; 0 is censoring

- ip, se, utime = _calc_incidence_right(time, status)

- ip is a list of CIF arrays for each cause (e.g., ip[0] for cause 1, ip[1] for cause 2)
- se is a corresponding list of SE arrays (or None if weights are provided)

C. Input validation

- Before running computations, call _checkargs(time, status, entry, freq_weights, exog) to ensure input consistency.

D. Class-based CIF estimation

- Create an instance of CumIncidenceRight with the required data.
- Use the instance to perform CIF estimation and extract results (actual methods depend on the full class implementation).

Examples of caveats and considerations
--------------------------------------

- Data consistency is critical: time and status must align in length; entry times (if any) must occur before event times.
- Censoring handling is restricted to right-censoring; left-truncation is supported via entry times to adjust risk sets.
- When using frequency weights, standard errors for CIF are not returned; use unweighted estimates if SE is required.
- Survfunc probabilities are clamped to avoid numerical underflow; this may influence SE in boundary regions.

Testing and validation guidance
-------------------------------

- Unit tests should cover:
  - Basic survival calculation with and without weights.
  - CIF calculation for one and multiple causes.
  - Behavior with entry times (delayed entry) and without.
  - Handling of edge cases (all censored, all events at same time, zero risk set).
- Compare results against analytical expectations on small datasets or against established survival analysis libraries for cross-validation.

Performance considerations
--------------------------

- The algorithm processes data by unique event/censoring times, which scales with the number of unique times rather than the number of observations.
- Weight handling uses bincount operations, which are efficient in NumPy but may memory-load for very large datasets.
- The compress flag helps reduce memory usage by discarding times without events.
