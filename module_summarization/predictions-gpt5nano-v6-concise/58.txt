# optimization.rst

Module: optimization

Summary
- This module provides general, reusable optimization utilities that are not tied to any specific model class (e.g., LikelihoodModel). The goal is to offer a common optimization backbone that can be reused across different fitting problems.

Key components
- _check_method(method, methods): Utility to validate a requested optimization method against a set of supported methods.
- Optimizer: A lightweight class that exposes a single entry point, _fit, to perform optimization of an arbitrary objective function with an optional gradient and (optionally) Hessian.

What is this module for
- The Optimizer and its _fit method are designed to be general-purpose tools for fitting any model that can be characterized by:
  - An objective function to be minimized or maximized (in practice, typically a log-likelihood to maximize or a negative log-likelihood to minimize).
  - An optional gradient (score) function.
  - Optional Hessian information.
- The approach decouples the optimization routine from a specific model class, enabling reuse across different modeling contexts.

API reference

- _check_method(method, methods)
  - Purpose: Validate that a requested optimization method is supported.
  - Parameters:
    - method: str, the name of the optimization method to use.
    - methods: Iterable[str], the set of supported methods.
  - Behavior: Raises ValueError if method not in methods.

- Optimizer
  - _fit(self, objective, gradient, start_params, fargs, kwargs,
           hessian=None, method='newton', maxiter=100, full_output=True,
           disp=True, callback=None, retall=False)
  - Purpose: Fit a generic objective function using a chosen solver. Intended to be usable with any model that can supply:
    - objective(x, *fargs, **kwargs) -> float
    - gradient(x, *fargs, **kwargs) -> array_like  (optional)
    - hessian(x, *fargs, **kwargs) -> array_like  (optional)
  - Parameters:
    - objective: callable
        The objective function to optimize. Should return a scalar value.
    - gradient: callable
        The gradient (score) function with respect to parameters. Can be None if the solver should approximate the gradient.
    - start_params: array_like
        Initial guess of the solution for the optimization.
    - fargs: tuple
        Extra positional arguments passed to the objective function.
    - kwargs: dict
        Extra keyword arguments passed to the objective function.
    - hessian: callable, optional
        Function computing the Hessian or Hessian-vector product, if provided.
    - method: str, default 'newton'
        Optimization method to use. Options include:
        - 'newton', 'nm', 'bfgs', 'powell', 'cg', 'ncg', 'basinhopping', 'minimize'
        - A generic 'minimize' string is treated as a wrapper around scipy.optimize.minimize (when available).
        - The exact supported methods and their arguments depend on the underlying solver (SciPy).
    - maxiter: int, default 100
        Maximum number of iterations to perform.
    - full_output: bool, default True
        If True, return additional solver output in the return value.
    - disp: bool, default True
        If True, print convergence messages.
    - callback: callable, optional
        Called after each iteration, with the current parameter vector.
    - retall: bool, default False
        If True, return a list of solutions at each iteration (where supported).
  - Returns:
    - xopt: array
        The optimized parameter vector.
    - retvals: dict or None
        If full_output is True, a dictionary with solver-specific output. Otherwise None.
    - optim_settings: dict
        A dictionary containing the parameters passed to the solver (the optimization settings).
  - Notes:
    - The 'basinhopping' solver ignores explicit arguments maxiter, retall, and full_output.
  - Solver-specific optional arguments (illustrative; see SciPy solver docs for exact keys):
    - 'newton'
      - tol: Relative tolerance for convergence of parameters.
    - 'nm' (Nelder-Mead)
      - xtol, ftol: Convergence tolerances for parameters and objective value.
      - maxfun: Maximum function evaluations.
    - 'bfgs'
      - gtol: Stop when gradient norm is below this threshold.
      - norm: Order of the norm used to compute the gradient norm.
      - epsilon: Step size for numerical gradient if the gradient is approximated.
    - 'lbfgs'
      - m, pgtol, factr, maxfun, epsilon, approx_grad
    - 'cg'
      - gtol, norm, epsilon
    - 'ncg'
      - fhess_p, avextol, epsilon
    - 'powell'
      - xtol, ftol, maxfun, start_direc
    - 'basinhopping'
      - niter, niter_success, T, stepsize, interval, minimizer
      - minimizer is a dict of kwargs to scipy.optimize.minimize, e.g.,
        method for the inner minimization, tolerance, etc.
        Other mappings include:
        - args <- fargs
        - jac <- gradient
        - h <- (content not shown in current docstring)
  - Notes on design:
    - The implementation emphasizes reusability by decoupling solver choice and model specifics.
    - It is intended to be used as a foundation for model-fitting workflows, not as a feature-complete replacement for SciPy optimize APIs.

Usage examples

Example 1: Fit a simple quadratic objective with a known gradient
- Objective: f(x) = 0.5 * x^T A x - b^T x
- Gradient: ∇f(x) = A x - b
- Hessian: H = A

Code (illustrative; assumes availability of a suitable A, b, and a properly configured Optimizer):
- import numpy as np
- from optimization import Optimizer

- A = np.array([[3.0, 0.0], [0.0, 1.0]])
- b = np.array([1.0, 2.0])
- def objective(x, A=A, b=b):
-     return 0.5 * x.dot(A).dot(x) - b.dot(x)
- def gradient(x, A=A, b=b):
-     return A.dot(x) - b
- x0 = np.zeros(2)

- opt = Optimizer()
- xopt, retvals, optim_settings = opt._fit(
-     objective=objective,
-     gradient=gradient,
-     start_params=x0,
-     fargs=(),
-     kwargs={},
-     hessian=None,         # If you have a Hessian function, supply it here
-     method='newton',
-     maxiter=100,
-     full_output=True,
-     disp=True,
-     callback=None,
-     retall=False
- )

- print("Optimized x:", xopt)
- print("Solver info:", retvals)
- print("Settings:", optim_settings)

Example 2: Use a generic objective with SciPy minimize (method='minimize')
- Define objective and gradient as in Example 1.
- Call _fit with method='minimize' (the exact kwargs depend on SciPy's minimize interface).

Example 3: Basinhopping (global optimization)
- Define objective and (optionally) gradient.
- Configure basinhopping-specific parameters via the solver options (see Notes above).
- Call _fit with method='basinhopping' and appropriate minimizer config.

Notes for developers
- This module is designed to be decoupled from any concrete model class, enabling reuse in diverse fitting scenarios.
- The _fit method abstracts the optimization routine, leaving model-specific details (objective, gradient, and optional Hessian) to the caller.
- The documentation in the code includes extensive notes on solver options. When integrating with actual SciPy versions, be mindful of any changes in the underlying solver interfaces and adjust the inner argument mappings accordingly.
- The basinhopping path may ignore some explicit arguments (maxiter, retall, full_output); plan your usage accordingly if you rely on those features.

See also
- SciPy optimize: scipy.optimize (for the exact solver options and behavior per solver)
- LikelihoodModel (in your project) for context on how this module is intended to be reused independently of model classes

Appendix: design considerations
- The _fit API is intentionally flexible to accommodate various objective forms and solver backends.
- If your project uses a wrapper around SciPy optimizers, you can map your objective and gradient to this interface and reuse this optimization backbone.
