# General Plotting Routines

This document describes the general plotting utilities implemented in Masci-tools for interactive plotting with Bokeh. It covers the public API, data handling, defaults management, and practical usage patterns aimed at developers extending or using the plotting package.

## Overview

- The plotting layer is built on top of a Bokeh backend via the `BokehPlotter` class.
- User-facing helpers allow quick creation of interactive plots (scatter and related variants) while keeping data handling and figure configuration consistent.
- Data can be provided as raw arrays or as a data source (e.g., pandas DataFrame). The internal helper `process_data_arguments` harmonizes inputs and creates Bokeh-friendly sources.
- Defaults for the Bokeh backend are centralized and can be inspected, saved, loaded, or reset.
- A decorator (`ensure_plotter_consistency`) enforces that plotting calls share a consistent plotter configuration.

## Core Components

- `plot_params`: a global instance of the Bokeh backend plotter (`BokehPlotter`) used by the general plotting routines.
- `BokehPlotter`: the backend class that manages defaults, figure creation, plot rendering, tooltips, legends, and finalization.
- `process_data_arguments`: internal helper that processes `data`, `x`, `y`, and related options to return a plot-ready data source.
- `get_special_kpoint_ticks`, `NestedPlotParameters`, and `ensure_plotter_consistency`: supporting utilities for tick handling, parameter nesting, and plotter consistency.

## Public API

- Defaults and help
  - `set_bokeh_plot_defaults(**kwargs)`: Update the default plotting options for the Bokeh backend. Defaults are defined by `BokehPlotter`.
  - `reset_bokeh_plot_defaults()`: Reset to hardcoded defaults.
  - `show_bokeh_plot_defaults()`: Print the current defaults.
  - `get_bokeh_help(key)`: Print the description for a specific default key.
  - `load_bokeh_defaults(filename='plot_bokeh_defaults.json')`: Load defaults from a JSON file.
  - `save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)`: Save current defaults to a JSON file. If `save_complete` is `True`, also stores the hardcoded defaults.

- General plotting routines
  - `bokeh_scatter(x, y=None, *, xlabel='x', ylabel='y', title='', figure=None, data=None, saveas='scatter', copy_data=False, **kwargs)`
    - Create an interactive scatter plot with Bokeh.
    - `x` and `y` can be array-like or keys to columns in `data`.
    - `data` can be a pandas DataFrame or another suitable data source (e.g., `ColumnDataSource`).
    - Deprecated usage: passing the data source as the first positional argument. If detected, a DeprecationWarning is emitted and inputs are reinterpreted.
    - Internals: data preparation via `process_data_arguments`, automatic column source creation, and tooltip/interaction handling.
    - Returns: a Bokeh figure object.
  - `bokeh_multi_scatter(x, y=None, *, data=None, figure=None, xlabel='x', ylabel='y', title='', saveas='scatter', copy_data=False, set_default_legend=True, **kwargs)`
    - Create a scatter plot that can display multiple data series.
    - Similar input rules as `bokeh_scatter`, with additional options to manage multiple datasets and legends.
    - Returns: a Bokeh figure object.

> Note: The docstrings indicate that plot arguments are passed through to the underlying Bokeh plotting function and that unrecognized arguments are forwarded to the Bokeh plotting routine (e.g., `scatter`).

## Data Input and Preparation

- Inputs
  - `x`: array-like, or a key in `data` (column name). If a string key is provided, its column in `data` is used for the x-axis.
  - `y`: array-like, or a key in `data` (column name). If a string key is provided, its column in `data` is used for the y-axis.
  - `data`: a data source (e.g., a pandas DataFrame, or a `ColumnDataSource`). When provided, `x` and `y` can be keys (column names) in this source.
  - `figure`: optional existing Bokeh figure to add to.
  - `copy_data`: if `True`, data arguments are copied.
  - `saveas`: output file name or key used by the plotting backend to decide how to finalize the plot (e.g., file, notebook, or inline display).

- Deprecation notice
  - Passing the data source as the first positional argument to `bokeh_scatter` is deprecated. Use `data` keyword and pass `x`/`y` as explicit arguments (e.g., `bokeh_scatter('x', 'y', data=df)`).

- Internal workflow
  - `process_data_arguments` harmonizes inputs to a consistent internal representation.
  - The function returns a mapping of plot entries; the code extracts `entry` and `source` from this mapping for plotting.
  - The plotter is configured via `set_defaults` and `set_parameters`, allowing per-plot customization and consistent styling.

## Plot Customization

- Global defaults are managed via `plot_params` and the helper functions above.
- Plot-level settings:
  - Axis labels, title, and figure
  - Legend handling (in `bokeh_multi_scatter`, controlled by `set_default_legend`)
  - Tooltips and hover interactions (added via `plot_params.add_tooltips`)
  - Plot range and tick handling (via `plot_params.set_limits` and related helpers)
  - Saving and export behavior (`save_plot` via `saveas`)

## Examples

1) Simple scatter with raw arrays

```python
import numpy as np
from masci_tools.vis.plotting import bokeh_scatter, set_bokeh_plot_defaults

# Optional: adjust defaults
set_bokeh_plot_defaults(plot_width=600, plot_height=400)

# Data as numpy arrays
x = np.linspace(0, 10, 100)
y = np.sin(x)

p = bokeh_scatter(x, y, xlabel='Energy (eV)', ylabel='sin(E)', title='Simple Scatter')
# p is a Bokeh Figure; you can display or save it depending on your environment
```

2) Scatter with a DataFrame

```python
import pandas as pd
import numpy as np
from masci_tools.vis.plotting import bokeh_scatter

df = pd.DataFrame({'x': np.linspace(0, 10, 100),
                   'y': np.sin(np.linspace(0, 10, 100))})

p = bokeh_scatter('x', 'y', data=df, xlabel='Energy (eV)', ylabel='sin(E)', title='Scatter from DataFrame')
```

3) Managing defaults

```python
from masci_tools.vis.plotting import set_bokeh_plot_defaults, show_bokeh_plot_defaults

set_bokeh_plot_defaults(plot_width=800, plot_height=600, toolbar_location='above')
show_bokeh_plot_defaults()
```

4) Resetting defaults

```python
from masci_tools.vis.plotting import reset_bokeh_plot_defaults
reset_bokeh_plot_defaults()
```

5) Saving and loading defaults

```python
from masci_tools.vis.plotting import save_bokeh_defaults, load_bokeh_defaults

save_bokeh_defaults(filename='my_plot_defaults.json', save_complete=False)
load_bokeh_defaults(filename='my_plot_defaults.json')
```

## Developer Guidance

- Extending plotting capabilities
  - Implement new general plotting routines by following the pattern used by `bokeh_scatter` and `bokeh_multi_scatter`.
  - Ensure input handling goes through `process_data_arguments` to maintain consistency.
  - Use the `ensure_plotter_consistency(plot_params)` decorator to enforce plotter consistency across multiple plotting calls.
  - Leverage `plot_params` methods for:
    - Defaults management (`set_defaults`, `reset_defaults`, `get_dict`, `get_description`, `load_defaults`, `save_defaults`)
    - Figure creation and customization (`prepare_figure`, `plot_kwargs`, `add_tooltips`, `draw_straight_lines`, `set_limits`, `save_plot`)
- Testing
  - Write unit tests that validate:
    - Correct handling of `data`, `x`, `y` inputs (including column-name-based inputs)
    - Deprecation warnings when the old data-as-first-argument pattern is used
    - Correct defaults application and reset behavior
    - Plot creation when a `figure` is provided or when a new figure is created
- Documentation
  - Keep docstrings aligned with the public API and ensure examples reflect current behavior.
  - Document any deprecations or changes in input conventions.

## Troubleshooting

- Warning: Deprecation of first-argument data source
  - If you see a deprecation warning, update your calls to pass the data via the `data` keyword and provide `x` and `y` accordingly.
- Inconsistent data lengths
  - Ensure `x` and `y` have the same length when providing raw arrays. When using `data`, provide columns with matching lengths or enable `same_length` in the data preparation call.
- Tooltip or hover issues
  - Check that the target data is exposed via the `ColumnDataSource` and that `plot_params` tooltips are correctly configured.
- Defaults not applying
  - Use `show_bokeh_plot_defaults()` to verify current defaults. If needed, reset with `reset_bokeh_plot_defaults()` or reload from a file with `load_bokeh_defaults(...)`.
