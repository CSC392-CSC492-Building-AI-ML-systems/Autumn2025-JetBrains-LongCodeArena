# Nextcord Introduction

This document provides a concise introduction to Nextcord with a focus on the widget-related API (WidgetChannel, WidgetMember, and Widget). It explains what widgets are, how the classes are structured, and how to use them in practice.

## Overview

Nextcord is an asynchronous Python library for interacting with the Discord API. It provides high-level abstractions for common Discord concepts, including guild widgets. Widgets expose lightweight, partial representations of a guild’s channels and members for external (public) use, such as displaying information on a server’s widget page.

Key widget classes:
- WidgetChannel: Represents a “partial” widget channel.
- WidgetMember: Represents a “partial” widget member (a guild member visible in the widget).
- Widget: Represents a guild widget, including accessible channels and online members.

This section documents the API surface and typical usage patterns for these types.

## API Reference

The following classes are part of the widget subsystem:

### WidgetChannel

Represents a partial widget channel.

- __init__(id: int, name: str, position: int) -> None
  - id: The channel’s ID.
  - name: The channel’s name.
  - position: The channel’s position.

- __str__(self) -> str
  - Returns the channel’s name.

- __repr__(self) -> str
  - Returns a string representation: `<WidgetChannel id=<id> name=<name> position=<position>>`.

- mention(self) -> str (property)
  - Returns a string that can be used to mention the channel: `<#<id> >`.

- created_at(self) -> datetime.datetime (property)
  - Returns the channel’s creation time in UTC (derived from the channel’s ID).

Usage example:
```python
from nextcord import WidgetChannel

channel = WidgetChannel(id=123, name="General", position=0)
print(channel)          # General
print(channel.mention)  # <#123>
```

### WidgetMember

Represents a “partial” member of the widget’s guild.

- __init__(*, state: ConnectionState, data: WidgetMemberPayload, connected_channel: Optional[WidgetChannel] = None) -> None
  - state: The internal connection state (provided by Nextcord’s internals).
  - data: Payload data for the widget member (including fields like id, name, discriminator, status, nick, avatar, and activity).
  - connected_channel: The channel the member is connected to (optional).

- display_name(self) -> str (property)
  - Returns the member’s display name (nick if available, otherwise the user’s name).

- __repr__(self) -> str
  - Returns a representation including the member’s name, discriminator, bot flag, and nick.

- Attributes (select):
  - id: The member’s ID.
  - name: The member’s username.
  - discriminator: The member’s discriminator.
  - bot: Whether the member is a bot.
  - status: The member’s Status.
  - nick: The member’s nickname (optional).
  - avatar: The member’s avatar hash (optional).
  - activity: The member’s activity (BaseActivity or Spotify) (optional).
  - deafened: Whether the member is deafened (optional).
  - muted: Whether the member is muted (optional).
  - suppress: Whether the member is suppressed (optional).
  - connected_channel: The WidgetChannel the member is currently connected to (optional).

Notes:
- WidgetMember extends BaseUser, and status and activity are derived from the payload.

Usage example:
```python
# NOTE: state is provided by Nextcord's internals in real usage.
state = ...  # ConnectionState (internal state object)
data = {
    'id': 42,
    'name': 'Alice',
    'discriminator': '0001',
    'status': 'online',
    'nick': 'Ally',
    # ... other widget-related fields ...
}
member = WidgetMember(state=state, data=data)
print(member.display_name)  # Ally
```

### Widget

Represents a Guild widget.

- id: The guild’s ID.
- name: The guild’s name.
- channels: List[WidgetChannel] – The accessible voice channels in the guild.
- members: List[WidgetMember] – The online members in the server (offline members do not appear).

- Operations:
  - x == y, x != y
  - str(x) returns the widget’s JSON URL (as a string representation).

- Note:
  - The widget contains partial data about channels and members available to the widget view.

Usage example:
```python
# This is a conceptual example; in real usage, the library provides a Widget instance
# after fetching widget data from Discord.
widget = ...  # instance of nextcord.Widget
print(widget.name)
for ch in widget.channels:
    print(ch.name)
for m in widget.members:
    print(m.display_name)
```

## Quickstart

- Install
  - pip install nextcord

- Basic bot (minimal example)
```python
import nextcord
from nextcord.ext import commands

intents = nextcord.Intents.default()
bot = commands.Bot(command_prefix="!", intents=intents)

@bot.event
async def on_ready():
    print(f"Logged in as {bot.user}")

bot.run("YOUR_BOT_TOKEN")
```

- Working with widgets (conceptual)
  - Enable guild widgets in Discord settings if you plan to rely on widget data.
  - Use Nextcord’s widget-related classes to access partial channel and member information exposed by the widget.
  - Example usage shows how to inspect channel names and member display names from the widget data.

## Remarks

- WidgetChannel, WidgetMember, and Widget are designed to represent partial data exposed by Discord’s guild widget feature.
- These classes provide straightforward accessors and a small set of utilities (e.g., mention, creation time, display name).
- For full widget data, consult Nextcord’s widget parsing and state management in your project, as the library wires payloads from Discord into these Python objects.
