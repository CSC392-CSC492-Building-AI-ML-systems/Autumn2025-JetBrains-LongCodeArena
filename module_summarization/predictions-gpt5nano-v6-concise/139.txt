# Controlling chunking

Overview
- In Dask Array, data is partitioned into chunks. The size and shape of these chunks determine memory usage, parallelism, and the granularity of scheduling.
- You can control chunking when creating arrays, and you can change it later with rechunking. Proper chunking can improve performance and memory behavior for a given workload.
- Operations between arrays (e.g., arithmetic, reductions) may require aligning chunking along axes; Dask can rechunk automatically or you can specify a target chunking pattern.

Creating arrays with explicit chunks
- You can specify the chunking pattern at creation time. This is useful to bound memory per task and to match the computation pattern.

Examples
- Create a 2D NumPy array and wrap it as a Dask array with a specific chunk shape:
  - Chunk shape of (250, 250) on a (1000, 1000) array:

  import numpy as np
  import dask.array as da

  np_arr = np.arange(1_000_000).reshape(1000, 1000)
  A = da.from_array(np_arr, chunks=(250, 250))
  # A has 16 chunks (4 x 4 blocks)

- Create arrays directly filled with zeros or ones with a given chunk shape:
  A_zeros = da.zeros((1000, 1000), chunks=(100, 200))
  A_ones  = da.ones((1000, 1000), chunks=(150, 150))

- When building from sources that expose their own chunking, you can still override via chunks:
  B = da.from_array(np_arr, chunks=(500, 250))

Rechunking: changing chunk patterns
- Rechunking creates a new Dask array with a different chunk layout while preserving data.
- Use da.rechunk(array, new_chunks) to specify the desired chunks along each axis.
- Some APIs also expose a .rechunk(...) method on the Dask Array.

Examples
- Change a 2D array’s chunking to a new pattern:
  A = da.random.random((1000, 1000), chunks=(250, 250))
  B = A.rechunk((500, 200))  # new chunks: (500, 200)

- Rechunk with a single value (same for all axes) or with per-axis specifications:
  C = A.rechunk((100, 100))  # roughly 100x100 blocks
  D = A.rechunk((100, 200))  # 100 along axis 0, 200 along axis 1

- Rechunking for alignment with another array:
  A = da.random.random((1000, 1000), chunks=(250, 400))
  E = da.random.random((1000, 1000), chunks=(500, 250))

  # Align A and E to a common chunking pattern before arithmetic:
  F = (A + E).rechunk((500, 400))
  # Now A and E have a common chunking pattern, minimizing automatic rechunking during the operation.

Aligning chunks for operations
- Binary operations (e.g., +, -, *, @) require compatible shapes and aligned chunking along each axis.
- If two inputs have different chunking along an axis, Dask will typically rechunk to a common pattern automatically, or you can specify a target pattern explicitly.

Examples
- Align two arrays with different chunking before addition:
  A = da.random.random((1000, 1000), chunks=(250, 250))
  B = da.random.random((1000, 1000), chunks=(500, 125))

  # Let Dask pick a common, efficient rechunking:
  C = A + B  # may trigger automatic rechunking

  # Or explicitly set a target common chunking:
  C = (A + B).rechunk((500, 250))

Best practices
- Choose chunk sizes to fit memory per task:
  - If you have a memory budget per worker, pick chunk sizes so that the array block size (dtype-dependent) fits comfortably within that budget.
- Align chunk boundaries with common reductions or groupings:
  - If you frequently reduce along an axis, consider chunk sizes that align with the reduction axis to minimize data movement.
- Avoid unnecessary small chunks:
  - Very small chunks increase task overhead; too-large chunks may reduce parallelism or increase per-task memory usage.
- When combining arrays, aim for a common chunking pattern:
  - Before elementwise operations on large arrays, rechunk to a shared pattern to minimize automatic rechunking during the computation.

Querying and inspecting chunking
- To inspect current chunking:
  - You can print the .chunks attribute of a Dask Array, which shows the size of each chunk along each axis.
  - For example, A.chunks -> ((250, 250, 250, 250), (250, 250, 250, 250))

Notes
- The actual chunk shapes you pick may depend on your compute backend, memory constraints, and the specific operation you are performing.
- The terminology “chunk” in Dask corresponds to a block of the array that is computed and stored as a task in the graph.

See also
- dask.array.rechunk: Rechunk arrays to a new chunking pattern.
- Array creation functions with chunks parameter (from_array, zeros, ones, empty, etc.).
- Blockwise and map_blocks for fine-grained control over chunked computations.

This document provides guidance for developers on how to control and optimize chunking to influence memory usage, scheduling, and performance when using Dask arrays.
