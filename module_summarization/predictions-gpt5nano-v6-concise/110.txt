BestEffortCallback
==================

Overview
--------
BestEffortCallback is a developers-facing callback intended for live monitoring of a data acquisition run. It integrates with the plotting and table utilities (LivePlot, LiveGrid, LiveScatter, LiveTable) and provides a pragmatic, “best effort” visualization and table display of data as it is produced. The callback is designed to be tolerant of irregularities in descriptors and dimensional hints, and it offers toggles to enable/disable heading output, table display, baseline data, and plotting.

Key concepts
------------
- Live visualization: Plots and grids (LivePlot, LiveGrid, LiveScatter) are created on demand based on descriptor information and hints.
- Live table: A lightweight table of primary stream readings can be printed during a run.
- Dimensional handling: The callback attempts to infer independent variables (dimensions) from run start document hints or metadata, with a fallback “guess” when hints are unavailable.
- Robustness: The callback is designed to function even if some streams do not provide consistent hints or if the dimensional hints are incompatible.

Notes and design considerations
- The callback is designed to be permissive with compatibility across old and new descriptor formats. When hints are missing or inconsistent across streams, it falls back to a conservative dimension guess and logs warnings.
- The “dimensions” hint is treated as a structured list of (fields, stream_name) pairs. For each dimension, only the first field is used for plotting, while all fields are tracked for completeness.
- The internal architecture supports multiple live visualizations (plots, grids, scatters) and a live table, with per-descriptor associations stored in dictionaries keyed by descriptor UID.
- The public options allow users to tailor the verbosity and scope of the visualization (heading, table, baseline, plots, etc.).

Usage examples
--------------

Basic usage
- Create the callback with defaults and attach to your RunEngine (RE) or experiment controller.

.. code-block:: python

   from your_module import BestEffortCallback

   cb = BestEffortCallback()

   RE = ...  # your RunEngine setup
   RE.subscribe(cb)

Custom figure factory
- If you want to customize figure creation, provide a fig_factory.

.. code-block:: python

   def my_factory():
       fig, ax = plt.subplots()
       return fig, ax

   cb = BestEffortCallback(fig_factory=my_factory)
   RE.subscribe(cb)

Toggle features
- Enable or disable heading, table, baseline, and plots as needed.

.. code-block:: python

   cb = BestEffortCallback()
   cb.enable_table()
   cb.disable_plots()
   cb.enable_heading()
   RE.subscribe(cb)

Notes for developers

- When extending this callback, consider adding more explicit handling for additional descriptor shapes or richer hints, and ensure consistency between the dimension inference logic and the plotting capability.
