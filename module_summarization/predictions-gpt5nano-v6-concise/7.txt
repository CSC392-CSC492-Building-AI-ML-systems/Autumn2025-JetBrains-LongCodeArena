Using lists
============

Overview
--------
Lists are a fundamental data structure in the ftrack API and in Python in general. In the ftrack ecosystem you will commonly:
- collect multiple entity identifiers (IDs) to perform batch operations,
- accumulate results returned from the API into a list-like collection,
- manipulate and transform lists with Python list utilities and standard library modules.

The API frequently returns or accepts list-like objects (Python lists or ftrack_api.collection.Collection). Treat these as ordinary sequences: index, slice, iterate, and pass to functions that expect iterables.

Common patterns
---------------
- Build lists of IDs from a set of entities for batch operations.
- Convert any iterable of entities into a Python list for materialization or persistence.
- Filter, sort, and deduplicate results using standard Python techniques.
- Process large lists in chunks to respect API limits or performance constraints.
- Serialize lists (e.g., to JSON) for transport or storage.

List-like types in the API
--------------------------
- Python lists: standard mutable sequences used throughout Python code.
- ftrack_api.collection.Collection (and similar): behave like lists, often used for API results. They support iteration and common sequence operations and can be converted to Python lists when needed.

Best practices
--------------
- Materialize only as needed: avoid loading extremely large result sets into memory if you can stream or paginate.
- Validate elements in lists: ensure items are of the expected type (e.g., entity IDs are strings/UUIDs).
- Use chunking for bulk operations: when sending large batches to the server, split into manageable chunks.
- Prefer list comprehensions for readability and performance.
- When serializing to JSON, convert complex objects to primitive types (e.g., extract IDs with a list comprehension).

Examples
--------

1) Materializing a collection of entities to a Python list
- Retrieve a collection from the API and convert to a standard Python list for further processing.

Code:
- entities = session.query('Task').all()
- entities_list = list(entities)
- # Now you can index, slice, or pass to functions expecting a Python list

2) Building a list of IDs from entities
- Collect IDs for a batch operation (e.g., bulk update or fetch)

Code:
- ids = [e.id for e in entities_list]

3) Filtering and sorting a list of entities
- Simple filter and sort using native Python

Code:
- tasks = [e for e in entities_list if getattr(e, 'type', None) == 'Task']
- tasks_sorted = sorted(tasks, key=lambda t: getattr(t, 'name', ''))
- # tasks_sorted is now ordered by name

4) Deduplicating IDs
- Ensure uniqueness when combining multiple sources

Code:
- combined_ids = list(set(list_a_ids) + list_b_ids))

5) Processing large lists in chunks
- Break a large list into chunks to respect API limits or improve responsiveness

Code:
- import itertools
- def chunks(iterable, size):
-     it = iter(iterable)
-     while True:
-         chunk = list(itertools.islice(it, size))
-         if not chunk:
-             break
-         yield chunk

- for chunk in chunks(ids, 50):
-     # perform batch operation with chunk
-     process_batch(chunk)

6) Serializing a list of IDs to JSON
- Prepare data for storage or HTTP transport

Code:
- import json
- payload = json.dumps([e.id for e in entities_list])

7) Combining lists with itertools
- Merge multiple sources without creating intermediate lists

Code:
- import itertools
- all_entities = list(itertools.chain(source1, source2))

8) Selecting a subset by index or slice
- Quick access to a subset of results

Code:
- first_five = entities_list[:5]
- last_two = entities_list[-2:]

Performance and safety tips
----------------------------
- Large result sets: prefer streaming or server-side pagination when possible; materialize only what you need.
- Validation: assert or validate element types when composing lists from heterogeneous sources.
- Memory usage: converting a very large Collection to a Python list increases memory usage; consider iterating over the collection directly when possible.
- Consistency: when combining lists from multiple sources, ensure they share the same element type and structure (e.g., all IDs are strings).

Reference
---------
- Python lists: standard mutable sequences with indexing, slicing, iteration, and many utilities.
- ftrack_api.collection.Collection: a list-like API result type; supports iteration and conversion to Python lists via list(collection).
