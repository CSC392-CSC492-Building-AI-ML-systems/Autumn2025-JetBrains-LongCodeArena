# Biclustering

Biclustering, or subspace clustering, aims to find patterns that are coherent in both rows and columns of a data matrix. Unlike standard clustering, which groups rows (or samples) or columns (features) independently, biclustering seeks simultaneous groupings of rows and columns that reveal meaningful, localized structure (e.g., checkerboard patterns). This is especially useful for datasets where clusters exist only on subsets of features or observations, such as gene expression data or user-item matrices with contextual subgroups.

## Why use biclustering

- Discover localized patterns that are not visible with global clustering.
- Handle heterogeneous data where different submatrices exhibit distinct structure.
- Provide interpretable blocks (biclusters) that align rows and columns.

## Algorithms implemented

The biclustering module provides spectral approaches that are well-suited for discovering checkerboard-like structures in data.

- SpectralBiclustering
  - Finds row and column clusters by applying spectral methods followed by iterative refinement.
  - Produces a checkerboard-like partition of the data matrix.
  - Suitable when you expect multiple, potentially overlapping biclusters with varying row/column memberships.

- SpectralCoclustering
  - Aims to partition both rows and columns into blocks that best approximate a block-constant (or piecewise-constant) structure.
  - Often more robust to noise and can yield more globally consistent coclusterings.

## API overview (Python interface)

The biclustering estimators live in the scikit-learn API under the clustering module. Typical usage:

- Import:
  - from sklearn.cluster import SpectralBiclustering, SpectralCoclustering

- Common usage pattern:
  - model = SpectralBiclustering(n_clusters=(n_row_clusters, n_col_clusters), ...)

  - model.fit(X)
  - row_labels_  # array of shape (n_rows,)
  - column_labels_  # array of shape (n_columns,)

  - For Coclustering, similar attributes exist:
    - row_labels_
    - column_labels_

Notes:
- X must be a 2D array-like (n_samples, n_features).
- The estimators assign each row to one of the row clusters and each column to one of the column clusters, effectively revealing bicluster blocks.

Common parameters (details vary slightly by class):
- n_clusters: int or tuple (n_row_clusters, n_col_clusters)
  - Specifies the number of row and column clusters.
- random_state: int, RandomState instance, or None
  - Controls randomness for initialization.
- Other method-specific options (e.g., for SpectralBiclustering) fine-tune the spectral decomposition and block discovery.

## Outputs and interpretation

- row_labels_: array of shape (n_rows,)
  - Each entry gives the row cluster assignment for the corresponding row.
- column_labels_: array of shape (n_columns,)
  - Each entry gives the column cluster assignment for the corresponding column.

Bicluster blocks can be inferred by cross-referencing row_labels_ and column_labels_:
- For each pair (r, c) of row cluster and column cluster, the intersection forms a bicluster block.

## Examples

Toy example: create a matrix with two distinct biclusters and run SpectralBiclustering.

```python
import numpy as np
from sklearn.cluster import SpectralBiclustering

# Seeded synthetic data with two biclusters
rng = np.random.RandomState(0)
X = rng.normal(loc=0.0, scale=1.0, size=(100, 80))

# Inject two clear biclustered blocks
X[20:40, 10:30] += 5.0
X[60:90, 40:70] += 4.0

# Biclustering
model = SpectralBiclustering(n_clusters=(2, 2), random_state=0)
model.fit(X)

# Access results
row_labels = model.row_labels_      # shape (100,)
col_labels = model.column_labels_   # shape (80,)

print("Row cluster counts:", np.bincount(row_labels))
print("Column cluster counts:", np.bincount(col_labels))
```

Visual inspection (optional): you can map row/column labels back to a heatmap to see the checkerboard blocks.

Additional example: Spectral Coclustering on the same data.

```python
from sklearn.cluster import SpectralCoclustering

model = SpectralCoclustering(n_clusters=(2, 2), random_state=0)
model.fit(X)

row_labels = model.row_labels_
column_labels = model.column_labels_
```

## When to choose biclustering

- You expect localized patterns that only involve subsets of features and observations.
- Global clustering fails to capture structure due to varying densities or block-like organization.
- You need interpretable blocks that can guide downstream analyses (e.g., identifying gene modules or user segments with specific feature combinations).

## Implementation notes for developers

- These estimators are implemented in the biclustering module and expose a standard fit API compatible with scikit-learn pipelines.
- Outputs are provided via row_labels_ and column_labels_ attributes to enable easy downstream analysis and visualization.
- Memory usage is influenced by the spectral decomposition and the size of X; for large matrices consider data sparsity patterns or reducing dimensionality prior to biclustering if applicable.

## Performance considerations

- Spectral methods scale with the size of the data matrix; for very large matrices, consider preprocessing or subsampling.
- The quality of biclustering depends on the inherent block structure in the data and the chosen number of row/column clusters.
