Gaussian Processes in gaussian_process.rst
==========================================

This documentation covers Gaussian Processes (GPs) as implemented in the project, focusing on kernel-based representations, hyperparameter management, and practical usage for developers building or using GP models.

Overview
--------

- A Gaussian Process defines a distribution over functions. Any finite set of function values is jointly Gaussian.
- A GP is fully specified by a mean function m(x) and a covariance (kernel) function k(x, x′).
- In regression/classification with Gaussian noise, GP priors induce tractable posterior distributions, enabling closed-form inference and analytic gradients with respect to hyperparameters.
- Kernels (covariance functions) are the primary means of encoding prior assumptions about function smoothness, amplitude, periodicity, etc.
- Kernels can be composed: sum for additive effects, product for multiplicative interactions, and exponentiation for scaling. Scalar values in expressions are automatically converted to constant kernels.

Key Concepts
------------

- Gaussian Process: f ~ GP(m, k) where m(x) is the mean function and k(x, x′) is the covariance function.
- Kernel as covariance: k(x, x′) encodes relationships between function values at x and x′.
- Hyperparameters: Kernel hyperparameters control amplitude, length scales, smoothness, noise, etc. They can be optimized by gradient-based methods.
- Hyperparameter bounds: The search space is rectangular; bounds are specified for each hyperparameter. Hyperparameters can be declared fixed to exclude them from optimization.
- Anisotropic vs isotropic length scales:
  - Isotropic: a single length-scale parameter.
  - Anisotropic: a length-scale vector, one parameter per input dimension.
  The implementation checks dimensional consistency when anisotropic length scales are used.

Kernel Abstractions
-------------------

- Hyperparameter: A named, typed description of a single hyperparameter.
  - name: parameter name (e.g., "length_scale" or "constant_value").
  - value_type: currently supports "numeric".
  - bounds: (lower, upper) bounds, or "fixed" to fix the parameter.
  - n_elements: number of elements if the parameter is vector-valued.
  - fixed: whether the parameter is fixed; if None, inferred from bounds.
- Kernel: Abstract base class for all kernels.
  - Objects support: combining via + and * operators, exponentiation via **, and scalar-to-kernel conversion in compositions.
  - Analytic gradients: all kernels provide gradients w.r.t. hyperparameters for efficient optimization.
  - Hyperparameter exposure: kernel.hyperparameters yields a sequence of Hyperparameter objects.
  - Parameter introspection: get_params allows retrieving parameters for inspection or cloning.

Practical Usage
--------------

- Build kernels using composition and constants:
  - Basic kernels (e.g., ConstantKernel, RBFKernel, MaternKernel) can be combined with + and *.
  - Scalar terms in expressions are automatically converted to constant kernels.
  - Example: a typical composite kernel might be RBF(length_scale) + WhiteKernel(noise_level).

- Hyperparameters:
  - Each kernel exposes a set of Hyperparameter objects via kernel.hyperparameters.
  - You can print and inspect hyperparameters, and retrieve their bounds via get_params().
  - Bounds define the optimization search space. If a parameter is "fixed", it will not be optimized.

- Optimization:
  - Kernels support analytic gradient-based hyperparameter optimization.
  - The search space is rectangular (independent bounds per parameter).
  - Fixed hyperparameters are excluded from optimization.

- Length scales:
  - Use _check_length_scale to validate anisotropic length scales.
  - Anisotropic kernels require length_scale.shape[0] equal to the input dimensionality.

API Highlights
--------------

- Hyperparameter(name, value_type, bounds, n_elements=1, fixed=None)
  - Example: Hyperparameter("length_scale", "numeric", bounds=(0.1, 2.0), n_elements=2)
- Kernel base class
  - kernel.hyperparameters: iterable of Hyperparameter objects
  - kernel.get_params(deep=True): dictionary of constructor parameters for cloning and inspection
  - Supports operator overloading for composition (e.g., kernel1 + kernel2, kernel1 * kernel2)
- GaussianProcessRegressor/Classifier integration (conceptual)
  - Kernels are passed to the GP model as the covariance function.
  - Hyperparameters are optimized during training if configured (subject to bounds and fixed flags).

Examples
--------

1) Basic Constant Kernel with bounds

```python
from sklearn.gaussian_process.kernels import ConstantKernel
# A constant kernel with a tunable constant_value in [0.0, 10.0]
kernel = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 10.0))

# Inspect hyperparameters
for hyperparameter in kernel.hyperparameters:
    print(hyperparameter)

# Inspect parameters
params = kernel.get_params()
for key in sorted(params):
    print(f"{key} : {params[key]}")
```

Notes for Developers
--------------------

- Design philosophy:
  - Kernels are first-class objects with clear, inspectable hyperparameters.
  - Users can compose kernels to encode prior beliefs about functions without writing custom covariance code.
  - Gradient-based optimization is supported to learn hyperparameters from data.

- Extensibility:
  - Implement new kernels by subclassing Kernel and providing:
    - A constructor that exposes hyperparameters as Hyperparameter objects.
    - Methods to compute k(x, x′) and its gradients with respect to hyperparameters.
  - Ensure anisotropic support if you expose length-scale-like parameters.

- Validation and error handling:
  - Enforce dimensional consistency for anisotropic length scales.
  - When hyperparameters are provided with shape mismatches, raise informative errors.

- Compatibility:
  - The Hyperparameter framework is designed to integrate with a GP regression/classification API that uses hyperparameter optimization.
  - The get_params method supports cloning and parameter introspection, aligning with scikit-learn conventions.

Design Notes
------------

- Hyperparameter metadata is stored as immutable namedtuples for memory efficiency and easy comparison.
- Bounds handling:
  - If bounds is "fixed", the parameter is not optimized.
  - Otherwise, bounds are stored as 2D arrays to support per-element specifications when n_elements > 1.
- Kernel composition:
  - The library supports expressive kernel engineering via operator overloading, enabling a flexible search space for model selection.
