# Generate Tutorial: Working with Text Data

This document provides a practical guide for developers to generate feature vectors from text documents using scikit-learn’s text data utilities. It covers preprocessing, tokenization, stop words handling, n-grams, and common vectorizers.

## Overview

The text feature extraction utilities in `sklearn.feature_extraction.text` convert raw text into numeric representations suitable for machine learning models. Key components include:

- Preprocessing and tokenization pipelines
- Accent stripping and HTML tag removal helpers
- Stop words handling
- Vectorizers: CountVectorizer, TfidfVectorizer, and HashingVectorizer

## Core concepts

### Preprocessing and analysis pipeline

Text processing is composed by chaining several steps:

- Preprocessor: custom text normalization (e.g., removing HTML, lowercasing)
- Tokenizer: split text into tokens
- N-gram generation: create token sequences (unigrams, bigrams, etc.)
- Analyzer: a higher-level composition that can replace several steps

The internal helpers provide flexible composition:
- `_preprocess(doc, accent_function=None, lower=False)`
- `_analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)`

While these internal functions exist in the library, the public API provides convenient vectorizers that expose and document these capabilities.

### Accent stripping

- `strip_accents_unicode(s)`: remove accented characters using Unicode normalization with preservation of ASCII equivalents when possible.
- `strip_accents_ascii(s)`: transliterate to ASCII by removing or replacing accented characters.

### HTML/XML stripping

- `strip_tags(s)`: quick HTML/XML tag stripper using a regular expression. Useful as a preprocessing step before tokenization.

### Stop words

- Built-in English stop words are available as `ENGLISH_STOP_WORDS`.
- At preprocessing time you can pass a stop word list to filter tokens.
- `_check_stop_list(stop)` resolves the stop word specification (e.g., `"english"`, a path-like string, or a custom collection).

## Vectorizers

### CountVectorizer

Converts a collection of text documents to a matrix of token counts.

Example:
```python
from sklearn.feature_extraction.text import CountVectorizer

texts = [
    "This is the first document.",
    "This is the second document.",
    "And this is the third one."
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts)

print(X.shape)  # (3, number_of_features)
print(vectorizer.get_feature_names_out())  # feature names
```

Key options:
- `analyzer`, `preprocessor`, `tokenizer`, `ngram_range` for custom tokenization
- `stop_words` to filter stop words (e.g., `'english'` or a custom list)
- `strip_accents` to control accent stripping (examples: `strip_accents_unicode`, `strip_accents_ascii`)

### TfidfVectorizer

Computes TF-IDF features from the text documents.

Example:
```python
from sklearn.feature_extraction.text import TfidfVectorizer

texts = [
    "This is the first document.",
    "This is the second document.",
    "And this is the third one."
]

tfidf = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
X = tfidf.fit_transform(texts)

print(X.shape)  # (3, number_of_features)
print(tfidf.get_feature_names_out())  # feature names
```

Key options:
- `stop_words` (e.g., `'english'` or a custom list)
- `ngram_range` to enable unigrams and bigrams (e.g., `(1, 2)`)
- Accent stripping and HTML stripping can be incorporated via preprocessing

### HashingVectorizer

Maps text data into a fixed-dimensional sparse feature space using the hashing trick. No vocabulary is stored, which can be advantageous for large corpora.

Example:
```python
from sklearn.feature_extraction.text import HashingVectorizer

texts = [
    "This is the first document.",
    "This is the second document.",
    "And this is the third one."
]

hasher = HashingVectorizer(n_features=2**20, alternate_sign=False)
X = hasher.fit_transform(texts)

print(X.shape)  # (3, 1048576)
```

Notes:
- Memory usage is predictable due to fixed `n_features`.
- No dependency on a dictionary of terms.

## Practical usage patterns

### Simple text classification pipeline

```python
from sklearn.pipeline import Pipeline
from sklearn.linear_model import LogisticRegression
from sklearn.feature_extraction.text import TfidfVectorizer

pipeline = Pipeline([
    ('tfidf', TfidfVectorizer(
        stop_words='english',
        ngram_range=(1, 2)  # unigrams and bigrams
    )),
    ('clf', LogisticRegression(max_iter=1000))
])

texts = [
    "Free money!!! Act now",
    "Hi there, how are you today?",
    "Earn money quickly with this simple trick"
]
labels = [1, 0, 1]

pipeline.fit(texts, labels)
pred = pipeline.predict(["This is a money making scheme"])
```

### Custom preprocessing and tokenization

You can customize the preprocessing and tokenization steps by supplying your own functions to the vectorizers.

```python
from sklearn.feature_extraction.text import CountVectorizer
import re

def remove_html(text):
    return re.sub(r'<[^>]+>', ' ', text)

def custom_tokenizer(text):
    # simple whitespace tokenizer with lowercasing
    return text.lower().split()

texts = [
    "<p>Sample <b>text</b> with HTML</p>",
    "Another sample text."
]

vectorizer = CountVectorizer(preprocessor=remove_html, tokenizer=custom_tokenizer)
X = vectorizer.fit_transform(texts)
print(vectorizer.get_feature_names_out())
```

### Accent handling and HTML stripping in preprocessing

```python
from sklearn.feature_extraction.text import TfidfVectorizer

texts = [
    "résumé",
    "café",
    "naïve approach"
]

tfidf = TfidfVectorizer(
    analyzer='word',
    stop_words=None,
    lowercase=True,
    strip_accents='unicode'  # or 'ascii' via a custom path if needed
)
X = tfidf.fit_transform(texts)
print(tfidf.get_feature_names_out())
```

## Best practices

- Start with `TfidfVectorizer` for text classification tasks to downweight common terms.
- Use `stop_words='english'` (or a custom stop list) to reduce noise.
- Use `ngram_range=(1, 2)` or `(1, 3)` when phrase information improves performance, balancing features and sparsity.
- For very large corpora, consider `HashingVectorizer` to avoid building a vocabulary, trading interpretability for scalability.
- If you need full interpretability of features, prefer `CountVectorizer` or `TfidfVectorizer` with a defined vocabulary.

## Troubleshooting tips

- If you encounter a NotFittedError, ensure you call `fit` (or `fit_transform`) before `transform` or using the pipeline.
- If the feature space is too large, adjust `ngram_range` or switch to `HashingVectorizer` with a reasonable `n_features`.
- If you observe poor performance, inspect the most informative features from the vectorizer (e.g., via `get_feature_names_out()` and relevance scores after training a model).

## API reference (high level)

- `CountVectorizer`: token counts to a sparse feature matrix.
- `TfidfVectorizer`: TF-IDF weighted features.
- `HashingVectorizer`: fixed-size, memory-efficient hashing-based features.
- Accent stripping: `strip_accents_unicode`, `strip_accents_ascii`.
- Tag stripping: `strip_tags`.
- Stop words: built-in with `ENGLISH_STOP_WORDS`, and `_check_stop_list` for various stop list inputs.

This guide provides a practical foundation for generating and using text features in machine learning workflows. For deeper invocation details, refer to the public API documentation of the respective vectorizers in scikit-learn.
