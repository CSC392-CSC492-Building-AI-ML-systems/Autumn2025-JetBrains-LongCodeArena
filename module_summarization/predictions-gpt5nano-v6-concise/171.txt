# Calculator Interface

This document describes the Calculator Interface used by sasmodels to query and evaluate model parameters across all available models. It covers discovery of model parameters, how the command-line interface works, and guidance for developers extending the interface with new models.

## Overview

- The Calculator Interface provides a uniform way to discover which parameters are used by which models.
- It exposes a lightweight query mechanism to list all parameters (and optionally the models that use them) and to filter by parameter kind.
- Core interactions rely on:
  - list_models(): returns the available model names.
  - load_model_info(name): loads metadata for a given model, including its parameters.
  - Model parameter metadata: each model exposes kernel_parameters with attributes such as name and type.

## Core concepts

- Parameter discovery: collect all parameters used by all models.
- Parameter kind: a parameter classification (e.g., volume, orientation, sld) used to filter which parameters are listed.
- Output formats:
  - Names only: just the parameter names.
  - Names with models: parameter name followed by the list of models that use it.

## Key components (from the calculator code)

- find_pars(kind=None)
  - Scans all models returned by list_models().
  - For each model, iterates over model_info.parameters.kernel_parameters.
  - If kind is None or p.type == kind, adds the parameter name and associated models to a table.
  - Returns a dictionary: { parameter_name: [model, model, ...] }

- list_pars(names_only=True, kind=None)
  - Builds the parameter table via find_pars(kind).
  - If names_only is True, prints the sorted list of parameter names.
  - If names_only is False, prints lines in the form: "parameter: model1, model2, ..."

- main() (CLI entry point)
  - Accepts:
    - -v / --verbose: print models that use each parameter (i.e., not just the parameter names).
    - kind: optional positional argument to filter by parameter kind. Accepted values: volume, orientation, sld, none, any.
      - The argument handling maps 'any' and 'none' to None (no filtering); other values pass through as a standard kind.
  - Usage: python -m sasmodels.list_pars [-v] [volume|orientation|sld|none|any]

- Example usage (CLI)
  - List all parameter names:
    python -m sasmodels.list_pars
  - Verbose listing (parameter -> models):
    python -m sasmodels.list_pars -v
  - List only volume-type parameters:
    python -m sasmodels.list_pars volume

## API surface

- list_models(): returns a list of model names available in the sasmodels registry.
- load_model_info(model_name): returns an object containing model metadata, including:
  - parameters.kernel_parameters: an iterable of parameter objects.
  - Each parameter object has:
    - name: the parameter name (string).
    - type: the parameter kind, e.g., 'volume', 'orientation', 'sld', etc.

- Kernel parameters
  - Represent the parameters that are used by the model’s kernel (the mathematical form used to compute model outputs).
  - Each parameter has at least a name and a type that indicates its kind.

## Developer usage and workflow

- Discovering parameters across models
  - Use find_pars() to collect a mapping from parameter names to the models that use them.
  - You can filter by type by passing kind (e.g., kind='volume').

- Extending with new models
  - Add a new model to your registry so that list_models() includes the model name.
  - Ensure the model defines kernel_parameters with appropriate .name and .type attributes.
  - After adding, run the calculator CLI to verify parameter discovery reflects the new model.

- Filtering and reporting
  - Use the CLI to quickly understand parameter usage across models.
  - Use the verbose flag (-v) when you need to know which models rely on a given parameter.

## Examples

- List all parameter names (names_only):
  - Command:
    python -m sasmodels.list_pars
  - Output (example):
    q, scale, sld_core, volfraction, ...

- List parameters with the models that use them (verbose):
  - Command:
    python -m sasmodels.list_pars -v
  - Output (example):
    q: modelA, modelB
    scale: modelA, modelC
    sld_core: modelB

- List parameters by kind (volume):
  - Command:
    python -m sasmodels.list_pars volume
  - Output (example):
    volume: modelA, modelC
    thickness: modelA

- Python API usage (conceptual)
  - from sasmodels import list_models, load_model_info
  - models = list_models()
  - for name in models:
      info = load_model_info(name)
      for p in info.parameters.kernel_parameters:
          print(p.name, p.type)

Note: The exact attributes and availability depend on the current model registry and the model metadata implementation.

## Notes and considerations

- The -v flag makes the output more verbose by including the models that use each parameter; without it, only parameter names are printed.
- The kind argument supports volume, orientation, sld, none, and any (where none/any are treated as no filtering).
- The interface is designed to be lightweight and introspective to aid in model development, debugging, and documentation generation.
