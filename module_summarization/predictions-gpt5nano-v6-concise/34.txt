# w_stateprobs: Developer Documentation

Overview
- w_stateprobs is a compatibility shim around the DStateProbs tool used for computing state-to-state kinetics by tracing weighted-ensemble trajectories.
- It exposes a subcommand named `trace` (via the WStateProbs class) and is wired to the main w_direct-style interface.
- This wrapper is deprecated in favor of using w_direct directly. A deprecation warning is emitted when w_stateprobs is invoked.

Key design notes for developers
- The shim subclass (WStateProbs) sets up the subcommand, help text, and default files to preserve backward compatibility with existing scripts that call w_stateprobs trace.
- The top-level command (WDirect) is a WestMasterCommand + WestParallelTool that provides the unified CLI surface via subcommands. Here, it exposes a single subcommand (trace) implemented by WStateProbs.
- Entry point behavior: calling w_stateprobs prints a deprecation warning and then delegates to WDirect.main(), inserting the trace subcommand if none is provided. This makes older scripts work while guiding users toward w_direct.
- Default files:
  - Output file: stateprobs.h5 (default)
  - Kinetics file: assign.h5 (default, used to hold kinetics/assignments for the analysis)
- Functionality remains the same as the underlying DStateProbs tool; the wrapper simply adapts it to the legacy w_stateprobs CLI surface.

Datasets and output format
The command computes averages and confidence intervals for state populations from weighted-ensemble data. If evolution-mode is requested, additional evolution data is produced.

- Output file (default: stateprobs.h5)
  - /avg_state_pops [state]
    - Structured dataset: population of each state across the requested range.

- Optional evolution data (if --evolution-mode is specified):
  - /state_pop_evolution [window][state]
    - Structured dataset: state populations over windows of iterations, with:
      - iter_start: integer, inclusive start iteration of the window
      - iter_stop: integer, exclusive end iteration of the window
      - expected: float, mean rate within the window (units: 1/tau)
      - ci_lbound: float, lower CI bound (units: 1/tau)
      - ci_ubound: float, upper CI bound (units: 1/tau)
      - corr_len: integer, correlation length within the window (units: tau)
    - Attributes stamping the datasets:
      - mcbs_alpha: float, alpha value for bootstrap CIs
      - mcbs_nsets: integer, number of bootstrap datasets used
      - mcbs_acalpha: float, alpha used for determining correlation lengths

Recommended invocation flow
- Deprecated wrapper (for compatibility):
  - w_stateprobs trace
  - This prints a deprecation warning and delegates to the underlying w_direct-based flow, ensuring the same options are available as with w_direct.
- Modern usage (preferred):
  - w_direct trace
  - This is functionally equivalent and avoids the deprecation path.

Common command-line options (inferred from DStateProbs and wrapper)
- -o, --output FILE
  - Output HDF5 file (default: stateprobs.h5)
- --kinetics-file FILE
  - Kinetics/assignment file (default: assign.h5)
- --evolution-mode {cumulative, blocked}
  - Enable computation of state-pop evolution datasets
- --start-iter N
  - Iteration at which the averaging window starts (inclusive)
- --step-iter N
  - Window width increment or window step size (depending on mode)
- Other options are inherited from DStateProbs (e.g., state/bin definitions and the state-to-state mappings). The wrapper exists to preserve the legacy option surface and defaults; the actual option parsing is performed by DStateProbs via the w_direct entry path.

Examples
- Basic usage (default outputs, same kinetics file):
  - w_stateprobs trace
- Specify an output file:
  - w_stateprobs trace -o my_stateprobs.h5
- Enable evolution data (cumulative windows):
  - w_stateprobs trace --evolution-mode cumulative --start-iter 0 --step-iter 100
- Use a non-default kinetics/assignment file:
  - w_stateprobs trace --kinetics-file custom_assign.h5
- Modern usage (recommended): use w_direct instead of the deprecated wrapper
  - w_direct trace

Implementation notes for maintainers
- WStateProbs:
  - subcommand = 'trace'
  - help_text = 'averages and CIs for path-tracing kinetics analysis'
  - default_output_file = 'stateprobs.h5'
  - default_kinetics_file = 'assign.h5'
- WDirect:
  - prog = 'w_stateprobs'
  - subcommands = [WStateProbs]
  - The description block documents the output format and datasets, mirroring the long-form documentation in the code.
- entry_point():
  - Emits a deprecation warning:
    "{} is being deprecated. Please use w_direct instead.".format(WDirect.prog)
  - Ensures the subcommand 'trace' is present in argv, then dispatches to WDirect().main().
- Future direction:
  - As w_direct becomes the canonical interface, consider deprecating/removing the w_stateprobs shim entirely and migrating any user-facing docs to reference w_direct trace only.
