Auto Augment (NVIDIA DALI) — auto_augment.rst
==============================================

Overview
--------
This module provides helpers to apply NVIDIA DALI's AutoAugment policies to batches
of images or videos. It exposes two primary entry points:

- auto_augment: applies a predefined AutoAugment policy to a batch.
- auto_augment_image_net: a convenience wrapper that applies the image_net policy.

The policies implement the augmentation operators described in the AutoAugment paper
(ArXiv:1805.09501) and are exposed via NVIDIA DALI's auto_aug package. The module
supports control over translation-related magnitudes, interpolation, padding, and
randomization seed to enable reproducible augmentation pipelines.

Public API
----------

Functions

- auto_augment(data, policy_name='image_net', shape=None, fill_value=128, interp_type=None, max_translate_abs=None, max_translate_rel=None, seed=None) -> DataNode

  Applies one of the predefined AutoAugment policies to a batch of samples.

  Parameters
  - data: DataNode
      Batch of samples to process. Supported: images (HWC) and videos (FHWC) with dtype uint8.
  - policy_name: str, optional
      Name of the policy to apply. Allowed values:
      'image_net', 'reduced_image_net', 'svhn', 'reduced_cifar10'. Defaults to 'image_net'.
  - shape: DataNode or tuple(int, int), optional
      Size (height, width) of images/frames. If provided, translation magnitude scales with the shape
      (via max_translate_rel). If omitted, magnitudes are constrained by max_translate_abs.
  - fill_value: int, optional
      Padding value used by warp_affine operations (translation, shear, rotate). If None, padding
      uses border value repeated (clamped).
  - interp_type: types.DALIInterpType, optional
      Interpolation method for warp operations. Typically types.INTERP_LINEAR (default) or types.INTERP_NN.
  - max_translate_abs: int or (int, int), optional
      Maximum translation in pixels when shape is not provided. If a tuple, (height_limit, width_limit).
  - max_translate_rel: float or (float, float), optional
      Maximum translation as a fraction of the image shape when shape is provided. If a tuple, (height_frac, width_frac).
  - seed: int, optional
      Random seed for sampling augmentations and magnitudes.

  Returns
  - DataNode
      Transformed batch.

  Notes
  - If policy_name is one of the policies that do not depend on image shape (see policy list below),
    providing shape arguments will raise an exception.
  - Requires NumPy to be installed.

- auto_augment_image_net(data, shape=None, fill_value=128, interp_type=None, max_translate_abs=None, max_translate_rel=None, seed=None) -> DataNode

  Convenience wrapper that applies the image_net policy via auto_augment.

  Parameters are as in auto_augment, with policy_name fixed to 'image_net'.

  Returns
  - DataNode
      Transformed batch.

Internal Policy Details (for developers)
----------------------------------------

Supported predefined policies

- image_net: get_image_net_policy
- reduced_image_net: get_reduced_image_net_policy
- svhn: get_svhn_policy
- reduced_cifar10: get_reduced_cifar10_policy

Policy behavior considerations

- shape-dependent translations: When shape is provided, translation magnitudes are scaled relative to
  the shape using max_translate_rel. This allows larger translations for larger inputs.

- shape-independent translations: When shape is not provided, translations are bounded by max_translate_abs.

- policies_without_translation: The set of policies that do not rely on image shape includes reduced_image_net.
  If policy_name is in this set, supplying a non-null shape argument will raise an exception.

Error handling and validation

- If policy_name is not one of the allowed values, an exception is raised with a message listing
  the supported policy names.
- If a shape-related argument is supplied for a policy that does not use shape (e.g., reduced_image_net),
  an exception is raised indicating the incompatible combination.

Usage patterns
--------------

1) Basic usage with default policy

- Apply the image_net policy to a batch with default magnitudes and padding.

  Example:

  .. code-block:: python

     from nvidia.dali.auto_aug.auto_augment import auto_augment
     # data_batch is a DataNode containing a batch of images (HWC, uint8)
     augmented = auto_augment(data_batch)

2) Use shape to scale translations

- Enable translation magnitudes to scale with the provided shape.

  Example:

  .. code-block:: python

     augmented = auto_augment(
         data_batch,
         policy_name='image_net',
         shape=(224, 224),
         max_translate_rel=0.25,
         seed=123
     )

3) Use a policy that does not rely on shape

- Reduced Image Net policy does not depend on shape; avoid passing shape.

  Example:

  .. code-block:: python

     augmented = auto_augment(
         data_batch,
         policy_name='reduced_image_net',
         max_translate_abs=None,  # explicitly not used
         seed=42
     )

4) Using a non-default policy (SVHN, CIFAR-10 variants)

- Apply the SVHN policy (or reduced CIFAR-10) to a batch.

  Example:

  .. code-block:: python

     augmented_svhn = auto_augment(data_batch, policy_name='svhn', seed=7)

5) Convenience alias: image_net policy

- Use auto_augment_image_net to apply the image_net policy directly.

  Example:

  .. code-block:: python

     augmented = auto_augment_image_net(data_batch, shape=(128, 128), max_translate_rel=0.5, seed=1)

Requirements and compatibility
------------------------------

- NumPy must be installed for the augmentation examples to run. If NumPy is not available, importing
  the module will raise a RuntimeError instructing to install NumPy.

- Dependency: NVIDIA DALI with auto_aug policies (nvidia.dali.auto_aug).

Notes for developers
--------------------

- The module uses internal helpers and translation utilities (e.g., _get_translations, _pretty_select)
  to describe and select augmentations. These are internal implementation details and not part
  of the public API.

- To extend with new policies, implement a new get_<policy>_policy function in the
  NVIDIA DALI auto_aug policy module and add the corresponding entry to the predefined_policies
  mapping in auto_augment.
