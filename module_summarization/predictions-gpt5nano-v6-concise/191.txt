# Validation curves: plotting scores to evaluate models

Validation curves provide a simple, yet powerful way to study how a model’s performance depends on a hyperparameter. By plotting training and validation scores across a range of parameter values, you can diagnose bias/variance trade-offs and select a value that yields good generalization.

- Primary goal: identify a hyperparameter value that gives high, stable validation performance while avoiding overfitting.
- What you plot: scores (e.g., accuracy, F1, mean squared error) on both training and validation sets as a function of the hyperparameter.
- How to use: compare curves to understand underfitting (low training and test scores), overfitting (high training score, low test score), and where the gap between curves is minimized.

Related concepts
- Learning curves show how performance changes with the size of the training set.
- Validation curves focus on a single hyperparameter, keeping the dataset and model configuration otherwise fixed.

API overview
- validation_curve(estimator, X, y, param_name, param_range, scoring=None, cv=None, n_jobs=None, verbose=0, fit_params=None)
  - Computes train_scores and test_scores for each value in param_range.
  - Param_range can be a sequence of values (linear or log-spaced).
  - Returns: train_scores, test_scores, param_range (the same object you passed as param_range).
- ValidationCurveDisplay
  - A plotting helper that renders the validation curve. Use ValidationCurveDisplay.from_estimator(...) for a convenience API, or ValidationCurveDisplay.from_scores(...) if you already have scores.
  - The display shows mean scores across CV folds with optional error bands, and labels the x-axis with the used parameter.

Typical usage pattern
- Compute scores with validation_curve, then plot with ValidationCurveDisplay (or plot manually if you prefer).
- Use cross-validated scores (the default) to assess generalization error.

Examples
1) Basic usage with Logistic Regression (C regularization strength)
- Goal: study how accuracy changes with C on a binary classification task.
- Code:

```python
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import validation_curve
from sklearn.linear_model import LogisticRegression
import numpy as np

X, y = load_breast_cancer(return_X_y=True)

param_range = np.logspace(-4, 4, 6)  # [1e-4, 1e-3, 1e-2, 1e-1, 1, 10, 1e4]
train_scores, test_scores = validation_curve(
    LogisticRegression(max_iter=1000, solver="lbfgs"),
    X, y,
    param_name="C",
    param_range=param_range,
    cv=5,
    scoring="accuracy",
    n_jobs=None,
)

# Optional: quick plot
import matplotlib.pyplot as plt
train_mean = train_scores.mean(axis=1)
train_std  = train_scores.std(axis=1)
test_mean  = test_scores.mean(axis=1)
test_std   = test_scores.std(axis=1)

plt.semilogx(param_range, train_mean, label="Training score")
plt.semilogx(param_range, test_mean, label="Validation score")
plt.fill_between(param_range, train_mean - train_std, train_mean + train_std, alpha=0.2)
plt.fill_between(param_range, test_mean - test_std, test_mean + test_std, alpha=0.2)
plt.xlabel("C (inverse regularization strength)")
plt.ylabel("Accuracy")
plt.legend()
plt.title("Validation Curve for Logistic Regression (C)")
plt.show()
```

2) Validation curve using a RandomForestClassifier
- Goal: evaluate how the number of trees affects accuracy.
- Code:

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import validation_curve
import numpy as np

X, y = load_breast_cancer(return_X_y=True)

param_range = [10, 50, 100, 200, 500]
train_scores, test_scores = validation_curve(
    RandomForestClassifier(n_estimators=100, random_state=0),
    X, y,
    param_name="n_estimators",
    param_range=param_range,
    cv=5,
    scoring="accuracy",
    n_jobs=-1,
)

# Plotting using mean scores and confidence bands (as above) or via ValidationCurveDisplay
```

3) Convenient plotting with ValidationCurveDisplay.from_estimator
- Goal: concise plotting with a single call.
- Code:

```python
from sklearn.datasets import load_breast_cancer
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import ValidationCurveDisplay

X, y = load_breast_cancer(return_X_y=True)
param_range = [10, 50, 100, 200, 500]

display = ValidationCurveDisplay.from_estimator(
    estimator=RandomForestClassifier(n_estimators=100, random_state=0),
    X=X,
    y=y,
    param_name="n_estimators",
    param_range=param_range,
    cv=5,
    scoring="accuracy",
)
display.plot()
```

Interpretation tips
- If both training and validation scores are low across all parameter values, the model is underfitting.
- If training score is high but validation score is poor, the model is overfitting.
- A "sweet spot" is a parameter value where the validation score is maximized and the curves are reasonably close to each other.
- Use log-spaced ranges for parameters like regularization strength (C), learning rate, or gamma to capture wide ranges.

Notes
- Parameter names must correspond to an actual hyperparameter of the chosen estimator.
- The scoring parameter accepts any valid scorer (e.g., accuracy, f1, neg_mean_squared_error). If a loss-based scorer is used (e.g., neg_mean_squared_error), higher scores may indicate worse performance depending on the metric; interpret accordingly.
- Cross-validation (cv) controls the folds used to estimate generalization performance.

See also
- sklearn.model_selection.learning_curve: for how curves can be used in conjunction with learning curves to analyze data sufficiency.
- ValidationCurveDisplay (from_estimator, from_scores) for convenient plotting of validation curves.
