Dataset Loading Utilities
=========================

This module provides utilities for loading datasets from various sources, including OpenML. The main function in this module is `fetch_openml`, which allows users to download datasets from the OpenML repository. The module also includes helper functions for handling network errors and caching downloaded files.

Functions
---------

1. `fetch_openml`
    - This function downloads a dataset from the OpenML repository and returns it as a `Bunch` object.
    - Parameters:
        - `name` (str): The name of the dataset to download.
        - `version` (int, optional): The version of the dataset to download. Defaults to the latest version.
        - `data_home` (str, optional): The directory where the dataset will be cached. If `None`, no caching will be applied.
        - `return_X_y` (bool, optional): Whether to return the data and target as separate arrays. Defaults to `False`.
        - `as_frame` (bool, optional): Whether to return the data and target as pandas DataFrames. Defaults to `False`.
    - Returns:
        - `Bunch`: A dictionary-like object with the dataset attributes.

2. `_get_local_path`
    - This function constructs the local path for caching a dataset file.
    - Parameters:
        - `openml_path` (str): The OpenML URL of the dataset.
        - `data_home` (str): The directory where the dataset will be cached.
    - Returns:
        - `str`: The local path for caching the dataset file.

3. `_retry_with_clean_cache`
    - This decorator function retries a decorated function if it encounters an exception, cleaning the cache before retrying.
    - Parameters:
        - `openml_path` (str): The OpenML URL of the dataset.
        - `data_home` (str, optional): The directory where the dataset will be cached.
        - `no_retry_exception` (Exception, optional): An exception type that should not trigger a retry.
    - Returns:
        - `Callable`: A decorated function that handles retries with cache cleaning.

4. `_retry_on_network_error`
    - This decorator function retries a decorated function if it encounters a network error, up to a specified number of retries.
    - Parameters:
        - `n_retries` (int, optional): The number of retries. Defaults to 3.
        - `delay` (float, optional): The delay between retries. Defaults to 1.0.
        - `url` (str, optional): The URL being accessed for additional error information.
    - Returns:
        - `Callable`: A decorated function that handles retries on network errors.

5. `_open_openml_url`
    - This function retrieves a resource from OpenML.org, optionally caching it to a specified directory.
    - Parameters:
        - `openml_path` (str): The OpenML URL of the resource.
        - `data_home` (str, optional): The directory where the resource will be cached.
        - `n_retries` (int, optional): The number of retries for HTTP errors. Defaults to 3.
        - `delay` (float, optional): The delay between retries. Defaults to 1.0.
    - Returns:
        - `stream`: A stream to the OpenML resource.

6. `_get_json_content_from_openml_api`
    - This function loads JSON data from the OpenML API and handles network errors.
    - Parameters:
        - `url` (str): The URL of the OpenML API endpoint.
        - `error_message` (str, optional): An error message to display on failure.
        - `data_home` (str, optional): The directory where the data will be cached.
        - `n_retries` (int, optional): The number of retries for network errors. Defaults to 3.
        - `delay` (float, optional): The delay between retries. Defaults to 1.0.
    - Returns:
        - `Dict`: The JSON data loaded from the OpenML API.

Classes
-------

1. `OpenMLError`
    - This class represents a specific OpenML error with the HTTP 412 status code, indicating a generic error.
