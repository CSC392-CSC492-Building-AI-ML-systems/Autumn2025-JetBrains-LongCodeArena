Generate documentation for this module
=======================================

The `StorageService` class provided in this module is designed for MongoDB storage and depends on the `pymongo` and `motor` libraries. It offers methods for interacting with a MongoDB database, such as retrieving data, inserting or updating documents, and deleting documents.

Usage
-----

To use the `StorageService` class, you need to instantiate it with the required parameters:

- `app`: The ASAB application instance.
- `service_name`: The name of the storage service.
- `config_section_name`: The name of the configuration section containing MongoDB connection details.

Example:

```python
storage = StorageService(app, "my_storage_service")
```

Methods
-------

1. `upsertor(collection: str, obj_id=None, version=0) -> MongoDBUpsertor`:
   Returns an instance of `MongoDBUpsertor` for the specified collection, object ID, and version.

2. `get(collection: str, obj_id, decrypt=None) -> dict`:
   Retrieves a document from the specified collection based on the object ID. Optionally decrypts specified fields.

3. `get_by(collection: str, key: str, value, decrypt=None) -> dict`:
   Retrieves a document from the specified collection based on a key-value pair. Optionally decrypts specified fields.

4. `collection(collection: str) -> motor.motor_asyncio.AsyncIOMotorCollection`:
   Returns an `AsyncIOMotorCollection` object for the specified collection, useful for custom operations.

5. `delete(collection: str, obj_id) -> str`:
   Deletes a document from the specified collection based on the object ID.

6. `MongoDBUpsertor` class:
   - `generate_id() -> bson.objectid.ObjectId`: Generates a new ObjectId for MongoDB documents.
   - `execute(custom_data: typing.Optional[dict] = None, event_type: typing.Optional[str] = None)`: Executes the upsert operation with custom data and event type.

Additional Notes
----------------

- The `StorageService` class utilizes the `motor` library for asynchronous MongoDB operations.
- Custom data and event types can be provided when executing the upsert operation using the `MongoDBUpsertor` class.
- Webhook URIs can be configured in the storage service for handling events related to document updates.
