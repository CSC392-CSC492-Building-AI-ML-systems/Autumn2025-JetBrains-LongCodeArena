Models
======

Model Usage
-----------

The `Model` class in Tortoise ORM represents a database table. It is used to define the structure of the data stored in the database and provides methods for interacting with the database table.

Attributes
~~~~~~~~~~

- `MODEL`: A type variable representing a model class.

Functions
~~~~~~~~~~

- `get_together(meta: "Model.Meta", together: str) -> Tuple[Tuple[str,...], ...]`: Retrieves the `together` attribute from the model's meta class.
- `prepare_default_ordering(meta: "Model.Meta") -> Tuple[Tuple[str, Order], ...]`: Prepares the default ordering for the model.
- `_fk_setter(self: "Model", value: "Optional[Model]", _key: str, relation_field: str, to_field: str) -> None`: Setter function for foreign key fields.
- `_fk_getter(self: "Model", _key: str, ftype: "Type[Model]", relation_field: str, to_field: str) -> Awaitable`: Getter function for foreign key fields.
- `_rfk_getter(self: "Model", _key: str, ftype: "Type[Model]", frelfield: str, from_field: str) -> ReverseRelation`: Getter function for reverse foreign key fields.
- `_ro2o_getter(self: "Model", _key: str, ftype: "Type[Model]", frelfield: str, from_field: str) -> "QuerySetSingle[Optional[Model]]"`: Getter function for reverse one-to-one fields.
- `_m2m_getter(self: "Model", _key: str, field_object: ManyToManyFieldInstance) -> ManyToManyRelation`: Getter function for many-to-many fields.
- `_get_comments(cls: "Type[Model]") -> Dict[str, str]`: Retrieves comments from the source code of the model class.

Class: MetaInfo
~~~~~~~~~~~~~~~

A class that holds metadata information about the model.

Attributes
^^^^^^^^^^^

- `abstract`: Indicates if the model is abstract.
- `db_table`: The name of the database table associated with the model.
- `app`: The name of the app the model belongs to.
- `unique_together`: A tuple of unique together constraints.
- `indexes`: A tuple of indexes defined on the model.
- `_default_ordering`: The default ordering of the model.
- `_ordering_validated`: Indicates if the ordering has been validated.
- `fields`: A set of field names in the model.
- `db_fields`: A set of database field names in the model.
- `m2m_fields`: A set of many-to-many field names in the model.
- `fk_fields`: A set of foreign key field names in the model.
- `o2o_fields`: A set of one-to-one field names in the model.
- `backward_fk_fields`: A set of backward foreign key field names in the model.
- `backward_o2o_fields`: A set of backward one-to-one field names in the model.
