# Publishing Versions

The `ftrack` module provides functionality for interacting with an ftrack server and performing various operations. This documentation covers the different versions of the module and their features.

## Version 1.0.0

- Initial release of the `ftrack` module.
- Includes basic functionality for connecting to an ftrack server, authenticating users, and interacting with entities.
- Supports automatic fetching of entity attributes from the server.
- Provides a session authentication mechanism for attaching authentication information to requests.

## Version 1.1.0

- Added support for plugins by specifying paths to search for plugins.
- Introduced caching mechanisms for improving performance.
- Enhanced event handling capabilities with an embedded event hub for publishing and subscribing to events.
- Improved schema caching functionality for storing cache in a specified folder path.

## Version 1.2.0

- Enhanced logging capabilities for better tracking and debugging.
- Improved error handling and exception reporting.
- Added support for weak references to methods.
- Optimized memory usage and performance improvements.

## Version 1.3.0

- Updated dependencies to the latest versions for improved compatibility.
- Fixed bugs and addressed security vulnerabilities.
- Streamlined code structure and improved code readability.
- Enhanced documentation and examples for better understanding and usage.

This documentation provides an overview of the different versions of the `ftrack` module and the features introduced in each version. Users are encouraged to refer to the specific version documentation for detailed information and usage guidelines.
