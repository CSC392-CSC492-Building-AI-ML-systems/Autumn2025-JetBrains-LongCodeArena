=====================

This document provides guidance on migrating to nextcord from the previous version.

Future Compatibility
--------------------

nextcord uses annotations from the `__future__` module to support type hinting. Ensure that your codebase is compatible with this feature.

Type Annotations
----------------

nextcord uses type annotations for improved code readability and maintainability. Ensure that your codebase includes type hints for parameters, return types, and class attributes.

WidgetChannel Class
-------------------

The `WidgetChannel` class represents a "partial" widget channel. It has the following attributes:

- `id`: The channel's ID.
- `name`: The channel's name.
- `position`: The channel's position.

WidgetMember Class
------------------

The `WidgetMember` class represents a "partial" member of the widget's guild. It has the following attributes:

- `id`: The member's ID.
- `name`: The member's username.
- `discriminator`: The member's discriminator.
- `bot`: Whether the member is a bot.
- `status`: The member's status.
- `nick`: The member's nickname.
- `avatar`: The member's avatar hash.
- `activity`: The member's activity.
- `deafened`: Whether the member is currently deafened.
- `muted`: Whether the member is currently muted.
- `suppress`: Whether the member is currently being suppressed.
- `connected_channel`: Which channel the member is connected to.

Widget Class
------------

The `Widget` class represents a Guild widget. It has the following attributes:

- `id`: The guild's ID.
- `name`: The guild's name.
- `channels`: The accessible voice channels in the guild.
- `members`: The online members in the server. Offline members do not appear in the widget.

Please review your codebase and make the necessary changes to migrate to nextcord successfully.
