Generate documentation for this module
=======================================

This module contains a standard suite of augmentations used by AutoAugment policy for ImageNet, RandAugment, and TrivialAugmentWide. The augmentations are implemented in terms of DALI operators.

The `@augmentation` decorator handles the computation of the decorated transformation's parameter. When called, the decorated augmentation expects:
- a single positional argument: batch of samples
- `magnitude_bin` and `num_magnitude_bins` instead of the parameter. The parameter is computed as if by calling `mag_to_param(magnitudes[magnitude_bin] * ((-1) ** random_sign))`, where `magnitudes=linspace(mag_range[0], mag_range[1], num_magnitude_bins)`.

The augmentations in this module are defined with example setups passed to `@augmentation`. The parameters can be easily adjusted. For instance, to increase the magnitudes range of `shear_x` from 0.3 to 0.5, you can create `my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))`.

--------------

   - Parameters:
     - `data`: Input data.
     - `shear`: Shear magnitude.
     - `fill_value`: Value to fill the empty space after transformation.
     - `interp_type`: Interpolation type.
   - Returns: Transformed data.

2. `shear_y(data, shear, fill_value=128, interp_type=None)`: Applies shear transformation along the y-axis.

     - `interp_type`: Interpolation type.
   - Returns: Transformed data.

3. `translate_x(data, rel_offset, shape, fill_value=128, interp_type=None)`: Translates the data along the x-axis.
   - Parameters:
     - `data`: Input data.
     - `rel_offset`: Relative offset for translation.
     - `shape`: Shape of the data.
     - `fill_value`: Value to fill the empty space after transformation.
     - `interp_type`: Interpolation type.
