FITS Headers
============

This module provides functionality for working with FITS headers. FITS (Flexible Image Transport System) headers are metadata stored in FITS files that describe the data contained within the file. The headers contain information such as the data type, dimensions, and any additional information about the data.

Classes
-------

VerifyError
    Exception class for verification errors related to FITS headers.

VerifyWarning
    Warning class for verification warnings related to FITS headers.

_NotifierMixin
    Mixin class that provides services for registering listeners to changes on objects related to FITS headers.

Functions
---------

cmp(a, b)
    Compare two values `a` and `b` and return -1 if `a` is less than `b`, 0 if they are equal, and 1 if `a` is greater than `b`.

Attributes
----------

VERIFY_OPTIONS
    List of options for verifying FITS headers, including "ignore", "warn", "exception", "fix", "silentfix", and various combinations of these options.

path_like
    Types that are considered path-like, including strings, bytes, and objects that implement the `os.PathLike` protocol.

all_integer_types
    Tuple containing all integer types, including `int` and `numpy.integer`.

Constants
---------

Note: astropy.io.fits uses zero-based indexing.
