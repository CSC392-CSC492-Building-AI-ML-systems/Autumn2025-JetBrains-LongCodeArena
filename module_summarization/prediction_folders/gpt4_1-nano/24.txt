w_init Command
==============

Synopsis
--------
w_init [options]

Description
-----------
The `w_init` command initializes a new WEST (Weighted Ensemble Simulation Toolkit) simulation. It creates the necessary HDF5 file and prepares the first iteration's segments. The initialization process involves setting up initial states, target states, and start states based on user-specified inputs or default configurations.

Features
--------
- Creates a new simulation environment with a specified or default initial state.
- Supports reading basis, target, and start states from files or command-line arguments.
- Allows customization of the number of segments per state.
- Provides options to overwrite existing simulation data.
- Integrates with WESTPA's work management system for parallel execution.

Options
-------
--force
    Overwrite any existing simulation data.

--bstate-file, --bstates-from BSTATE_FILE
    Read basis state names, probabilities, and optional data references from the specified file.

--bstate, --bstate (multiple)
    Add a basis state specified as 'label,probability[,auxref]' to the list of basis states. Can be used multiple times to specify multiple states.

--tstate-file, --tstates-from TSTATE_FILE
    Read target state names and representative progress coordinates from the specified file. The entire bin containing the progress coordinate will be used as a recycling sink.

--tstate, --tstate (multiple)
    Add a target state specified as 'label,pcoord0[,pcoord1[,...]]'. Multiple target states can be added by repeating this option.

--sstate-file, --sstates-from SSTATE_FILE
    Read start state names, probabilities, and optional data references from the specified file.

--sstate, --sstate (multiple)
    Add a start state specified as 'label,probability[,auxref]'. Multiple start states can be added by repeating this option.

--segs-per-state N
    Set the number of segments to initialize from each basis state (default: 1).

--no-we, --shotgun
    Do not run the weighted ensemble bin/split/merge algorithm on newly-created segments.

Usage
-----
The command is typically invoked with various options to specify initial states, target states, and other parameters. It sets up the simulation environment accordingly, preparing the system for the first iteration.

Notes
-----
- If neither basis states nor start states are specified, a default basis state with probability one, label "basis", and state ID zero will be created.
- The command integrates with the WESTPA work management system to handle parallel execution and resource management.
- Proper configuration of input files and parameters is essential for accurate simulation setup.

Example
-------
```bash
w_init --bstate-file basis_states.txt --tstates-from target_states.txt --segs-per-state 10 --force
```

This example initializes a simulation with basis states and target states read from files, sets 10 segments per basis state, and forces overwriting existing data.
