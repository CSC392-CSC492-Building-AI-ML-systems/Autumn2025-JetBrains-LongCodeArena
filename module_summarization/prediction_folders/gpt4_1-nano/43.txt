Config Module
=============

Overview
--------
The `Config` module provides an enhanced configuration parser built upon Python's standard `configparser`. It offers additional features such as default configuration values, support for including other configuration files, URL validation, and integration with external systems like Zookeeper. This module simplifies managing application settings and ensures flexible, secure, and organized configuration handling.

Classes
-------
ConfigParser
~~~~~~~~~~~~
The `ConfigParser` class extends the standard `configparser.ConfigParser` with additional capabilities:

- **Default Values**: Contains a set of predefined default configuration values accessible via the `_default_values` attribute.
- **Adding Defaults**: The `add_defaults()` method allows merging default values into an existing configuration, filling in missing options.
- **Including Files**: Supports including other configuration files through the `_traverse_includes()` method, enabling nested includes and glob pattern matching.
- **Zookeeper Support**: Can include configuration from Zookeeper servers if specified via environment variables.
- **Configuration Loading**: The `_load()` method initializes the configuration, reading from specified files and handling include directives.

Configuration Structure
-----------------------
The configuration is organized into sections, each containing key-value pairs. Default sections and options include:

- **general**: Basic application settings such as `config_file`, `tick_period`, `var_dir`, and daemonization options (`pidfile`, `working_dir`, `uid`, `gid`).
- **asab:metrics**: Metrics-related settings like `native_metrics`, `web_requests_metrics`, and `expiration`.
- **asab:doc**: Documentation-related options such as `default_route_tag`.
- **logging**: Logging configuration including verbosity, application name, structured data ID (`sd_id`), and log level.
- **logging:console**: Console logging format and date formatting.
- **logging:syslog**: Syslog logging settings, including enabling/disabling, address, and format.
- **logging:file**: File logging options such as path, format, rotation, and backup settings.
- **library**: Library-specific options, e.g., Azure cache path.
- **passwords**: Secure storage for passwords, referenced via environment variables or configuration.
- **housekeeping**: Scheduled tasks configuration, including run times and limits.

Usage
-----
To utilize the configuration system:

1. Instantiate the `ConfigParser` class.
2. Call the `_load()` method once to initialize and load configuration files.
3. Use `add_defaults()` to merge default values into your configuration.
4. Access configuration options via standard `configparser` methods, e.g., `get()`, `set()`, `has_option()`.

Example
-------
```python
config = ConfigParser()
config._load()
config.add_defaults(ConfigParser._default_values)
log_level = config.get('logging', 'level')
```

Notes
-----
- The configuration system supports environment variable expansion within values.
- Includes can be nested and support glob patterns.
- Zookeeper integration allows dynamic inclusion of configuration data.
- Default values are provided for common settings, but can be overridden in configuration files.
- Secure password storage is facilitated through a dedicated `passwords` section, preventing passwords from being exposed in logs or API outputs.

For detailed options and advanced usage, refer to the inline comments within the code and the method documentation.
