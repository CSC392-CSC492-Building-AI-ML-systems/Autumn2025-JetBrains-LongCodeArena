Configuration Options
=====================

This section describes the configuration options available for managing the import and export of various components within the PyAEDT environment.

Options Class
-------------

The `ConfigurationsOptions` class provides a comprehensive set of configurable parameters that allow users to enable or disable the import/export of specific components such as variables, setups, optimizations, parametrics, boundaries, mesh operations, coordinate systems, materials, object properties, and datasets.

Initialization
--------------

Upon instantiation, all options are set to their default values, typically enabled (`True`) for export/import components, and a default object mapping tolerance of `1e-9`.

Properties
----------

### Object Mapping Tolerance

- **object_mapping_tolerance**: Float value specifying the tolerance used in object mapping processes, such as boundary matching. Default is `1e-9`.

### Export Options

- **export_variables**: Boolean indicating whether variables should be exported to a JSON file. Default is `True`.
- **export_setups**: Boolean indicating whether setups should be exported. Default is `True`.
- **export_optimizations**: Boolean indicating whether optimizations should be exported. Default is `True`.
- **export_parametrics**: Boolean indicating whether parametric studies should be exported. Default is `True`.
- **export_boundaries**: Boolean indicating whether boundary conditions should be exported. Default is `True`.
- **export_mesh_operations**: Boolean indicating whether mesh operations should be exported. Default is `True`.
- **export_coordinate_systems**: Boolean indicating whether coordinate systems should be exported. Default is `True`.
- **export_materials**: Boolean indicating whether materials should be exported. Default is `True`.
- **export_object_properties**: Boolean indicating whether object properties should be exported. Default is `True`.
- **export_datasets**: Boolean indicating whether datasets should be exported. Default is `True`.

### Import Options

- **import_datasets**: Boolean indicating whether datasets should be imported from JSON files. Default is `True`.
- **import_variables**: Boolean indicating whether variables should be imported. Default is `True`.
- **import_setups**: Boolean indicating whether setups should be imported. Default is `True`.
- **import_optimizations**: Boolean indicating whether optimizations should be imported. Default is `True`.
- **import_parametrics**: Boolean indicating whether parametric studies should be imported. Default is `True`.
- **import_boundaries**: Boolean indicating whether boundary conditions should be imported. Default is `True`.
- **import_mesh_operations**: Boolean indicating whether mesh operations should be imported. Default is `True`.
- **import_coordinate_systems**: Boolean indicating whether coordinate systems should be imported. Default is `True`.
- **import_materials**: Boolean indicating whether materials should be imported. Default is `True`.

Additional Settings
-------------------

- **skip_import_if_exists**: Boolean flag to skip importing components if they already exist in the project, preventing overwriting.

Usage
-----

These options can be customized programmatically to tailor the import/export behavior according to project requirements, ensuring efficient data management and consistency across simulation setups.
