Unified Renderable Interface
==============================

Overview
--------
The unified renderable interface provides a flexible and extensible way to render templates with associated context data and CSS classes. It is designed to be subclassed to create custom renderable components with consistent rendering behavior and styling capabilities.

AbstractRenderable
------------------
The `AbstractRenderable` class serves as the base class for all renderable objects. It defines the core rendering mechanism and template management.

Methods:
- `get_template_names()`: Returns the template name(s) for rendering. Must be set or overridden.
- `get_context_data(request=None)`: Provides context data for rendering. Defaults to including the instance itself under the key `'me'`.
- `render(request=None, extra_context_data=None)`: Renders the template with the combined context data.

Usage:
Create subclasses of `AbstractRenderable` and specify the `template_name`. Override `get_template_names()` if multiple templates are needed.

AbstractRenderableWithCss
-------------------------
Extends `AbstractRenderable` with a unified API for managing CSS classes.

Features:
- `get_css_classes_list()`: Returns a list of CSS classes for the component. Override to customize.
- `css_classes`: Property that joins the CSS classes into a string, optionally including test CSS classes if enabled.
- `get_test_css_class_suffixes_list()`: Provides suffixes for test CSS classes, filtered through a template tag.

Usage:
Override `get_css_classes_list()` to specify CSS classes. Use the `css_classes` property to obtain the class string for rendering.

AbstractBemRenderable
---------------------
Provides support for BEM (Block Element Modifier) CSS naming conventions.

Features:
- Initialization with `bem_block`, `bem_element`, and `bem_variant_list` to define BEM structure.
- `get_bem_block()`, `get_bem_element()`, and `get_bem_variant_list()`: Accessors for BEM components.
- `bem_block_or_element`: Returns the block or element, prioritizing the block.
- Validation ensures correct BEM structure during initialization.

Usage:
Instantiate with BEM parameters to generate CSS class names following BEM conventions. Extend or override methods for custom behavior.

Summary
-------
The unified renderable interface standardizes how components render templates, manage context data, and handle CSS classes, including support for BEM naming conventions. It encourages consistent implementation and easy extension for complex rendering needs in Django applications.
