Configurations Options
======================

The `ConfigurationsOptions` class provides a comprehensive set of options to control the import and export behavior of various components within a project configuration. This class allows users to enable or disable the import/export of specific elements such as variables, setups, optimizations, parametrics, boundaries, mesh operations, coordinate systems, materials, datasets, and object properties.

Class: ConfigurationsOptions
----------------------------

### Overview

This class is designed to manage configuration options related to the serialization and deserialization of project data. It is particularly useful when exporting project settings to JSON files or importing them back, allowing fine-grained control over which components are included or excluded.

### Attributes and Properties

- **object_mapping_tolerance** (`float`):  
  Specifies the tolerance value used in object mapping processes, such as mapping boundaries.  
  *Default:* `1e-9`  
  *Example:*  
  ```python
  config_options.object_mapping_tolerance = 1e-6
  ```

- **export_variables** (`bool`):  
  Determines whether variables should be exported to the JSON file.  
  *Default:* `True`  
  *Example:*  
  ```python
  config_options.export_variables = False  # Disable variables export
  ```

- **export_setups** (`bool`):  
  Controls the export of setup configurations to the JSON file.  
  *Default:* `True`  
  *Example:*  
  ```python
  config_options.export_setups = False  # Disable setups export
  ```

- **export_optimizations** (`bool`):  
  Enables or disables the export of optimization setups.  
  *Default:* `True`  
  *Example:*  
  ```python
  config_options.export_optimizations = False  # Disable optimizations export
  ```

- **export_parametrics** (`bool`):  
  Controls whether parametric setups are exported.  
  *Default:* `True`  
  *Example:*  
  ```python
  config_options.export_parametrics = False  # Disable parametrics export
  ```

- **export_boundaries** (`bool`):  
  Determines if boundary conditions are included in the export.  
  *Default:* `True`  
  *Example:*  
  ```python
  config_options.export_boundaries = False  # Disable boundaries export
  ```

- **export_mesh_operations** (`bool`):  
  Controls the export of mesh operations.  
  *Default:* `True`

- **export_coordinate_systems** (`bool`):  
  Enables or disables the export of coordinate systems.  
  *Default:* `True`

- **export_materials** (`bool`):  
  Determines if material definitions are exported.  
  *Default:* `True`

- **export_object_properties** (`bool`):  
  Controls the export of object properties.  
  *Default:* `True`

- **export_datasets** (`bool`):  
  Specifies whether datasets should be exported.  
  *Default:* `True`

- **import_datasets** (`bool`):  
  Controls the import of datasets from JSON files.  
  *Default:* `True`

- **import_variables** (`bool`):  
  Enables or disables the import of variables.  
  *Default:* `True`

- **import_setups** (`bool`):  
  Controls the import of setup configurations.  
  *Default:* `True`

- **import_optimizations** (`bool`):  
  Enables or disables the import of optimization setups.  
  *Default:* `True`

- **import_parametrics** (`bool`):  
  Controls the import of parametric setups.  
  *Default:* `True`

- **import_boundaries** (`bool`):  
  Determines if boundary conditions are imported.  
  *Default:* `True`

- **import_mesh_operations** (`bool`):  
  Controls the import of mesh operations.  
  *Default:* `True`

- **import_coordinate_systems** (`bool`):  
  Enables or disables the import of coordinate systems.  
  *Default:* `True`

- **import_materials** (`bool`):  
  Controls the import of material definitions.  
  *Default:* `True`

- **import_object_properties** (`bool`):  
  Specifies whether object properties are imported.  
  *Default:* `True`

- **skip_import_if_exists** (`bool`):  
  If set to `True`, the import process will skip components that already exist in the project.  
  *Default:* `False`

### Usage Example

```python
from pyaedt import Hfss

hfss = Hfss()
# Disable exporting variables and boundaries
hfss.configurations.options.export_variables = False
hfss.configurations.options.export_boundaries = False

# Set a custom tolerance for object mapping
hfss.configurations.options.object_mapping_tolerance = 1e-6
```

Helper Function: `_find_datasets`
---------------------------------

This internal function recursively searches through a nested dictionary structure to find dataset references, particularly those containing piecewise linear (PWL) data. It collects these dataset identifiers into a provided list for further processing.

Parameters:
- `d` (`dict`): The dictionary to search.
- `out_list` (`list`): The list to append found dataset identifiers.

This function is used internally to assist in managing dataset imports and exports.
