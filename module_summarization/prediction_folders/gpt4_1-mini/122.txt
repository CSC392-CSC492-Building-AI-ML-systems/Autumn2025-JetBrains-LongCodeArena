Unified Renderable Interface
============================

This documentation describes a unified interface for renderable components, providing a consistent way to render templates with context data and manage CSS classes. The interface is designed to be extended and customized for various rendering needs.

AbstractRenderable
------------------

The `AbstractRenderable` class is an abstract base class that defines the core interface for rendering. It provides helper methods centered around the `render` method, which renders a template using the object itself as context.

### Attributes

- `template_name`  
  The default template name(s) used by the `render` method. Must be set or overridden.

### Methods

- `get_template_names()`  
  Returns the template name(s) to be used for rendering.  
  Raises `NotImplementedError` if `template_name` is not set.

- `get_context_data(request=None)`  
  Returns a dictionary of context data for rendering.  
  By default, returns `{'me': self}`.

- `render(request=None, extra_context_data=None)`  
  Renders the template(s) returned by `get_template_names` with the context data from `get_context_data`, optionally extended with `extra_context_data`.  
  The `request` parameter, if provided, is forwarded to `get_context_data` and the template rendering function.

AbstractRenderableWithCss
-------------------------

`AbstractRenderableWithCss` extends `AbstractRenderable` by adding a unified API for managing CSS classes.

### Methods

- `get_base_css_classes_list()`  
  Returns a list of base CSS classes. Defaults to an empty list.

- `get_extra_css_classes_list()`  
  Returns a list of extra CSS classes. Defaults to an empty list.

- `get_css_classes_list()`  
  Returns the combined list of CSS classes by extending the base and extra CSS classes lists.  
  Override this method to define the CSS classes for the component.

- `get_test_css_class_suffixes_list()`  
  Returns a list of CSS class suffixes to include when running automatic tests.  
  These suffixes are processed through a template tag to generate test-specific CSS classes.

### Properties

- `css_classes`  
  Returns a string of CSS classes joined from `get_css_classes_list()`.  
  Includes test CSS classes if enabled in settings.  
  This property should be used instead of overriding `get_css_classes_string()`.

### Deprecated

- `get_css_classes_string()`  
  Deprecated in favor of the `css_classes` property.

AbstractBemRenderable
---------------------

`AbstractBemRenderable` is a base class for renderables that use the BEM (Block Element Modifier) methodology for CSS class naming. It provides a natural way to work with BEM conventions.

### Initialization Arguments

- `bem_block` (str)  
  The BEM block name. Cannot be supplied if `bem_element` is provided.

- `bem_element` (str)  
  The BEM element name. Must contain `__` and cannot be supplied if `bem_block` is provided.

- `bem_variant_list` (list)  
  A list of BEM variants (modifiers) for the block or element. Only the variant part (after `--`) should be included.

- `extra_css_classes_list` (list)  
  A list of additional CSS classes to include.

### Methods

- `get_test_css_class_suffixes_list()`  
  Returns a list of CSS class suffixes to include when running automatic tests, similar to `AbstractRenderableWithCss`.

- `bem_block_or_element` (property)  
  Returns the BEM block if set, otherwise the BEM element.

- `get_bem_block()`  
  Returns the BEM block string.

- `get_bem_element()`  
  Returns the BEM element string.

Usage Summary
-------------

The unified renderable interface allows components to:

- Define which template(s) to render.
- Provide context data for rendering.
- Render templates with optional request and extra context.
- Manage CSS classes in a consistent manner, supporting both generic and BEM-style class naming.
- Include test-specific CSS classes when enabled.

This interface is designed to be extended and customized to fit the needs of different rendering scenarios while maintaining a consistent API.
