WSucc command (w_succ.rst)
===========================

Overview
--------
w_succ lists segments that successfully reach a target state by reporting, for each iteration, the recycled segments and their final progress-coordinate values. The command scans through iterations, selects segments whose endpoint_type is SEG_ENDPOINT_RECYCLED, and prints per-segment data including iteration, seg_id, weight, and the final pcoord values.

Key Components
--------------
- WSucc class
  - Inherits CommonOutputMixin, WESTDataReaderMixin, and WESTAnalysisTool.
  - Core method: find_successful_trajs
    - Builds a per-iteration list of recycled segments.
    - Retrieves the pcoord dataset for the iteration.
    - Extracts final progress-coordinate values for recycled segments.
    - Outputs a line per recycled segment containing iteration, seg_id, weight, and final pcoord values.
  - Behavior notes:
    - If headers are not suppressed, prints a header describing the columns:
      # successful (recycled) segments
      # column 0:    iteration
      # column 1:    seg_id
      # column 2:    weight
      # column>2:    final progress coordinate value
    - Uses a pcoord_formats mapping to format different pcoord dtypes.
    - Skips iterations with no recycled segments to avoid zero-length selections.
- pcoord_formats
  - A mapping from numpy dtype strings to printf-style format specifiers, e.g.:
    'u8': '%20d', 'f4': '%14.7g', 'f8': '%23.15g', etc.
- entry_point function
  - Creates a WSucc instance and a CLI parser.
  - Adds WESTPA common args via westpa.rc.add_args(parser) and wsucc.add_args(parser).
  - Exposes an output option:
    -o, --output OUTPUT_FILE: store output in OUTPUT_FILE (default: stdout).
  - Processes arguments and runs wsucc.find_successful_trajs().
- Command-line flow
  - The tool relies on WESTPA’s argument processing (westpa.rc) and the WSucc instance to read trajectory data and write output.

Usage
-----
Basic usage:
- w_succ

With explicit output file:
- w_succ -o succs.txt

Notes:
- By default, output is written to standard output. Use -o to redirect to a file.
- The header describing the columns is printed unless headers are suppressed by the configuration.

Details
-------
- Iteration range:
  - The tool iterates for n_iter in 1..(current_iteration - 1).
- Segment selection:
  - For each iteration, it obtains the segment index via get_seg_index(n_iter).
  - Recycled segments are identified as those with endpoint_type == Segment.SEG_ENDPOINT_RECYCLED.
- Final progress coordinates:
  - The final pcoords are taken from the last pcoord frame (index pcoord_len - 1) for each recycled segment.
  - The resulting final_pcoords array is reshaped to (n_recycled, pcoord_ndim) for per-segment output.
- Output format:
  - Each line begins with:
    iteration, seg_id, weight
  - Followed by the final pcoord values formatted according to pcoord_formats.
  - Example header (if not suppressed):
    # successful (recycled) segments
    # column 0:    iteration
    # column 1:    seg_id
    # column 2:    weight
    # column>2:    final progress coordinate value
- Code excerpts (from the implementation):
  - Header output:
    '''
    # successful (recycled) segments
    # column 0:    iteration
    # column 1:    seg_id
    # column 2:    weight
    # column>2:    final progress coordinate value
    '''
  - Per-line output (illustrative):
    self.output_file.write('%8d    %8d    %20.14g' % (n_iter, seg_id, seg_index[seg_id]['weight']))
    fields = ['']
    for field in final_pcoords[ipc]:
        fields.append(pcoord_formats.get(field.dtype.str[1:], '%s') % field)
    self.output_file.write('    '.join(fields))
    self.output_file.write('\n')
- Code structure (highlights):
  - Class: WSucc(CommonOutputMixin, WESTDataReaderMixin, WESTAnalysisTool)
  - Method: find_successful_trajs(self)
  - Entry point: entry_point()
  - CLI integration: adds WESTPA args, processes config, and runs the analysis
