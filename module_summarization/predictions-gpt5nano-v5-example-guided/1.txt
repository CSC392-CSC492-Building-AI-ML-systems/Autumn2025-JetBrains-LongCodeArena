# Using custom attributes

## Overview
Custom attributes are user-defined metadata attached to ftrack entities, allowing teams to extend the information tracked for objects such as tasks, assets, or shots. They are defined on the server side and exposed through the API, enabling reading, writing, filtering, and reporting based on these attributes.

This documentation covers the concept and basic usage within the ftrack API context, including how to establish a session and how authentication is handled when communicating with the server.

## Key Components
- ftrack_api.attribute: Module used to interact with custom attribute definitions and their values.
- Entity objects: Instances representing ftrack entities (e.g., tasks, shots) that may have custom attributes.
- Session: API session used to connect to the ftrack server and perform attribute-related operations.
- SessionAuthentication: Internal mechanism that attaches authentication headers to outgoing HTTP requests.

Code references from the source:
- SessionAuthentication attaches authentication headers to requests:
```
class SessionAuthentication(requests.auth.AuthBase):
    '''Attach ftrack session authentication information to requests.'''

    def __init__(self, api_key, api_user):
        '''Initialise with *api_key* and *api_user*.'''
        self.api_key = api_key
        self.api_user = api_user
        super(SessionAuthentication, self).__init__()

    def __call__(self, request):
        '''Modify *request* to have appropriate headers.'''
        request.headers.update({
            'ftrack-api-key': self.api_key,
            'ftrack-user': self.api_user
        })
        return request
```

- Session initialization (example usage):
```
session = Session(
    server_url='https://your.ftrack.server',
    api_key='your_api_key',
    api_user='your_username'
)
```

- Alternative session initialization using environment variables (fallbacks described in the code):
```
import os
session = Session()  # Uses FTRACK_SERVER, FTRACK_API_KEY, FTRACK_API_USER if provided
```

## Usage

- Prerequisites
  - You must have access to a running ftrack server.
  - Obtain an API key and user name for authentication.

- Defining custom attributes
  - Custom attributes are defined on the server (per entity type). They can be created and configured via the server UI or corresponding server-side API endpoints.

- Accessing custom attributes via the API
  - After attributes are defined, they are accessible through the API on the relevant entity types. The exact API calls depend on the wrapper you are using, but the workflow generally involves authenticating with a Session and retrieving or updating attribute values on target entities.

- Reading and writing values
  - Read values by querying the entity’s custom attribute data.
  - Write values by updating the attribute data on the entity and persisting changes through the API.

- Filtering and reporting
  - Once custom attributes exist, you can use them to filter, group, and report on entities in your workflows, enabling more granular queries and insights.

## Details

- Authentication
  - The SessionAuthentication class demonstrates how authentication data (api_key and api_user) is attached to each HTTP request via headers:
    - ftrack-api-key: your_api_key
    - ftrack-user: your_username

- Session initialization
  - The session can be created explicitly by providing server_url, api_key, and api_user, or implicitly by relying on environment variables FTRACK_SERVER, FTRACK_API_KEY, and FTRACK_API_USER:
    - Explicit:
      session = Session(
          server_url='https://your.ftrack.server',
          api_key='your_api_key',
          api_user='your_username'
      )
    - Implicit (via environment):
      session = Session()  # reads env vars as a fallback

- Notes
  - Custom attributes are part of the server-side schema. Ensure the attributes exist for the target entity type before attempting to read or write them through the API.
  - The exact API methods for manipulating custom attribute values may vary depending on the client wrapper version; consult the latest ftrack API documentation for concrete method names and usage patterns.
