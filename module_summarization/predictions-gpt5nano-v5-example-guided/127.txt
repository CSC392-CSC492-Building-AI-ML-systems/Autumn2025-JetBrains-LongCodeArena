# Convolution module built-in kernels

## Overview
This section documents the built-in kernel support used by the Convolution modeling framework. It describes the 1D and 2D kernel representations (Kernel, Kernel1D, Kernel2D), the boundary handling options, and the utilities that enable efficient, robust convolution (including FFT-based performance optimizations and input preparation). Convolution relies on a C-accelerated routine for performance, accessed via _convolveNd_c, and enforces normalization and size constraints to ensure correct results.

Key components provided by the module include utilities for FFT padding, kernel validation, and input handling when masks or NaN handling are required.

## Key Components
- Kernel, Kernel1D, Kernel2D
  - Built-in kernel models used by the Convolution operation to specify the convolution kernel.
- Convolution integration
  - The convolution operation uses the built-in kernels to perform fast, boundary-aware convolution.
- _convolveNd_c
  - A C-extension implementation used for high-performance multi-dimensional convolution.
- _next_fast_lengths(shape)
  - Determines optimal padding lengths for FFT-based convolution to maximize performance.
  - If SciPy is available, it uses scipy.fft.next_fast_len; otherwise it falls back to a predefined set of good sizes.
  - Example snippet (internal behavior):
    - Tries to import scipy.fft and compute next_fast_len for each dimension.
    - If SciPy is unavailable, computes a suitable length from a precomputed list (_good_sizes) scaled by powers of ten.
- _copy_input_if_needed(input, dtype, order, nan_treatment, mask, fill_value)
  - Prepares the input array for convolution.
  - Handles copied inputs when NaN treatment or masking is involved, ensuring safe operation and preserving input integrity.
  - Important for cases with masked arrays or when nan_treatment="fill" is used.
- MAX_NORMALIZATION
  - The maximum allowed normalization factor for kernels, ensuring numerical stability.
- KernelSizeError, has_even_axis, raise_even_kernel_exception
  - Utilities for validating kernel sizes and axis requirements. Some configurations require odd-sized kernels along certain axes; these helpers enforce that constraint and raise a meaningful error if violated.
- BOUNDARY_OPTIONS
  - Allowed boundary handling modes:
    - None
    - "fill"
    - "wrap"
    - "extend"

## Usage
- Create and use built-in kernels with the Convolution interface.

1) 1D kernel example
- Create a simple 1D kernel:
  - The Kernel1D object can be constructed from a 1D array of weights.
- Apply convolution:
  - Instantiate a Convolution object with the kernel and apply it to your 1D data.

Code example:
- 1D kernel
  - kernel_1d = Kernel1D([0.25, 0.5, 0.25])

- Convolution usage
  - conv = Convolution(kernel_1d)
  - result = conv(data_1d)

2) 2D kernel example
- Create a simple 2D kernel:
  - The Kernel2D object can be constructed from a 2D array of weights.
- Apply convolution:
  - Instantiate a Convolution object with the kernel and apply it to your 2D image data.

Code example:
- 2D kernel
  - kernel_2d = Kernel2D([[1, 2, 1],
                          [2, 4, 2],
                          [1, 2, 1]])

- Convolution usage
  - conv = Convolution(kernel_2d)
  - result = conv(image_2d)

Note:
- Boundary handling can be configured using the boundary option, selecting from the allowed BOUNDARY_OPTIONS. The default behavior will depend on the kernel and convolver configuration.

## Details
- Performance and FFT padding
  - For FFT-based convolution, the module computes optimal (or near-optimal) padding lengths using _next_fast_lengths.
  - If SciPy is installed, scipy.fft.next_fast_len is used; otherwise, a deterministic fallback based on a predefined list of good sizes is used.
- Input preparation
  - When inputs contain NaNs, masks, or explicit fill requirements (nan_treatment="fill"), _copy_input_if_needed creates a safe copy of the input with appropriate dtype, order, and fill behavior to avoid altering the original data.
- Kernel validation
  - The module enforces constraints on kernel sizes via has_even_axis and raise_even_kernel_exception to ensure kernels align with the chosen boundary handling and normalization rules.
- Normalization
  - MAX_NORMALIZATION ensures that the kernel normalization remains within stable limits, preventing numerical issues during convolution.
- Doctest and SciPy dependency
  - doctest can require SciPy for certain doctests (convolve_fft relies on SciPy). See __doctest_requires__ gating for details.
- Boundary handling
  - The boundary behavior during convolution is configurable, with options None, "fill", "wrap", and "extend" available via BOUNDARY_OPTIONS. This controls how data outside the input boundaries is treated during the convolution operation.

Code references (high-level)
- _next_fast_lengths(shape): selects efficient FFT lengths for each dimension.
- _copy_input_if_needed(input, ...): ensures safe, correctly-typed copies when masking or NaN filling is needed.
- BOUNDARY_OPTIONS: [None, "fill", "wrap", "extend"]
- MAX_NORMALIZATION: normalization constraint for kernels
- Kernel, Kernel1D, Kernel2D: core kernel representations used by Convolution
