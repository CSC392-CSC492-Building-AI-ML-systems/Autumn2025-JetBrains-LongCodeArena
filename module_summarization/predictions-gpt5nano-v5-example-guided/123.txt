Implements the preview+confirm+delete workflow
=============================================

Overview
--------
This module provides a DeleteView workflow for Django Cradmin that presents a preview of the object to be deleted, asks for confirmation, and performs the deletion with user feedback. It defines a reusable DeleteViewMixin and a ready-to-use WithinRoleDeleteView that integrates with Cradmin’s role context and templates.

Key Components
--------------
- DeleteViewMixin
  - Adds the preview/confirm/delete lifecycle to a Django DeleteView.
  - Core methods and behavior:
    - get_pagetitle: default page title for the confirmation page.
    - get_action_label / get_delete_button_label: label used for actions.
    - get_cancel_button_label: label for cancel action.
    - get_object_preview: string representation of the object to delete.
    - get_confirm_message: message shown in the focus area, confirming deletion.
    - get_success_url: URL to redirect to after successful deletion (defaults to app index).
    - get_success_message: customizable success message displayed after deletion.
    - add_success_messages: hooks into Django’s messaging framework to show the success message.
    - add_delete_view_mixin_context_data: populates template context with relevant data (object preview, labels, pagetitle, etc.).
    - delete: override that preserves the object preview for messaging, then delegates to the parent and adds success messages.
- WithinRoleDeleteView
  - A concrete view class wired for Cradmin “Within Role” contexts.
  - Inherits from QuerysetForRoleMixin, DeleteViewMixin, DjangoDeleteView, CommonCradminViewMixin, and WithinRoleViewMixin.
  - Uses the template django_cradmin/viewhelpers/formview/within_role_delete_view.django.html.
  - get_context_data: augments the template context with necessary Cradmin and mixin data.

Usage
-----
To use this workflow, create a delete view for your model by subclassing WithinRoleDeleteView (or Compose your own view with DeleteViewMixin).

Example: delete view for MyModel in a Cradmin app

```python
# myapp/views.py
from django.utils.translation import gettext_lazy
from django_cradmin import viewhelpers
from myapp.models import MyModel

class MyModelDeleteView(viewhelpers.formview.WithinRoleDeleteView):
    model = MyModel
    queryset = MyModel.objects.all()

    # Optional: customize the success message
    def get_success_message(self, object_preview):
        return gettext_lazy('Deleted "%(what)s"') % {'what': object_preview}
```

In your urls (example):

```python
from django.urls import path
from .views import MyModelDeleteView

urlpatterns = [
    path('mymodel/<int:pk>/delete/', MyModelDeleteView.as_view(), name='mymodel_delete'),
]
```

Notes
-----
- The view uses the template path specified in WithinRoleDeleteView:
  django_cradmin/viewhelpers/formview/within_role_delete_view.django.html
- The default success URL is the current app’s index view, but you can customize get_success_url.
- The confirm message derives from the object preview:
  Are you sure you want to delete "<object_preview>"?
- The workflow integrates with Django’s messages framework via add_success_messages.

Details
-------
- Preview and confirmation flow:
  - The object to delete is presented via get_object_preview() (defaults to str(obj)).
  - The confirmation prompt uses get_confirm_message(), which includes the object preview.
  - The page button labels come from get_delete_button_label() and get_cancel_button_label().
- Post-deletion behavior:
  - delete() captures the object preview, calls the parent delete, and then invokes add_success_messages() to show feedback to the user.
  - By default, a success message is shown using get_success_message(object_preview).
- Context data:
  - add_delete_view_mixin_context_data() injects keys like object, object_preview, pagetitle, confirm_message, and button labels into the template context.
  - For WithinRoleDeleteView, get_context_data() also wires in Cradmin-specific context via add_javascriptregistry_component_ids_to_context and add_common_view_mixin_data_to_context.

This documentation covers the preview+confirm+delete workflow implemented by the DeleteViewMixin and the WithinRoleDeleteView concrete class, including how to customize messages and integrate into a Cradmin-powered application.
