## Overview
This module provides the command implementations used to manage Alembic migration environments. It includes commands to list available templates, initialize a new migration environment, and create revision files (with optional autogeneration and SQL output). The primary functions are designed to be invoked by higher-level CLI glue or programmatic callers.

Key dependencies include the Alembic Config, ScriptDirectory, and the autogeneration helpers, along with utility helpers for messaging and errors.

Main capabilities:
- List available environment templates
- Initialize a new migration environment directory with optional package layout
- Create new revision files, with optional autogeneration, SQL dump, and revision graph customization

## Usage

- List available templates
  - This is useful to discover templates you can use with init.
  - Example (conceptual):
    - list_templates(config)

- Initialize a new environment
  - Creates the scripts directory, a versions subdirectory, and copies or generates template files.
  - Example:
    - init(config, directory="./migrations", template="generic", package=False)

- Create a new revision
  - Basic usage with a message:
    - revision(config, message="create accounts table")
  - With autogeneration (recommended when you want the script to reflect model changes):
    - revision(config, message="add users table", autogenerate=True)
  - Dump as SQL:
    - revision(config, message="create accounts table", autogenerate=True, sql=True)
  - Advanced customization via process_revision_directives:
    - revision(config, process_revision_directives=my_callable)

Examples from the source reflect typical usage patterns of these commands, including parameter-driven behavior and integration with the ScriptDirectory and autogenerate machinery.
