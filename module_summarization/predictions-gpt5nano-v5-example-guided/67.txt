Text feature extraction
=======================

Overview
--------
The feature_extraction.text submodule provides utilities for converting text documents into numeric feature vectors suitable for machine learning. It includes end-to-end vectorizers (CountVectorizer, TfidfVectorizer, HashingVectorizer), a transformer (TfidfTransformer), and various preprocessing helpers (e.g., strip_accents, strip_tags). The module relies on SciPy sparse matrices for efficient storage and operations on large text corpora.

Key Components
--------------
- HashingVectorizer
  - A stateless vectorizer that converts text to a matrix of token features using a hashing trick. Useful for large-scale text classification where vocabulary construction is impractical.
- CountVectorizer
  - Converts a collection of text documents to a matrix of token counts. Supports customization of tokenizer, preprocessor, and n-gram ranges.
- TfidfTransformer
  - Transforms a count matrix to a TF-IDF representation, applying term frequency and inverse document frequency weighting.
- TfidfVectorizer
  - End-to-end vectorizer that combines CountVectorizer and TfidfTransformer for convenient text-to-TF-IDF conversion.
- ENGLISH_STOP_WORDS
  - A predefined set of common English stop words, suitable for filtering out frequent but non-informative terms.
- strip_accents_unicode
  - Converts accented Unicode characters to their closest ASCII counterpart or removes them, preserving readability where possible.
- strip_accents_ascii
  - Strips accents by transliterating Unicode characters to ASCII where possible.
- strip_tags
  - Basic HTML/XML tag stripper using a regular expression.
- _preprocess
  - Internal helper to chain text preprocessing steps (lowercasing, accent handling).
  - Example (signature):
    def _preprocess(doc, accent_function=None, lower=False)
- _analyze
  - Internal helper to chain processing steps from a document to tokens/ngrams, with options for a custom analyzer, tokenizer, preprocessor, and stop words.
  - Example (signature):
    def _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
- _VectorizerMixin
  - Provides common tokenization-related functionality shared by vectorizers (e.g., decode of input data, word n-grams generation helper).
  - Example (decode method signature):
    def decode(self, doc)
- _word_ngrams
  - Internal helper to generate word n-grams from token sequences, with optional stop words filtering.

Usage
-----
 Typical usage with end-to-end vectorizers:

.. code-block:: python

  from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer

  docs = [
      "The quick brown fox jumps over the lazy dog.",
      "Never jump over the lazy dog quickly."
  ]

  # Simple bag-of-words representation with English stop words removed
  cv = CountVectorizer(stop_words="english")
  X_counts = cv.fit_transform(docs)
  print(X_counts.shape)

  # TF-IDF representation
  tfidf = TfidfVectorizer()
  X_tfidf = tfidf.fit_transform(docs)
  print(X_tfidf.shape)

  # End-to-end TF-IDF using a single vectorizer
  tfidf_vec = TfidfVectorizer(stop_words="english", ngram_range=(1, 2))
  X_tfidf_vec = tfidf_vec.fit_transform(docs)

Using the hashing-based vectorizer (stateless, suitable for large-scale scenarios):

.. code-block:: python

  from sklearn.feature_extraction.text import HashingVectorizer

  docs = [
      "The quick brown fox jumps over the lazy dog.",
      "Never jump over the lazy dog quickly."
  ]

  hasher = HashingVectorizer(n_features=2**20, alternate_sign=False)
  X_hash = hasher.transform(docs)  # note: hashing vectorizer is stateless

Preprocessing helpers
---------------------
- strip_accents_ascii
  - Example:
.. code-block:: python

  from sklearn.feature_extraction.text import strip_accents_ascii
  text = "Café naïve"
  print(strip_accents_ascii(text))  # "Cafe naive"

- strip_accents_unicode
  - Example:
.. code-block:: python

  from sklearn.feature_extraction.text import strip_accents_unicode
  text = "Café naïve"
  print(strip_accents_unicode(text))

- strip_tags
  - Example:
.. code-block:: python

  from sklearn.feature_extraction.text import strip_tags
  html = "<p>Hello <b>world</b></p>"
  print(strip_tags(html))  # "Hello  world "

Stop words
----------
- ENGLISH_STOP_WORDS
  - A built-in set of English stop words. Usage example:
.. code-block:: python

  from sklearn.feature_extraction.text import ENGLISH_STOP_WORDS
  print(len(ENGLISH_STOP_WORDS))

Notes
-----
- The module uses SciPy sparse matrices for efficient storage and operations on large corpora.
- Input sources can be text content or, depending on vectorizer configuration, filenames or file-like objects. The internal decode method handles reading and decoding inputs as needed.

Details
-------
- The module's API is exposed via __all__, including:
  - "HashingVectorizer", "CountVectorizer", "ENGLISH_STOP_WORDS",
    "TfidfTransformer", "TfidfVectorizer", "strip_accents_ascii",
    "strip_accents_unicode", "strip_tags"
- Internal helpers (_preprocess, _analyze, _VectorizerMixin) manage the chaining of preprocessing, tokenization, and analysis steps, enabling flexible composition of custom text processing pipelines.
- Example usage demonstrates common end-to-end workflows as well as specialized preprocessing steps.
