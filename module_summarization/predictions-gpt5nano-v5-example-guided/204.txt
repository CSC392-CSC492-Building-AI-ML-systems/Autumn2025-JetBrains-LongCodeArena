# Random Projection: A simple and computationally efficient way to reduce the dimensionality of the data

## Overview
Random projection is a fast and scalable technique to reduce data dimensionality by projecting data onto a lower-dimensional space using a random matrix. This approach trades a controlled amount of accuracy (in the form of additional variance) for faster processing and smaller model sizes. The projection preserves pairwise distances between samples with high probability, in line with the Johnson-Lindenstrauss lemma.

The projection can be implemented using a dense Gaussian random matrix or a sparse Achlioptas-style matrix. The target dimensionality is chosen to ensure an eps-embedding with high probability, as described below.

Key theoretical reference:
- Johnson-Lindenstrauss lemma: distances between points are preserved up to a (1 ± eps) factor after projection onto a lower-dimensional space. See: https://en.wikipedia.org/wiki/Johnson%E2%80%93Lindenstrauss_lemma

## Key Components
- GaussianRandomProjection: Transformer that uses a dense Gaussian random matrix (entries ~ N(0, 1/n_components)).
- SparseRandomProjection: Transformer that uses a sparse random matrix (sparsity controlled by density, often following Li et al. guidance).
- johnson_lindenstrauss_min_dim(n_samples, eps=0.1): Utility to compute the minimal number of components required to achieve an eps-embedding with high probability.
  - Example-derived formula: n_components >= 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)
- _gaussian_random_matrix(n_components, n_features, random_state=None): Internal helper to generate the dense Gaussian projection matrix.
- Density handling helpers (_check_density) and input validation helpers (_check_input_size).

Note: The module also ties into common scikit-learn estimator interfaces (fit/transform) for easy integration into pipelines.

## Usage
- Compute the minimal number of components for an eps-embedding:
  ```
  >>> from sklearn.random_projection import johnson_lindenstrauss_min_dim
  >>> johnson_lindenstrauss_min_dim(1e6, eps=0.5)
  663

  >>> johnson_lindenstrauss_min_dim(1e6, eps=[0.5, 0.1, 0.01])
  array([    663,   11841, 1112658])

  >>> johnson_lindenstrauss_min_dim([1e4, 1e5, 1e6], eps=0.1)
  array([ 7894,  9868, 11841])
  ```
- Basic usage with transformers:
  - Gaussian projection
    ```
    from sklearn.random_projection import GaussianRandomProjection
    grp = GaussianRandomProjection(n_components=100, random_state=42)
    X_new = grp.fit_transform(X)
    ```
  - Sparse projection
    ```
    from sklearn.random_projection import SparseRandomProjection
    spr = SparseRandomProjection(n_components=100, density='auto', random_state=42)
    X_new = spr.fit_transform(X)
    ```

- Density parameter notes:
  - density can be "auto" (default for some implementations), or a value in (0, 1].
  - "auto" maps to 1 / sqrt(n_features) for likelihood-based sparse matrices.

## Details
- Purpose and guarantees:
  - Random projections aim to preserve pairwise distances with high probability while reducing dimensionality.
  - The relationship is governed by the Johnson-Lindenstrauss lemma. The projection can be implemented via a random Gaussian matrix or a sparse Achlioptas matrix.

- Dimensionality bound:
  - The minimal number of components required for an eps-embedding is:
    n_components >= 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)
  - The dimensionality bound depends on the number of samples (n_samples) and the distortion parameter (eps), not on the original number of features.

- Practical aspects:
  - The random matrix is typically scaled so that projected coordinates maintain comparable variance.
  - The approach is suitable for fast dimensionality reduction prior to downstream processing, clustering, or linear modeling.

- Reference implementation notes:
  - The module exposes: SparseRandomProjection, GaussianRandomProjection, and johnson_lindenstrauss_min_dim.
  - Internal helpers include _gaussian_random_matrix for matrix generation and density checks via _check_density and _check_input_size.
  - See the module source for precise parameter validation and internal behavior.

- See also:
  - Johnson-Lindenstrauss lemma: https://en.wikipedia.org/wiki/Johnson%E2%80%93Lindenstrauss_lemma
