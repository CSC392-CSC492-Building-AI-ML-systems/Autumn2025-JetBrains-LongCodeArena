======================================

Overview
--------
This module defines the core migration operation objects used by the autogeneration and migration framework. It provides a base MigrateOperation class, an extensible AddConstraintOp for adding constraints, and DropConstraintOp for removing constraints. The system is designed to be extensible via a dispatcher that selects concrete constraint-operation implementations based on the type of constraint encountered.

Key Components
-------------
- MigrateOperation
  - Base class for migration commands and their organization.
  - Attributes/methods:
    - info: memoized dictionary for arbitrary per-operation data.
    - _mutations: empty immutable set of mutation types.
    - reverse(): abstract method to obtain the reverse operation.
    - to_diff_tuple(): abstract method to render the operation as a diff tuple.

- AddConstraintOp
  - Base class for operations that add constraints.
  - add_constraint_ops: a dispatcher mapping constraint types to concrete implementations.
  - constraint_type (abstract): property that concrete subclasses must implement to indicate the constraint type.
  - register_add_constraint(type_: str) -> Callable: decorator to register a concrete AddConstraintOp subclass for a given constraint type.
  - from_constraint(constraint: Constraint) -> AddConstraintOp: factory that dispatches based on constraint.__visit_name__.
  - to_constraint(self, migration_context=None) -> Constraint: abstract method to render the constraint back to a SQLAlchemy Constraint.
  - reverse(): returns the corresponding DropConstraintOp for the constraint produced by to_constraint().
  - to_diff_tuple(): returns a tuple like ("add_constraint", constraint).

- DropConstraintOp
  - Registered operation for dropping constraints.
  - Decorated with:
    - @Operations.register_operation("drop_constraint")
    - @BatchOperations.register_operation("drop_constraint", "batch_drop_constraint")
  - __init__(constraint_name, table_name, type_=None, schema=None, _reverse=None)
    - constraint_name: optional name of the constraint.
    - table_name: name of the table containing the constraint.
    - type_: optional constraint type (e.g., "foreignkey", "primary", "unique", "check").
    - schema: optional schema name.
    - _reverse: optional AddConstraintOp used to reconstruct the original constraint.
  - reverse(): returns the AddConstraintOp corresponding to this constraint.
  - to_diff_tuple(): returns a diff tuple; uses "remove_fk" for foreign keys, otherwise "remove_constraint".
  - from_constraint(constraint: Constraint) -> DropConstraintOp: converts a SQLAlchemy Constraint into a DropConstraintOp, mapping visit names to types and populating table/schema information.
  - to_constraint(self) -> Constraint: reconstructs the original Constraint from _reverse, or raises if not available.
  - drop_constraint(operations, constraint_name, table_name, type_=None, *, schema=None) -> None: classmethod to drop a constraint.
  - batch_drop_constraint(operations: BatchOperations, constraint_name, type_=None) -> None: classmethod for batch-dropped constraints (omits table_name and schema).

Usage
-----
- Create a drop constraint operation:
.. code-block:: python

    DropConstraintOp.drop_constraint(
        operations,
        constraint_name="fk_user_role",
        table_name="user",
        type_="foreignkey",
        # schema="public"  # if needed
    )

- Create a batch drop constraint operation:
.. code-block:: python

    DropConstraintOp.batch_drop_constraint(
        batch_operations,
        constraint_name="fk_user_role",
        type_="foreignkey",
    )

- Reconstruct an AddConstraintOp from a SQLAlchemy Constraint, and obtain its diff:
.. code-block:: python

    add_op = AddConstraintOp.from_constraint(constraint)
    diff = add_op.to_diff_tuple()  # e.g., ("add_constraint", constraint)

- Reconstruct a DropConstraintOp’s constraint from its reverse, then obtain diff:
.. code-block:: python

    drop_op = DropConstraintOp.from_constraint(constraint)
    reverse_constraint = drop_op.to_constraint()  # yields the original Constraint
    diff = drop_op.to_diff_tuple()  # e.g., ("remove_fk", constraint)

Details
-------
- Extensibility: AddConstraintOp uses a Dispatcher (add_constraint_ops) to map constraint visit names to concrete AddConstraintOp subclasses. Subclasses implement constraint_type and to_constraint, enabling support for various constraint kinds.
- Reversibility: Each operation can provide its reverse operation, enabling bidirectional migrations (apply and rollback).
- Diff representation: to_diff_tuple returns a machine-readable representation of the operation, suitable for diffing and applying changes.
- Batch operations: DropConstraintOp supports batch form via BatchOperations, allowing batch-based migration commands without repeating table/schema details.
- Type hints: The module uses TYPE_CHECKING to import numerous SQLAlchemy types for static type checking without incurring runtime import costs.

Notes
-----
- The concrete behavior for specific constraint types is expected to be provided by subclasses registered via AddConstraintOp.register_add_constraint. The from_constraint path dispatches based on constraint.__visit_name__.
- The to_diff_tuple output adapts to constraint_type (e.g., "remove_fk" for foreign keys, "remove_constraint" for others).
