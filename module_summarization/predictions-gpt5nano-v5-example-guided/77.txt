# Generate documentation and example for this module

## Overview
This module provides essential utilities used by image transformations in the library. It defines:
- InterpolationMode: an enum of supported interpolation methods for image resizing.
- Helpers for mapping between interpolation modes and PIL/Tensor representations.
- Image format utilities: get_dimensions, get_image_size, get_image_num_channels.
- Conversions between PIL images, NumPy arrays, and PyTorch tensors: to_tensor and pil_to_tensor.
- Internal helpers for PIL and NumPy image checks (_is_pil_image, _is_numpy, _is_numpy_image).

These components support consistent handling of image data across different backends and workflows (PIL, NumPy, and PyTorch tensors).

## Key Components
- InterpolationMode (Enum)
  - Available: NEAREST, NEAREST_EXACT, BILINEAR, BICUBIC, BOX, HAMMING, LANCZOS
  - Used to specify interpolation methods for resizing and related operations.
- _interpolation_modes_from_int(i: int) -> InterpolationMode
  - Maps integer codes to InterpolationMode values.
- pil_modes_mapping
  - Maps InterpolationMode values to corresponding PIL mode integers.
- _is_pil_image
  - Internal helper to detect PIL Images.
- _is_numpy, _is_numpy_image
  - Internal helpers to detect NumPy arrays and NumPy image shapes.
- get_dimensions(img: Tensor) -> List[int]
  - Returns image dimensions as [channels, height, width].
- get_image_size(img: Tensor) -> List[int]
  - Returns image size as [width, height].
- get_image_num_channels(img: Tensor) -> int
  - Returns the number of image channels.
- to_tensor(pic) -> Tensor
  - Convert a PIL Image or NumPy array to a PyTorch tensor. Handles various dtypes and image modes, and scales to [0, 1] for byte inputs.
- pil_to_tensor(pic: Any) -> Tensor
  - Convert a PIL Image to a tensor of the same underlying type (deep copy).

## Usage
Examples illustrating common usage of these components.

- Convert a PIL image to a tensor
```python
from PIL import Image
from torchvision.transforms import to_tensor

img = Image.open("path/to/image.jpg").convert("RGB")
t = to_tensor(img)
print(t.shape)  # e.g., [3, H, W]
```

- Convert a NumPy array to a tensor
```python
import numpy as np
from torchvision.transforms import to_tensor

np_img = np.random.randint(0, 255, (256, 256, 3), dtype=np.uint8)
t = to_tensor(np_img)
print(t.shape)  # [3, 256, 256]
```

- Get image dimensions and size
```python
from torchvision.transforms import get_dimensions, get_image_size

# Assuming t is a tensor obtained from an image
dims = get_dimensions(t)      # e.g., [3, H, W]
size = get_image_size(t)        # e.g., [W, H]
```

- Map an interpolation mode to a PIL mode integer
```python
from torchvision.transforms import InterpolationMode, pil_modes_mapping

mode = InterpolationMode.BILINEAR
pil_mode = pil_modes_mapping[mode]  # e.g., 2
```

- Check PIL/NumPy image helpers (internal usage)
```python
from torchvision.transforms import _is_pil_image, _is_numpy, _is_numpy_image

print(_is_pil_image(img))        # True if PIL Image
print(_is_numpy(np_img))         # True if NumPy array
```

- Convert a PIL image to a tensor with explicit PIL handling
```python
from torchvision.transforms import pil_to_tensor
from PIL import Image

img = Image.open("path/to/image.png")
t = pil_to_tensor(img)
print(t.shape)
```

## Details
- to_tensor(pic)
  - Accepts a PIL Image or a NumPy array. If the input is a 2D array, it is expanded to include a channel dimension.
  - For NumPy arrays:
    - If 2D, expands to 3D and reorders to CHW format.
    - If the array type is uint8, scales to [0, 1] using the default floating dtype.
  - For PIL images:
    - Supports modes such as "1" (binary) and converts to a suitable NumPy dtype based on mode.
    - Converts to a PyTorch tensor in CHW format and scales to [0, 1] for 8-bit data.
  - If accimage is available, it can be used as an alternative source and converted accordingly.
- pil_to_tensor(pic)
  - Converts a PIL Image to a tensor preserving the underlying data type (deep copy).
- get_dimensions, get_image_size, get_image_num_channels
  - Provide convenient access to common image metadata for downstream transforms.
- InterpolationMode and pil_modes_mapping
  - Facilitate consistent selection of interpolation methods across PIL-based resizing operations.

Notes:
- API usage is logged via _log_api_usage_once for non-scripting contexts.
- Some helpers (_is_pil_image, _is_numpy, _is_numpy_image) are intended for internal use but may be useful for advanced users writing custom pipelines.
