## Overview
This module provides miscellaneous models that use Student-t distributed errors for robust likelihood-based estimation. It includes:
- TLinearModel: Maximum Likelihood Estimation of a linear model with t-distributed errors. It extends GenericLikelihoodModel and uses a Student-t likelihood with an optional degrees-of-freedom parameter that can be fixed or estimated.
- TArma: Univariate ARMA model with t-distributed errors. It extends Arma and adopts a t-distribution for the innovations, with df and scale as parameters.

The t-distribution provides heavier tails than the normal distribution, offering robustness to outliers and heavy-tailed data. Note that in these implementations, the scale parameter does not directly equal the standard deviation of the error term.

References and context are included in the source, with design notes and TODOs indicating typical usage as a generic MLE example.

Key ideas:
- Robust likelihood-based estimation under Student-t errors
- Flexible parameterization with optional df estimation (or fixed df)
- Numerical derivatives for gradients and Hessians (except where defined)

## Key Components

- TLinearModel
  - Purpose: Maximum Likelihood Estimation of a linear model with t-distributed errors.
  - Inheritance: GenericLikelihoodModel.
  - Parameters:
    - beta: regression coefficients
    - df: degrees of freedom (can be fixed or estimated)
    - scale: scale parameter (positive)
  - Features:
    - Option to fix df via a fix_df flag (default is to estimate df and scale together with beta)
    - Start values derived from an OLS fit (with optional kurtosis-based df initialization)
    - loglike and nloglikeobs implemented to use the t-distribution likelihood
    - predict method for fitted values
  - Notes:
    - When df is fixed, the parameter vector excludes df and includes only scale as an extra parameter.
    - The variance of the error is not equal to scale; for df > 2, var(error) = df/(df-2) * scale^2.

- TArma
  - Purpose: Univariate ARMA model with t-distributed errors.
  - Inheritance: Arma.
  - Parameters:
    - df: degrees of freedom
    - scale: scale parameter (positive)
  - Features:
    - loglike delegates to nloglikeobs, which encodes the t-distribution likelihood for residuals
    - nloglikeobs computes the per-observation log-likelihood contributions using the standardized residuals
  - Notes:
    - Uses the standard t-distribution; the implied error variance is variance = df/(df-2) * scale^2 (for df > 2).
    - This may be refined by adopting a standardized t-distribution with a scale aligned to variance.

Code excerpts (illustrative)

- TLinearModel initialization and parameter handling
  - Supports both fixed df and estimated df:
    - If fix_df is False: parameters include df and scale
    - If fix_df is True: df is fixed, only scale (plus betas) are estimated
  - Start values are derived from an ordinary least squares fit

- nloglikeobs (TLinearModel)
  - Formulates the log-likelihood under Student-t errors:
    - Uses residuals x = (y - Xβ) / scale
    - log-density components built with gammaln terms and the t-density formula
    - Returns negative log-likelihood contributions (as is customary for minimization)

- nloglikeobs (TArma)
  - Computes per-observation log-likelihood for ARMA with t-distributed errors
  - Uses errors returned by geterrors(params[:-2])
  - Includes df and scale in the likelihood via the t-distribution

- Fitting a T-linear model
  - Note: adjust imports to your environment; this shows typical usage pattern.

## Details

- Mathematical and statistical notes
  - TLinearModel.nloglikeobs(params) and TArma.nloglikeobs(params) implement the negative log-likelihood under Student-t errors.
  - The t-distribution parameters:
    - df: degrees of freedom (either estimated or fixed)
    - scale: scale parameter (absolute value is used to ensure positivity)
  - For TArma, the implied error variance when df > 2 is scale^2 * df/(df-2).
  - The likelihood is computed per observation and then aggregated (sum) to obtain the total log-likelihood.
  - The implementation uses numerical differentiation for gradients and Hessians (except where overridden), as noted in the class description.
- Parameter handling
  - The model may fix df (fix_df) or estimate it along with beta and scale.
  - Start values are derived from a standard OLS fit; the df initialization can optionally use kurtosis-derived values.

  - This module serves as an educational/generic MLE example within StatsModels, illustrating how to wrap a custom likelihood into the GenericLikelihoodModel framework.
  - The implementation prioritizes clarity and extensibility for exploring robust linear and ARMA models with t-distributed errors.
