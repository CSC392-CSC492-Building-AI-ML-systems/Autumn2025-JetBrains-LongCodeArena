# FleurXMLModifier

## Overview
FleurXMLModifier is a helper class for organizing and grouping changes to a Fleur input file (inp.xml) in a robust, low-level manner. It collects modification tasks (via the XML setter interfaces provided by masci_tools) and applies them in a single, consolidated pass. This mirrors a low-level approach to modifications separate from higher-level modifiers such as FleurinpModifier.

Key ideas:
- Modifications are represented as tasks (ModifierTask) and stored internally until applied.
- Setter methods come from three groups:
  - XPath-based setters (XPATH_SETTERS)
  - Schema-dict setters (SCHEMA_DICT_SETTERS)
  - NMMPMAT-related setters (NMMPMAT_SETTERS)
- The class can be extended with extra setter methods via _extra_functions.
- Tasks can be created individually or from a list (fromList), and later applied with modify_xmlfile.

Example usage (high level):
- Create a modifier, add tasks, then apply to inp.xml:
  - fmode = FleurXMLModifier()
  - fmode.set_inpchanges({'Kmax': 4.0})
  - fmode.shift_value({'Gmax': 5.0})
  - fmode.set_species('all-Fe', {'lo': [{'n':3, 'l': 's', 'type': 'SCLO'},
                                      {'n':3, 'l': 'p', 'type': 'SCLO'}]})
  - new_xmltree, additional_files = fmode.modify_xmlfile('/path/to/input/file/inp.xml')

## Key Components
- ModifierTask: A NamedTuple that stores a single modification task with fields:
  - name: str
  - args: tuple[Any, ...] = ()
  - kwargs: dict[str, Any] = {}

- FleurXMLModifier class attributes:
  - _xpath_functions: mapping of setter names to callables (XPATH_SETTERS)
  - _schema_dict_functions: mapping of setter names to callables (SCHEMA_DICT_SETTERS)
  - _nmmpmat_functions: mapping of setter names to callables (NMMPMAT_SETTERS)
  - _extra_functions: optional extra functions for each category (default empty)

- Dynamic composition (via __new__):
  - On first instantiation, the class builds:
    - xpath_functions = _xpath_functions merged with any extra xpath setters
    - schema_dict_functions = _schema_dict_functions merged with any extra schema_dict setters
    - nmmpmat_functions = _nmmpmat_functions merged with any extra nmmpmat setters
  - This enables extending the set of available actions without altering core dictionaries.

- Core methods:
  - __init__(validate_signatures: bool = True)
    - Initializes an empty task list and stores the validation flag.
  - fromList(cls, task_list, *args, **kwargs)
    - Convenience constructor: creates an instance, adds tasks from task_list, and returns it.
    - task_list should be a list of tuples: (method_name, {'kwarg_name': value, ...})
  - add_task_list(self, task_list)
    - Adds a list of tasks by calling the corresponding available action methods.
    - Resolves methods via get_avail_actions() and raises a ValueError if an unknown method is requested.
  - _validate_signature(name, *args, **kwargs)
    - Deprecated helper that emits a deprecation warning and delegates to _validate_arguments.
  - _get_setter_function_and_prefix(self, name)
    - Determines the appropriate setter function and a prefix path that indicates where to apply substitutions for the arguments.
    - If the name is found in xpath_functions, returns (func, ('xmltree',)).
    - If in schema_dict_functions, returns (func, ('xmltree', 'schema_dict')).
    - If in nmmpmat_functions, returns (func, ('xmltree', 'nmmplines', 'schema_dict')).
    - Raises a ValueError if the setter name is not found.

Note: get_avail_actions and _validate_arguments are part of the broader toolchain and are used by FleurXMLModifier to resolve and validate available actions. They are not fully shown in the provided excerpt but are relied upon for task management and validation.

## Usage

- Basic instantiation and task addition:
  - fmode = FleurXMLModifier()
  - fmode.set_inpchanges({'Kmax': 4.0})  # Set Kmax
  - fmode.shift_value({'Gmax': 5.0})    # Add to Gmax
  - fmode.set_species('all-Fe', {'lo': [{'n': 3, 'l': 's', 'type': 'SCLO'},
                                      {'n': 3, 'l': 'p', 'type': 'SCLO'}]})

- From a predefined list of tasks:
  - task_list = [
      ('set_inpchanges', {'Kmax': 4.0}),
      ('shift_value', {'Gmax': 5.0}),
    ]
  - fmode = FleurXMLModifier.fromList(task_list)

- Applying changes to an input file:
  - new_xmltree, additional_files = fmode.modify_xmlfile('/path/to/input/file/inp.xml')

- Example from the class docstring showing typical workflow:

## Details

- Purpose and design
  - FleurXMLModifier provides a structured, low-level interface for batching changes to Fleur’s inp.xml. By collecting changes as tasks, it enables robust modification flows, reduces repeated file parsing, and aligns with the XML-based setter infrastructure used elsewhere in Masci-tools.

- How actions are organized
  - The available actions are categorized into three dictionaries:
    - XPath-based actions (xpath_functions)
    - Schema-dict-based actions (schema_dict_functions)
    - NMMPMAT-related actions (nmmpmat_functions)
  - These dictionaries are sourced from XPATH_SETTERS, SCHEMA_DICT_SETTERS, and NMMPMAT_SETTERS, and can be augmented via _extra_functions for each category.

- Dynamic extension
  - The __new__ method lazily assembles the effective action sets on first instantiation:
    - xpath_functions = _xpath_functions + any extra xpath actions
    - schema_dict_functions = _schema_dict_functions + any extra schema actions
    - nmmpmat_functions = _nmmpmat_functions + any extra nmmpmat actions

- Task representation
  - Each modification is stored as a ModifierTask with a name and its associated positional and keyword arguments. This decouples task specification from execution, enabling pre-collection, reordering, and batch application.

- Applying changes
  - The modify_xmlfile method (not shown in the excerpt) applies the collected tasks to the specified input file, returning the updated XML tree and any additional files generated as part of the change process.

- Deprecation note
  - _validate_signature is marked as deprecated; prefer using _validate_arguments with explicit name, *args, and **kwargs.

- Compatibility
  - The class is designed to be compatible with the broader Masci-tools XML modification utilities, leveraging the same setter interfaces used across the project.

For further details, refer to the surrounding modules (masci_tools.util.xml.collect_xml_setters, masci_tools.util.xml.xml_setters_names, masci_tools.util.xml.common_functions, and related schema utilities) that provide the concrete setter implementations and XML handling primitives used by FleurXMLModifier.
