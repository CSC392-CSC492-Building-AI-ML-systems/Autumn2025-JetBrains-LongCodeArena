## Overview
The sklean.naive_bayes module implements Naive Bayes classifiers, which are supervised learning methods based on Bayes' theorem with strong (naive) feature independence assumptions. The module provides an abstract base class for Naive Bayes estimators and concrete implementations such as GaussianNB, along with other Naive Bayes variants (BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB) exposed via the module's public API.

The Gaussian Naive Bayes implementation supports online updates through partial_fit and is described here as a primary example of the approach.

## Key Components

- _BaseNB (abstract)
  - Purpose: Abstract base class for Naive Bayes estimators.
  - Key abstract methods:
    - _joint_log_likelihood(X): Compute the unnormalized posterior log probability of X, i.e., log P(c) + log P(x|c) for all samples and classes.
    - _check_X(X): Perform dataset-specific input validation/checks (used by predict* methods).
  - Public methods (built on top of the abstract methods):
    - predict_joint_log_proba(X): Returns joint log probabilities log P(x, y) for each sample and class.
    - predict(X): Classifies samples and returns predicted labels.
    - predict_log_proba(X): Returns log-probability estimates P(y|x) in log-space.
    - predict_proba(X): Returns probability estimates P(y|x) by exponentiating the log-probabilities.

- GaussianNB
  - Description: Gaussian Naive Bayes implementation. Can be updated online via partial_fit.
  - Parameters
    - priors: Prior class probabilities (optional). If provided, priors are not adjusted from the data.
    - var_smoothing: Portion of the largest variance of all features added to variances for numerical stability (default 1e-9).
  - Attributes (set after fit):
    - class_count_: Number of training samples observed in each class.
    - class_prior_: Prior probability of each class.
    - classes_: Known class labels.
    - epsilon_: Absolute additive value to variances (for numerical stability).
    - n_features_in_: Number of features seen during fit.
    - feature_names_in_: Feature names seen during fit (if provided as strings).
    - var_: Per-class feature variances.
    - theta_: Per-class feature means.
  - Notes: Inherits predict*, predict_proba*, predict_log_proba*, and predict_joint_log_proba from _BaseNB with class-specific likelihood computations.

- Other Naive Bayes variants (exposed in the module's API)
  - BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB
  - Each variant provides a specific distributional assumption for P(x|y) and corresponding training logic.

See also: BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB.

## Usage

- Basic GaussianNB example
  - Fit and predict:
    - from sklearn.naive_bayes import GaussianNB
    - gnb = GaussianNB()
    - gnb.fit(X_train, y_train)
    - y_pred = gnb.predict(X_test)
  - Probabilities and log-probabilities:
    - proba = gnb.predict_proba(X_test)
    - log_proba = gnb.predict_log_proba(X_test)
  - Joint log probabilities:
    - jll = gnb.predict_joint_log_proba(X_test)

- Online updates (partial_fit)
  - GaussianNB supports incremental learning via partial_fit to update means, variances, and class statistics without re-fitting from scratch.

## Details

- Joint log likelihood
  - _joint_log_likelihood(X) computes log P(y) + log P(x|y) for each sample and class.

- Prediction and probabilities
  - predict_proba uses predict_log_proba and exponentiation to return P(y|x).
  - predict_log_proba computes log P(y|x) by normalizing jll with log P(x) = log sum exp_jll across classes.
  - The normalization constant P(x) is obtained via logsumexp over the joint log-likelihoods.

- Input and fitting considerations
  - _check_X(X) provides dataset-specific validation for predictions.
  - check_is_fitted is used to ensure the estimator has been fitted before making predictions.
  - The GaussianNB implementation includes epsilon_ for numerical stability and var_smoothing to stabilize variance estimates.

- Feature names and inputs
  - n_features_in_ records the number of features observed during fit.
  - feature_names_in_ records feature names if provided as strings at fit time.

- Variants and extensibility
  - The module provides multiple Naive Bayes classifiers, each implementing its own _joint_log_likelihood and _check_X logic to handle different data distributions (e.g., Gaussian, Bernoulli, Multinomial).
  - See Also: BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB for alternative Naive Bayes classifiers.
