Nextcord Introduction
=====================


Overview
--------

This introduction covers the widget-related components provided by Nextcord. It focuses on representing guild widgets in a lightweight, partial form suitable for inspection and display. The key classes are:

- WidgetChannel: a lightweight representation of a widget channel with id, name, and position.
- WidgetMember: a partial member in a widget, including identity, status, nick, activity, and connection information.
- Widget: represents a Guild widget, including its id, name, accessible channels, and online members (offline members do not appear in the widget).

The implementation emphasizes efficiency with __slots__ and provides helpful string and representation methods for debugging and display.


Key Components
--------------

WidgetChannel
- Represents a "partial" widget channel.
- Core attributes: id (int), name (str), position (int).
- Key behaviors:
  - __str__ returns the channel's name.
  - __repr__ returns a descriptive string with id, name, and position.
  - mention property returns a string that can be used to mention the channel: <#<id>>.
  - created_at property returns the channel's creation time in UTC (via snowflake_time).

Example:
.. code-block:: python

    w = WidgetChannel(id=123, name="General", position=1)
    print(w)            # General
    print(w.mention)    # <#123>
    print(w.created_at)   # datetime.datetime(...)  (creation time in UTC)


WidgetMember
- Represents a "partial" member of the widget's guild.
- Core attributes (as documented):
  - id, name, discriminator
  - bot
  - status (Status)
  - nick (optional)
  - avatar (optional)
  - activity (optional, BaseActivity or Spotify)
  - deafened, muted, suppress (optional booleans)
  - connected_channel (optional WidgetChannel)
- __slots__ define the set of instance attributes for memory efficiency.
- display_name property returns nick if available, otherwise the username.
- __init__ initializes fields from a data payload and associates an activity if present.
- __repr__ provides a concise representation including name, discriminator, bot, and nick.

Usage note:
WidgetMember requires a runtime state and a payload dictionary as produced by Nextcord’s internal state handling.

Example:
.. code-block:: python

    # Given a payload dict and a state, you might instantiate (simplified)
    data = {
        'id': 42,
        'name': 'Alice',
        'discriminator': '0001',
        'nick': 'Ally',
        'status': 'online',
        'game': {'name': 'Chess'}
        # other keys as provided by the widget payload
    }
    member = WidgetMember(state=state, data=data)
    print(member.display_name)  # Ally
    if member.activity:
        print(member.activity.name)  # Chess


Widget
- Represents a Guild widget.
- Core attributes (as described in the code comments):
  - id (guild ID)
  - name (guild name)
  - channels: list[WidgetChannel] — the accessible voice channels in the guild
  - members: list[WidgetMember] — the online members in the server (offline members are omitted)

Notes
- The widget data is subject to Discord limitations. Offline members may not appear in the widget data.
- The implementation and documentation emphasize lightweight representations suitable for quick UIs and display.

Example:
.. code-block:: python

    # In typical usage, you would obtain a Widget object from Nextcord’s state/guild widget data
    widget = Widget(id=987, name="My Guild", channels=[WidgetChannel(1, "General", 0)], members=[...])
    print(widget.name)  # My Guild
    print(len(widget.channels))  # number of accessible voice channels
    print(len(widget.members))   # number of online members in the widget


Details
-------

- Equality and hashing: WidgetChannel, WidgetMember, and Widget support standard Python comparisons and hashing as part of their container behaviors.
- String representations:
  - WidgetChannel.__str__ returns the channel name.
  - WidgetChannel.__repr__ provides a detailed representation, e.g., <WidgetChannel id=... name='General' position=...>.
  - WidgetMember.__repr__ summarizes key identity fields (name, discriminator, bot, nick).
- Display helpers:
  - WidgetMember.display_name returns the display name, preferring nick over the username.
- Time information:
  - WidgetChannel.created_at uses snowflake_time to compute creation time from the channel ID.
