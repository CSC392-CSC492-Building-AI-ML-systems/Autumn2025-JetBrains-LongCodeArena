Overview
--------
This module provides Bluesky plan builders for reading detectors and performing scans. It defines:

- count(detectors, num=1, delay=None, *, per_shot=None, md=None): Take one or more readings from detectors, with optional per-shot customization and metadata.
- list_scan(detectors, *args, per_step=None, md=None): Scan over one or more motor variables in a multi-dimensional inner product, constructing a detailed metadata dictionary and delegating to a generic scanning plan.
- rel_list_scan(detectors, *args, per_step=None, md=None): Placeholder for relative-scans (not yet implemented).

The plans are implemented as generators using Bluesky decorators to integrate with a Run Engine. They construct rich metadata dictionaries (_md) describing the plan, its detectors, motors, and dimensional hints, and they leverage utilities from plan_patterns and other helper modules.

Key Components
--------------
- count(detectors, num=1, delay=None, *, per_shot=None, md=None)
  - Purpose: Read one or more detectors a specified number of times with an optional delay.
  - Behavior:
    - Allows num=None to indicate indefinite acquisition.
    - Computes num_intervals as num-1 (or None if num is None).
    - Builds an _md metadata dictionary containing detectors, points, plan name, and hints.
    - Uses per_shot (default: bps.one_shot) to customize the inner step.
    - Executes: yield from inner_count(), which yields from bps.repeat(partial(per_shot, detectors), num=num, delay=delay).
  - Notes: The inner function is decorated with stage and run decorators to integrate with the Run Engine.

  Example excerpt from code:
  .. code-block:: python
     def count(detectors, num=1, delay=None, *, per_shot=None, md=None):
         if num is None:
             num_intervals = None
         else:
             num_intervals = num - 1
         _md = {'detectors': [det.name for det in detectors],
                'num_points': num,
                'num_intervals': num_intervals,
                'plan_args': {'detectors': list(map(repr, detectors)), 'num': num},
                'plan_name': 'count',
                'hints': {}
                }
         _md.update(md or {})
         _md['hints'].setdefault('dimensions', [(('time',), 'primary')])
         if per_shot is None:
             per_shot = bps.one_shot

         @bpp.stage_decorator(detectors)
         @bpp.run_decorator(md=_md)
         def inner_count():
             return (yield from bps.repeat(partial(per_shot, detectors),
                                           num=num, delay=delay))

         return (yield from inner_count())

- list_scan(detectors, *args, per_step=None, md=None)
  - Purpose: Scan over one or more variables in steps in an inner-product fashion.
  - Behavior:
    - Validates that *args contains pairs of (motor, [points...]).
    - Ensures all motor point lists have the same length; otherwise raises ValueError.
    - Builds a metadata dictionary (_md) describing detectors, motors, number of points, and the inner scan pattern.
    - Determines x_fields from motors’ hints and sets default dimension hints accordingly.
    - Creates a full cycler via plan_patterns.inner_list_product(args) and delegates to scan_nd(detectors, full_cycler, per_step, md=_md).
  - Notes:
    - The plan pattern is recorded in _md with plan_name 'list_scan' and plan_pattern 'inner_list_product', including plan_pattern_module and plan_pattern_args.
    - The actual inner loop logic is provided by scan_nd and plan_patterns.

Usage
-----
Plans are designed to be used within a Bluesky Run Engine. They are generator-based and typically invoked via yield from within another plan or by directly enqueuing them in a Run Engine.

Examples:

- Counting readings from detectors:
  .. code-block:: python
     plan = count(detectors=[det1, det2], num=5, delay=0.1)
     RE(plan)

- Scanning over motors with points for each motor:
  .. code-block:: python
     plan = list_scan(detectors=[det1], motor1, [0, 1, 2],
                                     motor2, [10, 20, 30],
                                     per_step=None)
     RE(plan)

- Note: rel_list_scan is not yet implemented; attempting to use it will require awaiting future updates.

Details
-------
Metadata and plan structure
- Both count and list_scan build a metadata dictionary (_md) that describes:
  - detectors: list of detector names
  - num_points / length: number of points per motor sweep
  - num_intervals: num_points - 1 (or None if num is None)
  - plan_args: representation-friendly arguments to reconstruct the plan
  - plan_name: 'count' or 'list_scan'
  - hints: metadata hints, including dimension hints
- For count, default hints set dimensions to time as primary:
  _md['hints'].setdefault('dimensions', [(('time',), 'primary')])

- For list_scan, dimensions are derived from motor hints, allowing automatic axis labeling when motors expose 'fields' in their hints:
  x_fields = []
  for motor in motors:
      x_fields.extend(getattr(motor, 'hints', {}).get('fields', []))
  default_dimensions = [(x_fields, 'primary')]
  if x_fields:
      _md['hints'].update(dimensions=default_dimensions)

Implementation notes
- The count plan uses a decorator-driven inner function (inner_count) to wrap the generator that yields readings. It uses:
  - @bpp.stage_decorator(detectors)
  - @bpp.run_decorator(md=_md)
  to integrate with the Run Engine and attach metadata.
- The inner loop leverages bps.repeat(partial(per_shot, detectors), num=num, delay=delay), enabling per-shot customization of how readings are produced.
- list_scan constructs an inner cycler via plan_patterns.inner_list_product and delegates to scan_nd(detectors, full_cycler, per_step=per_step, md=_md). This aligns with Bluesky’s approach to scan generation and metadata propagation.
- rel_list_scan is present as a placeholder with a TODO comment, indicating future implementation.
