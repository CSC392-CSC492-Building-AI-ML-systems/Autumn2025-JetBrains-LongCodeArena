# w_stateprobs

## Overview
w_stateprobs is a deprecated shim that provides backward-compatible access to the state-to-state kinetics analysis functionality via a trace subcommand. It delegates to the w_direct tool under the hood and issues a deprecation warning encouraging users to switch to w_direct directly. The wrapper ensures the appropriate functions are available so it can be invoked as a regular tool rather than a subcommand.

The command computes averages and confidence intervals for state populations traced through weighted ensemble trajectories. It relies on bin assignments (including macrostate definitions) produced by prior steps (e.g., w_assign).

Key features include:
- Default output file: stateprobs.h5
- Default kinetics file: assign.h5
- Long-form description and output layout provided by the underlying tool

Code reference (key components):
- WStateProbs class (subcommand and defaults)
- WDirect master command (program wrapper and description)
- entry_point that warns about deprecation and routes to WDirect

```
class WStateProbs(DStateProbs):
    subcommand = 'trace'
    help_text = 'averages and CIs for path-tracing kinetics analysis'
    default_output_file ='stateprobs.h5'
    default_kinetics_file = 'assign.h5'


class WDirect(WESTMasterCommand, WESTParallelTool):
    prog = 'w_stateprobs'
    subcommands = [WStateProbs]
    subparsers_title = 'calculate state-to-state kinetics by tracing trajectories'
    description = '''\
Calculate average populations and associated errors in state populations from
weighted ensemble data. Bin assignments, including macrostate definitions,
are required. (See "w_assign --help" for more information).

-----------------------------------------------------------------------------
Output format
-----------------------------------------------------------------------------

The output file (-o/--output, usually "stateprobs.h5") contains the following
dataset:

  /avg_state_pops [state]
    (Structured -- see below) Population of each state across entire
    range specified.

If --evolution-mode is specified, then the following additional dataset is
available:

  /state_pop_evolution [window][state]
    (Structured -- see below). State populations based on windows of
    iterations of varying width.  If --evolution-mode=cumulative, then
    these windows all begin at the iteration specified with
    --start-iter and grow in length by --step-iter for each successive
    element. If --evolution-mode=blocked, then these windows are all of
    width --step-iter (excluding the last, which may be shorter), the first
    of which begins at iteration --start-iter.

The structure of these datasets is as follows:

  iter_start
    (Integer) Iteration at which the averaging window begins (inclusive).

  iter_stop
    (Integer) Iteration at which the averaging window ends (exclusive).

  expected
    (Floating-point) Expected (mean) value of the rate as evaluated within
    this window, in units of inverse tau.

  ci_lbound
    (Floating-point) Lower bound of the confidence interval on the rate
    within this window, in units of inverse tau.

  ci_ubound
    (Floating-point) Upper bound of the confidence interval on the rate
    within this window, in units of inverse tau.

  corr_len
    (Integer) Correlation length of the rate within this window, in units
    of tau.

Each of these datasets is also stamped with a number of attributes:

  mcbs_alpha
    (Floating-point) Alpha value of confidence intervals. (For example,
    *alpha=0.05* corresponds to a 95% confidence interval.)

  mcbs_nsets
    (Integer) Number of bootstrap data sets used in generating confidence
    intervals.

  mcbs_acalpha
    (Floating-point) Alpha value for determining correlation lengths.


-----------------------------------------------------------------------------
Command-line options
-----------------------------------------------------------------------------
'''
```

        # If we're not really supporting subcommands...
        import sys

            if sys.argv[1]!= 'trace':

## Usage
- Basic (legacy) invocation:
  - w_stateprobs
  - This will emit a deprecation warning and run the trace subcommand (defaulting to the appropriate inputs and outputs).

- Explicit invocation (via trace subcommand):
  - w_stateprobs trace -o stateprobs.h5 [options]
  - By default, the output file is stateprobs.h5 and the kinetics file is assign.h5 unless overridden.

- Evolution modes (when enabled with --evolution-mode):
  - The tool can produce the state_pop_evolution dataset with windows defined by --start-iter and --step-iter.
  - Modes include cumulative (windows grow) or blocked (fixed width).

- Default behavior:
  - Output file: stateprobs.h5
  - Kinetics file: assign.h5

- Important notes:
  - Bin assignments and macrostate definitions are required input (see w_assign --help).
  - This command is a compatibility shim for older usage; for new workflows, use w_direct directly.

## Details
Output layout for state probabilities (as described in the command description):
- Primary dataset:
  - /avg_state_pops [state]
    - Population of each state across the analyzed range.

- Optional dataset (if evolution mode is specified):
  - /state_pop_evolution [window][state]
    - Population evolution across specified windows.

Dataset structure (per window):
- iter_start: Integer, window start (inclusive)
- iter_stop: Integer, window stop (exclusive)
- expected: Floating-point, mean rate within the window (inverse tau)
- ci_lbound: Floating-point, lower CI bound (inverse tau)
- ci_ubound: Floating-point, upper CI bound (inverse tau)
- corr_len: Integer, correlation length (in tau)

Attributes stamped on each dataset:
- mcbs_alpha: Floating-point, CI alpha
- mcbs_nsets: Integer, number of bootstrap samples
- mcbs_acalpha: Floating-point, alpha for correlation length determination

Deprecation notice
- This wrapper is being deprecated in favor of using w_direct directly. The code path explicitly warns, then delegates to WDirect:
  - “w_stateprobs is being deprecated. Please use w_direct instead.”
