Delayed interface (dask.delayed)
=================================

Overview
--------
The dask.delayed interface provides a lightweight way to build task graphs by turning
regular Python computations into lazy, parallelizable tasks. It exposes:

- Delayed: a lightweight object representing a lazy computation or a subgraph
- delayed (factory decorator): a constructor used to wrap values or functions as delayed tasks
- Utilities to normalize Python objects into executable task graphs (unpack_collections, to_task_dask)

Public API (highlights)
- Delayed: Core class representing a lazy task or subgraph
- delayed: Factory to create Delayed objects from values or callables
- unpack_collections: Normalize Python objects containing Delayed objects into a runnable task and its merged dask graph
- finalize: Create a Delayed that runs a finalization step after computing a collection
- to_task_dask: Internal helper to normalize and merge subgraphs (deprecated in favor of unpack_collections)

Key Components
--------------
- Delayed class
  - Represents a lazy computation within a Dask graph
  - Integrates with Dask’s graph machinery (keys, graphs, postcompute hooks)
  - Public API surface exposed as Delayed (class) and delayed (factory)

- delayed
  - Public entry point to wrap values or functions as Delayed objects
  - Examples in code illustrate creating delayed values with explicit keys
  - Facilitates composition of lazy tasks into larger graphs

- unpack_collections
  - Normalizes a Python object containing Delayed objects and/or subgraphs
  - Replaces Delayed objects with their keys, converts literals to schedulable values
  - Extracts all embedded graphs and merges them into a single HighLevelGraph
  - Supports common Python containers (lists, tuples, sets, dicts) and dataclasses/namedtuples
  - Examples:
    - a = delayed(1, 'a'); b = delayed(2, 'b')
    - task, collections = unpack_collections([a, b, 3])
      - task -> ['a', 'b', 3]
      - collections -> (Delayed('a'), Delayed('b'))
    - task, collections = unpack_collections({a: 1, b: 2})
      - task -> (<class 'dict'>, [['a', 1], ['b', 2]])
      - collections -> (Delayed('a'), Delayed('b'))

- finalize
  - Creates a finalizing step for a given collection, returning a Delayed
  - Useful for post-processing or aggregating results after a computation
  - Example (conceptual): name = "finalize-" + tokenize(collection); layer = {name: (finalize, keys) + args}

- to_task_dask (internal, deprecated)
  - Legacy helper that mirrors unpack_collections behavior
  - Emits a deprecation warning directing users to unpack_collections

Usage
-----
Creating delayed values
- Basic usage (as shown in the source)
  - a = delayed(1, 'a')
  - b = delayed(2, 'b')

Normalizing and merging subgraphs
- Use unpack_collections to build a runnable task and merged graph from mixed inputs
  - task, collections = unpack_collections([a, b, 3])
  - task  # ['a', 'b', 3]
  - collections  # (Delayed('a'), Delayed('b'))

Examples
- Unpacking a simple list
  >>> import dask
  >>> a = delayed(1, 'a')
  >>> b = delayed(2, 'b')
  >>> task, collections = unpack_collections([a, b, 3])
  >>> task
  ['a', 'b', 3]
  >>> collections
  (Delayed('a'), Delayed('b'))

- Unpacking a dict
  >>> task, collections = unpack_collections({a: 1, b: 2})
  >>> task
  (<class 'dict'>, [['a', 1], ['b', 2]])
  >>> collections
  (Delayed('a'), Delayed('b'))

Deprecation notice
- The to_task_dask function is deprecated in favor of unpack_collections:
  - It emits a warning advising users to use unpack_collections instead

Details
-------
How normalization works
- If an expression is a Delayed, the task is its key and the collection includes the Delayed
- If the expression is a dask collection, it is finalized into a Delayed, and its graph is merged
- Iterables (list/tuple/set) are recursively unpacked; output type is preserved
- Dicts are transformed into (dict, [[key, value], ...]) form and subgraphs are extracted
- Dataclasses and namedtuples are supported; fields are unpacked and reconstructed
- For custom __init__ or init=False fields, appropriate errors are raised to guide users

Graph merging and execution
- unpack_collections gathers all subgraphs and returns:
  - A runnable task specification (task)
  - A collection of Delayed objects representing subgraphs (collections)
- The resulting HighLevelGraph can be used with Dask’s schedulers to compute results

Public API surface
- __all__ = ["Delayed", "delayed"]
- DEFAULT_GET defines the default scheduler (threads or sync)

Code references (from the source)
- Unpacked logic and examples are demonstrated in unpack_collections:
  - Replacement of Delayed with their keys
  - Conversion of literals to schedulable values
  - Extraction and merging of graphs
- finalize constructs a new Delayed object that wraps a finalize function and its arguments
- to_task_dask is present for backward compatibility but deprecated in favor of unpack_collections

Notes
- The delayed interface integrates with Dask’s core graph machinery, enabling parallel execution of Python code via lazy evaluation
- Use unpack_collections to convert arbitrary Python objects containing Delayed values into a graph and task for execution
