SANS to SESANS conversion
==========================

Overview
--------
This documentation describes the sans_to_sesans.rst module, which provides utilities used in the SANS to SESANS workflow within sasmodels. The primary focus is an inspection utility that enumerates parameters used across available models, aiding the construction and validation of SESANS-related conversions.

While the module is not itself a full SANS→SESANS conversion engine, it supports the workflow by exposing which model parameters (kernel parameters) influence the SESANS calculation. This facilitates debugging, model selection, and parameter tracking in SESANS analyses.

Key Components
--------------
- Parameter discovery across models
  - Scans all registered models and collects kernel parameters.
  - Optional filtering by parameter kind (volume, orientation, SLD) or returning all parameters.
- Printing/formatting utility
  - Provides a concise or detailed listing:
    - Names only (default)
    - Names with the models that use them (verbose mode)

Core functions (from the source)
- find_pars(kind=None)
  - Returns a mapping: {parameter_name: [model, model, ...]}
  - If kind is provided, only parameters of that kind are included.
  - Example signature and behavior:
    def find_pars(kind=None):
        """
        Find all parameters in all models.

        Returns the reference table *{parameter: [model, model, ...]}*
        """
- list_pars(names_only=True, kind=None)
  - Prints all parameters across models.
  - If names_only is True, prints only parameter names; otherwise prints "name: model1, model2, ..."
  - Relies on find_pars(kind) for its data.
  - Example signature and behavior:
    def list_pars(names_only=True, kind=None):
        """
        Print all parameters in all models.

        If *names_only* then only print the parameter name, not the models it
        occurs in.
        """
- main()
  - CLI entry point for the parameter-listing tool.
  - Supports:
    - -v / --verbose: list models which use each parameter
    - kind: optional positional to filter by volume/orientation/sld/none/any
  - Example signature and behavior:
    def main():
        """
        Program to list the parameters used across all models.
        """
        parser = argparse.ArgumentParser(...)
        parser.add_argument('-v', '--verbose', action='store_true', help="list models which use this argument")
        parser.add_argument('kind', default="any", nargs='?', metavar="volume|orientation|sld|none|any", choices=['volume', 'orientation', 'sld', None, 'any'], type=lambda v: None if v == 'any' else None if v == 'none' else v, help="only list arguments of the given kind")
        args = parser.parse_args()
        list_pars(names_only=not args.verbose, kind=args.kind)

Usage
-----
CLI usage for discovering which model parameters participate in SESANS-related calculations:

- List all parameters (names only):
  python -m sasmodels.list_pars

- List parameters with the models that use them (verbose):
  python -m sasmodels.list_pars -v

- Filter by parameter kind (volume, orientation, sld) or show all:
  - Without any filtering (all parameters):
    python -m sasmodels.list_pars
  - Filter to volume parameters only:
    python -m sasmodels.list_pars volume
  - Filter to orientation parameters only:
    python -m sasmodels.list_pars orientation
  - Filter to SLD parameters only:
    python -m sasmodels.list_pars sld
  - Show all parameters across all kinds:
    python -m sasmodels.list_pars any
  - Note: passing "none" or "any" maps to a non-filtered view due to the internal type handling.

Details
-------
- Purpose in the SANS→SESANS workflow:
  - This tooling helps users identify which model parameters impact SESANS conversions.
  - By listing kernel_parameters across registered models, researchers can target parameterization and diagnostics for SESANS-involved modeling work.

- Parameter kinds:
  - volume: parameters associated with volumetric properties
  - orientation: orientation-related parameters
  - sld: scattering length density-related parameters
  - none/any: special handling in the CLI to indicate no filtering (i.e., all parameters)

Notes
-----
- The sans_to_sesans.rst documentation focuses on how to inspect and track model parameters that are relevant to SESANS conversions. It does not implement the SESANS kernel itself but provides a practical utility to support SESANS-related modeling workflows.
