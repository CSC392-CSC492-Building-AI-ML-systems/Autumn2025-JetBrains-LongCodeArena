The Metadata Routing API
========================

Overview
--------

The Metadata Routing API provides utilities to route and forward metadata across sub-estimators within meta-estimators and composite estimators. It exposes a small set of primitives that help determine when routing is enabled, how to map method-level metadata to the appropriate sub-estimators, and how to process user-provided routing configurations.

Key Components
--------------

- MetadataRouter
  - Central orchestrator for routing metadata to sub-estimators.
  - Enables or disables routing based on configuration and context.
- MethodMapping
  - Structure describing how methods or metadata should be routed to sub-estimators.
- process_routing
  - Helper to convert user-provided routing specifications into internal routing structures understood by MetadataRouter.
- _raise_for_params
  - Internal validation helper that raises informative errors when required routing parameters are missing or invalid.
- _routing_enabled
  - Internal flag/function to determine whether routing is currently active for a given configuration.

Usage
-----

The API is intended for use within meta-estimators and other composite estimators that need to forward or route metadata to their sub-estimators. Typical usage involves creating a routing specification, processing it, and applying it via a MetadataRouter.

- Public API surface (names imported from the module):
  - MetadataRouter
  - MethodMapping
  - process_routing
  - _raise_for_params (internal)
  - _routing_enabled (internal)

Code example (illustrative)
:: python

from your_project.utils.metadata_routing import MetadataRouter, process_routing, MethodMapping

# Define routing rules (illustrative; adapt to your actual routing needs)
routing_spec = {
    "get_feature_names": "base_estimator",
    "get_params": "base_estimator",
}

# Process routing spec into internal representation
routing_info = process_routing(routing_spec)

# Create a router for an underlying estimator
base_estimator = ...  # your sub-estimator
router = MetadataRouter(estimator=base_estimator, routing=routing_info)

# Use `router` in a higher-level meta-estimator where metadata routing is desired

Details
-------

- The routing primitives are designed to support scenarios where meta-estimators need to forward or adapt metadata (e.g., parameter names, feature names, or other metadata) to their contained estimators.
- Internal helpers (_raise_for_params, _routing_enabled) are primarily for internal use and validation; external users should rely on the documented public API components (MetadataRouter, MethodMapping, process_routing).
- This API is leveraged by multi-output and meta-estimators to extend single-output estimators to multiple outputs while preserving or distributing relevant metadata across sub-estimators.
