# Parallel Random Number Generation

## Overview
This module provides parallel-friendly random number generation by exposing multiple bit generators (including MT19937, Philox, PCG64, and SFC64) through NumPy’s random framework. The build structure compiles a shared set of C sources and Cython extensions to deliver fast, independent random streams suitable for parallel workloads. Independent streams can be created using seed sequences to ensure reproducibility across parallel tasks.

Key components implemented in this repository include a central BitGenerator interface and dedicated backends for each algorithm, enabling flexible parallel usage via separate streams.

## Key Components
- npyrandom static library
  - Core C sources for distributions and random-related utilities:
    - src/distributions/logfactorial.c
    - src/distributions/distributions.c
    - src/distributions/random_mvhg_count.c
    - src/distributions/random_mvhg_marginals.c
    - src/distributions/random_hypergeometric.c
  - Built as a static library and installed under random/lib.
- Cython extensions for numpy.random
  - _bounded_integers
    - Generated via a template system (custom_target) to produce _bounded_integers.pxd and _bounded_integers.pyx from templates.
  - _common, _mt19937, _philox, _pcg64, _sfc64
    - Implement the respective bit generators:
      - MT19937 (Mersenne Twister)
      - Philox
      - PCG64
      - SFC64
  - bit_generator
    - Core interface that unifies the various bit generators for use by higher-level NumPy random APIs.
  - _generator, mtrand
    - Provide the bridging and legacy interfaces for compatibility.
- Build and installation workflow
  - Uses Meson to compile npyrandom and the Python/Cython extensions.
  - Custom generation step for _bounded_integers.pxd ensures proper integration with the generated Cython code.
  - Extensions install under numpy/random with tests, data, and examples included.
- Seed management and parallel streams
  - Although not shown in code, the integration supports parallel usage by enabling independent streams via SeedSequence spawning.

## Usage
- Basic usage with a single bit generator:
  - Example using PCG64:
    - from numpy.random import Generator
    - from numpy.random import PCG64
    - bg = PCG64(12345)
    - g = Generator(bg)
    - g.integers(0, 10, size=5)

- Creating multiple independent parallel streams:
  - Example using SeedSequence to spawn streams:
    - import numpy as np
    - ss = np.random.SeedSequence(1234)
    - child_seeds = ss.spawn(4)
    - rngs = [Generator(PCG64(s)) for s in child_seeds]
    - for i, r in enumerate(rngs):
        - print(i, r.integers(0, 100, size=5))

- Bounded integer generation
  - Example:
    - g = Generator(PCG64(123))
    - g.integers(0, 100, size=1000)

- Notes on algorithm choice for parallel workloads
  - For parallel scalable workloads, Philox, PCG64, and SFC64 are typically preferred for their parallel-friendly properties.
  - MT19937 is suitable for single-threaded usage but is less ideal for producing independent parallel streams without careful seeding strategies.

## Details
- The parallel natively supported RNG backends
  - MT19937: Mersenne Twister
  - Philox
  - PCG64
  - SFC64
- Build-time details
  - A static npyrandom library is built from the core distribution sources and installed under numpy/random/lib.
  - Cython extensions are generated for each bit generator and wired together through the bit_generator interface.
  - A special custom_target is used to generate _bounded_integers.pxd from a template to ensure correct C API exposure.
- File and installation layout
  - Core library: numpy/random/lib (via np_dir/random/lib)
  - Extensions live under numpy/random in the Python package
    - _bounded_integers, _common, _mt19937, _philox, _pcg64, _sfc64, bit_generator, mtrand, and _generator
  - Tests, data, and examples are installed under numpy/random/tests, numpy/random/tests/data, and numpy/random/_examples respectively
