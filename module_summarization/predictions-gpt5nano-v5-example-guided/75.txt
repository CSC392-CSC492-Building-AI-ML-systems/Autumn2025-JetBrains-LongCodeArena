Generate tutorial about work with text data
===========================================

Overview
--------
This module provides utilities to convert text documents into numeric feature vectors suitable for machine learning. It supports tokenization, normalization, stop words removal, n-gram generation, and weighting (e.g., TF-IDF). Key vectorizers include CountVectorizer, TfidfVectorizer, and HashingVectorizer, along with a built-in stop words list (ENGLISH_STOP_WORDS) and text cleaning helpers (strip_accents_unicode, strip_accents_ascii, strip_tags).

Key Components
--------------
- CountVectorizer
  - Converts a collection of text documents to a matrix of token counts.
  - Options include analyzer, tokenizer, preprocessor, stop_words, ngram_range, max_features, etc.
- TfidfTransformer
  - Transforms a count matrix to a TF-IDF representation.
- TfidfVectorizer
  - Convenience wrapper that combines CountVectorizer and TfidfTransformer into a single step.
- HashingVectorizer
  - Uses the hashing trick to map text to a fixed-dimensional feature space without an explicit vocabulary.
- ENGLISH_STOP_WORDS
  - Built-in set of common English stop words.
- strip_accents_unicode
  - Remove accent marks from Unicode characters (preserves ASCII equivalents when possible).
- strip_accents_ascii
  - Strip accents by transliterating to ASCII where possible.
- strip_tags
  - Remove HTML/XML-like tags using a simple regular expression.
- _check_stop_list
  - Internal helper to resolve stop word specifications (e.g., 'english', a collection, or None).

Usage
-----
Basic usage with CountVectorizer
- Tokenize and count word occurrences in a corpus.

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   corpus = [
       "The quick brown fox jumps over the lazy dog.",
       "The dog is quick and the fox is brown."
   ]

   vectorizer = CountVectorizer()
   X = vectorizer.fit_transform(corpus)
   print(vectorizer.get_feature_names_out())
   print(X.toarray())

Using English stop words
- Remove common English words during vectorization.

.. code-block:: python

   vectorizer = CountVectorizer(stop_words='english')
   X = vectorizer.fit_transform(corpus)

Preprocessing options (strip HTML and accents)
- You can supply preprocessor callables, such as HTML tag stripping or accent stripping, to clean text before tokenization.

.. code-block:: python

   from sklearn.feature_extraction.text import strip_tags

   docs = ["<p>The cat sits on the mat.</p>", "<div>The dog barks.</div>"]
   vectorizer = CountVectorizer(preprocessor=strip_tags)
   X = vectorizer.fit_transform(docs)

   from sklearn.feature_extraction.text import strip_accents_unicode
   preprocessed = [strip_accents_unicode(doc) for doc in docs]

Combining with TF-IDF
- Use TfidfVectorizer for TF-IDF weighting in a single step.

.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   corpus = [
       "The quick brown fox jumps over the lazy dog.",
       "The dog is quick and the fox is brown."
   ]

   tfidf = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
   X = tfidf.fit_transform(corpus)

Hashing-based vectorization
- For large-scale or streaming text data, HashingVectorizer avoids a fixed vocabulary.

.. code-block:: python

   from sklearn.feature_extraction.text import HashingVectorizer

   corpus = [
       "The quick brown fox",
       "Lazy dogs sleep"
   ]

   hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
   X = hasher.fit_transform(corpus)

Notes on internal processing
----------------------------
- _preprocess(doc, accent_function=None, lower=False)
  - Chains optional steps: lowercasing and accent handling before further processing.
- _analyze(...)
  - Chains preprocessing steps (preprocessor, tokenizer, ngrams) unless a high-level analyzer is provided.
  - If analyzer is provided, only the decoder is applied; otherwise, the preprocessor/tokenizer/ngrams path is used.
- decode(self, doc)
  - Handles input formats such as filenames, file-like objects, or strings, with appropriate decoding.
- _check_stop_list(stop)
  - Converts stop list input to a usable form, supporting 'english', a collection, or None.
- strip_accents_unicode and strip_accents_ascii
  - Provide different strategies for removing accents depending on language requirements.
- strip_tags
  - Removes HTML/XML-like tags using a simple regex for lightweight preprocessing before tokenization.

Details
-------
- The module focuses on transforming text into numerical features for ML workflows.
- Stop words can be supplied via stop_words='english' or by using ENGLISH_STOP_WORDS directly.
- The internal decode logic allows input flexibility (filenames, file objects, or raw strings).
- The provided helpers enable lightweight, modular preprocessing steps suitable for pipelines.
