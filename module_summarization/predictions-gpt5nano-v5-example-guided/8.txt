# Using review sessions

## Overview
A review session provides an isolated context for interacting with an ftrack server. It is implemented by the Session class and supports authentication, optional embedded event hub connectivity, caching, and operation recording. A review session can be configured to automatically populate entity data on access, publish and subscribe to local or remote events, and cache schema information for faster startup.

Key capabilities include:
- Authentication via API key and user
- Optional embedded event hub (local and/or non-local events)
- Caching with a configurable top-level memory cache
- Automatic schema caching and optional disabling of caching
- Tracking of pending operations
- Optional plugin argument reduction during discovery

## Key Components
- Session
  - An isolated client context for interacting with the ftrack server.
  - Parameters include:
    - server_url: URL of the ftrack server (including port if needed)
    - api_key: API key for authentication
    - api_user: API user name
    - auto_populate: whether entity attributes fetch from the server on access
    - plugin_paths: paths to search for plugins
    - cache: a cache instance or a callable returning one
    - cache_key_maker: key maker for cache entries
    - auto_connect_event_hub: enables embedded event hub connectivity in the background
    - schema_cache_path: folder path for schema caching (or False to disable)
    - plugin_arguments: mapping of keyword arguments to pass to plugin registrations
  - Internal state:
    - recorded_operations: holds pending operations
    - record_operations: boolean flag to enable/disable operation recording
- SessionAuthentication
  - Attaches authentication headers to requests:
    - ftrack-api-key
    - ftrack-user
- Event hub
  - When enabled via auto_connect_event_hub, the session connects to the event server in a background thread.
  - Subscribing to events does not require a connected hub, but some plugins may require it.
- Cache
  - The session uses a layered cache with a MemoryCache at the top level by default.
  - If a custom cache is provided, it is integrated into this layered cache.
- Schema caching
  - schema_cache_path enables caching of schema information to speed up session startup.
  - If not provided, FTRACK_API_SCHEMA_CACHE_PATH is consulted.
  - If neither is set, a temporary directory is used.
  - Setting schema_cache_path to False disables schema caching entirely.
- Plugin arguments
  - plugin_arguments allows passing extra arguments to plugin registration.
  - If a plugin signature is incompatible, the discovery mechanism will reduce the arguments to those accepted by the plugin and log a warning.

## Usage
- Basic instantiation (required parameters)
  - server_url, api_key, and api_user can be provided directly or sourced from environment variables:
    - FTRACK_SERVER for server_url
    - FTRACK_API_KEY or FTRACK_APIKEY for api_key
    - FTRACK_API_USER for api_user
- Example: provide credentials directly
```python
from ftrack_api import Session

session = Session(
    server_url='https://ftrack.example.com',
    api_key='YOUR_API_KEY',
    api_user='YOUR_USERNAME'
)
```

- Example: rely on environment variables (and optional features)
```python
# Environment variables:
# FTRACK_SERVER, FTRACK_API_KEY (or FTRACK_APIKEY), FTRACK_API_USER
# Optional: FTRACK_API_SCHEMA_CACHE_PATH for schema caching
session = Session(auto_connect_event_hub=True)
```

- Auto-populate behavior
```python
# If auto_populate is True (default), accessing entity attributes will fetch
# missing data from the server.
```

- Connecting the event hub
```python
session = Session(
    server_url='https://ftrack.example.com',
    api_key='YOUR_API_KEY',
    api_user='YOUR_USERNAME',
    auto_connect_event_hub=True
)
```

- Disabling schema caching
```python
session = Session(
    server_url='https://ftrack.example.com',
    api_key='YOUR_API_KEY',
    api_user='YOUR_USERNAME',
    schema_cache_path=False
)
```

- Providing plugin arguments
```python
session = Session(
    server_url='https://ftrack.example.com',
    api_key='YOUR_API_KEY',
    api_user='YOUR_USERNAME',
    plugin_arguments={'log_level': 'DEBUG'}
)
```

## Details
- Required parameters and environment fallbacks
  - server_url, api_key, and api_user are required unless their corresponding environment variables provide values.
  - If required values are missing, a TypeError is raised with guidance on how to supply them.
- auto_populate
  - When enabled, entity attribute access triggers a server fetch for data not yet loaded.
- Operational recording
  - The session maintains pending operations in recorded_operations and can enable/disable recording via record_operations.
- Embedded event hub
  - The event hub connection is performed in a background thread to improve startup time.
  - Plugins may require a connected hub; if so, they should check the hub status explicitly.
  - Subscribing to events does not require a connected hub.
- Schema caching
  - The presence of a schema cache path enables caching of schema data for faster restarts.
  - Environment variable FTRACK_API_SCHEMA_CACHE_PATH is consulted if schema_cache_path is not provided.
  - Setting to False disables caching entirely.
- Plugin arguments
  - During plugin discovery, passed arguments are filtered to those accepted by the plugin if necessary, with a warning logged when reductions occur.
- Security headers
  - The SessionAuthentication class ensures requests include the ftrack-api-key and ftrack-user headers for authentication.
