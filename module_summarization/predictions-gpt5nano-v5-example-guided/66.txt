# Generate tutorial about pipelines

## Overview
The sklearn.pipeline module provides utilities to build a composite estimator as a chain of transforms and estimators. Pipelines allow you to apply multiple steps in a sequence, where intermediate steps are transformers (fit/transform) and the final step is an estimator (fit).

From the source:
- The Pipeline class enables chaining transforms and a final estimator.
- Steps are named and can be configured or replaced using parameter syntax (e.g., step__param).
- The module also exposes helpers like FeatureUnion, and convenience constructors make_pipeline and make_union.

Key example demonstrates integrating scaling with a classifier and tuning parameters via hierarchical names.

## Key Components
- Pipeline
  - Chains a list of steps (name, transform) tuples, ending with an estimator.
  - Intermediate steps must implement fit and transform; the final step must implement fit.
  - Optional memory parameter to cache fitted transformers.
  - verbose option to print progress as steps are fitted.
  - Parameters can be set using hierarchical names (e.g., svc__C).
  - Can replace a step with another estimator or remove it by using 'passthrough' or None.
  - Access steps via named_steps or steps.
  - If memory is set, the last step is never cached.
  - set_output method to configure output format for transform/fit_transform (e.g., default or pandas).
  - Attributes include:
    - named_steps: dictionary-like access to each step by name
    - classes_, n_features_in_, feature_names_in_ (availability depends on underlying estimator)
- FeatureUnion
  - Combines features from multiple transformer outputs in parallel.
- make_pipeline
  - Convenience function to create a Pipeline from a sequence of estimators without naming each step.
- make_union
  - Convenience function to create a FeatureUnion from a sequence of transformers without naming each step.

Notes from the implementation:
- The Pipeline API is designed to support cross-validation and grid search by allowing parameter setting with the '__' syntax.
- The final estimator can be a classifier, regressor, or any estimator implementing fit.

Relevant example usage is shown in the next section.

## Usage
Typical usage shows constructing a pipeline with a scaler followed by a classifier, fitting on training data, and evaluating on test data.

Code example:
```python
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.pipeline import Pipeline

X, y = make_classification(random_state=0)
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=0)

pipe = Pipeline([('scaler', StandardScaler()), ('svc', SVC())])
pipe.fit(X_train, y_train).score(X_test, y_test)

# Tune a parameter of the final estimator
pipe.set_params(svc__C=10).fit(X_train, y_train).score(X_test, y_test)
```

For convenience, you can use the make_pipeline helper:
```python
from sklearn.pipeline import make_pipeline

pipe = make_pipeline(StandardScaler(), SVC())
```

Further composition can be done with FeatureUnion to combine features from multiple transformers:

## Details
- The Pipeline class is the primary composition tool for building end-to-end workflows that can be cross-validated or tuned with grid search.
- Parameters can be set using the syntax step_name__param_name, enabling flexible experimentation without reconstructing the pipeline.
- The order of steps matters: intermediate steps must transform data, and the final step must be an estimator that can be fitted.
- Memory caching can significantly speed up repeated fittings of expensive transformers; however, the final estimator is never cached.
- The set_output method allows configuring the output type of transform and fit_transform across all steps (e.g., default or pandas DataFrame output).
- Attributes such as named_steps, classes_, n_features_in_, and feature_names_in_ provide convenient access to internal state after fitting.
