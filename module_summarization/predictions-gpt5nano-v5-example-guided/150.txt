# Supporting data to plotting functions in multiple ways

## Overview
This module provides flexible data handling to support plotting functions across multiple data formats. It allows:
- Using common data containers (dicts, pandas DataFrames) or an optional Bokeh ColumnDataSource.
- Single data sources or multiple sources (lists) with automatic alignment and expansion into plotting sets.
- Mapping of data keys to plot inputs through a lightweight, iterable interface.
- Optional copying of sources to avoid mutating the original data.
- Optional enforcement of data length consistency and restriction on new keys.

Core component: PlotData

- The PlotData class lets plotting code work with various input types (dicts, DataFrames, and optionally ColumnDataSource from Bokeh) and exposes a uniform, iterable interface for per-plot-set data.
- It supports defining which keys in the data correspond to x, y, z, color, size, etc., and automatically expands to multiple plotting sets if any of the provided key values are lists.

Key supporting helpers:
- _is_bokeh_cds(data): Detects if a given object is a Bokeh ColumnDataSource (when Bokeh is installed).
- _copy_bokeh_cds(data): Creates a copy of a Bokeh ColumnDataSource to avoid in-place mutation.

Data holders supported:
- With Bokeh installed: dict, pandas.DataFrame, ColumnDataSource
- Without Bokeh: dict, pandas.DataFrame

The following features are configurable via constructor arguments:
- use_column_source: Integrates with column data sources when available.
- same_length: If True and dealing with dict-like sources, ensures all entries have compatible lengths.
- strict_data_keys: If True, disallows adding new data keys after initialization.
- copy_data: If True, makes a shallow copy of the provided data sources.

In addition, keyword arguments define the columns for the plotting data. If any keyword value is a list, the data expands into multiple plot sets, each represented by a namedtuple with the specified fields.

Code references (high level):
- PlotData(data, use_column_source=False, same_length=False, strict_data_keys=False, copy_data=False, **kwargs)
- Internals:
  - _column_spec = namedtuple('Columns', list(kwargs.keys()))
  - self.columns: list of per-set column specifications
  - self.data_keys: property returning the field names of the current column specification
  - add_data_key(data_key, keys=None): allows adding a new data key (subject to strict_data_keys)

Usage
- Basic usage with a single source (dict):
  - Map x and multiple y values; iterate over the resulting plot sets.

Code example (corrected for syntax):
```
from masci_tools.vis.data import PlotData
import numpy as np

# Example data: one energy grid with several functions
x = np.linspace(-10, 10, 100)
data = {'x': x, 'y1': np.sin(x), 'y2': np.cos(x), 'y3': x**2}

p = PlotData(data, x='x', y=['y1', 'y2', 'y3'])

# Iterate over plot sets: each iteration provides the data for one y-key
for entry in p:
    print(entry.x)  # 'x' data
    print(entry.y)  # 'y1' data in first iteration, then 'y2', then 'y3'
```

- Basic usage with a DataFrame:
```
import pandas as pd

df = pd.DataFrame({'x': x, 'y1': np.sin(x), 'y2': np.cos(x), 'y3': x**2})
p = PlotData(df, x='x', y=['y1','y2','y3'])

for entry in p:
    print(entry.x, entry.y)
```

- Multiple plot sets via list-like kwargs:
```
# If any of the y entries were provided as a list, PlotData expands to multiple sets.
p = PlotData(data, x='x', y=['y1', 'y2', 'y3'])  # expands internally to 3 plotting sets
```

- Copy safety and data integrity options:
```
# Copy sources to avoid mutating original data
p = PlotData(data, x='x', y=['y1','y2','y3'], copy_data=True)

# Enforce that no new data keys are introduced after initialization
p = PlotData(data, x='x', y=['y1','y2','y3'], strict_data_keys=True)
```

- Advanced: handling multiple data sources
```
sources = [ {'x': x, 'y1': np.sin(x)}, {'x': x, 'y1': np.cos(x)} ]
p = PlotData(sources, x='x', y=['y1','y2'], same_length=True)
for entry in p:
    # entry.x and entry.y reflect per-plot-set data
    pass
```

Details
- Data keys and sets
  - The plotting interface is defined by a namedtuple Columns created from the provided kwargs.
  - If a value in kwargs is a list, the number of plotting sets is determined by the maximum length among those lists.
  - Each set in self.columns is a snapshot (via a per-set namedtuple) of the specified keys for that plot.
  - The data_keys property returns the field names of the current column specification.

- Data normalization and length checks
  - When same_length is enabled and the data is dict-like, entries are normalized to ensure consistent lengths across keys.
  - If data is a list of sources, each element is normalized accordingly (where applicable).

- Bokeh integration
  - If Bokeh is installed, the module can work with ColumnDataSource in addition to standard Python data holders.
  - Copying for Bokeh sources is handled to avoid unintended mutations.

- Extensibility
  - add_data_key allows dynamic extension of the plotting keys (subject to strict_data_keys constraints). This enables evolving plot configurations without reconstructing the entire PlotData instance.

Notes
- The module aims to keep plotting code flexible: you can feed it raw data structures (dicts, DataFrames) or Bokeh sources, while keeping a consistent, iterable interface for generating per-plot-set data.
- The exact behavior of iteration and some internal edge cases depend on the internal implementation details not shown in this excerpt (e.g., _normalize_dict_entries, iteration protocol). The documentation above reflects the documented usage and purpose as implemented.
