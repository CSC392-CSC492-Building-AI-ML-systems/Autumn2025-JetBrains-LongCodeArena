Kernel approximation (scikit-learn)
===================================

Overview
--------
The sklearn.kernel_approximation module implements several approximate kernel
feature maps based on Fourier transforms and Count Sketches. In particular,
the PolynomialCountSketch class provides a Tensor Sketch-based approximation
for the polynomial kernel:

  K(X, Y) = (gamma * <X, Y> + coef0) ^ degree

The implementation uses Count Sketches with FFT (Fast Fourier Transforms) to
efficiently approximate the outer product features. See the user guide entry
polynomial_kernel_approx for details.

Key Components
--------------
- PolynomialCountSketch
  - Purpose: Approximate the feature map of a polynomial kernel using Tensor Sketch.
  - Inherits: ClassNamePrefixFeaturesOutMixin, TransformerMixin, BaseEstimator.
  - Core idea: Uses degree-wise random hash functions (Count Sketch) to map
    input features into a lower-dimensional feature space of size n_components.
  - Parameters:
    - gamma (float, default=1.0): Kernel coefficient for <X, Y>.
    - degree (int, default=2): Degree of the polynomial kernel to approximate.
    - coef0 (float, default=0): Constant term added to the polynomial kernel.
    - n_components (int, default=100): Output feature dimensionality.
    - random_state (int, RandomState, default=None): Seed for reproducible random hashes.
  - Attributes:
    - indexHash_ (ndarray, shape (degree, n_features)): 2-wise independent hash indices for Count Sketch.
    - bitHash_ (ndarray, shape (degree, n_features)): 2-wise independent hash bits in {+1, -1}.
    - n_features_in_ (int): Number of features seen during fit.
    - feature_names_in_ (ndarray, shape (n_features_in_,), optional): Names of input features if provided.
  - Notes:
    - If coef0 != 0, the feature space expands by 1 to accommodate the constant term.
    - _n_features_out is set to n_components.
    - Uses check_random_state for reproducible hash initialization.
  - See Also: AdditiveChi2Sampler, Nystroem, RBFSampler, SkewedChi2Sampler, and kernel_metrics helpers.

Usage
-----
Example workflow using PolynomialCountSketch:

.. code-block:: python

    from sklearn.kernel_approximation import PolynomialCountSketch
    from sklearn.linear_model import SGDClassifier

    X = [[0, 0], [1, 1], [1, 0], [0, 1]]
    y = [0, 0, 1, 1]

    ps = PolynomialCountSketch(degree=3, random_state=1)
    X_features = ps.fit_transform(X)

    clf = SGDClassifier(max_iter=10, tol=1e-3)
    clf.fit(X_features, y)
    score = clf.score(X_features, y)
    print(score)  # Expected to be 1.0 on this toy example

Details
-------
- The module targets approximate kernel feature maps rather than exact kernel computations.
- Tensor Sketch relies on Count Sketch and FFT-based computations to efficiently
  approximate outer products of input features.
- Random hashing components:
  - indexHash_: shape (degree, n_features) with integers in [0, n_components)
  - bitHash_: shape (degree, n_features) with values in {+1, -1}
- Data handling:
  - fit validates X (dense or sparse CSC) and records the number of input features.
  - If coef0 ≠ 0, the effective feature count increases by one to account for the bias term.
- Version notes (in the class docstring):
  - Tensor Sketch was added in version 0.24.
- Implementation details:
  - Internally uses _fit_context(prefer_skip_nested_validation=True) for validation flow.
  - Random state handling via check_random_state to ensure reproducibility.
- Related components:
  - This approach complements other kernel-approximation methods in sklearn.kernel_approximation
    such as AdditiveChi2Sampler, Nystroem, RBFSampler, and SkewedChi2Sampler.
