# Multiclass and multioutput algorithms

## Overview
This module provides meta-estimators that extend single-output estimators to handle multioutput tasks (both regression and classification). The estimators in this module require a base estimator supplied at construction and then apply that estimator to each target dimension, or in a chained fashion across targets.

Public API includes:
- MultiOutputRegressor
- MultiOutputClassifier
- ClassifierChain
- RegressorChain

Internally, the module defines helpers and a base class used by these meta-estimators:
- _MultiOutputEstimator: base class for multioutput meta-estimators
- _fit_estimator: fit a sub-estimator on a single target
- _partial_fit_estimator: support for partial_fit on sub-estimators
- _available_if_estimator_has: conditional availability of features based on sub-estimator capabilities

These components enable fitting and predicting across multiple targets by cloning and applying the supplied base estimator, with optional parallelization via the n_jobs parameter.

## Key Components

- MultiOutputRegressor: Fit one regressor per target variable.
- MultiOutputClassifier: Fit one classifier per target variable; validates that targets are suitable for classification.
- ClassifierChain: Train classifiers in a predefined chain, feeding the previous outputs as additional features to subsequent classifiers.
- RegressorChain: Similar to ClassifierChain but for regression tasks.

Note: The public API is designed for meta-estimators that wrap a base estimator implementing fit and predict.

## Implementation highlights

- Base class and constraints
  - _MultiOutputEstimator(MetaEstimatorMixin, BaseEstimator, metaclass=ABCMeta)
  - Parameter constraints indicate:
    - estimator must have fit and predict methods
    - n_jobs is an integer or None
  - Example constructor:
    - __init__(self, estimator, *, n_jobs=None)
    - Stores the base estimator and the number of parallel jobs

  Code excerpt:
  - Base class signature
    def __init__(self, estimator, *, n_jobs=None):
        self.estimator = estimator
        self.n_jobs = n_jobs

  - Parameter constraints
    _parameter_constraints: dict = {
        "estimator": [HasMethods(["fit", "predict"])],
        "n_jobs": [Integral, None],
    }

- Fitting sub-estimators for each target
  - _fit_estimator clones the base estimator and fits it to X and y (optionally using sample_weight)
  - If sample_weight is provided, it is passed to the sub-estimator during fit

  Code excerpt:
  def _fit_estimator(estimator, X, y, sample_weight=None, **fit_params):
      estimator = clone(estimator)
      if sample_weight is not None:
          estimator.fit(X, y, sample_weight=sample_weight, **fit_params)
      else:
          estimator.fit(X, y, **fit_params)
      return estimator

- Partial fitting support
  - _partial_fit_estimator provides support for partial_fit on sub-estimators
  - First-time calls clone the estimator; subsequent calls reuse the clone
  - Supports passing classes for incremental learning if the sub-estimator supports it

  Code excerpt:
  def _partial_fit_estimator(
      estimator, X, y, classes=None, partial_fit_params=None, first_time=True
  ):
      partial_fit_params = {} if partial_fit_params is None else partial_fit_params
      if first_time:
          estimator = clone(estimator)

      if classes is not None:
          estimator.partial_fit(X, y, classes=classes, **partial_fit_params)
      else:
          estimator.partial_fit(X, y, **partial_fit_params)
      return estimator

- Conditional exposure of sub-estimator attributes
  - _available_if_estimator_has(attr) returns a function that checks whether the sub-estimator (or any of the sub-estimators) has a given attribute
  - This is used to enable/disable certain features only when the base estimator supports them

  Code excerpt:
  def _available_if_estimator_has(attr):
      """Return a function to check if the sub-estimator(s) has(have) `attr`."""
      def _check(self):
          if hasattr(self, "estimators_"):
              return all(hasattr(est, attr) for est in self.estimators_)

          if hasattr(self.estimator, attr):
              return True

          return False

      return available_if(_check)

- Classification target validation
  - The module imports check_classification_targets to validate that targets are suitable for classification in MultiOutputClassifier scenarios

## Usage

Examples illustrate applying a base estimator to handle multiple targets.

- MultiOutputClassifier example

  - Model: base estimator is LogisticRegression
  - Purpose: handle multi-label or multioutput classification

  Example:
  - from sklearn.linear_model import LogisticRegression
  - from sklearn.multioutput import MultiOutputClassifier
  - model = MultiOutputClassifier(LogisticRegression(max_iter=1000))
  - model.fit(X, y)
  - y_pred = model.predict(X)

- MultiOutputRegressor example

  - Model: base estimator is RandomForestRegressor
  - Purpose: fit one regressor per target

  Example:
  - from sklearn.ensemble import RandomForestRegressor
  - from sklearn.multioutput import MultiOutputRegressor
  - reg = MultiOutputRegressor(RandomForestRegressor(n_estimators=10), n_jobs=-1)
  - reg.fit(X, y)
  - y_pred = reg.predict(X)

- ClassifierChain example

  - Model: base estimator is a classifier (e.g., LogisticRegression)
  - Purpose: chain classifiers so each predicts and passes its output as a feature to the next

  Example:
  - from sklearn.linear_model import LogisticRegression
  - from sklearn.multioutput import ClassifierChain
  - chain = ClassifierChain(LogisticRegression(max_iter=1000), order='roman')
  - chain.fit(X, y)
  - y_pred = chain.predict(X)

- RegressorChain example

  - Model: base estimator is a regressor
  - Purpose: chain regressors with outputs passed as features along the chain

  Example:
  - from sklearn.ensemble import RandomForestRegressor
  - from sklearn.multioutput import RegressorChain
  - chain = RegressorChain(RandomForestRegressor(n_estimators=100), order=[0, 1, 2])
  - chain.fit(X, y)
  - y_pred = chain.predict(X)

## Details

- Check targets
  - MultiOutputClassifier applies classification target checks via check_classification_targets to ensure proper target formatting for classification tasks.

- Parallelization
  - The n_jobs parameter controls the degree of parallelization when fitting multiple sub-estimators.

- Internal helpers
  - The module includes internal helpers (_fit_estimator, _partial_fit_estimator, _available_if_estimator_has) that encapsulate common logic for fitting, incremental learning, and feature exposure across the multioutput estimators.

- Relationship to single-output estimators
  - Each target dimension can be modeled independently (in many cases) or sequentially via a chain, using a supplied base estimator that implements fit and predict.
