# Interaction with smart contracts over Web3 connector

## Overview
This module defines a base Contract class used to generate contract proxy classes for interacting with Ethereum smart contracts via a Web3 connector. It supports deploying contracts, encoding ABI calls, and subscribing to contract events. Classes are typically created via a factory (e.g., construct_contract_factory) that derives from this base class.

Key capabilities include:
- Managing contract ABI, bytecode, and source information
- Deploying contracts to the blockchain
- Encoding function calls to ABI-compatible payloads
- Subscribing to and handling contract events

The class assumes Web3 is initialized and available via the class attribute `web3`.

## Key Components

- Contract (base class)
  - Class attributes
    - web3: Web3 instance (must be initialized before use)
    - _abi, _code, _code_runtime, _source: class-level contract definitions (overrideable per instance)
  - Instance attributes
    - address: deployed contract address
  - Important properties
    - abi: returns the contract ABI or raises if not provided
    - code: returns the contract bytecode or raises if not provided
    - code_runtime: returns the runtime bytecode or raises if not provided
    - source: returns contract source if provided
  - Initialization
    - __init__(abi=None, address=None, code=None, code_runtime=None, source=None)
    - Validates Web3 initialization and applies overrides for abi/code/code_runtime/source
  - Deployment
    - deploy(transaction=None, args=None, kwargs=None)
    - Deploys the contract to the blockchain using provided transaction parameters and constructor arguments
    - Validates that code is present and that unsupported fields (data, to) are not specified
    - Uses _encode_constructor_data(args, kwargs) to prepare deployment data
    - Returns the transaction hash
    - Example:
      - 
        ```python
        txn_hash = MyContract.deploy(
            transaction={
                'from': web3.eth.accounts[1],
                'value': 12345,
            },
            args=('DGD', 18),
        )
        # returns: '0x5c504ed432cb51138bcf09aa5e8a410dd4a1e204ef84bfed1be16dfba1b22060'
        ```
  - Encoding ABI
    - encodeABI(cls, fn_name, args=None, kwargs=None, data=None)
    - Classmethod (decorated with coerce_return_to_text)
    - Encodes function arguments for the contract function that matches the given name
    - If `data` is not provided, uses the function selector
    - Returns the encoded ABI payload
  - Event handling
    - on(self, event_name, filter_params=None, *callbacks)
    - Registers callbacks to be triggered on matching events
    - Supports filtering and formatting of event data
    - Uses helper utilities to locate the relevant event ABI, construct filters, and extract event data
    - Example usage (subscribe to a contract event):
      - 
        ```python
        def handle_transfer(log):
            print(log)

        my_contract = MyContract(address=address, abi=abi)
        my_contract.on('Transfer', filter_params={'fromBlock': 'latest'}, handle_transfer)
        ```

- Helper and utility functions (imported)
  - ABI utilities (e.g., get_abi_input_types, get_abi_output_types, function_abi_to_4byte_selector, etc.)
  - Event utilities (get_event_data, construct_event_filter_params, PastLogFilter)
  - Decorators (combomethod, coerce_return_to_text)
  - Web3 encoding/formatting helpers (encode_hex, add_0x_prefix, remove_0x_prefix, force_bytes, etc.)

Note: Several internal methods (e.g., _get_function_info, _encode_abi, _encode_constructor_data, _find_matching_event_abi) are used by the public API but are not shown in the excerpt. They are part of the overall contract proxy generation and ABI handling workflow.

## Usage

- Prerequisites
  - Initialize Web3 and build a contract proxy class (typically via a factory like construct_contract_factory) that derives from Contract.
  - Provide ABI, bytecode, and optional source information either at class level or per-instance.

- Creating a contract proxy instance
  - Provide address and/or ABI as needed:
    - instance = MyContract(abi=abi, address=address)

- Deploying a contract
  - Requires that the contract class has associated code
  - Example:
    - 
      ```python
      txn_hash = MyContract.deploy(
          transaction={
              'from': web3.eth.accounts[1],
              'value': 12345,
          },
          args=('DGD', 18),
      )
      ```

- Encoding function calls
  - Use encodeABI to obtain the payload for a function call
  - Example:
    - 
      ```python
      data = MyContract.encodeABI('transfer', args=(recipient, amount))
      ```

- Subscribing to events
  - Use on to register callbacks for contract events
  - Example:
    - 
      ```python
      def handle_transfer(log_entry):
          print(log_entry)

      my_contract = MyContract(address=address, abi=abi)
      my_contract.on('Transfer', filter_params={'fromBlock': 'latest'}, handle_transfer)
      ```

## Details

- Initialization requirements
  - The Contract class must be initialized through the Web3 contract interface (i.e., `Contract.web3` must be set). If not initialized, instantiating a Contract subclass will raise an AttributeError.
- Error handling
  - Deploy will raise ValueError if attempting to deploy a contract without associated code or if the deployment transaction contains unsupported fields like `data` or `to`.
- Accessor behavior
  - The properties `abi`, `code`, `code_runtime`, and `source` will raise AttributeError if their corresponding class-level attributes are not provided.
- Event filtering
  - Event handling relies on contract ABI matching, filter construction, and a Web3 log filter. A data formatter is applied via a log entry formatter to present event data in a usable form.
