# CockroachEngine

## Overview
CockroachEngine is an extension of PostgresEngine that enables CockroachDB support within the Piccolo framework. It identifies itself with the engine type "cockroach" and includes a Cockroach-specific startup step to configure cluster settings when possible. If the running user lacks permission to alter cluster settings, a warning is issued.

Key behaviors:
- Inherits from PostgresEngine
- Uses a lazy loader for asyncpg
- Attempts to enable an experimental column-type alteration setting at startup
- Emits a warning if the user does not have sufficient privileges

## Key Components

- CockroachEngine class
  - Inherits from PostgresEngine
  - engine_type = "cockroach"
  - min_version_number = 0 (Cockroach versioning is not strictly supported)
  - __init__(config, extensions, log_queries, log_responses, extra_nodes)
  - prep_database() to set up Cockroach-specific cluster settings

- Lazy loading of asyncpg
  - Uses LazyLoader to import asyncpg lazily:
    - asyncpg = LazyLoader("asyncpg", globals(), "asyncpg")

- Error handling during preparation
  - If the user lacks permission to set cluster options, a colored warning is emitted:
    - "=> Unable to set up Cockroach DB functionality may not behave as expected. Make sure your database user has permission to set cluster options."

Code reference from the module:

```python
from __future__ import annotations

import typing as t

from piccolo.utils.lazy_loader import LazyLoader
from piccolo.utils.warnings import Level, colored_warning

from .postgres import PostgresEngine

asyncpg = LazyLoader("asyncpg", globals(), "asyncpg")


class CockroachEngine(PostgresEngine):
    """
    An extension of
    :class:`PostgresEngine <piccolo.engine.postgres.PostgresEngine>`.
    """

    engine_type = "cockroach"
    min_version_number = 0  # Doesn't seem to work with cockroach versioning.

    def __init__(
        self,
        config: t.Dict[str, t.Any],
        extensions: t.Sequence[str] = (),
        log_queries: bool = False,
        log_responses: bool = False,
        extra_nodes: t.Dict[str, CockroachEngine] = None,
    ) -> None:
        super().__init__(
            config=config,
            extensions=extensions,
            log_queries=log_queries,
            log_responses=log_responses,
            extra_nodes=extra_nodes,
        )

    async def prep_database(self):
        try:
            await self._run_in_new_connection(
                "SET CLUSTER SETTING sql.defaults.experimental_alter_column_type.enabled = true;"  # noqa: E501
            )
        except asyncpg.exceptions.InsufficientPrivilegeError:
            colored_warning(
                "=> Unable to set up Cockroach DB "
                "functionality may not behave as expected. Make sure "
                "your database user has permission to set cluster options.",
                level=Level.medium,
            )
```

## Usage

- Instantiate a CockroachEngine instance (configuration will be passed to the underlying PostgresEngine setup):

```python
from piccolo.engine.cockroach_engine import CockroachEngine

engine = CockroachEngine(
    config={
        "database": "mydb",
        "user": "myuser",
        "password": "secret",
        "host": "localhost",
        "port": 26257,
    },
    log_queries=True
)
```

- Prepare the database (enables Cockroach-specific cluster setting if permitted):

```python
await engine.prep_database()
```

## Details
- This module defines CockroachEngine as an extension of PostgresEngine with specific behavior for CockroachDB.
- The min_version_number is set to 0 since Cockroach versioning is not reliably supported in this context.
- The prep_database method attempts to enable the experimental alter column type capability:
  - Command: SET CLUSTER SETTING sql.defaults.experimental_alter_column_type.enabled = true;
- If the current database user lacks privileges to modify cluster settings, a warning is issued via colored_warning, guiding the user to check permissions.
