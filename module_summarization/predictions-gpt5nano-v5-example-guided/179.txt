# Cross decomposition

## Overview
Cross decomposition in this module refers to extracting latent variables by decomposing the cross-covariance between X (predictors) and Y (targets). This is achieved through approaches based on the leading singular vectors of X'Y, either via full SVD or via a power-method iterative scheme. The cross-decomposition machinery underpins the PLS family (Canonical, Regression, and SVD-based variants).

Key helpers and components are implemented to:
- compute the first cross-covariance singular vectors,
- center and optionally scale X and Y,
- ensure consistent sign conventions after decomposition,
- and support compatibility with different SciPy versions for pseudo-inverses.

Public cross-decomposition estimators exposed by this module include PLSCanonical, PLSRegression, and PLSSVD.

## Key Components
- _get_first_singular_vectors_power_method(X, Y, mode="A", max_iter=500, tol=1e-06, norm_y_weights=False)
  - Returns the first left and right singular vectors of X'Y using a power iteration method.
  - Modes:
    - A: computes X-driven vectors
    - B: computes Y-driven vectors (with precomputed pseudo-inverses)
  - Supports iteration control via max_iter and tol; can normalize Y weights if requested.
  - If Y residual is constant, raises StopIteration.

- _get_first_singular_vectors_svd(X, Y)
  - Computes the first left and right singular vectors via the full SVD of C = X^T Y.
  - Returns U[:, 0] and Vt[0, :], i.e., the leading left and right singular vectors.

- _center_scale_xy(X, Y, scale=True)
  - Centers X and Y (subtracts their means) and, if scale is True, scales each to unit standard deviation.
  - Returns: (X, Y, x_mean, y_mean, x_std, y_std)

- _svd_flip_1d(u, v)
  - Helper to align the signs of 1D vectors to maintain a deterministic sign convention, analogous to svd_flip but operating in 1D in place.

- _PLS
  - Abstract base class implementing the common machinery for PLS variants (PLSCanonical, PLSRegression, PLSSVD).
  - Uses various mixins (TransformerMixin, RegressorMixin, etc.) and serves as the foundation for cross-decomposition estimators.

Note: The _get_first_singular_vectors_power_method docstring references Wegelin’s review and describes a correspondence to Section 11.3 when norm_y_weights is True in mode A.

## Usage
These helpers are used internally by the cross-decomposition estimators, but they illustrate the core steps of the cross-decomposition process.

- Center and (optionally) scale data:
  - Xc, Yc, x_mean, y_mean, x_std, y_std = _center_scale_xy(X, Y, scale=True)

- Compute the leading cross-covariance vectors:
  - u, v = _get_first_singular_vectors_svd(Xc, Yc)
  - or
  - u, v, n_iter = _get_first_singular_vectors_power_method(Xc, Yc, mode="A", max_iter=500, tol=1e-06, norm_y_weights=False)

- Ensure consistent sign:
  - _svd_flip_1d(u, v)

- In practice, these steps are embedded within the concrete PLS estimators (PLSCanonical, PLSRegression, PLSSVD) to construct the full cross-decomposition model.

## Details
- Cross-decomposition relies on extracting latent factors that maximize the cross-covariance between X and Y. The leading singular vectors of X^T Y provide those latent directions.
- The module provides two computational strategies:
  - Full SVD of C = X^T Y via _get_first_singular_vectors_svd.
  - Iterative power-method approach via _get_first_singular_vectors_power_method, which can be beneficial for large-scale problems or certain conditioning.
- Sign consistency is addressed with _svd_flip_1d to avoid arbitrary sign flips in the computed vectors.
- Centering and optional scaling is performed by _center_scale_xy to ensure that the variables contribute comparably to the decomposition.
- SciPy version compatibility for pseudo-inverses is handled:
  - If scipy version >= 1.7, pinv is used through a compatibility alias (pinv2); otherwise, pinv2 from SciPy is used directly, ensuring consistent behavior across versions.

Public API note:
- __all__ = ["PLSCanonical", "PLSRegression", "PLSSVD"]
- The concrete cross-decomposition estimators (PLSCanonical, PLSRegression, PLSSVD) expose the functionality to users, built upon the internal cross-decomposition machinery.
