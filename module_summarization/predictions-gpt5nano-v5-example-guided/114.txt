# simulation.rst — Visualization, summarization, and limit checking for Bluesky plans

## Overview
This module provides utilities to visualize raster plans, print concise summaries of plans, and verify that plans respect motor limits. It is designed to work with Bluesky Msg objects and relies on matplotlib for plotting.

Key capabilities:
- Plot the raster path implied by a plan using two motors.
- Produce a compact summary of a plan for quick inspection.
- Check that plan messages stay within device limits, issuing warnings when limits cannot be validated.

## Key Components

### plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)
Plot the raster path for a plan.

- plan: Iterable of Msg objects. Must yield Msgs and not be a coroutine.
- x_motor, y_motor: Names of the x and y motors (strings).
- ax: Optional matplotlib.axes.Axes to plot on; if None, a new figure/axes are created.
- probe_size: Optional float. If provided, renders each visited point as a circle with this radius (in the same units as motor positions).
- lw: Optional float. Line width for the path.

Behavior:
- Iterates messages, tracking cur_x and cur_y on 'set' commands and appending a point to the trajectory on 'save' commands.
- Draws the path between recorded points.
- If probe_size is provided, draws circle markers at each trajectory point; otherwise uses a simple scatter of the points.
- Returns a dict with keys 'path' (the Line2D object) and 'events' (the scatter/collection of read points).

Code excerpt (from the source):
```python
def plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2):
    import matplotlib.pyplot as plt
    from matplotlib import collections as mcollections
    from matplotlib import patches as mpatches
    if ax is None:
        ax = plt.subplots()[1]
    ax.set_aspect('equal')

    cur_x = cur_y = None
    traj = []
    for msg in plan:
        cmd = msg.command
        if cmd =='set':
            if msg.obj.name == x_motor:
                cur_x = msg.args[0]
            if msg.obj.name == y_motor:
                cur_y = msg.args[0]
        elif cmd =='save':
            traj.append((cur_x, cur_y))

    x, y = zip(*traj)
    path, = ax.plot(x, y, marker='', linestyle='-', lw=lw)
    ax.set_xlabel(x_motor)
    ax.set_ylabel(y_motor)
    if probe_size is None:
        read_points = ax.scatter(x, y, marker='o', lw=lw)
    else:
        circles = [mpatches.Circle((_x, _y), probe_size,
                                   facecolor='black', alpha=0.5)
                   for _x, _y in traj]

        read_points = mcollections.PatchCollection(circles,
                                                   match_original=True)
        ax.add_collection(read_points)
    return {'path': path, 'events': read_points}
```

### check_limits(plan)
Verify that a plan will not move devices beyond their limits.

- plan: Iterable of Msg objects. Must yield Msg objects.
- Behavior:
  - Iterates messages; for 'set' commands, if the destination object has a check_value method, it calls obj.check_value(value).
  - If an object lacks check_value, a warning is emitted and the object is ignored for subsequent checks.

Code excerpt (from the source):
```python
def check_limits(plan):
    """
    Check that a plan will not move devices outside of their limits.

    Parameters
    ----------
    plan : iterable
        Must yield `Msg` objects
    """
    ignore = []
    for msg in plan:
        if msg.command =='set' and msg.obj not in ignore:
            if hasattr(msg.obj, "check_value"):
                msg.obj.check_value(msg.args[0])
            else:
                warn(f"{msg.obj.name} has no check_value() method"
                     f" to check if {msg.args[0]} is within its limits.")
                ignore.append(msg.obj)
```

## Usage

- Visualize a plan raster path:
```python
from simulation import plot_raster_path
# plan should be an iterable of Bluesky Msg objects
plot_raster_path(plan, x_motor='x', y_motor='y')
```

- Print a concise summary of a plan:
```python
from simulation import print_summary
# plan should be an iterable of Bluesky Msg objects
print_summary(plan)
```

- Check that a plan respects device limits:
```python
from simulation import check_limits
# plan should be an iterable of Bluesky Msg objects
check_limits(plan)
```

## Details

- Dependencies: matplotlib, and Bluesky (for Msg structure and print_summary_wrapper).
- Expected plan structure: An iterable of Msg objects where:
  - 'set' messages indicate motor position updates (msg.obj.name matches the motor name).
  - 'save' messages denote a recorded event; the current (x, y) position is appended to the trajectory.
- plot_raster_path uses a simple mechanism to reconstruct the path:
  - Tracks current x and y coordinates as 'set' messages arrive.
  - Appends to traj on 'save' messages.
  - Produces a 2D path and either point markers or circular probes at visited points.
- check_limits provides a lightweight, non-destructive check, warning when an object lacks a check_value method, and suppressing further checks for that object.
