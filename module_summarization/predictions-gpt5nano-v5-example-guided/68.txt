Gaussian Process Kernels
========================

Overview
--------
This module provides kernels for Gaussian process regression and classification. The kernels support kernel-engineering, allowing you to:

- Combine kernels using addition (+), multiplication (*) or exponentiation (** by a scalar).
- Include scalar constants by automatically converting them to a constant kernel.

Key features:
- Analytic gradient-based hyperparameter optimization for all kernels.
- Hyperparameter spaces defined by lower and upper bounds (a rectangular search space).
- Hyperparameters can be declared as "fixed" to exclude them from optimization.
- Anisotropic length-scales are supported; validation is performed to ensure consistent dimensionality.
- Kernels can be combined into complex expressions, enabling flexible kernel design.

The module is inspired by the kernel component of the george package.

Key Components
--------------

Hyperparameter
------------
A kernel hyperparameter is specified by the Hyperparameter namedtuple:

- name: str
  The name of the hyperparameter (e.g., constant_value). The kernel using this hyperparameter should expose an attribute with this name and a corresponding ..._bounds attribute.

- value_type: str
  Type of the hyperparameter (currently supports "numeric").

- bounds: array-like or "fixed"
  The lower and upper bounds for the parameter. If n_elements > 1, a pair of 1D arrays with n_elements each may be provided. If "fixed" is used, the hyperparameter cannot be changed.

- n_elements: int (default=1)
  Number of elements of the hyperparameter value. 1 means scalar; >1 corresponds to vector-valued parameters (e.g., anisotropic length-scales).

- fixed: bool (default=None)
  Whether the value is fixed (excluded from optimization). If None, it is derived from the bounds.

Example (from the Hyperparameter docstring):
.. code-block:: python

   >>> from sklearn.gaussian_process.kernels import Hyperparameter
   >>> # Example hyperparameter definition (illustrative)
   >>> hyper = Hyperparameter(name='constant_value', value_type='numeric',
   ...                        bounds=[[0.0, 10.0]], n_elements=1, fixed=False)

The Hyperparameter class includes a stable __eq__ method to compare hyperparameters.

_Usage example (documented in the Hyperparameter docstring)_
.. code-block:: python

   >>> from sklearn.gaussian_process.kernels import ConstantKernel
   >>> from sklearn.datasets import make_friedman2
   >>> from sklearn.gaussian_process import GaussianProcessRegressor
   >>> from sklearn.gaussian_process.kernels import Hyperparameter
   >>> X, y = make_friedman2(n_samples=50, noise=0, random_state=0)
   >>> kernel = ConstantKernel(constant_value=1.0,
   ...    constant_value_bounds=(0.0, 10.0))

   >>> for hyperparameter in kernel.hyperparameters:
   ...    print(hyperparameter)
   Hyperparameter(name='constant_value', value_type='numeric',
   bounds=array([[ 0., 10.]]), n_elements=1, fixed=False)

   >>> params = kernel.get_params()
   >>> for key in sorted(params): print(f"{key} : {params[key]}")
   constant_value : 1.0
   constant_value_bounds : (0.0, 10.0)

_length scale validation
-----------------------
The module includes a helper function to validate anisotropic length-scales:

.. code-block:: python

   length_scale = _check_length_scale(X, length_scale)

.. code-block:: python

   def _check_length_scale(X, length_scale):
       length_scale = np.squeeze(length_scale).astype(float)
       if np.ndim(length_scale) > 1:
           raise ValueError("length_scale cannot be of dimension greater than 1")
       if np.ndim(length_scale) == 1 and X.shape[1] != length_scale.shape[0]:
           raise ValueError(
               "Anisotropic kernel must have the same number of "
               "dimensions as data (%d!=%d)" % (length_scale.shape[0], X.shape[1])
           )
       return length_scale

Kernel base class
-----------------
The module defines a base abstract class Kernel that serves as the interface for all kernels. It provides shared utility for parameter management, including a get_params method that introspects constructor arguments to expose kernel parameters in a scikit-learn style.

- Kernel
  - get_params(deep=True): Retrieve parameter names and values for the kernel, enabling standard parametrization, cloning, and hyperparameter management.

Usage
-----
- Kernel construction and combination:
  - Kernels can be created (e.g., ConstantKernel) and combined using addition, multiplication, or exponentiation to form complex kernels.
  - Scalar values can be included directly and are automatically converted to constant kernels.

- Hyperparameter optimization:
  - Each kernel exposes hyperparameters with bounds or a fixed flag.
  - Hyperparameters are optimized analytically (where possible) using gradient-based methods.
  - The search space for hyperparameters is rectangular, defined by bounds, with the option to fix certain parameters.

- Anisotropic length-scales:
  - Length scales can be scalar or a length-scale vector, validated by _check_length_scale.

Notes
-----
- The module supports kernel-engineering by allowing kernels to be combined and exponentiated.
- Hyperparameters can be bounded or fixed, enabling flexible hyperparameter optimization strategies.
- The implementation is designed to be compatible with analytic gradients for hyperparameter optimization.
- The module is strongly inspired by the george kernel package.
