# Encoding media

## Overview
This module provides support for encoding media for transmission to an ftrack server. It focuses on the authentication and session-management aspects required to encode and upload media assets securely and efficiently. The key components expose how requests are authenticated and how a session is established and configured to interact with the server.

The primary building blocks are:
- SessionAuthentication: attaches ftrack authentication headers to outgoing HTTP requests.
- Session: central entry point for interacting with an ftrack server, handling server URL, API key, API user, caching, plugin loading, and event hub behavior.

## Key Components

- SessionAuthentication
  - Purpose: Attach ftrack session authentication information to requests.
  - How it works: Injects the following HTTP headers into each request:
    - ftrack-api-key
    - ftrack-user
  - Source excerpt:
    - __call__ implementation that updates request headers and returns the request.

  Code example:
  - class SessionAuthentication(requests.auth.AuthBase):
      '''Attach ftrack session authentication information to requests.'''

    def __init__(self, api_key, api_user):
        '''Initialise with *api_key* and *api_user*.'''
        self.api_key = api_key
        self.api_user = api_user
        super(SessionAuthentication, self).__init__()

    def __call__(self, request):
        '''Modify *request* to have appropriate headers.'''
        request.headers.update({
            'ftrack-api-key': self.api_key,
            'ftrack-user': self.api_user
        })
        return request

- Session
  - Purpose: Manage an isolated session for interaction with an ftrack server.
  - Responsibilities:
    - Validate and store server_url, api_key, and api_user (with environment-variable fallbacks).
    - Provide access to pending operations, caching, and plugin configuration.
    - Optionally enable an embedded event hub and schema caching.
  - Notable configuration options (from the constructor docstring):
    - server_url: URL of the ftrack server (required; can fall back to FTRACK_SERVER env var).
    - api_key: API key for authentication (required; can fall back to FTRACK_API_KEY or FTRACK_APIKEY env vars).
    - api_user: API user name (required; can fall back to FTRACK_API_USER env var or system user).
    - auto_populate: If True, automatically fetch entity attributes on access.
    - plugin_paths: Paths to search for plugins.
    - cache: Cache instance or callable to configure caching.
    - cache_key_maker: Key maker for cache entries (defaults to StringKeyMaker).
    - auto_connect_event_hub: If True, connect the embedded event hub automatically.
    - schema_cache_path: Directory for schema cache; can be disabled or read from FTRACK_API_SCHEMA_CACHE_PATH.
    - plugin_arguments: Optional mapping of arguments provided to plugin registration.
  - Source excerpt (class header and constructor docstring):
    - class Session(object):
        '''An isolated session for interaction with an ftrack server.'''
      and a long docstring describing the parameters and behavior.

## Usage

- Basic usage: create a session with required credentials and server URL.
  - Example:
    - from ftrack_api import Session
      session = Session(
          server_url="https://your-ftrack-server",
          api_key="YOUR_API_KEY",
          api_user="YOUR_USERNAME"
      )

- Environment-based configuration (alternative to explicit arguments):
  - server_url may be read from FTRACK_SERVER.
  - api_key may be read from FTRACK_API_KEY (or FTRACK_APIKEY for backward compatibility).
  - api_user may be read from FTRACK_API_USER or inferred from the current user if possible.

- Example: attaching authentication to requests
  - import requests
  - auth = SessionAuthentication(api_key="YOUR_API_KEY", api_user="YOUR_USERNAME")
  - response = requests.get("https://your-ftrack-server/path", auth=auth)

- Advanced usage: customize caching, plugins, and event hub behavior
  - You can provide a custom cache implementation, a cache key maker, plugin-search paths, and plugin arguments to tailor how media encoding and related operations are performed.

## Details

- Authentication headers:
  - The module provides a dedicated SessionAuthentication class that automatically injects the necessary ftrack headers into each HTTP request:
    - ftrack-api-key
    - ftrack-user

- Session lifecycle and configuration:
  - The Session class encapsulates the server connection details and user identity, ensuring that operations related to encoding and media upload are performed within a properly authenticated session.
  - It supports:
    - Automatic environment variable fallback for required credentials.
    - An optional embedded event hub, which can publish/subscribe to events beyond the local process.
    - Schema caching to improve startup performance when encoding media assets with related metadata.
    - A layered caching strategy with a top-level MemoryCache by default.
    - Plugin discovery and argument passing to plugins at discovery time.

- Environment variables (convenience):
  - FTRACK_SERVER: server URL
  - FTRACK_API_KEY or FTRACK_APIKEY: API key
  - FTRACK_API_USER: API user

- Important notes:
  - If required parameters are not supplied and corresponding environment variables are missing, a TypeError is raised.
  - The event hub connection is performed asynchronously to improve startup time; plugins should verify connection status if they rely on the hub.
  - If schema caching is enabled, provide a valid directory path via schema_cache_path or the FTRACK_API_SCHEMA_CACHE_PATH environment variable; set to False to disable.

- Relevant behaviors from the source:
  - The session tracks pending operations via recorded_operations and maintains a flag to enable or disable recording of operations (self.record_operations).
  - A default cache strategy is applied, with self.cache_key_maker defaulting to a StringKeyMaker if none is provided.
