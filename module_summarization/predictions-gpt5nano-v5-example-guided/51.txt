## Overview
This module defines the base class for one-parameter exponential family distributions used by GLM. It provides the core interface and utilities (such as IRLS weights, starting values for the mean, deviance computations, and fitted values) that concrete GLM families extend. The class interacts with link functions and variance functions to model the mean-variance relationship and the link transformation.

Key methods include starting_mu, weights, deviance, resid_dev, and fitted. The class is designed to be subclassed by specific one-parameter exponential family distributions (e.g., Poisson, Binomial) which implement distribution-specific details (notably _resid_dev).

Notes:
- The link and variance objects shape how mu (the mean) is transformed and how its variance behaves.
- When constructing a Family, pass an instance of a Link subclass and a variance function instance.
- If a link class (not an instance) is provided, a DeprecationWarning is raised and an instance is created internally.

Refer to the source for the exact method behavior and formulas.

## Key Components
- Family class
  - link: property that sets/gets the link function for the family.
  - variance: variance function associated with the family.
  - starting_mu(y): starting value for mu in the IRLS algorithm.
  - weights(mu): IRLS weights, computed as 1 / (g'(mu)^2 * Var(mu)).
  - deviance(endog, mu, var_weights=1., freq_weights=1., scale=1.): deviance given data and means.
  - resid_dev(endog, mu, var_weights=1., scale=1.): deviance residuals.
  - fitted(lin_pred): fitted values corresponding to linear predictors.
- Link and Variance integration
  - The methods rely on self.link.deriv(mu) and self.variance(mu) to compute quantities.
  - See the Links and VarFuncs modules for concrete implementations.

Key excerpts from the source:
- starting_mu:
  mu_0 = (Y + overline{Y}) / 2
  Note: Only the Binomial family may use a different initialization.
- weights:
  w = 1 / (g'(mu)^2 * Var(mu))
- deviance:
  D = 2 * sum_i freq_weights_i * var_weights * (llf(endog_i, endog_i) - llf(endog_i, mu_i))
  Internally, the implementation uses:
  D = sum_i freq_weights_i * var_weights * resid_dev_i / scale
- resid_dev:
  resid_dev_i = sign(y_i - mu_i) * sqrt(D_i)
  where D_i is computed by the _resid_dev method of the concrete family.

Deprecation note:
- If you pass a link class rather than an instantiated link, a DeprecationWarning is issued. Example (deprecated):
  self.link = link()
  Use an instantiated link object instead.

## Details

- The Family class acts as a base class for one-parameter exponential families used in GLM contexts. It delegates distribution-specific calculations (such as the actual resid_dev computation) to concrete subclasses via the _resid_dev method.
- The link attribute must be an instance of a Link subclass. Passing a class rather than an instance emits a DeprecationWarning and still works by constructing an instance, but you should pass an instance directly.
- The starting_mu method provides a warm start for iterative reweighted least squares (IRLS) algorithms. The default is:
  mu_0 = (Y + average(Y)) / 2
- The weights method computes IRLS weights as:
  w = 1 / (g'(mu)^2 * Var(mu))
  where g is the link function and Var is the variance function.
- The deviance and resid_dev methods provide distribution-specific deviation measures and residuals, using the per-observation contributions defined by _resid_dev (to be implemented by concrete families).
