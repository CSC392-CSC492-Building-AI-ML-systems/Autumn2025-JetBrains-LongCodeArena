# Tuning the hyper-parameters of an estimator

## Overview
Grid search is a systematic approach to hyper-parameter optimization. It exhaustively evaluates all combinations of specified hyper-parameter values to identify the configuration that yields the best performance according to a chosen scoring metric, typically using cross-validation. It is commonly used to improve model performance by selecting the best parameters for an estimator or a pipeline.

Key concepts:
- Estimator: the model whose hyper-parameters you want to tune.
- Parameter grid: the set of hyper-parameter values to evaluate.
- Cross-validation: splitting data into training/validation folds to estimate performance.
- Scoring: the metric used to compare parameter configurations.
- Refitting: optionally refitting the best parameters on the full training data.

## Key Components
- GridSearchCV: performs an exhaustive search over a parameter grid with cross-validation.
- Parameter grid: a dictionary (or list of dictionaries) mapping estimator parameters to lists of values.
- Estimator: the model or pipeline whose hyper-parameters are tuned.
- Scoring: a string (e.g., "accuracy", "f1") or a callable specifying how performance is measured. Can also be a dict for multi-metric scoring.
- CV: cross-validation splitter or integer specifying the number of folds.
- Refitting: whether to refit the best parameters on the full dataset after search; can be a metric name when using multi-metric scoring.
- n_jobs: number of parallel jobs to run; enables faster searches on multi-core machines.

## Usage
- Define the estimator and the parameter grid, then create and run GridSearchCV.
- Access the best parameters and score, and inspect cross-validation results.

Example 1: Tuning a RandomForest classifier
```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV

param_grid = {
    'n_estimators': [100, 200],
    'max_depth': [None, 10, 20],
    'min_samples_split': [2, 5],
}

grid_search = GridSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_grid=param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1
)

grid_search.fit(X, y)

print("Best parameters:", grid_search.best_params_)
print("Best cross-validation accuracy:", grid_search.best_score_)
```

Example 2: Using a Pipeline
```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', LogisticRegression(max_iter=1000)),
])

param_grid = {
    'clf__C': [0.1, 1.0, 10.0],
    'clf__penalty': ['l2'],
}

grid_search = GridSearchCV(pipe, param_grid, cv=5, scoring='roc_auc', n_jobs=-1)
grid_search.fit(X_train, y_train)

print("Best parameters:", grid_search.best_params_)
print("Best ROC AUC:", grid_search.best_score_)
```

What you can obtain from GridSearchCV:
- best_params_: the parameter setting that produced the best score.
- best_score_: the best mean cross-validation score achieved.
- best_estimator_: the estimator configured with the best parameters.
- cv_results_: a detailed record of the scores and parameters for all tested configurations.
- refit_: whether the best parameters are used to fit on the full training data after search (default is True).

Accessing results:
```python
results = grid_search.cv_results_
# Example: show mean test scores for each parameter combination
import pandas as pd
results_df = pd.DataFrame(grid_search.cv_results_)
results_df.sort_values(by='mean_test_score', ascending=False).head()
```

Advanced usage:
- Multi-metric scoring: provide scoring as a dict of metrics and choose the metric for refitting with the refit parameter.
- Parameter grids: param_grid can be a list of dictionaries to explore different hyper-parameter combinations or conditional grids.
- Pipeline parameters: use the double-underscore syntax to reference parameters of steps, e.g., 'stepname__param'.

## Details
- Parameter grid specification:
  - param_grid can be a dict or a list of dicts. Each dict forms a separate grid to search.
  - For pipelines, prefix parameter names with the step name followed by two underscores, e.g., 'clf__C' for a parameter C in the estimator named 'clf'.
- Grid size considerations:
  - The total number of configurations is the product of the lengths of all parameter value lists. Large grids may be computationally expensive.
  - To manage cost, you can reduce grid size, use RandomizedSearchCV as an alternative, or set a limit on n_jobs and verbose.
- Scoring and refitting:
  - If scoring is None, the estimator’s default score method is used.
  - Multi-metric scoring allows evaluating several metrics simultaneously. When using multi-metric scoring, refit can be set to a metric name to optimize that metric for selecting the best estimator.
- Practical tips:
  - Use a reproducible seed for estimators that rely on randomness.
  - Start with a coarse grid to identify promising regions, then refine with a finer grid.
  - Use cross-validation with a reasonable number of folds to balance bias and variance in the score estimates.
  - Consider standardizing data preprocessing steps within a Pipeline to ensure consistent evaluation across configurations.

Note: The examples above illustrate typical usage of grid search in model tuning. Adjust estimator types, parameters, and scoring to your specific problem and data.
