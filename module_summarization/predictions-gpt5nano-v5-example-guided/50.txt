## Overview
This module provides spline-based smoothing utilities for Generalized Additive Models (GAMs). It focuses on constructing and evaluating B-spline bases, knot placement, and related derivatives. Core functionality includes generating knot vectors, building spline bases (and their derivatives), and obtaining ready-to-use basis matrices for integration into GAM design matrices or GLMGam models.

Key functions include:
- _equally_spaced_knots(x, df)
- _R_compat_quantile(x, probs)
- _eval_bspline_basis(x, knots, degree, deriv='all', include_intercept=True)
- compute_all_knots(x, df, degree)
- make_bsplines_basis(x, df, degree)
- get_knots_bsplines(x=None, df=None, knots=None, degree=3, spacing='quantile', lower_bound=None, upper_bound=None, all_knots=None)

These utilities enable users to generate B-spline bases using either quantile-based knot placement (as commonly used in Patsy-GAM) or equal-spaced knots (as used in P-Splines), and to evaluate the resulting basis and its derivatives at the data points.

## Key Components
- _equally_spaced_knots(x, df)
  - Returns a set of n_knots equally spaced between the minimum and maximum of x.
  - n_knots = df - 2

- _R_compat_quantile(x, probs)
  - Computes quantiles of x corresponding to the given probabilities, emulating R-style quantile behavior.

- _eval_bspline_basis(x, knots, degree, deriv='all', include_intercept=True)
  - Computes B-spline basis (and optionally its first and second derivatives) for input values x given knots and spline degree.
  - Requires SciPy (scipy.interpolate.splev). Raises NotImplementedError if any x lies outside the knot range.

- compute_all_knots(x, df, degree)
  - Determines the full knot vector for B-splines based on data x, desired degrees of freedom df, and polynomial degree.
  - Returns all_knots, lower_bound, upper_bound, inner_knots.

- make_bsplines_basis(x, df, degree)
  - Convenience wrapper to generate the basis and its derivatives for x using the computed knots.

- get_knots_bsplines(x=None, df=None, knots=None, degree=3, spacing='quantile', lower_bound=None, upper_bound=None, all_knots=None)
  - Primary entry point to obtain knot vectors for B-splines.
  - spacing can be 'quantile' (default) or 'equal' for knot placement.
  - Supports passing precomputed all_knots or x/df/degree to compute knots.

## Usage

End-to-end example: generate a B-spline basis for a sample dataset and inspect its shape.

```python
import numpy as np
# Example data
x = np.linspace(0, 1, 100)
df = 8       # degrees of freedom (influences the number of basis functions)
degree = 3   # cubic splines

# Build B-spline basis and derivatives
basis, der_basis, der2_basis = make_bsplines_basis(x, df, degree)

print("Basis shape:", basis.shape)        # (100, n_bases)
print("First derivative shape:", der_basis.shape)
print("Second derivative shape:", der2_basis.shape)
```

Example: obtain knot vector using quantile spacing and then build the basis

```python
# Get knots for quantile-based spacing
knots = get_knots_bsplines(x=x, df=df, degree=degree, spacing='quantile')
print("Number of knots:", len(knots))

# Alternatively, compute all knots and then evaluate
basis2, der2, der2_b = _eval_bspline_basis(x, knots, degree)
print("Basis2 shape:", basis2.shape)
```

Example: using equal spacing (P-Splines style) for knots

```python
knots_equal = get_knots_bsplines(x=x, df=df, degree=degree, spacing='equal',
                                lower_bound=x.min(), upper_bound=x.max())
basis_equal, der_equal, der2_equal = _eval_bspline_basis(x, knots_equal, degree)
print("Basis (equal spacing) shape:", basis_equal.shape)
```

Integrating with GLMGam or a GAM design:
- The produced basis matrices (basis, der_basis, etc.) can be supplied as predictor terms in the model design.
- For example, you can add basis as a term in the exog design matrix or combine with Patsy-style formulas if you wrap the basis into a DataFrame.

Notes:
- The implementation relies on SciPy and Patsy; ensure SciPy is installed for spline evaluation via splev.
- _eval_bspline_basis enforces that all x values lie within the knot range; data outside outermost knots raises NotImplementedError.
- The include_intercept flag controls whether a constant term is included in the basis (affects the number of basis functions).
