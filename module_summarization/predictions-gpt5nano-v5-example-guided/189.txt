# Kernel Ridge Regression

## Overview
Kernel ridge regression (KRR) combines ridge regression (linear least squares with L2 regularization) with the kernel trick. It learns a linear function in the feature space induced by a chosen kernel, which corresponds to a nonlinear function in the original input space for non-linear kernels. The KRR model has the same functional form as support vector regression (SVR) but uses squared-error loss with L2 regularization, enabling a closed-form solution that is typically faster for medium-sized datasets. Unlike SVR, the learned model in KRR is not sparse, so predictions can be slower for very large datasets.

This estimator supports multi-target regression (y can be a 2D array with shape [n_samples, n_targets]).

See the User Guide for Kernel Ridge Regression for more details.

## Key Components

- KernelRidge class
  - Implements multi-output regression with kernelized linear models.

- Parameters
  - alpha: float or array-like of shape (n_targets,), default=1.0
    Regularization strength. Must be positive. If an array is provided, each target gets its own penalty.
  - kernel: str or callable, default="linear"
    Kernel to use. Passed to pairwise_kernels unless kernel is "precomputed".
    If a string, it must be one of the kernels in pairwise.PAIRWISE_KERNEL_FUNCTIONS or "precomputed".
    If "precomputed", X is treated as a precomputed kernel matrix. If callable, it is invoked on each pair of samples; callables from sklearn.metrics.pairwise are not allowed.
  - gamma: float, default=None
    Gamma parameter for certain kernels (e.g., RBF, polynomial, sigmoid). Ignored by other kernels.
  - degree: int, default=3
    Degree for polynomial kernel. Ignored by other kernels.
  - coef0: float, default=1
    Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.
  - kernel_params: dict, default=None
    Additional parameters for the kernel function when kernel is a callable.

- Attributes
  - dual_coef_: ndarray of shape (n_samples,) or (n_samples, n_targets)
    Representation of weight vectors in kernel space.
  - X_fit_: {ndarray, sparse matrix} of shape (n_samples, n_features)
    Training data used for fitting. If kernel == "precomputed", this stores the precomputed training kernel matrix of shape (n_samples, n_samples).
  - n_features_in_: int
    Number of features seen during fit.
    Version added: 0.24
  - feature_names_in_: ndarray of shape (n_features_in_,)
    Names of features seen during fit (when X has string feature names).
    Version added: 1.0

- Internal helpers and constraints
  - _get_kernel(X, Y=None)
    Builds the kernel matrix. If kernel is callable, uses kernel_params; otherwise uses gamma, degree, and coef0 for parameterization.
    Implementation (from source):
    def _get_kernel(self, X, Y=None):
        if callable(self.kernel):
            params = self.kernel_params or {}
        else:
            params = {"gamma": self.gamma, "degree": self.degree, "coef0": self.coef0}
        return pairwise_kernels(X, Y, metric=self.kernel, filter_params=True, **params)

  - _parameter_constraints
    Validates parameter types and ranges, including:
    - alpha: positive real or array-like
    - kernel: one of the supported pairwise kernel functions or "precomputed" or a callable
    - gamma, degree, coef0: appropriate types and bounds
    - kernel_params: dict or None

- Examples
  The following example demonstrates fitting a KernelRidge model:
  ```python
  from sklearn.kernel_ridge import KernelRidge
  import numpy as np

  n_samples, n_features = 10, 5
  rng = np.random.RandomState(0)
  y = rng.randn(n_samples)
  X = rng.randn(n_samples, n_features)

  krr = KernelRidge(alpha=1.0)
  krr.fit(X, y)
  KernelRidge(alpha=1.0)
  ```

## Usage

- Instantiate the model with desired hyperparameters.
- Fit with training data X and target y (1D for single target or 2D for multiple targets).
- Predict on new samples via the standard fit/predict workflow (predict uses the learned dual coefficients in kernel space).

Key usage notes:
- If kernel is "precomputed", provide a kernel matrix as X (training data) and use the corresponding test kernel with Y accordingly.
- If kernel is callable, it should operate on two samples (rows) and return a single kernel value. Callables from pairwise metrics are not allowed in this path.
- The alpha parameter corresponds to 1/(2C) in other linear models; larger alpha yields stronger regularization.

## Details

Kernel ridge regression learns a function in the feature space induced by the chosen kernel, solving a closed-form system via the dual representation. This makes KRR particularly convenient for medium-sized datasets where a fast, non-sparse solution is desirable. Compared to SVR, KRR uses squared error loss and yields a non-sparse model, which can be less efficient for very large datasets but often simpler to tune.

The implementation supports multi-target regression by allowing y to be a 2D array, enabling simultaneous learning of multiple outputs. Regularization is controlled by alpha (and its per-target variant if provided as an array).

When using a non-precomputed kernel, the internal kernel matrix is constructed via pairwise_kernels with the appropriate kernel parameters (gamma, degree, coef0) or via the provided callable. The internal helper _get_kernel encapsulates this logic.
