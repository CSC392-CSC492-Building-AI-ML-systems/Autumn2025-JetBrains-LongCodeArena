# Kernel Ridge Regression (sklearn.kernel_ridge)

## Overview
Kernel Ridge Regression (KRR) combines ridge regression (linear least squares with L2 regularization) with the kernel trick. It learns a linear function in the feature space induced by the selected kernel, allowing non-linear relationships in the original data. KRR is the closed-form counterpart to SVR with a squared loss, typically faster for medium-sized datasets, though the resulting model is not sparse.

This estimator supports multi-target regression (y can be a 2D array of shape [n_samples, n_targets]).

See also the user guide entry on kernel ridge regression for more details.

## Key Components
- Class: KernelRidge
- Inheritance: MultiOutputMixin, RegressorMixin, BaseEstimator
- Core ideas:
  - Builds a dual representation in kernel space and solves the dual system in closed form
  - Uses the kernel matrix on the training data (or a precomputed kernel matrix)
  - Supports multiple targets and a variety of kernels

- Parameters
  - alpha: float or array-like of shape (n_targets,), default=1.0
    Regularization strength; must be positive. Larger values increase regularization.
    If an array is provided, penalties apply per target.
    Note: alpha corresponds to 1 / (2C) in some linear models.
  - kernel: str or callable, default="linear"
    Kernel mapping used internally. Passed to pairwise_kernels unless kernel is a callable.
    If a string, must be one of the functions in pairwise.PAIRWISE_KERNEL_FUNCTIONS or "precomputed".
    If "precomputed", X is assumed to be a kernel matrix.
    If a callable is provided, it is called on pairs of samples. Callables from
    sklearn.metrics.pairwise are not allowed (they operate on matrices, not on individual samples).
  - gamma: float, default=None
    Gamma parameter for certain kernels (e.g., RBF, laplacian, polynomial, exponential chi2, sigmoid). Ignored by kernels that do not use it.
  - degree: int, default=3
    Degree for polynomial kernel. Ignored by other kernels.
  - coef0: float, default=1
    Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.
  - kernel_params: dict, default=None
    Additional keyword arguments for the kernel function when kernel is a callable.

- Internal methods
  - _get_kernel(X, Y=None): Builds or returns the kernel matrix for X (and optionally Y). If kernel is callable, uses provided kernel_params; otherwise, uses gamma, degree, and coef0.

- Attributes
  - dual_coef_: ndarray of shape (n_samples,) or (n_samples, n_targets)
    Representation of the weight vector(s) in kernel space.
  - X_fit_: Training data (or the precomputed kernel matrix if kernel="precomputed").
    Shape is (n_samples, n_features) or (n_samples, n_samples) for precomputed.
  - n_features_in_: int
    Number of features observed during fit.
  - feature_names_in_: ndarray of shape (n_features_in_,)
    Names of features observed during fit (present if X has feature names).

- Validation
  - Validates input via internal utilities; supports per-target regularization and kernel parameter validation.

- See Also
  - GaussianProcessRegressor, Ridge, RidgeCV, SVR

## Usage
Basic example (default linear kernel):
```python
from sklearn.kernel_ridge import KernelRidge
import numpy as np

n_samples, n_features = 10, 5
rng = np.random.RandomState(0)
y = rng.randn(n_samples)
X = rng.randn(n_samples, n_features)

krr = KernelRidge(alpha=1.0)
krr.fit(X, y)
print(krr)
```

Example with a non-linear kernel (e.g., RBF):
```python
krr = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.5)
krr.fit(X, y)
```

Example with precomputed kernel:
```python
# Compute kernel matrix K of shape (n_samples, n_samples)
K = pairwise_kernels(X, X, metric="rbf", gamma=0.5)
krr = KernelRidge(alpha=1.0, kernel="precomputed")
krr.fit(K, y)
```

## Details
- KernelRidge learns a linear function in the feature space induced by the chosen kernel and regularizes with L2 (ridge) to control variance.
- The model is computed in closed form via solving a system that uses the kernel matrix; thus, fitting is generally faster than iterative solvers for moderate datasets.
- The learned representation is typically non-sparse, unlike some SVR formulations that promote sparsity.
- Multi-target support allows simultaneous regression across several output dimensions.
- Internal kernel handling relies on pairwise_kernels for standard kernels or a user-supplied callable; if kernel="precomputed", the input X is treated as the kernel matrix.
- The class uses _solve_cholesky_kernel to solve the dual system and _get_kernel to obtain kernel evaluations as needed.
