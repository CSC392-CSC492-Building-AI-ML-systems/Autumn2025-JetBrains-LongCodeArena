# Using lists

## Overview
Python lists are the primary, ordered, mutable container type for collecting items. In the context of working with the ftrack API, lists are commonly used to gather entity IDs, accumulate results from queries, batch-process items, or compose configuration values. This guide covers standard list usage and patterns that are commonly helpful when interacting with ftrack-related workflows.

## Key Components
- Creating and modifying lists
  - List literals and the list constructor
  - Mutation methods: append, extend, insert, remove, pop
- Accessing and slicing
  - Indexing with positive and negative indices
  - Slicing to obtain sublists
- Iteration and comprehensions
  - List comprehensions for concise transformations
  - When to prefer generators for memory efficiency
- Copying and mutability
  - Shallow vs. deep copies
- Common patterns
  - Concatenation and merging
  - Batch processing through slicing
  - Converting between generators and lists when needed

## Usage

Examples are general Python patterns that are commonly useful when handling lists in ftrack API workflows.

- Create and transform a list from objects
  ```python
  # Suppose 'entities' is an iterable of objects with an 'id' attribute
  entities = [...]  # provided by your code
  ids = [e.id for e in entities]  # extract IDs into a list
  ```

- Batch processing with slicing
  ```python
  batch_size = 100
  for i in range(0, len(ids), batch_size):
      batch = ids[i:i + batch_size]
      process_batch(batch)  # your processing function
  ```

- Concatenate lists
  ```python
  a = [1, 2, 3]
  b = [4, 5]
  combined = a + b  # [1, 2, 3, 4, 5]
  ```

- Copying lists
  ```python
  import copy

  shallow_copy = a.copy()        # shallow copy
  deep_copy = copy.deepcopy(a)   # deep copy (if elements are mutable)
  ```

- Using comprehensions for transformation
  ```python
  # Transform a list of entities to their names
  names = [e.name for e in entities]
  ```

- Converting a generator to a list
  ```python
  # If you have a generator yielding items
  def generate_items():
      yield from range(10)

  items = list(generate_items())  # materialize as a list when needed
  ```

## Details

- Lists are mutable and order-preserving. Use them when you need to maintain sequence and allow in-place modifications.
- Be mindful of memory usage with large datasets. Prefer generator expressions where possible and only materialize lists when necessary (e.g., when you need to index or repeatedly access items).
- When collecting results from operations (e.g., API calls or query results), consider batching via slicing to avoid large single operations.
- If you require fixed-size, immutable sequences, consider using tuples instead of lists.
