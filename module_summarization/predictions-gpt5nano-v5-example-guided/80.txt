# MapPlotter

## Overview
MapPlotter is a utility class for plotting a spectrum map from a data cube with interactive capabilities. It creates a 2D plane from a cube, displays it (optionally with publication-quality rendering via APLpy), and provides interactive features to inspect and overplot spectra, as well as to average over circular regions. It is designed as an interactive component of a larger cube-analysis package rather than a standalone publication plotter.

The class integrates with Astropy WCS headers and can fall back to Matplotlib if APLpy is unavailable. It uses a header flattening function from the cubes module and can construct a WCS object when available.

Key dependencies (optional): APLpy, astropy.wcs, astropy.io.fits (or pyfits as fallback)

## Key Components

- MapPlotter class
  - Purpose: Plot a spectrum map from a cube and provide interactive analysis controls.
  - Docstring: 
    - The class docstring states: “Class to plot a spectrum. See `mapplot` for use documentation; this docstring is only for initialization.”
  - Core attributes (initialized in __init__):
    - Cube: the input cube data
    - header: flattened FITS header obtained from cubes.flatten_header(Cube.header, delete=True)
    - wcs: WCS object created from header (if pywcs is available)
    - figure, axis: Matplotlib figure/axes or APLpy FITSFigure constituents
    - FITSFigure: APLpy FITSFigure instance if APLpy is available
    - _click_marks, _circles, _clickX, _clickY: interactive state for clicks and drawn markers
    - overplot_colorcycle, overplot_linestyle: styling for overplotted spectra
    - plane: 2D map extracted from the cube (created by makeplane)
  - Key methods:
    - __init__(self, Cube=None, figure=None, doplot=False, **kwargs)
      - Initializes plotting resources, prepares header and WCS, and optionally plots immediately if doplot is True.
    - __call__(self, **kwargs)
      - Convenience alias to mapplot(**kwargs).
    - mapplot(self, convention='calabretta', colorbar=True, useaplpy=True,
                vmin=None, vmax=None, cmap=None, plotkwargs={}, **kwargs)
      - Main plotting routine. Creates or resets the figure, builds the plane via makeplane, and renders the map.
      - Uses APLpy when available and requested; otherwise uses Matplotlib imshow.
      - Determines vmin/vmax automatically if not provided.
      - Supports an optional colorbar and passes plotting kwargs to the backend.
      - Removes estimator from kwargs if present to avoid interference with plotting.
      - Connects interactive event handlers via _connect().
    - _connect(self)
      - Connects mouse and keyboard events to interactive handlers:
        - button_press_event -> click
        - button_release_event -> plot_spectrum
        - key_press_event -> plot_spectrum
    - _disconnect(self)
      - Disconnects the interactive event handlers to reset the interactive state.
    - makeplane(...) [not shown in snippet]
      - Responsible for producing self.plane from self.Cube using the chosen estimator (details not included in the snippet).

- Interactivity and events
  - Interactive modes include:
    - Left-click (Button 1) or keyboard '1': plot the selected pixel’s spectrum in a new window and mark with an 'x'
    - Middle-click (Button 2) or keyboard 'o': overplot the selected spectrum in the external plot window
    - Button 3: disconnect the interactive viewer
    - Click-and-drag with Button 1 to average over a circular region, or use 'c' to set center and 'r' to set radius
  - Event handlers are wired via the _connect() method.

- Data and plotting backends
  - If APLpy is available (icanhasaplpy and useaplpy), the map is shown with APLpy’s FITSFigure:
    - A FITS file is built from the plane and header: pyfits.PrimaryHDU(data=self.plane, header=self.header)
    - self.FITSFigure.show_colorscale(...) is used for rendering
    - A colorbar can be added via self.FITSFigure.add_colorbar()
  - If APLpy is not available, the map is shown with Matplotlib:
    - self.axis.imshow(self.plane, vmin=vmin, vmax=vmax, cmap=cmap, **plotkwargs)
    - A colorbar can be added with self._pyplot.colorbar(self.axis.images[0])

- Dependencies and compatibility
  - Attempts to import astropy.wcs as pywcs and astropy.io.fits as pyfits; falls back to pyfits/pywcs if needed; sets pywcsOK accordingly.
  - Attempts to import aplpy; if unsuccessful, icanhasaplpy is False and Matplotlib path is used.
  - Header handling uses cubes.flatten_header to strip the header while preserving necessary WCS information.
  - The code is designed to be compatible with both Py2 and Py3 (from __future__ import print_function, use of six).

- Notes
  - The actual implementation of makeplane, plot_spectrum, and the detailed interactive behavior are not shown in the snippet but are referenced by mapplot and event-handling code.
  - The interactive capabilities are intended for analysis rather than publication-quality plots, though APLpy can still yield publication-quality visuals.

## Usage

- Basic usage (with a Cube object available as mycube):
  - Initialize and plot:
    - mp = MapPlotter(Cube=mycube)
    - mp.mapplot()
  - Or plot directly on initialization:
    - mp = MapPlotter(Cube=mycube, figure=1, doplot=True)

- Simple explicit plotting call:
  - mp = MapPlotter(Cube=mycube)
  - mp.mapplot(vmin=None, vmax=None, cmap=None)

- Access to the interactive plotting workflow:
  - The interactive controls enable per-pixel spectrum inspection, overplotting additional spectra, and radial averaging via circle operations as described in the docstring.

## Details

- Design goals
  - Provide an interactive, exploratory tool for cube data, enabling quick inspection of spectra across spatial positions.
  - Utilize APLpy when available for high-quality plotting, while maintaining a robust fallback to Matplotlib when APLpy is absent.
  - Integrate with the cubes module for header handling and WCS support, ensuring consistent behavior across the project.

- Implementation notes (from source)
  - The MapPlotter class encapsulates plotting, interactivity, and plotting backend selection.
  - It relies on the cube header to establish a WCS context for plotting.
  - It supports optional colorbars and customizable color mapping via vmin, vmax, and cmap.
  - It ensures that estimator-related kwargs do not interfere with plotting by removing them from kwargs when calling mapplot.
