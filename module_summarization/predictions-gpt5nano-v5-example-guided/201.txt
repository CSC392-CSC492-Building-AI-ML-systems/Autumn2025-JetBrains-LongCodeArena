Partial Dependence and Individual Conditional Expectation plots
==============================================================

Overview
--------
Partial Dependence Plots (PDP) and Individual Conditional Expectation (ICE) plots are visualization techniques for understanding how a machine learning model’s predictions depend on one or more features.

- PDP shows the average predicted outcome as a function of a feature.
- ICE shows individual prediction lines for each observation, illustrating heterogeneity across samples. PDP can be viewed as the average of ICE curves.

Key Concepts
------------
- For a selected feature, predictions are computed after varying the feature’s value across a grid while keeping all other features fixed (often to observed values).
- ICE reveals sample-specific relationships; PDP summarizes this relationship by averaging across samples.
- Proper handling of feature names or indices is important for selecting the feature to plot and for mapping feature names to their corresponding indices.

Key Components
--------------
- _check_feature_names(X, feature_names=None)
  Validates or derives feature names for plotting. If feature_names is None, it uses column names (for pandas) or generates generic names for NumPy arrays. It also checks for duplicates.

  Example (from source code):

  .. code-block:: python

     def _check_feature_names(X, feature_names=None):
         """Check feature names.

         Parameters
         ----------
         X : array-like of shape (n_samples, n_features)
             Input data.

         feature_names : None or array-like of shape (n_names,), dtype=str
             Feature names to check or `None`.

         Returns
         -------
         feature_names : list of str
             Feature names validated. If `feature_names` is `None`, then a list of
             feature names is provided, i.e. the column names of a pandas dataframe
             or a generic list of feature names (e.g. `["x0", "x1", ...]`) for a
             NumPy array.
         """
         if feature_names is None:
             if hasattr(X, "columns") and hasattr(X.columns, "tolist"):
                 # get the column names for a pandas dataframe
                 feature_names = X.columns.tolist()
             else:
                 # define a list of numbered indices for a numpy array
                 feature_names = [f"x{i}" for i in range(X.shape[1])]
         elif hasattr(feature_names, "tolist"):
             # convert numpy array or pandas index to a list
             feature_names = feature_names.tolist()
         if len(set(feature_names)) != len(feature_names):
             raise ValueError("feature_names should not contain duplicates.")

         return feature_names

- _get_feature_index(fx, feature_names=None)
  Maps a feature identifier (index or name) to a numeric index, given the feature_names list. This is useful when selecting the feature to plot by name or by index.

  Example (from source code):

  .. code-block:: python

     def _get_feature_index(fx, feature_names=None):
         """Get feature index.

         Parameters
         ----------
         fx : int or str
             Feature index or name.

         feature_names : list of str, default=None
             All feature names from which to search the indices.

         Returns
         -------
         idx : int
             Feature index.
         """
         if isinstance(fx, str):
             if feature_names is None:
                 raise ValueError(
                     f"Cannot plot partial dependence for feature {fx!r} since "
                     "the list of feature names was not provided, neither as "
                     "column names of a pandas data-frame nor via the feature_names "
                     "parameter."
                 )
             try:
                 return feature_names.index(fx)
             except ValueError as e:
                 raise ValueError(f"Feature {fx!r} not in feature_names") from e
         return fx

Usage
-----
- Selecting and plotting a feature for PDP/ICE typically involves resolving the feature to an index and then generating the plot for that feature.
- The helper functions above are used to derive a valid feature name list and a corresponding index from either a feature name or an index.

Example workflow (conceptual; actual plotting API may vary):
- Resolve feature names
  .. code-block:: python

     X = ...  # dataset of shape (n_samples, n_features)
     feature_names = _check_feature_names(X, feature_names=None)

- Obtain the feature index by name or by numeric index
  .. code-block:: python

     idx = _get_feature_index("age", feature_names=feature_names)
     # or
     idx = _get_feature_index(0, feature_names=feature_names)

- Plot PDP or ICE for the selected feature
  .. code-block:: python

     # Using a plotting utility (e.g., from scikit-learn)
     from sklearn.inspection import PartialDependenceDisplay
     # estimator: a fitted model; X: training data
     PartialDependenceDisplay.from_estimator(estimator, X, features=[idx], kind="average")  # PDP
     PartialDependenceDisplay.from_estimator(estimator, X, features=[idx], kind="both")     # ICE + PDP

Details
-------
- Partial Dependence Plots (PDP)
  - Show the average effect of a feature on the predicted value.
  - Useful for understanding global trends and general feature influence.

- Individual Conditional Expectation (ICE) plots
  - Show the effect for individual instances, highlighting heterogeneity in the data.
  - Can reveal interactions between features that PDP might obscure when averaged.

- Practical considerations
  - Feature names: Ensure a unique and stable set of feature names for mapping to indices.
  - Grid resolution: Choose an appropriate grid for continuous features to balance detail and compute cost.
  - Correlated features: Be cautious interpreting PDP when features are correlated; ICE can help diagnose issues by revealing sample-level variations.
