# Optimization: General-Purpose Optimizer

## Overview
This module provides general-purpose optimization utilities that can be used for fitting any model by supplying an objective function and its gradient. The design intentionally decouples these utilities from LikelihoodModel so they can be reused across different modeling contexts. The core component is the Optimizer class, which exposes a flexible _fit method capable of leveraging multiple SciPy optimization solvers.

Key ideas:
- Objective and gradient-based fitting for arbitrary models.
- Support for multiple solvers (Newton, Nelder–Mead, BFGS, Powell, Conjugate Gradient, Newton-CG, basinhopping, and a generic wrapper around scipy.optimize.minimize).
- Optional Hessian support and detailed solver settings when available.
- Basin-hopping solver has special behavior and may ignore some explicit arguments.

## Key Components
- _check_method(method, methods)
  - Validates the chosen optimization method against a list of supported methods and raises a ValueError if unknown.

- Optimizer
  - _fit(self, objective, gradient, start_params, fargs, kwargs, hessian=None, method='newton', maxiter=100, full_output=True, disp=True, callback=None, retall=False)
    - Primary entry point for fitting an objective function.
    - Parameters:
      - start_params: Initial guess for the solution.
      - objective: Function to maximize (or minimize, depending on convention).
      - gradient: Function that returns the gradient of the objective.
      - fargs: Extra positional arguments passed to the objective.
      - kwargs: Extra keyword arguments passed to the objective.
      - hessian: Optional Hessian or Hessian operator.
      - method: Optimization method (e.g., 'newton', 'nm', 'bfgs', 'powell', 'cg', 'ncg', 'basinhopping', 'minimize').
      - maxiter: Maximum number of iterations (ignored by some solvers like basinhopping).
      - full_output: If True, return a dictionary with solver output in mle_retvals.
      - disp: If True, print convergence messages.
      - callback: Optional callback function called after each iteration.
      - retall: If True, return a list of solutions at each iteration.
    - Returns:
      - xopt: Optimized parameters.
      - retvals: Solver output dictionary (or None if full_output is False).
      - optim_settings: Dictionary containing the solver parameters actually used.
    - Notes:
      - The basinhopping solver ignores maxiter, retall, and full_output explicit arguments.
      - Optional solver-specific arguments (e.g., tol, xtol, ftol, maxfun, gtol, etc.) are documented in the notes and align with SciPy's solver interfaces.

Notes on solver options
- The method exposes a range of solver-specific options. For example:
  - 'newton': tol
  - 'nm' (Nelder-Mead): xtol, ftol, maxfun
  - 'bfgs': gtol, norm, epsilon
  - 'lbfgs': m, pgtol, factr, maxfun, epsilon, approx_grad
  - 'cg': gtol, norm, epsilon
  - 'ncg': fhess_p, avextol, epsilon
  - 'powell': xtol, ftol, maxfun, start_direc
  - 'basinhopping': niter, niter_success, T, stepsize, interval, minimizer (with mapping of args to the minimizer)
- These options correspond to SciPy’s optimization interfaces and can be supplied through the appropriate bridging of arguments.

## Usage
- Typical usage pattern involves defining an objective function and its gradient, then calling _fit with a starting parameter vector.

Example: basic custom objective with a gradient
- Objective function and gradient (example)
  - Minimize or maximize a simple quadratic function as a placeholder
  - In practice, objective should be the model’s log-likelihood (or negative log-likelihood, depending on convention)

Code example:
- Note: The following is a minimal illustration of using the Optimizer with a simple objective. Adapt objective/gradient to your model.

```python
import numpy as np
from optimization import Optimizer

def objective(x, *args):
    # Example: simple quadratic (to minimize). If maximizing, negate as needed.
    return np.sum(x**2)

def gradient(x, *args):
    return 2 * x

start_params = np.zeros(3)

opt = Optimizer()
xopt, retvals, optim_settings = opt._fit(
    objective=objective,
    gradient=gradient,
    start_params=start_params,
    fargs=(),
    kwargs={},
    hessian=None,
    method='newton',
    maxiter=100,
    full_output=True,
    disp=True,
    callback=None,
    retall=False
)

print("Optimized parameters:", xopt)
print("Solver outputs:", retvals)
print("Solver settings:", optim_settings)
```

Usage with a Hessian or alternative solver
- If you have a Hessian or a Hessian-vector product, you can pass it via hessian.
- To use a different solver, set method to one of the supported options, e.g., 'bfgs', 'cg', 'basinhopping', or 'minimize'.

Example: using the generic minimize interface
```python
xopt, retvals, optim_settings = opt._fit(
    objective=objective,
    gradient=gradient,
    start_params=start_params,
    fargs=(),
    kwargs={},
    hessian=None,
    method='minimize',
    maxiter=200,
    full_output=True,
    disp=False,
    callback=None,
    retall=False
)
```

## Details
- Purpose: Provides a reusable, solver-agnostic interface to optimize an arbitrary objective function with optional gradient and Hessian information.
- Return contract:
  - xopt: Optimal parameter values found.
  - retvals: Solver output dictionary when full_output is True; otherwise None.
  - optim_settings: Dictionary containing the solver parameters actually used for the run.
- Design note: This module is intended to be independent of any specific model class (e.g., LikelihoodModel) so it can be reused across different fitting problems.
- Relationship to SciPy: The methods and options are wrappers around SciPy's optimization routines, with some additional handling for solver-specific nuances (notably basinhopping).
