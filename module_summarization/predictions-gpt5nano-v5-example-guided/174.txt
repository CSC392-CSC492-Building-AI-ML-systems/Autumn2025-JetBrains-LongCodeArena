Array API support
===================

Overview
--------

This document describes how Array API compatibility is approached in the repository via array_api.rst. The code in this area focuses on ensuring array operations have reliable backends by pinning minimum dependency versions (e.g., NumPy, SciPy) and providing a small utility to query these requirements. There is no explicit implementation of the Array API standard in this module; compatibility is achieved by relying on standard NumPy/SciPy array interfaces and consistent minimum versions.

Key Components
--------------

- Dependency specifications: dependent_packages defines the minimum versions for core array-related libraries and related tooling.
- PyPy compatibility: NumPy minimum version depends on the Python implementation.
- Inverse mapping for CI tags: tag_to_packages is generated to map extras tags to package/version requirements.
- Minimal CLI for minimum versions: A small command-line interface allows querying the minimum version for a given package, useful for CI.

- Inverse mapping used by CI (generated at runtime)
.. code-block:: python

   tag_to_packages: dict = defaultdict(list)
   for package, (min_version, extras) in dependent_packages.items():
       for extra in extras.split(", "):
           tag_to_packages[extra].append("{}>={}".format(package, min_version))


- Minimal CLI to query minimum versions
.. code-block:: python

   if __name__ == "__main__":
       parser = argparse.ArgumentParser(description="Get min dependencies for a package")

       parser.add_argument("package", choices=dependent_packages)
       args = parser.parse_args()
       min_version = dependent_packages[args.package][0]
       print(min_version)


Usage
-----

- Retrieve the minimum version for a package:
  Example:
  .. code-block:: console

     python array_api.py numpy

  Example output:
  .. code-block:: text

     1.17.3

Notes
-----

- The Array API support demonstrated here is indirect: ensuring core array libraries meet minimum versions to maintain compatibility across array backends.
- PyPy builds may require a newer NumPy version than CPython builds. This is reflected in the NUMPY_MIN_VERSION selection.
