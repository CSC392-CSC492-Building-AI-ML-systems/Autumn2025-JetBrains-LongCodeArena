# Configuration Options for Import/Export

## Overview
This module provides configuration options for controlling the import and export of various elements in JSON-based configuration files within PyAEDT. It defines the ConfigurationsOptions class, which exposes a set of boolean flags and parameters that determine which components are exported to or imported from configuration data. A helper function, _find_datasets, supports extraction of dataset references (notably piecewise-linear, or "pwl", values) from nested dictionaries.

## Key Components

- ConfigurationsOptions
  - Purpose: Encapsulates flags and tolerances that govern exporting and importing configuration components.
  - Default behavior: Most components are exported/imported by default (True), with a small object_mapping_tolerance of 1e-9.

  - Important properties (defaults shown):
    - object_mapping_tolerance: 1e-9
      - Description: Tolerance used when mapping objects (e.g., during boundary mapping). Affects precision in object matching.
    - export_variables: True
      - Description: Export variables to the JSON configuration.
    - export_setups: True
      - Description: Export setups to the JSON configuration.
    - export_optimizations: True
      - Description: Export optimization studies to the JSON configuration.
    - export_parametrics: True
      - Description: Export parametric definitions to the JSON configuration.
    - export_boundaries: True
      - Description: Export boundary definitions to the JSON configuration.
    - export_mesh_operations: True
      - Description: Export mesh operation data to the JSON configuration.
    - export_coordinate_systems: True
      - Description: Export coordinate system definitions to the JSON configuration.
    - export_materials: True
      - Description: Export material definitions to the JSON configuration.
    - export_object_properties: True
      - Description: Export object properties to the JSON configuration.
    - export_datasets: True
      - Description: Export datasets to the JSON configuration.
    - import_datasets: True
      - Description: Import datasets from the JSON configuration.
    - import_variables: True
      - Description: Import variables from the JSON configuration.
    - import_setups: True
      - Description: Import setups from the JSON configuration.
    - import_optimizations: True
      - Description: Import optimizations from the JSON configuration.
    - import_parametrics: True
      - Description: Import parametrics from the JSON configuration.
    - import_boundaries: True
      - Description: Import boundary definitions from the JSON configuration.
    - import_mesh_operations: True
      - Description: Import mesh operation data from the JSON configuration.
    - import_coordinate_systems: True
      - Description: Import coordinate system definitions from the JSON configuration.
    - import_materials: True
      - Description: Import material definitions from the JSON configuration.
    - import_object_properties: True
      - Description: Import object properties from the JSON configuration.
    - skip_import_if_exists: False
      - Description: If True, skip importing components that already exist in the target design.

  - Usage example:
    - Create an application instance and customize export/import behavior.
      - from pyaedt import Hfss
      - hfss = Hfss()
      - hfss.configurations.options.export_variables = False  # Disable variables export

- _find_datasets(d, out_list)
  - Purpose: Internal helper to recursively search a nested dictionary (and lists) for dataset references used in piecewise-linear values.
  - Behavior:
    - Traverses dictionaries (including OrderedDict) recursively.
    - If a value is a list, iterates elements and, when possible, looks for entries with a "free_form_value" containing a "pwl" reference, extracting the substring that starts at "$" and ends at ",".
    - If a value is a string containing "pwl", extracts the substring from "$" to ",".
    - Safely handles missing keys or type mismatches (KeyError, TypeError) without failing.
  - Example:
    - This function collects dataset references (e.g., "$dataset_name,") from complex configuration structures into out_list for later processing.

## Usage

- Creating and configuring export/import options:
  - from pyaedt import Hfss
  - hfss = Hfss()
  - # Access the configuration options
  - opts = hfss.configurations.options
  - # Disable export of variables
  - opts.export_variables = False
  - # Enable import of materials
  - opts.import_materials = True
  - # Control object mapping tolerance
  - opts.object_mapping_tolerance = 1e-8

- Extracting datasets from a configuration structure (internal utility):
  - datasets = []
  - _find_datasets(configuration_dict, datasets)
  - # 'datasets' now contains extracted dataset references (if any)

## Details

- The ConfigurationsOptions class provides a centralized location to toggle export/import behavior for a wide range of configuration components, enabling precise control over what gets serialized to or deserialized from JSON configuration files.
- The object_mapping_tolerance parameter enables fine-grained control over coordinate or object mapping precision during operations that rely on matching entities between models or configurations.
- The _find_datasets helper is intended for internal use to support dataset resolution within nested configuration data structures. It is robust to common data structure variations (dicts, lists, strings) and focuses on identifying "pwl" (piecewise-linear) dataset references embedded in the free_form_value fields.
