# Cross-validation: evaluating estimator performance

## Overview
This module provides visualization utilities for cross-validation results, enabling evaluation of estimator performance as a function of training set size (learning curves) or hyperparameter values (validation curves). It builds on the results from sklearn.model_selection.learning_curve and sklearn.model_selection.validation_curve to render informative plots that compare training and test scores across folds.

Key visualization classes include LearningCurveDisplay, which presents learning curves to assess how performance improves with more training data.

## Key Components

- _BaseCurveDisplay
  - Internal base class for curve displays. Responsible for plotting curves of training and/or test scores with options for how to display variability (standard deviation) across cross-validation folds.
  - Core features:
    - Plot training and/or test scores as means over cross-validation folds.
    - Optional display of variability via:
      - fill_between: shaded region around the mean
      - errorbar: error bars per point
      - None: no variability visualization
    - Support for plotting either "train", "test", or "both" score types.
    - Score naming and labeling support through score_name.
    - Deprecation handling for log_scale (use ax_.set_xscale / set_yscale instead).
    - Automatic selection of x-axis scale when not using log_scale, based on the spread of x_data.
  - Core method:
    - _plot_curve(x_data, ax=None, negate_score=False, score_name=None, score_type="test",
                  std_display_style="fill_between", line_kw=None, fill_between_kw=None,
                  errorbar_kw=None)

- LearningCurveDisplay
  - Visualization for learning curves, i.e., performance vs. training set size.
  - Attributes (relevant to plotting outputs):
    - ax_: Matplotlib Axes containing the plot
    - figure_: Matplotlib Figure containing the plot
    - train_sizes: array of training set sizes
    - train_scores: scores on training sets
    - test_scores: scores on test sets
    - errorbar_/lines_/fill_between_: handles for the displayed variability depending on std_display_style
  - Recommended usage: create via from_estimator to produce a ready-to-plot display.
  - See Also: sklearn.model_selection.learning_curve (computes the underlying data)

## Usage

- Typical workflow (recommended):
  - Compute learning curve data and display:
    - Example (conceptual):
      - Compute data with learning_curve
      - Create a LearningCurveDisplay via from_estimator
      - Plot the display

  - Example:
    ```
    from sklearn.model_selection import learning_curve
    from sklearn.model_selection import LearningCurveDisplay
    from sklearn.ensemble import RandomForestClassifier
    import numpy as np

    # X, y: dataset
    estimator = RandomForestClassifier(random_state=0)
    cv = 5
    train_sizes, train_scores, test_scores = learning_curve(estimator, X, y, cv=cv)

    display = LearningCurveDisplay.from_estimator(estimator, X, y, cv=cv,
                                                train_sizes=train_sizes)
    display.plot()
    ```

- Alternative usage (direct display construction):
  - If you already have train_sizes, train_scores, and test_scores:
    ```
    display = LearningCurveDisplay(
        train_sizes=train_sizes,
        train_scores=train_scores,
        test_scores=test_scores,
        score_name="Accuracy",
    )
    display.plot(std_display_style="fill_between")
    ```

- Variability display options:
  - Fill between (mean ± std):
    ```
    display.plot(std_display_style="fill_between")
    ```
  - Error bars:
    ```
    display.plot(std_display_style="errorbar")
    ```
  - No variability shading/bars:
    ```
    display.plot(std_display_style=None)
    ```

## Details

- _BaseCurveDisplay._plot_curve behavior
  - Validates Matplotlib support and creates axes if not provided.
  - Handles score negation for metrics where higher is worse by negating the arrays when needed.
  - Validates std_display_style values: "errorbar", "fill_between", or None.
  - Builds a mapping of scores to display:
    - score_type == "train" -> {"Train": train_scores}
    - score_type == "test"  -> {"Test": test_scores}
    - score_type == "both"  -> {"Train": train_scores, "Test": test_scores}
  - Display modes:
    - fill_between (default) or None:
      - Plots mean scores (axis=1) for each entry in scores.
      - If fill_between, overlays shaded regions representing mean ± std (axis=1).
    - errorbar:
      - Plots mean with error bars corresponding to std (axis=1) for each entry in scores.
  - Axis and labeling:
    - If score_name is not provided, uses the object's score_name attribute.
    - Sets the y-axis label to the score name.
    - Sets the x-axis scale:
      - If the log_scale parameter is used (deprecated), a corresponding scale is set and a FutureWarning is issued.
      - Otherwise, selects an appropriate scale (linear, log, or symlog) based on _interval_max_min_ratio(x_data).
  - Maintains references to created plot elements:
    - self.ax_, self.figure_
    - self.lines_, self.errorbar_, self.fill_between_ depending on display style
  - Always adds a legend to the axis.

- Score naming and negation
  - The score_name parameter customizes the y-axis label.
  - If negate_score is True, scores are negated to display lower is better metrics appropriately.

- Deprecation note
  - The log_scale parameter is deprecated as of version 1.3. Use the axis scale methods:
    - display.ax_.set_xscale(...)
    - display.ax_.set_yscale(...)

- See Also
  - sklearn.model_selection.learning_curve: Computes the learning curve data used to populate LearningCurveDisplay.
  - sklearn.model_selection.validation_curve: Computes validation curves for hyperparameter analysis (for display via ValidationCurveDisplay).

- Notes
  - The visualization API is designed to be flexible and integrated with the broader sklearn plotting utilities, and emphasizes mean performance over cross-validation folds with optional uncertainty visualization.
