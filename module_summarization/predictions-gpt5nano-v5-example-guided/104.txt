Diophantine
===========

Overview
--------

This module provides a solver for Diophantine equations. It exposes the
diophantine() function, which converts a polynomial equation into a product
of terms that should equal zero and solves each factor independently. The
solutions are returned as a set of tuples, where each tuple contains the
solutions for the corresponding variables, ordered alphabetically.

Key Components
--------------

- diophantine(eq, param=symbols('t', integer=True), syms=None)
  - Primary entry point for solving Diophantine equations.
  - Converts the equation into a product of terms that must vanish and solves
    each term via diop_solve, merging results according to the equation type.
  - Parameters:
    - eq: Relational or Expr — the equation to solve.
    - param: Symbol, optional — the parameter to use for the solution (default is a new integer parameter t).
    - syms: list of Symbol, optional — determines the order of variables in the returned tuples.
  - Returns: A set of tuples containing the solutions for each variable, ordered by the
    alphabetic order of the variables.
  - Examples (from the source):
    - diophantine(x**2 - y**2)
      -> {(t_0, -t_0), (t_0, t_0)}
    - diophantine(x*(2*x + 3*y - z))
      -> {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}
    - diophantine(x**2 + 3*x*y + 4*x)
      -> {(0, n1), (3*t_0 - 4, -t_0)}

- classify_diop
  - Helper utility (imported) used to classify a polynomial into a Diophantine type
    (e.g., linear, binary_quadratic, general_sum_of_squares, etc.). Its output
    guides how solutions are merged.

- diop_solve (via the solver pathway)
  - Subroutine used to solve individual diophantine subproblems for a given base term
    and parameter. The diophantine() function delegates to this routine for actual
    solving.

- Auxiliary helpers (internal, used by diophantine)
  - _is_int(i): Check if i can be interpreted as an integer.
  - _sorted_tuple(*i): Return a sorted tuple of inputs.
  - _remove_gcd(*x): Divide arguments by their gcd if possible.
  - _rational_pq(a, b): Return (numerator, denominator) of a/b with gcd removed.
  - _nint_or_floor(p, q): Return the nearest integer to p/q (tie -> floor).
  - _odd(i): True if i is odd.
  - _even(i): True if i is even.

- diophantine known types
  - diop_known: A set of known Diophantine problem types such as
    'binary_quadratic', 'cubic_thue', 'general_pythagorean', 'general_sum_of_squares',
    'linear', 'univariate', etc. This helps in routing to appropriate solving strategies.

- __all__ = 'diophantine', 'classify_diop'
  - Public API surface exported by the module.

Usage
-----

Basic usage (solve a simple Diophantine equation):

.. code-block:: python

    from sympy import symbols
    from your_module import diophantine  # adjust import path as appropriate

    x, y = symbols('x y')
    diophantine(x**2 - y**2)
    # -> {(t_0, -t_0), (t_0, t_0)}

Advanced usage (specifying parameter and symbol order):

.. code-block:: python

    t = symbols('t', integer=True)
    diophantine(x*(2*x + 3*y - z), param=t)

Notes
-----

- The function returns a set of solutions. Each element is a tuple whose entries
  correspond to the variables sorted alphabetically (unless a specific order is
  provided via syms).

- If the equation has denominators, the solver handles them by solving the
  denominator and numerator parts appropriately and combining results.

- If no nontrivial solutions exist, but the equation is satisfied by the trivial
  zero vector, the trivial solution is included in the result.

- If the input cannot be treated as a polynomial with rational coefficients or
  requires unsupported diophantine types, the function raises a TypeError.
