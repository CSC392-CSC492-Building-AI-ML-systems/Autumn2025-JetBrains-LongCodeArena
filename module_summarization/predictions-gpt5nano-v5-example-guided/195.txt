# Learn Gaussian Mixture Models

## Overview
This module defines the base infrastructure for Gaussian mixture models used with the Expectation-Maximization (EM) algorithm. It provides an abstract BaseMixture class that handles common functionality such as parameter validation, initialization strategies, multi-stage fitting (n_init initializations), convergence control, and API conventions. Concrete Gaussian mixture models should subclass BaseMixture and implement the abstract methods to specify model-specific details (e.g., parameter checks and initialization of component parameters).

Key features include:
- EM-based fitting with multiple initializations (n_init) and convergence tolerance (tol)
- Flexible parameter initialization via several strategies
- Integration with a scikit-learn-like API (fit, fit_predict, etc.)
- Parameter constraints for robust validation

## Key Components

- _check_shape(param, param_shape, name)
  - Utility to validate that a given parameter has the expected shape.

  Example:
  - Ensures a parameter matches a specified shape and raises a ValueError otherwise.

  Code excerpt:
  ```
  def _check_shape(param, param_shape, name):
      """Validate the shape of the input parameter 'param'.

      Parameters
      ----------
      param : array

      param_shape : tuple

      name : str
      """
      param = np.array(param)
      if param.shape != param_shape:
          raise ValueError(
              "The parameter '%s' should have the shape of %s, but got %s"
              % (name, param_shape, param.shape)
          )
  ```

- BaseMixture (abstract base class)
  - Inherits from DensityMixin and BaseEstimator and defines a common interface for mixture models.
  - Core responsibilities:
    - Parameter constraints specification
    - Initialization of model parameters via various strategies
    - Fitting via EM across multiple initializations
    - Abstract methods to be implemented by concrete subclasses:
      - _check_parameters(self, X)
      - _initialize(self, X, resp)

  Main constructor parameters:
  - n_components: number of mixture components
  - tol: convergence tolerance for the EM algorithm
  - reg_covar: regularization for covariance matrices to ensure numerical stability
  - max_iter: maximum number of EM iterations per initialization
  - n_init: number of initializations to perform
  - init_params: method used to initialize the responsibilities (e.g., "kmeans", "random", "random_from_data", "k-means++")
  - random_state: random state or seed controlling randomness
  - warm_start: if True, reuse solution across fit calls
  - verbose: verbosity level
  - verbose_interval: how often to print progress

  Important implementation detail:
  - Initialization strategies are implemented in _initialize_parameters and can be one of:
    - "kmeans": initialize using KMeans first, then set one-hot responsibilities
    - "random": random soft assignments normalized to sum to 1 per sample
    - "random_from_data": select random component indices and assign corresponding one-hot rows
    - "k-means++": initialize using k-means++ centroids and set one-hot responsibilities

  Example (from the source):
  ```
  def _initialize_parameters(self, X, random_state):
      """Initialize the model parameters.

      Parameters
      ----------
      X : array-like of shape  (n_samples, n_features)

      random_state : RandomState
          A random number generator instance that controls the random seed
          used for the method chosen to initialize the parameters.
      """
      n_samples, _ = X.shape

      if self.init_params == "kmeans":
          resp = np.zeros((n_samples, self.n_components))
          label = (
              cluster.KMeans(
                  n_clusters=self.n_components, n_init=1, random_state=random_state
              )
              .fit(X)
              .labels_
          )
          resp[np.arange(n_samples), label] = 1
      elif self.init_params == "random":
          resp = random_state.uniform(size=(n_samples, self.n_components))
          resp /= resp.sum(axis=1)[:, np.newaxis]
      elif self.init_params == "random_from_data":
          resp = np.zeros((n_samples, self.n_components))
          indices = random_state.choice(
              n_samples, size=self.n_components, replace=False
          )
          resp[indices, np.arange(self.n_components)] = 1
      elif self.init_params == "k-means++":
          resp = np.zeros((n_samples, self.n_components))
          _, indices = kmeans_plusplus(
              X,
              self.n_components,
              random_state=random_state,
          )
          resp[indices, np.arange(self.n_components)] = 1

      self._initialize(X, resp)
  ```

  - fit(self, X, y=None)
    - Public method to fit the model parameters using EM. It delegates to fit_predict and returns self.

  - fit_predict(self, X, y=None)
    - Public method to fit the model and return predicted component labels for X.
    - Performs validation of input data and returns the most probable component labels for each sample.

  Note: This class is abstract. Concrete Gaussian mixture implementations should provide:
  - _check_parameters(self, X)
  - _initialize(self, X, resp)

## Usage

Because BaseMixture is an abstract base class, you typically use a concrete subclass (e.g., GaussianMixture) that implements the specific Gaussian mixture model. The API mirrors scikit-learn estimators.

- Example (illustrative; replace with a concrete subclass in your codebase):
  - Create and fit a Gaussian mixture model with 3 components, using k-means initialization and multiple restarts:
    - model = GaussianMixture(n_components=3, tol=1e-3, max_iter=100, n_init=5, init_params="kmeans", random_state=0)
    - model.fit(X)
- After fitting, you can obtain labels:
  - labels = model.fit_predict(X)

## Details

- Parameter constraints (as enforced by _parameter_constraints):
  - n_components: integer in [1, ∞)
  - tol: real, >= 0.0
  - reg_covar: real, >= 0.0
  - max_iter: integer, >= 0
  - n_init: integer, >= 1
  - init_params: one of {"kmeans", "random", "random_from_data", "k-means++"}
  - random_state: random_state
  - warm_start: boolean
  - verbose: verbose
  - verbose_interval: integer, >= 1

- Convergence and warnings:
  - The EM procedure iterates up to max_iter per initialization and uses tol to determine convergence.
  - If convergence is not achieved within the allotted iterations, a ConvergenceWarning may be raised.

- Dependencies:
  - numpy
  - scipy (logsumexp)
  - sklearn-like utilities within the package (cluster.KMeans, kmeans_plusplus, etc.)

- Relationship to concrete models:
  - BaseMixture provides the shared EM framework and initialization logic.
  - Concrete models must implement _check_parameters and _initialize to handle model-specific parameters (e.g., Gaussian component means, covariances, and weights).

This documentation summarizes the purpose and usage of the mixture module, emphasizing its role as a reusable base for Learn Gaussian Mixture Models and the EM-based fitting workflow.
