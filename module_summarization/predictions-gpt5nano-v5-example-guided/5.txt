Publishing versions
===================

Overview
--------
This module defines the session infrastructure used when publishing versioned content to an ftrack server. It provides:
- A small authentication helper that attaches ftrack credentials to HTTP requests
- A Session class that encapsulates connection details, credentials, caching, plug-in support, and event hub interaction
- Mechanisms for tracking and recording operations performed during a publishing workflow

Key Components
--------------
- SessionAuthentication
  - Attaches ftrack authentication information to requests via HTTP headers.
  - Example:
    def __init__(self, api_key, api_user):
        '''Initialise with *api_key* and *api_user*.'''
        self.api_key = api_key
        self.api_user = api_user
        super(SessionAuthentication, self).__init__()

    def __call__(self, request):
        '''Modify *request* to have appropriate headers.'''
        request.headers.update({
            'ftrack-api-key': self.api_key,
            'ftrack-user': self.api_user
        })
        return request

- Session
  - Central entry point for interacting with an ftrack server to publish versions.
  - Initialization parameters include:
    - server_url: server address (required)
    - api_key / api_user: credentials (required)
    - auto_populate: fetch missing data automatically when accessed
    - plugin_paths: locations to discover plugins
    - cache / cache_key_maker: caching layer and key generation
    - auto_connect_event_hub: automatically connect to the event hub for non-local events
    - schema_cache_path: enable/disable and locate schema caching
    - plugin_arguments: arguments passed to discovered plugins
  - Behavior:
    - Reads server URL and credentials from environment if not supplied explicitly
    - Maintains recorded operations for ongoing work via self.recorded_operations
    - Uses a layered cache with a top-level MemoryCache to optimize performance
  - Example (constructor signature excerpt):
    class Session(object):
        def __init__(
            self, server_url=None, api_key=None, api_user=None, auto_populate=True,
            plugin_paths=None, cache=None, cache_key_maker=None,
            auto_connect_event_hub=None, schema_cache_path=None,
            plugin_arguments=None
        ):
            ...

Usage
-----
Preparing to publish versions requires establishing a session with authentication and, optionally, an event hub connection for publishing and subscribing to events.

1) Prepare credentials
   - You can supply credentials directly or via environment variables:
     - FTRACK_SERVER for server_url
     - FTRACK_API_KEY for api_key
     - FTRACK_API_USER for api_user

2) Create a session
   - Example:
     from ftrack_api import Session

     session = Session(
         server_url='https://ftrack.example.com',
         api_key='your_api_key',
         api_user='your_username'
     )

3) Enable event hub publishing (optional)
   - If you need to publish or subscribe to non-local events during publishing, enable the embedded event hub:
     session = Session(..., auto_connect_event_hub=True)

4) Publish versions
   - Use the session's capabilities and the ftrack API to create or update versioned entities.
   - The current module provides the session scaffold and operation-tracking facility (recorded_operations) used by publishing workflows.

Details
-------
- Environment integration
  - The session supports deriving configuration from environment variables:
    - FTRACK_SERVER for the server URL
    - FTRACK_API_KEY and FTRACK_API_USER for authentication
    - FTRACK_API_SCHEMA_CACHE_PATH to enable schema caching location
    - FTRACK_EVENT_PLUGIN_PATH to specify plugin discovery paths

- Auto-population and plugin support
  - If auto_populate is True, missing attributes can be fetched on demand from the server.
  - Plugins may be discovered and registered at session creation, with optional plugin_arguments to control discovery behavior. If a plugin cannot accept the provided arguments, the mechanism will attempt to adapt and warn if necessary.

- Event hub
  - The session can connect to an embedded event hub to publish and subscribe to non-local events, with automatic connection controlled by auto_connect_event_hub.

- Operation tracking
  - Publishing workflows can record operations through self.recorded_operations for later processing or commit.

This module provides the foundational capabilities for publishing versions through the ftrack API, including authenticated communication, session management, plugin support, event publishing, and operation tracking.
