Kernel approximation feature maps
===============================

Overview
--------
The sklearn.kernel_approximation module implements several approximate kernel feature maps based on Fourier transforms and Count Sketches. These maps enable efficient approximations of kernel computations in high-dimensional spaces.

Key Components
--------------
- PolynomialCountSketch
  - Purpose: Approximate the feature map of the polynomial kernel using Tensor Sketch.
  - Kernel form approximated:
    K(X, Y) = (gamma * <X, Y> + coef0)^degree
  - Core idea: Efficiently compute a Count Sketch of the outer product of a vector with itself using Fast Fourier Transforms (FFT). See the user guide on polynomial kernel approximation for details.
  - Version: Added in 0.24

  Parameters
  - gamma: float, default=1.0
    Parameter of the polynomial kernel whose feature map will be approximated.
  - degree: int, default=2
    Degree of the polynomial kernel whose feature map will be approximated.
  - coef0: float, default=0
    Constant term of the polynomial kernel whose feature map will be approximated.
  - n_components: int, default=100
    Dimensionality of the output feature space.
  - random_state: int, RandomState instance, default=None
    Determines random number generation for indexHash and bitHash initialization.

  Attributes (set after fit)
  - indexHash_: ndarray of shape (degree, n_features), dtype=int64
    Hash indices in [0, n_components) used for the Count Sketch.
  - bitHash_: ndarray of shape (degree, n_features), dtype=float32
    Random entries in {+1, -1} used for the Count Sketch.
  - n_features_in_: int
    Number of features seen during fit.
  - feature_names_in_: ndarray of shape (n_features_in_,)
    Names of features seen during fit (if provided).
    Defined only when X has string feature names.
  - _n_features_out: int
    The output dimensionality (n_components).

  Notes
  - If coef0 != 0, the feature space is expanded by 1 to account for the constant term.

  See Also
  --------
  AdditiveChi2Sampler, Nystroem, RBFSampler, SkewedChi2Sampler, sklearn.metrics.pairwise.kernel_metrics

Usage
-----
Example from the source:

>>> from sklearn.kernel_approximation import PolynomialCountSketch
>>> from sklearn.linear_model import SGDClassifier
>>> X = [[0, 0], [1, 1], [1, 0], [0, 1]]
>>> y = [0, 0, 1, 1]
>>> ps = PolynomialCountSketch(degree=3, random_state=1)
>>> X_features = ps.fit_transform(X)
>>> clf = SGDClassifier(max_iter=10, tol=1e-3)
>>> clf.fit(X_features, y)
SGDClassifier(max_iter=10)
>>> clf.score(X_features, y)
1.0

Details
-------
PolynomialCountSketch implements Tensor Sketch to approximate the feature map of the polynomial kernel
K(X, Y) = (gamma * <X, Y> + coef0)^degree. It uses a 2-wise independent hash function representation
for the Count Sketch (indexHash and bitHash) and leverages FFT-based convolution to obtain the
approximate feature mapping efficiently.

Important implementation notes
- gamma controls the scaling of the dot product.
- degree sets the polynomial degree of the kernel being approximated.
- coef0 adds a constant term to the kernel.
- n_components governs the dimensionality of the approximate feature space.
- random_state ensures reproducible hashing/signs for the sketch.

Validation and compatibility
- The _fit_context decorator is used for validation during fit.
- Works with dense and sparse input formats; supports feature name tracking when provided.
