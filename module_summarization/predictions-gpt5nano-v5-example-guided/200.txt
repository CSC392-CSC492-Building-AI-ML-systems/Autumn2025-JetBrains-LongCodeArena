Novelty and Outlier Detection
==============================

Overview
--------
This module covers techniques for identifying anomalous observations in data. It distinguishes between:
- Novelty detection: identifying new, unseen outliers relative to a model trained on normal (inlier) data. The model is trained on labeled or assumed-normal data, and is then used to judge new samples.
- Outlier detection: identifying anomalies within an unlabeled dataset, typically trained on the same data used for evaluation.

Key Concepts
------------
- Inliers vs. outliers: In novelty detection, inliers are the known normal observations seen during training; outliers are new, unseen anomalies. In outlier detection, all data may be considered for labeling anomalies.
- Training vs scoring: Models are fit on training data (often normal data for novelty detection). Scoring/prediction identifies which samples are outliers.
- Interfaces: Common APIs include fit, predict, decision_function, and score_samples. Different detectors may provide one or more of these methods to obtain binary labels or anomaly scores.

Key Components
--------------
- Novelty Detection Algorithms
  - OneClassSVM: One-class classification to identify normal vs. anomalous samples.
  - Local Outlier Factor (LOF): Density-based anomaly detection that flags samples with lower density than neighbors.
  - EllipticEnvelope: Parametric detector assuming an elliptically distributed data (Gaussian-like).
  - Minimum Covariance Determinant (MCD) variants: Robust covariance-based detectors.

- Outlier Detection Algorithms
  - IsolationForest: Ensemble method that isolates anomalies via random partitions.
  - EllipticEnvelope: As above, used for outlier detection in unsupervised settings.
  - Local Outlier Factor (LOF): Density-based, can be used for unsupervised outlier detection.

- Common APIs
  - fit(X, y=None, ...): Train the detector.
  - predict(X): Return labels (+1 for inliers, -1 for outliers) where supported.
  - decision_function(X): Return anomaly scores (higher scores typically mean inliers).
  - score_samples(X): Return per-sample anomaly scores.

Usage
-----
End-to-end examples for typical scenarios:

1) Novelty detection with OneClassSVM
- Train on normal data
- Score new samples to detect novelty

Code example:
.. code-block:: python
   :language: python

   from sklearn.svm import OneClassSVM

   # X_train contains only normal observations
   model = OneClassSVM(nu=0.05, kernel='rbf', gamma='scale')
   model.fit(X_train)

   # X_test contains both normal and novel observations
   pred = model.predict(X_test)           # 1 for inliers, -1 for outliers
   scores = model.decision_function(X_test)  # higher scores -> inliers

2) Outlier detection with IsolationForest
- Train on the full dataset (unsupervised)
- Identify anomalies in new data

Code example:
.. code-block:: python
   :language: python

   from sklearn.ensemble import IsolationForest

   clf = IsolationForest(
       n_estimators=200,
       max_samples='auto',
       contamination=0.05,
       random_state=42
   )
   clf.fit(X)

   # Predictions for anomalies in X
   pred = clf.predict(X)                 # -1 for outliers, 1 for inliers
   scores = clf.decision_function(X)     # higher scores -> inliers

3) Internal pattern: fitting estimators via a clone (advanced)
Some internal utilities use a pattern that clones the estimator and then fits it, ensuring clean, repeatable training behavior. This mirrors a common approach in meta-estimators.

Code example:
.. code-block:: python
   :language: python

   from sklearn.base import clone

   def _fit_estimator(estimator, X, y=None, sample_weight=None, **fit_params):
       estimator = clone(estimator)
       if sample_weight is not None:
           estimator.fit(X, y, sample_weight=sample_weight, **fit_params)
       else:
           estimator.fit(X, y, **fit_params)
       return estimator

Details
-------
- When to use novelty vs. outlier detection
  - Novelty detection is appropriate when you have a clean, representative training set of normal behavior and want to detect future deviations.
  - Outlier detection is suitable when the dataset contains anomalies and you want to identify them without prior labeling.

- Data preparation
  - Feature scaling can improve performance for distance- and density-based methods.
  - Robust methods (e.g., robust covariance detectors) can be more resilient to outliers in the training data.

- Evaluation
  - Use metrics that reflect anomaly detection quality (e.g., ROC-AUC, precision-recall, F1 for labeled contexts).
  - Be mindful of class imbalance and the chosen contamination or expected anomaly rate when interpreting scores and thresholds.

- Model selection tips
  - If anomalies are a rare and unknown proportion, ensemble methods like IsolationForest are often effective.
  - For monotonic or Gaussian-like data distributions, EllipticEnvelope or robust covariance detectors can perform well.
  - Local methods like LOF are useful when anomalies have a local density deviation rather than a global footprint.

API and Implementation Notes
----------------------------
- Typical methods across detectors:
  - fit(X, y=None, ...): Train the detector.
  - predict(X): Binary labels (+1 inliers, -1 outliers) when supported.
  - decision_function(X): Anomaly score; higher values indicate inliers.
  - score_samples(X): Per-sample anomaly scores (often similar to decision_function).

- Internal patterns
  - Some internal utilities, such as _fit_estimator, clone, and meta-estimator behavior, illustrate how detectors can be composed and trained in a consistent manner. See example above for a clone-based fit pattern.
