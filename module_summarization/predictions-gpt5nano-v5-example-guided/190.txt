Linear and Quadratic Discriminant Analysis
============================================

Overview
--------
Linear Discriminant Analysis (LDA) and Quadratic Discriminant Analysis (QDA) are probabilistic classifiers that model each class with Gaussian distributions and use Bayes’ rule to assign class labels. LDA assumes a shared covariance matrix across classes, resulting in linear decision boundaries. QDA allows each class to have its own covariance, producing quadratic decision boundaries. They provide a principled approach to classification when features are continuous and approximately Gaussian within each class.

Key Components
--------------
- LinearDiscriminantAnalysis (LDA): Classifier that uses a single, shared covariance matrix across all classes to compute linear decision boundaries.
- QuadraticDiscriminantAnalysis (QDA): Classifier that uses class-specific covariance matrices, resulting in quadratic decision boundaries.
- Common features:
  - fit(X, y): Learn class means, covariances (shared for LDA; per-class for QDA), and class priors.
  - predict(X): Assign class labels by maximizing the discriminant function.
  - priors: Optional class prior probabilities that can be provided or estimated from data.
  - store_covariance: Optional flag to store estimated covariance matrices in the fitted model.
  - shrinkage (LDA only): Regularization of the covariance estimate when using certain solvers.
- Differences between LDA and QDA:
  - LDA uses a pooled (shared) covariance Σ, leading to linear decision boundaries.
  - QDA uses Σ_k for each class k, leading to quadratic decision boundaries and potentially better flexibility when class covariances differ.

Usage
-----
Basic classification

.. code-block:: python

    from sklearn.discriminant_analysis import LinearDiscriminantAnalysis, QuadraticDiscriminantAnalysis

    # X: feature matrix, y: class labels
    X, y = ...  # prepare your data

    # Linear Discriminant Analysis
    lda = LinearDiscriminantAnalysis()
    lda.fit(X, y)
    y_pred_lda = lda.predict(X)

    # Quadratic Discriminant Analysis
    qda = QuadraticDiscriminantAnalysis()
    qda.fit(X, y)
    y_pred_qda = qda.predict(X)

Optional configuration
- priors: Specify class priors as a 1D array-like of length n_classes.
- store_covariance: If True, store the estimated covariance matrices in the fitted model (useful for inspection).
- shrinkage (LDA only): Regularization value (e.g., for the eigenvalue-based solver) to stabilize the covariance estimate.

Notes
- Use LDA when class covariances are similar across classes and you want a simpler, more robust model.
- Use QDA when class covariances differ substantially and you have enough data to estimate per-class covariances.

Details
-------
Assumptions
- Each class k is modeled as a multivariate Gaussian distribution with mean μ_k.
- LDA assumes a common covariance Σ across all classes.
- QDA allows a distinct covariance Σ_k for each class.

Discriminant functions

- Linear Discriminant Analysis (LDA)
  - Discriminant function for class k:
    δ_k(x) = x^T Σ^-1 μ_k − 0.5 μ_k^T Σ^-1 μ_k + log π_k
  - Decision rule: predict the class k with the largest δ_k(x).
  - Where:
    - μ_k is the mean vector for class k
    - Σ is the shared covariance matrix
    - π_k is the prior probability of class k

- Quadratic Discriminant Analysis (QDA)
  - Discriminant function for class k:
    δ_k(x) = −0.5 log|Σ_k| − 0.5 (x − μ_k)^T Σ_k^-1 (x − μ_k) + log π_k
  - Decision rule: predict the class k with the largest δ_k(x).
  - Where:
    - μ_k is the mean vector for class k
    - Σ_k is the covariance matrix for class k
    - π_k is the prior probability of class k

Covariance estimation and regularization
- LDA uses a pooled estimate of the covariance Σ, optionally regularized via shrinkage when supported by the solver.
- QDA estimates a separate covariance Σ_k for each class k.
- Priors π_k can be the observed class frequencies or user-specified.

Practical considerations
- Feature scaling can impact performance; standardize features when they have different scales.
- With high-dimensional data, QDA may require more data to reliably estimate per-class covariances.
- Both estimators provide a consistent API and integrate with scikit-learn’s cross-validation and pipelines.
