Generate documentation for this code
===================================

Overview
--------
This module generates a Sphinx-friendly docstring at import time. It builds a collection of hypergeometric expansion formulas (FormulaCollection), converts each expanded expression to LaTeX, and assembles them into a series of math blocks. The resulting string is assigned to __doc__, enabling automatic documentation of the hypergeometric expansions in Sphinx.

Key Components
-------------
- FormulaCollection: container holding formulae used to expand hypergeometric functions.
- Eq: equation constructor used to relate a hypergeometric form to its closed form.
- hyper: function to form hypergeometric expressions from parameters.
- latex: formatter that converts expressions to LaTeX for inclusion in docs.
- add_formulae: helper that populates a knowledge base of known formulas (0F0, 1F0, 2F1, etc.) and related representations.
- HyperRep_*: various wrappers used for representing hypergeometric expansions.
- The on-import doc-generation loop:
  - Iterates over c.formulae
  - Builds an equation object comparing hyper(f.func.ap, f.func.bq, f.z) to f.closed_form.rewrite('nonrepsmall')
  - Appends a corresponding LaTeX math directive to the doc string

Code example
------------
The following snippet illustrates the core doc-generation logic present in the module:

.. code-block:: python

    from ..core import Eq
    from ..functions import hyper
    from ..printing import latex
    from hyperexpand import FormulaCollection

    c = FormulaCollection()
    doc = ''

    for f in c.formulae:
        obj = Eq(hyper(f.func.ap, f.func.bq, f.z),
                 f.closed_form.rewrite('nonrepsmall'))
        doc += f'.. math::\n  {latex(obj)}\n'

    __doc__ = doc

Usage
-----
- Importing this module triggers on-the-fly generation of the documentation string:
  - The code builds a math block for each formula and concatenates them into a single docstring.
  - This docstring is then assigned to __doc__, which Sphinx can render as part of the documentation.

- This approach centralizes the generation of mathematical expressions for documentation, ensuring consistency between the implemented formulas and their displayed representations.

Details
-------
- The module includes an extensive comment block describing the expansion of Hypergeometric and Meijer G functions into named special functions, referencing Roach (1997) and related literature.
- It outlines a framework for extending to Meijer G functions with:
  - Shift operators (A_J, B_J, C_J, D_J) and their effects on parameters.
  - Order reduction criteria when parameters differ by non-negative integers.
  - The concept of index quadruples, reachability, and the notion of “suitable” (irreducible) quadruples.
- The code uses a helper _mod1 to bucket arguments by their fractional part, aiding the organization of formulae.
- The add_formulae function builds a knowledge base by registering base cases (0F0, 1F0, 2F1, etc.) and their hyper-representations, including matrix-based representations and logarithmic forms.
- This module serves both as a demonstration of dynamic documentation generation and as a repository of known hypergeometric expansions that can be reflected in the generated docs.
