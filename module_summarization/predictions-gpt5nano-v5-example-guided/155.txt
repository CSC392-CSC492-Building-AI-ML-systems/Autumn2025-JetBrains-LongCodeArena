Getting Started
===============

Overview
--------
This module defines functionality common to master and parent processes and is
sent to child contexts that are about to become a parent due to recursive
connections. It sets up core utilities, environment checks, and helpers that
are used when preparing new child contexts for execution.

Key Components
--------------
- get_log_level()
  - Returns the effective logging level for the module.
- get_sys_executable()
  - Returns sys.executable if set; otherwise returns '/usr/bin/python' and logs a one-time warning.
  - Example:
    .. code-block:: python

       from mitogen.getting_started import get_sys_executable
       exe = get_sys_executable()
       print(exe)
- _get_core_source()
  - Returns the Mitogen core source code. In master contexts this may be replaced
    with a minification-specific version; in non-master contexts it simply fetches
    the cached Mitogen core via the import mechanism.
- get_core_source_partial()
  - Lazily initializes and returns a PartialZlib-wrapped core source to avoid
    redundant work. Ensures thread-safety by guarding with a lock.
  - Example:
    .. code-block:: python

       from mitogen.getting_started import get_core_source_partial
       get_core_source_partial()  # initializes the cached partial core source
- _ioctl_cast(n)
  - Cast helper for Linux/BSD compatibility when preparing ioctl request codes.
  - Note: unsigned vs signed handling is performed for older Python versions.
- System/Environment Constants
  - LINUX_TIOCGPTN: PTY number ioctl request (cast for platform compatibility).
  - LINUX_TIOCSPTLCK: PTY lock ioctl request (cast for platform compatibility).
  - IS_LINUX: True if running on a Linux system.
  - SELINUX_ENABLED: True if SELinux enforcement is active; used to avoid PTY usage
    when appropriate.
  - SC_OPEN_MAX: Maximum number of file descriptors supported by the system.
  - BROKER_SHUTDOWN_MSG: Standard message for broker shutdown scenarios.
  - OPENPTY_MSG: Error message for PTY creation failures with guidance.
  - SYS_EXECUTABLE_MSG: Warning message if sys.executable is unset.
- Other Internal Helpers
  - _preexec_hook: Optional pre-exec hook invoked before exec() of a new child process.
  - SIGNAl_BY_NUM: Mapping of signal numbers to their names for debugging and logging.
  - _core_source_lock and _core_source_partial: Synchronization primitives for lazy core source minification.

Usage
-----
Getting started typically involves relying on these utilities when preparing a child
context in a master or parent process.

- Determine the Python executable to use for child contexts:
  .. code-block:: python

     from mitogen.getting_started import get_sys_executable
     exe = get_sys_executable()
     print("Using Python executable:", exe)

- Prepare the core source in a thread-safe, lazy manner (useful for pre-warming
  or when spawning multiple children):
  .. code-block:: python

     from mitogen.getting_started import get_core_source_partial
     get_core_source_partial()  # initializes the partial core source

- Be mindful of platform constraints:
  - SELINUX_ENABLED guards against PTY usage on SELinux-enabled systems.
  - The module defines PTY-related constants (LINUX_TIOCGPTN, LINUX_TIOCSPTLCK)
    for proper operation across Linux/BSD environments.

Details
-------
- This module is designed to support both master and child contexts with a focus
  on getting started quickly and safely. It includes compatibility helpers for
  Python 2.x and 3.x, such as xrange aliasing and closure/self-reference handling.
- It uses a small, centralized set of global state to coordinate expensive
  operations (like minifying the Mitogen core) across threads, ensuring that
  multiple threads do not redundantly perform the same work.
- The get_sys_executable function emits a one-time warning if Python’s executable
  location cannot be determined, defaulting to '/usr/bin/python' to preserve
  behavior when a proper path is unavailable.
- When starting new child processes, a _preexec_hook may be invoked (if set)
  to customize scheduling or environment preparation prior to exec().
- The module relies on Mitogen’s core facilities (e.g., PartialZlib) to efficiently
  transfer and reuse the core source across contexts.

Note
