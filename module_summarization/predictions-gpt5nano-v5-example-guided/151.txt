# General Plotting routines

## Overview
This module provides general and specialized Bokeh-based plotting utilities for interactive visualizations. It centralizes plot configuration through a single BokehPlotter instance and supplies convenience wrappers for common scatter plotting tasks. The key entry points are bokeh_scatter and bokeh_multi_scatter, supported by a set of helper functions to manage Bokeh defaults and metadata.

Core components are designed to:
- Consistently apply plotting parameters via a shared BokehPlotter
- Handle data wiring and alignment through process_data_arguments
- Offer a simple API for interactive visuals with sensible defaults and tooltips

Key imports and helpers:
- BokehPlotter: central plot parameter management
- ensure_plotter_consistency: decorator ensuring a consistent plotter is used
- process_data_arguments: standardizes input data into a suitable data source
- get_special_kpoint_ticks: helper for specialized axis ticks (imported for use in related routines)

Auxiliary utilities:
- set_bokeh_plot_defaults, reset_bokeh_plot_defaults, show_bokeh_plot_defaults
- get_bokeh_help, load_bokeh_defaults, save_bokeh_defaults

These utilities enable fine-grained control over the Bokeh backend and help preserve reproducibility of plots.

## Key Components

- plot_params
  - A global instance of BokehPlotter used to configure and render plots.

- Defaults management
  - set_bokeh_plot_defaults(**kwargs): update the default plotting parameters.
  - reset_bokeh_plot_defaults(): reset to hardcoded defaults.
  - show_bokeh_plot_defaults(): print the current defaults.
  - get_bokeh_help(key): show description for a specific default key.
  - load_bokeh_defaults(filename='plot_bokeh_defaults.json'): load defaults from a JSON file.
  - save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False): save current defaults to a JSON file.

- General plotting routines
  - bokeh_scatter(...)
    - Creates an interactive scatter plot with Bokeh.
    - Data handling: accepts x and y as data arrays or data source keys. If first positional argument is a data source (dict, DataFrame, or ColumnDataSource), a deprecation warning is emitted and the function adapts to use data as the source with xdata/ydata for axes.
    - Data plumbing: uses process_data_arguments to construct a single data source suitable for Bokeh.
    - Plot construction: prepares the figure, generates plot keywords via the BokehPlotter, draws the scatter, attaches tooltips, applies axis limits, and saves/draws as configured.
    - Returns: the created Bokeh figure.
  - bokeh_multi_scatter(...)
    - Similar to bokeh_scatter but supports multiple data sets to be displayed in a single figure.
    - Follows the same data handling via process_data_arguments and uses the BokehPlotter for defaults and styling.
    - Returns: the created Bokeh figure.

Note: Both bokeh_scatter and bokeh_multi_scatter are decorated with ensure_plotter_consistency(plot_params) to ensure the internal plotter state remains consistent across calls.

## Usage

Typical workflow:
- Configure defaults (optional)
  - set_bokeh_plot_defaults(color='navy', tooltips=[...], etc.)
  - save or load defaults for reproducibility
- Prepare data
  - Provide data as a pandas DataFrame or as separate x/y arrays, or pass a single data source via the data parameter.
- Create plots
  - bokeh_scatter(x='x', y='y', data=df, xlabel='X', ylabel='Y', title='My Scatter')
  - bokeh_multi_scatter(x='time', y=['val1', 'val2'], data=df, copy_data=True, set_default_legend=True)

Code examples (from the module):

- Single scatter
  - from masci_tools.plotting import bokeh_scatter
  - p = bokeh_scatter(x='x', y='y', data=df, xlabel='X', ylabel='Y', title='My Scatter', saveas='scatter_plot')
  - This returns a Bokeh Figure object.

- Defaults management
  - set_bokeh_plot_defaults(grid=True, tools="pan,reset,box_select,lasso_select")
  - show_bokeh_plot_defaults()
  - save_bokeh_defaults(filename='plot_bokeh_defaults.json')
  - load_bokeh_defaults(filename='plot_bokeh_defaults.json')

Important notes:
- If you pass the data source as the first positional argument to bokeh_scatter, a DeprecationWarning is emitted. The preferred usage is to pass the data via the data parameter and specify xdata/ydata through the function arguments (or via x and y as data keys).
- The internal data handling via process_data_arguments ensures that x and y coordinates align in length and are ready for a Bokeh data source.

## Details

- Data handling
  - process_data_arguments(data=data, x=x, y=y, copy_data=copy_data, single_plot=True, same_length=True, use_column_source=True) is used to prepare a consistent data source from various input forms.
  - The function returns a mapping of data fields (entry.x, entry.y) and the associated data source for Bokeh plotting.

- Plot customization
  - plot_params (BokehPlotter) centralizes plot styling, defaults, tooltips, and axis limits.
  - The plotting pipeline typically includes: prepare_figure, plot_kwargs (for scatter), scatter call, optional tooltips, level handling, drawing of straight lines, axis limits, and saving the figure.

- Extensibility
  - The structure supports adding more general or specialized plotting helpers by reusing the same BokehPlotter instance and the same data processing workflow, ensuring consistency across plots.

- Dependencies
  - pandas, numpy, warnings, pprint, and Bokeh (via BokehPlotter) are required.
  - The code relies on Masci-tools’ internal utilities (process_data_arguments, BokehPlotter, etc.) for consistent behavior.
