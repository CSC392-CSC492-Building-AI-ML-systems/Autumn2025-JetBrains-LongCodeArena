# w_kinavg

## Overview
w_kinavg is a deprecated shim that provides a backward-compatible interface for achieving averages and confidence intervals (CIs) for path-tracing kinetics analysis. It wraps the functionality of the w_direct command by exposing a WKinAvg subcommand and delegating execution through a WDirect master command.

The shim ensures compatibility with older workflows by defaulting to trace-mode analysis and using conventional input/output file names.

Key code paths are exposed in the source as:
- WKinAvg: a subclass of DKinAvg with a fixed subcommand and defaults.
- WDirect: a WESTMasterCommand/WESTParallelTool entry point that aggregates subcommands.
- entry_point: a compatibility wrapper that emits a deprecation warning and forwards to WDirect.

Example code (from the source):
```python
class WKinAvg(DKinAvg):
    subcommand = 'trace'
    help_text = 'averages and CIs for path-tracing kinetics analysis'
    default_kinetics_file = 'kintrace.h5'
    default_output_file = 'kinavg.h5'


class WDirect(WESTMasterCommand, WESTParallelTool):
    prog = 'w_kinavg'
    subcommands = [WKinAvg]
    subparsers_title = 'direct kinetics analysis schemes'
    description = '''\
Calculate average rates and associated errors from weighted ensemble data. Bin
assignments (usually "assignments.h5") and kinetics data (usually
"kintrace.h5" or "kinmat.h5") data files must have been previously generated
(see "w_assign --help" and "w_kinetics --help" for information on generating
these files).

... (long description of output formats, datasets, and attributes) ...
'''


def entry_point():
    warn('{} is being deprecated.  Please use w_direct instead.'.format(WDirect.prog))
    import sys

    try:
        if sys.argv[1]!= 'trace':
            sys.argv.insert(1, 'trace')
    except Exception:
        sys.argv.insert(1, 'trace')
    WDirect().main()
```

## Usage
- Basic usage
  - w_kinavg (defaults to trace mode due to the entry_point shim)
  - w_kinavg trace (explicitly selects the trace subcommand)
  - w_kinavg --help (displays help for the deprecated shim and underlying w_direct interface)
- Notes
  - This module is deprecated; users should migrate to using w_direct directly.
  - The shim maps to the same core functionality used by w_direct for direct kinetics analysis.

## Details
Output and data organization are as described in the long description of WDirect:

- Output format
  - /avg_rates [state,state]
    - Structured dataset containing state-to-state rates over the selected window of iterations.

- In trace mode, additional datasets are produced:
  - /avg_total_fluxes [state]
  - /avg_conditional_fluxes [state,state]

- Evolution-mode support (when specified):
  - /rate_evolution [window][state][state]
    - State-to-state rates across windows of varying widths.
  - /target_flux_evolution [window,state]
  - /conditional_flux_evolution [window,state,state]

- Structure of these datasets
  - iter_start (Integer)
    Iteration at which the averaging window begins (inclusive).
  - iter_stop (Integer)
    Iteration at which the averaging window ends (exclusive).
  - expected (Floating-point)
    Mean rate within the window, in units of inverse tau.
  - ci_lbound (Floating-point)
    Lower bound of the confidence interval for the rate in the window.
  - ci_ubound (Floating-point)
    Upper bound of the confidence interval for the rate in the window.
  - corr_len (Integer)
    Correlation length of the rate within the window, in units of tau.

- Attributes stamped on each dataset
  - mcbs_alpha (Floating-point)
    Alpha value for the confidence intervals (e.g., alpha=0.05 for 95% CI).
  - mcbs_nsets (Integer)
    Number of bootstrap datasets used for CI estimation.
  - mcbs_acalpha (Floating-point)
    Alpha value for determining correlation lengths.

- Input/Output files
  - Kinetics input: kintrace.h5 (default)
  - Output: kinavg.h5 (default)

- Compatibility note
  - The module exists primarily to preserve older workflows; direct usage of w_direct is recommended going forward.
