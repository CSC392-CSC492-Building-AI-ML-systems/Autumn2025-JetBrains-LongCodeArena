# Three types of algorithms for the estimation of the parameters of a model

## Overview
This module provides an Optimizer to fit an objective function (e.g., a log-likelihood) with a variety of optimization algorithms. It supports gradient-based local methods, gradient-free local methods, and global optimization approaches, enabling flexible parameter estimation for different modeling scenarios.

Key entry point: Optimizer._fit(objective, gradient, start_params, fargs, kwargs, …, method, …)

## Key Components
- _check_method(method, methods): Validates the selected solver against the supported list.
- Optimizer class with _fit: Core fitting routine that delegates to SciPy solvers. Signature excerpt:

  - start_params: initial parameter guess
  - method: solver name; options include 'newton', 'nm', 'bfgs', 'powell', 'cg', 'ncg', 'basinhopping', 'minimize'
  - maxiter, full_output, disp, callback, retall: solver-specific controls
  - fargs: extra args passed to the objective (e.g., loglike(x, *args))
  - gradient (and optional hessian): user-supplied gradient and Hessian if available
  - Returns xopt, retvals, optim_settings

- Supported solvers (and general purpose): 
  - 'newton' (Newton-Raphson)
  - 'nm' (Nelder-Mead)
  - 'bfgs' (Broyden-Fletcher-Goldfarb-Shanno)
  - 'powell' (Powell's method)
  - 'cg' (conjugate gradient)
  - 'ncg' (Newton-conjugate gradient)
  - 'basinhopping' (global basin-hopping)
  - 'minimize' (wrapper around scipy.optimize.minimize)

- Notes: The basinhopping solver ignores maxiter, retall, and full_output explicit arguments.

## Usage
- Prepare:

  - gradient: gradient function (score) or None

## Details
Three algorithmic categories implemented by the available methods:

- Local gradient-based optimization (derivative-based, fast convergence for smooth objectives)
  - Methods: 'newton', 'bfgs', 'cg', 'ncg'
  - Characteristics:
    - Require gradient (and sometimes Hessian)
    - Efficient for well-behaved problems
    - Examples of usage:
      - Newton-Raphson: rapid convergence near the optimum when Hessian information is available
      - BFGS: robust quasi-Newton updating without requiring Hessian
      - CG / NCG: gradient-based with conjugate gradient variants
  - Notes:
    - Hessian (hessian) can be supplied; otherwise may be approximated
    - Parameter controls (tol/gtol/epsilon) are solver-specific

- Local gradient-free optimization (derivative-free, useful when gradients are unavailable or noisy)
  - Methods: 'nm' (Nelder-Mead), 'powell'
  - Characteristics:
    - Do not require gradient information
    - More robust to non-smooth landscapes but may be slower
  - Notes:
    - Provide xtol/ftol/maxfun and related options to control convergence

- Global optimization (exploration of parameter space to avoid local minima)
  - Methods: 'basinhopping'
  - Characteristics:
    - Global search strategy that can escape local minima
    - Often combined with a local minimizer for refinement
  - Notes:
    - Basinhopping ignores maxiter, retall, and full_output arguments
    - Configurable through niter, niter_success, temperature T, stepsize, interval, and minimizer options

- General-purpose wrapper
  - 'minimize': a wrapper around scipy.optimize.minimize that can dispatch to various underlying solvers
  - Usage follows SciPy conventions; arguments are solver-specific

- Parameters and behavior
  - maxiter: maximum iterations (solver-dependent)
  - full_output: when True, returns a dictionary of solver output in mle_retvals
  - disp: print convergence messages
  - retall: return the list of solutions at each iteration (if supported)
  - retvals: solver output when full_output is True
  - optim_settings: dictionary of parameters passed to the solver
 
- Example snippet (structure)
  - def _fit(self, objective, gradient, start_params, fargs, kwargs, hessian=None, method='newton', maxiter=100, full_output=True, disp=True, callback=None, retall=False)

- Practical guidance
  - For well-behaved models with reliable gradient/Hessian, prefer gradient-based local methods ('newton', 'bfgs', 'cg', 'ncg')
  - For non-smooth objectives or when gradients are unavailable, consider gradient-free methods ('nm', 'powell')
  - To mitigate local minima risk, use global optimization ('basinhopping') or pair global search with a local optimizer
  - When in doubt, start with 'bfgs' or 'cg' and switch to 'basinhopping' if convergence is problematic

- Source reference (core API)
  - The module exposes the Optimizer class and its _fit method, which orchestrates solver selection and argument handling. Example supported methods are enumerated in the docstring and code, including 'newton', 'nm', 'bfgs', 'powell', 'cg', 'ncg', 'basinhopping', and 'minimize'.
