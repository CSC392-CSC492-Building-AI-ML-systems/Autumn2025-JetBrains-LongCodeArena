# Isotonic Regression

## Overview
Isotonic regression enforces a monotonic relationship between input values and targets by projecting data onto the set of monotone functions. The implementation supports optional sample weights, value bounds, and automatic determination of monotonicity direction. It is available as a functional interface and as an estimator object.

Key features:
- Monotone increasing or decreasing fits (with an auto option).
- Optional bounds on predicted values (y_min, y_max).
- Handling of out-of-bounds inputs during prediction (nan, clip, or raise).
- Support for sample weights.
- Internal use of efficient isotonic projection algorithms.

Core components implemented in isotonic.rst:
- check_increasing: determine whether y is monotonically correlated with x using Spearman correlation.
- isotonic_regression: compute the isotonic fit for a 1D target y.
- IsotonicRegression: estimator wrapper with fit/predict-like interface and configurable out-of-bounds behavior.

References:
- Active set algorithms for isotonic regression (Best & Chakravarti, section 3).

## Key Components

- check_increasing(x, y)
  - Determines whether y is monotonically increasing with respect to x (or decreasing) based on Spearman correlation.
  - Returns a boolean indicating increasing vs decreasing.
  - Warns if the 95% confidence interval for Spearman's rho spans zero.

- isotonic_regression(y, *, sample_weight=None, y_min=None, y_max=None, increasing=True)
  - Solves the isotonic regression problem for a 1D target y.
  - Parameters:
    - y: 1D array-like of shape (n_samples,)
    - sample_weight: array-like of shape (n_samples,), default None (weights = 1)
    - y_min: lower bound on the minimum predicted value (default None => -inf)
    - y_max: upper bound on the maximum predicted value (default None => +inf)
    - increasing: if True (default) fit is increasing; if False, fit is decreasing
  - Returns:
    - y_: ndarray of shape (n_samples,) containing the isotonic fit
  - Implementation details:
    - Validates input, handles weights, applies the isotonic projection in-place, and clips to bounds if provided.

- IsotonicRegression
  - Estimator wrapper (RegressorMixin, TransformerMixin, BaseEstimator).
  - Parameters:
    - y_min: lower bound for the minimum predicted value (default=None => -inf)
    - y_max: upper bound for the maximum predicted value (default=None => +inf)
    - increasing: bool or 'auto' (default=True). Determines monotonic direction; 'auto' decides based on Spearman rho from check_increasing.
    - out_of_bounds: {'nan', 'clip', 'raise'}, default 'nan'. Specifies how to handle X values outside the training domain during prediction.
  - Attributes (post-fit):
    - X_min_: minimum value of input X_ used to determine the left bound
    - X_max_: maximum value of input X_ used to determine the right bound
    - X_thresholds_: unique ascending X values used to interpolate the monotone function
    - y_thresholds_: corresponding thresholds for the monotone function
  - Notes:
    - The estimator leverages the same core isotonic projection as isotonic_regression, with an interface compatible with scikit-learn pipelines and cross-validation workflows.

## Usage

1) Quick monotone fit for 1D targets

- Determine monotonicity direction automatically
- Compute the isotonic fit with optional bounds

Example:
```python
import numpy as np
from sklearn.isotonic import check_increasing, isotonic_regression

x = np.array([0.0, 2.0, 1.0, 3.0, 4.0])
y = np.array([0.0, 2.1, 1.9, 3.8, 4.2])

# Determine whether y increases with x
increasing = check_increasing(x, y)

# Isotonic fit (direction determined by 'increasing')
y_fit = isotonic_regression(y, increasing=increasing)

# With bounds
y_fit_bounded = isotonic_regression(y, increasing=increasing, y_min=0.0, y_max=5.0)
```

2) Estimator interface (fit/predict)

Example:
```python
import numpy as np
from sklearn.isotonic import IsotonicRegression

X = np.array([0.0, 1.0, 2.0, 3.0]).reshape(-1, 1)
y = np.array([0.0, 2.0, 1.0, 3.0])

ir = IsotonicRegression(increasing='auto', y_min=None, y_max=None, out_of_bounds='nan')
ir.fit(X, y)
y_pred = ir.predict(X)
```

Notes:
- out_of_bounds controls handling of predictions for X values outside the training domain:
  - 'nan': predictions are NaN
  - 'clip': predictions are clipped to the nearest training interval endpoint
  - 'raise': raises a ValueError

## Details

- Input validation and types:
  - y must be a 1D array-like of real numbers (float32/float64).
  - sample_weight, if provided, must be compatible with y in length.
  - y_min/y_max accept real numbers or None; when None, bounds default to -inf/+inf.

- Monotonic direction:
  - increasing can be True, False, or 'auto'. If 'auto', the direction is decided by the sign of the Spearman correlation between x and y (via check_increasing).

- Bounds handling:
  - If y_min or y_max is provided, isotonic_regression clips the result to [y_min, y_max] after the isotonic projection.

- Algorithm:
  - The core projection uses an in-place contiguous isotonic regression routine (_inplace_contiguous_isotonic_regression). For the class-based API, this aligns with a transparent fit/predict workflow.
  - The Best & Chakravarti framework is referenced for the underlying methodology (section 3 of the cited work).

- References:
  - Best, M. J., & Chakravarti, N. (article on Active set algorithms for isotonic regression).
  - Fisher transformation context is used in check_increasing for CI awareness.
