Create Cradmin menu system
==========================

Overview
--------
This module provides a set of renderable classes used to build the Cradmin header menu in Django applications. It integrates with the Cradmin renderable system and uses BEM-style CSS classes to structure menu items, expandable menus, separators, and the menu toggle. Templates are defined for each renderable to control HTML output.

Key Components
--------------

- get_default_expandable_menu_renderable
  - Returns the default expandable menu renderable instance based on the DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS setting, or None if not configured.
  - Code example:
    ```python
    def get_default_expandable_menu_renderable(**kwargs):
        """
        Get the default expandable menu renderable.

        The one set in the :setting:`DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS`.

        Args:
            **kwargs: Kwargs for the expandable menu class constructor.

        Returns:
            AbstractMenuRenderable: Expandable menu renderable object or ``None``.
        """
        if not getattr(settings, 'DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS', None):
            return None
        menu_class = import_string(settings.DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS)
        return menu_class(**kwargs)
    ```
  - Uses import_string on the configured class path to instantiate the menu renderable.

- BaseMenuLinkRenderable
  - Base class for link-like menu items. Renders as a link item with label and URL and supports an active state.
  - Key attributes: label, url, is_active, parent_bem_block.
  - BEM element is derived from the parent block (default: adminui-page-header → __navlink).
  - Active state adds the "active" BEM variant.
  - Code excerpt:
    ```python
    class BaseMenuLinkRenderable(renderable.AbstractBemRenderable):
        template_name = 'django_cradmin/crmenu/menuitem/link.django.html'

        def __init__(self, label, url, is_active=False,
                     bem_variant_list=None, parent_bem_block=None,
                     **kwargs):
            """
            Parameters:
                label: A label shown in the menu.
                url: The url to go to whem the user clicks the menu item.
                is_active: Should be ``True`` if the menuitem should be styled as active.
            """
            self.label = label
            self.url = url
            self.is_active = is_active
            self.parent_bem_block = parent_bem_block
            super(BaseMenuLinkRenderable, self).__init__(bem_variant_list=bem_variant_list, **kwargs)

        def get_fallback_parent_bem_block(self):
            return 'adminui-page-header'

        def get_parent_bem_block(self):
            return self.parent_bem_block or self.get_fallback_parent_bem_block()

        def get_bem_element(self):
            return '{}__navlink'.format(self.get_parent_bem_block())

        def get_active_bem_variant_list(self):
            return [
                'active'
            ]

        def get_bem_variant_list(self):
            bem_variant_list = super(BaseMenuLinkRenderable, self).get_bem_variant_list()
            if self.is_active:
                bem_variant_list.extend(self.get_active_bem_variant_list())
            return bem_variant_list
    ```

- NavLinkItemRenderable
  - Simple renderable used to add navigation links to the main menu.
  - Inherits from BaseMenuLinkRenderable.

- NavLinkButtonItemRenderable
  - Subclass of NavLinkItemRenderable for links that should be rendered as a button.
  - Overrides the BEM variant to include "button".
  - Code excerpt:
    ```python
    class NavLinkButtonItemRenderable(NavLinkItemRenderable):
        """
        Use this to add links styled as a button to the main menu.
        """
        def get_bem_variant_list(self):
            return ['button']
    ```

- ExpandableMenuItem
  - Represents an item that toggles an expandable submenu in the header.
  - Uses its own template: 'django_cradmin/crmenu/menuitem/expandable-menu-item.django.html'.
  - BEM blocks:
    - Fallback parent: 'expandable-menu'
    - Link element: '{}__link'
    - List item element: '{}__listitem'
  - Code excerpt:
    ```python
    class ExpandableMenuItem(BaseMenuLinkRenderable):
        """
        Use this to add links to the main menu.
        """
        template_name = 'django_cradmin/crmenu/menuitem/expandable-menu-item.django.html'

        def get_fallback_parent_bem_block(self):
            return 'expandable-menu'

        def get_bem_element(self):
            return '{}__link'.format(self.get_parent_bem_block())

        def get_list_element_bem_element(self):
            return '{}__listitem'.format(self.get_parent_bem_block())
    ```

- ExpandableMenuSeparator
  - Renderable for a separator within an expandable menu.
  - Template: 'django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html'
  - Manages its own parent block with a fallback to 'expandable-menu'.
  - Code excerpt:
    ```python
    class ExpandableMenuSeparator(renderable.AbstractBemRenderable):
        template_name = 'django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html'

        def __init__(self, parent_bem_block=None, **kwargs):
            self.parent_bem_block = parent_bem_block
            super(ExpandableMenuSeparator, self).__init__()

        def get_fallback_parent_bem_block(self):
            return 'expandable-menu'

        def get_parent_bem_block(self):
            return self.parent_bem_block or self.get_fallback_parent_bem_block()

        def get_bem_element(self):
            return '{}__separator'.format(self.get_parent_bem_block())
    ```

- MenuToggleItemItemRenderable
  - Renderable for the expandable menu toggle control in the header.
  - Template: 'django_cradmin/crmenu/menuitem/menutoggle.django.html'
  - Provides label, icons, and BEM class names for the toggle.
  - Uses translation via pgettext_lazy for the default label.
  - Code excerpt:
    ```python
    class MenuToggleItemItemRenderable(renderable.AbstractBemRenderable):
        """
        Use this to add an expandable menu toggle to the menu.
        """
        template_name = 'django_cradmin/crmenu/menuitem/menutoggle.django.html'

        def __init__(self, parent_bem_block=None, **kwargs):
            self.parent_bem_block = parent_bem_block
            super(MenuToggleItemItemRenderable, self).__init__()

        def get_label(self):
            return pgettext_lazy('cradmin default header menu toggle label', 'Menu')

        def get_fallback_parent_bem_block(self):
            return 'adminui-page-header'

        def get_parent_bem_block(self):
            return self.parent_bem_block or self.get_fallback_parent_bem_block()

        def get_bem_element(self):
            return '{}__navmenutoggle'.format(self.get_parent_bem_block())

        def get_label_css_class(self):
            return '{}__navmenutoggle-label'.format(self.get_parent_bem_block())

        def get_icon_css_class(self):
            return '{}__navmenutoggle-icon'.format(self.get_parent_bem_block())
    ```

    - button_renderable_class
  - Methods:
    - __init__(request, cradmin_instance=None)
    - iter_renderables()
    - get_link_renderable_class()

    - get_separator_renderable_class()

Usage
-----

- Configuring a default expandable menu
  - Set the class path for the default expandable menu in Django settings:
    - DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS
  - Retrieve an instance in code:
    ```python
    from django_cradmin.crmenu import get_default_expandable_menu_renderable

    menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin)
    ```
  - This allows a centralized, configurable menu structure to be injected where the header is rendered.

- Building a custom menu
  - Create a concrete subclass of AbstractMenuRenderable and assign the renderable classes you want to use:
    ```python
    class MyMenuRenderable(AbstractMenuRenderable):
        link_renderable_class = NavLinkItemRenderable
        separator_renderable_class = ExpandableMenuSeparator
        button_renderable_class = NavLinkButtonItemRenderable

        def __init__(self, request, cradmin_instance=None):
            super().__init__(request, cradmin_instance)
            self.renderable_list = [
                NavLinkItemRenderable(label='Dashboard', url='/admin/dashboard/', is_active=True),
                ExpandableMenuItem(label='Users', url='/admin/users/'),
            ]
    ```
  - Use the menu renderable in your templates where Cradmin renders the header, passing the request and current Cradmin instance as needed.

Details
-------

- Template-driven rendering
  - Each renderable specifies a template_name, determining the exact HTML structure. Examples include:
    - Base link item: 'django_cradmin/crmenu/menuitem/link.django.html'
    - Expandable menu item: 'django_cradmin/crmenu/menuitem/expandable-menu-item.django.html'
    - Expandable menu separator: 'django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html'
    - Menu toggle: 'django_cradmin/crmenu/menuitem/menutoggle.django.html'
  - The actual HTML output is controlled by these templates and the BEM CSS classes provided by the renderables.

- Styling and accessibility
  - BEM blocks and elements are used for predictable styling hooks.
  - Active state is surfaced via an "active" BEM variant on link renderables.

- Translation support
  - Menu toggle label text is translated using pgettext_lazy to provide context-sensitive translations.

- Extensibility
  - The design allows substituting alternative renderables and templates to customize the menu's appearance and behavior without changing the core logic.

Examples
--------

- Retrieve and render the default expandable menu (when configured):
  ```python
  # settings.py
  DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.menus.MyExpandableMenu'

  # usage in a view or context where the header is built
  menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin)
  # The 'menu' object can then be used in templates to render the header menu
  ```

- Build a minimal custom menu:
  ```python
  from django_cradmin.crmenu import AbstractMenuRenderable, NavLinkItemRenderable, ExpandableMenuItem

  class MyMenuRenderable(AbstractMenuRenderable):
      link_renderable_class = NavLinkItemRenderable
      def __init__(self, request, cradmin_instance=None):
          super().__init__(request, cradmin_instance)
          self.renderable_list = [
              NavLinkItemRenderable(label='Home', url='/admin/home/', is_active=True),
              ExpandableMenuItem(label='Users', url='/admin/users/'),
          ]
  ```

Notes
-----
- The crmenu system is designed to be flexible and template-driven, enabling custom HTML while providing sensible defaults for common Cradmin header menu patterns.
