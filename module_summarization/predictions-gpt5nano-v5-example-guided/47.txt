Contingency Tables
===================

Overview
--------
This module provides classes and functions for analyzing two-way contingency tables
(i.e., frequency tables of observations cross-classified by two categorical
variables). The main entities are:

- Table: operations that can be applied to any two-way contingency table.
- SquareTable: methods applicable to square two-way contingency tables.
- Table2x2: utilities for 2x2 contingency tables.
- StratifiedTable: tools for a collection of 2x2 tables.
- Additional functionality for conducting McNemar's test and Cochran's q test.

Helper routines include utilities for preparing data (e.g., converting to square form)
and a simple container for results.

Key components
--------------
- _make_df_square(table)
  - Reindexes a pandas DataFrame to become square by ensuring the row and column
    indices contain the same values in the same order. If necessary, extends the
    index to include all values appearing in either the row or column labels.

  Example behavior (from code):
  - If the input is not a DataFrame, it is returned unchanged.
  - If the DataFrame is not square, it computes the union of row and column labels,
    sorts them, and reindexes both axes with fill_value=0. It then aligns the row and
    column order to be identical.

- _Bunch
  - Lightweight container used to return results from analyses with a readable string
    representation.

  The class provides:
  - __repr__ and __str__ for a compact, readable display of results.

- Table
  A two-way contingency table with convenience methods and data handling.

  Construction
  - __init__(self, table, shift_zeros=True)
    - Accepts a 2D array-like contingency table.
    - If shift_zeros is True and any cell is zero, all zero values are replaced with 0.5
      to avoid issues with certain statistical procedures.

  - table_orig
    - The original input table cached as table_orig.

  - table
    - The data converted to a NumPy array of float64 for analysis.

  - from_data(cls, data, shift_zeros=True)
    - Classmethod to construct a Table from raw data by creating a contingency table
      from the first two columns using pandas crosstab.

    Example (from code documentation):
    - If data is a DataFrame, uses data.iloc[:, 0] and data.iloc[:, 1] to form the cross-tabulation.
    - Otherwise uses data[:, 0] and data[:, 1].

  - __str__(self)
    - Returns a string describing the 2D contingency table and its counts.

  - test_nominal_association(self)
    - Assesses independence for nominal (unordered) factors using a chi-squared test.
    - statistic: sum of chi-square contributions (chi2_contribs)
    - df: degrees of freedom (product of (rows - 1) and (cols - 1))
    - pvalue: chi-squared p-value

    Returns a _Bunch with attributes: statistic, df, pvalue.

  - test_ordinal_association(self, row_scores=None, col_scores=None)
    - Assesses independence for ordinal (ordered) factors using a linear-by-linear
      (Cochran–Armitage) trend test.
    - row_scores: numeric scores for the row categories (default: 0..n-1)
    - col_scores: numeric scores for the column categories (default: 0..m-1)

    Returns a _Bunch with attributes:
    - statistic: the test statistic
    - null_mean: expected value under the null hypothesis
    - null_sd: standard deviation under the null hypothesis
    - zscore: the Z-statistic for the test
    - pvalue: the p-value for the test

    Notes
    - The default scores (0..n-1 and 0..m-1) yield the Cochran–Armitage trend test.
    - The scores define the alternative along which the test has most power.

Usage
-----
Below are representative usage patterns illustrating how to work with this module.

Prerequisites
- Requires NumPy, SciPy, pandas, and Statsmodels (for decorations and utilities).

Example 1: Constructing a Table
.. code-block:: python

    from contingency_tables import Table
    # A sample 3x3 contingency table
    tbl = [[10, 0, 3],
           [0, 5, 2],
           [1, 4, 0]]
    t = Table(tbl)

    print(t)

Example 2: Handling zeros (shifts zeros to avoid zero-cell issues)
.. code-block:: python

    t_shift = Table(tbl, shift_zeros=True)
    print(t_shift)

Example 3: Creating a Table from data
.. code-block:: python

    import pandas as pd
    data = pd.DataFrame({
        'A': ['x', 'x', 'y', 'y', 'x'],
        'B': ['u', 'v', 'u', 'u', 'v']
    })
    t_from_data = Table.from_data(data)
    print(t_from_data)

Example 4: Nominal association test
.. code-block:: python

    res = t.test_nominal_association()
    print("statistic:", res.statistic)
    print("df:", res.df)
    print("p-value:", res.pvalue)

Example 5: Ordinal association test (Cochran–Armitage trend test)
.. code-block:: python

    row_scores = [0, 1, 2]
    col_scores = [0, 1, 2]
    res_ord = t.test_ordinal_association(row_scores=row_scores, col_scores=col_scores)
    print("statistic:", res_ord.statistic)
    print("p-value:", res_ord.pvalue)

Details
-------
- The underlying inference procedures assume independent and identically distributed
  units, each classified with respect to two categorical variables.
- The module is designed to work with standard contingency-table data and provides
  a streamlined interface for common tests of association between the two factors.
- The test_nominal_association method uses the chi-squared distribution with df = (rows - 1) * (cols - 1).
- The test_ordinal_association method targets ordered alternatives by leveraging provided
  row and column scores; by default, it implements the Cochran–Armitage trend test.

Notes
-----
- The internal helper _make_df_square is used to ensure square DataFrames when needed.
- The _Bunch class provides a compact, attribute-accessible results container with a
  readable string representation.
