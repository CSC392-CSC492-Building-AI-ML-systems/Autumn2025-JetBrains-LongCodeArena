# Working with assignments and allocations

## Overview
This module provides a session-based interface for interacting with an ftrack server, enabling operations related to assignments and allocations through the ftrack API. The core components are a Session class for managing server connections, authentication handling, and optional features such as auto-populated entities, caching, event hub integration, and plugin support. While the sample code focuses on session management and authentication, it underpins working with assignments and allocations by providing authenticated, stateful access to the ftrack server where such resources are managed.

Key capabilities include:
- Secure HTTP authentication via headers injected automatically into requests
- Configurable server connection using explicit arguments or environment variables
- Lazy loading of entity attributes (auto_populate)
- Caching support with pluggable key makers
- Optional embedded event hub connection for non-local events
- Schema caching and plugin argument handling

## Key Components

- SessionAuthentication
  - Attaches ftrack authentication information to HTTP requests.
  - Example from the source:

    ```
    class SessionAuthentication(requests.auth.AuthBase):
        '''Attach ftrack session authentication information to requests.'''

        def __init__(self, api_key, api_user):
            '''Initialise with *api_key* and *api_user*.'''
            self.api_key = api_key
            self.api_user = api_user
            super(SessionAuthentication, self).__init__()

        def __call__(self, request):
            '''Modify *request* to have appropriate headers.'''
            request.headers.update({
                'ftrack-api-key': self.api_key,
                'ftrack-user': self.api_user
            })
            return request
    ```

- Session
  - An isolated session for interaction with an ftrack server.
  - Core features:
    - Server URL resolution
      - If *server_url* is not provided, the value is read from the environment variable FTRACK_SERVER.
      - If still not provided, a TypeError is raised.
    - API credentials
      - *api_key* and *api_user* can be provided directly or sourced from environment variables FTRACK_API_KEY (or FTRACK_APIKEY for backward compatibility) and FTRACK_API_USER.
      - If not provided, a TypeError is raised.
    - Auto-population of entities
      - If *auto_populate* is True (default), accessing an entity’s attribute will trigger a fetch from the server if not already loaded.
    - Cache support
      - Accepts a cache instance or a callable that returns a cache; uses a top-level MemoryCache internally.
      - Uses a *cache_key_maker* (default: StringKeyMaker) to generate cache keys.
    - Event hub
      - If *auto_connect_event_hub* is True, an embedded event hub can publish/subscribe to non-local events. The hub runs in a background thread to improve startup time.
    - Schema caching
      - Optional schema caching controlled by *schema_cache_path*. If not set, environment variable FTRACK_API_SCHEMA_CACHE_PATH is used. Set to False to disable schema caching.
    - Plugin arguments
      - *plugin_arguments* allows passing keyword arguments to plugin discovery; compatibility handling is provided for plugins with differing signatures.

## Usage

- Create a session with explicit credentials:

  ```
  from ftrack_api.assignments_and_allocations import Session
  session = Session(
      server_url='https://ftrack.example.com',
      api_key='YOUR_API_KEY',
      api_user='your.username'
  )
  ```

- Or rely on environment variables:

  - FTRACK_SERVER for server_url
  - FTRACK_API_KEY (or FTRACK_APIKEY) for api_key
  - FTRACK_API_USER for api_user

  ```
  session = Session()
  ```

- Working with the session
  - Accessing entities will auto-populate when auto_populate is True (default).
  - Use the session to perform operations related to assignments and allocations via the underlying ftrack API (the module provides the session scaffold and authentication mechanism; exact resource access is performed through the API exposed by the session).

- Authentication in HTTP requests
  - The SessionAuthentication class can be used to attach credentials to requests:

    ```
    auth = SessionAuthentication('my_api_key', 'my_user')
    response = requests.get('https://server/endpoint', auth=auth)
    ```

- Event hub
  - If auto_connect_event_hub is True, the session will connect to the event hub in the background, enabling non-local event publishing/subscribing when supported by plugins.

## Details

- Environment and authentication
  - If required values are missing, the session raises TypeError with guidance on how to supply them.
  - The session uses a layered caching approach with a MemoryCache at the top level to optimize repeated access to assignments and allocations-related data.

- Extensibility
  - Plugin paths and arguments support dynamic extension of the session capabilities as new features (potentially including enhanced handling for assignments and allocations) are discovered and registered.

- Schema caching
  - Optional for performance; controlled via schema_cache_path or FTRACK_API_SCHEMA_CACHE_PATH. Disable by setting to False.

- Notes
  - The code relies on internal ftrack_api subsystems (operations, cache, event hub, etc.). This documentation focuses on the surface features visible in the provided implementation and how they underpin working with assignments and allocations via authenticated sessions.
