# w_crawl: Parallel per-iteration crawling of WestPA datasets

## Overview
w_crawl is a command-line tool that crawls a weighted ensemble dataset by executing a user-supplied callable for every iteration. It runs tasks in parallel (per iteration) and allows optional coordination via a custom crawler class to handle initialization, per-iteration result processing, and finalization. Each per-iteration task receives the iteration number and the corresponding iteration data, and returns a result that can be consumed by the crawler.

Key points:
- Parallel execution by iteration
- Optional crawler to coordinate setup, per-iteration results, and teardown
- Per-iteration data accessed via a data manager helper
- Callable provided as a module.function path
- Non-deterministic task completion order is allowed

## Key Components

- WESTPACrawler
  - Base class for coordination on the master node.
  - Methods to implement/override:
    - initialize(iter_start, iter_stop): prepare crawling before tasks start
    - process_iter_result(n_iter, result): handle the result of a per-iteration task
    - finalize(): perform any cleanup after all tasks complete

  Example (from the source):
  ```python
  class WESTPACrawler:
      '''Base class for general crawling execution. This class
      only exists on the master.'''

      def initialize(self, iter_start, iter_stop):
          '''Initialize this crawling process.'''
          pass

      def finalize(self):
          '''Finalize this crawling process.'''
          pass

      def process_iter_result(self, n_iter, result):
          '''Process the result of a per-iteration task.'''
          pass
  ```

- _remote_task(n_iter, taskfn)
  - Internal helper that retrieves the per-iteration data and executes the user-supplied callable.
  - Accesses the data manager, opens backing in read mode, and calls:
  - return n_iter, taskfn(n_iter, data_manager.get_iter_group(n_iter))

  - Example (from the source):
  ```python
  def _remote_task(n_iter, taskfn):
      data_manager = westpa.rc.get_data_manager()  # gaahhh...globals
      data_manager.open_backing(mode='r')
      return n_iter, taskfn(n_iter, data_manager.get_iter_group(n_iter))
  ```

- WCrawl (command class)
  - Subclass of WESTParallelTool with:
    - prog = 'w_crawl'
    - description: explains that a callable is run for each iteration, with results optionally processed by a crawler
  - Core components:
    - progress: ProgressIndicatorComponent
    - data_reader: WESTDataReader
    - iter_range: IterRangeSelection
    - crawler: WESTPACrawler (optional; defaults to a basic WESTPACrawler if not provided)
    - task_callable: the user-provided callable loaded from a module path

  - CLI options (via add_args):
    - Data reader options (via self.data_reader)
    - Iteration range options (via self.iter_range)
    - -c / --crawler-instance: pathname to a crawler instance (module.instance)
    - task_callable: pathname to the callable (module.function)
    - Progress options (via self.progress)

  - Processing flow (in go()):
    - Open the dataset for reading
    - Initialize the crawler (if provided)
    - Dispatch per-iteration tasks and process results as they complete
    - For each completed task, pass the result to crawler.process_iter_result(n_iter, result)
    - Finalize the crawler after all tasks complete

  - Key behavior:
    - Tasks are dispatched by iteration and run in parallel
    - No guarantees are made about evaluation order
    - The per-iteration callable is invoked as taskfn(n_iter, iter_group)

  - Example usage (from the source):
  ```python
  class WCrawl(WESTParallelTool):
      prog = 'w_crawl'
      description = '''\
  Crawl a weighted ensemble dataset, executing a function for each iteration.
  This can be used for postprocessing of trajectories, cleanup of datasets,
  or anything else that can be expressed as "do X for iteration N, then do
  something with the result". Tasks are parallelized by iteration, and
  no guarantees are made about evaluation order.

  -----------------------------------------------------------------------------
  Command-line options
  -----------------------------------------------------------------------------
  '''

      def __init__(self):
          super().__init__()

          # These are used throughout
          self.progress = ProgressIndicatorComponent()
          self.data_reader = WESTDataReader()
          self.iter_range = IterRangeSelection(self.data_reader)

          self.crawler = None
          self.task_callable = None

      # ... (arguments and processing)
  ```

  - The entry point and main execution path:
  ```python
  def go(self):
      iter_start = self.iter_range.iter_start
      iter_stop = self.iter_range.iter_stop
      iter_count = iter_stop - iter_start
      self.data_reader.open('r')
      pi = self.progress.indicator
      with pi:
          pi.operation = 'Initializing'
          self.crawler.initialize(iter_start, iter_stop)

          try:
              pi.new_operation('Dispatching tasks & processing results', iter_count)
              task_gen = ((_remote_task, (n_iter, self.task_callable), {}) for n_iter in range(iter_start, iter_stop))
              for future in self.work_manager.submit_as_completed(task_gen, self.max_queue_len):
                  n_iter, result = future.get_result(discard=True)
                  if self.crawler is not None:
                      self.crawler.process_iter_result(n_iter, result)
                  pi.progress += 1
          finally:
              pi.new_operation('Finalizing')
              self.crawler.finalize()
  ```

## Usage

What you need:
- A callable that operates on per-iteration data
- Optional crawler to coordinate initialization, per-iteration result handling, and finalization
- Timing and range control via WESTDataReader and IterRangeSelection

CLI syntax:
- w_crawl <TASK_CALLABLE> [options]
  - TASK_CALLABLE: module.function path to the callable to execute for each iteration
  - -c/--crawler-instance: path to a crawler instance (module.instance) to coordinate the crawl
  - Iteration range options are provided by IterRangeSelection (via WESTDataReader)

Examples:
- Minimal usage with a simple callable (no crawler coordination):
  - w_crawl my_module.my_callable

- Using a custom crawler to coordinate initialization and result handling:
  - w_crawl my_module.my_callable -c my_crawler_module.my_crawler.instance

- With an explicit iteration range (range options come from the data reader/iteration selection):
  - w_crawl my_module.my_callable --start 0 --end 100
  - Note: The exact range flags come from IterRangeSelection; see its documentation for specifics.

Notes:
- The per-iteration task receives (n_iter, iter_group) and should return a result. The result is then handed to the crawler via process_iter_result(n_iter, result).
- If no crawler instance is provided, a default WESTPACrawler is used (which performs no special coordination).
- All per-iteration data is accessed through the data manager via get_iter_group(n_iter).

## Details

- The w_crawl command is implemented as part of the WestPA parallel tool framework. It leverages:
  - WESTParallelTool for parallel task execution
  - WESTDataReader and IterRangeSelection to define the iteration space
  - ProgressIndicatorComponent to provide user feedback during execution
  - get_object to dynamically load the user-provided callable and optional crawler instance
- The core workflow is designed to be simple and flexible:
  - The master initializes, dispatches tasks by iteration, processes results as they complete, and finalizes.
  - Individual iterations are isolated: each task operates on its own iter_group data.
- Code references (from the source):
  - Base crawler interface (WESTPACrawler)
  - Remote task runner (_remote_task)
  - WCrawl command implementation (go(), argument processing, and entry point)
  - Entry point for CLI usage:
    - entry_point()
    - if __name__ == '__main__': entry_point()
