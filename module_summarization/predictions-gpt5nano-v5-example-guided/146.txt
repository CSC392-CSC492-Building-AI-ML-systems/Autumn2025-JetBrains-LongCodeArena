# Quick Start

## Overview
This quick start guide covers the core geometric-path components used by TOPP-RA and how to generate appropriate interpolation grids and OpenRAVE trajectory wrappers. It highlights how a geometric path is evaluated and how gridpoints are proposed to ensure interpolation accuracy.

Key ideas:
- AbstractGeometricPath defines the interface for geometric paths.
- propose_gridpoints generates a grid of path positions with controlled interpolation error.
- RaveTrajectoryWrapper wraps OpenRAVE trajectories into a geometric-path compatible form.

## Key Components

- propose_gridpoints
  - Generates gridpoints that sufficiently cover a given geometric path.
  - Uses an error estimate based on the path’s second derivative.

  Signature:
  - def propose_gridpoints(path, max_err_threshold=1e-4, max_iteration=100, max_seg_length=0.05)

  Important excerpt:
  - The function evaluates the maximum estimated error per segment and refines the grid until the error is within the threshold or the maximum iterations are reached.

  Key behavior:
  - Repeatedly subdivides segments where the estimated error exceeds max_err_threshold.
  - Ensures all segment lengths are below max_seg_length.

  Source reference (summary):
  - The function description and error estimation are implemented as described in the docstring and inline comments.

- AbstractGeometricPath
  - Abstract base class for geometric paths.

  Core interface (example signature):
  - def __call__(self, path_positions: Union[float, np.ndarray], order: int = 0) -> np.ndarray

  - order:
    - 0: position
    - 1: first-order derivative (velocity)
    - 2: second-order derivative (acceleration)

  Required properties:
  - @property
    - def dof(self) -> int: Degrees of freedom of the path.
  - @property
    - def path_interval(self): Starting and ending path positions.

  Optional:
  - @property
    - def waypoints(self): Tuple[ndarray, ndarray] or None

  Source excerpt:
  - class AbstractGeometricPath:
      def __call__(self, path_positions: Union[float, np.ndarray], order: int = 0) -> np.ndarray:
          """Evaluate the path at given positions..."""

- RaveTrajectoryWrapper
  - An interpolator that wraps OpenRAVE's GenericTrajectory.

  Purpose:
  - Wraps a trajectory as a geometric path, supporting quadratic or cubic interpolation.

  Key points:
  - Determines interpolation method from the trajectory configuration.
  - Uses OpenRAVE trajectory waypoints and durations to construct a piecewise polynomial (PPoly).

  Notable methods/attributes (from source):
  - static method _extract_interpolation_method(spec)
  - __init__(self, traj, robot): stores trajectory, duration, and dof, and builds internal representation
  - Handles only "quadratic" or "cubic" interpolations

  Source excerpt (summary):
  - The class reads the trajectory configuration, extracts interpolation type, and builds a polynomial representation of the trajectory, raising an error if an unsupported interpolation is encountered.

- OpenRAVE integration
  - The module conditionally imports OpenRAVE if FOUND_OPENRAVE is True.

  Source snippet:
  - if FOUND_OPENRAVE:
      import openravepy as orpy  # pylint: disable=import-error

## Usage

1) Generate gridpoints for a geometric path

- Purpose: Ensure interpolation points cover the path within a specified error.

Code example:
```python
# Assuming `path` is an instance of a class that implements AbstractGeometricPath
from toppra.quickstart import propose_gridpoints

gridpoints = propose_gridpoints(path)
```

2) Wrap an OpenRAVE trajectory (optional)

- Purpose: Treat an OpenRAVE trajectory as a geometric path for TOPP-RA processes.

Code example:
```python
from toppra.quickstart import RaveTrajectoryWrapper

# traj: orpy.RaveTrajectory, robot: orpy.Robot
wrapper = RaveTrajectoryWrapper(traj, robot)
```

3) Evaluate the geometric path

- Purpose: Compute positions, velocities, or accelerations from the path.

Code example:
```python
# Path evaluation
positions = path(np.array([0.0, 0.5, 1.0]), order=0)
velocities = path(np.array([0.0, 0.5, 1.0]), order=1)
accelerations = path(np.array([0.0, 0.5, 1.0]), order=2)
```

4) Optional: Create a path from an OpenRAVE trajectory

- If you have a trajectory, wrap it and use its representation as a geometric path as part of a TOPP-RA workflow.

Notes:
- OPENRAVE integration is optional and guarded by FOUND_OPENRAVE.
- The Quick Start demonstrates how to move from a geometric path abstraction to gridpoint generation and, if available, to an OpenRAVE-based trajectory wrapper.

## Details

- The gridpoint generation algorithm is designed to distribute points more densely where the path curvature is higher, improving approximation quality.
- The error estimate used for refinement is based on the path’s second derivative and segment length:
  - err_est ≈ 0.5 * max(|p'' * d_segment^2|)

- The AbstractGeometricPath interface is deliberately minimal, enabling different backends (e.g., OpenRAVE, custom splines) to plug into TOPP-RA’s planning pipeline.

- The RaveTrajectoryWrapper translates an OpenRAVE trajectory into a piecewise polynomial representation, exposing a consistent interface through the AbstractGeometricPath API.

- For advanced usage, you can inspect the source code directly to see the detailed docstrings, method implementations, and internal helpers that support these operations.
