Async Plans
===========

Overview
--------
This module provides asynchronous Bluesky plan helpers for data acquisition. It exposes plans that interact with detectors and motors to perform time-resolved readings and scans. The key functions are:
- count: take one or more readings from detectors, with optional per-shot customization and metadata.
- list_scan: scan over one or more motors in an inner-product fashion, recording detector readings at each point.
- rel_list_scan: placeholder for relative scans (not yet implemented).

Key Components
--------------

count(detectors, num=1, delay=None, *, per_shot=None, md=None)
- Description
  Take one or more readings from detectors. The function yields a plan that reads detectors multiple times, with optional delay between readings and an optional per-shot hook.
- Parameters
  - detectors: list of readable objects (detectors)
  - num: integer, number of readings to take (default 1). If None, continue until canceled
  - delay: iterable or scalar, time delay between successive readings (default 0)
  - per_shot: callable, optional hook for inner-loop action (messages per step)
  - md: dict, optional metadata
- Behavior
  - If delay is iterable, it must have at least num - 1 entries.
  - Uses decorators to set up stage and run metadata.
- Returns
  A generator-based plan yielded to the RunEngine.
- Example
  .. code-block:: python

     # Take 5 readings from two detectors with a 0.5s delay between shots
     RE(count([det1, det2], num=5, delay=0.5))

list_scan(detectors, *args, per_step=None, md=None)
- Description
  Scan over one or more variables in steps simultaneously (inner product). Produces a plan that iterates over specified motor positions and collects detector data at each step.
- Parameters
  - detectors: list of readable objects
  - *args: For one dimension, motor, [points...]; generally motor1, [points...], motor2, [points...], ...
  - per_step: callable, optional hook for inner-loop actions
  - md: dict, optional metadata
- Notes
  - All position lists must have the same length; otherwise a ValueError is raised.
  - The plan uses plan_patterns.inner_list_product to generate the full sequence.
- Returns
  A generator-based plan yielded to the RunEngine.
- Example
  .. code-block:: python

     # Simple 2D scan: motorA over [0,1,2], motorB over [10,20,30]
     RE(list_scan([det], motorA, [0,1,2], motorB, [10,20,30]))

rel_list_scan(detectors, *args, per_step=None, md=None)
- Description
  Scan over one variable in steps relative to the current position. Not yet implemented in this module.
- Parameters
  - detectors: list of readable objects
  - *args: motor, [relative steps], ... (relative to current position)
  - per_step: callable, optional hook for inner-loop actions
  - md: dict, optional metadata
- Status
  TODO: Implementation placeholder
- Returns
  A generator-based plan (not implemented yet).

Usage
-----
- Typical pattern (Bluesky RunEngine):
  .. code-block:: python

     from bluesky import RunEngine
     from bluesky.preprocessors import stage
     from bluesky.plans import count, list_scan

     # detectors: list of detector objects
     # motors: one or more controllable devices
     # md: optional metadata dictionary

     RE(count(detectors, num=5, delay=0.2))
     RE(list_scan(detectors, motor1, [0, 1, 2], motor2, [10, 20, 30]))

Details
-------
- Decorators and plan structure:
  - The count plan is wrapped with @bpp.stage_decorator(detectors) and @bpp.run_decorator(md=_md), ensuring detectors are staged and metadata is attached.
  - The inner logic uses bps.repeat with a per_shot hook to perform the inner loop.
- Metadata (md) handling:
  - The plan metadata includes detectors, plan name, number of points, and plan arguments, along with hints for plotting dimensions.
  - Default hints set the primary dimension to time if not specified.
- Plan patterns:
  - list_scan uses plan_patterns.inner_list_product to generate the sequence for an inner product of motors and points.
