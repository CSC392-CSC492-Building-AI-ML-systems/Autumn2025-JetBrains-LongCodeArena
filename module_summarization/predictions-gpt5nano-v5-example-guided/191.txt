# Validation curves: plotting scores to evaluate models

## Overview
Validation curves assess how a model’s performance changes as you vary a hyperparameter. By computing training and cross-validated scores across a range of parameter values, you can identify optimal settings and diagnose bias/variance behavior. This module provides utilities to compute validation curves and visualize them, including options to display uncertainty.

Key components in this workflow include:
- The validation_curve API to compute scores across a parameter range.
- Display utilities (e.g., ValidationCurveDisplay) to plot training and validation scores with configurable uncertainty visualization.

The plotting machinery is built on a common base that supports multiple display styles (mean with shaded std, error bars, or simple lines) and automatic axis scaling.

## Key Components

- _BaseCurveDisplay
  - Core plotting helper used by curve displays. Its _plot_curve method accepts:
    - x_data: array-like, the parameter grid (e.g., C values for an SVM).
    - ax: optional matplotlib Axes to plot on.
    - negate_score: if True, plots the negative of the scores (useful when minimizing is desired).
    - score_name: label for the y-axis (defaults to inferred or provided value).
    - score_type: "test", "train", or "both" to select which curves to display.
    - std_display_style: "fill_between", "errorbar", or None to visualize uncertainty.
    - line_kw, fill_between_kw, errorbar_kw: styling keyword arguments for the respective plot styles.
  - Supports automatic axis scaling, including a deprecation path for log_scale and an intelligent choice between linear/symlog/log scales based on the x_data spread.
  - Handles multiple score curves (e.g., training and validation) and updates the legend accordingly.

- LearningCurveDisplay
  - Visualization for learning curves (scores vs. training set size). While focused on learning curves, it demonstrates the same display patterns used by validation curves (mean ± std shading, or error bars, etc.).
  - Public API emphasizes using from_estimator to construct the display and then plot it.

- ValidationCurveDisplay (recommended API for validation curves)
  - Public plotting wrapper to visualize validation curves derived from varying a hyperparameter.
  - Typically used via from_estimator(estimator, X, y, param_name, param_range, ...), then plot().

- validation_curve (function)
  - Computes training and cross-validated scores for a given estimator by varying a hyperparameter over a specified range.
  - Returns the parameter grid and the corresponding train_scores and test_scores, which can be consumed by ValidationCurveDisplay.

## Usage

- Recommended visualization approach
  - Use ValidationCurveDisplay.from_estimator to construct a display from an estimator, data, and a hyperparameter range.
  - Then call display.plot() to render the curves.

- Example: basic validation curve visualization
  - This demonstrates computing and plotting a validation curve for a classifier by varying a hyperparameter (e.g., C for an SVM).

  ```
  from sklearn.model_selection import ValidationCurveDisplay
  from sklearn.svm import SVC
  from sklearn.datasets import load_iris

  X, y = load_iris(return_X_y=True)
  param_name = "C"
  param_range = [0.1, 1, 10, 100]

  # Construction and plotting via estimator-based API
  display = ValidationCurveDisplay.from_estimator(
      estimator=SVC(), X=X, y=y,
      param_name=param_name, param_range=params_range,
      scoring="accuracy", cv=5
  )
  display.plot()
  ```

- Example: customizing uncertainty representation
  - You can choose the style to visualize uncertainty:
    - std_display_style="fill_between" (mean ± std as a shaded region)
    - std_display_style="errorbar" (error bars at each point)
    - std_display_style=None (only lines)

  ```
  display.plot(std_display_style="fill_between", fill_between_kw={"alpha": 0.4})
  ```

- Parameter and label handling
  - score_name may be inferred from the scoring parameter, but can be overridden to customize the y-axis label.
  - The display supports negating the score when a lower value is preferable (negate_score).

## Details

- Scoring and naming
  - Validation curves rely on the scoring parameter passed to the computation function. The display derives a human-friendly score_name for labeling the y-axis, optionally overridden by the caller.

- Uncertainty visualization
  - std_display_style controls how uncertainty is shown:
    - "fill_between": mean score with shaded area spanning mean ± std.
    - "errorbar": error bars representing the standard deviation.
    - None: no uncertainty visualization; only the mean curves are drawn.

- Axis scaling and appearance
  - The plotting routine auto-selects an appropriate x-axis scale (linear, log, or symlog) based on the distribution of the parameter values, with a heuristic to switch scales when the range varies widely.
  - Legends are added to distinguish curves (e.g., Train vs Test).

- Deprecations
  - Some legacy parameters (e.g., log_scale) are deprecated in favor of using axis methods (ax_.set_xscale / set_yscale). The plotting function may emit a FutureWarning if such parameters are used.

- See Also
  - sklearn.model_selection.learning_curve: Utilities for computing and visualizing learning curves.
  - ValidationCurveDisplay: Public API for plotting validation curves (from estimator, param_name, param_range, etc.).

## Notes
- The visualization API aligns with the established curve display design used for learning curves, enabling consistent styling and flexible uncertainty visualization across different types of curve plots.
- When using validation curves, prefer the estimator-based display (ValidationCurveDisplay.from_estimator) for a seamless workflow that ties parameter ranges directly to plots.
