# Imputation (MICE)

## Overview
This module implements Multiple Imputation through Chained Equations (MICE) for handling missing data in statistical analyses. The procedure follows these steps:

- Impute missing values with the mean of observed values for the same variable.
- For each variable with missing values (the focus variable):
  - Fit an imputation model (a regression of the focus variable on observed and current imputed values of other variables).
  - Impute the missing values for the focus variable using predictive mean matching (PMM).
- Once all variables are imputed, fit the analysis model.
- Repeat the above steps multiple times and combine results using a combining rule to obtain point estimates and standard errors.

Imputations for each variable are based on an imputation model specified by a model class and a regression formula. The default model is Ordinary Least Squares (OLS) with a formula that includes main effects for all other variables.

The MICE procedure can be used in two ways:

- If the goal is only to generate imputed data sets, use the MICEData class to wrap a dataframe, providing facilities for imputation and summary plots to assess performance.
- If the imputed data sets will be used to fit an additional analysis model, use a MICE instance. After running, results can be combined with the combine method, and summary plots and results are available.

Key ideas:
- Imputation model per variable, possibly conditioned on all or a subset of the remaining variables (main effects, transformations, interactions, etc.).
- Perturbation methods to set parameter estimates in imputation models:
  - Gaussian perturbation: draw parameters from the Gaussian approximation to the fit distribution.
  - Bootstrap perturbation: use parameters from a bootstrap version of the data.

## Key Components
- MICEData: Wraps a Pandas dataframe, associating imputation models with variables that have missing values. Provides facilities for producing multiply imputed datasets, as well as plotting and a history hook for saving intermediate results.
- MICE: Takes a MICEData object and an analysis model specification. Runs the multiple imputation, fits the analysis models, and combines results into a MICEResults object. The summary method exposes key estimands and inferential quantities.
- PatsyFormula: A small wrapper for a string to be interpreted as a Patsy formula. It prefixes the provided formula with "0 + " to ensure proper term handling.
- History/callbacks: history_callback in MICEData allows saving features of interest from intermediate imputations.

Notes:
- Default memory considerations: MICEData saves very little information between iterations to conserve memory. The input dataset is copied on entry, then overwritten with new imputations. If using MICE, fitted analysis models and results are saved.
- History callback: A hook available to capture intermediate information from imputed datasets.

## Usage

- Producing imputed datasets only:
  - Wrap a dataframe with MICEData to perform imputations and generate multiple complete datasets.
  - Use provided plotting utilities to assess imputation quality.
- Using imputed data to fit an analysis model:
  - Create a MICE instance with a MICEData object and an analysis model specification.
  - Run the imputation process and combine the results with the combine method.
  - Access results and summary plots via the MICEResults object.

Code examples from the module:

Example 1: Imputing with a specific imputer and iterating
```python
>>> imp = mice.MICEData(data)
>>> imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
>>> for j in range(20):
...     imp.update_all()
...     imp.data.to_csv('data%02d.csv' % j)
```

Example 2: Iterating over imputed datasets
```python
>>> imp = mice.MICEData(data)
>>> j = 0
>>> for data in imp:
...     imp.data.to_csv('data%02d.csv' % j)
...     j += 1
```

## Details

- Imputation models: Each variable with missing data has an associated imputation model. Models can condition on all or a subset of other variables, using main effects, transformations, and interactions as desired. The default for imputation is OLS with a simple formula.
- Analysis model: If provided, the analysis model is fitted to each imputed dataset, and results are combined to produce final estimates and standard errors.
- Perturbation methods:
  - Gaussian: Fit the model, then draw parameter vectors from a Gaussian approximation to the distribution of the estimated parameters.
  - Bootstrap: Use the parameter vector obtained from fitting the conditional model to a bootstrapped dataset.

- Memory considerations: MICEData stores minimal information between iterations to save memory. The user-provided dataset is copied upon entry; subsequent iterations overwrite the working dataset. If using MICE, the fitted analysis models and results are preserved.

- History and hooks: history_callback allows arbitrary information from intermediate datasets to be saved for later use and analysis.
