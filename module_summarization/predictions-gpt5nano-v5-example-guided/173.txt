# Fitting SESANS Data

## Overview
This documentation covers the Sesans fitting parameter discovery utility used in SESANS data analysis. The module provides tools to enumerate all parameters used across available SESANS models and to see which models utilize specific parameters. It helps in understanding model parameterization, enabling consistent fitting and comparison across SESANS models.

The functionality is exposed via a small CLI (python -m sasmodels.list_pars) and is driven by model metadata loaded from sasmodels' model registry.

## Key Components
- find_pars(kind=None)
  - Scans all available models and collects parameters from each model's kernel_parameters.
  - Returns a dictionary mapping parameter names to a list of models that use that parameter.
  - If kind is provided, only parameters of that kind are included (e.g., 'volume', 'orientation', 'sld').
  - Example return: { 'q': ['ModelA', 'ModelB'], 'rho': ['ModelC'] }

- list_pars(names_only=True, kind=None)
  - Prints parameters found by find_pars.
  - If names_only is True, prints only the parameter names (one per line, columnized).
  - If names_only is False, prints "parameter: model1, model2, ..." with the models that use each parameter.
  - The output can be filtered by kind as described above.

- main()
  - Command-line entry point for the utility.
  - Usage: python -m sasmodels.list_pars [-v]
  - -v / --verbose: if set, print models that use each parameter; otherwise, print only parameter names.
  - kind argument: can be volume, orientation, sld, none, or any (default: any)
  - The kind argument is parsed and passed to list_pars to filter the results.

## Usage
- List all parameters (names only):
  - python -m sasmodels.list_pars

- List all parameters with the models that use them:
  - python -m sasmodels.list_pars -v

- List only parameters of a given kind (e.g., volume parameters):
  - python -m sasmodels.list_pars volume

- Full usage snippet (from the source):
  Usage:

      python -m sasmodels.list_pars [-v]

  If '-v' is given, then list the models containing the parameter in
  addition to just the parameter name.

  def main():
      """
      Program to list the parameters used across all models.
      """
      parser = argparse.ArgumentParser(
          description="Find all parameters in all models",
          )
      parser.add_argument(
          '-v', '--verbose',
          action='store_true',
          help="list models which use this argument")
      parser.add_argument(
          'kind', default="any", nargs='?',
          metavar="volume|orientation|sld|none|any",
          choices=['volume', 'orientation', 'sld', None, 'any'],
          type=lambda v: None if v == 'any' else None if v == 'none' else v,
          help="only list arguments of the given kind")
      args = parser.parse_args()

      list_pars(names_only=not args.verbose, kind=args.kind)

    if __name__ == "__main__":
        main()

  This shows how to run the utility and how the kind argument is interpreted.

## Details
- What you can filter by (kind)
  - volume: parameters related to volume (geometric or size-related)
  - orientation: parameters describing orientation-related factors
  - sld: scattering length density related parameters
  - none / any: select all parameters (no filtering)

- How the output is produced
  - The tool aggregates parameter usage across all models by inspecting each model's kernel_parameters.
  - The output is columnized for readability when listing parameter names, and includes model associations when verbose is enabled.

- Implementation notes (reference to source)
  - The core logic relies on:
    - list_models() to enumerate available models
    - load_model_info(name) to access model metadata
    - model_info.parameters.kernel_parameters to access parameters
    - Each parameter p has attributes such as p.name and p.type
    - columnize() is used to format output in the non-verbose case

Code references (key excerpts from the source):
- find_pars(kind=None)

  - Uses find_pars(kind) and prints either just names or name-to-model mappings
