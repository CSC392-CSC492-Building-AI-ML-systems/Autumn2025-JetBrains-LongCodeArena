# Memory Alignment

## Overview
Memory alignment concerns how NumPy allocates and accesses memory for arrays, particularly in the context of SIMD (Single Instruction, Multiple Data) optimizations. The alignment.rst content reflects build- and run-time information that affects alignment and vectorized memory access:

- Build-time and host CPU information that influence alignment constraints and available SIMD paths.
- Detection of SIMD extensions and features supported by the runtime CPU.
- A show() utility that exposes the configuration used to build NumPy and what the running environment supports, which in turn informs alignment and vectorization decisions.

The module exposes information about the build configuration and SIMD capabilities through a small API described below.

## Key Components

- DisplayModes(Enum)
  - stdout: Print a human-friendly representation (YAML if pyyaml is available, otherwise JSON).
  - dicts: Return the configuration as a Python dictionary.

- _cleanup(d)
  - Internal helper that recursively removes empty values from a dictionary. Used to prune build-time data.

- CONFIG
  - The central configuration dictionary containing sections such as:
    - Compilers: Details for C, C++, and Cython compilers (name, linker, version, commands, arguments).
    - Machine Information: Host, build, and cross-compiled flags.
    - Build Dependencies: BLAS and LAPACK information (name, found, version, include/lib directories, etc.).
    - Python Information: Path and version.
    - SIMD Extensions: Baseline CPU features and feature availability.
      - baseline: The CPU baseline features from __cpu_baseline__.
      - found: Features available on the running CPU (from __cpu_dispatch__ and __cpu_features__).
      - not found: Features not available on the running CPU.

  The SIMD data is derived from:
  - __cpu_baseline__: baseline features.
  - __cpu_dispatch__: all possible SIMD features (with true/false indicating availability).
  - __cpu_features__: actual feature availability.

- _check_pyyaml()
  - Lazy loader for PyYAML; used to provide nicer human-readable output when printing CONFIG.

- show(mode)
  - Public API to display the configuration.
  - mode: 'stdout' (default) or 'dicts'.
    - 'stdout': Attempts to print CONFIG in YAML via PyYAML; falls back to JSON if PyYAML is unavailable, and emits a warning suggesting to install PyYAML for nicer output.
    - 'dicts': Returns CONFIG as a Python dictionary.
  - Example:
    - show('dicts') returns a dict; show('stdout') prints to console.

- Notes on SIMD and memory alignment
  - The SIMD Extensions block in CONFIG reflects the vectorization capabilities considered at build-time and checked at run-time.
  - Availability of certain SIMD features can influence alignment guarantees and striding for memory accesses in vectorized paths.

## Usage

- Print a human-readable configuration to stdout (YAML if PyYAML is installed):
  - from alignment import show, DisplayModes
  - show(DisplayModes.stdout.value)

- Retrieve the configuration as a Python dictionary:
  - from alignment import show, DisplayModes
  - config = show(DisplayModes.dicts.value)
  - baseline = config['SIMD Extensions']['baseline']
  - found_features = config['SIMD Extensions']['found']

- Inspect example keys of interest:
  - SIMD Extensions:
    - baseline: value from __cpu_baseline__
    - found: list of features available on the running CPU
    - not found: list of features not available on the running CPU

## Details

- The CONFIG dictionary structure is designed to summarize both the build-time environment and the runtime capabilities that impact memory alignment and SIMD paths:
  - Host and build CPU information help determine expected alignment guarantees and vectorized behavior.
  - SIMD Extensions provide explicit visibility into which alignment- or vectorization-related features are present or missing.
- The show() function is the primary mechanism to expose this information:
  - It defaults to stdout for convenient inspection.
  - When mode='dicts', it returns a dictionary for programmatic use.
