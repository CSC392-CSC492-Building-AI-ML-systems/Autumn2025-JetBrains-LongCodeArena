# Using timers

## Overview
Timers schedule the execution of a function after a delay or at regular intervals. Python provides built-in support for timers via the threading module (e.g., threading.Timer) and other facilities (sched, asyncio). This page outlines common patterns for using timers, including single-shot timers, repeating timers, cancellation, and best practices when integrating with other lifecycle management (such as cleanup at exit).

Note: The provided source code does not include a dedicated timer module. The examples below show typical Python approaches that you can adapt for usage in projects that rely on the standard library and threading.

## Key Components
- Timer object (threading.Timer): schedules a callback to run after a specified delay.
- Callback: the function to be executed when the timer fires.
- Cancellation: stopping a timer before it fires using `Timer.cancel()`.
- Repeating timer: achieved by rescheduling the timer at the end of the callback.
- Lifecycle considerations: daemon threads, cleanup at program exit (atexit), exception handling within the callback.

## Usage

### Single-shot timer
Schedules a function to run once after a delay.

```python
from threading import Timer

def greet():
    print("Hello after 3 seconds")

t = Timer(3.0, greet)  # 3-second delay
t.start()

# Optional: cancel before it fires
# t.cancel()
```

### Repeating timer (periodic callback)
Use a timer to re-schedule itself at the end of each execution to achieve periodic behavior.

```python
from threading import Timer

def tick():
    print("tick")
    # Schedule next tick
    t = Timer(1.0, tick)
    t.start()

# Start after an initial delay
initial = Timer(1.0, tick)
initial.start()
```

> Note: Repeating timers create new threads each interval. For long-running periodic work, consider a dedicated thread with a loop or using sched/asyncio to manage lifecycle more explicitly.

### Cancelling a timer
You can stop a timer before it executes.

```python
from threading import Timer

def work():
    print("This will not run if canceled")

t = Timer(5.0, work)
t.start()

# Cancel if a condition occurs
t.cancel()
```

### Timer integrated with program exit
Ensure timers do not keep the interpreter alive or leave resources open on exit.

```python
import atexit
from threading import Timer

def cleanup():
    print("Cleaning up before exit")

atexit.register(cleanup)

t = Timer(2.0, lambda: print("Timer fired before exit"))
t.start()
```

## Details

- Threading model:
  - threading.Timer creates a daemon thread that runs the callback after the specified delay.
  - Daemon threads do not prevent program termination; the program can exit even if timers are still pending.
- Exception handling:
  - Exceptions raised in a timer callback propagate within the timer thread and won’t crash the main thread. Consider wrapping callbacks with try/except and logging.
- Cancellation:
  - Call `cancel()` before the timer fires to prevent the callback from running. If the timer has already fired or completed, `cancel()` has no effect.
- Alternatives:
  - sched.scheduler: a general-purpose event scheduler that runs callbacks in the main thread (requires a running event loop).
  - asyncio: for asynchronous timers in an event-driven context, suitable for IO-bound or cooperative concurrency.
- Complexity considerations:
  - Repeated timers can accumulate threads if not careful; prefer a single worker thread with a loop for long-running periodic tasks.
  - For high-frequency or highly reliable timing requirements, consider dedicated timing services or event-driven architectures.
