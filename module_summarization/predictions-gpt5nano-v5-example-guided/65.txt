# Tutorial: Probability Calibration

## Overview
This module provides probability calibration for classifiers using either isotonic regression or Platt’s sigmoid (logistic regression). It uses cross-validation to estimate calibration parameters and, optionally, to produce an ensemble of calibrated classifiers whose predicted probabilities are averaged.

Key features:
- Supports two calibration methods: sigmoid (Platt scaling) and isotonic regression.
- Cross-validated calibration with an ensemble option for robust probability estimates.
- Prefit mode to calibrate an already-fitted estimator without additional cross-validation.
- Calibration relies on the estimator’s decision_function when available, otherwise on predict_proba.

This documentation centers on the CalibratedClassifierCV class, which implements probability calibration in a flexible, cross-validated framework.

## Key Components

- CalibratedClassifierCV
  - A meta-estimator that calibrates the predicted probabilities of a base estimator using cross-validation.
  - Supports two calibration methods:
    - sigmoid: Platt’s method (logistic regression on the base estimator’s scores)
    - isotonic: non-parametric isotonic regression
  - Can operate in ensemble mode (averaged probabilities across CV folds) or using cross_val_predict for unbiased calibration.
  - Prefit mode: calibrates a pre-fitted estimator without CV; data for fitting and calibration must be disjoint if used manually.

- Main parameters
  - estimator: The classifier to calibrate (default: LinearSVC in the docstring). The base estimator’s decision_function or predict_proba is used for calibration.
  - method: {'sigmoid', 'isotonic'}, default 'sigmoid'
  - cv: Cross-validation strategy. Can be an integer (folds), CV splitter, iterable of train/test indices, or "prefit".
  - n_jobs: Number of parallel jobs. None means 1 by default; -1 uses all processors.
  - ensemble: If True, trains a separate calibrated classifier per CV fold and averages the outputs. If False, uses cross_val_predict to obtain unbiased predictions for calibration and trains a single estimator on all data for prediction.
  - base_estimator: Deprecated alias for estimator (to be removed in future releases).

- Attributes
  - classes_: Class labels seen during fit.
  - n_features_in_: Number of features seen during fit.
  - feature_names_in_: Feature names seen during fit (if supported by the estimator).
  - calibrated_classifiers_: List of calibrated estimators (one per CV fold or a single estimator for "prefit"/non-ensemble cases).

- Example usage (illustrative)
  - Basic calibration with sigmoid on a classifier:
    - calib = CalibratedClassifierCV(estimator=SomeClassifier(), method='sigmoid', cv=5)
    - calib.fit(X, y)
    - proba = calib.predict_proba(X_test)
  - Prefit calibration (no cross-validation):
    - est = SomeClassifier().fit(X, y)
    - calib = CalibratedClassifierCV(estimator=est, method='sigmoid', cv='prefit')
    - calib.fit(X, y)

## Usage

- Basic calibration with a base estimator
  - Purpose: Produce calibrated probability estimates for predictions.
  - Example:
    ```python
    from sklearn.calibration import CalibratedClassifierCV
    from sklearn.svm import LinearSVC
    from sklearn.datasets import make_classification

    X, y = make_classification(n_samples=1000, n_features=20, random_state=42)
    base = LinearSVC()
    calib = CalibratedClassifierCV(base, method='sigmoid', cv=5)
    calib.fit(X, y)
    probs = calib.predict_proba(X[:5])
    ```

- Prefit calibration
  - Purpose: Calibrate a pre-fitted estimator without cross-validation.
  - Example:
    ```python
    est = LinearSVC().fit(X, y)
    calib = CalibratedClassifierCV(est, method='sigmoid', cv='prefit')
    calib.fit(X, y)  # Uses all data for calibration
    probs = calib.predict_proba(X[:5])
    ```

- Calibration methods
  - sigmoid: Platt scaling (logistic regression on the estimator’s scores).
  - isotonic: Non-parametric isotonic regression; be cautious with small calibration samples (tends to overfit if very few samples).

- Key behavioral notes
  - The calibration uses the estimator’s decision_function when available; otherwise it uses predicted probabilities (predict_proba) if the estimator provides them.
  - If cv='prefit', cross-validation is not used and all data are used for calibration.
  - When ensemble=True, predictions are averaged across calibrated classifiers from each CV fold.
  - When ensemble=False, calibration uses cross_val_predict to obtain unbiased predictions for calibration, and the final predictor is the estimator trained on all data.

## Details

- Cross-validation strategy (cv)
  - If y is binary or multiclass, StratifiedKFold is used by default.
  - Otherwise, KFold is used.
  - The default cv is 5-fold when cv=None (as of the documented behavior).

- Calibration methods
  - sigmoid:
    - Implemented as Platt’s scaling: fits a logistic regression model to map the base estimator’s scores to calibrated probabilities.
  - isotonic:
    - Non-parametric calibration that fits an Isotonic Regression model to the scores/probabilities.
    - Not recommended for very small calibration sets (<< 1000 samples) due to overfitting risk.

- Output and compatibility
  - The calibrated estimator provides predict_proba, and, depending on the base estimator, may also expose the base estimator’s attributes (e.g., classes_, feature_names_in_).
  - The class is designed to be used with scikit-learn’s API, including integration with pipelines and grid searches.

- Reference and guidance
  - For more on calibration, see the user guide (Calibration) in the documentation.
