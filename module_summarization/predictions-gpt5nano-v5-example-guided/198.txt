# Nearest Neighbors

## Overview
This module provides efficient nearest-neighbor search using tree-based data structures. It builds specialized Ball Tree representations (and their 64-bit and 32-bit variants) from input data and supports fast k-nearest-neighbor queries under a variety of distance metrics. Key concepts include reduced distances for faster computation, node bounding radii for pruning, and optional sample weights for weighted centroids.

The implementation emphasizes two ideas:
- Ball Tree structure to partition space via centroided balls with associated radii.
- Distance metrics (and their reduced forms) used to prune search space during queries.

Supported metrics include a broad set of distance functions (Euclidean, Manhattan, Hamming, Mahalanobis, Minkowski variants, and more), with per-variant names such as EuclideanDistance64 / EuclideanDistance32, etc.

## Key Components
- BallTree64 and BallTree32: 64-bit and 32-bit specialized Ball Tree implementations, inheriting from the BinaryTree variants.
- BinaryTree: The generic binary-tree foundation used by the Ball Tree implementations.
- NodeData: Stores per-node information (e.g., radius, idx_start, idx_end) used during search.
- Distance metrics: A collection of metric implementations, including reduced-distance variants to accelerate pruning. Examples of supported metrics include EuclideanDistance, ManhattanDistance, CanberraDistance, ChebyshevDistance, JaccardDistance, HammingDistance, HaversineDistance, and several Minkowski/Mahalanobis-based variants.
- Data and indexing: Internal structures such as tree.data (dataset), tree.idx_array (point indices), and tree.node_bounds (per-node centroids and radii) facilitate fast pruning and traversal.

Note: The code defines multiple implementations for different dtypes, exposed as BallTree{{name_suffix}} (e.g., BallTree64, BallTree32) to optimize performance for float64 and float32 data.

## Usage
- Build a Ball Tree from your dataset and perform k-nearest-neighbor queries.

Example (high-level usage pattern):
```python
# Pseudocode: user-facing API
from your_package import BallTree

# X is your data matrix (n_samples, n_features)
tree = BallTree(X, metric='euclidean')  # 64-bit or 32-bit variant chosen under the hood

# X_query contains query points (m_samples, n_features)
distances, indices = tree.query(X_query, k=5)  # find 5 nearest neighbors for each query point
```

## Details
- Reduced distance: For certain metrics, a reduced distance (e.g., squared distance for Euclidean) is computed to speed up comparisons while preserving the relative ordering of true distances. This enables efficient pruning of tree nodes during search.
- Node data and bounding: Each node stores a centroid and a radius, plus the index range of the points it covers. The radius is derived from the maximum distance from the centroid to points in that node.
- Weighted data support: If tree.sample_weight is provided, centroids are computed as weighted means, and the total weight influences the radius calculation.
- Distance pruning: The search uses min_dist and max_dist helpers to prune candidates by comparing the query point to node bounds before evaluating individual points.
- Metrics availability: The implementation exposes a broad set of metrics for both 64-bit and 32-bit variants, for example:
  - BrayCurtisDistance64 / BrayCurtisDistance32
  - CanberraDistance64 / CanberraDistance32
  - ChebyshevDistance64 / ChebyshevDistance32
  - DiceDistance64 / DiceDistance32
  - EuclideanDistance64 / EuclideanDistance32
  - HammingDistance64 / HammingDistance32
  - HaversineDistance64 / HaversineDistance32
  - JaccardDistance64 / JaccardDistance32
  - MahalanobisDistance64 / MahalanobisDistance32
  - ManhattanDistance64 / ManhattanDistance32
  - MinkowskiDistance64 / MinkowskiDistance32
  - PyFuncDistance64 / PyFuncDistance32
  - RogersTanimotoDistance64 / RogersTanimotoDistance32
  - RussellRaoDistance64 / RussellRaoDistance32
  - SokalMichenerDistance64 / SokalMichenerDistance32
  - SokalSneathDistance64 / SokalSneathDistance32
  - WMinkowskiDistance64 / WMinkowskiDistance32

- Internal API (for advanced users): Functions like allocate_data{{name_suffix}}, init_node{{name_suffix}}, min_dist{{name_suffix}}, max_dist{{name_suffix}}, and min_max_dist{{name_suffix}} show how the tree is built and how distances are computed and pruned at the Cython level.

Performance considerations:
- The Ball Tree structure enables sublinear average-case query performance for moderate to high-dimensional data when using suitable metrics.
- Build-time cost is higher than flat brute-force search, but query-time performance typically dominates for large datasets.
- The 64-bit and 32-bit variants allow trade-offs between numerical precision and memory usage.

## API Reference (high level)
- BallTree: Base Ball Tree interface for performing nearest-neighbor searches.
- BallTree64: 64-bit variant optimized for double-precision data.
- BallTree32: 32-bit variant optimized for single-precision data.
- BinaryTree: Base class providing common tree behaviors and interfaces.
- NodeData: Per-node data structure storing radius and index ranges.
- Distances: A suite of distance functions and their reduced forms used for pruning.

If you need a quick start, import BallTree from the package, fit it on your data, and issue a query for the k nearest neighbors of your query points. The underlying Cython code demonstrates the specialized instantiations and the core node initialization and distance-pruning logic used in these Nearest Neighbors searches.
