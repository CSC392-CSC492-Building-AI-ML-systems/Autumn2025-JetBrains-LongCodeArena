# w_ipa (WESTPA Interactive Python Analysis)

## Overview
w_ipa is a stateful command-line tool integrated with WESTPA that provides an interactive Python analysis environment for the current analysis scheme and iteration. It enables you to run traces, inspect weights and progress coordinates, view states and bins, and perform interactive analyses against the active WESTPA configuration. Access and control is via the familiar WESTPA CLI namespace (w or west).

Key characteristics
- Stateful: data pulled from the current analysis scheme and iteration.
- Default behavior uses the first analysis scheme in west.cfg unless changed.
- Interaction via properties and methods exposed on the w or west namespace.
- Visual output can be generated with the default matplotlib interface or rendered in a terminal with a text-based interface.

Reference from the source
- The main tool is implemented as a class WIPI (WESTPA Interactive Python Analysis), which is a WESTParallelTool.
- The docstring for WIPI describes typical usage and available properties, including how to access iteration, schemes, states, bins, and the current iteration data via w.current.
- Example: “ALL PROPERTIES ARE ACCESSED VIA w or west … to see the current iteration, try: w.iteration OR west.iteration … to set it, simply plug in a new value: w.iteration = 100.”
- The tool can run a trace over a seg_id: “w.trace(seg_id)” and caches results because traces can be expensive.

Code reference (highlights)
- Core class and initialization:
  - WIPI inherits from WESTParallelTool and wires in a data reader, progress indicator, and default interface.
  - Default interface is matplotlib; can be switched to text with a plotting flag.
  - The current analysis scheme is resolved from the west configuration, preferring enabled schemes.
- Key methods:
  - add_args: defines runtime options such as --analysis-only, --reanalyze, --ignore-hash, --debug, and --terminal.
  - process_args: reads the current iteration, selects the scheme, and exposes runtime options (analysis_mode, reanalyze, ignore_hash, debug_mode).
  - hash_args: builds a unique hash to determine if arguments/files have changed, with optional debug output.
  - stamp_hash: stamps an HDF5 file with the argument hash (internal integrity mechanism).

Usage
- Access and modify core state from the w or west namespace:
  - View or set iteration
    - w.iteration
    - west.iteration
    - w.iteration = 100
  - List and select analysis schemes
    - w.list_schemes
    - w.scheme = <scheme_name> (e.g., OUTPUT FROM w.list_schemes)
  - Inspect states and bins
    - w.states
    - w.bin_labels
  - Inspect current iteration data
    - w.current  (walkers, summary, states, seg_id, weights, parents, kinavg, pcoord, bins, populations, auxdata)
  - Run a trace
    - w.trace(seg_id)
- Command-line options (exposed via add_args)
  - --analysis-only / -ao: Run the analysis and return to the terminal
  - --reanalyze / -ra: Delete existing files and reanalyze
  - --ignore-hash / -ih: Ignore the argument hash and avoid regenerating files
  - --debug / -d: Enable debug output
  - --terminal / -t: Plot output in the terminal (text interface)
  - -f / --extra: Hidden/experimental option
- Example interaction (conceptual)
  - Start interactive session
  - w.iteration = 2
  - w.scheme = 'SOME_ENABLED_SCHEME'
  - w.current  # inspect current iteration data
  - w.trace(12345)  # retrieve trace information for a segment
- Internal mechanisms (for advanced usage)
  - hash_args(args, extra=None, path=None) computes a stable hash to determine if reanalysis is needed.
  - stamp_hash(h5file_name, new_hash) stamps an HDF5 file with the computed arg_hash to track changes.

Details
- Purpose and scope: w_ipa provides convenient access to iteration-level data and analysis state within WESTPA, enabling interactive exploration of traces, weights, progress coordinates, and related quantities.
- Data flow: The tool uses WESTDataReader to fetch data for the current iteration and scheme, then exposes this data through a consistent, user-friendly interface.
- Configuration: The active analysis scheme is determined from westrc configuration by scanning the analysis_schemes section and selecting the first scheme that is enabled (or None if not applicable).

Notes
- w_ipa is designed to be extensible and integrated with WESTPA’s workflow. It emphasizes interactive analysis while remaining consistent with WESTPA’s configuration and iteration management.
- If you need to rely on internal hashing behavior or positional hashing for reproducibility, refer to hash_args and stamp_hash for the underlying implementation details.
