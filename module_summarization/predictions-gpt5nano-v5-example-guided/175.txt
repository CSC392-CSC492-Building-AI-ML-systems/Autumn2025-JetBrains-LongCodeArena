# Biclustering

## Overview
Biclustering (co-clustering) simultaneously clusters rows and columns of a data matrix to identify submatrices (biclusters) where a subset of rows exhibits coherent patterns across a subset of columns. This approach is particularly useful when patterns are localized to specific portions of the data, such as in gene expression analysis or collaborative filtering.

In scikit-learn, biclustering is implemented primarily through spectral methods that reorder the data to reveal checkerboard patterns. The two main classes are SpectralBiclustering and SpectralCoclustering. After fitting, you can inspect the row and column cluster assignments to extract the corresponding biclusters.

Common attributes after fitting:
- row_labels_: cluster labels for rows
- column_labels_: cluster labels for columns

## Key Components
- SpectralBiclustering
  - Finds checkerboard patterns in the data by applying spectral methods to a bipartite representation.
  - Provides row and column cluster labels to identify biclusters.
- SpectralCoclustering
  - A coclustering variant that groups rows and columns into a consistent set of coclusters.
  - Also provides row_labels_ and column_labels_ after fitting.
- API highlights
  - fit(X): learn the row and column cluster structure from the data matrix X.
  - row_labels_: cluster assignment for each row (shape: n_samples)
  - column_labels_: cluster assignment for each column (shape: n_features)
  - Generated biclusters can be formed by selecting rows with a given row_label and columns with a given column_label.

## Usage
- Typical workflow: prepare a data matrix, choose the number of row/column clusters, fit the model, and inspect the resulting labels to extract biclusters.
- Example with SpectralBiclustering:

```python
from sklearn.cluster import SpectralBiclustering
import numpy as np

# Example data
X = np.random.RandomState(0).normal(size=(100, 80))

# Fit biclustering model (5 row clusters and 4 column clusters)
model = SpectralBiclustering(n_clusters=(5, 4), random_state=0)
model.fit(X)

# Access labels
row_labels = model.row_labels_        # shape: (100,)
col_labels = model.column_labels_     # shape: (80,)

# Form a single bicluster (for demonstration)
i, j = 1, 2  # specific bicluster indices
rows = np.where(row_labels == i)[0]
cols = np.where(col_labels == j)[0]
bicluster = X[np.ix_(rows, cols)]
```

- Example with SpectralCoclustering:

```python
from sklearn.cluster import SpectralCoclustering
import numpy as np

X = np.random.RandomState(0).normal(size=(100, 80))

model_cc = SpectralCoclustering(n_clusters=5, random_state=0)
model_cc.fit(X)

row_labels_cc = model_cc.row_labels_      # shape: (100,)
col_labels_cc = model_cc.column_labels_   # shape: (80,)

# Example: extract a cocluster
i = 0  # cocluster row group
j = 1  # cocluster column group
rows = np.where(row_labels_cc == i)[0]
cols = np.where(col_labels_cc == j)[0]
cocluster = X[np.ix_(rows, cols)]
```

## Details
- Concepts
  - Biclustering discovers patterns that depend on both a subset of rows and a subset of columns, resulting in a checkerboard-like partition of the data.
  - Spectral methods leverage eigen/decomposition techniques to identify coherent submatrices efficiently.
- Preprocessing
  - Standardization or normalization can influence bicluster detection. Consider centering or scaling features (columns) as appropriate for your application.
  - For some datasets, row-wise or column-wise normalization may help reveal meaningful patterns.
- How to interpret
  - Each pair of row_label and column_label defines a bicluster: the submatrix formed by the corresponding rows and columns.
  - Check for meaningful patterns within each bicluster (e.g., high/low mean, consistent structure) rather than relying solely on cluster IDs.
- Limitations
  - Spectral biclustering assumes the data contains checkerboard-like structure; performance degrades with noise or non-checkerboard patterns.
  - Scalability is suitable for medium-sized matrices; very large matrices may require preprocessing, dimensionality reduction, or more scalable variants.
- Related approaches
  - Biclustering is complementary to traditional clustering (which assigns each row or column to a single cluster) and to full clustering of either dimension only.
- References
  - For a practical implementation and in-depth discussion, see the biclustering documents and tutorials in scikit-learn’s clustering module.
