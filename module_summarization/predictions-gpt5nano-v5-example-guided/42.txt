# How to generate schema

## Overview
This module provides utilities to generate and apply the SQL schema for a database client. It uses the client’s schema generator to create the SQL statements and can optionally apply them to the database.

Key functions:
- get_schema_sql(client, safe) -> str: Generate the SQL to create the schema for the given client.
- generate_schema_for_client(client, safe) -> None: Generate and apply the SQL schema directly to the given client.

## Key Components
- get_schema_sql(client: "BaseDBAsyncClient", safe: bool) -> str
  - Description: Generates the SQL schema for the given client.
  - Behavior: Calls client.schema_generator(client).get_create_schema_sql(safe) and returns the SQL string.
  - Signature:
    ```python
    def get_schema_sql(client: "BaseDBAsyncClient", safe: bool) -> str:
        generator = client.schema_generator(client)
        return generator.get_create_schema_sql(safe)
    ```
- generate_schema_for_client(client: "BaseDBAsyncClient", safe: bool) -> None
  - Description: Generates and applies the SQL schema directly to the given client.
  - Behavior: Retrieves the SQL via get_schema_sql, logs the SQL at debug level, and, if non-empty, applies it using the generator.
  - Signature:
    ```python
    async def generate_schema_for_client(client: "BaseDBAsyncClient", safe: bool) -> None:
        generator = client.schema_generator(client)
        schema = get_schema_sql(client, safe)
        logger.debug("Creating schema: %s", schema)
        if schema:  # pragma: nobranch
            await generator.generate_from_string(schema)
    ```

## Usage
- Generate the SQL for a client without applying:
  ```python
  sql = get_schema_sql(client, safe=True)
  # sql contains the CREATE TABLE statements (or similar) for the client's schema
  ```
- Generate and apply the SQL schema for a client:
  ```python
  await generate_schema_for_client(client, safe=True)
  ```
- Notes:
  - The safe parameter controls whether tables are created with existence checks (e.g., CREATE TABLE IF NOT EXISTS) or not, via the schema generator.
  - If the generated schema is empty, no action is taken.
  - The process uses the client’s schema_generator to obtain a generator, then delegates SQL creation and application to that generator.

## Details
- The module relies on a BaseDBAsyncClient-like interface for typing (imported under TYPE_CHECKING).
- The schema generation flow:
  1) Acquire a generator: generator = client.schema_generator(client)
  2) Produce SQL: generator.get_create_schema_sql(safe)
  3) Optionally apply: await generator.generate_from_string(schema)
- Debug logging reflects the exact SQL being applied:
  ```python
  logger.debug("Creating schema: %s", schema)
  ```
- Guarding against empty schemas ensures no unnecessary operations are performed.
