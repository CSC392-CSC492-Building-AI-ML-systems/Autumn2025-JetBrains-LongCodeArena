# Analyzing a collection of text documents

## Overview

This module provides utilities to transform a collection of text documents into numerical feature vectors suitable for machine learning. It covers preprocessing, tokenization, stop-word handling, n-gram generation, and conversion to sparse matrices via various vectorizers (e.g., CountVectorizer, TfidfVectorizer, HashingVectorizer). It also includes helper functions for text normalization and simple HTML/XML stripping.

## Key Components

- HashingVectorizer
  - Converts text into a fixed-dimensional sparse matrix using the hashing trick.
- CountVectorizer
  - Builds a vocabulary from a collection of text documents and counts term occurrences (bag-of-words representation).
- TfidfTransformer
  - Transforms a count matrix to a TF-IDF representation.
- TfidfVectorizer
  - Combines CountVectorizer and TfidfTransformer into a single step to produce TF-IDF features.
- ENGLISH_STOP_WORDS
  - A built-in set of common English stop words to exclude during tokenization.
- strip_accents_unicode
  - Strips accentuated characters by normalizing to NFKD form and removing combining marks.
- strip_accents_ascii
  - Strips accents by transliterating to ASCII when possible.
- strip_tags
  - Simple HTML/XML tag stripper using a regular expression.
- _preprocess
  - Chainable preprocessing step (e.g., lowercasing, applying an accent function).
- _analyze
  - Orchestrates preprocessing, tokenization, and optional n-gram extraction.
- _word_ngrams
  - Helper for constructing word n-grams with optional stop-word filtering.
- _check_stop_list
  - Resolves stop-word specifications (e.g., "english", a custom list, or None).
- _VectorizerMixin
  - Shared tokenization/decode logic for text vectorizers.

## Usage

- Basic bag-of-words with default settings
  ```python
  from sklearn.feature_extraction.text import CountVectorizer

  docs = [
      "This is the first document.",
      "This is the second document.",
      "And this is the third one."
  ]

  vect = CountVectorizer()
  X = vect.fit_transform(docs)
  print(X.shape)  # (3, number_of_features)
  print(vect.get_feature_names_out())  # feature names
  ```

- TF-IDF representation
  ```python
  from sklearn.feature_extraction.text import TfidfVectorizer

  docs = [
      "This is the first document.",
      "This is the second document.",
      "And this is the third one."
  ]

  tfidf_vect = TfidfVectorizer(stop_words='english')
  X_tfidf = tfidf_vect.fit_transform(docs)
  print(X_tfidf.shape)
  ```

- Stop words and n-grams
  ```python
  vect = CountVectorizer(stop_words='english', ngram_range=(1, 2))
  X = vect.fit_transform(docs)
  print(vect.get_feature_names_out())
  ```

- Accent stripping and HTML stripping (via preprocessing)
  ```python
  from sklearn.feature_extraction.text import CountVectorizer
  from sklearn.feature_extraction.text import strip_accents_ascii, strip_tags

  preprocessor = lambda s: strip_accents_ascii(strip_tags(s))
  vect = CountVectorizer(preprocessor=preprocessor)
  X = vect.fit_transform(docs)
  ```

- Custom analyzer and decoding
  - You can override the analysis steps by supplying an analyzer callable, or customize with preprocessor, tokenizer, and ngrams.
  - The decode method supports input sources such as filenames or file-like objects when input is set accordingly.

## Details

- Preprocessing chain
  - The preprocessing pipeline is composed of optional steps: preprocessor, tokenizer, and n-grams. If an analyzer is provided, it replaces preprocessor/tokenizer/ngrams, and only the decoder argument is used for decoding input.
  - The _preprocess function applies lowercasing (if requested) and an optional accent function.

- Tokenization and n-grams
  - Tokenization and n-gram generation can be customized. The _word_ngrams helper handles stop-word filtering and construction of word-level n-grams.

- Stop words
  - Use ENGLISH_STOP_WORDS or supply a custom list. The _check_stop_list utility resolves stop-word specifications, including the built-in set when "english" is provided.

- Accent stripping
  - strip_accents_unicode and strip_accents_ascii are provided to normalize text by removing or transliterating accented characters. These can be integrated into preprocessing or as part of an analyzer.

- HTML/XML stripping
  - strip_tags provides a simple way to remove markup before tokenization when documents contain HTML/XML.

- Input handling
  - The vectorizers include decoding logic to handle various input sources (e.g., filenames, file-like objects) and convert them to strings for processing.

- Internal utilities
  - The module exposes several internal helpers (_preprocess, _analyze, _word_ngrams, _check_stop_list) that facilitate flexible and customizable text processing pipelines.
