Computational Performance
=========================

Overview
--------
This module provides multi-output meta-estimators and internal helpers that train multiple target estimators, typically by cloning a single base estimator per output. Computational performance is influenced by parallelization (n_jobs), cloning overhead, per-output training cost, and memory usage. The internal helpers shown below illustrate how per-output estimators are created and trained, either in parallel or incrementally.

Key Components
--------------
- _fit_estimator(estimator, X, y, sample_weight=None, **fit_params)
  - Clones the base estimator and fits it to the data. If sample_weight is provided, it is passed to fit.
  - Performance implication: cloning creates independent estimator instances for each output, enabling isolation but incurring clone overhead.

  Example from source:
  def _fit_estimator(estimator, X, y, sample_weight=None, **fit_params):
      estimator = clone(estimator)
      if sample_weight is not None:
          estimator.fit(X, y, sample_weight=sample_weight, **fit_params)
      else:
          estimator.fit(X, y, **fit_params)
      return estimator

- _partial_fit_estimator(estimator, X, y, classes=None, partial_fit_params=None, first_time=True)
  - Supports incremental learning via partial_fit. If first_time is True, the estimator is cloned to avoid side effects on the original estimator.
  - Performance implication: enables online or streaming learning with reduced memory pressure per update; however, there is overhead from cloning and managing partial_fit parameters.

  Example from source:
  def _partial_fit_estimator(
      estimator, X, y, classes=None, partial_fit_params=None, first_time=True
  ):
      partial_fit_params = {} if partial_fit_params is None else partial_fit_params
      if first_time:
          estimator = clone(estimator)

      if classes is not None:
          estimator.partial_fit(X, y, classes=classes, **partial_fit_params)
      else:
          estimator.partial_fit(X, y, **partial_fit_params)
      return estimator

- _available_if_estimator_has(attr)
  - Creates a conditional availability check based on whether sub-estimators expose a given attribute. This enables dynamic capability checks for nested estimators.
  - Performance note: helps avoid unnecessary computations if a feature is not supported by the base estimator.

  Example fragment from source:
  def _available_if_estimator_has(attr):
      """Return a function to check if the sub-estimator(s) has(have) `attr`."""
      def _check(self):
          if hasattr(self, "estimators_"):
              return all(hasattr(est, attr) for est in self.estimators_)
          if hasattr(self.estimator, attr):
              return True
          return False
      return available_if(_check)

- _MultiOutputEstimator
  - Base class for meta-estimators that operate on a base estimator and manage parallelization via n_jobs.
  - Parameter constraints include:
    - estimator: must have fit and predict methods
    - n_jobs: integer or None
  - Performance note: n_jobs controls parallel training across outputs; higher values can reduce wall-clock time on multi-core machines but increase memory usage.

Usage
-----
Typical usage leverages parallel training across outputs when supported by the meta-estimator and the underlying base estimator.

Example:
- Training a multi-output regressor with parallelization across outputs (n_jobs):
  from sklearn.multioutput import MultiOutputRegressor
  from sklearn.linear_model import Ridge
  from sklearn.datasets import make_regression

  X, y = make_regression(n_samples=200, n_features=20, n_targets=3, random_state=0)
  base = Ridge(alpha=1.0)
  mor = MultiOutputRegressor(base, n_jobs=4)
  mor.fit(X, y)

Notes on Performance
--------------------
- Parallelization
  - The module uses Parallel and delayed (via sklearn.utils.parallel) to train multiple output estimators concurrently when n_jobs > 1.
  - Benefits: can significantly reduce wall-clock time on multi-core systems.
  - Trade-offs: higher memory consumption due to multiple estimator copies; overhead from managing parallel tasks and data routing.

- Cloning Overhead
  - Each output gets a cloned copy of the base estimator. While this ensures independence, it adds cloning cost and memory usage.

- Incremental Learning
  - Partial fit supports streaming or large-scale data. The first-time clone ensures the original estimator isn’t modified, which can affect memory and performance depending on the base estimator’s internals.

- Data and Model Characteristics
  - The performance scales with the number of outputs and the complexity of the base estimator. For expensive base estimators, parallelization across outputs yields larger gains.
  - If outputs are highly correlated, consider joint learning approaches; otherwise, multi-output wrappers provide clean parallel training but may incur per-output overhead.

- Base class for multi-output meta-estimators:
  class _MultiOutputEstimator(MetaEstimatorMixin, BaseEstimator, metaclass=ABCMeta):
      _parameter_constraints: dict = {
          "estimator": [HasMethods(["fit", "predict"])],
          "n_jobs": [Integral, None],
      }
      @abstractmethod
      def __init__(self, estimator, *, n_jobs=None):
          self.estimator = estimator
          self.n_jobs = n_jobs

Additional considerations
-------------------------
- When configuring n_jobs, balance the number of parallel tasks with available memory. Each parallel task may hold its own copy of the estimator and data.
- For very large numbers of outputs or very large datasets, consider incremental or chunked processing to limit peak memory usage.
