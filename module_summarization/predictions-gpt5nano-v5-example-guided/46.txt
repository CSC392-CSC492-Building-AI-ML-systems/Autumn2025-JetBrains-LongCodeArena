# MongoDB Storage Service (storage.rst)

## Overview
This module provides a MongoDB-backed storage service for ASAB. It implements StorageServiceABC and uses motor (async MongoDB driver) together with pymongo and bson to perform asynchronous data operations. It supports upsert-style updates via a MongoDBUpsertor that queues modifications and applies them atomically. Configuration defaults are provided so the service can connect to MongoDB and select a database.

Key features:
- Async MongoDB integration via motor
- Simple get, get_by, collection, and delete operations
- Upsert support with set, inc, push, and unset operations
- Automatic handling of Versioning and Object IDs
- Optional decryption support for retrieved fields

Configuration defaults (config section asab:storage):
- mongodb_uri: mongodb://localhost:27017
- mongodb_database: asabdb

Example:
- mongodb_uri is used to initialize the AsyncIOMotorClient
- mongodb_database selects the database with tz-aware timezone handling

  class StorageService(StorageServiceABC):
      '''
      StorageService for MongoDB. Depends on `pymongo` and `motor`.
      '''

      def __init__(self, app, service_name, config_section_name='asab:storage'):
          super().__init__(app, service_name)
          self.Client = motor.motor_asyncio.AsyncIOMotorClient(asab.Config.get(config_section_name,'mongodb_uri'))

          self.Database = self.Client.get_database(
              asab.Config.get(config_section_name,'mongodb_database'),
              codec_options=bson.codec_options.CodecOptions(tz_aware=True, tzinfo=datetime.timezone.utc),
          )
          assert self.Database is not None

      def upsertor(self, collection: str, obj_id=None, version=0):
          return MongoDBUpsertor(self, collection, obj_id, version)

- Basic retrieval methods
  async def get(self, collection: str, obj_id, decrypt=None) -> dict:
      coll = self.Database[collection]
      ret = await coll.find_one({'_id': obj_id})
      if ret is None:
          raise KeyError("NOT-FOUND")
      if decrypt is not None:
          for field in decrypt:
              if field in ret:
                  ret[field] = self.aes_decrypt(ret[field])
      return ret

## Key Components

- StorageService
  - Purpose: MongoDB-backed storage layer for ASAB.
  - Dependencies: PyMongo, Motor, BSON.
  - Core responsibilities:
    - Initialize MongoDB client and database with timezone-aware codec options
    - Provide upsertor factory for per-collection upserts
    - Retrieve documents by _id or by arbitrary field
    - Delete documents by _id
    - Expose raw collection objects for advanced usage
  - Important configuration: asab:storage section with mongodb_uri and mongodb_database

- MongoDBUpsertor
  - Purpose: Upsertor implementation for MongoDB-backed storage.
  - Core responsibilities:
    - Build MongoDB update operations ($set, $inc, $push, $unset)
    - Support upsert behavior with versioning (_v)
    - Handle ObjectId generation for new objects
    - Execute updates with proper error handling (DuplicateKeyError → DuplicateError)
    - Optional webhook payload generation for upsert events
  - Key methods:
    - generate_id: returns a new ObjectId
    - execute: performs the upsert using MongoDB, applying modifications and handling errors
  - Notes:
    - Uses internal ModSet, ModInc, ModPush, ModUnset collections from UpsertorABC
    - If Version is 0 or None, performs an insert (upsert)
    - If Version is set, updates existing document matching _v and id

## Usage
- General flow:
  - Configure and initialize the storage service
  - Obtain an upsertor for a collection
  - Queue modifications (set, inc, push, unset)
  - Execute the upsert
  - Retrieve or delete as needed

- Example: basic usage
  - Configuration and service setup
    from storage import StorageService

    storage = StorageService(app, "storage", "asab:storage")

  - Create a new object (insert)
    up = storage.upsertor("users", obj_id=None, version=0)
    up.ModSet["name"] = "Alice"
    up.ModSet["email"] = "alice@example.com"
    await up.execute()
    # up.ObjId now contains the generated ObjectId

  - Update an existing object
    up = storage.upsertor("users", obj_id=existing_id, version=0)
    up.ModSet["name"] = "Alice Smith"
    await up.execute()

  - Retrieve an object
    user = await storage.get("users", existing_id)

  - Retrieve by a custom key
    user = await storage.get_by("users", "email", "alice@example.com")

  - Delete an object
    deleted_id = await storage.delete("users", existing_id)

- Notes on decryption
  - get and get_by support an optional decrypt parameter:
    ret = await storage.get("collection", obj_id, decrypt=["sensitive_field"])
  - If decrypt is provided, the specified fields are decrypted using self.aes_decrypt

- Webhooks
  - When WebhookURIs is configured, upsert operations may trigger webhook notifications with a structured payload containing collection, event_type, custom data, and upsertor data (excluding fields starting with "__")

## Details
- Configuration
  - The module adds default configuration for the storage:
    asab.Config.add_defaults(
      {
        'asab:storage': {
          'mongodb_uri':'mongodb://localhost:27017',
          'mongodb_database': 'asabdb',
        }
      }
    )
  - You can override mongodb_uri and mongodb_database in your application config.

- Dependencies
  - motor (async MongoDB driver)
  - pymongo
  - bson
  - asab (framework)
  - The service relies on UpsertorABC (base class for upsert operations) and DuplicateError for duplicate-key handling.

- Error handling
  - If a document is not found during get/get_by or delete, a KeyError("NOT-FOUND") is raised.
  - DuplicateKeyError from PyMongo is translated to DuplicateError by the upsertor.

- Performance and concurrency
  - Updates are performed atomically via MongoDB find_one_and_update with upsert semantics.
  - Versioning (_v) is used to manage optimistic concurrency during upserts.

- Extensibility
  - The design allows additional updates via ModSet, ModInc, ModPush, and ModUnset.
  - Raw collection access is available for custom MongoDB operations through the collection() method.

---

For developers integrating ASAB with MongoDB-backed storage, this module provides a straightforward, asynchronous interface for managing objects with upsert semantics, including conveniences for object creation, updates, retrieval, and deletion, along with optional decryption and webhook support.
