# Utilities for Developers

## Overview
This module provides a collection of compatibility utilities used by the project to support multiple Python, NumPy, and SciPy versions. It also includes helpers for threadpool management, resource access, and small wrappers that normalize behavior across library versions. These utilities are primarily for developers and internal use; they help maintain a stable API across a range of environments.

## Key Components
- Version and NaN checks
  - _object_dtype_isnan(X): Detects NaN in object-dtype arrays via the property X != X.
  - Example:
    - return X != X

- NumPy/SciPy version compatibility for percentile
  - _percentile(a, q, *, method="linear", **kwargs): Wrapper to support NumPy < 1.22 where the keyword was interpolation.
  - If NumPy < 1.22, percentile is provided via _percentile; otherwise, use numpy.percentile directly.
  - Example:
    - percentile(a, q, interpolation=method, **kwargs)

- Threadpool management
  - _get_threadpool_controller(): Returns a global ThreadpoolController for threadpoolctl >= 3.0.0 if available.
  - threadpool_limits(limits=None, user_api=None): Apply thread-limits using the global controller when present.
  - threadpool_info(): Retrieve current threadpool configuration.
  - These wrappers enable consistent thread usage behavior across environments.
  - Example:
    - with threadpool_limits(limits=1):
        do_work()

- Deprecated wrapper
  - delayed(function): Backwards-compatible import path forwarding to sklearn.utils.parallel.delayed with a deprecation note.
  - Example:
    - @deprecated(...)
      def my_func(...)

- Mode compatibility for SciPy
  - _mode(a, axis=0): Compatibility wrapper for scipy.stats.mode across SciPy versions.
  - Handles differences in output shape and behavior (e.g., axis=None with keepdims=True).
  - Example:
    - mode = _mode(a, axis=0)

- Sparse linear algebra compatibility
  - _sparse_linalg_cg(A, b, **kwargs): Compatibility wrapper for SciPy’s cg.
  - For older SciPy versions, it maps rtol -> tol and sets a default atol="legacy".
  - Example:
    - x, info = _sparse_linalg_cg(A, b, tol=1e-6)

- Importlib.resources backport for data access
  - _open_text(data_module, data_file_name): Open a text data file packaged with a module.
  - _open_binary(data_module, data_file_name): Open a binary data file packaged with a module.
  - _read_text(descr_module, descr_file_name): Read text data from a packaged resource.
  - _path(data_module, data_file_name): Obtain a path-like object to a packaged resource.
  - These helpers support Python versions without full importlib.resources support.
  - Example:
    - with _open_text(some_module, "data.txt") as f:
        content = f.read()

- Sparse container type awareness
  - CSR_CONTAINERS, CSC_CONTAINERS, COO_CONTAINERS, LIL_CONTAINERS, DOK_CONTAINERS, BSR_CONTAINERS
  - Lists of accessible sparse container types (augmented with sparse arrays when available in SciPy >= 1.8).
  - Useful for generic handling of sparse matrices across versions.

- SciPy line search import fallback
  - Supports importing line search functions from the appropriate SciPy module depending on version:
    - line_search_wolfe1, line_search_wolfe2 (SciPy >= 1.8)
    - line_search_wolfe2, line_search_wolfe1 (older SciPy)

- Notes on internal APIs
  - Several utilities are intended for internal use and provide backward-compatibility shims rather than public APIs.

- Version-specific notes (maintenance guidance)
  - These shims are kept to support older environments. When the minimum supported versions advance (e.g., NumPy >= 1.22, SciPy >= 1.12, Python >= 3.9), remove the corresponding compatibility code.
  - Example decisions:
    - NumPy percentile: remove _percentile and rely on numpy.percentile directly once NumPy 1.22+ is the minimum.
    - _mode: remove version branches once SciPy minimum reaches a version with consistent mode behavior.

## Usage
- Restricting thread usage during computations:
  - with threadpool_limits(limits=1):
      run_heavy_computation()

- Reading packaged data:
  - with _open_text(my_module, "description.txt") as f:
      text = f.read()

- Memory-efficient sparse Solvers (internal usage):
  - x, info = _sparse_linalg_cg(A, b, tol=1e-6)

- Accessing mode results:
  - m = _mode(array, axis=0)

- Accessing a packaged resource path:
  - p = _path(my_module, "data.bin")

## Details
- This module emphasizes compatibility across Python and library upgrades.
- The internal utilities are designed to minimize breaking changes when dependencies evolve.
- For maintainers: if you add a compatibility fix, annotate the version at which the fix is no longer required, so future removals can be scheduled cleanly.
- Examples and behavior shown here reflect the current implementation, including version checks such as:
  - NumPy: < 1.22 uses _percentile; >= 1.22 uses percentile directly
  - SciPy: 1.8+ adds sparse arrays; 1.12+ affects cg behavior
  - Python: 3.9+ uses importlib.resources enhancements
