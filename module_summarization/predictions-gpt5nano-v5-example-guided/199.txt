# Neural network models (supervised)

## Overview
This module provides low-level utilities for supervised neural network models, including in-place activation functions, their derivatives for backpropagation, and common loss functions. The functions are designed to work with NumPy arrays (shape: n_samples × n_features) and support in-place updates to minimize memory usage.

Key capabilities include:
- In-place activations: identity, logistic (sigmoid), tanh, relu, softmax
- In-place derivatives for backpropagation: identity, logistic, tanh, relu
- Common loss functions: squared loss, logistic (cross-entropy) loss, binary logistic loss

Dependencies: NumPy and SciPy (for the logistic sigmoid and log-loss computations).

## Key Components

### In-place activations
- identity
- logistic (sigmoid)
- tanh
- relu
- softmax

These are exposed via the ACTIVATIONS mapping, which maps a string name to the corresponding in-place function.

Example:
- Apply ReLU in place:
  ```python
  import numpy as np
  X = np.array([[-1.0, 2.0], [3.0, -0.5]])
  ACTIVATIONS["relu"](X)
  # X is modified in place
  ```

### Activation derivatives (for backpropagation)
- identity
- logistic
- tanh
- relu

These are exposed via the DERIVATIVES mapping and modify the backpropagated error delta in place, using the forward-pass activation output Z.

Example:
- Backprop through logistic:
  ```python
  Z = np.array([[0.2, 0.8], [0.5, 0.3]])
  delta = np.ones_like(Z)
  DERIVATIVES["logistic"](Z, delta)
  # delta is modified in place
  ```

### Loss functions
- squared_loss(y_true, y_pred)
  - Regression loss: 0.5 * mean((y_true - y_pred)^2)
- log_loss(y_true, y_prob)
  - Multiclass logistic loss (cross-entropy) from predicted probabilities
- binary_log_loss(y_true, y_prob)
  - Binary cross-entropy loss (useful for multilabel cases)

Example:
- Squared loss:
  ```python
  y_true = np.array([[1.0], [0.0], [1.0]])
  y_pred = np.array([[0.9], [0.2], [0.8]])
  loss = squared_loss(y_true, y_pred)
  ```
- Cross-entropy loss for probabilities:
  ```python
  y_true = np.array([[1, 0, 0], [0, 1, 0]])
  y_prob = np.array([[0.7, 0.2, 0.1], [0.1, 0.8, 0.1]])
  loss = log_loss(y_true, y_prob)
  ```
- Binary cross-entropy (multilabel context):
  ```python
  y_true = np.array([[1], [0], [1]])
  y_prob = np.array([[0.9], [0.2], [0.8]])
  loss = binary_log_loss(y_true, y_prob)
  ```

### Loss function registry
- LOSS_FUNCTIONS: A mapping of loss function names to implementations (e.g., "squared", "log", "binary_log"). This provides a convenient registry for selecting loss functions by name.

## Usage

- Activation and derivative usage in a training loop (conceptual):
  - Forward pass:
    - Apply activation in place to the pre-activation outputs:
      ```python
      Z = pre_activations.copy()
      ACTIVATIONS["relu"](Z)
      ```
  - Backward pass:
    - Compute delta (error signal) and propagate through the activation:
      ```python
      delta = initial_delta.copy()
      DERIVATIVES["relu"](Z, delta)
      ```
- Loss computation:
  - Choose appropriate loss for your task (regression or classification) and compute it from true labels and predictions:
    ```python
    loss = squared_loss(y_true, y_pred)
    # or for probabilities in classification
    loss = log_loss(y_true, y_prob)
    ```

## Details

- In-place operations save memory by overwriting the input arrays during activation and derivative computations.
- The logistic activation uses SciPy’s expit for numerical stability.
- Softmax in-place implementation subtracts the row-wise maximum to prevent overflow, then exponentiates and normalizes.

This module is intended as a utility layer for higher-level neural network components, providing reusable building blocks for supervised learning workflows.
