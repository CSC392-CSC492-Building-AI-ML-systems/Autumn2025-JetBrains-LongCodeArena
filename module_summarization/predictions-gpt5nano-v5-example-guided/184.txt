# Gaussian Processes — Kernels and Hyperparameters

## Overview
This module provides kernels for Gaussian process regression and classification. Kernels can be combined to form complex covariances (via addition, multiplication, and scalar exponentiation) and can include scalar values that are automatically converted to a constant kernel. All kernels support analytic gradient-based hyperparameter optimization. Hyperparameters are defined with bounds (forming a rectangular search space) or can be marked as fixed to exclude them from optimization. The implementation is inspired by the kernel module of the george package.

Key features include:
- Kernel composition: a + b, a * b, and kernel scaling with ** (e.g., scalar exponents) to build expressive covariance structures.
- Hyperparameter management: bounds, fixed parameters, and vector-valued hyperparameters (e.g., anisotropic length scales).
- Gradient-based optimization: enables efficient hyperparameter tuning within a specified search space.

---

## Key Components

- Hyperparameter
  - A namedtuple describing a kernel hyperparameter with fields:
    - name: parameter name (e.g., constant_value)
    - value_type: e.g., "numeric"
    - bounds: 2D array of shape (n_elements, 2) or the string "fixed"
    - n_elements: number of elements (default 1; >1 for vector-valued parameters)
    - fixed: whether the parameter is fixed (cannot be optimized)
  - Supports vector-valued hyperparameters (anisotropic parameters) and derived fixed behavior when bounds indicate it.
  - Example representation:
    - Hyperparameter(name='constant_value', value_type='numeric',
      bounds=array([[ 0., 10.]]), n_elements=1, fixed=False)

  - Example usage (from the docstring):
    - Access hyperparameters of a kernel:
      - for hyperparameter in kernel.hyperparameters: print(hyperparameter)
    - Inspect parameters via get_params:
      - params = kernel.get_params()
      - for key in sorted(params): print(f"{key} : {params[key]}")

- Kernel (base class)
  - Abstract base class for all kernels used in Gaussian processes.
  - Supports standard interfaces expected by scikit-learn-like estimators (e.g., get_params for parameter introspection and copying via clone).
  - The formatting and validation of hyperparameters are centralized within concrete kernel implementations that derive from this base.

- _check_length_scale(X, length_scale)
  - Helper function to validate and normalize length-scale parameters.
  - Ensures the length-scale is 1D and compatible with the data dimensionality.
  - Raises informative errors if anisotropic length-scales are specified inconsistently with the input data.

- Kernel composition and scalar exponentiation
  - Kernels can be combined using:
    - Addition: k1 + k2
    - Multiplication: k1 * k2
  - Can be exponentiated with a scalar: k ** scalar
  - Scalar values in expressions are converted to a constant kernel automatically.

- Hyperparameter optimization
  - Each hyperparameter can have lower and upper bounds, defining a rectangular search space.
  - Hyperparameters can be declared "fixed" to exclude them from optimization.
  - All kernels support analytic gradients to facilitate gradient-based optimization of hyperparameters.

## Usage

- Basic kernel construction and hyperparameters
  - Example (ConstantKernel from the docstring example):
    - from sklearn.gaussian_process.kernels import ConstantKernel
    - kernel = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 10.0))

- Hyperparameter introspection
  - Access the hyperparameters defined for a kernel:
    - for hyperparameter in kernel.hyperparameters:
        print(hyperparameter)

- Hyperparameter and parameter inspection
  - params = kernel.get_params()
  - for key in sorted(params):
        print(f"{key} : {params[key]}")

- Example workflow (from the Hyperparameter docstring)
  - X, y = make_friedman2(n_samples=50, noise=0, random_state=0)
  - kernel = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 10.0))

- Length-scale validation helper:

## Details

- Composition and flexiblity
  - Kernels can be experienced with modular composition to form rich covariance structures suitable for Gaussian process regression and classification.

- Hyperparameter search space
  - The bounds define a rectangular search space, enabling straightforward gradient-based optimization routines to explore hyperparameters efficiently.

- Fixed hyperparameters
  - Hyperparameters can be declared fixed, excluding them from optimization while still being used in kernel evaluation.

- Anisotropic length scales
  - The module supports anisotropic length scales (n_elements > 1) provided the dimension matches the input data.

- Inspirations and references
  - This module is strongly inspired by the kernel module of the george package and aligns with scikit-learn-like APIs for ease of use and compatibility.
