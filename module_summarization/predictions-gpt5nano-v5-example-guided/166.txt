filenode - simulating a filesystem with PyTables
-----------------------------------------------

Overview
--------
The filenode module provides a binary file interface to PyTables nodes stored inside PyTables database files. It allows treating a PyTables node as a regular file-like object (readable and writable according to the opened mode). New nodes can be created with new_node() and existing ones opened with open_node(). Read access is always available; write access is enabled when the file node is opened with a writable mode (e.g., appending). Only binary I/O is supported. For usage instructions, see filenode_usersguide. Since version 3.0 the module is fully compliant with the interfaces defined in the io module.

Key Components
--------------
- NodeType
  - Value: 'file'
  - Purpose: internal node system attribute representing the type of node.

- NodeTypeVersions
  - Values: [1, 2]
  - Purpose: supported values for the NODE_TYPE_VERSION system attribute.

- RawPyTablesIO
  - Base class: io.RawIOBase
  - Purpose: file-like binary I/O for PyTables nodes. Implements a PyTables-backed raw binary stream usable with Python’s io interface.
  - Core behavior:
    - Maps versioned data shapes via _size_to_shape, allowing compatibility with multiple NODE_TYPE_VERSION values.
    - Operates on a PyTables node (self._node) and reflects its attributes (nrows, atom, NODE_TYPE_VERSION, etc.).
    - Provides standard IO methods and properties such as mode, tell, seek, seekable, fileno, close, flush, truncate, readable, writable, readinto, and readline.
  - Version handling:
    - _version is derived from node.attrs.NODE_TYPE_VERSION.
    - _vshape is chosen from _size_to_shape[self._version] to determine how data is shaped when read.
  - Mode handling:
    - The IO object respects the requested mode or defaults to the node’s file mode.
    - readable() and writable() reflect the requested/opened mode.
  - Notes:
    - Read path uses node.read(start, stop) to fetch data.
    - Truncate grows the file and relies on internal _append_zeros to extend the underlying data.
    - Close releases the reference to the node to allow the host file to close.

Usage
-----
1) Obtain or create a PyTables file node (via new_node/open_node, as per the filenode workflow).
2) Wrap the node with RawPyTablesIO to interact using Python’s file-like semantics.
3) Use standard IO operations (read, seek, tell, etc.) according to the opened mode.

Example
-------
The following illustrates basic usage with a PyTables node (pseudo-code to show intended pattern):

.. code-block:: python

    # Obtain a PyTables file node (the actual creation/opening is done via PyTables APIs)
    node = new_node(...)           # or open_node(...)

    # Open as a binary file-like object
    fio = RawPyTablesIO(node, mode='r+')  # read/write, depending on node and mode

    # Read into a pre-allocated buffer
    buf = bytearray(4096)
    n = fio.readinto(buf)
    data = buf[:n]

    # Seek and continue reading or writing
    fio.seek(0)                    # go to start
    # ... further IO operations ...

    # When finished
    fio.close()

Notes
-----
- Read access is always available; write access depends on the mode (and underlying node state).
- Truncation is designed to allow growth of the file node; rewrites/deletes are not supported in the current implementation.
- The module is designed to be compliant with Python’s io interfaces (as of version 3.0).

Details
-------
- The filenode module defines a binary file interface to PyTables nodes, enabling file-like interaction inside PyTables database files.
- Primary class: RawPyTablesIO, which extends io.RawIOBase and provides low-level binary I/O bound to a PyTables node.
- Versioning: NODE_TYPE_VERSION guides data shape handling for compatibility across versions; supported versions are [1, 2].
- Data model: Uses node attributes and methods (e.g., _v_file, nrows, atom.dtype) to implement IO behavior. Data is read via node.read(start, stop) and written via the underlying PyTables mechanisms (append semantics and growth through _append_zeros for truncate).
- Documentation references: See filenode_usersguide for detailed use instructions and examples in the broader PyTables context.
- Change history: As of version 3.0, the module was rewritten to conform fully to the io module interfaces.
