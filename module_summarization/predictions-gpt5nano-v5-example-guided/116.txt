# Utilities for Searching ASDF Trees

## Overview
This module provides AsdfSearchResult, a representation of the results from an ASDF file search. It supports chaining filters, adjusting display formatting, and applying replacements to matched leaf nodes. It is designed to help locate and manipulate nodes within an ASDF tree based on keys, types, values, or custom filters.

Key capabilities include:
- Building complex search queries by combining key, type, value, and custom filters.
- Controlling how results are rendered (rows, columns, and whether primitive values are shown).
- Replacing matched leaf nodes in place with a new value.

The primary public surface is the AsdfSearchResult class, which is returned by methods like AsdfFile.search.

## Key Components

### AsdfSearchResult
Represents the result of a search performed on an ASDF tree. It encapsulates:
- identifiers: path-like identifiers to matched nodes
- node: the root of the tree being searched
- filters: a list of filter callables that refine the result set
- parent_node: the parent container of matched leaf nodes
- max_rows, max_cols: rendering limits for display
- show_values: whether to display primitive values in the representation

Core methods:

- format(max_rows=NotSet, max_cols=NotSet, show_values=NotSet)
  - Change formatting parameters of the rendered tree.
  - Parameters
    - max_rows: Maximum number of lines to print; may be an int, None (no limit), a tuple (per-depth limits), or NotSet to retain the current value.
    - max_cols: Maximum line length; may be an int, None (unconstrained), or NotSet to retain the current value.
    - show_values: If False, do not display primitive values; NotSet to retain the current value.
  - Returns: A new AsdfSearchResult with updated formatting settings.
  - Example:
    ```python
    res2 = res.format(max_rows=50, max_cols=120, show_values=False)
    ```

- search(key=NotSet, type=NotSet, value=NotSet, filter=None)
  - Narrow the search results further by:
    - key: constrain by the node key or dict/list index. Accepts NotSet (unconstrained), a string (regular expression), or any other value (must equal the queried key/index).
    - type: constrain by node type. Accepts NotSet, a string (regular expression on the fully-qualified type name), a compiled regex (typing.Pattern), or a builtins.type (node must be an instance of this type).
    - value: constrain by node value. Accepts NotSet, a string (regular expression on the string representation of the node), a compiled regex, or any other object (must equal the queried value). If the value is a regex and the node is a container, the search avoids matching container representations.
    - filter: a callable that accepts (node, identifier) and returns True to retain the node.
  - Returns: A new AsdfSearchResult containing the additional filter criteria.
  - Notes:
    - Strings passed for type and value are compiled to regex patterns when convenient.
    - Type matching supports both regex patterns and direct type checks.
  - Example:
    ```python
    # Narrow by key pattern
    res = af.search(key='^gravity_.*')
    # Narrow by exact type (builtins.type) or by regex on fully-qualified type name
    res2 = af.search(type='builtins.int')
    # Use a custom filter
    res3 = af.search(filter=lambda node, idx: isinstance(node, int))
    ```

- replace(value)
  - Assign a new value in place of all leaf nodes that pass the current filters.
  - Mechanism:
    - Collects (identifier, parent) pairs for leaf nodes matching the filters.
    - Iterates over the collected pairs and sets parent[identifier] = value.
  - Example:
    ```python
    # Replace all matched leaf values with None
    res.replace(None)
    ```

- node (property)
  - Retrieve the leaf node of a tree when there is exactly one search result.

- Internal helpers (brief)
  - _maybe_compile_pattern(query): If the query is a string, compile it into a regular expression; otherwise, return the query as-is.
  - _safe_equals(a, b): Safely compare two values, returning False on any exception.
  - _get_fully_qualified_type(value): Return the fully-qualified type name for a value:
    - If the type is from builtins, return just the type name.
    - Otherwise, return "module.name".
  - Note: The implementation uses additional internal helpers (e.g., for wrapping filters) not exposed publicly.

## Usage

Typical workflow:
- Open or read an ASDF file and perform a search on it.
- Narrow results by key, type, or value as needed.
- Optionally format the output for display.
- Optionally replace matched leaf nodes.

Code example snippet:
```
# Assume af is an opened AsdfFile and NotSet is imported
# Basic search by key pattern
res = af.search(key='^gravity_.*')

# Further narrow by type (regex on fully-qualified type name)
res = res.search(type='builtins.int')

# Limit display and hide primitive values
res = res.format(max_rows=60, max_cols=100, show_values=False)

# Replace matched leaf values with a placeholder
res.replace(None)
```

## Details

- NotSet is used as a sentinel to indicate that a parameter should retain its current value during format and subsequent searches.
- The search method supports flexible inputs:
  - key: NotSet, exact value, or a regex on the node identifier
  - type: NotSet, a regex string, a compiled Pattern, or a builtins.type
  - value: NotSet, a regex on the node’s string representation, or exact equality
  - filter: a callable that receives (node, identifier) and returns a boolean
- Type matching uses fully-qualified type names for regex comparisons, except when a direct builtins.type is supplied (which enforces isinstance checks).

This module focuses on enabling powerful, composable, and in-place queries over ASDF trees, with emphasis on flexible filtering, safe comparisons, and convenient rendering controls.
