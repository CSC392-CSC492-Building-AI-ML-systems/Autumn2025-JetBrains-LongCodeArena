# Polys: Polynomial factorization routines (characteristic zero)

## Overview
This module provides factorization routines for polynomials over characteristic zero domains. It supports:
- Univariate and multivariate factoring over the integers (Z) via Zassenhaus and Enhanced Zassenhaus (EEZ) methods.
- Multivariate factoring over algebraic number fields using Trager-style approaches.
- Internal utility routines for trial division and bound computations.

Key features include automatic handling of content, primitive parts, and multiplicities, as well as integration with the project's domain/ring abstractions and error handling.

The implementation draws on classical algorithms for polynomial factorization, notably Zassenhaus-type methods for Z[x] and related multivariate lifting approaches. See the references for details.

References
- Gathen and Gerhard, Modern Computer Algebra (e.g., Zassenhaus-based factoring)
- See internal documentation strings for algorithm notes (e.g., Trager’s method for algebraic number fields)

## Key Components

- _Factor (Mixin class)
  - Core internal factoring utilities used by the higher-level factorization routines.
  - Methods:
    - _trial_division(self, f, factors)
      - Performs simple trial-division of a polynomial f by a list of factors, returning the factors with their multiplicities, sorted.
      - Source excerpt:

    - _aa_factor_trager(self, f)
      - Factor multivariate polynomials over algebraic number fields using Trager-style techniques.
      - Handles ground polynomials, square-free parts, and Cofactor-based factorization with linear changes of variables.
      - Source excerpt:
        - def _aa_factor_trager(self, f): ...
    - _zz_factor(self, f)
      - Factor polynomials in Z[X], covering both univariate and multivariate cases.
      - Univariate path:
        - Factor content and primitive part.
        - Optional irreducibility/cyclotomic checks based on configuration flags.
        - Use Zassenhaus or EEZ-based factoring, followed by trial-division to recover multiplicities.
      - Multivariate path:
        - Reduce to primitive parts, project to a univariate component, factor, and lift factors back to the multivariate ring.
      - Source excerpt:
        - def _zz_factor(self, f): ...
    - _zz_mignotte_bound(self, f)
      - Compute the Mignotte bound for multivariate polynomials in Z[X], used to bound possible factor sizes during Zassenhaus-type factoring.
      - Source excerpt:
        - def _zz_mignotte_bound(self, f): ...
    - _zz_zassenhaus(self, f)
      - Factor a primitive, square-free polynomial in Z[x] using a Zassenhaus-based approach.
      - Includes prime selection where the polynomial is square-free modulo p, modular factorization, Hensel lifting, and combinatorial lifting of factors.
      - Source excerpt:

## Usage

Examples illustrating typical usage through the high-level factoring interface (factor_list):

- Univariate factoring over ZZ (integers):
  ```
  >>> _, x = ring('x', ZZ)
  >>> (2*x**4 - 2).factor_list()
  (2, [(x - 1, 1), (x + 1, 1), (x**2 + 1, 1)])
  ```

- Multivariate factoring over ZZ:
  ```
  >>> _, x, y = ring('x y', ZZ)
  >>> (2*x**2 - 2*y**2).factor_list()
  (2, [(x - y, 1), (x + y, 1)])
  ```

These examples demonstrate the typical interface: factor_list returns a tuple with the content and a list of irreducible factors paired with their multiplicities.

## Details

- Univariate vs. Multivariate paths
  - Univariate: Factorization proceeds on the primitive part, with optional irreducibility and cyclotomic checks controlled by configuration flags (e.g., USE_IRREDUCIBLE_IN_FACTOR, USE_CYCLOTOMIC_FACTOR). The implementation then applies Zassenhaus (or EEZ) factoring, followed by trial division to recover multiplicities.
  - Multivariate: The polynomial is reduced to a primitive, square-free core, and factoring proceeds via a combination of projection to univariate components, lifting (e.g., Wang or Hensel lifting), and recursive factoring in extension rings. The result is reassembled into the multivariate ring.

- Algorithms
  - Zassenhaus-based factoring for Z[x], including bound computations (Mignotte bounds) and prime selection to ensure favorable modular factorizations.
  - Enhanced Extended Zassenhaus (EEZ) and related lifting techniques for robust multivariate factorization.
  - Trager-style factoring for algebraic number fields (via _aa_factor_trager) to handle multivariate polynomials over algebraic extensions.

- Configuration and dependencies
  - The module depends on project-wide components such as domain/ring abstractions, modular arithmetic primitives, factor utilities, and error types.
  - Configuration queries (e.g., USE_IRREDUCIBLE_IN_FACTOR, USE_CYCLOTOMIC_FACTOR) influence the factoring strategy.

- References
  - Gathen, J. von, and J. Gerhard. Modern Computer Algebra (context for Zassenhaus-type methods and bounds)
  - In-code documentation notes (e.g., _zz_zassenhaus, _aa_factor_trager) provide algorithmic insights and representative examples.
