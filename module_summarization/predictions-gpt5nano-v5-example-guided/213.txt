Nonparametric Methods
=====================


Overview
--------

This module provides univariate nonparametric density estimation using kernel methods. The primary interface is the KDEUnivariate class, which fits a kernel density estimate (KDE) to a one-dimensional sample and exposes the estimated density on a grid (support). It supports FFT-based and non-FFT implementations, various kernel choices, and flexible bandwidth selection.

Key components
--------------

- KDEUnivariate
  - Univariate kernel density estimator with a rich set of options for kernel, bandwidth, grid size, and post-estimation quantities.
  - Core methods:
    - __init__(endog): initialize with the data to be estimated.
    - fit(kernel, bw, fft, weights, gridsize, adjust, cut, clip): fit the KDE with configurable options.
  - Outputs:
    - density: estimated density on the support grid.
    - support: grid points where the density is evaluated.
    - bw: chosen bandwidth.
    - kernel: kernel object used for the estimation (instantiated with the chosen bandwidth).
    - density-related attributes (e.g., cdf, if requested) are available after fitting.
  - Notes:
    - If fft is True, only Gaussian kernel is supported and weights are not implemented.
    - If bw is a string, it selects a predefined rule; otherwise bw can be a float or a callable.
    - If a callable is provided for bw, it should take exactly two parameters (fn, kern) and return a float.

- kernel_switch
  - Mapping from short kernel codes to kernel classes:
    - gau (Gaussian)
    - epa (Epanechnikov)
    - uni (Uniform)
    - tri (Triangular)
    - biw (Biweight)
    - triw (Triweight)
    - cos (Cosine)
    - cos2 (Cosine2)
  - Each kernel class is instantiated with h=bw (bandwidth).

- _checkisfit(self)
  - Internal helper to verify that fit has been called before accessing estimated quantities (e.g., cdf).

- References and context
  - Kernel density estimation in one dimension; related literature and resources are provided in the module header, including Racine (2008) and Silverman (Density Estimation for Statistics and Data Analysis).

Usage
-----

- Basic workflow (univariate KDE):

```
import numpy as np
import statsmodels.api as sm

# Sample data
nobs = 300
np.random.seed(1234)
data = np.random.normal(size=nobs)

# Initialize and fit KDE
dens = sm.nonparametric.KDEUnivariate(data)
dens.fit()  # uses default kernel 'gau' and 'normal_reference' bandwidth

# Access results
density = dens.density      # estimated density values on the support
support = dens.support      # grid points (x-values)
bw_used = dens.bw            # chosen bandwidth
kernel_used = dens.kernel    # kernel instance (Gaussian by default)
```

- Accessing the CDF (if supported by the kernel/density machinery):

```
import matplotlib.pyplot as plt

plt.plot(dens.cdf)  # requires that fit has been called
plt.show()
```

- Customization options (examples):

```
# Different kernel
dens.fit(kernel="epa")

# Custom bandwidth (float)
dens.fit(bw=0.5)

# User-supplied bandwidth function
def bw_fn(x, kern):
    # user-defined how bandwidth varies with data or grid
    return 0.75
dens.fit(bw=bw_fn)

# FFT-based path (Gaussian kernel only)
dens.fit(fft=True, kernel="gau")

# Non-FFT path with a specific grid size
dens.fit(gridsize=1000)
```

Details
-------

- Kernel choices
  - The estimator supports several kernel functions (Gaussian, Epanechnikov, Uniform, Triangular, Biweight, Triweight, Cosine, Cosine2). The kernel influences density estimation and, in particular, the shape of the estimated density.

- Bandwidth selection
  - Default bandwidth methods include:
    - "scott": 1.059 * A * nobs^(-1/5), where A = min(std(x), IQR/1.34)
    - "silverman": 0.9 * A * nobs^(-1/5), same A
    - "normal_reference": C * A * nobs^(-1/5), where C depends on the kernel
  - A numeric value can be supplied as the bandwidth.
  - A callable can be supplied to compute a data- or kernel-dependent bandwidth. The callable should implement the signature fn(x, kern).

- FFT vs. direct convolution
  - FFT-based density estimation is faster for large samples but currently supports only the Gaussian kernel (kernel="gau") and does not implement weights.
  - The non-FFT path performs direct computation of the KDE on a grid, which is more flexible with respect to kernels and weighting.

- Outputs and attributes
  - density: estimated density values on the support grid
  - support: x-values of the grid where density is evaluated
  - bw: bandwidth used in the estimation
  - kernel: kernel instance configured with the selected bandwidth
  - _cache: internal cache to store intermediate results between operations

- Related functionality
  - KDEUnivariate is designed for univariate, continuous data and is significantly faster than multivariate counterparts for the univariate case.
  - For multivariate nonparametric methods and mixed data, consider KDEMultivariate (not shown here but referenced in the module).

Notes
-----

- The exact implementations of kdensity and kdensityfft are used under the hood; the public API exposed to users is via KDEUnivariate.fit and the resulting attributes (density, support, bw, kernel, etc.).
- See Also: KDEMultivariate for multivariate or mixed-data nonparametric density estimation.
