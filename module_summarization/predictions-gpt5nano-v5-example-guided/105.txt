# Generate documentation for this code

## Overview
This module constructs a Sphinx-ready documentation string at import time. It builds a complete set of mathematical formulas representing generated functions in the meijerint package by populating a lookup table and formatting the results as reStructuredText with embedded LaTeX.

Key steps:
- Creates a defaultdict-based lookup table t and fills it via _create_lookup_table(t).
- Iterates the sorted contents of t to generate sections and math directives.
- Distinguishes elementary functions from other function groups and formats headers accordingly.
- Outputs the final documentation as the module’s __doc__ attribute.

The resulting documentation is intended to be consumed by Sphinx, providing an auto-generated, LaTeX-friendly reference of the generated formulas.

## Key Components
- t: dict[tuple[type,...], list[typing.Any]]
  - A defaultdict(list) acting as a categorized lookup table for generated functions.
- _create_lookup_table
  - Populates t with the generated formulas, their groupings, conditions, and hints.
- doc
  - A string accumulator that builds the final documentation content.
- The generation loop
  - Iterates over sorted(t.items(), key=default_sort_key).
  - If the category key about is empty (), it creates the "Elementary functions:" section header.
  - Otherwise, it creates a "Functions involving ..." header, listing involved functions via LaTeX.
  - For each entry (formula, gs, cond, hint) in a category:
    - Determines the generated expression g (either a Symbol marked as generated or an additive combination of factors).
    - Builds the equation object obj = Eq(formula, g).
    - Formats any condition as a LaTeX string when present.
    - Appends a Sphinx math directive:
      .. math::
        <latex(obj)><cond>
- __doc__
  - The final documentation string assigned to the module’s __doc__ attribute.

Code reference (key logic excerpt):
- Building sections and formulas
  for about, category in sorted(t.items(), key=default_sort_key):
      if about == ():
          doc += 'Elementary functions:\n\n'
      else:
          doc += ('Functions involving'+
                  ', '.join(f'`{latex(list(category[0][0].atoms(func))[0])}`'
                            for func in about) + ':\n\n')
      for formula, gs, cond, hint in category:
          if not isinstance(gs, list):
              g = Symbol('\\text{generated}')
          else:
              g = Add(*[fac*f for (fac, f) in gs])
          obj = Eq(formula, g)
          if cond is True:
              cond = ''
          else:
              cond = f',\\text{{ if }} {latex(cond)}'
          doc += f'.. math::\n  {latex(obj)}{cond}\n\n'

## Usage
- How it works at runtime:
  - Importing this module triggers the docstring construction, and __doc__ is populated with the generated documentation.
- Accessing the generated documentation:
  - After import, the documentation can be read from the module’s __doc__ attribute.
  - This makes it convenient for Sphinx to include as part of the API reference.

Usage example:
- Basic usage
  - import the module to trigger generation
  - print(g_functions.__doc__)  # or reference in Sphinx

Code example (illustrative, using the module’s mechanism):
- from ..meijerint import _create_lookup_table
- t = collections.defaultdict(list)
- _create_lookup_table(t)
- # Import g_functions to have __doc__ populated
- import g_functions
- print(g_functions.__doc__)

Notes:
- The exact content of the generated documentation depends on the data produced by _create_lookup_table.
- The output uses LaTeX via the latex printer and emits math blocks suitable for Sphinx with the math directive.

## Details
- Structure of the generated documentation:
  - Uses two main sections:
    - Elementary functions: (about == ()) 
    - Functions involving ...: (other groupings)
  - For each entry, the documentation presents an equation of the form:
    .. math::
      <LaTeX representation of Eq(formula, g)><optional condition>
- Function naming in headers:
  - For non-elementary groups, the header is built by listing involved functions extracted with:
    ', '.join(f'`{latex(list(category[0][0].atoms(func))[0])}`' for func in about)
  - This provides a human-readable indication of the functions involved in that group.
- Generated vs. explicit:
  - If gs is not a list, the generated expression is represented as Symbol('\\text{generated}').
  - If gs is a list, the expression is constructed as Add(*[fac*f for (fac, f) in gs]).
