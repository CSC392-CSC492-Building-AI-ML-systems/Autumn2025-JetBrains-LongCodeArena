# Naive Bayes methods

## Overview
The sklearn.naive_bayes module implements Naive Bayes classifiers. These are supervised learning methods based on Bayes’ theorem with strong (naive) feature independence assumptions. The implementation provides several NB variants optimized for different data types and use cases, including online updates via partial_fit.

Key variants include GaussianNB, BernoulliNB, MultinomialNB, ComplementNB, and CategoricalNB. All share a common interface via the _BaseNB abstract base class.

## Key Components

- _BaseNB
  - Abstract base class for Naive Bayes estimators.
  - Core responsibilities:
    - Define the interface for computing the joint log-likelihood: _joint_log_likelihood(X).
    - Provide a sequence of public methods that operate on the joint log-likelihood:
      - predict_joint_log_proba(X): returns log P(x, y) for each class.
      - predict(X): returns predicted class labels.
      - predict_log_proba(X): returns log-probabilities for each class (log P(y|x)).
      - predict_proba(X): returns probabilities P(y|x).
  - Internal mechanics:
    - X is checked via _check_X(X) before passing to _joint_log_likelihood.
    - Uses check_is_fitted to ensure the model has been trained.
    - predict_log_proba uses logsumexp to normalize and obtain posterior log-probabilities.

- GaussianNB
  - Gaussian Naive Bayes implementation for continuous features.
  - Online updates supported via partial_fit.
  - Key parameters:
    - priors: array-like of shape (n_classes,), optional prior probabilities for the classes. If provided, priors are not adjusted from the data.
    - var_smoothing: float, default 1e-9. Adds a portion of the largest variance of all features to variances for numerical stability.
  - Attributes (learned after fit):
    - class_count_: number of training samples observed in each class.
    - class_prior_: prior probability of each class.
    - classes_: array of class labels.
    - epsilon_: absolute additive value to variances (derived from var_smoothing).
    - n_features_in_: number of features seen during fit.
    - feature_names_in_: names of features seen during fit (if provided as strings).
    - var_: per-class feature variances (n_classes, n_features).
    - theta_: per-class feature means (n_classes, n_features).
  - See Also: BernoulliNB, MultinomialNB, ComplementNB, CategoricalNB for other Naive Bayes variants.

- Other NB variants (listed in __all__):
  - BernoulliNB: Naive Bayes for binary/boolean features.
  - MultinomialNB: Naive Bayes for discrete counts (e.g., word counts in text classification).
  - ComplementNB: Variant of MultinomialNB designed to improve performance on imbalanced data.
  - CategoricalNB: Naive Bayes for categorical features.

Notes:
- The core mathematics involve computing the joint log probability log P(y) + log P(x|y) for each class. The class with the highest joint log probability is selected as the prediction.
- The posterior log-probabilities are obtained by normalizing the joint log-likelihood across classes using the evidence P(x), computed via logsumexp.

## Usage

- Basic usage (GaussianNB as a representative example):

  - Fitting a model:
    - from sklearn.naive_bayes import GaussianNB
    - gnb = GaussianNB()
    - gnb.fit(X_train, y_train)

  - Making predictions:
    - y_pred = gnb.predict(X_test)
    - y_proba = gnb.predict_proba(X_test)

  - Getting log-probabilities:
    - y_log_proba = gnb.predict_log_proba(X_test)

  - Online updates:

- Other variants can be used similarly by importing the corresponding class from sklearn.naive_bayes, e.g. BernoulliNB, MultinomialNB, ComplementNB, or CategoricalNB, and calling fit/predict as above.

## Details

- The Naive Bayes assumption: features are conditionally independent given the class label. This allows P(x|y) to factorize as a product of feature-wise likelihoods P(x_i|y).
- Model behavior:
  - Priors (class probabilities) can be provided explicitly via priors. If priors are not provided, they are estimated from the training data.
  - For GaussianNB, feature likelihoods P(x_i|y) are modeled with Gaussian distributions characterized by per-class means (theta_) and variances (var_). A small smoothing term (epsilon_) is added to variances for numerical stability (controlled by var_smoothing).
- Output interpretations:
  - predict_proba(X) yields P(y|X) for each class, i.e., posterior probabilities.
  - predict_log_proba(X) yields log P(y|X).
  - predict_joint_log_proba(X) yields log P(x, y) = log P(y) + log P(x|y) for each class.
- Implementation notes:
  - The _BaseNB class provides a consistent interface for all Naive Bayes estimators, including methods that operate on the joint log-likelihood and handle input validation and fitting checks.
  - The module uses logsumexp for stable log-probability normalization.
  - The design supports online learning via partial_fit for GaussianNB and, in general, the NB family where applicable.

This documentation summarizes the Naive Bayes interface and the GaussianNB specifics, aligning with the implementation provided.
