# Contributing to Qiskit Metal

## Overview
This guide outlines the standard workflow, conventions, and practices for contributing to the Qiskit Metal project. It covers code structure, licensing, documentation, testing, and the steps to submit code changes (PRs) that integrate smoothly with the project.

Qiskit Metal is organized into modules (for example, analyses and CPW calculations). Example components include:
- CPW-related analyses (e.g., guided_wavelength, lumped_cpw)
- Submodules such as cpw_calculations and lumped_capacitive
Contributions should follow the project’s licensing, documentation, and testing standards to ensure consistency and quality.

## Key Components

- Analyses
  - Submodules:
    - cpw_calculations: Functions for calculating coplanar waveguide parameters.
    - lumped_capacitive: Tools for lumped-capacitor calculations in CPW contexts.
  - Example function usage:
    - guided_wavelength(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45)

      Example:
      ```python
      from qiskit_metal.analyses.cpw_calculations import guided_wavelength

      lambdaG, etfSqrt, q = guided_wavelength(
          freq=5e9,
          line_width=10e-6,
          line_gap=6e-6,
          substrate_thickness=760e-6,
          film_thickness=200e-9
      )
      ```

- Documentation
  - Docstrings and Sphinx autosummary usage help generate API docs.
  - Modules and functions should include clear docstrings with parameter types, return values, and examples.

## Usage (How to Contribute)

1. Get the code
   - Fork the repository and clone your fork.
   - Create a feature branch for your work.

2. Make your changes
   - Follow existing coding conventions (PEP 8, Black formatting).
   - Include or update docstrings and, if applicable, Sphinx autosummary entries.
   - Add or update unit tests to cover your changes.

3. Run tests locally
   - Install dependencies and run the test suite.
   - Example commands (adjust for your environment):
     - pytest -k "cpw or lumped_capacitive"  # run CPW-related tests
     - pytest -q

4. Build documentation (optional but encouraged)
   - Ensure your changes are reflected in the API docs.
   - Build docs with Sphinx to verify formatting.

5. Submit a pull request
   - Provide a clear PR description, including motivation, impact, and a brief testing summary.
   - Link related issue tickets if applicable.

6. Review and iteration
   - Respond to feedback from reviewers.
   - Update tests, docs, or code as requested.

## Details

- Coding and Documentation Standards
  - Each new module should begin with a module docstring and provide a clear description of purpose.
  - Include type hints where applicable and maintain consistent naming conventions.
  - Use the project’s autosummary blocks for public submodules (when applicable) to enable Sphinx to generate API docs.

- Licensing and Attribution
  - All new files must retain the project’s Apache-2.0 license header and copyright notice, as shown in the existing source code headers.

- Testing and Quality Assurance
  - Add unit tests for new functionality or behavior changes.
  - Ensure tests are deterministic and do not rely on external state.
  - Run the test suite locally before opening a PR.

- Example Contribution: CPW Calculations
  - The CPW analysis module exposes calculations such as guided_wavelength and lumped_cpw. When contributing:
    - Use the existing function signatures and return formats.
    - Provide usage examples in docstrings or the module’s user guide.
  - Example usage:
    ```python
    from qiskit_metal.analyses.cpw_calculations import guided_wavelength

    lambdaG, etfSqrt, q = guided_wavelength(
        freq=5e9,
        line_width=10e-6,
        line_gap=6e-6,
        substrate_thickness=760e-6,
        film_thickness=200e-9
    )
    ```
  - If extending functionality, add tests demonstrating the expected behavior, and update any affected docs.

## Quick Start Checklist

- [ ] Add or update unit tests for new features or fixes.
- [ ] Include a module-level docstring and function docstrings with examples.
- [ ] Ensure license header is present in new files.
- [ ] Run test suite and fix any issues.
- [ ] Update documentation and example usage as needed.
- [ ] Create a clear, concise PR description.

This guide provides the essential practices for contributing to Qiskit Metal, ensuring contributions are consistent, well-documented, and maintainable.
