Utilities for searching ASDF trees
==================================

Overview
--------
This module provides a light-weight API for querying and manipulating the results
of searches over ASDF trees. The central class, AsdfSearchResult, represents a
set of matched nodes along with any accumulated filtering logic. It supports
refining searches, controlling the rendering of results, and replacing leaf
values within the matched sub-tree.

API Reference
-------------

AsdfSearchResult
- Purpose: Represents the result of a call to AsdfFile.search, including the
  matched identifiers, the corresponding node, and any filters accumulated
  along the search path.

Constructor
- __init__(identifiers, node, filters=[], parent_node=None, max_rows=DEFAULT_MAX_ROWS, max_cols=DEFAULT_MAX_COLS, show_values=DEFAULT_SHOW_VALUES)
  - identifiers: sequence describing the path to the matched node(s) from the root.
  - node: the root node being searched.
  - filters: list of filter callables applied to prune results.
  - parent_node: the immediate parent structure of the current node for updates.
  - max_rows: maximum number of display lines when rendering the tree.
  - max_cols: maximum line length when rendering the tree.
  - show_values: whether primitive values are displayed for matched nodes.

Public methods
- format(max_rows=NotSet, max_cols=NotSet, show_values=NotSet)
  - Description: Change formatting parameters of the rendered tree.
  - Parameters:
    - max_rows: int, tuple, None, or NotSet
      Maximum number of lines to print. If None, display all lines. If a tuple,
      constrain lines per node at the depth corresponding to the tuple index.
      If NotSet, retain the existing value.
    - max_cols: int, None or NotSet
      Maximum length of each line. If None, line length is unconstrained.
    - show_values: bool or NotSet
      Set to False to disable display of primitive values. NotSet retains the existing value.
  - Returns: AsdfSearchResult with updated rendering options.

- search(key=NotSet, type=NotSet, value=NotSet, filter=None)
  - Description: Further narrow the search by key, type, or value, or by a custom filter.
  - Parameters:
    - key: NotSet, str, or any other object
      Constrains the node key or list index. If NotSet, unconstrained.
      If str, interpreted as a regular expression pattern over keys/indexes.
      Otherwise, requires an exact match to the queried key.
    - type: NotSet, str, or builtins.type or typing.Pattern
      Constrains the node type. If NotSet, unconstrained.
      If str, interpreted as a regular expression over the fully-qualified type name.
      If builtins.type, requires the node to be an instance of the given type.
    - value: NotSet, str, or any other object
      Constrains the node value. If NotSet, unconstrained.
      If str, interpreted as a regular expression over the node’s string value.
      Otherwise, requires equality to the queried value.
    - filter: callable
      Optional callable that receives (node, identifier) and returns True to retain
      the node or False to drop it.
  - Behavior: Patterns and strings are compiled as appropriate; NotSet remains
    an unconstrained value; a provided filter further reduces results.
  - Returns: AsdfSearchResult representing the narrowed search with the
    accumulated filters.

- replace(value)
  - Description: Assign a new value in place of all leaf nodes in the search results.
  - Behavior: Traverses the tree breadth-first to collect leaf matches that satisfy
    all active filters, then assigns the new value to each corresponding parent
    container element.
  - Returns: None

- node (property)
  - Description: Retrieve the leaf node for a search result that yields a single match.
  - Returns: The leaf node if exactly one result exists; otherwise behavior is not shown
    in the snippet.

Internal helpers (private)
- _maybe_compile_pattern(query)
  - If query is a string, returns a compiled regular expression. Otherwise returns query unchanged.

- _safe_equals(a, b)
  - Safely compares a and b, returning False if the comparison raises an exception.

- _get_fully_qualified_type(value)
  - Determines the fully-qualified type name for a value. If the type originates
    from the builtins module, only the type name is returned; otherwise the module
    and type name are joined with a dot.

Notes
- The API uses NotSet as a sentinel value to distinguish “no constraint” from an
  explicit constraint.
- The search supports both exact matching and pattern-based matching for keys,
  types, and values.
- The AsdfSearchResult object can be chained to progressively refine results and
  adjust rendering without mutating the underlying tree.

Usage tips
- Start with an AsdfFile.search(...) call to obtain an AsdfSearchResult, then
  chain additional .search(...) calls to narrow results, use .format(...) to
  control rendering, and call .replace(new_value) to modify matched leaf nodes
  in place. If exactly one match is found, access result.node for direct access to
  the leaf node.
