Security Roles
==============

Working with user security roles
--------------------------------

Overview
--------
Ftrack uses a security model to control access to data and operations. Security roles define a set of permissions that can be granted to users or groups at various scopes (for example, at the project or global level). This document explains the concepts and common tasks involved in working with user security roles.

Key concepts
------------
- User: An individual with access to the ftrack server.
- Role: A named collection of permissions that can be assigned to users or groups.
- Permission: An action that can be performed (e.g., read, write, administer).
- Scope: The level at which a role applies (global, project, or other hierarchical scopes).

Managing roles
--------------
- Viewing roles: Roles assigned to a user can be viewed in the UI or via server-side APIs.
- Assigning roles: Attach one or more roles to a user to grant the corresponding permissions.
- Revoking roles: Remove roles from a user to revoke the associated permissions.
- Auditing: Keep a record of role assignments and changes for accountability.

Using the Python client (authentication context)
-----------------------------------------------
The Python client authenticates using a user and an API key. Role changes are typically performed through server-side interfaces (UI or REST API) rather than direct client calls, but establishing an authenticated session is a prerequisite for any operation that requires permissions.

Example: creating an authenticated session
.. code-block:: python

    import ftrack_api
    session = ftrack_api.Session(
        server_url='https://your.ftrack.server',
        api_key='your_api_key',
        api_user='your_username'
    )

Best practices
--------------
- Apply the principle of least privilege: grant only the roles necessary for a user's responsibilities.
- Use groups to simplify role management when multiple users share the same permissions.
- Regularly review role assignments to ensure they remain appropriate.
- Keep an audit trail of role changes for compliance and troubleshooting.

Troubleshooting
---------------
- If you cannot view or modify roles, verify that your user account has the required permissions at the target scope.
- Ensure you are connected to the correct server and using valid API credentials.
