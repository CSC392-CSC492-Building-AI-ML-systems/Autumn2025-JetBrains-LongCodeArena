Cross decomposition
===================

Overview
--------
The cross_decomposition module provides three estimators that perform two-block partial least squares (PLS) and related cross-decomposition analyses to model the relationship between two data matrices, X (predictors) and Y (targets). The estimators extract latent variables that capture the maximal shared structure between the two blocks and can be used for prediction (PLSRegression) or exploratory analysis of associations (PLSCanonical, PLSSVD).

Estimators
----------
- PLSRegression
  - Purpose: Regression of Y from X using latent variables.
  - Key ideas: Builds components that maximize the covariance between X scores and Y scores, enabling prediction of Y from X.
  - Public attributes after fitting (examples): 
    - x_weights_ (loadings of X on components)
    - y_weights_ (loadings of Y on components)
    - x_scores_ (scores for X)
    - y_scores_ (scores for Y)
    - x_loadings_ (loadings of X)
    - y_loadings_ (loadings of Y)
    - coef_ (regression coefficients to predict Y from X)
  - Public methods: fit(X, Y), transform(X), predict(X)

- PLSCanonical
  - Purpose: Canonical-like cross-decomposition between X and Y, emphasizing associations between the two blocks without a predefined regression target.
  - Key ideas: Finds latent variables that maximize the correlation (or covariance) between X and Y components.
  - Public attributes (common with other estimators): x_weights_, y_weights_, x_scores_, y_scores_, x_loadings_, y_loadings_
  - Public methods: fit(X, Y), transform(X), transform_y(Y) (if provided)

- PLSSVD
  - Purpose: SVD-based cross-decomposition, providing a fast decomposition of the cross-covariance between X and Y.
  - Key ideas: Relies on singular value decomposition to obtain latent structure that links the two blocks.
  - Public attributes: similar to other estimators (weights, scores, loadings)

Common concepts
--------------
- Latent variables: Each estimator extracts a sequence of latent components (one per component) that capture shared structure between X and Y.
- Centering and scaling: By default, X and Y can be centered, and optionally scaled to unit variance before decomposition.
- Output representations:
  - x_weights_ and y_weights_: weights that define the latent directions in X and Y.
  - x_scores_ and y_scores_: representations of samples in the latent space.
  - x_loadings_ and y_loadings_: loadings that map latent components back to the original feature spaces.
- Prediction: Only PLSRegression provides direct prediction capabilities via coef_ to map X to Y.

Algorithmic notes
---------------
Internally, these estimators employ iterative approaches to compute latent components:
- Power-method based schemes to obtain first singular vectors (for certain modes).
- Optional SVD-based computation to obtain the same quantities via a complete singular value decomposition of X^T Y.
The exact implementation details are encapsulated, but the public API exposes the common interface and attributes described above.

Usage example
-------------
- Fitting a PLS regression model:
  - from sklearn.cross_decomposition import PLSRegression
  - pls = PLSRegression(n_components=2)
  - pls.fit(X, Y)
  - Y_pred = pls.predict(X_new)

Notes
-----
- The choice between PLSRegression, PLSCanonical, and PLSSVD depends on whether the goal is prediction, exploratory analysis of associations, or a fast SVD-based decomposition.
- After fitting, use the available attributes to inspect the latent structure and, for PLSRegression, to make predictions.
