Channel Options
===============

Overview
--------
In Vega-Lite (as exposed through Altair), visual properties of a chart are controlled by encoding channels. An encoding channel maps a data value (or a fixed value) to a visual property such as position, color, size, or tooltip. Each channel can be configured with a variety of options to control how data is rendered and how the chart should behave.

What is a channel?
- A channel represents a visual property (e.g., x, y, color, size, shape, opacity, tooltip, text, detail).
- A channel can be defined with:
  - FieldDef (mapping a data field to the channel)
  - ValueDef (assigning a fixed value to the channel)
  - A conditional definition (depend on data or predicates)

Common channel options
- field / value
  - field: the data field used for the channel (for FieldDef)
  - type: data type hint (nominal, ordinal, quantitative, temporal)
  - value: a fixed literal value (for ValueDef)
- bin, timeUnit, aggregate
  - bin: enable/disable binning for quantitative fields
  - timeUnit: time granularity for temporal fields
  - aggregate: aggregation function (sum, mean, count, etc.)
- scale
  - scale: configuration for the underlying scale (domain, range, nice, clamp, etc.)
- axis
  - axis: axis-specific configuration (title, labelAngle, grid, domain, ticks, etc.)
- legend
  - legend: legend configuration (title, font, orientation, etc.)
- sort
  - sort: ordering of values (by field, by value, or a specified order)
- title
  - title: override the axis or legend title
- condition (for conditional encodings)
  - allows different channel values based on a predicate or test

Common channels and their typical uses
- x, y
  - Map data fields to positions. Often used with Bin/TimeUnit, Aggregation, and scales.
- color / fill / stroke
  - Map a qualitative or quantitative field to color; supports color scales and legends.
- size
  - Control mark size via a qualitative/quantitative field or a fixed value.
- shape
  - Map a field to symbol shape (circle, square, etc.) with an associated legend.
- opacity
  - Control transparency, often used for layered plots or to de-emphasize outliers.
- tooltip
  - Attach informative fields to appear on hover. Can be a single field or a list.
- text
  - For text marks, maps a field to the displayed text content.
- detail
  - Define a secondary key for faceting or grouping within a single view.
- order
  - Control stacking or drawing order when multiple marks share a channel.

Examples
- Basic x and y encodings with field mappings
  - x = alt.X('Horsepower:Q', axis=alt.Axis(title='Horsepower'))
  - y = alt.Y('Miles_per_Gallon:Q', axis=alt.Axis(title=' mpg'))

- Color encoding with legend
  - color = alt.Color('Origin:N', scale=alt.Scale(scheme='category20'), legend=alt.Legend(title='Origin'))

- Tooltip with multiple fields
  - tooltip = ['Name:N', 'Origin:N', 'Horsepower:Q']

Notes
- Altair/Vega-Lite automatically infers the appropriate encoding channel type from the provided FieldDef/ValueDef and the data type.
- Channel options can be nested within an encoding declaration. For example, x, y, color, and tooltip can be combined to produce rich, interactive charts.
