Generate documentation for this module
======================================

Overview
--------
This module provides support utilities and data models used by Piccolo for
discovering and organizing apps, migrations, and table definitions. It
defines lightweight module-like wrappers, a helper for discovering Table
subclasses, and core configuration objects used to register apps and their
tables.

API reference
-------------

MigrationModule
  A lightweight ModuleType subclass used to describe a migration module.

  Attributes
    ID: str
      Identifier for the migration module.

    VERSION: str
      Version string for the migration module.

    DESCRIPTION: str
      Description of the migration module.

  Methods
    forwards() -> None (async)
      Placeholder coroutine that should apply forward migrations.

PiccoloAppModule
  A lightweight ModuleType subclass used to describe a Piccolo app module.

  Attributes
    APP_CONFIG: AppConfig
      The AppConfig instance for the app.

table_finder(modules, include_tags=None, exclude_tags=None, exclude_imported=False) -> List[type[Table]]
-----------------------------------------------------------------------------------------------
Locate and return all Table subclasses from the given modules.

  Parameters
    modules: Sequence[str]
      Module paths to inspect for Table subclasses (e.g., ['blog.tables']).
      The path should be relative to the project root.
    include_tags: Sequence[str] | None
      If a Table subclass has any of these tags, it will be imported.
      The special tag '__all__' imports all Table subclasses found.
    exclude_tags: Sequence[str] | None
      If a Table subclass has any of these tags, it will be excluded.
      Exclusion overrides include_tags.
    exclude_imported: bool
      If True, only Table subclasses defined in the target module are used;
      imported ones from other modules are ignored.

  Returns
    List[type[Table]]
      A list of Table subclasses discovered in the specified modules.

Command
  Dataclass used to represent a callable command that can be registered with an app.

  Fields
    callable: Callable
      The function or coroutine to execute.
    aliases: List[str]
      Optional aliases for the command.

AppConfig
  Dataclass describing configuration for a Piccolo app.

  Attributes
    app_name: str
      Name of the app (e.g., 'article').

    migrations_folder_path: str
      Path to the folder containing this app's migrations.

    table_classes: List[type[Table]]
      Table subclasses registered for this app. Defaults to an empty list.

    migration_dependencies: List[str]
      A list of module paths for apps that this app depends on for migrations.

    commands: List[Union[Callable, Command]]
      Functions and/or Command objects to be registered with the Piccolo CLI.

  Special handling
    __post_init__()
      - Converts non-Command callables in commands into Command objects.
      - Ensures migrations_folder_path is stored as a string.
      - Initializes _migration_dependency_app_configs to None for caching.

  Methods
    register_table(table_class: type[Table]) -> type[Table]
      Add a table class to the app's registry and return it.

    migration_dependency_app_configs property -> List[AppConfig]
      Lazily loads and caches AppConfig objects for the app's migration dependencies
      by importing each dependency module (expected to expose APP_CONFIG).

    get_table_with_name(table_class_name: str) -> type[Table]
      Retrieve a registered Table subclass by its class name. Raises ValueError
      if no matching table exists.

AppRegistry
  Class that records all Piccolo apps in the project. This information is
  typically kept in piccolo_conf.py.

  Attributes
    apps: List[str]
      Paths to Piccolo apps (e.g., ['blog.piccolo_app']).

    app_configs: Dict[str, AppConfig]
      Mapping from app name to AppConfig. (Description reflects the intent;
      the exact implementation may store additional metadata as needed.)

  Notes
    This class acts as a registry and accessor for AppConfig instances based
    on the provided app paths, enabling coordinated migrations, table
    registration, and command integration across the project.

Usage examples
--------------
- Discover all Table subclasses from specified modules, honoring include/exclude tags:

  tables = table_finder(
      modules=['blog.tables', 'shop.tables'],
      include_tags=['__all__'],
      exclude_tags=['deprecated'],
  )

- Create an AppConfig and register tables and commands:

  app_config = AppConfig(
      app_name='article',
      migrations_folder_path='article/migrations',
      table_classes=[],
      migration_dependencies=['other_app.piccolo_conf'],
      commands=[some_function, Command(other_callable, aliases=['start'])]
  )
  app_config.register_table(ArticleTable)

- Access dependency AppConfig objects for migrations:

  deps = app_config.migration_dependency_app_configs
