Image HDU Data
==============

The ImageHDU stores image pixel data in its Data attribute. The data is a NumPy array whose shape is determined by the FITS header keywords NAXIS and NAXISn. The header (header) provides metadata such as BitPix (the pixel data type), units (BUNIT), and other image-specific information.

Data attribute
- What it is: The pixel data array for the image extension.
- Type: A NumPy ndarray. The dtype is derived from the FITS BITPIX keyword.
- Shape: Given by NAXIS and NAXISn in the header (e.g., a typical 2-D image has shape (ny, nx)).

Accessing data
- Example
  .. code-block:: python

     from astropy.io import fits
     with fits.open('image.fits') as hdul:
         hdu = hdul[0]
         data = hdu.data
         header = hdu.header
         print(data.shape)
         print(data.dtype)
         print(header.get('BUNIT', None))

Writing and updating data
- If the file is opened in update mode, the Data array is writable and changes can be saved back to disk.
- Example
  .. code-block:: python

     from astropy.io import fits
     with fits.open('image.fits', mode='update') as hdul:
         hdu = hdul[0]
         hdu.data[...] = 0  # modify pixels
     # changes are written upon closing the file

Notes
- The pixel value semantics and valid ranges depend on BITPIX and any associated header keywords.
- The header may include units (BUNIT) and other metadata describing the data values.
- For non-standard data representations or large images, the Data array may be memory-mapped depending on how the FITS file was opened.
