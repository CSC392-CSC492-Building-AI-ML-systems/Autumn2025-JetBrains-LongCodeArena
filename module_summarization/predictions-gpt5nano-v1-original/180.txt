Cross-validation: evaluating estimator performance
==================================================

Overview
--------
Cross-validation provides an robust way to estimate how an estimator will generalize to an independent dataset. By repeatedly splitting the available data into training and validation parts and averaging the results, cross-validation mitigates the risk of overfitting to a particular train/test split and provides a more reliable measure of expected performance.

Key ideas:
- Split the data into training and validation (test) sets in a systematic way.
- Train the estimator on the training portion and evaluate it on the held-out portion.
- Aggregate scores across folds to obtain a performance estimate (and often a measure of its variability).

Scoring and evaluation
----------------------
- The metric used to evaluate performance is determined by the scoring parameter (or by the estimator’s built-in score method if scoring is not specified).
- cross_val_score returns the test scores for each fold; the mean (and standard deviation) provides the overall estimate.
- cross_validate can return multiple scores (e.g., train_score, test_score) and timing information for each fold.
- For regression, possible scorers include R^2, MSE, RMSE, etc. For classification, common scorers include accuracy, precision, recall, F1, ROC AUC, etc.
- Negative scores indicate that a scorer is designed as a loss (e.g., neg_mean_squared_error); take the negative if you want the actual loss.

Cross-validation strategies
---------------------------
- K-fold (default): Split data into k equal folds, train on k-1 folds and evaluate on the remaining fold, repeated k times.
- StratifiedKFold: Ensures class distribution is preserved in each fold (recommended for classification with imbalanced datasets).
- LeaveOneOut (LOO): Each sample acts as a validation set once.
- GroupKFold: Maintain group integrity by ensuring that all samples from the same group are either in the training or the validation set.
- TimeSeriesSplit: For time-ordered data, respects the order and prevents leakage from future data into training.

Common workflows
----------------
- Estimate generalization performance with cross_val_score:
  - Example:
    - from sklearn.model_selection import cross_val_score
    - scores = cross_val_score(model, X, y, cv=5)
    - mean_score = scores.mean()
    - std_score = scores.std()
- Obtain richer information with cross_validate (including train scores and timing):
  - Example:
    - results = cross_validate(model, X, y, cv=5, return_train_score=True, return_estimator=True)
- Explore how performance changes with a parameter using validation_curve:
  - Example:
    - from sklearn.model_selection import validation_curve
    - param_range = np.linspace(0.1, 1.0, 5)
    - train_scores, test_scores = validation_curve(model, X, y, param_name="alpha", param_range=param_range, cv=5)
- Inspect how performance varies with training set size using learning_curve:
  - Example:
    - from sklearn.model_selection import learning_curve
    - train_sizes, train_scores, test_scores = learning_curve(model, X, y, cv=5)

Visualizations (optional)
-------------------------
- Learning curves show how training and validation scores evolve with the size of the training data. Use the LearningCurveDisplay helper to visualize:
  - from sklearn.model_selection import learning_curve
  - from sklearn.model_selection import LearningCurveDisplay
  - display = LearningCurveDisplay.from_estimator(model, X, y, cv=5)
- Validation curves illustrate the effect of a hyperparameter on performance:
  - from sklearn.model_selection import validation_curve
  - train_scores, test_scores = validation_curve(model, X, y, param_name="gamma", param_range=gamma_range, cv=5)
  - Visualize with a suitable plotting tool or the ValidationCurveDisplay helper.

Practical considerations
------------------------
- Use StratifiedKFold for classification to preserve label distribution across folds.
- When tuning hyperparameters, consider nested cross-validation to avoid optimistic bias from hyperparameter selection.
- Ensure proper data preprocessing (scaling, encoding) is included inside the cross-validation loop to avoid data leakage.
- For imbalanced datasets, consider appropriate scoring metrics (e.g., macro-averaged F1, ROC AUC) and stratified splits.
