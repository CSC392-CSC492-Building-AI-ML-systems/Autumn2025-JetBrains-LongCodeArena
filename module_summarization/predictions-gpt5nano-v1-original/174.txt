Array API support
=================

Overview
--------

This project relies on the Array API-compatible behavior provided by NumPy and SciPy. Array-like operations
are performed through NumPy arrays (with SciPy support where relevant), ensuring compatibility with the Array
API standard wherever NumPy/SciPy implementations expose it. The minimum dependency versions defined here
guarantee a consistent Array API surface for building, testing, and documentation.

Notes
-----

- The dependency set is defined in a mapping that assigns each package a minimum version and a set of tags describing its usage (build, install, docs, tests, maintenance, etc.).
- A script (invokable as a module) can print the minimum version for a given package, useful for CI or reproducible environments:
  - python -m <module> <package>  (returns the minimum version for that package)

CI and maintenance
------------------

- The “build” and “install” tags are used to structure metadata for CI, and are not always included in runtime extras.
- The dependency mapping supports generating an inverse tag-to-packages view for CI checks and lockfile generation.

Multi-output estimators and Array API
------------------------------------

The codebase includes multi-output estimators that operate on array-like data using NumPy/SciPy interfaces. Operations conform to the array-oriented expectations of the Array API via the underlying numerical libraries, ensuring consistent behavior across supported backends when arrays are passed through estimators.
