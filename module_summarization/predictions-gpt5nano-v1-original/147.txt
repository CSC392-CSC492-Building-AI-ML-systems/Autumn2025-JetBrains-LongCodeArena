# Fleur input/output parsers

This document describes the Fleur parsers implemented in fleur_parser. The parsers extract useful parts of Fleur XML files to aid versioning, validation, and downstream processing.

## get_fleur_modes(xmltree, schema_dict, logger=None)

Determine the calculation modes of Fleur for the given Fleur XML file. These modes describe settings that may influence produced files or outputs (e.g., out.xml).

- Parameters
  - xmltree: XML-like object representing the Fleur XML document
  - schema_dict: either an InputSchemaDict or an OutputSchemaDict describing the Fleur schema version
  - logger: optional Logger for warnings and errors

- Returns
  - dict[str, Any] containing the extracted calculation modes. Keys include:
    - jspin: number of spins (from the jspins attribute)
    - noco: non-collinear flag (l_noco)
    - soc: spin-orbit coupling flag (l_soc)
    - relax: structural relaxation flag (l_f)
    - spex: Spex/GW related mode (0 or a non-zero value)
    - force_theorem: presence of a Force Theorem calculation
    - cf_coeff: presence of cFCoeffs coefficients (coefficients for cFCoeffs)
    - plot: plotting flag (iplot) with schema-version dependent interpretation
    - film: film system flag (filmPos)
    - ldau: LDA+U flag (ldaU within species)
    - dos: density of states flag (dos)
    - band: band structure calculation flag (band)
    - bz_integration: Brillouin-zone integration mode (mode under bzIntegration)
    - greensf: Greens function calculation flag (greensfCalculation or related torque/torgue tags)
    - ldahia: LDA-HIA flag (ldaHIA within species)

- Notes
  - The function uses FleurXMLContext for robust XML traversal and version-aware interpretation.
  - Version-specific logic handles options such as expertModes, cFCoeffs, plotting, Greens functions, and ldaHIA.

## get_nkpts(xmltree, schema_dict, logger=None)

Get the number of k-points used in the calculation described by the Fleur XML file. This function is version-dispatched to handle differences between input/output schema versions.

- Parameters
  - xmltree: XML-like object representing the Fleur XML document
  - schema_dict: either an InputSchemaDict or an OutputSchemaDict describing the Fleur schema version
  - logger: optional Logger for warnings and errors

- Returns
  - int: the number of k-points

- Behavior
  - Determines the active k-point list by name (listName)
  - Collects all available names from kPointList entries under kPointLists
  - Validates that the selected listName exists; raises ValueError if not
  - Locates the specific kPointList with the matching name and reads its count attribute (nkpts)
  - Returns nkpts as an integer
  - For older Fleur versions (before Max5) a warning is mentioned in the docstring about potential mismatches between kPointCount and actual k-points

- Notes
  - Uses FleurXMLContext to access attributes and locate the correct kPointList with version-aware handling
  - The function is decorated with schema_dict_version_dispatch(output_schema=False) to select the appropriate implementation for the given schema version

- Example
  - nkpts = get_nkpts(xmltree, schema_dict)

This documentation describes the public Fleur parsers available in the module. They enable consistent extraction of calculation modes and k-point counts across Fleur XML files and versions.
