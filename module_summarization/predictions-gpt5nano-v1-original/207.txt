Decision Trees
================

Overview
--------

Decision trees are a family of predictive models that recursively partition the feature space into regions by answering a sequence of axis-aligned questions. They can be used for both classification and regression, handle nonlinear relationships, and provide intuitive, human-interpretable rules.

Core concepts
-------------

Impurity and splits
- Splits are chosen to reduce impurity: the algorithm evaluates how good a split is at partitioning the data into homogeneous groups.
- The impurity criterion defines how node quality is measured and how split quality is scored.

Criterion interface (high-level)
- The impurity calculation is implemented via a Criterion interface, which encapsulates:
  - Initialization on a node's data
  - Handling of missing values
  - Computation of node impurity
  - Computation of left/right child impurities
  - Evaluation of node predictions
  - Efficiency helpers for split search (e.g., proxy impurity improvement)

Key methods (as implemented in the code)
- init(y, sample_weight, weighted_n_samples, sample_indices, start, end)
  - Initialize the criterion for the samples at the current node.
- init_missing(n_missing)
  - Prepare handling for missing values when present.
- reset() and reverse_reset()
  - Reset impurity calculations at the start or end of the current candidate split position.
- update(new_pos)
  - Move samples from the right child to the left as the potential split position advances.
- node_impurity()
  - Compute the impurity of the current node (the parent node).
- children_impurity(impurity_left, impurity_right)
  - Compute the impurities of the left and right child nodes for a candidate split.
- proxy_impurity_improvement()
  - Fast, surrogate score used to speed up the search for the best split.
- impurity_improvement(impurity_parent, impurity_left, impurity_right)
  - Exact improvement in impurity due to a split, based on the parent and child impurities.
- node_value(dest)
  - Compute and store the predicted value for the node.
- clip_node_value(dest, lower_bound, upper_bound)
  - Optional clipping of the node value to specified bounds.
- middle_value()
  - Value used for monotonicity constraints; specialized in classification and regression implementations.
  
Data representation and weights
- Sample indices and sample weights are used to represent which samples are considered at a node and their importance.
- weighted_n_samples tracks the total weight of samples in the current node.
- There is support for missing values, placed contiguously at the end of the sample index mask to simplify handling.

Performance and implementation notes
- EPSILON (Poisson criterion stability) is defined to aid numerical stability in impurity calculations.
- The implementation is in Cython, using memoryviews and nogil regions for performance.
- The Criterion class serves as a base for concrete impurity measures (e.g., Gini, entropy, mean squared error, etc.).

Monotonicity and extension points
- middle_value is a hook for monotonicity constraints, with concrete subclasses providing specific behavior for classification and regression criteria.

Architecture and usage
- The Criterion abstraction enables flexible incorporation of different impurity measures while sharing common data handling (weights, indices, missing values) and split-search infrastructure.
- A decision tree learner uses these criteria to select the best feature and threshold at each node, recursively constructing the tree until stopping criteria are met.
