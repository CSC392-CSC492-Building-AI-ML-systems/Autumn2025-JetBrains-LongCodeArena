How to generate schema
======================

Overview
--------
This module provides helpers to generate the SQL schema for a database client and, optionally, apply it to the database.

API
---
- get_schema_sql(client, safe) -> str

  Generates the SQL schema for the given client.

  - client: The DB client implementing a schema_generator.
  - safe: When true, the generated CREATE statements use CREATE TABLE IF NOT EXISTS semantics.

  Returns: A SQL string representing the create schema statements. If empty, no schema should be applied.

- generate_schema_for_client(client, safe) -> None

  Async helper that generates and applies the SQL schema directly to the given client.

  - client: The DB client implementing a schema_generator.
  - safe: When true, the CREATE statements use CREATE TABLE IF NOT EXISTS semantics.

  Behavior: obtains the generator via client.schema_generator(client), creates the schema with get_schema_sql, logs the schema, and if non-empty, applies it with generator.generate_from_string(schema).

Usage
-----
Generate SQL only:
.. code-block:: python

   sql = get_schema_sql(client, safe=True)
   print(sql)

Generate and apply:
.. code-block:: python

   await generate_schema_for_client(client, safe=True)

Notes
-----
- The generator object is retrieved via client.schema_generator(client) and is expected to provide:
  - get_create_schema_sql(safe) -> str
  - generate_from_string(schema: str) -> None (async)

- The operation logs the generated schema at DEBUG level:
  logger.debug("Creating schema: %s", schema)
