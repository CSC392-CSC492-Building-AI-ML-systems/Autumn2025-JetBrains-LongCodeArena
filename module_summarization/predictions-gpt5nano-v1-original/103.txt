Code generation utilities (diofant.utilities.codegen)
====================================================

Overview
--------
This module provides a framework for generating compilable code that evaluates diofant expressions
in C, C++, Fortran77, Fortran90 and Octave/Matlab. It is a work in progress; only milestones marked
with a '+' in the list below have been completed.

How this differs from diofant.printing.ccode
-------------------------------------------
- Printing routines generate a single string, while this code generator targets full routines, headers, and multi-language outputs.
- It can be extended to support .pyf for f2py and to handle programming-technical concerns (inputs/outputs, arrays, library headers, etc.).
- It enables sharing common intermediate results (e.g., via common subexpression elimination) when evaluating multiple diofant functions in one routine.
- Expressions are partially evaluated in the code generator (constants are folded where possible), a different approach from simple printing.

Basic concepts
--------------
- Routine: A generic description of a routine to be translated into a target language. It captures all features supported by the backends.
- CodeGen: An abstract class; descendants translate multiple Routine instances into compilable code for a specific language.
- Language backends: CCodeGen, FCodeGen, OctaveCodeGen implement translations for C, Fortran, and Octave/Matlab respectively.
- Friendly API: Lightweight helpers that wrap Routine/CodeGen functionality for ease of use, but with less flexibility.

Public API
----------
- Routine: Generic description of a routine to be translated.
- DataType, default_datatypes, get_default_datatype: Datatype handling for inputs/outputs.
- Argument, InputArgument, OutputArgument, Result: Build the interface and return/result structure.
- CodeGen, CCodeGen, FCodeGen, OctaveCodeGen: Backends that translate Routine(s) into target language code.
- codegen, make_routine: Friendly, higher-level APIs wrapping the core functionality.

Getting started
---------------
- Define a Routine with a name, a set of arguments (InputArgument, OutputArgument, etc.), results, local_vars, and global_vars.
- Choose a CodeGen subclass corresponding to the target language (CCodeGen, FCodeGen, or OctaveCodeGen).
- Generate the code from the Routine object (via the appropriate CodeGen workflow or the friendly API).

Notes
-----
- The module relies on existing diofant core elements (Expr, Symbol, Integer, etc.) and the existing printers (ccode, fcode, octave).
- The codebase documents its goals and current limitations within the module docstring.
