Sync LDAP users
================

Overview
--------
Keeps ftrack user accounts in sync with an LDAP directory. The sync process provisions new users from LDAP, updates existing user attributes, assigns group memberships, and can deactivate users who no longer exist in LDAP.

Prerequisites
-------------
- An LDAP directory accessible from the ftrack server.
- Administrative access to configure LDAP synchronization.
- Sufficient permissions to read user and group information from LDAP.
- An ftrack server capable of hosting LDAP-based user provisioning.

Configuration
-------------
Configure LDAP connection details, attribute mappings, and provisioning behavior. Typical settings include:
- LDAP server URL and credentials for binding
- Base DN and filters for locating user entries
- Attribute mappings for username, email, first name, and last name
- Group search base and membership attribute mappings
- Provisioning policy (create/update users) and default roles
- Sync interval (frequency of the LDAP sync)

Example configuration (YAML)
.. code-block:: yaml
   ldap:
     url: "ldap://ldap.example.com"
     bind_dn: "cn=admin,dc=example,dc=com"
     bind_password: "secret"
     user_search_base: "ou=users,dc=example,dc=com"
     user_filter: "(objectClass=person)"
     user_id_attr: "uid"
     user_email_attr: "mail"
     first_name_attr: "givenName"
     last_name_attr: "sn"
     group_search_base: "ou=groups,dc=example,dc=com"
     group_membership_attr: "member"
   provisioning:
     enabled: true
     default_role: "Viewer"
   sync:
     interval_seconds: 3600
     on_missing_in_ldap: "deactivate"

Operation
---------
- LDAP entries are mapped to ftrack users using configured attribute mappings.
- New LDAP users are provisioned in ftrack according to the provisioning policy.
- Existing ftrack users are updated with LDAP attribute values and group memberships.
- Users no longer present in LDAP can be deactivated or left untouched based on configuration.

Running and scheduling
----------------------
- Run the sync manually with the configured tool or script.
- Schedule the sync (e.g., via cron or a systemd timer) to occur at the desired interval.

Logging and monitoring
----------------------
- Ensure logs capture provisioning actions, attribute updates, and deactivation events.
- Monitor for errors such as LDAP connectivity issues or attribute mapping mismatches.

Troubleshooting
---------------
- LDAP server unreachable or credentials invalid: verify URL, bind DN, and password.
- Attribute mapping issues: confirm LDAP schema and attribute names match configuration.
- Permission errors: ensure the LDAP bind user has read access to necessary entries.
- Deactivation behavior: review policy for users missing in LDAP and adjust as needed.

Security considerations
-----------------------
- Protect LDAP bind credentials; use secure storage and access controls.
- Limit exposure of directory data and follow your organization’s privacy policies.
