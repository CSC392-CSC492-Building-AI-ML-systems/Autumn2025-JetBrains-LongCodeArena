Using Timers
============

Overview
--------

Timers let you measure how long operations take and schedule actions to run after a delay or periodically. They are commonly used for profiling, timeout handling, and regular background tasks.

Measuring elapsed time
----------------------

For high-resolution timing, use time.perf_counter(). For monotonic timing (not affected by system clock changes), use time.monotonic().

Example:

.. code-block:: python

    import time

    start = time.perf_counter()
    # perform some work
    end = time.perf_counter()
    duration = end - start
    print("Elapsed time: {:.6f} seconds".format(duration))

Scheduling one-off actions
--------------------------

Use threading.Timer (a daemon thread) for a one-time delayed execution.

Example:

.. code-block:: python

    from threading import Timer

    def say_hello():
        print("Hello after delay")

    t = Timer(5.0, say_hello)  # run after 5 seconds
    t.start()

Repeating actions
-----------------

To repeat an action at regular intervals, you can reschedule a timer at the end of the callback, or use a small helper:

.. code-block:: python

    from threading import Timer

    def periodic(interval, func, *args, **kwargs):
        def wrapper():
            func(*args, **kwargs)
            Timer(interval, wrapper).start()
        Timer(interval, wrapper).start()

    def tick():
        print("Tick")

    periodic(2.0, tick)  # tick every 2 seconds

Note: Repeating timers can accumulate drift over time; for precise scheduling consider an event loop or sched module.

Cleanup on exit
---------------

Register cleanup functions to run when the process exits using atexit.

.. code-block:: python

    import atexit

    def cleanup():
        print("Cleaning up before exit")

    atexit.register(cleanup)

Best practices
--------------

- Prefer time.perf_counter() for elapsed time measurements.
- Use time.monotonic() if you need a clock that never goes backward.
- Keep timer callbacks short; long-running work should be delegated to other threads or processes.
- For long-running or complex scheduling, consider an event loop or task scheduler.

Common tasks
------------

- Delayed execution of a function after a given delay.
- Periodic background tasks with controlled intervals.
- Simple profiling of code sections.

Notes
-----

- threading.Timer creates a daemon thread; if the main program exits, non-daemon timers may be terminated. Ensure proper shutdown if needed.
