Internal organization of NumPy arrays
=====================================

Overview
--------
NumPy arrays (ndarray) are the core multi-dimensional, homogeneous data containers used for numerical computing. Each array exposes a data buffer containing elements of a single dtype, plus metadata describing shape, strides, dtype, and ownership. The Python-facing type hints for arrays are defined in NumPy’s typing layer (NDArray, _ArrayLike, _ShapeLike, DTypeLike, etc.), while the low-level implementation relies on a C-backed ndarray object and a rich set of internal helpers (e.g., flagsobj) to describe memory layout and capabilities.

The ndarray object
------------------
- Data buffer: A contiguous or strided block of raw bytes holding the elements. The buffer is the underlying memory store for all elements in the array.
- Shape: A tuple describing the length of the array along each axis.
- Strides: A tuple describing how many bytes to skip to advance by one element along each axis. Strides determine how the logical index maps to the physical memory.
- Dtype: Describes the element type (including size, endianness, and field structure). The dtype defines itemsize and how an element is interpreted.
- Offset and data ownership: The array may view a sub-region of a larger buffer; ownership semantics are tracked to manage memory lifetime and views.
- Flags: A set of boolean attributes describing capabilities and layout (e.g., whether the data is writable, and whether it is C- or Fortran-contiguous). These are exposed via the internal flags machinery (flagsobj).

Memory layout and contiguity
----------------------------
- Contiguity: By default, arrays have a canonical memory layout, but slicing and advanced indexing can yield views with non-trivial strides. Contiguity can be C-style (row-major) or Fortran-style (column-major).
- Strides and indexing: Strides determine how memory is traversed for multi-dimensional indexing. Changing the strides changes how the same data buffer is interpreted as higher-dimensional arrays.
- Views and ownership: Slices and transpositions create views that reference the same memory buffer without copying data. Ownership and reference semantics ensure memory is freed when no longer needed.

Dtypes and the dtype engine
---------------------------
- Primitive and structured dtypes: The dtype object describes how to interpret each element, including primitive numeric types and structured (record) dtypes with multiple named fields.
- Internal type codes: NumPy maintains internal mappings for primitive kinds via code tables such as _BoolCodes, _UByteCodes, _UShortCodes, _UIntCCodes, _UIntCodes, _ULongLongCodes, _ByteCodes, _ShortCodes, _IntCCodes, _IntCodes, _LongLongCodes, _SingleCodes, _DoubleCodes, _LongDoubleCodes. These support efficient representation, dispatch, and typing of element types.
- Dtype-related typing: Static typing uses DTypeLike and related aliases to describe array element types in type hints, enabling type checkers to reason about array element kinds.

Typing and internal interfaces
------------------------------
- NDArray and friends: The Python-facing type aliases NDArray, _ArrayLike, _ShapeLike, DTypeLike, _DTypeLike, and others live in the numpy._typing module. They describe the shape, dtype, and compatibility of arrays in type-checked code.
- Platform-specific ctypes and mypy plugin: NumPy’s internal notes indicate the use of a mypy plugin to import the correct platform-specific ctypes._SimpleCData[int] sub-type. This ensures correct type representation when NumPy interops with C-level data on different platforms.

Flags and memory characteristics
-------------------------------
- Flagsobj: An internal helper that encapsulates array flags such as writability, alignment, and contiguity. These flags influence how NumPy optimizes operations and how memory can be safely accessed or mutated.

Notes for developers
--------------------
- The internal structure of ndarray is designed to be efficient for numerical workloads, enabling fast element access, broadcasting, and vectorized operations.
- The integration between dtype definitions, memory layout (shapes, strides), and flags allows NumPy to support a wide range of data types (including structured dtypes) and memory layouts with minimal data copies.
- Type hints and internal typing abstractions provide a stable programming interface for static type checkers and for developers extending NumPy’s core.

This organization underpins the core capabilities of NumPy arrays: efficient storage, flexible views, rich dtype support, and robust type-oriented tooling.
