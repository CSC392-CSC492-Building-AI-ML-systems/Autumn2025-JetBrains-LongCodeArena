# General Plotting Routines

This module provides general and special Bokeh plotting routines for Masci-tools. It uses a centralized BokehPlotter to manage defaults and figure configuration, and exposes convenient plotting functions for interactive visualization.

## Plotter and defaults

- plot_params
  - A global BokehPlotter instance used to configure defaults and generate figures.

- set_bokeh_plot_defaults(**kwargs)
  - Set defaults for the Bokeh backend according to the provided keyword arguments.
  - Available defaults are defined in Masci-tools' BokehPlotter.

- reset_bokeh_plot_defaults()
  - Reset the Bokeh backend defaults to the hardcoded defaults.

- show_bokeh_plot_defaults()
  - Print the currently set defaults for the Bokeh backend.

- get_bokeh_help(key)
  - Print the description of the given key in the Bokeh backend.

- load_bokeh_defaults(filename='plot_bokeh_defaults.json')
  - Load defaults for the Bokeh backend from a JSON file.

- save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)
  - Save the current Bokeh defaults to a JSON file.
  - If save_complete is True, both the modified user defaults and the unmodified hardcoded defaults are stored.

> Note: The defaults and descriptions referenced here are managed by the BokehPlotter instance and its internal schema.

## General plotting routines

### bokeh_scatter

Create an interactive scatter plot with Bokeh.

Signature
- bokeh_scatter(x, y=None, *, xlabel='x', ylabel='y', title='', figure=None, data=None, saveas='scatter', copy_data=False, **kwargs)

Description
- Builds a single scatter plot using Bokeh, with data provided by:
  - x: array-like or a key for data for the x-axis (or a data source passed via keyword).
  - y: array-like or a key for data for the y-axis (optional if x provides a full source).
  - data: data source (e.g., a pandas DataFrame) that contains the x/y data.
- If x is a dict, DataFrame, or ColumnDataSource (or None), a deprecation warning is emitted advising to pass the data source via the keyword argument data and xdata/ydata as the first arguments.
- Data handling is performed by process_data_arguments to obtain a plot entry (with x/y data) and a data source.
- The function sets default plot type/name, prepares the figure, and constructs the scatter plot using Bokeh’s scatter method.
- Tooltips are added, and optional styling/limits are applied according to the current defaults.
- The function returns the created Bokeh figure.

Behavior details
- Default type is set to 'function' and the default name is entry.y.
- Additional keyword arguments are forwarded to the underlying Bokeh scatter call.
- If a figure is provided, the plot is added to that figure.
- The function enforces consistency via the ensure_plotter_consistency decorator.

Usage notes
- If you pass a direct data source as the first positional argument, you should instead pass it via the data keyword and provide xdata/ydata as the first arguments.

### bokeh_multi_scatter

Create an interactive scatter plot that supports multiple data sets.

Signature
- bokeh_multi_scatter(x, y=None, *, data=None, figure=None, xlabel='x', ylabel='y', title='', saveas='scatter', copy_data=False, set_default_legend=True, **kwargs)

Description
- Similar to bokeh_scatter but supports multiple data sets. Data can be provided in a single data source, and the function will generate a legend (by default using the data names) when set_default_legend is True.
- The function handles data preprocessing, figure preparation, plotting for multiple data series, and legend configuration according to the current defaults.
- Returns the created Bokeh figure.

Usage notes
- The function passes additional keyword arguments through to the underlying Bokeh plotting calls and uses the central plotting configuration defined by the BokehPlotter.

## Data handling and helpers

- process_data_arguments (imported from masci_tools.vis.data)
  - Central utility used to interpret the provided data, x, and y inputs, and to return a suitable data source for the Bokeh plotting call.

- get_special_kpoint_ticks (imported)
  - Helper available in the module for handling special x-axis ticks in related plots (not shown in the general plotting functions above but available in the plotting utilities).

---

## Decorators and behavior

- ensure_plotter_consistency(plot_params)
  - A decorator applied to plotting routines to ensure that the current plotter configuration is consistent across calls.

- The plotting functions use:
  - plot_params.prepare_figure(...) to create or reuse a Bokeh figure with the configured title and axis labels.
  - plot_params.plot_kwargs(...) to assemble the keyword arguments for the Bokeh plotting call.
  - plot_params.add_tooltips(...) to attach informative tooltips to the plotted elements.
  - plot_params.draw_straight_lines(...) and plot_params.set_limits(...) to adjust the visual appearance and axis limits.
  - plot_params.save_plot(...) to save or display the resulting plot according to the chosen save location/name.
