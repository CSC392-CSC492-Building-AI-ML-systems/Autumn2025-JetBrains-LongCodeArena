Generate Tutorial: Pipelines
=============================

This tutorial explains how to compose simple and complex data processing and modeling workflows using pipelines. A pipeline chains a sequence of transformers and a final estimator, enabling convenient parameter management, cross-validation, and reuse of intermediate results.

Overview
--------
- A pipeline assembles several steps (transformers and a final estimator) that are applied sequentially.
- All intermediate steps that transform data must implement fit and transform; the final estimator must implement fit.
- Pipelines support parameter setting using a double-underscore syntax (step_name__parameter) to tune or replace individual components.
- Caching of transformed data is supported via the memory parameter to speed up repeated fits.

Key concepts
------------
- Steps: A list of (name, transform/estimator) pairs. The last step is the final estimator.
- Memory: Optional caching for fitted transformers. The final estimator is not cached.
- Output configuration: The set_output method can adjust the output format (e.g., default, pandas) for all steps.
- Parameter access: Access and set parameters of nested steps using the step name and parameter name (e.g., pipeline.set_params(clf__C=1.0)).

Constructing a Pipeline
-----------------------
- Steps must be a list of (name, transform) tuples, where transforms implement fit and transform, and the last step is an estimator that implements fit.

Example: simple preprocessing + classifier
Code example
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', LogisticRegression())
])

# Fit and evaluate as with any estimator
pipe.fit(X_train, y_train)
score = pipe.score(X_test, y_test)

Parameter tuning and nesting
----------------------------
- Use __ to address parameters inside nested steps. For example:
pipe.set_params(clf__C=0.5, scaler__with_mean=False)

Cross-validation and grid search
-------------------------------
Pipelines are especially powerful when used with tools like GridSearchCV, enabling joint optimization of preprocessing and estimator hyperparameters.

Example: grid search with a pipeline
from sklearn.model_selection import GridSearchCV

param_grid = {
    'scaler__with_mean': [True, False],
    'clf__C': [0.1, 1, 10]
}
gs = GridSearchCV(pipe, param_grid, cv=5)
gs.fit(X, y)

Memory and performance
----------------------
- The memory parameter enables caching of transformed data. This can speed up repeated fits, especially when fitting pipelines on large datasets or expensive transformations.
- The last step is never cached, even if it is a transformer.

Output customization
--------------------
- set_output allows configuring how transform results are returned (default, pandas DataFrame, or leaving the configuration unchanged).

Example: set output to pandas
pipe.set_output(transform='pandas')

Inspecting pipelines
--------------------
- named_steps: Access steps by name via the named_steps attribute (a dict-like object).
- Steps can be inspected to understand the composition and to adjust parameters or replace components.

Code snippet: inspecting components
for name, step in pipe.named_steps.items():
    print(name, type(step))

Notes
-----
- The pipeline mechanism enables cross-validation to be performed without leaking test data into training.
- When replacing a step, you can substitute a new estimator or transformer, or set a step to 'passthrough' to skip that step.

See Also
--------
- make_pipeline: Convenience function to construct a pipeline with unnamed steps.
- FeatureUnion: Combine several transformers in parallel and concatenate their outputs.

Examples
--------

This tutorial provides a concise guide to building, tuning, and using pipelines to streamline machine learning workflows, ensuring consistent preprocessing and model evaluation.
