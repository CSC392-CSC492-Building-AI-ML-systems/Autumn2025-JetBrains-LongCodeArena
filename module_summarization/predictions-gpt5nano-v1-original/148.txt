# General HDF5 file reader (HDF5Parser)

Overview
- This module provides a framework of predefined transformations for the HDF5Reader.
- Transformations are designed to be robust against three data forms:
  - a) an h5py.Dataset that may need to be converted to a NumPy array
  - b) a NumPy array
  - c) a dict mapping keys to datasets/arrays
- All transformations are registered and exposed through the HDF5Reader class, enabling consistent data processing of HDF5 content.

Transformation mechanism
- Registration
  - Transformations are registered using the hdf5_transformation decorator.
  - When a function is decorated, it gets added to HDF5Reader._transforms under its function name.
  - If attribute_needed is True, the function name is also added to HDF5Reader._attribute_transforms, indicating it should operate on a previously processed attribute value.
  - Each registered transformation is wrapped to catch exceptions and re-raise as HDF5TransformationError with a descriptive message.
- Error handling
  - If a transformation fails, an HDF5TransformationError is raised:
    - Message format: "The HDF5 transformation {function_name} failed with {exception}"
- Data forms
  - All transformations gracefully handle:
    - h5py.Dataset → transformed using standard array-like operations
    - numpy.ndarray → transformed directly
    - dict → each value is transformed independently, producing a dict of results

Available transformations
- get_first_element(dataset)
  - Purpose: Get the first element of the dataset.
  - Behavior: Delegates to index_dataset(dataset, 0).
  - Works for: h5py.Dataset, numpy.ndarray, or dict of datasets/arrays.

- index_dataset(dataset, index)
  - Purpose: Get the n-th element of the dataset.
  - Behavior:
    - If dataset is a dict: {key: data[index] for key, data in dataset.items()}
    - Else: dataset[index]
  - Works for: h5py.Dataset, numpy.ndarray, or dict.

- slice_dataset(dataset, slice_arg)
  - Purpose: Slice the dataset with the given slice argument.
  - Behavior:
    - If dataset is a dict: {key: data[slice_arg] for key, data in dataset.items()}
    - Else: dataset[slice_arg]
  - Works for: h5py.Dataset, numpy.ndarray, or dict.

- get_shape(dataset)
  - Purpose: Get the shape of the dataset.
  - Behavior:
    - If dataset is a dict: {key: data.shape for key, data in dataset.items()}
    - Else: dataset.shape
  - Works for: h5py.Dataset, numpy.ndarray, or dict.

- get_name(dataset, full_path=False)
  - Purpose: Get the name of the dataset.
  - Behavior:
    - If dataset is a list or numpy array → raises ValueError (requires a dataset)
    - If dataset is a dict:
      - full_path=True: {key: data.name for key, data in dataset.items()}
      - full_path=False: {key: data.name.split('/')[-1] for key, data in dataset.items()}
    - Else (single dataset):
      - full_path=True: dataset.name
      - full_path=False: dataset.name.split('/')[-1]
  - Works for: h5py.Dataset, numpy.ndarray (only via error for non-dataset types), or dict.

- tile_array(dataset, n_repeats)
  - Purpose: Repeat the array n times using numpy.tile.
  - Behavior:
    - If dataset is a dict: {key: np.tile(dataset, n_repeats) for key, data in dataset.items()}
    - Else: np.tile(dataset, n_repeats)
  - Note: In the dict case, the function applies tile to the entire dataset mapping (as coded), producing a dict of tiled values.

- repeat_array(dataset, n_repeats)
  - Purpose: Repeat each element in the array n times using numpy.repeat.
  - Behavior:
    - If dataset is a dict: {key: np.repeat(dataset, n_repeats) for key, data in dataset.items()}
    - Else: np.repeat(dataset, n_repeats)

- get_all_child_datasets(group, ignore=None, contains=None)
  - Purpose: Retrieve all datasets contained in the given HDF5 group.
  - Parameters:
    - group: an h5py group-like object
    - ignore: a string or iterable of strings; keys containing any of these phrases are ignored
    - contains: a string or iterable of strings; only keys containing all phrases are kept
  - Behavior:
    - Builds a dict of {dataset_name: dataset} for all h5py.Dataset items in the group, applying the ignore/contains filters
  - Returns: dict of contained datasets (by their keys)

Notes and tips
- Data shapes and names
  - get_shape and get_name help extract metadata about datasets, whether you have a single dataset or a dict of datasets.
- Dict handling
  - All transformations are designed to operate uniformly on a single dataset or on every value in a dict, making it convenient to work with multiple datasets in parallel.
- Error conditions
  - If a transformation fails, you will receive a clear HDF5TransformationError indicating which transformation failed and why.

Usage context
- These transformations are registered on the HDF5Reader class and can be invoked by name to process datasets read from HDF5 files.
- They provide a consistent, reusable toolkit for common operations when parsing and transforming HDF5 data in material science workflows.
