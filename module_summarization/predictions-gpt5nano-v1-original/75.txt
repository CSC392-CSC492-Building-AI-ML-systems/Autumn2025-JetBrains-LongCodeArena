Working with text data
======================

This tutorial demonstrates how to work with text data using scikit-learn’s text feature extraction utilities. It covers preprocessing, tokenization, n-grams, stop words, and vectorization methods (CountVectorizer, TfidfVectorizer, HashingVectorizer). It also shows a complete end-to-end example of building a simple text classifier.

Overview
--------

Text data must be converted into numeric feature vectors before it can be used by machine learning models. The typical workflow is:

- Normalize and clean the text (lowercasing, removing accents, stripping HTML tags).
- Tokenize the text and optionally form n-grams.
- Optionally remove stop words.
- Convert tokens to numerical features (counts or TF-IDF).

Core concepts
-------------

- Preprocessing and analysis
  - _preprocess(doc, accent_function=None, lower=False) applies lowercasing and an optional accent-stripping function.
  - _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None) chains together the steps to obtain tokens or n-grams from a document. If an analyzer is provided, it replaces the preprocessor, tokenizer, and n-grams steps.
- Accent stripping
  - strip_accents_unicode(s) removes accented characters in a Unicode string (slower but comprehensive).
  - strip_accents_ascii(s) transliterates accented characters to ASCII where possible.
- Tag stripping
  - strip_tags(s) removes basic HTML/XML tags using a simple regular expression.
- Stop words
  - _check_stop_list(stop) resolves stop word lists; the built-in English stop words are available as ENGLISH_STOP_WORDS.

Vectorization options
---------------------

- CountVectorizer: converts a collection of text documents to a matrix of token counts.
- TfidfVectorizer: performs a CountVectorizer step followed by TF-IDF weighting.
- TfidfTransformer: transforms a term-frequency matrix to TF-IDF representation (used when you already have a count matrix).
- HashingVectorizer: uses the hashing trick to produce a fixed-size feature vector without an explicit vocabulary mapping (suitable for streaming data and very large corpora).

End-to-end example
------------------

A common workflow is to build a scikit-learn Pipeline that combines vectorization with a classifier. The following is a minimal end-to-end example that:
- tokenizes text using a word-level analyzer,
- removes English stop words,
- considers unigrams and bigrams,
- applies TF-IDF weighting,
- trains a simple linear classifier.

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer, TfidfTransformer
    from sklearn.pipeline import Pipeline
    from sklearn.linear_model import SGDClassifier

    # Example data
    texts = [
        "This is a sample document.",
        "This document is another example.",
        "Text data requires preprocessing before modeling."
    ]
    labels = [0, 1, 0]

    # Build a pipeline: vectorize -> TF-IDF -> classifier
    model = Pipeline([
        ("vect", CountVectorizer(stop_words="english", ngram_range=(1, 2))),
        ("tfidf", TfidfTransformer()),
        ("clf", SGDClassifier(loss="hinge", max_iter=1000, random_state=42)),
    ])

    # Train
    model.fit(texts, labels)

    # Predict
    preds = model.predict(["New text data to classify."])
    print(preds)

Practical tips
--------------

- When dealing with large text corpora, consider using HashingVectorizer for streaming data or very large vocabularies.
- Choose n-gram ranges (e.g., (1, 2)) based on the dataset size and the importance of local context.
- Use stop words to reduce noise, but be mindful that removing them can affect performance for some tasks.
- If you need custom preprocessing (e.g., stripping HTML more aggressively or language-specific normalization), supply a custom preprocessor or analyzer to the vectorizer.
- Validate and compare models with cross-validation and keep an eye on classifier choice and regularization.
