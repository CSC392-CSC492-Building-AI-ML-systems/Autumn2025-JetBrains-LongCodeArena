AutoAugment augmentations
==========================

Overview
--------
This module defines a standard suite of augmentations used by AutoAugment policy for ImageNet, RandAugment and TrivialAugmentWide. The augmentations are implemented using NVIDIA DALI operators and are exposed via an @augmentation decorator that handles parameter computation.

Key ideas
- Each augmentation is defined with a mag_range, an optional randomly_negate flag, and a mag_to_param mapper.
- When invoked, the decorated augmentation expects a single positional argument (a batch of samples). Instead of passing the raw parameter, you provide magnitude_bin and num_magnitude_bins; the parameter is computed as mag_to_param(magnitudes[magnitude_bin] * random_sign), where magnitudes are linearly spaced within mag_range.
- The module includes example setups passed to @augmentation. You can adjust the magnitudes by creating e.g. my_shear_x = shear_x.augmentation(mag_range=(0, 0.5)).

Augmentations included
-----------------------
- shear_x
  - mag_range: (0, 0.3)
  - randomly_negate: True
  - mag_to_param: warp_x_param
  - Description: applies horizontal shear via a computed shear value.

- shear_y
  - mag_range: (0, 0.3)
  - randomly_negate: True
  - mag_to_param: warp_y_param
  - Description: applies vertical shear via a computed shear value.

- translate_x
  - mag_range: (0, 1)
  - randomly_negate: True
  - mag_to_param: warp_x_param
  - shape-aware version (uses rel_offset and shape to compute offset)

- translate_x_no_shape
  - mag_range: (0, 250)
  - randomly_negate: True
  - mag_to_param: warp_x_param
  - name: translate_x
  - Description: translation along x with an explicit offset (no shape input required)

- translate_y
  - mag_range: (0, 1)
  - randomly_negate: True
  - mag_to_param: warp_y_param
  - shape-aware version (uses rel_offset and shape to compute offset)

- translate_y_no_shape
  - mag_range: (0, 250)
  - randomly_negate: True
  - mag_to_param: warp_y_param
  - name: translate_y
  - Description: translation along y with an explicit offset (no shape input required)

- rotate
  - mag_range: (0, 30)
  - randomly_negate: True
  - default: rotate_keep_size = True
  - Description: rotation by an angle in degrees.

- brightness
  - mag_range: (0, 0.9)
  - randomly_negate: True
  - mag_to_param: shift_enhance_range
  - Description: brightness adjustment; magnitudes are mapped to [0, 2] with a neutral value around 1.

- contrast
  - mag_range: (0, 0.9)
  - randomly_negate: True
  - mag_to_param: shift_enhance_range
  - Description: contrast adjustment; computed to follow a channel-weighted mean center.

Helper functions
----------------
- warp_x_param(magnitude): maps a magnitude to a 2D affine parameter [x, y] for horizontal transforms.
- warp_y_param(magnitude): maps a magnitude to a 2D affine parameter [x, y] for vertical transforms.
- shift_enhance_range(magnitude): converts a [0, 1] magnitude into an enhancement factor in [0, 2], with 1 representing neutral.

Usage notes
------------
- The decorator computes the actual parameter from mag_range, magnitude_bin, and num_magnitude_bins, then passes it to the underlying DALI operator.
- You can adjust the magnitude space by creating a customized augmentation, e.g., my_shear_x = shear_x.augmentation(mag_range=(0, 0.5)).
- These augmentations are implemented via DALI operators (e.g., fn.warp_affine, fn.rotate, fn.brightness, etc.) and support common data layouts (FHWC or HWC) as needed (notably in contrast).

Example (conceptual)
- Create a customized shear:
  my_shear_x = shear_x.augmentation(mag_range=(0, 0.5))
- Apply to a batch by selecting a magnitudeBin:
  augmented_batch = my_shear_x(batch, magnitude_bin=2, num_magnitude_bins=10)

Notes
-----
- The module contains a docstring describing the parameter computation model and how to customize augmentations.
- The implementation relies on NVIDIA DALI operators for GPU-accelerated augmentation.
