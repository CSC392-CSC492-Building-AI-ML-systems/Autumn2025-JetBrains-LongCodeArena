Simulation Utilities
====================

Overview
--------
This module provides helpers for working with Bluesky plans in simulations:
- plot_raster_path: visualizes the raster path traced by a plan.
- summarize_plan (and its alias print_summary): prints a compact summary of a plan.
- check_limits: validates that a plan does not move devices outside their limits.

API
---

plot_raster_path(plan, x_motor, y_motor, ax=None, probe_size=None, lw=2)
----------------------------------------------------------------------------

Plot the raster path for a plan.

Parameters
  - plan: iterable of Msg objects (must yield Msgs and not be a coroutine)
  - x_motor, y_motor: str
      Names of the x and y motors
  - ax: matplotlib.axes.Axes, optional
      The axes to plot to; if None a new figure/axes are created
  - probe_size: float, optional
      If provided, use as radius of probe (in the same units as motor positions)
  - lw: float, optional
      Line width for the path

Returns
  - dict with keys:
      - 'path': the PathLine2D object produced by ax.plot
      - 'events': the scatter artists or a PatchCollection representing read points

Notes
  - The function uses matplotlib for plotting and builds the trajectory by tracking
    set commands for the specified x and y motors, collecting positions on save commands.

Example
  .. code-block:: python

     plot_raster_path(plan, x_motor='x', y_motor='y', probe_size=0.5)

summarize_plan(plan)
--------------------

Print a minimal, human-readable summary of a plan.

Parameters
  - plan: iterable of Msg objects (must yield Msgs)

Returns
  - None (prints the summary)

Alias
  - print_summary = summarize_plan  # back-compat

check_limits(plan)
------------------

Check that a plan will not move devices outside of their limits.

Parameters
  - plan: iterable of Msg objects (must yield Msgs)

Returns
  - None (prints warnings as needed)

Behavior
  - Iterates through the plan; for each msg with command == 'set' and the target
    object not in the ignore list:
    - If the object has a check_value method, it is called with the new value.
    - Otherwise, a warning is emitted, and the object is added to an ignore list
      to avoid repeated warnings.

Notes
  - Uses warnings.warn to report missing check_value implementations.
  - The ignore list prevents repeated complaints about the same object.

Dependencies
------------
- matplotlib
- bluesky (print_summary_wrapper)

Examples
--------
- Plot and inspect a plan, then summarize and validate limits:
  .. code-block:: python

     plot_raster_path(plan, x_motor='x', y_motor='y', probe_size=None)
     summarize_plan(plan)
     check_limits(plan)

See Also
--------
- bluesky.preprocessors.print_summary_wrapper
- matplotlib.pyplot and related plotting utilities

Notes
-----
- The module expects Msg objects with attributes command and obj, where command can be 'set' or 'save'.
