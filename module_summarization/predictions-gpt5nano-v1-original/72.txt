Pairwise distance utilities (sklearn.metrics.pairwise)
=====================================================

Overview
--------
This module provides internal utilities used for validating and preparing inputs
for pairwise distance computations. It exposes helpers for inferring the
appropriate floating dtype and for ensuring that input matrices X and Y have
the correct shape and type before distance calculations are performed.

Functions
---------

_return_float_dtype(X, Y)
~~~~~~~~~~~~~~~~~~~~~~~~
Determine the default floating dtype for inputs X and Y.

Parameters
  X : {array-like, sparse matrix}
      First input matrix.

  Y : {array-like, sparse matrix} or None
      Second input matrix or None. If None, Y is treated as X.

Returns
  X, Y, dtype
      - X and Y converted as needed to arrays (if not already sparse).
      - dtype is float32 if both X and Y are dense and have dtype float32; otherwise float.

check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)
---------------------------------------------------------------------------------------------------------------
Set X and Y appropriately and check inputs for pairwise distance computations.

Specifically, this function:
- Converts inputs to arrays (or uses the same object if Y is X or None).
- Determines the dtype to use (defaults to a dtype inferred by _return_float_dtype).
- Validates sparsity and finite values according to the provided options.
- Ensures the feature dimension compatibility between X and Y (or between X and a precomputed distance matrix when precomputed is True).

Parameters
  X : {array-like, sparse matrix} of shape (n_samples_X, n_features)
      First input.

  Y : {array-like, sparse matrix} of shape (n_samples_Y, n_features) or None
      Second input or None.

  precomputed : bool, default=False
      True if X is treated as a precomputed distance matrix.

  dtype : str, type, list of type, default=None
      Desired dtype for X and Y. If None, inferred from inputs.

  accept_sparse : {str, bool} or list of str, default='csr'
      Allowed sparse formats. If the input is sparse but not in an allowed
      format, it will be converted to the first listed format. True allows any
      sparse format. False raises an error for sparse input.

  force_all_finite : bool or 'allow-nan', default=True
      Whether to require finite values. If True, inf and NaN are rejected.
      If 'allow-nan', only NaN and missing values are allowed; infinite values are not.

  copy : bool, default=False
      If True, make a copy of the inputs when converting.

Returns
  safe_X, safe_Y
      - safe_X: array-like, shape (n_samples_X, n_features)
      - safe_Y: array-like, shape (n_samples_Y, n_features) or a reference to X if Y is None

check_paired_arrays(X, Y)
--------------------------
Set X and Y appropriately and check inputs for paired distances.

Description excerpt (from code docstring):
Set X and Y appropriately and checks inputs for paired distances. All paired
distance metrics should use this function first to assert that the given
parameters are correct and safe to use. Specifically, this function first
ensures that both X and Y are arrays, then checks that they are at least two
dimensional while ensuring that their elements are floats.
