Computational Performance
=========================

Overview
--------
This module emphasizes scalable and fast computation for multi-output estimators. It provides meta-estimators that operate on a base estimator and can parallelize work across targets or data partitions. Performance hinges on efficient cloning, parallel execution, and the ability to leverage optimized numerical backends through compiled extensions.

Parallelism and Scaling
-----------------------
- n_jobs controls the level of parallelism for fitting and prediction. It is an integer or None and propagates to sub-estimators where applicable.
- Parallel and delayed utilities are used to distribute work across multiple workers, enabling concurrent fitting or evaluation of independent tasks.
- threadpoolctl is used to probe and limit threading of underlying libraries (e.g., OpenMP, MKL) to avoid oversubscription and contention on multicore systems.
- Proper parallelism can substantially reduce wall-clock time for multi-output tasks, but memory usage increases with the number of concurrent tasks due to replicated state and data.

Fitting and Incremental Learning
-------------------------------
- _fit_estimator(estimator, X, y, sample_weight=None, **fit_params) clones the base estimator to ensure isolation between workers and performs fitting with or without sample_weight.
- _partial_fit_estimator(estimator, X, y, classes=None, partial_fit_params=None, first_time=True) supports incremental learning; on the first call, the estimator is cloned, and subsequent calls reuse the same instance with updated parameters.
- The design favors parallel execution of independent fitting tasks when applicable, improving performance on multi-output problems.

Profiling and Timing
--------------------
- Timing utilities (e.g., _print_elapsed_time) are available to help measure and report elapsed time during training and evaluation, aiding performance tuning and benchmarking.

Performance Considerations
--------------------------
- Memory usage grows with parallelism due to multiple estimators coexisting in memory; monitor memory when increasing n_jobs.
- The use of compiled extensions (via cython and optimized BLAS/LAPACK libraries) is essential for achieving good performance.
- Consistent environment behavior is encouraged by pinning key dependencies (numpy, scipy, joblib, threadpoolctl) to known versions.

Best Practices
--------------
- Enable parallelism when multiple outputs or independent data partitions can be processed concurrently.
- Use threadpoolctl to limit external library threading to prevent oversubscription on multi-core machines.
- Profile and benchmark with representative data; adjust n_jobs and data batching accordingly.
- Be mindful of random_state to ensure reproducible performance measurements.
