Pipelines and composite estimators
==================================

The sklearn.pipeline module provides utilities to build composite estimators, as a chain of transforms and estimators. A Pipeline assembles several steps that can be cross-validated together, allowing you to set parameters for different steps using a single parameter grid. In addition to Pipeline, the module exposes utilities for composing estimators, such as FeatureUnion, and helper constructors like make_pipeline and make_union.

What is a Pipeline
-------------------
A Pipeline is a sequence of transformers followed by a final estimator. Intermediate steps must implement fit and transform, while the final step only needs to implement fit. The pipeline can cache fitted transformers using the memory argument, which can speed up repeated fits on the same data.

Key ideas:
- Each step is a (name, estimator) pair. The last step must be an estimator (i.e., it must implement fit).
- Steps can be replaced or removed by using special values (e.g., 'passthrough' or None) for parameter updates.
- The pipeline exposes the parameters of all steps using a single namespace, with the step name separated from the parameter name by a double underscore (e.g., pipe__svc__C).

Parameters
----------
steps : list of tuple
    List of (name, transform) tuples (implementing fit/transform) that are chained in sequential order. The last transform must be an estimator.

memory : str or object with the joblib.Memory interface, default=None
    Used to cache the fitted transformers of the pipeline. The last step will never be cached, even if it is a transformer. If a string is given, it is the path to the caching directory. Enabling caching triggers a clone of the transformers before fitting. The pipeline’s internal transformers cannot be inspected directly; use the attribute named_steps or steps to inspect estimators within the pipeline.

verbose : bool, default=False
    If True, the time elapsed while fitting each step will be printed as it is completed.

Attributes
----------
named_steps : Bunch
    Dictionary-like object, with attributes for each step by name. This provides read-only access to any step parameter by user-given name.

classes_ : ndarray
    The classes labels. Only defined if the last step of the pipeline is a classifier.

n_features_in_ : int
    Number of features seen during fit. Defined if the first estimator in steps exposes such an attribute.

feature_names_in_ : ndarray
    Names of features seen during fit. Defined if the estimator exposes such an attribute when fit.

Notes
-----
- The final estimator is not cached, even if memory is used for previous steps.
- A step’s parameters can be set or queried using the standard get_params and set_params interfaces. For example, to set the C parameter of an SVC inside a pipeline named 'svc', you would use: pipe.set_params(svc__C=10).
- The set_output method can configure the output container for transform and fit_transform calls across all steps.

Examples
--------
Basic usage
~~~~~~~~~~~
>>> from sklearn.svm import SVC
>>> from sklearn.preprocessing import StandardScaler
>>> from sklearn.pipeline import Pipeline
>>> from sklearn.datasets import make_classification
>>> X, y = make_classification(random_state=0)
>>> pipe = Pipeline([('scaler', StandardScaler()), ('svc', SVC())])
>>> # The pipeline can be used as any other estimator
>>> score = pipe.fit(X, y).score(X, y)
>>> print(score)

Parameter tuning with GridSearchCV
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
>>> # The parameter of a step can be set via its name using '__'
>>> pipe.set_params(svc__C=10).fit(X, y).score(X, y)

Convenience constructor
~~~~~~~~~~~~~~~~~~~~~~~
For simpler pipelines, use make_pipeline, which automatically names steps and builds a Pipeline instance:
>>> from sklearn.pipeline import make_pipeline
>>> from sklearn.preprocessing import StandardScaler
>>> from sklearn.decomposition import PCA
>>> pipe = make_pipeline(StandardScaler(), PCA(n_components=2))
>>> pipe.fit(X, y)
