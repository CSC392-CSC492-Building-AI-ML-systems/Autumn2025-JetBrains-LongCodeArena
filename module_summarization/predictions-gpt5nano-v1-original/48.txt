Discret e models: Zero-Inflated Count Models
===========================================

Generate documentation for this module.

Overview
--------
This module provides discrete count models with zero-inflation. It exposes specialized Zero-Inflated Poisson, Zero-Inflated Generalized Poisson, and Zero-Inflated Negative Binomial P models, as well as a generic base class GenericZeroInflated that implements the common zero-inflated structure used by these models.

Public API
----------
- ZeroInflatedPoisson
- ZeroInflatedGeneralizedPoisson
- ZeroInflatedNegativeBinomialP
- GenericZeroInflated

GenericZeroInflated
-------------------
GenericZeroInflated is a base class for zero-inflated count models and inherits from CountModel.

Inheritance
-----------
CountModel

Docstring

Generiz Zero Inflated model for count data

Parameters
----------
endog : array_like
    The endogenous response variable.

exog : array_like
    The exogenous design matrix for the main count model.

exog_infl : array_like, optional
    Exogenous design for the zero-inflation model. If None, a column of ones is used.

offset : array_like, optional
    Offset is added to the linear prediction with coefficient equal to 1.

inflation : {'logit', 'probit'}, default 'logit'
    The model for the zero inflation, either Logit (default) or Probit.

exposure : array_like, optional
    Log(exposure) is added to the linear prediction with coefficient equal to 1.

missing : {'none', 'drop', 'raise'}, default 'none'
    How to handle missing data.

Notes on initialization
-----------------------
- If exog_infl is None, the class creates a constant inflation design with a single parameter.
- The inflation model is either a Logit or Probit GLM fitted to exog_infl.
- The main model is a count model whose parameters are separate from the inflation parameters.

Core implementation details
----------------------------
- self.model_infl is the GLM for the zero-inflation process, initialized with zeros and exog_infl.
- self._hessian_inflate points to the appropriate Hessian method for the inflation model (logit or probit).
- self.inflation stores the chosen inflation type ('logit' or 'probit').
- self.k_inflate and self.k_extra reflect the number of inflation-related parameters.

Log-likelihood
--------------
- loglike(params): Full log-likelihood of the zero-inflated model.
- loglikeobs(params): Per-observation log-likelihood, combining the inflation probability with the main model’s likelihood.

Fitting
-------
- fit(start_params=None, method='bfgs', maxiter=35, full_output=1, disp=1, callback=None, cov_type='nonrobust', cov_kwds=None, use_t=None, **kwargs)
  - If start_params is not provided, a start is computed from the current model state.
  - Includes a workaround for perfect-separation scenarios via a callback.
  - Delegates the optimization to the underlying CountModel fit with the specified method and options.

Notes
-----
- The log-likelihood couples the zero-inflation process with the main count model:
  L_i = w_i + (1 - w_i) * P_main for y_i = 0
  L_i = (1 - w_i) * P_main for y_i > 0
  where w_i is the zero-inflation probability from the inflation model, P_main is the main count model likelihood.

Parameters for the zero-inflation component
-------------------------------------------
_exog_infl_ (and related inflation parameters) are created and named as inflate_... based on the selected inflation model.

References to internal utilities
-------------------------------
- _doc_zi_params documents the parameters for the zero-inflation portion:
  exog_infl, offset, exposure, inflation
- Uses approx_fprime, approx_hess, cache_readonly, and ConvergenceWarning from Statsmodels utilities as part of optimization and numerical differentiation.

Example usage (conceptual)
--------------------------
- Construct a GenericZeroInflated model with endog, exog, and appropriate exog_infl.
- Choose inflation='logit' or 'probit'.
- Call fit() to estimate parameters for both the inflation and main count models.

Note: This documentation reflects the public surface and core behavior of the GenericZeroInflated class and its interaction with specialized Zero-Inflated models exposed by __all__ in this module.
