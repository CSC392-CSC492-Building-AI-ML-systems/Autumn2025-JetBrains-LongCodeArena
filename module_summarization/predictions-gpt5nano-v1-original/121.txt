Create Cradmin Menu System
==========================

Overview
---------
This module provides a set of renderables for building a Cradmin menu in Django Cradmin. It includes base classes and concrete renderables for links, buttons, expandable menus, separators, and a menu toggle. The default expandable menu class can be configured via Django settings.

Default expandable menu
-----------------------
- get_default_expandable_menu_renderable(**kwargs)
  - Returns the default expandable menu renderable instance defined by the Django setting DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS.
  - Behavior:
    - If DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS is not set, returns None.
    - Otherwise, imports the configured class with import_string and instantiates it with the provided kwargs.

Menu renderables
----------------
- BaseMenuLinkRenderable
  - Inherits from renderable.AbstractBemRenderable.
  - Purpose: render a single menu link item.
  - Template: django_cradmin/crmenu/menuitem/link.django.html
  - Initialization parameters:
    - label: text shown for the menu item.
    - url: destination URL when the item is clicked.
    - is_active: whether the item should be styled as active (default False).
    - bem_variant_list, parent_bem_block: optional styling hints.
  - Key behavior:
    - Resolves a parent BEM block with get_parent_bem_block(), using a fallback of adminui-page-header.
    - Uses a BEM element of navlink under the parent block.
    - Active items receive an "active" bem variant.
- NavLinkItemRenderable
  - Subclass of BaseMenuLinkRenderable.
  - Intended for adding links to the main menu.
- NavLinkButtonItemRenderable
  - Subclass of NavLinkItemRenderable.
  - Overrides get_bem_variant_list to return ['button'], rendering the link as a button-styled item.
- ExpandableMenuItem
  - Subclass of BaseMenuLinkRenderable.
  - Enables links that expand to reveal additional items.
  - Template: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
  - Behavior:
    - get_fallback_parent_bem_block() returns 'expandable-menu'.
    - get_bem_element() returns a link element under the expandable menu block.
    - get_list_element_bem_element() returns a list item element under the expandable menu block.
- ExpandableMenuSeparator
  - Inherits from renderable.AbstractBemRenderable.
  - Represents a separator within an expandable menu.
  - Template: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
  - Initialization accepts an optional parent_bem_block.
  - Behavior:
    - get_fallback_parent_bem_block() returns 'expandable-menu'.
    - get_parent_bem_block() resolves the explicit or fallback block.
    - get_bem_element() returns the separator element under the parent block.
- MenuToggleItemItemRenderable
  - Inherits from renderable.AbstractBemRenderable.
  - Represents a toggle for expanding/collapsing the menu.
  - Template: django_cradmin/crmenu/menuitem/menutoggle.django.html
  - Initialization accepts an optional parent_bem_block.
  - Localization:
    - get_label() returns a translated label via pgettext_lazy with domain 'cradmin default header menu toggle label' and default 'Menu'.
  - Behavior:
    - get_fallback_parent_bem_block() returns 'adminui-page-header'.
    - get_parent_bem_block() resolves the explicit or fallback block.
    - get_bem_element() returns the nav toggle element under the parent block.
    - get_label_css_class() returns the label CSS class for the toggle.
    - get_icon_css_class() returns the icon CSS class for the toggle.

Abstract menu rendering
-----------------------
- AbstractMenuRenderable
  - Base class for rendering a menu.
  - Provides a common structure for managing the items to render.
  - Public attributes (to be configured by subclasses):
    - link_renderable_class: The renderable class used for links.
    - separator_renderable_class: The renderable class used for separators.
    - button_renderable_class: The renderable class used for buttons.
  - Initialization:
    - __init__(self, request, cradmin_instance=None)
      - Stores the incoming request and CrAdmin instance.
      - Initializes an internal renderable_list to hold the items to render.
  - Public methods:
    - iter_renderables(): returns an iterator over the renderable_list.
    - get_link_renderable_class():
      - Returns the link_renderable_class.
      - If not defined, raises ValueError indicating no link_renderable_class is defined for this class.
    - get_separator_renderable_class():
      - Returns the separator_renderable_class (behavior described similarly to links).
    - (Additional methods for obtaining button renderables and other renderable classes follow the same pattern.)

Templates
---------
The following templates are used by the renderables:
- Link item: django_cradmin/crmenu/menuitem/link.django.html
- Expandable menu item: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
- Expandable menu separator: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
- Menu toggle: django_cradmin/crmenu/menuitem/menutoggle.django.html

Usage and customization
-----------------------
- To customize the default expandable menu, set DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS in your Django settings to the import path of your custom expandable menu class.
- To build a custom menu, subclass AbstractMenuRenderable, assign your own renderable classes to link_renderable_class, separator_renderable_class, and button_renderable_class, and manage the renderable_list to render your menu structure.
- The menu system relies on BEM-style class names for styling, with sensible defaults for parent blocks such as adminui-page-header and expandable-menu.

Translations
------------
- Menu toggle label is provided via pgettext_lazy with the domain 'cradmin default header menu toggle label', enabling translation of the default label "Menu".
