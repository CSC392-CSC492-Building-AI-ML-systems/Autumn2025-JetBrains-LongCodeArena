Table Data
==========

Table data represents tabular information as a collection of columns. Each column holds a 1-D array of values and all columns share the same number of rows. A Table can store heterogeneous data types across columns and may carry per-column units as well as descriptive metadata.

Structure
---------
- Rows: Records across all columns.
- Columns: Named 1-D arrays; each column has a length equal to the number of rows.
- Heterogeneous types: Different columns may have different dtypes (integers, floats, booleans, strings, datetimes, etc.).

Column data types
-----------------
- Numeric types (ints, floats, complex).
- Booleans.
- Strings and character data.
- Datetime and timedelta types.
- Object-dtype columns for arbitrary Python objects.
- Masked data support for representing missing values.

Units and metadata
------------------
- Per-column units can be attached (e.g., with astropy.units).
- Per-column metadata: description, units, and other annotations.
- Table-wide metadata stores information applicable to the entire table.

Accessing data
--------------
- Access a column by name: t['column_name'].
- Access a row by index: t[i], which yields a row-like view.
- Add, remove, or rename columns:
  - t['new_col'] = values
  - t.remove_columns(['old_col'])
  - t.rename_column('old', 'new')
- Indexing and filtering:
  - Select rows with a boolean mask based on column values.

Masked / missing data
---------------------
- Missing values can be represented using masked columns or masked arrays.
- Operations on a Table propagate masks to preserve data integrity.

I/O and formats
--------------
- Table data can be read from and written to multiple formats (e.g., FITS tables, CSV, and other tabular formats) with appropriate IO utilities.
- Metadata and units are preserved where supported by the format.

Example
-------
Code example illustrating creation and basic operations:

.. code-block:: python

   from astropy.table import Table
   import numpy as np

   t = Table()
   t['time'] = np.array([1.0, 2.0, 3.0])
   t['flux'] = np.array([0.1, 0.2, 0.3])
   t['flag'] = np.array([True, False, True])

   # Access
   times = t['time']
   first_row = t[0]

   # Add a column
   t['quality'] = np.array([0.99, 0.95, 0.98])

See Also
--------

- astropy.io.fits: Interfaces for FITS binary and ASCII tables.

Notes
-----
- Table data structures are designed to be memory-efficient for large datasets and to integrate with Astropy units and metadata infrastructure.
