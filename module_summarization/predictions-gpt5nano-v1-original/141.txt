Connect to remote data
======================

Overview
--------

This utility enables connecting to remote data sources via fsspec-backed filesystems
and returns lazy Dask objects that can be composed into a computation. It supports
single paths, multiple files, and glob patterns, including remote filesystems
such as S3, HDFS, and others.

API
---

read_bytes(urlpath, delimiter=None, not_zero=False, blocksize="128 MiB",
           sample="10 kiB", compression=None, include_path=False, **kwargs)

- urlpath : string or list
  Absolute or relative filepath(s). Prefix with a protocol like "s3://" to read
  from remote filesystems. Multiple files can be read by passing a globstring
  or a list of paths; all must use the same protocol.

- delimiter : bytes
  Optional delimiter to split blocks on (e.g., b'\n'). Blocks are aligned to
  delimiter boundaries.

- not_zero : bool
  If True, seek the start-of-file delimiter and discard the header.

- blocksize : int or str
  Chunk size in bytes. Defaults to "128 MiB". Set to None to disable chunking
  and read whole files as single blocks.

- compression : string or None
  Compression type (e.g., 'gzip', 'xz') or None. If compression is set (and
  blocksize is not None), chunked reads are not supported for compressed files.

- include_path : bool
  If True, include the file path corresponding to each block in the returned
  structure.

- kwargs
  Storage options that apply to the target filesystem (e.g., host, port,
  username, password, etc.).

Returns
-------

sample : bytes
  The header sample read from the first file (unless sampling is disabled).

blocks : list of lists of Delayed
  Each inner list corresponds to one file; each element is a Dask Delayed object
  that computes to a block of bytes from that file.

paths : list of strings (only if include_path is True)
  The corresponding paths for each block in the same order as blocks.

Notes
------

- urlpath may be a single path, a globstring, or a list/tuple of paths. When
  multiple files are provided, they must all use the same protocol.

- If blocksize is None, offsets/lengths are empty and each file is read in one
  piece. Otherwise, the code computes per-file offsets and lengths to produce
  blocks that align with the requested chunk size. If a file is empty, it yields
  no blocks for that file.

- Compression inference (compression="infer") is supported per file prior to
  chunking. If any file is compressed, chunked reads are not allowed and you
  must set blocksize=None.

- The function returns a sample header by reading from the first file. If
  delimiter is provided, the sample is read up to the delimiter boundary;
  otherwise, a raw byte sample is read.

Examples
--------

>>> sample, blocks = read_bytes('2015-*-*.csv', delimiter=b'\\n')  # doctest: +SKIP
>>> sample, blocks = read_bytes('s3://bucket/2015-*-*.csv', delimiter=b'\\n')  # doctest: +SKIP
>>> sample, paths, blocks = read_bytes('2015-*-*.csv', include_path=True)  # doctest: +SKIP

Notes for remote data sources
-----------------------------

- Requires fsspec-compatible filesystem implementations (e.g., S3, GCS, HDFS).
- The returned blocks are lazy; computation is avoided until explicitly invoked.
- Use read_bytes in conjunction with Dask to build scalable, parallel data ingestion
  pipelines from remote storage.
