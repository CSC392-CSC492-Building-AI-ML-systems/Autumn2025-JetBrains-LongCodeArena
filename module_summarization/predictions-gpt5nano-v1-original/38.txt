Examples
--------

Below are illustrative usages of the base Field type and how it behaves in practice.

Example 1: Simple custom field with fixed SQL type
.. code-block:: python

    from tortoise.models import Model
    from tortoise.fields import Field

    class TextField(Field):
        SQL_TYPE = "VARCHAR(255)"

    class User(Model):
        name = TextField(null=False, default="Anonymous")

Example 2: Field type inferred from Python type via mixin
.. code-block:: python

    class Text(Field, str):
        SQL_TYPE = "VARCHAR(100)"

    class User(Model):
        name = Text(null=False)

Example 3: Per-dialect overrides for a field
.. code-block:: python

    from pypika.terms import Term

    class _db_sqlite:
        SQL_TYPE = "VARCHAR(40)"
        skip_to_python_if_native = False

        def function_cast(self, term: Term) -> Term:
            # Example cast for a specific dialect
            from pypika import functions
            return functions.Cast(term, "NUMERIC")

    class User(Model):
        nickname = classmethod(lambda cls: None)  # placeholder to show usage
        # The actual field would be defined with a Field subclass here.
        # During SQLite dialect usage, Tortoise will apply the overrides
        # from this inner class.

Example 4: Value conversion helpers
.. code-block:: python

    class EmailField(Field):
        SQL_TYPE = "VARCHAR(320)"
        null = False
        unique = True

    class User(Model):
        email = EmailField(null=False)

    f = TextField(null=True)
    db_val = f.to_db_value("hello@example.com", None)   # "hello@example.com"
    py_val = f.to_python_value("hello@example.com")     # "hello@example.com"

Example 5: Using the base Field in a model
.. code-block:: python

    from tortoise.models import Model
    from tortoise.fields import Field

    class SimpleField(Field):
        SQL_TYPE = "VARCHAR(50)"

    class User(Model):
        username = SimpleField(pk=True, null=False)
        # A single field with primary key semantics is implied by pk=True

Usage
-----

Field base and its common usage

- Field(options)
  - source_field: Optional source column name in the DB if it should differ from the attribute name.
  - generated: Whether the DB should generate this field.
  - pk: Mark this field as primary key. Only one primary key per model is typically allowed.
  - null: Allow NULL in the DB for this field.
  - default: Default value for the field if not provided on creation. Can be a value or a callable.
  - unique: Enforce a unique constraint on this field.
  - index: Create an index on this field.
  - description: Human-readable description; included in model descriptions and DB comments.
  - model: Reference to the Model class to which this field belongs.
  - Other keyword arguments are accepted and stored as needed by concrete field types.

Class attributes (to override or use as-is)

- field_type: The Python type of the field. Set automatically by the metaclass when a field is defined with multiple bases (mixins).
- indexable: Whether the field can be indexed. Defaults to True.
- has_db_field: Does this field map to a DB column? False for virtual fields.
- skip_to_python_if_native: If the DB driver natively supports this Python type, skip conversion to/from Python.
- allows_generated: Whether the field can be DB-generated.
- function_cast: Optional casting term function to apply for DB emulation.
- SQL_TYPE: The SQL type as a string used by the DB for DDL.
- GENERATED_SQL: SQL used to instruct the DB to auto-generate this field (if applicable).

Per-DB overrides

- You can provide an inner class named _db__{dialect} (e.g., _db_sqlite) to override attributes and methods for a specific dialect.
- Overrides may include SQL_TYPE, skip_to_python_if_native, and a custom function_cast implementation.

Internal behaviors and conversions

- to_db_value(value, instance): Converts a Python value to the DB representation. If value is None or already of the target Python type, it is returned as-is; otherwise it is cast to the field_type.
- to_python_value(value): Converts a DB value to the Python type. If value is None or already of the target Python type, it is returned as-is; otherwise it is cast to the field_type.

Notes

- The metaclass _FieldMeta supports creating a field type by combining Field with a mixin base containing the desired Python type. In that case, field_type is automatically set to the mixin type.
- Validations in __init__:
  - If not indexable and (unique or index) is requested, a ConfigurationError is raised.
  - pk and null cannot both be True.
  - If pk is True, index and unique are set to True automatically.
