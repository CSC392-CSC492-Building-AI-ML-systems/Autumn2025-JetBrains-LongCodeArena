Fitting SESANS Data
===================

This section documents how SESANS data fitting is handled within sasmodels and how to explore the parameter space used by SESANS models.

Overview
--------
SESANS data are modeled using kernel-based scattering models. Each model defines a set of parameters that describe size, shape, composition, orientation, and other physical properties. Parameters may be shared across models or be specific to a particular model. Understanding which parameters exist and which models use them is essential for fitting SESANS data and diagnosing fitting behavior.

Parameter space and kinds
-------------------------
- Each model exposes its parameters via model_info.parameters.kernel_parameters.
- A parameter has:
  - name: the identifier used in fitting and reporting.
  - type: the kind of parameter, which can indicate how the parameter is varied in fit (for example, volume-related, orientation-related, or SLD-related). The kinds supported include volume, orientation, sld, None, or any.
- A single parameter can be used by multiple models.

Finding and listing parameters
-----------------------------
The parameter space can be explored with the sasmodels parameter-listing utilities.

- find_pars(kind=None)
  - Returns a mapping: {parameter_name: [model_name, model_name, ...]}
  - If kind is provided, only parameters of that kind are included.
- list_pars(names_only=True, kind=None)
  - Prints a list of parameters.
  - If names_only is True, prints only the parameter names.
  - If names_only is False, prints each parameter followed by the models that use it.
  - If kind is provided, filters parameters to the specified kind.
- The models and parameters are discovered from the installed model registry via list_models() and load_model_info().

Command-line usage
------------------
- python -m sasmodels.list_pars [-v]
  - -v, --verbose: list models that use each parameter in addition to the parameter name.
- python -m sasmodels.list_pars [volume|orientation|sld|none|any]
  - kind filter to show only parameters of a given kind.
  - any (or omitted) shows all parameters; none limits to no kind constraint.

Notes
-----
- The listing relies on the model information available to sasmodels at runtime.
- This facility is useful for preparing fitting strategies, understanding parameter correlations, and documenting which models rely on which parameters.

Example workflow
----------------
1) List all parameter names across models:
   python -m sasmodels.list_pars

2) List parameter names along with the models that use them:
   python -m sasmodels.list_pars -v

3) List only volume-related parameters:
   python -m sasmodels.list_pars volume

4) Inspect the complete parameter-to-model mapping for documentation or debugging:
   python -m sasmodels.list_pars -v volume

Implementation notes
--------------------
- The parameter collection iterates over all available models (list_models) and collects kernel_parameters from each model’s metadata (load_model_info).
- The list_pars function delegates formatting to columnize for a clean tabular output when verbose listing is requested.

This documentation reflects the parameter discovery and reporting utilities that support fitting SESANS data by clarifying which parameters exist, which models use them, and how to filter the parameter space by kind.
