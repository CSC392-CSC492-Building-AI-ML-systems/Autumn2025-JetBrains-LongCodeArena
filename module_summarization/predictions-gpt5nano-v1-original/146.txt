Quick Start
===========

 toppra provides a compact, path-centric interface for generating gridpoints along a geometric path and preparing for trajectory optimization. The core pieces are:

 - AbstractGeometricPath: a minimal interface for geometric paths
 - propose_gridpoints(path, ...): adaptively sample a path so interpolation error stays within bounds
 - RaveTrajectoryWrapper: optional adapter for OpenRAVE trajectories (if OpenRAVE is available)

Usage overview
--------------

- Implement a geometric path by subclassing AbstractGeometricPath and implementing:
  - __call__(path_positions, order=0): evaluate position (order=0), velocity (order=1), or acceleration (order=2)
  - dof: number of joints / degrees of freedom
  - path_interval: start and end positions along the path
  - waypoints: optional tuple/array of waypoints

- Generate gridpoints with propose_gridpoints to ensure the interpolation error stays below max_err_threshold.

- If you have an OpenRAVE trajectory, wrap it with RaveTrajectoryWrapper to obtain a geometric path and then generate gridpoints (only if OpenRAVE is available).

Minimal example
---------------

.. code-block:: python

    from toppra.geometric_path import AbstractGeometricPath, propose_gridpoints

    import numpy as np

    class ConstantPath(AbstractGeometricPath):
        def __init__(self, dof=2):
            self._dof = dof
            self._interval = np.array([0.0, 1.0])

        @property
        def dof(self):
            return self._dof

        @property
        def path_interval(self):
            return self._interval

        @property
        def waypoints(self):
            return None

        def __call__(self, path_positions, order=0):
            p = np.asarray(path_positions)
            if order == 0:
                return np.tile(p.reshape(-1, 1), (1, self.dof))
            elif order == 1:
                return np.zeros((len(p), self.dof))
            else:
                return np.zeros((len(p), self.dof))

    path = ConstantPath(dof=2)
    gridpoints = propose_gridpoints(path)

OpenRAVE integration (optional)
--------------------------------

If OpenRAVE is installed and FOUND_OPENRAVE is true, you can wrap an OpenRAVE trajectory:

.. code-block:: python

    from toppra.geometric_path import RaveTrajectoryWrapper, FOUND_OPENRAVE

    if FOUND_OPENRAVE:
        # traj is an OpenRAVE RaveTrajectory, robot is an OpenRAVE Robot
        path = RaveTrajectoryWrapper(traj, robot)
        gridpoints = propose_gridpoints(path)

Notes
-----

- propose_gridpoints refines the grid by evaluating the path’s second derivative and segment lengths to keep interpolation error under the specified threshold.
- The default thresholds are suitable for many robotics applications, but you can tune max_err_threshold, max_iteration, and max_seg_length as needed.

API quick reference
-------------------

- propose_gridpoints(path, max_err_threshold=1e-4, max_iteration=100, max_seg_length=0.05)
- AbstractGeometricPath
  - __call__(path_positions, order=0) -> np.ndarray
  - dof -> int
  - path_interval -> np.ndarray(2,)
  - waypoints -> Optional[Tuple[np.ndarray, ...]]
- RaveTrajectoryWrapper(traj, robot) (OpenRAVE integration)
