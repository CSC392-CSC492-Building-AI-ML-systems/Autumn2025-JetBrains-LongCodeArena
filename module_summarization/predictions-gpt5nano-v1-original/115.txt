ASDF Configuration
==================

Module overview
---------------
This module provides the AsdfConfig class, which serves as the central container for
global ASDF configuration options. Users typically access configuration via the
module-level APIs (e.g., get_config and config_context) which expose a shared
AsdfConfig instance.

Default values
--------------
The following defaults are defined for configuration before any user customization:

- DEFAULT_VALIDATE_ON_READ = True
- DEFAULT_DEFAULT_VERSION = str(versioning.default_version)
- DEFAULT_LEGACY_FILL_SCHEMA_DEFAULTS = True
- DEFAULT_IO_BLOCK_SIZE = -1  (auto)
- DEFAULT_ARRAY_INLINE_THRESHOLD = None

AsdfConfig
----------
Container for ASDF configuration options. Instances are managed internally; users
should obtain the configuration via the module-level API (get_config, config_context).

Properties
----------
resource_mappings
    Get the list of registered resource mapping instances. Defaults to mappings
    registered via entry points. The list is lazily loaded and cached, with
    thread-safe initialization.

resource_manager
    The asdf.resource.ResourceManager instance. Includes resources from the
    registered resource mappings and any runtime-added mappings. Created lazily
    and cached with thread-safety.

extensions
    The list of registered extensions. Defaults to extensions registered via
    entry points. Loaded lazily and cached.

default_version
    The default ASDF Standard version used for new files. Getter returns a string;
    setter validates the version using validate_version.

io_block_size
    The block size used when reading and writing files. A value of -1 selects the
    filesystem's preferred block size.

array_inline_threshold
    (Internal/advanced) Threshold controlling inlining behavior for arrays.
    Present in the configuration initialization; see implementation for current
    usage details.

Methods
-------
add_resource_mapping(mapping)
    Register a new resource mapping. The new mapping takes precedence over existing
    mappings. The mapping is wrapped with ResourceMappingProxy.maybe_wrap.

remove_resource_mapping(mapping=None, *, package=None)
    Remove a registered resource mapping. Requires at least one of mapping or package.
    If mapping is provided, it is wrapped as a ResourceMappingProxy for comparison.

reset_resources()
    Reset resource mappings to the default list provided by entry points.

add_extension(extension)
    Register a new extension. The new extension takes precedence over existing ones.
    The extension is wrapped with ExtensionProxy.maybe_wrap.

remove_extension(extension=None, *, package=None)
    Remove a registered extension. Can remove by exact extension (or URI pattern of
    extension.extension_uri) or by package. Requires at least one of extension or package.

reset_extensions()
    Reset extensions to the default list registered via entry points.

Property behavior and thread-safety
-----------------------------------
- All mutating operations acquire a re-entrant lock (thread-safe updates).
- Resource mappings and extensions are loaded lazily and cached to optimize startup.
- Changes to mappings or extensions invalidate the associated cached resource_manager
  or extension lists as needed.

Usage examples (high level)
----------------------------
- Retrieve the global configuration:
  cfg = get_config()

- Customize defaults:
  cfg.default_version = "1.8.0"
  cfg.io_block_size = 4096

- Add a resource mapping at runtime:
  cfg.add_resource_mapping(my_mapping)

- Remove an extension by package:
  cfg.remove_extension(package="my_package")

Note
----
The module also exposes module-level helpers (get_config, config_context) to work with
the shared AsdfConfig instance, enabling convenient context-managed or global-style
configuration in applications.
