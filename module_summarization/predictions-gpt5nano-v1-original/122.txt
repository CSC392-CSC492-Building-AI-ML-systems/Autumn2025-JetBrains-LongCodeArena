Unified renderable interface
============================

This documentation describes the unified renderable interface provided by the code. A renderable is an object that knows how to render itself into a template using Django’s render_to_string, with a consistent API that supports context data, optional CSS classes, and optional BEM-style naming.

Core interface: AbstractRenderable
----------------------------------

- Purpose
  - Defines the minimal contract for rendering an object into a template.

- Template selection
  - template_name attribute
  - get_template_names(self)
    - Returns the template name(s) to render.
    - Defaults to the value of template_name.
    - Raises NotImplementedError if no template is defined.

- Context data
  - get_context_data(self, request=None)
    - Returns the context dictionary for rendering.
    - Default: { 'me': self }

- Rendering
  - render(self, request=None, extra_context_data=None)
    - Renders the template(s) returned by get_template_names() using the context from get_context_data().
    - Merges extra_context_data (if provided) with the default context.
    - Forwards request to render_to_string as well as to get_context_data.

- Usage example (conceptual)
  - Subclasses typically set template_name and override get_context_data to add more context.

Core extension: AbstractRenderableWithCss
----------------------------------------

- Purpose
  - Extends AbstractRenderable with a unified CSS class API.

- CSS class sources
  - get_base_css_classes_list(self)
  - get_extra_css_classes_list(self)
  - Both defaults return [].

- CSS class construction
  - get_css_classes_list(self)
    - Returns a list of CSS class names (base + extra, in that order).
  - css_classes (property)
    - Joins the list from get_css_classes_list() into a string.
    - If DJANGO_CRADMIN_INCLUDE_TEST_CSS_CLASSES is enabled, also appends test CSS classes generated for each suffix from get_test_css_class_suffixes_list().
  - get_css_classes_string(self)
    - Deprecated alias for css_classes; emits a DeprecationWarning and returns css_classes.

- Test CSS support
  - get_test_css_class_suffixes_list(self)
    - Returns a list of suffixes to include as test CSS classes.

- Helper
  - join_css_classes_list(css_classes_list)
    - Utility to join a list of CSS classes into a string using two spaces as the separator (see implementation: '  '.join(...)).

Core extension: AbstractBemRenderable
-------------------------------------

- Purpose
  - Base class for renderables using BEM (Block-Element-Modifier) naming.

- Initialization
  - __init__(self, bem_block=None, bem_element=None, bem_variant_list=None, extra_css_classes_list=None)
    - bem_block (str): BEM block name.
    - bem_element (str): BEM element name; must contain '__' and cannot be supplied together with bem_block.
    - bem_variant_list (list): List of BEM variants (suffixes after --).
    - extra_css_classes_list (list): Additional CSS classes.
    - Validation:
      - If both bem_block and bem_element are provided, raises ValueError.
      - If bem_element is provided and does not contain '__', raises ValueError.
    - Stores values to internal attributes for later use.

- Test CSS suffixes
  - get_test_css_class_suffixes_list(self)
    - Returns a list of suffixes used to generate test CSS classes (empty by default).

- Bem accessors and helpers
  - bem_block_or_element (property)
    - Returns get_bem_block() if set, otherwise get_bem_element().
  - get_bem_block(self)
    - Returns the stored bem_block.
  - get_bem_element(self)
    - Returns the stored bem_element.

Utility and integration details
-------------------------------

- Template rendering
  - Rendering is performed via Django’s render_to_string, with:
    - template_name obtained from get_template_names()
    - context_data merged from get_context_data(request) and any extra_context_data provided
    - request forwarded if provided

- Context shape
  - By default, the context contains 'me': self, enabling templates to access the renderable instance.

- CSS and test hooks
  - CSS class strings are produced by css_classes, which can include optional test CSS classes when enabled by settings.

- BEM support
  - AbstractBemRenderable provides a structured way to define BEM-based naming and related CSS classes via bem_block, bem_element, and bem_variant_list.

Example usage (conceptual)
--------------------------

- Simple renderable
  - A subclass sets template_name and overrides get_context_data to augment context.

- Renderable with CSS
  - Subclass AbstractRenderableWithCss, override get_base_css_classes_list and/or get_extra_css_classes_list to declare CSS classes; access via the css_classes property.

- BEM renderable
  - Subclass AbstractBemRenderable, provide bem_block or bem_element (with the required constraints), and optionally bem_variant_list and extra_css_classes_list to influence CSS composition.
