Nonparametric Methods
===================== 

Kernel Density Estimation (KDE)
------------------------------

Nonparametric kernel density estimation is used to estimate the distribution of a univariate random variable without assuming a parametric form for the distribution. The estimator uses kernel functions to smooth the observed data.

KDEUnivariate
--------------

Univariate Kernel Density Estimator.

Class
^^^^^

KDEUnivariate(endog)

Notes
^^^^^

- If cdf, sf, cumhazard, or entropy are computed, they are computed based on the definition of the kernel rather than the FFT approximation, even if the density is fit with FFT = True.
- KDEUnivariate is typically faster than KDEMultivariate due to its FFT-based implementation. It is preferred for univariate, continuous data. KDEMultivariate also supports mixed data.

Usage
^^^^^

Parameters
----------

endog : array_like
    The variable for which the density estimate is desired.

Notes on the fit method
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
- kernel: String specifying the kernel to use. Options include:
  - "biw" (Biweight)
  - "cos" (Cosine)
  - "epa" (Epanechnikov)
  - "gau" (Gaussian)
  - "tri" (Triangular)
  - "triw" (Triweight)
  - "uni" (Uniform)
- bw: Bandwidth selector. Options include:
  - "scott" – 1.059 * A * nobs ** (-1/5), where A = min(std(x), IQR/1.34)
  - "silverman" – .9 * A * nobs ** (-1/5), same A
  - "normal_reference" – C * A * nobs ** (-1/5), with kernel-determined C
  - A float value: used directly as the bandwidth
  - A callable: should accept two parameters (fn, kern) and return a float
- fft: If True, use FFT-based computation. FFT is more efficient but currently only supports the Gaussian kernel. Weights are not implemented with FFT.
- gridsize: Grid size for the evaluation grid. If None, max(len(endog), 50) is used.
- cut: Extends the grid beyond the observed range to ensure the kernel reaches zero at the ends. The extent is +/- cut * bw * min(x) or max(x).
- adjust: Adjustment factor for the bandwidth: bw_final = bw * adjust.
- weights: Optional weights for the observations. If provided, they are normalized.

Returns
^^^^^^^
KDEUnivariate
    The instance with the density estimate attached.

Attributes after fit
^^^^^^^^^^^^^^^^^^^^
- density: The estimated density evaluated on the support grid.
- support: The grid (support) where the density is evaluated.
- bw: The final bandwidth used.
- kernel: The kernel object corresponding to the chosen kernel with bandwidth bw.
- _cache: Internal cache used for performance reasons.

Kernel Switch
^^^^^^^^^^^^^

A mapping from short kernel names to kernel classes, e.g.:

- gau -> Gaussian
- epa -> Epanechnikov
- uni -> Uniform
- tri -> Triangular
- biw -> Biweight
- triw -> Triweight
- cos -> Cosine
- cos2 -> Cosine2

Examples
--------

Basic usage:

.. code-block:: python

   import numpy as np
   import statsmodels.api as sm

   nobs = 300
   np.random.seed(1234)
   data = np.random.normal(size=nobs)
   dens = sm.nonparametric.KDEUnivariate(data)
   dens.fit()
   # Access density and support
   density = dens.density
   support = dens.support
   bw = dens.bw

More options:

.. code-block:: python

   dens = sm.nonparametric.KDEUnivariate(data)
   dens.fit(kernel="gau", bw="silverman", fft=True)
   density = dens.density

Notes on FFT
^^^^^^^^^^^^^^
- FFT implementation is available only for Gaussian kernel.
- Weights are not implemented with FFT.
