SANS to SESANS conversion
=========================

Overview
--------
This documentation describes the SANS-to-SESANS conversion utilities exposed by the sans_to_sesans module. The tools provide a pathway to translate data and models between Small-Angle Neutron Scattering (SANS) representations (I(q), q-space) and Spin-Echo Small-Angle Neutron Scattering (SESANS) representations (P(y), y-space). The approach maps scattering coordinates through a projection-based geometry, enabling cross-compatibility of analyses and visualizations between the two measurement modalities.

Key ideas
--------
- Coordinate projection: A projection-based transformation connects q-space to SESANS y-space. The implementation leverages a Guyou-like style of mapping to relate the two coordinate systems.
- Elliptic function machinery: The projection relies on elliptic functions to compute the necessary intermediate quantities. The core routines use NumPy broadcasting for scalar- and array-valued inputs and SciPy’s elliptic function implementations.
- Flexible inputs: The transformation supports scalar or array inputs, enabling batch processing of datasets and models.

Implementation notes
--------------------
- Core routines are implemented with NumPy and SciPy, using elliptic functions to perform the necessary coordinate transforms.
- The module provides an array-friendly interface, broadcasting inputs as needed to produce corresponding outputs in SESANS space or back to SANS space.

Usage
-----
- Converting from SANS to SESANS:

  - Input: q-values and corresponding I(q) measurements.
  - Output: y-values (spin-echo length) and P(y) polarization profile.

- Inverse conversion (SESANS to SANS):

  - Input: y-values and SESANS polarization P(y).
  - Output: q-values and reconstructed I(q) if the inversion is well-posed for the given data.

- Typical workflow:

  - Prepare q-space data I(q) with uncertainties.
  - Call the conversion routine to obtain SESANS P(y) versus y.
  - Use P(y) for model comparisons or visualization in SESANS space.

Notes on the mathematics
------------------------
- The transformation relies on a projection that translates q-space coordinates into SESANS path-length space. Due to the use of elliptic functions, the implementation can handle a variety of geometries and sampling schemes.
- Broadcasting and vectorization are employed to support both single- and multi-dimensional datasets efficiently.

API (high level)
----------------
- sans_to_sesans.convert(q, Iq, y_grid=None) -> (y, Py)
  - Converts a SANS I(q) dataset to a SESANS P(y) representation.

- sans_to_sesans.invert(y, Py, q_grid=None) -> (q, Iq)
  - Converts a SESANS P(y) profile back to SANS I(q), where possible.

- Internal helpers (ellipticJi, _ellipticJi_real, _ellipticJi_imag)
  - Implement the elliptic-function-based components of the projection.

Examples
--------
- Simple conversion:

  - y, Py = sans_to_sesans.convert(q, Iq)

- Inverse conversion:

  - q, Iq = sans_to_sesans.invert(y, Py)

Limitations
-----------
- Inversion from SESANS to SANS may be ill-posed for certain data sets or limited y/Q ranges.
- Realistic error propagation requires careful treatment of uncertainties in both representations.
