Outlier and Novelty Detection
==============================

Overview
--------
Outlier detection (anomaly detection) and novelty detection are techniques for identifying data that do not conform to expected patterns. In scikit-learn, these tasks are solved with "one-class" or unsupervised estimators that model normality and flag deviations.

Key concepts
------------
- Outlier detection: Detects anomalies in unlabeled data. The model is trained on data that is assumed to be representative of normal behavior and then used to score new samples.
- Novelty detection: Detects anomalies in new data after being trained on a set of normal data. The model is fit on normal data only and evaluated on unseen data.

Common algorithms in scikit-learn
---------------------------------

One-class classifiers (suitable for novelty detection)
- OneClassSVM
  - Idea: Learn a boundary around normal data and label points outside the boundary as anomalies.
  - Typical usage: Train on normal data, then predict on new samples.
  - Example:
    from sklearn.svm import OneClassSVM
    model = OneClassSVM(nu=0.05, kernel='rbf', gamma='scale')
    model.fit(X_train_normal)
    y_pred = model.predict(X_test)

- LocalOutlierFactor (LOF) in novelty mode
  - Idea: Detects anomalies by local density deviation. In novelty mode, LOF can be fitted on normal data and used to score new samples.
  - Important: Use novelty=True to enable scoring of new data.
  - Example:
    from sklearn.neighbors import LocalOutlierFactor
    lof = LocalOutlierFactor(n_neighbors=20, contamination=0.1, novelty=True)
    lof.fit(X_train_normal)
    y_pred = lof.predict(X_test)

- EllipticEnvelope
  - Idea: Assumes data follow a Gaussian (elliptical) distribution and detects outliers accordingly. Can be used for novelty detection when trained on normal data.
  - Example:
    from sklearn.covariance import EllipticEnvelope
    ee = EllipticEnvelope(contamination=0.1)
    ee.fit(X_train_normal)
    y_pred = ee.predict(X_test)

Outlier detection (unsupervised or semi-supervised)
- IsolationForest
  - Idea: Detects anomalies by isolation; random trees isolate anomalies faster. Can operate in novelty mode when trained on normal data only.
  - Example:
    from sklearn.ensemble import IsolationForest
    iso = IsolationForest(contamination=0.1, random_state=42)
    iso.fit(X_train_normal)
    y_pred = iso.predict(X_test)

- LocalOutlierFactor (LOF) in standard mode
  - Idea: Detects anomalies based on local density without requiring novelty mode. Used for unsupervised outlier detection on the training data.
  - Example:
    from sklearn.neighbors import LocalOutlierFactor
    lof = LocalOutlierFactor(n_neighbors=20, contamination=0.1)
    y_pred = lof.fit_predict(X)

Guidance for choosing an algorithm
-------------------------------
- When you have a clean set of normal data and want to score new observations (novelty detection), use:
  - OneClassSVM, LOF with novelty=True, IsolationForest with novelty=True, or EllipticEnvelope.
- When you have unlabeled data and want to detect anomalies within that data, use:
  - LOF (standard), IsolationForest, EllipticEnvelope.
- Consider data characteristics:
  - High-dimensional data: ensemble methods like IsolationForest can be effective.
  - Data with a Gaussian-like spread: EllipticEnvelope can be appropriate.
  - Local density variations: LOF can capture local anomalies, but tuning of n_neighbors and contamination is important.
- Contamination parameter:
  - Use contamination to reflect the expected proportion of anomalies in the data; it influences decision thresholds and scoring.

Evaluation and metrics
----------------------
- Common metrics: precision, recall, F1-score, ROC AUC, PR AUC.
- Be cautious with unsupervised methods: without labeled ground truth, you may rely on domain knowledge or synthetic benchmarks.
- For novelty detectors, evaluate on a held-out set of normal data and a separate set of known anomalies if available.

Practical tips
-------------
- Preprocessing matters: scale features when required (especially for OneClassSVM and LOF).
- LOF is sensitive to the choice of n_neighbors; experiment with this parameter.
- For novelty detection, ensure the training data truly represents normal behavior (avoid leakage of anomalous data into the training set).
- Use cross-validation carefully with novelty detectors; traditional cross-validation may not be appropriate for novelty scoring.

Example workflow
----------------
1. Prepare normal data (training set).
2. Choose an algorithm and fit on the normal data.
3. Score or predict on new data to identify anomalies.
4. Validate results using domain knowledge or labeled anomalies if available.
