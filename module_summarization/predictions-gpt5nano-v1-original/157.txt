Nextcord Introduction
=====================

Nextcord is a modern Python wrapper for the Discord API, designed to be easy to use and actively maintained. It offers high-level abstractions for common Discord features and a typed, Pythonic API.

Widget API
----------

Nextcord exposes lightweight representations for guild widgets, which provide a read-only snapshot of a guild's voice channels and online members. Widgets are useful for dashboards, bots, or external UIs that want to display guild state without performing real-time synchronization.

WidgetChannel
~~~~~~~~~~~~

Represents a channel in a guild widget. A WidgetChannel is a partial channel with:
- id: The channel's ID
- name: The channel's name
- position: The channel's position

It supports:
- __str__() -> channel name
- mention property -> a string to mention the channel, e.g. <#123>

WidgetMember
~~~~~~~~~~~~

Represents a member in the widget's guild. Important attributes include:
- id, name, discriminator
- bot
- status (Status)
- nick (optional)
- avatar (optional)
- activity (optional; BaseActivity or Spotify)
- deafened, muted, suppress
- connected_channel (optional WidgetChannel)

Display name:
- display_name returns nick if available, otherwise the username.

Widget
~~~~~~

Represents a Guild widget. Key attributes:
- id
- name
- channels: List[WidgetChannel] — the accessible voice channels in the guild
- members: List[WidgetMember] — online members in the server; offline members may not be included due to Discord limitations

Usage notes:
- The widget data is provided by Discord and may be limited by permissions and the server's widget configuration.
