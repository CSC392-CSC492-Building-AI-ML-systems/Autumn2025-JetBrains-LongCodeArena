How to use QuerySet to build your queries
===========================================

Overview
--------
QuerySet provides a fluent API to build and execute database queries for your models. You can filter data, combine complex conditions, traverse relations, prefetch related records, annotate results with aggregations, select specific fields, and fetch results in various shapes (model instances, dictionaries, or lists).

Starting a query
---------------
- Basic filtering and retrieval
  .. code-block:: python

     from myapp.models import User
     # fetch users matching simple field filters
     users = await User.filter(age=30)

- Get a single object (may raise DoesNotExist or MultipleObjectsReturned)
  .. code-block:: python

     user = await User.get(id=42)

Filtering and combining conditions
----------------------------------
- Simple filters
  .. code-block:: python

     active_users = await User.filter(is_active=True)

- Complex filters using Q objects
  .. code-block:: python

     from tortoise.expressions import Q
     q = Q(age__gte=18) & (Q(status="active") | Q(status="premium"))
     adult_active_users = await User.filter(q)

- Nesting and traversing relations in filters
  You can filter on fields of related models using double-underscore notation.
  .. code-block:: python

     # users whose author's name is "John Doe" (example related model)
     books = await Book.filter(author__name="John Doe")

Joins and related data
----------------------
- Query across related models automatically creates joins as needed.
  Use traversal in filter expressions (e.g., related_model__field) to filter on related data.
- Prefetch related data to avoid additional queries
  .. code-block:: python

     users = await User.filter(is_active=True).prefetch_related("groups", "posts")

Aggregations and annotations
----------------------------
- Annotate results with aggregations (e.g., counts) and use them in ordering or retrieval
  .. code-block:: python

     from tortoise.functions import Count
     users = await User.annotate(post_count=Count("posts")).order_by("-post_count")

Selecting and shaping results
-----------------------------
- Fetch as dictionaries (values) or as tuples/lists (values_list)
  .. code-block:: python

     # list of dictionaries with selected fields
     users_dicts = await User.values("id", "username", "email")

     # list of tuples (id, username)
     users_tuples = await User.values_list("id", "username")

- Select only certain fields on model instances
  .. code-block:: python

     partial_users = await User.all().only("id", "username")

Ordering
--------
- Order results by one or more fields. Use a leading '-' for descending order.
  .. code-block:: python

     ordered_users = await User.filter(is_active=True).order_by("-last_login", "username")

- Ordering by related fields or annotations is supported via traversal or explicit annotations
  .. code-block:: python

     # order by related field (via traversal)
     ranked_users = await User.filter(is_active=True).order_by("profile__rating")

     # or order by an annotation
     # (assuming you annotated a field named 'score' earlier)
     ranked_users = await User.annotate(score=Count("points")).order_by("-score")

Notes and common patterns
-------------------------
- When using filters, you can combine multiple conditions with Q objects using standard Python operators (&, |).
- Filtering by relations directly (i.e., trying to filter on a whole related object) is not supported; filter on fields of related models using nested field names (e.g., related__field).
- Queries that involve aggregates may require grouping; the implementation handles this as needed when there are joins or HAVING clauses.
- Exceptions to handle for single-object retrieval:
  - DoesNotExist: no object matched the query
  - MultipleObjectsReturned: more than one object matched the query

Typical end-to-end example
--------------------------
.. code-block:: python

   from tortoise.expressions import Q
   from tortoise.functions import Count
   from myapp.models import User, Post

   # Find active users older than 18, who have at least 5 posts, ordered by newest activity
   users = await (
       User
       .filter(Q(is_active=True) & Q(age__gte=18))
       .annotate(post_count=Count("posts"))
       .filter(post_count__gte=5)
       .order_by("-last_login")
       .prefetch_related("posts")
   )
