Plans
=====


Module overview
- This module provides planning helpers for Bluesky-style plans:
  - count: Take a sequence of readings from detectors with optional timing.
  - list_scan: Perform a Cartesian product scan over one or more motors while recording detector readings.
  - rel_list_scan: Intended to scan relative to the current position (not yet implemented).

Functions
---------

count(detectors, num=1, delay=None, *, per_shot=None, md=None)
--------------------------------------------------------------------------------
Take one or more readings from detectors.

Parameters
- detectors: list of readables (e.g., ophyd objects) to read.
- num: integer, optional; number of readings to take (default 1). If None, capture until canceled.
- delay: iterable or scalar, optional; time delay in seconds between successive readings (default 0).
- per_shot: callable, optional; hook for customizing action of the inner loop (messages per step).
  Signature: def f(detectors: Iterable[OphydObj]) -> Generator[Msg]: ...
- md: dict, optional; metadata to attach to the plan.

Notes
- If delay is iterable, it must have at least num - 1 entries; otherwise a ValueError is raised during iteration.
- Internally builds an MD dict including detectors, num_points, and other plan metadata.
- If per_shot is None, it defaults to bps.one_shot.
- Executes an inner loop via bps.repeat(partial(per_shot, detectors), num=num, delay=delay).

Returns
- A generator-based plan to be yielded from (via Bluesky run engine).

Examples
- Basic usage: count([det1, det2], num=5, delay=0.1)

list_scan(detectors, *args, per_step=None, md=None)
---------------------------------------------------
Scan over one or more variables in steps simultaneously (inner product).

Parameters
- detectors: list of readables to record at each point.
- *args: For one dimension, motor, [points...]. For multiple dimensions, pairs of motor and corresponding point lists.
  Example: motor1, [p1, p2], motor2, [q1, q2], ...
- per_step: callable, optional; hook for customizing action of inner loop (messages per step).
  Signature: f(detectors, motor, step) -> plan (generator)
- md: dict, optional; metadata to attach to the plan.

Notes
- All provided point lists must have the same length; otherwise a ValueError is raised.
- Builds metadata including plan_name='list_scan', plan_pattern='inner_list_product', and related plan_pattern information.
- The function constructs md_args from the motors and their point lists and uses plan_patterns.inner_list_product(args) to generate the cycler.
- Sets up default hints for plotting dimensions based on motor hints, if available.
- Delegates to scan_nd(detectors, full_cycler, per_step=per_step, md=_md) to execute the scan.

Returns
- A generator-based plan to be yielded from (via Bluesky run engine).

See Also
- bluesky.plans.rel_list_scan
- bluesky.plans.list_grid_scan
- bluesky.plans.rel_list_grid_scan

Examples
- Basic usage: list_scan([det1], motor1, [0, 1, 2], motor2, [10, 20, 30])

rel_list_scan(detectors, *args, per_step=None, md=None)
-------------------------------------------------------
Scan over one variable in steps relative to the current position.

Parameters
- detectors: list of readables to record at each point.
- *args: Pairs of motor and [relative points], where points are relative to the motor's current position.
- per_step: callable, optional; hook for customizing action of inner loop (messages per step).
- md: dict, optional; metadata to attach to the plan.

Notes
- This function is currently a placeholder (TODO) and not implemented.

Returns
- Not implemented yet; plan skeleton left for future development.

See Also
- bluesky.plans.list_scan
- bluesky.plans.list_grid_scan
- bluesky.plans.rel_list_grid_scan

Changelog and dependencies
- Uses plan_patterns for complex plan structures and patterns.
- Uses utilities (utils, Msg) for message handling and metadata formatting.
- Falls back to cytools.partition if available, with a fallback to toolz.partition.
- Decorated with preprocessors (bpp) for stage/run context management and plan_stubs (bps) for standard plan helpers.

Notes
- The rel_list_scan function is declared and documented here for future implementation.
- The module is designed to be integrated with Bluesky’s plan execution engine and metadata conventions.
