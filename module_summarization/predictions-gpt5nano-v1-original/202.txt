Permutation feature importance
===============================

Overview
--------
Permutation feature importance measures how much a trained model’s predictive performance decreases when the values of a single feature are randomly permuted. By breaking the association between a feature and the target, this method estimates how important that feature is for the model’s predictions. Higher drops in the score indicate more important features.

Algorithm
---------
1. Compute the baseline score using the provided scoring metric on the original data (X, y).
2. For each feature:
   - Permute the feature’s values across samples (shuffling).
   - Recompute the score with the permuted feature.
   - Repeat the permutation-and-evaluation process n_repeats times.
3. For each feature, compute the permutation importance as the difference between the baseline score and the permuted scores.
4. Aggregate results:
   - importances_mean: mean of the differences across repeats.
   - importances_std: standard deviation of the differences across repeats.
   - importances: raw differences for each feature across repeats.
5. If a subset of samples is used (max_samples < n_samples), perform bootstrap sampling as needed for speed.

Parameters
----------
estimator : estimator
    Trained estimator implementing fit and a scoring interface.

X : array-like
    Input data of shape (n_samples, n_features).

y : array-like or None
    Target values. If None, some estimators supporting unsupervised scoring may be used.

scoring : str, callable, list, tuple, dict, None, or multi-metric
    Scoring function or name used to evaluate the estimator. If None, the estimator’s default scoring is used.

n_repeats : int
    Number of permutation repeats for each feature. Higher values yield more stable estimates.

n_jobs : int or None
    Number of parallel jobs to run. None means 1 job. (-1 may denote using all CPUs depending on the environment.)

random_state : int, RandomState instance or None
    Seed or random state for reproducibility of the permutation.

sample_weight : array-like or None
    Sample weights to pass to the scoring function, if supported.

max_samples : int or RealNotInt or None
    If set, use a subset of samples with bootstrap sampling for efficiency. Otherwise, use all samples.

Returns
-------
result : Bunch
    A dictionary-like object with the following attributes:
    - importances_mean : ndarray of shape (n_features,)
        Mean permutation importances across repeats.
    - importances_std : ndarray of shape (n_features,)
        Standard deviation of permutation importances across repeats.
    - importances : ndarray of shape (n_features, n_repeats)
        Raw permutation importance scores for each feature and repeat.

Notes
------
- Permutation importance is model-agnostic and can be applied to any estimator that supports scoring.
- It can handle multi-metric scoring; in that case, the aggregate function combines per-metric scores appropriately.
- If max_samples is used, the method internally samples a subset of the data to trade off speed and accuracy.
- Feature names (if available) can be used to label results; otherwise, features are indexed numerically.

Examples
--------
Below is a minimal example of computing permutation feature importances for a fitted classifier.

.. code-block:: python

    from sklearn.ensemble import RandomForestClassifier
    from sklearn.datasets import load_iris
    from sklearn.inspection import permutation_importance

    data = load_iris()
    X, y = data.data, data.target

    clf = RandomForestClassifier(random_state=0)
    clf.fit(X, y)

    result = permutation_importance(clf, X, y, scoring='accuracy', n_repeats=30, random_state=0)

    print("Mean importances:", result.importances_mean)
    print("Importances std:", result.importances_std)
