PyTables File Format
=====================

Overview
--------
PyTables stores data in HDF5 files using a hierarchy of Groups and Datasets. In addition to the standard HDF5 objects, PyTables maintains its own file-format constructs to support fast queries, such as column indexes. This document summarizes the parts of the on-disk format that are specific to PyTables, focusing on the indexing mechanism exposed by the Index class.

Index objects
-------------
- Purpose: An index represents the indexing information for a single column of a table. Each Index is associated with exactly one column.
- Versioning: The on-disk format version for Index objects is controlled by obversion. The current default is "2.1" (used by PyTables Pro 2.1 and newer). Older versions exist for compatibility.
- Path and naming:
  - Index objects are stored in the file under the table’s group, using a path component that begins with a prefix “_i_” to denote an index. The helper logic extracts the table path and the column path from the overall index pathname.
  - The function _table_column_pathname_of_index(indexpathname) parses the full index path, locating the segment that starts with "_i_", and returns:
    - tablepathname: the path to the table group
    - colpathname: the path to the indexed column within the table
- Key properties:
  - kind: The granularity of the index. It translates an internal size (indsize) to one of:
    - ultralight (size 1)
    - light (size 2)
    - medium (size 4)
    - full (size 8)
    This controls how much of the row position is tracked.
  - filters: The I/O filters applied to the index datasets (e.g., compression level, library, shuffling, integrity checks). Implemented via a Filters instance.
  - dirty: A flag indicating whether the index is out of sync with the table data. When DIRTY is True, the index cannot be used reliably for queries until it is rebuilt or updated.
  - column: The Column object associated with the index (derived from the path).
  - table: The Table object containing the indexed column (derived from the path).
  - nblockssuperblock: The number of blocks per superblock. Calculated as superblocksize divided by blocksize.
  - nslicesblock: The number of slices per block. Calculated as blocksize divided by slicesize.
  - nchunkslice: The number of chunks per slice. Calculated as slicesize divided by chunksize.
  - nsuperblocks: The total number of superblocks for the index (derived from the total size and superblock size; exact calculation depends on dataset dimensions and layout).
- Storage details:
  - Index-related datasets and attributes are stored in a hierarchical structure adjacent to the indexed table, organized to support fast access and independent filtering.
  - The DIRTY attribute is stored in the index’s attributes and is consulted to determine whether the index can be used for queries without rebuilding.

Default settings and terminology
-------------------------------
- Default indexing behavior:
  - default_auto_index: True. PyTables will automatically maintain indexes after operations that modify the table data, and will reindex if needed.
  - default_index_filters: A Filters instance specifying compression and IO behavior for index data. Example defaults include complevel=1, complib='zlib', shuffle=True, fletcher32=False.
- Sorting and search optimizations:
  - defsort: The default sorting algorithm used in internal operations (e.g., "mergesort" for deterministic stable sorting).
  - opt_search_types: The list of numeric types for which optimized search implementations exist (e.g., int8, int16, int32, int64, uint8, uint16, uint32, uint64, float32, float64).
- Limits:
  - max32: Upper limit for 32-bit unsigned integers (2**32).

Additional notes
----------------
- The Index class inherits behavior from PyTables’ internal mixins and extensions, reflecting PyTables’ object model (e.g., NotLoggedMixin, Group) and the index extension module.
- The on-disk representation combines PyTables’ internal object model with HDF5 storage semantics, enabling efficient indexing while remaining compatible with the overall PyTables data layout.

This document describes the essential aspects of the PyTables file format as it pertains to column indexes and their on-disk representation, including naming conventions, versioning, layout parameters, and default settings.
