Using metadata
===============

Overview
--------
The ftrack API client attaches metadata to every HTTP request to identify and authenticate
the caller. This metadata is transmitted as HTTP headers added automatically by the
SessionAuthentication handler:

- ftrack-api-key: the API key
- ftrack-user: the API user

Providing metadata
------------------
Credentials and server information can be supplied in three ways:

- server_url
  - The URL of the ftrack server to connect to.
  - If not provided, the environment variable FTRACK_SERVER is used.
  - If neither is provided, an error is raised.

- api_key
  - The API key to use for authentication.
  - If not provided, the environment variable FTRACK_API_KEY is used. For backwards compatibility, FTRACK_APIKEY is also checked.

- api_user
  - The username to operate against in ftrack.
  - If not provided, FTRACK_API_USER is used. If that is not set, the current system user is used via getpass.getuser() when possible.

Notes
-----
- Credentials can be provided directly to Session(...) or resolved from environment variables.
- If required information cannot be determined, a TypeError is raised.

Environment variables
---------------------
- FTRACK_SERVER: server URL
- FTRACK_API_KEY or FTRACK_APIKEY: API key
- FTRACK_API_USER: API user

Examples
--------
Basic usage with explicit credentials:

.. code-block:: python

   from ftrack_api import Session
   s = Session(
       server_url='https://ftrack.example.com',
       api_key='your_api_key',
       api_user='your_username'
   )

Using environment variables (no explicit credentials):

.. code-block:: python

   # Environment must provide these variables
   # FTRACK_SERVER, FTRACK_API_KEY (or FTRACK_APIKEY), FTRACK_API_USER
   s = Session()
