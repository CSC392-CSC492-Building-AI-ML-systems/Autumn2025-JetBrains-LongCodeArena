w_trace
=======

Overview
--------
Trace a trajectory segment back to its origin within a WESTPA data store. The command
reconstructs the lineage by following parent segments from a specified end segment to the
initial state, collecting per-segment metadata (iteration, segment id, weight, walltime,
CPU time, and final coordinated state).

How it works
------------
- The trace is constructed from the data manager’s iteration groups. For each segment along the
  lineage, information such as weight, walltime, cputime, and the final pcoord is recorded.
- The trace supports both current and older HDF5 layouts. When necessary, initial-state data is
  retrieved from the data manager or reconstructed from the iteration data.
- The trace stores a summary array with per-segment entries and also keeps references to
  endpoint_type, basis_state, and initial_state.

Trace data model
---------------
Trace consists of:
- summary: a structured NumPy array with fields:
  - n_iter (n_iter_dtype)
  - seg_id (seg_id_dtype)
  - weight (weight_dtype)
  - walltime (utime_dtype)
  - cputime (utime_dtype)
  - final_pcoord (pcoord_dtype with shape pcoord_pt_shape)
- endpoint_type: type of the trajectory’s final endpoint
- basis_state: basis state used for the initial segment (if available)
- initial_state: the initial state object for the traced trajectory
- data_manager: data manager used to access datasets

Key methods
-----------
- Trace.from_data_manager(n_iter, seg_id, data_manager=None)
  Constructs a Trace for the trajectory ending at (n_iter, seg_id) by walking backward through
  the parent_id chain. Returns a Trace instance containing the summary, endpoint_type,
  basis_state, and initial_state.

- Trace.get_segment_data_slice(datafile, dsname, n_iter, seg_id, slice_=None, index_data=None, iter_prec=None)
  Retrieve a slice of data for a given segment from a dataset within the appropriate iteration
  group. Supports indexing via index_data for fast lookup of (n_iter, seg_id) pairs.

- Trace.trace_timepoint_dataset(dsname, slice_=None, auxfile=None, index_ds=None)
  Return a time-ordered trace along this trajectory for a dataset laid out as [seg_id][timepoint][...].
  Accounts for overlapping values at segment boundaries. Returns (data_trace, weight),
  where data_trace is the time-series data along the trajectory and weight is the corresponding
  trajectory weight at each time point.

Usage
-----
w_trace <n_iter> <seg_id> [options]

Options (examples; exact flags may vary by version)
- --dataset DSNAME: dataset to trace along (e.g., pcoord)
- --slice SLICE: numpy-like slice to apply to the retrieved data
- --index INDEX_DS: dataset to use for index-based lookup of (n_iter, seg_id)
- --auxfile FILE: path to an auxiliary file for additional data
- --iter-prec N: override iteration precision for HDF5 groups
- --verbose: increase output verbosity

Output
------
- A Trace object containing:
  - summary: per-segment record with fields (n_iter, seg_id, weight, walltime, cputime, final_pcoord)
  - endpoint_type, basis_state, initial_state
  - data_manager reference
- Optional timepoint data trace via Trace.trace_timepoint_dataset if requested

Notes
-----
- If the data store uses an old HDF5 layout, initial_state may be constructed from the
  iter_group pcoord data.
- The implementation is designed to minimize file I/O by caching opened auxiliary files.

Examples
--------
- Basic trace of a specific end segment:
  w_trace 10 5

- Trace a particular dataset along the trajectory:
  w_trace 10 5 --dataset pcoord

- Retrieve a timepoint trace for a dataset:
  w_trace 10 5 --dataset pcoord --slice [0:100] --auxfile my_aux.h5

This page documents the w_trace command's behavior as implemented by the Trace class and its
associated data access methods.
