w_fork
======

Synopsis
--------

w_fork [OPTIONS]

Description
-----------

Prepare a new WestPA weighted-ensemble simulation from an existing one at a particular iteration. A new HDF5 file is generated. For executable propagation, the user is responsible for arranging the new simulation directory so that the old simulation’s restart data from the chosen iteration is available as the new simulation’s initial state data. A mapping of old simulation segments to new initial states is created, both in the new HDF5 file and in a flat text file to aid in traceability. The target states and basis states for the new simulation are taken from those in the original simulation.

Options
-------

- -i, --input
  dest='input_h5file'
  Create simulation from the given INPUT_H5FILE (default: read from configuration).

- -I, --iteration
  dest='n_iter'
  type=int
  Take the initial distribution for the new simulation from iteration N_ITER (default: last complete iteration).

- -o, --output
  dest='output_h5file'
  default='forked.h5'
  Save the new simulation HDF5 file as OUTPUT (default: forked.h5).

- --istate-map
  default='istate_map.txt'
  Write a text file describing the mapping of existing segments to new initial states in ISTATE_MAP (default: istate_map.txt).

- --no-headers
  action='store_true'
  Do not write headers to ISTATE_MAP.

What w_fork does
----------------

- Opens the existing (old) HDF5 file and determines the relevant iteration.
- Creates a new HDF5 file and copies the target states.
- Copies the basis states into the new file.
- Transforms each old segment into exactly one new initial state and one corresponding new segment:
  - Creates initial states with appropriate iteration counters, type, and status, and assigns the final pcoord from the old segment.
  - Creates a new segment in the new iteration with the same weight as the old segment, linking it to the new initial state via parent IDs, and initializes its pcoord to include the initial point.
- Updates the new file’s initial states and prepares the first iteration with the created segments.
- Sets the new file’s current iteration to 1, then closes both old and new files.
- Writes a mapping file (default istate_map.txt) describing how old segments map to new initial states. The file can include a header (unless suppressed with --no-headers) and contains lines with old_n_iter, old_seg_id, and new_istate_id.

Output
------

- A new HDF5 file containing the forked simulation (OUTPUT, default forked.h5).
- A flat-text mapping file describing the old-to-new initial-state mapping (ISTATE_MAP, default istate_map.txt).

Notes
-----

- The mapping file aids in tracing how the old simulation’s segments were reinitialized as the new simulation’s initial states.
- The new initial states and segments are created only for the old segments present in the chosen iteration. Further adjustments can be made with other WestPA tools if needed.
