Getting started: Generate getting started tutorial
=============================================

This document describes how to create a self-contained, getting-started experience for Mitogen’s master/parent process functionality by leveraging the shared core source and runtime helpers provided in the common module.

Overview
--------
The module provides functionality common to master and parent processes and is designed to be sent to child contexts that become parents in recursive connections. It includes utilities for:
- Robustly determining the Python interpreter path
- Accessing and partially compressing the Mitogen core source for embedding in child processes
- Handling PTY and OS-specific quirks (e.g., Linux SELinux, open file descriptor limits)
- Exposing helpful, pre-defined messages for common failure modes (broker shutdown, PTY creation, missing Python executable)

Key concepts
------------
- Master/child architecture: The code manages context creation and inter-process communication between master, parent, and child processes in Mitogen.
- PTY handling: The module contains constants and logic related to pseudo-terminal (PTY) usage and related failures.
- Environment resilience: It includes fallbacks and warnings when system characteristics (such as sys.executable) are not available.
- Core source provisioning: The core Mitogen source can be retrieved and embedded in child contexts via a partial, compressed representation to minimize startup overhead.

Public helpers and messages
---------------------------
- get_log_level(): Read the effective log level for the module.
- get_sys_executable(): Return sys.executable if set, otherwise log a warning and fall back to a default.
- get_core_source_partial(): Retrieve a cached, partially compressed representation of the Mitogen core source for embedding in child contexts.
- SYS_EXECUTABLE_MSG: Message shown when Python cannot determine the original program name.
- OPENPTY_MSG: Guidance for PTY creation failures (often related to system limits or configuration).
- BROKER_SHUTDOWN_MSG: Notification used when the broker is shutting down.

Usage notes
-----------
- Determining the interpreter:
  - Use get_sys_executable() to obtain a reliable Python executable path, with a safe fallback when sys.executable is not set.
  - If the fallback is used, a warning is logged once to avoid noisy logs.

- Embedding Mitogen core in children:
  - Use get_core_source_partial() to obtain a compressed representation of mitogen.core without incurring the cost of repeatedly accessing the full source.
  - This is designed to reduce startup latency in child contexts when the core needs to be available without a full import-time dependency load.

- Platform considerations:
  - SELinux: The code avoids using socketpairs in environments with SELinux enabled.
  - Linux specifics: The module defines Linux-specific ioctl helpers and counts on the system’s open file descriptor limits to decide when to create PTYs.

Example
-------
- Retrieve a safe Python executable:
  from mitogen.core import get_sys_executable
  exe = get_sys_executable()
  print(exe)

- Prepare a partial core source for embedding in a child:
  from mitogen.core import get_core_source_partial
  core_partial = get_core_source_partial()
  # core_partial is a PartialZlib-wrapped, compressed representation of mitogen.core
  # It can be used by child contexts to reconstruct the necessary core behavior.

Notes on minification and threading
-----------------------------------
- _get_core_source() fetches the full Mitogen core source.
- get_core_source_partial() ensures this operation is performed safely in multi-threaded contexts by using a lock and caching the result.
- The PartialZlib wrapper is used to minimize the memory footprint and startup latency when injecting the core into child processes.

Error handling and messaging
----------------------------
- If the Python executable cannot be determined, SYS_EXECUTABLE_MSG is logged and the default fallback is used.
- If PTY creation fails due to system limits, OPENPTY_MSG provides guidance on remedies (e.g., adjusting kernel parameters or configuration).
- If a broker begins to shut down, BROKER_SHUTDOWN_MSG informs the user of the cancellation.

Prerequisites and compatibility
-------------------------------
- Works with Python 2.x and Python 3.x environments (with historical notes referencing Python 2.4/2.5 behavior in older contexts).
- Designed to operate in environments where inter-process communication and PTY management are required for Mitogen’s orchestration of master/child processes.

Summary
-------
This getting-started guide explains how to generate a self-contained, minimal tutorial surface by leveraging the core source provisioning and environment-safe utilities in the common module. By retrieving a compressed representation of mitogen.core and resolving the interpreter path robustly, you can create a portable, easy-to-distribute getting-started experience for users exploring Mitogen’s master/parent process capabilities.
