Contingency Tables (two-way)

This module provides classes and helpers for analyzing two-way contingency tables (frequency tables cross-classified by two categorical variables). The core class is Table, with additional utilities for DataFrame handling and result encapsulation. The module is designed around the standard statistical inference for contingency tables, with support for nominal and ordinal association testing.

Module API

- _make_df_square(table)
  - Purpose: Reindex a pandas DataFrame so that it becomes square (row and column indices contain the same values in the same order). If the input is not a DataFrame, it is returned unchanged.
  - Returns: A square DataFrame or the original input.

- _Bunch
  - A lightweight container for results returned by test methods. Provides readable representations via __repr__ and __str__.

- Table
  - Description: Represents a two-way contingency table and provides methods for common analyses.
  - Initialization
    - __init__(self, table, shift_zeros=True)
      - table: array_like
        Contingency table of observed counts.
      - shift_zeros: boolean
        If True and any cell count is zero, add 0.5 to all values in the table.
      - Attributes
        - table_orig: The original input table (cached).
        - table: The table as a NumPy array of dtype float64 (modified if shift_zeros is True).

  - from_data(cls, data, shift_zeros=True)
    - Class method to construct a Table from raw data.
    - data: array_like or DataFrame
      If a DataFrame, uses the first two columns to build a contingency table via a cross-tabulation.
    - Returns: A Table instance.

  - __str__(self)
    - Returns a human-readable string describing the 2D contingency table and its contents.

  - test_nominal_association(self)
    - Purpose: Assess independence for nominal (unordered) factors using a chi-squared test.
    - Returns: A _Bunch with attributes:
      - statistic: float (the chi-squared statistic)
      - df: int (degrees of freedom)
      - pvalue: float (p-value for the test)
    - Notes: The statistic is computed as the sum of the chi-square contributions across cells.

  - test_ordinal_association(self, row_scores=None, col_scores=None)
    - Purpose: Assess independence between two ordinal variables using a linear-by-linear (Cochran-Armitage) trend test.
    - Parameters:
      - row_scores: array_like (scores for rows; default is 0..n-1)
      - col_scores: array_like (scores for columns; default is 0..m-1)
    - Returns: A _Bunch with attributes common to the trend test (statistic, null_mean, null_sd, zscore, pvalue).
    - Notes: The provided scores define the test’s sensitivity to ordered alternatives. If not provided, the default equally spaced scores are used.

Dependencies

- numpy
- pandas
- scipy (scipy.stats)
- warnings
- statsmodels.iolib
- statsmodels.tools.sm_exceptions

Example

- Build a Table from raw data and perform a nominal association test:

  - from statsmodels.stats.contingency_tables import Table
  - data = [...]  # your data with at least two categorical variables
  - tbl = Table.from_data(data)
  - res = tbl.test_nominal_association()
  - print(res.statistic, res.df, res.pvalue)

Notes

- The module follows the standard inference model assuming independence and identical distribution of units classified by two categorical variables.
- The Top-level docstring mentions additional classes and tests (e.g., SquareTable, Table2x2, StratifiedTable, McNemar’s test, Cochran’s q test). The core implementation shown here centers on Table and the associated utilities for nominal/ordinal tests. Refer to the full module for any additional classes and tests that may be present.
