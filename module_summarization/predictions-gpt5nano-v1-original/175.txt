Biclustering
==============

Biclustering (also known as co-clustering) aims to simultaneously cluster the rows and columns of a data matrix to reveal submatrices (biclusters) with coherent patterns. This is particularly useful in applications where different subsets of features are informative for different subsets of samples, such as gene expression analysis and collaborative filtering.

Estimators
----------

The scikit-learn biclustering module provides spectral-based biclustering estimators that seek checkerboard-like patterns in data. The two primary estimators are:

- SpectralBiclustering
  - Discovers row and column clusters by applying spectral decomposition to a normalized data matrix and then partitioning the components into biclusters.
  - Produces row and column cluster assignments that define coherent submatrices.
  - Typical outputs:
    - row_labels_: array of cluster indices for each row
    - column_labels_: array of cluster indices for each column

- SpectralCoclustering
  - Models the data as a bipartite structure and finds co-clusters that maximize a checkerboard pattern in both dimensions.
  - Produces row and column cluster assignments similar to SpectralBiclustering, but with a different objective that emphasizes coclustering structure.

Parameters
----------

Both estimators follow the scikit-learn convention and expose parameters to control the number of clusters and randomness. Common concepts include:

- n_clusters (or n_clusters=(n_row, n_col) for SpectralBiclustering)
  - Number of row/column clusters to find.
- random_state
  - Seed for randomness to enable reproducible results.
- (Optional) n_components, tol, max_iter, and other convergence-related options
  - Controls for the internal spectral decomposition and optimization.

Outputs
-------

- row_labels_: 1-D array with the cluster label for each row.
- column_labels_: 1-D array with the cluster label for each column.

Usage
-----

Typical workflow:

- Create an estimator with desired n_clusters (and other options).
- Call fit(X) on the data matrix X.
- Inspect row_labels_ and column_labels_ to interpret the biclusters.

Example
-------

.. code-block:: python

   from sklearn.cluster import SpectralBiclustering

   # X is a real-valued data matrix (rows: samples, columns: features)
   model = SpectralBiclustering(n_clusters=(3, 3), random_state=0)
   model.fit(X)
   row_labels = model.row_labels_
   col_labels = model.column_labels_

Notes
-----

- Biclustering is particularly useful when patterns are localized to submatrices rather than the entire matrix.
- Spectral-based methods can be sensitive to scaling; consider standardizing data if appropriate for your domain.
- The interpretability of biclusters depends on the quality and preprocessing of the data.
