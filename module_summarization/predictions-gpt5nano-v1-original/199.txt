Neural network models (supervised)
==================================

This section documents the utilities used for supervised neural network models, including in-place activation functions, their derivatives for backpropagation, and common loss functions.

Activation functions (in-place)
-------------------------------
Activations are implemented as in-place functions to minimize memory usage. Each function takes:
- X: {array-like, sparse matrix}, shape (n_samples, n_features)
  The data to transform (modified in place).

Available activations (mapped by name):
- identity: leaves X unchanged
- tanh: hyperbolic tangent
- logistic: logistic sigmoid
- relu: rectified linear unit
- softmax: K-way softmax across features for each sample

API
- ACTIVATIONS: mapping from activation name (string) to the corresponding in-place function.

Activation derivatives (in-place)
----------------------------------
Derivatives used for backpropagation are also implemented as in-place operations. Each function takes:
- Z: {array-like, sparse matrix}, shape (n_samples, n_features)
  The data output from the activation during the forward pass.
- delta: {array-like}, shape (n_samples, n_features)
  The backpropagated error signal to be modified in place.

Available derivatives (mapped by name):
- identity: no change to delta
- tanh: delta *= 1 - Z**2
- logistic: delta *= Z * (1 - Z)
- relu: delta[Z == 0] = 0

API
- DERIVATIVES: mapping from activation name (string) to the corresponding derivative function.

Loss functions
--------------
Loss functions measure the discrepancy between true values and model predictions.

- squared_loss(y_true, y_pred)
  - Description: Computes 1/2 * mean((y_true - y_pred) ** 2)
  - Returns: scalar loss

- log_loss(y_true, y_prob)
  - Description: Logistic (cross-entropy) loss for multi-class classification
  - Details:
    - Uses a small epsilon for numerical stability
    - If y_prob has one column, expands to two classes
    - If y_true has one column, expands to two classes
  - Returns: scalar loss

- binary_log_loss(y_true, y_prob)
  - Description: Binary cross-entropy loss for multilabel/classification
  - Details:
    - Uses a small epsilon for numerical stability
    - Operates on probabilities with shape (n_samples, 1)
  - Returns: scalar loss

API
- Loss functions available: squared_loss, log_loss, binary_log_loss
- LOSS_FUNCTIONS: mapping from loss name to corresponding function implementation (e.g., "squared" -> squared_loss). The exact full mapping may include additional entries elsewhere in the codebase.

Usage notes
- All activation and derivative functions modify their second argument (delta) or input (X) in place to reduce temporary allocations.
- The public API enables concise switching of activation functions and their derivatives during model construction and training.

Example
.. code-block:: python

   import numpy as np
   X = np.array([[0.5, -1.2], [0.0, 2.0]])
   # Apply ReLU activation in place
   ACTIVATIONS['relu'](X)

   Z = X.copy()
   delta = np.ones_like(Z)
   # Backpropagate through ReLU
   DERIVATIVES['relu'](Z, delta)

   y_true = np.array([[1, 0], [0, 1]])
   y_prob = np.array([[0.7, 0.3], [0.2, 0.8]])
   loss = log_loss(y_true, y_prob)
