Transforms API
==============

Overview
--------
This module provides utilities used by image transformation operations. It includes
an InterpolationMode enum, mappings between PIL and internal representations, and
helpers to convert images between PIL/NumPy formats and PyTorch tensors.

Public API
- InterpolationMode (Enum)
  - NEAREST, NEAREST_EXACT, BILINEAR, BICUBIC, BOX, HAMMING, LANCZOS
  - Description: Supported interpolation methods for image resizing and related transforms.

- _interpolation_modes_from_int(i: int) -> InterpolationMode
  - Maps an integer to the corresponding InterpolationMode.

- pil_modes_mapping (dict)
  - Maps InterpolationMode to the corresponding PIL mode index.

- _is_pil_image (bool or function reference)
  - Flag indicating whether an object is a PIL image, sourced from the internal functional module.

- get_dimensions(img: Tensor) -> List[int]
  - Returns image dimensions as [channels, height, width].

- get_image_size(img: Tensor) -> List[int]
  - Returns image size as [width, height].

- get_image_num_channels(img: Tensor) -> int
  - Returns the number of image channels.

- to_tensor(pic) -> Tensor
  - Converts a PIL Image or NumPy array to a PyTorch Tensor.
  - Supports input types: PIL.Image.Image or numpy.ndarray.
  - Returns a float32 tensor in [0, 1] for 8-bit images; preserves dtype for other cases.
  - Handles NumPy arrays (2D/3D), PIL Images, and optional accimage inputs if available.

- pil_to_tensor(pic: Any) -> Tensor
  - Converts a PIL Image to a tensor of the same type (does not script).

Example
-------

Convert a PIL image to a PyTorch tensor and inspect its properties:

.. code-block:: python

    from PIL import Image
    from torchvision.transforms import to_tensor, get_dimensions, get_image_size, get_image_num_channels
    import numpy as np
    import torch

    # Load a PIL image
    img = Image.open("example.jpg").convert("RGB")

    # Convert to tensor
    t = to_tensor(img)

    # Tensor shape is (C, H, W)
    print("Tensor shape:", t.shape)                # e.g., (3, 512, 768)
    print("Tensor dtype:", t.dtype)                  # torch.float32
    print("Tensor min/max:", t.min(), t.max())

    # Inspect image properties using helpers
    dims = get_dimensions(t)
    size = get_image_size(t)
    channels = get_image_num_channels(t)

    print("Dimensions:", dims)       # [3, H, W]
    print("Size:", size)               # [W, H]
    print("Channels:", channels)        # 3

Another example: convert a NumPy array to a tensor

.. code-block:: python

    import numpy as np
    from torchvision.transforms import to_tensor

    arr = np.random.randint(0, 256, (480, 640, 3), dtype=np.uint8)
    t2 = to_tensor(arr)

    print("t2 shape:", t2.shape)      # (3, 480, 640)
    print("t2 dtype:", t2.dtype)      # torch.float32

Notes
-----
- to_tensor supports both PIL Images and NumPy arrays. If a NumPy array is provided,
  it is interpreted as a typical HWC image and converted to CHW layout.
- If accimage is installed and the input is an accimage.Image, it will be converted accordingly.
- The implementation logs API usage for monitoring purposes via _log_api_usage_once.
