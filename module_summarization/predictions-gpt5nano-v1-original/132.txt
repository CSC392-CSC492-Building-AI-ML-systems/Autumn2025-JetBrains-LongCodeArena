Markov chain Monte Carlo (MCMC) framework
=========================================

Overview
--------

This module provides a framework for equilibrium sampling from a given thermodynamic
state of a biomolecule using a Markov chain Monte Carlo (MCMC) scheme.

Supported moves
---------------

The framework currently offers support for:
- Langevin dynamics (assumed to be free of integration error; use at your own risk)
- Hybrid Monte Carlo (HMC)
- Generalized Hybrid Monte Carlo (GHMC)
- Monte Carlo barostat moves

These moves can be combined through the SequenceMove and WeightedMove classes.

Defaults and configuration
--------------------------

By default, the MCMC moves use the fastest OpenMM platform available and a shared
global ContextCache that minimizes the number of OpenMM contexts created. The
examples below show how to configure these aspects.

References
----------

Jun S. Liu. Monte Carlo Strategies in Scientific Computing. Springer, 2008.

Examples
--------

Create the initial state (thermodynamic and microscopic) for an alanine dipeptide
system in vacuum.

.. code-block:: python

   from simtk import unit
   from openmmtools import testsystems, cache
   from openmmtools.states import ThermodynamicState, SamplerState

   test = testsystems.AlanineDipeptideVacuum()
   thermodynamic_state = ThermodynamicState(system=test.system,
                                           temperature=298*unit.kelvin)
   sampler_state = SamplerState(positions=test.positions)

Create an MCMC move to perform at every iteration of the simulation, and
initialize a sampler instance.

.. code-block:: python

   ghmc_move = GHMCMove(timestep=1.0*unit.femtosecond, n_steps=50)
   langevin_move = LangevinDynamicsMove(n_steps=10)
   sampler = MCMCSampler(thermodynamic_state, sampler_state, move=ghmc_move)

You can combine them to form a sequence of moves

.. code-block:: python

   sequence_move = SequenceMove([ghmc_move, langevin_move])
   sampler = MCMCSampler(thermodynamic_state, sampler_state, move=sequence_move)

or create a move that selects one of them at random with given probability
at each iteration.

.. code-block:: python

   weighted_move = WeightedMove([(ghmc_move, 0.5), (langevin_move, 0.5)])
   sampler = MCMCSampler(thermodynamic_state, sampler_state, move=weighted_move)

By default the MCMCMove use a global ContextCache that creates Context on the
fastest available OpenMM platform. You can configure the default platform to use
before starting the simulation

.. code-block:: python

   reference_platform = openmm.Platform.getPlatformByName('Reference')
   cache.global_context_cache.platform = reference_platform
   cache.global_context_cache.time_to_live = 10  # number of read/write operations

Minimize and run the simulation for few iterations.

.. code-block:: python

   sampler.minimize()
   sampler.run(n_iterations=2)

If you don't want to use a global cache, you can create local ones.

.. code-block:: python

   local_cache1 = cache.ContextCache(capacity=5, time_to_live=50)
   local_cache2 = cache.ContextCache(platform=reference_platform, capacity=1)
   sequence_move = SequenceMove([HMCMove(), LangevinDynamicsMove()],
                                context_cache=local_cache1)
   ghmc_move = GHMCMove(context_cache=local_cache2)

If you don't want to cache Context at all but create one every time, you can use
the DummyCache.

.. code-block:: python

   dummy_cache = cache.DummyContextCache(platform=reference_platform)
   ghmc_move = GHMCMove(context_cache=dummy_cache)

Module contents
---------------

MCMCMove
~~~~~~~~

Abstract base class for all Markov chain Monte Carlo moves.

- Purpose: Encapsulate an action that can be applied to a thermodynamic state and a
  sampler state to advance the Markov chain.
- Implementation note: Concrete moves must implement the apply method.
- Key method:
  - apply(thermodynamic_state, sampler_state): Apply the move, potentially modifying
    the thermodynamic state and/or the sampler state.

MCMCSampler
~~~~~~~~~~

Basic Markov chain Monte Carlo sampler.

- Parameters
  - thermodynamic_state: The initial openmmtools.states.ThermodynamicState
    representing the thermodynamic conditions.
  - sampler_state: The initial openmmtools.states.SamplerState
    representing the microscopic configuration.
  - move_set: A container of MCMCMove objects. If a list/tuple, all moves are run
    each iteration in the given sequence. If a dictionary, unnormalized weights are
    used to select moves (e.g. {move1: 0.3, move2: 0.5, move3: 0.9}).

- Attributes
  - thermodynamic_state: Current thermodynamic state.
  - sampler_state: Current sampler state.
  - move_set: Moves to attempt during MCMC runs. Supports sequence or weighted
    selections as described above.

- Typical usage
  - Construct MCMCSampler with an initial state and a move (or sequence/weighted
    moves), then call minimize() and run(n_iterations).

Move composition
~~~~~~~~~~~~~~~~

SequenceMove and WeightedMove allow combining multiple MCMCMove implementations
to form more complex sampling strategies:
- SequenceMove: apply moves in a fixed order each iteration.
- WeightedMove: randomly select among moves according to defined weights.

Platform and caching
~~~~~~~~~~~~~~~~~~~~

OpenMM Platform and caching configuration can be controlled via the
openmmtools.cache module. A global ContextCache can be configured or local
caches can be supplied to specific moves, enabling flexible performance tuning
or avoiding caching entirely with a DummyContextCache.
