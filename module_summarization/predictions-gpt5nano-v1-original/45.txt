Logging
=======

Overview
--------

This module provides a configurable logging subsystem for the application. It defines a custom log level named NOTICE (25) and a helper class that wires console, file, and syslog logging according to application configuration.

Constants
---------

LOG_NOTICE
  Numeric value for the custom NOTICE log level (25). Registers the level name "NOTICE". This level is intended to be visible in non-verbose mode and should not be used for warnings or errors.

NAME_TO_LEVEL
  Mapping from textual log level names to their numeric values. Includes common levels such as NOTSET, DEBUG, INFO, WARNING, ERROR, and CRITICAL, plus variants for NOTICE and related aliases (e.g., "LOG_NOTICE", "LOG NOTICE").

Classes
-------

Logging
  A helper class that configures the global logging for the application. Initialization requires an app object (with an event loop) and creates the following internal handlers as needed:

  - RootLogger: the global logger instance
  - ConsoleHandler: optional console stream handler
  - FileHandler: optional rotating file handler
  - SyslogHandler: optional syslog handler

  Behavior
  - If the root logger has no handlers, the module configures logging based on runtime conditions:
    - Console logging is enabled if stdout is a TTY or if the ASABFORCECONSOLE environment variable is not '0'.
  - File logging is configured if a file path is provided in the configuration.
    - Ensures the target directory exists.
    - Uses RotatingFileHandler with backup_count and backup_max_bytes from configuration.
    - Applies a StructuredDataFormatter using format, datefmt, and sd_id from configuration.
  - Log rotation can be scheduled via the rotate_every configuration key.
    - Accepts values like "<n><unit>" where unit is H, M, d, or s.
    - Schedules a Timer to trigger rotation checks at the specified interval.
  - Syslog logging is configured if enabled in configuration.
    - Supports UNIX domain sockets or network syslog (tcp/udp) and several address formats.
    - Applies a formatter based on logging:syslog:format:
      - 'm' -> MacOSXSyslogFormatter
      - '5' -> SyslogRFC5424Formatter
      - '5micro' -> SyslogRFC5424microFormatter
      - default -> SyslogRFC3164Formatter
    - Attaches the chosen syslog formatter to the syslog handler and registers it with the root logger.
  - The implementation relies on external components (Config, Timer, AsyncIOHandler, and various formatters) and is intended to be used with an application object that provides an event loop (app.Loop).

Public API
----------

- LOG_NOTICE: See above in Constants.
- NAME_TO_LEVEL: See above in Constants.
- Logging: Class to initialize and manage logging configuration for the application.

Configuration
-------------

The behavior is driven by the application configuration (Config). Key configuration entries include:

- logging:file
  - path: Path to the log file. If empty, file logging is disabled.
  - backup_count: Number of backup files to retain.
  - backup_max_bytes: Maximum size of a log file before rotation.
  - format: Format string passed to StructuredDataFormatter for file logs.
  - datefmt: Date format for file logs.
- logging:file/rotate_every: Optional rotation interval, e.g. "5M" or "1H".
- logging:sd_id: Source designator ID used by formatters.
- logging:syslog
  - enabled: Enable or disable syslog logging.
  - address: Syslog server address. UNIX paths (starting with /) or network addresses (URL-like, e.g., tcp://host:port, udp://host:port, unix-connect://path, unix-sendto://path).
  - format: Formatter type for syslog ('m', '5', '5micro', or default).
  - sd_id: Source designator ID used by syslog formatters.

Notes
-----

- The module expects an application object with an event loop accessible as app.Loop.
- The logging infrastructure depends on several components not defined in this snippet (StructuredDataFormatter, MacOSXSyslogFormatter, SyslogRFC5424Formatter, SyslogRFC5424microFormatter, SyslogRFC3164Formatter, AsyncIOHandler, Timer).
- Custom log level NOTICE allows more granular informational output in non-verbose modes.

Usage example
-------------

Instantiate the logging subsystem during application startup:

  logging = Logging(app)

This will configure the root logger and attach the appropriate handlers based on the current configuration.
