Publishing versions
===================

Overview
--------

This documentation explains how to publish version-related information using the ftrack API. When you create a Session with auto_connect_event_hub enabled, an embedded event hub is automatically connected to the event server, allowing you to publish and subscribe to non-local events. Subscribing to events does not require a connected event hub.

Configuring a session for publishing
-------------------------------------

To publish versions, you typically create a Session with the following configuration:

- server_url: URL of the ftrack server (including port if needed)
- api_key: your API key for authentication
- api_user: the user name in ftrack to publish under
- auto_connect_event_hub: enable automatic connection to the event hub for publishing

Notes:
- If server_url, api_key, or api_user are not provided, you can also supply them via environment variables FTRACK_SERVER, FTRACK_API_KEY, and FTRACK_API_USER, respectively.
- The event hub connection is performed in a background thread to improve startup time.

Example
-------

.. code-block:: python

    import ftrack_api

    session = ftrack_api.Session(
        server_url='https://ftrack.example.com',
        api_key='YOUR_API_KEY',
        api_user='your_username',
        auto_connect_event_hub=True  # enable publishing/subscribing to non-local events
    )

Publishing workflow
-------------------

- Create or obtain the objects whose versions you intend to publish.
- Ensure the event hub is connected (automatic if auto_connect_event_hub is True).
- Publish version-related events to the event server as appropriate for your workflow.
- Subscribe to relevant events if you need to respond to version changes or broadcasts from other clients.

Important considerations
------------------------

- Enabling auto_connect_event_hub provides the ability to publish non-local events, but subscribing to events is possible for local events without a connected hub.
- The session will include a layered cache to optimize performance; do not remove the top-level memory cache for typical usage.
- If a plugin or workflow relies on a connected event hub, verify the hub's connection status before attempting to publish or subscribe.

Troubleshooting
---------------

- If required credentials are missing, ensure server_url, api_key, and api_user are supplied or available via environment variables.
- If publishing does not appear to function, verify that auto_connect_event_hub is set to True and that the event hub successfully connects in the background.
- Check for exceptions related to the event hub or session initialization in the logs.
