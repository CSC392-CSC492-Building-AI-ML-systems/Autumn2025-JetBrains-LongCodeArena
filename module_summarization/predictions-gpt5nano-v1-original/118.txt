Configuration Options
=====================

This module provides the ConfigurationsOptions class, which controls which parts of a project are exported to or imported from JSON during configuration processing in PyAEDT. It also includes a helper function used to discover dataset references within nested configuration structures.

Overview
--------

- ConfigurationsOptions: Options class for the configurations. Users can enable or disable import/export of various components when generating or reading configuration data.

- _find_datasets: Internal helper to recursively gather dataset identifiers from nested dictionaries/lists in configuration data.

Class: ConfigurationsOptions
---------------------------

The ConfigurationsOptions class defines a set of flags that determine what is exported or imported when handling configurations. It also stores a tolerance value used for object mapping.

Initialization
--------------

When a ConfigurationsOptions object is created, the following defaults are applied:

- object_mapping_tolerance: 1e-9
- export_variables: True
- export_setups: True
- export_optimizations: True
- export_parametrics: True
- export_boundaries: True
- export_mesh_operations: True
- export_coordinate_systems: True
- export_materials: True
- export_object_properties: True
- export_datasets: True
- import_datasets: True
- import_variables: True
- import_setups: True
- import_optimizations: True
- import_parametrics: True
- import_boundaries: True
- import_mesh_operations: True
- import_coordinate_systems: True
- import_materials: True
- import_object_properties: True
- skip_import_if_exists: False

Public properties
-----------------

- object_mapping_tolerance (float)
  - Get/Set the tolerance value used in object mapping (e.g., for boundaries).
  - Default: 1e-9

  Examples
  --------
  >>> from pyaedt import Hfss
  >>> hfss = Hfss()
  >>> hfss.configurations.options.object_mapping_tolerance = 1e-8

- export_variables (bool)
  - Define if variables should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> from pyaedt import Hfss
  >>> hfss = Hfss()
  >>> hfss.configurations.options.export_variables = False  # Disable variable export

- export_setups (bool)
  - Define if setups should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.export_setups = False  # Disable setup export

- export_optimizations (bool)
  - Define if optimizations should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.export_optimizations = False  # Disable optimization export

- export_parametrics (bool)
  - Define if parametrics should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.export_parametrics = False  # Disable parametrics export

- export_boundaries (bool)
  - Define if boundaries should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.export_boundaries = False  # Disable boundary export

- import_datasets (bool)
  - Define if datasets should be imported from the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.import_datasets = False  # Disable dataset import

- export_datasets (bool)
  - Define if datasets should be exported to the JSON file. Default: True.

  Examples
  --------
  >>> hfss.configurations.options.export_datasets = False  # Disable dataset export

- skip_import_if_exists (bool) [internal/private]
  - Internal flag indicating whether to skip import when the target already exists. This is initialized but not exposed as a public property in this snippet.

Internal helper
---------------

- _find_datasets(d, out_list)
  - Purpose: Recursively scan a nested dictionary (and mixed structures) to collect dataset identifiers used in free-form value strings. It looks for entries containing "pwl" and extracts the dataset token found between the first "$" and the first ",".
  - Parameters:
    - d: input dictionary (potentially nested)
    - out_list: list to accumulate discovered dataset identifiers
  - Returns: None (populates out_list in place)

Usage example
-------------

- Create a configuration options object and tweak exports as needed:
  >>> from pyaedt import Hfss
  >>> hfss = Hfss()
  >>> opts = hfss.configurations.options
  >>> opts.export_variables = False
  >>> opts.export_setups = True
  >>> opts.import_datasets = True

Notes
-----

- The defaults favor exporting and importing most configuration components to preserve full project data.
- The _find_datasets function is intended for internal use within the configuration handling logic and is not part of the public API.
