Probability calibration
=======================

CalibratedClassifierCV provides probability calibration for a classifier using either isotonic regression or Platt’s sigmoid method (logistic regression). The calibration is performed in conjunction with cross-validation: for each cross-validation split, a copy of the base estimator is trained on the training subset and calibrated on the testing subset. Predictions from the calibrated estimators can be averaged across folds to produce final calibrated probabilities.

Estimator
---------
estimator : estimator instance, default=None
    The classifier whose output needs to be calibrated to provide more accurate
    predict_proba outputs. The default classifier is a LinearSVC. This parameter
    was added in version 1.2.

method
------
method : {'sigmoid', 'isotonic'}, default='sigmoid'
    The calibration method to use:
    - 'sigmoid' corresponds to Platt’s method (a logistic regression model)
    - 'isotonic' is a non-parametric approach.
    Isotonic calibration can overfit when calibration samples are too few
    (<< 1000).

cv
---
cv : int, cross-validation generator, iterable or "prefit", default=None
    Determines the cross-validation splitting strategy.
    - None: use a default 5-fold cross-validation
    - int: number of folds
    - CV splitter or an iterable yielding (train, test) splits
    - "prefit": assumes estimator has been fitted already; all data is used for calibration

    If integer/None, StratifiedKFold is used for binary or multiclass targets;
    otherwise KFold is used. See the cross-validation user guide for details.

    When "prefit" is passed, no cross-validation is performed and all data is
    used for calibration.

    versionchanged:: 0.22
        Default cv changed from 3-fold to 5-fold when None.

n_jobs
------
n_jobs : int, default=None
    Number of jobs to run in parallel. None means 1 unless inside a
    joblib.parallel_backend context. -1 uses all processors.

ensemble
--------
ensemble : bool, default=True
    Determines how the calibrator is fitted when cv is not 'prefit'. If True,
    the estimator is fitted on training data and calibrated on testing data
    for each cross-validation fold, and the final prediction is the average of
    the calibrated probabilities from all folds. If False, cross_val_predict is
    used to obtain unbiased predictions for calibration, and the final estimator
    is the estimator trained on all the data. This option mirrors behavior in
    some SVM implementations with probabilities.

base_estimator
--------------
base_estimator : estimator instance
    Deprecated. Use estimator instead. This parameter was deprecated in version
    1.2 and will be removed in 1.4.

Attributes
----------
classes_ : ndarray of shape (n_classes,)
    The class labels.

n_features_in_ : int
    Number of features seen during fit. Defined if the underlying estimator
    exposes such an attribute when fit.

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features seen during fit. Defined if the underlying estimator exposes
    an attribute when fit.

calibrated_classifiers_ : list
    List of calibrated classifiers. Length is equal to the number of cross-validation
    folds, or 1 if cv="prefit" or ensemble=False.

Notes
-----
- The calibration uses the model's decision_function if available; otherwise it uses
  predict_proba.
- For detailed usage and advanced options, refer to the User Guide section on
  calibration (calibration) and cross-validation.
