Instrumentation events (SQLAlchemy ORM)

Overview
This module defines SQLAlchemy's system of class instrumentation. It is usually not directly visible to user applications, but it plays a central role in registering end-user classes for state tracking. It interacts closely with state.py and attributes.py, which implement per-instance and per-attribute instrumentation, respectively. The system can be customized on a per-class or global basis via the sqlalchemy.ext.instrumentation package, which provides alternative instrumentation forms.

Notes
- Version note: The instrumentation extension system was moved out of the ORM into the external sqlalchemy.ext.instrumentation package (versionchanged: 0.8).

Public API and core concepts
- ClassManager
  - Tracks state information at the class level for a given mapped class.
  - Inherits from HasMemoized, Dict[str, QueryableAttribute[Any]], Generic[_O], and EventTarget.
  - Core attributes:
    - dispatch: event target (dispatcher) for InstanceEvents related to this class manager.
    - MANAGER_ATTR, STATE_ATTR: default attribute names used for instrumentation.
    - _state_setter: helper to set state attributes.
    - expired_attribute_loader: protocol for loading expired attributes (formerly named deferred_scalar_loader).
    - init_method, original_init: customization hooks for class initialization instrumentation.
    - factory: optional manager factory used to create managers for related classes.
    - declarative_scan: optional weak reference to a MapperConfig for declarative scans.
    - registry: registry of managers; populated during instrumentation setup.
    - class_: the Python class being instrumented.
    - _bases: list of base class Managers for base classes that participate in instrumentation.
  - Deprecations
    - deferred_scalar_loader is deprecated in favor of expired_attribute_loader (since 1.4); the property forwards to the new name with a deprecation warning.
  - Behavior
    - __init__(self, class_): initializes internal state, collects base managers, updates its state from bases, notifies InstanceEvents, and performs initial instrumentation setup. Warns if __del__ is defined on the class (as it can create unwanted reference cycles).
    - _update_state(...): updates internal state when binding to a mapper, registry, declarative scan, or an expired attribute loader; can also set an initialization method.
    - _finalize(): marks the manager as finalized, triggers instrumentation of the class __init__, and finalizes the instrumentation setup (details depend on the rest of the module).

- _ExpiredAttributeLoaderProto (Protocol)
  - Defines the callable used to load attributes that have expired, given a state, a set of attributes to load, and a passive flag.

- _ManagerFactory (Protocol)
  - Defines the callable that, given a class, returns a ClassManager for that class.

- DEL_ATTR
  - A memoized symbol used internally by the instrumentation system to denote deletion of attributes.

- TYPE_CHECKING imports
  - The module references several types for typing checks without importing them at runtime.

Interaction and workflow
- The ClassManager ties a Python class to the ORM’s state and attribute instrumentation by constructing per-class state and collaborating with base class managers, the mapper, and the registry.
- It notifies InstanceEvents via its dispatch when new class-level instrumentation is established.
- Finalization triggers the enabling of instrumentation hooks for the class’s __init__ and related behavior, integrating the class into the ORM’s state-tracking system.

This module is part of SQLAlchemy’s internal machinery for ORM class instrumentation and is typically used indirectly through higher-level ORM configuration and the instrumentation extension.
