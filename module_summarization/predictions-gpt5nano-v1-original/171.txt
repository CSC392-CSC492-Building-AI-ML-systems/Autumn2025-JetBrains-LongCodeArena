Calculator Interface
====================

Overview
--------

The Calculator Interface defines how model calculations are performed within the framework. A calculator is responsible for computing the model intensity I(q) for a given q-range and a set of model parameters. Calculators should be fast, support vectorized inputs, and return deterministic results for a given input.

Public API
----------

- CalculatorBase (abstract)
  - name: str
    The human-friendly name of the calculator.
  - description: str
    Short description of the calculator’s purpose and capabilities.
  - compute(self, q, params, **options) -> numpy.ndarray
    Compute the model intensity I(q) for the provided q-values and parameter set.
    - q: array_like
      Scattering vector magnitudes (1/Angstrom).
    - params: dict[str, float]
      Mapping of parameter names to numeric values.
    - options: dict
      Optional keyword arguments (e.g., scale, background, solvent contrast, polydispersity settings).
  - required_params: list[str]
    Names of parameters that must be provided for computation.
  - param_types: dict[str, str]
    Optional classification of parameters by kind. Values are:
    - 'volume'
    - 'orientation'
    - 'sld'
    - 'none'
    - 'any'
    This allows filtering parameters by their physical meaning when assembling calculators.
  - supports_q_matrix (optional): bool
    Indicates whether the calculator can consume a matrix of q-values efficiently (vectorized work).

Model compatibility
-------------------

Calculators rely on a model’s parameter metadata to determine which parameters are required and how they should be interpreted. Each parameter may have a type/kind (volume, orientation, sld, none, any) which can be used to select appropriate calculators or to validate input.

Output
------

- compute(...) returns a numpy array with the same shape as the input q, representing I(q).
- Additional optional outputs (if supported by a calculator) may be provided via the options dict or a separate return structure.

Usage
-----

- Create or select a calculator that matches your model and data needs.
- Provide a parameter dictionary with all required parameter values.
- Call compute with a q-array to obtain the model intensity.

Parameter handling
------------------

- Parameter names correspond to model parameters.
- Each parameter may carry a kind/type, informing how it should be treated in filtering and UI.
- Common kinds include:
  - volume: shape/size-related
  - orientation: orientation distribution or alignment
  - sld: scattering length density
  - none: not exposed to the user
  - any: used when the parameter is not constrained to a specific kind

Extending the interface
-----------------------

- Subclass CalculatorBase and implement compute(...)
- Register your calculator with the framework’s calculator registry
- Provide descriptive name and parameter metadata (required_params, param_types)
