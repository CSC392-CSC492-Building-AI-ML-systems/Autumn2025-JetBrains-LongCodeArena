Feature extraction utilities
============================

.. currentmodule:: torchvision.feature_extraction

Overview
--------

This module provides two public APIs for creating feature extractors from PyTorch models:

- create_feature_extractor: wrap a model to return intermediate feature maps
  for user-specified graph nodes.
- get_graph_node_names: obtain the list of node names available for feature extraction
  from a given model.

The implementation uses an FX-based tracer (NodePathTracer) that records a per-node
qualname mapping and supports treating certain modules as leaves via a LeafModuleAwareTracer.
This enables you to specify exactly which intermediate activations to extract during a
forward pass.

Public API
----------

- create_feature_extractor(model, return_nodes)
  - Returns an nn.Module that, when called with an input, yields a dictionary of
    feature tensors corresponding to the keys provided in return_nodes.
  - return_nodes: a mapping from graph node name(s) to desired output names. For example:
    {"layer4": "feat_layer4", "avgpool": "feat_pool"}.
  - The resulting module preserves the original model’s behavior for all other parts of the graph.

- get_graph_node_names(model)
  - Returns a list of available graph node names that can be used in return_nodes.
  - Useful for discovering which intermediate nodes can be used for feature extraction.

Usage example
-------------

Basic usage with a pretrained model:

.. code-block:: python

   import torch
   from torchvision.models import resnet18
   from torchvision.feature_extraction import create_feature_extractor, get_graph_node_names

   # Build a model
   model = resnet18(pretrained=False)

   # Discover available node names for feature extraction
   node_names = get_graph_node_names(model)
   # Example node names might include: "layer4", "avgpool", etc.

   # Define which nodes to extract and what to call them in the output
   return_nodes = {"layer4": "feat_layer4", "avgpool": "feat_pool"}

   # Create a feature extractor
   feature_extractor = create_feature_extractor(model, return_nodes=return_nodes)

   # Run a forward pass
   x = torch.randn(1, 3, 224, 224)
   features = feature_extractor(x)

   # Features is a dict: {"feat_layer4": Tensor, "feat_pool": Tensor}
   for name, feat in features.items():
       print(name, feat.shape)

Advanced usage
--------------

- Leaf modules: Some modules can be treated as leaves so that their internal operations are not traced. You can influence tracing behavior via tracer configuration used inside create_feature_extractor (e.g., by marking specific modules as leaves).

Notes
-----

- Node names depend on the model graph and may differ between training and evaluation graphs. If there are differences between train and eval graphs, you may need to specify separate output nodes for each mode.

Example with a custom module
----------------------------

You can apply the same pattern to any PyTorch model, including custom architectures, by specifying the desired intermediate nodes to extract.
