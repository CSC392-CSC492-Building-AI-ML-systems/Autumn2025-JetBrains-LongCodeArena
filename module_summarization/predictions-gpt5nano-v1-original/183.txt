Feature selection and dimensionality reduction on sample sets
============================================================

Overview
--------
Feature selection and dimensionality reduction are techniques for simplifying data by reducing the number of features (columns) observed across samples (rows).

- Feature selection: chose a subset of the original features that are most useful for the task (classification, regression, etc.). The feature space remains interpretable because you work with real features from the data.
- Dimensionality reduction: transform the data to a new, lower-dimensional feature space. The resulting features are usually combinations or nonlinear mappings of the original features and are often less interpretable.

Typical use
------------
- Improve model performance and generalization
- Reduce training time and memory usage
- Alleviate multicollinearity and overfitting
- Visualize high-dimensional data in 2D or 3D

Feature selection
-----------------

Unsupervised vs supervised
- Unsupervised: selects features without using the target variable (e.g., VarianceThreshold).
- Supervised: uses the target variable to select features that are predictive (e.g., SelectKBest with f_classif, mutual_info_classif, etc.).

Common methods
- Variance-based:
  - VarianceThreshold: remove features with low variance
- Univariate statistical tests:
  - SelectKBest, SelectPercentile
  - f_classif, f_regression (ANOVA F-value)
  - chi2 (chi-squared test for non-negative features)
  - mutual_info_classif, mutual_info_regression (mutual information)
- Model-based and embedded methods:
  - SelectFromModel: select features based on model attributes (e.g., coefficients, feature importances)
  - Recursive Feature Elimination (RFE) and RFECV: recursively remove least-important features
- Other:
  - Mutual information-based selectors for nonlinear associations

Example (SelectKBest and VarianceThreshold)
.. code-block:: python

   from sklearn.feature_selection import VarianceThreshold, SelectKBest
   from sklearn.feature_selection import f_classif
   from sklearn.pipeline import Pipeline

   # Remove low-variance features, then select top k features by ANOVA F-value
   pipeline = Pipeline([
       ('var_thresh', VarianceThreshold()),
       ('select_k', SelectKBest(score_func=f_classif, k=20))
   ])
   X_reduced = pipeline.fit_transform(X, y)

Dimensionality reduction
------------------------

Linear methods
- Principal Component Analysis (PCA): linearly projects data to maximize variance
- IncrementalPCA: memory-efficient online PCA for large datasets
- TruncatedSVD: suitable for sparse data (e.g., text data)
- Independent Component Analysis (ICA): seeks statistically independent components
- FactorAnalysis: probabilistic factor analysis

Nonlinear and manifold methods
- Isomap, LocallyLinearEmbedding (LLE), SpectralEmbedding: preserve local/neighborhood structure
- t-Distributed Stochastic Neighbor Embedding (t-SNE): good for visualization of high-dimensional data
- Other nonlinear methods may be useful for uncovering complex structure

Notes
- Dimensionality reduction emphasizes capturing structure in the data, not feature interpretability.
- For sparse, high-dimensional data (e.g., text), TruncatedSVD is often preferred over PCA.

Example (PCA)
.. code-block:: python

   from sklearn.decomposition import PCA
   pca = PCA(n_components=2)
   X_pca = pca.fit_transform(X)

Example (TruncatedSVD for sparse data)
.. code-block:: python

   from sklearn.decomposition import TruncatedSVD
   svd = TruncatedSVD(n_components=50)
   X_svd = svd.fit_transform(X_sparse)

Choosing a method
- If interpretability of features is important, prefer feature selection (retain original features).
- If visualization or computational efficiency is the goal, dimensionality reduction (learned components) can be more effective.
- For text data or very high-dimensional sparse data, use TruncatedSVD or similar sparse-friendly methods.
- Use cross-validation to assess how reduction or selection affects model performance.
