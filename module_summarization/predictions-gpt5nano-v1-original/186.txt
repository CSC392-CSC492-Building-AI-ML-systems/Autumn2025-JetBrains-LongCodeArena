Imputation of missing values
==============================

Imputers fill in missing values in arrays or dataframes using a variety of strategies. The imputation process is implemented in the BaseImputer and concrete implementations such as SimpleImputer.

Overview
--------

- Imputers replace missing values along each feature (column) using a chosen strategy.
- Optional indicator features can be added to flag which values were imputed.
- Supports numeric and, for some strategies, non-numeric data.
- Integration with scikit-learn’s fit/transform API.

Base functionality
------------------

BaseImputer provides common functionality shared by imputers:

- missing_values: the placeholder for missing values. Accepts numeric NaN, None, pandas.NA, or other user-specified placeholders.
- add_indicator: if True, a MissingIndicator is fitted and transformed, and its results are concatenated to the imputed data as additional features.
- keep_empty_features: controls behavior when a feature (column) is entirely empty.
- _fit_indicator, _transform_indicator: helper methods to manage the optional missing-value indicator.
- _concatenate_indicator: utility to concatenate the imputed data with the indicator (if enabled).
- _concatenate_indicator_feature_names_out: updates feature names when an indicator is added.
- _more_tags: reports capability such as allow_nan.

The implementation supports using a MissingIndicator when add_indicator is True, and automatically handles feature-name augmentation when indicators are added.

SimpleImputer
-------------

The SimpleImputer is a univariate imputer that completes missing values with a simple strategy along each column.

Purpose
- Replace missing values with a descriptive statistic or a fixed constant.

Supported strategies
- mean: Replace missing values using the mean along each column. Numeric data only.
- median: Replace missing values using the median along each column. Numeric data only.
- most_frequent: Replace missing values with the most frequent value along each column. Works for strings or numeric data. If multiple values tie for most frequent, the smallest value is returned.
- constant: Replace missing values with a fixed value given by fill_value. Works with strings or numeric data.

Parameters
- missing_values: The placeholder for the missing values. Default: np.nan. For pandas’ nullable dtypes, missing_values can be np.nan or pd.NA.
- strategy: {'mean', 'median', 'most_frequent', 'constant'}, default 'mean'.
- fill_value: Used to replace missing values when strategy='constant'. If None, fill_value defaults to 0 for numeric data and "missing_value" for strings/object data types.
- copy: bool, default True. If True, a copy of X is created; if False, imputation may be done in place when possible.
- add_indicator, keep_empty_features: Inherited from BaseImputer. When add_indicator is True, an additional binary indicator feature is added per input feature indicating whether the original value was missing.

Additional notes
- Input validation: _check_inputs_dtype ensures that X is numeric where appropriate, and that missing_values type is compatible. If missing_values is a pandas NA, numerical checks may be bypassed to allow pd.NA as a missing value placeholder.
- _most_frequent helper: used to compute the most frequent value with careful tie-breaking to mirror scipy.stats.mode behavior. Handles both numeric and object (string) data efficiently.
- The SimpleImputer’s description references the user guide for more details:
  Read more in the User Guide imputation: :ref:`User Guide <impute>`.

Usage example
- Basic mean imputation:
  from sklearn.impute import SimpleImputer
  imputer = SimpleImputer(strategy='mean')
  X_imputed = imputer.fit_transform(X)

- Add missing-value indicator:
  imputer = SimpleImputer(strategy='mean', add_indicator=True)
  X_imputed, indicator = imputer.fit_transform(X)
