Nearest Neighbors
===================

Overview
--------
The Nearest Neighbors functionality uses a Ball-Tree based data structure to efficiently find
the k nearest neighbors of a query point under a variety of distance metrics. The implementation
is specialized for different floating point precisions (BallTree64 and BallTree32) and can be
compiled from the BinaryTree base class into BallTree variants. Queries exploit the concept of
reduced distance to prune the search space and accelerate neighbor lookups.

Algorithms and Data Structures
------------------------------
- BallTree (and specialized BallTree64/BallTree32) are built from BinaryTree, storing:
  - data: the training samples
  - idx_array: indices into the data
  - node_bounds: bounding boxes for tree nodes
  - node_data: per-node data such as radius, start/end indices
- The tree uses reduced distance for pruning where applicable (e.g., squared Euclidean for Euclidean),
  while still returning true distances to the user.
- Node radius and centroid calculations are used to prune branches that cannot contain closer neighbors.

Supported Distance Metrics
--------------------------
The implementation supports a variety of distance metrics. Internal metric classes are named with
suffixes for the 32-bit/64-bit precision variants, but public usage exposes standard metric names.
Commonly supported metrics include (list not exhaustive):
- Bray-Curtis Distance
- Canberra Distance
- Chebyshev Distance
- Dice Distance
- Euclidean Distance
- Hamming Distance
- Haversine Distance
- Jaccard Distance
- Mahalanobis Distance
- Manhattan Distance
- Minkowski Distance
- PyFunc Distance
- Rogers-Tanimoto Distance
- Russell-Rao Distance
- Sokal-Michener Distance
- Sokal-Sneath Distance
- Weighted Minkowski Distance (WMinkowski)

Note: The exact internal class names include numeric suffixes (e.g., EuclideanDistance64, EuclideanDistance32) to reflect
the corresponding BallTree precision variants.

Usage
-----
The public API exposes the NearestNeighbors interface (and the BallTree variants) to perform neighbor
searches. Typical usage:

- Instantiate with a dataset and a chosen algorithm/metric:
  - algorithm='ball_tree' (or rely on auto-selection)
  - metric can be a standard distance name (e.g., 'euclidean', 'manhattan', 'minkowski', etc.)

- Fit on training data:
  - nn = NearestNeighbors(n_neighbors=5, algorithm='ball_tree', metric='euclidean')
  - nn.fit(X)

- Query for neighbors:
  - distances, indices = nn.kneighbors(X_query)
  - Or use radius_neighbors for all points within a given radius

Notes
-----
- The BallTree implementation is designed for efficiency with large datasets and supports 32-bit and 64-bit
  floating point precisions via BallTree32/BallTree64.
- Reduced distance is used to prune branches during search while ensuring correct true distances are returned.

Example
-------
.. code-block:: python

   from sklearn.neighbors import NearestNeighbors
   # Euclidean distance with BallTree
   nn = NearestNeighbors(n_neighbors=5, algorithm='ball_tree', metric='euclidean')
   nn.fit(X_train)
   dist, idx = nn.kneighbors(X_test)

   # Radius-based search
   indices = nn.radius_neighbors(X_query, radius=0.5, return_distance=False)

Implementation Notes
--------------------
- Internally, BallTree derives from BinaryTree and specializes node data for efficient nearest-neighbor queries.
- Data types and precision-specific implementations are selected via BallTree64 and BallTree32.
- The internal routines allocate per-node data (bounds, radius, and index ranges) and compute centroids and radii
  to enable pruning during search.

For detailed API and supported metrics, see the Nearest Neighbors documentation in the reference guide.
