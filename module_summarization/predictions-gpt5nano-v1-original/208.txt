Utilities for Developers
=========================

These utilities provide compatibility support for older versions of Python, NumPy, SciPy, and related packages. They help maintain a consistent internal API across a range of environments and handle behavior changes introduced in newer releases.

Overview
--------
- The module collects small helpers and compatibility shims used across the project.
- It includes safeguards for sparse matrix containers, thread-pool management, numerical helpers, and resource access.
- Some utilities are intended for internal use only; public-facing behavior may evolve and deprecated paths may be removed in future versions.

Thread-pool management
----------------------
- threadpool_limits(limits=None, user_api=None)
  - Sets thread pool limits if a global thread-pool controller is available; otherwise falls back to threadpoolctl’s default behavior.
- threadpool_info()
  - Returns information about the active thread pool configuration.
- Implementation notes:
  - For threadpoolctl >= 3.0.0, a global ThreadpoolController may be created on first use to avoid scanning all loaded libraries repeatedly.
  - A lightweight shim is provided to maintain a consistent API across environments.

Percentile compatibility
------------------------
- percentile (NumPy compatibility)
  - NumPy < 1.22 uses a local _percentile wrapper that translates the older keyword usage.
  - NumPy >= 1.22 exposes numpy.percentile directly. The module aliases percentile to the NumPy function in this case.

Object dtype NaN detection
------------------------
- _object_dtype_isnan(X)
  - A simple NaN check that works for object-dtype arrays: X != X.

Mode compatibility
------------------
- _mode(a, axis=0)
  - Wrapper around scipy.stats.mode that handles differences across SciPy versions.
  - Uses keepdims where available and adjusts results for changes in shape when SciPy’s implementation evolves.

Sparse linear algebra compatibility
-----------------------------------
- _sparse_linalg_cg(A, b, **kwargs)
  - Normalizes keyword arguments to accommodate changes in SciPy versions:
    - Maps rtol to tol if provided.
    - Sets a default atol for legacy behavior when not supplied.
  - Provides a consistent CG solver interface across SciPy versions.

Sparse matrix containers
------------------------
- CSR_CONTAINERS, CSC_CONTAINERS, COO_CONTAINERS, LIL_CONTAINERS, DOK_CONTAINERS, BSR_CONTAINERS
  - Lists of sparse matrix container types supported by the code base.
  - SciPy 1.8 introduced sparse array variants (csr_array, csc_array, etc.); these are appended to the corresponding containers if available.

Compatibility wrappers for importlib.resources (Python 3.9 backport)
------------------------------------------------------------------
- _open_text(data_module, data_file_name)
- _open_binary(data_module, data_file_name)
- _read_text(descr_module, descr_file_name)
- _path(data_module, data_file_name)
  - Provide Python 3.9’s importlib.resources API in environments running older Python versions.
  - Use resources.files(...).joinpath(...).open/read_text() when available; otherwise fall back to the older resources API.

Resource access helpers
-----------------------
- _read_text(descr_module, descr_file_name)
  - Reads a text resource in a version-agnostic way.
- _path(data_module, data_file_name)
  - Returns a context-managed path-like object for a resource when supported.

Deprecated API compatibility
----------------------------
- delayed(function)
  - Backwards-compatible wrapper that imports delayed from sklearn.utils.parallel.
  - Decorated with a deprecation notice indicating the move to sklearn.utils.parallel and future removal in 1.5+.

Notes on versions and deprecations
----------------------------------
- Several utilities include version gates to remain compatible with older environments while allowing newer features when available.
- The deprecation notice for delayed signals the planned removal of an old import path and encourages updating import locations.

Public API surface (summary)
----------------------------
- threadpool_limits, threadpool_info
- percentile (alias to numpy.percentile on NumPy >= 1.22)
- _object_dtype_isnan
- _mode
- _sparse_linalg_cg
- _open_text, _open_binary, _read_text, _path
- CSR_CONTAINERS, CSC_CONTAINERS, COO_CONTAINERS, LIL_CONTAINERS, DOK_CONTAINERS, BSR_CONTAINERS
- delayed (deprecated compatibility wrapper)

Usage examples
--------------
- Managing thread pools:
  - with threadpool_limits(limits=2): ... perform computation ...
- Reading a resource with Python 3.9+ API:
  - with _open_text(my_package, "data.txt") as f: contents = f.read()
- Safe percentile across NumPy versions:
  - p = percentile(data, 50)
- CG solver compatibility:
  - x = _sparse_linalg_cg(A, b, rtol=1e-5)
