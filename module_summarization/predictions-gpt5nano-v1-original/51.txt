================================

Overview
--------
The one-parameter exponential families form the foundation for generalized linear models (GLMs) and related estimating equations (GEE) approaches. This module defines the base class Family, which encapsulates a link function and a variance function for a single-parameter exponential family. Concrete families (e.g., Poisson, Binomial) are typically implemented by subclassing Family and providing a distribution-specific implementation of the deviance residual calculation.

Example
-------
The following is a self-contained illustration of using the base family interface by defining a minimal Poisson-like family and performing a few common operations.

Python
------
from __future__ import print_function

import numpy as np

# Import the base class and supporting components from the package
# (adjust the imports to match your package layout)
from gee.families import Family
from gee.families import links as L
from gee.families import varfuncs as V

FLOAT_EPS = np.finfo(float).eps

class Poisson(Family):
    """
    Minimal Poisson-like family with a log link and Poisson variance.
    This is a simple illustrative subclass for documentation purposes.
    """
    def __init__(self, link=None):
        if link is None:
            link = L.Log()  # canonical log link
        # variance function for Poisson: Var(mu) = mu
        variance = getattr(V, 'poisson', lambda mu: mu)
        super(Poisson, self).__init__(link, variance)

    def _resid_dev(self, endog, mu):
        """
        Deviance contributions D_i for Poisson:
        D_i = 2 * (y_i * log(y_i / mu_i) - (y_i - mu_i))
        """
        y = np.asarray(endog, dtype=float)
        m = np.asarray(mu, dtype=float)
        m = np.maximum(m, FLOAT_EPS)
        y = np.maximum(y, 0.0)
        # Use a safe log to handle y == 0
        with np.errstate(divide='ignore', invalid='ignore'):
            term = y * np.log(np.maximum(y / m, FLOAT_EPS))
        return 2.0 * (term - (y - m))

# Example data
y = np.array([2.0, 0.0, 3.0, 1.0, 5.0])

# Instantiate a Poisson-like family
poisson = Poisson()

# Starting value for mu (IRLS)
mu0 = poisson.starting_mu(y)

# Weights for IRLS steps
w = poisson.weights(mu0)

# Deviance for the current mu
D = poisson.deviance(endog=y, mu=mu0)

# Deviance residuals
resid_dev = poisson.resid_dev(endog=y, mu=mu0)

# Fitted values from a linear predictor (for Poisson with log link)
lin_pred = np.log(np.maximum(mu0, FLOAT_EPS))
mu_fitted = poisson.fitted(lin_pred)

print("mu0:", mu0)
print("weights:", w)
print("deviance D:", D)
print("deviance residuals:", resid_dev)
print("fitted mu from lin_pred:", mu_fitted)

Notes
-----
- The Poisson example above defines a minimal subclass to illustrate usage of the base class interface. In a full implementation, you would provide complete variance, deviance, and possibly additional diagnostic helpers for each specific distribution.

This example demonstrates how to:
- Instantiate a concrete Family with a link and a variance function.
- Compute starting values for the IRLS algorithm.
- Compute observation weights for IRLS.
- Evaluate the total deviance and individual deviance residuals.
- Recover fitted values from a linear predictor via the inverse link.
