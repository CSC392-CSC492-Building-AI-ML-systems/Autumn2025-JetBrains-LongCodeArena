Encoding media

Introduction
Media assets uploaded to Ftrack can be encoded into multiple formats and resolutions to support streaming, distribution, and archival workflows. This document describes the high-level concepts and workflows for encoding media within Ftrack.

Overview of the encoding workflow
- Prepare the source media: Attach or upload the original media file to an asset version or media item.
- Create an encoding request: Specify the target formats and resolutions (presets) you need for downstream usage (e.g., streaming, archival, or review).
- Server-side processing: The encoding request is processed asynchronously by server-side encoding plugins. Encoding may run in the background and can be triggered by events on the server.
- Access encoded artifacts: Once encoding completes, the generated artifacts are associated with the original asset and can be retrieved via their dedicated URLs or asset references.
- Monitor status: Encoding jobs expose status so you can track progress, handle failures, and react accordingly in your workflows.

Presets and formats
- Video formats commonly used for distribution:
  - MP4 (H.264) at multiple resolutions (e.g., 1080p, 720p, 480p)
  - WebM (VP9 or AV1) for web playback
- Audio formats:
  - MP3, AAC
- Image formats (for previews and thumbnails):
  - JPEG, PNG, WebP at various sizes

Managing encoding in workflows
- Triggering encodes: Initiate encoding as part of asset preparation or as a separate post-upload step, depending on your pipeline needs.
- Versioning: Encoded outputs are linked to the source asset version and are accessible as derived artifacts.
- Quality control: Use the encoding presets to enforce consistent output quality across your projects.
- Automation: Encode requests can be integrated with automation tools and event-driven plugins to fit into CI/CD-like media pipelines.

Operational notes
- Encoding is typically asynchronous; plan for processing time in your workflow.
- Server plugins and presets define which formats and resolutions are produced. See your server/plugin configuration to customize outputs.
- Permissions: Ensure the user or service account performing encoding has the necessary permissions to read source assets and create encoded artifacts.
- Logging and debugging: Enable verbose logging for encoding-related plugins to diagnose failures or performance issues.

Example usage (high level)
- Attach source media to an asset version.
- Submit an encoding request for target presets (e.g., 1080p and 720p MP4).
- Monitor the encoding job until completion.
- Retrieve and publish the resulting encoded artifacts for consumption in your downstream tasks.

Notes
- Encoding configurations are managed on the server side via plugins and presets. To customize outputs, adjust server plugin settings rather than client-side code.
- For non-local encoding events, ensure the event hub is connected and the encoding plugins are loaded in your environment.

This documentation covers the general concept of Encoding media within Ftrack workflows. For project-specific configurations, refer to your server-side plugin documentation and environment setup.
