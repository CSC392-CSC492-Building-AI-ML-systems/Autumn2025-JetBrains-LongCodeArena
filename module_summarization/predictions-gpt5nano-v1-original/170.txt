Handling Large Datasets

Overview
- This module provides multiple strategies for computing moment maps from large spectral cubes in a memory-friendly way.
- Available strategies: slice-wise, ray-wise, cubewise, plus an automatic selector that picks a strategy based on the cube and axis.
- Moments supported: order 0 (integrated intensity), order 1 (centroid), and higher orders (centered moments).

Key concepts
- A moment map is computed along a specified axis of a 3D cube, producing a 2D map with shape equal to the cube shape with the chosen axis removed.
- The computation relies on the cube object (typically a SpectralCube) providing methods and attributes such as:
  - _get_filled_data(fill, view): retrieve data for a given 2D slice, filling missing values with fill.
  - _pix_size_slice(axis): physical size of a pixel along the given axis.
  - _pix_cen(): pixel centers; used to compute moments.
  - _iter_rays(axis): iterator over rays along the given axis for ray-wise processing.
  - flattened(slc): obtain a flattened data view for a specific slice.
  - _mask, _wcs, and _wcs_tolerance: mask and world-coordinate system information used to select valid data.
- The allbadtonan utility is used to convert NaN sums to a well-defined representation for order 0.

Moment computation strategies
- Slice-wise (moment_slicewise)
  - Processes data one slice at a time along the target axis.
  - Lower memory footprint than cubewise, suitable when there is limited RAM.
  - Implementations include:
    - _slice0: 0th moment (integrated intensity) computed slice-by-slice with NaN handling.
    - _slice1: 1st moment (centroid) computed slice-by-slice with centroid weighting.
    - General order: uses precomputed moments to accumulate higher-order moments slice-by-slice.
- Ray-wise (moment_raywise)
  - Processes one ray (a line of sight) at a time.
  - Efficient for very large cubes when slices are still too large to hold in memory.
  - Handles 0th, 1st, and higher orders by accumulating along each ray with appropriate weighting.
- Cubewise (moment_cubewise)
  - Loads the entire data along the specified axis into memory and computes moments in one go.
  - Fastest method but requires memory proportional to the full cube along the axis.
  - Supports 0th, 1st, and higher orders; includes broadcasting logic to align shapes for higher-order calculations.
- Automatic selection (moment_auto)
  - Chooses among cubewise, raywise, or slice-wise based on iterator_strategy(cube, axis).
  - Returns the result of the selected strategy.
  - Ready-to-use when you want a balance between speed and memory without manual tuning.

Usage notes
- Use moment_auto(cube, order, axis) to let the system pick a strategy automatically.
- For extreme data sizes or limited memory, prefer slice-wise or ray-wise strategies.
- For small to moderately large cubes with ample memory, cubewise can be the fastest option.
- The order parameter supports 0, 1, and higher integers, enabling the computation of standard moments and higher-order statistics.

API overview (functions of interest)
- moment_slicewise(cube, order, axis): slice-wise moment computation.
- moment_raywise(cube, order, axis): ray-wise moment computation.
- moment_cubewise(cube, order, axis): cubewise moment computation.
- moment_auto(cube, order, axis): automatic strategy selection.
- _moment_shp(cube, axis): helper to determine the moment map shape.
- _slice0(cube, axis): 0th moment, slice-wise.
- _slice1(cube, axis): 1st moment, slice-wise.

Performance and memory considerations
- Slice-wise: moderate memory, good for large datasets where only slices fit in memory.
- Ray-wise: low to moderate memory, but potentially more I/O if many reads are required.
- Cubewise: maximal speed, but requires memory to hold the relevant data.
- Higher-order moments increase computational load; precomputing the first moment (for order > 1) can improve performance in the slice-wise implementation.

Notes
- The implementation uses NaN handling for missing or invalid data, ensuring consistent results when data gaps exist.
- The moment map shape is the original cube shape with the chosen axis removed, e.g., for axis=0 the result shape is (ny, nx).

Example usage (conceptual)
- result = moment_auto(cube, order=2, axis=0)
- result = moment_slicewise(cube, order=0, axis=2)
- result = moment_raywise(cube, order=1, axis=1)

This documentation describes the strategies and options available for efficiently handling large datasets when computing moment maps from spectral cubes.
