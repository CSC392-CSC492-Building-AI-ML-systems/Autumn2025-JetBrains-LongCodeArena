Kernel ridge regression
=======================

Kernel ridge regression (KRR) combines ridge regression (linear least-squares with L2 regularization) with the kernel trick. It learns a linear function in the feature space induced by the chosen kernel. For non-linear kernels, this corresponds to a non-linear function in the original space. The model learned by KRR is identical to that learned by support vector regression (SVR) with squared loss and L2 regularization, but KRR has a closed-form solution and is typically faster for medium-sized datasets. Compared to SVR, the learned model is non-sparse and prediction-time inference is correspondingly slower.

This estimator supports multi-output regression (y with shape [n_samples, n_targets]). Read more in the User Guide: kernel_ridge.

Estimator
---------

KernelRidge
    MultiOutputMixin, RegressorMixin, BaseEstimator

Parameters
----------

- alpha : float or array-like of shape (n_targets,), default=1.0  
  Regularization strength; must be a positive float. Regularization improves conditioning and reduces the variance of the estimates. Larger values specify stronger regularization. If an array is passed, penalties are assumed to be specific to the targets and must correspond to the number of targets.

- kernel : str or callable, default="linear"  
  Kernel mapping used internally. This parameter is directly passed to pairwise_kernels. If kernel is a string, it must be one of the metrics in pairwise.PAIRWISE_KERNEL_FUNCTIONS or "precomputed". If kernel is "precomputed", X is assumed to be a kernel matrix. Alternatively, if kernel is a callable, it is called on each pair of instances (rows) and the resulting value recorded. The callable should take two rows from X as input and return the corresponding kernel value as a single number. Callables from sklearn.metrics.pairwise are not allowed, as they operate on matrices rather than single samples.

- gamma : float, default=None  
  Gamma parameter for the RBF, Laplacian, polynomial, exponential chi2 and sigmoid kernels. Its interpretation is kernel-dependent. Ignored by kernels that do not use gamma.

- degree : int, default=3  
  Degree of the polynomial kernel. Ignored by kernels that are not polynomial.

- coef0 : float, default=1  
  Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

- kernel_params : dict, default=None  
  Additional parameters (keyword arguments) for the kernel function passed as a callable.

Attributes
----------

- dual_coef_ : ndarray of shape (n_samples,) or (n_samples, n_targets)  
  Representation of the weight vector(s) in the kernel space.

- X_fit_ : {ndarray, sparse matrix} of shape (n_samples, n_features)  
  Training data used for fitting. If kernel == "precomputed", this is the training kernel matrix of shape (n_samples, n_samples).

- n_features_in_ : int  
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)  
  Names of features seen during fit (only defined when X has feature names that are all strings). Added in v1.0.

Notes
-----

- KernelRidge learns a non-sparse model in the kernel space and solves a closed-form linear system. This can be advantageous for moderate-sized datasets but may be slower or memory-intensive for very large datasets.

- If kernel == "precomputed", the input X to fit/predict should be the kernel matrix rather than raw features.

- The implemented kernel is selected via the kernel parameter and parameters gamma, degree, and coef0 (when applicable) are passed to the kernel function.

Examples
--------

.. code-block:: python

   from sklearn.kernel_ridge import KernelRidge
   import numpy as np

   n_samples, n_features = 10, 5
   rng = np.random.RandomState(0)
   y = rng.randn(n_samples)
   X = rng.randn(n_samples, n_features)

   krr = KernelRidge(alpha=1.0)
   krr.fit(X, y)
   KernelRidge(alpha=1.0)
