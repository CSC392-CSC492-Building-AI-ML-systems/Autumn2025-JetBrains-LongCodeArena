Getting Started
===============

Overview
--------
This project’s core module provides functionality common to master and parent processes. It is sent to child contexts that are about to become a parent due to recursive connection. The module defines utilities, constants, and helpers used when managing child processes and inter-process communication, including:

- Logging level access via `get_log_level()`.
- Determining the Python executable to use for child processes via `get_sys_executable()`.
- Accessing the Mitogen core source for minification and startup via `_get_core_source()` and `get_core_source_partial()` (the latter caches a partial, minified form to speed up child startup).
- Standard messages for error handling, such as `BROKER_SHUTDOWN_MSG`, `OPENPTY_MSG`, and `SYS_EXECUTABLE_MSG`.

Key concepts
------------
- Master and parent processes share common functionality that is re-used when spawning or communicating with child contexts.
- The code supports Python 2 and Python 3 environments and contains compatibility logic for differences between versions.
- Some operations (like obtaining the core source) can be expensive; caching and pre-execution hooks are used to optimize startup.

Prerequisites
-------------
- Python (compatible with both Python 2.x and 3.x environments as indicated by the codebase’s compatibility handling).
- POSIX-like environment capable of PTY operations (PTY creation, openpty, etc.).
- Optional SELinux awareness on systems where SELinux is enabled.

Installation
------------
- Install via package manager (e.g., `pip install mitogen`) or from source as appropriate for your environment.

Getting started
---------------
- Import and inspect basic runtime configuration:
  - `get_log_level()` to determine the current logging level.
  - `get_sys_executable()` to determine the Python interpreter that should be used for child processes (falls back to `'/usr/bin/python'` with a warning if necessary).
- Optimize startup by using pre-minified core source:
  - `get_core_source_partial()` preloads a partial minified form of the Mitogen core to speed up child initialization.
- Be aware of common error messages used by the system:
  - `BROKER_SHUTDOWN_MSG` when the broker begins shutting down.
  - `OPENPTY_MSG` when PTY creation fails due to system limits.
  - `SYS_EXECUTABLE_MSG` when the Python executable cannot be determined.

Basic usage examples
--------------------
- Retrieve the effective log level:
  - ``level = get_log_level()``

- Determine the Python executable for child processes:
  - ``exe = get_sys_executable()``

- Prepare a partial core source for faster child startup:
  - ``partial = get_core_source_partial()``

Notes
-----
- `_get_core_source()` is an internal helper that fetches the Mitogen core source (used by non-minified startup paths; in some contexts it is overridden for minification during startup).
- `_sys_executable_warning_logged` ensures the warning about an unset `sys.executable` is logged only once.

Troubleshooting
---------------
- If you see PTY-related failures, review `OPENPTY_MSG` for guidance on system limits and kernel parameters.
- If the Python executable cannot be determined, check `SYS_EXECUTABLE_MSG` usage and ensure an appropriate Python interpreter is available at runtime.
- If a broker shutdown occurs unexpectedly, `BROKER_SHUTDOWN_MSG` indicates the cause and may guide reconnection logic.
