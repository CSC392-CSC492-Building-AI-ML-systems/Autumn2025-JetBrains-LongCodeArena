distutils_utilities
===================

Documentation for the distutils_utilities module.

Overview
--------
This module provides a small collection of utilities used to support distutils-based build workflows. It includes a lightweight container for installable libraries, helpers to determine parallel build job counts, and path-related utilities that handle POSIX-style paths as well as arbitrary Python frames.

Public API
-----------

InstallableLib
- Class that acts as a container for information about an installable library.
  - name: str
    Name of the installed library.
  - build_info: dict
    Dictionary holding build information.
  - target_dir: str
    Absolute path specifying where to install the library.
  - Notes: The class stores these parameters as attributes and includes a brief class docstring describing its purpose and usage.

get_num_build_jobs()
- Returns the number of parallel build jobs allowed for a build.
  - Behavior:
    - Reads the --parallel setting via numpy.distutils.core.get_distribution if present.
    - If not set, checks the environment variable NPY_NUM_BUILD_JOBS.
    - If still unset, falls back to the system processor count, capped at 8 to avoid overloading the system.
  - Returns: int
    The number of parallel build jobs that can be run.

quote_args(args)
- Quote list of arguments.
  - Note: This function is deprecated (deprecated:: 1.22) and issues a DeprecationWarning when called.
  - Behavior:
    - Accepts an iterable of arguments, converts to a list, and quotes items as needed.
  - Returns: list
    A new list with quoted arguments where appropriate.

allpath(name)
- Convert a /-separated pathname to one using the OS's path separator.
  - Returns: str
    The path with the OS-specific separators.

rel_path(path, parent_path)
- Return path relative to parent_path.
  - Behavior:
    - Uses realpath to avoid issues with symlinks.
    - Computes the path relative to the given parent_path, handling edge cases such as paths being the same or parent_path being a prefix of path.
  - Returns: str
    The relative path, or the original path if it cannot be made relative.

get_path_from_frame(frame, parent_path=None)
- Return path of the module given a frame object from the call stack.
  - Behavior:
    - Attempts to derive the file path from the frame's __file__; if unavailable, tries the module name and resolves its file path.
    - If a parent_path is provided, returns the path relative to it.
  - Returns: str
    The path (relative to parent_path if given) or '.' as a fallback.

njoin(*path)
- Join two or more pathname components.
  - Features:
    - Converts any POSIX-style separators to the OS-specific separator.
    - Resolves .. and . components.
    - Accepts either multiple string arguments, a single sequence argument, or a mixture thereof.
  - Returns: str
    The joined and normalized path.

Internal state and helpers
- _tdata
  Thread-local storage used to hold per-thread temporary directory data.
- _tmpdirs
  List of all created temporary directories for cleanup on exit.
- clean_up_temporary_directory()
  Cleanup function registered with atexit to remove temporary directories created during execution.

Notes
- The public API is exposed via __all__ in the module, which includes various utilities for build configuration and environment handling (e.g., get_numpy_include_dirs, default_config_dict, dict_append, appendpath, generate_config_py, get_cmd, allpath, get_mathlibs, terminal_has_colors, ..., exec_mod_from_location). The documentation above focuses on the functions and class directly implemented in the provided excerpt.
- Some functions reference numpy.distutils and distutils-related behavior, indicating this module is part of a larger distutils-based build system.
