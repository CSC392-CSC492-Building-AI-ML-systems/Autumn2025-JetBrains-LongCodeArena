Support Vector Machines (SVM)
=============================

Overview
--------

Support Vector Machines are a family of supervised learning methods used for classification and regression. They work by finding a decision boundary that maximizes the margin between classes. In scikit-learn, two families are available:

- SVC: kernel-based SVMs that support non-linear decision boundaries through kernels (e.g., linear, polynomial, RBF).
- LinearSVC: linear (kernel-free) SVMs optimized for large feature spaces and fast training, implemented via LIBLINEAR.

LinearSVC (Linear Support Vector Classification)
------------------------------------------------

LinearSVC provides a linear decision boundary and is optimized for speed on large-scale data. It supports both dense and sparse input and uses a one-vs-rest scheme for multiclass problems.

Key concepts
------------

- The main differences between LinearSVC and SVC lie in the loss function used by default and in the handling of intercept regularization.
- LinearSVC is backed by LIBLINEAR and relies on a linear kernel (no feature mapping is performed).

Parameters
----------

- penalty: {'l1', 'l2'}, default='l2'
  The norm used in regularization. 'l2' is the standard choice; 'l1' can yield sparse coef_.

- loss: {'hinge', 'squared_hinge'}, default='squared_hinge'
  The loss function to optimize. 'hinge' is the standard SVM loss, while 'squared_hinge' squares the hinge loss. The combination penalty='l1' and loss='hinge' is not supported.

- dual: "auto" or bool, default=True
  Selects the optimization problem to solve (dual or primal). 'auto' chooses based on data shape, loss, penalty, and multiclass setting. For some configurations, dual may be more efficient; for others, primal is preferred.

- tol: float, default=1e-4
  Tolerance for the stopping criteria.

- C: float, default=1.0
  Regularization parameter. The strength of the regularization is inversely proportional to C. Must be strictly positive.

- multi_class: {'ovr', 'crammer_singer'}, default='ovr'
  Strategy for multiclass classification. 'ovr' trains one-vs-rest classifiers; 'crammer_singer' optimizes a joint objective over all classes. If 'crammer_singer' is chosen, other options are ignored.

- fit_intercept: bool, default=True
  Whether to fit an intercept term. If True, an intercept feature is added to the model (and the interpretation of coefficients changes accordingly).

- intercept_scaling: float, default=1.0
  When fit_intercept is True, the instance vector becomes [x1, ..., xn, intercept_scaling], i.e., a synthetic feature with a constant value equal to intercept_scaling is appended. The intercept term is penalized by the regularization like other features.

Notes
-----

- LinearSVC supports both dense and sparse input and uses a one-vs-rest multiclass strategy by default.
- It differs from SVC with a linear kernel primarily in the loss function and the treatment of intercept regularization.
- For non-linear decision boundaries, consider SVC with a non-linear kernel (e.g., kernel='rbf' or kernel='poly').

Choosing between LinearSVC and SVC
----------------------------------

- Use LinearSVC for large-scale linear problems where training speed is critical and a linear decision boundary is appropriate.
- Use SVC when you need non-linear decision boundaries via kernel functions.

Examples
--------

Basic usage with default settings:

.. code-block:: python

   from sklearn.svm import LinearSVC
   clf = LinearSVC()
   clf.fit(X, y)

Advanced usage (example):

.. code-block:: python

   clf = LinearSVC(C=0.5, penalty='l2', loss='hinge', dual='auto', multi_class='ovr')
   clf.fit(X, y)
