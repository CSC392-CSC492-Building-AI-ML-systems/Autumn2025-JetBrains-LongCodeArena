w_init
======

Initialize a new WEST simulation, creating the WEST HDF5 file and preparing the first iteration’s segments.

Description
-----------

The w_init command initializes a WESTPA simulation. It reads basis states, target states, and start states from either files or command-line specifications, and prepares the initial set of segments for the first iteration. If no basis states or target states are provided, sensible defaults are created:

- Basis states: If neither --bstates-from nor --bstate is supplied, a default basis state with probability 1 identified by state ID zero and label "basis" is created. A warning is printed to remind you of this behavior.
- Target states: If neither --tstates-from nor --tstate is supplied, an equilibrium simulation (without any sinks) will be performed.

State sources
-------------

Basis states
^^^^^^^^^^^^^^

- --bstate-file / --bstates-from BSTATE_FILE
  Read basis state names, probabilities, and (optionally) data references from BSTATE_FILE.

- --bstate
  Add a basis state specified as a string 'label,probability[,auxref]'. May be provided multiple times; states are appended in the order given.

Target states
^^^^^^^^^^^^^^

- --tstate-file / --tstates-from TSTATE_FILE
  Read target state names and representative progress coordinates from TSTATE_FILE. The representative progress coordinate of a target state is used to create a recycling sink from the entire corresponding bin.

- --tstate
  Add a target state specified as a string 'label,pcoord0[,pcoord1[,...]]'. May be provided multiple times; states are appended in the order given.

Start states
^^^^^^^^^^^^

- --sstate-file / --sstates-from SSTATE_FILE
  Read start state names, probabilities, and (optionally) data references from SSTATE_FILE.

- --sstate
  Add a start state specified as a string 'label,probability[,auxref]'. May be provided multiple times; states are appended in the order given.

Common options
--------------

- --segs-per-state N
  Initialize N segments from each basis state (default: 1).

- --force
  Overwrite any existing simulation data.

- --no-we / --shotgun
  Do not run the weighted ensemble bin/split/merge algorithm on newly-created segments.

Work manager integration
------------------------

The command wires into the WESTPA work manager. Additional work-manager-specific arguments can be provided via the WM configuration (via --wm-args or equivalent, depending on the installed WESTPA version). The command builds the initial states and then delegates to the configured work manager to manage the simulation lifecycle.

Input formats
-------------

- Basis state line format: label,probability[,auxref]
- Start state line format: label,probability[,auxref]
- Target state line format: label,pcoord0[,pcoord1[,...]]

Examples
--------

- Initialize from files with default behavior:
  w_init --bstates-from basis_states.txt --tstates-from target_states.txt

- Initialize with inline basis and target states and multiple segments per state:
  w_init --bstate "basis,1.0" --tstate "target1,0.0" --segs-per-state 2

- Force overwrite of existing data and disable WE for new segments:
  w_init --force --no-we
