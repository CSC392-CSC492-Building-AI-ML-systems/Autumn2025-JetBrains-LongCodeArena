Stochastic Gradient Descent
===========================

Overview
--------
This documentation describes the stochastic gradient descent (SGD) implementation that aggregates Stochastic Average Gradient (SAG) and SAGA algorithms. The code provides fused-type support for two floating-point precisions and exposes a Multinomial Logistic Loss suitable for multiclass classification. Numerical stability helpers and thresholding utilities are included to support efficient, differentiable optimization.

Supported precisions
--------------------
- 64-bit: c_type = double, np_type = np.float64, suffix = 64
- 32-bit: c_type = float,  np_type = np.float32, suffix = 32

Key components
--------------
- SAG and SAGA implementations: Generated code duplicates for each supported dtype, enabling fused-type performance.
- LossFunction interface: Interacts with the _sgd_fast module to define loss and gradient behavior.
- Logarithmic loss for multiclass (MultinomialLogLoss): Computes per-sample multiclass logistic loss and its gradient.
- Numerical stability helpers:
  - logsumexp: Stable computation of log(sum(exp(prediction))) for a vector of class scores.
  - fmax: Inline maximum helper used in numerical routines.
- Soft thresholding: Non-linear shrinkage operation used in certain optimization steps.

Multinomial Logistic Loss
------------------------
The loss for a single sample in multiclass logistic regression is computed as:
  loss = (logsumexp(prediction) - prediction[y]) * sample_weight
where:
- prediction is the vector of class scores for the sample (length n_classes),
- y is the index of the correct class,
- sample_weight scales the contribution of the sample.

Gradient computation (per-sample, per-class):
  gradient[c] = exp(prediction[c] - logsumexp(prediction)) * sample_weight
  if c == y: gradient[c] -= 1
The resulting gradient vector is scaled by the sample weight.

Numerical stability
-------------------
- logsumexp is implemented in a numerically stable way by normalizing with the maximum value in the vector:
  log(sum(exp(arr))) = log(sum(exp(arr - max))) + max
- The code uses dedicated helpers to ensure stability during both loss and gradient calculations.

Interfaces and data types
-------------------------
- LossFunction: Core interface for loss evaluation used by the SGD framework.
- Log and SquaredLoss: Supporting loss abstractions exposed by the _sgd_fast module.
- Data structures:
  - SequentialDataset32
  - SequentialDataset64
These are used to feed data efficiently into the SGD routines.

Implementation notes
--------------------
- The code employs Cython cdef and nogil blocks to enable release of the Python GIL during heavy numerical work.
- Inline functions (e.g., _logsumexp and fmax) are used to optimize performance.
- The MultinomialLogLoss class provides:
  - __reduce__ support for pickling.
  - Methods to compute loss and gradient given a prediction vector, true label, number of classes, and sample weight.

Usage notes
-----------
- The implementation is designed to work with both SAG and SAGA optimization strategies, leveraging fused types for performance.
- MultinomialLogLoss is suitable for multiclass classification problems with softmax-like decision boundaries.
- Numerical stability helpers (logsumexp, etc.) help prevent underflow/overflow in practice.
