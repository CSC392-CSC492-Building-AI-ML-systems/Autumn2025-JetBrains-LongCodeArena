Partial Dependence and ICE Plots
================================

Overview
--------
Partial Dependence Plots (PDP) visualize the average effect of one or more features on the predicted outcome of a model. They show how the model prediction changes when a feature is varied, averaging out the effects of all other features.

Individual Conditional Expectation (ICE) plots display the predicted response for each individual observation as a feature is varied. ICE plots reveal heterogeneity in feature effects that may be hidden in the average PDP.

Partial Dependence Plots (PDP)
------------------------------
Concept
- For a given feature (or pair of features), PDP integrates out the other features by averaging over the data distribution, producing the average predicted response as a function of the feature values.
- Useful for understanding the monotonicity or general shape of the model’s dependence on a feature.

When to use
- You want a global view of how the model responds to a feature on average.
- You are interested in the overall trend, not per-sample variation.

How computed
- For a single feature: vary the feature on a grid of values and average the model predictions over all samples with other features fixed to their observed values (or drawn from the data distribution).
- For two features: a 2D surface showing the joint average effect.

Using scikit-learn (PDP)
- Typical workflow uses PartialDependenceDisplay to plot the average effect.

.. code-block:: python

   from sklearn.inspection import PartialDependenceDisplay
   # Average effect for one or more features
   display = PartialDependenceDisplay.from_estimator(
       estimator, X, features=[0, 1], grid_resolution=20, kind="average"
   )
   display.plot()

Individual Conditional Expectation (ICE) Plots
----------------------------------------------
Concept
- ICE plots show, for each individual sample, how the model prediction changes as the feature varies.
- Each line corresponds to a single observation. The collection of lines reveals heterogeneity in the feature effect and potential interactions with other features.

When to use
- You want to assess whether the average PDP is representative or if there is substantial variation across instances.
- You want to detect feature interactions and non-linearities that are not visible in the PDP alone.

How computed
- For a feature, for every sample, vary the feature along a grid and compute the prediction while keeping the other features fixed to that sample’s values.

Using scikit-learn (ICE)
- Visualization can be produced by requesting ICE lines alongside the average effect.

.. code-block:: python

   from sklearn.inspection import PartialDependenceDisplay
   # ICE lines only
   display = PartialDependenceDisplay.from_estimator(
       estimator, X, features=[0], grid_resolution=20, kind="individual"
   )
   display.plot()

Combined PDP and ICE
-------------------
- You can request both average and individual lines to compare the overall trend with per-sample behavior.

.. code-block:: python

   display = PartialDependenceDisplay.from_estimator(
       estimator, X, features=[0], grid_resolution=20, kind="both"
   )
   display.plot()

Interpretation tips
- PDP:
  - A flat line indicates little dependence of the predicted outcome on the feature.
  - Increasing or decreasing curves suggest positive or negative associations.
- ICE:
  - If ICE lines are tightly clustered, the effect is similar across instances.
  - If lines vary widely or cross, there are interactions with other features or heterogeneous effects.
- Be mindful of extrapolation: PDP and ICE rely on the data distribution used to generate predictions; outside regions with little data, interpretations may be unreliable.
- For categorical features, use a grid that reflects the categories or distinct values.

Limitations and caveats
- PDP marginalizes over other features and may obscure interactions.
- ICE can be sensitive to noise when the grid is fine or the model is unstable.
- Feature correlations can affect interpretation; ensure the feature of interest is meaningful in the data context.
