Mixed Linear Models (Linear Mixed Effects Models)
===============================================

Overview
--------
Linear mixed effects models are regression models for dependent data. They can be used to estimate regression relationships involving both means and variances. These models are also known as multilevel linear models and hierarchical linear models.

The MixedLM class fits linear mixed effects models to data and provides support for common post-estimation tasks. This is a group-based implementation that is most efficient when the data can be partitioned into independent groups. Some models with crossed effects can be handled by specifying a model with a single group.

Model and data structure
------------------------
The data are partitioned into disjoint groups. For group i, the probability model is:

Y = X * beta + Z * gamma + epsilon

where

- Y is a n_i-dimensional response vector (endog in MixedLM)
- X is a n_i x k_fe design matrix for the fixed effects (exog in MixedLM)
- beta is a k_fe-dimensional vector of fixed effects parameters (fe_params in MixedLM)
- Z is a design matrix for the random effects with n_i rows (exog_re in MixedLM). The number of columns in Z can vary by group
- gamma is a random vector with mean 0. The covariance matrix for the first k_re elements of gamma (cov_re in MixedLM) is common to all groups. The remaining elements are variance components as discussed below. Each group receives its own independent realization of gamma.
- epsilon is a n_i-dimensional vector of iid normal errors with mean 0 and variance sigma^2; the epsilon values are independent both within and between groups

Y, X and Z must be observed. beta, Psi (cov_re), and sigma^2 are estimated using ML or REML; gamma and epsilon are random and define the probability model.

The marginal mean structure is E[Y | X, Z] = X * beta. If only the mean structure is of interest, Generalized Estimating Equations (GEE) provide an alternative to linear mixed models.

Random effects and variance components
-------------------------------------
Two types of random effects are supported:

- Standard random effects are correlated with each other in arbitrary ways. Every group has the same number (k_re) of standard random effects, with the same joint distribution (independent realizations across groups).

- Variance components are uncorrelated with each other, and with the standard random effects. Each variance component has mean zero, and all realizations of a given variance component have the same variance parameter. The number of realized variance components per variance parameter can differ across groups.

Estimation and likelihood
-------------------------
The fixed effects parameters (beta), the random effects covariance (Psi, referred to as cov_re), and the error variance (scale, i.e., sigma^2) are estimated via ML or REML. The random effects gamma and the residuals epsilon are random, defining the probability model.

The likelihood, gradient, and Hessian calculations closely follow the standard references for linear mixed effects models.

Parameterizations
-----------------
Three parameterizations are used in different parts of the implementation. The regression slopes (fe_params) are identical in all three parameterizations, but the variance parameters differ. The parameterizations are:

- User parameterization: cov(endog) = scale * I + Z * cov_re * Z'
  This is the main parameterization visible to the user.

- Profile parameterization: cov(endog) = I + Z * cov_re1 * Z'
  This parameterization is used in the profile likelihood maximization. The user’s cov_re is equal to the profile cov_re1 scaled by the scale.

- Square root parameterization: operate with the Cholesky factor of cov_re1 instead of cov_re directly. This is hidden from the user.

All three parameterizations can be packed into a vector by concatenating fe_params, the lower-triangle or Cholesky square root of the dependence structure, and the variance parameters for the variance components. When unpacking, it is important to square or reflect the dependence structure depending on the parameterization used.

Optimization and scoring
------------------------
Two score methods are implemented:

- Score with respect to the random effects covariance matrix elements (used for inference after the maximum likelihood estimate is reached).

- Score with respect to the parameters of the Cholesky square root of the random effects covariance matrix (used for optimization).

Numerical optimization uses GLS to avoid explicitly optimizing over the fixed effects parameters. The likelihood that is optimized is profiled over both the scale parameter and the fixed effects parameters. As a result, it is difficult and unnecessary to calculate the Hessian of the profiled log-likelihood function, so Hessian-based methods such as Newton-Raphson are not used for model fitting.

Notation
--------
- cov_re is the random effects covariance matrix (referred to as Psi) and scale is the error variance (scalar). For a single group, the marginal covariance matrix of endog given exog is scale * I + Z * cov_re * Z'.

- vcomp is a vector of variance parameters. Its length is determined by the number of keys in either the exog_vc argument to MixedLM, or the vc_formula argument when using formulas to fit a model.

Notes
-----
- Three parameterizations can be collapsed into a single vector, with differences in how the random effects structure is represented and scaled.
- The Hessian of the profiled likelihood is not implemented; Newton-Raphson is not suitable for this model fitting.
- This module emphasizes GLS-based profiling to efficiently estimate fixed effects and variance components.
