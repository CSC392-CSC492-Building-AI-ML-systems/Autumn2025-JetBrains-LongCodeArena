Documentation for the code module
===================================

Overview
--------

This module provides Bluesky plan helpers to read detectors and to perform structured scans over motors. It exposes three primary plan generators:

- count: take a sequence of readings from detectors.
- list_scan: perform a full inner-product scan over one or more motors.
- rel_list_scan: placeholder for a relative-list-scan implementation (not yet implemented).

The implementations are designed to integrate with Bluesky preprocessors and stubs, and to producing rich metadata (via the md argument) for plan execution and downstream processing.

API reference
-------------

count(detectors, num=1, delay=None, *, per_shot=None, md=None)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Take one or more readings from detectors.

Parameters
- detectors: list
  - A list of "readable" objects (detectors) to read from.
- num: int, optional
  - Number of readings to take; default is 1.
  - If None, data capture continues until canceled.
- delay: iterable or scalar, optional
  - Time delay in seconds between successive readings; default is 0.
- per_shot: callable, optional
  - Hook for customizing action of the inner loop (messages per step).
  - Signature: def f(detectors: Iterable[OphydObj]) -> Generator[Msg, ...]
- md: dict, optional
  - Metadata dictionary that will be merged into the plan metadata.

Notes
- If delay is an iterable, it must have at least num - 1 entries, or the plan will raise ValueError during iteration.

Returns
- A generator yielded by inner_count, i.e., the plan steps for counting.

Example
.. code-block:: python

    yield from count([det1, det2], num=5, delay=[0.1, 0.2], per_shot=my_hook)

-----

list_scan(detectors, *args, per_step=None, md=None)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Scan over one or more variables in steps simultaneously (inner product).

Parameters
- detectors: list
  - List of detectors to read during the scan.
- *args:
  - For one dimension: motor, [point1, point2, ...]
  - For multiple dimensions: motor1, [points...], motor2, [points...], ..., motorN, [points...]
  - Motors can be any settable object (motor, temperature controller, etc.).
- per_step: callable, optional
  - Hook for customizing action of the inner loop (messages per step).
  - Signature: f(detectors, motor, step) -> plan (generator)
- md: dict, optional
  - Metadata dictionary for the plan.

Returns
- A generator produced by scan_nd(detectors, full_cycler, per_step=per_step, md=_md).

Implementation notes
- Validates that the number of motor/point arguments is even (each motor should be paired with a points list).
- All point lists must be the same length; otherwise a ValueError is raised.
- Builds a metadata dictionary (_md) containing detectors, motors, number of points, and plan details.
- Uses plan_patterns.inner_list_product(args) to create the full cycling pattern (full_cycler).
- Integrates with default hints for dimension information derived from motor hints.
- Delegates to scan_nd to perform the actual execution.

Example
.. code-block:: python

    yield from list_scan([det1], motor1, [0, 1, 2], motor2, [10, 20, 30], per_step=my_step)

-----

rel_list_scan(detectors, *args, per_step=None, md=None)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Scan over one variable in steps relative to the current position.

Parameters
- detectors: list
  - List of detectors to read during the scan.
- *args:
  - For one dimension: motor, [points...], where points are relative to the current position.
  - General form mirrors list_scan, but points are interpreted as offsets from the current motor position.
- per_step: callable, optional
  - Hook for customizing action of the inner loop (messages per step).
- md: dict, optional
  - Metadata dictionary for the plan.

Returns
- Not implemented yet. This function currently contains a placeholder (TODO).

Notes
- The rel_list_scan implementation is marked as TODO and is not functional in this version.

Examples
.. code-block:: python

    # Placeholder usage (not yet implemented)
    # yield from rel_list_scan([det1], motor1, [0.5, 1.0, -0.5])

Dependencies and related modules
--------------------------------

- numpy (np)
- cytools (partition) or toolz (partition) as a fallback
- Bluesky plan primitives:
  - plan_patterns (for inner_list_product)
  - preprocessors as bpp
  - plan_stubs as bps
  - Msg (for message objects)

Internal design notes
---------------------

- count builds a minimal metadata dictionary (_md) and augments it with user-provided md.
- list_scan computes a consistent dimensional space by extracting motor hints and applying default hints for dimensions.
- rel_list_scan is a planned feature and currently a placeholder.

See also
--------

- plan_patterns.inner_list_product
- bluesky.plans.scan_nd
- bluesky.plans.rel_list_scan
- Msg class for message objects

Examples of usage
-----------------

- Basic detector readout:
  - yield from count([det1], num=3, delay=0.1)

- Multi-motor inner product scan:
  - yield from list_scan([det1], motorA, [0, 1, 2], motorB, [10, 20, 30])

Notes
-----

- The module relies on relative imports (plan_patterns, utils, Msg, bpp, bps) and may require appropriate package structure to resolve imports.
- rel_list_scan is not yet implemented; use list_scan for now to perform structured scans.
