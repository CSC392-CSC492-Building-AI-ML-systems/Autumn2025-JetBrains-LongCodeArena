w_red
===== 

Overview
--------

w_red computes raw and RED-corrected rate constants from a WestPA
HDF5 simulation file. It reads the rate evolution data and, when requested,
applies the RED correction based on the distribution of event durations.

Key data used:
- rate_evolution dataset, accessing the field "expected" for rate values.
- durations dataset (for RED correction), using fields "duration", "weight", "istate", "fstate".

Core concepts
-------------

- get_raw_rates(directh5, istate, fstate, n_iters=None)
  Returns the raw (unadjusted) rate evolution value for the given initial
  and final state over the specified iteration slice.

- DurationCorrector
  Encapsulates the RED correction logic. Constructed from a kinetics file slice
  of durations/weights, along with a dtau. Provides:
  - event_duration_histogram and cumulative_event_duration_histogram properties
  - correction(iters, freqs=None) to compute the RED correction factor for a
    given subset of iterations (optionally weighted by freqs)

- calc_avg_rate(directh5_path, istate, fstate, **kwargs)
  Compute either the raw or RED-corrected rate constant, with an optional
  confidence-interval-like context. Options include:
  - n_iters: number of iterations to include (default all)
  - report_interval (default 20)
  - n_steps_iter (default 1000)
  - red: enable RED correction
  - callback: if provided, receives a dict like {"correction": dc}

- calc_rates(directh5_path, istate, fstate, **kwargs)
  Return both raw and RED-corrected rate constants as a function of iterations.

Command-line interface
----------------------

w_red is a command to compute rate constants with optional RED correction from a WestPA HDF5 file.

Arguments
- directh5_path: Path to the WestPA direct HDF5 file.
- istate: Initial state index (integer).
- fstate: Final state index (integer).

Options
- --n_iters N or -n N
  Number of iterations to include in the computation. If omitted, all iterations are used.

- --report_interval N
  Interval used to define dtau when calculating dtau (default 20).

- --n_steps_iter N
  Number of steps per iteration used to compute dtau (default 1000).

- --red
  Enable RED correction. When this flag is set, w_red computes and applies the
  RED correction factor to the rate.

- --callback FUNC
  Optional callback function to receive intermediate results. If provided,
  w_red passes a dict like {"correction": dc} where dc is the DurationCorrector
  instance.

Notes
- If RED is requested but there is no meaningful duration distribution data,
  the correction may be 0.0 or effectively no correction.
- The RED correction factor multiplies the raw rate to yield a corrected rate.

Output
------

- The primary output is the rate constant. When using RED, the function returns
  the RED-corrected rate; otherwise it returns the raw rate.

- If a callback is provided, it will be invoked with a dictionary containing
  the correction object for inspection or further processing.

Algorithmic details (brief)
----------------------------

- DurationCorrector builds a mapping from observed durations to a tau-bin index.
- It computes a histogram-like function f_tilde(tau) from weights with a normalization
  that accounts for the observation window (maxduration) and bin width dtau.
- f_tilde is integrated to obtain f_int1, then integrated again to obtain integral2.
- If integral2 is nonzero, the RED correction factor is maxduration / integral2.
- The rate is multiplied by this factor to yield the RED-corrected rate.

Data structures referenced
--------------------------

- rate_evolution: Array-like dataset with entries per iteration, initial state (istate),
  final state (fstate), containing a field "expected" representing the rate.
- durations: Dataset containing per-walker event data with fields
  "duration", "weight", "istate", "fstate".

Examples
--------

- Compute the raw rate for istate=0, fstate=1 from the file direct.h5:

  w_red direct.h5 0 1

- Compute the RED-corrected rate:

  w_red --red direct.h5 0 1

- Compute rates with all defaults and capture the correction via a callback (Python usage):

  rc = __import__('w_red_module')  # hypothetical import path
  rate = rc.calc_avg_rate('direct.h5', 0, 1, red=True, n_iters=None,
                          callback=lambda correction: print("Correction provided"))
