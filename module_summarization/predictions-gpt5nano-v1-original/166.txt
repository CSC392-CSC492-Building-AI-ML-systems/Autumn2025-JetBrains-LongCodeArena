Filenode - simulating a filesystem with PyTables
===============================================

The filenode module provides a file-like interface to PyTables nodes, enabling you to simulate a filesystem inside a PyTables database file. It exposes a pair of high-level helpers to create or open a file-like node and a low-level IO class that implements the Python file API for binary data stored in a PyTables node.

Overview
--------
- Create a brand new filenode with new_node() and obtain a Python file-like object that supports read/write/seek/truncate semantics.
- Open an existing filenode with open_node() for read-only or read-write access. Read access is always available.
- All IO is binary; text encoding is outside the scope of this interface.
- The implementation adheres to the io module interfaces as of Python 3.x, enabling seamless use with standard Python I/O patterns.
- Version 3.0 introduced a complete rewrite to be fully compliant with the standard io interfaces.

Public API
----------
- NodeType: 'file'
  - A system attribute value identifying filenode objects as file-type nodes.
- NodeTypeVersions: [1, 2]
  - Supported NODE_TYPE_VERSION values for filenode system attribute.

- new_node(...)
  - Create a new filenode inside a PyTables file. The returned object behaves like a file opened for writing and reading as appropriate.

- open_node(...)
  - Open an existing filenode created with new_node() for read-only or read-write access. Read access is always available.

- RawPyTablesIO
  - Base class implementing binary I/O on PyTables nodes. Subclasses provide the actual file-like behavior. It implements key methods such as tell(), seek(), readable(), writable(), readinto(), writeable-by-mode handling, truncate(), flush(), close(), and related behavior consistent with Python’s io.RawIOBase.

Usage notes
-----------
- Access modes:
  - Read access is always available.
  - Write access depends on how the filenode was created or opened (e.g., appending modes may be required for writes).
- Binary I/O only: the interface is designed for raw byte streams; no text encoding is applied.

- The file-like object supports:
  - tell(): current position
  - seek(pos, whence): reposition within the node
  - readinto(b): read data into a pre-allocated buffer
  - read(): (through the high-level usage patterns that leverage readinto and related operations)
  - write(data): append or write depending on mode
  - truncate(pos=None): extend the filenode size (growth only)
  - flush(): flush any internal buffers
  - close(): release resources

- Limitations:
  - This is a binary file interface backed by a PyTables node inside an HDF5 file; it is not a general-purpose filesystem and does not support all filesystem semantics (e.g., random-access file rewrites to arbitrary data blocks beyond append growth in some modes).

Example
-------
Below is a minimal illustrative usage pattern. Adapt to your exact API signatures as provided by your PyTables filenode module.

.. code-block:: python

    import tables as tb
    # from filenode import new_node, open_node

    with tb.open_file("mydb.h5", "a") as h5f:
        # Create a new filenode at path "/myfs.bin"
        fnode = new_node(h5f, "/myfs.bin", mode="w+b")

        # Use a separate handle to read/write
        with open_node(h5f, "/myfs.bin", mode="r+b") as af:
            af.write(b"hello world")
            af.seek(0)
            data = af.read(11)
            af.flush()

API reference (high level)
--------------------------
- Filenode objects implement the standard Python file-like API for binary data.
- The underlying storage is a PyTables node; data is accessed via numpy-backed buffers and PyTables I/O primitives.

Version history
---------------
- Version 3.0: The module was rewritten to be fully compatible with the Python io module interfaces.
