Text feature extraction
=======================

The sklearn.feature_extraction.text submodule provides utilities to convert text documents into numeric feature vectors. It offers several vectorizers and helpers to preprocess text, tokenize, and transform counts into meaningful representations such as TF–IDF.

Public API
----------

- HashingVectorizer
- CountVectorizer
- ENGLISH_STOP_WORDS
- TfidfTransformer
- TfidfVectorizer
- strip_accents_ascii
- strip_accents_unicode
- strip_tags

Key components
--------------

- _preprocess(doc, accent_function=None, lower=False)
  Internal helper to apply optional lowercasing and accent-stripping to a document before tokenization.

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  Internal pipeline to obtain tokens or n-grams from a document. Supports using a full analyzer or composing preprocessor, tokenizer, and n-grams steps.

- strip_accents_unicode(s)
  Remove accentuated characters by normalizing to NFKD and stripping combining marks. Faster for ASCII-compatible inputs when possible.

- strip_accents_ascii(s)
  Remove accentuated characters by transliterating to ASCII via NFKD and encoding to ASCII.

- strip_tags(s)
  Basic HTML/XML tag remover using a simple regular expression. Intended for lightweight preprocessing.

- _VectorizerMixin
  Shared logic for text vectorizers, including:
  - decode: convert input (filename, file-like, or string) to a unicode string, handling encodings and error modes.
  - _word_ngrams(tokens, stop_words=None): convert tokens into word n-grams with optional stop-word filtering.

Usage notes
-----------

- CountVectorizer converts a collection of text documents into a matrix of token counts.
- TfidfVectorizer builds on top of CountVectorizer to produce TF–IDF features.
- TfidfTransformer can convert a term-count matrix to a TF–IDF representation without re-vectorizing the text.
- HashingVectorizer uses the hashing trick to map tokens to a fixed-dimensional feature space, avoiding a vocabulary lookup.
- ENGLISH_STOP_WORDS provides a built-in stop-word list suitable for many English text mining tasks.
- The accent-stripping and tag-stripping helpers offer lightweight preprocessing options that can be composed in the preprocessing pipeline.

Example
-------

Basic TF–IDF vectorization:

.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   docs = [
       "The quick brown fox jumps over the lazy dog.",
       "Never jump over the lazy dog quickly."
   ]

   vectorizer = TfidfVectorizer(stop_words="english")
   X = vectorizer.fit_transform(docs)
