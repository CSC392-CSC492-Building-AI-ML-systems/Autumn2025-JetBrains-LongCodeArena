Migrating to nextcord
===================== 

This guide documents the widget-related API in nextcord and how to migrate code that interacts with guild widgets from previous versions.

Widget model
------------

Nextcord provides three widget-related classes that mirror the data returned by Discord's widget endpoints:

WidgetChannel
- Represents a "partial" widget channel.
- Attributes:
  - id (int): The channel's ID.
  - name (str): The channel's name.
  - position (int): The channel's position.
- Operations:
  - __str__ returns the channel's name.
  - __repr__ returns a representation including id, name, and position.
- Properties:
  - mention (str): The string to mention the channel (e.g. "<#12345>").
  - created_at (datetime): The channel's creation time (UTC) derived from its ID via snowflake_time.

WidgetMember
- Represents a "partial" member of the widget's guild.
- Inherits from BaseUser.
- Attributes:
  - id (int)
  - name (str)
  - discriminator (str)
  - bot (bool)
  - status (Status)
  - nick (Optional[str])
  - avatar (Optional[str])
  - activity (Optional[Union[BaseActivity, Spotify]])
  - deafened (Optional[bool])
  - muted (Optional[bool])
  - suppress (Optional[bool])
  - connected_channel (Optional[WidgetChannel])
- Initialization:
  - Requires state and data payload, with an optional connected_channel.
  - activity is created from the payload's game data using create_activity.
- display_name:
  - Returns nick if set, otherwise the member's name.

Widget
- Represents a Guild widget.
- Attributes:
  - id (int): The guild's ID.
  - name (str): The guild's name.
  - channels (List[WidgetChannel]): Accessible voice channels in the guild.
  - members (List[WidgetMember]): Online members in the server. Offline members do not appear in the widget.
- Notes:
  - Widget data reflects current activity and presence where available; data may be limited by Discord and may omit offline members.

Migration notes
--------------

- Import paths:
  - Use nextcord.WidgetChannel, nextcord.WidgetMember, and nextcord.Widget as appropriate for your codebase.
- Activity data:
  - If your code previously relied on a different key for activity data, map it to the activity field, which is created from the payload's game data via create_activity.
- Presence data:
  - Widget.members contains online members only; handle absence of offline members accordingly.
- Data shape:
  - Widget-related payloads may vary; rely on the provided attributes and properties (e.g., WidgetChannel.mention, WidgetMember.display_name) for robust code.

Example usage (conceptual)
---------------------------

- Iterate over channels in a guild widget:
  - for ch in widget.channels:
      print(ch.name, ch.mention)

- Access a member's display name:
  - print(member.display_name)

- Check a member's status and activity:
  - print(member.status, member.activity)
