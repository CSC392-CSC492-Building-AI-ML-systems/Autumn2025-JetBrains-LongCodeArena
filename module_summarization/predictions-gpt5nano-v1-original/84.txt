piccolo_apps
=============

Documentation for the piccolo_apps module

Overview
--------
This module provides utilities and data structures to configure and manage Piccolo applications and migrations. It includes helpers for discovering table classes, representations of app configurations, and a registry for all apps in a project.

Public API
-----------

MigrationModule
---------------
A ModuleType subclass used as a marker for migration modules. It defines:
- ID: str
- VERSION: str
- DESCRIPTION: str
- forwards(): an async method intended to perform forward migrations (no-op by default)

PiccoloAppModule
----------------
A ModuleType subclass used as a marker for modules that contain app configuration. It defines:
- APP_CONFIG: AppConfig

table_finder(...)
--------------
Imports and collects Table subclasses from given modules. Supports filtering by tags.

Parameters
- modules: Sequence[str]
  Module paths to search for Table subclasses (e.g., ['blog.tables']).
- include_tags: Sequence[str] (optional)
  Tags that a Table subclass must have to be included. '__all__' includes all.
- exclude_tags: Sequence[str] (optional)
  Tags that, if present on a Table subclass, will exclude it. Takes precedence over include_tags.
- exclude_imported: bool (optional)
  If True, only Tables defined in the given modules are used; imported Tables are ignored.

Returns
- List[Type[Table]]: A list of Table subclasses that match the criteria.

Notes
- The function imports modules dynamically and inspects attributes to find subclasses of Table (excluding the base Table class itself).

Command
-------
A small data structure to represent a callable command and its aliases.

Fields
- callable: t.Callable
  The function or coroutine to be registered as a command.
- aliases: List[str]
  Optional aliases for the command.

AppConfig
---------
Represents the configuration for a single Piccolo application.

Fields
- app_name: str
  The name of the app (e.g., 'article').
- migrations_folder_path: str
  Path to the folder containing the app's migration files.
- table_classes: List[Type[Table]]
  Table classes registered for the app to enable auto migrations.
- migration_dependencies: List[str]
  List of other app module paths that this app depends on. Dependencies are migrated first.
- commands: List[Union[Callable, Command]]
  List of callables or Command objects exposed to the Piccolo CLI for this app.

Methods and Properties
- __post_init__()
  Wraps plain callables in Command objects and normalizes migrations_folder_path to a string. Initializes internal migration dependency cache.
- register_table(table_class: Type[Table]) -> Type[Table]
  Adds a table class to the app's registry and returns the class.
- migration_dependency_app_configs (property)
  Returns a List[AppConfig] for all apps listed in migration_dependencies. Caches the result for efficiency.
- get_table_with_name(table_class_name: str) -> Type[Table]
  Retrieves a registered Table subclass by its class name. Raises ValueError if not found.

AppRegistry
-----------
Keeps track of all Piccolo apps in the project (typically declared in piccolo_conf.py).

Constructor
- __init__(apps: List[str] = None)
  Creates a registry. Accepts an optional list of app module paths.

Notes
- The registry stores app paths and groups their AppConfig instances. The implementation details (such as the exact structure of app_configs) are defined in the module and used by the rest of the system to coordinate migrations and CLI commands.

Usage overview
--------------
- Use table_finder(...) to discover Table subclasses across modules based on tags.
- Define AppConfig instances for each app, including app_name, migrations_folder_path, and any registered table_classes or commands.
- Access migration_dependency_app_configs from an AppConfig to resolve and prepare dependent apps before running migrations.
- Retrieve specific Table subclasses within an AppConfig using get_table_with_name(...).
- Collect all apps in an AppRegistry to coordinate across the project.
