filenode.rst
============

Overview
--------

The filenode module provides a file-like interface to PyTables nodes within HDF5 files. It enables reading and writing to PyTables file nodes using a Pythonic IO API. Use new_node() to create a brand-new file node and open_node() to open an existing one. Read access is always available; write access is permitted on new files or files opened with mode 'a+' and only allows appending data. Currently only binary I/O is supported.

This module is designed to be fully compliant with the interfaces defined in the Python io module. See the filenode_usersguide for detailed usage instructions.

Public API
----------

Module-level constants
- NodeType: 'file'
  Value used for NODE_TYPE node system attribute.
- NodeTypeVersions: [1, 2]
  Supported values for NODE_TYPE_VERSION node system attribute.

Public class
- RawPyTablesIO(io.RawIOBase)
  Base class for raw binary I/O on PyTables file nodes.

  Purpose
  - Provides a raw binary file-like interface to a PyTables node, enabling low-level read/write semantics consistent with Python's IO API.

  Key attributes
  - mode: read/write mode for the IO object (readable and writable depending on mode)

  Important methods
  - tell() -> int
    Return the current stream position.

  - seek(pos, whence=0) -> int
    Change the stream position.
    whence values:
    - 0: start of stream
    - 1: current position
    - 2: end of stream

  - seekable() -> bool
    Return True if the object supports random access.

  - fileno() -> int
    Return underlying file descriptor, if available.

  - close()
    Flush buffers and close the IO object.

  - flush()
    Flush write buffers, if applicable.

  - truncate(pos=None) -> int
    Truncate the file to the given size. If pos is None, use the current position. Truncation is primarily meaningful for growing files in this context.

  - readable() -> bool
    Return True if the IO object was opened for reading.

  - writable() -> bool
    Return True if the IO object was opened for writing.

  - readinto(b) -> int
    Read up to len(b) bytes into the writable buffer b. Returns the number of bytes read.

  - readline(limit=-1) -> bytes
    Read and return a line from the stream, up to limit bytes if specified.

Notes
-----

- The module was rewritten in version 3.0 to be fully compliant with the Python io interfaces.
- The file interface targets PyTables’ internal file nodes and is intended for binary data.
- Read access is always available; write access is permitted in modes that enable writing (e.g., new files or files opened with mode 'a' or '+'). Writing only supports appending for file nodes.
