Async Plans Documentation
=========================

Overview
--------

This module provides asynchronous planning helpers for taking readings from detectors and performing scans. It defines the following plans:

- count: Take one or more readings from detectors.
- list_scan: Scan over one or more motors in an inner-product style sweep.
- rel_list_scan: Scan over one variable in steps relative to the current position (not yet implemented in this version).

Functions
---------

count(detectors, num=1, delay=None, *, per_shot=None, md=None)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Description
    Take one or more readings from detectors.

Parameters
    detectors : list
        List of 'readable' objects (detectors).
    num : int, optional
        Number of readings to take; default is 1. If None, capture data until canceled.
    delay : iterable or scalar, optional
        Time delay in seconds between successive readings; default is 0.
    per_shot : callable, optional
        Hook for customizing action of the inner loop (messages per step).
        Signature:
        def f(detectors: Iterable[OphydObj]) -> Generator[Msg]
    md : dict, optional
        Metadata to attach to the plan.

Notes
    If delay is an iterable, it must contain at least num - 1 entries; otherwise a ValueError is raised during iteration.

Return
    A generator that yields control to the inner count loop and returns the results
    of the inner_count plan.

Usage example
    .. code-block:: python

       yield from count(detectors, num=10, delay=0.1)

Additional implementation details
    - The plan constructs metadata (_md) describing detectors, number of points, and plan arguments.
    - If per_shot is not provided, it defaults to bps.one_shot.
    - Uses bpp.stage_decorator and bpp.run_decorator to integrate with the bluesky plan runtime.

list_scan(detectors, *args, per_step=None, md=None)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Description
    Scan over one or more variables in steps simultaneously (inner product).

Parameters
    detectors : list
        List of 'readable' objects (detectors).
    *args
        For one dimension: motor, [point1, point2, ...].
        In general, a sequence of motor, [points] pairs for each dimension.
        Motors can be any 'settable' object (motor, temp controller, etc.).
    per_step : callable, optional
        Hook for customizing action of inner loop (messages per step).
        Signature:
        f(detectors, motor, step) -> plan (generator)
    md : dict, optional
        Metadata for the plan.

Notes
    - Validates that the list of arguments has an even number of elements (motor, points pairs).
    - Ensures all position lists have the same length.
    - Builds plan metadata including plan_pattern and plan_pattern_args.
    - Uses plan_patterns.inner_list_product to construct the full cycler.
    - Delegates to scan_nd(detectors, full_cycler, per_step=per_step, md=_md) to perform the scan.

Return
    A generator that yields control to the inner scan, returning results from scan_nd.

Usage example
    .. code-block:: python

       yield from list_scan([det1, det2],
                            motor1, [0, 1, 2],
                            motor2, [10, 20, 30],
                            per_step=your_step_hook)

Notes
    - The function builds a detailed _md (metadata) payload describing detectors, motors, and the argument structure.
    - If hints are present on motors, default dimensions are attached to the plan via hints.

Dependencies and implementation details
---------------------------------------

- The code relies on:
  - plan_patterns for complex plan patterns (e.g., inner_list_product).
  - preprocessors (bpp) and plan_stubs (bps) for decorators and stubs.
  - A partition function from cytools or toolz for argument packing.
- The count and list_scan plans build and propagate metadata to aid tooling and introspection.
- The rel_list_scan plan is a placeholder for future functionality.

See Also
--------

- bluesky.plans (for reference on similar plans)
- plan_patterns (inner_list_product, etc.)
- scan_nd (inner scanning primitive used by list_scan)

Examples
--------

- Basic count usage:

  yield from count([det1, det2], num=5, delay=0.2)

- List scan with two motors:

  yield from list_scan([det], motorA, [0, 1, 2], motorB, [10, 20, 30])

- Relative list scan (to be implemented):

  yield from rel_list_scan([det], motor, [0.5, 1.0, 1.5])

Notes
-----

- rel_list_scan is currently a placeholder and will raise a NotImplementedError when invoked, pending implementation.
