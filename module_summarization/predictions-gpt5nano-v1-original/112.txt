Metadata
========

Overview
--------

This module provides Bluesky plan generators for acquiring data from detectors and performing scans. It exposes three public plans:

- count: take one or more readings from detectors with optional per-shot customization and metadata.
- list_scan: perform an inner-product scan over one or more motors (settable objects) and collect detector data.
- rel_list_scan: placeholder for a relative scan over one variable (not yet implemented).

Public API
----------

count(detectors, num=1, delay=None, *, per_shot=None, md=None)
    Take one or more readings from detectors.

    Parameters
    - detectors: list of readable objects
    - num: int, optional, number of readings to take (default 1)
    - delay: iterable or scalar, optional, delay between readings (default 0)
    - per_shot: callable, optional, hook for customizing action per step
    - md: dict, optional, metadata

    Notes
    - If num is None, the plan captures data until canceled.
    - When per_shot is not provided, a default one_shot hook from plan stubs is used.
    - Returns a generator plan yielding messages for each step.

list_scan(detectors, *args, per_step=None, md=None)
    Scan over one or more variables in an inner product.

    Parameters
    - detectors: list of readable objects
    - *args: pairs of (motor, [points...]) describing the scan axes
    - per_step: callable, optional, hook for customizing action per step
    - md: dict, optional, metadata

    Notes
    - All point lists must be the same length; otherwise a ValueError is raised.
    - Builds a compact metadata dictionary describing detectors, motors, and the scan pattern.
    - Delegates the actual stepping to scan_nd with a precomputed full cycle generator from plan_patterns.

rel_list_scan(detectors, *args, per_step=None, md=None)
    Scan over one variable in steps relative to the current position.

    Parameters
    - detectors: list of readable objects
    - *args: pairs of (motor, [relative points...])
    - per_step: callable, optional, hook for customizing action per step
    - md: dict, optional, metadata

    Notes
    - This function is currently a placeholder (TODO) and not yet implemented.

Implementation notes
--------------------

- The module uses helper decorators from preprocessors (bpp) and plan stubs (bps) to build executable plans.
- Metadata is assembled into a dictionary (_md) and extended with user-provided md.
- Plans rely on plan_patterns for generating inner-product sequences and are designed to integrate with Bluesky runtimes.
