Pairwise Metrics, Affinities and Kernels
=========================================

Overview
--------
This section documents utilities for computing pairwise quantities between samples. Pairwise computations yield either distances (dissimilarities) or similarities (affinities) between samples in X and, optionally, Y. The implementation supports dense and sparse inputs, input validation, and optional use of precomputed matrices.

Input validation and data types
-------------------------------
- _return_float_dtype(X, Y):
  - Ensures operations use an appropriate floating-point dtype.
  - If both inputs are float32, returns float32; otherwise defaults to Python float.
- check_pairwise_arrays(X, Y, precomputed=False, dtype=None, accept_sparse='csr',
                        force_all_finite=True, copy=False):
  - Converts inputs to numpy arrays (or validates sparse formats) and enforces consistent feature dimensions.
  - If Y is the same as X or is None, X and Y are treated as a single array pair.
  - When precomputed is True, validates that the input represents a valid distance matrix with matching dimensions.
  - Returns safe_X and safe_Y as numpy arrays (or pointers for precomputed cases).
- check_paired_arrays(X, Y):
  - Validates inputs for paired distance metrics, ensuring correct shapes and numeric types.

Pairwise distances (metrics)
----------------------------
- Computes pairwise dissimilarities between samples in X and Y.
- Common metrics are derived from standard distance notions (e.g., Euclidean, Manhattan). The implementation relies on robust input validation and can operate on dense or sparse data.
- Supports optional precomputation: a distance matrix can be supplied directly via the precomputed pathway for reuse.

Affinities and kernels
----------------------
- Affinities represent similarities between samples, often obtained via kernel functions.
- Kernels convert data into a similarity space, producing kernel matrices that can be used in clustering, kernelized learning, and spectral methods.
- Internal fast kernels are provided for specific fast computations, such as:
  - _chi2_kernel_fast: fast computation of the chi-squared kernel for histogram-like data.
  - _sparse_manhattan: efficient handling of Manhattan-like measures on sparse inputs.
- The framework supports computing pairwise kernels with dense or sparse inputs and can operate with precomputed kernel matrices.

Practical notes
---------------
- Precomputed mode: If you already have a distance or kernel matrix, you can bypass feature-wise computations by supplying a precomputed matrix and setting precomputed accordingly.
- Sparse support: Inputs may be CSR (or other accepted formats). If a sparse input is provided in an unsupported format, it will be converted to the first allowed format.
- Efficiency: Parallelization and optimized routines are used where appropriate to speed up large-scale computations.
- Output: The resulting matrix has shape (n_samples_X, n_samples_Y) for standard pairwise computations, or (n_queries, n_indexed) when using precomputed distance matrices as indices.

Example (conceptual)
---------------------
- Compute pairwise distances:
  - D = pairwise_distances(X, Y=None, metric='euclidean')
- Compute pairwise affinities via a kernel:
  - K = pairwise_kernels(X, Y=None, metric='rbf', gamma=0.5)
- Use a precomputed matrix:
  - D = check_pairwise_arrays(precomputed_distance_matrix, None, precomputed=True)
  - or K = check_pairwise_arrays(precomputed_kernel_matrix, None, precomputed=True)

Notes
-----
- The implementation emphasizes robust input handling, dtype control, and support for both dense and sparse data structures.
- Fast paths are available for specific kernel computations to improve performance on large datasets.
