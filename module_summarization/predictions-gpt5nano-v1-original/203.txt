Transforming the prediction target
==================================

In regression tasks, transforming the target variable (the prediction target) can stabilize variance, handle skewness, or improve model performance. This is achieved in scikit-learn with TransformedTargetRegressor, which applies a transformation to the target y before fitting and then inverts the transformation for predictions.

When to use
- y is highly skewed or heteroscedastic and a transformation can stabilize the variance.
- A monotonic transformation (e.g., log, Box-Cox) makes the relationship between X and y more linear for the regressor.
- You want predictions in the original scale, which are obtained by applying the inverse transformation to model outputs.

How it works
- A regressor is wrapped with a target transformation.
- The transformation func is applied to y during fitting.
- Predictions are automatically inverse-transformed to the original scale.

API (high level)
- TransformedTargetRegressor wraps a regressor and applies:
  - func: transformation applied to y before fitting
  - inverse_func: inverse transformation applied to predictions
  - check_inverse: whether to validate that the inverse is consistent

Example
- Fit a model on the log-transformed target and predict in the original scale:

.. code-block:: python

    from sklearn.compose import TransformedTargetRegressor
    from sklearn.linear_model import LinearRegression
    import numpy as np

    reg = TransformedTargetRegressor(
        regressor=LinearRegression(),
        func=np.log1p,
        inverse_func=np.expm1
    )

    reg.fit(X, y)
    y_pred = reg.predict(X)

Notes
- The chosen transformation must have a valid inverse for all predicted values.
- For functions like log, y must be positive (or use a safe variant like log1p for y > -1).
- If you need predictions on the original scale for evaluation, the inverse transformation handles this automatically.
