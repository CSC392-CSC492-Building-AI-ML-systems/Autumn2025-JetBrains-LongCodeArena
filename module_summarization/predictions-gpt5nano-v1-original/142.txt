Delayed API
===========

The dask.delayed interface provides a lightweight, lazy wrapper for building Dask graphs from regular Python code. The core concept is the Delayed object, which represents a value that will be computed as part of a Dask graph.

Delisted objects and the public API
-----------------------------------

- Delayed: The primary object type representing a delayed value in a Dask graph. It is constructed by the delayed factory/function and participates in the Dask graph machinery via standard mixins used by Dask collections.

- delayed: The public factory that creates Delayed objects. It is exposed as part of the module’s public API (see __all__: ["Delayed", "delayed"]).

Key helpers and concepts
------------------------

- finalize(collection)
  - Creates a finalization node for a Dask collection. The function names the finalization key using the collection’s tokenize, collects the collection’s keys, and attaches the finalization operation with its arguments into a HighLevelGraph. Returns a Delayed representing the finalized result.
  - Use: finalize is internally used when normalizing or finalizing a collection within a graph.

- unpack_collections(expr)
  - Normalize a Python object into a task graph:
    - Replaces Delayed values with their keys.
    - Converts literals and collections into forms schedulers can handle.
    - Extracts embedded Dask graphs from enclosed values.
  - Returns a tuple (task, collections) where task is a normalized representation of the computation to perform, and collections is a tuple of sub-collections involved.
  - Handles: Delayed objects, existing Dask collections, lists/tuples/sets, dicts, slices, dataclasses, and namedtuples.
  - Examples (illustrative):
    - a = delayed(1, 'a'); b = delayed(2, 'b')
      task, collections = unpack_collections([a, b, 3])
      task  -> ['a', 'b', 3]
      collections -> (Delayed('a'), Delayed('b'))
    - task, collections = unpack_collections({a: 1, b: 2})
      task  -> (<class 'dict'>, [['a', 1], ['b', 2]])
      collections -> (Delayed('a'), Delayed('b'))

- to_task_dask(expr)  (Deprecated)
  - Deprecated alias of unpack_collections. Warns that the function has been deprecated in favor of unpack_collections.
  - Returns (task, dask) where dask is a merged dictionary-like graph.
  - Behavior mirrors unpack_collections for various Python structures and Delayed values.
  - Note: This helper is deprecated; prefer unpack_collections.

Notes on graph construction
---------------------------

- Keys and graphs are constructed in a way that integrates with Dask’s scheduling and optimization mechanisms:
  - Keys for finalization are named using a prefix (e.g., "finalize-") plus a token derived from the input collection.
  - The Delayed object participates in the graph via standard Dask interfaces, including __dask_keys__ and __dask_postcompute__ for finalization and post-processing of results.

Usage patterns (high level)
---------------------------

- Create delayed computations
  - a = delayed(1, 'a')
  - b = delayed(2, 'b')
  - Combine Delayed objects with Python data to build a graph. Use unpack_collections to inspect how inputs are normalized into a task and a set of sub-graphs.

- Compute or persist
  - Once a graph is built with Delayed objects, compute, visualize, or persist it using Dask’s compute/persist functions, leveraging the Delayed objects in the graph.

Examples
--------

- Basic unpacking example:
  - a = delayed(1, 'a')
  - b = delayed(2, 'b')
  - task, collections = unpack_collections([a, b, 3])
  - task -> ['a', 'b', 3]
  - collections -> (Delayed('a'), Delayed('b'))

- Dictionary-like unpacking:
  - task, collections = unpack_collections({a: 1, b: 2})
  - task -> (dict, [['a', 1], ['b', 2]])
  - collections -> (Delayed('a'), Delayed('b'))

 Why use unpack_collections
 --------------------------

- It provides a robust, recursive normalization of arbitrary Python objects into a task graph suitable for Dask schedulers.
- It handles a variety of Python container types, dataclasses, and namedtuples, preserving input shapes when reconstructing results.
