The Metadata Routing API

Overview
The Metadata Routing API provides utilities to route metadata and method calls from meta-estimators to their constituent sub-estimators. This enables consistent behavior when combining multiple estimators under a single wrapper, such as multi-output or chained meta-estimators, and allows centralized validation and forwarding of fit/predict-related operations to the appropriate sub-estimators.

Public API

- MetadataRouter
  - Core coordinator for routing rules across sub-estimators.
  - Maintains mappings that describe how parent-level methods and metadata should be forwarded to child estimators.
  - Used by meta-estimators to determine when and how to delegate operations to sub-estimators.

- MethodMapping
  - Describes a single rule for forwarding a method or piece of metadata from a parent object to a sub-estimator.
  - Encapsulates how a given parent method or attribute should be resolved on the target sub-estimator.

- process_routing(config)
  - Builds a routing plan from a configuration specification.
  - The configuration describes how metadata and methods should be routed to sub-estimators.
  - Returns a routing object (or plan) that can be consumed by MetadataRouter.

- _routing_enabled(obj)
  - Helper function to check whether routing should be applied for a given object.
  - Used as a gate to enable or disable routing logic in certain contexts.

- _raise_for_params(params, routing)
  - Internal validation helper.
  - Ensures that routing parameters are valid and consistent with the routing plan.

Usage concepts

- Routing purpose
  - Forward appropriate metadata, method calls, and validation checks from a meta-estimator to its sub-estimators.
  - Ensure compatibility across heterogeneous sub-estimators and maintain consistent interfaces.

- Typical integration
  - A meta-estimator defines or discovers sub-estimators.
  - A routing plan is constructed (often via process_routing) to express how calls should be delegated.
  - MetadataRouter applies the routing plan during fit, partial_fit, predict, or other relevant operations.
  - Helpers like _routing_enabled and _raise_for_params guard and validate routing behavior.

- Relationship to other components
  - Works alongside MetaEstimatorMixin, BaseEstimator, and other meta-estimator infrastructure.
  - Used internally by multi-estimator strategies such as MultiOutputRegressor/Classifier and their chaining variants to propagate method calls to sub-estimators.

Notes

- Several core elements are marked private (-prefixed with an underscore) to indicate they are intended for internal use. The public-facing surface is provided by MetadataRouter, MethodMapping, and process_routing, which are designed to be stable integration points for meta-estimators.
