Geometric image transformations
================================

Overview
--------

Geometric image transformations manipulate image coordinates and 3D points under camera models and linear/non-linear mappings. They are essential for camera calibration, 3D reconstruction, image warping, and projecting between world, camera, and image coordinates. This documentation highlights a batched Pinhole camera model and related geometric utilities used for transforming points and coordinates.

Pinhole camera model
--------------------

The PinholeCamera class represents the batched pinhole camera model with full 4x4 calibration (intrinsics) and pose (extrinsics) matrices.

Constructor
-----------

PinholeCamera(intrinsics, extrinsics, height, width)

- intrinsics: tensor of shape (B, 4, 4)
  Full 4x4 camera calibration matrix containing intrinsic parameters.
- extrinsics: tensor of shape (B, 4, 4)
  Full 4x4 rotation-translation matrix encoding pose of the camera.
- height: tensor of shape (B)
  Image height for each batch element.
- width: tensor of shape (B)
  Image width for each batch element.

Note: The class assumes batch form to leverage PyTorch parallelism.

Internal validation
-------------------

The class performs shape validations to ensure consistent batch sizes and valid shapes for intrinsics, extrinsics, height, and width. Valid shapes are described as:

- intrinsics and extrinsics: (B, 4, 4) or (B, N, 4, 4)
- height and width: (B)

Public API (properties)
-----------------------

- intrinsics (B, 4, 4)
  The full 4x4 intrinsics matrix for each batch element.

- extrinsics (B, 4, 4)
  The full 4x4 extrinsics matrix for each batch element.

- batch_size
  The batch size B inferred from intrinsics.

- fx (B)
  Focal length in x-direction, intrinsics[..., 0, 0].

- fy (B)
  Focal length in y-direction, intrinsics[..., 1, 1].

- cx (B)
  Principal point x-coordinate, intrinsics[..., 0, 2].

- cy (B)
  Principal point y-coordinate, intrinsics[..., 1, 2].

- tx (B)
  Translation component along x, extrinsics[..., 0, -1].

- ty (B)
  Translation component along y, extrinsics[..., 1, -1].

- tz (B)
  Translation component along z, extrinsics[..., 2, -1].

- rt_matrix (B, 3, 4)
  Rotation-translation matrix derived from extrinsics, (3x4).

- camera_matrix (B, 3, 3)
  The 3x3 intrinsic camera matrix, intrinsics[..., :3, :3].

- rotation_matrix (B, 3, 3)
  The 3x3 rotation part of the extrinsics, extrinsics[..., :3, :3].

- translation_vector (B, 3, 1)
  The translation vector from extrinsics, extrinsics[..., :3, -1:].

Usage notes
-----------

- The intrinsics and extrinsics are treated in batch form to enable efficient, parallel processing on GPUs.
- The class exposes convenient accessors to commonly used camera parameters (focal lengths, principal point, rotation/translation components, etc.), enabling straightforward geometric calculations and projections.

Geometric utilities (related)
-----------------------------

- inverse_transformation
  Utility to invert a transformation representing a camera or world-to-camera pose.

- transform_points
  Utility to apply a transformation to a set of points, enabling projection, re-projection, or coordinate frame changes.

Usage example (conceptual)
--------------------------

This example demonstrates how the PinholeCamera class fits into geometric image transformations. It is representative; exact usage may vary with the surrounding codebase.

.. code-block:: python

   import torch
   from your_module.geometry.subpix import PinholeCamera
   from kornia.geometry.linalg import transform_points, inverse_transformation

   B, H, W = 4, 480, 640
   intrinsics = torch.eye(4).unsqueeze(0).repeat(B, 1, 1)
   extrinsics = torch.eye(4).unsqueeze(0).repeat(B, 1, 1)
   height = torch.full((B,), H, dtype=torch.int64)
   width  = torch.full((B,), W, dtype=torch.int64)

   cam = PinholeCamera(intrinsics, extrinsics, height, width)

   # Example points in world or camera coordinates
   pts = torch.randn(B, 1000, 3)

   # Transform points using a pose or extrinsic transformation
   # transformed_pts = transform_points(pts, cam.extrinsics)  # depending on library API
