Condition syntax in PyTables

Introduction
- A condition is a boolean expression used to filter rows of a Table or to select data with methods such as read_where, where, or where_list. Conditions are written as strings using column names and Python-like operators. PyTables evaluates the expression against each row and returns only the rows for which the expression is true.

What can be used in a condition
- Column references
  - Refer to a column by its name or by its path within the table’s group hierarchy.
  - Example: age, height, "group1/group2/value"
- Literals
  - Numeric literals (integers and floating point values)
  - String literals (quoted with single or double quotes)
  - Boolean literals: True, False
- Operators
  - Comparison operators: ==, !=, <, <=, >, >=
  - Membership tests: in, not in (the right-hand side is typically a Python-like iterable, e.g. (1, 2, 3) or ["a", "b"])
  - Logical operators: and, or, not
- Parentheses for grouping and operator precedence
  - Expressions can be combined and nested using parentheses to control order of evaluation.
- Supported expression style
  - Expressions resemble Python boolean expressions but must refer only to column names and literals.
  - Examples:
    - age > 21 and status == "active"
    - height >= 1.70 and (gender == "F" or gender == "Female")
    - id in (100, 200, 300) and score <= 99.9

Special considerations
- NaN handling
  - Floating point NaN values follow IEEE 754 semantics in comparisons, which means standard comparisons with NaN evaluate to False. PyTables uses internal nan-aware comparators to ensure consistent behavior when combining conditions; exact nan-handling details are implemented by the internal comparator mechanisms.
- Indexes and performance
  - If a condition references indexed columns, PyTables may use the corresponding index to accelerate evaluation. Complex conditions may benefit from the presence of suitable indexes (see the Index class and indexing documentation for details).
- Path and scope
  - Columns must exist in the target table. Referencing a non-existent column raises an error.
  - For nested group structures, use the full path to the column (group/subgroup/column).

Examples
- Read rows where age is at least 18:
  where('age >= 18')
- Read rows where age is between 18 and 65 and status is "active":
  where('(age >= 18) and (age <= 65) and (status == "active")')
- Read rows where a value is one of several options:
  where('region in ("north", "south", "east")')
- Read rows where score is NaN or below a threshold (NaN handling is internal; use standard comparisons for finite values):
  where('(score != score) or (score < 50)')

Notes
- The condition syntax is designed to be clean and Python-like, focusing on readability and ease of use for quick data filtering.
- For advanced usage and performance tuning (e.g., maximizing index usage), refer to the Index and Table indexing documentation.
