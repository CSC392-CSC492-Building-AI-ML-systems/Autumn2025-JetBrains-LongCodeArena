Controlling chunking
====================

Dask arrays are partitioned into chunks along their axes. The chunk structure determines memory usage, task granularity, and parallelism. You can control chunking at creation, by rechunking, and by using chunk-aware operations that preserve or restructure blocks.

Creating arrays with explicit chunks
------------------------------------

When you create a Dask array, you can specify the chunk sizes per axis with the chunks parameter. For example:

.. code-block:: python

    import numpy as np
    import dask.array as da

    a = np.arange(1_000_000).reshape(1000, 1000)
    x = da.from_array(a, chunks=(250, 250))
    print(x.chunks)
    # ((250, 250, 250, 250), (250, 250, 250, 250))

Rechunking existing arrays
--------------------------

To change the chunking of an existing array, use rechunk:

.. code-block:: python

    y = da.rechunk(x, chunks=(500, 1000))
    print(y.chunks)
    # ((500, 500), (1000, 1000, 1000, 1000, 1000, 1000))

Preserving and controlling chunking with map_blocks and blockwise
---------------------------------------------------------------

Operations can be applied to each chunk independently using map_blocks or blockwise. These functions preserve the chunk layout (unless the operation changes the shape of blocks).

.. code-block:: python

    def f(block):
        return block + 1

    z = x.map_blocks(f, dtype=x.dtype)
    print(z.chunks)
    # same as x.chunks

Masked array operations and chunking
-----------------------------------

Some masked-array helpers are implemented to work block-wise while preserving chunk structure. Internal wrappers apply NumPy masked-array operations to each block via blockwise, ensuring the resulting array maintains the input chunk layout.

.. code-block:: python

    # Conceptual example: per-block masked operation preserved by blockwise
    # (actual implementations in dask.array use this pattern internally)
    import numpy as np
    import dask.array as da

    a = da.from_array(np.ma.masked_array(np.ones((1000, 1000), dtype=float)), chunks=(250, 250))
    m = da.ma.masked_equal  # used internally with blockwise to preserve chunks
    # z = m(a, 1)  # output shares a's chunk layout

Dispatch and backends
---------------------

Dask array uses a dispatch mechanism to route operations to backend-specific implementations. The code defines registries for various operations (for example: concatenate, tensordot, einsum, empty, divide, percentile, numel, nannumel, to_numpy_dispatch, to_cupy_dispatch) so that the correct chunked backend implementation is selected for the active array type. This enables consistent chunking behavior across CPU and accelerator backends.
