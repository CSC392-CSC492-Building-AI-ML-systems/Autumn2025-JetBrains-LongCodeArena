Connecting Steps Together
==========================

Overview
--------

In this workflow, a build is composed by connecting the outputs of one step to the inputs of another. Each Step exposes handles for its inputs and outputs, and these handles can be linked by edges to form a directed graph. An edge represents a single data dependency: the output of the source step flowing into the input of the destination step.

Step handles
------------

- Input handles
  - Obtain a handle to a step input with:
    - step.i('input_name') or step.get_input_handle('input_name')
  - The handle is a tuple of the form: (step_name, 'inputs', 'input_name')

- Output handles
  - Obtain a handle to a step output with:
    - step.o('output_name') or step.get_output_handle('output_name')
  - The handle is a tuple of the form: (step_name, 'outputs', 'output_name')

- Convenience
  - shorthands:
    - step.i(name) → get_input_handle
    - step.o(name) → get_output_handle
  - Retrieve all handles:
    - step.all_input_handles()
    - step.all_output_handles()

Edge: connecting two handles
-----------------------------

Edges connect a source output to a destination input. An edge is created by supplying the two handles to the Edge constructor:

- edge = Edge(src_handle, dst_handle)

Where:
- src_handle = source_step.o('output_name')
- dst_handle = destination_step.i('input_name')

Graph: collecting and connecting edges
--------------------------------------

Create a graph and add edges to define the data flow between steps:

- graph = Graph()
- graph.connect(edge)

You can connect multiple edges to form a complete pipeline. The graph ensures that outputs feed into compatible inputs and can validate connections as needed.

Notes and tips
--------------

- Outputs may be named or tagged. The Step class supports tagged outputs (where an output entry can be a dict). When wiring, use the actual output handle names produced by the step.
- Steps can be cloned for reuse or copied as needed:
  - new_step = Step.__new__(Step)
  - new_step._config = copy.deepcopy(old_step._config)
  - new_step.step_dir = old_step.step_dir
- Steps may have no inputs or no outputs. Use all_input_handles/all_output_handles to inspect what a step provides.
- You can pre- or post-extend commands on a step:
  - step.pre_extend_commands([...])
  - step.extend_commands([...])

Example: connecting two steps
------------------------------

- Create steps
  - A = Step('steps/stepA')
  - B = Step('steps/stepB')

- Connect A's output to B's input
  - src = A.o('data_out')
  - dst = B.i('data_in')
  - edge = Edge(src, dst)

- Add edge to a graph
  - graph = Graph()
  - graph.connect(edge)

This establishes that the data produced by stepA (data_out) will be consumed by stepB (data_in) during the build.
