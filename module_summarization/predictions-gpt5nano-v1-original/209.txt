Analyzing a collection of text documents
=========================================

The text feature extraction utilities in scikit-learn provide a convenient way to convert a collection of text documents into numerical feature vectors. This enables machine learning algorithms to operate on text data. Key components include vectorizers, preprocessing helpers, and stop word handling.

Overview
--------

- The sklearn.feature_extraction.text submodule gathers utilities to build feature vectors from text documents.
- Main vectorizers:
  - CountVectorizer: converts a collection of text documents to a matrix of token counts.
  - TfidfVectorizer: converts documents to a matrix of TF–IDF features.
  - HashingVectorizer: uses the hashing trick to produce feature vectors without an explicit vocabulary.
- Preprocessing and normalization helpers:
  - strip_accents_unicode and strip_accents_ascii: remove or normalize accented characters.
  - strip_tags: remove basic HTML/XML tags.
  - _preprocess: chain optional preprocessing steps (lowercasing, accent handling, etc.).
  - _analyze: chain together preprocessing, tokenization, and n-gram generation (or use an analyzer directly).
- Stop words:
  - ENGLISH_STOP_WORDS provides a built-in list of common English stop words.
  - _check_stop_list converts various stop-word specifications into a usable set.

Preprocessing and normalization
--------------------------------

Text preprocessing is performed before tokenization and n-gram generation. Common steps include:

- Lowercasing:
  - Optional via the _preprocess function.
- Accent handling:
  - strip_accents_unicode: convert accented characters to their basic form but preserve non-ASCII content when possible.
  - strip_accents_ascii: convert accented characters to ASCII equivalents or remove them.
- Tag stripping:
  - strip_tags: remove HTML/XML tags with a basic regular expression.
- Other normalization:
  - Chaining preprocessing steps is supported, enabling consistent handling across the pipeline.

Analyzing a single document
---------------------------

The _analyze function chains together the steps to transform a single document into tokens or n-grams:

- If a decoder is provided, it is applied to the document first.
- If an analyzer is provided, it replaces the preprocessor, tokenizer, and n-gram steps.
- Otherwise, optional preprocessor, tokenizer, and n-grams steps are applied in sequence.
- Stop words may be supplied and are filtered from tokens as needed.

Stop words
----------

- ENGLISH_STOP_WORDS is a built-in stop word list used by default filtering.
- _check_stop_list(stop) resolves different stop-word specifications:
  - "english" → ENGLISH_STOP_WORDS
  - a string that is not a built-in list raises an error
  - None means no stop words
  - a collection is converted to a frozenset

Tokenization and n-grams
------------------------

- Tokenization and n-gram generation are configurable:
  - tokenizer: custom tokenization function
  - preprocessor: custom preprocessing function
  - ngrams: function to produce n-grams from tokens
  - stop_words: optional filtering of common words before or during n-gram construction
- The internal logic supports producing token sequences of varying lengths (unigrams, bigrams, trigrams, etc.) depending on how ngrams is configured.

Vectorizers and common infrastructure
-------------------------------------

- _VectorizerMixin provides common behavior for text vectorizers, including input handling and a basic decoding pipeline:
  - input: how the raw data is provided (e.g., filenames, file-like objects)
  - encoding and decode_error: control how byte data is decoded to text
  - decode(doc): reads and decodes input documents as needed
- decode supports input formats such as:
  - "filename": read from a file path
  - "file": read from a file-like object
- The codebase includes functions to strip accents and tags as part of the preprocessing stage:
  - strip_accents_unicode
  - strip_accents_ascii
  - strip_tags

Main vectorizers
--------------

- CountVectorizer:
  - Builds a sparse document-term matrix of token counts.
  - Tokenization, preprocessing, and stop-word handling are configurable.
- TfidfVectorizer:
  - Builds a sparse matrix of TF–IDF features.
  - Combines term frequency with inverse document frequency to downscale frequent terms.
- TfidfTransformer:
  - Transforms a count matrix to a TF–IDF representation (useful when you already have a count matrix).
- HashingVectorizer:
  - Uses the hashing trick to map tokens to a fixed-dimensional feature space.
  - Useful when a vocabulary is difficult to build or when streaming large datasets.

How to use (high-level)
------------------------

- Create and configure a vectorizer (CountVectorizer, TfidfVectorizer, or HashingVectorizer) with optional:
  - analyzer, tokenizer, preprocessor, ngrams
  - stop_words (including "english" for ENGLISH_STOP_WORDS)
  - strip_accents options or custom accent handling
  - strip_tags if HTML/XML content is present
- Fit the vectorizer on a collection of documents and transform them into a sparse document-term matrix.
- For TF–IDF, either use TfidfVectorizer directly or compose CountVectorizer with TfidfTransformer.

Notes
-----

- Input handling is flexible: vectorizers can accept raw strings, file paths, or file-like objects, with decoding performed as needed.
- The preprocessing and analysis pipeline is designed to be composable, allowing simple defaults or customized processing chains as required by the application.
