w_ipa
=====

WESTPA Interactive Python Analysis (w_ipa) is a WESTPA Parallel Tool that provides an interactive Python environment for examining and analyzing trajectory data from the current analysis scheme and iteration. It is designed to let you run traces, inspect weights and progress coordinates, and explore the current state of the simulation workflow.

Overview
--------
- Welcome to w_ipa (WESTPA Interactive Python Analysis)!
- From here you can run traces, inspect weights, progress coordinates, and other analysis data.
- This is a stateful tool: data pulled is always from the current analysis scheme and iteration.
- By default, the first analysis scheme in west.cfg is used, and you are set at iteration 1.

Getting and using data
----------------------
- ALL PROPERTIES ARE ACCESSED VIA w or west.

- Current iteration:
  - w.iteration
  - west.iteration
  - To set: w.iteration = <value>

- Analysis schemes:
  - List schemes: w.list_schemes
  - Set scheme: w.scheme = <scheme_name_from_list>

- Current analysis scheme data:
  - States and bins: w.states, w.bin_labels

- Information about the current iteration is available in an object called 'current':
  - current contains walkers, summary, states, seg_id, weights, parents, kinavg, pcoord, bins, populations, and auxdata (if it exists)

- Tracing:
  - w.trace(seg_id) runs a trace over a seg_id in the current iteration and returns a dictionary with the seg_id history. Traces can be expensive; it’s advisable to store them if needed.

- Help and discovery:
  - Run help on any function or property for more information!

Runtime options
---------------
w_ipa exposes a set of runtime command-line options to control analysis behavior:
- --analysis-only, -ao
  - Run the analysis and return to the terminal.
- --reanalyze, -ra
  - Delete existing files and reanalyze.
- --ignore-hash, -ih
  - Ignore the argument hash and avoid regenerating files.
- --debug, -d
  - Enable debug output (development use).
- --terminal, -t
  - Plot output in the terminal (text interface).
- -f, --f
  - Extra, suppressed option (internal use).
- Defaults:
  - compression is enabled by default.

Internal tooling and state
--------------------------
- WIPI inherits from WESTParallelTool and coordinates with:
  - A data reader (WESTDataReader) for loading iteration data
  - A progress indicator component (ProgressIndicatorComponent)
- Interface by default is matplotlib; can be switched to text when plotting is requested.
- Core versioning and configuration:
  - version = "1.0B"
  - interface = 'matplotlib' by default
  - _scheme tracks the active analysis scheme
  - _iter starts at 1
  - config_required = True

Data handling and hashing
-------------------------
- hash_args(args, extra=None, path=None)
  - Create a unique hash stamp based on the arguments to determine if outputs need regeneration.
  - Normalizes path elements and prints detailed argument information if debug_mode is enabled.
- stamp_hash(h5file_name, new_hash)
  - Open the HDF5 file and stamp it with the new argument hash; returns the file opened in read-only mode.

Notes
-----
- The tool is designed to pull data from the current analysis scheme and iteration to ensure consistency.
- Traces can be computationally expensive; consider storing results when exploring history.
- Access and manipulate the current state via the w namespace (or west) for interactive exploration.
