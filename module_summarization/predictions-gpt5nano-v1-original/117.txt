Resources
=========

Overview
--------
This module provides support for plugins that provide access to resources (for example, schemas). It defines:
- a DirectoryResourceMapping that reads resource content from a directory or directory tree
- a ResourceMappingProxy that wraps a resource mapping and carries package information
- a ResourceManager that combines multiple mappings into a single interface
- a JsonschemaResourceMapping that fetches metaschemas from the jsonschema package
- a helper get_json_schema_resource_mappings to obtain the standard jsonschema mappings

Public API
-----------


DirectoryResourceMapping
------------------------
A resource mapping that reads resource content from a directory or directory tree.

- Inherits from asdf_standard.DirectoryResourceMapping
- See the class docstring for details:
  "A resource mapping that reads resource content from a directory or directory tree. See :class:`~asdf_standard.resource.DirectoryResourceMapping` for details."

ResourceMappingProxy
--------------------
Wrapper around a resource mapping that carries additional information on the providing package.

Key features:
- maybe_wrap(delegate): returns a ResourceMappingProxy wrapping the delegate, or returns the delegate if already wrapped
- __init__(delegate, package_name=None, package_version=None): validates the delegate implements Mapping and stores package information
- __getitem__, __len__, __iter__: delegate to the wrapped mapping
- delegate: access to the wrapped mapping
- package_name, package_version: information about the providing package (or None if added at runtime)
- class_name: fully qualified class name of the mapping
- __eq__, __hash__, __repr__: identity and representation behavior

ResourceManager
-------------
Wraps multiple resource mappings into a single interface with simple, predictable lookup behavior.

- __init__(resource_mappings): accepts an iterable of mappings; if multiple mappings define the same URI, the first mapping takes precedence
- __getitem__(uri): returns the resource content for a given URI, raising KeyError if unavailable
- __len__, __iter__, __contains__: provide standard Mapping-like behavior
- __repr__: concise representation including the number of resources

JsonschemaResourceMapping
-------------------------
Resource mapping that fetches metaschemas from the jsonschema package.

- __getitem__(uri): looks up the corresponding filename from _JSONSCHEMA_URI_TO_FILENAME and returns the resource data from jsonschema's package
- __len__, __iter__, __repr__: standard Mapping behavior
- _JSONSCHEMA_URI_TO_FILENAME: mapping of well-known JSON Schema URIs to filenames used inside the jsonschema package (e.g., "http://json-schema.org/draft-04/schema" -> "draft4.json")

get_json_schema_resource_mappings
-------------------------------
Returns a list of JsonschemaResourceMapping instances.

Implementation detail:
- Currently returns [JsonschemaResourceMapping()]

Notes
-----
- Exposed names (via __all__): ResourceMappingProxy, DirectoryResourceMapping, ResourceManager, JsonschemaResourceMapping, get_json_schema_resource_mappings
- This module facilitates plugin-based access to resources (especially schemas) and provides convenient composition and metadata about the source package. 

Usage example
-------------
Here is a minimal usage pattern (illustrative):

.. code-block:: python

   from resources import (
       DirectoryResourceMapping,
       ResourceMappingProxy,
       ResourceManager,
       get_json_schema_resource_mappings,
   )

   # Create a mapping from a directory
   dir_map = DirectoryResourceMapping("/path/to/resources")
   proxy = ResourceMappingProxy(dir_map, package_name="my_plugin", package_version="0.2.0")

   # Combine mappings
   manager = ResourceManager([proxy] + get_json_schema_resource_mappings())

   # Retrieve a resource by URI
   data = manager["my/resource.json"]  # bytes or str depending on mapping
