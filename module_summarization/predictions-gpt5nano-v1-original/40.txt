How to generate Pydantic Models from Tortoise Models
====================================================

This document describes how to generate Pydantic models from Tortoise ORM models using the pydantic_model_creator helper and the PydanticMeta configuration.

Overview
--------

- Generate a Pydantic model class that mirrors a Tortoise Model.
- The generator returns a subclass of PydanticModel suitable for data validation and serialization with Pydantic.
- You can customize the generated model via a nested PydanticMeta class on the Tortoise model and via parameters to pydantic_model_creator.

API
---

pydantic_model_creator
~~~~~~~~~~~~~~~~~~~~~~~

Signature

- pydantic_model_creator(cls, *, name=None, exclude: Tuple[str,...] = (), include: Tuple[str,...] = (), computed: Tuple[str,...] = (), allow_cycles: Optional[bool] = None, sort_alphabetically: Optional[bool] = None, _stack: tuple = (), exclude_readonly: bool = False) -> Type[PydanticModel]

What it does

- Builds and returns a Pydantic model class representing the given Tortoise Model (cls).
- The returned type is a subclass of PydanticModel.
- The created model respects field visibility and “computed” fields defined on the source model.

Parameters

- cls: The Tortoise Model class to convert.
- name: Custom name for the generated Pydantic model. If omitted, a deterministic name is created from the model and options.
- exclude: Tuple[str, ...] of field names to exclude from the Pydantic model.
- include: Tuple[str, ...] of field names to include from the Tortoise model.
- computed: Tuple[str, ...] of computed field names to include in the Pydantic model.
- allow_cycles: If True, allow cycles in recursive/self-referential models; otherwise cycles are guarded.
- sort_alphabetically: If True, sort fields alphabetically in the generated model.
- exclude_readonly: If True, build a subset model that excludes read-only fields.
- _stack: Internal recursion tracking; do not rely on external usage.
- The function uses internal recursion protection to avoid infinite expansion in recursive schemas.

PydanticMeta (on the Tortoise Model)
------------------------------------

You may configure generated Pydantic models by declaring a nested PydanticMeta class inside your Tortoise Model.

Example:

.. code-block:: python3

    class User(Model):
        id = fields.IntField(pk=True)
        username = fields.CharField(max_length=50)
        password = fields.CharField(max_length=128)

        class PydanticMeta:
            exclude = ("password",)
            computed = ("full_name",)

            # Optional controls (defaults shown)
            backward_relations: bool = True
            max_recursion: int = 3
            allow_cycles: bool = False
            exclude_raw_fields: bool = True
            sort_alphabetically: bool = False

Fields

- include: Only fields listed here will be included in the Pydantic model.
- exclude: Fields listed here will be excluded from the Pydantic model.
- computed: Names of computed properties to expose in the Pydantic model.
- backward_relations: Use backward relations without annotations (not recommended for large data).
- max_recursion: Maximum allowed recursion depth for nested models.
- allow_cycles: Allow cycles in recursion (use with care).
- exclude_raw_fields: Exclude raw relation fields (ending with _id) from the Pydantic model.
- sort_alphabetically: If True, sort fields alphabetically; otherwise keep declaration order.

Usage Examples
--------------

Basic usage: generate a Pydantic model for a simple Tortoise model.

.. code-block:: python3

    from tortoise.models import Model
    from tortoise import fields
    from tortoise.contrib.pydantic import pydantic_model_creator

    class User(Model):
        id = fields.IntField(pk=True)
        username = fields.CharField(max_length=50)

    UserPyd = pydantic_model_creator(User)

This creates a Pydantic model class (UserPyd) with the fields mirrored from User.

Advanced: customize fields with PydanticMeta

.. code-block:: python3

    class User(Model):
        id = fields.IntField(pk=True)
        username = fields.CharField(max_length=50)
        email = fields.CharField(max_length=100, null=True)

        @property
        def display_name(self) -> str:
            return self.username or self.email or ""

        class PydanticMeta:
            exclude = ("email",)
            computed = ("display_name",)

    UserPyd = pydantic_model_creator(User)

The generated Pydantic model will include display_name as a computed field and exclude email.

Custom model name

.. code-block:: python3

    UserPD = pydantic_model_creator(User, name="UserPublicModel")

Notes
-----

- The pydantic_model_creator function generates a Pydantic model class at runtime.
- You can create multiple variants of the same Tortoise model by providing different include/exclude/computed options; each variant will have a deterministic generated name (or the custom name you supply).
- Recursion protection ensures that deeply nested or self-referential relationships do not produce infinite models, honoring max_recursion and allow_cycles.

Importants
-----------

- Import path for the creator:
  from tortoise.contrib.pydantic import pydantic_model_creator
- Generated models are standard Pydantic models (subclasses of PydanticModel) and can be used wherever Pydantic models are accepted (validation, serialization, etc.).

This documentation provides the essentials to generate and customize Pydantic models from Tortoise ORM models using pydantic_model_creator and PydanticMeta.
