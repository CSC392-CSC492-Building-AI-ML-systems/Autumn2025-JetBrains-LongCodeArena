Camera geometry
================

Work with images or data cubes related to Cherenkov Cameras
----------------------------------------------------------------

This section describes how to use camera geometry to work with per-event images and with data cubes built from many events. The CameraGeometry class provides the layout of the camera’s pixels (their positions, shapes, and sometimes areas), which is essential for turning per-pixel measurements into viewable images or into stacked data cubes for analysis.

Key concepts
- Pixel layout: The camera geometry exposes the pixel coordinates (e.g., pix_x, pix_y) that define where each camera pixel lies in a 2D plane.
- Per-event images: A single event is typically represented as a 1D array of pixel values (one value per pixel). To visualize or analyze, you map these values to a 2D representation using the pixel coordinates.
- Data cubes: By stacking many per-event images along a new axis, you obtain a 3D data cube (events × pixels, or events × image grid), which can be used for time-resolved studies, stacked analyses, or advanced visualization.

Working with a single event: turning per-pixel data into an image or visualization
- Access the camera geometry for the telescope/camera you are analyzing.
- Use the pixel coordinates (pix_x, pix_y) to map each pixel value to a 2D representation.
- Visualization options:
  - Scatter plot: visualize per-pixel values directly at their (pix_x, pix_y) locations.
  - 2D image: create a regular grid and place values at corresponding locations (nearest-neighbor or interpolation). Note that many Cherenkov cameras have irregular geometries, so a regular rectangular grid may require interpolation or masking of empty regions.

Example (conceptual; adapt to your codebase)
- Given:
  - geometry: a CameraGeometry instance
  - values: 1D array with length geometry.n_pixels (one value per pixel)

  - Scatter visualization (no regular grid required):
    - plt.scatter(geometry.pix_x, geometry.pix_y, c=values, s=5, cmap='viridis')
    - plt.colorbar(label='Charge (a.u.)')
    - plt.xlabel('pix_x'); plt.ylabel('pix_y'); plt.title('Camera Image (per-pixel values)')

  - Regular grid image (if you need a 2D array for image display):
    - Define a grid covering min/max of pix_x and pix_y.
    - Create an empty 2D array (height × width) filled with NaNs or a background value.
    - For each pixel i, assign grid_y[i], grid_x[i] <- values[i] (nearest neighbor or interpolated).
    - Display with imshow, masking NaNs as needed.

Data cubes: stacking many events
- Build a data cube by stacking per-event images along a new axis, e.g.:
  - cube = np.stack([image_from_event(e) for e in events], axis=0)
  - The shape will be (n_events, n_pixels) if you keep a flat pixel axis, or (n_events, height, width) if you map to a 2D grid per event.
- If you keep the irregular geometry, you may store the cube as (n_events, n_pixels) and map to 2D for visualization on a per-event basis, or convert to a regular grid when possible for quick visualization or certain analyses.
- Common workflows:
  - Time-resolved analyses: treat the first axis as time (event index) and analyze evolution of pixel values.
  - Image-based statistics: compute mean, median, or PCA across the event axis after mapping to a common image representation (either via flat pixel space or a regular grid).

Tips and best practices
- Use the geometry’s pixel coordinates to avoid relying on hard-coded image shapes that don’t match the camera.
- When visualizing, prefer scatter or masked imshow when dealing with irregular camera geometries.
- For large data sets, consider lazy loading or chunked processing, and store cubes in formats that support efficient slicing (e.g., HDF5, Zarr) rather than loading all events into memory.

References to common tools
- CameraGeometry provides pixel positions (pix_x, pix_y) and the mapping to a 2D plane for visualization and analysis.
- For multi-event cubes, standard numpy operations (stack, mean, std, etc.) and, when needed, array masking with NaNs for missing pixels.

Example snippet (conceptual)
- Visualize a single-event 'values' with per-pixel coordinates:
  - geometry: CameraGeometry
  - values: 1D array, length geometry.n_pixels

  - plt.scatter(geometry.pix_x, geometry.pix_y, c=values, s=5, cmap='viridis')
  - plt.colorbar(label='Charge (a.u.)')
  - plt.xlabel('pix_x'); plt.ylabel('pix_y'); plt.title('Camera Image')

- Build a simple data cube across N events (using a 2D grid representation per event when possible):
  - images = [map_event_to_grid(e, geometry) for e in events]  # returns 2D arrays or NaN-masked arrays
  - cube = np.stack(images, axis=0)  # shape: (n_events, height, width) or (n_events, n_pixels)

This documentation provides guidance for working with images and data cubes in Cherenkov camera data using the camera geometry abstraction. For concrete function names and utilities, refer to your codebase’s CameraGeometry implementation and visualization helpers.
