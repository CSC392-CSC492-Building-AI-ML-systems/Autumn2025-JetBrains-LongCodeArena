Imputation (MICE) module
========================

Overview
--------

This module implements the Multiple Imputation through Chained Equations (MICE)
approach for handling missing data in statistical analyses. The process
follows these general steps:

- Impute each missing value with a default or specified method (e.g., mean
  imputation initially).
- For each variable with missing values (the focus variable), fit an
  imputation model (a regression of the focus variable on observed and
  current imputed values of the other variables) and impute its missing values
  (currently using predictive mean matching, PMM).
- After imputing all variables, fit the analysis model to the completed data.
- Repeat cycles and combine results using a combining rule to obtain
  point estimates and standard errors for the analysis model parameters.

Imputations for each variable rely on an imputation model specified by a model
class and a regression formula. The default model is ordinary least squares (OLS),
with a formula that includes main effects for all other variables.

Two primary usage modes are supported:

- Imputed data sets only: Use MICEData to wrap a data frame and perform imputations.
  Summary plots and a history mechanism are available for assessing imputation
  performance.

- Imputed data sets for analysis: Use MICE with an analysis model. After running,
  results can be combined with the combine method, and summary plots are available
  for inference and diagnostics.

Terminology
-----------

- Analysis model: The model whose parameters are of interest for inference.
  If not provided, the focus is on generating imputed data sets for future use.

- Imputation model: A regression model used to impute a particular variable with
  missing values, conditioned on a subset or all of the other variables,
  including transformations and interactions as needed.

- Perturbation method: Strategy for setting the parameter estimates in an
  imputation model. Supported methods include:
  - gaussian: Draws parameters from the Gaussian approximation to the posterior
    (typically after a fit).
  - boot: Uses a fitted parameter vector from a bootstrap-resampled data set.

Class structure
---------------

Two main classes are provided:

- MICEData
  - Wraps a Pandas DataFrame and stores per-variable imputation models.
  - Used to generate multiply imputed data sets that can be processed further
    or shared with others.
  - Offers plotting facilities to visualize imputation results and missing-data
    patterns.
  - history_callback hook (optional): A function invoked after each complete
    imputation cycle; its return value is stored for later analysis.

  Example usage:
  .. code-block::

     imp = mice.MICEData(data)
     imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
     for j in range(20):
         imp.update_all()
         imp.data.to_csv('data%02d.csv' % j)

  Another usage pattern using the object as an iterator:
  .. code-block::

     imp = mice.MICEData(data)
     j = 0
     for data in imp:
         imp.data.to_csv('data%02d.csv' % j)
         j += 1

- MICE
  - Accepts a MICEData object and an analysis-model specification.
  - Runs the multiple imputation, fits the analysis model(s), and combines
    the results to produce a MICEResults object.
  - The MICEResults object provides a summary method to view key estimands and
    inferential quantities.

  Example usage (conceptual):
  .. code-block::

     mice_model = mice.MICE(imp, analysis_model_formula)
     mice_model.run()
     results = mice_model.combine()
     print(results.summary())

Notes
-----

- By default, MICEData is memory-conscious: it saves minimal information across
  iterations. The input data is copied on entry and overwritten with new
  imputations as the process proceeds. If using MICE, fitted analysis models and
  results are saved for later inspection.
- MICEData includes a history_callback hook to persist arbitrary information from
  intermediate datasets for future use.
