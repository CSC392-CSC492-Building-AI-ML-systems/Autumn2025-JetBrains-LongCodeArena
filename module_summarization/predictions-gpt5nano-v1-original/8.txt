Using review sessions
=====================

Overview
--------
A review session provides an isolated, configurable environment for interacting with an ftrack server. It supports automatic data population, optional event hub connectivity for non-local events, configurable caching, and plugin discovery. The session records pending operations and can be customized to fit review workflows.

Getting started
---------------
Prerequisites
- Access to an ftrack server.
- API key and API user (or environment variables).

Creating a review session
-------------------------
Example (Python):
.. code-block:: python

    from ftrack_api import Session

    session = Session(
        server_url='https://ftrack.myserver',
        api_key='YOUR_API_KEY',
        api_user='YOUR_USERNAME',
        auto_populate=True,
        plugin_paths=['/path/to/plugins'],
        cache=None,                # Optional: a cache implementing ftrack_api.cache.Cache
        cache_key_maker=None,      # Optional: a cache key maker
        auto_connect_event_hub=True,
        schema_cache_path='/path/to/schema_cache',
        plugin_arguments={}
    )

Notes
- If server_url is omitted, the environment variable FTRACK_SERVER is used.
- If api_key is omitted, FTRACK_API_KEY (or FTRACK_APIKEY for backwards compatibility) is used.
- If api_user is omitted, FTRACK_API_USER or the current system user (via getpass) is used.

Configuration options
---------------------
- auto_populate: When True, accessing entity attributes will fetch data from the server if not already loaded.
- plugin_paths: List of paths to search for plugins.
- cache: A cache instance implementing ftrack_api.cache.Cache, or a callable that returns one.
- cache_key_maker: Instance implementing ftrack_api.cache.KeyMaker; defaults to StringKeyMaker.
- auto_connect_event_hub: If True, the embedded event hub will be connected in the background to support non-local events.
- schema_cache_path: Path to enable schema caching. Set to False to disable, or omit to use environment variable FTRACK_API_SCHEMA_CACHE_PATH or a temporary directory.
- plugin_arguments: Optional mapping of arguments to pass to plugin registration.

Operational behavior
--------------------
- The session tracks pending operations via session.recorded_operations and operates under session.record_operations being enabled.
- A top-level memory cache is used as part of a layered cache setup.

Event hub usage
--------------
- The session can automatically connect to the event hub in a background thread when auto_connect_event_hub is True.
- Subscribing to and publishing non-local events requires an active hub connection.

Caching and schema behavior
---------------------------
- Schema caching improves performance by caching schema information locally when enabled.
- The schema cache path is configurable; environment variable FTRACK_API_SCHEMA_CACHE_PATH is consulted if provided.
- Disable caching by setting schema_cache_path to False.

Working with review data
------------------------
- Accessing entities will auto-populate data if needed.
- Use the ftrack_api.query and related collection APIs to locate items under review and inspect or modify them.
- Changes can be recorded in the session’s pending operations and later processed as part of the review workflow.

Shutting down and cleanup
-------------------------
- When finished, properly close the session to ensure resources, caches, and the event hub (if connected) are cleaned up.

Environment reference
---------------------
- FTRACK_SERVER: default server URL.
- FTRACK_API_KEY and FTRACK_API_USER: credentials for authentication.
- FTRACK_API_SCHEMA_CACHE_PATH: path to cache schema information.
- Optional: plugin directories via plugin_paths in the Session constructor.

See also
--------
- ftrack_api.cache for caching interfaces
- ftrack_api.event.hub for event hub usage
- ftrack_api.query for querying entities
- ftrack_api.operation for recording operations
