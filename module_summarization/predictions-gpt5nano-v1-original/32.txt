w_run
=====

Start/continue a WEST simulation.

Synopsis
--------

w_run [OPTIONS]

Description
-----------

w_run is the command-line entry point for starting or continuing a WEST simulation. It builds the WestPA runtime environment by:

- parsing WestPA runtime configuration options (via westpa.rc.add_args)
- parsing work-manager options (via work_managers.environment.add_wm_args)
- optionally propagating only a single segment (via --oneseg)

It then creates and configures the active work manager and the core WESTPA drivers (simulation manager, system driver, data manager, propagator, and WE driver), wires them together, and executes the simulation under the work manager.

Options
-------

- --oneseg
  - dest: only_one_segment
  - action: store_true
  - help: only propagate one segment (useful for debugging propagators)

- WestPA runtime configuration options
  - Provided by westpa.rc.add_args(parser). These control runtime behavior of the WESTPA environment.

- Work manager options
  - Provided by work_managers.environment.add_wm_args(parser). These configure how the work manager runs (e.g., parallelization, resource management).

Behavior
--------

The command flow is:

- Instantiate the work manager and assign it to westpa.rc.work_manager via make_work_manager().
- Retrieve the sim_manager, system driver, data manager, propagator, and WE driver from westpa.rc.
- Wire components together:
  - propagator.system = system
  - data_manager.system = system
  - we_driver.system = system
  - sim_manager.data_manager = data_manager
  - sim_manager.system = system
  - sim_manager.propagator = propagator
  - sim_manager.we_driver = we_driver
- Enter the work manager context:
  - If master:
    - install SIGINT handler
    - load plugins
    - prepare_run()
    - run()
    - finalize_run()
    - Handle KeyboardInterrupt and other exceptions, reporting status via westpa.rc.pstatus and logging errors/tracebacks
  - If not master:
    - run the work manager

Notes
-----

- This command centralizes the run lifecycle (prepare, run, finalize) and exception handling for WEST simulations.
- All critical components are accessed through the WESTPA runtime configuration (westpa.rc) and are wired before execution.
