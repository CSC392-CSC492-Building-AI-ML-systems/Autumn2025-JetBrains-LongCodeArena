Miscellaneous features
=====================

This section summarizes a few internal utilities used by astropy.io.fits for miscellaneous tasks, including a lightweight verification framework and an internal notifier mixin.

Verification utilities
----------------------

Astropy.io.fits includes a small internal verification framework to check values and report or fix issues encountered during operations. The key components are:

- VerifyError and VerifyWarning
  - Exception class used when verification fails.
  - Warning class used to report verification issues without raising exceptions.

- VERIFY_OPTIONS
  - Supported option strings for verification, including:
    - "ignore", "warn", "exception", "fix", "silentfix"
    - Combinations of "fix" or "silentfix" with "+ignore", "+warn", or "+exception" (e.g., "fix+warn").

- _Verify class
  - Shared methods for performing verification and reporting results.
  - run_option(option="warn", err_text="", fix_text="Fixed.", fix=None, fixable=True)
    - Executes a single verification option. Returns a tuple (fixable, text).
    - Behavior:
      - If option is "warn" or "exception", fixes are not attempted (fixable = False).
      - If the error is unfixable, prefixes the message with "Unfixable error: ".
      - If fix is provided and the error is fixable, calls fix() and appends the fix_text to the message.
  - verify(option="warn")
    - Verifies all values in the instance.
    - option must be one of the recognized options (see VERIFY_OPTIONS); otherwise, raises ValueError.
    - "ignore" returns immediately (no reporting).
    - Splits combined options (e.g., "fix+warn") into fix_opt and report_opt.
    - Handles “silentfix” behavior (suppressing output for fixable errors if appropriate).
    - Collects and formats error messages, optionally warning or raising an exception based on the chosen options.
    - Includes a note about zero-based indexing: "Note: astropy.io.fits uses zero-based indexing."

- _ErrList
  - A nested list structure used to hold verification errors from different class levels.
  - __init__(val=(), unit="Element")
    - Initializes the error list and a unit label (used when printing).
  - __str__()
    - Returns a string representation of the error list.
  - iter_lines(filter=None, shift=0)
    - Iterates through the nested error structure, yielding lines with indentation appropriate to their level.
    - Helps format hierarchical error messages for display.

Notes
- The verification system is designed for internal use and flexible reporting and fixing of issues encountered during FITS file handling.
- Output is designed to be readable and informative, including appropriate headers and indentation to reflect the structure of the errors.

Notifier mixin for internal event listening
-------------------------------------------

NotifierMixin provides a lightweight facility for objects to notify registered listeners about changes.

- Purpose
  - Allows internal components to register listeners to changes on an object.
  - Uses weak references for listeners to avoid reference cycles and unnecessary retention.

- API (all methods are intended for internal use; underscored)
  - _add_listener(listener)
    - Registers a listener with a weak reference so it can be garbage-collected if no other references exist.
  - _notify(change_type, *args, **kwargs)
    - Notifies all registered listeners by calling a corresponding update method on each listener, if present.
    - The notification calls listener._update_change_type(*args, **kwargs).
  - Behavior
    - If a listener does not implement the required update method, it is ignored (no error raised).
  - Example
    - A minimal example showing a widget-like class that registers for notifications and updates its state when notified.

Example (from the documented usage)
-----------------------------------
A small illustration of how a class can participate in the notification system:

.. code-block:: python

   class Widget(NotifierMixin):
       state = 1

       def __init__(self, name):
           self.name = name

       def update_state(self):
           self.state += 1

   # Usage would involve creating a notifier instance, registering listeners,
   # and calling _notify with a change type to propagate updates.

Internal usage
- These features are designed for internal coordination and robustness of the astropy.io.fits codebase.
- They are not intended as public, user-facing APIs.
