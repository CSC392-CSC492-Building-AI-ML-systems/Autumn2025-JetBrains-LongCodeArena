w_states
========

Display or manipulate basis (initial) or target (recycling) states for a WEST simulation.

Synopsis
--------

w_states [options]

Description
-----------

w_states displays or updates the basis (initial) or target (recycling) states used by a WEST simulation. By default, states are displayed (or dumped to files). You may replace the current states for the next iteration or append new states to the existing set (note: appending basis states is not permitted).

Ground rules and notes
- Appending basis states is not permitted, as this would require renormalizing basis state probabilities in ways that may be error-prone. To add new basis states, edit a file produced by w_states and then update using --replace with that file.
- States can be read from or written to files:
  - Basis states: --bstate-file
  - Target states: --tstate-file
- Basis states are represented as strings of the form: label,probability[,auxref]
- Target states are represented as: label,pcoord0[,pcoord1[,...]]
- In show mode, w_states writes the current states to the specified files (or to stdout if no file is given) and prefixes the output with a header.

Modes of operation
------------------

--show
    Display current basis/target states (or dump to files).
--append
    Append the given basis/target states to those currently in use.
--replace
    Replace current basis/target states with those specified.

Notes:
- The mode defaults to show if not specified.
- In replace mode for basis states, the tool aggregates states from the file (--bstate-file) and any --bstate arguments, validates total probability sums to unity, rescales if necessary, assigns progress coordinates, and updates the in-memory and on-disk state representations for the next iteration.

Options
-------

--bstate-file BSTATE_FILE
    Read (--append/--replace) or write (--show) basis state names, probabilities, and data references from/to BSTATE_FILE.

--bstate
    Add a basis state to the list, specified as a string 'label,probability[,auxref]'.
    May be specified multiple times; states are appended in the order given.

--tstate-file TSTATE_FILE
    Read (--append/--replace) or write (--show) target state names and representative progress coordinates from/to TSTATE_FILE.

--tstate
    Add a target state to the list, specified as a string 'label,pcoord0[,pcoord1[,...]]'.
    May be specified multiple times; states are appended in the order given.

Examples
--------

- Show current states (basis and target) to stdout:
  w_states

- Show basis states to a file:
  w_states --show --bstate-file=bstates.txt

- Replace basis states using a file and inline definitions:
  w_states --replace --bstate-file=bstates.txt --bstate="newlabel,0.3" --bstate="another,0.7,aux"

- Replace target states using a file:
  w_states --replace --tstate-file=tstates.txt

Implementation notes
--------------------

- In show mode, basis and target states are retrieved from the data manager and written to files (or stdout) with headers identifying the iteration.
- In replace mode, basis states are constructed from a file and/or command-line specifications, their probabilities validated and possibly rescaled to sum to unity, and then assigned progress coordinates before being written back to the WEST HDF5 backing store.
- Target state processing in the provided snippet is incomplete; the available code shows the basis-state handling path in replace mode and standard read/write for show mode.
