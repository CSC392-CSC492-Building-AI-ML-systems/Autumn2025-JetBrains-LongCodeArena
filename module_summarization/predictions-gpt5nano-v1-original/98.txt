Contracts API
=============

Overview
--------

This module provides a base class, Contract, used to generate and interact with
smart contract proxies via a Web3 connection. Subclasses are typically created
from Solidity contract ABIs using a factory (e.g., construct_contract_factory)
and then used to deploy contracts, encode function calls, and subscribe to
contract events.

Contract class
--------------

The Contract class serves as the base for all generated contract proxies. It is
designed to be subclassed from a factory that supplies ABI, bytecode, and source
information. The class itself focuses on interaction patterns with deployed
contracts.

Initialization and attributes
-----------------------------

- web3: Class-level attribute that must be initialized before instances can be used.
  If this is None, instantiation will raise an AttributeError instructing to use the
  Web3 contract interface to create the contract class.

- _abi, _code, _code_runtime, _source: Class-level attributes that can be overridden
  at the instance level to provide ABI, deployed bytecode, runtime bytecode, and
  source code respectively.

- address: Instance-level attribute representing the deployed contract address.

Construction
------------

__init__(self, abi=None, address=None, code=None, code_runtime=None, source=None)

- Creates a new smart contract proxy object.
- If any of abi, code, code_runtime, or source are provided, they override the
  corresponding class-level values for this instance.
- Sets the contract address.

Properties
----------

- abi: Returns the contract ABI. If not provided, raises AttributeError with guidance
  that a contract ABI must be specified.

- code: Returns the contract bytecode. If not provided, raises AttributeError.

- code_runtime: Returns the runtime bytecode. If not provided, raises AttributeError.

- source: Returns the contract source. If not provided, raises AttributeError.

Public API
----------

deploy(transaction=None, args=None, kwargs=None)
-----------------------------------------------

Deploys the contract onto the blockchain.

- transaction: Optional dict of transaction parameters (e.g., from, value).
- args: Positional constructor arguments for the contract.
- kwargs: Keyword constructor arguments for the contract.
- Returns: The transaction hash of the deployment.

Notes:
- The deployment requires that the class provides bytecode (code).
- Certain keys ('data', 'to') must not be present in the deployment transaction.
- The constructor data is encoded via _encode_constructor_data and included as the
  transaction data before sending.

encodeABI(fn_name, args=None, kwargs=None, data=None)
-----------------------------------------------

Encodes the arguments for a contract function (or a pre-constructed function selector)
into the Ethereum ABI format.

- fn_name: Name of the contract function.
- args: Positional arguments for the function.
- kwargs: Keyword arguments for the function.
- data: Optional; if provided, is used as the data portion of the call. If not
  provided, the 4-byte function selector is used as the data.
- Returns: The ABI-encoded function call data as text.

Notes:
- This is a classmethod decorated with coerce_return_to_text to ensure a text
  representation is returned.

on(event_name, filter_params=None, *callbacks)
----------------------------------------------

Registers a callback to be triggered when a matching event is observed.

- event_name: The event name to listen for.
- filter_params: Optional dict of filter parameters for event filtering. May contain
  a 'filter' key whose value specifies argument filters.
- callbacks: One or more callback callables to be invoked when events are detected.

Process:
- Determines the matching event ABI.
- Constructs event filter parameters based on the contract address and provided filters.
- Creates a log filter, assigns data filters and a log entry formatter, and returns
  a filter handle (note: the code snippet ends with wiring up the filter and formatter).

Usage examples
--------------

- Subclass creation and deployment (conceptual; specifics depend on the factory used):

  - Define a contract class from an ABI, then deploy with optional constructor args.

- Encoding a function call:

  - encoded = MyContract.encodeABI('transfer', args=(recipient, amount))

- Deploying a contract:

  - txn_hash = MyContract.deploy(transaction={'from': my_account, 'value': 0}, args=(constructor_arg1,))

- Listening to events:

  - contract_instance.on('Transfer', filter_params={'from': my_account}, callback1, callback2)

Notes and error handling
------------------------

- If Contract.web3 is not initialized, attempting to create an instance raises an
  AttributeError explaining how to initialize the contract class via the Web3
  interface.

- Deploy requires that the contract class provides bytecode; otherwise a ValueError
  is raised.

- The on method wires up a log filter and associates data and formatting logic for
  event processing.

See also
--------

- web3.contract (factory utilities for creating contract proxy classes)
- web3.utils and eth_abi utilities used by the contract proxy for ABI encoding/decoding,
  event filtering, and data formatting.
