Multiclass and multioutput estimators
=====================================

Overview
--------
This module provides meta-estimators that extend single-output estimators to handle multiple target variables (multioutput) for both regression and classification. Each estimator wraps a base estimator supplied at construction time and applies it to every target (or to a chain of targets). The primary benefit is reusing a familiar estimator to tackle multioutput problems without implementing new algorithms from scratch.

Estimators
----------

MultiOutputRegressor
^^^^^^^^^^^^^^^^^^^
- Purpose: Extend a regressor to multioutput regression by fitting one regressor per target.
- What it does: For a dataset with y having shape (n_samples, n_outputs), it clones the base estimator and fits one instance to each column y[:, i].
- Key arguments:
  - estimator: estimator implementing fit and predict.
  - n_jobs: number of parallel jobs (if supported by the base estimator).
- Typical usage:
  - from sklearn.multioutput import MultiOutputRegressor
  - from sklearn.linear_model import LinearRegression
  - regr = MultiOutputRegressor(LinearRegression())
  - regr.fit(X, y)
  - y_pred = regr.predict(X)
- Notes: Predictions have shape (n_samples, n_outputs). Uses standard estimator interfaces for scoring and evaluation.

MultiOutputClassifier
^^^^^^^^^^^^^^^^^^^^^
- Purpose: Extend a classifier to multioutput classification by fitting one classifier per target.
- What it does: For each target column i, it fits a separate classifier on X and y[:, i].
- Key arguments:
  - estimator: classifier implementing fit and predict.
  - n_jobs: number of parallel jobs (if supported by the base estimator).
- Typical usage:
  - from sklearn.multioutput import MultiOutputClassifier
  - from sklearn.linear_model import LogisticRegression
  - moc = MultiOutputClassifier(LogisticRegression())
  - moc.fit(X, y)
  - y_pred = moc.predict(X)
- Notes: predict_proba is available per target; results are returned per target as appropriate for the base classifier.

ClassifierChain
^^^^^^^^^^^^^^^
- Purpose: Build a multioutput classifier by chaining binary/multiclass classifiers.
- What it does: Trains a sequence (chain) of classifiers where each stage receives the original features plus the predictions of previous stages as additional features.
- Key arguments:
  - estimator: base classifier implementing fit and predict.
  - order: sequence specifying the order of targets or None for a random order.
  - verbose: int controlling the verbosity.
- Typical usage:
  - from sklearn.multioutput import ClassifierChain
  - from sklearn.ensemble import RandomForestClassifier
  - chain = ClassifierChain(RandomForestClassifier(), order=None)
  - chain.fit(X, y)
  - y_pred = chain.predict(X)
- Notes: The chain order can impact performance; error propagation is possible along the chain. Useful when targets are related and sequential dependencies can be exploited.

RegressorChain
^^^^^^^^^^^^^^
- Purpose: Build a multioutput regressor by chaining regressors.
- What it does: Similar to ClassifierChain but for regression; each stage regresses its target using features plus previous predictions.
- Key arguments:
  - estimator: base regressor implementing fit and predict.
  - order: sequence specifying the order of targets or None for a random order.
  - verbose: int controlling the verbosity.
- Typical usage:
  - from sklearn.multioutput import RegressorChain
  - from sklearn.ensemble import RandomForestRegressor
  - chain = RegressorChain(RandomForestRegressor(), order=None)
  - chain.fit(X, y)
  - y_pred = chain.predict(X)
- Notes: As with ClassifierChain, the chosen order can affect performance and there is potential for error propagation through the chain.

General guidance
--------------
- Data shape: y should be a 2D array of shape (n_samples, n_outputs).
- Base estimators: All base estimators must support fit and predict (and optionally partial_fit for incremental learning in some cases).
- Parallelism: n_jobs (when supported by the base estimator) controls parallelism.
- See also: OneVsRestClassifier for a flat multioutput approach; potentially use in combination with other multioutput strategies.
