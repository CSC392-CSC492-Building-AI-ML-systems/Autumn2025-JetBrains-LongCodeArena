ImPACT
=====

Image Pixel-wise fit for Atmospheric Cherenkov Telescopes (ImPACT)
-----------------------------------------------------------------

ImPACT implements a template-based image fitting method for atmospheric Cherenkov telescopes. It compares observed camera images with a library of precomputed templates to perform a maximum likelihood fit of shower parameters such as the shower axis, energy, and height of maximum. The approach follows the template-based reconstruction concept described by Parsons & Hinton (2014).

Overview
--------

- Purpose: Reconstruct gamma-ray induced air showers by fitting observed telescope images to interpolated template images.
- Core idea: Use a library of simulated templates for different telescope types and shower parameters, then maximize the likelihood between data and model.
- Performance considerations: Computationally intensive; fixed-unit handling is used inside the class for speed.
- References: Parsons & Hinton, Astroparticle Physics 56 (2014), pp. 26–34.

Key features
------------

- Per-telescope template interpolation via TemplateNetworkInterpolator
- Optional time-gradient interpolation via TimeGradientInterpolator
- Energy and depth priors to regularise fits
- Support for multiple telescope types and their template files
- Atmosphere profile integration for depth-related conversions

API
---

Classes
-------

ImPACTReconstructor
~~~~~~~~~~~~~~~~~~~

A ctapipe Component that implements the image-pixel wise template fitting algorithm.

- Purpose
  - Provide an interface to perform template-based reconstruction of gamma-ray showers using image data from multiple telescopes.
  - Compute likelihoods between observed images and template predictions to estimate shower axis, energy, and height of maximum.

- Construction parameters
  - root_dir: str
    Path to the directory containing template files.
  - minimiser: str
    Choice of minimiser (e.g. "minuit").
  - prior: str
    Priors to apply to the likelihood (e.g. energy/xmax priors).
  - template_scale: float
    Scaling factor for templates to adjust model amplitudes.
  - xmax_offset: float
    Offset to apply to the predicted depth of maximum.
  - use_time_gradient: bool
    If True, use time-gradient templates in addition to spatial templates.

- Internal data and attributes (highlights)
  - file_names: dict
    Mapping from telescope type to template filenames (and time templates where applicable).
  - ped_table: dict
    Pedestal (noise) width for each telescope type (hard-coded for now).
  - spe: float
    Single photoelectron distribution width (hard-coded).
  - thickeness_profile, altitude_profile
    Atmospheric profiles obtained from ctapipe.instrument.get_atmosphere_profile_functions.
  - pixel_x, pixel_y, image, time
    Per-event pixel coordinates and data.
  - tel_types, tel_id
    Telescope type and identifiers in the event.
  - tel_pos_x, tel_pos_y
    Telescope positions in the array.
  - peak_x, peak_y, peak_amp
    Peak position and amplitude per telescope (Hillas-based helper).
  - hillas_parameters, ped
    Internal containers for per-telescope Hillas parameters and pedestal values.
  - prediction, time_prediction
    Interpolated template predictions and time-gradient predictions per telescope type.
  - array_direction, array_return, nominal_frame
    Metadata describing the array geometry and coordinate handling.
  - template_scale, xmax_offset, use_time_gradient
    Runtime configuration options for template scaling, depth offset, and timing.

- Methods
  - initialise_templates(tel_type)
    Ensure templates for each telescope type are initialised and available for the fit.
    Creates TemplateNetworkInterpolator instances for each telescope type and, if enabled, TimeGradientInterpolator instances for timing information.
  - get_hillas_mean()
    Compute peak positions and amplitudes per telescope from the per-telescope Hillas parameters; used to characterise image peaks for downstream calculations.

Functions
---------

guess_shower_depth(energy)
-------------------------

Simple estimation of the depth of shower maximum (Xmax) based on the energy.

- Parameters
  - energy: float
    Shower energy in TeV
- Returns
  - float: Expected Xmax (depth of shower maximum)

energy_prior(energy, index=-1)
------------------------------

Gaussian-type prior on the energy parameter in the likelihood, implemented as a log-prior.

- Parameters
  - energy: float
    Shower energy
  - index: int
    Power index (default -1)

- Returns
  - float: log-prior contribution for energy

xmax_prior(energy, xmax, width=100)
----------------------------------

Prior on the depth of maximum (Xmax) relative to the predicted Xmax for a given energy.

- Parameters
  - energy: float
    Shower energy
  - xmax: float
    Observed/predicted Xmax
  - width: float
    Width of the Gaussian prior on the difference

- Returns
  - float: log-prior contribution for Xmax

Usage
-----

The ImPACTReconstructor is intended to be used within a ctapipe-based workflow. An example workflow (conceptual) is:

.. code-block:: python

   from imapct import ImPACTReconstructor

   recon = ImPACTReconstructor(
       root_dir="/path/to/templates",
       minimiser="minuit",
       prior="default",
       template_scale=1.0,
       xmax_offset=0.0,
       use_time_gradient=True,
   )

   # Prepare event data (pixel_x, pixel_y, image, time, tel_types, etc.)
   # Initialise templates for the telescope types in the event
   recon.initialise_templates(event.tel_types)

   # Run the fit (high-level call shown conceptually)
   # best_fit = recon.fit(event)

Notes
-----

- Template files are telescope-type specific and overlayed by the file_names map.
- Atmosphere profiles are loaded using ctapipe’s atmosphere utilities (Paranal in the default setup).
- Inference is performed in the same unit system inside the class (radians for angles, metres for distances, TeV for energy).
- The module relies on ctapipe, iminuit, scipy, numpy, and astropy.
