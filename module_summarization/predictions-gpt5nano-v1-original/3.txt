Publishing for web review
=========================

Overview
--------
This section documents how to publish content to the web review workflow using the ftrack API session. The Session class manages connection to the ftrack server, authentication, caching, and optional embedded event hub publishing. Enabling the embedded event hub allows publishing and subscribing to non-local events, which is commonly used for web-based review workflows.

Prerequisites
-------------
- Server URL to the ftrack server (server_url)
- API key (api_key)
- API user (api_user)
  These can be provided as arguments to Session or via environment variables:
  - FTRACK_SERVER for server_url
  - FTRACK_API_KEY (and FTRACK_API_USER) for credentials

Session setup for web review publishing
--------------------------------------
- Create a Session with the required connection data.
- Optionally enable the embedded event hub by setting auto_connect_event_hub to True. The event hub runs in a background thread, enabling publishing and subscribing to non-local events needed for web review workflows.

  Example concept (not code):
  - session = Session(server_url=..., api_key=..., api_user=..., auto_connect_event_hub=True)

Event publishing via the embedded hub
------------------------------------
- When the event hub is connected, you can publish events that drive the web review process. These events are handled by the hub and propagated to connected listeners (including the web review interface).
- Subscribing to events is also supported, allowing reactive workflows related to web review updates.

Authentication and requests
---------------------------
- Requests sent through the session use HTTP headers for authentication:
  - ftrack-api-key: your API key
  - ftrack-user: your API user
- This authentication is provided by the SessionAuthentication class, which can be applied to HTTP requests to ensure proper authorization when interacting with the ftrack API outside of the standard session methods.

Caching and schema
-------------------
- Schema caching improves performance by storing API schema locally.
  - Enable by setting schema_cache_path to a folder path.
  - If not set, FTRACK_API_SCHEMA_CACHE_PATH environment variable is used.
  - If both are not specified, a temporary directory is used.
  - Set to False to disable schema caching entirely.

Operations tracking
--------------------
- The Session initializes a recorder for pending operations:
  - recorded_operations holds pending operations
  - record_operations defaults to True, enabling automatic tracking of operations performed during the session

Notes
-----
- Auto-connected event hub is designed to improve startup time; plugins may rely on a connected hub, so ensure it is enabled if web review publishing depends on event publishing.
- The exact publishing actions depend on the event types supported by the ftrack web review integration and the event hub configuration in your environment.
