w_fluxanl
==========

Synopsis
--------
w_fluxanl [options]

Description
-----------
Extract fluxes into pre-defined target states from WEST data, average them,
and construct confidence intervals. Monte Carlo bootstrapping is used to
account for the correlated and possibly non-Gaussian statistical error in
flux measurements.

All non-graphical output (including that to the terminal and HDF5) assumes
that the propagation/resampling period tau is equal to unity; to obtain results
in familiar units, divide all fluxes and multiply all correlation lengths by
the true value of tau.

Notes
- Outputs are produced for each target state (by label when available, otherwise by index).
- Flux units are in per-tau; convert to physical units by multiplying fluxes by tau
  (and apply the appropriate scaling for correlation lengths as described above).
- The tool reads flux information from the WEST HDF5 file using the same data
  management interface as other WEST tools, with logic branching on the file version.

Output format
-------------
Format version: 2

- Non-graphical outputs include terminal printouts and HDF5 outputs.
- For each target, results are stored as a 1-D array of flux entries with fields:
  - n_iter: iteration number (type n_iter_dtype)
  - flux: flux value (type weight_dtype)
  - count: count of recycling events (unsigned int)

- Summary per target (when available) is returned as:
  - target_label: human-readable label for the target (type vstr_dtype)
  - mean_flux: mean flux over the considered iterations (type weight_dtype)
  - mean_flux_ci_lb: lower bound of the bootstrap confidence interval (type weight_dtype)
  - mean_flux_ci_ub: upper bound of the bootstrap confidence interval (type weight_dtype)
  - mean_flux_correl_len: estimated correlation length (unsigned int)

Workflow
--------
- The tool reads iterations within a specified range (iter_start to iter_stop).
- For HDF5 files with version < 7, fluxes are extracted from the recycling table, using either
  the weight field (pre-v7) or the flux field (v7+).
- For HDF5 files with version >= 7, fluxes are gathered from the new_weights group of the
  following iteration after recycling events, aggregating by target state label.
- The last contiguous run where a target is available is determined, and results are kept
  for that range.

Inputs
------
- WEST data reader configuration (provided by WESTDataReader).
- Iteration range selection (provided by IterRangeSelection).
- Output destination/format configuration (output options group).

Output destinations
-------------------
- Terminal (stdout)
- HDF5 (as part of the WEST output structure)

Example
-------
Example usage (conceptual):

  w_fluxanl --input WEST_DATA.h5 --start 1 --stop 1000 --output flux_results.h5

This will extract fluxes for iterations 1 through 999, compute mean fluxes with
bootstrap confidence intervals, and store the results in the specified output.

Compatibility
-------------
Output format is version 2. The tool adapts its flux extraction method depending on
the we_h5file_version of the data manager, ensuring compatibility with both older
and newer WEST HDF5 file layouts.
