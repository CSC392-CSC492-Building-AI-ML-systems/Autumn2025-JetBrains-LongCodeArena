Best Effort Callback
====================

Overview
--------

This callback provides a lightweight, “best-effort” live view during data collection. It integrates with the Bluesky run engine to:
- print a header with timing and identifiers at the start of a run
- announce new streams as they appear
- optionally display a live table of readings
- optionally plot data live via LivePlot, LiveGrid, and LiveScatter
- accumulate peak statistics via PeakStats

Key features include toggling of the heading, table, baseline, and plotting, as well as control over what is plotted and how much is shown.

Class reference
---------------

BestEffortCallback(QtAwareCallback)
---------------------------------

A Qt-aware callback that tries to surface useful live information during a scan without being overly intrusive.

Initialization
--------------

__init__(self, *, fig_factory=None, table_enabled=True, **kwargs)

- fig_factory: Optional factory for creating figures when needed.
- table_enabled: Enable or disable the LiveTable output by default.
- kwargs: Passed to the superclass initializer.

Internal state (example of fields set up by __init__)
- _start_doc: Start document for the scan
- _descriptors: Mapping of descriptor uids to descriptor documents
- _table: LiveTable instance (if used)
- _heading_enabled: Whether to print the header at the start
- _table_enabled: Whether to print a table
- _baseline_enabled: Whether to plot baseline fields
- _plots_enabled: Whether to plot plotted fields
- _fig_factory: Figure factory for plots
- _live_plots / _live_grids / _live_scatters: Dictionaries holding live plotting objects
- _peak_stats: Peak statistics storage
- _cleanup_motor_heuristic: Flag to indicate heuristic cleanup of motor dimensions
- _stream_names_seen: Set of stream names encountered
- Public options:
  - overplot: Enable overplotting when multiple streams provide data
  - noplot_streams: Streams to skip plotting (default ['baseline'])
  - omit_single_point_plot: Do not plot if there is only a single point
- Public data:
  - peaks: Peak statistics container (PeakResults)
- Misc:
  - _buffer: StringIO used to handle bottom border of the table
  - _baseline_toggle: Internal toggle for baseline handling

Public methods
--------------

enable_heading() / disable_heading()
- Enable or disable printing of the transient scan header (timestamp and IDs) at the top of a run.

enable_table() / disable_table()
- Enable or disable printing of hinted readings from the 'primary' stream in a LiveTable.

enable_baseline() / disable_baseline()
- Enable or disable printing of hinted fields from the 'baseline' stream.

enable_plots() / disable_plots()
- Enable or disable plotting for streams not in noplot_streams.

__call__(self, name, doc, *args, **kwargs)
- Gate the callback so that it only runs if at least one of table, baseline, or plots is enabled. Delegates to the superclass otherwise.

start(self, doc)
- Initialize state for a new run.
- Determine hints about dimensions (independent variables) and the fields to plot against.
- Print a heading with the scan ID and timestamp if enabled.
- Build internal lists of dimension fields and the corresponding stream.

descriptor(self, doc)
- Handle a new descriptor document:
  - Track the descriptor by uid and the stream name.
  - If a new stream is seen and table output is enabled, print a notice.
  - Determine which columns are available from the descriptor (hinted fields).
  - For legacy documents, potentially reconstruct dimension fields from object hints.
  - Remove independent variables from plotting columns to identify plotted fields.
  - Decide whether to plot data for this descriptor based on current settings and hints.
  - If plotting is enabled, set up the appropriate live plot objects (LivePlot/LiveGrid/LiveScatter) and related state (not shown in the excerpt).

Notes
-----

- The callback is designed to work with the LivePlot, LiveGrid, and LiveScatter infrastructure, as well as a LiveTable for tabular output.
- It uses a heuristic to determine dimensions from either motor information or time, and may fall back to a default guess if hints are insufficient or incompatible.
- The code is intended to be “best effort” and safe in the presence of multiple streams; if streams cannot be combined, it will fall back to simpler behavior and warn the user.

Usage
-----

- Instantiate the callback and attach it to the run engine (RE).
- Adjust toggles as needed:
  - BestEffortCallback(enable_heading), enable_table, enable_plots, etc.
- For testing and demonstration, see tests/interactive/best_effort_cb.py.

Example (conceptual)
--------------------

- cb = BestEffortCallback(table_enabled=True, fig_factory=my_factory)
- RE(targets, callbacks=[cb])

Testing
-------

Behavior is demonstrated in tests/interactive/best_effort_cb.py. Use this script to verify live output, table updates, and plotting under simulated conditions.
