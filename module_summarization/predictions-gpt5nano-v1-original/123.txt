Delete view (form view)
=======================

Implements the preview+confirm+delete workflow used by django_cradmin form views.

Overview
--------

This module provides a DeleteView mixin and a WithinRoleDeleteView that together implement a three-step workflow:

1) Preview the object to be deleted
2) Confirm the deletion
3) Perform the deletion and redirect with a success message

DeleteViewMixin
---------------

Purpose
    A mixin that adds the preview, confirm, and delete behavior to a form-style delete view.

Key behaviors
- Page title
  - get_pagetitle: defaults to "Confirm delete".
- Action and button labels
  - get_action_label: "Delete"
  - get_delete_button_label: defaults to get_action_label
  - get_cancel_button_label: "Cancel"
- Object preview
  - get_object_preview: returns the string representation of the object (uses __str__ of the model)
- Confirm message
  - get_confirm_message: "Are you sure you want to delete "<object_preview>"?"
- Success handling
  - get_success_url: defaults to the app index URL for the current Cradmin app
  - get_success_message: default "Deleted "<what>""
  - add_success_messages: if a non-empty success message is returned by get_success_message, it is added via Django's messages framework
- Context data for templates
  - add_delete_view_mixin_context_data populates:
    - model_verbose_name
    - success_url
    - object_preview
    - pagetitle
    - confirm_message
    - delete_button_label
    - cancel_button_label
- Deletion flow
  - delete: captures object_preview, performs the deletion via the superclass, then adds success messages

WithinRoleDeleteView
--------------------

Purpose
    A DeleteView configured for use when a Cradmin role is involved, providing the correct context and template.

Inheritance
    QuerysetForRoleMixin, DeleteViewMixin, DjangoDeleteView, CommonCradminViewMixin, and javascriptregistry.viewmixin.WithinRoleViewMixin

Template
    template_name = 'django_cradmin/viewhelpers/formview/within_role_delete_view.django.html'

Context data
    get_context_data augments the template context with JavaScript component IDs and common Cradmin view data, in addition to the DeleteViewMixin context data.

Usage notes
- Import path: you should import this class with from django_cradmin import viewhelpers, and refer to it as viewhelpers.formview.WithinRoleDeleteView.
- Customization: override get_success_message(object_preview) to customize the success banner; the default is "Deleted "<object_preview>"".
- Object representation: the preview uses str(get_object()), i.e., the object’s __str__ representation.
- Redirect: after a successful delete, the user is redirected to the app index URL by default.

Context data keys introduced by the delete mixin
------------------------------------------------

- model_verbose_name: the model’s verbose name
- success_url: URL to redirect to after deletion
- object_preview: string representation of the object
- pagetitle: page title (e.g., "Confirm delete")
- confirm_message: message asking for confirmation
- delete_button_label: label for the delete action button
- cancel_button_label: label for the cancel button

Template
--------

WithinRoleDeleteView uses the template:
- django_cradmin/viewhelpers/formview/within_role_delete_view.django.html

See also
---------

- The delete flow is implemented via the delete() method on DeleteViewMixin, which delegates to the parent deletion logic and then emits a success message.
