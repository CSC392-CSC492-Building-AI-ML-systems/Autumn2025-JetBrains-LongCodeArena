Model usage
===========

This guide describes how to declare and work with models using the core Model abstraction. It covers defining fields, configuring model metadata, relationships, default ordering, and how descriptive comments can be attached to fields.

Defining a Model
----------------
Models are defined by subclassing the base Model class and declaring fields as class attributes using field types.

Example:
- Note: Import styles may vary; common approach uses the fields module.

.. code-block:: python

    from tortoise.models import Model
    from tortoise import fields

    class User(Model):
        id: int = fields.IntField(pk=True)
        username: str = fields.CharField(max_length=50, unique=True)
        email: str = fields.CharField(max_length=100, null=True)
        is_active: bool = fields.BooleanField(default=True)
        created_at: 'datetime' = fields.DatetimeField(auto_now_add=True)

    class Post(Model):
        id: int = fields.IntField(pk=True)
        author: User = fields.ForeignKeyField("models.User", related_name="posts")
        title: str = fields.CharField(max_length=200)
        content: str = fields.TextField()
        published_at: 'datetime' = fields.DatetimeField(null=True)

Meta options
------------
Models can declare an inner Meta class to configure various options:

- table: database table name
- app: application name
- abstract: whether the model should be abstract
- unique_together: constraints enforcing uniqueness across multiple fields
- indexes: additional database indexes
- ordering: default ordering for queries (see below)

Example:
.. code-block:: python

    class User(Model):
        id = fields.IntField(pk=True)
        username = fields.CharField(max_length=50)
        email = fields.CharField(max_length=100)

        class Meta:
            table = "user"
            app = "myapp"
            unique_together = (("username", "email"),)
            indexes = (("username",),)

            # default ordering is defined as a sequence of ordering strings
            ordering = ("username ASC", "created_at DESC")

Notes:
- unique_together and indexes are validated and worked with by the ORM when creating or querying data.
- default ordering is resolved by QuerySet._resolve_ordering_string for each entry in the ordering sequence.

Fields and relationships
------------------------
Fields declare the columns stored in the database. Relationships enable navigation between models.

Examples:

.. code-block:: python

    class User(Model):
        id = fields.IntField(pk=True)
        username = fields.CharField(max_length=50)
        profile = fields.OneToOneField("models.Profile", related_name="user", null=True)

    class Profile(Model):
        id = fields.IntField(pk=True)
        bio = fields.TextField()
        user = fields.OneToOneField("models.User", related_name="profile")

    class Post(Model):
        id = fields.IntField(pk=True)
        author = fields.ForeignKeyField("models.User", related_name="posts")
        tags = fields.ManyToManyField("models.Tag", related_name="posts")

    class Tag(Model):
        id = fields.IntField(pk=True)
        name = fields.CharField(max_length=30)

Notes:
- ForeignKeyField and OneToOneField support lazy loading via internal getters.
- ManyToManyField creates a ManyToManyRelation, enabling easy access to related objects.

Comments on fields
------------------
Comments can be attached to fields using a special syntax placed directly above the field declaration. The ORM extracts these comments from the source and attaches them to the corresponding field metadata, with an optional placeholder for the model name.

Example:
.. code-block:: python

    class User(Model):
        #: The user's login name
        username = fields.CharField(max_length=50)

        #: The user's email address
        email = fields.CharField(max_length=100, null=True)

How comments are collected:
- The _get_comments utility reads the class source, looks for lines starting with #:, and associates them with subsequent fields.
- The placeholder "{model}" in the comment is replaced by the actual model class name.

Querying and default behavior
-----------------------------
Models expose a QuerySet for convenient data access:

.. code-block:: python

    # all users
    users = await User.all()

    # filter
    active_users = await User.filter(is_active=True)

    # ordering
    recent_users = await User.all().order_by("-created_at")

    # limiting
    latest = await User.all().limit(5)

Ordering and meta
-----------------
- Default ordering is determined from Meta.ordering if provided.
- Ordering strings are resolved by QuerySet._resolve_ordering_string during model initialization.

Transactions and signals (brief)
--------------------------------
- Transactions: use in_transaction helpers to run multiple operations atomically.
- Signals: the Signals facility allows subscribing to model lifecycle events (save, update, etc.).

Example transaction:
.. code-block:: python

    from tortoise.transactions import in_transaction

    async with in_transaction():
        user = await User.create(username="alice")
        await Post.create(author=user, title="Hello")

This documentation reflects the core usage of the Model abstraction: declaring fields, configuring Meta options, wiring relationships, leveraging default ordering, and utilizing field-level comments for self-documenting models.
