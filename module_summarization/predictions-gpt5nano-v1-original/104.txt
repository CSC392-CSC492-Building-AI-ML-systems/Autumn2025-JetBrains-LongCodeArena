diophantine
===========

Overview
--------

This module provides a solver for Diophantine equations by transforming the equation into a product of terms that should equal zero. Each term is solved independently (via diop_solve) and the results are merged to form the complete solution set. The output is a set of tuples, where each tuple contains the solutions for the variables ordered alphabetically by their symbols. The public API exposed by this module includes diophantine and classify_diop.

Public API
----------

diophantine(eq, param=symbols('t', integer=True), syms=None)

- Purpose
  Simplifies the solution procedure for a Diophantine equation by converting it into a product of terms that must be zero and solving each term.

- Parameters
  - eq: Relational or Expr
      The equation (or expression) to be solved.
  - param: Symbol, optional
      The parameter symbol to be used in the solution. Defaults to t (an integer symbol).
  - syms: list of Symbol, optional
      The variables to determine the order of the elements in the returned tuple. If provided, only variables present in the equation are considered, and the order is determined by the specified sequence.

- Returns
  A set of tuples. Each tuple contains a solution for each variable in alphabetical order (or in the order determined by syms if provided).

- Notes
  - If the equation is provided as Eq(lhs, rhs), it is converted to lhs - rhs.
  - If the equation reduces to 0, a trivial solution containing the parameter is returned.
  - The function attempts to factor and decompose the equation. If the equation cannot be treated as a polynomial with rational coefficients, a TypeError is raised (with a message indicating the expectation).
  - The function may skip certain factoring steps for performance reasons (e.g., to avoid hanging on general_sum_of_squares expressions).

- Examples
  - diophantine(x**2 - y**2)
      Returns {(t_0, -t_0), (t_0, t_0)}.
  - diophantine(x*(2*x + 3*y - z))
      Returns {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}.
  - diophantine(x**2 + 3*x*y + 4*x)
      Returns {(0, n1), (3*t_0 - 4, -t_0)}.

See Also
--------

- diofant.solvers.diophantine.diop_solve
  The underlying solver used to handle individual diophantine terms.

- classify_diop
  Helper for classifying diophantine terms (imported as part of the module’s public API).

Internal helpers (private)
--------------------------

These helpers are used internally by diophantine and are not part of the public API. They assist with integer checks, tuple handling, gcd normalization, and simple rational arithmetic.

- _is_int(i)
  Checks whether i can be interpreted as an integer (uses as_int).

- _sorted_tuple(*i)
  Returns a sorted tuple of inputs.

- _remove_gcd(*x)
  Divides all inputs by their greatest common divisor and returns the reduced tuple. If gcd cannot be determined, returns the original inputs.

- _rational_pq(a, b)
  Returns a reduced rational pair (numerator, denominator) for a/b with sign handled in the numerator.

- _nint_or_floor(p, q)
  Returns the nearest integer to p/q; in case of a tie, returns floor(p/q).

- _odd(i)
  True if i is odd.

- _even(i)
  True if i is even.

Examples
--------

- Basic usage with a simple difference of squares:
  diophantine(x**2 - y**2)

- A linear combination with a product:
  diophantine(x*(2*x + 3*y - z))

- A quadratic in one variable:
  diophantine(x**2 + 3*x*y + 4*x)

Notes
-----

- The function returns solutions in terms of a parameter (default t) and symbols that appear in the equation.
- If no non-trivial solutions exist, a trivial solution involving the parameter may be returned depending on the equation.
- The behavior relies on polynomial factoring and solving strategies; not all Diophantine equations may be solvable with this approach, and some expressions may raise TypeError if they cannot be treated as polynomials with rational coefficients.
