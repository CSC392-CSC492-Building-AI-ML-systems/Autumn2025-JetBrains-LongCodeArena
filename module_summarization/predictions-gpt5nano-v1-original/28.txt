w_multi_west
=============
Tool designed to combine multiple WESTPA simulations while accounting for reweighting.

Overview
--------
The w_multi_west tool merges results from several WESTPA simulations into a single multi-simulation output file, with support for reweighting across runs. It is implemented as a WESTMultiTool subclass and uses WESTPA’s HDF5 I/O facilities to read per-simulation data and write a consolidated output.

Command-line options
--------------------
- -o, --output-file
  The name of the output file to store results in. Default: multi.h5

- -W, --west, --WEST_H5FILE
  The name of the main .h5 file inside each simulation directory. Default: west.h5

- -a, --aux
  Names of additional auxiliary datasets to be combined. Can be specified multiple times.

- -aa, --auxall
  Combine all auxiliary datasets. Default: False

- -nr, --no-reweight
  Do not reweight. Default: False

- -ib, --ibstates
  Attempt to combine ibstates dataset. Default: False

Internal behavior
-----------------
- Opens and writes to a single output HDF5 file (default multi.h5) and stamps creator metadata.
- Collects input WESTPA data from the specified west.h5 files in each simulation directory.
- Computes the total number of walkers across iterations for proper aggregation.
- Supports optional reweighting across simulations unless disabled with --no-reweight.
- Can merge auxiliary datasets either selectively (via -a) or all available auxiliaries (-aa).
- If present, ibstates can be merged (--ibstates).

Notes
-----
- The tool relies on WESTPA’s HDF5 I/O utilities (h5io) and the WESTMultiTool framework.
- A progress indicator component is used to report tool progress.
- The implementation includes helpers for bin topology lookup and ID type conversion to ISTATE structures, used during reweighting and data integration.

Usage example
-------------
w_multi_west -o multi.h5 -W west.h5 -aa -nr
This merges multiple simulations, optionally including all auxiliary datasets and disabling reweighting.
