Learn Gaussian Mixture Models
=============================

Overview
--------

Gaussian Mixture Models (GMMs) provide a probabilistic framework for clustering data by assuming that the data are generated from a mixture of underlying Gaussian components. Each component k has a prior weight pi_k, a mean mu_k, and a covariance Sigma_k. The model assigns soft mixture memberships to samples, allowing for flexible modeling of complex data distributions.

Training
--------

GMMs are trained via the Expectation-Maximization (EM) algorithm. Given data X, EM iteratively updates

- E-step: responsibilities gamma_nk = P(z_n = k | x_n, theta)
- M-step: parameters are updated to maximize the expected complete-data log-likelihood (or a related lower bound)

Training runs multiple initializations (n_init) and selects the best solution based on the objective (log-likelihood or lower bound). Each initialization performs up to max_iter iterations or until the change in the objective falls below tol. If convergence is not achieved, a ConvergenceWarning is issued.

Initialization
--------------

The initial parameter values for EM can significantly affect convergence. Four initialization strategies are supported:

- kmeans: initialize responsibilities using a single run of KMeans with n_clusters = n_components
- random: initialize with random responsibilities (per-sample) and normalize to sum to 1
- random_from_data: select a subset of data points as initial representatives for components
- k-means++: initialize using the k-means++ strategy

Hyperparameters and Parameter Validation
----------------------------------------

- n_components: int
  Number of mixture components (clusters). Must be at least 1.

- tol: float
  Convergence threshold. EM stops when the change in the objective is less than this value.

- reg_covar: float
  Non-negative regularization added to the diagonals of covariance matrices for numerical stability.

- max_iter: int
  Maximum number of EM iterations per initialization.

- n_init: int
  Number of initializations to perform; the best solution is kept.

- init_params: {'kmeans', 'random', 'random_from_data', 'k-means++'}
  Method used to initialize the parameters.

- random_state: int, RandomState instance or None
  Seed or random number generator for stochastic initializations.

- warm_start: bool
  If True, reuse the previous solution and do not reinitialize on subsequent calls to fit.

- verbose: int
  Level of verbosity for progress messages during fitting.

- verbose_interval: int
  Frequency of progress messages (in iterations) when verbose > 0.

Convergence and Warnings
------------------------

Fitting may raise a ConvergenceWarning if the algorithm fails to converge within max_iter iterations for a given initialization. Upon successful fitting, the estimator exposes learned parameters and diagnostic information.

Fitted attributes (typical)
---------------------------

- weights_: array-like of shape (n_components,)
  Mixture weights pi_k.

- means_: array-like of shape (n_components, n_features)
  Component means mu_k.

- covariances_: array-like
  Component covariances Sigma_k (format depends on the implemented parameterization).

- converged_: bool
  Indicates whether the last fit converged.

- lower_bound_: float
  Final value of the lower bound on the log-likelihood (or equivalent objective).

Notes
-----

- Inputs X are validated to ensure proper shape and data type, requiring at least two samples.
