Module: alembic.command

Overview
- This module exposes the top-level command helpers used by the Alembic CLI to manage migration environments and revisions. It provides:
  - list_templates: list available environment templates
  - init: initialize a new migration environment
  - revision: create a new revision file

Functions

list_templates(config)
- Description: List available templates.
- Parameters:
  - config (Config): a Config object.
- Behavior:
  - Prints the available templates to stdout.
  - Reads each template directory under the template directory, opens its README, and prints a synopsis line.
  - Includes usage notes showing how templates are used (e.g., via the init command).
- Returns: None

init(config, directory, template='generic', package=False)
- Description: Initialize a new scripts directory.
- Parameters:
  - config (Config): a Config object.
  - directory (str): string path of the target directory.
  - template (str): name of the migration environment template to use (default: 'generic').
  - package (bool): if True, write __init__.py files into the environment location as well as the versions/ location.
- Behavior:
  - If the target directory exists and is not empty, raises a CommandError.
  - Validates that the chosen template exists.
  - Creates the target directory if it does not exist.
  - Creates a versions subdirectory under the target directory.
  - Copies template files into the target directory; if a mako template is present, generates the configuration file.
  - Optionally creates __init__.py files in the target and versions directories.
  - Prompts the user to edit configuration/connection/logging settings in the generated config file before proceeding.
- Returns: None

revision(config, message=None, autogenerate=False, sql=False, head='head', splice=False, branch_label=None, version_path=None, rev_id=None, depends_on=None, process_revision_directives=None)
- Description: Create a new revision file.
- Parameters:
  - config (Config): a Config object.
  - message (str | None): message to apply to the revision (the -m option).
  - autogenerate (bool): whether to autogenerate the script from the database; corresponds to --autogenerate.
  - sql (bool): whether to dump the script as SQL to stdout; corresponds to --sql.
  - head (str): head revision to build the new revision upon as a parent; corresponds to --head.
  - splice (bool): whether to create the new revision as a splice (needed when the given head is not itself a head); corresponds to --splice.
  - branch_label (str | None): label to apply to the branch; corresponds to --branch-label.
  - version_path (str | None): version path symbol identifying a specific version path; corresponds to --version-path.
  - rev_id (str | None): optional revision identifier to use instead of generating one; corresponds to --rev-id.
  - depends_on (str | None): optional list of dependencies (depends-on); corresponds to --depends-on.
  - process_revision_directives (ProcessRevisionDirectiveFn | None): callable applied to the structure generated by the revision process (available via programmatic use).
- Behavior:
  - Creates a ScriptDirectory from the config.
  - Prepares a revision context (RevisionContext) with the given command arguments.
  - Determines the runtime environment based on the config and options; if autogenerate is used, environment is enabled, and certain invalid combinations (e.g., --sql with --autogenerate) raise an error.
  - Uses the revision machinery to generate the actual revision script(s) (details as implemented in the module).
- Returns:
  - A Script object or a list of Script objects, depending on how revisions are created (Optional[Script] or List[Optional[Script]]).
