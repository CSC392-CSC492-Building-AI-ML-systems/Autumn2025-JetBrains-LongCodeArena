Storage Module: MongoDB (Motor)
=================================

Overview
--------

This module implements a MongoDB-backed storage service for asab apps. It provides:

- StorageService: a concrete implementation of the asab storage service using motor (async MongoDB driver) and pymongo.
- MongoDBUpsertor: an Upsertor implementation for performing atomic updates/inserts against MongoDB, with support for versioning, incremental updates, pushes, unsets, and optional webhook notifications.

Configuration
-------------

Default configuration (added by the module at import time):

- asab:storage.mongodb_uri
  - Default: mongodb://localhost:27017
- asab:storage.mongodb_database
  - Default: asabdb

Notes:
- The database is opened as timezone-aware (tz_aware=True) with UTC tzinfo.
- The configuration is read via asab.Config to initialize the client and database.

Classes and Interfaces
----------------------

StorageService (extends StorageServiceABC)
-----------------------------------------

Purpose:
- Provide MongoDB-backed storage for asab applications.
- Manage a MongoDB client and database connection.
- Offer a simple API to obtain an upsertor, fetch documents, and delete documents.

Key responsibilities and methods:

- __init__(self, app, service_name, config_section_name='asab:storage')
  - Creates an AsyncIOMotorClient using the configured mongodb_uri.
  - Obtains the database using configured mongodb_database with timezone-aware codec options.

- upsertor(self, collection: str, obj_id=None, version=0) -> MongoDBUpsertor
  - Returns a MongoDBUpsertor instance bound to this storage, collection, and identity/version information.

- get(self, collection: str, obj_id, decrypt=None) -> dict
  - Retrieves a document by its _id from the specified collection.
  - If decrypt is provided (iterable of field names), decrypts those fields using aes_decrypt.

- get_by(self, collection: str, key: str, value, decrypt=None) -> dict
  - Retrieves a document by an arbitrary field=value query.
  - If decrypt is provided, decrypts those fields.

- collection(self, collection: str) -> motor.motor_asyncio.AsyncIOMotorCollection
  - Returns the underlying AsyncIOMotorCollection for direct operations.
  - Includes a descriptive docstring with usage examples.

- delete(self, collection: str, obj_id)
  - Deletes a document by its _id from the specified collection.
  - Returns the deleted object's _id or raises NOT-FOUND if absent.

Notes:
- The class relies on asab.Config for configuration and uses AES decryption (aes_decrypt) for specified fields during get/get_by.

MongoDBUpsertor (extends UpsertorABC)
--------------------------------------

Purpose:
- Implement the update/insert logic for documents in MongoDB with upsert semantics.
- Support versioning, incremental updates, sets, unsets, and pushes.

Key responsibilities and methods:

- generate_id(cls) -> Any
  - Class method that generates a new MongoDB ObjectId.

- execute(self, custom_data: typing.Optional[dict] = None, event_type: typing.Optional[str] = None)
  - Core operation performing the upsert.

  Behavior (high level):
  - Builds MongoDB update objects from:
    - ModSet -> $set
    - ModInc -> $inc
    - ModPush -> $push with $each
    - ModUnset -> $unset
  - Builds a filter based on the identity field (via get_id_name) and Version.
  - Executes find_one_and_update with upsert semantics:
    - Upsert is true if Version is 0 or None; otherwise false.
    - Returns the document after update.
  - Handles duplicate key errors by raising DuplicateError with context.
  - If an object is returned, updates self.ObjId with the identifier value.
  - If webhooks are enabled (Storage.WebhookURIs is not None), prepares and sends webhook data including collection, custom data, event_type, and upsertor-derived payload (id, version, updated fields, etc.).

  Notes:
  - The implementation relies on the surrounding UpsertorABC framework (e.g., ModSet, ModInc, ModPush, ModUnset, Collection, Storage, ObjId, Version, id_name).
  - The code snippet ends partway through the webhook-upsertor data construction; the documented behavior reflects the visible logic and intended purpose.

Exceptions
----------

- DuplicateError: Raised when a MongoDB duplicate key condition is detected during upsert operations.
- KeyError with message "NOT-FOUND": Raised when a requested document does not exist (during get/get_by or after an attempted update).

Usage Examples
--------------

- Create storage service and obtain an upsertor:

  - storage = StorageService(app, "mongodb-storage")
  - upsertor = storage.upsertor("my-collection")

- Fetch a document by id:

  - doc = await storage.get("my-collection", some_id, decrypt=["secret_field"])

- Fetch a document by a field:

  - doc = await storage.get_by("my-collection", "username", "alice")

- Delete a document by id:

  - await storage.delete("my-collection", some_id)
