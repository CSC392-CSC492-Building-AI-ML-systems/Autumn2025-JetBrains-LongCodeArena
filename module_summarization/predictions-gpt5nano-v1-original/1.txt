Using Custom Attributes
=======================

Custom attributes are user-defined fields that can be attached to entities in Ftrack to store additional project- or workflow-specific information. They extend the standard data model by allowing teams to capture metadata that is meaningful to their processes, without modifying core entity definitions.

Overview
--------

- Custom attributes (CAs) are defined per project or entity type and can be of various data types (e.g., string, integer, float, boolean, date, enum).
- They are accessed and manipulated through the Ftrack API (ftrack_api.attribute and related components).
- When a Session is created with auto_populate enabled (the default), retrieving an entity will lazily fetch custom attribute values as needed.

Defining Custom Attributes
--------------------------

Custom attributes are typically defined via the Ftrack user interface (Admin > Custom Attributes) or programmatically through the API. Each CA has:

- A name
- A data type
- Optional constraints (e.g., allowed values for enums)
- Applicability (which entity types or projects the CA applies to)

Notes

- Definitions are stored in the Ftrack schema and can be shared across users and sessions.
- CA definitions may be reused across multiple entity types, or scoped to a specific type or project.

Accessing Custom Attribute Values
---------------------------------

Once defined, you can read the values of custom attributes on entity instances. Depending on your version of the API and the session configuration, CA values may be loaded lazily or may require an explicit fetch.

Typical access patterns (conceptual; API names may vary):

- Retrieve a value by attribute name:
  - value = entity.get_custom_attribute_value('AttributeName')
  - or:
  - value = entity.custom_attribute_values['AttributeName']

- Inspect all custom attributes for an entity:
  - for name, value in entity.custom_attribute_values.items():
  -     print(name, value)

Creating and Updating Custom Attribute Values
---------------------------------------------

To set or update custom attribute values on an entity, use the appropriate API methods to assign a new value and persist the change.

Typical update pattern (conceptual):

- entity.set_custom_attribute_value('AttributeName', new_value)
- session.commit()  # Or the equivalent persistence call in your API usage

Be mindful of type constraints; ensure that new_value matches the CA’s declared data type and any enumeration constraints.

Data Types and Constraints
--------------------------

Common CA data types include:

- String
- Integer
- Float
- Boolean
- Date/Datetime
- Enum (one of a predefined set of values)
- User (references to a user in the system)
- Asset or other entity links

Constraints may include:
- Required or optional
- Default values
- Allowed value sets (for enums)
- Validation rules

Performance and Best Practices
------------------------------

- Use auto-population to reduce the number of explicit fetches, but be aware of the potential for larger initial payloads.
- Cache CA definitions where possible to minimize repetitive API calls.
- When querying by custom attribute values, leverage server-side filtering if supported by the API to avoid transferring large result sets.

See Also
--------

- ftrack_api.attribute: module handling custom attribute definitions and values
- Session (in the ftrack_api) for how attributes are loaded with respect to auto_populate

Examples
--------

Code samples illustrate typical usage patterns (adjust to match your API version):

.. code-block:: python

# Pseudo-code: read a custom attribute value
task = session.find_one('Task', {'id': 'abc123'})
estimate = task.get_custom_attribute_value('Estimate')

# Pseudo-code: set a custom attribute value
task.set_custom_attribute_value('Estimate', 5)

# Persist changes
session.commit()

This documentation provides an overview of using custom attributes within the Ftrack Python API. For exact API calls and class/method names, refer to the ftrack_api.attribute module in your installed version of the library.
