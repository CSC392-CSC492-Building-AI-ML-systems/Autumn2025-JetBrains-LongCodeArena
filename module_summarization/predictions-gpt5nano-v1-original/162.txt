Parallel Random Number Generation
=================================

Overview
--------
This project provides parallel-friendly random number generation by offering multiple RNG engines that support independent streams suitable for multi-threaded or multi-process workloads. The RNG backends are implemented as C extensions and exposed to Python through the numpy.random interface, enabling consistent usage across engines while preserving parallel safety.

RNG backends
------------
MT19937 with jump-ahead
- Mersenne Twister with a jump-ahead capability to create independent streams for parallel tasks.
- Implemented with the standard MT19937 core plus a jump routine (mt19937.c and mt19937-jump.c) to advance the state by large steps without generating intermediate numbers.
- Useful when you want separate, reproducible streams across threads or processes without overlap.

Philox
- A counter-based RNG designed for parallel throughput and distributed environments.
- Exposed via a Python extension (_philox) and a C implementation (philox.c).
- Provides non-overlapping subsequences by advancing the counter space deterministically, making it a strong choice for parallel workloads.

PCG64
- A 64-bit Permuted Congruential Generator with a modern, efficient design.
- Includes pcg64.c and pcg64.pyx, with optional inlining controlled by compiler flags.
- Suitable for parallel generation when paired with independent seeds or counters.

SFC64
- Simple Fast Counter 64-bit generator, optimized for speed and parallel usage.
- Delivered as a Python extension (_sfc64) with accompanying C source (sfc64.c).
- Offers fast, parallel-friendly random number generation.

Bounded integers helper
-----------------------
_ bounded_integers module (_bounded_integers) provides safe, unbiased conversion for sampling bounded integers, ensuring correct behavior when generating integers within a specified range in parallel contexts.

API and modules
---------------
- The Python-facing bit_generator module acts as a common interface to the underlying RNG engines.
- Each engine is exposed as a separate extension:
  - _mt19937 (MT19937 with jump-ahead)
  - _philox (Philox)
  - _pcg64 (PCG64)
  - _sfc64 (SFC64)
  - bit_generator (core bit-level generator interface)
- The higher-level Generator object can be constructed with a chosen bit generator to create streams suitable for parallel usage.

Usage in parallel workloads
---------------------------
- For multi-threaded or multi-process code, instantiate independent Generator objects using different bit generators or different seeds/counters.
- MT19937 with jump-ahead is convenient when you want deterministic, independent streams without coordinating seeds.
- Counter-based engines (Philox) are especially well-suited for parallel workloads and distributed computation where streams must remain non-overlapping across workers.

Notes
-----
- The implementation includes both C sources and Cython wrappers to provide fast, parallel-friendly random number generation within numpy.random.
- The distributions module under npyrandom offers additional functionality for statistical distributions built atop these RNG backends.
