Gaussian Processes
====================

Overview
--------

This package provides kernels for Gaussian process regression and classification. Kernels define the covariance structure of a Gaussian process and can be used to express prior assumptions about function behavior. The kernel module supports flexible kernel engineering by combining simple kernels into complex ones.

Key features
------------

- Kernel composition: Combine kernels using addition (+), multiplication (*) and exponentiation (**). Scalar values in expressions are automatically converted to a ConstantKernel.
- Gradient-based hyperparameter optimization: All kernels support analytic gradients with respect to their hyperparameters, enabling efficient optimization.
- Hyperparameter search space: Hyperparameters can be constrained by bounds, defining a rectangular search space for optimization.
- Fixed hyperparameters: Hyperparameters can be declared as fixed, excluding them from optimization.
- Anisotropic length scales: Support for anisotropic kernels where different input dimensions have different length scales; proper validation ensures consistency with data dimensionality.
- Hyperparameter exposure: Kernels expose their hyperparameters via a dedicated API, facilitating inspection and optimization.

Hyperparameters
--------------

Hyperparameters describe the tunable parameters of a kernel. They are defined as a Hyperparameter object with the following fields:

- name: The parameter name (e.g., "length_scale" or "constant_value").
- value_type: The type of the hyperparameter (currently "numeric").
- bounds: A pair of floats >= 0 or the string "fixed". If n_elements > 1, bounds may be a 2D array with shape (n_elements, 2). If "fixed" is used, the parameter cannot be changed.
- n_elements: The number of elements of the hyperparameter value (default 1). n_elements > 1 indicates a vector-valued parameter (e.g., anisotropic length-scales).
- fixed: Whether the hyperparameter is fixed (can be None, in which case it is inferred from bounds).

Notes
-----

- If the bounds are not a scalar "fixed" indication, they are stored as a 2D array of bounds for each element.
- The Hyperparameter class provides equality comparison to help test kernel configurations.

Hyperparameter usage example
----------------------------

The following example illustrates hyperparameter inspection:

.. code-block:: python

   from sklearn.gaussian_process.kernels import ConstantKernel
   from sklearn.datasets import make_friedman2
   from sklearn.gaussian_process import GaussianProcessRegressor
   from sklearn.gaussian_process.kernels import Hyperparameter

   X, y = make_friedman2(n_samples=50, noise=0, random_state=0)
   kernel = ConstantKernel(constant_value=1.0,
                           constant_value_bounds=(0.0, 10.0))

   for hyperparameter in kernel.hyperparameters:
       print(hyperparameter)

   params = kernel.get_params()
   for key in sorted(params):
       print(f"{key} : {params[key]}")

Anisotropic length scales
-------------------------

When using anisotropic kernels, length scales can be specified per input dimension. The helper _check_length_scale ensures that:

- length_scale is a 1D array (no higher-dimensional shapes).
- If anisotropic, its length matches the data dimensionality.

Kernel base class
-----------------

All kernels derive from a common Kernel base class, which provides:

- A unified interface for parameter access and manipulation.
- Mechanisms for combining kernels and exposing their hyperparameters.
- Support for gradient-based optimization through analytic derivatives with respect to hyperparameters.

Implementation notes
--------------------

- This module is strongly inspired by the kernel module of the george package.
- It relies on NumPy and SciPy (e.g., cdist, pdist, squareform, gamma, kv) and integrates with a broader machine learning framework (e.g., parameter cloning and validation utilities).
- Access to kernel parameters is facilitated through methods such as get_params and the hyperparameters attribute, enabling cloning, inspection, and optimization workflows.

Usage in Gaussian Processes
---------------------------

Kernels are the building blocks of Gaussian Process models. They define the covariance function k(x, x') between inputs x and x'. By composing kernels, you can encode rich prior beliefs about function behavior (e.g., smoothness, periodicity, scaling) and tailor models to different data regimes. Hyperparameters control the scale and characteristic length scales of these beliefs and are typically learned from data via gradient-based optimization.
