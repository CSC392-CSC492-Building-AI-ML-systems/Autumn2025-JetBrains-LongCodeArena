PlotData: Supporting data to plotting functions in multiple ways

Overview
- PlotData provides a flexible interface to supply data for plotting, allowing common data holders such as dictionaries, pandas DataFrames, and, when available, Bokeh ColumnDataSource.
- It supports single plots and multiple plot sets (batches) in a single object, enabling multi-plot workflows without duplicating data.
- Data can be provided as a single source or as a list of sources. When multiple plot sets are required, you can describe per-set values using lists in the keyword arguments.

Supported data sources
- If Bokeh is installed:
  - dict, pandas.DataFrame, and bokeh.models.ColumnDataSource
- If Bokeh is not installed:
  - dict and pandas.DataFrame
- When a list of data sources is provided, each element must be one of the above types.

Initialization and options
- data: a single data source or a list of data sources (dicts, DataFrames, or ColumnDataSource when available).
- use_column_source: optional flag related to columnar data usage (affects integration with ColumnDataSource).
- same_length: if True and the data are dict-like, ensure all entries have the same length across all keys.
- strict_data_keys: if True, no new data keys may be added after initialization via add_data_key.
- copy_data: if True, data sources are copied to avoid mutating the original data.

Column mapping with kwargs
- You pass keyword arguments to declare which data to plot and how to extract them from the data source.
- The keys of the keyword arguments become the fields of a per-set namedtuple (Columns).
- If any value in kwargs is a list, a multi-set (multiple plot sets) is created:
  - num_sets is the maximum length among the list-valued kwargs.
  - For each set, the value for a key is taken as value[index].
  - If a value is not a list, it is replicated across all sets.
- Example:
  - data = {'x': x, 'y1': y1, 'y2': y2}
  - PlotData(data, x='x', y=['y1', 'y2']) creates two plot sets, with x fixed to x and y set to y1 then y2.

Data keys and columns
- data_keys property returns the list of data keys (fields) defined for the current instance.
- The internal structure uses a namedtuple Columns to expose the available keys per plot set.
- If multiple sets are defined, iteration yields one entry per set, with attributes named after the declared keys.

Copying and same_length
- copy_data: when enabled, the underlying data sources are copied (or copied via a Bokeh-safe method if needed) to avoid modifying the original data.
- same_length: when True and data are dict-like, the data are normalized to ensure consistent lengths across keys.

Multi-set plotting support
- By providing lists as values in kwargs, you can define multiple plot sets in a single PlotData instance.
- The internal logic determines the number of sets and builds a corresponding list of Columns entries.
- Each set yields an entry containing the per-set values for the declared keys.

Data integrity and extension
- strict_data_keys controls whether new keys can be added after initialization via add_data_key.
- add_data_key(data_key, keys=None) allows extending the dataset with a new key:
  - If keys is a list, its length must match the number of sets.
  - If keys is a scalar, it is replicated across all sets.
  - If strict_data_keys is True, adding a new key raises an error.

Bokeh integration
- When Bokeh is available, ColumnDataSource objects can be used as data sources.
- The module provides helpers to detect and copy ColumnDataSource objects safely:
  - _is_bokeh_cds(data)
  - _copy_bokeh_cds(data)

Iteration semantics
- PlotData is designed to be iterable.
- Each iteration yields an entry (a small object, typically a namedtuple) with attributes named after the declared keys.
- If a key’s value in kwargs was a list (defining multiple sets), the per-set value is used for that key in the corresponding entry.
- Access pattern example:
  - for entry in PlotData(...):
      print(entry.x)   # data for the x-axis for this set
      print(entry.y)   # data or a key label for this set, depending on kwargs

Examples

Example 1: Simple dict with a single y key and multiple y sets
- data = {'x': x, 'y1': y1, 'y2': y2, 'y3': y3}
- p = PlotData(data, x='x', y=['y1', 'y2', 'y3'])
- for entry in p:
  - print(entry.x)  # x data
  - print(entry.y)  # 'y1', then 'y2', then 'y3' for successive sets

Example 2: Data with a list of data sources (multiple plots)
- ds1 = {'x': x1, 'y': y1}
- ds2 = {'x': x2, 'y': y2}
- p = PlotData([ds1, ds2], x='x', y=['y1', 'y2'])
- for entry in p:
  - entry.x corresponds to the x for this set
  - entry.y corresponds to the key name for this set (e.g., 'y1', 'y2') if using the multi-set pattern

Example 3: Pandas DataFrame
- df = pandas.DataFrame({'x': x, 'y': y})
- p = PlotData(df, x='x', y=['y'])
- for entry in p:
  - access entry.x and entry.y (where entry.y is the actual data or the defined key)

Example 4: Copying and same_length
- data = {'x': x, 'y': y}
- p = PlotData(data, x='x', y=['y'], copy_data=True, same_length=True)

Example 5: Extending data with add_data_key
- data = {'x': x, 'y': y}
- p = PlotData(data, x='x', y=['y'])
- p.add_data_key('z', keys=[z1, z2])
- for entry in p:
  - entry.z will exist (values per set) if z was provided during initialization or via add_data_key

API reference (high level)
- PlotData(data, use_column_source=False, same_length=False, strict_data_keys=False, copy_data=False, **kwargs)
- data_keys (property): list of declared data keys
- add_data_key(data_key, keys=None): add a new data key to the existing plot sets

Notes
- The exact behavior of entry attributes depends on how you pass kwargs:
  - If all kwargs values are scalars, each plot set uses those constant values but yields one entry per set.
  - If any kwargs value is a list, multiple plot sets are created with per-set values taken from those lists.
- Bokeh integration is conditional on Bokeh being installed. If Bokeh is unavailable, ColumnDataSource and related copying helpers are inactive.
