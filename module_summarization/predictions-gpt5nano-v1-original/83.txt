CockroachEngine
===============

An extension of PostgresEngine that targets CockroachDB. This class adapts the base PostgreSQL engine to accommodate CockroachDB quirks while reusing the core PostgresEngine functionality.

Overview
--------

- Inherits from: PostgresEngine
- Purpose: Provide CockroachDB-compatible behavior by extending the PostgreSQL engine
- Key behavior: Attempts to enable a CockroachDB cluster setting during initialization

Class reference
--------------

- Class: CockroachEngine(PostgresEngine)

Attributes
----------

- engine_type: "cockroach"
- min_version_number: 0
  - Note: Versioning is not reliably enforced for CockroachDB in this implementation.

Initialization
--------------

__init__(config, extensions=(), log_queries=False, log_responses=False, extra_nodes=None) -> None

- config: dict
  - Connection configuration
- extensions: Sequence[str]
  - Optional extensions to load
- log_queries: bool
  - If True, log all executed queries
- log_responses: bool
  - If True, log responses received from the database
- extra_nodes: dict[str, CockroachEngine] | None
  - Optional mapping of additional nodes for cross-database operations

Description
-----------

Calls the superclass constructor with the provided parameters. This class does not introduce new initialization logic beyond what PostgresEngine provides.

Methods
-------

prep_database() -> None

- Behavior:
  - Attempts to run the following statement on a new connection:
    SET CLUSTER SETTING sql.defaults.experimental_alter_column_type.enabled = true;
  - If an asyncpg.exceptions.InsufficientPrivilegeError is raised, emits a medium-level warning via colored_warning:

    "=> Unable to set up Cockroach DB functionality may not behave as expected. Make sure your database user has permission to set cluster options."

Notes
-----

- Uses LazyLoader to lazily import asyncpg (via asyncpg = LazyLoader("asyncpg", globals(), "asyncpg")).
- The prep_database step is CockroachDB-specific and guards against insufficient privileges by warning rather than failing hard.

See also
---------

- PostgresEngine: The base engine this class extends to support CockroachDB.
- Piccolo Engine Documentation: For general engine usage and configuration patterns. 

Examples
--------

Basic usage:

.. code-block:: python

   from piccolo.engine.cockroach_engine import CockroachEngine

   cfg = {
       "database": "mydb",
       "user": "dbuser",
       "password": "secret",
       "host": "localhost",
       "port": 26257,
   }

   engine = CockroachEngine(config=cfg)
   await engine.connect()
   # ... use engine as with PostgresEngine ...
