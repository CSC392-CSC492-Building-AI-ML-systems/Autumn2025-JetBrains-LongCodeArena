Random Projection transformers
==============================

A simple and computationally efficient way to reduce the dimensionality of the data
by trading a controlled amount of accuracy (as additional variance) for faster
processing times and smaller model sizes.

Overview
--------

Random projections reduce dimensionality by projecting data onto a lower‑dimensional
subspace using a random matrix. The dimensions and distribution of the projection
matrix are chosen to preserve pairwise distances between samples with high probability.

Johnson–Lindenstrauss lemma
---------------------------

The main theoretical result behind the efficiency of random projection is the Johnson–
Lindenstrauss lemma. It states that a small set of points in a high‑dimensional space
can be embedded into a space of much lower dimension such that pairwise distances are
nearly preserved. The projection can be viewed as an orthogonal projection or as a random
matrix transform.

In particular, for data samples u and v drawn from a dataset of shape (n_samples, n_features),
there exists an eps‑embedding with a random projection p such that

  (1 - eps) ||u - v||^2 < ||p(u) - p(v)||^2 < (1 + eps) ||u - v||^2

with good probability, where eps ∈ (0, 1).

Random projection in scikit-learn
---------------------------------

This module provides two transformer implementations:

- GaussianRandomProjection: projects data using a dense Gaussian random matrix.
  The components are drawn from N(0, 1/n_components).

- SparseRandomProjection: projects data using a sparse random matrix (Achlioptas‑style
  distributions). A density parameter controls the fraction of non‑zero entries. If
  density is "auto", it is set to 1 / sqrt(n_features) by default.

Minimal dimensionality for an eps‑embedding
-------------------------------------------

To safely choose the number of components, use:

  johnson_lindenstrauss_min_dim(n_samples, eps=0.1)

This returns the minimal number of components required to guarantee an eps‑embedding with
a given number of samples. The bound depends on eps and n_samples and is independent of
the original number of features.

API and usage
--------------

Typical usage involves creating a transformer and applying fit_transform to your data:

.. code-block:: python

   from sklearn.random_projection import GaussianRandomProjection

   rp = GaussianRandomProjection(n_components=100)
   X_projected = rp.fit_transform(X)

Notes
-----

- Random projections trade a small amount of accuracy (variance) for significant
  gains in speed and memory efficiency, especially on very high‑dimensional data.

- The random matrix can be dense (Gaussian) or sparse, affecting memory usage and speed.
