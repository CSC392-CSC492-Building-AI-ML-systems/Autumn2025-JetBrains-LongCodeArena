Executing notebooks
===================

Overview
--------
This API documents components for executing notebook code cells programmatically. The primary entry point is the ExecutePreprocessor, which runs all code cells in a notebook, captures outputs, and updates metadata accordingly. It relies on nbclient's NotebookClient mechanisms to manage kernel execution and communicates status via kernel_info and execution events.

ExecutePreprocessor
-------------------
Inheritance
- Preprocessor
- NotebookClient

Purpose
- Executes all the cells in a notebook and updates outputs as part of the preprocessing step.

Constructor
- __init__(**kw)
  - If nb is not provided, a new NotebookNode is created.
  - Initializes both base classes (Preprocessor and NotebookClient) with the given keyword arguments.

API
- preprocess(nb, resources=None, km=None) -> (NotebookNode, dict)
  - nb: NotebookNode being processed (modified in place)
  - resources: Optional resources dictionary carried through the process
  - km: Optional KernelManager to reuse an existing kernel
  - Returns: (executed_nb, resources)
  - Behavior:
    - Reinitializes kernel state via NotebookClient.__init__(nb, km)
    - Resets execution trackers
    - Assigns resources if provided
    - Establishes a kernel using setup_kernel()
    - Retrieves kernel_info and updates nb.metadata["language_info"]
    - Iterates over notebook cells and invokes preprocess_cell for each
    - Applies widget metadata after execution
- preprocess_cell(cell, resources, index) -> (cell, dict)
  - cell: NotebookCell being processed
  - resources: Resources dictionary
  - index: Cell index
  - Behavior: Executes the cell via execute_cell(index, store_history=True) and returns the updated cell and resources

Notes
- The preprocess method internally calls NotebookClient.__init__ again to reset execution state, which is intentional to align with execution semantics.

Deprecation
- executenb(*args, **kwargs)
  - Deprecated alias that forwards to nbclient.execute.
  - Emits a FutureWarning recommending direct use of nbclient.execute.

HighlightMagicsPreprocessor
--------------------------
Purpose
- Detects and tags code cells that use a different language via magic extensions (e.g., %%R, %%bash).
- Tags cell metadata with magics_language to inform syntax highlighting and processing.

Default languages
- A mapping from magic extensions to pygments lexers, e.g.:
  - "%%R": "r"
  - "%%bash": "bash"
  - "%%cython": "cython"
  - "%%javascript": "javascript"
  - "%%julia": "julia"
  - "%%latex": "latex"
  - "%%octave": "octave"
  - "%%perl": "perl"
  - "%%ruby": "ruby"
  - "%%sh": "sh"
  - "%%sql": "sql"

User-configurable languages
- languages: Traitlets Dict allowing users to override or extend default_languages.

Internal setup
- __init__(config=None, **kw)
  - Calls super().__init__(config=config, **kw)
  - Merges user-defined languages into default_languages
  - Compiles a regular expression to detect language magics at the start of a code cell

Helper
- which_magic_language(source) -> str or None
  - Returns the associated lexer name if a magic extension is detected, otherwise None

preprocess_cell(cell, resources, cell_index) -> (cell, resources)
- If cell is of type "code" and a magic language is detected, adds:
  - cell["metadata"]["magics_language"] = detected_language
- Returns the (possibly modified) cell and resources

See also
- This preprocessor is often used in conjunction with ExecutePreprocessor to ensure proper language highlighting for cells containing magic language extensions.
