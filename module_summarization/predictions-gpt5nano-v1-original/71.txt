Kernel Ridge Regression
=======================

Module :mod:`sklearn.kernel_ridge` implements kernel ridge regression.

Overview
--------
Kernel ridge regression (KRR) combines ridge regression (linear least squares with l2-norm regularization) with the kernel trick. It learns a linear function in the space induced by the chosen kernel. For non-linear kernels, this corresponds to a non-linear function in the original space.

This estimator supports multi-target regression (i.e., when `y` is a 2D array of shape `[n_samples, n_targets]`). The model learned by KRR is equivalent to that of support vector regression (SVR) with a squared loss, but with closed-form fitting and typically faster training for medium-sized datasets. The learned model is non-sparse, in contrast to SVR.

Read more in the User Guide kernel_ridge.

Class
-----
KernelRidge
    Kernel ridge regression estimator.

    Parameters
    ----------
    alpha : float or array-like of shape (n_targets,), default=1.0
        Regularization strength. Must be a positive float. When an array is
        provided, penalties are assumed to be specific to the targets and must
        have the same length as the number of targets.

    kernel : {str, callable}, default="linear"
        Kernel mapping used internally. This is passed to
        :func:`sklearn.metrics.pairwise.pairwise_kernels` when `kernel` is a string.
        If `kernel` is a string, it must be one of the metrics in
        ``PAIRWISE_KERNEL_FUNCTIONS`` or "precomputed".
        If `kernel` is "precomputed", X is assumed to be a kernel matrix.
        If a callable is provided, it is called on each pair of instances
        (rows) and should return the kernel value for that pair. Callables
        from :mod:`sklearn.metrics.pairwise` operate on matrices and are not
        allowed here.

    gamma : float, default=None
        Gamma parameter for certain kernels (RBF, Laplacian, polynomial,
        exponential chi2, sigmoid). Its interpretation depends on the kernel.
        Ignored by kernels where it is not applicable.

    degree : int, default=3
        Degree of the polynomial kernel. Ignored by other kernels.

    coef0 : float, default=1
        Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

    kernel_params : dict, default=None
        Additional parameters for the kernel function, passed as keyword
        arguments when the kernel is called.

    Attributes
    ----------
    dual_coef_ : ndarray of shape (n_samples,) or (n_samples, n_targets)
        Representation of weight vector(s) in kernel space.

    X_fit_ : {ndarray, sparse matrix} of shape (n_samples, n_features)
        Training data used for fitting. If `kernel == "precomputed"`, this
        contains the precomputed training kernel matrix of shape
        (n_samples, n_samples).

    n_features_in_ : int
        Number of features seen during fit.

    feature_names_in_ : ndarray of shape (`n_features_in_`,)
        Names of features seen during fit. Defined only when the input has
        feature names that are all strings.

    See Also

    sklearn.gaussian_process.GaussianProcessRegressor : Gaussian process regression with kernels.
    sklearn.linear_model.Ridge : Linear ridge regression.
    sklearn.linear_model.RidgeCV : Ridge regression with built-in cross-validation.
    sklearn.svm.SVR : Support Vector Regression with various kernels.

    References
    ----------
    * Kevin P. Murphy, "Machine Learning: A Probabilistic Perspective",
      MIT Press, 2012, ch. 14.4.3, pp. 492-493

    Examples
    --------
    >>> from sklearn.kernel_ridge import KernelRidge
    >>> import numpy as np
    >>> n_samples, n_features = 10, 5
    >>> rng = np.random.RandomState(0)
    >>> y = rng.randn(n_samples)
    >>> X = rng.randn(n_samples, n_features)
    >>> krr = KernelRidge(alpha=1.0)
    >>> krr.fit(X, y)
    KernelRidge(alpha=1.0)

Notes
-----
- This estimator supports multi-target regression (2D `y`).
- If `kernel == "precomputed"`, pass a kernel matrix instead of feature data.
- The model is non-sparse; predictions scale with the number of training samples.

Parameters Constraints
----------------------
- alpha must be a positive float or a non-negative array-like with length equal to the number of targets.
- degree must be a non-negative integer.
- Other parameters follow the constraints described in the class documentation.

Usage
-----
Example usage with a linear kernel (default is "linear"):

.. code-block:: python

   from sklearn.kernel_ridge import KernelRidge
   import numpy as np

   n_samples, n_features = 10, 5
   rng = np.random.RandomState(0)
   y = rng.randn(n_samples)
   X = rng.randn(n_samples, n_features)

   krr = KernelRidge(alpha=1.0)
   krr.fit(X, y)
   y_pred = krr.predict(X)

Example with a non-linear kernel (RBF):

.. code-block:: python

   krr_rbf = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.5)
   krr_rbf.fit(X, y)
   y_pred = krr_rbf.predict(X)

Version and Compatibility
-------------------------
This class is available in scikit-learn and aligns with the public API as of the referenced release. The capability for multi-output regression was introduced to support handling multiple targets in a single model.
