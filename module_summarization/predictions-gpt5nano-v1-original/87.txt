Running migrations (module API)
===============================

Overview
--------
This module provides utilities to reverse migrations across one or more apps and to query the status of migrations. It includes a back‑wards migration manager, convenience wrappers for running backwards migrations, and a skeleton for a migration status checker.

API Reference
-------------

BackwardsMigrationManager
~~~~~~~~~~~~~~~~~~~~~~~~~
class BackwardsMigrationManager(BaseMigrationManager)

Constructor
```
__init__(
    app_name: str,
    migration_id: str,
    auto_agree: bool = False,
    clean: bool = False,
    preview: bool = False,
)
```
Parameters:
- app_name: Target app name (or 'all' to operate on all apps).
- migration_id: Specific migration id to start from or 'all' to reverse all migrations.
- auto_agree: If True, skip interactive prompts.
- clean: If True, delete the migration files from disk after reversal.
- preview: If True, only print SQL/queries without executing them.

Key methods
```
async def run(self) -> MigrationResult
```
- Creates the migration table if needed and runs the backwards migrations for the configured app.

```
async def run_migrations_backwards(self, app_config: AppConfig)
```
- Internal workflow to determine which migrations to reverse and execute them in reverse order. Returns a MigrationResult.

Notes:
- Uses Migration.get_migrations_which_ran, Migration.delete, and direct database operations to reverse state.
- Supports a preview mode and optional cleanup of migration files from disk.

run_backwards
~~~~~~~~~~~~~
async def run_backwards(
    app_name: str,
    migration_id: str = "1",
    auto_agree: bool = False,
    clean: bool = False,
    preview: bool = False,
) -> MigrationResult

- Reverses migrations for a single app or all apps.
- If app_name == 'all', iterates through apps in reverse order, prompting (unless auto_agree) and reversing all migrations for each app.
- Returns a MigrationResult indicating success or failure.

backwards
~~~~~~~~
async def backwards(
    app_name: str,
    migration_id: str = "1",
    auto_agree: bool = False,
    clean: bool = False,
    preview: bool = False,
)

- High‑level async function that wraps run_backwards and exits the process with a non‑zero status if the operation failed.
- Intended as a user‑facing entry point for reversing migrations.

Usage
-----

Undo the most recent migration for a single app
```
from your_module import backwards

# Async usage
import asyncio
async def main():
    await backwards(app_name="myapp", migration_id="1", auto_agree=True)

asyncio.run(main())
```

Undo all migrations for all apps
```
from your_module import run_backwards

import asyncio
async def main():
    await run_backwards(app_name="all", auto_agree=True)

asyncio.run(main())
```

Preview a reversal (no changes applied)
```
from your_module import backwards

import asyncio
async def main():
    await backwards(app_name="myapp", preview=True)

asyncio.run(main())
```

Notes
-----
- The module integrates with Piccolo’s migration system (Migration table, AppConfig, MigrationModule).
- Use auto_agree to bypass interactive confirmations.
- The clean option will remove migration files from disk after reversal when applicable.
- The CheckMigrationManager class is provided for potential status checks, but its core method is not fully implemented in the present code.
