w_ntop — select walkers from bins

Overview
- w_ntop selects walkers from bin/state assignments for each iteration.
- It requires an assignments file that maps walkers to bins at each timepoint (see w_assign --help for generation details).
- By default, w_ntop selects the highest-weight walkers from each bin; other modes (lowest-weight or random) are available.

Output file format
The output is an HDF5 file (default ntop.h5) containing the following datasets:

- /n_iter [iteration]
  - Integer. Iteration numbers corresponding to the entries in the other datasets.

- /n_segs [iteration][bin]
  - Integer. Number of segments in each bin/state for the given iteration.
  - Typically equals the requested count, but may be smaller if fewer matching walkers exist.

- /seg_ids [iteration][bin][segment]
  - Integer. Matching segment identifiers for each iteration and bin.
  - For a given iteration n_iter, only the first n_segs[n_iter][bin] entries are valid.
  - Example: seg_ids[0][0][:n_segs[0][0]] yields the valid segment IDs for bin 0 in the first stored iteration.

- /weights [iteration][bin][segment]
  - Floating-point. Weights corresponding to the matching segments in /seg_ids.

Dataset details
- The output is shaped as (n_iter, nbins) for /n_segs, and (n_iter, nbins, count) for /seg_ids and /weights.
- nbins is determined from the assignments file as nbins + 1 (nbins is stored in the file’s attributes).

Command-line interface
Input and common options
- -a, --assignments ASSIGNMENTS
  - Path to the assignments file (default: assign.h5). This file maps walkers to bins at each timepoint.

- -o, --output OUTPUT
  - Path to the output file (default: ntop.h5).

- --progress
  - Enables a progress indicator for long runs.

Selection and timepoint options
- -n, --count COUNT
  - Integer. Number of walkers to select from each bin (default: 1).

- -t, --timepoint TIMEPOINT
  - Integer. Base selection on the given timepoint within each iteration.
  - Default: -1, which corresponds to the last timepoint.

- Mutually exclusive selection methods
  - --highweight: Select COUNT highest-weight walkers from each bin (default).
  - --lowweight: Select COUNT lowest-weight walkers from each bin.
  - --random: Select COUNT walkers randomly from each bin.

Note: Exactly one of --highweight, --lowweight, or --random is active at a time. The default is --highweight.

Additional data-reading options
- The tool also accepts arguments provided by WESTDataReader and IterRangeSelection (for selecting iteration ranges), as well as standard WESTTool options.

How it works (high-level)
- For each iteration in the requested range:
  - It loads the assignments for the chosen timepoint, yielding the set of walkers assigned to each bin.
  - It retrieves the corresponding walker weights from the trajectory data.
  - For each bin, it selects up to COUNT walkers according to the chosen method (highest weight, lowest weight, or random).
  - It writes the selected segment IDs and their weights into /seg_ids and /weights, and records the number of selected segments per bin in /n_segs.

Examples
- Select 3 top-weight walkers per bin from the last timepoint and write to ntop.h5:
  w_ntop -a assign.h5 -n 3 -o ntop.h5

- Select 2 random walkers per bin at timepoint 0:
  w_ntop -a assign.h5 -n 2 -t 0 --random -o random_ntop.h5

- Select 4 lowest-weight walkers per bin from the 10th timepoint:
  w_ntop -a assign.h5 -n 4 -t 10 --lowweight -o lowweight_ntop.h5

Notes and requirements
- The assignments file must provide the necessary binning information for each timepoint and iteration.
- The output file includes n_iter, n_segs, seg_ids, and weights with appropriate data types (n_iter uses n_iter_dtype; weights use weight_dtype; seg_ids use seg_id_dtype).
- The number of bins in the output is determined by the nbins attribute in the assignments file (nbins + 1).
