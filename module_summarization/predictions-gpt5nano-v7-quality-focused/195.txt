Learn Gaussian Mixture Models
=============================

Overview
--------
This documentation describes the base framework for mixture models used to learn Gaussian mixtures. The BaseMixture class provides the EM (Expectation-Maximization) workflow, parameter initialization, and multi-run support. Concrete Gaussian Mixture Model implementations subclass this base to define how the means, covariances, and weights are updated in the M-step.

Key concepts
------------
- Mixture model: A probabilistic model that represents data as a combination of several component distributions (typically Gaussians).
- Responsibilities: The E-step computes the probability that each data point belongs to each component.
- Parameters:
  - Weights (mixing proportions): how much each component contributes to the mixture.
  - Means: centers of the Gaussian components.
  - Covariances: shapes and orientations of the Gaussian components.
- EM algorithm: Alternates between updating responsibilities (E-step) and updating parameters (M-step) until convergence or until the maximum number of iterations is reached.
- Initialization: The starting responsibilities influence convergence. This base class provides several initialization strategies.

API and behavior
-----------------
- Base class: BaseMixture (abstract)
- Core parameters (constructor):
  - n_components: number of mixture components (must be >= 1)
  - tol: convergence tolerance for the EM loop (must be >= 0)
  - reg_covar: regularization for covariances to ensure numerical stability (>= 0)
  - max_iter: maximum EM iterations per initialization
  - n_init: number of random restarts to perform
  - init_params: method to initialize responsibilities; one of {"kmeans", "random", "random_from_data", "k-means++"}
  - random_state: seed or RNG for reproducibility
  - warm_start: reuse solution across fit calls if True
  - verbose: verbosity level
  - verbose_interval: interval of iterations for verbose output
- Abstract methods to implement in concrete subclasses:
  - _check_parameters(X): validates and checks parameters for the specific model.
  - _initialize(X, resp): initializes model parameters (means, covariances, weights) given the initial responsibilities.
- Initialization of responsibilities:
  - "kmeans": uses KMeans to assign hard labels, then converts to one-hot responsibilities.
  - "random": initializes with random responsibilities and normalizes per data point.
  - "random_from_data": selects random component indices and creates a one-hot responsibility vector accordingly.
  - "k-means++": uses k-means++ style seeding to form initial responsibilities.
- Fitting:
  - fit(X, y=None): fits the model and returns self.
  - fit_predict(X, y=None): validates X, runs EM over multiple inits, selects the best solution, and returns the component labels for X.
  - The concrete mixture (e.g., GaussianMixture) exposes learned parameters such as weights_, means_, covariances_, and possibly a lower_bound_ per iteration.
- Validation:
  - Input X is validated for type (float64/float32) and minimum sample size (at least 2 samples).

Data validation and initialization (as implemented)
---------------------------------------------------
- The _initialize_parameters method chooses an initialization strategy based on init_params, produces a responsibility matrix resp of shape (n_samples, n_components), and delegates to _initialize to set the model parameters.
- Shapes are validated via internal helpers (e.g., _check_shape) to ensure parameters have expected dimensions.

Usage example
-------------
Typical usage relies on a concrete Gaussian Mixture implementation derived from BaseMixture (for example, GaussianMixture in many libraries). The base class handles initialization and the multi-start EM loop; the concrete class defines how to update means, covariances, and weights.

Example with a standard Gaussian Mixture implementation:

.. code-block:: python

  from sklearn.mixture import GaussianMixture
  import numpy as np

  # X should be an array of shape (n_samples, n_features)
  X = np.random.randn(300, 2)

  # Create a Gaussian Mixture model
  gm = GaussianMixture(n_components=3, covariance_type='full', random_state=0, init_params='kmeans')

  # Fit the model
  gm.fit(X)

  # Predict component labels for X
  labels = gm.predict(X)

  # Access learned parameters (example attributes present in concrete implementations)
  weights = gm.weights_
  means = gm.means_
  covariances = gm.covariances_

  print("Weights:", weights)
  print("Means:\n", means)
  print("Covariances:\n", covariances)

Notes:
- The exact attribute names (weights_, means_, covariances_) come from the concrete Gaussian Mixture implementation. They are typically available after a successful fit.
- You can also use fit_predict in one step:
  labels = gm.fit_predict(X)

Practical tips
--------------
- Initialization and restarts:
  - Use n_init > 1 to improve chances of finding a good local optimum.
  - Choose init_params to balance speed and robustness (e.g., 'kmeans' or 'k-means++' for sensible starts).
- Convergence:
  - tol controls when EM stops early; small values yield more precise convergence but longer training.
  - The base class emits a ConvergenceWarning if a run does not converge within max_iter.
- Stability:
  - reg_covar helps prevent singular covariance matrices, especially in high-dimensional settings.
- Reproducibility:
  - Set random_state to reproduce results across runs.
- Warm start:
  - If you plan to continue training, enable warm_start to reuse the existing solution.

Common pitfalls
---------------
- Too small tol or too few iterations may cause unnecessary computation or non-convergence warnings.
- Highly imbalanced component weights can lead to numerical issues; consider reg_covar and proper initialization.
- Ensure X has enough samples relative to n_components to avoid degenerate solutions.
