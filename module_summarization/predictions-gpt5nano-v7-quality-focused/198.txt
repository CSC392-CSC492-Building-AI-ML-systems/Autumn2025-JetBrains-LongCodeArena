Nearest Neighbors
===================

Overview
--------
This module provides fast nearest-neighbor search using tree-based indexes. The primary implementation is BallTree, with specialized 64-bit and 32-bit variants to optimize memory and compute for different data types. BallTree supports a wide range of distance metrics and can be used to perform exact nearest-neighbor queries as well as radius-based searches.

Public API
----------
- BallTree: Base BallTree implementation for generic use.
- BallTree64: 64-bit floating-point variant of BallTree.
- BallTree32: 32-bit floating-point variant of BallTree.

Notes
- BallTree variants inherit from a common BinaryTree base and are selected automatically based on input data types. Internally, distances are computed using 64-bit floating-point arithmetic for numerical stability.
- A set of distance metrics is supported. Each metric has a 64-bit and a 32-bit variant, named with the suffix 64 or 32 (for example, EuclideanDistance64 and EuclideanDistance32).

Supported distance metrics
--------------------------
The following metric families are available, with 64-bit and 32-bit variants:

- BrayCurtisDistance64 / BrayCurtisDistance32
- CanberraDistance64 / CanberraDistance32
- ChebyshevDistance64 / ChebyshevDistance32
- DiceDistance64 / DiceDistance32
- EuclideanDistance64 / EuclideanDistance32
- HammingDistance64 / HammingDistance32
- HaversineDistance64 / HaversineDistance32
- JaccardDistance64 / JaccardDistance32
- MahalanobisDistance64 / MahalanobisDistance32
- ManhattanDistance64 / ManhattanDistance32
- MinkowskiDistance64 / MinkowskiDistance32
- PyFuncDistance64 / PyFuncDistance32
- RogersTanimotoDistance64 / RogersTanimotoDistance32
- RussellRaoDistance64 / RussellRaoDistance32
- SEuclideanDistance64 / SEuclideanDistance32
- SokalMichenerDistance64 / SokalMichenerDistance32
- SokalSneathDistance64 / SokalSneathDistance32
- WMinkowskiDistance64 / WMinkowskiDistance32

Internal concepts
-----------------
- Reduced distance: For some metrics, a reduced distance is used during pruning. It preserves the relative ordering of true distances while allowing faster bounding computations.
  - Example: For Euclidean distance, the reduced distance is the squared Euclidean distance.
- Node structure: Each node stores a centroid and a radius that bounds all data points within the node. During queries, nodes with a potential to improve the current best distance are explored; others are pruned to speed up search.
- Data layout: The tree is built over a data matrix of shape (n_samples, n_features). Each node stores bounds (centroid and radius) for fast pruning.

Usage
-----
- Basic nearest-neighbor query with Euclidean distance:

.. code-block:: python

   from sklearn.neighbors import BallTree
   import numpy as np

   # Sample data: 100 points in 3D
   X = np.random.random((100, 3))

   # Build a BallTree (automatic selection between 64/32 bit variants)
   tree = BallTree(X, leaf_size=40, metric='euclidean')

   # Query the five nearest neighbors for the first five points
   dist, ind = tree.query(X[:5], k=5)

   print(dist)
   print(ind)

- Different metric usage (example with Manhattan distance):

.. code-block:: python

   tree = BallTree(X, leaf_size=40, metric='manhattan')
   dist, ind = tree.query(X[:5], k=3)

- Weighting (advanced): The tree build path can take weighted samples if sample weights are provided. When present, weights influence the centroid computation during node initialization.

Performance notes
-----------------
- Building the tree costs roughly O(n log n) time.
- Querying scales logarithmically with the number of points for well-behaved metrics, with pruning reducing the practical cost.
- Using 64-bit or 32-bit implementations can impact memory usage and numerical stability; choose 64-bit for higher precision and 32-bit for reduced memory.

Internal implementation details (for advanced users)
--------------------------------------------------
- BallTree variants specialize a BinaryTree by computing centroids and radii for each node and by using reduced distances to prune search space.
- The internal distance computations produce distances in 64-bit floating point, even when the data are stored as 32-bit floats.
- The implementation supports a variety of metrics, with the public API mapping user-specified metric names to the corresponding internal metric classes (e.g., EuclideanDistance64/EuclideanDistance32).
