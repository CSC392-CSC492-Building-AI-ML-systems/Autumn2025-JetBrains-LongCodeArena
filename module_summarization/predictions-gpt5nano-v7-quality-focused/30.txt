w_pdist
=======

Calculate time-resolved, multi-dimensional probability distributions of WE datasets.

Overview
--------
w_pdist computes normalized histograms of user-supplied data across iterations. The first axis of the resulting histograms corresponds to iteration number; the remaining axes correspond to the dimensions of the input data. The tool writes an HDF5 file (default name: pdist.h5) containing the histograms and the bin metadata necessary for plotting or further analysis.

Source data
-----------
Source data can come from one of two approaches, or fall back to the default progress coordinate:

- Custom function
  - Provide a Python function using standard MODULE.FUNCTION syntax via --construct-dataset.
  - The function is called as function(n_iter, iter_group) for each iteration.
  - The function must return data that supports indexing as [segment][timepoint][dimension].
- Data set specifications
  - Use --dsspecs to list datasets to flavor the histogram input.
  - Each specification has the form NAME[,file=FILENAME,slice=SLICE].
  - NAME is the dataset to use; FILENAME defaults to the main WEST HDF5 file west.h5; SLICE selects a subset of the dataset (Python slice syntax).
  - The slice option is especially useful for selecting one column (or a few columns) from a multi-dimensional dataset.
- Default
  - If neither --construct-dataset nor --dsspecs is provided, the tool uses the progress-coordinate dataset pcoord.

Histogram binning
-----------------
By default, w_pdist uses 100 bins in each input dimension.

Bin specification via -b/--bins supports several forms:

- A single integer N
  - Use N uniformly spaced bins in every dimension.
- A comma-separated sequence of integers N1,N2, ...
  - Use N1 bins in the first dimension, N2 in the second, etc.
- A list of lists [[B11, B12, B13, ...], [B21, B22, B23, ...], ...]
  - Provide explicit bin boundaries for each dimension.
  - Boundaries need not be uniform.
  - These expressions are evaluated with Python eval, with numpy (np) available for convenience.
  - Using explicit per-dimension bounds avoids scanning all data to determine min/max.

Notes
- The first two forms (single integer or sequence of integers) trigger a scan of all data to determine min and max per dimension.
- The list-of-lists form provides explicit boundaries without an on-the-fly scan.
- Bin boundaries are independent of WE sampling bins.

Output format
-------------
The output file is specified with -o/--output (default: pdist.h5). The file contains:

- histograms
  - Normalized histograms. The first axis is iteration; remaining axes correspond to the input data’s dimensions.
- /binbounds_0, /binbounds_1, ...
  - Per-dimension bin boundary arrays.
- /midpoints_0, /midpoints_1, ...
  - Per-dimension bin midpoints.
- n_iter
  - Vector of iteration numbers corresponding to the stored histograms (i.e., the first axis of histograms).

Subsequent processing
---------------------
The produced HDF5 file can be fed to plothist to generate plots or other processed outputs. See plothist --help for details.

Parallelization
---------------
w_pdist supports parallelized binning and data reading by default. For some simple cases (e.g., pre-computed input data or modest numbers of segments), serial processing (--serial) can be more efficient.

Command-line options
--------------------
Key options include:

- -o, --output
  - Path to the output HDF5 file (default: pdist.h5).
- -b, --bins
  - Bin specification (see Histogram binning above).
- --construct-dataset
  - Python function in MODULE.FUNCTION syntax to construct the input data.
- --dsspecs
  - Data set specifications for input data (NAME[,file=FILENAME,slice=SLICE]).
- --serial
  - Run in serial instead of parallel (may improve performance for some workloads).

Examples
--------
- Compute a default distribution using the built-in pcoord dataset:

.. code-block:: console

   w_pdist -o pdist.h5

- Use a specific dataset specification from the main West HDF5 file and 100 bins in each dimension:

.. code-block:: console

   w_pdist --dsspecs pcoord --bins 100 -o pdist.h5

- Provide explicit per-dimension bin boundaries (nonuniform) without scanning data:

.. code-block:: console

   w_pdist --bins [[0.0, 0.5, 1.0, 1.5], [0.0, 2.0, 4.0]] -o pdist.h5

- Use a custom Python function to derive the input data for binning:

.. code-block:: console

   w_pdist --construct-dataset mymodule.compute_pcoord --bins 100 -o pdist.h5

- Read multiple datasets as input (e.g., a multi-dimensional progress coordinate) by specifying datasets with per-dataset slices:

.. code-block:: console

   w_pdist --dsspecs "coords,file=trajectory.h5,slice=[0:2]" --dsspecs "times,file=trajectory.h5,slice=[2:3]" -b 50 -o pdist.h5

Notes
- The binning process is time-resolved: each iteration produces a histogram corresponding to that iteration’s data.
- The resulting histograms are normalized within each iteration.
- The stored datasets are intended to be compatible with plothist for visualization.
