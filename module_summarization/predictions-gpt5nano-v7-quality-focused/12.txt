A Quick Dive into Using the API
===============================

This quick guide shows how to start using the ftrack_api Session, configure common options, and perform lightweight tasks such as authenticating HTTP requests and enabling convenient features like on-demand data loading and schema caching.

Getting started: required configuration
---------------------------------------

A Session requires three core pieces of information:

- server_url: the ftrack server to connect to
- api_key: your API key for authentication
- api_user: the username for the API user

You can provide these as constructor arguments, or use environment variables (FTRACK_SERVER, FTRACK_API_KEY, FTRACK_API_USER). For backwards compatibility, FTRACK_APIKEY is also recognized.

Explicit session creation
-------------------------

# Provide all three values explicitly
import ftrack_api

session = ftrack_api.Session(
    server_url='https://my.ftrack.server',
    api_key='your_api_key',
    api_user='your_user'
)

Via environment variables
-------------------------

Set the environment variables and construct the Session with no arguments:

# Example (do this in your shell)
# export FTRACK_SERVER=https://my.ftrack.server
# export FTRACK_API_KEY=your_api_key
# export FTRACK_API_USER=your_user

import ftrack_api

session = ftrack_api.Session()

Optional configuration
----------------------

Tune common features to fit your workflow.

Auto-populate
--------------

Controls on-demand loading of entity attributes.

- Default: enabled (auto_populate = True)
- Effect: accessing an attribute fetches it from the server if not already loaded

session = ftrack_api.Session(auto_populate=False)

Caching
-------

Sessions work with a layered in-memory cache. You can supply your own cache instance or rely on the built-in layered setup.

- Provide a top-level cache (e.g. MemoryCache) and an optional key maker

from ftrack_api.cache import MemoryCache, StringKeyMaker

session = ftrack_api.Session(
    cache=MemoryCache(),
    cache_key_maker=StringKeyMaker()
)

Plugin discovery
----------------

Specify paths where plugins live.

session = ftrack_api.Session(
    plugin_paths=['/path/to/plugins']
)

Event hub behavior
------------------

Control how the embedded event hub connects to the server.

- auto_connect_event_hub = True connects the hub in the background automatically
  so you can publish/subscribe to non-local events without extra setup.
- If you disable automatic connection, you can connect later using the EventHub API (see ftrack_api.event.hub).

session = ftrack_api.Session(auto_connect_event_hub=True)

Schema caching
--------------

Enable per-project schema caching to speed up repeated access.

session = ftrack_api.Session(schema_cache_path='/path/to/schema/cache')

Plugin arguments
----------------

Pass arguments to discovered plugins if they declare accepted parameters.

session = ftrack_api.Session(plugin_arguments={'arg1': 'value1'})

Authentication for raw HTTP requests
------------------------------------

The SessionAuthentication class attaches ftrack-specific headers to HTTP requests. You can use it directly with a requests.Session for ad-hoc HTTP calls outside the high-level API.

import requests
from ftrack_api import SessionAuthentication

auth = SessionAuthentication(api_key='your_api_key', api_user='your_user')

with requests.Session() as s:
    s.auth = auth
    r = s.get('https://my.ftrack.server/rest/endpoint')

Notes
-----

- If auto_populate is enabled, you can access entity attributes and they will be fetched on demand if missing.
- The session will integrate any provided cache into a pre-configured layered cache with a MemoryCache at the top level.
- Environment variables provide a convenient quick-start path when you don’t want to pass explicit arguments in code.

This quick dive covers the essentials to get up and running with the API and introduces the main configuration knobs you’ll use in typical workflows.
