Diophantine Diophantine Equations Solver
=========================================

Overview
--------
The diophantine module provides a function to solve Diophantine equations symbolically. The main entry point is diophantine(eq, param, syms), which transforms the given equation into a product of terms that equals zero, solves each term independently, and then merges the results into a set of parametric solutions. Solutions are returned as sets of tuples, where each tuple contains the values for the variables in a fixed order.

Key ideas
- The equation is converted to a product-of-terms form: (term1) * (term2) * ... = 0.
- Each term is solved using diop_solve (from the diophantine solver) and then merged into solutions for the entire equation.
- Variables are ordered either by a supplied syms sequence or alphabetically.
- The function can handle equations with rational coefficients and denominators by separating numerator and denominator and solving appropriately.
- If factoring would cause issues for certain known types (e.g., general_sum_of_squares), the function may skip factoring and rely on other handling.

API
----
diophantine(eq, param=symbols('t', integer=True), syms=None)

- eq
  - Type: Relational or Expr
  - Description: The equation to solve. If it is a Relational (Eq), the equation is reduced to lhs - rhs.
- param
  - Type: Symbol
  - Description: The parameter symbol used to express infinite families of solutions. Default is a symbol t with integer attribute.
- syms
  - Type: sequence of Symbol
  - Description: Optional sequence specifying the variable order in the returned solution tuples. If provided, only variables present in the equation are kept and ordered according to this sequence; otherwise, variables are ordered alphabetically.

Return value
------------
A set of tuples. Each tuple corresponds to a complete solution, with components ordered according to the final variable order (as determined by syms or default ordering). Each component is a SymPy expression representing the solution for the corresponding variable.

Example results
- diophantine(x**2 - y**2)
  -> {(t_0, -t_0), (t_0, t_0)}
- diophantine(x*(2*x + 3*y - z))
  -> {(0, n1, n2), (t_0, t_1, 2*t_0 + 3*t_1)}
- diophantine(x**2 + 3*x*y + 4*x)
  -> {(0, n1), (3*t_0 - 4, -t_0)}

Notes
-----
- The function is designed to return all parametric, independent families of solutions by treating each factor as a separate Diophantine component and combining results where appropriate.
- If no nontrivial solution exists but the zero assignment satisfies the equation, the trivial solution is returned.
- If the equation is not a polynomial with rational coefficients, or if an unhandled type is encountered, the function raises TypeError or NotImplementedError as appropriate.
- diophantine relies on internal helpers and other components such as classify_diop, diop_solve, factor_list, and signsimp to determine types, factor terms, and solve subproblems.

Internal helpers (private)
--------------------------
These helpers support the solving process and are not part of the public API:

- _is_int(i)
  - Returns True if i is an integer, otherwise False.
- _sorted_tuple(*i)
  - Returns a sorted, immutable tuple of the given items.
- _remove_gcd(*x)
  - Attempts to divide all elements by their gcd; returns the reduced tuple or original if inapplicable.
- _rational_pq(a, b)
  - Returns a reduced rational pair (numer, denom) for a/b, removing common gcd and placing the sign on the numerator.
- _nint_or_floor(p, q)
  - Returns the nearest integer to p/q; ties resolve to floor(p/q).
- _odd(i), _even(i)
  - Helpers returning whether i is odd or even.

Associated and related functionality
------------------------------------
- classify_diop(eq)
  - Classifies a polynomial equation into known Diophantine problem types. The code uses this to decide whether to factor or skip factoring for certain complex types.
- diop_solve(base, param)
  - Solves a single Diophantine subproblem for a base term, returning candidate solutions which are merged into full solutions.
- factor_list(eq)
  - Factors a polynomial expression into a list of terms to be processed.
- merge_solution(var, var_t, solution)
  - Merges the solutions for a subterm into a full solution vector consistent with the variable ordering.
- sign, signsimp, _mexpand
  - Utilities used to simplify expressions and maintain correct signs during processing.
- __all__ exports
  - diophantine, classify_diop

Known Diophantine problem types
--------------------------------
The solver recognizes a set of known problem types (the diop_known set). Examples include:
- binary_quadratic
- cubic_thue
- general_pythagorean
- general_sum_of_even_powers
- general_sum_of_squares
- homogeneous_general_quadratic
- homogeneous_ternary_quadratic
- homogeneous_ternary_quadratic_normal
- inhomogeneous_general_quadratic
- inhomogeneous_ternary_quadratic
- linear
- univariate

Usage tips
----------
- Use syms to control the order of variables in the returned solution tuples.
- If you expect a family of solutions parameterized by t, keep param as the parameter symbol (default t).
- For complex equations, be prepared for the possibility that factoring may be skipped for certain types to avoid hanging on difficult factorizations.

Examples
--------
Python code snippets illustrate common usage:

- Solve a simple difference of squares:
  from diophantine import diophantine
  diophantine(x**2 - y**2)

- Solve a linear combination with a third variable:
  diophantine(x*(2*x + 3*y - z))

- Solve a polynomial in two variables:
  diophantine(x**2 + 3*x*y + 4*x)
