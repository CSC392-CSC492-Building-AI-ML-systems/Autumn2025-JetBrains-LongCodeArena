basics.indexing.rst

Overview
========

basics.indexing provides utilities for manipulating structured NumPy arrays.  
These helpers simplify working with multi-field and nested (structured)
dtypes, including extracting field names, flattening descriptions, and
filling one structured array with data from another.

Key ideas
---------

- Work with structured and nested dtypes (recarrays, MaskedArray records, etc.)
- Support nested field structures and provide convenient dtype descriptions
  that are easy to reconstruct
- Include small, focused helpers for common tasks such as filling fields and
  listing field names

Public API (implemented in this module)
=======================================

recursive_fill_fields
---------------------

Fill fields from an input array into an output array, with support for nested
structures.

Parameters
- input: ndarray
  Input array containing data to copy from.
- output: ndarray
  Output array to be filled. Should be at least the same size as input.

Notes
- Iterates over the fields in the output dtype. For each field, if the input
  contains a corresponding field, its data is copied into the output.
- If a field is itself a structured array (has nested names), the function
  recurses into the sub-field.
- Non-matching fields are skipped.

Returns
- The filled output array (same object as provided).

Examples
--------

>>> from numpy.lib import recfunctions as rfn
>>> a = np.array([(1, 10.), (2, 20.)], dtype=[('A', np.int64), ('B', np.float64)])
>>> b = np.zeros((3,), dtype=a.dtype)
>>> rfn.recursive_fill_fields(a, b)
array([(1, 10.), (2, 20.), (0,  0.)], dtype=[('A', '<i8'), ('B', '<f8')])

Internal helpers
----------------

_get_fieldspec(dtype)
---------------------

Produce a list describing the field structure of a dtype. The list consists of
name/dtype pairs suitable for reconstructing the dtype, with careful handling
of subarray dtypes. This is an internal helper and may be used by other
public routines.

- If the dtype has no named fields, returns [('', dtype)].
- If the dtype has fields, returns a list of (name, field_dtype) pairs.
  For fields with substructure, preserves the nested naming when possible.

get_names
---------

Return the field names of a dtype as a tuple. Supports nested structures by
returning a nested tuple structure that mirrors the dtype hierarchy.

Parameters
- adtype: dtype
  The input dtype with fields.

Returns
- A tuple of field names, with nesting represented as nested tuples.

Examples
--------

>>> from numpy.lib import recfunctions as rfn
>>> rfn.get_names(np.empty((1,), dtype=[('A', int)]).dtype)
('A',)
>>> rfn.get_names(np.empty((1,), dtype=[('A',int), ('B', float)]).dtype)
('A', 'B')
>>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
>>> rfn.get_names(adtype)
('a', ('b', ('ba', 'bb')))

get_names_flat
--------------

Return the flattened field names of a dtype as a tuple. Nested structures are
flattened into a simple sequence of names.

Parameters
- adtype: dtype
  The input dtype with fields.

Returns
- A tuple of flattened field names.

Examples
--------

>>> from numpy.lib import recfunctions as rfn
>>> rfn.get_names_flat(np.empty((1,), dtype=[('A', int)]).dtype) is None
False
>>> rfn.get_names_flat(np.empty((1,), dtype=[('A',int), ('B', str)]).dtype)
('A', 'B')
>>> adtype = np.dtype([('a', int), ('b', [('ba', int), ('bb', int)])])
>>> rfn.get_names_flat(adtype)
('a', 'b', 'ba', 'bb')

flatten_descr
-------------

Flatten a structured dtype description to a sequence of (name, dtype) pairs.

Parameters
- ndtype: dtype
  The input dtype to flatten.

Returns
- A tuple of (name, dtype) pairs describing all fields at all nesting levels.

Examples
--------

>>> from numpy.lib import recfunctions as rfn
>>> ndtype = np.dtype([('a', '<i4'), ('b', [('ba', '<f8'), ('bb', '<i4')])])
>>> rfn.flatten_descr(ndtype)
(('a', dtype('int32')), ('ba', dtype('float64')), ('bb', dtype('int32')))

Notes
- If the input dtype is not structured, the result is a single entry with an
  empty name and the original dtype.
- Useful for reconstructing dtypes or for describing field layouts in a flat
  manner.

Usage tips
----------

- Use get_names_flat to quickly enumerate all leaf fields in a nested dtype.
- Use flatten_descr to obtain a compact, reconstruction-friendly description
  of a nested dtype.
- Use recursive_fill_fields when you need to propagate data from one
  structured array into another, especially when dealing with nested fields.

Examples: combining functionality
----------------------------------

- List all nested field names, then create a new array with a subset of fields
  by selecting from get_names_flat and corresponding dtypes from flatten_descr.

Internal notes
--------------

- This module focuses on readability and practical handling of nested dtypes.
- Public functions are designed to be composable with NumPy’s structured array
  support, and to work cleanly with numpy.ma MaskedArray types where relevant.

See Also
--------

- Other utilities in this module handle creating, dropping, and joining fields
  within structured arrays (not all shown here). Refer to the module
  documentation for a complete API surface.
