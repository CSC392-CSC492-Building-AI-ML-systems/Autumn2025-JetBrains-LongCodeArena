Handling Large Datasets: Moment Maps (big_data.rst)

Overview
- This documentation describes efficient computation of moment maps for large spectral cubes.
- Three strategies are provided to balance speed and memory:
  - Cubewise: compute moments using the full cube in memory.
  - Slicewise: compute moments one 2D plane at a time along a chosen axis.
  - Raywise: compute moments by iterating over rays along the chosen axis (streaming-friendly).
- The code exposes moment computation through moment_auto, which selects a strategy based on the data.

Public API for large-data moment computations
- moment_auto(cube, order, axis)
  - Builds a moment map for the given axis and order using an automatic strategy.
  - Strategy is chosen by iterator_strategy(cube, axis).
  - Returns a 2D moment map with the axis collapsed.
- moment_cubewise(cube, order, axis)
  - Computes the moment using the entire data array at once (memory-intensive).
  - Suitable for smaller cubes where memory is not a concern.
- moment_raywise(cube, order, axis)
  - Computes the moment by processing one ray at a time.
  - Memory-efficient and suitable for very large datasets.
- moment_slicewise(cube, order, axis)
  - Computes the moment slice-by-slice (2D planes) along the given axis.
  - Balances memory usage with computation by processing one plane at a time.

Internal helpers (for completeness)
- _moment_shp(cube, axis)
  - Returns the shape of the moment map after collapsing along axis.
- _slice0(cube, axis)
  - 0th moment computed slicewise.
- _slice1(cube, axis)
  - 1st moment computed slicewise.
- These internal functions support the slicewise strategy and are not public API.

Key concepts and behaviors
- Axis convention
  - The axis parameter follows NumPy conventions (0, 1, 2 for the three dimensions).
  - The moment maps returned by the public APIs have the specified axis collapsed.
- Data handling and NaNs
  - Slicewise 0th moment handles NaNs by accumulating with NaN-to-zero conversion and marking invalid positions as NaN.
  - Higher-order moments rely on weighted sums with pixel sizes; masked or missing data are handled through the cube’s internal masking and fill logic.
- Weighting
  - 1st moment uses the intensity-weighted centroid along the axis, with weights based on pixel sizes.
  - Higher-order moments use deviations from the centroid (mom1) raised to the requested order, weighted by data and pixel size.
- Masking and validity
  - Raywise mode uses the cube’s mask to determine which rays to include; if no pixels are valid, the ray is skipped.
  - Cubewise and slicewise modes rely on the cube’s fill behavior and masking through the cube’s internal methods.

Choosing a strategy for large datasets
- Cubeways (moment_cubewise)
  - Fastest when the data fits in memory.
  - Not suitable for very large cubes due to high memory usage.
- Slicewise (moment_slicewise)
  - Processes one 2D plane at a time; memory footprint is modest.
  - Useful when memory is constrained and axis orientation yields manageable plane sizes.
- Raywise (moment_raywise)
  - Streams data along rays; efficient in memory usage for very large datasets.
  - May incur Python loop overhead; best when streaming access is advantageous and axis-oriented rays are tractable.

Example usage
- Compute 0th moment along the spectral axis (axis=2) with automatic strategy
  - cm0 = moment_auto(cube, order=0, axis=2)
- Compute 1st moment along the spectral axis with automatic strategy
  - cm1 = moment_auto(cube, order=1, axis=2)
- Compute higher-order moments along the spectral axis with automatic strategy
  - cm2 = moment_auto(cube, order=2, axis=2)

Forcing a specific strategy (alternative approaches)
- If you want to explicitly choose a strategy, call the specific function directly:
  - cm0_cubewise = moment_cubewise(cube, order=0, axis=2)
  - cm0_slicewise = moment_slicewise(cube, order=0, axis=2)
  - cm0_raywise = moment_raywise(cube, order=0, axis=2)

Notes and tips
- The shapes of moment maps are cube.shape[:axis] + cube.shape[axis+1:], i.e., the axis is removed.
- Practical choice of axis often depends on data layout and plane sizes; choosing the axis that minimizes the per-slice plane size can reduce memory and improve speed for slicewise processing.
- If your dataset is extremely large or contains many masked regions, raywise or slicewise processing often offers better scalability than loading the entire cube into memory.

API Reference (summary)
- moment_auto(cube, order, axis): selects an automatic strategy and computes the moment.
- moment_cubewise(cube, order, axis): computes using the full cube in memory.
- moment_raywise(cube, order, axis): computes by iterating over rays.
- moment_slicewise(cube, order, axis): computes slice-by-slice along the axis.
- Internal helpers:
  - _moment_shp, _slice0, _slice1 (used by slicewise strategy)

Implementation notes
- The code uses cube internals such as _get_filled_data, _pix_size_slice, _pix_cen, and _iter_rays to perform computations.
- NaN handling and masking are integral to robust moment calculations on real astronomical data.
