filenode
--------

A file interface to nodes for PyTables databases.

The FileNode module provides a file interface for using inside of PyTables database files. Use the new_node() function to create a brand new file node which can be read and written as any ordinary Python file. Use the open_node() function to open an existing node (created with new_node()) for read-only or read-write access. Read access is always available. Write access (enabled on new files and files opened with mode 'a+') only allows appending data to a file node. Currently only binary I/O is supported.

See filenode_usersguide for instructions on use.

Version notes:
- version 3.0: The module is rewritten to be fully compliant with the interfaces defined in the io module.

Public constants
----------------

NodeType
    Value for NODE_TYPE node system attribute.

NodeTypeVersions
    Supported values for NODE_TYPE_VERSION node system attribute.

Public classes
--------------

RawPyTablesIO
    Base class for raw binary I/O on HDF5 files using PyTables.

    Description
        Provides a low-level, binary IO interface that conforms to io.RawIOBase.
        It operates on a PyTables file node and exposes a minimal set of
        operations needed for binary streaming.

    Inheritance
        io.RawIOBase

    Key attributes (internal)
        - _node: PyTables node instance backing this IO
        - _mode: file-like mode used for I/O
        - _pos: current stream position (in bytes)
        - _version: NODE_TYPE_VERSION value from the node
        - _vshape: function mapping length to a shape, per version
        - _vtype: underlying data type of the node's atom

    Important internal helpers (not part of the public API)
        - _check_node(node): validates the node is a file node
        - _check_attributes(node): validates required attributes exist
        - _check_mode(mode): validates a provided mode string
        - _cross_check_mode(mode, current_mode): ensures mode compatibility with the underlying file

    Public API

    mode
        Read-only attribute that returns the current file mode.

    tell()
        Return the current stream position.

    seek(pos, whence=0)
        Change the stream position to a byte offset. Supports:
        - whence = 0 (start of stream)
        - whence = 1 (current position)
        - whence = 2 (end of stream)
        Returns the new absolute position.

    seekable()
        Returns True, as random access is supported.

    fileno()
        Returns the underlying file descriptor, if available. Raises IOError
        if no file descriptor exists for the host PyTables file.

    close()
        Flushes and closes the IO object. After closing, the node reference is
        released to allow the host file to be closed.

    flush()
        Flushes write buffers if applicable. Not implemented for read-only
        or non-blocking streams.

    truncate(pos=None)
        Truncates the file to the given size in bytes. If pos is None, uses the
        current position. Truncation is intended for growing the file; data
        rewriting or deletion is not supported. Returns the new size.

    readable()
        Returns True if the object was opened for reading (mode includes 'r' or '+').

    writable()
        Returns True if the object was opened for writing (mode includes 'w', 'a',
        or '+').

    readinto(b)
        Read up to len(b) bytes into the writable buffer b. Returns the number of
        bytes read, or 0 on EOF. Updates the current position.

        Notes:
        - Uses the node's read(start, stop) under the hood.
        - If the requested stop exceeds the node length, it reads only up to
          the available data.

    readline(limit=-1)
        Read and return a line from the stream. If limit is provided, reads at most
        limit bytes. (Implementation detail: exact behavior follows binary line
        scanning semantics.)

Usage
-----

The module exposes a binary IO interface that integrates with Python's io
infrastructure. You obtain a PyTables file node (via new_node or open_node) and
wrap it in a RawPyTablesIO instance to perform low-level binary operations.

Example: basic binary read after opening a node

.. code-block:: python

    from filenode import new_node
    from filenode import RawPyTablesIO

    # Create a new file node (read/write, binary)
    node = new_node('example_node', mode='w+')

    # Obtain a raw binary IO object for the node
    f = RawPyTablesIO(node, mode='r+')

    # Move to start and read data (if any)
    f.seek(0, 0)
    buf = bytearray(1024)
    n = f.readinto(buf)
    data = bytes(buf[:n])

    # Append more data by truncating to a larger size or writing through the node
    f.truncate(1024)       # grows the node by appending zeros if needed
    f.seek(0)
    f.close()

Notes
- The available operations mirror the capabilities of the underlying PyTables file node.
- Read access is always available; write access is controlled by the mode string.
- The implementation relies on the NODE_TYPE and NODE_TYPE_VERSION attributes
  to determine how to translate binary IO calls into PyTables operations.
