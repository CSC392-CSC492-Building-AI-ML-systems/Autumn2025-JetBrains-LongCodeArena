Pipelines and composite estimators
===================================

Overview
--------
The pipeline utilities in this module enable building composite estimators by chaining
transforms with a final estimator. A pipeline executes transforms in sequence and passes
the transformed data to the final estimator. All intermediate steps are transformers
(with fit and transform methods), while the final step must be an estimator (with fit).

Key ideas:
- Chain transforms and a final estimator into a single object.
- Set parameters for any step using the double-underscore syntax (step__param).
- Optionally cache fitted transformers to speed up repeated fits with the memory parameter.
- Access steps via a convenient named_steps attribute.

API reference
-------------
Pipeline
- Purpose: Chain a sequence of transforms and a final estimator.
- Construction: Pipeline(steps, memory=None, verbose=False)
  - steps: list of (name, transform) pairs. All but the last must be transformers; the last must be an estimator.
  - memory: None or a path or object implementing a caching interface (joblib.Memory). When provided, fitted transformers are cached. The last step is never cached.
  - verbose: If True, prints timing information for each step as it completes.

- Important behavior:
  - Parameters can be set for any step using the __ syntax, e.g., pipeline.set_params(svc__C=1.0).
  - The pipeline itself exposes get_params for parameter introspection, including nested parameters of contained steps (deep=True).

- Output configuration:
  - The set_output method configures the output container for transform and fit_transform calls across all steps.
  - transform options:
    - "default": default output format of a transformer
    - "pandas": output as a pandas DataFrame
    - None: transform configuration remains unchanged
  - Example:
    pipe.set_output(transform="pandas")

- Attributes:
  - named_steps: A dictionary-like object (Bunch) mapping step names to step instances, for convenient access.
  - classes_, n_features_in_, feature_names_in_: metadata about the fitted pipeline
    - classes_: only present if the last step is a classifier
    - n_features_in_: number of features seen during fit (added in version 0.24)
    - feature_names_in_: feature names seen during fit (added in version 1.0)

- Design note:
  - The pipeline is a _BaseComposition, enabling composition of estimators and reuse with utility functions like make_pipeline.

Usage examples
--------------
Basic usage

.. code-block:: python

   from sklearn.pipeline import Pipeline
   from sklearn.preprocessing import StandardScaler
   from sklearn.svm import SVC

   # Create a simple pipeline: scale features, then classify
   pipe = Pipeline([
       ('scaler', StandardScaler()),
       ('svc', SVC())
   ])

   # Fit and score
   # X_train, y_train, X_test, y_test are assumed to be defined
   pipe.fit(X_train, y_train).score(X_test, y_test)

Parameter setting with __

.. code-block:: python

   pipe.set_params(svc__C=10).fit(X_train, y_train).score(X_test, y_test)

Caching with memory
-------------------
Enable caching of transformers to speed up repeated fits. The last step is never cached.

.. code-block:: python

   pipe = Pipeline([
       ('scaler', StandardScaler()),
       ('pca', PCA()),
       ('clf', LogisticRegression())
   ], memory='tmp_cache_dir')

Transform output configuration
------------------------------
Configure how transform results are returned from the pipeline steps.

.. code-block:: python

   pipe.set_output(transform="pandas")

Notes
-----
- The final estimator in the pipeline must implement fit; intermediate steps must implement fit and transform.
- The parameter syntax allows replacing or removing steps:
  - Replace a step by setting its parameter to a new estimator
  - Remove a step by setting it to 'passthrough' or None
- The docs reference examples that show grid searches across pipeline parameters via __ notation.

See also
--------
- FeatureUnion: combines multiple transformer outputs in parallel.
- make_pipeline: a convenience function to build pipelines succinctly.
- make_union: a convenience function for building unions of transformers.

Version notes
-------------
- n_features_in_ attribute is available when the underlying estimator exposes it (added in 0.24).
- feature_names_in_ is available when feature names are provided or inferred (added in 1.0).
