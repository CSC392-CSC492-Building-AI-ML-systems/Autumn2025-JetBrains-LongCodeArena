Condition Syntax in PyTables
==============================

Overview
--------
PyTables evaluates user-supplied boolean expressions, referred to as conditions, against rows of a table. Each symbol in a condition corresponds to a column (or field) of the table. Conditions are used to filter data when iterating, reading, or locating specific rows.

Supported constructs
--------------------
- Column references: Use column names that exist in the table. Each reference in an expression maps to a column’s value for the current row.
- Constants: Numeric, string, and boolean literals are supported in expressions.
- Comparisons: Standard operators are supported:
  - Equal: ==
  - Not equal: !=
  - Greater than: >
  - Greater or equal: >=
  - Less than: <
  - Less or equal: <=
- Logical operators: Combine comparisons with:
  - and
  - or
  - not
  Expressions can be grouped with parentheses for clarity and precedence.

Notes on NaN and missing values
--------------------------------
- PyTables uses NaN-aware comparison helpers for numeric data. In practice, that means NaN values are treated as incomparable in most comparisons, and expressions involving NaNs yield False for the comparison part. This behavior helps avoid surprising results when filters encounter missing data.

Using conditions with a Table
-----------------------------
PyTables provides convenient methods to apply conditions:

- Table.where(expression)
  - Returns a generator yielding rows that satisfy the condition.
  - Example:
    - for row in table.where('(age > 20) and (score >= 80): print(row)')
- Table.readWhere(expression)
  - Reads and returns all rows that satisfy the condition as a NumPy structured array.
  - Example:
    - data = table.readWhere('name == "John"')
- Table.get_where_list(expression)
  - Returns the row indices that satisfy the condition as a NumPy array.
  - Example:
    - idx = table.get_where_list('age > 50')

Performance considerations
--------------------------
- If the table has an index on the involved columns, PyTables uses the index to speed up evaluation of the condition.
- A condition cache exists at the table level to optimize repeated queries. The cache is invalidated automatically when the underlying data changes in a way that would affect the result.
- Enabling and tuning automatic indexing (Table.autoindex) can improve performance for frequent queries.

Practical examples
------------------
- Simple filter on a numeric column:
  - for row in table.where('age >= 18 and status == True'):
      print(row['name'], row['age'])
- Read all rows matching a string condition:
  - data = table.readWhere("city == 'New York'")
- Retrieve indices of rows that satisfy a condition:
  - idx = table.get_where_list('score < 50')

Tips and gotchas
---------------
- Use parentheses to make precedence explicit and avoid ambiguous results.
- Quote string literals with single quotes inside the condition string.
- If you modify data that affects a condition’s result, the internal condition cache updates automatically.
- Invalid column names in a condition raise an error; ensure column names exactly match the table’s schema.
