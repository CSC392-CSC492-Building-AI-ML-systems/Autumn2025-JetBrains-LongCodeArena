Transforming the prediction target
===================================

Overview
--------

Sometimes the distribution of the target variable y is skewed or heteroscedastic, which can hamper regression performance. Transforming the target before fitting a model and then inverting the transformation on predictions can improve model fit and interpretability.

Key concept
-----------

Use TransformedTargetRegressor to apply a forward transformation to the target y during fitting and automatically apply the inverse transformation to predictions returned by the trained model.

API
---

- regressor: The estimator to wrap (e.g., LinearRegression, RandomForestRegressor).
- func: Callable that transforms the target y to a transformed space.
- inverse_func: Callable that inverts the transformation, mapping predictions back to the original space.
- check_inverse: Bool, default True. If True, validates that inverse_func is indeed the inverse of func (on a sample).

Notes
-----

- The transformation applies only to the target y, not to the input features X.
- Predictions produced by the wrapped regressor are mapped back to the original target space via inverse_func.
- Works with single-output and multi-output regression if the underlying regressor supports multi-output targets.

Usage
-----

Example: apply a log1p transform to the target to stabilize variance, then invert with expm1 for predictions.

```python
from sklearn.compose import TransformedTargetRegressor
from sklearn.linear_model import LinearRegression
import numpy as np

# Define the transformation and its inverse
regressor = TransformedTargetRegressor(
    regressor=LinearRegression(),
    func=np.log1p,
    inverse_func=np.expm1
)

# Fit and predict
regressor.fit(X, y)
y_pred = regressor.predict(X_new)  # y_pred is in the original space
```

Example: disable inverse check if you’re certain about the transformation

```python
regressor = TransformedTargetRegressor(
    regressor=LinearRegression(),
    func=np.log1p,
    inverse_func=np.expm1,
    check_inverse=False
)
```

Advanced tips
-------------

- If you require a custom, potentially non-invertible transform, ensure both func and inverse_func are properly defined and handle edge cases (e.g., non-positive values for log transforms).
- For multi-output targets, ensure the transformation function can operate on the full target array or apply a per-output transform as needed, consistent with the regressor’s expectations.
