Using metadata
==============

Overview
--------

In ftrack_api, metadata describes auxiliary information that accompanies API usage to optimize and streamline interactions with the server. This includes:

- Authentication metadata attached to HTTP requests
- Metadata about server schema used for caching (schema metadata)
- Per-session caching metadata (cache and key generation)
- Plugin-related metadata provided to the discovery process

Authentication metadata
-----------------------

Requests are authenticated by attaching metadata to each request via the SessionAuthentication class. This adds the following headers to outgoing requests:

- ftrack-api-key
- ftrack-user

Example
-------

.. code-block:: python

# Create an authentication object that provides metadata for requests
from ftrack_api.session import SessionAuthentication  # or the appropriate module path

auth = SessionAuthentication(api_key="YOUR_API_KEY", api_user="YOUR_USERNAME")

# Use the auth object with a HTTP client (e.g., requests)
import requests
response = requests.get("https://server/api", auth=auth)

Schema metadata (schema caching)
--------------------------------

Schema metadata is used to cache server schema information to speed up startup and repeated operations.

- Enable by setting schema_cache_path to a folder path.
- If not set, the environment variable FTRACK_API_SCHEMA_CACHE_PATH is used.
- If neither is set, a temporary directory is used.
- Set to False to disable schema caching entirely.

Parameters related to schema caching are exposed on Session via the schema_cache_path argument.

Examples
--------

.. code-block:: python

# Enable schema caching by providing a cache path
session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    schema_cache_path="/path/to/schema_cache"
)

# Disable schema caching
session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    schema_cache_path=False
)

Cache metadata
--------------

Caching stores object data locally to avoid repeated server round-trips. The session uses a layered cache with a top-level in-memory cache to ensure consistent identity across the session.

- cache: Can be an instance implementing ftrack_api.cache.Cache or a callable that receives the session and returns a cache instance (or None if no suitable cache is configured).
- cache_key_maker: Determines how keys for cached objects are generated. If not provided, a StringKeyMaker is used by default.

Examples
--------

.. code-block:: python

# Use a dedicated cache instance
from ftrack_api.cache import MemoryCache, StringKeyMaker

memory_cache = MemoryCache()

session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    cache=memory_cache
)

# Provide a custom cache via a callable
def build_cache(session):
    # Return a cache instance or None if not applicable
    return MemoryCache()

session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    cache=build_cache
)

Key maker
---------

The key maker influences how objects are stored and retrieved in the cache.

- If you do not specify cache_key_maker, StringKeyMaker is used by default.
- You can supply a custom key maker implementing the KeyMaker interface to customize cache keys.

Example
-------

.. code-block:: python

from ftrack_api.cache import StringKeyMaker

session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    cache_key_maker=StringKeyMaker(prefix="obj:")
)

Plugin metadata
---------------

Plugin discovery can receive optional keyword arguments that are passed to plugin register functions.

- plugin_arguments: A mapping of keyword arguments passed to plugin registrations upon discovery.
- If a plugin signature is incompatible with the provided arguments, the discovery mechanism attempts to reduce the arguments to those the plugin accepts and logs a warning.

Example
-------

.. code-block:: python

session = Session(
    server_url="https://server",
    api_key="YOUR_API_KEY",
    api_user="YOUR_USERNAME",
    plugin_arguments={
        "arg1": "value1",
        "arg2": 42
    }
)

Notes
-----

- The top-level cache is a memory cache, ensuring consistent identity for cached objects within a session.
- If you configure cache and key maker, the session uses them to manage metadata about cached entities.
- The environment and default behaviours for schema caching are documented in the Session initializer docstring and are reflected here as part of metadata usage.
