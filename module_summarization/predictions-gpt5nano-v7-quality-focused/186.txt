Imputation of missing values
============================

This section documents the imputation components used to fill in missing values
in tabular data. The primary user-facing estimator is SimpleImputer, a
univariate imputer that fills each feature (column) using a specified
strategy or a constant value. An optional missing-value indicator can be
appended to the transformed data.

Overview
--------

- SimpleImputer replaces missing values with a per-feature statistic or a
  fixed value.
- Per-feature statistics are learned during fit and applied to transform.
- Supports an optional missing-value indicator that is appended to the
  imputed data via add_indicator.
- Maintains feature order and supports common data types; can operate on dense
  numpy arrays or sparse matrices.
- For pandas data with nullable dtypes (e.g., pd.NA), missing_values can be
  set to pd.NA, enabling imputation with numeric arrays containing pandas NA
  entries.

Key classes
-----------

- SimpleImputer: Univariate imputer that fills missing values along each column.
  Strategies: mean, median, most_frequent, or constant.
- _BaseImputer: Internal base class that provides common functionality, including
  support for add_indicator and feature concatenation.

Supported strategies
--------------------

- mean: Replace missing values with the mean of each column. Requires numeric data.
- median: Replace missing values with the median of each column. Requires numeric data.
- most_frequent: Replace missing values with the most frequent value in each column
  (works with numeric or non-numeric data; for ties, the smallest value is chosen).
- constant: Replace missing values with fill_value. Use fill_value to specify the
  constant to imput.

Parameters
----------

SimpleImputer parameters (examples shown; refer to the code for defaults):

- missing_values: The placeholder for missing values. Could be np.nan, None, or pandas.NA.
  For pandas DataFrames with nullable integer dtypes, missing_values can be np.nan or pd.NA.

- strategy: {'mean', 'median', 'most_frequent', 'constant'}; default is 'mean'.

- fill_value: Used only with strategy 'constant'. The value to use for imputation.
  If None, fill_value defaults to 0 for numeric data and "missing_value" for strings
  or object data.

- copy: bool, default True. If False, imputation may modify X in place when possible.

- add_indicator: bool, default False. If True, appends a missing-value indicator
  feature that marks locations of missing values in the original data.

- keep_empty_features: bool, default False. If True, keeps features (columns) that
  contain only missing values, ensuring the output preserves the input shape.

How imputation works
--------------------

- During fit, SimpleImputer computes per-feature statistics (mean, median, or
  most_frequent) or records the constant fill_value.
- During transform, missing values in each column are replaced by the corresponding
  statistic or the fill_value.
- If a column has no non-missing values, the computed statistic may be NaN, and
  the corresponding column after imputation may contain NaN values.
- If add_indicator is enabled, the transformed data is augmented with an
  indicator matrix produced by a MissingIndicator, and the resulting feature
  names (when supported) include the indicator feature names.

Handling of missing_values and dtypes
-------------------------------------

- If X is numeric (float, int, unsigned int) and missing_values is not a real number,
  a ValueError is raised to ensure consistency between X and the missing value
  placeholder.
- If missing_values is a pandas NA, the estimator allows imputing numeric arrays
  that use pd.NA as missing markers.

Fitting and transforming
------------------------

- fit(X, y=None) learns the per-feature statistics and, when add_indicator is True,
  fits a MissingIndicator to X.
- transform(X) imputes missing values using the learned statistics or fill_value.
- fit_transform(X) is a convenience that performs fit followed by transform.

Usage examples
--------------

Mean imputation (numeric data):

```python
from sklearn.impute import SimpleImputer
import numpy as np

X = np.array([[1.0, 2.0], [3.0, np.nan]])
imputer = SimpleImputer(strategy="mean")
X_imputed = imputer.fit_transform(X)
# X_imputed -> array([[1.0, 2.0], [3.0, 2.0]])
```

Median imputation:

```python
X = np.array([[1.0, 4.0], [3.0, np.nan]])
imputer = SimpleImputer(strategy="median")
X_imputed = imputer.fit_transform(X)
```

Most frequent imputation (works with numbers or strings):

```python
X = np.array([[1, "a"], [1, "b"], [np.nan, "a"]], dtype=object)
imputer = SimpleImputer(strategy="most_frequent")
X_imputed = imputer.fit_transform(X)
```

Constant imputation:

```python
X = np.array([[1.0, np.nan], [np.nan, 3.0]])
imputer = SimpleImputer(strategy="constant", fill_value=0)
X_imputed = imputer.fit_transform(X)
```

Indicator when adding missing-value mask:

```python
X = np.array([[1.0, np.nan], [np.nan, 3.0]])
imputer = SimpleImputer(strategy="mean", add_indicator=True)
X_imputed = imputer.fit_transform(X)
# X_imputed contains the imputed data followed by indicator features
```

Notes and tips
--------------

- Use add_indicator in pipelines to preserve information about missingness for downstream models.
- Keep_empty_features=False removes features that are entirely missing; set True to preserve shape.
- For DataFrame inputs, outputs generally preserve column order; when add_indicator is used, indicator
  features are appended to the right.
- When a column has all values missing and the chosen strategy cannot compute a finite statistic,
  the imputed values may remain NaN.
