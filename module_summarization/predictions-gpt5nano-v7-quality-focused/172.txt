SANS to SESANS conversion
=========================

Overview
--------
This document describes the SANS-to-SESANS conversion provided by the sans_to_sesans module. The module converts a SANS prediction I(q) (intensity as a function of momentum transfer q) into a SESANS polarization profile P(z) as a function of spin-echo length z. The conversion uses a one-dimensional transform with a kernel that encodes the SESANS experimental geometry and instrument response.

Key ideas
- Input: I(q) from a SANS model (or a callable that returns I(q) for given q).
- Output: P(z) for a collection of z values (spin-echo lengths).
- Transform: P(z) is computed via a weighted integral of I(q) over q with a SESANS kernel K(q, z):
  P(z) = ∫ I(q) · K(q, z) dq
- Kernel: The kernel represents the spin-echo geometry and instrumental broadening. The module supports a default kernel and allows kernel customization via parameters.
- Vectorization: The implementation supports broadcasting, enabling efficient calculation for multiple q and z values.

API reference
------------
The sans_to_sesans module exposes a user-friendly transform that maps an I(q) curve to a P(z) polarization curve.

Public surface (typical usage):
- Function: sans_to_sesans
  - Purpose: Compute SESANS polarization P(z) from SANS intensity I(q).
  - Parameters:
    - q: 1D array of q-values for which I(q) is known or evaluated.
    - Iq: 1D array of intensities corresponding to q. If a callable is provided instead of an array, it will be evaluated as needed.
    - z: 1D array of spin-echo lengths for which to compute P(z).
    - kernel: string or object specifying the SESANS kernel to use (e.g., 'default'). Optional.
    - kernel_params: dict of additional kernel parameters (e.g., width, shape) for customization. Optional.
    - method: string, optional; selects the internal computation method (e.g., 'auto', 'quad', 'fft'). Optional.
  - Returns:
    - Pz: 1D array of SESANS polarization values corresponding to each z.

Sample signature (illustrative):
  def sans_to_sesans(q, Iq, z, kernel='default', kernel_params=None, method='auto'):
      ...

Example
-------
Compute P(z) from a SANS I(q) using the default kernel.

.. code-block:: python

    import numpy as np
    from sasmodels.sans_to_sesans import sans_to_sesans

    q = np.linspace(0, 0.5, 256)          # SANS q-values
    Iq = some_model(q)                    # I(q) from a SANS model
    z = np.linspace(0, 50, 200)           # Spin-echo lengths
    Pz = sans_to_sesans(q, Iq, z, kernel='default')
    print(Pz.shape)  # (200,)

If you already have a callable for I(q), you can pass it directly:

.. code-block:: python

    def I_of_q(q):
        return some_model(q)

    Pz = sans_to_sesans(q, I_of_q, z, kernel='default')

Implementation notes
--------------------
- Kernel customization: Users can adjust kernel parameters to reflect different instrument configurations or sample environments.
- Performance: The transform is vectorized for performance. Where possible, the implementation uses efficient quadrature or fast transforms.
- Compatibility: The interface is designed to work with NumPy arrays and with I(q) supplied as arrays or callables.

Inputs and data shapes
----------------------
- q: shape (N,), where N is the number of q-points.
- Iq: shape (N,), corresponding intensities I(q).
- z: shape (M,), where M is the number of SESANS spin-echo lengths.
- Result Pz: shape (M,)

Parameter and kernel guidance
-----------------------------
- kernel: Choose 'default' for standard SESANS geometry. Other kernels can model instrument-specific responses or sample broadening.
- kernel_params: Provide any kernel-specific settings such as:
  - width: positive float controlling kernel spread
  - shape: string or numeric descriptor for kernel shape
  - normalization: boolean to apply normalization to the kernel
- method: Select computation strategy:
  - 'auto': Let the implementation pick the best available method
  - 'quad': Use numerical quadrature
  - 'fft': Use a fast transform-based approach when applicable

Validation and testing
----------------------
- Compare P(z) against reference SESANS polarization for well-known test cases.
- Validate shapes: ensure Pz.shape == (len(z),)
- Check energy-conservation-like sanity: non-negative modulation in expected regimes and smooth behavior as z increases.

Extensibility
------------
- Add new kernels by implementing a kernel object or function and wiring it through the kernel parameter.
- Extend I(q) sources by accepting callables or data from any SANS model integration.
