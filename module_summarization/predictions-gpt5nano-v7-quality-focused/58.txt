Optimization
===============
This module provides a general, reusable optimization conduit for model fitting. It focuses on fitting an objective function (often a log-likelihood or its negative) independent of any specific model class, so the functionality can be reused across different modeling scenarios.

Overview
--------
- Provides a general Optimizer class with a flexible _fit method.
- Wraps solvers from scipy.optimize, enabling Newton-Raphson, Nelder-Mead, BFGS, CG, Powell, basin-hopping, and a generic minimize wrapper.
- Allows passing extra arguments to the objective function and its gradient, and supports returning detailed solver information.

Public API
----------
- Optimizer
  - _fit(objective, gradient, start_params, fargs, kwargs, hessian=None, method='newton', maxiter=100, full_output=True, disp=True, callback=None, retall=False)
    - Fitting entry point for an arbitrary objective function.

Notes on _fit
-------------
- objective: The objective to minimize (commonly the negative log-likelihood).
- gradient: Function that computes the gradient of the objective with respect to parameters.
- start_params: Initial guess for the parameters.
- fargs: Extra positional arguments passed to the objective (loglike(x, *fargs)).
- kwargs: Keyword arguments passed to the objective.
- hessian: Optional Hessian (not always required; used by some solvers).
- method: Solver name. Supported: newton, nm, bfgs, powell, cg, ncg, basinhopping, minimize. Uses the corresponding solver from scipy.optimize. If an unsupported method is provided, a ValueError is raised.
- maxiter: Maximum number of iterations (basinhopping ignores this).
- full_output: If True, returns a dictionary with solver outputs (where supported).
- disp: If True, prints convergence messages.
- callback: Callback function called after each iteration with the current parameter vector.
- retall: If True, returns the full list of parameter vectors from each iteration (where supported).

Returns
-------
- xopt: Array of optimized parameters.
- retvals: Solver-specific output dictionary (only when full_output is True; otherwise None).
- optim_settings: Dictionary of the solver parameters that were used.

Solver-specific notes
---------------------
- basinhopping ignores maxiter, retall, and full_output.
- Optional solver arguments (shown here for reference) include:
  - newton: tol
  - nm: xtol, ftol, maxfun
  - bfgs: gtol, norm, epsilon
  - lbfgs: m, pgtol, factr, maxfun, epsilon, approx_grad
  - cg: gtol, norm, epsilon
  - ncg: fhess_p, avextol, epsilon
  - powell: xtol, ftol, maxfun, start_direc
  - basinhopping: niter, niter_success, T, stepsize, interval, minimizer (dict of extra kwargs for scipy.optimize.minimize)

Example
-------
The following demonstrates a simple usage with a quadratic objective and its gradient.

.. code-block:: python

    import numpy as np
    from optimization import Optimizer

    def objective(x, a=1.0, b=0.0):
        # Simple quadratic: (x - [a,b])^2
        return np.sum((x - np.array([a, b]))**2)

    def gradient(x, a=1.0, b=0.0):
        return 2.0 * (x - np.array([a, b]))

    opt = Optimizer()
    start_params = np.array([0.0, 0.0])
    fargs = ()      # no extra positional args
    kwargs = {'a': 1.0, 'b': 0.0}

    xopt, retvals, optim_settings = opt._fit(
        objective=objective,
        gradient=gradient,
        start_params=start_params,
        fargs=fargs,
        kwargs=kwargs,
        method='bfgs',
        maxiter=200,
        full_output=True,
        disp=True
    )

See Also
--------
- scipy.optimize: The underlying optimization routines used by the wrapper.

Caveats
-------
- The internal _check_method helper validates the requested method and raises a ValueError for unknown methods.
- The basinhopping solver behaves differently with respect to several explicit arguments and may ignore some of them as noted above.
