Ploterr
========

Overview
--------

Ploterr produces plots of error-bar data extracted from an HDF5 dataset. The command is organized as a top-level Ploterr command with a Generic interval subcommand that reads a dataset and plots its expected values with confidence intervals.

Subcommand
----------

Generic interval
~~~~~~~~~~~~~~~~

- Description: Plots generic expectation and confidence-interval data from a user-specified HDF5 file and dataset. The path must be of the form FILENAME/PATH[SLICE]. If the dataset is not 1-dimensional, a SLICE must be provided.
- Example of dataset access: kinavg.h5/rate_evolution[:,0,1] accesses the rate evolution for states 0 to 1.

Usage
-----

- Basic command:
  west ploterr generic FILENAME/PATH[SLICE] -o OUTPUT.[ext]

- Example:
  west ploterr generic kinavg.h5/rate_evolution[:,0,1] -o rate_evolution_ci.pdf

- Dataset path syntax:
  FILENAME is the HDF5 file to read, PATH is the dataset path, and SLICE (optional) is a NumPy-style slice in brackets. If no SLICE is provided, the tool uses a full-collection slice (...).

- If the dataset is not 1-dimensional, you must provide a SLICE.

CLI options
-----------

Common options (Ploterr, via CommonPloterrs)
- --xscale
  Choices: linear (default), log, symlog
  Use a scaling for the x-axis.

- --yscale
  Choices: linear (default), log, symlog
  Use a scaling for the y-axis.

- --xrange
  Format: "xmin,xmax"
  Restrict the x-axis to this range. Default determined from input data.

- --yrange
  Format: "ymin,ymax"
  Restrict the y-axis to this range. Default determined from input data.

- --xlabel
  X-axis label. Default varies; if not provided, a default is used.

- --ylabel
  Y-axis label. Default varies; if not provided, a default is used.

- --title
  Plot title. Default varies; if not provided, no title is set.

- --terminal, -t
  Plot output in terminal. Sets the interface to text mode.

- The command uses a progress indicator via the ProgressIndicatorComponent.

Data format and requirements
----------------------------

- The dataset must provide the following fields:
  - iter_start
  - iter_stop
  - expected
  - ci_lbound
  - ci_ubound

- The plotting code computes the x-values as iter_stop - 1.

- The data is read as a 1D array (structured data with the required fields). If the loaded data is not 1D, the command raises an error.

- The HDF5 file path is resolved with h5io.resolve_filepath in read mode.

Output
------

- Output is written to OUTPUT, whose format is inferred from the file extension. Common defaults include PDFs, PNGs, and other image formats supported by the chosen backend.

- By default, the output file name is errbars.pdf unless overridden with -o/--output.

Interface behavior
------------------

- If a graphical display is available (DISPLAY is set), the plotting backend uses Matplotlib to render and save the plot.
- In headless environments (DISPLAY is not set) or when --terminal is used, the interface switches to text mode (no graphical plotting performed by default).

Examples
--------

- Plot a specific rate-evolution series to a PDF:
  west ploterr generic kinavg.h5/rate_evolution[:,0,1] -o rate_evolution_ci.pdf

- Plot with explicit ranges and labels:
  west ploterr generic kinavg.h5/rate_evolution[:,0,1] \
    -o rates_ci.png \
    --xrange 0,1000 --yrange -0.1,0.5 \
    --xlabel "Iteration" --ylabel "Rate" --title "Rate evolution (states 0→1)"

- Run in a headless environment (text interface):
  west ploterr generic kinavg.h5/rate_evolution[:,0,1] -t -o rate_evolution_ci.txt

Implementation notes
--------------------

- The common options are implemented in CommonPloterrs, including x/y scales, ranges, labels, and title. The defaults and validation reflect the code behavior described above.
- The GenericIntervalSubcommand handles input parsing for FILENAME/PATH[SLICE], using a regular expression to split pathname and optional slice. If a slice is provided, it is converted to a NumPy index expression; otherwise, a full Ellipsis slice is used.
- Data loading is performed through h5io.resolve_filepath and dataset slicing, followed by a dimensionality check (must be 1D).

Code references (conceptual)
----------------------------

- Dataset fields accessed during plotting: iter_stop, expected, ci_lbound, ci_ubound
- X-values derived from iter_stop minus 1
- Plotting sequence: x-values, then plots of expected and confidence bounds, followed by axis configuration and saving to output file

Notes
-----

- The documentation reflects the behavior visible in the code: plotting uses Matplotlib when available; otherwise, the interface defaults to text mode.
- For precise behavior of defaults (labels, title), rely on the defaults defined in the code paths that set xlabel, ylabel, and title.
