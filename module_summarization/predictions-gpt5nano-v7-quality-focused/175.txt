Biclustering
==============

Overview
--------
Biclustering jointly finds clusters of rows and columns in a data matrix, revealing submatrices (biclusters) where a subset of samples co-varies with a subset of features. This is useful for domains such as gene expression analysis, text mining, and collaborative filtering where coherent patterns span both dimensions.

Supported algorithms
--------------------
The biclustering module provides two primary algorithms:

- SpectralBiclustering
  - Discovers biclusters by applying spectral decomposition to a bipartite representation of the data.
  - Produces row and column cluster assignments that emphasize coherent blocks within the matrix.

- SpectralCoclustering
  - Performs co-clustering via a similar spectral approach but focuses on jointly partitioning rows and columns into blocks with high within-block similarity.

Key concepts
------------
- A bicluster is a pair (row block, column block) indicating which rows and which columns belong together.
- After fitting, each algorithm exposes membership information that lets you inspect which rows and columns participate in each bicluster.
- The methods work best on non-negative data and are suitable for discovering rectangular patterns in large matrices.

API overview
------------
Common interface
- fit(X, y=None)
  - Fits the model to the data matrix X.
  - X must be a 2D array of shape (n_samples, n_features) or a sparse matrix.
- transform(X=None)
  - Optional transformation to reveal the bicluster structure on new data (if supported by the estimator).
  - May return a representation indicating bicluster membership or scores.

Important attributes (after fit)
- rows_ : array-like of shape (n_row_clusters, n_rows)
  - Binary masks or indicators showing which rows belong to each row cluster.
- cols_ : array-like of shape (n_col_clusters, n_cols)
  - Binary masks or indicators showing which columns belong to each column cluster.
- row_labels_ (optional in some variants)
  - Cluster labels assigned to rows for a particular row clustering interpretation.
- col_labels_ (optional in some variants)
  - Cluster labels assigned to columns for a particular column clustering interpretation.

Parameters (typical)
- n_clusters: int or tuple
  - Specifies the number of row and column clusters.
  - Example: (3, 4) creates 3 row clusters and 4 column clusters.
- random_state: int or None
  - Controls randomness for reproducibility.
- max_iter or n_iter (if present): int
  - Maximum iterations for iterative optimization (varies by implementation).
- normalize: bool (if present)
  - Whether to normalize input data prior to clustering (implementation dependent).

Usage pattern
-------------
- Prepare a non-negative data matrix X of shape (n_samples, n_features).
- Instantiate a biclustering estimator with desired cluster counts.
- Call fit(X) to learn the biclusters.
- Inspect rows_ and cols_ to determine the composition of each bicluster.
- Optionally transform or extract submatrices corresponding to each bicluster for further analysis.

Code example
------------
Example with synthetic data:

.. code-block:: python

    import numpy as np
    from sklearn.cluster import SpectralBiclustering

    # Synthetic matrix with two clear biclusters
    X = np.array([
        [5, 5, 0, 0, 0, 0],
        [4, 4, 0, 0, 0, 0],
        [0, 0, 6, 6, 6, 0],
        [0, 0, 5, 5, 5, 0],
        [0, 0, 0, 0, 0, 7],
        [0, 0, 0, 0, 0, 6],
    ], dtype=float)

    # Fit biclustering model
    model = SpectralBiclustering(n_clusters=(2, 3), random_state=0)
    model.fit(X)

    # Inspect bicluster composition
    # Rows belong to row clusters; columns belong to column clusters
    print("Row clusters (rows_):")
    print(model.rows_)
    print("Column clusters (cols_):")
    print(model.cols_)

Practical notes
-------------
- Choice of n_clusters directly influences the granularity of discovered biclusters; larger values yield more, smaller biclusters.
- Data preprocessing can impact results:
  - Non-negativity often helps, particularly for methods based on co-clustering of additive patterns.
  - Centering or scaling may alter the interpretation of blocks; apply domain-appropriate preprocessing.
- If noisy or highly overlapping patterns exist, consider trying different cluster counts or alternative biclustering methods.

Interoperability and related tools
----------------------------------
- Biclustering complements standard clustering by focusing on coherent submatrices rather than global structure.
- Results can feed into downstream tasks such as visualization, feature selection, or domain-specific pattern analysis.

Notes
-----
- The exact parameter names and available variants may evolve across versions of the library. Refer to the latest API reference for precise signatures and attributes.
- Some variants expose row and column labels or membership masks via attributes such as rows_ and cols_. Use these to map biclusters back to the original matrix indices.
