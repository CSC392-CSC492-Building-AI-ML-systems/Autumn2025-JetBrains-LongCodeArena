ASDF: Search utilities for ASDF trees (AsdfSearchResult)

Overview
- This documentation describes the AsdfSearchResult class, a result object produced by ASDF tree searches. It supports chaining search criteria, formatting the rendered tree, and replacing matched leaf nodes.
- The implementation focuses on key-based, type-based, and value-based filtering, plus an optional custom filter.
- Key concepts:
  - identifers: path-like sequence identifying the location of nodes in the tree
  - node: the current subtree or leaf being considered
  - filters: a list of callables applied to each node during search
  - NotSet: a sentinel used to denote “no constraint” for search/format options

API Reference
- AsdfSearchResult
  - Purpose
    - Represents the result of a search over an ASDF tree. It can be further refined with additional search criteria, formatted for display, or used to mutate leaf values in the matched results.
  - Constructor
    - __init__(identifiers, node, filters=[], parent_node=None, max_rows=DEFAULT_MAX_ROWS, max_cols=DEFAULT_MAX_COLS, show_values=DEFAULT_SHOW_VALUES)
    - Parameters
      - identifiers: list of keys/indices identifying the path to the target node
      - node: the root of the (sub)tree being examined
      - filters: a list of filter callables applied to candidate nodes
      - parent_node: the parent container of the leaf nodes
      - max_rows: maximum number of rows to display (default uses DEFAULT_MAX_ROWS)
      - max_cols: maximum line width (default uses DEFAULT_MAX_COLS)
      - show_values: whether to render primitive values (default uses DEFAULT_SHOW_VALUES)
  - format(max_rows=NotSet, max_cols=NotSet, show_values=NotSet)
    - Purpose
      - Returns a new AsdfSearchResult with updated formatting parameters.
    - Behavior
      - If an argument is NotSet, it preserves the existing value.
      - max_rows: int, tuple, None, or NotSet
        - Max lines to print; None means no limit.
        - Tuple constrains lines per node at a given depth.
      - max_cols: int, None, or NotSet
        - Max line length; None means no limit.
      - show_values: bool or NotSet
        - False disables display of primitive values.
    - Returns
      - A new AsdfSearchResult with the updated formatting parameters.
    - Example
      - result2 = result.format(max_rows=40, max_cols=80, show_values=False)
  - _maybe_compile_pattern(query)
    - Internal helper
    - If query is a string, compiles it to a regular expression; otherwise returns the query unchanged.
  - _safe_equals(a, b)
    - Internal helper
    - Safely compares two values, returning False if an exception occurs or if the comparison yields a non-bool.
  - _get_fully_qualified_type(value)
    - Internal helper
    - Returns a fully qualified type name for a value:
      - If the type belongs to builtins, returns just the type name (e.g., int, str).
      - Otherwise returns module.name (e.g., mymodule.MyType).
  - search(key=NotSet, type=NotSet, value=NotSet, filter=None)
    - Purpose
      - Adds a new filter to narrow the results. Returns a new AsdfSearchResult with the composed filter.
    - Parameters
      - key: NotSet, str, or any object
        - If NotSet, no constraint on the node’s key/index.
        - If str, treat as a regular expression against the node’s key/index.
        - Otherwise, requires equality with the node’s key/index.
      - type: NotSet, str, typing.Pattern, or builtins.type
        - If NotSet, no constraint on the node’s type.
        - If str, treat as a regular expression against the node’s fully qualified type name.
        - If typing.Pattern, test if the node’s fully qualified type matches the pattern.
        - If builtins.type, require isinstance(node, type).
      - value: NotSet, str, or any object
        - If NotSet, no constraint on the node’s value.
        - If str, treat as a regular expression against the node’s string representation.
          - If the node is a container, pattern matching is skipped to avoid including child values.
        - Otherwise, require equality with the node’s value.
      - filter: callable(node, identifier) -> bool
        - Optional; can apply arbitrary criteria. The callable receives the node and its key/index.
    - Returns
      - A new AsdfSearchResult with the added filter.
    - Behavior notes
      - type parameter can be a string (regex) or a builtins.type or a Pattern.
      - The internal filter checks key, type, value, and the optional custom filter in sequence.
  - replace(value)
    - Purpose
      - Replaces all leaf nodes in the current search results with the given value.
    - Process
      - Walks the tree breadth-first, collecting leaf nodes whose path passes all filters.
      - Assigns value to the corresponding leaf positions in their parents.
    - Example
      - result.replace(None)  # clears matched leaf values
  - node (property)
    - Purpose
      - Retrieves the leaf node when the search yields a single result.
    - Returns
      - The leaf node for a tree with one search result. (If multiple results exist, behavior depends on the surrounding code.)

Internal helpers (for reference)
- _maybe_compile_pattern(query)
  - Converts a string to a compiled regex; leaves non-strings untouched.
- _safe_equals(a, b)
  - Performs a robust equality check with exception safety.
- _get_fully_qualified_type(value)
  - Produces a string like builtins.str or package.module.Type for type matching.

Default formatting and behavior
- The module uses defaults from:
  - DEFAULT_MAX_ROWS
  - DEFAULT_MAX_COLS
  - DEFAULT_SHOW_VALUES
- These defaults guide initial display size and whether primitive values render in the tree.

Semantics and usage notes
- The search method supports rich filtering:
  - Key/index matching via strings or patterns
  - Type matching via fully qualified names or type objects
  - Value matching via strings (regex) or direct equality
  - Optional custom filter for arbitrary logic
- Pattern matching applies to the node’s key/index and type as described:
  - If a pattern is provided for key, a node is kept only if the identifier matches the pattern.
  - If a pattern is provided for type, a node is kept only if its fully qualified type matches.
  - For value patterns, container nodes are excluded to avoid confusing matches with internal representations.
- The format method allows precise control of how the search results render, without altering the underlying results.

Examples
- Basic usage (pseudocode; assume a root_node from an ASDF tree):
  - Start from a root search result
    result = AsdfSearchResult(['root'], root_node)

  - Narrow by a key pattern (regex)
    by_key = result.search(key=r'^vol')

  - Narrow by type (fully qualified name pattern)
    by_type = by_key.search(type=r'^builtins\.str$')

  - Narrow by value (regex on string form)
    by_value = by_type.search(value=r'^[0-9]+$')

  - Apply a custom filter
    by_custom = by_value.search(filter=lambda node, ident: hasattr(node, 'shape'))

  - Replace matched leaf values
    by_custom.replace(None)

  - Change display options
    display = by_custom.format(max_rows=40, max_cols=80, show_values=False)

  - Access single leaf node (if there is exactly one result)
    leaf = display.node

Notes
- NotSet is used to indicate that a parameter should retain its existing value during format or search calls.
- The internal implementation emphasizes robustness in value comparisons and pattern matching, with careful handling of patterns versus direct equality.
