Nonparametric Methods
=====================


Overview
--------
This section documents the nonparametric kernel density estimation (KDE) facilities provided for univariate data. The primary interface is KDEUnivariate, which builds a density estimate on a regular grid using a chosen kernel and bandwidth. The implementation favors speed via an FFT-based approach when possible.

Key concepts
- Density estimate: p̂(x) on a grid of evaluation points.
- Kernel: shape function used to smooth observations. A set of discrete kernels is available.
- Bandwidth (bw): smoothing parameter. Can be selected automatically or supplied by the user.
- FFT mode: accelerates computation for Gaussian kernels; other kernels are not supported in FFT mode.

KDE Univariate API
--------------------

KDEUnivariate(endog)
- endog
  - Array-like, one-dimensional data for which the density is estimated.

fit(...)
- kernel: str
  - Kernel to use. Options and codes:
    - "gau" → Gaussian
    - "epa" → Epanechnikov
    - "uni" → Uniform
    - "tri" → Triangular
    - "biw" → Biweight
    - "triw" → Triweight
    - "cos" → Cosine
    - "cos2" → Cosine2
- bw: str, float, or callable
  - Bandwidth selection or value. Options:
    - "scott" – 1.059 × A × nobs^(-1/5), with A = min(std(x), IQR/1.34)
    - "silverman" – 0.9 × A × nobs^(-1/5), with A as above
    - "normal_reference" – C × A × nobs^(-1/5), where C depends on the kernel
    - float – uses the given value directly
    - callable – must accept exactly two parameters (fn(x, kern)) and return a float. The callable is invoked with the clipped input x and the kernel instance.
- fft: bool
  - If True, uses the FFT-based implementation. Only the Gaussian kernel is supported with FFT. Weights are not implemented with FFT.
- gridsize: int or None
  - Number of grid points for evaluation. If None, uses max(len(endog), 50).
- adjust: float
  - Adjustment factor applied to the bandwidth: bw × adjust.
- cut: float
  - Defines the grid extent beyond the observed data. The grid extends to +/- cut × bw × (min or max of x) to ensure the kernel goes to zero at the boundaries.
- clip: tuple (lower, upper)
  - Clipping bounds for input data before fitting.

Returns
- KDEUnivariate
  - The fitted instance with attributes populated:
    - density: estimated density values on the support grid
    - support: grid of evaluation points
    - bw: final bandwidth used
    - kernel: the instantiated kernel (with h=bw)
    - weights (optional): if provided, normalized and used in the kernel
  - A cache is maintained to speed repeated operations; re-fitting resets the cache.

Notes on bandwidth and kernels
- When bw is a string, the method stores the string in bw_method (not the numeric bandwidth).
- When bw is a float or callable, the numeric bandwidth is used to initialize the kernel with h=bw.
- The kernel instance is created as kernel_switch[kernel](h=bw). The available codes map to concrete kernel implementations.
- If weights are provided, they are attached to the kernel and normalized by their sum.

Output and accessibility
- density: array of density estimates on the evaluation grid
- support: array of grid points corresponding to density
- bw: bandwidth used for the estimation
- kernel: kernel instance used for density evaluation (accessible properties include domain and density function)

CDF and related quantities
- cdf: property that returns the cumulative distribution function evaluated at the support points. It requires fit to have been called. It uses the kernel’s density function for computation and respects the kernel’s domain if defined; otherwise it assumes an infinite domain.

Kernel options at a glance
- Gau (Gaussian)
- Epa (Epanechnikov)
- Uni (Uniform)
- Tri (Triangular)
- Biw (Biweight)
- Triw (Triweight)
- Cos (Cosine)
- Cos2 (Cosine2)

Usage example
-------------
The following example demonstrates fitting a univariate KDE with a Gaussian kernel using the normal-reference bandwidth, then inspecting the results.

Code
```python
import numpy as np
from statsmodels.nonparametric.kde import KDEUnivariate

np.random.seed(123)
x = np.random.normal(size=300)

# Create and fit KDE
dens = KDEUnivariate(x)
dens.fit(kernel="gau", bw="normal_reference", fft=True)

# Access results
print("Density shape:", dens.density.shape)
print("Support shape:", dens.support.shape)
print("Bandwidth:", dens.bw)

# Optional: access CDF (computed on demand)
cdf_values = dens.cdf  # aligned with dens.support
```

Notes
- FFT mode is available only for the Gaussian kernel and when weights are not used.
- The implementation favors speed for univariate data; KDEMultivariate handles mixed data types and higher dimensions.

Related functionality
---------------------
- KDEMultivariate: multivariate kernel density estimator (not covered here).
- kdensity and kdensityfft: internal routines used by KDEUnivariate for non-FFT and FFT implementations, respectively.
- See Also: KDEMultivariate, kdensity, kdensityfft.

Practical guidance
- Use FFT mode for large univariate datasets with the Gaussian kernel to gain speed.
- For other kernels or when not using FFT, ensure gridsize is appropriate for the data scale to capture distribution features.
- The bandwidth choice significantly affects the smoothness of the estimate; leverage "normal_reference" or "scott"/"silverman" for common defaults, or supply a custom value or callable for advanced control.
