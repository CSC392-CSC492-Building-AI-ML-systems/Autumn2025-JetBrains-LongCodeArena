Pairwise metrics, Affinities and Kernels
=========================================

This section documents the utilities used to compute pairwise metrics between samples. It covers pairwise distances, affinities (kernel-based similarities), and kernel-related helpers. The focus is on input validation, handling of dense and sparse matrices, and the behavior of core utilities that prepare data for pairwise computations.

Overview
--------

- Pairwise computations compare samples (rows) of one or two matrices to produce distances or similarities.
- The utilities support dense numpy arrays and sparse matrices, with configurable sparse formats.
- Precomputed distances are supported: a user may pass a distance matrix as input.
- Data type handling ensures numerical stability by selecting appropriate floating-point dtype.
- Validation ensures compatibility of input shapes and detects invalid values.

Key Utilities
-------------

The following functions are central to preparing inputs for pairwise computations.

- _return_float_dtype(X, Y)
  - Purpose: Determine the appropriate floating-point dtype for X and Y.
  - Behavior:
    - If both X and Y are float32, returns dtype float32.
    - Otherwise returns a general float dtype (float).
  - Notes:
    - Converts non-array inputs to numpy arrays as needed.
  - Typical usage: called internally when dtype is not explicitly provided.

- check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)
  - Purpose: Validate and prepare X and Y for pairwise computations.
  - Behavior:
    - Ensures X and Y are array-like (dense) or sparse, converting as needed.
    - If Y is None or Y is X, both are validated as the same array (pointer semantics for Y).
    - Determines the dtype using _return_float_dtype unless a dtype is supplied.
    - If precomputed is True:
      - Checks that X.shape[1] == Y.shape[0].
    - If precomputed is False:
      - Checks that X.shape[1] == Y.shape[1].
    - Returns:
      - safe_X: array-like, shape (n_samples_X, n_features)
      - safe_Y: array-like, shape (n_samples_Y, n_features)
  - Parameters:
    - X: array-like or sparse matrix, shape (n_samples_X, n_features)
    - Y: array-like or sparse matrix, shape (n_samples_Y, n_features) or None
    - precomputed: True if X is treated as a distance matrix to Y
    - dtype: desired data type for X and Y; if None, inferred
    - accept_sparse: allowed sparse formats (or True/False)
    - force_all_finite: True, False, or 'allow-nan' controlling nan/inf handling
    - copy: if True, forces a copy
  - Typical use:
    - X, Y = check_pairwise_arrays(X, Y, precomputed=False)

- check_paired_arrays(X, Y)
  - Purpose: Validate inputs for paired distance metrics (paired comparisons between two sets).
  - Behavior: Ensures X and Y are properly shaped arrays and ready for pairwise computations.
  - Notes: Used by pairwise metrics that operate on paired inputs.

Input Formats
-------------

- Dense: numpy arrays of shape (n_samples, n_features).
- Sparse: sparse matrices (e.g., CSR/CSC). Use accept_sparse to control allowed formats.
- Precomputed distances: when computing with a precomputed distance matrix, set precomputed=True so that the shapes X.shape[1] and Y.shape[0] are compatible.

Precomputed Distances
---------------------

- When precomputed=True, X represents distance (or similarity) values between queries and indexed samples.
- The required condition is:
  - X.shape[1] == Y.shape[0]
- This mode is useful when distances are computed externally and supplied to the metric routines.

Data Types and Finite Values
----------------------------

- _return_float_dtype determines the resulting dtype:
  - If both inputs are float32, dtype is float32.
  - Otherwise, dtype is float (typically float64).
- force_all_finite controls validation of finite values:
  - True: all values must be finite.
  - False: allows inf, NaN, and missing values (pd.NA becomes NaN).
  - 'allow-nan': allows NaN and pd.NA but not infinite values.
- check_pairwise_arrays propagates these checks to ensure safe computations.

Examples
--------

1) Basic pairwise distance between two sets

```python
import numpy as np
from sklearn.metrics.pairwise import pairwise_distances

X = np.array([[0.0, 0.0], [1.0, 1.0], [2.0, 2.0]])
Y = np.array([[0.0, 1.0], [1.0, 0.0]])
D = pairwise_distances(X, Y, metric='euclidean')
```

2) Validating inputs for a pairwise computation (internal utility usage)

```python
from sklearn.metrics.pairwise import check_pairwise_arrays

X = [[0, 1], [1, 0]]
Y = X
safe_X, safe_Y = check_pairwise_arrays(X, Y, precomputed=False, dtype=None, copy=False)
```

3) Using precomputed distances (internal usage)

```python
from sklearn.metrics.pairwise import check_pairwise_arrays

# X is treated as a distance matrix to Y
X = [[0.0, 1.2], [1.2, 0.0]]
Y = X
safe_X, safe_Y = check_pairwise_arrays(X, Y, precomputed=True, dtype=None, copy=False)
```

Kernels, Affinities and Related Concepts
-----------------------------------------

- Affinities (kernel-based similarities) evaluate how similar two samples are, often via kernel functions such as linear, polynomial, RBF, or chi-squared variants.
- Kernels enable methods that rely on inner products in a high-dimensional feature space without explicit feature construction.
- The codebase provides optimized fast-paths for certain kernels:
  - _chi2_kernel_fast: fast implementation of the chi-squared kernel (useful for histogram-based features).
  - _sparse_manhattan: efficient implementation of Manhattan-like distances for sparse inputs.
- Kernel-based metrics may be used in clustering, nearest-neighbor retrieval, and kernelized learning algorithms.

Notes for Developers
---------------------

- The validation utilities are shared by many pairwise metric implementations to ensure consistent input handling, dtype decisions, and precomputed support.
- When extending metrics, rely on:
  - _return_float_dtype to choose an appropriate dtype.
  - check_pairwise_arrays to validate shapes, dtypes, and finiteness.
  - Accept_sparse to control sparse input formats and avoidance of unnecessary conversions.

This coverage ensures robust, consistent behavior across pairwise distance, affinity, and kernel computations.
