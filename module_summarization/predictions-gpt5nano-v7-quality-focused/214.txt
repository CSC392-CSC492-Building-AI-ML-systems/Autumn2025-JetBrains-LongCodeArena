Optimization: Parameter Estimation Algorithms
==============================================

Overview
--------
This module provides a general-purpose fitting helper that estimates model parameters
by optimizing an objective function. It supports three broad classes of algorithms
for parameter estimation, plus a generic wrapper that delegates to SciPy's minimize:

- Local gradient-based methods
  - Newton-Raphson (newton)
  - Broyden-Fletcher-Goldfarb-Shanno (bfgs)
  - Conjugate gradient (cg)
  - Newton-conjugate gradient (ncg)
- Derivative-free local methods
  - Nelder-Mead (nm)
  - Powell’s method (powell)
- Global optimization
  - Basin-hopping (basinhopping)
- Generic wrapper
  - minimize (wrapper around scipy.optimize.minimize)

Supported methods
-----------------
- newton: Newton-Raphson update using gradient and Hessian when available
- nm: Nelder-Mead, a derivative-free simplex search
- bfgs: Quasi-Newton method that builds an approximate inverse Hessian
- powell: Powell’s method, a derivative-free line-search approach
- cg: Conjugate gradient, uses gradient information
- ncg: Newton-conjugate gradient, combines Newton steps with CG
- basinhopping: Global basin-hopping search (ignores some local-fit options)
- minimize: Generic wrapper for scipy.optimize.minimize (any supported SciPy solver)

How the fit function is used
----------------------------
The core fitting function is _fit, which accepts:

- start_params: Initial guess for the parameters
- objective: Function to be minimized (or maximized if you pass the negative objective)
- gradient: Function returning the gradient of the objective
- fargs: Extra positional arguments passed to the objective
- kwargs: Additional keyword arguments passed to the objective
- method: One of the methods listed above
- maxiter: Maximum number of iterations
- full_output: If True, returns a detailed retvals dictionary from the solver
- disp: If True, prints convergence messages
- callback: Function called after each iteration with the current parameters
- retall: If True, returns the history of solutions at each iteration

Return values
-------------
- xopt: The optimized parameter vector
- retvals: A dictionary with solver-specific information (only if full_output is True)
- optim_settings: The dictionary of explicit arguments passed to the solver

Notes and solver specifics
--------------------------
- The basinhopping solver ignores maxiter, retall, and full_output explicitly.
- Each solver exposes its own set of optional arguments. See the solver’s documentation
  (via scipy.optimize) for details on tolerances, maximum evaluations, gradient handling, etc.

------------------
1) Gradient-based local optimization (BFGS)

- Objective: optimize f(x) with available gradient
- Typical usage: provide objective and gradient, choose method 'bfgs'

def objective(x, *args, **kwargs):
    # return scalar objective
    pass

def gradient(x, *args, **kwargs):
    # return gradient vector
    pass

start_params = np.zeros(3)

xopt, retvals, optim_settings = opt._fit(
    objective, gradient, start_params,
    fargs=(), kwargs={},
    method='bfgs', maxiter=100, full_output=True, disp=True

2) Derivative-free local optimization (Nelder-Mead)

s = "If gradient is unavailable or unreliable"
start_params = np.array([1.0, 1.0, 1.0])

xopt, retvals, optim_settings = opt._fit(
    objective, gradient, start_params,
    fargs=(), kwargs={},
    method='nm', maxiter=200, full_output=True, disp=True
)

3) Global optimization (basinhopping)

s = "When seeking a global minimum among multiple local minima"
start_params = np.array([0.0, 0.0, 0.0])

xopt, retvals, optim_settings = opt._fit(
    objective, gradient, start_params,
    fargs=(), kwargs={},
    method='basinhopping', maxiter=50, full_output=True, disp=True,
    callback=None
)

4) Generic wrapper (minimize)

xopt, retvals, optim_settings = opt._fit(
    objective, gradient, start_params,
    fargs=(), kwargs={},
    method='minimize', maxiter=200, full_output=True, disp=True
)

What to expect from retvals
---------------------------
- retvals contains solver-specific information returned by the underlying SciPy solver.
- The exact contents depend on the solver used (the solver determines what information is exposed).

Choosing a method
---------------
- Use gradient-based methods (newton, bfgs, cg, ncg) when you have a reliable gradient (and possibly Hessian) and the problem is smooth.
- Use derivative-free methods (nm, powell) when gradients are unavailable or expensive to compute, or the objective is noisy or non-differentiable.
- Use basinhopping when you suspect multiple local minima and you seek a global minimum, possibly in combination with a local optimizer.
- Use minimize as a flexible wrapper to leverage any SciPy solver available for your problem.

References to the code
----------------------
- Method list and descriptions are defined in the _fit method’s method parameter and its docstring.
- Basinhopping behavior and solver-specific argument mappings are documented in the Notes and Optional arguments sections of _fit.
- Return values follow from the solver’s outputs when full_output is True.
