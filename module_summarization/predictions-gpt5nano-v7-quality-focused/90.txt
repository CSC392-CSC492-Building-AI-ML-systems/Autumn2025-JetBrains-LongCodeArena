Auto Augment
============

Overview
--------

This module exposes high-level helpers to apply NVIDIA DALI’s AutoAugment policies to batches of images or videos. It provides a single entry point, auto_augment, which selects one of several predefined policies, and a convenience wrapper auto_augment_image_net for the common ImageNet policy.

API
---

auto_augment
------------

Applies one of the predefined AutoAugment policies to the input batch.

Signature
- data: DataNode
- policy_name: str = 'image_net'
- shape: DataNode or Tuple[int, int] = None
- fill_value: int = 128
- interp_type: types.DALIInterpType = None
- max_translate_abs: int or (int, int) = None
- max_translate_rel: float or (float, float) = None
- seed: int = None

Returns
- DataNode: A batch of transformed samples.

Description
- data: A batch of samples to process. Supports images in HWC layout and videos in FHWC layout with dtype uint8.
- policy_name: Name of the predefined policy. Accepted values are:
  - image_net
  - reduced_image_net
  - svhn
  - reduced_cifar10
- shape: Optional. If provided, specifies the height and width of the image/video frames as a DataNode or a (H, W) tuple. When shape is provided, translation magnitudes scale with the shape according to max_translate_rel.
- fill_value: Padding value used by warp_affine-based augmentations (translation, shear, rotate). If None, padding uses the border value (clamped).
- interp_type: Interpolation type used by warp_affine operations. Typical values are types.INTERP_LINEAR (default) and types.INTERP_NN.
- max_translate_abs: Optional maximal translation magnitude, in pixels. Used only when shape is not provided. Can be an int or a (height, width) tuple. Default is 250.
- max_translate_rel: Optional maximal translation as a fraction of the shape. Used only when shape is provided. Can be a float or a (height, width) tuple. Default is 1 (entirely translates beyond the canvas).
- seed: Optional seed for random sampling of augmentations and magnitudes.

Policy behavior and validation
- The function maintains a mapping from policy_name to policy builders:
  - image_net -> get_image_net_policy
  - reduced_image_net -> get_reduced_image_net_policy
  - svhn -> get_svhn_policy
  - reduced_cifar10 -> get_reduced_cifar10_policy
- Policies without translation (currently reduced_image_net) do not rely on image shape. If shape (or related translation arguments) are provided with such a policy, an exception is raised.
- If policy_name is invalid, the function raises a descriptive error listing allowed values.

Usage examples
- Basic usage with default ImageNet policy:
  output = auto_augment(data)

- Specify a target shape and enable translation-based augmentations:
  output = auto_augment(
      data,
      policy_name='image_net',
      shape=(224, 224),
      max_translate_rel=0.5,
      seed=42
  )

- Use a policy that does not depend on the image shape:
  output = auto_augment(
      data,
      policy_name='reduced_image_net',
      seed=123
  )

- Parallel convenience for the ImageNet policy:
  output = auto_augment_image_net(data, shape=(256, 256), seed=7)

auto_augment_image_net
----------------------

Convenience wrapper that applies the ImageNet AutoAugment policy.

Signature
- data: DataNode
- shape: DataNode or Tuple[int, int] = None
- fill_value: int = 128
- interp_type: types.DALIInterpType = None
- max_translate_abs: int or (int, int) = None
- max_translate_rel: float or (float, float) = None
- seed: int = None

Returns
- DataNode: A batch of transformed samples.

Description
- Equivalent to calling auto_augment with policy_name='image_net'.
- Accepts the same parameters as auto_augment, except policy_name is fixed to 'image_net'.

Notes
------

- This documentation reflects the behavior implemented in the module’s public API. For internal policy construction and translation handling, the actual policy builders and utilities live in the NVIDIA DALI auto_aug package.
