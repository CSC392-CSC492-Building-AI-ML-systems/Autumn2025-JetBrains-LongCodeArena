Generalized Additive Models (GAM) — Spline Smoothers (BSplines)
===============================================================

This section demonstrates how to generate and use B-spline bases for GAM smoothing with the functions in the smoothing module. The key public entry points are make_bsplines_basis and get_knots_bsplines.

What you will learn
- How to generate a B-spline basis for a 1D predictor.
- How to fit a simple linear model using the generated spline basis.
- How to obtain knot configurations for B-splines.

Quick example
- Goal: Build a B-spline basis for a 1D predictor x and fit a smooth function to y using a least-squares fit.

Code
```python
import numpy as np
from statsmodels.gam.smooth_basis import make_bsplines_basis
# Optional: for knot generation
from statsmodels.gam.smooth_basis import get_knots_bsplines

np.random.seed(0)

# Generate sample data
n = 200
x = np.linspace(0, 1, n)
y = np.sin(2 * np.pi * x) + 0.2 * np.random.randn(n)

# Build a B-spline basis for x
# df roughly corresponds to the number of basis functions; degree is spline degree
basis, der_basis, der2_basis = make_bsplines_basis(x, df=8, degree=3)

# Fit a simple linear model using the spline basis
# y ~ basis columns (no intercept term is added here; you can add one if desired)
beta, *_ = np.linalg.lstsq(basis, y, rcond=None)
y_fit = basis @ beta

# Evaluate fit quality
rmse = np.sqrt(np.mean((y - y_fit) ** 2))
print("RMSE:", rmse)
print("Basis shape:", basis.shape)

# Optional: inspect knots used for the basis
knots = get_knots_bsplines(x, df=8, degree=3, spacing='quantile')
print("Knots length:", len(knots))
```

Notes
- make_bsplines_basis(x, df, degree) returns:
  - basis: the design matrix with one column per basis function
  - der_basis: first-derivative basis (if needed)
  - der2_basis: second-derivative basis (if needed)
- The df parameter controls the number of basis functions (roughly, subject to the degree and knot layout).
- get_knots_bsplines(x, df, degree, spacing) provides knot locations aligned with the chosen spacing (e.g., 'quantile' or 'equal'). Use these knots if you want to customize the basis construction or to mimic P-Splines behavior.

Implementation notes (high level)
- The basis generation relies on internal routines that compute knot placements, evaluate B-spline basis functions, and optionally their derivatives via SciPy's splev.
- For practical GAM usage, these bases are combined with a linear or generalized linear model, or integrated into GLMGam smooth terms in the larger GAM framework.
