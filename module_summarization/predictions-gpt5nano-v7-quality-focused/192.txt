# Manifold Learning: Non-linear Dimensionality Reduction

Manifold learning seeks low-dimensional representations of high-dimensional data by uncovering and exploiting non-linear, intrinsic structures (manifolds) embedded in the data. Among non-linear DR techniques, multidimensional scaling (MDS) preserves pairwise relationships between samples either as distances (metric MDS) or as monotone ranks (non-metric MDS). The approach below is based on the SMACOF algorithm, a robust solver for both metric and non-metric MDS.

## SMACOF-based Multidimensional Scaling (MDS)

SMACOF (Scaling by MAjorizing a COmplicated Function) optimizes a stress objective to embed n samples in a low-dimensional space. It can operate in two modes:

- Metric MDS: the input dissimilarities are treated as distances to be preserved.
- Non-metric MDS: only the rank order of dissimilarities is preserved; a monotone regression (Isotonic Regression) maps dissimilarities to disparities during optimization.

Key components of the implementation

- Dissimilarities matrix
  - Input must be a symmetric n-by-n matrix of pairwise dissimilarities.
  - For non-metric MDS, zeros are treated as missing values in the dissimilarities and are ignored during the monotone regression step.

- Initialization
  - If an initial embedding is not provided, a random embedding is generated.

- Iterative update (Guttman transform)
  - Compute current Euclidean distances between embedded points.
  - If metric, use the input dissimilarities as disparities.
  - If non-metric, compute disparities by monotone regression of the squared dissimilarities against the observed distances.
  - Compute stress: a measure of how well the embedding preserves dissimilarities.
  - Update the embedding using the Guttman transform.
  - Check convergence via a relative change in stress. Stop when changes drop below a tolerance (eps).

- Normalized stress (optional)
  - In non-metric mode, you can request a normalized stress (Stress-1) for comparability across problems.

- Internal utilities
  - Checks for symmetry and handles random state for reproducibility.
  - Uses IsotonicRegression for non-metric mapping.
  - Employs Euclidean distance calculations for current embedding.

Public behavior and return values

- The internal function _smacof_single returns:
  - X: the embedding coordinates of shape (n_samples, n_components).
  - stress: the final stress value (measure of misfit between dissimilarities and embedding distances).
  - n_iter: the number of iterations performed.

- Important parameters
  - dissimilarities: symmetric array-like of shape (n_samples, n_samples).
  - metric: boolean, True for metric MDS, False for non-metric MDS.
  - n_components: target embedding dimensionality (default 2).
  - init: optional starting embedding of shape (n_samples, n_components).
  - max_iter: maximum iterations per run (default 300).
  - verbose: verbosity level (0, 1, 2, ...).
  - eps: convergence tolerance (relative change in stress).
  - random_state: seed or RandomState for reproducible initialization.
  - normalized_stress: whether to return Stress-1 (only for non-metric).

- Notes
  - Non-metric MDS requires metric=False and uses monotone regression to derive disparities from the rank-ordered dissimilarities.
  - Convergence depends on initialization and the intrinsic difficulty of the embedding task.

## When to use SMACOF MDS

- You have a symmetric measure of dissimilarity between samples and want a 2D or 3D embedding that preserves those dissimilarities as faithfully as possible.
- You aim to compare embeddings across datasets or problems using a normalized stress measure (in non-metric mode).
- You want a robust non-linear embedding method that does not assume a global linear structure.

## Practical usage

You can use the public MDS interface to obtain a low-dimensional embedding from either raw data or a precomputed dissimilarity matrix. The public API (as provided by the full implementation) typically supports:

- Metric MDS
  - Input: data matrix X (samples × features) or a precomputed dissimilarity matrix D (with dissimilarity='precomputed').
  - Embedding: fit_transform(X) or fit_transform(D) depending on the input type.
  - Example:
    ```python
    from sklearn.manifold import MDS
    from sklearn.metrics import pairwise_distances
    import numpy as np

    X = np.random.randn(100, 5)
    mds = MDS(n_components=2, metric=True, random_state=0)
    Y = mds.fit_transform(X)  # embedding from raw data
    ```

- Non-metric MDS (precomputed dissimilarities)
  - Input: a symmetric dissimilarity matrix D
  - Embedding: fit_transform(D) with dissimilarity='precomputed'
  - Example:
    ```python
    import numpy as np
    from sklearn.manifold import MDS

    D = np.loadtxt('dissimilarities.txt')  # symmetric n×n matrix
    mds = MDS(n_components=2, metric=False, dissimilarity='precomputed', random_state=0)
    Y = mds.fit_transform(D)  # embedding based on rank preservation
    ```

Note: The exact public API may differ slightly across versions. The internal SMACOF implementation underlying both modes follows the workflow described above.

## Tips and best practices

- Precompute dissimilarities only if you have a meaningful non-distance measure or you want to preserve a specific notion of similarity. Otherwise, feed raw data to MDS and let the algorithm compute distances.
- For non-metric MDS, the monotone regression step can handle non-distance-like dissimilarities, but results depend on the rank order of the input.
- Larger datasets incur higher memory usage due to the n-by-n dissimilarity matrix. Consider using a subset or sparse representations if applicable.
- Run multiple restarts (where supported) to mitigate local minima and select the embedding with the lowest final stress.
