Configuration Options
=====================


Overview
--------
This module exposes the ConfigurationsOptions class, which controls which parts of a PyAEDT project are exported to or imported from JSON files. It includes toggles for variables, setups, optimizations, parametrics, boundaries, datasets, materials, object properties, and more. It also provides a tolerance setting used for object mapping during imports/exports.

Public API
----------
- ConfigurationsOptions
  - A settings container that exposes boolean toggles for export/import categories and a mapping tolerance value.

Internal helper
----------------
- _find_datasets(d, out_list)
  - Recursively traverses a nested dictionary/list structure to collect dataset references. It specifically looks for free_form_value entries containing "pwl" and extracts the corresponding dataset tokens. This function is internal and not part of the public API.

Properties
----------
The following properties are available on ConfigurationsOptions. Each has a getter and setter and a default value as initialized in __init__.

- object_mapping_tolerance
  - Description: Get/Set the tolerance value used in object mapping (e.g., for boundaries).
  - Default: 1e-9

- export_variables
  - Description: Determines if variables are exported to JSON. Default is True.
  - Default: True

- export_setups
  - Description: Determines if setups are exported to JSON. Default is True.
  - Default: True

- export_optimizations
  - Description: Determines if optimizations are exported to JSON. Default is True.
  - Default: True

- export_parametrics
  - Description: Determines if parametrics are exported to JSON. Default is True.
  - Default: True

- export_boundaries
  - Description: Determines if boundaries are exported to JSON. Default is True.
  - Default: True

- export_mesh_operations
  - Description: Determines if mesh operations are exported to JSON. Default is True.
  - Default: True

- export_coordinate_systems
  - Description: Determines if coordinate systems are exported to JSON. Default is True.
  - Default: True

- export_materials
  - Description: Determines if materials are exported to JSON. Default is True.
  - Default: True

- export_object_properties
  - Description: Determines if object properties are exported to JSON. Default is True.
  - Default: True

- export_datasets
  - Description: Determines if datasets are exported to JSON. Default is True.
  - Default: True

- import_datasets
  - Description: Determines if datasets are imported from JSON. Default is True.
  - Default: True

- import_variables
  - Description: Determines if variables are imported from JSON. Default is True.
  - Default: True

- import_setups
  - Description: Determines if setups are imported from JSON. Default is True.
  - Default: True

- import_optimizations
  - Description: Determines if optimizations are imported from JSON. Default is True.
  - Default: True

- import_parametrics
  - Description: Determines if parametrics are imported from JSON. Default is True.
  - Default: True

- import_boundaries
  - Description: Determines if boundaries are imported from JSON. Default is True.
  - Default: True

- import_mesh_operations
  - Description: Determines if mesh operations are imported from JSON. Default is True.
  - Default: True

- import_coordinate_systems
  - Description: Determines if coordinate systems are imported from JSON. Default is True.
  - Default: True

- import_materials
  - Description: Determines if materials are imported from JSON. Default is True.
  - Default: True

- import_object_properties
  - Description: Determines if object properties are imported from JSON. Default is True.
  - Default: True

- skip_import_if_exists
  - Description: If True, skip importing items that already exist. Default is False.
  - Default: False

Usage examples
--------------
Access and modify the options through the public API of a supported APP (e.g., Hfss, Icepak).

- Disable exporting variables:
  from pyaedt import Hfss
  hfss = Hfss()
  hfss.configurations.options.export_variables = False

- Disable importing datasets and set a custom object mapping tolerance:
  hfss.configurations.options.import_datasets = False
  hfss.configurations.options.object_mapping_tolerance = 1e-8

Notes
-----
- Defaults are conservative: most categories export and import by default (True), with object_mapping_tolerance set to 1e-9 and skip_import_if_exists set to False.
- The _find_datasets helper is for internal use to extract dataset references (not part of the public API).
