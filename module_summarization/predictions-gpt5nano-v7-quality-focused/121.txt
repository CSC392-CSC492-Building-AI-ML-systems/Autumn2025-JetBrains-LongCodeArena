Cradmin Menu System (crmenu)
============================

This document describes the Cradmin menu system implemented in crmenu. It provides renderable menu items, an expandable menu, and utilities to obtain default menu renderables from configuration.

Overview
--------

The crmenu module exposes a set of renderable classes used to build navigation menus within Cradmin pages. It relies on Django Cradmin’s renderable framework and follows a BEM-inspired naming convention for CSS classes.

Key concepts
------------

- Link renderables render individual navigation items (links or buttons) in the main menu.
- Expandable menu items render links that may toggle an expanded sub-list.
- A dedicated toggle renderable provides the UI element to open/close the expandable menu.
- The AbstractMenuRenderable base class coordinates a list of renderable items and selects the appropriate renderable classes for links, separators, and buttons.

Public API
----------

Functions

- get_default_expandable_menu_renderable(**kwargs)

  - Description: Returns the default expandable menu renderable as configured by DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS. If the setting is not defined, returns None.
  - Behavior:
    - Reads DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS from settings.
    - Imports the class path and instantiates it with the provided kwargs.
  - Signature: get_default_expandable_menu_renderable(**kwargs)

Classes

- BaseMenuLinkRenderable(renderable.AbstractBemRenderable)

  - Purpose: Base class for link-like menu items.
  - Template: django_cradmin/crmenu/menuitem/link.django.html
  - Initialization:
    - __init__(self, label, url, is_active=False, bem_variant_list=None, parent_bem_block=None, **kwargs)
    - Stores label, url, is_active, and parent BEM block. Delegates to the parent constructor with bem_variant_list.
  - Key methods:
    - get_fallback_parent_bem_block(self) -> str: Returns 'adminui-page-header' as default parent block.
    - get_parent_bem_block(self) -> str: Uses explicit parent_bem_block or falls back.
    - get_bem_element(self) -> str: Returns the BEM element for the link: "<parent>__navlink".
    - get_active_bem_variant_list(self) -> list: Returns ['active'].
    - get_bem_variant_list(self) -> list: Extends the parent variant list and adds 'active' if is_active is True.

- NavLinkItemRenderable(BaseMenuLinkRenderable)

  - Purpose: Use this to add links to the main menu.

- NavLinkButtonItemRenderable(NavLinkItemRenderable)

  - Purpose: Use this to add links styled as a button to the main menu.
  - Behavior: get_bem_variant_list returns ['button'] to render as a button.

- ExpandableMenuItem(BaseMenuLinkRenderable)

  - Purpose: Represents a link that expands to reveal a sub-list.
  - Template: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
  - Specializations:
    - get_fallback_parent_bem_block(self) -> str: Returns 'expandable-menu'.
    - get_bem_element(self) -> str: Returns '<parent>__link'.
    - get_list_element_bem_element(self) -> str: Returns '<parent>__listitem'.

- ExpandableMenuSeparator(renderable.AbstractBemRenderable)

  - Purpose: Renders a separator within an expandable menu.
  - Template: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
  - Initialization: __init__(self, parent_bem_block=None, **kwargs) stores parent_bem_block.
  - Behavior:
    - get_fallback_parent_bem_block(self) -> str: Returns 'expandable-menu'.
    - get_parent_bem_block(self) -> str: Uses explicit parent_bem_block or falls back.
    - get_bem_element(self) -> str: Returns '<parent>__separator'.

- MenuToggleItemItemRenderable(renderable.AbstractBemRenderable)

  - Purpose: Renders the expandable menu toggle in the header.
  - Template: django_cradmin/crmenu/menuitem/menutoggle.django.html
  - Initialization: __init__(self, parent_bem_block=None, **kwargs) stores parent_bem_block.
  - Key methods:
    - get_label(self) -> LazyString: Returns the translated label for the toggle. Default is "Menu".
      - Uses pgettext_lazy('cradmin default header menu toggle label', 'Menu').
    - get_fallback_parent_bem_block(self) -> str: Returns 'adminui-page-header'.
    - get_parent_bem_block(self) -> str: Uses explicit parent_bem_block or falls back.
    - get_bem_element(self) -> str: Returns '<parent>__navmenutoggle'.
    - get_label_css_class(self) -> str: Returns '<parent>__navmenutoggle-label'.
    - get_icon_css_class(self) -> str: Returns '<parent>__navmenutoggle-icon'.

- AbstractMenuRenderable(renderable.AbstractRenderableWithCss)

  - Purpose: Base class for rendering a menu as a collection of renderables.
  - Attributes (class-level defaults):
    - link_renderable_class = None
    - separator_renderable_class = None
    - button_renderable_class = None
  - Initialization:
    - __init__(self, request, cradmin_instance=None)
    - Stores request and cradmin_instance; initializes renderable_list as an empty list.
  - Core behavior:
    - iter_renderables(self) -> iterator: Returns an iterator over renderable_list.
    - get_link_renderable_class(self) -> class: Returns the configured link renderable class; raises ValueError if not set.
    - get_separator_renderable_class(self) -> class: Returns the configured separator renderable class.
  - Notes:
    - Subclasses may override the renderable classes to customize the composition of the menu.

How to use
----------

- Build a menu by composing renderables into renderable_list and pass the appropriate context (request, cradmin_instance).

Example: simple menu with two items
-----------------------------------

- Creates a menu using NavLinkItemRenderable for a standard link and NavLinkButtonItemRenderable for a button-like link.

```python
from django_cradmin.crmenu import AbstractMenuRenderable, NavLinkItemRenderable, NavLinkButtonItemRenderable

class SimpleMenu(AbstractMenuRenderable):
    link_renderable_class = NavLinkItemRenderable
    separator_renderable_class = None
    button_renderable_class = NavLinkButtonItemRenderable

    def __init__(self, request, cradmin_instance=None):
        super(SimpleMenu, self).__init__(request, cradmin_instance)
        self.renderable_list = [
            NavLinkItemRenderable(label='Home', url='/home/', is_active=True),
            NavLinkButtonItemRenderable(label='Settings', url='/settings/')
        ]
```

Example: expandable menu item
-----------------------------

```python
from django_cradmin.crmenu import ExpandableMenuItem, ExpandableMenuSeparator

expandable = ExpandableMenuItem(label='Reports', url='/reports/', is_active=False)
separator = ExpandableMenuSeparator()

# Add to a menu’s renderable_list as appropriate.
```

Configuration
-------------

- DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS

  - Purpose: Specifies the class used to render the default expandable menu.
  - Type: String path to a Python class.
  - Behavior: get_default_expandable_menu_renderable(**kwargs) imports and instantiates this class with the provided kwargs. If the setting is not defined, the function returns None.

Usage example
--------------

In settings.py, define:

```python
DJANGO_CRADMIN_DEFAULT_EXPANDABLE_MENU_CLASS = 'myapp.menus.DefaultExpandableMenu'
```

Then obtain an instance via:

```python
from django_cradmin.crmenu import get_default_expandable_menu_renderable

menu = get_default_expandable_menu_renderable(request=request, cradmin_instance=cradmin_instance)
```

Templates
---------

The renderable classes reference specific Django templates. Key templates include:

- Django link item: django_cradmin/crmenu/menuitem/link.django.html
- Expandable menu item: django_cradmin/crmenu/menuitem/expandable-menu-item.django.html
- Expandable menu separator: django_cradmin/crmenu/menuitem/expandable-menu-separator.django.html
- Menu toggle: django_cradmin/crmenu/menuitem/menutoggle.django.html

Notes
-----

- CSS class naming follows a BEM-like convention, driven by get_parent_bem_block, get_bem_element, and get_bem_variant_list methods.
- The default label for the header menu toggle uses a lazy translation key: cradmin default header menu toggle label with the default text "Menu".
