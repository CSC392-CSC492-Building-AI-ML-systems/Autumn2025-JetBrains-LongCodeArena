Delete view with preview, confirm, and delete (Within Role)

Overview
- Implements a preview+confirm+delete workflow for deleting a model instance.
- Provides a dedicated WithinRoleDeleteView that is wired for Django CRAdmin roles and a sane template.
- Builds all necessary context for the template, including object preview, messages, and button labels.
- Handles success messaging and redirection after deletion.

What you get

1) DeleteViewMixin
- get_pagetitle: Returns the page title. Defaults to "Confirm delete".
- get_action_label: Returns the action prefix used for titles and defaults. Returns "Delete".
- get_delete_button_label: Label on the delete button. Defaults to the action label.
- get_cancel_button_label: Label on the cancel button. Defaults to "Cancel".
- get_object_preview: Returns a string representation of the object (the preview shown to the user).
- get_confirm_message: Message shown in the focus area asking for confirmation. Uses the object preview.
- get_success_url: URL to redirect to after a successful delete. Defaults to the app index for the current CRAdmin app.
- get_success_message: Optional message shown after deletion. Defaults to 'Deleted "<object_preview>"'.
- add_success_messages: Adds the success message (if any) to Django messages after deletion.
- add_delete_view_mixin_context_data: Populates template context with:
  - model_verbose_name
  - success_url
  - object_preview
  - pagetitle
  - confirm_message
  - delete_button_label
  - cancel_button_label
- delete: Performs the actual deletion, then adds the success message.

2) WithinRoleDeleteView
- A concrete, ready-to-use delete view for CRAdmin views that have a role.
- Inherits: QuerysetForRoleMixin, DeleteViewMixin, DjangoDeleteView, CommonCradminViewMixin, and WithinRoleViewMixin.
- template_name: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html
- get_context_data: Extends context with JavaScript component IDs and common CRAdmin context, then includes the DeleteViewMixin data.

Template
- Path: django_cradmin/viewhelpers/formview/within_role_delete_view.django.html
- Context data provided to the template includes:
  - object
  - object_preview
  - model_verbose_name
  - pagetitle
  - confirm_message
  - delete_button_label
  - cancel_button_label
  - success_url

Usage

Import and subclass

- Import path (recommended): from django_cradmin import viewhelpers; refer to the view as viewhelpers.formview.WithinRoleDeleteView
- Example:

  from django.db import models
  from django.utils.translation import gettext_lazy
  from django_cradmin.viewhelpers.formview import WithinRoleDeleteView

  class MyModel(models.Model):
      name = models.CharField(max_length=200)

  class MyModelDeleteView(WithinRoleDeleteView):
      model = MyModel

      def get_success_message(self, object_preview):
          return gettext_lazy('Successfully deleted "%(what)s"') % {'what': object_preview}

  # URL configuration (example)
  # path('mymodel/<int:pk>/delete/', MyModelDeleteView.as_view(), name='mymodel-delete')

What to customize

- Page title and labels (optional):
  - get_pagetitle: Override to customize the page title.
  - get_action_label: Override to customize the action prefix.
  - get_delete_button_label: Override to customize the delete button text.
  - get_cancel_button_label: Override to customize the cancel button text.
- Preview and messages:
  - get_object_preview: Override to control the object preview text.
  - get_confirm_message: Override to customize the confirmation prompt.
  - get_success_url: Override to redirect somewhere else after deletion.
  - get_success_message: Override to customize the flash message after deletion.
- Context data:
  - add_delete_view_mixin_context_data: Override to extend or modify the context as needed.

Preview+Confirm+Delete workflow (summary)

- User lands on the delete view.
- Page shows a focus area with:
  - The object preview (via get_object_preview)
  - A confirmation message (via get_confirm_message)
  - A delete button (label from get_delete_button_label)
  - A cancel button (label from get_cancel_button_label)
  - Page title from get_pagetitle
- User clicks Delete (POST request):
  - The object is deleted (via Django’s DeleteView mechanism)
  - A success message is added (via add_success_messages and get_success_message)
  - User is redirected to get_success_url

Notes

- The WithinRoleDeleteView is tailored for CRAdmin’s role-based views and uses the within_role_delete_view.django.html template.
- All user-facing text supports translation via gettext_lazy.
