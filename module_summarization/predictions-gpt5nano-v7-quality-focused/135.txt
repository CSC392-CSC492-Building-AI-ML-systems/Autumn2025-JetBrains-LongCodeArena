Camera Geometry for Cherenkov Cameras
=============================

Overview
--------

This document describes the data structures and APIs used to represent the geometry of a Cherenkov telescope subarray. It focuses on how images or data cubes relate to the physical layout of telescopes on the ground and how to work with the subarray geometry in code.

Key concepts
------------

- SubarrayDescription: Encapsulates the collection of telescopes that form a subarray, including their ground positions and telescope descriptions.
- Telescopes (tel_id): Each telescope in the subarray is identified by a unique telescope id (tel_id) and has an associated TelescopeDescription.
- tel_positions: A mapping from tel_id to a 3D ground position. Each position is a 3-tuple of quantities (x, y, z) with units, typically expressed in meters.
- tel_descriptions (tels): A mapping from tel_id to a TelescopeDescription instance that describes the telescope’s optical and mechanical properties.
- GroundFrame: The coordinate frame used for ground positions. Positions are converted to a GroundFrame SkyCoord for downstream calculations.
- CameraGeometry / CameraReadout (imported in the module): Used elsewhere to describe camera layouts and readout schemes for image/cube data. This document focuses on the subarray geometry that underpins data handling.

API: SubarrayDescription
-------------------------

Constructor
-----------

SubarrayDescription(name, tel_positions=None, tel_descriptions=None, reference_location=None)

- name: String name of the subarray.
- tel_positions: Dict[int, sequence]. Maps telescope id to a 3-tuple (x, y, z) of quantities with units (meters). Each p in the sequence is expected to provide .to and .to_value for unit conversion.
- tel_descriptions: Dict[int, TelescopeDescription]. Maps telescope id to a TelescopeDescription.
- reference_location: EarthLocation representing the subarray reference position in the chosen coordinate system.

Example
-------

- Define telescope ground positions (x, y, z) in meters:

  tel_positions = {
      1: (0*u.m, 0*u.m, 0*u.m),
      2: (20*u.m, 0*u.m, 0*u.m),
      3: (0*u.m, 20*u.m, 0*u.m),
  }

- Provide corresponding telescope descriptions (real objects, not shown here):

  tel_descriptions = {1: tel1_desc, 2: tel2_desc, 3: tel3_desc}

- Create the subarray description:

  sub = SubarrayDescription(name="TestSubarray",
                          tel_positions=tel_positions,
                          tel_descriptions=tel_descriptions)

Attributes and properties
-------------------------

- name
  - The subarray name (string).
- positions
  - The raw input mapping of tel_id to ground position (x, y, z) with units.
- tels
  - Dictionary mapping tel_id to TelescopeDescription.
- reference_location
  - EarthLocation used as the reference point for the ground positions (if provided).

- tel (property)
  - Returns the dictionary of telescope descriptions (tel_id -> TelescopeDescription).

- n_tels (property)
  - Returns the number of telescopes in the subarray.

- tel_coords (lazy property)
  - Returns the telescope positions as an Astropy SkyCoord array in the GroundFrame.
  - Access pattern:
    coords = sub.tel_coords
    print(coords)

- tel_ids (lazy property)
  - Returns a NumPy array of telescope IDs in the subarray.

- tel_indices (lazy property)
  - Returns a dictionary mapping tel_id to a sequential index (0-based) used for fixed-length array packing.

- tel_index_array (lazy property)
  - Returns a 1D NumPy array that maps tel_id to its index in the fixed-length representation.
  - For tel_id beyond the largest ID, gaps are filled with -1.

- tel_ids_to_indices(tel_ids)
  - Maps a telescope id or a list/array of tel_ids to their corresponding fixed-length indices.
  - Example:
    indices = sub.tel_ids_to_indices([1, 3])

- tel_ids_to_mask(tel_ids)
  - Converts a list of telescope ids to a boolean mask of length n_tels where True indicates the presence of a telescope with the given id.
  - Example:
    mask = sub.tel_ids_to_mask([1, 2, 3])

Useful patterns for working with images and data cubes
-----------------------------------------------------

- Coordinate conversions
  - Use sub.tel_coords to obtain a GroundFrame-based SkyCoord array for downstream alignment with camera frames or event coordinates.
- Data alignment
  - When aggregating camera data across a subarray, leverage tel_index_array or tel_indices to pack per-telescope data into fixed-length arrays suitable for image cubes or per-telescope feature maps.
- Telescope filtering
  - Use tel_ids_to_mask to select data from a subset of telescopes when constructing sub-cubes or applying subarray-level cuts.

Operational notes
-----------------

- Versioning
  - CURRENT_TAB_VERSION is set to "2.0".
  - COMPATIBLE_VERSIONS contains {"2.0"}.
  - These values help manage storage or interchange formats for subarray descriptions.
- Information printing
  - info(printer=...) prints a human-readable summary of the subarray, including counts of telescopes per type and a list of telescope IDs per type. The output references a footprint value when available.

Example: Inspecting subarray geometry
-------------------------------------

- Access the coordinates and IDs:

  coords = sub.tel_coords
  print(coords)

  ids = sub.tel_ids
  print(ids)

- Map telescope IDs to fixed indices and back:

  indices = sub.tel_ids_to_indices([1, 3])
  print(indices)

  idx_map = sub.tel_index_array
  print(idx_map)

- Create a boolean mask for a subset of telescopes:

  mask = sub.tel_ids_to_mask([1, 2])
  print(mask)

Notes
-----

- The tel_positions values must be a sequence of three quantity objects (x, y, z), each with length units (e.g., meters). Accessing p[0], p[1], p[2] yields quantities that can be converted to meters via .to_value(u.m).
- The tel_descriptions dictionary must provide valid TelescopeDescription objects for each tel_id present in tel_positions.

Relation to Cherenkov camera data
--------------------------------

- The SubarrayDescription provides the spatial context needed to interpret per-telescope camera data and to assemble multi-telescope data products (image cubes, event arrays, etc.) in a geometrically consistent manner.
- The ground-based geometry serves as the basis for projecting camera-plane data into a common coordinate system, aligning with other frames used in reconstruction and analysis workflows.

This documentation reflects the SubarrayDescription API and its role in managing the geometry of Cherenkov camera arrays.
