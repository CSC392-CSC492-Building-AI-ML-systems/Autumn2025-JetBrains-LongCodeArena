Web Review Publishing with ftrack Session
=========================================

Overview
--------
Publish web review artefacts by interacting with an ftrack server through a Session. The Session encapsulates server connection, authentication, optional caching, plugin loading, and an embedded event hub for publishing and subscribing to events (including non-local ones when enabled).

Getting started
---------------
Environment and requirements
- Server URL: Provide via the server_url parameter or the FTRACK_SERVER environment variable.
- API credentials: Provide api_key and api_user or read from environment variables:
  - FTRACK_API_KEY (or FTRACK_APIKEY for backwards compatibility)
  - FTRACK_API_USER
- Optional features:
  - Plugin discovery paths
  - Cache configuration
  - Schema caching
  - Embedded event hub (real-time web review publishing)

Important environment variable mappings
- FTRACK_SERVER: server URL (e.g. https://ftrack.yourdomain.com)
- FTRACK_API_KEY or FTRACK_APIKEY: API key
- FTRACK_API_USER: API user
- FTRACK_EVENT_PLUGIN_PATH: default plugin search path
- FTRACK_API_SCHEMA_CACHE_PATH: folder path for schema caching

Constructor and parameters
--------------------------
Session(server_url=None, api_key=None, api_user=None, auto_populate=True,
        plugin_paths=None, cache=None, cache_key_maker=None,
        auto_connect_event_hub=None, schema_cache_path=None,
        plugin_arguments=None)

- server_url: URL of the ftrack server (required if not set via FTRACK_SERVER)
- api_key: API key for authentication (required if not set via FTRACK_API_KEY)
- api_user: API user name (required if not set via FTRACK_API_USER)
- auto_populate: Accessing entity attributes auto-fetches data if not already loaded
- plugin_paths: List of paths to search for plugins; defaults to FTRACK_EVENT_PLUGIN_PATH
- cache: Cache instance implementing ftrack_api.cache.Cache or a callable returning a cache; integrated into a layered cache with MemoryCache on top
- cache_key_maker: Object implementing ftrack_api.cache.KeyMaker; defaults to StringKeyMaker
- auto_connect_event_hub: If True, connects the embedded event hub in the background for non-local events
- schema_cache_path: Folder path for schema caching; False disables caching
- plugin_arguments: Optional mapping of keyword arguments passed to plugin registration functions

Authentication
--------------
SessionAuthentication attaches ftrack authentication information to requests:
- Adds headers:
  - ftrack-api-key: api_key
  - ftrack-user: api_user
- Implemented via the __call__ method, suitable for use with requests’ authentication mechanisms

Behavior and validation
-----------------------
- If server_url is not provided, the constructor raises:
  TypeError: Required "server_url" not specified. Pass as argument or set in environment variable FTRACK_SERVER.
- If api_key is not provided, the constructor raises:
  TypeError: Required "api_key" not specified. Pass as argument or set in environment variable FTRACK_API_KEY.
- If api_user is not provided, the constructor raises:
  TypeError: Required "api_user" not specified. Pass as argument, set in environment variable FTRACK_API_USER or one of the standard environment variables used by Python's getpass module.

Publishing for web review: usage considerations
-----------------------------------------------
- Use a Session to establish a connection to the web review backend and to publish review-related entities.
- Enable auto_connect_event_hub if your publishing workflow relies on events (e.g., triggering updates or notifications in real time).
- Leverage schema caching to speed up repeated interactions with the server when publishing large review bundles.
- Configure plugin_arguments to tailor plugin loading for your web review environment without breaking compatibility.

Examples
--------
Example 1: Basic web review publishing session (environment-based)
- Reads credentials from environment variables and connects to the server.

import os
from ftrack_api import Session

server_url = os.environ['FTRACK_SERVER']
api_key = os.environ['FTRACK_API_KEY']
api_user = os.environ['FTRACK_API_USER']

session = Session(
    server_url=server_url,
    api_key=api_key,
    api_user=api_user,
    auto_populate=True,
    auto_connect_event_hub=True
)

Example 2: Custom cache and plugin paths
- Uses a provided cache and includes a custom plugin path.

from ftrack_api.cache import MemoryCache
session = Session(
    server_url='https://ftrack.example.com',
    api_key='your_api_key',
    api_user='your_api_user',
    cache=MemoryCache(),
    plugin_paths=['/path/to/web/review/plugins'],
    auto_connect_event_hub=False
)

Notes
-----
- The event hub connection runs in a background thread to improve startup time.
- Subscribing to events does not require a connected event hub, but publishing non-local events does.
- Schema caching can be disabled by setting schema_cache_path to False.
