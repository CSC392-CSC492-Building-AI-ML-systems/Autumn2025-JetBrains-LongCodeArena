w_succ.rst
==========

Overview
--------
The w_succ command lists segments that successfully reach a target state by reporting
recycled segments from each iteration. For every iteration, it prints the iteration
number, the segment identifier, the segment weight, and the final progress-coordinate
values for each recycled segment.

Synopsis
--------
w_succ [OPTIONS]

Options
-------
- -o, --output OUTPUT_FILE
  Store output in OUTPUT_FILE (default: write to standard output). The file is opened
  for writing text.

Notes:
- The command also accepts standard WestPA configuration options provided via westpa.rc
  and options contributed by the CommonOutputMixin (e.g., header control). The
  explicit option shown above is the primary user-facing control for output destination.

Output format
-------------
If headers are not suppressed, the output begins with the following comment lines:

# successful (recycled) segments
# column 0:    iteration
# column 1:    seg_id
# column 2:    weight
# column>2:    final progress coordinate value

Each subsequent line corresponds to a recycled segment in a given iteration and has the form:

iteration  seg_id  weight  pcoord_0  pcoord_1  ...  pcoord_{N-1}

- iteration: integer, the WESTPA iteration number
- seg_id: integer, the segment identifier
- weight: floating-point, the segment weight
- pcoord_i: the i-th final progress-coordinate value for the segment, formatted
  according to the dtype of the coordinate (see the pcoord_formats mapping in the source)

Notes on the data
- The command processes iterations from 1 up to current_iteration - 1.
- For each iteration, it selects segments whose endpoint_type is RECYCLED.
- If no recycled segments exist for an iteration, that iteration contributes no lines.
- The final progress coordinates are taken from the last pcoord entry for each recycled
  segment (i.e., the final coordinate values within that iteration).

Examples
--------
Print to standard output (default):

$ w_succ

Print to a file:

$ w_succ -o succ.txt

Sample output (header and one data line):

# successful (recycled) segments
# column 0:    iteration
# column 1:    seg_id
# column 2:    weight
# column>2:    final progress coordinate value
   3        12        0.01234567890123    0.1000000
   4        7         0.02345678901234    0.2500000  0.7500000

Notes
- Headers are included by default; you can suppress them via the appropriate CommonOutputMixin
  options (as exposed by WESTPA’s argument handling).
- The exact number of final-progress-coordinate columns equals the dimensionality of
  the pcoord at the final step for each recycled segment (pcoord_ndim).

Implementation details (for developers)
- The command is implemented as a WESTAnalysisTool subclass named WSucc.
- It reads segment indices and final pcoords using WESTDataReaderMixin helpers:
  - get_seg_index(n_iter)
  - get_pcoord_dataset(n_iter)
- It filters segments with endpoint_type == Segment.SEG_ENDPOINT_RECYCLED.
- It formats final coordinate values using a dtype-aware formatter map (pcoord_formats).
- Output defaults to sys.stdout; the -o/--output option overrides this destination.
