Channel Options
===============

Overview
--------
Channel options define how data is encoded in Vega-Lite through its encoding channels
(x, y, color, size, shape, etc.). In this documentation, “Channel Options” refer to the
settings you place inside the Vega-Lite spec under the encoding block. The magic
cell allows you to render a Vega-Lite spec that may include detailed channel options.

How to use
----------
- You provide a Vega-Lite specification in the cell. The spec may be JSON or YAML (if YAML is available).
- You can optionally supply a notebook variable that contains a pandas DataFrame to be used as the dataset. The magic injects the processed data into the spec.
- The cell magic supports:
  - DataFrame binding after the magic name (e.g., %%vegalite df)
  - JSON input with --json
  - YAML input when YAML is available
  - Vega-Lite version selection (default v5)

Encoding channels in a Vega-Lite spec
--------------------------------------
Channel options live under the encoding field of the spec. Each channel defines a mapping
from data fields to visual properties. Example channels include:
- x: position along the horizontal axis
- y: position along the vertical axis
- color: color encoding
- size: point size
- shape: marker shape
- tooltip: hover information

You customize the appearance and behavior of each channel by adding properties such as:
- field and type (to specify the data field and the data type)
- scale, axis, legend, and title options
- binning, aggregations, and other transforms
- interaction and conditionals via channel-level rules

Practical examples
------------------

1) Basic scatter plot with x and y channels
.. code-block:: json

   {
     "mark": "point",
     "encoding": {
       "x": {"field": "Horsepower", "type": "quantitative"},
       "y": {"field": "Miles_per_Gallon", "type": "quantitative"}
     }
   }

2) Adding a color channel
.. code-block:: json

   {
     "mark": "point",
     "encoding": {
       "x": {"field": "Horsepower", "type": "quantitative"},
       "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
       "color": {"field": "Origin", "type": "nominal"}
     }
   }

3) Customizing channel options (axis titles and scales)
.. code-block:: json

   {
     "mark": "line",
     "encoding": {
       "x": {
         "field": "Year",
         "type": "temporal",
         "axis": {"title": "Year"},
         "scale": {"domain": [1950, 2020]}
       },
       "y": {
         "field": "Value",
         "type": "quantitative",
         "axis": {"title": "Value"},
         "scale": {"nice": true}
       }
     }
   }

Using a DataFrame as the data source
------------------------------------
- To bind a DataFrame stored in a notebook variable, pass the variable name after the magic name:

.. code-block:: text

   %%vegalite df
   {
     "mark": "point",
     "encoding": {
       "x": {"field": "Horsepower", "type": "quantitative"},
       "y": {"field": "Miles_per_Gallon", "type": "quantitative"},
       "color": {"field": "Origin", "type": "nominal"}
     }
   }

- When a DataFrame is provided, the data portion becomes a transformed dataset suitable for Vega-Lite using the library's data_transformers.

Input formats
-------------
- JSON input: Use the --json flag to parse the cell as JSON.
- YAML input: If PyYAML is installed, you can provide a YAML spec. Otherwise, you must use JSON.
- If --json is not used and YAML is unavailable, the cell content must be valid JSON (a YAML parse error guides you otherwise).

Versioning
----------
- The magic defaults to Vega-Lite version 5 (v5). You can select a different version with --version, e.g., --version='v5'.
- Internally, the code maps the version label to the appropriate Vega-Lite class (e.g., "5" -> VegaLite from vegalite_v5).

Notes
-----
- The encoding channel options you configure are standard Vega-Lite options defined in the spec.
- The magic handles loading the spec, parsing input, and injecting a transformed data object when a DataFrame is used.
- This documentation describes how to specify and customize channel options via the Vega-Lite encoding block, not through a separate API in this module.
