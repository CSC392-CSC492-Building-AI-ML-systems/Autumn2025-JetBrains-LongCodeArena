Kernel Ridge Regression
=======================

Module Overview
---------------
The module provides KernelRidge, an estimator that performs kernel ridge regression (KRR). KRR combines ridge regression with the kernel trick, learning a linear function in the feature space induced by the chosen kernel. For non-linear kernels, this corresponds to a non-linear function in the original input space. The learned model in KRR is equivalent to that of SVM regression with squared-error loss and L2 regularization, but KRR affords a closed-form solution and is typically faster for medium-sized datasets. It does not produce a sparse model, as opposed to SVR.

This estimator supports multi-target regression, i.e., y can be a 2D array of shape (n_samples, n_targets). See the User Guide for kernel_ridge for more details.

Class: KernelRidge
------------------

Description
-----------
KernelRidge implements kernel ridge regression. It learns a linear function in the kernel-induced feature space and provides a closed-form solution. It supports both single-target and multi-target regression.

Key behaviors
- Uses the kernel trick: predictions are made in kernel space.
- Supports predefined kernels via strings from the pairwise kernel functions or a user-supplied callable.
- If kernel is set to "precomputed", X is treated as a kernel matrix.
- For callable kernels, the kernel function is invoked on pairs of instances; the callable should accept two 1D arrays (rows) and return a scalar.
- Multi-target support: y can be 2D, yielding dual_coef_ with shape (n_samples, n_targets).

Parameters
----------
alpha : float or array-like of shape (n_targets,), default=1.0
    Regularization strength. Must be positive. If an array is provided, penalties
    apply per target and must match the number of targets.

kernel : str or callable, default="linear"
    Kernel mapping used internally. Passed directly to pairwise kernels.
    If a string, it must be one of the kernels in pairwise.PAIRWISE_KERNEL_FUNCTIONS
    or "precomputed". If "precomputed", X is treated as a kernel matrix.
    If a callable, it is called on each pair of instances (rows) to compute the
    kernel value for that pair. Callables from sklearn.metrics.pairwise are not
    allowed because they operate on matrices rather than on pairwise rows.

gamma : float, default=None
    Gamma parameter for certain kernels (e.g., RBF, polynomial, sigmoid). The
    interpretation depends on the selected kernel. Ignored by kernels that do not
    require it.

degree : int, default=3
    Degree of the polynomial kernel. Ignored by other kernels.

coef0 : float, default=1
    Zero coefficient for polynomial and sigmoid kernels. Ignored by other kernels.

kernel_params : dict, default=None
    Additional keyword parameters for the kernel function when the kernel is
    provided as a callable.

Attributes
----------
dual_coef_ : ndarray of shape (n_samples,) or (n_samples, n_targets)
    Representation of the weight vector(s) in kernel space.

X_fit_ : {ndarray, sparse matrix} of shape (n_samples, n_features)
    Training data used for fitting. If kernel == "precomputed", this stores the
    precomputed kernel matrix of shape (n_samples, n_samples).

n_features_in_ : int
    Number of features seen during fit.

feature_names_in_ : ndarray of shape (n_features_in_,)
    Names of features seen during fit. Defined only if X has string feature
    names. See versionadded notes in the docstring.

Notes
-----
- KernelRidge uses a closed-form solution, which makes fitting straightforward for
  moderate-sized datasets but may become memory-intensive for very large datasets.
- When kernel = "precomputed", both X and Y must adhere to kernel matrix conventions:
  X_fit_ stores the training kernel matrix, and prediction uses the learned dual
  coefficients accordingly.
- The _get_kernel method computes the kernel matrix using pairwise_kernels with the
  appropriate parameters (gamma, degree, coef0) for non-callable kernels, or uses
  the provided callable with kernel_params for callable kernels.

Examples
--------

Basic usage with an RBF kernel
--------------------------------
from sklearn.kernel_ridge import KernelRidge
import numpy as np

# Sample data
rng = np.random.RandomState(0)
n_samples, n_features = 50, 5
X = rng.randn(n_samples, n_features)
y = rng.randn(n_samples)

# Create model
model = KernelRidge(alpha=1.0, kernel="rbf", gamma=0.1)

# Fit and predict
model.fit(X, y)
y_pred = model.predict(X)

# y_pred has shape (n_samples,)

Multi-target regression
-----------------------
# Generate multi-target data
Y = rng.randn(n_samples, 3)

model_mt = KernelRidge(alpha=1.0, kernel="linear")
model_mt.fit(X, Y)
Y_pred = model_mt.predict(X)

Prediction with a precomputed kernel
-----------------------------------
# Compute kernel matrix K for training data
K = pairwise_kernels(X, X, metric="linear")
model_precomputed = KernelRidge(alpha=1.0, kernel="precomputed")
model_precomputed.fit(K, y)
# For prediction, provide the kernel matrix between new samples and training samples
K_test = pairwise_kernels(X, X, metric="linear")  # or compute between X_test and X_train
y_pred_test = model_precomputed.predict(K_test)

Notes for Implementation
------------------------
- The internal _get_kernel(X, Y=None) handles parameter preparation for kernel computations.
- If kernel is a callable, the callable is invoked for kernel computations with the provided kernel_params.
- If kernel is a string, gamma, degree, and coef0 are used to configure the kernel via pairwise_kernels.
