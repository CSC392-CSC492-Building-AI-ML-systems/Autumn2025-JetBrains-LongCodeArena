# plotting.md

General plotting routines using the Bokeh backend

This module provides a small set of general plotting utilities built on top of the Bokeh backend. It exposes a central plotter instance and helper functions to configure defaults, inspect help text, and create common plots.

## Plotter and defaults

- plotter instance
  - plot_params = BokehPlotter()
  - This object controls defaults, figure creation, plotting options, tooltips, and saving behavior for Bokeh plots.

- Default management
  - set_bokeh_plot_defaults(**kwargs)
    - Sets default options for the Bokeh backend. These defaults are defined by the BokehPlotter class.
    - Example:
      - set_bokeh_plot_defaults(title='My Plot', width=800)
  - reset_bokeh_plot_defaults()
    - Resets the Bokeh defaults to hardcoded values in the code.
  - show_bokeh_plot_defaults()
    - Prints the currently active defaults (as a dictionary).
  - get_bokeh_help(key)
    - Prints the description of a specific default key from the BokehPlotter.
  - load_bokeh_defaults(filename='plot_bokeh_defaults.json')
    - Loads defaults for the Bokeh backend from a JSON file.
  - save_bokeh_defaults(filename='plot_bokeh_defaults.json', save_complete=False)
    - Saves the current defaults to a JSON file.
    - If save_complete is True, both modified user defaults and unmodified hardcoded defaults are stored.

## General plots

The module provides two primary general plotting routines based on Bokeh: a single-scatter plot and a multi-scatter plot. Both rely on the shared data handling and figure setup provided by the BokehPlotter and the data preparation routine process_data_arguments.

### bokeh_scatter
Create an interactive scatter plot with a single data set.

- Signature
  - bokeh_scatter(x, y=None, *, xlabel='x', ylabel='y', title='', figure=None, data=None, saveas='scatter', copy_data=False, **kwargs)
- Behavior
  - Accepts x and y as either array-like data or keys into a provided data source (e.g., a DataFrame). If you pass a data source as the first positional argument, a deprecation warning is emitted and the data source is moved to the `data` parameter.
  - If the first argument is a data source (dict, DataFrame, or ColumnDataSource) or x is None, the function expects data to be supplied via the `data` keyword. The deprecated-path behavior extracts x and y from kwargs using keys 'xdata' and 'ydata' (defaulting to 'x' and 'y' if not provided).
  - Data preparation is done via process_data_arguments with:
    - single_plot=True
    - same_length=True
    - use_column_source=True
  - The prepared data yields an entry with x and y column names and a corresponding ColumnDataSource.
  - The plot is created with the BokehPlotter helper functions:
    - Prepare the figure with title and axis labels
    - Apply plot defaults and kwargs
    - Draw tooltips for the data point(s)
    - Optionally apply a defined line level, vertical/horizontal references, and limits
    - Save the figure if requested (saveas)
- Output
  - Returns a Bokeh Figure object
- Example
  - Simple scatter from explicit arrays:
    - p = bokeh_scatter([0, 1, 2, 3], [10, 15, 13, 17], xlabel='Time (s)', ylabel='Value', title='Sample Scatter')
  - Scatter from a DataFrame:
    - import pandas as pd
    - df = pd.DataFrame({'time': [0, 1, 2, 3], 'value': [10, 15, 13, 17]})
    - p = bokeh_scatter(x='time', y='value', data=df, xlabel='Time (s)', ylabel='Value', title='Time vs Value')

Notes
- The function delegates a lot of behavior to BokehPlotter, including how tooltips are shown and how defaults are applied.
- Unknown kwargs are passed through to the underlying Bokeh scatter call after defaults and parameter validation.

### bokeh_multi_scatter
Create an interactive scatter plot that supports multiple data sets (multi-series).

- Signature
  - bokeh_multi_scatter(x, y=None, *, data=None, figure=None, xlabel='x', ylabel='y', title='', saveas='scatter', copy_data=False, set_default_legend=True, **kwargs)
- Behavior
  - Similar data handling as bokeh_scatter, but designed to plot multiple data series.
  - If set_default_legend is True, data names are used to generate default legend labels.
  - Uses ColumnDataSource for data sharing across series and to enable interactivity.
  - The function passes kwargs to the underlying Bokeh plotting machinery and applies plotter defaults, tooltips, and styling.
- Output
  - Returns a Bokeh Figure object
- Example
  - Suppose you have a DataFrame with multiple series defined in separate columns and a common x-axis:
    - df = pd.DataFrame({'x': [0, 1, 2, 3],
                         'series1': [1, 2, 1.5, 2.5],
                         'series2': [2, 2.2, 2.8, 3.1]})
    - p = bokeh_multi_scatter(x='x', data=df, y=['series1', 'series2'], title='Multiple Series', set_default_legend=True)

Implementation notes
- Both plotting helpers rely on process_data_arguments for consistent data handling, including support for copying data, ensuring equal lengths, and using a column-based data source.
- The plotting pipeline uses:
  - plot_params.set_defaults(...) to configure default plot type and names
  - plot_params.set_parameters(...) to validate and merge user kwargs
  - plot_params.prepare_figure(...) to instantiate or reuse a Bokeh figure
  - plot_params.plot_kwargs(plot_type=...) to obtain plotting arguments appropriate for the plot type
  - plot_params.add_tooltips(...) to attach helpful tooltips
  - plot_params.draw_straight_lines(...) and plot_params.set_limits(...) to enforce visual constraints
  - plot_params.save_plot(...) to save the figure when requested

Practical tips
- Use data frames with named columns and pass x and y as strings to specify keys within data for clean and flexible plotting:
  - df = pandas.DataFrame({'x': [..], 'y': [..]})
  - p = bokeh_scatter(x='x', y='y', data=df, xlabel='X', ylabel='Y', title='Plot')
- To customize defaults globally, call set_bokeh_plot_defaults with desired keys (these keys are defined by BokehPlotter).
- Inspect current defaults with show_bokeh_plot_defaults and adjust as needed before plotting.

For more details, see the BokehPlotter class methods:
- set_defaults, reset_defaults, get_dict, get_description, load_defaults, save_defaults
- prepare_figure, plot_kwargs, add_tooltips, draw_straight_lines, set_limits, save_plot

These general plotting routines provide a consistent, script-friendly way to produce interactive plots with a focus on ease of use and integration with the Masci-tools data processing flow.
