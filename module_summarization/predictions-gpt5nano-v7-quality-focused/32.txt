w_run
=====


Overview
--------
w_run starts or continues a WEST simulation. It builds a work manager, loads the simulation manager and drivers, wires them together, and executes the run. The command supports both master and worker roles, with the master orchestrating the run and workers executing their assigned tasks under the work manager.

Invocation
----------
w_run is a command-line entry point that uses an argument parser configured by WESTPA. The parser is augmented with global WESTPA arguments and work-manager specific options, then the simulation is started or continued.

Key behavior:
- Creates a work manager with make_work_manager() and stores it in westpa.rc.work_manager
- Loads the simulation manager and drivers (system, data manager, propagator, we driver)
- Links components (e.g., propagator.system = system)
- Enters a with work_manager: block to ensure proper resource management
- Master process performs run preparation, execution, and finalization
- Non-master processes participate as workers via the work manager
- Handles KeyboardInterrupt and generic exceptions gracefully, reporting status and logging details

Arguments and options
---------------------
- Global WESTPA arguments are added by westpa.rc.add_args(parser)
- --oneseg
  - dest: only_one_segment
  - action: store_true
  - help: "only propagate one segment (useful for debugging propagators)"
  - This option is parsed and available to the run environment via the common WESTPA argument infrastructure
- Work-manager arguments are added by work_managers.environment.add_wm_args(parser)

What w_run does (high level)
-----------------------------
- Parses arguments and configures WESTPA runtime environment
- Creates and configures the work manager
- Loads sim_manager and all drivers
- Binds components together (system, data_manager, propagator, we_driver)
- Runs the simulation under the work manager control
- Handles shutdowns and exceptions with status messages and logs

Inside the run flow
-------------------
- Master process:
  - Installs a SIGINT handler
  - Loads plugins through sim_manager.load_plugins()
  - Calls sim_manager.prepare_run()
  - Executes the run via sim_manager.run()
  - Calls sim_manager.finalize_run()
  - Catches KeyboardInterrupt to report interruption
  - Catches other Exceptions to report and log details (including a traceback)
- Worker processes:
  - Call work_manager.run() to participate in the distributed run

Example usage
-------------
- Start a new run (default configuration)
  - w_run
- Start or continue with the one-segment mode (for debugging propagators)
  - w_run --oneseg
- Combine with WESTPA configuration (environment and WM args can be provided as needed)
  - w_run [additional WM args as supported by your setup]

Implementation notes
--------------------
- Entry point: entry_point() creates the argument parser, registers arguments, processes configuration, and invokes run_simulation()
- Core entry: run_simulation() creates the work manager, wires up the sim_manager and drivers, and executes the run within a with work_manager: context
- Dependency wiring:
  - propagator.system = system
  - data_manager.system = system
  - we_driver.system = system
  - sim_manager.data_manager = data_manager
  - sim_manager.system = system
  - sim_manager.propagator = propagator
  - sim_manager.we_driver = we_driver
- Error handling: logs error messages and returns meaningful status to the user via westpa.rc.pstatus()
