Utilities for Developers
========================

Overview
--------
This module provides compatibility utilities and helpers used across the project to support older Python, NumPy, and SciPy versions. It also includes small wrappers that unify behavior across versions (e.g., percentile handling, sparse matrix helpers, and threadpool control). Internal helpers support data access and numerical routines in a version-stable way.

Public utilities
----------------
- threadpool_limits(limits=None, user_api=None)
  - Controls the number of threads used by third-party libraries via a threadpool controller.
  - Behavior adapts to whether a global threadpool controller is available (threadpoolctl 3.x+). Falls back to threadpoolctl’s own limits if no controller exists.
  - Example:
    .. code-block:: python
       from sklearn.utils.fixes import threadpool_limits
       with threadpool_limits(limits=4):
           # Run code that benefits from limited threading
           ...

- threadpool_info()
  - Returns information about the current threadpool configuration.
  - If a global controller exists, delegates to it; otherwise, queries threadpoolctl directly.
  - Example:
    .. code-block:: python
       from sklearn.utils.fixes import threadpool_info
       info = threadpool_info()
       print(info)

- delayed(function)
  - Deprecated wrapper that imports delayed from sklearn.utils.parallel.
  - Maintains compatibility for code paths importing delayed from older locations.
  - Usage:
    .. code-block:: python
       from sklearn.utils.fixes import delayed
       @delayed
       def f(x):
           return x * x

Internal and compatibility helpers
----------------------------------
- _object_dtype_isnan(X)
  - Checks for NaN values in object-dtype arrays: returns a boolean array where True indicates NaN.

- _percentile(a, q, *, method="linear", **kwargs)
  - NumPy 1.22+ exposes percentile with an interpolation keyword. Older NumPy versions map the keyword to interpolation.
  - Maintains a consistent signature across NumPy versions by using method as the public kwarg and translating it to interpolation internally.
  - Public percentile variable is wired to:
    - _percentile for NumPy < 1.22
    - numpy.percentile for NumPy >= 1.22

- _mode(a, axis=0)
  - Backport-friendly wrapper around scipy.stats.mode.
  - For SciPy >= 1.9.0, uses mode(..., keepdims=True) and handles shape changes for SciPy >= 1.10.999 when axis=None.
  - Falls back to scipy.stats.mode for older SciPy versions.
  - Note: This is an internal helper, intended for stable behavior across SciPy releases.

- _sparse_linalg_cg(A, b, **kwargs)
  - Compatibility wrapper for the conjugate gradient solver.
  - If SciPy >= 1.12.0, uses scipy.sparse.linalg.cg directly.
  - Otherwise, maps rtol to tol and defaults atol to "legacy" if not provided, then calls cg.

- Importlib.resources backport helpers (Python 3.9 compatibility)
  - _open_text(data_module, data_file_name)
  - _open_binary(data_module, data_file_name)
  - _read_text(descr_module, descr_file_name)
  - _path(data_module, data_file_name)
  - These helpers provide a uniform way to access package data across Python versions:
    - For Python >= 3.9, use importlib.resources.files(...).joinpath(...).open/read.
    - For older Python versions, use importlib.resources.open_text/open_binary/read_text or as appropriate.

Version and compatibility notes
-------------------------------
- SciPy 1.8+: Sparse array types (csr_array, csc_array, coo_array, lil_array, dok_array, bsr_array) are supported in addition to the traditional sparse matrices.
- Import of line_search_wolfe1/line_search_wolfe2:
  - SciPy >= 1.8 uses SciPy’s internal _linesearch module.
  - Older SciPy versions fall back to scipy.optimize.linesearch.
- NumPy < 1.22:
  - percentile uses the internal _percentile wrapper, preserving behavior with the method keyword via interpolation.
- SciPy >= 1.12:
  - cg is referenced directly as scipy.sparse.linalg.cg.
  - Older SciPy versions route through a compatibility wrapper to align argument names.
- Python 3.9+:
  - importlib.resources backport utilities use the modern API (resources.files) when available.
  - Older Python versions rely on the legacy importlib.resources API.

Examples
--------
- Limit threadpool usage for a computation:
  .. code-block:: python
     from sklearn.utils.fixes import threadpool_limits
     with threadpool_limits(limits=2):
         result = heavy_computation()

- Access a data file from a package using the backport API:
  .. code-block:: python
     from sklearn.utils.fixes import _open_text
     with _open_text(some_package, "data.txt") as f:
         content = f.read()

- Internal helpers (prefixed with an underscore) exist to stabilize behavior across dependencies and platforms. They are not part of the public API and may change without notice.
