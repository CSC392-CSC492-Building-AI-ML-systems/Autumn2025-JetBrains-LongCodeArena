Getting Started
===============

Overview
--------

This module defines functionality shared by master and parent processes. It is intended to be sent to child contexts that are about to become a parent due to recursive connections. It also provides utilities and constants used when coordinating process creation, PTY handling, logging, and core source transmission to children.

Prerequisites and environment
-----------------------------

- The code adapts to Python 2.x and Python 3.x, with specific branches for each runtime. If mitogen.core.PY3 is true, certain aliases are set (e.g., xrange becomes range).
- On some systems, PTY usage is restricted or limited. The module detects SELinux enforcement and avoids socketpairs when SELinux is enabled.
- PTY-related constants are exposed for internal use:
  - LINUX_TIOCGPTN
  - LINUX_TIOCSPTLCK
- Platform and messaging support:
  - IS_LINUX indicates Linux environments
  - BROKER_SHUTDOWN_MSG and OPENPTY_MSG provide consistent user-facing error and status messages
  - SYS_EXECUTABLE_MSG warns when sys.executable is unset

Getting started with core utilities
-----------------------------------

- Inspect the current log level
  - get_log_level() returns the effective log level for the module.

- Determine the Python executable for child processes
  - get_sys_executable() returns sys.executable when it is set.
  - If sys.executable is unset, it falls back to '/usr/bin/python' and logs SYS_EXECUTABLE_MSG once.

- Access the core Mitogen source for sending to children
  - _get_core_source() returns the Mitogen core source code (used internally).
  - get_core_source_partial() prepares a lazily-initialized, compressed representation of the core source. This helps avoid expensive work when multiple threads need the data concurrently.

- Pre-exec customization (advanced)
  - _preexec_hook is a global hook invoked prior to exec() of any new child process. It is used by utilities that need to adjust scheduling, such as resetting process affinity.

- SELinux-aware behavior
  - The module detects SELINUX_ENABLED by probing /sys/fs/selinux/enforce. If SELinux is enforced, Mitogen avoids certain socketpair-based IPC paths to maintain compatibility.

Code excerpts (illustrative)
--------------------------------

- Retrieve the Python executable for a child process:

  from mitogen.core import get_sys_executable
  child_python = get_sys_executable()

- Prepare the core source for transmission to a child:

  from mitogen.core import get_core_source_partial
  core_source = get_core_source_partial()

- Check Linux-specific PTY handling (internal use):

  from mitogen.core import IS_LINUX
  if IS_LINUX:
      # Use Linux PTY handling paths here
      pass

- Optional: customize pre-exec behavior for child processes:

  import mitogen.core
  def my_preexec():
      # e.g., set CPU affinity or other scheduling hints
      pass
  mitogen.core._preexec_hook = my_preexec

Common messages you may encounter
---------------------------------

- OPENPTY_MSG: Indicates failure to create a PTY, often due to reaching the maximum number of PTYs. Guidance suggests increasing system limits or reconfiguring to reduce PTY usage.
- BROKER_SHUTDOWN_MSG: Signals that the associated Broker began to shut down, and the connection is cancelled.
- SYS_EXECUTABLE_MSG: Warns that Python’s sys.executable is unset, and that '/usr/bin/python' is used by default for child contexts.

Practical example: Quick start in a parent context
--------------------------------------------------

- Prepare to launch a child that may become a parent:

  from mitogen.core import get_sys_executable, get_core_source_partial, _preexec_hook
  exe = get_sys_executable()
  core = get_core_source_partial()
  # Pass 'exe' and 'core' to the child as part of the initialization payload.

- Optionally set a pre-exec hook for scheduling traits:

  def set_affinity():
      # Implement affinity logic here
      pass
  _preexec_hook = set_affinity

Notes and further reading
--------------------------

- The _get_core_source() function behavior varies between master and non-master contexts:
  - Non-masters fetch the cached mitogen.core source code directly via import mechanisms.
  - Masters replace it with a version that performs minification inline.

- The get_core_source_partial() function ensures thread-safe initialization using a lock to avoid contention when multiple threads attempt to prepare the compressed core source simultaneously.

Next steps
----------

- Integrate these utilities into your master/parent code paths to ensure consistent, portable child initialization.
- If you rely on PTY-backed communication, monitor for OPENPTY_MSG and adjust system limits or configuration accordingly.
- Use get_sys_executable() to robustly determine the Python interpreter for remote contexts, especially on diverse environments.
