Imputation with MICE (Multiple Imputation by Chained Equations)
===============================================================

Overview
--------

This module implements the Multiple Imputation through Chained Equations (MICE)
approach for handling missing data in statistical analyses.

Key ideas
- Imputations follow a cyclical process across variables with missing values.
- For each focus variable, an imputation model is fitted and used to generate imputations.
- Predictive mean matching (PMM) is used for imputing missing values in the focus variable.
- After imputing all variables, an analysis model is fit to the completed data.
- Repeating the cycle and combining results yields point estimates and standard errors for the analysis model parameters.

Imputation workflow (high level)
- Start by imputing missing values with simple initial values (e.g., means).
- For each variable with missing data (the focus variable):
  - Fit an imputation model regressing the focus variable on other variables.
  - Impute the missing values for the focus variable (using PMM).
- After all variables are imputed, fit the analysis model.
- Repeat the cycle and combine results using a combining rule to obtain parameter estimates and their standard errors.

Imputations are produced using models specified per variable via a model class and a formula. The default model is ordinary least squares (OLS) with a main-effects formula over the remaining variables.

Usage modes
- MICEData: Wraps a Pandas DataFrame to perform imputations directly. Useful for generating multiple imputed data sets and for visualizing imputation performance. A history hook (history_callback) can save intermediate information across imputations.
- MICE: Combines MICEData with an analysis model. Runs imputation, fits the analysis models, and combines results into a MICEResults object. The MICEResults object provides a summary of estimands and inferential quantities.

Terminology
- Analysis model: The statistical model of interest used for inference on the completed data.
- Imputation model: A regression model for a focus variable conditioned on other variables; one such model exists per variable with missing values.
- Perturbation method: How parameter estimates are perturbed during imputation (e.g., 'gaussian' or 'boot'). The Gaussian method draws from a Gaussian approximation to the posterior; the bootstrap method uses parameter estimates from a bootstrapped data set.
- MICEData: Wraps a Pandas DataFrame to manage imputations and related data structures.
- MICE: Orchestrates the MICE process using a MICEData object and an analysis model, then combines results.
- MICEResults: The result object from MICE that summarizes the analysis after pooling across imputations.

Class structure
- MICEData
  - Wraps a Pandas DataFrame and stores imputation models for variables with missing values.
  - Provides facilities to produce multiply imputed data sets.
  - Includes plotting utilities to visualize imputation results and missing data patterns.
  - history_callback hook allows saving user-specified information from intermediate datasets.
  - Basic workflow shown in examples:
    - Define imputers for variables using set_imputer.
    - Update imputations with update_all.
    - Iterate over imputations when desired.

- MICE
  - Accepts a MICEData object and an analysis model specification.
  - Runs the full imputation cycle, fits the analysis models, and combines results.
  - Produces a MICEResults object, whose summary method exposes key estimands and inferential quantities.

Notes
- By default, MICEData conserves memory by overwriting the working data with new imputations between iterations.
- When used with MICE, fitted analysis models and results are saved for later inspection.
- The history_callback hook in MICEData enables users to persist arbitrary information from intermediate imputations.

Usage examples
--------------

Imputing data with MICEData (imputations only)
---------------------------------------------
.. code-block:: python

    imp = mice.MICEData(data)
    imp.set_imputer('x1', formula='x2 + np.square(x2) + x3')
    for j in range(20):
        imp.update_all()
        imp.data.to_csv('data%02d.csv' % j)

Iterating through imputations
------------------------------
.. code-block:: python

    imp = mice.MICEData(data)
    j = 0
    for data in imp:
        imp.data.to_csv('data%02d.csv' % j)
        j += 1

Fitting an analysis model and combining results
-----------------------------------------------
.. code-block:: python

    mice = MICE(imp, analysis_model_formula='y ~ x1 + x2 + x3')
    results = mice.run()
    print(results.summary())
