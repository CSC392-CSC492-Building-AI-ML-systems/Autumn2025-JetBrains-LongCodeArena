Permutation feature importance
==============================

This document describes the permutation_importance function and how it computes feature importances by measuring the effect of permuting each feature on the model’s score.

Overview
--------

- For each feature, the feature’s values are randomly permuted across samples to break associations with the target.
- The model’s score is re-evaluated with the permuted data.
- The decrease in score relative to the baseline (unpermuted data) is the feature’s importance.
- Repeats (n_repeats) are performed to obtain a distribution of importances.
- The result is returned as a Bunch with per-feature importances and uncertainty.

API
---

Permutation importance is computed as:

- estimator: Trained estimator with a fit method.
- X: array-like or DataFrame of shape (n_samples, n_features).
- y: array-like of shape (n_samples,) or None (depending on scorer).
- scoring: Scoring function(s) to use. Can be a string, callable, list/tuple, or dict of scorers (see score utilities in the package).
- n_repeats: Number of permutation repeats per feature (minimum 1).
- n_jobs: Number of parallel jobs to run. Use None or a positive integer.
- random_state: Random state for reproducibility.
- sample_weight: Sample weights to use when scoring (or None).
- max_samples: Controls the number of samples used for permutation. Accepts either:
  - an int >= 1 (number of samples), or
  - a float in [0, 1] (fraction of samples).
  If max_samples < n_samples, a subset of rows is used; otherwise the full dataset is used.

Return value
------------

A Bunch with:
- importances_mean: ndarray of shape (n_features,)
  Mean decrease in score over repeats for each feature.
- importances_std: ndarray of shape (n_features,)
  Standard deviation of the importances over repeats.
- importances: ndarray of shape (n_features, n_repeats)
  Raw permutation importance scores for each feature and repeat.

Notes on features and feature names
-----------------------------------

- If X is a pandas DataFrame (or otherwise provides column names), feature names are inferred automatically from X.columns.
- If feature names need to be provided or manipulated, the internal helpers support mapping feature identifiers (by name or index) to their corresponding position.

How it works (high level)
-------------------------

- Compute baseline_score using the provided scorer on the original X, y (and optional sample_weight).
- For each feature (column), permute its values across samples and compute the score on the permuted data.
- Repeat the permutation n_repeats times for each feature.
- Compute importances as baseline_score - permuted_score.
- Return the mean and std of the importances across repeats, along with the full importances matrix.

Internal helpers (brief)
------------------------

- _weights_scorer: Applies the scorer, respecting optional sample_weight.
- _calculate_permutation_scores: Performs the per-feature permutation and scoring across repeats.
- _create_importances_bunch: Builds the final Bunch with mean, std, and raw importances.

Usage examples
--------------

Basic usage with a fitted estimator and a DataFrame X:

- from sklearn.inspection import permutation_importance
- res = permutation_importance(clf, X, y, scoring="accuracy", n_repeats=10, random_state=0, n_jobs=-1)
- print(res.importances_mean)
- print(res.importances_std)

Access to per-feature importances:

- features = X.columns if hasattr(X, "columns") else [f"x{i}" for i in range(X.shape[1])]
- for name, mean in zip(features, res.importances_mean):
-     print(f"{name}: {mean:.4f}")

Notes
-----

- Random state controls reproducibility of the permutation order.
- n_jobs enables parallel computation across features.
- If X is a DataFrame, feature names are used automatically; otherwise, generic names (x0, x1, ...) are created.
- The function supports single or multi-metric scoring depending on the scoring argument; importances are returned for the chosen metrics.
