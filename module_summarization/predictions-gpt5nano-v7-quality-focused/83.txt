CockroachEngine
=================

An extension of Piccolo's Postgres engine tailored for CockroachDB.

Overview
--------

The CockroachEngine class provides CockroachDB-specific behavior on top of the standard Postgres engine. It inherits all querying, connection, and ORM features from PostgresEngine, while enabling a CockroachDB-specific startup step to opt into experimental column type alteration support when possible.

Key characteristics
-------------------

- engine_type: "cockroach"
- min_version_number: 0
  - Note: This is a placeholder because CockroachDB's versioning does not align with the standard versioning used by Piccolo.

Behavior

- The constructor delegates to the base class (PostgresEngine) using the same arguments.
- This class does not add new members beyond what PostgresEngine provides; it primarily specializes behavior via prep_database.

CockroachDB-Specific Preparation
--------------------------------

CockroachEngine implements a CockroachDB-specific preparation step via prep_database. This method attempts to enable an experimental cluster setting that allows altering column types without full table rewrites.

- Action:
  - Executes the SQL statement:
    SET CLUSTER SETTING sql.defaults.experimental_alter_column_type.enabled = true;
  - This is performed on a new database connection using the engine’s internal _run_in_new_connection method.

- Error handling:
  - If the operation raises asyncpg.exceptions.InsufficientPrivilegeError, the engine emits a warning to inform that functionality may not behave as expected.
  - Warning details:
    - Message: "=> Unable to set up Cockroach DB functionality may not behave as expected. Make sure your database user has permission to set cluster options."
    - Level: Level.medium

Code excerpt (for reference)

.. code-block:: python

   async def prep_database(self):
       try:
           await self._run_in_new_connection(
               "SET CLUSTER SETTING sql.defaults.experimental_alter_column_type.enabled = true;"
           )
       except asyncpg.exceptions.InsufficientPrivilegeError:
           colored_warning(
               "=> Unable to set up Cockroach DB "
               "functionality may not behave as expected. Make sure "
               "your database user has permission to set cluster options.",
               level=Level.medium,
           )

Lazy imports
------------

- The module uses a lazy loader for the asyncpg package:
  - asyncpg = LazyLoader("asyncpg", globals(), "asyncpg")
- This defers the import of asyncpg until it is actually needed, reducing import-time dependency requirements.

Usage
-----

Example: creating a CockroachEngine instance

.. code-block:: python

   from piccolo.engine.cockroach_engine import CockroachEngine

   engine = CockroachEngine(
       config={
           "database": "mydb",
           "user": "db_user",
           "password": "secret",
           "host": "localhost",
           "port": 26257,  # CockroachDB default port
       }
   )

Notes
-----

- The prep_database step is optional in environments where the user lacks sufficient privileges. In such cases, a warning is emitted and normal operation continues (with potential caveats related to column type changes).
- The min_version_number is set to 0 due to CockroachDB’s versioning behavior, which does not map cleanly to Piccolo’s versioning scheme.
