Working with Text Data
======================

This tutorial shows how to convert raw text documents into numeric feature vectors that you can feed into machine learning models. It covers preprocessing helpers, tokenization and n-grams, stop words handling, and the main vectorizers: CountVectorizer, TfidfVectorizer, and HashingVectorizer.

Overview
--------

Text data must be transformed into numerical features before most algorithms can process it. The sklearn.feature_extraction.text module provides:

- Tokenization and n-gram generation
- Stop words handling
- Several vectorizers to create document-term matrices
  - CountVectorizer: counts token occurrences
  - TfidfVectorizer: term frequency–inverse document frequency weighting
  - HashingVectorizer: fixed-size, hash-based feature representation
- Lightweight preprocessing helpers for common text cleaning tasks

Preprocessing utilities
-----------------------

You can apply simple preprocessing steps to documents before tokenization. The module provides:

- strip_accents_unicode: replaces accented characters with their base characters (Unicode normalization)
- strip_accents_ascii: transliterates characters to ASCII where possible
- strip_tags: removes simple HTML/XML tags using a regular expression

Examples
--------

Strip HTML tags from a string and normalize accents:

.. code-block:: python

   from sklearn.feature_extraction.text import strip_tags, strip_accents_ascii

   text = "<p>Café naïve résumé</p>"
   clean = strip_tags(text)
   clean = strip_accents_ascii(clean)
   print(clean)  # "Cafe naive resume"

Tokenization, preprocessing, and analysis
-----------------------------------------

Text processing can be customized by composing:

- preprocessor: a function applied before tokenization (e.g., remove tags)
- tokenizer: a function that converts text into tokens
- analyzer: a higher-level function that can replace preprocessor, tokenizer, and n-grams
- ngrams: a function to generate token n-grams
- stop_words: a list of words to filter out before n-gram generation

If you supply an analyzer, the preprocessor/tokenizer/ngrams steps are bypassed in favor of the analyzer’s logic.

Common customization examples

- Custom analyzer that lowercases, strips HTML and accents, and splits on whitespace
- Use preprocessor to strip tags and a tokenizer that splits on whitespace
- Enable 2-grams with ngram_range

Examples
--------

Custom analyzer example:

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer
   from sklearn.feature_extraction.text import strip_tags, strip_accents_ascii

   def my_analyzer(doc):
       doc = strip_tags(doc)
       doc = strip_accents_ascii(doc)
       return doc.lower().split()

   docs = ["<p>Hello world</p>", "Hello again, world!"]
   vectorizer = CountVectorizer(analyzer=my_analyzer)
   X = vectorizer.fit_transform(docs)
   print(X.shape)  # (2, number_of_features)
   print(vectorizer.get_feature_names_out())

Using preprocessor and tokenizer:

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   def preprocessor(doc):
       return strip_tags(doc)

   def tokenizer(doc):
       return doc.lower().split()

   docs = ["<p>Text Data</p>", "Data processing with text."]
   vectorizer = CountVectorizer(preprocessor=preprocessor, tokenizer=tokenizer)
   X = vectorizer.fit_transform(docs)

Next, enable n-grams (unigram and bigram)

.. code-block:: python

   vectorizer = CountVectorizer(ngram_range=(1, 2))
   X = vectorizer.fit_transform(docs)
   print(vectorizer.get_feature_names_out())

Stop words handling
-------------------

The module provides a built-in English stop words list ENGLISH_STOP_WORDS, and you can reference it in two ways:

- Use a built-in option: stop_words='english'
- Supply a custom collection: stop_words=ENGLISH_STOP_WORDS or any set/list of words

Examples

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, ENGLISH_STOP_WORDS

   docs = ["This is a sample document.", "This document is a sample."]
   vec_en = CountVectorizer(stop_words='english')
   X_en = vec_en.fit_transform(docs)
   print(vec_en.get_feature_names_out())

   vec_custom = CountVectorizer(stop_words=ENGLISH_STOP_WORDS)
   X_custom = vec_custom.fit_transform(docs)

Vectorizers
-----------

CountVectorizer
- Produces a document-term matrix with token counts
- Uses a tokenizer and optional analyzer
- Supports stop words, n-grams, and vocabulary control

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer

   docs = ["The cat sat on the mat.", "The dog ate my homework."]
   cv = CountVectorizer()
   X = cv.fit_transform(docs)
   print(X.shape)  # (2, number_of_features)
   print(cv.vocabulary_)

TfidfVectorizer
- Computes TF-IDF-weighted features
- Often improves performance for text classification tasks

.. code-block:: python

   from sklearn.feature_extraction.text import TfidfVectorizer

   docs = ["The cat sat on the mat.", "The dog ate my homework."]
   tfidf = TfidfVectorizer()
   X = tfidf.fit_transform(docs)
   print(X.shape)
   print(tfidf.vocabulary_)

Tip: You can obtain the same effect by chaining a CountVectorizer with TfidfTransformer

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, TfidfTransformer

   docs = ["The cat sat on the mat.", "The dog ate my homework."]
   count = CountVectorizer()
   X_counts = count.fit_transform(docs)
   transformer = TfidfTransformer()
   X_tfidf = transformer.fit_transform(X_counts)
   print(X_tfidf.shape)

HashingVectorizer
- Uses the hashing trick to produce a fixed-size feature space
- Does not require a vocabulary; suitable for large-scale or streaming data
- Use n_features to control the dimensionality; alternate_sign controls sign of hashed features

.. code-block:: python

   from sklearn.feature_extraction.text import HashingVectorizer

   docs = ["The cat sat on the mat.", "The dog ate my homework."]
   hv = HashingVectorizer(n_features=2**16, alternate_sign=False)
   X = hv.transform(docs)  # or hv.fit_transform(docs)
   print(X.shape)

Input handling and decoding
---------------------------

Vectorizers can accept raw strings or file-like inputs. The internal decode logic handles:

- input='filename': reads a file path as binary, then decodes to text
- input='file': reads from a file-like object
- If the input is bytes, it decodes using the vectorizer’s encoding and error handling

These features help you work with datasets stored as files or as in-memory strings.

Practical workflow: end-to-end example
--------------------------------------

1) Prepare data: a list of text documents
2) Choose a vectorizer (Count or TF-IDF) and optional options (stop words, n-grams)
3) Fit on the training data and transform the data
4) Use the resulting sparse matrix as input to a classifier or other model

.. code-block:: python

   from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer

   docs = [
       "I love machine learning. Machine learning is fun.",
       "Text data requires preprocessing. Preprocessing helps models.",
       "I love coding in Python for ML."
   ]

   # Unigram counts
   cv = CountVectorizer(stop_words='english')
   X_counts = cv.fit_transform(docs)

   # TF-IDF features with bigrams
   tfidf = TfidfVectorizer(ngram_range=(1, 2), stop_words='english')
   X_tfidf = tfidf.fit_transform(docs)

   print(X_counts.shape)
   print(X_tfidf.shape)

Best practices
--------------

- Normalize and tokenize consistently: consider customizing the analyzer to enforce lowercase, strip HTML, and remove accents as needed.
- Use stop words judiciously: English stop words often improve performance, but domain-specific text may need custom lists.
- Experiment with n-grams: unigrams capture individual terms; bigrams/trigrams can reveal common phrases.
- For large datasets, HashingVectorizer offers memory-efficient, streaming-friendly feature extraction.
- If you need deterministic vocabulary control, prefer CountVectorizer/TfidfVectorizer over hashing.
