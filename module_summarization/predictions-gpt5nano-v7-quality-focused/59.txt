QuantReg — Quantile Regression model (IRLS with kernel-based covariance)

Overview
- Implements quantile regression by Iteratively Reweighted Least Squares (IRLS).
- Estimates the conditional quantile q of the response given the predictors.
- Uses kernel density estimation to obtain the asymptotic covariance matrix of the parameters.
- Special case: q = 0.5 reduces to the Least Absolute Deviations (LAD) regression.
- Providers robust (heteroskedastic) standard errors by default; option to compute iid standard errors.

API

Class
- QuantReg(RegressionModel)
  - Inherits common regression functionality from Statsmodels’ RegressionModel base.

Constructor parameters
- endog: array-like
  - Endogenous (response) variable.
- exog: array-like
  - Explanatory (design) matrix.

Public methods
- whiten(self, data)
  - Returns data unchanged. The model’s whitener is effectively a no-op.

- fit(self, q=0.5, vcov='robust', kernel='epa', bandwidth='hsheather',
       max_iter=1000, p_tol=1e-6, **kwargs)
  - Solves the quantile regression by IRLS.
  - Parameters:
    - q: float in [0, 1]
      - Target quantile. The default is 0.5 (median/LAD).
    - vcov: {'robust', 'iid'}
      - Method for estimating the variance-covariance matrix of the parameters.
      - 'robust' uses heteroskedasticity-robust standard errors.
      - 'iid' uses iid errors standard errors.
    - kernel: {'epa', 'cos', 'gau', 'par'}
      - Kernel to use in the kernel density estimation for the asymptotic covariance.
    - bandwidth: {'hsheather', 'bofinger', 'chamberlain'}
      - Bandwidth selection method for the kernel density estimation.
    - max_iter: int
      - Maximum number of IRLS iterations.
    - p_tol: float
      - Convergence tolerance for the change in parameter estimates.
  - Returns:
    - A RegressionResultsWrapper containing estimated coefficients, standard errors, and related statistics.

Notes
- The Least Absolute Deviation (LAD) estimator corresponds to q = 0.5.
- The asymptotic covariance matrix follows Greene (2008) methodology, computed via kernel density estimation with the chosen kernel and bandwidth.
- Kernel options (as per fit docstring) include Epanechnikov (epa), Cosine (cos), Gaussian (gau), and Parzen (par). See kernel/bandwidth references in the module docstring for background.
- Bandwidth choices (hsheather, bofinger, chamberlain) align with standard references in quantile regression literature.

Usage example
- Basic usage with median regression and robust standard errors:
  - mod = QuantReg(endog, exog)
  - res = mod.fit(q=0.5, kernel='epa', bandwidth='hsheather', max_iter=1000, p_tol=1e-6)
  - print(res.summary())
  - beta_hat = res.params
  - se = res.bse

- Example for a different quantile and iid standard errors:
  - res = mod.fit(q=0.25, vcov='iid', kernel='gau', bandwidth='bofinger', max_iter=800)

References (conceptual)
- General: Birkes & Dodge (1993); Green (2008); Koenker (2005); LeSage (1999).
- Kernels and bandwidths: Green (2008); Hall & Sheather (1988); Bofinger (1975); Chamberlain (1994).

Implementation notes
- The method relies on a weighted least squares step within each IRLS iteration to update the parameter vector.
- Weights are derived from residuals relative to the target quantile q and the current parameter estimates.
- The Peters–type variance estimator is computed through kernel density estimation with the selected kernel and bandwidth.
