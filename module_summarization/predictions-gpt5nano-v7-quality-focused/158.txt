Migrating to Nextcord: Widget API
=================================

Overview
--------

Nextcord exposes a widget API that represents a guild's public widget data using strong-typed objects instead of plain dictionaries. The core classes are:

- WidgetChannel: a partial representation of a widget channel.
- WidgetMember: a partial representation of a widget member (guild member).
- Widget: a representation of a guild widget, including accessible channels and online members.

These classes help you work with widget data more reliably and with richer typing, including activity objects, status enums, and timestamp conversion.

WidgetChannel
-------------

Represents a partial widget channel.

- Attributes
  - id: int — the channel's ID.
  - name: str — the channel's name.
  - position: int — the channel's position.

- Behaviors
  - str(widget_channel) returns the channel's name.
  - repr(widget_channel) returns a concise identity string.
  - mention: property that returns a string to mention the channel, e.g. "<#123>".
  - created_at: property that returns the channel's creation time (UTC) as a datetime.

- Example
  .. code-block:: python

     wc = WidgetChannel(id=123, name="general", position=1)
     print(wc)            # general
     print(wc.mention)    # <#123>
     print(wc.created_at) # datetime in UTC

WidgetMember
------------

Represents a partial member of the widget’s guild.

- Inheritance: extends BaseUser, so user-related fields come from BaseUser.
- Attributes (subset shown for clarity)
  - id: int — member ID (via BaseUser).
  - name: str — member's username.
  - discriminator: str — member's discriminator.
  - bot: bool — whether the member is a bot.
  - status: Status — member's online status.
  - nick: Optional[str] — member's nickname (may be None).
  - avatar: Optional[str] — member's avatar hash.
  - activity: Optional[Union[BaseActivity, Spotify]] — member's activity, created from the payload’s game data.
  - deafened: Optional[bool] — whether the member is currently deafened (maps from data['deaf'] or data['self_deaf']).
  - muted: Optional[bool] — whether the member is currently muted (maps from data['mute'] or data['self_mute']).
  - suppress: Optional[bool] — whether the member is being suppressed.
  - connected_channel: Optional[WidgetChannel] — the channel the member is connected to.
- display_name: property that returns nick if set, otherwise the member’s name.

- Data mapping from payload
  - nick is taken from data['nick'].
  - status is parsed via try_enum(Status, data.get('status')).
  - deafened is data.get('deaf', False) or data.get('self_deaf', False).
  - muted is data.get('mute', False) or data.get('self_mute', False).
  - suppress is data.get('suppress', False).
  - activity is built from data['game'] using create_activity(game) if present.
  - connected_channel is optionally provided at construction.

- Example
  .. code-block:: python

     data = {
         'nick': 'Alice',
         'name': 'Alice',
         'discriminator': '1234',
         'status': 'online',
         'deaf': False,
         'self_deaf': False,
         'mute': False,
         'self_mute': False,
         'suppress': False,
         'game': {'name': 'Coding', 'type': 0},
     }
     member = WidgetMember(state=state, data=data)
     print(member.display_name)  # Alice

Widget
------

Represents a Guild widget.

- Attributes
  - id: int — the guild's ID.
  - name: str — the guild's name.
  - channels: List[WidgetChannel] — accessible voice channels in the guild.
  - members: List[WidgetMember] — online members in the server; offline members may be omitted by Discord.

- Notes
  - Widget data is partial and may be subject to Discord limitations. When present, you access channels and online members via these structured objects rather than raw dictionaries.

- Example
  .. code-block:: python

     # widget is provided by Nextcord's widget retrieval API
     print(widget.name)
     for ch in widget.channels:
         print(ch.name, ch.position)
     for m in widget.members:
         print(m.display_name, m.status)

Migration guidance
------------------

If you previously consumed widget data as plain dictionaries, migrate to the typed classes above:

- Replace dict access with properties
  - Use widget.channels and widget.members instead of parsing payloads manually.
  - For a channel, use channel.mention to mention it and channel.created_at for timestamps.
  - For a member, use member.display_name for display name, and member.activity for current activity.

- Field name mappings to be aware of
  - Member deaf/mute status: use member.deafened and member.muted (they map from payload['deaf']/'self_deaf' and payload['mute']/'self_mute'] respectively).
  - Member activity: data['game'] payload is translated into an Activity object (BaseActivity or Spotify) via create_activity.

- Activity handling
  - If a member has a game field, Nextcord converts it into a proper Activity object (BaseActivity or Spotify) for richer interaction.

- Time handling
  - Use member_widget.created_at (on channels) for creation time in UTC.

- Display utilities
  - Use member.display_name instead of combining name and nickname manually.

Notes
-----

- WidgetChannel.created_at returns a datetime in UTC derived from the channel’s ID.
- WidgetMember.display_name falls back to the member’s name if nick is not set.
- Activity types supported include BaseActivity and Spotify, produced by create_activity.

API references
--------------

- WidgetChannel: id, name, position, __str__, __repr__, mention, created_at
- WidgetMember: Inherits BaseUser; nick, status, deafened, muted, suppress, activity, connected_channel, display_name
- Widget: id, name, channels, members

These changes provide a more consistent and type-safe way to work with guild widget data in Nextcord.
