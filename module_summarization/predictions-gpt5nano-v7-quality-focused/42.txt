Schema generation
===================

Overview
--------
This module provides utilities to generate SQL for the database schema and to apply it directly to a database client. It relies on each client exposing a schema generator object capable of producing the necessary SQL and executing it.

Key concepts
- get_schema_sql(client, safe): Produces the SQL string that creates the required schema for the given client.
- generate_schema_for_client(client, safe): Generates the SQL and executes it against the client to apply the schema.
- The SQL generation uses the client's schema_generator(client) to obtain a generator object, which provides:
  - get_create_schema_sql(safe): returns the SQL string (or an empty string) for creating the schema. The safe flag controls whether CREATE TABLE IF NOT EXISTS behavior is used.
  - generate_from_string(schema): asynchronously applies the provided SQL to the database.

API
----

get_schema_sql
--------------

Generates the SQL schema for the given client.

Parameters
- client: The DB client to generate Schema SQL for. Type: BaseDBAsyncClient (used for type checking).
- safe: When true, creates the table only if it does not already exist.

Returns
- str: The SQL string to create the schema (empty if no schema is needed).

Example
.. code-block:: python

    sql = get_schema_sql(client, safe=True)
    print(sql)

generate_schema_for_client
------------------------

Generates and applies the SQL schema directly to the given client.

Parameters
- client: The DB client to generate Schema SQL for. Type: BaseDBAsyncClient (used for type checking).
- safe: When true, creates the table only if it does not already exist.

Returns
- None

Behavior
- Builds a schema generator via client.schema_generator(client).
- Computes the schema SQL with get_schema_sql(client, safe).
- Logs the generated SQL at debug level: "Creating schema: %s".
- If the generated SQL is non-empty, executes it asynchronously using generator.generate_from_string(schema).

Example
.. code-block:: python

    await generate_schema_for_client(client, safe=True)

Notes
- The module uses a logger named "tortoise".
- Type hints reference BaseDBAsyncClient inside TYPE_CHECKING, allowing forward references without runtime imports.
- If the generated schema is an empty string, no SQL is executed.

Usage scenarios
--------------

- Quick inspection of the SQL needed to create the schema:
  - Call get_schema_sql with the target client and safe flag.
- Apply the schema to a database:
  - Call generate_schema_for_client with the target client and safe flag.

See also
--------

- Client implementations that provide a schema_generator client method.
- The generator object methods:
  - get_create_schema_sql(safe)
  - generate_from_string(schema)
