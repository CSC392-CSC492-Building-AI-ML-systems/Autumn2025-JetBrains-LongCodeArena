Working with Text Data: Analyzing a collection of text documents
===============================================================

This section covers how to analyze a collection of text documents using scikit-learn’s text feature extraction utilities. It explains the preprocessing, decoding, tokenization, stop-word handling, accent stripping, and n-gram options, and provides practical, ready-to-run examples.

Overview
--------

- You represent a text corpus as a collection (e.g., list) of documents (strings).
- A vectorizer converts the collection into a document–term matrix (sparse).
- Typical pipelines combine a vectorizer with a transformer (e.g., TF–IDF) for downstream machine learning.

Key concepts
------------

- Decoding input: Vectorizers can accept raw strings, files, or filenames. The decode step handles reading file content and decoding bytes to text.
- Preprocessing: Optional steps such as lowercasing and applying a custom accent function.
- Analyzing: A pipeline that may include a decoder, a custom analyzer, a preprocessor, a tokenizer, and an n-gram generator.
- Stop words: You can supply a built-in stop list (e.g., "english"), or a custom collection. Stop words are filtered from tokens before or during n-gram generation.
- Stripping accents: Control over removing diacritics via strip_accents_unicode or strip_accents_ascii.
- Stripping tags: Basic HTML/XML tag stripping with a dedicated helper.
- Tokenization and n-grams: Tokens are produced and optionally combined into n-grams (e.g., bigrams, trigrams).
- Output: A sparse document–term matrix; optionally transformed by TF–IDF or other transformers.

Preprocessing and analysis helpers
----------------------------------

- _preprocess(doc, accent_function=None, lower=False)
  - Applies optional lowercasing and an accent function to a single document.
  - Parameters:
    - doc: input string
    - accent_function: function applied to the text to handle accents
    - lower: whether to lowercase the text
  - Returns: preprocessed string

- _analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
  - Chains optional processing steps from a single document to tokens/ngrams.
  - If analyzer is provided, it replaces preprocessor/tokenizer/ngrams steps.
  - Returns: sequence of tokens or n-grams

- strip_accents_unicode(s)
  - Converts accented Unicode characters to their closest ASCII-like form (preserves characters where no ASCII equivalent exists).
  - Performance: relatively slower, but accurate for many languages.

- strip_accents_ascii(s)
  - Transliterates to ASCII where possible, discarding characters without ASCII equivalents.

- strip_tags(s)
  - Strips basic HTML/XML tags using a simple regular expression.

- _check_stop_list(stop)
  - Converts stop-word specifiers to a usable collection.
  - Accepts:
    - "english" to use ENGLISH_STOP_WORDS
    - None for no stop words
    - A collection of stop words

Using CountVectorizer for a basic analysis
------------------------------------------

CountVectorizer converts a collection of text documents to a document–term matrix of token counts.

Example: basic usage with English stop words and unicode accent stripping

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer, ENGLISH_STOP_WORDS

    docs = [
        "The quick brown fox jumps over the lazy dog.",
        "A quick fox and a lazy dog stay quiet."
    ]

    vectorizer = CountVectorizer(
        stop_words="english",  # uses ENGLISH_STOP_WORDS
        strip_accents="unicode"  # or "ascii" or None
    )
    X = vectorizer.fit_transform(docs)

    # Feature names (vocabulary)
    # Newer scikit-learn versions provide get_feature_names_out()
    feature_names = (
        vectorizer.get_feature_names_out()
        if hasattr(vectorizer, "get_feature_names_out")
        else vectorizer.get_feature_names()
    )
    print(feature_names)

    # Document-term matrix shape
    print(X.shape)

    # Dense representation (for small examples)
    print(X.toarray())

Notes:
- stop_words can also be ENGLISH_STOP_WORDS by importing ENGLISH_STOP_WORDS and passing stop_words=ENGLISH_STOP_WORDS (instead of "english").
- strip_accents accepts "unicode", "ascii", or None.

Advanced customization: HTML stripping and accent handling
---------------------------------------------------------

You can customize preprocessing by composing functions in the preprocessor argument. For example, strip HTML tags first, then normalize accents.

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer
    from __main__ import strip_tags, strip_accents_unicode  # or import from the module

    docs = [
        "<p>The quick brown fox!</p>",
        "<div>The lazy dog &amp; fox.</div>"
    ]

    vectorizer = CountVectorizer(
        preprocessor=lambda s: strip_accents_unicode(strip_tags(s)),
        stop_words=None
    )
    X = vectorizer.fit_transform(docs)
    print(vectorizer.get_feature_names_out())
    print(X.shape)

Notes:
- strip_tags removes simple HTML/XML tags.
- You can combine with ENGLISH_STOP_WORDS for effective cleaning:
  preprocessor=lambda s: strip_accents_unicode(strip_tags(s))
  stop_words=ENGLISH_STOP_WORDS

Using TfidfVectorizer to emphasize important terms
--------------------------------------------------

TfidfVectorizer combines token counts with TF–IDF weighting, highlighting terms that are informative for documents.

.. code-block:: python

    from sklearn.feature_extraction.text import TfidfVectorizer

    docs = [
        "The quick brown fox",
        "The quick fox jumps over the lazy dog"
    ]

    tfidf = TfidfVectorizer(
        stop_words="english",
        strip_accents="unicode",
        ngram_range=(1, 2)  # unigrams and bigrams
    )
    X = tfidf.fit_transform(docs)

    feature_names = (
        tfidf.get_feature_names_out()
        if hasattr(tfidf, "get_feature_names_out")
        else tfidf.get_feature_names()
    )
    print(feature_names)
    print(X.shape)

Combining with a transformer
----------------------------

Most analyses use a pipeline: vectorize text into a document–term matrix, then apply a transformer such as TfidfTransformer.

.. code-block:: python

    from sklearn.feature_extraction.text import CountVectorizer, TfidfTransformer

    docs = ["The quick brown fox", "The quick fox jumps over the lazy dog"]

    vec = CountVectorizer(stop_words="english")
    X_counts = vec.fit_transform(docs)

    transformer = TfidfTransformer()
    X_tfidf = transformer.fit_transform(X_counts)

    print(X_tfidf.shape)

Using HashingVectorizer for streaming data or large corpora
----------------------------------------------------------

HashingVectorizer avoids building a vocabulary and maps tokens to fixed-length feature vectors via hashing. This is useful for very large corpora or streaming data.

.. code-block:: python

    from sklearn.feature_extraction.text import HashingVectorizer

    docs = ["The quick brown fox", "The quick fox jumps over the lazy dog"]

    hv = HashingVectorizer(n_features=2**18, alternate_sign=False)
    X = hv.transform(docs)  # or hv.fit_transform(docs) in API variants

    print(X.shape)
    # Note: HashingVectorizer does not provide feature names, since the features are hashed.

Notes on the internal helper usage
----------------------------------

- _VectorizerMixin.decode handles:
  - input="filename": reads the file in binary and decodes it to text
  - input="file": reads from a file-like object
  - binary content is decoded using the vectorizer’s encoding/decoding settings
- The decode step raises a ValueError for invalid inputs such as np.nan.
- The _analyze function respects an explicit analyzer if provided; otherwise it applies optional preprocessing, tokenization, and n-gram creation.
- _check_stop_list translates "english" to ENGLISH_STOP_WORDS or accepts a custom collection of stop words.

Practical tips
------------

- Start with a simple pipeline (CountVectorizer with a small corpus) to inspect the vocabulary and term frequencies.
- Use stop_words="english" or ENGLISH_STOP_WORDS to reduce common words that carry little discriminative power.
- Experiment with ngram_range to capture phrases (e.g., (1, 2) for unigrams and bigrams).
- Use strip_accents="unicode" or "ascii" to improve robustness across languages with diacritics.
- For large datasets, consider HashingVectorizer to avoid building a large vocabulary; note that feature names are not recoverable with hashing.

API references (selected)
-------------------------

- CountVectorizer: Build a sparse term-frequency matrix from a collection of text documents.
- TfidfVectorizer: Build a TF–IDF weighted document-term matrix.
- HashingVectorizer: Stream-friendly vectorizer that uses hashing to map tokens to features.
