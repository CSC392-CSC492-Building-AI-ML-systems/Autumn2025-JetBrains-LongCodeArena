distutils.rst

Module overview
----------------
This module provides a collection of utilities used by distutils-related tooling. It includes:
- helpers for path manipulation and normalization
- management of temporary directories tied to threads
- a lightweight container for information about installable libraries
- utilities for determining the number of parallel build jobs
- a deprecated function for quoting argument lists
- helper functions to work with call frames to locate module paths

Public API
----------
InstallableLib
    A simple container that holds information about an installable library.

    Parameters
    - name (str): Name of the installed library.
    - build_info (dict): Dictionary containing build information.
    - target_dir (str): Absolute path where the library installs.

    Attributes
    - name: Stores the library name.
    - build_info: Stores build-related information.
    - target_dir: Stores the installation target directory.

    See Also
    - Configuration.add_installed_library

    Notes
    - The three parameters are stored as attributes with the same names.

get_num_build_jobs()
    Determines the number of parallel build jobs allowed.

    Behavior
    - Checks for a command-line setting (via setup.py) and the environment
      variable NPY_NUM_BUILD_JOBS.
    - If not specified, uses the system CPU count (capped at 8 to avoid overload).
    - If a distribution configuration is present, it inspects the parallel setting
      of build-related commands (build, build_ext, build_clib) and uses the largest
      specified value if any are set.

    Return
    - int: The number of parallel build jobs that can run.

quote_args(args)  (deprecated)
    Quote a list of arguments.

    Deprecation
    - This function is deprecated (as of version 1.22).

    Behavior
    - Emits a deprecation warning.
    - Converts items containing spaces to quoted strings where needed.
    - Returns the possibly-modified list of arguments.

allpath(name)
    Convert a slash-separated pathname to the OS-specific path separator.

    Example
    - allpath("foo/bar/baz") -> "foo/bar/baz" on POSIX, or "foo\bar\baz" on Windows
      (depending on the OS).

rel_path(path, parent_path)
    Return the path relative to parent_path.

    Behavior
    - Normalizes both paths to real paths.
    - If the absolute path of path is shorter than parent_path, returns path unchanged.
    - If path equals parent_path, returns an empty string.
    - If parent_path is a prefix of path, returns the portion after parent_path with a
      leading path separator removed.
    - Otherwise, returns path.

get_path_from_frame(frame, parent_path=None)
    Return the path of the module associated with a given frame.

    Behavior
    - Attempts to obtain __file__ from the frame's global/locals.
    - If __file__ is not available, imports the module by name and uses its __file__.
    - If neither is available, defaults to the current directory.
    - If parent_path is provided, returns the path relative to that directory
      (using rel_path).

    Return
    - str: The module path, relative to parent_path if given, otherwise absolute.

njoin(*path)
    Join path components with normalization.

    Description
    - Joins two or more pathname components.
    - Converts slash-separated components to the OS-specific separator.
    - Resolves ".." and "." in the path.
    - Accepts either multiple string arguments (njoin('a', 'b')) or a single
      iterable (njoin(['a', 'b'])).
    - Can handle mixtures of argument types by flattening sequences as needed.

    Notes
    - Uses internal helpers is_sequence and is_string (not shown here) to
      determine argument types.

Internal/private state and helpers
----------------------------------
_tdata
    Thread-local storage object used to hold temporary directory information per thread.

_tmpdirs
    A list of created temporary directories. Used so they can be cleaned up on exit.

clean_up_temporary_directory()
    Cleanup function registered with atexit to remove all tracked temporary
    directories listed in _tmpdirs.

    Behavior
    - Iterates through _tmpdirs and attempts to remove each directory tree.
    - Suppresses OSError exceptions to avoid failing at exit time.

Module lifecycle
----------------
atexit.register(clean_up_temporary_directory)
    Registers the temporary directory cleanup function to run when Python exits.

Notes on usage
--------------
- Temporary directories created by utilities in this module are tracked per thread
  and globally, ensuring proper cleanup at program termination.
- The get_num_build_jobs function integrates with numpy.distutils configuration
  and system capabilities to determine a safe level of parallelism.
- The quote_args function is deprecated; prefer constructing argument lists directly.

Examples
--------
Determine parallel build jobs:
,code-block:: python

    from distutils import r

    jobs = r.get_num_build_jobs()
    print("Parallel build jobs:", jobs)

Join path components portably:
,code-block:: python

    from distutils.rst import allpath  # example import path as placeholder

    path = allpath("foo/bar/baz")
    print(path)

Get a module path from a frame:
,code-block:: python

    import sys
    import traceback
    from distutils.rst import get_path_from_frame

    def where_am_i():
        frame = traceback.extract_stack(limit=2)[0]
        # Note: get_path_from_frame expects a frame object, adapt as needed.
        # This is a conceptual example; in real usage, pass a frame object.
