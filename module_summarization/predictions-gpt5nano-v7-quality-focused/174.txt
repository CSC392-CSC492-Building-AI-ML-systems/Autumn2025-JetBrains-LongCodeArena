Array API support
===================

This document describes how the project aligns with Array API-style usage by standardizing on array-backed inputs and a stable set of minimum dependencies. It focuses on the NumPy-based array interface used throughout the codebase and the version requirements that enable reliable array operations, testing, and documentation.

Overview
--------
- The project relies on NumPy arrays as the primary array backend and uses SciPy for scientific computing needs.
- Minimum dependency versions are defined to ensure compatibility with array-like operations, test frameworks, and documentation tooling.
- The dependency specification is used by CI to pin exact versions for builds, installs, tests, and docs.

Minimum dependencies for Array API support
------------------------------------------
Core array backend
- numpy: minimum version depends on Python implementation
  - PyPy: 1.19.2 or newer
  - CPython and others: 1.17.3 or newer
- scipy: 1.5.0 or newer
- joblib: 1.1.1 or newer
- threadpoolctl: 2.0.0 or newer
- cython: 0.29.33 or newer

Notes
- The dependent_packages dictionary defines for each package a minimum version and a set of tags describing where the package is used (e.g., build, install, docs, tests, benchmarks, examples, maintenance).
- The script also builds an inverse mapping (tag_to_packages) to determine which packages correspond to a given tag. This supports CI configurations that install only the necessary dependencies for a given task.

How to verify and use the minimum versions
------------------------------------------
- The script includes a small CLI to query the minimum version for a given package. Run the script and pass a package name to retrieve its minimum version.
- Example (conceptual):
  - python array_api_min_deps.py numpy
  - Output: 1.17.3 (or 1.19.2 on PyPy)

Practical guidance
------------------
- For development and testing on CPython, ensure numpy >= 1.17.3 and scipy >= 1.5.0 are installed in your environment.
- For PyPy users, prefer numpy >= 1.19.2 to satisfy the minimum requirement.
- When deploying documentation and tests, install the subset of packages tied to docs/tests (as indicated by the tags in dependent_packages) to keep environments lean.

Code references (from the source)
--------------------------------
- The minimum versions and tag associations are defined in the dependent_packages mapping.
- The inverse mapping tag_to_packages is constructed by iterating over dependent_packages and splitting the comma-separated tags.
- A simple command-line interface allows querying the minimum version for a given package when the module runs as a script.

Example structure (summarized)
- dependent_packages: {
    "numpy": (NUMPY_MIN_VERSION, "build, install"),
    "scipy": (SCIPY_MIN_VERSION, "build, install"),
    ...
  }
- tag_to_packages: dict mapping each tag to a list of strings like "numpy>=1.17.3" for all packages associated with that tag.

Appendix: matching the Array API spirit
---------------------------------------
- The project adheres to a NumPy-based array interface, using standard numpy arrays and common array-processing patterns.
- There is no separate, explicit Array API namespace; compatibility is achieved through consistent use of NumPy array semantics and the supported dependency set described above.
