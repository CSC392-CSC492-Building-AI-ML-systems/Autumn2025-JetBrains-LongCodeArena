polys.rst

Polynomial factorization routines in characteristic zero
======================================================

Overview
--------

This module provides the core factorization routines used for polynomials over
characteristic zero domains. It defines a mixin class, _Factor, which offers
high-level factorization strategies such as trial division, algebraic-number
field factorization (Trager-style), and Zassenhaus-based factorization for
univariate and multivariate polynomials over Z. The routines are designed to
be used by higher-level polynomial classes within the library and rely on a
common interface exposed by the surrounding factoring framework.

Key concepts
------------

- Trial division: quick factoring step by dividing by a given list of factors and
  collecting multiplicities.
- Algebraic-number-field factorization: factors multivariate polynomials using
  Trager-style methods over algebraic extensions.
- Zassenhaus-based factorization: robust factorization for polynomials in Z[X],
  including multivariate cases via primitive decomposition, square-free checks,
  and Hensel lifting (EEZ/extended versions).
- Mignotte bound: numeric bound used in Zassenhaus-style lifting to constrain search
  space for possible factors.

Public API
----------

The following members are provided by the _Factor mixin class.

_class: _Factor_

Mixin class for factorization routines. It implements the core, lower-level
routines used by higher-level factorization methods.

_trial_division
--------------

def _trial_division(self, f, factors)

- Purpose
  - Performs trial division of the polynomial f by each factor in the provided
    iterable 'factors', collecting multiplicities.

- Parameters
  - f: The polynomial to factor (within the current domain).
  - factors: An iterable of factors (as polynomials or compatible objects)
    to trial-divide by.

- Returns
  - A sorted list of pairs (factor, k), where 'factor' is a factor and 'k' its
    multiplicity in f.

- Notes
  - Uses repeated division by each factor until a non-zero remainder is found.

- Example
  - Given f and a list of candidate factors, returns their multiplicities
    after straightforward division.

_aa_factor_trager
--------------

def _aa_factor_trager(self, f)

- Purpose
  - Factor multivariate polynomials over algebraic number fields using a
    Trager-style approach.

- Parameters
  - f: The polynomial to factor (multivariate).

- Returns
  - A pair (lc, factors), where 'lc' is the leading coefficient of the input
    polynomial when made monic, and 'factors' is the list of irreducible factors
    (each paired with its multiplicity after trial-division normalization).

- Process (highlights)
  - Obtains the leading coefficient and makes the polynomial monic.
  - If the polynomial is ground (no variables), returns with no factors.
  - Splits into square-free part, computes a norm, and factors the square-free
    part. If the square-free factorization yields multiple irreducibles, lifts
    factors via cofactorization and a shift corresponding to a unit in the base
    domain.

- Notes
  - Relies on domain handling, square-free factorization (sqf_part, sqf_norm),
    and cofactor computation to assemble the final factor list.

_zz_factor
----------

def _zz_factor(self, f)

- Purpose
  - Complete factorization of a polynomial with integer coefficients (in Z[X]).
    Handles both univariate and multivariate cases, returning a content and a
    factorization into irreducibles over Z.

- Parameters
  - f: The polynomial to factor. May be multivariate; for univariate inputs, the
    routine reduces the problem to primitive, square-free univariate factorization.

- Returns
  - A tuple (content, factors), where 'content' is the content of f and
    'factors' is a list of (factor, multiplicity) tuples.

- Behavior (highlights)
  - For univariate polynomials:
    - Reduces to primitive form, handles degree 0/1 cases directly.
    - Optionally uses irreducibility checks and cyclotomic reduction as per
      configuration flags (USE_IRREDUCIBLE_IN_FACTOR, USE_CYCLOTOMIC_FACTOR).
    - Performs Zassenhaus/EEZ-based factorization and applies trial division to
      recover multiplicities.
  - For multivariate polynomials:
    - Extracts primitive part and the content, reduces to a univariate
      drive via substitution of generators, and factors incrementally.
    - Uses Zassenhaus-based factorization on the univariate projection and then
      propagates factors back into the multivariate ring.

- Notes
  - Integrates with the overall factorization workflow, including primitive
    decomposition and ring extension handling.

_zz_mignotte_bound
------------------

def _zz_mignotte_bound(self, f)

- Purpose
  - Computes the Mignotte bound for multivariate polynomials in Z[X], used to
    bound possible factor coefficients during Zassenhaus lifting.

- Parameters
  - f: The polynomial for which the bound is computed.

- Returns
  - A numeric bound in the domain (as a field element) representing the
    maximum coefficient size to consider during lifting.

- Details
  - Uses a = max_norm of f, b = |LC|, and n = total sum of degrees over all
    generators. The bound equals sqrt(n+1) * 2^n * a * b in the domain.

_zz_zassenhaus
--------------

def _zz_zassenhaus(self, f)

- Purpose
  - Factor primitive square-free polynomials in Z[x] (univariate Z[x] focus).

- Preconditions
  - Assumes the polynomial is univariate and that the domain is the Integer Ring.

- Returns
  - A list of irreducible factors of f over Z, possibly with multiplicities
    tracked via subsequent processing.

- Algorithm overview
  - Computes degrees and initial constants (leading coefficient, norm, etc.).
  - Selects a prime p such that f mod p is square-free; tests a few candidate
    primes to minimize the number of factors when reduced modulo p.
  - Maps the problem to a finite field Z_p and factors there, collecting a
    candidate list of irreducible factors modulo p.
  - Performs Hensel lifting from Z_p to Z using an appropriate l and a lifted
    factor set G representing the product of candidate factors.
  - Uses combinatorial subset lifting to reconstruct the actual factorization over Z.
  - Returns the list of factors after assembling them from lift results.

- Notes
  - This implementation follows classical Zassenhaus lifting with optional
    enhancements (EEZ) and uses a bounded search guided by the Mignotte bound.

Usage and dependencies
----------------------

- Requires a mathematical domain abstraction (domain) that defines:
  - Integer ring, finite fields, polynomial rings, and units.
  - Operations such as primitive(), monic(), sqf_part(), sqf_norm(),
    factor_list(), is_squarefree, and compose/eject/inject helpers.
- Relies on external factorization primitives:
  - factorint, isprime, nextprime, and symmetric_residue from the ntheory
    package.
  - Circular/cofactor machinery via self.cofactors and related domain-aware
    polynomial operations.
- Error handling and utility helpers:
  - polyerrors: CoercionFailedError, DomainError, EvaluationFailedError,
    ExtraneousFactorsError.
  - polyutils._sort_factors to order factor lists consistently.

Examples
--------

The module documents a standard Zassenhaus-based factorization flow via its
docstrings. A representative example can be found in the _zz_zassenhaus method
docstring, showing the behavior on simple univariate Z[x] polynomials:

  >>> _, x = ring('x', ZZ)
  >>> (2*x**4 - 2).factor_list()
  (2, [(x - 1, 1), (x + 1, 1), (x**2 + 1, 1)])

This demonstrates content extraction and factorization into irreducibles over Z.

Notes
-----

- The factorization strategies implemented here are designed for characteristic
  zero domains and integrate with the broader factoring framework in the
  codebase.
- The configuration-driven behavior (e.g., USE_IRREDUCIBLE_IN_FACTOR,
  USE_CYCLOTOMIC_FACTOR) allows tuning of which factorization strategies are
  attempted under different circumstances.
- The algorithms reflect classical methods (Zassenhaus, EEZ, Trager’s algebraic
  number field approach) and modern optimizations, including lifting and
  trial-division to recover multiplicities.

References

  especially when introducing new configuration flags or alternative lifting
  strategies.
