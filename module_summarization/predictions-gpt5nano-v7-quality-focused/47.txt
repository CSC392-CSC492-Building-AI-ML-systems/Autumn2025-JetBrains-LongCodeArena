Contingency Tables: Two-Way Analysis
====================================

Overview
--------

This module provides tools for analyzing two-way contingency tables (cross-classifications of two categorical variables). It focuses on computing common tests of association and related statistics for nominal and ordinal data.

Public API
----------

Key components

- _make_df_square(table): Internal helper that reindexes a pandas DataFrame to be square by aligning row and column indices and filling missing cells with zeros. If the input is not a DataFrame, it returns the input unchanged.

- _Bunch: Lightweight container used to hold results from statistical tests. It implements printable representations via __repr__ and __str__.

- Table: Core class for a two-way contingency table. It supports construction from an existing table or from raw data, optional zero-shift handling, and tests for nominal and ordinal association.

  - __init__(table, shift_zeros=True)
    - table: array-like contingency table.
    - shift_zeros: if True and any cell is zero, adds 0.5 to all values to stabilize computations.

  - from_data(data, shift_zeros=True) (classmethod)
    - Builds a Table from raw data by constructing a contingency table from the first two columns (using a pandas Crosstab).
    - Returns a Table instance.

  - __str__(): Produces a readable description including the table shape and contents.

  - test_nominal_association()
    - Assesses independence between rows and columns treating them as nominal factors.
    - Performs a chi-squared test of independence.
    - Returns a _Bunch with:
      - statistic: chi-square statistic
      - df: degrees of freedom
      - pvalue: p-value

  - test_ordinal_association(row_scores=None, col_scores=None)
    - Assesses independence for ordinal factors using a linear-by-linear (trend) test.
    - row_scores: numeric scores for the row categories (default: 0,1,2,...)
    - col_scores: numeric scores for the column categories (default: 0,1,2,...)
    - Validates lengths of score vectors against table dimensions.
    - statistic: test statistic (weighted by provided scores)
    - null_mean, null_sd: null distribution mean and standard deviation
    - zscore: Z-score for the statistic
    - pvalue: p-value for the test
    - Notes: Default scores yield the Cochran-Armitage trend test.

Notes and Details
-----------------

- Data handling
  - If shift_zeros is True and the table contains zeros, the implementation adds 0.5 to all cells to avoid zero-count complications in chi-square-based procedures.
  - from_data uses pandas.crosstab to construct the contingency table from two categorical variables.

- Inference foundation
  - The nominal test uses Pearson’s chi-squared framework.
  - The ordinal test targets linear-by-linear association by incorporating scores for ordered categories.

- Related utilities
  - The module mentions additional functions for McNemar's test and Cochran's q test (not shown in the excerpt). These provide specialized tests for paired data and k-sample kappa-like analyses, respectively.

Examples
--------

Example 1: Build a table from raw data and run a nominal association test

.. code-block:: python

  from statsmodels.stats.contingency_tables import Table
  import pandas as pd

  # Example data: two categorical variables
  data = pd.DataFrame({
      'var1': ['A', 'A', 'B', 'A', 'B', 'B', 'A'],
      'var2': ['X', 'Y', 'X', 'X', 'Y', 'Y', 'Z']
  })

  t = Table.from_data(data)
  print(t)

  result_nominal = t.test_nominal_association()
  print("Chi-square statistic:", result_nominal.statistic)
  print("df:", result_nominal.df)
  print("p-value:", result_nominal.pvalue)

Example 2: Ordinal association test with custom scores

.. code-block:: python

  # Using the same table as above
  row_scores = [1, 2]      # scores for the two rows (A, B)
  col_scores = [1, 2, 3]   # scores for the three columns (X, Y, Z)

  result_ordinal = t.test_ordinal_association(row_scores=row_scores, col_scores=col_scores)
  print("Ordinal test statistic:", result_ordinal.statistic)
  print("Null mean:", result_ordinal.null_mean)
  print("Null sd:", result_ordinal.null_sd)
  print("Z-score:", result_ordinal.zscore)
  print("p-value:", result_ordinal.pvalue)

Internal helpers and representation
-----------------------------------

- _make_df_square(table) ensures that any DataFrame-based table is square, which is often needed for certain analyses that assume equal class sets for rows and columns.
- _Bunch provides a simple, print-friendly container for test results.

Notes
-----

- The statistical procedures rely on the assumption that observations are independent and identically distributed, unless otherwise noted by a specific test.
- The public API is designed to be straightforward for constructing tables from data and performing common association tests.
