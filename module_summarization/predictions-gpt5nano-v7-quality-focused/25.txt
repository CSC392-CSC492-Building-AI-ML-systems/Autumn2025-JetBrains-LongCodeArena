w_ipa.rst
==========

Overview
--------
w_ipa is the WESTPA Interactive Python Analysis tool. It provides a stateful, in-session interface to the current analysis scheme and iteration. You access all relevant data through the w or west objects, inspect and manipulate the current analysis state, and run traces to inspect walker histories.

Key concepts
------------
- The interface is stateful: data you access comes from the current analysis scheme and iteration.
- By default you operate on the first analysis scheme in west.cfg and at iteration 1.
- All important properties are accessible via w (the w-namespace) or west (the global namespace).
- You can run a trace for a specific seg_id using w.trace(seg_id); traces can be costly, so store them if you reuse them.

Getting started
---------------
Start with an interactive session and inspect common properties:

- Access current iteration and scheme
  - w.iteration
  - west.iteration
  - To set: w.iteration = 100
- List and select analysis schemes
  - w.list_schemes
  - w.scheme = <scheme_index_or_name>
- Inspect states and bins for the current scheme
  - w.states
  - w.bin_labels
- Inspect current iteration data
  - w.current  (provides walkers, summary, states, seg_id, weights, parents, kinavg, pcoord, bins, populations, and auxdata when present)

Running a trace
---------------
- Use w.trace(seg_id) to run a trace over a segment in the current iteration.
- The function returns a dictionary with all relevant history data for that seg_id; traces can be expensive, so consider saving the result if you reuse it.

Examples
--------
- Inspect core properties
  - w.iteration
  - w.scheme
  - w.states
  - w.bin_labels

- Change iteration and scheme
  - w.iteration = 42
  - w.scheme = 0  # or a scheme name if supported

- Run a trace
  - trace_info = w.trace(123)

- View current iteration data
  - current = w.current
  - walkers = current.walkers
  - weights = current.weights

CLI/runtime options
-------------------
w_ipa accepts several runtime options to control behavior and debugging:

- --analysis-only, -ao
  - Run the analysis and return to the terminal.
- --reanalyze, -ra
  - Delete existing files and reanalyze from scratch.
- --ignore-hash, -ih
  - Ignore the stored argument hash and do not regenerate files.
- --debug, -d
  - Enable verbose debugging output for development.
- --terminal, -t
  - Plot output in the terminal (text interface).
- -f, --extra
  - Internal placeholder option; use only as needed by environments.

Interface and configuration notes
---------------------------------
- The default interface is matplotlib unless you enable terminal plotting.
- The current iteration and analysis scheme are determined from west.cfg settings unless overridden by w.
- The tool is designed to be extended and used in scripts via its properties and methods; use help on any function or property for more details.

Internal helpers (advanced)
---------------------------
- hash_args(args, extra=None, path=None)
  - Creates a unique hash identifying a run based on arguments. Useful for reproducibility checks.
- stamp_hash(h5file_name, new_hash)
  - Writes a stamp to an HDF5 file to reflect the argument hash. Intended for tracking and consistency.

Notes
-----
- Use help on any property or method for detailed information.
- Traces are potentially expensive; store results if you plan to reuse them.
