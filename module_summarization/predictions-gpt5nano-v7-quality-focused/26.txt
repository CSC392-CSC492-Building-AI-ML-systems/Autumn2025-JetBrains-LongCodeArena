w_kinavg
========

Deprecation notice
------------------
.. warning::
   w_kinavg is being deprecated. Please use w_direct instead.

Overview
--------
The w_kinavg command provides averages and confidence intervals (CIs) for
path-tracing kinetics analysis. It is a thin wrapper around the DKinAvg tool
from westpa.cli.tools.w_direct and is invoked via a master command
WDirect. The default subcommand is trace, and the command operates on
previously generated data files (assignments and kinetics data).

- Input data: assignments.h5 (bin assignments) and kinetics data (kintrace.h5 or kinmat.h5)
  generated by prior steps (e.g., w_assign and w_kinetics).
- Defaults:
  - Kinetics file: kintrace.h5
  - Output file: kinavg.h5

Usage
-----
w_kinavg [options]

Notes:
- If you invoke w_kinavg with no subcommand, the wrapper inserts the trace subcommand
  automatically (see entry point in the source). You can also call it explicitly as:
  w_kinavg trace [options]

- Options are inherited from DKinAvg (via the w_direct interface). Typical options include
  controls for input/output files and options related to evolution-mode and windowing.

Output format
-------------
The output file (default kinavg.h5) contains the following datasets:

- /avg_rates [state, state]
  Structured dataset: state-to-state rates computed over the entire selected window.

In trace mode, additional datasets appear:

- /avg_total_fluxes [state]
  Structured dataset: total flux into each state over the selected window.

- /avg_conditional_fluxes [state, state]
  Structured dataset: state-to-state fluxes over the selected window.

If you specify --evolution-mode, the following dataset is available:

- /rate_evolution [window, state, state]
  Structured dataset: state-to-state rates computed over windows of varying width.
  If evolution mode is cumulative, windows begin at --start-iter and grow by --step-iter
  for each successive element. If evolution mode is blocked, windows all have width
  --step-iter (last may be shorter), with the first window starting at --start-iter.

If --evolution-mode is specified in trace mode, additional datasets are available:

- /target_flux_evolution [window, state]
  Structured: total flux into a given macro state using windows of varying width.

- /conditional_flux_evolution [window, state, state]
  Structured: state-to-state fluxes using windows of varying width.

Dataset structure (applies to the datasets above)
-------------------------------------------------
Each windowed dataset contains:
- iter_start
  Integer: iteration at which the averaging window begins (inclusive).

- iter_stop
  Integer: iteration at which the averaging window ends (exclusive).

- expected
  Floating-point: expected (mean) value of the rate within this window (units: 1/tau).

- ci_lbound
  Floating-point: lower bound of the confidence interval on the rate (units: 1/tau).

- ci_ubound
  Floating-point: upper bound of the confidence interval on the rate (units: 1/tau).

- corr_len
  Integer: correlation length of the rate within this window (units: tau).

Attributes (present on each dataset)
------------------------------------
- mcbs_alpha
  Floating-point: alpha value for confidence intervals (e.g., alpha=0.05 -> 95% CI).

- mcbs_nsets
  Integer: number of bootstrap datasets used to generate confidence intervals.

- mcbs_acalpha
  Floating-point: alpha value for determining correlation lengths.

Command-line options
--------------------
The command uses the options defined by DKinAvg (via the WEST master/parallel tool
infrastructure). Key defaults to note:

- --kinetics-file: selects the kinetics data file (default kintrace.h5)
- --output: selects the output file (default kinavg.h5)

Examples
--------
- Basic usage (trace mode, default files):
  w_kinavg

- Explicitly request trace subcommand:
  w_kinavg trace

- Override input/output files:
  w_kinavg --kinetics-file kintrace.h5 --output kinavg.h5

See also
--------
- w_direct: master command wrapper that w_kinavg relies on
- w_assign: generates bin assignments
- w_kinetics: generates kinetics data (kintrace.h5 or kinmat.h5)

Notes
-----
- This command is a deprecated wrapper. Prefer using w_direct directly for future needs.
