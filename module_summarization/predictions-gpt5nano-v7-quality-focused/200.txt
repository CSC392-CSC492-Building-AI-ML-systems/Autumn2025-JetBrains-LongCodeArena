Outlier Detection and Novelty Detection
======================================

Overview
--------
Outlier detection identifies anomalous samples in a dataset, often without
label guidance. Novelty detection models are trained on “normal” data and then
identify previously unseen abnormal samples in new data. In practice, novelty
detection is used when you hold out a clean, normal dataset for training and
need to flag deviations in future observations.

Key distinctions
- Outlier detection (unsupervised): learns from the full training set and flags
  anomalies within that data or similar data. Examples include standard use of
  LOF, Isolation Forest, and Elliptic Envelope without a novelty mode.
- Novelty detection (semi-supervised): trains on normal data only, then detects
  anomalies in new, unseen data. This mode requires the estimator to support
  novelty detection (e.g., novelty=True for several algorithms).

Algorithms at a glance
----------------------
- Isolation Forest (sklearn.ensemble.IsolationForest)
  - Can operate in novelty mode. Trains on normal data and predicts anomalies in
    new data with predict/decision_function.
  - Useful for high-dimensional data and large datasets.
- Elliptic Envelope (sklearn.covariance.EllipticEnvelope)
  - Can operate in novelty mode. Fits a Gaussian-like envelope to data.
  - Works well when normal data roughly follow a Gaussian distribution.
- Local Outlier Factor (sklearn.neighbors.LocalOutlierFactor)
  - Supports novelty mode. Detects anomalies based on local density.
  - In default mode, trained on the input data itself; in novelty mode, fits on normal data
    and evaluates new samples.
- One-Class SVM (sklearn.svm.OneClassSVM)
  - Used for novelty detection by training on normal data and evaluating new data.
  - Works with various kernels and is suitable for smaller to medium datasets.
- Other notes
  - Contamination parameter expresses the expected proportion of outliers.
  - Feature scaling is often important; apply a standard scaler or robust scaler as needed.

API usage patterns
------------------
- Outlier detection (no novelty mode)
  - Fit on the dataset and predict on the same or new data.
  - Typical outputs: -1 for outliers, 1 for inliers; decision_function or score_samples
    provide continuous anomaly scores.
- Novelty detection (novelty mode)
  - Choose an estimator that supports novelty detection (e.g., Isolation Forest,
    Elliptic Envelope, Local Outlier Factor with novelty=True).
  - Fit on normal data only; predict on unseen data to flag novelties.

Worked examples
---------------
The following examples illustrate novelty detection and standard outlier detection
using common scikit-learn estimators.

Example 1: Novelty detection with Isolation Forest
```
from sklearn.ensemble import IsolationForest
import numpy as np

# Normal data for training
X_train = np.random.normal(loc=0.0, scale=1.0, size=(1000, 2))

# Unseen data contains some anomalies
X_test = np.concatenate([
    np.random.normal(loc=0.0, scale=1.0, size=(20, 2)),
    np.random.normal(loc=3.0, scale=0.5, size=(20, 2))  # anomalies
])

clf = IsolationForest(
    n_estimators=200,
    contamination=0.02,
    random_state=42,
    novelty=True,  # enables novelty detection
)
clf.fit(X_train)

# Predict on unseen data
preds = clf.predict(X_test)          # 1 for inliers, -1 for outliers
scores = clf.decision_function(X_test)  # higher scores = more normal
```

Example 2: Novelty detection with Elliptic Envelope
```
from sklearn.covariance import EllipticEnvelope
import numpy as np

X_train = np.random.normal(size=(1000, 2))
X_test = np.random.normal(size=(40, 2))

env = EllipticEnvelope(contamination=0.02, random_state=42, novelty=True)
env.fit(X_train)

preds = env.predict(X_test)          # 1 for inliers, -1 for outliers
scores = -env.mahalanobis(X_test)     # optional score proxy (if available)
```

Example 3: Standard outlier detection with Local Outlier Factor (no novelty)
```
from sklearn.neighbors import LocalOutlierFactor
import numpy as np

X = np.random.normal(size=(1000, 2))

lof = LocalOutlierFactor(n_neighbors=20)
labels = lof.fit_predict(X)  # -1 for outliers, 1 for inliers
```

Example 4: Novelty detection with Local Outlier Factor (novelty mode)
```
from sklearn.neighbors import LocalOutlierFactor
import numpy as np

# Train on normal data only
X_train = np.random.normal(size=(1000, 2))

X_test = np.random.normal(size=(40, 2))

lof = LocalOutlierFactor(n_neighbors=20, novelty=True, contamination=0.02)
lof.fit(X_train)

preds = lof.predict(X_test)
```

Evaluation and metrics
----------------------
- ROC AUC: useful when you have continuous anomaly scores.
- Precision/Recall, F1: relevant when you know the contamination level.
- For novelty detection, focus on performance on unseen data; calibrate using
  a held-out normal/abnormal split if available.

Best practices
--------------
- Scale features when algorithms assume standardized inputs (e.g., SVM-based
  methods).
- Set contamination to reflect the expected anomaly rate in your domain.
- Prefer novelty-enabled models when you must detect anomalies in future data
  after training on normal data only.
- Use visualization (e.g., 2D projections) to sanity-check the separation between
  inliers and outliers.

Notes
-----
- The examples assume a basic familiarity with scikit-learn estimators and
  fit/predict workflows.
- For large-scale data, prefer algorithms with good scalability (e.g., Isolation
  Forest) and avoid overly complex kernels without sufficient data.
