w_trace
=======

Overview
--------

w_trace traces a trajectory back to its origin by assembling the sequence
of segments that lead to a specified last segment. It builds a Trace object
from the West data manager and exposes the per-segment summary, initial state,
and basis state information, along with utilities to fetch segment data and
timepoint data along the trace.

Key concepts
------------

- Trace
  - Represents a trajectory back to its origin.
  - Contains a per-segment summary, the endpoint type, the basis state, the
    initial state, and a data manager reference.
- Summary
  - A NumPy structured array with one row per segment in the traced history.
  - Fields include: n_iter, seg_id, weight, walltime, cputime, final_pcoord.
- Initial state / basis state
  - Initial state is the trajectory’s state at the first segment.
  - Basis state is the underlying state from which the trajectory starts.

Building a trace
----------------

The primary entry point is Trace.from_data_manager(n_iter, seg_id, data_manager=None).

- n_iter
  - The iteration number of the last segment in the trace.
- seg_id
  - The segment identifier of the last segment in the trace.
- data_manager
  - Optional data manager. If omitted, the global West data manager is used.

Example
-------

from westpa.core.data_manager import weight_dtype, n_iter_dtype
from westpa.tools import WESTDataReader
from westpa.trace import Trace  # hypothetical import path; adapt to your package

# Build a trace for the last segment identified by (n_iter, seg_id)
trace = Trace.from_data_manager(n_iter=100, seg_id=42)

# Inspect the summary
print("Endpoint type:", trace.endpoint_type)
print("Number of segments in trace:", len(trace))
for row in trace:
    print(row['n_iter'], row['seg_id'], row['weight'], row['final_pcoord'])

# Access the initial and basis state
initial = trace.initial_state
basis = trace.basis_state

Accessing trace data
--------------------

Trace.summary
- The summary is a NumPy structured array with the following fields:
  - n_iter: iteration number (n_iter_dtype)
  - seg_id: segment identifier (seg_id_dtype)
  - weight: trajectory weight (weight_dtype)
  - walltime: wall clock time (utime_dtype)
  - cputime: CPU time (utime_dtype)
  - final_pcoord: final pcoord vector for the segment (pcoord_dtype, shape pcoord_pt_shape)

- The per-row data corresponds to segments along the traced history, in order
  from the initial segment to the last segment.

Initial and basis state
- initial_state: object representing the trajectory’s initial state. Retrieved
  from the data manager when available; falls back to an older HDF5 path if needed.
- basis_state: the basis state corresponding to the initial state. May be None
  for older HDF5 data layouts.

Fetching segment data slices
---------------------------

Trace.get_segment_data_slice(datafile, dsname, n_iter, seg_id, slice_=None, index_data=None, iter_prec=None)

- datafile
  - Open h5py.File object representing the West data file.
- dsname
  - Dataset name within the appropriate iteration group.
- n_iter, seg_id
  - Target iteration and segment for the slice.
- slice_
  - Optional slice tuple to further subset the dataset (e.g., timepoints).
- index_data
  - Optional iterable of (n_iter, seg_id) pairs. If provided, the function
    locates the matching pair and returns the corresponding data.
- iter_prec
  - Optional iteration precision (number of digits) for the iteration group
    naming. If omitted, the data manager’s default_iter_prec is used.

Behavior
- If index_data is provided, the function searches for the matching (n_iter, seg_id)
  pair within index_data and uses that position to index the dataset.
- If index_data is not provided, the function derives the iteration group name
  (e.g., iter_0000000100 with the appropriate precision) and fetches
  dataset[seg_id, ...] at that path. The default behavior uses the data
  manager’s default_iter_prec unless overridden.

Timepoint tracing along a trajectory
------------------------------------

Trace.trace_timepoint_dataset(dsname, slice_=None, auxfile=None, index_ds=None)

- dsname
  - Dataset name laid out as [seg_id][timepoint][...].
- slice_
  - Optional slice to apply to the timepoint data.
- auxfile
  - Optional auxiliary file handle or name if the dataset resides in an aux file.
- index_ds
  - Optional dataset name used to index into the trace (for advanced indexing).

Return value
- A tuple (data_trace, weight), where:
  - data_trace is a time series of the requested dataset along the trajectory.
  - weight is the corresponding trajectory weight at each time point.

Notes on data formats
---------------------

- The code handles both newer and older HDF5 data formats.
- For older formats, initial state and parent information may be obtained
  via alternative paths (e.g., parents dataset). The Trace.from_data_manager
  method contains explicit fallbacks to maintain compatibility.

Formatting and data types
-------------------------

- The summary dtype is constructed with fields:
  - n_iter (n_iter_dtype)
  - seg_id (seg_id_dtype)
  - weight (weight_dtype)
  - walltime (utime_dtype)
  - cputime (utime_dtype)
  - final_pcoord (pcoord_dtype, with shape pcoord_pt_shape)
- pcoord_pt_shape derives from the pcoord dataset shape (dimensions after time
  and segment axes).

Implementation notes
--------------------

- Trace uses a cache for auxiliary files to minimize I/O overhead.
- It builds the initial segment, iterating backward from the last segment,
  collecting (n_iter, seg_id, weight, walltime, cputime, final_pcoord) for
  each step until reaching the initial state.

Related tools
-------------

- WESTTool and WESTDataReader are part of the WestPA tooling ecosystem and may be used
  to interact with data readers or to extend command-line functionality.
- The Trace class relies on the data manager to fetch segments, initial states,
  and basis states.
