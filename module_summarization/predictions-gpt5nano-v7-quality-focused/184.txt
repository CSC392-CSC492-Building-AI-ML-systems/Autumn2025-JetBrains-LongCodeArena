Gaussian Processes
==================

Overview
--------
Gaussian Processes (GPs) provide a principled, probabilistic approach to regression and classification. In this documentation, we focus on the kernel machinery that underpins GP models in this package. Kernels (covariance functions) encode assumptions about function smoothness, amplitude, and relevant length scales. They support algebraic construction and analytic gradient-based hyperparameter optimization.

Key concepts
------------
- Kernel calculus: Kernels can be combined using standard arithmetic:
  - Addition: a sum of kernels represents the sum of independent latent functions.
  - Multiplication: a product encodes interactions between kernels.
  - Exponentiation: a scalar exponentiates a kernel (e.g., k(x, x')**p).
  - Scalars in expressions are automatically converted to a constant kernel.
- Hyperparameters: Each kernel exposes a set of hyperparameters that control its behavior (e.g., length scales, amplitudes).
- Optimization: All kernels support analytic gradient-based optimization of hyperparameters. The search space is defined by lower and upper bounds for each hyperparameter.
- Fixed hyperparameters: Hyperparameters can be declared as fixed, excluding them from optimization.
- Anisotropic vs isotropic length scales: Length-scale hyperparameters can be scalar (isotropic) or a vector (anisotropic) with a per-dimension interpretation.

Kernel API and hyperparameters
------------------------------
- Hyperparameter: Describes a single hyperparameter with fields:
  - name: parameter name (e.g., "length_scale").
  - value_type: typically "numeric".
  - bounds: [lower, upper] bounds or the string "fixed" (fixed value cannot be optimized).
  - n_elements: number of elements (1 for scalar, >1 for vector-valued parameters).
  - fixed: whether the hyperparameter is fixed (defaults to derived behavior from bounds).
- Kernel: Base class for all kernels. Kernels expose parameters via their signature and allow deep inspection of nested estimators via get_params.

Implementation notes
--------------------
- _check_length_scale(X, length_scale): Validates the length scale for anisotropic kernels. It ensures:
  - length_scale is 1D and not multi-dimensional.
  - If anisotropic (vector) length_scale is provided, its length matches the data dimensionality (X.shape[1]).

Practical usage
---------------
- Building a composite kernel:
  - Use +, *, and ** to combine kernels.
  - Scalars in the expression convert to a constant kernel automatically.
- Specifying hyperparameters:
  - Bounds define the rectangular search space for optimization.
  - Use "fixed" to lock certain hyperparameters.
- Accessing hyperparameters:
  - Each kernel provides a hyperparameters attribute (a collection of Hyperparameter instances) describing the optimization space.

Example
-------
Below demonstrates building a composite kernel, inspecting hyperparameters, and using a kernel with a Gaussian Process Regressor.

Code examples (conceptual; adjust imports to your environment):

- Create a composite kernel with bounds
  from sklearn.gaussian_process.kernels import RBF, ConstantKernel
  # Constant kernel with bounds, multiplied by an RBF kernel
  kernel = ConstantKernel(1.0, bound=(0.0, 10.0)) * RBF(length_scale=1.0, length_scale_bounds=(1e-2, 1e2))

- Inspect hyperparameters
  for hp in kernel.hyperparameters:
      print(hp)

- Use with a GaussianProcessRegressor
  from sklearn.gaussian_process import GaussianProcessRegressor
  gpr = GaussianProcessRegressor(kernel=kernel, alpha=1e-6, normalize_y=True)

- Fit and predict
  gpr.fit(X_train, y_train)
  y_mean, y_std = gpr.predict(X_test, return_std=True)

Extending kernels
-----------------
- To introduce new kernels, subclass Kernel and implement the required interface.
- Ensure hyperparameters are exposed via the kernel’s constructor signature so they can be discovered by get_params and hyperparameter inspection.
- Leverage the Hyperparameter structure to define bounds and fixed parameters for analytic gradient optimization.

Notes
-----
- The kernel framework is designed to be compatible with scikit-learn’s estimator conventions, enabling integration into pipelines and hyperparameter tuning workflows.
- The documentation here reflects the kernel module for Gaussian process regression and classification, including the described algebra, hyperparameter machinery, and optimization capabilities.
