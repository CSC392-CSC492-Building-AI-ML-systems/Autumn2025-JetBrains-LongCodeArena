# Model usage

This section covers how to define and use models in your project. It focuses on how to declare models, configure metadata, and work with common field types and relationships.

## Defining a model

Models are created by subclassing the base Model class and declaring fields as class attributes. Each field maps to a database column. Use the appropriate Field subclass for the data type.

Example:

```python
from tortoise import fields, models

class Group(models.Model):
    id = fields.IntField(pk=True)
    name = fields.CharField(max_length=100)
```

- pk=True marks the field as the primary key.
- Use suitable field types for your data (IntField, CharField, DatetimeField, etc.).

## Meta configuration

The inner `Meta` class configures how a model maps to the database and how it behaves in queries.

Common options:
- `table` (str): database table name.
- `app` (Optional[str]): app label used for migrations and organization.
- `abstract` (bool): if true, the model is not instantiated as a database table.
- `unique_together` (Tuple[Tuple[str, ...], ...]): composite unique constraints.
- `indexes` (Tuple[Tuple[str, ...], ...]): database indexes.
- `ordering` (Tuple[str, ...]): default ordering for queries (strings are resolved to (field, Order) pairs).

Example:

```python
class User(models.Model):
    id = fields.IntField(pk=True)
    username = fields.CharField(max_length=50)
    created_at = fields.DatetimeField(auto_now_add=True)

    class Meta:
        table = "users"
        app = "accounts"
        unique_together = (("username",),)
        indexes = (("created_at",),)
        ordering = ("-created_at", "username")
```

Notes:
- Ordering strings are resolved to a pair of (field_name, Order) using internal utilities. A leading minus sign indicates descending order.

## Fields and basic types

Use field classes to declare column types. Common ones include:
- `IntField`, `BigIntField`
- `CharField`, `TextField`
- `DatetimeField`, `DateField`, `TimeField`
- `BooleanField`, `FloatField`, `DecimalField`

Example:

```python
class Product(models.Model):
    id = fields.IntField(pk=True)
    name = fields.CharField(max_length=200)
    price = fields.DecimalField(max_digits=10, decimal_places=2)
    in_stock = fields.BooleanField(default=True)
```

## Relationships

Model relationships use dedicated field types to express foreign keys, one-to-one, and many-to-many associations.

- Foreign key
  - Declared with `ForeignKeyField` and a related name for reverse queries.
- One-to-one
  - Declared with `OneToOneField` or equivalent.
- Many-to-many
  - Declared with `ManyToManyField` and an optional related name.

Example:

```python
class Group(models.Model):
    id = fields.IntField(pk=True)
    name = fields.CharField(max_length=100)

class User(models.Model):
    id = fields.IntField(pk=True)
    username = fields.CharField(max_length=50)
    group = fields.ForeignKeyField('models.Group', related_name='users')
    groups = fields.ManyToManyField('models.Group', related_name='members')
```

Notes:
- Reverse relations and placeholder relation objects exist internally (e.g., `BackwardFKRelation`, `ReverseRelation`). These are typically managed automatically and used by the ORM to resolve related entities.

## Field comments and documentation

The system can extract comments from source to document fields. Format comments using a public, multi-line-friendly style that the extractor recognizes:

```python
class User(models.Model):
    #: The user’s login name
    username = fields.CharField(max_length=50)

    #: The user’s email address
    email = fields.CharField(max_length=255, null=True)
```

- The placeholder `{model}` in comments is replaced with the model class name during extraction.
- Comments must use the `#:` format (public) or a multiline sequence preceding the field declaration.

## Default and custom ordering

Default ordering is configured in the model’s `Meta` via the `ordering` attribute. Each entry is resolved to a tuple of `(field_name, Order)` where `Order` is typically `asc` or `desc` (as determined by the parser).

Example:

```python
class Article(models.Model):
    id = fields.IntField(pk=True)
    title = fields.CharField(max_length=200)
    published_at = fields.DatetimeField()

    class Meta:
        ordering = ("-published_at", "title")
```

This sets the default order to newest first by `published_at`, then by `title`.

## Internal metadata handling (overview)

The ORM maintains internal metadata through structures like `MetaInfo`. It collects:
- field sets (including primary keys, foreign keys, M2M fields)
- table mappings and DB fields
- index and unique constraints
- default ordering (and validation state)

These internals drive query construction, joins, and relationship resolution. As a user, you interact with the model API and do not usually manipulate these internals directly.

## Basic usage patterns

- Create and save a model instance
  - Instantiate with field values and call `save()` (asynchronously if using the async backend).
- Querying
  - Use the provided query API to filter, order, and fetch results.
  - Examples assume the ORM’s standard asynchronous API.

```python
# Create
user = User(username="alice", email="alice@example.com")
await user.save()

# Query
user_qs = User.filter(username="alice")
user_obj = await user_qs.first()

# Access related objects
group = await user_obj.group  # foreign key relation
user_groups = await user_obj.groups.all()  # many-to-many relation
```

Note: The exact syntax for queries may vary slightly with the ORM version, but the concepts (filters, awaits for async operations, and relation resolution) remain consistent.
