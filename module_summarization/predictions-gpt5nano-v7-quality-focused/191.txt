Validation curves and learning curves
=====================================

Overview
--------

Validation curves plot scores as a function of a hyperparameter (e.g., regularization strength) or as the training size, enabling you to assess model performance across cross-validated folds. Learning curves specifically track how scores evolve with increasing training set size. Both visualizations help diagnose bias, variance, and data sufficiency.

Key concepts
------------

- Score data shape: train_scores and test_scores have shape (n_points, n_folds). The plot shows the mean across folds (axis=1) and optionally the standard deviation.
- Plot styles for the variability:
  - fill_between: shows mean ± std with shaded area.
  - errorbar: shows mean with error bars representing std.
  - None: only the mean lines are drawn.
- Curve selection (score_type):
  - train: plot training scores only.
  - test: plot test/validation scores only.
  - both: plot both training and test scores.
- Negating scores: negate_score flips the sign, useful when your scoring metric is inverted (e.g., error) and you want higher is better.
- Axis labeling: score_name labels the y-axis. It can be inferred from the scoring parameter, or overridden via the score_name argument.
- Axis scales: the x-axis scale is chosen automatically unless you explicitly pass the deprecated log_scale parameter. The algorithm selects an appropriate scale (linear, log, or symlog) based on the spacing of x_data.

Public API
----------

- LearningCurveDisplay
  - Visualizes learning curves (scores vs. training sizes).
  - Common usage: create a display via LearningCurveDisplay.from_estimator(...) or LearningCurveDisplay.from_learning_curve(...) and render with .plot().
  - Attributes:
    - ax_: Matplotlib Axes containing the plot.
    - figure_: Matplotlib Figure containing the plot.
    - train_scores, test_scores: raw data used to compute means and stds.
    - lines_, errorbar_, fill_between_: handles for plotted artists depending on std_display_style.
    - score_name: label used for the y-axis (derived from scoring or overridden).
- _BaseCurveDisplay (internal)
  - _plot_curve(...) handles the actual drawing logic, including:
    - Validation of std_display_style ("errorbar", "fill_between", or None).
    - Selection of which curves to plot based on score_type ("train", "test", "both").
    - Plotting of mean scores and optional dispersion (std) according to std_display_style.
    - Handling of negate_score and score_name.
    - Automatic x-axis scaling and y-axis labeling.

Usage examples
--------------

Learning curve (scores vs. training size)

.. code-block:: python

    import numpy as np
    import matplotlib.pyplot as plt
    from sklearn.model_selection import learning_curve
    from sklearn.linear_model import LogisticRegression
    from sklearn.datasets import load_digits
    from sklearn.model_selection import LearningCurveDisplay

    X, y = load_digits(return_X_y=True)
    estimator = LogisticRegression(max_iter=1000)

    train_sizes, train_scores, test_scores = learning_curve(
        estimator, X, y, train_sizes=np.linspace(0.1, 1.0, 5), cv=5, scoring="accuracy"
    )

    disp = LearningCurveDisplay.from_estimator(
        estimator, X, y, train_sizes=np.linspace(0.1, 1.0, 5),
        cv=5, scoring="accuracy"
    )
    disp.plot()  # or disp.plot(std_display_style="fill_between")
    plt.show()

Validation curve (scores vs. a hyperparameter)

.. code-block:: python

    import numpy as np
    from sklearn.model_selection import validation_curve
    from sklearn.svm import SVC
    from sklearn.datasets import load_digits
    from sklearn.model_selection import ValidationCurveDisplay
    import matplotlib.pyplot as plt

    X, y = load_digits(return_X_y=True)
    param_range = np.logspace(-3, 3, 7)

    train_scores, test_scores = validation_curve(
        SVC(), X, y, param_name="C", param_range=param_range,
        cv=5, scoring="accuracy"
    )

    disp = ValidationCurveDisplay.from_estimator(
        SVC(), X, y, param_name="C", param_range=param_range,
        cv=5, scoring="accuracy"
    )
    disp.plot()
    plt.show()

Customization options
---------------------

- std_display_style:
  - "fill_between" (default for many plots): shows mean ± std as a shaded region.
  - "errorbar": shows mean with error bars representing std.
  - None: plots only the mean lines without dispersion shading or error bars.
- score_type:
  - "train": plot training scores.
  - "test": plot test/validation scores.
  - "both": plot both training and test scores.
- negate_score:
  - Set to True to flip the sign of both train and test scores.
- score_name:
  - Override the y-axis label. If not provided, it derives from the scoring parameter.
- log_scale (deprecated):
  - This parameter is deprecated and will be removed in a future release. Use axis methods to adjust scales instead. The code falls back to automatic x-axis scaling if not provided.
- Axis scaling:
  - The x-axis scale is chosen automatically to reflect the spacing of x_data (e.g., training sizes or hyperparameter values).

Internal behavior notes
-----------------------

- The plot uses the mean across cross-validation folds (score.mean(axis=1)) for each x_data point.
- Standard deviation is computed as score.std(axis=1) and used according to the chosen dispersion style.
- The x-axis scale is determined by inspecting the interval ratio of x_data; the implementation prefers log or symlog when appropriate, otherwise linear.
- The y-axis label is set to the chosen score_name, providing a clear description of what is being plotted.

Notes
-----

- Matplotlib is required for plotting. The plotting routines check for Matplotlib support and raise a helpful error if it is unavailable.
- The API supports both learning curves and validation curves through a common display class hierarchy, enabling consistent styling and interaction.
