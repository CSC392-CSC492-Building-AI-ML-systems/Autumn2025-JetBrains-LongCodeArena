Calculator Interface
====================

Overview
--------

The Calculator Interface provides a command-line utility to discover which parameters are used across all models in the SAS Models suite. The primary entry point is the sasmodels.list_pars module, which collects kernel-level parameters from every model and presents them either as a simple list or with the corresponding models that use each parameter.

Key features:
- Aggregates all parameter names across models
- Optional filtering by parameter kind (volume, orientation, sld)
- Quiet (names only) or verbose (names with model lists) output

Usage
-----

- Basic usage (names only):
  python -m sasmodels.list_pars

- Verbose usage (include models that use each parameter):
  python -m sasmodels.list_pars -v

- Filter by parameter kind (optional):
  python -m sasmodels.list_pars volume
  python -m sasmodels.list_pars orientation
  python -m sasmodels.list_pars sld

Notes:
- The positional argument kind accepts: volume, orientation, sld, none, or any.
- In the implementation, any or none map to no filtering; only volume/orientation/sld restrict the results.

CLI Arguments
-------------

- -v, --verbose
  List the models that use each parameter in addition to the parameter name.

- kind (positional)
  Optional filter for parameter kind. Accepts:
  - volume
  - orientation
  - sld
  - none
  - any

  Behavior:
  - volume, orientation, sld filter to parameters of the specified kind.
  - none or any apply no filtering (show all parameters).

Output formats
--------------

- Non-verbose output (names_only = True):
  A single column of parameter names, sorted alphabetically.

- Verbose output (names_only = False):
  One line per parameter, formatted as:
  parameter_name: model1, model2, ...

Implementation details
----------------------

- Core data
  - The tool uses list_models() to enumerate models.
  - For each model, load_model_info(name) accesses model_info.parameters.kernel_parameters to collect parameters.

- Parameter collection
  - find_pars(kind=None) returns a dictionary:
    { parameter_name: [model, model, ...] }

- Output generation
  - list_pars(names_only=True, kind=None) prints:
    - If names_only: a sorted list of parameter names (via columnize for alignment).
    - If not names_only: parameter lines with a comma-separated list of models.

- Main entry point
  - main() wires up an argparse-based CLI and calls list_pars with appropriate arguments.

API reference
-------------

- find_pars(kind=None)
  - Description: Finds all parameters across models, optionally filtering by kind.
  - Returns: dict[str, list[str]] where keys are parameter names and values are lists of model names that use the parameter.
  - Parameters:
    - kind: None or a string indicating the parameter kind (e.g., 'volume', 'orientation', 'sld').

- list_pars(names_only=True, kind=None)
  - Description: Prints all parameters found across models.
  - Behavior:
    - If names_only is True, prints only parameter names (alphabetically).
    - If names_only is False, prints lines of the form "name: model1, model2, ..."
  - Returns: None
  - Parameters:
    - names_only: boolean to control output format.
    - kind: same filtering as find_pars.

- main()
  - Description: CLI entry point for the script.
  - Behavior: Builds and parses the command-line arguments, then invokes list_pars accordingly.

Examples
--------

- List all parameter names (names only):
  python -m sasmodels.list_pars

- List all parameter names with the models that use them:
  python -m sasmodels.list_pars -v

- List only parameters of type 'volume':
  python -m sasmodels.list_pars volume

- List parameters of type 'sld' with model usage:
  python -m sasmodels.list_pars sld

Notes
-----

- The output order for the verbose form reflects the order in which models are discovered and appended to each parameter’s model list.
- The non-verbose form presents a clean, alphabetically sorted list of parameter names for quick scanning.
