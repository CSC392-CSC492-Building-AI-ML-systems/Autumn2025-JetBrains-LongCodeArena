Random Projection transformers
==============================

This module implements random projection transformers that reduce the dimensionality of data by projecting it into a lower-dimensional space using a random matrix. The projection trades some accuracy for faster processing and smaller model sizes. The approach preserves pairwise distances with high probability, in line with the Johnson-Lindenstrauss lemma.

Theoretical background
----------------------

- Distances between any two samples are preserved within a factor of (1 ± eps) after projection:
  (1 - eps) ||u - v||^2 < ||p(u) - p(v)||^2 < (1 + eps) ||u - v||^2
  where u and v are rows from the dataset, and p is the random projection.
- The projection uses a random Gaussian matrix (or an Achlioptas-like sparse matrix) with shape (n_components, n_features).
- The required number of components n_components to achieve an eps-embedding is approximately:
  n_components ≥ 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)
- The dimensionality depends on the dataset size (n_samples) rather than the original number of features.

----------

- SparseRandomProjection
- GaussianRandomProjection
- johnson_lindenstrauss_min_dim

johnson_lindenstrauss_min_dim
--------------------------------

Find a safe, minimal number of components to randomly project to in order to achieve an eps-embedding with high probability.

Signature
- johnson_lindenstrauss_min_dim(n_samples, eps=0.1)

Parameters
- n_samples: int or array-like of int
  Number of samples that should be projected. If an array is provided, the function computes a safe number of components element-wise.
- eps: float or array-like of shape (n_components,), default 0.1
  Maximum distortion factor (0, 1). If an array is provided, the function computes a safe number of components element-wise.

Returns
- n_components: int or ndarray of int
  The minimal number of components to guarantee, with high probability, an eps-embedding for the given n_samples.

Notes
- eps must lie strictly between 0 and 1.
- The function supports array-like inputs and computes results element-wise.
- See also the references and user guide for more details.

Examples
--------

>>> from sklearn.random_projection import johnson_lindenstrauss_min_dim
>>> johnson_lindenstrauss_min_dim(1e6, eps=0.5)
663

>>> johnson_lindenstrauss_min_dim(1e6, eps=[0.5, 0.1, 0.01])
array([    663,   11841, 1112658])

>>> johnson_lindenstrauss_min_dim([1e4, 1e5, 1e6], eps=0.1)
array([ 7894,  9868, 11841])

Internal helpers
----------------

The module defines several internal utility functions used to support random projection generation. These helpers are not part of the public API but are documented for completeness.

- _check_density(density, n_features)
  - Validates and canonicalizes the density parameter for sparse projection matrices.
  - If density is "auto", it uses density = 1 / sqrt(n_features).
  - If density is not in (0, 1], it raises a ValueError.
  - Returns the density as a float.

- _check_input_size(n_components, n_features)
  - Validates that n_components > 0 and n_features > 0.
  - Raises ValueError if invalid.

- _gaussian_random_matrix(n_components, n_features, random_state=None)
  - Generates a dense Gaussian random matrix with entries drawn from N(0, 1.0 / n_components).
  - Returns a NumPy array of shape (n_components, n_features).
  - The distribution is chosen to control variance and preserve distances after projection.

Usage notes
-----------

- The public estimators (SparseRandomProjection and GaussianRandomProjection) rely on these helpers to generate the random projection matrix and to validate input sizes.
- A consistent random state is available via the module utilities to ensure reproducibility.
- The approach is suitable for large-scale dimensionality reduction where exact methods are impractical.
