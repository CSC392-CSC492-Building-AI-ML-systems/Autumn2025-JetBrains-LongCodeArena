Cross-validation: evaluating estimator performance
===============================================

This section describes how to evaluate estimator performance with cross-validation using visualizations. The visualizations help you understand how performance scales with training data and how it responds to changes in hyperparameters.

What you can visualize
-----------------------

- Learning curve
  - Shows training and validation scores as a function of the number of training samples.
  - Useful for diagnosing bias, variance, and whether adding more data helps.
  - Rendered by LearningCurveDisplay.

- Validation curve
  - Shows a chosen score as a function of a hyperparameter value.
  - Useful for hyperparameter tuning and understanding sensitivity.
  - Rendered by ValidationCurveDisplay.

Key concepts
------------

- Scores come from cross-validation splits. For each training set size (or hyperparameter value), scores are averaged across folds.
- Visualizations show mean scores and, optionally, variability (via standard deviation) across folds.
- The display API accepts different styles for representing variability:
  - fill_between: shading around the mean score (default)
  - errorbar: error bars for the standard deviation
  - None: only the mean curve (no variability visualization)
- Score name and labeling:
  - score_name controls the y-axis label.
  - score_type allows you to plot training scores, test scores, or both.
- Axis scaling:
  - The x-axis scale is chosen automatically to suit the data (log/symlog/linear) unless you opt into the deprecated log_scale parameter.

API overview
------------

- LearningCurveDisplay
  - Visualizes learning curves (train vs. test scores) as training size increases.
  - Typically constructed via from_estimator or from_records, wrapping results from sklearn.model_selection.learning_curve.

- ValidationCurveDisplay
  - Visualizes how a score changes with a hyperparameter value.
  - Typically constructed via from_estimator or from_records, wrapping results from sklearn.model_selection.validation_curve.

- Internal plotting behavior (for reference)
  - The plotting routine handles:
    - Negating scores if requested
    - Selecting which curves to plot (train, test, or both)
    - Choosing display style for variability ('fill_between', 'errorbar', or None)
    - Styling via line_kw, fill_between_kw, and errorbar_kw
    - Automatic x-axis scale selection (unless deprecated log_scale is used)
    - Legend, axis labels, and saving references to axes and figures

Usage patterns
--------------

1) Learning curve (estimate how performance grows with more data)

- Compute and plot the learning curve for a given estimator and data.

.. code-block:: python

   from sklearn.model_selection import learning_curve
   from sklearn.model_selection import LearningCurveDisplay
   from sklearn.linear_model import LogisticRegression
   from sklearn.datasets import load_iris
   import numpy as np

   X, y = load_iris(return_X_y=True)
   estimator = LogisticRegression(max_iter=200)

   train_sizes, train_scores, test_scores = learning_curve(
       estimator, X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 5),
       scoring="accuracy",
   )

   disp = LearningCurveDisplay.from_estimator(
       estimator, X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 5),
       scoring="accuracy"
   )
   disp.ax_.set_title("Learning curve (Logistic Regression on Iris)")
   disp.figure_.tight_layout()

2) Validation curve (how score varies with a hyperparameter)

- Compute and plot a validation curve for a hyperparameter.

.. code-block:: python

   from sklearn.model_selection import validation_curve
   from sklearn.model_selection import ValidationCurveDisplay
   from sklearn.svm import SVC
   from sklearn.datasets import load_wine
   import numpy as np

   X, y = load_wine(return_X_y=True)
   estimator = SVC()

   param_name = "C"
   param_range = [0.5, 1.0, 2.0, 5.0, 10.0]

   train_scores, test_scores = validation_curve(
       estimator, X, y, param_name=param_name, param_range=param_range,
       cv=5, scoring="accuracy"
   )

   disp = ValidationCurveDisplay.from_estimator(
       estimator, X, y, param_name=param_name, param_range=param_range,
       cv=5, scoring="accuracy"
   )
   disp.ax_.set_title("Validation curve for SVC on Wine dataset")
   disp.figure_.tight_layout()

3) Customize variability display

- Choose how to show score variability via std_display_style.

.. code-block:: python

   disp = LearningCurveDisplay.from_estimator(
       estimator, X, y, cv=5, train_sizes=np.linspace(0.1, 1.0, 5),
       scoring="accuracy",
       std_display_style="errorbar"  # or "fill_between" or None
   )

Options and customization
-------------------------

- std_display_style:
  - "fill_between": shading between mean - std and mean + std (default)
  - "errorbar": explicit error bars
  - None: only the mean curves are drawn
- score_type:
  - "train", "test", or "both"
- score_name:
  - Sets the y-axis label. If not provided, derived from the scoring parameter or defaults.
- line_kw, fill_between_kw, errorbar_kw:
  - Pass styling options to the respective plotting calls to customize colors, line styles, and transparency.
- log_scale (deprecated):
  - Using this parameter triggers a FutureWarning and is replaced by using axis scaling methods on the display object.

Notes
-----

- The visualizations are built on results from cross-validation (learning_curve or validation_curve).
- The plots include an automatically chosen x-axis scale to best represent the data, with a fallback to linear scale if required.
- The API favors transparency and quick interpretation of how well an estimator generalizes and how sensitive it is to hyperparameters.
