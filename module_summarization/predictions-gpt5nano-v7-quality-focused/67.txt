Feature extraction utilities for text data (feature_extraction.rst)

Module overview
----------------
The feature_extraction.text submodule provides tools to convert text documents into numerical feature vectors. It includes vectorizers that build vocabularies, transform raw text into token counts, and compute TF–IDF representations. It also offers simple, importable text preprocessing helpers such as accent-stripping and HTML/XML tag stripping, plus a built-in set of English stop words.

Public API
----------

HashingVectorizer
    A transformer that converts a collection of text documents to a matrix of token counts using the hashing trick. It does not require an explicit vocabulary and can efficiently handle large corpora. Typical usage:

    from sklearn.feature_extraction.text import HashingVectorizer

    hv = HashingVectorizer(n_features=2**20, alternate_sign=False)
    X = hv.fit_transform(docs)  # fit is a no-op; transform computes the features

CountVectorizer
    Converts a collection of text documents to a matrix of token counts. Supports tokenization, optional pre-processing, and n-gram generation. Typical usage:

    from sklearn.feature_extraction.text import CountVectorizer

    cv = CountVectorizer(stop_words='english', ngram_range=(1, 2))
    X = cv.fit_transform(docs)

ENGLISH_STOP_WORDS
    Built-in set of common English stop words (as a frozenset). Use this to filter out stop words during vectorization, e.g.:

    from sklearn.feature_extraction.text import ENGLISH_STOP_WORDS

    stop_words = ENGLISH_STOP_WORDS

TfidfTransformer
    Transforms a count matrix to a TF–IDF representation. It is typically used after CountVectorizer, or with any matrix of non-negative counts. Typical usage:

    from sklearn.feature_extraction.text import TfidfTransformer

    counts = cv.fit_transform(docs)
    tfidf = TfidfTransformer()
    X = tfidf.fit_transform(counts)

TfidfVectorizer
    Combines CountVectorizer and TfidfTransformer into a single step. It produces TF–IDF features directly from raw text documents. Typical usage:

    from sklearn.feature_extraction.text import TfidfVectorizer

    tfidf_vec = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
    X = tfidf_vec.fit_transform(docs)

strip_accents_ascii
strip_accents_unicode
strip_tags
    Simple preprocessing helpers exposed at the module level.

    - strip_accents_unicode(s): Replaces accented Unicode characters with their simple counterparts. Slower but capable of handling any Unicode symbol with a known ASCII counterpart.
    - strip_accents_ascii(s): Transliterates accented characters to ASCII (best for languages with direct ASCII transliterations).
    - strip_tags(s): Strips basic HTML/XML tags using a regular expression.

Internal helpers and utilities (documentation-oriented summary)
--------------------------------------------------------------

_preprocess(doc, accent_function=None, lower=False)
    Chains optional preprocessing steps to a document. Applies lowercasing (if requested) and an optional accent-stripping function. Returns the preprocessed string.

_analyze(doc, analyzer=None, tokenizer=None, ngrams=None, preprocessor=None, decoder=None, stop_words=None)
    Chains text processing steps to produce n-grams from a single document. If an analyzer is provided, it replaces preprocessor, tokenizer, and ngrams steps; otherwise, it applies the provided preprocessor, tokenizer, and ngrams, optionally filtering with stop words.

_check_stop_list(stop)
    Resolves a stop word specification to a concrete stop word set:
    - 'english' returns ENGLISH_STOP_WORDS
    - None returns None
    - a string raises an error unless it matches a built-in list
    - a collection returns a frozenset of words

_VectorizerMixin
    Shared logic for text vectorizers, including tokenization and decoding of input documents.

decode(doc)
    Converts input to a Unicode string based on the vectorizer’s input/source configuration:
    - 'filename' reads a file path as binary, then decodes
    - 'file' reads a file-like object
    - Handles bytes input by decoding with the configured encoding and error handling

_word_ngrams(tokens, stop_words=None)
    Converts token sequences into n-grams, optionally filtering out stop words.

Usage examples
--------------

Example 1: Basic CountVectorizer and TfidfVectorizer
- Build a bag-of-words representation and a TF–IDF representation for a small corpus.

  from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer

  docs = [
      "The quick brown fox jumps over the lazy dog.",
      "Never jump over the lazy dog quickly."
  ]

  # Count vectors with English stop words removed and bigrams included
  cv = CountVectorizer(stop_words='english', ngram_range=(1, 2))
  X_counts = cv.fit_transform(docs)

  # TF–IDF vectors directly from raw text
  tfidf_vec = TfidfVectorizer(stop_words='english', ngram_range=(1, 2))
  X_tfidf = tfidf_vec.fit_transform(docs)

Example 2: HashingVectorizer for large corpora
- Use hashing to avoid building a vocabulary, suitable for large-scale or streaming data.

  from sklearn.feature_extraction.text import HashingVectorizer

  docs = [
      "Data science combines statistics with programming.",
      "Machine learning enables predictive analytics."
  ]

  hasher = HashingVectorizer(n_features=2**18, alternate_sign=False)
  X = hasher.fit_transform(docs)  # fit is a no-op; use transform/fit_transform

Notes and best practices
------------------------

- Accent stripping: Use strip_accents_unicode for full Unicode coverage or strip_accents_ascii for ASCII transliteration when you need compact keys or compatibility with ASCII-only pipelines.
- HTML/XML cleaning: Use strip_tags to remove simple tags before tokenization if raw HTML is present, but for serious HTML parsing consider dedicated libraries.
- Stop words: ENGLISH_STOP_WORDS provides a built-in English stop word list. Supply a custom set or list of stop words via the stop_words parameter to vectorizers.
- Internal helpers: The _preprocess, _analyze, and related internal utilities support flexible and modular customization of preprocessing, tokenization, and n-gram generation. They are primarily of interest to advanced users implementing custom analyzers or extending the API.

Formatting and extensibility
----------------------------

- All public vectorizers follow a consistent transformer API (fit, transform, fit_transform), enabling easy chaining and compatibility with scikit-learn pipelines.
- The module is designed to interoperate with other preprocessing and feature extraction steps in scikit-learn, providing a cohesive text-processing workflow.
