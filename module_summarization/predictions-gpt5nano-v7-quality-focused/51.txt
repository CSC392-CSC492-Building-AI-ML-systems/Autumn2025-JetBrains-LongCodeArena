The one-parameter exponential family distributions used by GLM (gee)

Overview
- The module defines the base class Family, which represents a one-parameter exponential family used in generalized linear models (GLM).
- A Family combines:
  - a link function (to transform the mean to the linear predictor space)
  - a variance function (to model Var(mu) as a function of mu)
- Core capabilities provided:
  - starting_mu(y): starting value for the IRLS algorithm
  - weights(mu): IRLS weights
  - deviance(endog, mu, var_weights, freq_weights, scale): deviance for a set of observations
  - resid_dev(endog, mu, var_weights, scale): deviance residuals
  - fitted(lin_pred): fitted values corresponding to a linear predictor
- Subclasses implement the analytic _resid_dev(endog, mu), which defines the distribution-specific deviance contribution.

Public API
- __init__(self, link, variance)
  - Sets the link (must be a Link instance) and the variance function.
- link (property)
  - Accessor for the link function; validates that the provided link is compatible with the family.
- starting_mu(self, y)
  - Returns the starting mean mu for IRLS.
- weights(self, mu)
  - Returns IRLS weights: 1 / (g'(mu)^2 * Var(mu)).
- deviance(self, endog, mu, var_weights=1., freq_weights=1., scale=1.)
  - Computes the (scaled) deviance.
- resid_dev(self, endog, mu, var_weights=1., scale=1.)
  - Computes the deviance residuals.
- fitted(self, lin_pred)
  - Returns fitted values from linear predictors (mu = g^{-1}(eta)).

Example
This example shows how to define a minimal, self-contained subclass of Family and use it to compute starting values, weights, deviance, and deviance residuals.

```python
import numpy as np
from gee.family import Family
from gee.family import links as L

# Minimal identity-like link for demonstration
class IdentityLink(L.Link):
    def __call__(self, mu):
        return mu
    def deriv(self, mu):
        return np.ones_like(mu)

# Simple Gaussian-like family using an identity link
class SimpleGaussianFamily(Family):
    # Allow IdentityLink as a valid link for this family
    links = [IdentityLink]

    def _resid_dev(self, endog, mu):
        # Simple squared-error contribution for demonstration
        resid = endog - mu
        return resid**2

# Variance function (constant variance for this simple example)
variance = lambda mu: np.ones_like(mu)

# Build the family
link = IdentityLink()
fam = SimpleGaussianFamily(link=link, variance=variance)

# Data
y = np.array([0.2, 0.5, 0.9])

# Compute quantities
mu0 = fam.starting_mu(y)  # starting mu for IRLS
w = fam.weights(mu0)       # IRLS weights
D = fam.deviance(y, mu0)   # deviance
r = fam.resid_dev(y, mu0)  # deviance residuals

print("mu0 =", mu0)
print("weights =", w)
print("deviance =", D)
print("deviance residuals =", r)
```

Notes
- In practice, provide a realistic variance function V(mu) for the chosen distribution.
- Implement _resid_dev(endog, mu) to reflect the analytic deviance contribution for your distribution. The base class uses this to compute resid_dev and deviance.
- To use real GLM families (Poisson, Binomial, etc.), extend Family with a concrete _resid_dev implementation and select an appropriate variance function and valid links (as defined in the library’s links module).
