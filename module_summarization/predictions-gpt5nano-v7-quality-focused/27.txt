w_kinetics
==========

Overview
--------

w_kinetics is a deprecated wrapper around the direct kinetics tooling. It provides a trace subcommand to compute state-to-state kinetics by tracing trajectories and then relies on the w_kinavg tool for normalization into rates with confidence intervals. It is implemented as a backwards-compatibility shim that forwards to the w_direct workflow and emits a deprecation warning.

Deprecation and behavior
------------------------

- Running w_kinetics prints a deprecation warning: “w_kinetics is being deprecated. Please use w_direct instead.”
- If invoked without the trace subcommand, the command automatically inserts the trace subcommand and forwards to the underlying w_direct logic.
- The recommended replacement is to use w_direct with the trace subcommand.

Usage
-----

w_kinetics trace [options]

- The default output file is kintrace.h5 unless overridden with -o/--output.
- A bin assignment file (usually assign.h5) that labels trajectories is required. See w_assign --help for details on generating this file.

Output format
-------------

The output file (default kintrace.h5) contains the following datasets:

- /conditional_fluxes [iteration][state][state]
  Floating-point. Macrostate-to-macrostate fluxes. Not normalized by the initial macrostate population.

- /conditional_arrivals [iteration][stateA][stateB]
  Integer. Number of trajectories arriving at stateB in a given iteration, given that they departed from stateA.

- /total_fluxes [iteration][state]
  Floating-point. Total flux into a given macrostate.

- /arrivals [iteration][state]
  Integer. Number of trajectories arriving at a given state in a given iteration, regardless of origin.

- /duration_count [iteration]
  Integer. The number of event durations recorded in each iteration.

- /durations [iteration][event duration]
  Structured data. Event durations for transitions that end during a given iteration. Stored as:
  - istate: Integer. Initial state of the transition.
  - fstate: Integer. Final state of the transition.
  - duration: Floating-point. Duration of the transition, in units of tau.
  - weight: Floating-point. Weight of the trajectory at the end of the transition (not normalized by initial state population).

Notes
-----

- The flux datasets are not normalized by the initial macrostate population. To obtain rates, normalize using the w_kinavg tool, which also accounts for statistical fluctuations and correlations. Target fluxes (total flux into a given state) require no normalization.

Command-line options
---------------------

w_kinetics accepts options from the underlying w_direct tool (with the trace subcommand). The most relevant option is:

- -o, --output FILE
  Specify the output file name (default: kintrace.h5).

Input requirements
------------------

- Requires a bin assignment file containing trajectory labeling (commonly assign.h5). See w_assign --help for details on generating this file.

Examples
--------

- Basic usage (default output):

  w_kinetics trace -o kintrace.h5

- Ensure an assignment file is present (name may vary; refer to your workflow):

  w_kinetics trace -o kintrace.h5

Related tools
-------------

- w_kinavg trace: Converts the flux data in kintrace.h5 into average rates with confidence intervals.
- w_direct: The recommended direct tool to perform kinetics analysis without the w_kinetics shim.
- w_assign: Generates the required trajectory labeling/assignment file.

Source note
-----------

w_kinetics serves as a backwards-compatibility shim for older workflows. The deprecation warning guides users to migrate to w_direct while preserving existing functionality.
