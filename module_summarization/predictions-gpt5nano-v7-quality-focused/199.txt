Neural Networks – Supervised
============================

This module provides utilities for supervised neural network models. It focuses on in-place activation functions, their derivatives, and common loss functions used in training classifiers and regressors.

In-place activation functions
-----------------------------

Activation functions operate on data arrays in place to reduce memory usage during forward passes.

- Identity: leaves data unchanged
- Logistic (sigmoid): maps inputs to (0, 1)
- Tanh: maps inputs to (-1, 1)
- ReLU: applies maximum(0, x)
- Softmax: maps each row to a probability distribution over classes

API
- inplace_identity(X)
- inplace_logistic(X)
- inplace_tanh(X)
- inplace_relu(X)
- inplace_softmax(X)

All functions mutate their input array X in place.

Key mapping
- ACTIVATIONS: a dictionary mapping activation names to their in-place functions
  - "identity" → inplace_identity
  - "tanh" → inplace_tanh
  - "logistic" → inplace_logistic
  - "relu" → inplace_relu
  - "softmax" → inplace_softmax

Usage example
- Apply an activation in place to a data matrix X (shape: n_samples × n_features):

  .. code-block:: python

     import numpy as np
     from neural_networks_supervised import ACTIVATIONS

     X = np.random.randn(100, 20)
     ACTIVATIONS["relu"](X)
     # X is updated in place

In-place derivatives of activations
-----------------------------------

Derivative functions compute the backward signal in place, given the forward activation outputs.

- Identity derivative: inplace_identity_derivative(Z, delta)  # does nothing
- Tanh derivative: inplace_tanh_derivative(Z, delta)        # delta *= 1 - Z^2
- Logistic derivative: inplace_logistic_derivative(Z, delta)  # delta *= Z * (1 - Z)
- ReLU derivative: inplace_relu_derivative(Z, delta)          # zeroes delta where Z == 0

API
- DERIVATIVES: dictionary mapping activation names to their in-place derivative functions
  - "identity" → inplace_identity_derivative
  - "tanh" → inplace_tanh_derivative
  - "logistic" → inplace_logistic_derivative
  - "relu" → inplace_relu_derivative

Usage example
- Backpropagate a gradient delta through an activation:

  .. code-block:: python

     import numpy as np
     from neural_networks_supervised import DERIVATIVES

     Z = np.array([[0.2, -0.5], [0.8, 0.0]])  # output from forward pass
     delta = np.ones_like(Z)
     DERIVATIVES["relu"](Z, delta)
     # delta is updated in place

Loss functions
--------------

The module provides standard loss functions used for supervised learning.

1. Squared loss (regression)
   - Definition: ((y_true - y_pred) ** 2).mean() / 2
   - y_true, y_pred: array-like, shapes compatible for elementwise difference
   - Returns: scalar loss

   Example:
   .. code-block:: python

      from neural_networks_supervised import squared_loss
      y_true = np.array([[1.0], [2.0]])
      y_pred = np.array([[0.5], [2.5]])
      loss = squared_loss(y_true, y_pred)

2. Logistic loss (multiclass classification)
   - Definition: -sum(xlogy(y_true, y_prob)) / n_samples
   - y_true: one-hot encoded (n_samples, n_classes) or similar
   - y_prob: predicted probabilities (n_samples, n_classes)
   - The function clips probabilities to [eps, 1 - eps] for numerical stability
   - If y_prob has a single class (n_classes == 1), it is extended to two columns
   - If y_true has a single class (n_classes == 1), it is extended to two columns

   Example:
   .. code-block:: python

      from neural_networks_supervised import log_loss
      y_true = np.array([[1, 0], [0, 1]])
      y_prob = np.array([[0.7, 0.3], [0.4, 0.6]])
      loss = log_loss(y_true, y_prob)

3. Binary logistic loss (binary classification, kept for multilabel use)
   - Definition: -(xlogy(y_true, y_prob).sum() + xlogy(1 - y_true, 1 - y_prob).sum()) / n_samples
   - y_true: (n_samples, 1) or (n_samples, 1) binary labels
   - y_prob: predicted probabilities (n_samples, 1)

   Example:
   .. code-block:: python

      from neural_networks_supervised import binary_log_loss
      y_true = np.array([[1], [0], [1]])
      y_prob = np.array([[0.9], [0.2], [0.8]])
      loss = binary_log_loss(y_true, y_prob)

Notes
- All loss functions operate on NumPy arrays
- For numerical stability, log_loss and binary_log_loss clip probabilities using the input dtype’s epsilon

Usage example: selecting a loss function
- The module exposes LOSS_FUNCTIONS mapping loss names to functions (exact keys may vary)
- You can call a loss function directly if you know the function reference, e.g. squared_loss, log_loss, binary_log_loss

  .. code-block:: python

     from neural_networks_supervised import squared_loss, log_loss, binary_log_loss
     loss1 = squared_loss(y_true, y_pred)
     loss2 = log_loss(y_true, y_prob)
     loss3 = binary_log_loss(y_true_binary, y_prob_binary)

Relation to supervised neural network models
--------------------------------------------

These utilities support forward-pass activations, backward-pass derivatives, and common loss computations used in supervised neural network training, including:
- Feedforward activations for hidden and output layers
- Backpropagation signals through activation functions
- Loss calculations for regression and classification tasks

Implementation notes
- The in-place operations aim to minimize memory overhead during training
- Derivatives rely on the forward-pass outputs (Z) to compute the gradient signals in delta
- Loss functions are designed to handle common shapes seen in multiclass and binary setups
