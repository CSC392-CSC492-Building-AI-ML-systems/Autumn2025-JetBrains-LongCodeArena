Partial Dependence and Individual Conditional Expectation (ICE) plots
=====================================================================

Overview
--------
Partial Dependence Plots (PDP) visualize the average effect of one or two features on the predicted outcome of a trained model. Individual Conditional Expectation (ICE) plots extend PDP by displaying the effect for each individual instance, revealing heterogeneity and potential interactions across samples.

Key ideas
- PDP averages the model’s prediction over all samples for fixed values of the target feature(s).
- ICE shows a curve for each sample, illustrating how predictions change as the target feature(s) vary.
- Using both views helps identify feature effects that are consistent across data or vary by region, group, or sample.

API and usage (high level)
---------------------------
- Compute PDP or ICE for one or two features using the inspection utilities.
- Features to plot can be specified by index or by feature name. Internal helpers resolve names and indices:
  - _check_feature_names(X, feature_names=None) ensures a valid, unique list of feature names.
  - _get_feature_index(fx, feature_names=None) resolves a feature spec (name or index) to an integer index.
- Plotting can be done via high-level display utilities or by extracting the data and plotting manually.

Feature naming and indexing
---------------------------
- If X is a DataFrame, and feature_names is None, the implementation uses the DataFrame’s column names.
- If X is a NumPy array, and feature_names is None, a generic list like ["x0", "x1", ...] is created.
- You can pass a feature name (string) or a feature index (int). Internally:
  - If a string is provided, the code looks up its position in feature_names.
  - If a numeric index is provided, it is used directly.
- Duplicates in feature_names are rejected to avoid ambiguity.

Code examples
-------------
The following examples illustrate typical usage with scikit-learn-style APIs.

Example 1: PDP for a single feature by name (DataFrame X)
::
from sklearn.ensemble import RandomForestRegressor
from sklearn.inspection import partial_dependence, PartialDependenceDisplay
import matplotlib.pyplot as plt

# Assume X is a pandas DataFrame and y is the target
# and estimator is a fitted model
estimator = RandomForestRegressor(n_estimators=200, random_state=0)
estimator.fit(X, y)

# PDP for a single feature named 'feature_1'
features = ['feature_1']
disp = PartialDependenceDisplay.from_estimator(estimator, X, features, grid_resolution=50)
disp.plot()
plt.show()

Example 2: ICE for a single feature by index (NumPy array X)
::
# X is a NumPy array, feature 0 is the target
features = [0]  # or [(0,)] depending on API variation
# Depending on the API, specify kind='individual' to get ICE
disp_ice = PartialDependenceDisplay.from_estimator(estimator, X, features, grid_resolution=50, kind='individual')
disp_ice.plot()
plt.show()

Example 3: PDP for a pair of features
::
# Pair of features by index
features = [(0, 2)]
disp_pair = PartialDependenceDisplay.from_estimator(estimator, X, features, grid_resolution=20)
disp_pair.plot()
plt.show()

Example 4: PDP using feature names with explicit feature_names
::
# If X is a NumPy array, supply feature_names to enable name-based access
feature_names = ['size', 'age', 'weight', 'color_score']
features = ['age']  # or [(1,)]
disp_named = PartialDependenceDisplay.from_estimator(estimator, X, features, feature_names=feature_names, grid_resolution=40)
disp_named.plot()
plt.show()

Notes
-----
- PDP computes the average prediction across the dataset for fixed values of the target feature(s).
- ICE draws one curve per sample, showing the prediction as the target feature(s) vary.
- The features argument accepts:
  - A single feature specification (int index or str name).
  - A list of single features for multiple PDP curves.
  - A tuple of two features to produce a 2D PDP (and optionally ICE for the pair).
- If feature_names is provided or inferred from X, string specifications map to the correct indices through internal resolution.

Implementation details (internal helpers)
---------------------------------------
- _check_feature_names(X, feature_names=None)
  - Determines a validated list of feature names.
  - If feature_names is None:
    - Uses X.columns.tolist() for pandas DataFrames.
    - Otherwise generates ["x0", "x1", ...] for NumPy arrays.
  - Ensures all feature_names are unique; raises ValueError on duplicates.
- _get_feature_index(fx, feature_names=None)
  - If fx is a string, looks up its index in feature_names.
  - If fx is an int, returns it directly.
  - Produces clear errors when a feature name is not found.

Compatibility and tips
----------------------
- DataFrame input is preferred when you want to reference features by name.
- When using NumPy arrays, consider providing feature_names so you can reference features by meaningful names.
- For large datasets, ICE can be more computationally intensive than PDP due to per-sample curves.
- Use the display utilities (PartialDependenceDisplay) for convenient plotting and consistent aesthetics.
