# fleur_parser.md

Overview
- This module extracts distinct parts of Fleur XML files to support versioning and reuse.
- It provides high-level helpers to read calculation modes and k-point settings from Fleur input/output XMLs.

Public API
- get_fleur_modes(xmltree, schema_dict, logger=None) -> dict[str, Any]
- get_nkpts(xmltree, schema_dict, logger=None) -> int
- Both functions operate on a FleurXMLContext, which handles version-aware XML access.

get_fleur_modes(xmltree, schema_dict, logger=None) -> dict[str, Any]
- Purpose
  - Detects calculation modes that influence outputs (e.g., out.xml, post-processing).
- Parameters
  - xmltree: XMLLike
    - The Fleur XML tree (parsed with lxml or similar).
  - schema_dict: fleur_schema.InputSchemaDict | fleur_schema.OutputSchemaDict
    - Version-aware schema dictionary used to interpret the XML structure.
  - logger: Logger | None
    - Optional logger for warnings/errors.
- Returns
  - A dictionary containing the extracted calculation modes.
- How values are determined
  - jspin: value of the jspins attribute
  - noco: l_noco attribute (default False)
  - soc: l_soc attribute (default False)
  - relax: l_f attribute (default False)
  - spex: 0 or the gw/spex value (uses expertModes section if present, with version-specific fallback)
  - force_theorem: presence of forceTheorem tag (depends on version)
  - cf_coeff: True if cFCoeffs exist and contain potential or chargeDensity entries
  - plot: plotting mode (interpreted differently depending on version)
  - film: presence of filmPos
  - ldau: presence of ldaU for species
  - dos: value of the dos attribute
  - band: value of the band attribute
  - bz_integration: value of the mode attribute inside bzIntegration
  - greensf: greensfCalculation presence (and torqueCalculation/torgueCalculation for older versions)
  - ldahia: presence of ldaHIA for species
- Version behavior and dependencies
  - Behavior adapts to schema_dict.inp_version.
  - Some fields (e.g., spex, force_theorem, greensf, ldahia) depend on specific version thresholds.
- Example
  - Assume you already have xmltree and a valid schema_dict:
    - modes = get_fleur_modes(xmltree, schema_dict)
    - print(modes)
  - Expected keys include: jspin, noco, soc, relax, spex, force_theorem, cf_coeff, plot, film, ldau, dos, band, bz_integration, greensf, ldahia

get_nkpts(xmltree, schema_dict, logger=None) -> int
- Purpose
  - Returns the number of k-points used in the calculation described by the Fleur XML.
- Decorator
  - schema_dict_version_dispatch(output_schema=False) selects the appropriate implementation for the file version.
- Parameters
  - xmltree: XMLLike
    - The Fleur XML tree.
  - schema_dict: fleur_schema.InputSchemaDict | fleur_schema.OutputSchemaDict
    - Version-aware schema dictionary used to interpret the XML structure.
  - logger: Logger | None
    - Optional logger for warnings/errors.
- Returns
  - An integer with the number of k-points.
- How it works
  - Reads the selected k-point set name from the listName attribute.
  - Collects all available k-point list names that exist in kPointLists and kPointList elements.
  - Validates that the chosen listName exists; otherwise raises ValueError listing available names.
  - Locates the corresponding kPointList element by name and reads its count attribute as nkpts.
- Version considerations
  - Behavior relies on the version-dispatch mechanism to handle differences across Fleur XML schema versions.
- Error handling
  - If the selected k-point list name is invalid, raises a ValueError with the available options.
  - If nkpts cannot be evaluated as an int, raises an error (as implemented in the code path).
- Example
  - Assume you have xmltree and a valid schema_dict:
    - nkpts = get_nkpts(xmltree, schema_dict)
    - print(nkpts)

Usage tips
- Prerequisites
  - Provide a valid XML tree (XMLLike) representing Fleur input or output.
  - Supply a schema dictionary (InputSchemaDict or OutputSchemaDict) that matches the file version.
- Practical notes
  - get_fleur_modes helps drive downstream decisions (e.g., reporting, filtering, or generating versioned metadata).
  - get_nkpts is useful for validating and comparing k-point sampling across runs or versions.
- Integration pattern
  - Typical workflow reads an XML file, builds a schema dict corresponding to its version, and calls the parsers:
    - xmltree = etree.parse('path/to/file.xml')
    - schema_dict = fleur_schema.determine_schema(xmltree)  # example placeholder
    - modes = get_fleur_modes(xmltree, schema_dict, logger=logger)
    - nkpts = get_nkpts(xmltree, schema_dict, logger=logger)

Internal details (brief)
- get_fleur_modes uses FleurXMLContext to access fields in a version-aware way.
- get_nkpts relies on a version-dispatch decorator to select the correct implementation for the given Fleur version.
- The code handles missing or optional attributes with sensible defaults or version-aware fallbacks.
