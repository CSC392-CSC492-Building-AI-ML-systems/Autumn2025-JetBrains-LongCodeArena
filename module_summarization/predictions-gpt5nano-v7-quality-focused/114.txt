Simulation utilities
====================

Overview
--------
This document describes the small set of utilities found in simulation.rst. They operate on Bluesky plan generators that yield Msg objects and provide helpers to visualize, summarize, and validate plans that manipulate motor positions.

API
---

plot_raster_path
~~~~~~~~~~~~~~~~
Plot the raster path for a given plan.

Parameters
- plan (iterable)
  Must yield Msg objects and not be a coroutine.
- x_motor (str)
  Name of the x motor.
- y_motor (str)
  Name of the y motor.
- ax (matplotlib.axes.Axes, optional)
  Axes to plot to. If None, a new figure and axes are created.
- probe_size (float, optional)
  If provided, uses this as the radius of the probe at each saved point (in the same units as motor positions).
- lw (float, optional)
  Width of the lines drawn between points.

Returns
- dict with keys:
  - 'path': Path object (Line2D) representing the drawn path.
  - 'events': A scatter plot object (PathCollection) or a PatchCollection of probes, depending on probe_size.

Description
- The function tracks the current x and y motor values by consuming 'set' messages that update the motor values when msg.obj.name matches x_motor or y_motor.
- On 'save' messages, it records the current (x, y) as a raster point.
- After processing the plan, it draws the path through saved points.
- If probe_size is None, a scatter plot marks the saved points; otherwise, circular probes are drawn at each point.

Example
- Basic usage (default axis, no probes):

.. code-block:: python

   plan = [
       Msg('set', DummyObj('x'), (0.0, ), ()),
       Msg('set', DummyObj('y'), (0.0, ), ()),
       Msg('save', None, (), ()),
       Msg('set', DummyObj('x'), (1.0, ), ()),
       Msg('set', DummyObj('y'), (0.5, ), ()),
       Msg('save', None, (), ()),
   ]
   plot_raster_path(plan, x_motor='x', y_motor='y')

summarize_plan
~~~~~~~~~~~~~
Print a minimal, human-readable summary of a plan.

Parameters
- plan (iterable)
  Must yield Msg objects.

Returns
- None. The function prints to stdout via the Bluesky print_summary_wrapper.

Description
- Iterates over the plan using bluesky’s print_summary_wrapper to produce a concise summary.
- The function itself does not yield values.

Alias
- print_summary is an alias for summarize_plan (backwards compatibility).

Example
.. code-block:: python

   plan = [...]  # iterable of Msg objects
   summarize_plan(plan)  # or: print_summary(plan)

check_limits
~~~~~~~~~~~~
Check that a plan does not move devices beyond their limits.

Parameters
- plan (iterable)
  Must yield Msg objects.

Returns
- None. Emits warnings for devices that lack a check_value method or for out-of-bounds attempts.

Description
- Iterates the plan to inspect 'set' messages.
- For each object (msg.obj) not in the ignore list:
  - If the object has a check_value method, calls msg.obj.check_value(msg.args[0]).
  - Otherwise, emits a warning that the object has no check_value() method and appends the object to an ignore list to avoid repeated warnings.
- If an object defines check_value and the value violates limits, the check_value call would typically raise an exception or warning depending on the implementation.

Example
.. code-block:: python

   plan = [...]  # iterable of Msg objects
   check_limits(plan)

Notes
-----
- plot_raster_path expects the plan to yield messages with:
  - cmd 'set' messages updating motor values via msg.obj.name and msg.args[0]
  - cmd 'save' messages marking raster points
- The function set uses cur_x/cur_y as the current motor values; if a 'save' occurs before corresponding 'set' messages for a motor, None may be recorded.
- plot_raster_path requires matplotlib. If probe_size is provided, the function uses circular probes at each saved point; otherwise it marks points with a scatter marker.

See also
--------
- Bluesky message structure (Msg) used by plan iterables
- Bluesky print_summary_wrapper for plan inspection

Code references
- plot_raster_path
  - Imports: matplotlib.pyplot as plt, matplotlib.collections as mcollections, matplotlib.patches as mpatches
  - Returns: dict with 'path' (Line2D) and 'events' (scatter or PatchCollection)
- summarize_plan and print_summary
  - Uses bluesky.preprocessors.print_summary_wrapper
- check_limits
  - Uses warnings.warn, and requires msg.args[0] to supply the value checked against limits
  - If msg.obj has no check_value, emits a warning and tracks ignored objects
