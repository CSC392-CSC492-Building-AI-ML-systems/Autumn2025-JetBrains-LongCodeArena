Mixed Linear Models (mixed_linear.rst)
======================================

Overview
--------

This module implements linear mixed effects models via the MixedLM class. It provides a group-based framework for fitting regression models that incorporate both fixed effects and random effects, enabling estimation of mean relationships and variance structure for dependent data. The implementation emphasizes models that can be partitioned into independent groups, while also supporting some crossed-effects configurations.

Key concepts
------------

- Endogenous data and design matrices:
  - endog (Y): n_i-dimensional response for each group i.
  - exog (X): n_i x k_fe design matrix for fixed effects.
  - exog_re (Z): design matrix for random effects; can vary in width by group.
- Parameters:
  - fe_params: fixed effects coefficients.
  - cov_re (Psi): covariance for the standard random effects (shared across groups).
  - vcomp: variance components (for additional, group-specific variance parameters).
  - sigma^2: residual error variance (scale).
  - gamma: random vector (per-group realization of random effects).
- Marginal mean: E[Y | X, Z] = X * beta.
- Variants of random effects:
  - Standard random effects: uniformly sized across groups with a common joint distribution.
  - Variance components: uncorrelated across components and with standard random effects; per-group realized values can differ in count.

Model structure
--------------

For each group i, the probability model is:

Y_i = X_i * beta + Z_i * gamma_i + epsilon_i

where:
- Y_i is the observed response for group i.
- X_i is the fixed-effects design matrix for group i.
- Z_i is the random-effects design matrix for group i.
- gamma_i is a random vector with mean 0; its covariance structure is shared across groups for the first k_re elements (cov_re), with additional variance components.
- epsilon_i ~ N(0, sigma^2 I) are iid errors within and between groups.
- Groups are disjoint; the data can be efficiently modeled when the group structure is preserved.

Parameterizations
-----------------

Three parameterizations are used in different parts of the implementation:

- User parameterization:
  cov(endog) = scale * I + Z * cov_re * Z'. This is the parameterization seen by users.
- Profile parameterization:
  cov(endog) = I + Z * cov_re1 * Z'. This parameterizes the profile likelihood; the user-facing cov_re is cov_re1 scaled by the scale.
- Square-root parameterization:
  uses the Cholesky factor of cov_re1. This is hidden from users but supported internally.

Parameter packing:
- The parameter vector can combine fe_params, the lower triangle or Cholesky factor of the random-effects covariance, and variance components. Depending on the parameterization, the packing may reflect or square the dependence structure accordingly.

Estimation and optimization
---------------------------

- Estimation uses GLS to avoid directly optimizing over fixed effects.
- The likelihood optimized is profiled over the scale parameter and fixed effects (when present).
- Hessian of the profiled log-likelihood is not implemented; consequently, Newton-Raphson or Hessian-based methods are not applicable for fitting.
- Two score methods are implemented:
  - Score with respect to the random-effects covariance elements (used for inference after fitting).
  - Score with respect to the Cholesky-square-root parameters of the covariance (used during optimization).

Post-estimation tasks
---------------------

- The MixedLM class provides support for common post-estimation tasks (e.g., retrieving parameter estimates, standard errors, and related statistics).
- The implementation follows established algorithms for linear mixed models, adapted for variance components.

Notational glossary
--------------------

- cov_re: random effects covariance matrix (Psi).
- scale: residual variance scalar (sigma^2).
- vcomp: vector of variance parameters for variance components.
- fe_params: fixed effects parameter vector.
- beta: fixed effects parameters (same as fe_params in user-facing terms).
- endog: observed response vector.
- exog: fixed-effects design matrix (X).
- exog_re: random-effects design matrix (Z).

Implementation notes
--------------------

- The module relies on numpy, pandas, patsy, and SciPy, including sparse matrix support.
- It uses the GLS framework to perform efficient estimation when focusing on fixed effects and variance components.
- Profiling nature of the likelihood means explicit Hessian calculation is unnecessary and not provided.

Usage interface
---------------

- MixLM class:
  - endog: array-like, shape (n_total,)
  - exog: array-like, shape (n_total, k_fe)
  - groups: array-like, length n_total, indicating group membership
  - exog_re: array-like or None, random-effects design matrix (per group)
  - exog_vc or vc_formula: variance components specification (optional)

Example
-------

- Basic usage (conceptual):

  import numpy as np
  import pandas as pd
  from statsmodels.regression.mixed_linear_model import MixedLM

  # y: response, X: fixed effects, Z: random effects design, groups: group labels
  model = MixedLM(endog, exog, groups=groups, exog_re=exog_re)
  result = model.fit()

  print(result.summary())

Notes for developers
--------------------

- The module is designed to be efficient for models with clearly partitioned data into independent groups.
- It supports certain crossed-effects configurations by modeling within a single group.
- The numerical core emphasizes profiling the likelihood over scale and fixed effects, reducing the need for Hessian-based optimization.
