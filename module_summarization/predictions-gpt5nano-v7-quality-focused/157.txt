Intro to Guild Widgets in nextcord
==================================

Nextcord exposes a lightweight representation of a guild widget. A widget provides a simplified view of a guild’s channels and members, suitable for external tools and quick integrations. The core objects are Widget, WidgetChannel, and WidgetMember.

Classes
-------

WidgetChannel
~~~~~~~~~~~~

Represents a partial channel exposed by a guild widget.

- Attributes
  - id: int — The channel's ID.
  - name: str — The channel's name.
  - position: int — The channel's position.

- Behaviors
  - __str__ returns the channel name.
  - __repr__ returns a concise representation including id, name, and position.
  - mention: str — A string you can use to mention the channel, e.g. "<#1234567890>".
  - created_at: datetime — The channel’s creation time (UTC), derived from its ID.

Usage example
::
    channel = widget.channels[0]
    print(channel.id)
    print(channel.name)
    print(channel.mention)
    print(channel.created_at)

WidgetMember
~~~~~~~~~~~~

Represents a partial member in the widget’s guild.

- Inherits from BaseUser and adds widget-specific fields.
- Attributes (among others)
  - name, discriminator, id, bot, avatar
  - status: Status — The member’s presence status.
  - nick: Optional[str] — The member’s nickname.
  - activity: Optional[Union[BaseActivity, Spotify]] — The member’s current activity, if any.
  - deafened: Optional[bool] — Whether the member is deafened.
  - muted: Optional[bool] — Whether the member is muted.
  - suppress: Optional[bool] — Whether the member is suppressed.
  - connected_channel: Optional[WidgetChannel] — The channel the member is connected to.

- Behaviors
  - __repr__ shows name, discriminator, bot, and nick.
  - display_name: str — Returns nick if present, otherwise the username.

Usage example
::
    member = widget.members[0]
    print(member.display_name)
    print(member.status)

Widget
~~~~~~

Represents a Guild widget.

- Attributes
  - id: int — The guild’s ID.
  - name: str — The guild’s name.
  - channels: List[WidgetChannel] — The accessible voice channels in the guild.
  - members: List[WidgetMember] — The online members in the server. Offline members do not appear in the widget.

Usage example
::
    # Iterate channels and members available in the widget
    for ch in widget.channels:
        print(ch.name, ch.id)

    for m in widget.members:
        print(m.display_name, m.status)

Notes
-----

- WidgetChannel and WidgetMember are lightweight representations meant to reflect the widget’s current view without requiring full guild object access.
- created_at uses the channel’s ID (a snowflake) to compute creation time.
- WidgetMember.activity is derived from the data payload’s 'game' field when present.

Code references (conceptual)
----------------------------

- WidgetChannel
  - __slots__ = ('id', 'name', 'position')
  - __str__ returns self.name
  - mention property returns "<#id>"
  - created_at property uses snowflake_time(self.id)

- WidgetMember
  - Inherits from BaseUser
  - status uses try_enum(Status, data.get('status'))
  - deafened: data.get('deaf') or data.get('self_deaf')
  - muted: data.get('mute') or data.get('self_mute')
  - suppress: data.get('suppress')
  - activity built from data['game'] if present via create_activity

- Widget
  - Represents a Guild widget with channels and members lists
  - Channels: List[WidgetChannel]
  - Members: List[WidgetMember]

Practical tip
-----------

When working with a widget payload, use the provided properties to quickly display channel names, format mentions, and show online members with their display names.
