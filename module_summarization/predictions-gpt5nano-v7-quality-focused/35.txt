w_states.rst
============

Display or manipulate basis (initial) or target (recycling) states for a WEST simulation.

Overview
--------

The w_states command lets you view, replace, or append basis and target states used by WEST-POOL simulations. By default, it only displays the current states for the active iteration; you can write these states to files for editing, or replace/append the states for the next iteration. Basis-state appending is not permitted in order to avoid renormalization issues; see the command help for details.

Key concepts
------------

- Basis states: the initial states used to seed simulations for an iteration.
- Target states: states used for recycling (reintroduction of trajectories) in subsequent iterations.
- Iteration transition: when the current iteration has completed segments, new states apply to the next iteration.

Usage
-----

w_states [OPTIONS]

Common options
- --show
  Display current basis/target states (or dump to files).

- --append
  Append the given basis/target states to those currently in use. Basis-state appending is not permitted by design; target-state appending is supported.

- --replace
  Replace current basis/target states with those specified.

- --bstate-file BSTATE_FILE
  Read (for append/replace) or write (for show) basis state names, probabilities, and optional auxiliary references to BSTATE_FILE.

- --bstate
  Append a basis state string to the list (format: label,probability[,auxref]).

- --tstate-file TSTATE_FILE
  Read (for append/replace) or write (for show) target state labels and their progress coordinates to TSTATE_FILE.

- --tstate
  Append a target state string to the list (format: label,pcoord0[,pcoord1[,...]]).

Defaults
- Default mode: show
- If you provide no --bstate-file/--bstate/--tstate-file/--tstate, w_states simply displays the current states.

File formats
------------

Basis states file (BSTATE_FILE)
- Each line represents a basis state as:
  label,probability[,auxref]
- When reading, states are converted to BasisState objects.
- When writing (show mode), BasisState.states_to_file writes a textual representation suitable for editing.

Target states file (TSTATE_FILE)
- Each line represents a target state as:
  label,pcoord0[,pcoord1[,...]]
- When reading, states are converted to TargetState objects.
- When writing (show mode), TargetState.states_to_file writes a textual representation suitable for editing.

Example formats
- Basis state line: "BS1,0.25,refA"
- Target state line: "TS1,0.15,0.3"

Behavior details by mode
------------------------

Show mode (--show)
- The command reads current states for the active iteration from the WEST data manager.
- If basis states exist, it writes them to stdout or to the file specified by --bstate-file, preceded by a header:
  "# Basis states for iteration <n>"
  followed by the textual basis-state list via BasisState.states_to_file.
- If target states exist, it writes them to stdout or to the file specified by --tstate-file, preceded by a header:
  "# Target states for iteration <n>"
  followed by the textual target-state list via TargetState.states_to_file.

Replace mode (--replace)
- The command replaces the current basis/target states for the next iteration.
- It computes the next iteration index:
  - If the current iteration has completed segments, it advances to iteration n+1.
- Basis states are built from:
  - --bstate-file (read basis states from file), and/or
  - --bstate (strings provided on the command line; parsed as label,probability[,auxref])
- If any basis states are provided:
  - It verifies that the total probability sums to unity. If not, it renormalizes all basis-state probabilities and logs a warning.
  - It assigns progress coordinates to basis states via sim_manager.get_bstate_pcoords(basis_states, n_iter).
  - It writes the basis states into the data store for iteration n_iter via data_manager.create_ibstate_group(basis_states, n_iter).
  - It reports the new basis states to the simulation manager via sim_manager.report_basis_states(basis_states).
- Target states are processed similarly (reading from --tstate-file and/or --tstate), and then applied to the next iteration as appropriate. (The snippet shows the target-state handling begins after the basis-state block; exact mechanics follow the same overall pattern in the full implementation.)

Append mode (--append)
- Appends given states to those currently in use for the next iteration.
- Basis-state appending is not permitted (per the command help) due to normalization concerns.
- Target-state appending follows the same append behavior as defined for target states in the full implementation.

Notes
- The command operates through the West data management stack (westpa.rc, data_manager, sim_manager, etc.).
- All operations that modify states apply to the next iteration, with an adjustment if the current iteration has completed segments.
- The EPS constant is used to determine tolerances when checking basis-state probabilities sum to one.

Examples
--------

Show current states to stdout
.. code-block:: text

  w_states --show

Show current states to files
.. code-block:: text

  w_states --show --bstate-file=bstates.txt --tstate-file=tstates.txt

Replace basis states from files and/or command line
.. code-block:: text

  w_states --replace --bstate-file=new_bstates.txt
  w_states --replace --bstate=NEW_BS1,0.5 --bstate=NEW_BS2,0.5

Append target states for next iteration
.. code-block:: text

  w_states --append --tstate=NEW_TS,0.2,0.7

Notes on behavior
- Basis states are required to sum to unity; if not, the system rescales all probabilities to enforce unity and logs a warning.
- The command supports reading states from files and/or direct on-the-command-line specifications.
- Appending basis states is not supported; use --bstate-file combined with --replace (or --show to inspect) instead.
