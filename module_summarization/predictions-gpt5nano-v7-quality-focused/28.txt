WMultiWest: Merging WESTPA Simulations with Reweighting

Overview
- w_multi_west implements combining multiple WESTPA simulations, with optional reweighting.
- It reads individual WESTPA HDF5 files, aggregates their data, and writes a combined output to a single HDF5 file.
- The tool supports combining auxiliary datasets, and optionally skipping reweighting or attempting to merge ibstates datasets.

Command name
- w_multi_west

Key features
- Aggregates results from multiple simulations (via WESTMultiTool infrastructure).
- Writes results to a single output file (default: multi.h5).
- Supports combining user-specified auxiliary datasets or all auxiliaries.
- Provides an option to disable reweighting.
- Attempts to combine ibstates if requested.
- Includes internal helpers for binning data and converting state records.

Usage synopsis
- w_multi_west [options]
- Common options include:
  - -o, --output-file: output HDF5 file name (default: multi.h5)
  - -W, --west, --WEST_H5FILE: name of the main.h5 file within each simulation directory (default: west.h5)
  - -a, --aux: names of additional auxiliary datasets to be combined (repeatable)
  - -aa, --auxall: combine all auxiliary datasets (default: False)
  - -nr, --no-reweight: do not apply reweighting (default: False)
  - -ib, --ibstates: attempt to combine ibstates dataset (default: False)

Options and arguments
- Output file
  - -o, --output-file
  - Description: The name of the output file to store results in.
  - Default: multi.h5
- West input file
  - -W, --west, --WEST_H5FILE
  - Description: The name of the main.h5 file inside each simulation directory.
  - Default: west.h5
- Auxiliary datasets
  - -a, --aux
  - Description: Names of additional auxiliary datasets to be combined.
  - Behavior: Can be supplied multiple times (action='append').
- All auxiliaries
  - -aa, --auxall
  - Description: Combine all auxiliary datasets. Default: False.
- Reweighting
  - -nr, --no-reweight
  - Description: Do not reweight. Default: False.
- ibstates
  - -ib, --ibstates
  - Description: Attempt to combine ibstates dataset. Default: False.

Internal data and helpers (conceptual)
- get_bin_mapper(we_h5file, hashval)
  - Purpose: Retrieve a bin mapper for a given hash value from the /bin_topologies group in a WESTPA HDF5 file.
  - Behavior:
    - Accepts a hash value or its hex digest.
    - Looks up index and pickles; unpickles the mapper and returns it.
    - Raises KeyError if the binning group or entry is not found.
  - Notes: This function is used to map binning information during reweighting or binning operations.
- create_idtype_array(input_array)
  - Purpose: Create a new array with istate_dtype while preserving existing data.
  - Behavior:
    - Allocates a new array with the same shape and istate_dtype.
    - Copies all fields from the input array.
    - Sets basis_auxref to empty bytes for all entries.
- Segment (inner class)
  - Purpose: Lightweight container for walker segment information during processing.
  - Fields:
    - weight: walker weight
    - iteration: iteration index
    - simid: simulation identifier
    - recycled_in: recycling information for the segment

File I/O and processing flow
- open_files
  - Opens the output file using WESTPAH5File with write mode.
  - Writes creator metadata via stamp_creator_data.
  - Builds a list ofWESTPA HDF5 files from the provided simulation paths via generate_file_list([self.west]).
  - Stores the mapping in self.westH5 for later processing.
  - Notes: The code indicates preparation for iteration counting, but explicit computation is not shown in the snippet.
- process_args
  - Maps command-line arguments into internal attributes:
    - self.output_file and self.output_file_name
    - self.west (path or identifier for the west.h5 in each sim)
    - self.sims (list of simulations)
    - self.aux and self.auxall (auxiliary datasets to combine)
    - self.reweight (inverted flag from no_reweight)
    - self.ibstates (whether to attempt ibstates combination)
  - Also passes through progress-related settings via self.progress.process_args(args)
- total_number_of_walkers
  - Computes total walkers across all loaded west HDF5 files.
  - Aggregates based on west['summary'] datasets, handling potential shape differences.
  - Result stored in self.total_walkers across iterations.
- go
  - Initializes a progress indicator from self.progress.indicator.
  - Sets self.istates = True, asserting that istates are assumed identical across runs.
  - Enters a progress context and begins progress tracking with pi.new as the starting point.

Implementation notes and limitations
- The provided code is a partial implementation:
  - The main data fusion and reweighting logic is not shown in the snippet.
  - go() currently only initializes progress and a flag; the actual merging workflow is not implemented here.
- The tool relies on base class functionality from WESTMultiTool for handling multiple simulations and I/O orchestration.
- The handling of auxiliary datasets, reweighting, and ibstates merging is determined by user-supplied options and the broader WESTPA framework integration.

Practical examples
- Combine two simulations with default settings (output to multi.h5)
  - w_multi_west -o multi.h5
  - Notes: Specify the simulations via theWESTMultiTool’s base behavior (not fully shown in this snippet). Use -W to set the per-simulation west.h5 if needed.
- Combine with all auxiliary datasets and disable reweighting
  - w_multi_west -o multi.h5 -aa -nr
- Include ibstates in the merge
  - w_multi_west -o multi.h5 -ib

Notes for developers
- The class WMultiWest inherits from WESTMultiTool and leverages a progress component for user feedback.
- The internal Segment class and helper functions provide a foundation for accumulating and organizing data during the merge process.
- To extend functionality, implement the core merging and reweighting logic within go() or via additional methods, using self.westH5 as the source data and self.output_file as the destination.
