Transforms
==========

This module provides utilities for converting between image formats (PIL, NumPy arrays) and PyTorch tensors, as well as helpers for querying image dimensions and channel information. It includes an enumeration for common interpolation modes and lightweight, internal utilities used by image-processing transforms.

API reference
--------------

InterpolationMode
~~~~~~~~~~~~~~~~~

Enumeration of supported interpolation methods. The values align with common image-processing terminology and are used by transforms that resize or resample images.

- NEAREST
- NEAREST_EXACT
- BILINEAR
- BICUBIC
- BOX
- HAMMING
- LANCZOS

Examples

- Access the enum:
  from torchvision.transforms import InterpolationMode

- Use with mappings to PIL-compatible codes:
  # internal usage example (not typically called by end users)
  from torchvision.transforms import pil_modes_mapping

get_dimensions(img)
~~~~~~~~~~~~~~~~~~

Returns the dimensions of an image as [channels, height, width].

- img: A PIL Image or a Torch Tensor.
- Returns: A list of integers [C, H, W].

Notes:
- If img is a Tensor, the function delegates to the tensor helper F_t.get_dimensions.
- If img is a PIL Image, it uses the PIL-based helper F_pil.get_dimensions.

get_image_size(img)
~~~~~~~~~~~~~~~~~~

Returns the size of an image as [width, height].

- img: A PIL Image or a Torch Tensor.
- Returns: A list of two integers [W, H].

Notes:
- For Tensor inputs, delegates to F_t.get_image_size.
- For PIL Images, delegates to F_pil.get_image_size.

get_image_num_channels(img)
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Returns the number of channels in an image.

- img: A PIL Image or a Torch Tensor.
- Returns: Integer number of channels.

Notes:
- For Tensor inputs, delegates to F_t.get_image_num_channels.
- For PIL Images, delegates to F_pil.get_image_num_channels.

to_tensor(pic)
~~~~~~~~~~~~~

Converts a PIL Image or NumPy array to a Torch Tensor.

- pic: PIL Image or NumPy ndarray. This function does not support TorchScript.
- Returns: A Torch Tensor with shape [C, H, W] and dtype matching the default floating type.
  - If the input is a NumPy array:
    - If 2D, a channel dimension is added (H, W, 1).
    - The array is transposed from HWC to CHW.
    - If the resulting tensor is ByteTensor, it is cast to the default float dtype and scaled to [0, 1].
  - If the input is a PIL Image:
    - Uses a mode-to-numpy dtype mapping to convert to a NumPy array, then to a Tensor.
    - For mode "1" (binary images), scales by 255.
    - Ensures the Tensor has shape [C, H, W].
    - If the Tensor is ByteTensor, scales to [0, 1] using the default float dtype.
  - If accimage.Image is available and the input is an accimage Image, converts via a NumPy intermediate.

- Examples:
  - PIL Image to tensor
    from PIL import Image
    from torchvision.transforms import to_tensor
    img = Image.open("path/to/image.jpg").convert("RGB")
    t = to_tensor(img)
    print(t.shape)  # torch.Size([3, H, W])
    print(t.dtype)

  - NumPy array to tensor
    import numpy as np
    arr = np.random.randint(0, 256, (64, 64, 3), dtype=np.uint8)
    t = to_tensor(arr)
    print(t.shape)  # torch.Size([3, 64, 64])
    print(t.dtype)

pil_to_tensor(pic)
~~~~~~~~~~~~~~~

Converts a PIL Image to a Tensor of the same type as the input image (i.e., the tensor preserves the input image’s dtype when feasible).

- pic: PIL Image
- Returns: Tensor with the same underlying type as the input image (e.g., uint8 for typical PIL RGB images).

Notes
- A deep copy of the underlying array is performed.
- This function is intended for cases where you want to preserve the input dtype, rather than scaling to [0, 1].

Internal utilities (usage notes)
--------------------------------

- _is_pil_image: Internal helper to check if an object is a PIL Image.
- _is_numpy: TorchScript-ignored helper to check if an object is a NumPy array.
- _is_numpy_image: TorchScript-ignored helper to check if a NumPy array has 2 or 3 dimensions.

Usage examples
--------------

Common scenarios for developers integrate this module into their image processing pipelines.

- Convert images to tensors for model input
  from PIL import Image
  from torchvision.transforms import to_tensor

  img = Image.open("sample.jpg").convert("RGB")
  x = to_tensor(img)
  print(x.shape)  # torch.Size([3, H, W])
  print(x.dtype)  # e.g., torch.float32

- Inspect image dimensions and channels in a mixed API
  import torch
  from PIL import Image
  from torchvision.transforms import get_dimensions, get_image_size, get_image_num_channels

  pil_img = Image.new("RGB", (128, 64), color=(255, 0, 0))
  # PIL input
  dims = get_dimensions(pil_img)           # [3, 64, 128]
  size = get_image_size(pil_img)             # [128, 64]
  chans = get_image_num_channels(pil_img)    # 3

  # Tensor input
  t = torch.zeros(3, 64, 128)
  dims2 = get_dimensions(t)                 # [3, 64, 128]

Notes
-----

- These helpers support both PIL Images and Torch Tensors. They route to the appropriate backend based on the input type.
- The to_tensor function does not support TorchScript; use pil_to_tensor if you need a TorchScript-compatible path in your workflow.
