Getting Started
===============

This guide helps you begin exploring Mitogen’s core internals used by master and child processes. It focuses on practical steps and small, tangible code examples you can run locally.

Prerequisites
-------------
- Python 3.x (Mitogen’s code is written to be compatible across Python versions; Python 3 is recommended)
- A clone of the Mitogen repository or access to the Mitogen package you will develop against

Installation
------------
- Install in editable mode (recommended for development):
  .. code-block:: sh

     pip install -e .

- Alternatively, install normally:
  .. code-block:: sh

     python setup.py install

What you’ll learn
------------------
- How Mitogen detects the runtime environment, including the OS and Python details
- How core source code is prepared and shared with child contexts
- How pre-exec hooks are intended to run before starting child processes
- OS-specific considerations, such as SELinux and PTY limits

Key Concepts and Quick Dites
---------------------------

1) sys.executable handling
- get_sys_executable() returns the path to the Python executable if it is set.
- If sys.executable is not set, it falls back to '/usr/bin/python' and logs a warning using SYS_EXECUTABLE_MSG.

Example:
.. code-block:: python

   from mitogen.core import get_sys_executable
   print(get_sys_executable())

2) SELinux awareness
- The module checks SELinux state at import time to decide whether to use socketpairs.
- SELINUX_ENABLED is a boolean that indicates if SELinux is enforcing.

Example:
.. code-block:: python

   from mitogen.core import SELINUX_ENABLED
   print("SELinux enforcing:", SELINUX_ENABLED)

3) Linux PTY constants
- Mitogen defines Linux PTY related constants to interact with pseudo-terminals.
- These are exposed as LINUX_TIOCGPTN and LINUX_TIOCSPTLCK.

Example:
.. code-block:: python

   from mitogen.core import LINUX_TIOCGPTN, LINUX_TIOCSPTLCK
   print(LINUX_TIOCGPTN, LINUX_TIOCSPTLCK)

4) Pre-exec hook
- _preexec_hook is a hook that can be invoked prior to exec() when starting a new child process.
- This is intended for internal scheduling or environment setup just before the child runs.

Example:
.. code-block:: python

   from mitogen.core import _preexec_hook

   def my_preexec():
       pass  # setup before child process starts

   _preexec_hook = my_preexec

5) Core source and minification
- _get_core_source() returns the Mitogen core source. In master contexts this is replaced to perform minification directly.
- get_core_source_partial() prepares a PartialZlib object containing a compressed version of the core source. This is used to lazily transfer core code to child contexts.

Examples:
.. code-block:: python

   from mitogen.core import _get_core_source, get_core_source_partial

   core_src = _get_core_source()
   print("Core source length:", len(core_src))

   partial = get_core_source_partial()
   print("Partial core prepared:", type(partial).__name__)

6) Common messages
- BROKER_SHUTDOWN_MSG communicates that a broker has begun shutdown and the connection is cancelled.
- OPENPTY_MSG provides guidance when PTY creation fails due to system limits.

Examples:
.. code-block:: python

   from mitogen.core import BROKER_SHUTDOWN_MSG, OPENPTY_MSG
   print(BROKER_SHUTDOWN_MSG)
   print(OPENPTY_MSG)

Getting hands-on: a small exploration script
--------------------------------------------
Run a small Python script to exercise a few of the items above and confirm your environment is wired for Mitogen internals:

.. code-block:: python

   from mitogen.core import get_sys_executable, SELINUX_ENABLED
   from mitogen.core import LINUX_TIOCGPTN, LINUX_TIOCSPTLCK
   from mitogen.core import _preexec_hook, get_core_source_partial

   print("sys.executable:", get_sys_executable())
   print("SELinux enabled:", SELINUX_ENABLED)

   print("PTY constants:", LINUX_TIOCGPTN, LINUX_TIOCSPTLCK)

   # Set a simple pre-exec hook (no-op example)
   def example_preexec():
       pass
   _preexec_hook = example_preexec

   partial = get_core_source_partial()
   print("Partial core type:", type(partial).__name__)

Notes for developers
--------------------
- The internal _preexec_hook and _get_core_source() are intended for internal use by the master/child initialization flow. Do not rely on them for public API guarantees.
- If your environment lacks a valid Python executable path, get_sys_executable() gracefully falls back to '/usr/bin/python' with a warning.
- On systems with SELinux enforcing, Mitogen avoids using socketpairs to maintain compatibility.
