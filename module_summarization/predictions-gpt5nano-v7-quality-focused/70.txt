Kernel approximation
====================

This module provides several approximate kernel feature maps based on Fourier transforms and Count Sketches. The primary goal is to enable scalable kernel methods by mapping inputs into a finite-dimensional feature space that approximates kernel evaluations.

Public API
----------

PolynomialCountSketch
^^^^^^^^^^^^^^^^^^^^^

Approximate feature map for the polynomial kernel using Tensor Sketch.

Summary
- Approximates the polynomial kernel K(X, Y) = (gamma * <X, Y> + coef0)^degree
- Uses Count Sketches and Fast Fourier Transforms (FFT) to efficiently compute the sketch of outer products
- Produces a fixed-dimensional feature representation of input samples

Parameters
- gamma : float, default=1.0
  Parameter of the polynomial kernel whose feature map is approximated. Non-negative.

- degree : int, default=2
  Degree of the polynomial kernel to approximate.

- coef0 : float, default=0
  Constant term of the polynomial kernel. See the polynomial kernel formula above.

- n_components : int, default=100
  Dimensionality of the output feature space. Typically, n_components should exceed the number of input features for good performance. A common guideline is roughly 10 times the number of input features, but the optimal value depends on the dataset and task.

- random_state : int, RandomState instance, default=None
  Seed for random number generation used to initialize the hash functions (indexHash) and sign hash (bitHash). Pass an int for reproducible results.

Attributes
- indexHash_ : ndarray of shape (degree, n_features), dtype=int64
  Array of indices in [0, n_components) used to represent the 2-wise independent hash functions for Count Sketch computation.

- bitHash_ : ndarray of shape (degree, n_features), dtype=float32
  Array with random entries in {+1, -1}, used to represent the 2-wise independent sign hashes for Count Sketch computation.

- n_features_out_ (internal): int
  The dimensionality of the output feature space (equals n_components). Exposed in some APIs as the number of features generated.

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit, if the input data provides feature names (strings). Present only when available.

Notes
- The class inherits from mixins that implement the standard fit/transform API.
- The internal hash indices and signs are initialized in fit using a provided random state.
- The transform method returns a transformed feature matrix of shape (n_samples, n_components).

Methods
- fit(X, y=None)
  Fit the transformer to the data X. Validates X, initializes random state, and constructs the hash indices and signs used for the Count Sketch. Returns self.

  Behavior details:
  - Validates X as a sparse CSC or dense array.
  - Computes n_features from X; if coef0 != 0, increments n_features by 1 to account for the constant term.
  - Initializes indexHash_ with random integers in [0, n_components) of shape (degree, n_features).
  - Initializes bitHash_ with random choices of -1 or +1 of shape (degree, n_features).
  - Sets the output feature dimensionality to n_components.

- transform(X)
  Generate the approximate kernel feature map for X.
  Returns an array-like of shape (n_samples, n_components) containing the transformed features.

Examples
- Basic usage with a small dataset:

  from sklearn.kernel_approximation import PolynomialCountSketch
  from sklearn.linear_model import SGDClassifier

  X = [[0, 0], [1, 1], [1, 0], [0, 1]]
  y = [0, 0, 1, 1]

  ps = PolynomialCountSketch(degree=3, random_state=1)
  X_features = ps.fit_transform(X)

  clf = SGDClassifier(max_iter=10, tol=1e-3)
  clf.fit(X_features, y)
  score = clf.score(X_features, y)
  print(score)  # 1.0

See Also
--------

- AdditiveChi2Sampler: Approximate feature map for additive chi-squared kernel.
- Nystroem: Approximate a kernel map using a subset of the training data.
- RBFSampler: Approximate an RBF kernel feature map using random Fourier features.

Implementation notes
--------------------

- The implementation relies on FFT-based Tensor Sketching to efficiently approximate polynomial kernels.
- Parameter validation and fit-context handling follow scikit-learn conventions.
