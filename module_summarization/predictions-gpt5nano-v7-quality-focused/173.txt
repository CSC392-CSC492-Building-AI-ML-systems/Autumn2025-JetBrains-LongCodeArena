Fitting SESANS Data
====================

This document describes how to fit SESANS data using sasmodels and, in particular, how to identify and manage the parameters involved in SESANS models. The primary focus here is on the parameter-discovery utility that helps you plan and execute fits by understanding which parameters appear in which models.

Overview
--------

SESANS fitting relies on models that describe the spin-echo small-angle neutron scattering signal as a function of system parameters (volumes, orientations, and scattering-length-density contrasts, among others). Before performing fits, you often need to know:

- which parameters exist across available SESANS models
- which models use a given parameter
- the kind of each parameter (volume, orientation, sld)

The sasmodels.list_pars utility provides this information.

Parameter discovery utility
---------------------------

What it does
- Lists all parameters used by models available to sasmodels.
- Optionally associates each parameter with the models that use it.
- Supports filtering by parameter kind (volume, orientation, sld) or no filtering.

Key functions (from sesans_fitting.rst perspective)
- find_pars(kind=None): returns a dictionary {parameter_name: [model, model, ...]} for all models, optionally filtered by kind.
- list_pars(names_only=True, kind=None): prints either just the parameter names or name: model list, and uses find_pars under the hood.
- main(): command-line entry point that parses arguments and invokes list_pars accordingly.

CLI usage
- List all parameter names (default): is useful for quick scanning.
  - Command: python -m sasmodels.list_pars
- List parameters with the models that use them (verbose):
  - Command: python -m sasmodels.list_pars -v
- Filter by parameter kind (volume, orientation, sld) or none/any:
  - Command: python -m sasmodels.list_pars -v volume
  - Command: python -m sasmodels.list_pars -v sld
  - Command: python -m sasmodels.list_pars       (default, no filtering)
  - Note: the kind option accepts volume, orientation, sld, none, or any (the default is any). The code maps any/none to an unconstrained filter.

Output
- Names-only mode (default): prints a sorted list of parameter names, one per line.
- Verbose mode (-v): prints lines of the form
  parameter_name: model1, model2, ...
  showing which models use each parameter.
- The underlying data structure is a mapping from parameter name to the list of models that reference it.

Where this helps with SESANS fitting
- Before fitting, identify which parameters exist across SESANS models and which models share a given parameter.
- Use the output to:
  - Decide which parameters to vary during fitting
  - Understand cross-model parameter usage to guide model selection
  - Prepare fixed vs. free parameters for robust SESANS fits

Practical examples
- Find all parameter names used by any SESANS model:
  - python -m sasmodels.list_pars
- Find parameters and the models that use them:
  - python -m sasmodels.list_pars -v
- Focus on volume-related parameters and see which models use them:
  - python -m sasmodels.list_pars -v volume
- Focus on sld-related parameters (scattering length density):
  - python -m sasmodels.list_pars -v sld

Implementation notes
- The tool iterates over all models returned by list_models(), loads model information with load_model_info(name), and inspects each model's kernel_parameters.
- It collects parameter.name and parameter.type, including all models that reference each parameter.
- The output relies on the parameter objects having attributes:
  - name: the parameter’s name
  - type: the parameter kind (volume, orientation, sld, etc.)
- The CLI uses the Python standard library argparse to parse -v/--verbose and the kind option.

References to related commands
- List models: sasmodels.list_models
- Show model information: sasmodels.core.load_model_info
- Compare and format output: sasmodels.compare.columnize

Code excerpts (for reference)
- Parameter discovery (simplified):
  - For each model in list_models():
      model_info = load_model_info(name)
      for p in model_info.parameters.kernel_parameters:
          if kind is None or p.type == kind:
              partable.setdefault(p.name, [])
              partable[p.name].append(name)

- Command-line interface (simplified):
  - parse -v/--verbose
  - parse kind (volume, orientation, sld, none, any)
  - call list_pars(names_only=not verbose, kind=kind)

Example minimal session
- Discover parameter space for SESANS fitting:
  - python -m sasmodels.list_pars
- See per-parameter model usage for planning fits:
  - python -m sasmodels.list_pars -v
- Narrow to volume-related parameters:
  - python -m sasmodels.list_pars -v volume
