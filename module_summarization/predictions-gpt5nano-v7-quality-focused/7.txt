Using lists
===========

Overview
--------

Python lists accumulate items and are used throughout the API to provide multiple values to parameters. In this documentation, we focus on how lists are used in the ftrack_api Session construction, particularly for plugin discovery.

Key point
---------

- The Session constructor accepts plugin_paths as a list of strings. If not provided, the discovery mechanism defaults to using the environment variable FTRACK_EVENT_PLUGIN_PATH.

Creating lists
--------------

- Basic creation:
  - my_list = [1, 2, 3]
- Empty list:
  - empty = []
- List of strings:
  - names = ["Alice", "Bob", "Charlie"]

Working with lists in the Session context
-----------------------------------------

plugin_paths
~~~~~~~~~~~~

- Type: list of strings
- Purpose: Paths to search for plugins during discovery
- Default behavior: If not specified, the system looks up FTRACK_EVENT_PLUGIN_PATH (via the discovery mechanism)

Example: providing multiple plugin paths

.. code-block:: python

   from ftrack_api import Session

   session = Session(
       server_url="https://ftrack.example.com",
       api_key="your_api_key",
       api_user="your_username",
       plugin_paths=[
           "/path/to/first/plugins",
           "/path/to/second/plugins"
       ]
   )

Best practices and tips
-----------------------

- If you need a single path, pass it as a one-element list:
  - plugin_paths=["/path/to/plugins"]
- If you want the default behavior, omit plugin_paths and rely on the environment variable.

Common patterns
---------------

- When a parameter expects a list, you can construct the list in advance and pass it as shown above.
- If you have a generator or other iterable, convert it to a list before passing to parameters that require a list:
  - path_iter = get_plugin_paths()
  - session = Session(..., plugin_paths=list(path_iter))

Examples
--------

- Minimal session using defaults for plugin paths:
  .. code-block:: python

     from ftrack_api import Session

     session = Session(
         server_url="https://ftrack.example.com",
         api_key="your_api_key",
         api_user="your_username"
     )

- Session with a custom list of plugin paths:
  .. code-block:: python

     session = Session(
         server_url="https://ftrack.example.com",
         api_key="your_api_key",
         api_user="your_username",
         plugin_paths=["/plugins/custom", "/opt/plugins"]
     )
