Probability calibration with CalibratedClassifierCV
====================================================

This documentation describes probability calibration using CalibratedClassifierCV. The class provides calibrated probability estimates for a base classifier by fitting a calibration model on top of the classifier’s outputs. Calibration can be performed using cross-validated predictions or with a pre-fit estimator.

Overview
--------

- Goal: produce better probability estimates (predict_proba) for classifiers whose raw outputs are not well-calibrated.
- Calibration methods:
  - sigmoid: Platt’s method (logistic regression) on the classifier scores or decision function.
  - isotonic: non-parametric isotonic regression.
- Cross-validation:
  - ensemble = True (default): for each cv split, fit a copy of the base estimator on the training split and calibrate on the testing split; predictions are averaged across folds.
  - ensemble = False: use cross_val_predict to obtain unbiased predictions for calibration; use the base estimator trained on all data for final predictions.
- Prefit mode:
  - cv = "prefit" calibrates an already-fitted estimator. No cross-validation is performed; all data are used for calibration. The user must ensure that the data used to train the estimator and to calibrate are disjoint.

Class
-----

CalibratedClassifierCV
----------------------

Probability calibration with isotonic regression or logistic regression.

- estmator: The base classifier to calibrate. Default is a LinearSVC when not provided.
- method: {'sigmoid', 'isotonic'}, default 'sigmoid'
  - 'sigmoid' implements Platt’s method (logistic regression on scores).
  - 'isotonic' implements a non-parametric isotonic calibration.
  - Isotonic calibration can overfit with very small calibration samples (<< 1000).
- cv: cross-validation strategy.
  - None or an integer: uses 5-fold cross-validation by default.
  - CV splitter or an iterable of (train, test) indices: use the provided splits.
  - "prefit": estimator is already fitted; no cross-validation is performed and all data are used for calibration.
  - If cv is None and y is binary or multiclass, StratifiedKFold is used; otherwise KFold is used.
- n_jobs: number of parallel jobs. Parallelism happens only when cv != "prefit".
- ensemble: whether to fit the estimator on training data and calibrate on testing data for each cv fold (True) or to use cross_val_predict for calibration (False).
- base_estimator: deprecated alias for estimator (use estimator instead).

Attributes
----------

- classes_: array-like of shape (n_classes,)
  The class labels discovered during fit.
- n_features_in_: int
  Number of features seen during fit.
- feature_names_in_: array-like of shape (n_features_in_,)
  Feature names seen during fit (if the base estimator provides them).
- calibrated_classifiers_: list
  The calibrated classifiers used (length equals the number of cv folds, or 1 if cv="prefit" or ensemble=False).

How it works
------------

- Calibration target:
  - If the base estimator provides a decision_function, calibration uses that output; otherwise, it uses predict_proba.
- Training/calibration flow:
  - ensemble = True:
    - For each fold in the cross-validation split:
      - Train a fresh copy of the base estimator on the training subset.
      - Calibrate its outputs using the held-out testing subset.
    - Predictions on new data are obtained by averaging the calibrated probabilities across folds.
  - ensemble = False:
    - Compute unbiased predictions with cross_val_predict to calibrate.
    - The final predictor uses the base estimator trained on all data for inference.
- Prefit mode:
  - Use a pre-fitted estimator and calibrate on the provided data without cross-validation.

Practical usage
---------------

- Typical workflow (default ensemble behavior):
  - Fit a base classifier with calibration.
  - Predict calibrated probabilities with predict_proba.
  - Example: calibrate a linear SVM (which does not natively output calibrated probabilities) using sigmoid calibration.

Code example
------------

Basic usage with a LinearSVC base estimator and sigmoid calibration:

.. code-block:: python

   from sklearn.svm import LinearSVC
   from sklearn.calibration import CalibratedClassifierCV

   # Base estimator
   base_clf = LinearSVC()

   # Calibrate probabilities using 5-fold cross-validation
   calibrated_clf = CalibratedClassifierCV(base_clf, method='sigmoid', cv=5)

   # Fit on training data
   calibrated_clf.fit(X_train, y_train)

   # Predict calibrated probabilities
   proba = calibrated_clf.predict_proba(X_test)

Prefit usage
-------------

If you already have a fitted estimator and want to calibrate it without cross-validation:

.. code-block:: python

   est = LinearSVC().fit(X_train, y_train)
   calib = CalibratedClassifierCV(est, method='sigmoid', cv='prefit')
   calib.fit(X_cal, y_cal)  # calibration on a separate set; ensure data are disjoint

Notes
-----

- The default behavior (ensemble=True) yields more stable probability estimates by averaging over folds, at the cost of fitting multiple models.
- The choice of method depends on data size and risks of overfitting:
  - Sigmoid is parametric and often works well with moderate data sizes.
  - Isotonic is non-parametric and can overfit with very small calibration sets.

Examples of common scenarios
----------------------------

- Calibrating a classifier that does not provide reliable probability estimates:
  - Use method='sigmoid' with cross-validated calibration.
- When you have a pretrained model and want to calibrate it on a separate calibration set:
  - Use cv='prefit' to avoid re-fitting the model.
