w_eddist
===========

Calculate time-resolved transition-event duration distribution from kinetics results.

Overview
--------
- w_eddist computes a histogram-based representation of durations extracted from kinetics results.
- It operates on the results produced by w_kinetics trace and supports parallel processing by default.

Source data
-----------
- Source data come from the results of w_kinetics trace (see w_kinetics trace --help for details on generating this data).
- A DurationDataset wrapper handles integration of masks into per-iteration data, ensuring that only masked segments contribute to the histogram for a given iteration.

Histogram binning
-----------------
- By default, histograms use 100 bins per dimension.
- Binning can be overridden with -b/--bins. The argument may be provided as:
  - a single integer N: N uniformly spaced bins in each dimension
  - a comma-separated sequence N1,N2,...: N1 bins for the first dimension, N2 for the second, etc.
  - a list of lists [[B11, B12, B13,...], [B21, B22, B23,...], ...]:
    explicit bin boundaries for each dimension. These boundaries need not be evenly spaced.
    The list-of-lists form is evaluated via Python's eval, with np available.
- The first two forms (integer or list of integers) trigger a scan of input data to determine min/max values per dimension, which can be expensive for large datasets.
- The list-of-lists form provides explicit boundaries and avoids data-range scanning.

Output format
-------------
The output file (specified with -o/--output; default: pdist.h5) contains:
- histograms
  - Normalized histograms. The first axis corresponds to iteration; remaining axes correspond to input-dataset dimensions.
- /binbounds_0, /binbounds_1, ...
  - Datasets containing bin boundaries for each dimension.
- /midpoints_0, /midpoints_1, ...
  - Datasets containing bin midpoints for each dimension.
- n_iter
  - Vector of iteration numbers that correspond to the stored histograms (i.e., the first axis of the histograms).

Subsequent processing
---------------------
- The produced file can be plotted with plothist. Use plothist --help for details.

Parallelization
----------------
- Parallel binning and input-data reading are supported and enabled by default.
- For simple scenarios (e.g., reading pre-computed input data or small numbers of segments), serial processing (--serial) may be faster.

How it works (implementation highlights)
----------------------------------------
- The code defines helper routines for remote processing:
  - _remote_min_max(ndim, dset_dtype, n_iter, dsspec):
    Computes per-dimension min and max values across the data available for a given iteration.
  - _remote_bin_iter(iiter, n_iter, dsspec, wt_dsspec, initpoint, binbounds, ignore_out_of_range):
    For a given iteration, builds a per-iteration histogram by scanning time-slice data and applying histnd with the provided weights. The histogram is then normalized with normhistnd.
- A DurationDataset wraps the underlying dataset with a mask, presenting, per iteration, a data array of shape (nsegs, 1, 1) or None if no segments are present.
- Binning uses the binbounds specification to determine the histogram axes. The number of axes equals the number of dimensions in the input data.

Command-line options
--------------------
- -o, --output
  - Specifies the output file (default: pdist.h5).
- -b, --bins
  - Controls histogram binning as described above.
- --serial
  - Forces serial (non-parallel) execution.

Examples
--------
Basic usage (default settings):
.. code-block:: bash

   w_eddist -o pdist.h5

Uniform bins across all dimensions (100 bins per dimension by default):
.. code-block:: bash

   w_eddist -o pdist.h5 -b 100

Explicit per-dimension bins (example; the exact numbers depend on your data):
.. code-block:: bash

   w_eddist -o pdist.h5 -b [[0, 1, 2, 3], [0, 0.5, 1.0, 1.5, 2.0]]

Explicit, nonuniform bin boundaries are evaluated with Python's eval (np is available):
.. code-block:: bash

   w_eddist -o pdist.h5 -b "[[0, 1, 3, 6], [0, 2, 5]]"

Serial processing (may be faster for some workloads):
.. code-block:: bash

   w_eddist --serial -o pdist.h5

Notes
-----
- The tool reads source data from the kinetics results produced by w_kinetics trace.
- The duration data are filtered per iteration using a mask, as implemented by DurationDataset.
- The output is designed to be fed to plothist for visualization and further analysis.
