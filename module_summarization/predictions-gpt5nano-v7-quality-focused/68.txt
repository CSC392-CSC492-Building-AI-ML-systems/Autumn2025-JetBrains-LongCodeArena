Kernels for Gaussian process regression and classification
=======================================================

This module provides a collection of kernels used for Gaussian process (GP) regression
and classification. Kernels define the covariance structure of the GP prior and can be
combined to form more expressive kernels.

Key features
------------
- Kernel engineering: kernels can be combined using addition (+), multiplication (*),
  and exponentiation (**). Scalar values are automatically converted to a constant kernel.
- Analytic gradients: all kernels support gradient-based hyperparameter optimization.
- Hyperparameter spaces: hyperparameters are specified by lower and upper bounds,
  forming a rectangular search space. Hyperparameters can also be declared as fixed,
  excluding them from optimization.

Core concepts
-------------
- Hyperparameters: Each kernel exposes hyperparameters that control its behavior
  (e.g., length scales, amplitudes). Hyperparameters are described by the Hyperparameter
  namedtuple, which includes the parameter name, type, bounds, number of elements,
  and whether the parameter is fixed.
- Bound specification: Bounds are provided as pairs of floats [lower, upper]. For
  vector-valued hyperparameters (n_elements > 1), bounds can be provided as a 2D array
  with one row per element. If bounds is the string "fixed", the hyperparameter becomes fixed
  and is not optimized.
- Anisotropic length scales: Length scales can be a vector with one entry per input
  dimension. Validation ensures the length_scale matches the data dimensionality.

Hyperparameter
--------------
The Hyperparameter class specifies a single hyperparameter for a kernel.

Attributes
- name (str): The hyperparameter name. For example, a kernel using a hyperparameter named
  "length_scale" must have attributes self.length_scale and self.length_scale_bounds.
- value_type (str): Type of the hyperparameter (currently only "numeric" is supported).
- bounds (array-like or str): A pair of floats (lower, upper) or "fixed" to mark the
  parameter as fixed. For vector-valued hyperparameters (n_elements > 1), bounds may be a
  2D array with n_elements rows.
- n_elements (int): Number of elements of the hyperparameter. 1 corresponds to a scalar
  hyperparameter; values > 1 indicate a vector-valued hyperparameter (e.g., anisotropic
  length-scales).
- fixed (bool): Whether the hyperparameter is fixed (not optimized). If None, it is inferred
  from bounds.

Example
>>> from sklearn.gaussian_process.kernels import Hyperparameter
>>> kernel = ConstantKernel(constant_value=1.0,
...    constant_value_bounds=(0.0, 10.0))
>>> for hyperparameter in kernel.hyperparameters:
...     print(hyperparameter)
Hyperparameter(name='constant_value', value_type='numeric',
bounds=array([[ 0., 10.]]), n_elements=1, fixed=False)

>>> params = kernel.get_params()
>>> for key in sorted(params): print(f"{key} : {params[key]}")
constant_value : 1.0
constant_value_bounds : (0.0, 10.0)

Notes
- Hyperparameters are exposed via the kernel’s API (e.g., kernel.hyperparameters) and
  can be inspected or altered for customized optimization.
- The Hyperparameter class is memory-efficient and designed to integrate with the
  kernel’s parameter handling.

Utility: _check_length_scale
----------------------------
The helper function _check_length_scale validates anisotropic length scales.

- It squeezes the input and converts to float.
- Ensures the length_scale is at most 1-dimensional.
- If length_scale is 1D, checks that its length matches the input dimensionality (X.shape[1]).
- Returns the validated length_scale.

Example
>>> import numpy as np
>>> X = np.random.randn(5, 3)
>>> length_scale = np.array([1.0, 2.0, 3.0])
>>> _check_length_scale(X, length_scale)
array([1., 2., 3.])

Kernels: base class
-------------------
Kernel is the abstract base class for all kernels.

- It provides a common interface for parameter handling and integration with
  hyperparameter optimization.
- Subclasses implement concrete kernel definitions (e.g., RBF, Matern, ConstantKernel)
  and support operator overloading for composition.

API overview
------------
- get_params(self, deep=True): Retrieve the kernel’s parameters as a dictionary.
  This introspects the kernel’s __init__ signature to identify parameter names and
  values. See the example in the Hyperparameter section for how parameters can be accessed.

Examples and usage
------------------
- Kernel composition
  - k = k1 + k2
  - k = k1 * k2
  - k = k1 ** 2.0
  - Scalar injection: k = k1 + 5  (scalar is converted to a ConstantKernel)

- Hyperparameter optimization
  - Define kernels with hyperparameters having bounds.
  - Use the rectangular search space to optimize hyperparameters via gradient-based methods.

- Accessing and inspecting parameters
  - params = kernel.get_params()
  - Print parameter names and values using a simple loop.

Notes
-----
- This module is inspired by the kernel module from the george package and
  follows a similar design philosophy: modular, composable kernels with
  transparent hyperparameter handling and optimization support.
- An implementation detail: the Hyperparameter structure is a memory-efficient
  namedtuple, intended to minimize the memory footprint of many kernels.

Examples
--------
- Anisotropic length-scale kernel construction (conceptual)
  >>> from sklearn.gaussian_process.kernels import RBF
  >>> X = np.random.randn(20, 3)
  >>> length_scale = np.array([1.0, 2.0, 3.0])  # one length-scale per feature
  >>> k = RBF(length_scale=length_scale)

- Composite kernel
  >>> k1 = RBF(length_scale=1.0)
  >>> k2 = ConstantKernel(constant_value=1.0, constant_value_bounds=(0.0, 5.0))
  >>> k = k1 + k2
  >>> k = (k1 * k2) ** 2.0
  >>> k  # doctest: +ELLIPSIS

Migration and compatibility
---------------------------
- The documented API follows the conventions used in scikit-learn-like Gaussian process kernels.
- Changes to hyperparameter handling may affect optimization behavior; always consult
  the latest API reference for details.
