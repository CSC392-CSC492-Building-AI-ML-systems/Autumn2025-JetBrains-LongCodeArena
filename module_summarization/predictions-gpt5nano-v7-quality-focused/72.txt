Metrics
=======

Overview
--------

This module provides internal utility functions used to validate and prepare inputs for pairwise distance computations. It focuses on ensuring that X and Y are properly shaped, typed, and compatible, and on selecting appropriate floating-point dtypes. These helpers are intended for use by higher-level distance metrics and are designed to be safe for both dense and sparse inputs.

API reference
-------------

_Returned values and behavior are described for each function as implemented in the module._

_return_float_dtype(X, Y)_
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Determine a safe floating-point dtype for pairwise computations and convert inputs as needed.

What it does
- If X (and potentially Y) are not sparse and not numpy arrays, converts them to numpy arrays as needed.
- Determines Y_dtype:
  - If Y is None, uses X.dtype.
  - If Y is not sparse and not a numpy array, converts Y to an array and uses Y.dtype.
  - Otherwise uses Y.dtype.
- Chooses the output dtype:
  - If both X and Y have dtype float32, dtype is float32.
  - Otherwise dtype is float (float64 on most platforms).
- Returns:
  - safe_X: possibly converted X
  - safe_Y: possibly converted Y (or a reference to X when Y is None or Y is X)
  - dtype: the selected floating-point dtype (np.float32 or float)

Syntax
.. code-block:: python

   safe_X, safe_Y, dtype = _return_float_dtype(X, Y)

Parameters
- X: array-like or sparse matrix (shape: n_samples_X × n_features)
- Y: array-like or sparse matrix, or None (shape: n_samples_Y × n_features)
  - If None, Y_dtype mirrors X.dtype
Returns
- safe_X: X (potentially converted to a numpy array if needed)
- safe_Y: Y (potentially converted), or a reference to X if Y is None or Y is X
- dtype: numpy.dtype, either np.float32 or float

Examples
.. code-block:: python

   import numpy as np
   from scipy.sparse import csr_matrix

   X = np.array([[1.0, 2.0], [3.0, 4.0]], dtype=np.float32)
   Y = None
   X2, Y2, dt = _return_float_dtype(X, Y)
   # X2 is X, Y2 is X, dt is np.float32

   X = np.array([[1.0, 2.0], [3.0, 4.0]], dtype=np.float64)
   Y = np.array([[5.0, 6.0]], dtype=np.float64)
   X2, Y2, dt = _return_float_dtype(X, Y)
   # X2 and Y2 are converted as needed, dt is float (float64)

_check_pairwise_arrays(X, Y, *, precomputed=False, dtype=None, accept_sparse="csr", force_all_finite=True, copy=False)_
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Prepare and validate X and Y for pairwise distance computations.

What it does
- Aligns X and Y to appropriate array formats, accounting for sparse inputs.
- Determines dtype:
  - If dtype is None, it uses the dtype inferred by _return_float_dtype(X, Y).
- If Y is X or Y is None, both X and Y are converted to arrays via check_array with the specified constraints.
- If Y is provided and distinct from X, both X and Y are individually validated via check_array with the same constraints.
- Validates shapes:
  - For precomputed distances: requires X.shape[1] == Y.shape[0]; otherwise raises a ValueError stating the mismatch.
  - For non-precomputed distances: requires X.shape[1] == Y.shape[1]; otherwise raises a ValueError with the relevant dimensions.
- Returns:
  - safe_X: X converted to an array (or Y’s reference if Y is None)
  - safe_Y: Y converted to an array (or a reference to X if applicable)

Parameters
- X: array-like or sparse matrix of shape (n_samples_X, n_features)
- Y: array-like or sparse matrix of shape (n_samples_Y, n_features)
- precomputed: bool, default False
  - True if X is treated as a precomputed distance matrix to the samples in Y
- dtype: data type, default None
  - If None, dtype is chosen by _return_float_dtype
- accept_sparse: {“csr”, “csc”, …}, bool or list/tuple of str, default 'csr'
  - Sparse formats that are accepted. If the input is sparse but not in an allowed format, it is converted to the first listed format. If True is passed, any sparse format is allowed. If False, sparse inputs raise an error.
- force_all_finite: bool or 'allow-nan', default True
  - Controls handling of infinite or NaN values:
    - True: all values must be finite
    - False: allows inf and NaN
    - 'allow-nan': allows NaN and pd.NA, but not infinite values
  - Note: pd.NA is converted to NaN for compatibility
- copy: bool, default False
  - If True, forces a copy during conversion; otherwise, copies may be avoided

Returns
- safe_X: array-like, shape (n_samples_X, n_features)
- safe_Y: array-like, shape (n_samples_Y, n_features) or a reference to X when appropriate

Examples
.. code-block:: python

   import numpy as np
   X = np.array([[1, 2], [3, 4]], dtype=np.float64)
   Y = None
   X2, Y2, dt = _return_float_dtype(X, Y)

   # Prepares X and Y for pairwise computations with default settings
   from metrics import check_pairwise_arrays
   X_valid, Y_valid = check_pairwise_arrays(X, Y, precomputed=False)

   # When using precomputed distances
   D = np.zeros((2, 2))
   X3, Y3 = check_pairwise_arrays(D, D, precomputed=True)

check_paired_arrays(X, Y)
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Validate inputs for paired distance metrics and ensure X and Y are proper arrays.

What it does
- Ensures that X and Y are arrays and are at least 2-D.
- Ensures their elements are valid numeric types (floats) suitable for distance computations.
- Serves as a preparatory validation step for paired distance metrics.

Returns
- X, Y: processed arrays suitable for subsequent distance calculations

Parameters
- X: array-like or sparse matrix
- Y: array-like or sparse matrix

Notes
- This function is intended to be called before performing paired distance computations to ensure inputs are well-formed and safe.
- Behavior aligns with the module’s general approach to input validation and dtype safety for numerical distances.

Examples
.. code-block:: python

   import numpy as np
   X = np.asarray([[1.0, 2.0], [3.0, 4.0]])
   Y = np.asarray([[5.0, 6.0]])
   X_valid, Y_valid = check_paired_arrays(X, Y)

See also
--------

- check_array (from utils) for general input validation and dtype handling
- _return_float_dtype for dtype selection logic used by pairwise computations

Notes
-----

- force_all_finite supports the string 'allow-nan' starting from version 0.22, which permits NaN values but disallows infinite values.
- pd.NA values are converted to NaN for compatibility.
- accept_sparse controls the handling and conversion of sparse inputs; True allows any sparse format, False disallows sparse inputs, and a list/tuple selects the allowed formats. If a sparse input is not in the allowed formats, it is converted to the first allowed format.
