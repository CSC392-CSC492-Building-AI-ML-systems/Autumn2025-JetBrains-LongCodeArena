Contributor Guide for Qiskit Metal
==================================

This guide describes how to contribute to Qiskit Metal, with emphasis on the analyses components (such as CPW calculations) and related numerical utilities used by components.

Overview
--------

- Goals: Enable professional developers to contribute new features, fixes, tests, and documentation in a clear, well-structured process.
- Scope: All sources under qiskit_metal, with particular focus on the analyses package (cpw_calculations, lumped_capacitive, and related utilities).

Getting started
---------------

Prerequisites
- Python 3.8+ and a working build environment.
- Familiarity with Git, Python packaging, and pytest.
- Basic knowledge of Sphinx/reST for documentation contribution.

Install from source (editable)
- Clone the repository and create a virtual environment.
- Install development extras (pytest, Sphinx, etc.) and the package in editable mode.

Example (bash)
- git clone https://github.com/Qiskit/qiskit-metal.git
- cd qiskit-metal
- python -m venv .venv
- source .venv/bin/activate  # on Windows use .venv\Scripts\activate
- pip install -e .[dev]

Running tests
- Run the full test suite: pytest -q
- Run only analyses-related tests: pytest tests/analyses -q
- To get a concise report, use pytest -q

Code structure overview (relevant to analyses)
- qiskit_metal/analyses
  - cpw_calculations: CPW-related calculations (e.g., guided_wavelength)
  - lumped_capacitive: Lumped-element equivalents for CPW geometries
- Each module provides small, focused numerical utilities used by components and tests.
- Public API should be documented, tested, and importable.

Contributing workflow
---------------------

1) Fork and branch
- Create a descriptive branch name, e.g. feat/cpw-guided-wavelength
- git checkout -b feat/cpw-guided-wavelength

2) Implement feature or fix
- Add or modify modules under qiskit_metal/analyses as appropriate.
- Ensure functions have clear docstrings (RST-style) and type hints where helpful.
- Example function signature (illustrative from CPW calculations):
  - guided_wavelength(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45)
  - lumped_cpw(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45, loss_tangent=1e-5, london_penetration_depth=30e-9)

3) Tests
- Write unit tests under tests/analyses/test_<module>.py
- Validate against known results or cross-check with alternate methods.
- Run tests locally: pytest tests/analyses -q

4) Documentation
- Update or add Sphinx-friendly docstrings for new/changed APIs.
- If adding a new module, export it in qiskit_metal/analyses/__init__.py and ensure autosummary entries exist in the relevant documentation pages.
- Example: autosummary entries may include:
  - cpw_calculations
  - lumped_capacitive

5) Documentation build
- Build docs locally to verify references:
  - make -C docs html
  - or sphinx-build -b html docs build/html

6) Submit
- Create a pull request with a concise summary of changes.
- Link tests and documentation updates in the PR description.
- Respond to feedback and iterate as needed.

Coding and documentation standards
----------------------------------

Code style
- Follow PEP 8 where applicable.
- Use clear, descriptive names for functions and variables.
- Keep functions small and focused; prefer pure functions with minimal side effects when possible.

Docstrings and API docs
- Provide thorough docstrings in reST format suitable for Sphinx.
- Include parameter descriptions, return values, and example usage where helpful.
- For public APIs, ensure examples demonstrate typical usage with real values.

Autosummary and docs integration
- When exposing modules under qiskit_metal.analyses, keep a clean export surface in __init__.py and update documentation autosummaries accordingly.
- Example entries (in docs):
  - .. autosummary::
      :toctree:

      cpw_calculations
      lumped_capacitive

Tests and quality
- Maintain a reasonable test coverage: unit tests for new functionality, integration tests if applicable.
- Ensure tests are deterministic; avoid relying on flaky external resources.
- Include regression tests for bug fixes.

Example of contributing to CPW-related utilities
-----------------------------------------------

- Add a module qiskit_metal/analyses/cpw_calculations.py implementing:
  - guided_wavelength(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45)
  - Returns lambdaG, etfSqrt, q (guided wavelength, effective dielectric constant, filling factor)
  - Docstring describes equations and usage

- Add a module qiskit_metal/analyses/lumped_capacitive.py implementing:
  - lumped_cpw(freq, line_width, line_gap, substrate_thickness, film_thickness, dielectric_constant=11.45, loss_tangent=1e-6, london_penetration_depth=30e-9)
  - Returns (Lk, Lext, C, G, Z0, etfSqrt**2, Cstar)
  - Include notes on units and applicability

- Update qiskit_metal/analyses/__init__.py to export new modules and align with autosummary in docs:
  - from .cpw_calculations import guided_wavelength, lumped_cpw
  - from .lumped_capacitive import lumped_cpw as lumped_capacitive

- Write tests that exercise basic input and verify output types and reasonable value ranges.

Code snippet (illustrative)
- Example usage of a contributed function:
  from qiskit_metal.analyses.cpw_calculations import guided_wavelength
  lambdaG, etfSqrt, q = guided_wavelength(
      freq=5e9,
      line_width=10e-6,
      line_gap=6e-6,
      substrate_thickness=760e-6,
      film_thickness=200e-9,
      dielectric_constant=11.45
  )

Documentation and licensing
---------------------------

- All contributions follow the project license (Apache 2.0) and include appropriate headers if files are modified.
- Follow the project’s contributor code of conduct and contribution guidelines.

Maintenance and contact
-----------------------

- Maintainers review PRs for correctness, style, tests, and documentation.
- For questions, raise an issue on the repository with a concise description and, if possible, a reproducible example.

Appendix: Quick pointers for contributors
- Keep changes small and focused.
- Write or update unit tests for any new functionality.
- Document new APIs with clear examples.
- Ensure documentation autosummaries reflect new or updated modules.
- Run the full test suite before submitting a PR.

Endnotes
--------

- The analyses package hosts numerical tools used by CPW-based components and quantum device simulations. This guide helps contributors add, modify, and validate these tools in a consistent, well-documented manner.
