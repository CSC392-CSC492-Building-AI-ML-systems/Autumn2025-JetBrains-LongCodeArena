MapPlotter
==========

Overview
--------
MapPlotter provides an interactive viewer to plot a 2D map (plane) extracted from a data cube and to interactively connect pixels to spectrum plots. It is designed for exploratory analysis rather than publication-quality plots, but it can leverage publication-quality backends when available (APLpy).

Key features
- Plots a map slice (plane) from a data cube.
- Uses APLpy for publication-quality figures when available; otherwise falls back to matplotlib.
- Interactive controls:
  - Left-click or key '1': plot the selected pixel’s spectrum in another window and mark the clicked pixel with an 'x'.
  - Button 2 or key 'o': overplot a second spectrum in an external plot window.
  - Button 3: disconnect the interactive viewer.
  - Click-and-drag with button 1: average over a circular region.
  - Keyboard: press 'c' to set the circle center and 'r' to set its radius.
- Handles both FITS-based WCS and pixel-based coordinates when possible.
- Optional colorbar, customizable color scale, and colormap.
- Converts cube header to a usable WCS header, and computes a 2D plane for plotting.

API and usage
--------------

Class: MapPlotter
-----------------

Initialization
```
MapPlotter(Cube=None, figure=None, doplot=False, **kwargs)
```

Parameters
- Cube: The data cube to plot. The class flattens the cube header for WCS and uses this to extract the plane.
- figure: An existing matplotlib Figure, or an integer figure number to create a new figure. If None, a new figure is created when plotting.
- doplot: If True, immediately creates and shows the map plot by calling mapplot.
- kwargs: Passed to the internal mapplot machinery (e.g., to influence how the plane is selected via makeplane, and to other plotting options).

Behavior
- Stores a reference to the cube and prepares a flattened header.
- Builds a WCS object if the necessary WCS support is available.
- If doplot is True, immediately renders the map plot.

Callable behavior
```
MapPlotter(...)  # after initialization
MapPlotter.__call__(**kwargs)  # alias for mapplot
```

Methods
1) mapplot(convention='calabretta', colorbar=True, useaplpy=True,
            vmin=None, vmax=None, cmap=None, plotkwargs={}, **kwargs)
   - Purpose: Plot a map based on the input cube.
   - Workflow:
     - Creates or reuses a figure. Clears and redraws if needed.
     - Calls makeplane(**kwargs) to produce self.plane (the 2D data to plot) and uses self.header for WCS.
     - Handles estimator argument passed via kwargs (popped before plotting).
     - Determines vmin/vmax if not provided (based on non-NaN values in the plane).
     - If APLpy is available and useaplpy is True:
       - Builds a FITS HDU from the plane and header.
       - Creates a FITSFigure (APLpy) and shows a colorscale with show_colorscale.
       - Sets self.axis to the internal axes of the APLpy figure, or to the standard axes if an older API is present.
       - Optionally adds a colorbar.
       - Sets self._origin to 0 (FITS convention; later logic can adjust for pixel-based units).
     - Otherwise (no APLpy or not requested):
       - Uses a standard matplotlib axis and imshow to render self.plane.
       - Optionally adds a colorbar via matplotlib.
       - Sets self._origin to 0 (normal convention).
     - Stores the figure’s canvas in self.canvas.
     - Calls _connect() to enable interactive event handling.
   - Parameters explained:
     - convention: If using APLpy, selects the projection convention (default 'calabretta').
     - colorbar: Whether to show a colorbar.
     - useaplpy: Prefer APLpy when a FITS header is available.
     - vmin/vmax: Clipping values for the color scale. If None, determined from the plane.
     - cmap: Colormap to use for plotting.
     - plotkwargs: Extra keyword arguments passed to show_colorscale (APLpy) or imshow (matplotlib).
     - kwargs: Passed to makeplane (e.g., to specify estimator or plane selection).

2) __call__(**kwargs)
   - Shorthand to invoke mapplot with the given keyword arguments.

3) _connect()
   - Connects interactive events to callbacks:
     - button_press_event → self.click
     - button_release_event → self.plot_spectrum
     - key_press_event → self.plot_spectrum

4) _disconnect()
   - Disconnects the interactive event handlers established by _connect.

Notes
- makeplane: The map is produced by the internal makeplane mechanism, which is part of the cubes module. The estimator keyword argument, when provided, is passed to makeplane to select the map plane.
- Header handling: The cube header is flattened via cubes.flatten_header(self.Cube.header, delete=True) to produce a WCS-compatible header for plotting.
- WCS support: If the environment supports astropy.wcs (pywcsOK), a WCS object is created for coordinate-aware plotting. If not, plotting proceeds with basic image coordinates.
- APLpy vs. matplotlib: The code prefers APLpy when available and requested, enabling publication-quality figures with a colorbar. When APLpy is unavailable, matplotlib’s imshow is used.

Usage examples
--------------

Basic usage (interactive viewer)
```
from cubes import MapPlotter

# Assuming `mycube` is a loaded cube object with the required interface
mp = MapPlotter(Cube=mycube, doplot=True)
# The map plot appears and is interactive
```

Non-interactive usage (manual control)
```
from cubes import MapPlotter

mp = MapPlotter(Cube=mycube)
mp.mapplot()  # renders the map in a new or existing figure
```

Customizing the plot
```
mp = MapPlotter(Cube=mycube)
mp.mapplot(
    convention='calabretta',
    colorbar=True,
    useaplpy=True,
    vmin=None,
    vmax=None,
    cmap='viridis',
    plotkwargs={'origin': 'lower'}
)
```

Interactivity overview
- Left-click (Button 1) or press '1': plots the selected pixel’s spectrum in a separate window and marks the clicked location.
- Button 2 or press 'o': overplots the selected spectrum in the external window.
- Button 3: disconnects the interactive viewer.
- Dragging with Button 1: averages over a circular region centered at the click.
- Press 'c' to set a circle center and 'r' to set the radius, enabling circular averaging.

Dependencies and compatibility
- Dependencies:
  - Optional: APLpy for publication-quality plotting (APLpy is used if available and useaplpy is True).
  - Optional: astropy.wcs or pyfits for WCS handling (pywcsOK flag).
- If APLpy is unavailable, the module gracefully falls back to matplotlib’s imshow with an optional colorbar.

This documentation covers the public API and typical usage of the MapPlotter class in the cubes module, including interactive features and plotting options.
