Random Projection
=================

A simple and computationally efficient method to reduce the dimensionality of data.

Overview
--------

- Random projections trade a controlled amount of accuracy for faster processing and smaller model sizes.
- We build a low-dimensional representation by projecting data with a random matrix.
- Distances between points are preserved with high probability, enabling effective downstream learning.

Key ideas
---------

- The projection uses a random matrix (dense Gaussian or sparse) to map data from high to low dimensions.
- The Johnson-Lindenstrauss lemma guarantees that pairwise distances are preserved within a factor (1 ± eps) with high probability.
- The minimum number of components (n_components) depends on the number of samples and the distortion eps, not on the original feature count.

  (1 - eps) ||u - v||^2 < ||p(u) - p(v)||^2 < (1 + eps) ||u - v||^2

  n_components >= 4 log(n_samples) / (eps^2 / 2 - eps^3 / 3)

- Projections can be dense (Gaussian) or sparse (Achlioptas-style) to improve efficiency.

API and core ideas
------------------

- Gaussian random projection uses a dense matrix with entries drawn from N(0, 1/n_components).
- Sparse random projection uses a sparse matrix to reduce computation and memory usage.
- The density of the sparse projection can be controlled:
  - density = "auto" maps to 1 / sqrt(n_features)
  - density ∈ (0, 1] specifies the fraction of non-zeros
- Input sizes are validated to be positive.

  - _check_input_size(n_components, n_features) enforces strictly positive dimensions.
  - _check_density(density, n_features) validates the density parameter.

- The helper function johnson_lindenstrauss_min_dim(n_samples, eps=0.1) computes a safe minimum n_components to achieve a given distortion eps for a dataset with n_samples samples.

  - Supports scalar or array-like n_samples and eps.
  - Valid for eps in (0, 1) and n_samples > 0.

Usage examples
--------------

Compute a safe number of components for a given dataset and distortion:

.. code-block:: python

   from sklearn.random_projection import johnson_lindenstrauss_min_dim

   n_samples = 1_000_000
   eps = 0.5
   n_components = johnson_lindenstrauss_min_dim(n_samples, eps)
   print(n_components)  # e.g., 663 for eps=0.5 and 1e6 samples

Create and apply a Gaussian random projection:

.. code-block:: python

   from sklearn.random_projection import GaussianRandomProjection
   from sklearn.datasets import make_blobs

   X, _ = make_blobs(n_samples=1000, n_features=500, random_state=0)

   transformer = GaussianRandomProjection(n_components=200, random_state=42)
   X_proj = transformer.fit_transform(X)

Create and apply a Sparse random projection:

.. code-block:: python

   from sklearn.random_projection import SparseRandomProjection

   transformer = SparseRandomProjection(n_components=200, density='auto', random_state=42)
   X_proj = transformer.fit_transform(X)

Notes
-----

- The number of components is chosen to balance accuracy and speed. Increasing n_components improves distance preservation but increases computation.
- You can decide between dense Gaussian and sparse projections based on your data size and available memory.
