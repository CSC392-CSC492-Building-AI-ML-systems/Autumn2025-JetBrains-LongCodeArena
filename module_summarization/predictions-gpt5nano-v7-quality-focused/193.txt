Metadata Routing API
====================

Overview
--------

The Metadata Routing API provides the infrastructure that enables meta-estimators to forward operations, parameters, and related metadata to their underlying base estimators. It supports composing estimators such as multi-output regressors/classifiers and chain estimators by coordinating how methods and attributes are routed to constituent estimators.

Public API
----------

The following symbols are part of the metadata routing utilities used by meta-estimators. Some are intended for public use, while others are internal helpers used within the implementation.

- MetadataRouter
  - Core component that coordinates routing of metadata across nested estimators.
  - Used to manage how methods and attributes propagate from a wrapper estimator to its sub-estimators.

- MethodMapping
  - Represents a mapping describing how methods or parameter access should be routed to sub-estimators.
  - Defines the routing relationships that the router applies when delegating operations.

- process_routing
  - Function that processes and applies routing rules to a routing configuration.
  - Produces a routed configuration that sub-estimators recognize and honor during evaluation.

- _routing_enabled
  - Predicate that indicates whether routing is currently active for a given routing configuration.
  - Used to guard routing logic in meta-estimators.

- _raise_for_params
  - Helper used to validate routing-related parameters.
  - Ensures routing configurations meet expected constraints and raises informative errors when they do not.

Internal helpers (underscore-prefixed)
--------------------------------------

The following items are intended for internal use within the metadata routing system and are not part of the public API surface. They support the implementation of routing behavior in meta-estimators.

- Underscored helpers referenced by the routing utilities to enforce invariants, perform validation, and manage routing state.

How it integrates with the Metadata Routing API
----------------------------------------------

- Meta-estimators import these utilities to implement consistent forwarding of methods and parameters to their underlying estimators.
- MetadataRouter acts as the central coordinator, while MethodMapping encodes how specific operations should be forwarded.
- process_routing applies routing configurations to a usable form, and _routing_enabled allows conditional routing to be turned on or off.
- _raise_for_params validates routing configurations to catch misconfigurations early.

Usage notes
-----------

- The routing utilities are primarily used by multi-output estimators (e.g., multi-output regressors/classifiers and chain estimators) to ensure that operations performed on the wrapper estimator correctly operate on the underlying base estimators.
- These tools are designed to be framework-consistent, enabling uniform behavior across different meta-estimators that rely on shared routing semantics.
- Since several components are internal (underscore-prefixed), users should treat these as advanced internals. Public usage focuses on the higher-level meta-estimator APIs, while routing utilities are used behind the scenes to implement delegation.

Examples and guidance
---------------------

- When implementing a custom meta-estimator that delegates to one or more sub-estimators, you can rely on the metadata routing utilities to:
  - Define how methods and parameters should be forwarded to sub-estimators.
  - Validate routing configurations to prevent invalid delegations.
  - Enable or disable routing as needed in different contexts (e.g., fitting, predicting, or evaluating).
