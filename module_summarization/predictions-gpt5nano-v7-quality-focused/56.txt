MixedGLM
=========

Bayesian inference for generalized linear mixed models (GLMMs)

Overview
--------
This module implements a Bayesian GLMM framework for families that do not have additional scale or shape parameters, specifically binomial and Poisson families. It provides two estimation approaches:

- Laplace approximation to the posterior (maximum a posteriori, MAP)
- Variational Bayes (mean-field approximation to the posterior)

All random effects are modeled as independent realizations in this implementation.

Key concepts
------------
- exog_vc: Design matrix for the random (random-effects) part of the model. Each column corresponds to an independent random effect realization.
- ident: Integer labels that determine which random terms share the same variance. Two columns of exog_vc with the same ident value share the same standard deviation parameter.
  - When using formulas (via patsy), columns derived from the same formula share the same variance parameter.
- Random effects and priors:
  - Random effects realizations (vc) are independent Gaussian given the variance-structure parameters (vcp).
  - The standard deviation parameters (vcp) follow Gaussian priors in log-space (i.e., log s with prior standard deviation vcp_p).
  - The log-standard-deviation parameters have log-normal priors.
- exog_vc sparsity: A sparse matrix is supported. If a dense matrix is provided, it is converted to sparse internally. When formulas are used, a temporary dense version of exog_vc may be created.

Model and parameterization
--------------------------
The joint density of data and parameters factors as:

.. math::

    p(y | vc, fep) p(vc | vcp) p(vcp) p(fe)

- p(y | vc, fep) is the GLM likelihood for the chosen family (binomial or Poisson) given the random effects vc and fixed-effects parameters fep.
- p(vc | vcp) is the independent Gaussian distribution of the random effects realizations, with variances determined by vcp. Random-effect realizations are independent within and across values of ident.
- p(vcp) and p(fe) are priors over the log-standard-deviation parameters and the fixed-effects parameters, respectively.
  - vcp: log-normal priors (mean 0, standard deviation vcp_p in log-space)
  - fe: Gaussian prior with mean 0 and standard deviation fe_p

Parameterization details
- endog: Response vector
- exog: Design matrix for fixed effects (columns align with fep)
- exog_vc: Design matrix for random effects (each column is a separate random effect realization)
- ident: Integer vector labeling random terms with a common variance
- vcp_p: Prior standard deviation for the variance-component parameters (log-space)
- fe_p: Prior standard deviation for fixed effects parameters
- family: A statsmodels.genmod.families instance (e.g., Binomial, Poisson)
- fep_names: Names for fixed effects parameters (default constructed if None)
- vcp_names: Names for variance-component parameters (default constructed if None)
- vc_names: Names for the random-effect realizations

Implementation notes
- The class supports a sparse exog_vc matrix via scipy.sparse. If a dense matrix is provided, it is converted internally to sparse.
- When using formulas, the columns of exog_vc created from a common formula share the same variance parameter.
- For some families (notably Binomial), setting vcp_p too large can hinder numerical convergence when locating the posterior mode. A value around 0.5 often works well.

Fitting methods
-------------
- fit_map: Fits using the Laplace approximation (MAP)
- fit_vb: Fits using variational Bayes (mean-field)

Both methods return a MixedGLMResults object.

Input and output
----------------
Inputs (constructor parameters)
- endog: Array-like, response values
- exog: Array-like, fixed-effects design matrix
- exog_vc: Array-like or scipy.sparse matrix, random-effects design matrix
- ident: Array-like of integers, groups for shared variance
- vcp_p: Float, prior std for variance-component parameters (in log-space)
- fe_p: Float, prior std for fixed effects parameters
- family: statsmodels.genmod.families instance (e.g., Binomial, Poisson)
- fep_names: List[str], names for fixed-effects parameters
- vcp_names: List[str], names for variance-component parameters
- vc_names: List[str], names for random-effect realizations

Output
- MixedGLMResults: Encapsulates parameter estimates for fixed effects (fep), random effects (vc), and variance components (vcp), along with standard errors or posterior summaries depending on the fitting method.

Example
-------
The following is a representative usage pattern:

- Create the model
  model = MixedGLM(endog, exog, exog_vc, ident, vcp_p=0.5, fe_p=2.0, family=sm.families.Binomial())

- Fit using Laplace MAP
  results_map = model.fit_map()

- OR fit using Variational Bayes
  results_vb = model.fit_vb()

Notes on formulas and sparsity
- When exog_vc is constructed via formulas, columns derived from the same formula share a variance parameter.
- If exog_vc is created from a formula and is dense, a temporary dense version may be generated before converting to sparse. There is currently no option to avoid this temporary representation in formula-based workflows.

Gauss-Legendre weights
----------------------
Internal Gauss-Legendre weights are used in the numerical integration steps for the posterior approximations.

Internal data
-------------
This module includes a placeholder for a logit example that demonstrates typical usage with a logit link; users should supply concrete endog, exog, exog_vc, and ident data consistent with their modeling scenario.
