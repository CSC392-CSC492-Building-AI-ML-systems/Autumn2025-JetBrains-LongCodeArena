w_stateprobs
==============

Overview
--------
w_stateprobs is a backward-compatible wrapper around the state-probability analysis tool. It exposes a subcommand trace that computes average state populations and associated confidence intervals by tracing trajectories in weighted ensemble data. This wrapper is deprecated in favor of using w_direct directly.

Subcommands
-----------
- trace
  - Calculates averages and confidence intervals for path-tracing kinetics analysis.
  - This subcommand is implemented by wrapping the DStateProbs tool.

Defaults
--------
- Output file: stateprobs.h5
- Kinetics file (used by the underlying analysis): assign.h5

Notes
-----
- The wrapper prints a deprecation warning and forwards control to the regular tool (w_direct). The message is:
  "w_stateprobs is being deprecated.  Please use w_direct instead."

Output format
-------------
The output file (specified with -o/--output, default stateprobs.h5) contains the following dataset:

- /avg_state_pops [state]
  - Population of each state across the entire analysis range.
  - Structure is described below.

If you enable evolution mode (via --evolution-mode), an additional dataset is produced:

- /state_pop_evolution [window][state]
  - State populations based on windows of iterations of varying width.
  - If evolution mode is cumulative, windows start at the iteration specified by --start-iter and grow by --step-iter for each successive element.
  - If evolution mode is blocked, windows have width --step-iter (the last window may be shorter), with the first window starting at --start-iter.

Dataset structure
-----------------
Each evolution window contains the following fields:

- iter_start
  - Integer: Iteration at which the averaging window begins (inclusive).

- iter_stop
  - Integer: Iteration at which the averaging window ends (exclusive).

- expected
  - Floating-point: Expected (mean) value of the rate within this window (inverse tau).

- ci_lbound
  - Floating-point: Lower bound of the confidence interval on the rate within this window (inverse tau).

- ci_ubound
  - Floating-point: Upper bound of the confidence interval on the rate within this window (inverse tau).

- corr_len
  - Integer: Correlation length of the rate within this window (in units of tau).

Attributes
----------
Each of the above datasets carries the following attributes:

- mcbs_alpha
  - Floating-point: Alpha value for the confidence intervals (e.g., alpha=0.05 for a 95% CI).

- mcbs_nsets
  - Integer: Number of bootstrap datasets used to generate confidence intervals.

- mcbs_acalpha
  - Floating-point: Alpha value for determining correlation lengths.

Command-line options
--------------------
- -o, --output
  - Path to the output HDF5 file (default: stateprobs.h5).

- --evolution-mode
  - Controls the generation of the /state_pop_evolution dataset.
  - Values include cumulative or blocked. If omitted, evolution data is not produced.

- --start-iter
  - Integer. Iteration at which the averaging window begins (inclusive).

- --step-iter
  - Integer. Step size between window starts (or window widths, depending on mode).

Notes
-----
- The actual state-bin definitions and macrostate definitions come from the underlying binning used by the tool (see w_assign for details).

Examples
--------
- Basic usage (defaults, deprecated wrapper forwards to trace):
  w_stateprobs

- Explicitly using the trace subcommand:
  w_stateprobs trace --output stateprobs.h5 --evolution-mode cumulative --start-iter 0 --step-iter 10

See Also
--------
- w_direct (westpa.cli.tools.w_direct): the underlying, current approach for direct use of this functionality.
- WESTMasterCommand, WESTParallelTool: base classes used by the shim.
