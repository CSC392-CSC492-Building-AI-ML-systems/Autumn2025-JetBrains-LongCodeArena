Grid Search for Hyper-Parameter Tuning
======================================

Overview
--------
GridSearchCV performs an exhaustive search over a user-specified parameter grid to find the best combination of hyper-parameters for a given estimator. It uses cross-validation to assess each parameter setting and returns the best parameters according to a chosen score.

Basic usage
-----------
- Create an estimator instance (e.g., a classifier or regressor).
- Define a parameter grid as a dictionary. Keys are parameter names; values are lists of values to try.
- Instantiate GridSearchCV with the estimator and the parameter grid.
- Fit on your data. Access the best parameters, best score, and cross-validation results.

Example
-------
A simple grid search over an SVM classifier:

.. code-block:: python

    from sklearn.datasets import load_iris
    from sklearn.svm import SVC
    from sklearn.model_selection import GridSearchCV

    X, y = load_iris(return_X_y=True)

    param_grid = {
        "C": [0.1, 1.0, 10],
        "gamma": ["scale", "auto"],
        "kernel": ["rbf"]
    }

    grid = GridSearchCV(SVC(), param_grid, scoring="accuracy", cv=5, n_jobs=-1)
    grid.fit(X, y)

    print("Best params:", grid.best_params_)
    print("Best score:", grid.best_score_)

Inspecting results
------------------
GridSearchCV stores detailed results for every parameter combination in cv_results_. You can convert it to a tabular form to inspect and sort by performance.

.. code-block:: python

    import pandas as pd
    results = pd.DataFrame(grid.cv_results_)
    results = results.sort_values(by="rank_test_score")
    print(results[["params", "mean_test_score", "std_test_score", "rank_test_score"]].head())

Defining parameter grids
------------------------
- Use a flat dictionary where each key is a parameter name and each value is a list of values to test.
- For nested objects (e.g., pipelines), refer to parameters with a double-underscore notation: "pipeline__step__param".
  Example with a pipeline:
  - If you have a pipeline named "sc" with a StandardScaler and an estimator "clf", you set:
    - "clf__C": [0.1, 1, 10]
    - "clf__kernel": ["linear", "rbf"]

Example: grid search with a pipeline
------------------------------------
.. code-block:: python

    from sklearn.pipeline import Pipeline
    from sklearn.preprocessing import StandardScaler
    from sklearn.linear_model import LogisticRegression
    from sklearn.model_selection import GridSearchCV
    from sklearn.datasets import load_iris

    X, y = load_iris(return_X_y=True)

    pipe = Pipeline([
        ("scaler", StandardScaler()),
        ("clf", LogisticRegression(max_iter=1000))
    ])

    param_grid = {
        "clf__C": [0.1, 1.0, 10.0],
        "clf__penalty": ["l2"]
    }

    grid = GridSearchCV(pipe, param_grid, scoring=" accuracy ", cv=5, n_jobs=-1)
    grid.fit(X, y)
    print("Best params:", grid.best_params_)
    print("Best score:", grid.best_score_)

Scoring and cross-validation
----------------------------
- The scoring parameter selects the metric used to compare parameter combinations. It can be:
  - A single string (e.g., "accuracy", "roc_auc").
  - A callable that follows the scikit-learn scoring contract.
  - A dict of scorers for multi-metric evaluation (in combination with refit to select which metric to optimize).
- The cv parameter controls the cross-validation strategy (folds). Options include:
  - An integer for k-fold cross-validation.
  - A cross-validation splitter (e.g., KFold, StratifiedKFold).

Refitting and best parameters
-----------------------------
- By default, GridSearchCV refits the estimator on the entire dataset using the best-found parameters.
- You can disable refitting with refit=False or choose a specific scorer to optimize when multi-metric scoring is used.

Parallelism and memory
----------------------
- Use n_jobs to parallelize across parameter combinations. -1 utilizes all available cores.
- Be mindful of memory usage: large parameter grids with expensive estimators can require substantial memory and processing time.

Practical tips
-------------
- Start with a coarse grid to identify promising regions of the parameter space.
- Narrow the grid around top-performing values and rerun.
- Consider RandomizedSearchCV when the parameter space is large; it samples parameter settings randomly and can be more efficient while still finding strong configurations.
- Use pipelines to ensure consistent preprocessing and model evaluation during grid search.

See Also
--------
- RandomizedSearchCV for stochastic search over parameter values.
- Pipeline for composing preprocessing and modeling steps.
- make_scorer for defining custom scoring functions.

Notes
-----
- Parameter names must match the estimator’s attributes. For nested objects (pipelines or feature selectors), use the double-underscore notation to reference subparameters.
- The grid size equals the product of the number of values for each parameter. Plan grids to balance thoroughness with compute time.
