w_bins
======

Display information and statistics about binning in a WEST simulation, or modify the binning for the current iteration of a WEST simulation.

Overview
--------

- The w_bins command provides two subcommands:
  - info: show current binning information for a given iteration.
  - rebin: rebuild the current iteration using a (potentially) new binning.

- The command relies on the WESTDataReader to access simulation data and on the BinMappingComponent to manage bin mappings.

Usage
-----

w_bins <subcommand> [options]

Subcommands
-----------

info
~~~~

Display information about binning for a specific iteration.

Options
- -n, --n-iter N
  - Consider the initial points of segment N_ITER (default: current iteration).

- --detail
  - Display detailed per-bin information in addition to summary information.

- Bin mapping options
  - Additional arguments from BinMappingComponent are added to this subcommand. These configure the bin mapper used for information display. (See BinMappingComponent documentation for exact options.)

Notes
- The command prints a header “Bin information for iteration N” and then outputs binning details via write_bin_info.

Example
- Display detailed bin information for the current iteration:
  w_bins info --detail

- Display bin information for a specific iteration (e.g., 7) with standard details:
  w_bins info -n 7

rebin
~~~~~

Rebuild the current iteration using a new or updated binning.

Options
- --confirm
  - Commit the revised iteration to the HDF5 file. Without this flag, the operation computes and prints the effects of the new binning without persisting changes.

- --detail
  - Display detailed per-bin information in addition to summary information (as with info).

- -n, --n-iter N
  - Consider the initial points of segment N_ITER (default: current iteration).

- Bin mapping options
  - BinMappingComponent arguments are added here, except for --bins-from-file which is suppressed in this subcommand.

- --bin-target-counts
  - (Provided by BinMappingComponent via add_target_count_args) Configure target bin counts for the new iteration.

Algorithm and behavior
- This subcommand is not supported for the very first WEST iteration (iteration 1). A message is written to stderr and the process exits with status 1:
  rebin is not supported for the first iteration; reinitialize with w_init instead

- Steps performed when rebin is invoked:
  1. Retrieve the current and previous iteration segments; load initial points for the current iteration.
  2. Determine available initial states and the states actually used in the current segments.
  3. Create a new iteration using the we_driver, with the current bin mapper and the configured target counts.
  4. Assign initial states and initialize; perform the rebin operation against the previous iteration’s segments.
  5. Compute weights and assignments for the new iteration and print bin information.
  6. If --confirm is set:
     - Prepare the iteration in the data manager and update endpoint statuses.
     - Save the updated binning to the HDF5 file and update initial states.
     - Flush changes to disk.

Output
------

- For info: prints “Bin information for iteration N” followed by bin-related information generated by write_bin_info.
- For rebin: prints bin information for the new iteration; if --confirm is set, changes are written to disk and the iteration is updated.

Implementation notes
--------------------

- Under the hood, w_bins uses:
  - WESTDataReader to access simulation data (target states, iterations, segments, pcoords, weights).
  - BinMappingComponent to manage and apply bin mappings.
  - write_bin_info to format and print binning information.
  - The WEST driver (via westpa.rc.get_we_driver) to perform the rebinning and iteration creation.

- Entry point
  - The entry_point function is defined as:
    def entry_point():
        WBinTool().main
    This constructs the tool but does not call main (missing parentheses). In typical usage, the CLI entry point is intended to invoke the tool’s main routine.

Examples
--------

- Show bin information for the current iteration with detail:
  w_bins info --detail

- Show bin information for iteration 5 without extra detail:
  w_bins info -n 5

- Rebin current iteration with details, then print the results without committing:
  w_bins rebin --detail -n 5

- Rebin current iteration and commit changes to HDF5:
  w_bins rebin --confirm -n 5

Notes
-----

- Bin mapping options are provided by BinMappingComponent; consult its documentation for exact flags and file formats.
- Rebinning operates on initial points of the current iteration and may require preparing a valid set of initial states. See WEST workflow guidance for proper prior steps.

Related references
------------------

- write_bin_info: utility used to format and print binning information.
- BinMappingComponent: manages bin mappings and related options.
- WESTDataReader: accesses simulation data from HDF5 files.
- we_driver: handles iteration creation, assignment, and rebinning during rebin operations.
