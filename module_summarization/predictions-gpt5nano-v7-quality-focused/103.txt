Title: Code Generation Utilities (diofant.utilities.codegen)

Introduction
- The codegen module provides facilities to generate compilable code (C, C++, Fortran77/Fortran90, and Octave/Matlab) that evaluates diofant expressions.
- It is a work in progress. The milestones marked with a plus sign (+) in the source tree indicate features that have been completed.
- This module distinguishes itself from diofant.printing.ccode by offering a dedicated code generator workflow that produces full source packages (including headers), supports multiple arguments, and enables combining multiple expressions into a single routine with potential shared intermediates.

What this module does
- Defines a generic Routine data structure that describes how to translate a mathematical routine into target-language code.
- Provides a CodeGen framework with language-specific backends (C, F77/F90, Octave/Matlab) to translate Routine instances into compilable code.
- Exposes friendly API helpers that simplify common tasks for users who do not want to manage the full Routine/CodeGen workflow directly.

Key concepts
- Routine: A description of a routine to be translated into code. It captures:
  - name: The routine’s identifier.
  - arguments: A list of argument descriptors (InputArgument, OutputArgument, InOutArgument) describing input, output, and in-out parameters.
  - results: A list of Result objects describing the outputs produced by the routine (as expressions).
  - local_vars: Variables defined locally within the routine.
  - global_vars: Variables that are not passed as arguments but are visible to the routine.
- Validation: The Routine constructor validates that every symbol used in the results (and in any expressions associated with outputs) is either provided as an input/inout argument, defined as a local variable, or available as a global variable. If symbols required by the outputs are not covered, it raises a ValueError.
- symbol accounting:
  - Collects input symbols and all symbols appearing in results.
  - Ensures all symbols used in the outputs are accounted for by the arguments, locals, or globals.
- Properties:
  - variables: Returns a set of all variables that may be used by the routine, including local variables, argument names, and result variable names.
  - result_variables: Returns a list of variables associated with the results (the exact interpretation depends on the results’ definitions).

Public API (high-level)
- Routine
  - Generic description of a single evaluation routine that can be translated into code for one or more languages.
  - Constructor signature (conceptual): Routine(name, arguments, results, local_vars, global_vars)
  - Behavior: Validates symbol coverage, stores routine metadata, and exposes variable introspection.
  - Key properties:
    - variables: set of all variables potentially used in the routine.
    - result_variables: list of variables corresponding to the results.

Notes on design
- The module separates the description of what to compute (Routine) from how to emit source code (CodeGen and language-specific backends). This separation enables:
  - Reuse of the same computational description across multiple target languages.
  - Advanced workflows such as sharing common subexpressions via a common subexpression elimination strategy.
  - Encapsulation of language-specific constraints (e.g., multiple return values in Python versus C).

Language backends
- The module provides language-specific code generators (CCodeGen, FCodeGen, OctaveCodeGen) to produce compilable source code tailored to each target language.
- A higher-level CCodePrinter, FCodePrinter, and OctaveCodePrinter are available for printing individual expressions; the CodeGen layer builds on these printers to generate full routines, headers, and language-specific boilerplate.

Usage example (conceptual)
- Define a routine description and translate it into target code via a CodeGen backend.
- The following is a conceptual illustration; exact constructor arguments depend on the concrete Argument and Result classes provided by the library.

# Conceptual example (exact API may differ)
# Define a routine that computes y = x + 1
name = 'increment'
arguments = [InputArgument('x')]
results = [Result('y', Symbol('x') + 1)]
local_vars = []
global_vars = []

routine = Routine(name, arguments, results, local_vars, global_vars)
