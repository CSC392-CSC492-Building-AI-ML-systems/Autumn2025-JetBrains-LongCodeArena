PolynomialCountSketch (Tensor Sketch for Polynomial Kernel)
=============================================================

Overview
--------

The PolynomialCountSketch class provides an approximate feature map for the polynomial kernel using Tensor Sketch. It targets kernels of the form

K(X, Y) = (gamma * <X, Y> + coef0) ^ degree

where <X, Y> denotes the inner product. The implementation uses Count Sketches and FFTs to efficiently compute a compact, high-dimensional feature representation that approximates the polynomial kernel.

Key idea: for each input feature, the method applies two randomized hashings per degree to form a compressed sketch of the outer product X ⊗ X, and then aggregates across degrees using FFT-based convolution. The resulting feature map has dimensionality n_components.

Algorithmic highlights
----------------------

- Uses Count Sketches to represent the outer product of input features.
- Applies Fast Fourier Transform (FFT) to efficiently combine sketches for different degrees.
- Reduces memory usage by using a compact random hashing scheme.
- Accounts for a constant term coef0 by optionally augmenting the feature space when coef0 != 0.
- The number of output features is controlled by n_components, with typical guidance suggesting a larger value relative to input features for better performance.

Parameters
----------

- gamma : float, default=1.0
  Parameter of the polynomial kernel whose feature map will be approximated.

- degree : int, default=2
  Degree of the polynomial kernel whose feature map will be approximated.

- coef0 : float, default=0
  Constant term of the polynomial kernel whose feature map will be approximated.

- n_components : int, default=100
  Dimensionality of the output feature space. Typically, n_components should exceed the number of input features to achieve good approximation quality. Practical balance between score and runtime often occurs around n_components ≈ 10 × n_features.

- random_state : int, RandomState instance, default=None
  Random state used to initialize the indexHash and bitHash arrays. Provides reproducible results across calls when set.

Attributes
----------

- indexHash_ : ndarray of shape (degree, n_features), dtype=int64
  Array of indices in [0, n_components) used to represent the 2-wise independent hash functions for Count Sketch computation.

- bitHash_ : ndarray of shape (degree, n_features), dtype=float32
  Array with random entries in {+1, -1}, used to represent the 2-wise independent hash functions for Count Sketch computation.

- _n_features_out : int
  Number of output features produced after fitting. This is set to n_components.

- n_features_in_ : int
  Number of features seen during fit.

- feature_names_in_ : ndarray of shape (n_features_in_,)
  Names of features seen during fit, defined only if the input has string feature names.

Notes
-----

- fit accepts sparse input and computes internal hashing parameters based on the number of input features.
- After fit, transform returns a (n_samples, n_components) array containing the approximate polynomial feature mapping for each sample.

Usage example
-------------

from sklearn.kernel_approximation import PolynomialCountSketch

# Create a sketch for a 3rd-degree polynomial kernel
ps = PolynomialCountSketch(degree=3, random_state=1)

# Compute the approximate feature map for X
X_features = ps.fit_transform(X)

# Use the features in a linear model
from sklearn.linear_model import SGDClassifier
clf = SGDClassifier(max_iter=10, tol=1e-3)
clf.fit(X_features, y)
score = clf.score(X_features, y)

>>> y = [0, 0, 1, 1]
>>> ps = PolynomialCountSketch(degree=3, random_state=1)
>>> X_features = ps.fit_transform(X)
>>> ps_score = X_features.shape
>>> ps_score
(4, 100)

Notes
-----

- The implementation relies on a combination of Count Sketches and FFT-based convolution to produce the approximate feature map efficiently for large input dimensions.
- The random hashing parameters (indexHash_ and bitHash_) are learned during fit and then reused for transform.
