hyperexpand.rst
===============

Overview
--------

Hyperexpand expands Hypergeometric (and Meijer G) functions into named special
functions. The module is designed to be displayed in Sphinx documentation by
dynamically cooking a docstring at import time. It builds a collection of
formulas, converts each into a symbolic equation, and renders the result as
LaTeX for inclusion in the generated documentation.

Workflow
--------

- The module obtains a FormulaCollection instance and iterates over its
  formulae.
- For each formula, it constructs an equality of the form
  Eq(hyper(ap, bq, z), closed_form), where ap/bq come from the formula and z is
  the argument.
- It rewrites the closed form using the 'nonrepsmall' option to simplify
  representations.
- It renders the resulting equation to LaTeX and appends it as a Sphinx math
  directive to the module docstring.
- The final __doc__ contains a sequence of math blocks that Sphinx can render.

Code structure (highlights)
---------------------------

- _mod1(x): helper used for bucket management during symbolic manipulations.
  It ensures numeric values are reduced modulo 1 and handles non-numeric
  expressions by splitting coefficients.
- add_formulae(formulae): public function that populates a provided list with
  Formula objects representing known hypergeometric expansions.
  - It defines a local helper add(ap, bq, res) to record a simple expansion.
  - It defines a local helper addb(ap, bq, B, C, M) to record expansions that
    involve matrix results and parameter shifts.
  - It registers several base cases, such as 0F0, 1F0, and 2F1, and more
    complex expansions using HyperRep and related constructs.
- Formula and Hyper_Function usage:
  - Hyper_Function(ap, bq) represents a hypergeometric pattern.
  - f.closed_form holds the closed-form result, which is rewritten before use.
  - HyperRep_* objects encode specific hypergeometric-to-special-function
    representations.
- The module relies on the FormulaCollection class (from the hyperexpand
  package) to obtain the set of formulas to document.

Key mathematical concepts (summary)
-----------------------------------

The documentation outlines a Meijer G-based expansion framework with these
principles:

- z**rho * G(ap, bq; z) = G(ap + rho, bq + rho; z)
- D denotes z * d/dz, used to express differential actions on G-functions
- G(1/z) introduces a symmetry where parameters ap and bq are interchanged
- Four shift operators:
  - A_J = b_J - D, J = 1, ..., n
  - B_J = 1 - a_j + D, J = 1, ..., m
  - C_J = -b_J + D, J = m+1, ..., q
  - D_J = a_J - 1 - D, J = n+1, ..., p
- A_J and C_J increment b_J; B_J and D_J decrement a_J
- Inverse-shift operators exist if there is no cancellation
- Order reduction rules: if b_j - a_i is a non-negative integer under certain index
  conditions, a quotient of gamma functions reduces to a polynomial, enabling a
  lower-order G-function representation
- Paired index theorems allow order reduction under specific integer-difference
  conditions
- An index quadruple is suitable if its order cannot be reduced; reachability
  concepts determine whether one quadruple can be transformed into another

Usage notes
-----------

- The module’s documentation generation occurs at import time. Importing the
  relevant package (which provides FormulaCollection) yields a __doc__ that
  contains a sequence of math directives corresponding to the documented
  formulas.
- The generated content serves as a reference for users of the expansion engine
  and for Sphinx-powered documentation.

API surface
-----------

- _mod1(x)
  - Internal helper used to compute x modulo 1 with special handling for numbers
    and symbolic expressions.
- add_formulae(formulae)
  - Public function that fills the provided list with Formula objects representing
    hypergeometric expansions.
  - Usage pattern:
    - formulae = []
    - add_formulae(formulae)
    - Each element in formulae is a Formula(ap, bq, z, res, ...) describing a
      specific expansion rule.
- FormulaCollection (external, imported)
  - Provides a collection of formulae used to generate the in-module documentation.

Example: generating the in-module documentation (conceptual)
-----------------------------------------------------------

- Create the formula collection and iterate over its formulae:
  For f in c.formulae:
    obj = Eq(hyper(f.func.ap, f.func.bq, f.z),
             f.closed_form.rewrite('nonrepsmall'))
    latex_doc_block = latex(obj)
- The resulting latex_doc_block is inserted into the module’s __doc__ as a
  math directive, forming the complete documentation for Sphinx.

Notes for maintainers
---------------------

- The module embeds heavy domain-specific logic for hypergeometric expansion.
  Keep the internal helper names (_mod1, add_formulae, etc.) stable to avoid
  breaking the generated documentation.
- Ensure the FormulaCollection source remains compatible with the code paths used
  to generate __doc__ during import.
